# Interview Log Phase

## 输入

- `speculo/.speculo/doc/<change>/fragments.md`
- 本轮访谈对话
- 用户明确指出的删除、重写、合并或保留偏好

## 产物

- `speculo/.speculo/doc/<change>/interview-log.md`，由 `../_templates/writing-interview-log-template.md` 填写

## 填写引导

1. 记录本轮追问方向，不要把 log 写成文章提纲。
2. 标出已采集 fragment 的类型：主张、小场景、锋利句子、半成形想法、引用、观察组等。
3. 标出下一轮最值得追问的 1-3 个方向。
4. 记录用户对语气、删改、保留的偏好。

## 边界

- 不重排 `fragments.md`。
- 不替用户决定最终文章结构。

## 完成准则

- `interview-log.md` 能让下一次会话继续访谈
- 已更新 `.status.json` 的 `fragment_count` 和 `writing_status`
- `interview-log.md` 无残留 `[TODO:]`
