## 功能说明

`triage` 通过一个小型**状态机**的分类角色将你项目追踪器上的 issue 流转起来——对它们进行分类、验证声明、必要时深入打磨成形，并留下可供 Agent 直接处理的简报。

它从不盲目打标签。每个被分类的条目恰好携带一个**类别**角色（`bug` / `enhancement`）和一个**状态**角色（`needs-triage`、`needs-info`、`ready-for-agent`、`ready-for-human`、`wontfix`），并且它*推荐并等待*——它会告诉你它的类别和状态判断及其理由，然后仅根据你的指示行动。在任何条目被提升到 `ready-for-agent` 之前，它会先验证声明：bug 会被复现，PR 会被 checkout 并运行测试。

## 何时使用

输入 `/triage` 并用自然语言描述你想做什么来调用它——Agent 不会自己触发它。例如："显示所有需要我关注的内容"、"来看看 #42"、"把 #42 移到 ready-for-agent"。

当你的问题追踪器上有原始、未经评估的报告，你希望它们被整理、验证，并转化为 Agent 或人类可以直接接手的工作时，使用它。要将一个已达成共识的对话转化为一份新规范，使用 [to-spec](https://aihero.dev/skills-to-spec)；要将现有规范拆分为 ticket，使用 [to-tickets](https://aihero.dev/skills-to-tickets)。`triage` 走的是相反方向——它处理的是*已经*落在追踪器里的内容。

## 前置条件

`triage` 会读取和写入你的问题追踪器，因此 [setup-matt-pocock-skills](https://aihero.dev/skills-setup-matt-pocock-skills) 必须先配置好追踪器和标签映射。上述角色名称是**规范名**——你在追踪器中实际使用的标签字符串可能不同，该映射由 setup 提供。配置中还决定了外部 PR 是否作为请求入口，以及谁算作外部贡献者。

## PR 即附带了代码的 issue

当追踪器将外部 Pull Request 视为请求入口时，`triage` 会对它们执行*相同*的处理流程：相同的类别角色、相同的状态、相同的流转——只是状态判断的依据是 diff 而非文字报告。`ready-for-agent` 表示已附加简报，Agent 应就代码采取下一步行动；`ready-for-human` 表示已准备好合并。发现流程只暴露外部 PR，但显式指定的 PR 无论如何都会被分类，不受作者身份影响。

## 先验证，再写简报

将 `triage` 与随意打标签区分开来的步骤是验证。它会根据报告者提供的步骤复现 bug，或 checkout PR 并运行测试，然后报告结果：通过代码路径确认、复现失败、或信息不充分（这本身就是一个强烈的 `needs-info` 信号）。它还会执行两项代码库检查——**冗余检查**（这个功能是否已经实现了？若是，则为 `wontfix`）和**过往拒绝检查**（`.out-of-scope/` 目录是否已经说过不做？）。经过确认的验证会让 Agent 简报有力得多；猜测则无济于事。

## 有效的标志

- 它处理的每个条目最终携带恰好一个类别角色和一个状态角色——从不零个，也从不两个冲突的状态。
- 它向你提交带理由的建议并等待确认，而非自行重新打标签。
- 在达到 `ready-for-agent` 之前，bug 被复现，PR 被实际运行。
- 它发布到追踪器的每条评论都以 `> *This was generated by AI during triage.*` 声明开头。

## 在系统中的位置

`triage` 是对问题追踪器的**定期维护**——每当报告堆积时运行它，以保持队列有序且 `ready-for-agent` 列的可信度。它位于追踪器的前端，在构建链的上游：它编写的简报是 [tdd](https://aihero.dev/skills-tdd) 后续用于实现的素材。当某个请求需要进一步细化时，它依赖 [grilling](https://aihero.dev/skills-grilling) 和 [domain-modeling](https://aihero.dev/skills-domain-modeling) 来一次一个问题地将其打磨成形。它的紧密邻居是 [to-spec](https://aihero.dev/skills-to-spec)，后者从新的对话中填充追踪器，而 `triage` 处理的是追踪器中已有的内容。当你不确定哪个技能或流程适合时，[ask-matt](https://aihero.dev/skills-ask-matt) 会为你导航。
