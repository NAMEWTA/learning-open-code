## 功能说明

`handoff` 将当前对话压缩为一份**交接文档**——一份新的 Agent 可以阅读并从中断处继续工作的单一文档。

它**不会**复述已经存在于其他地方的内容。任何已记录在规范、计划、ADR、issue、commit 或 diff 中的内容，都通过路径或 URL 引用，绝不复制。文档只携带实时线索——你当时在做什么、为什么做、以及下一步是什么——并保存到操作系统的临时目录中，而不是写入工作区，因此它永远不会成为又一个需要维护的产出物。

## 何时使用

你通过输入 `/handoff` 来调用它——Agent 不会自己主动使用它。你可以附上一条关于下一轮会话要做什么的说明，文档会据此进行定制。

当对话进行得足够久、上下文面临丢失风险时——你接近上下文上限、当天收工、或有意将工作交接给另一个 Agent——并且你希望保留工作线索而不必拖拽整个聊天记录时，就是使用它的时机。

## 文档承载的内容

- **实时线索**——正在进行的任务及其原因，以对话自身的术语表述，减去任何已在别处记录的内容。
- **推荐的 skills**——指向下一个 Agent 继续工作时应使用的 skills 的指针。
- **引用而非复制**——指向规范、计划、ADR、issue 和 diff 的链接和路径，这些是包含已确定细节的地方。
- **脱敏的密钥**——API 密钥、密码和个人身份信息在写入文档之前会被剥离。

需要记住的核心概念是**压缩**：一份交接文档是将对话压缩到仅剩可恢复的核心，让新的 Agent 继承的是势头，而非噪音。

## 在系统中的位置

`handoff` 是一个随时可用的独立工具——它位于两次会话之间的接缝处，而非构建链路之中。它自然地与那些产出物类 skills 搭配使用，因为它的输出指向这些产出物：[to-spec](https://aihero.dev/skills-to-spec)，因为一份完成的规范正是交接文档引用而非重复的那种已确定细节。当你不确定哪个 skill 适合当前场景时，[ask-matt](https://aihero.dev/skills-ask-matt) 会为你导航。
