## 功能说明

`writing-great-skills` 是你撰写和编辑 skills 时所对照的参考标准——它提供了使一个 skill 行为可预测的共享词汇和原则。

一个 skill 的职责是从随机系统中梳理出确定性，因此目标不是每次运行产生相同的*输出*，而是相同的*过程*。**可预测性**是根本的美德，每一个设计选择都以此为标准来评判——而不是看这个 skill 读起来有多聪明、多完备或多详尽。

## 何时使用

你通过输入 `/writing-great-skills` 来调用它——Agent 不会自己主动使用它。

当你正在编写一个新 skill 或编辑一个已有 skill，并希望它每次行为一致时，就是使用它的时机：决定调用模式、撰写描述、选择哪些内容放在 `SKILL.md` 中而哪些放在链接文件中，或者诊断一个 skill 为何行为异常。

## 认知负荷

整个参考标准围绕的核心概念是**认知负荷**——以及它的对应概念**上下文负荷**。每个 skill 都会消耗其中之一：

- **模型调用型** skill 在每一轮对话窗口中都保留描述，因此它消耗**上下文负荷**，但能自行触发。
- **用户调用型** skill 会移除该描述；它消耗零上下文负荷，但现在*你*成为了必须记住它存在的索引——这就是**认知负荷**。

这些 skills 中的大多数是用户调用型的，这就是为什么认知负荷是整个系统设计所要管理的压力所在：当用户调用型 skills 的数量增长到超出你脑中所能容纳的程度时，解决方案就是建立一个**路由 skill**，列出其他 skills 的名称以及何时使用它们。一旦你从这两种负荷的角度思考，大多数撰写决策——拆分还是不拆分、内联还是外链、模型调用还是用户调用——就都变成了在不同场景下做的同一笔权衡。

## 其他杠杆

参考标准的其余部分是合理花费这些负荷的工具箱：

- **引导词**——已在模型预训练中的紧凑概念（_tight_、_red_、_tracer bullet_），Agent 在运行 skill 时以此作为思维锚点。它以最少的 token 锚定执行和调用；要追查那些可以用单个词就能取代的冗长表述。
- **信息层级**——从 skill 内步骤，到 skill 内参考，再到通过**上下文指针**指向的外部参考。**渐进式披露**是沿此阶梯向下移动、让顶层保持清晰可读的方法。
- **修剪**——单一事实来源、相关性以及逐句进行的空操作测试，用于对抗**沉淀**和**蔓延**。
- **失败模式**——**过早完成**、**重复**、**沉淀**、**蔓延**、**空操作**——用于诊断一个行为异常的 skill。

## 在系统中的位置

这是一个随时可用的独立参考标准——你在构建整套 skills 时查阅的元 skill，而非链路中的一个环节。它的天然邻居是你维护的任何路由 skill，因为路由 skill 是用户调用型 skills 所累积认知负荷的直接解决方案；当你不确定哪个 skill 或流程适合某个任务时，[ask-matt](https://aihero.dev/skills-ask-matt) 会引导你遍历整套 skills。
