# 学习记录格式

学习记录存放在 `./learning-records/` 中，使用顺序编号：`0001-slug.md`、`0002-slug.md` 等。延迟创建目录 — 仅在第一条记录被写入时才创建。

它们是教学领域的 ADR：捕获非显而易见的经验、关键洞察以及将会指导未来会话的既有知识声明。它们用于计算最近发展区。

## 模板

```md
# {对学到或确立的内容的简短标题}

{1-3 句话：学到了什么（或确立了哪些既有知识），以及为什么它对未来会话重要。}
```

这就是全部格式。一条学习记录可以就是一个段落。其价值在于记录_这个_知识现在已知，以及_为什么_它会改变接下来教什么 — 而不是填满各个部分。

## 可选部分

仅当它们真正增加价值时才包含这些。大多数记录不需要它们。

- **Status** 前置元数据（`active | superseded by LR-NNNN`）— 当早期的理解后来被发现是错误的并被替换时有用。
- **Evidence** — 用户如何展示了理解（回答了一个问题、完成了一个练习、引用了先前经验）。当声明可能被重新审视时有用。
- **Implications** — 这为未来会话解锁了什么或排除了什么。当不显而易见时值得记录。

## 编号

扫描 `./learning-records/` 中的最高现有编号，然后加 1。

## 何时编写学习记录

当以下任一为真时编写：

1. **用户展示了对某个非平凡事物的真正理解** — 不仅仅是接触，而是有证据表明他们可以正确使用该概念。这为接下来教什么设定了新底线。
2. **用户披露了既有知识** — "我已经知道 X。"记录下来，这样未来的会话不会重复教它。同时记录所声称的_深度_。
3. **一个误解被纠正了** — 用户之前相信了错误的东西，现在明白了为什么。这些是高价值的：它们预测了相关主题未来的绊脚石。
4. **使命因学习而转变** — 用户发现他们关心的东西与之前想的不同。交叉链接到 [[MISSION.md]] 并更新它。

### 什么不算

- 仅仅覆盖过的材料。覆盖不等于学习。等有证据再说。
- 任何已简明地作为术语定义捕获在 [[GLOSSARY.md]] 中的内容。不要重复。
- 逐次会话的活动日志。学习记录不是日志 — 它们是决策级别的洞察。

## 取代

当后来的记录与之前的记录矛盾时（用户的理解深化了或被纠正了），将旧记录标记为 `Status: superseded by LR-NNNN`，而非删除它。理解如何演变的历史本身就是有用的信号。
