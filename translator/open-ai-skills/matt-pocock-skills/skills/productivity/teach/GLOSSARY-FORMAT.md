# GLOSSARY.md 格式

`GLOSSARY.md` 是该教学工作区的规范语言。所有讲解、练习和学习记录都应遵守其术语。构建它本身就是学习的一部分：将一个概念压缩成精确的定义，是用户理解它的证据。

## 结构

```md
# {主题} 术语表

{对该术语表所涵盖主题的一两句话描述。}

## Terms

**Hypertrophy**：
由反复训练过程中的机械张力和代谢压力驱动的肌肉增长。
_Avoid_: Bulking, getting big

**Progressive overload**：
随着时间系统地增加对肌肉的需求 — 通过负荷、训练量或强度。
_Avoid_: Pushing harder, levelling up

**RPE (Rate of Perceived Exertion)**：
对一组训练有多吃力的 1–10 分自评，10 表示力竭，8 表示还有两次重复的余力。
_Avoid_: Effort score, intensity rating
```

## 规则

- **仅在用户理解术语后添加。** 术语表是压缩知识的记录，不是用户可以阅读来学习的字典。如果用户刚接触一个概念，等到他们能正确使用它再将其提升到此。
- **要有主见。** 当存在多个表示同一概念的词时，选择最好的那个，将其余列为应避免的别名。这就是语言压缩的方式。
- **定义保持精炼。** 一到两句话。定义术语是什么，而不是它做什么或如何做。
- **在定义中使用术语表自身的术语。** 一旦一个术语被收录在术语表中，在任何地方都优先使用它 — 包括在其他定义中。这就是后续理解复杂术语变得更容易的原因。
- **当自然形成聚类时，用子标题分组**（例如 `## Anatomy`、`## Programming`）。当术语内在一致时，扁平列表也可以。
- **明确标记歧义。** 如果一个术语在更广泛的领域中被宽松使用，注明本工作区的选择："在本工作区中，'set' 始终表示正式组 — 热身组单独跟踪。"
- **随着理解深入而修订。** 用户在第一周写的定义到第六周可能是错的。就地更新；不要留下过时的条目。
