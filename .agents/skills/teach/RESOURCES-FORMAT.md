# RESOURCES.md 格式

`RESOURCES.md` 是此主题的精选可信来源集合。解释说明中的知识应从此处提取，而非依靠参数化猜测。智慧来自此处列出的社区。

## 结构

```md
# {主题} 资源

## 知识

- [书籍：_The Science and Practice of Strength Training_ — Zatsiorsky & Kraemer](https://example.com)
  关于训练计划和适应的基础著作。适用于：任何涉及周期化、恢复、强度区间的内容。
- [文章："How Much Should I Train?" — Greg Nuckols (Stronger By Science)](https://example.com)
  基于证据的训练量指标综述。适用于：每个肌群的每周组数目标。

## 智慧（社区）

- [r/weightroom](https://reddit.com/r/weightroom)
  高信号 subreddit，严格管理抵制伪科学。适用于：计划评审、平台期故障排查。
- 本地：{健身房名称} 周二力量课
  适用于：举重动作的实时教练反馈。
```

## 规则

- **仅高可信度来源。** 优先选择一手来源、公认专家、同行评审作品以及有严格管理的社区。如果某个资源是披着教育外衣的营销内容，不要收录。
- **为每个条目添加注释。** 光秃秃的链接三个月后就毫无意义了。加一行：涵盖什么内容以及何时查阅。
- **按知识 / 智慧分组。** 与 [SKILL.md](./SKILL.md) 中的教学哲学对应。一个资源只出现在一个分组中即可。
- **明确标注空白。** 如果使命需要的某个领域没有好的资源，写一个 `## 空白` 章节列出缺失内容。这会推动未来的搜索。
- **果断清理。** 一个被证明错误、肤浅或偏离使命的资源应当删除，而非埋没。宁可要五个精良来源，不要三十个平庸的。
- **记录社区偏好。** 如果用户选择不加入社区，在这里注明，这样未来的会话就不会一直推荐。
