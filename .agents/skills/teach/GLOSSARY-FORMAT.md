# GLOSSARY.md 格式

`GLOSSARY.md` 是本教学工作区的标准用语。所有讲解、练习和学习记录都应遵循其术语。构建词汇表本身就是学习的一部分：将一个概念压缩为简洁的定义，是用户理解该概念的证明。

## 结构

```md
# {主题} 词汇表

{用一两句话描述本词汇表涵盖的主题。}

## 术语

**肌肉肥大（Hypertrophy）**：
通过反复训练，由机械张力和代谢压力驱动的肌肉增长。
_避免使用_：增肌、变大

**渐进超负荷（Progressive overload）**：
通过负荷、训练量或强度随时间系统性增加对肌肉的需求。
_避免使用_：更努力地练、升级

**RPE（Rate of Perceived Exertion，自感用力程度）**：
对一组训练感受的 1-10 分自评，10 分为力竭，8 分表示还有两次可做。
_避免使用_：努力分数、强度评分
```

## 规则

- **仅在用户理解术语后才添加。** 词汇表是压缩知识的记录，不是用户用来学习的词典。如果用户刚刚接触一个概念，等他们能正确使用时再收录到这里。
- **要有主见。** 当同一概念存在多个词汇时，选择最好的一个，将其余列为应避免的别名。这就是语言压缩的方式。
- **定义要紧凑。** 一到两句话。定义术语是什么，而非它做什么或如何做。
- **在定义内部使用词汇表自身的术语。** 一旦术语进入词汇表，在各处优先使用——包括在其他定义中。这就是复杂术语后来更容易掌握的原因。
- **出现自然聚类时使用子标题分组**（例如 `## 解剖学`、`## 编程`）。当术语彼此关联时，扁平列表也可以。
- **明确标注歧义。** 如果某个术语在更广泛的领域中用法松散，注明解决方案："在本工作区中，'组'始终指正式训练组——热身组单独跟踪。"
- **随着理解加深而修订。** 用户在第一周写的定义到第六周可能是错的。原地更新；不要留下过时的条目。
