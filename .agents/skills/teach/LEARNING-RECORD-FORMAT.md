# 学习记录格式

学习记录存放在 `./learning-records/` 中，使用连续编号：`0001-slug.md`、`0002-slug.md` 等。目录按需创建——仅在写出第一条记录时才创建。

它们是教学领域的 ADR 等价物：记录非显而易见的经验教训、关键洞察以及会影响后续学习会话的已声明先验知识。它们用于计算最近发展区。

## 模板

```md
# {所学内容或已确立知识的简短标题}

{1-3 句话：学到了什么（或确立了哪些先验知识），以及为什么这对未来的学习会话很重要。}
```

这就是全部格式。一条学习记录可以只有一个段落。其价值在于记录 _这个知识点现在是已知的_ 以及 _它如何改变下一步的教学内容_——而不是填满各个章节。

## 可选章节

仅在这些内容真正增值时才包含。大多数记录不需要它们。

- **Status** frontmatter（`active | superseded by LR-NNNN`）——当早期的理解后来被证明是错误并被替代时有用。
- **Evidence**——用户如何证明这种理解（回答了一个问题、完成了一个练习、引用了先前的经验）。当这种说法可能被重新审视时有用。
- **Implications**——这对未来学习会话解锁了什么或排除了什么。当不明显时值得记录。

## 编号

扫描 `./learning-records/` 找到最高现有编号，加 1。

## 何时写学习记录

当以下任一情况成立时写一条：

1. **用户展示了对某个非平凡内容的真正理解**——不只是接触过，而是有证据表明他们能正确使用该概念。这为接下来的教学设定了新的底线。
2. **用户披露了先验知识**——"我已经知道 X。"记录下来，这样未来的会话就不会重新教它。同时记录声称的 _深度_。
3. **纠正了一个误解**——用户之前相信了错误的东西，现在明白了为什么。这些是高价值记录：它们能预测相关主题未来的绊脚石。
4. **使命因学习而转变**——用户发现他们关心的东西与之前想的不同。交叉链接到 [[MISSION.md]] 并更新它。

### 什么不算

- 仅仅覆盖过的内容。覆盖不是学习。等待证据。
- 已经简洁地作为术语定义记录在 [[GLOSSARY.md]] 中的内容。不要重复。
- 逐次会话的活动日志。学习记录不是日记——它们是决策级别的洞察。

## 替代

当后来的记录与之前的相矛盾（用户的理解深化或修正了），将旧记录标记为 `Status: superseded by LR-NNNN` 而不是删除它。理解如何演变的历史本身就是有用的信号。
