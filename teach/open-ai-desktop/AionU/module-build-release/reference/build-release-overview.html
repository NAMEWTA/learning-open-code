1. 模块职责

一句话概括：构建与发布模块把 AionU 源码、Electron 三端 bundle、aioncore runtime、Hub 离线资源、WebCLI tarball 和平台安装包组织成可分发、可更新、可校验的发布资产。

它不是独立 npm package，而是跨根脚本、桌面配置、共享脚本、Homebrew 模板、Dockerfile 和测试目录形成的“发布子系统”。从 L0 视角看，它对应 项目总览中的构建与部署层；它产出的桌面包会被 主入口模块 运行，WebCLI tarball 运行后则进入 Web 运行时模块。

2. 架构位置

方向连接对象说明
上游输入package.json、源码、public、resources、bun.lock命令入口、依赖锁、静态资源和 runtime 资源是构建输入。
编译层electron.vite.config.ts输出 main、preload、renderer，多页面 renderer 和内置 MCP server bundle。
打包层electron-builder.yml、build-with-builder.js生成 dmg/zip、NSIS、deb，并控制 asar、extraResources、afterPack、afterSign。
runtime 资源prepare-aioncore.js、prepareHubResources.js把 aioncore、managed resources、Hub index 和扩展压缩包放入可分发包。
发布整理prepare-release-assets.sh、verify-release-assets.sh把 CI 多平台产物整理成扁平 release-assets，并验证 updater metadata 与 WebCLI 资产。
安装分发install-web.sh、install-ubuntu.sh、homebrew/aionui.rb.example提供 WebCLI 一键安装、Ubuntu/Debian 安装和 Homebrew cask 模板。

6. 调用示例

6.1 桌面包：macOS 多架构

bun run build-mac
# 展开后：
node scripts/build-with-builder.js auto --mac --arm64 --x64

该命令会让构建主控识别多架构模式，先选择一个架构用于 Electron Vite 编译，再把 --arm64 --x64 传给 electron-builder。macOS 目标来自 open-ai-desktop/AionU/packages/desktop/electron-builder.yml：dmg 与 zip。

6.2 Windows 本地快速构建

bun run build-win:x64:fast
# package.json 中包含：
# ELECTRON_BUILDER_COMPRESSION_LEVEL=1
# --config.win.signAndEditExecutable=false

这个入口用于降低压缩成本并避开本地开发机上常见的可执行文件编辑或锁文件问题。单测会确认这三个关键片段仍在脚本中。

6.3 WebCLI tarball

PACK_PLATFORM=linux PACK_ARCH=x64 node scripts/pack-web-cli.js
# 产物：
# dist-web-cli/aionui-web-版本-linux-x86_64.tar.gz
# dist-web-cli/aionui-web-版本-linux-x86_64.tar.gz.sha256

WebCLI 包不带 node_modules。脚本用 Bun compile 生成独立可执行文件，再复制 out/renderer、同平台 aioncore bundle 和 stamped package.json。

6.4 发布资产整理

./scripts/prepare-release-assets.sh build-artifacts release-assets
./scripts/verify-release-assets.sh release-assets

第一条命令收集桌面安装包、WebCLI tarball、checksum、updater metadata 和 install-web.sh；第二条命令验证 metadata 指向的文件存在，并检查预期平台资产。

8. 风险与新发现

Dockerfile 疑似漂移：open-ai-desktop/AionU/Dockerfile 调用 bun run build:renderer:web 和 node scripts/build-server.mjs，但当前读取到的 open-ai-desktop/AionU/package.json 没有 build:renderer:web 脚本，优先文件清单中也未出现 scripts/build-server.mjs。因此本 L1 不把 Dockerfile 作为主发布路径，只标记为后续需要确认的容器化路径。

风险定位文件建议排查顺序
打包后 app 缺 aioncore 或 managed resourcesprepare-aioncore.js、afterPack.js、verify-bundled-aioncore-resources.js先看 prepare 输出目录，再看 afterPack missing 列表，最后对照单测 fixture。
macOS 有 `.app` 但没有 `.dmg`build-with-builder.js检查 DMG retry、hdiutil 清理和 --prepackaged 重试日志。
Windows 本地构建被锁文件阻断build-with-builder.js、NSIS 相关脚本检查 AionUi/Electron 进程、stale win output 清理和 retry without executable edit。
release-assets 缺 updater metadataprepare-release-assets.sh确认各平台 job 输出目录命名和 latest*.yml 文件名。
WebCLI 安装失败或 checksum mismatchpack-web-cli.js、install-web.sh确认 tarball 命名、版本 tag、mirror 路径和 SHA256 文件内容。
