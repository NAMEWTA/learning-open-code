1. 模块定位

Conversation runtime 位于 renderer 的 conversation 页面域，是 renderer 内最重要的业务运行时之一。它接住 /conversation/:id 路由，把“具体会话对象”装配成可交互工作台：上方是标题和模型控制，中间是消息与工具调用，右侧是 workspace，旁边还能挂 preview。它位于 L0 项目总览 之下、renderer-core 之内，向下依赖 common/chat 协议与 preview/workspace 子系统。

能力面主文件职责
页面入口packages/desktop/src/renderer/pages/conversation/index.tsx读取路由参数、加载会话、处理删除回退、切换会话时清理 preview。
会话总装packages/desktop/src/renderer/pages/conversation/components/ChatConversation.tsx把壳层、平台实现、模型选择和 workspace 信息组装成一个页面。
统一壳层packages/desktop/src/renderer/pages/conversation/components/ChatLayout/index.tsx管理标题栏、消息区、preview、workspace、响应式布局。
平台实现packages/desktop/src/renderer/pages/conversation/platforms/acp/AcpChat.tsx / packages/desktop/src/renderer/pages/conversation/platforms/aionrs/AionrsChat.tsx分别处理两种 agent runtime 的消息流、发送框、模型与模式配置。
消息渲染packages/desktop/src/renderer/pages/conversation/Messages/MessageList.tsx统一消费 text、tips、tool、plan、thinking、artifact 等消息。
运行时闸门packages/desktop/src/renderer/pages/conversation/runtime/useConversationRuntimeView.ts把 backend runtime summary 变成 UI 可用的“能否发送 / 是否处理中”快照。
候选回链packages/desktop/src/common/chat/chatLib.ts消息类型、slash command、导航拦截等协议定义；可回链到 common-adapter 主题继续读桥接层。

2. 对外接口与页面入口

入口 / 接口形式说明
/conversation/:id路由页面Renderer 中真正的单会话工作区入口。
ChatConversation页面总装组件对外暴露统一会话 UI，内部按 conversation type 分配平台实现。
ChatLayout布局容器承载 title、header actions、workspace、preview 和消息主体。
packages/desktop/src/renderer/pages/conversation/platforms/acp/AcpChat.tsxACP 平台组件把 ACP 消息流、发送框、权限与工具调用接到公共消息列表。
packages/desktop/src/renderer/pages/conversation/platforms/aionrs/AionrsChat.tsxaionrs 平台组件把本地 agent 运行时、模型选择与 workspace 绑定到公共布局。
packages/desktop/src/renderer/pages/conversation/runtime/useConversationRuntimeView.ts运行时接口为 sendbox、命令队列和停止按钮提供统一状态闸门。
packages/desktop/src/renderer/pages/conversation/Preview/context/PreviewContext.tsx跨页上下文让消息、workspace、导航工具都能打开预览并向 sendbox 回填内容。
packages/desktop/src/renderer/pages/conversation/Workspace/index.tsx右侧面板暴露文件树、变化列表、拖入导入与文件打开点。

6. 调用 / 阅读示例

示例 1：打开一个会话

packages/desktop/src/renderer/components/layout/Router.tsx 命中 /conversation/:id 后进入 packages/desktop/src/renderer/pages/conversation/index.tsx，再用 getConversationOrNull() 读取会话，最后把结果交给 ChatConversation。

示例 2：发送一条消息

平台 sendbox 先读 packages/desktop/src/renderer/pages/conversation/runtime/useConversationRuntimeView.ts 的 gate，再决定立即发送还是进入 packages/desktop/src/renderer/pages/conversation/platforms/useConversationCommandQueue.ts；消息到达后统一落进 packages/desktop/src/renderer/pages/conversation/Messages/MessageList.tsx。

示例 3：从消息打开文件预览

packages/desktop/src/renderer/pages/conversation/Messages/components/MessageText.tsx 解析 workspace 文件路径，调用 useLocalFilePreview()，最终由 packages/desktop/src/renderer/pages/conversation/Preview/context/PreviewContext.tsx 打开或复用 preview tab。

推荐阅读顺序：先读 packages/desktop/src/renderer/pages/conversation/index.tsx、packages/desktop/src/renderer/pages/conversation/components/ChatConversation.tsx 和 packages/desktop/src/renderer/pages/conversation/components/ChatLayout/index.tsx 建立骨架；再选一条竖切路径继续深挖：消息流走 packages/desktop/src/renderer/pages/conversation/platforms/acp/AcpChat.tsx / packages/desktop/src/renderer/pages/conversation/platforms/aionrs/AionrsChat.tsx 加 packages/desktop/src/renderer/pages/conversation/Messages/MessageList.tsx，文件工作台走 packages/desktop/src/renderer/pages/conversation/Workspace/index.tsx 加 packages/desktop/src/renderer/pages/conversation/Preview/context/PreviewContext.tsx，运行时门控走 packages/desktop/src/renderer/pages/conversation/runtime/useConversationRuntimeView.ts 加 packages/desktop/src/renderer/pages/conversation/platforms/useConversationCommandQueue.ts。
