2. 沿途层次

层主职责关键判断
创建 UITeamCreateModal 收集名称、leader assistant、workspace，形成 assistant-first payload。没有名称或 leader 不发请求；provider sentinel error 会转成用户可见错误。
common adapter / HTTPipcBridge.team 将 create/get/add/send/run-state 映射到 backend HTTP，将 run 与 agent 事件映射为 emitter。本 L2 不重复 L1 接口表，详见 common adapter 与 Team Mode 参考。
页面入口TeamIndex 用路由 id 调 team.get，把 TTeam 交给 TeamPage。创建成功不等于页面已完成事件订阅，页面必须从路由重新装载。
Team 页面状态TeamPage 组合 workspace、权限 provider、run view、tab slot 和 chat slot。页面只编排 renderer 状态，不实现 backend Team 调度。
成员 slotTeamTabsContext 保持 leader 在前、teammate 可排序，保存 active slot；TeamTabs 显示状态点和待确认徽标。active slot 被删除时回退 leader 或第一个 assistant。
发送注入TeamChatView 复用 ACP / aionrs Chat，只注入 teamSendMessage 与 teamRuntime。leader 调 team send；member 调 slot send。
运行态回流useTeamRunView 把 ack、run event、child turn event 和 reconnect reconcile 合成 slot 级 gate。终态 run 清空 active state；事件 team id 不匹配则忽略。
E2E 证据team-create.e2e.ts 与 team-agent-lifecycle.e2e.ts 验证创建、导航、成员添加和 assistant-first contract。测试环境没有可用 assistant 时按设计 skip，而不是误报产品失败。

3.1 添加成员支线

添加成员的 renderer 包装点在 useTeamSession.addAssistant()：它把新 teammate 输入交给 ipcBridge.team.addAgent，再刷新当前 Team record。生命周期 E2E 直接调用同一桥接契约，随后断言新 teammate tab 出现在 team-tab-bar，并用 team.get 证明 Team record 中保留了 assistant_id。

const addAssistant = useCallback(

    await mutateTeam();
  },
  [team.id, mutateTeam]
);

4. 运行与回流

ack 快速反馈

ITeamRunAck 被 ackToRunEvent() 转成带 slot_work 的 run event，避免 UI 等到 WebSocket 才显示 busy。

WebSocket 校准

runAccepted/runStarted/runUpdated 更新 active run；runCompleted/runCancelled/runFailed 清空终态运行视图。

child turn 映射

childTurnStarted 把 slot 标为 active；terminal child turn 移除该 slot 的 active turn。

slot gate

buildTeamSendRuntime() 根据 slotWorkBySlot 和 childTurnsBySlot 决定 sendbox 是否可发、是否显示 loading。

用户输入
  → TeamChatView.teamSendMessage
  → ipcBridge.team.sendMessage / sendMessageToAgent
  → common adapter HTTP provider
  → ITeamRunAck 立即进入 useTeamRunView
  → run* / childTurn* WebSocket 事件继续校准
  → TeamTabs 状态点、slot sendbox gate、stop handler 更新

真实订阅还包括 childTurnCancelled，以及 listChanged/sessionChanged/agentSpawned/agentRemoved/agentRenamed 触发的 reconcile()。这些结构变更事件会按 team_id 过滤，避免其他 Team 的事件污染当前页面。

5. 异常与边界

边界代码落点处理策略
创建时缺少名称TeamCreateModal.handleCreate()显示 warning、聚焦名称输入框、提前 return。
创建时未选 leaderTeamCreateModal.handleCreate()显示 warning、提前 return；确认按钮也会被禁用。
无支持的 assistantTeamCreateModal 与 team-create.e2e.tsUI 显示空态；E2E 截图后 skip。
bridge provider 返回 sentinel errorTeamCreateModal识别 __bridgeError 并展示创建错误，不调用 onCreated。
成员添加没有 slot_idteam-agent-lifecycle.e2e.ts清理已创建 Team，然后 skip，避免污染后续测试。
active tab 被删除TeamTabsContext优先切回 leader slot，否则回退到第一个 assistant。
停止 slot 时 run 已过期teamSendRuntime.isStaleTeamRunPauseError()触发 onRunStateStale() 重新拉取 run state；reconcile 失败才提示 stop error。
事件不属于当前 TeamuseTeamSession 与 useTeamRunView按 team_id 过滤，防止跨 Team 状态串扰。
