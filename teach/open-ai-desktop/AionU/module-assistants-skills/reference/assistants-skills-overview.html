1. 模块职责

能力面入口文件职责不负责
助手预设 open-ai-desktop/AionU/packages/desktop/src/renderer/pages/settings/AssistantSettings/index.tsx 展示 My / Official 助手、打开编辑器、创建、复制、启用、排序、删除、启动会话。 不直接持久化 assistant 数据；持久化走 ipcBridge.assistants。
助手编辑状态机 open-ai-desktop/AionU/packages/desktop/src/renderer/hooks/assistant/useAssistantEditor.ts 加载 detail、技能和 MCP catalog；处理 create/edit/duplicate/save/delete/toggle。 不实现 backend 校验；只做前端必填检查和错误提示。
技能中心 open-ai-desktop/AionU/packages/desktop/src/renderer/pages/settings/SkillsHubSettings.tsx 展示 custom、official、extension、auto-inject 技能；支持搜索、导入、删除、导入历史和 highlight。 不解析 Skill 包内容；解析、导入和路径管理走 /api/skills/*。
工具与 MCP open-ai-desktop/AionU/packages/desktop/src/renderer/pages/settings/ToolsSettings/index.tsx 提供独立 Tools 设置页，并嵌入 MCP 与图像生成工具配置。 不直接写 MCP 配置文件；CRUD、OAuth 和连接测试走 mcpService。
接口契约 open-ai-desktop/AionU/packages/desktop/src/common/adapter/ipcBridge.ts 把 renderer 调用包装为 assistants、skills、mcp 三组 HTTP API。 不包含 backend 业务实现。

3. 接口清单

3.1 Assistants: /api/assistants/*

adapterHTTP endpoint主要调用方用途
assistants.list.invoke()GET /api/assistantsuseAssistantList.ts加载统一排序后的 builtin + user + generated catalog。
assistants.get.invoke({ id, locale })GET /api/assistants/:id?locale=...useAssistantEditor.ts读取 profile、rules、prompts、defaults、capabilities、preferences。
assistants.create.invoke(request)POST /api/assistantshandleSave()创建 user assistant。
assistants.update.invoke(request)PUT /api/assistants/:idhandleSave()更新 assistant；builtin 仅提交 agent/defaults 子集。
assistants.delete.invoke({ id })DELETE /api/assistants/:idhandleDeleteConfirm()删除 user assistant。
assistants.setState.invoke(request)PATCH /api/assistants/:id/statereorderAssistants()、handleToggleEnabled()更新 enabled、sort_order、last_used_at。
assistants.import.invoke(request)POST /api/assistants/import当前模块未见直接调用批量导入 assistant 请求契约。

3.2 Skills 与 assistant rules: /api/skills/*

adapterHTTP endpoint主要调用方用途
fs.listAvailableSkills.invoke()GET /api/skillsSkillsHubSettings.tsx、useAssistantEditor.ts读取 builtin、custom、cron、extension 技能清单。
fs.importSkills.invoke({ skill_path })POST /api/skills/importSkillsHubSettings.tsx、handleSave()从目录或 zip 导入一个或多个 Skill。
fs.deleteSkill.invoke({ skill_name })DELETE /api/skills/:skill_nameSkillsHubSettings.tsx删除 custom skill。
fs.listSkillImportHistory.invoke()GET /api/skills/import-historySkillsHubSettings.tsx读取导入历史并按 operation 分组。
fs.getSkillImportLimits.invoke()GET /api/skills/import-limitsSkillsHubSettings.tsx显示导入大小限制和失败修复提示。
fs.detectCommonSkillPaths.invoke()GET /api/skills/detect-pathsadapter 暴露；当前 SkillsHubSettings.tsx 未直接调用，E2E helper / 外部源测试使用探测常见 Skill 路径。
fs.detectAndCountExternalSkills.invoke()GET /api/skills/detect-externaladapter 暴露；当前 SkillsHubSettings.tsx 未直接调用，E2E helper / 外部源测试使用读取外部源及其可导入 Skill。
fs.getCustomExternalPaths.invoke()GET /api/skills/external-pathsadapter 暴露；当前 SkillsHubSettings.tsx 未直接调用，E2E helper / 外部源测试使用读取用户添加的外部 Skill 源。
fs.addCustomExternalPath.invoke(data)POST /api/skills/external-pathsadapter 暴露；当前 SkillsHubSettings.tsx 未直接调用，E2E helper / 外部源测试使用添加命名外部源。
fs.removeCustomExternalPath.invoke({ path })DELETE /api/skills/external-paths?path=...adapter 暴露；当前 SkillsHubSettings.tsx 未直接调用，E2E helper / 外部源测试使用移除外部源。
fs.writeAssistantRule.invoke(data)POST /api/skills/assistant-rule/writepersistAssistantRules()保存非 builtin assistant 的 rules 文本。
fs.deleteAssistantRule.invoke({ assistant_id })DELETE /api/skills/assistant-rule/:assistant_idpersistAssistantRules()rules 为空时删除 assistant rule。
fs.materializeSkillsForAgent.invoke(data)POST /api/skills/materialize-for-agent会话运行时相关调用方把选中 Skill 物化给某个 agent，会话链路另讲。

3.3 MCP Tools: /api/mcp/*

adapterHTTP endpoint主要调用方用途
mcpService.listServers.invoke()GET /api/mcp/servershooks/mcp/catalog.ts读取 backend 管理的 MCP servers。
mcpService.createServer.invoke(data)POST /api/mcp/serversuseMcpServerCRUD.ts新增 MCP server。
mcpService.importServers.invoke(data)POST /api/mcp/servers/importuseMcpServerCRUD.ts批量导入 MCP server。
mcpService.updateServer.invoke(data)PUT /api/mcp/servers/:iduseMcpServerCRUD.ts、ToolsModalContent.tsx更新 server 配置；图像生成工具会同步 stdio env。
mcpService.deleteServer.invoke({ id })DELETE /api/mcp/servers/:iduseMcpServerCRUD.ts删除用户 MCP server。
mcpService.toggleServer.invoke({ id })POST /api/mcp/servers/:id/toggleuseMcpServerCRUD.ts、ToolsModalContent.tsx切换 server enabled 状态。
mcpService.getAgentMcpConfigs.invoke()GET /api/mcp/agent-configshooks/mcp/catalog.ts读取 agent 自带 MCP 配置并判断可导入性。
mcpService.testMcpConnection.invoke(server)POST /api/mcp/test-connectionuseMcpConnection.ts测试连接并回填 tools、auth 状态和错误。
mcpService.checkOAuthStatus.invoke(data)POST /api/mcp/oauth/check-statususeMcpOAuth.ts检查 HTTP/SSE MCP 的 OAuth 状态。
mcpService.loginMcpOAuth.invoke(data)POST /api/mcp/oauth/loginuseMcpOAuth.ts启动 OAuth 登录。
mcpService.logoutMcpOAuth.invoke(data)POST /api/mcp/oauth/logoutuseMcpOAuth.ts退出 MCP OAuth。

6. 测试证据与空白

测试文件覆盖行为对模块的意义
open-ai-desktop/AionU/tests/e2e/features/settings/skills/manual-import.e2e.tsmock 原生文件选择，导入文件夹 Skill，验证 custom source。证明 Skills Hub 与 dialog + importSkills 链路可端到端工作。
open-ai-desktop/AionU/tests/e2e/features/settings/skills/search.e2e.tsMy Skills 和 external skills 的命中/空结果搜索。锁定前端过滤行为。
open-ai-desktop/AionU/tests/e2e/features/settings/skills/refresh-empty-tabs.e2e.ts刷新、空状态、外部源 tab 切换。证明 UI 不自动感知外部变更，需刷新重新拉取。
open-ai-desktop/AionU/tests/e2e/features/settings/skills/batch-import.e2e.ts批量导入，已存在项跳过。验证导入结果应通过 bridge 再确认。
open-ai-desktop/AionU/tests/e2e/features/settings/skills/edge-cases.e2e.ts无外部源、无导出目标、highlight 指向不存在 skill。锁定异常参数不能让页面崩溃。
open-ai-desktop/AionU/tests/e2e/features/settings/skills/special-cases.e2e.ts特殊字符名称和较大数量渲染。提醒 data-testid 需要规范化 skill name。
open-ai-desktop/AionU/tests/e2e/specs/assistant-settings-crud.e2e.ts 等 assistant-settings-*.e2e.ts助手 CRUD、prompts、defaults、permissions、skills/MCP 默认项、迁移和会话默认值。证明 AssistantSettings 不是测试空白，改保存链路时应同步跑这些规格。
open-ai-desktop/AionU/tests/e2e/specs/ext-mcp.e2e.ts扩展 MCP 基本链路。Tools MCP 有较浅层端到端证据，但 CRUD、OAuth、图像生成 toggle 仍需要更直接覆盖。

测试空白：Skills Hub 有 dedicated features/settings/skills 套件；AssistantSettings 有 specs/assistant-settings-* 覆盖。Tools MCP 目前以 ext-mcp.e2e.ts 和单元测试为主，修改 ToolsModalContent 或 hooks/mcp 时，应补充 MCP CRUD、OAuth、图像生成 toggle 的直接测试。
