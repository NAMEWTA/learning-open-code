1. 模块定位

open-ai-desktop/AionU/mobile/ 是 AionU 的独立移动端源码工程，使用 Expo Router 组织页面，用 React Context 保存连接与会话状态，用 REST + WebSocket 直接连远端 AionU backend。它不承担模型执行、会话持久化或文件读取本体，而是把这些后端能力变成移动场景下的连接页、聊天页、文件页和设置页。

L0 回链：先看 00-overview 的模块候选 或 项目地图短课，再回到本页看 mobile/ 作为“远程入口”的局部结构。

层面主路径职责
页面入口open-ai-desktop/AionU/mobile/app/Expo Router 页面、Tabs、Drawer、连接与文件预览入口。
共享状态open-ai-desktop/AionU/mobile/src/context/连接、会话、聊天、工作区、文件标签页等长生命周期状态。
通信与协议open-ai-desktop/AionU/mobile/src/services/REST 配置、WebSocket 生命周期、bridge 请求/订阅协议、待发首条消息缓存。
UI 组件open-ai-desktop/AionU/mobile/src/components/聊天气泡、输入栏、确认卡、文件预览、侧边栏、主题文本等复用 UI。
Hooks 与配置open-ai-desktop/AionU/mobile/src/hooks/、src/constants/、src/i18n/主题色、平台 hook、国际化资源和翻译初始化。
适配逻辑open-ai-desktop/AionU/mobile/src/utils/消息拼装、时间线分组、工作区显示名、JWT 解析、UUID 生成。
工程配置app.config.ts、eas.json、metro.config.js、jest.config.ts、scripts/build.js、versions/Expo 构建、EAS 发布、Metro/Jest 配置、构建脚本和版本产物。
回归证据open-ai-desktop/AionU/mobile/__tests__/锁定协议层、工具函数和消息组合逻辑，保证移动端入口稳定。

2. 对外接口与页面入口清单

入口路径说明
根路由守门open-ai-desktop/AionU/mobile/app/index.tsx根据 isConfigured、connectionState、isRestoring 决定去 /connect 还是 /(tabs)/chat。
连接页open-ai-desktop/AionU/mobile/app/connect.tsx扫码或粘贴 /qr-login URL，换 JWT，等待 WebSocket 就绪后进入聊天页。
根 layoutopen-ai-desktop/AionU/mobile/app/_layout.tsx初始化 i18n、provider 栈与顶层 Stack。
Tabs layoutopen-ai-desktop/AionU/mobile/app/(tabs)/_layout.tsx定义 chat、files、settings 三个底部页签，并在未配置连接时退回根入口。
聊天抽屉 layoutopen-ai-desktop/AionU/mobile/app/(tabs)/chat/_layout.tsx把聊天页包进 Drawer，左侧是会话侧边栏，标题显示当前会话和工作区。
聊天页open-ai-desktop/AionU/mobile/app/(tabs)/chat/index.tsx在待建会话、空态、已选会话三种状态之间切换，并按会话创建 ChatProvider。
文件抽屉 layoutopen-ai-desktop/AionU/mobile/app/(tabs)/files/_layout.tsx把文件浏览页包进 Drawer，左侧是工作区文件树。
文件页open-ai-desktop/AionU/mobile/app/(tabs)/files/index.tsx根据当前工作区和标签页状态展示空态或文件内容。
设置页open-ai-desktop/AionU/mobile/app/(tabs)/settings/index.tsx展示连接状态、服务器地址、重连与断开入口。
文件预览页open-ai-desktop/AionU/mobile/app/file-preview.tsx按扩展名决定 markdown/code/html/diff/image/unsupported 预览方式，通过 bridge 读文件或图片 base64。

阅读顺序建议固定成一条链：先看 app/index.tsx 和 app/connect.tsx 的用户入口，再看 ConnectionContext.tsx、WebSocketContext.tsx、ConversationContext.tsx、ChatContext.tsx 的状态接力，最后回到 bridge.ts 与 websocket.ts 理解它们依赖的协议与恢复策略。UI、Hooks、i18n 和工程配置先作为边界层扫读，等进入具体页面或构建问题时再下钻。

4. Context 清单

文件核心状态 / 接口职责
open-ai-desktop/AionU/mobile/src/context/ConnectionContext.tsxconfig、connectionState、connect()、disconnect()、tryReconnect()保存服务器地址和 JWT，负责 SecureStore 持久化、token 刷新和连接恢复。
open-ai-desktop/AionU/mobile/src/context/WebSocketContext.tsxbridge、wsService把底层通信单例注入 React，并在应用回到前台时自动重连。
open-ai-desktop/AionU/mobile/src/context/ConversationContext.tsxconversations、availableAgents、activeConversationId、pendingAgent会话列表读取、Agent 列表读取、新建会话、重命名、删除、前台唤醒后刷新。
open-ai-desktop/AionU/mobile/src/context/ChatContext.tsxmessages、isStreaming、confirmations、thought、sendMessage()加载历史消息、订阅流式响应、拼装消息、处理确认卡片和停止生成。
open-ai-desktop/AionU/mobile/src/context/WorkspaceContext.tsxcurrentWorkspace、workspaceDisplayName、workspaceChanged从当前会话推导工作区身份，给聊天标题和文件页共享使用。
open-ai-desktop/AionU/mobile/src/context/FilesTabContext.tsxtabs、activeTabIndex、openTab()、closeAllTabs()管理文件页的多标签状态，并通过 AsyncStorage 做轻量持久化。

5. Service 与协议清单

文件作用关键点
open-ai-desktop/AionU/mobile/src/services/api.tsREST 客户端配置维护 axios baseURL 和 Bearer token；refreshToken() 走 /api/auth/refresh。
open-ai-desktop/AionU/mobile/src/services/websocket.ts长连接管理器token 作为 WebSocket protocol 传递，支持 ping/pong、鉴权过期、指数退避、死连接检测与重连。
open-ai-desktop/AionU/mobile/src/services/bridge.ts请求/订阅桥在 wsService 之上提供 request()、emit()、on()、once()，把业务事件变成命名调用。
open-ai-desktop/AionU/mobile/src/services/pendingInitialMessages.ts新会话首条消息暂存在会话创建和 ChatContext 真正挂载之间，用内存 Map 传递首条用户输入。

协议视角：移动端的业务 API 主要不是 HTTP Controller 的“页面式调用”，而是通过 bridge.request('database.get-user-conversations')、bridge.request('chat.send.message')、bridge.on('chat.response.stream') 这类命名事件直接复用 AionU 既有后端能力。

6. Utils 清单

文件用途典型消费者
open-ai-desktop/AionU/mobile/src/utils/messageAdapter.ts把 WebSocket 响应消息转换为移动端 TMessage，并负责流式文本/工具调用合并。ChatContext.tsx
open-ai-desktop/AionU/mobile/src/utils/groupingHelpers.ts按时间线和工作区分组会话列表，支持 pinned 会话和临时工作区标签。聊天侧边栏历史列表
open-ai-desktop/AionU/mobile/src/utils/timeline.ts计算 today / yesterday / recent7Days / earlier 标签。groupingHelpers.ts
open-ai-desktop/AionU/mobile/src/utils/workspace.ts判断临时工作区、提取展示名与末级目录名。WorkspaceContext.tsx、历史分组、文件页
open-ai-desktop/AionU/mobile/src/utils/jwt.ts无依赖解析 JWT payload。ConnectionContext.tsx、websocket.ts
open-ai-desktop/AionU/mobile/src/utils/uuid.ts生成消息、确认卡片等临时 ID。ChatContext.tsx、messageAdapter.ts

7. 关键依赖

依赖在 package.json 中的角色在模块中的体现
expo + expo-router移动工程运行时和文件路由决定 app/ 下页面结构和 `_layout.tsx` 的组织方式。
@react-navigation/*Tabs / Drawer / 导航内核聊天和文件页都叠了一层 Drawer，根 layout 使用 Tabs 和 Stack。
expo-secure-store安全保存连接配置与 JWTConnectionContext.tsx 在冷启动恢复连接时直接读取。
axiosREST 调用二维码登录和 token 刷新用它补上 WebSocket 之外的鉴权流程。
expo-camera扫码connect.tsx 用摄像头扫描桌面端生成的登录二维码。
i18next + react-i18next多语言连接页、设置页、工作区标签和错误提示统一走翻译键。
jest + jest-expo单元测试底座覆盖 service 与 utils，保证协议层和适配层可回归。
