模块职责

Main process 基础设施位于 Electron main 层和 aioncore backend 之间。它不直接实现聊天、Team 或 Cron 业务，而是负责让这些业务有稳定运行环境：Chromium 启动参数、用户数据目录、配置存储、legacy 数据迁移、IPC bridge、main process i18n、更新服务、backend 二进制解析、启动失败诊断、托盘/窗口/WebUI 配置。

在整体架构中，open-ai-desktop/AionU/packages/desktop/src/index.ts 是 Electron 总入口；open-ai-desktop/AionU/packages/desktop/src/process/index.ts 是基础设施汇合点；@aionui/web-host 和 aioncore 是后续运行时依赖。判断启动问题时，优先按“Electron app ready → initializeProcess → backend start → mode branch → window/WebUI”顺序定位。

入口与接口清单

这一节分成两部分看：先确认哪些文件是主进程基础设施的稳定入口，再确认这些入口真正向外暴露了哪些函数、常量或存储代理。L1 关注的是“别人从哪里接入”和“接入后能拿到什么”。

入口文件对外作用阅读重点
open-ai-desktop/AionU/packages/desktop/src/process/index.ts 导出 initializeProcess()，注册初始化副作用。 区分导入副作用与显式 await initStorage()。
open-ai-desktop/AionU/packages/desktop/src/process/utils/initStorage.ts 导出 ProcessConfig、ProcessEnv、getSystemDir() 等存储基础设施。 目录、配置拦截、助手/技能目录、legacy DB 迁移。
open-ai-desktop/AionU/packages/desktop/src/process/utils/initBridge.ts 副作用注册所有 main process bridge。 下一跳是 open-ai-desktop/AionU/packages/desktop/src/process/bridge/index.ts。
open-ai-desktop/AionU/packages/desktop/src/process/services/i18n/index.ts 初始化 main process i18n，并导出语言切换能力。 静态导入 locale，避免生产包运行时读文件失败。
open-ai-desktop/AionU/packages/desktop/src/process/backend/index.ts 对外再导出 backend 二进制解析能力。 向上层屏蔽 open-ai-desktop/AionU/packages/desktop/src/process/backend/binaryResolver.ts 的实现细节。
open-ai-desktop/AionU/packages/desktop/src/process/backend/binaryResolver.ts 实现 aioncore 二进制路径解析和诊断。 先查 packaged resources，再查系统 PATH，并附带诊断。
导出接口来源文件作用
initializeProcess() open-ai-desktop/AionU/packages/desktop/src/process/index.ts 主进程基础设施总入口；等待存储、迁移和系统目录提供者就绪。
ProcessConfig、ProcessEnv open-ai-desktop/AionU/packages/desktop/src/process/utils/initStorage.ts main process 的配置与环境变量存储代理，供其他模块统一读写。
ProcessChat、ProcessChatMessage open-ai-desktop/AionU/packages/desktop/src/process/utils/initStorage.ts 对旧版聊天会话和消息文件的读写代理，兼容 legacy 数据布局。
getSystemDir() open-ai-desktop/AionU/packages/desktop/src/process/utils/initStorage.ts 返回 cache、work、log 目录及平台/架构信息，供 bridge 暴露系统信息。
getAssistantsDir()、getSkillsDir()、getCronSkillsDir() open-ai-desktop/AionU/packages/desktop/src/process/utils/initStorage.ts 返回用户助手规则、用户技能和 cron job 技能目录路径。
BUILTIN_IMAGE_GEN_ID、getBuiltinMcpScriptPath() open-ai-desktop/AionU/packages/desktop/src/process/utils/initStorage.ts 内建 MCP 图像服务标识与 entry script 路径解析，供 main process 启动外部 node 进程。
BridgeDependencies、initAllBridges() open-ai-desktop/AionU/packages/desktop/src/process/bridge/index.ts bridge 聚合层的类型与注册入口，统一挂接 renderer 可调用的原生能力。
initApplicationBridge()、initDialogBridge()、initNotificationBridge()、initSystemSettingsBridge()、initThemeBridge()、initUpdateBridge()、initWindowControlsBridge()、initWebuiBridge() open-ai-desktop/AionU/packages/desktop/src/process/bridge/index.ts 分别暴露各子 bridge 的显式注册函数，便于测试或按需装配。
registerWindowMaximizeListeners()、disposeAllTeamSessions() open-ai-desktop/AionU/packages/desktop/src/process/bridge/index.ts 前者补充窗口最大化事件监听；后者当前是 team session 清理占位实现。
i18nReady、setInitialLanguage()、changeLanguage()、normalizeLanguageCode() open-ai-desktop/AionU/packages/desktop/src/process/services/i18n/index.ts 提供主进程 i18n 初始化完成信号、语言设置与语言码标准化能力。
default i18n open-ai-desktop/AionU/packages/desktop/src/process/services/i18n/index.ts 导出已初始化的 i18next 实例，供需要直接翻译的主进程服务复用。
resolveBinaryPath() open-ai-desktop/AionU/packages/desktop/src/process/backend/index.ts / open-ai-desktop/AionU/packages/desktop/src/process/backend/binaryResolver.ts 解析 aioncore 可执行文件位置；找不到时抛出附带诊断的错误。
BackendBinaryResolveDiagnostics open-ai-desktop/AionU/packages/desktop/src/process/backend/binaryResolver.ts 描述 bundled 路径、PATH 查找命令和目录枚举等诊断字段。

内部分层

packages/desktop/src/process/ ├── index.ts ← initializeProcess 汇合点 ├── utils/ ← 主进程工具与运行时配置 │ ├── configureChromium.ts ← app name、headless、CDP、GPU 恢复前置 │ ├── initStorage.ts ← 配置/目录/legacy DB 迁移 │ ├── initBridge.ts ← bridge 注册副作用 │ ├── webuiConfig.ts ← 桌面托管 WebUI 配置与恢复 │ ├── tray.ts ← 托盘与关闭到托盘状态 │ ├── windowBounds.ts ← 窗口尺寸持久化 │ └── deepLink.ts ← aionui:// 链接分发 ├── services/ ← 主进程服务 │ ├── i18n/index.ts ← main process 翻译资源 │ ├── autoUpdaterService.ts ← 自动更新状态机 │ ├── autoUpdateDiagnostics.ts← 更新诊断事件 │ ├── installerLastFailure.ts ← 安装器失败标记消费 │ └── database/ ← legacy SQLite 迁移与修复 ├── backend/ ← backend 二进制解析 │ ├── index.ts ← resolveBinaryPath 再导出 │ └── binaryResolver.ts ← bundled/PATH 查找与诊断 └── startup/ ← 相邻启动编排层（不属于本模块内部基础设施） ├── backendStartup.ts ← backend 启动成功/失败包装 ├── backendStartupFailure.ts← 失败分类 ├── quitCleanup.ts ← 退出清理 └── architectureCompatibility.ts ← macOS 架构兼容检查

open-ai-desktop/AionU/packages/desktop/src/process/startup/ 与本模块紧邻，负责消费这里准备好的基础设施继续编排启动链，但它不是 open-ai-desktop/AionU/packages/desktop/src/process/ 基础设施层的内部实现；阅读时要把“准备环境”和“驱动启动”区分开。

层关键依赖为什么重要
Electron appapp、BrowserWindow、ipcMain提供主进程 API、路径、窗口、IPC 和协议注册。
本地存储ConfigStorage、EnvStorage、better-sqlite3启动前必须建立目录、配置拦截和 legacy DB 基线。
Bridge@office-ai/platform logger、open-ai-desktop/AionU/packages/desktop/src/process/bridge/*统一注册 renderer 可调用的桌面原生能力。
BackendBackendLifecycleManager、resolveBinaryPath()决定 aioncore 从哪里启动，以及失败时提供何种诊断。
服务electron-updater、i18next、更新诊断支撑自动更新、语言、安装完整性与用户可见错误。
