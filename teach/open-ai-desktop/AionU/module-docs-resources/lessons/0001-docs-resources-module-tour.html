docs/resources 模块：判断资料是否进入产品契约

学习目标：15 分钟内能把 AionU 的文档、示例和静态资源分成正确层次，并能追到一个文件的实际调用方。

L1 模块总览 预计 12 分钟 主来源：docs README + public/build 配置 + extension 示例

回链：先看 L0 项目地图； 发布资源看 module-build-release； WebUI 静态服务看 module-web-runtime； 助手/技能扩展入口看 module-assistants-skills； 完整清单看 本模块 reference。 本模块位于产品叙事、扩展生态、Web 运行时和构建发布的交叉层。

1. 模块职责

这个模块的核心判断不是“哪些文件不是源码”，而是“哪些资料会影响用户、贡献者、发布包或扩展生态”。open-ai-desktop/AionU/docs/README.md 负责按读者意图导航；open-ai-desktop/AionU/examples/hello-world-extension/aion-extension.json 暴露扩展 manifest 契约；open-ai-desktop/AionU/resources/ 既服务 README 图片，也服务安装器与打包资源；open-ai-desktop/AionU/public/ 则会被 renderer 和 WebUI 当成运行时静态文件。

读它时先用一个规则：如果文件被 README、构建配置、WebHost、renderer 或 E2E 引用，它就是产品契约的一部分；如果只描述未来方案，就把它当成设计证据，并检查是否和现有源码漂移。

2. 一条主线：从文件到调用方

以 `public` 和 `resources` 为例，资料文件会沿两条路径进入产品。第一条是 WebUI：Vite 把 public/ 作为 renderer publicDir，registerPwa() 注册 ./sw.js，pet 页面用 ../pet-states/idle.svg 加载状态图。第二条是发布包：Electron Builder 把 public/**/*、图标、aioncore 和 Hub 离线资源放进安装包。

files:
  - public/**/*

extraResources:
  - from: public
    to: .
  - from: resources/app.png
    to: app.png
  - from: resources/bundled-aioncore
    to: bundled-aioncore
  - from: resources/hub
    to: hub
判断一：README 图片只影响产品叙事，但路径错了会直接破坏 GitHub 首页展示。
判断二：public/sw.js 和 public/manifest.webmanifest 会影响浏览器 WebUI，可交叉阅读 Web 运行时。
判断三：resources/windows/*.nsh、resources/app.ico、resources/hub 会影响打包和安装，可交叉阅读构建发布。

3. 分层地图

module-docs-resources/ ├── docs/ ← 用户指南、贡献者规则、PRD、README 翻译 ├── examples/ ← 扩展 manifest、skills、assistants、themes、channels 示例 ├── .aionui/ ← AI/开发协作需求模板与方案草案 ├── resources/ ← README 媒体、图标、安装器脚本、bundled runtime、Hub fallback └── public/ ← renderer/WebUI 可访问静态文件、PWA、pet-states

这张图也给出依赖方向：`docs` 和 `.aionui` 主要约束人和 AI 如何协作；`examples` 约束扩展生态如何声明能力；`resources` 和 `public` 会进入构建输出或运行时访问路径。修改前先问“调用方是谁”，比先问“这个文件像不像代码”更可靠。

4. 检索练习

暂停回忆：如果你要替换 WebUI 的 PWA 图标，同时更新 README 首页截图，应分别检查哪些路径和下游调用方？

参考答案与反馈

PWA 图标先查 open-ai-desktop/AionU/public/pwa/、public/manifest.webmanifest、renderer index.html 和 registerPwa.ts，再确认 electron-builder.yml 是否仍包含 public/**/*。README 截图先查 open-ai-desktop/AionU/resources/ 与 readme.md 的相对路径。若只替换图片而不检查构建配置或引用路径，你只完成了文件编辑，没有验证产品契约。

推荐主来源：继续阅读 接口清单，然后打开 docs/README.md 与 electron-builder.yml 对照本课的分层判断。下一步可以追问 agent：“请沿着一个 extension manifest 从 examples 追到 settings tabs 和 WebUI routes”。
