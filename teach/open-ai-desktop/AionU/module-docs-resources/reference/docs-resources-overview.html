3.1 docs：读者意图接口

目录或文件声明职责当前实际内容使用建议
docs/README.md文档总导航。定义 guides、contributing、architecture、specs、prds、readme 的放置规则。新增文档前先对照此文件，避免把指南/PRD/架构草稿放到根目录。

5. 调用示例

5.1 README 图片引用

<img src="./resources/aionui-banner-1.png" alt="AionUi - Cowork with AI Agents" width="100%">

root README 和翻译 README 大量引用 resources/ 图片/GIF。改名或移动资源会破坏 GitHub 首页与多语言文档。

5.2 public 进入 renderer 与发布包

electron.vite.config.ts
  renderer.publicDir = resolve('public')

electron-builder.yml
  files:
    - public/**/*
  extraResources:
    - from: public
      to: .

这说明 public/ 同时参与开发构建和桌面发布包，变更后要检查 WebUI/PWA/pet。

5.3 PWA service worker 注册

const SERVICE_WORKER_URL = './sw.js';
const registration = await navigator.serviceWorker.register(SERVICE_WORKER_URL, { scope: './' });

registerPwa.ts 只在非 Electron、支持 service worker、HTTP(S)/localhost 或 secure context 下注册；这就是 WebUI/PWA 的运行时边界。

5.4 pet 状态资源加载

const PET_STATES_BASE_PATH = '../pet-states';
function getStateAssetPath(state: string): string {
  return `${PET_STATES_BASE_PATH}/${state}.svg`;
}

pet 页面不是 import SVG，而是运行时用 object 加载相对路径，因此 public/pet-states 缺失会变成运行时问题。

5.5 extension manifest 到 UI 查询

examples/e2e-full-extension/aion-extension.json
  contributes.skills / themes / assistants / settingsTabs / channelPlugins

ipcBridge.extensions
  getSkills / getThemes / getAssistants / getSettingsTabs / getWebuiContributions

tests/e2e/specs/ext-capabilities.e2e.ts
  getExtensionSnapshot(page)

扩展示例不是孤立样板；E2E 会验证 contribute 类型能被 adapter 查询并在设置页打开。

5.6 Hub 离线资源进入发布包

scripts/build-with-builder.js
  -> node scripts/prepareHubResources.js
  -> resources/hub/index.json + extension packages
  -> electron-builder extraResources

Hub 资源是扩展生态与发布包的交叉点。若构建时跳过或下载失败，运行时 fallback 能力会下降。

7. 新发现与缺口

缺失架构/规格目录：open-ai-desktop/AionU/docs/README.md 指向 docs/architecture/overview.md 和 docs/specs/，但当前检出没有对应目录。L0 已记录 architecture 缺失；本模块补充记录 specs 也缺失。

发现证据影响建议
extension schema 缺失examples/hello-world-extension/aion-extension.json 引用 ../../schemas/aion-extension-v1.json，但当前没有 schemas/。贡献者无法用仓库内 schema 校验 manifest。后续补齐 schema 或移除/更新 $schema 引用。
development.md 脚本漂移文档列出 build:renderer:web、build:server、server:start 等 standalone server 命令，当前 package.json 未提供。贡献者照文档执行会失败。确认是否已迁移到 webui / web-cli 路径，更新脚本表。
scripts README 构建流漂移scripts/README.md 仍描述 Electron Forge / webpack，当前主路径是 electron-vite + electron-builder。发布排障时会误导读者。以 module-build-release reference 的构建路径重写脚本文档。
.aionui 模板存在旧布局FEATURE_DEV_TEMPLATE.md 示例使用旧 src/agent、src/webserver 等布局。AI worker 按模板生成改动时可能偏离当前 packages/desktop/src。用 docs/contributing/file-structure.md 的当前结构刷新模板。
README 翻译依赖大量资源相对路径readme.md 和 docs/readme/readme_ch.md 都引用多张 resources/* 图片/GIF。资源重命名需要多语言同步。改资源路径时用全文搜索覆盖 root README 与翻译副本。
