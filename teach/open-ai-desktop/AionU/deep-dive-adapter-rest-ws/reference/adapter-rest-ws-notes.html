1. 设计框架

问题判断落点
为什么还叫 ipcBridge这是 renderer 稳定调用面，不是传输承诺。common/index.ts 暴露调用面，ipcBridge.ts 决定真实传输。
什么能力走 REST真实业务状态归 aioncore，且结果可由请求/响应表达。httpGet/httpPost/httpPut/httpPatch/httpDelete
什么能力走 WS结果异步、连续、可由事件名分发。wsEmitter/wsMappedEmitter
什么能力继续 IPC副作用依赖 Electron main、本机窗口、原生对话框、自动更新、托盘、通知。bridge.buildProvider/buildEmitter
为什么 WebUI 重要浏览器无 preload，不能依赖 Electron IPC；same-origin /api/* 和 /ws 才能复用 renderer contract。httpBridge.ts、browser.ts、web-host

3. 错误与事件

主题规则维护动作
非 2xxhttpRequest 读取 body，JSON 优先，抛 BackendHttpError。新增 UI 分支时用 isBackendHttpError，不要解析错误字符串。
机器字段status/code/backendMessage/details/body 必须保留。后端新增错误码后，在 renderer 分类器里补映射。
软失败silentStatuses 只降低日志噪音，不吞错误。404 等预期失败仍应由调用方 catch。
backend WSwsEmitter 订阅事件名，消息取 name/event 与 data/payload。事件不进 UI 时先核对事件名和 payload shape。
重连断线指数退避；二次打开后派发 realtime.reconnected。长会话 UI 应考虑重连后刷新状态。
main bridge 事件adapter/main.ts 广播传统 bridge 事件到 BrowserWindow，emit 最后调用 broadcastToAll(name, data)，并有 50 MB payload 上限。不要把它误认为 backend stream；这是 Electron main 本地事件通道。

WebUI browser bridge 与 backend WS 的边界

分支连接方式可靠性行为维护注意
browser.ts 无 electronAPI连接同源 /ws，由 web-host 代理到 backend。发送前先 ensureSocket()；未打开时进入消息队列；断线指数退避重连；收到 ping 立即回 pong。认证缺失或过期属于 terminal error，会停止重连、清理 timer、关闭 socket，并跳转登录页。
httpBridge.ts WS singletonElectron renderer 走 ws://127.0.0.1:{port}/ws，WebUI browser mode 走同源 /ws。按事件名分发到 wsEmitter listener；二次打开后派发 realtime.reconnected。它承载 backend 业务事件，不负责 browser runtime bridge 的 emit 队列。

4. E2E 复用路径

Helper证明范围不能证明
helpers/httpBridge.tsrenderer context 可读取 window.__backendPort，HTTP 请求、DELETE 默认 body、响应 data 解包符合 app 路径。不能独立证明 backend 内部业务规则正确。
helpers/bridge/invoke.tslegacy dotted key 优先映射 HTTP route；route 不存在时才 fallback 到旧 subscribe-* IPC。不能保证所有 ipcBridge key 都已映射；只覆盖 E2E 实际需要的 key。
helpers/bridge/routes.ts测试侧显式记录 key 到 method/path/body mapper 的 contract。不是生产路由真值；生产真值仍在 ipcBridge.ts 与 backend route。
helpers/bridge/mappers.ts测试断言保持旧 camelCase 语义。不能替代生产 mapper；测试 mapper 只服务测试输入输出。

5. 性能与局限

区域当前特征已知局限可补强方向
HTTP helper保留 provider-like 调用形状，每次 invoke 直发 fetch。没有内建 retry、abort timeout、请求合并或幂等保护。对长耗时/可取消操作引入 AbortSignal 和调用方可配置超时。
WS singleton一个 renderer 共享一个 backend WS 连接，断线指数退避。重连只通知 realtime.reconnected，不会自动重放所有业务查询。为关键页面定义重连后 refresh contract。
main bridgeBrowserWindow 广播前序列化 payload，并拒绝超过 50 MB 的事件。大对象事件仍会消耗序列化成本；超限后只通过 bridge:error 告知。大 payload 改成文件引用、分页查询或 backend 拉取。
E2E route mapHTTP_ROUTES 只覆盖当前测试需要的 legacy key。不能证明所有 ipcBridge key 都已完成 REST 映射。增加 route drift 检测，或从生产 contract 生成测试映射基线。

6. 维护检查表

新增 adapter 能力时：

1. 判断真实状态/副作用归属：aioncore、Electron main，还是混合。

2. REST provider：确认 method、URL、params、body mapper、响应解包。

3. WS emitter：确认事件名、payload shape、是否需要 wsMappedEmitter。

4. 错误：确认后端错误 envelope 是否有 code/error/details，renderer 是否按 isBackendHttpError 分类。

5. mapper：确认 snake_case、枚举、路径、默认值是否会破坏旧前端 contract。

6. E2E：优先在 renderer context 复用 HTTP bridge；legacy key 需要加入 route map，或明确保留 IPC fallback。
