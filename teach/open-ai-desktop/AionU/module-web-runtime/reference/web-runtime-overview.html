1. 模块定位

Web 运行时位于 AionU 的“非 Electron 窗口宿主”层：向下继续复用同一个 backend 与 renderer 静态资源，向上则提供可以独立执行的 CLI 入口。它与 L0 总览 中的桌面壳并列，是应用的第二种宿主形态；与 main-entry 的关系是“桌面入口决定是否切到 WebUI”，与 renderer 的关系则是“只负责把打包后的静态资源和 API 代理暴露给浏览器”。

能力面主文件职责
运行时组合器packages/web-host/src/index.ts组合 backend 与静态服务器，返回统一的 WebHostHandle。
backend 生命周期packages/web-host/src/backend-launcher.ts解析 backend 二进制、选择端口、spawn、健康检查、停止与诊断。
静态与代理宿主packages/web-host/src/static-server.ts提供 SPA 静态服务、API 代理、登录路径转发、WebSocket/TCP splice。
agent 清理packages/web-host/src/agent-process-registry.ts在 backend 停止前清理历史登记的 agent 进程树。
CLI 启动入口packages/web-cli/src/index.ts解析命令、定位资源目录、启动 WebHost、处理自动开浏览器与优雅退出。
浏览器策略packages/web-cli/src/browser.ts决定默认是否自动打开浏览器，并按平台生成 opener 命令。
首启密码自举packages/web-cli/src/ensureAdminPassword.ts轮询认证状态，必要时调用 /api/webui/reset-password 生成初始密码。
用户操作文档docs/guides/webui.md从最终用户视角说明跨平台启动、远程访问和排障。

2. 对外接口清单

入口 / 接口形式说明
startWebHost(opts)库函数WebHost 主入口。接收已解析好的 app / staticDir / backend 选项，返回统一 handle。
startStaticServer(opts)库函数单独启动静态服务门面，供 WebHost 组合器或 frontend-only 模式使用。
stopStaticServer(handle)库函数关闭 TCP listener 与内部 HTTP server。
startBackend(opts)库函数独立 backend 启动函数，包装 BackendLifecycleManager。
stopBackend(handle)库函数停止 backend 句柄。
BackendLifecycleManager类长期持有 backend 进程、端口、状态与重启窗口。
buildSpawnArgs() / buildSpawnEnv()纯函数把 app metadata、目录配置和运行标志转换成 backend 启动参数。
findAvailablePort()纯函数寻找浏览器 fetch 兼容端口，跳过被浏览器禁用的端口号。
aionui-web startCLI 命令独立启动 WebUI，支持本地或远程访问。
aionui-web resetpassCLI 命令临时拉起 backend，重置管理员密码后立即关闭。
shouldAutoOpenBrowser()纯函数本地模式默认自动打开浏览器，远程模式默认关闭，可被 flag/env 覆盖。
ensureAdminPassword()服务函数确保首启时能给用户一组可登录的管理员凭据。

层次输入输出边界判断
CLI 决策层命令行参数、环境变量、打包目录结构WebHostOptions、日志提示、退出码负责“怎么启动”和“给用户看什么”，不直接实现网络转发。
WebHost 组合层WebHostOptionsWebHostHandle只组合 backend 与 static server，不管理持久配置。
backend 生命周期层app metadata、binary resolver、dirs、端口偏好BackendHandle、诊断错误负责 “aioncore 是否起来了”。
静态宿主层静态目录、backendPort、监听地址浏览器可访问 URL、API 代理与 WS 路由负责 “浏览器怎么接入 backend”。

4. 启动与调用链

场景主链你该先读什么
正常独立启动main() → runStart() → startWebHost() → startBackend() + startStaticServer()先读 packages/web-cli/src/index.ts，再下钻到 WebHost。
frontend-only 降级runStart() 发现 backend bin 缺失 → 直接 startStaticServer()先读 CLI 的 backendAvailable 判断，再看 static-server 的 502 行为。
远程访问模式--remote / env → allowRemote=true → static-server 绑定 0.0.0.0 并计算 networkUrl先读 resolveAllowRemote() 与 startStaticServer() 的 host 选择。
首启密码生成runStart() 成功后 → ensureAdminPassword() → /api/auth/status → /api/webui/reset-password先读 ensureAdminPassword.ts 与单元测试。
resetpassmain() → runResetPassword() → 临时 startWebHost() → POST reset-password → stop先读 CLI 的 runResetPassword()，不要从 WebHost 组合器开始猜。
页面能开但 API 失败前端请求到 static-server，若 backend 不通则得到 502 / ECONNREFUSED先判断是否 frontend-only 或 backend 未就绪，再看 forwardToBackend()。

推荐阅读顺序：先读 packages/web-cli/src/index.ts 的 runStart() 和 runResetPassword()，再看 packages/web-host/src/index.ts 的组合器，最后分支阅读 backend-launcher.ts 与 static-server.ts。如果你是从桌面入口跳过来的，先回看 main-entry 短课，确认 WebUI 模式是在哪里被选中的。

5. 关键依赖

依赖用途在本模块的体现
Node.js `child_process`启动与停止 backend 子进程backend-launcher.ts 的 spawn()、Windows 下 taskkill。
Node.js `http` / `net`静态服务、反代与 TCP splicestatic-server.ts 同时维护内部 HTTP server 与用户侧 TCP listener。
`serve-handler`静态资源与 SPA fallback把 staticDir 暴露为浏览器入口，并将未知路径重写到 index.html。
浏览器 fetch 端口规则避免 backend 选到被浏览器禁止的端口findAvailablePort() 会跳过例如 6000、6667 之类 fetch-blocked 端口。
打包目录约定让独立 CLI 能找到静态资源与 bundled backendresolveCliRoot()、resolveBackendBinary()、resolveStaticDir() 共同依赖 tarball 布局。
后端认证接口首启密码与 resetpass/api/auth/status、/api/webui/reset-password、/api/auth/internal/users/system。
