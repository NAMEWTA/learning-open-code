1. 概念边界

概念持久化/注册内容运行时语义常见误判
Assistantprofile、engine、rules、prompts、defaults、capabilities、preferences。创建会话时解析默认值和 override，生成 assistant snapshot。把 Assistant 当作 Skill 文件或 MCP server 的拥有者。
Skillbuiltin/custom/cron/extension 的技能目录项，字段包含 name、description、location、source、is_auto_inject。会话创建或发送时作为 skill id/name 注入，旧会话保留自己的 skills snapshot。以为取消自动 Skill 会删除 builtin Skill。
MCPserver id、name、enabled、transport、original_json、tools、last_test_status。会话创建时选择 user MCP ids 和 session MCP servers；连接测试仅是可用性检查。把 Tools 页的 connected 状态当作当前会话实时状态。
Extension通过 manifest 贡献 assistants、agents、skills、mcpServers 等条目。进入对应 registry，部分在设置页以只读方式展示。以为 extension assistant 自动绑定所有 extension skill/MCP。

2. 数据流

当前 Assistant 默认能力和会话快照链路里，所谓 Skill id 通常表现为 skill name；读 skill_ids、enabled_skills 或 resolved_skill_ids 时，应先按名称语义理解，再回查后端是否另有内部标识。

阶段入口关键动作落点
Assistant 编辑useAssistantEditor并行加载 assistant detail、Skill 列表、MCP catalog；保存 defaults 和 capabilities。API /api/assistants，API /api/skills/assistant-rule
Skill 导入SkillsHubSettings 或 pending skill 保存路径调用 importSkills，刷新 listAvailableSkills，Assistant 只保存 skill names。API /api/skills/import，API /api/skills
自动 builtin SkillAssistantEditorSections把 auto-inject builtin skill 合入显示值，取消时写 disabled list。disabled_builtin_skills
MCP catalogensureBackendMcpCatalog合并 backend user servers 与本地 builtin servers，按 user/builtin 名称去重。Assistant 默认 MCP 下拉与 Tools 页列表
MCP CRUDMcpManagement创建、批量导入、编辑、删除、toggle、测试连接、OAuth 登录。API /api/mcp/servers，API /api/mcp/test-connection
Extension 贡献examples manifestsmanifest 的 contributes 分别进入 extension endpoints。API /api/extensions/skills，API /api/extensions/mcp-servers，API /api/extensions/assistants
Guid 创建会话useGuidSend解析 Assistant 默认值与用户 override，发送 skill ids、disabled builtin ids、mcp ids、session MCP servers。API /api/conversations 与 backend snapshot
会话显示ChatConversation从 conversation extra 读取 loaded skills、loaded MCP names/statuses，传入 ACP/Aionrs chat context。发送框和附件菜单的 loaded capability 展示

5. 性能、可靠性、局限、改进

类别源码事实维护判断
性能useAssistantEditor 用 Promise.all 并行加载 detail、skills、MCP；useMcpConnection 批量测试默认 concurrency 为 4。设置页打开和批量导入后测试连接不会故意串行阻塞。
可靠性ensureBackendMcpCatalog 对 builtin/user 分开去重；useMcpConnection 成功回填 tools，失败只写 UI 状态并提示。MCP 目录避免明显重复；失败不会误写成会话状态。
可靠性McpServerHeader 明确提示手动检查不代表当前会话实时状态。排查“会话没用 MCP”时不要只看 Tools 页 connected。
局限指定的 ext-skills / ext-mcp 主要是页面可用性断言。需要补更强的 extension-to-assistant-to-conversation 链路测试。
局限docs/prds/assistants/README.md 为空。缺少可引用的产品设计背景，课程只能从源码反推。
局限extension MCP 在 McpManagement 中只读展示，edit/delete 为空操作。用户若想修改 extension MCP，需要改 extension 或导入为用户 MCP。
改进新增 E2E：选中 extension skill/MCP 创建 Assistant，再创建会话并断 snapshot。覆盖当前最薄弱的 extension 映射链路。
改进新增 bridge-level probe：读取 create payload 与 backend snapshot 的 skill/MCP ids。减少只靠 UI 文本长度判断的假阳性。
改进为 Assistant PRD 补充“目录、默认值、会话快照”三层模型。降低新维护者把 Assistant/Skill/MCP 混为一谈的概率。

最短排障路径：先问问题发生在哪个时间点。设置页保存前查 editor state；保存后新会话前查 API /api/assistants/{id}；创建会话时查 create payload；进入聊天页后查 conversation extra 和 snapshot。不要用一个页面状态推断所有层。
