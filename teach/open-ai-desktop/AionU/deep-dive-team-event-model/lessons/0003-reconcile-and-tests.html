Team Mode 事件模型：reconcile、slot gate 与测试缺口

学习目标：能判断结构事件为何要触发 reconcile，并识别 slot gate 的防串扰边界。

核心：reconcile + teamSendRuntime 测试证据与缺口
1. 结构事件触发 reconcile 2. slot gate 防串扰 3. 测试证据与缺口 4. 小任务

本课延续 核心合成机制，重点看状态丢失、结构变化和测试边界。

1. 结构事件触发 reconcile

WebSocket 事件不是可靠日志，renderer 也不是 Team 状态的最终事实源。useTeamRunView 因此在页面加载、实时连接恢复、Team 结构变化和 session 变化时调用 getRunState。结构事件包括 listChanged、sessionChanged、agentSpawned、agentRemoved、agentRenamed。它们本身不一定携带 run 明细，但会改变 slot 集合或名称，使旧的 slot work 索引有 stale 风险。

ipcBridge.realtime.reconnected.on(() => {
  void reconcile('realtime.reconnected');
}),
ipcBridge.team.listChanged.on((event) => {
  if (event.team_id === team_id) void reconcile('team.listChanged');
}),
ipcBridge.team.agentSpawned.on((event) => {
  if (event.team_id === team_id) void reconcile('team.agentSpawned');
}),
ipcBridge.team.agentRemoved.on((event) => {
  if (event.team_id === team_id) void reconcile('team.agentRemoved');
}),

同类订阅还包括 sessionChanged 和 agentRenamed。注意每个结构事件都按 team_id 过滤；这是跨 Team 防串扰的第一道门。

2. slot gate 防串扰

第二道门在 open-ai-desktop/AionU/packages/desktop/src/renderer/pages/team/components/teamSendRuntime.ts。它不问“是否存在 active run”，而是问“当前 slot 有没有 work 或 child turn”。因此 teammate 正在跑时，leader 输入框不一定被锁；leader 有 pending wake 时，leader 输入框才被锁。

const hasSlotWork =
  Boolean(work?.active_turn_id) ||
  (work?.pending_wake_count ?? 0) > 0 ||
  (work?.starting_child_count ?? 0) > 0;
if (hasSlotWork) return true;

const childStatus = runView.childTurnsBySlot[slot_id]?.status;
return childStatus === 'accepted' || childStatus === 'running' || childStatus === 'cancelling';

paused slot 是特例：源码优先返回 false，让用户可以继续 priority send。stop handler 也只会暂停当前 slot 的 work；如果后端说没有 active run 可暂停，才触发 stale reconcile。

3. 测试证据与缺口

单元测试对事件模型覆盖较好：open-ai-desktop/AionU/tests/unit/renderer/useTeamRunView.dom.test.tsx 验证 reconcile 清 stale、失败时保留本地状态、重连触发拉取，并直接覆盖 listChanged 的 team id 过滤；sessionChanged、agentSpawned、agentRemoved、agentRenamed 有源码订阅但缺少逐项测试。open-ai-desktop/AionU/tests/unit/renderer/teamSendRuntime.test.ts 验证 teammate-only work 不锁 leader，当前 slot child turn 才锁当前输入框。

E2E 证据要更谨慎。open-ai-desktop/AionU/tests/e2e/cases/teams/team-member-messaging.e2e.ts 会通过 UI 给成员发送消息，并断言消息可见、成员 active badge 可见或输入清空；team-communication.e2e.ts 验证 leader UI 发送后 Team record 仍有效。但这些测试没有直接监听或断言 ITeamRunAck、team.runUpdated、childTurnStarted 的合并顺序。

特征 / 局限维护含义
indexSlotWork() 每次按 slot 重建索引成本随 slot 数量增长，换来不累加旧 work 的可靠性。
reconcileSeq 防旧请求覆盖新状态重连和结构事件并发时，后返回的旧快照不会倒灌。
reconcile 会清空 childTurnsBySlot牺牲局部事件日志，换取 backend 快照校准后的简洁状态。
E2E 可观测性缺口UI 发送可证明用户路径，但仍需 bridge probe 或 unit 测试覆盖 reducer 顺序。

4. 小任务

任务：如果一个 bug 报告说“成员 A 正在处理时，成员 B 的输入框也被锁住”，你先看哪个文件、哪个判断？

参考答案：先看 teamSendRuntime.ts 的 slot gate，确认传入的 slot_id 是否是成员 B，以及 slotWorkBySlot / childTurnsBySlot 是否错误地写到了成员 B。再回到 useTeamRunView.ts 查事件是否按正确 slot 合并。
