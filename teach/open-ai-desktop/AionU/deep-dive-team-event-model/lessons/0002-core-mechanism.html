Team Mode 事件模型：核心合成机制

学习目标：能解释 ack、run event、child turn event 如何合成 `activeRun`、`slotWorkBySlot` 和 `childTurnsBySlot`。

核心文件：useTeamRunView.ts 前置：0001-problem-frame
1. ack 先占位 2. run event 覆盖 3. child turn 改局部 4. 判断练习

本课只看状态合成，不重讲 Team 创建流程。详细表格见 reducer 速查。

1. ack 先占位

TeamChatView 发送 leader 消息时调用 ipcBridge.team.sendMessage，成员 tab 则调用 sendMessageToAgent。两者返回的 ack 都交给 onTeamRunAck。在 open-ai-desktop/AionU/packages/desktop/src/renderer/pages/team/hooks/useTeamRunView.ts 中，ack 不单独存一份，而是被转成普通 run event。

const ackToRunEvent = (ack: ITeamRunAck): ITeamRunEvent => ({
  team_id: ack.team_id,
  team_run_id: ack.team_run_id,
  target_slot_id: ack.target_slot_id,
  target_role: ack.target_role,
  status: ack.status,
  active_child_count: 0,
  pending_wake_count: 0,
  starting_child_count: 0,
  slot_work: [
    {
      slot_id: ack.accepted_slot_id || ack.target_slot_id,
      role: ack.accepted_role || ack.target_role,
      pending_wake_count: 1,
      starting_child_count: 0,
    },
  ],
});

这个转换有意“保守”：没有 child turn 时，先让 accepted slot 的 gate 进入 pending。这样用户不会在同一个 slot 重复发送，后续事件到来后再把数量和 active turn 修正。

2. run event 覆盖

WebSocket 的 runAccepted、runStarted、runUpdated 也进入同一个 applyRunEvent()。如果 run 进入 completed、cancelled 或 failed，hook 直接回到空状态；否则用事件里的 slot_work 重建 slotWorkBySlot，但保留已有的 childTurnsBySlot。

if (TERMINAL_RUN_STATUSES.has(event.status)) {
  return emptyState;
}
return {
  activeRun: event,
  childTurnsBySlot: prev.childTurnsBySlot,
  slotWorkBySlot: indexSlotWork(event.slot_work),
};

这里的边界很重要：run event 是团队级快照，负责覆盖 slot work；child turn 是 slot 局部事实，不被普通 run event 顺手清空。否则后端短暂不带某个 slot work 时，前端可能误以为 child turn 已结束。

3. child turn 改局部

childTurnStarted 写入 childTurnsBySlot[slot_id]，并补充该 slot 的 active_turn_id。终态 child turn 则删除这个 slot 的 active turn；如果没有 pending wake、starting child、paused 或 suppressed work，才把该 slot 的 work 删除。

换句话说，run event 决定“这次 Team run 的全局骨架”，child turn event 决定“某个成员现在是否真的有 turn 在跑”。两者以 slot_id 汇合，而不是以 conversation id 或当前 active tab 汇合，这正是防串扰的关键。

4. 判断练习

判断：收到 runCompleted 后，前端是否应该保留旧的 childTurnsBySlot，等待每个 childTurnCompleted 再清理？

答案：不应该。源码把 completed、cancelled、failed 都视为团队级终态，直接返回 emptyState。终态 run 是比局部 child turn 更强的清理信号。
