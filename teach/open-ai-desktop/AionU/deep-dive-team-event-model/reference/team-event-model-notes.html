3. 状态合成规则

规则源码行为维护判断
ack 与 WS 同形ackToRunEvent() 将 ack 转成 ITeamRunEvent 后复用 applyRunEvent()。排查“刚发送不 busy”时，先看 ack 是否进入 hook。
终态 run 强清理completed、cancelled、failed 直接返回 emptyState。终态 run 比局部 child turn 更强，不等待后续 child 事件。
run event 覆盖 slot workslotWorkBySlot 由 event.slot_work 重建。slot work 是后端快照，不要在前端累加旧项。
child turn 局部更新childTurnStarted 只写对应 slot_id；terminal 也只清对应 slot。不要按 active tab 或 conversation id 猜测目标 slot。
reconcile 丢弃 child turn有 active run 快照时保留 active run 和 slot work，但 childTurnsBySlot 重置为空。reconcile 是快照校准，不试图恢复所有局部事件日志。
reconcile 失败保留本地状态catch 后返回 false，不清空现有 state。网络失败时避免误释放 gate；调用方可显示错误或稍后重试。
team id 过滤run、child、结构事件都按 event.team_id !== team_id 忽略。跨 Team 页面打开时，这是防串扰底线。
可靠性 / 性能特征取舍改进方向
indexSlotWork() 重建 slot 索引每次 run event O(slot work 数量)，但能避免旧 work 残留。如果 slot 数增长明显，再考虑 memo 或增量更新。
reconcileSeq 序号防止慢请求覆盖新快照，但不能取消已经发出的请求。可结合 AbortController 降低无效请求成本。
reconcile 清空 child turns承认快照不是事件日志，避免恢复不完整局部状态。后端若提供 active child turns 快照，可补全恢复。
E2E 主要观察 UI 结果接近用户路径，但难证明 reducer 合并顺序。增加 bridge-level run-state probe 或 DOM 单测断言 ack 注入。

4. slot gate 规则

输入状态当前 slot 是否锁理由
别的 slot 有 active_turn_id否buildTeamSendRuntime() 只查当前 slot_id。
当前 slot 有 pending_wake_count > 0是已有待唤醒工作，重复发送会串扰。
当前 slot 有 starting_child_count > 0是child turn 正在启动，输入框进入 processing。
当前 slot 的 child status 为 accepted/running/cancelling是即便 slot work 不完整，child turn 仍说明该 slot 忙。
当前 slot paused: true 且有 suppressed work否源码允许 priority send，不把 suppressed work 当作 processing。
stop 时后端返回 no active team run to pause不直接报错isStaleTeamRunPauseError() 触发 onRunStateStale() reconcile。
