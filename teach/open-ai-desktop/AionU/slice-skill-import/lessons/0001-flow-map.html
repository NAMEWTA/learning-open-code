1. 两个入口

当前源码里最直接的 skill 导入入口在 Skills Hub。用户打开“Add Skill”下拉菜单并选择手动导入后，页面先调用 Electron 原生文件选择，再把选中路径交给统一导入函数。下面是源码节选：注意 catch 分支只显示错误，不会刷新列表。

const handleManualImport = async () => {
  try {
    const result = await ipcBridge.dialog.showOpen.invoke({
      properties: ['openFile', 'openDirectory'],
      filters: [{ name: 'Skill folders or zip archives', extensions: ['zip'] }],
    });
    if (result && result.length > 0) {
      await handleImport(result[0]);
    }
  } catch (error) {
    console.error('Failed to open directory dialog:', error);
  }
};
const handleImport = async (skillPath: string) => {
  try {
    const result = await ipcBridge.fs.importSkills.invoke({ skill_path: skillPath });
    const notice = buildSkillImportNotice(result, t);
    if (notice.type === 'error') {
      Message.error(notice.message);
    } else if (notice.type === 'warning') {
      Message.warning(notice.message);
    } else {
      Message.success(notice.message);
    }
    if (notice.importedNames.length > 0) {
      setSearchQuery('');
      void fetchData();
    } else if (notice.type !== 'success') {
      void fetchData();
    }
  } catch (error) {
    console.error('Failed to import skill:', error);
    Message.error(getSkillImportErrorMessage(error, t));
  }
};

  onChange={(value) => {
    const nextValues = ((value as string[]) ?? []).filter(Boolean);
    const filteredValues = nextValues.filter((item) => item !== AUTO_SELECT_VALUE);
    if (filteredValues.length === 0) {
      setDefaultSkillsMode('auto');

2. 沿途每一层

层导入 skill绑定 assistant
设置页 UITalkToButlerButton 下拉到手动导入。AssistantEditorPage 保存按钮触发 actions.save。
页面状态SkillsHubSettings.fetchData() 读 skill、导入历史和限制。useAssistantEditor 读 assistant detail、skill catalog 和 MCP catalog。
文案与异常buildSkillImportNotice() 区分成功、部分成功、全失败。保存 pending skill 失败时复用 getSkillImportErrorMessage() 并中止保存。
adapteripcBridge.fs.importSkills 映射到 POST /api/skills/import。ipcBridge.assistants.create/update 映射到 /api/assistants。
刷新导入结果分支会按 importedNames 或失败 notice 决定是否 fetchData()；HTTP throw 的 catch 只弹错误，不刷新。保存后刷新 assistant catalog、detail cache，并发出 chat history refresh。

4. 边界与练习

最关键的失败路径在保存前导入 pending skill：一旦后端返回结构化错误，保存会显示 skill 导入错误并直接 return，不会继续创建或更新 assistant。

for (const pendingSkill of skillsToImport) {
  try {
    await ipcBridge.fs.importSkills.invoke({ skill_path: pendingSkill.path });
  } catch (error) {
    console.error(`Failed to import skill "${pendingSkill.name}":`, error);
    message.error(getSkillImportErrorMessage(error, t));
    return;
  }
}

if (skillsToImport.length > 0) {
  const skillsList = await ipcBridge.fs.listAvailableSkills.invoke();
  setAvailableSkills(skillsList);
}

检索练习：如果用户在 AssistantSettings 里选择了一个默认 skill 并保存，哪个字段把这个选择送给后端？如果导入 skill 时后端返回“文件过大”，为什么助手不会被创建？

参考答案：默认 skill 最终进入 enabled_skills 和 defaults.skills.value。文件过大属于 SKILL_IMPORT_FILE_TOO_LARGE，handleSave() 捕获后调用 getSkillImportErrorMessage() 并 return，因此不会执行 assistants.create/update。
