2. 分层图谱

层级导入 skill绑定 assistant 默认 skill状态刷新
用户动作Settings → Skills → Add Skill → Import Skills。Settings → Assistants → Create/Edit → Default Skills。用户不直接刷新；UI 在成功后主动重新读取。
UI 组件TalkToButlerButton 选择 manual action。DefaultsSection 多选控件。AssistantEditorPage 保存按钮。
页面 / hookSkillsHubSettings.handleImport()。AssistantEditorSections.handleSkillSelectionChange() 写入 selectedSkills 与 disabled auto skill。useAssistantEditor.refreshAssistantCatalog()、refreshAssistantDetailCaches()。
消息层skillImportMessages.ts 把后端 code 转为 success/warning/error。保存失败统一落到 common.failed；pending skill 导入失败走 skill import 文案。成功消息来自 common.createSuccess 或 common.saveSuccess。
adapterfs.importSkills → POST /api/skills/import。assistants.create/update → POST /api/assistants 或 PUT /api/assistants/:id。fs.listAvailableSkills、assistants.list、assistants.get。
后端边界aioncore 负责解析目录、父目录、SKILL.md 或 zip，并返回 skill_names / failed。aioncore 持久化 assistant capabilities 和 defaults。再次 GET 是本切片可观察的后端状态确认点。
测试手动导入 E2E mock dialog，最后用 HTTP helper 读取 /api/skills。assistant skills E2E 保存后重新打开 assistant 并断言选择摘要存在。refresh/empty E2E 用 bridge 预置 skill 后重新进入 Skills Hub。

4. 正常与异常路径

4.1 Skills Hub 正常路径

  1. fetchData() 初始读取 listAvailableSkills、listSkillImportHistory、getSkillImportLimits。
  2. handleManualImport() 通过原生 dialog 得到文件或目录路径。
  3. handleImport() 调用 fs.importSkills.invoke({ skill_path })。
  4. buildSkillImportNotice() 根据 skill_names 和 failed 生成成功、警告或错误消息。
  5. notice.importedNames.length > 0 时清空搜索并 fetchData()；notice.type !== 'success' 的 result 失败也会 fetchData()；HTTP throw 的 catch 只显示错误，不刷新。

4.2 AssistantSettings 正常路径

  1. useAssistantList() 从 assistants.list.invoke() 加载 catalog。
  2. handleCreate() 或 handleEdit() 读取 skill catalog，交给 DefaultsSection 渲染。
  3. 用户选择默认 skill 后，handleSkillSelectionChange() 更新 selectedSkills 和被禁用的自动注入 skill。
  4. handleSave() 把选择写入 enabled_skills 和 defaults.skills.value。
  5. 保存成功后刷新 assistant list、detail cache，并发出 chat.history.refresh。

4.3 异常 / 边界路径

边界触发点传播方式用户可见结果
文件过大后端返回 SKILL_IMPORT_FILE_TOO_LARGE。isBackendHttpError() 或 result.failed → getSkillImportCodeMessage()。显示“文件超过大小限制”的本地化错误；pending skill 保存链路中止。
部分成功父目录或 zip 内部分 skill 成功、部分失败。buildSkillImportNotice() 生成 warning，并格式化失败项。显示 warning，随后重新拉取 skill 列表和导入历史。
builtin assistant打开官方助手详情。AssistantEditorSections 判定 isReadOnlyAssistant。不渲染可编辑默认 skill Select，只显示只读摘要。
highlight 不存在URL 里带不存在的 ?highlight=。SkillsHubSettings 找不到 DOM ref，不清除 search param。页面仍显示 My Skills，不崩溃；E2E 断言 section 可见。

6. 边界说明

后端实现边界：本切片在 AionU 前端仓库内没有读到 /api/skills/import 的后端路由实现，后端内部解析和持久化细节应继续进入 aioncore 源码验证。本切片只把可观察边界讲到 ipcBridge HTTP contract、导入结果结构和 E2E 的后端状态读取。

测试资产演进：tests/e2e/features/settings/skills/ 中仍有外部源卡片、批量导入按钮等相邻证据；当前读到的 SkillsHubSettings.tsx 主入口集中在手动导入、导入历史、custom / official tab。本参考把手动导入作为 current source 主链路，把外部源相关测试视为相邻测试资产。

手动导入 selector 漂移：当前 SkillsHubSettings.tsx 给 TalkToButlerButton 传入 data-testid='btn-add-skill'，组件会把手动菜单项渲染为 btn-add-skill-manual。而 manual-import.e2e.ts 仍查找 btn-manual-import。因此它能作为链路意图证据，但在声明测试已通过前必须运行或修正 selector。
