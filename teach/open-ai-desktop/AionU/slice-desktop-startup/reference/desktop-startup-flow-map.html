4. 关键边界路径：backend 启动失败

触发：`backendManager.start()` 抛出非取消错误，或 debug 注入 `AIONUI_DEBUG_BACKEND_STARTUP_FAILURE`。

main 处理：markBackendStartupFailed() 设置 backendStartupFailed、backendStartupFailureInfo 和 globalThis.__backendStartupFailed；startBackendOrExit() 采集失败。在默认桌面模式下 exitOnFailure 为 false，所以流程继续走向窗口。

preload 传播：preload 通过 `get-backend-startup-failed` 和 `get-backend-startup-failure` 同步读取失败状态，暴露给 renderer。

renderer 收口：`renderer/main.tsx` 检查 `window.__backendStartupFailure`，若 reason 属于已知集合，则直接渲染 `BackendStartupFailureDialog`，不进入正常 Router。

这个边界解释了一个容易误判的现象：窗口出现不等于 backend 成功；body 有内容也不等于首屏业务可用。要证明正常首屏，需要在 E2E 中继续断言 `/guid` 或关键 UI 文案。
