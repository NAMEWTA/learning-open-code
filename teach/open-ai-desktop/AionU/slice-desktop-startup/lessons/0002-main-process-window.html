学习目标

本课只看 main process 的成功路径：handleAppReady() 如何先准备 process 基础设施和 backend，再进入默认桌面分支创建窗口。主要来源是 open-ai-desktop/AionU/packages/desktop/src/index.ts:603-850、open-ai-desktop/AionU/packages/desktop/src/process/index.ts:7-29 和 open-ai-desktop/AionU/packages/desktop/src/process/startup/backendStartup.ts:22-38。

process 前置不是只有一个函数

process/index.ts 的顶层 import 已经做了部分副作用：注册 Electron platform、配置 Chromium、按 packaged 状态设置 PREBUILDS_ONLY、导入 bridge 和 main i18n。真正的 initializeProcess() 当前只等待 initStorage()，但它依赖这些前置已经完成。

import '@/common/platform/register-electron';
import '@process/utils/configureChromium';
import initStorage from './utils/initStorage';
import './utils/initBridge';
import './services/i18n';

export const initializeProcess = async () => {
  await initStorage();
};

`initBridge` 继续调用 `initAllBridges()`，注册 dialog、application、window controls、system settings、notification、webui 和 theme 等 main bridge。也就是说 bridge 是启动前置的一部分，不是首屏后才按需注册。

backend 后创建窗口

handleAppReady() 在默认桌面分支前启动 backend。startBackendOrExit() 成功时把 port 交给 onStarted()；主入口再把它暴露到 globalThis.__backendPort，并在 backend 进入 running 时标记 ready、注册系统恢复通知和延迟迁移。

await initializeProcess();
const backendStartup = await startBackendOrExit({
  startBackend: async () => backendManager.start(...),
  onStarted: (backendPort) => {
    exposeBackendPort(backendPort);
    if (backendManager.status === 'running') {
      markBackendReady(backendPort, 'backendManager.start');
    }
  },
  exitOnFailure: isWebUIMode || isResetPasswordMode,
});

注意最后一行：默认桌面模式下 `exitOnFailure` 是 false。失败不会直接结束应用，而是留给 renderer 显示可恢复的启动失败 UI；这是第三课的边界路径。

窗口装载有两层兜底

进入桌面分支后，E2E 模式会禁用关闭到托盘；普通模式会读取托盘设置。随后 createWindow() 建立隐藏窗口、绑定 preload、等首帧或加载完成再显示，并按环境选择 dev URL 或构建后的 HTML。

mainWindow = new BrowserWindow({
  show: false,
  webPreferences: {
    preload: path.join(__dirname, '../preload/index.js'),
    webviewTag: true,
  },
});

if (!app.isPackaged && rendererUrl) {
  mainWindow.loadURL(rendererUrl).catch(() => mainWindow.loadFile(fallbackFile));
} else {
  mainWindow.loadFile(fallbackFile);
}

`ready-to-show`、`did-finish-load` 和 5 秒 fallback 共同避免窗口长期隐藏；`render-process-gone` 会尝试重新加载 renderer。这些边界不证明业务可用，但证明 Electron 壳努力把首屏推出来。

判断练习

判断：initializeProcess() 内部没有调用 initAllBridges()，所以 bridge 要等 renderer 发起 IPC 时才注册。

答案：错。process/index.ts 顶层导入 ./utils/initBridge，而 initBridge.ts 顶层调用 initAllBridges()；注册发生在 process 模块加载阶段。
