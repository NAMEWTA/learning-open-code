4. 调用时序

4.1 Electron `--webui`

sequenceDiagram
  participant Desktop as Electron main
  participant Backend as aioncore backend
  participant Config as webuiConfig
  participant Host as WebHost
  participant Static as Static server
  Desktop->>Backend: startBackendOrExit()
  Backend-->>Desktop: globalThis.__backendPort
  Desktop->>Config: loadUserWebUIConfig()
  Desktop->>Config: resolveWebUIPort() / resolveRemoteAccess()
  Desktop->>Host: startWebHost(useExistingBackend)
  Host->>Static: startStaticServer()
  Static-->>Desktop: localUrl / networkUrl
app.whenReady() -> handleAppReady() -> initializeProcess() -> startBackendOrExit() -> markBackendReady() 暴露 globalThis.__backendPort -> ensureAdminUserOnce() -> isWebUIMode 分支 -> loadUserWebUIConfig() -> resolveWebUIPort() / resolveRemoteAccess() -> startWebHost({ backend: useExistingBackend, port, allowRemote }) -> startStaticServer() -> will-quit 防止无窗口模式意外退出

关键点：`startWebHost()` 里的 backend 选项是 `useExistingBackend`，端口来自 `globalThis.__backendPort`。如果这个端口不存在，代码直接抛出 `[WebUI] Cannot start`，避免 WebUI 静默跑在无 backend 状态。

4.2 桌面设置恢复

普通桌面模式 -> createWindow() -> restoreDesktopWebUIFromPreferences() -> readWebUIDesktopPreferences() -> enabled=false：不启动 -> enabled=true：startDesktopWebUI({ port, allowRemote }) -> startWebHost({ backend: useExistingBackend }) -> 失败：writeWebUIDesktopEnabled(false)

这条路径说明 WebUI 不是只有命令行入口。用户在设置页保存的 `webui.desktop.enabled`、`allowRemote`、`port` 存在 backend 设置表中，启动后异步恢复；失败时会把 enabled 写回 false，避免 UI 继续显示“已开启”但端口拒绝连接。

4.3 独立 `aionui-web start`

main() -> parseArgs() -> runStart() -> resolveBackendBinary() / resolveStaticDir() / resolveDataDir() -> resolvePort() / resolveAllowRemote() -> backend 存在：startWebHost({ backend: ownBackend }) -> backend 缺失：startStaticServer({ backendPort: 0 }) -> ensureAdminPassword() -> shouldAutoOpenBrowser() -> SIGINT/SIGTERM 调用 handle.stop()

WebCLI 的使用场景更像独立部署包。它默认数据目录是 `~/.aionui-web`，生产默认端口是 `25808`，远程模式默认不自动打开浏览器，本地模式默认会尝试打开。

5. 异常与边界路径

边界触发条件实现行为排查意义
远程访问关闭没有 `--remote`、远程 env、host hint 或配置static-server 绑定 `127.0.0.1`，不返回 `networkUrl`。局域网设备访问失败时先确认 allowRemote。
显式远程访问`--remote` 或 `AIONUI_ALLOW_REMOTE=true`static-server 绑定 `0.0.0.0`，尝试取 LAN IP 并返回 `networkUrl`。网络可达还依赖防火墙和同网段。
端口配置无效端口为空、非数字、越界桌面路径 `parsePortValue()` 返回 null，继续使用下一个来源或默认值。不要把无效端口当成启动失败根因。
Electron backend 未就绪`globalThis.__backendPort` 未设置Electron WebUI 和桌面恢复都会抛错；headless 分支退出，桌面恢复会回写 enabled=false。说明要先查 backend 启动链路。
WebCLI backend 缺失找不到 `bundled-aioncore` 或 `AIONUI_BACKEND_BIN`降级为 frontend-only，只服务 SPA；API 代理会失败。页面可打开不等于业务可用。
首启密码生成失败认证状态超时或 reset 接口失败`ensureAdminPassword()` 只警告，不中止服务。用户可继续看到登录页，但需要运行 resetpass。
WebSocket/STT 路由`GET /ws` 或 `GET /api/stt/stream`外层 TCP listener 直接 splice 到 backend。普通 HTTP 代理正常不代表 WS 路径也正常。

6. 证据与漂移

实现证据

`web-host/src/index.ts` 明确声明 WebHost 不保存持久配置；调用方负责解析端口与远程访问。`static-server.ts` 明确服务 SPA、代理 `/api/*`、`/login`、`/logout`，并对 `/ws` 与 STT stream 做 TCP splice。

E2E 证据

`webui.e2e.ts` 验证设置页可渲染、启用开关可见、至少两个开关存在、默认端口或 WebUI 面板可见，并尝试启停服务。它是桌面设置路径的外部行为证据。

文档漂移

`docs/guides/webui.md` 第 25、470、478、497、503 行仍出现 `3000`，而实现默认端口是 `25808` 系列。Android 段和 E2E 更接近当前实现，已写 `25808`。
