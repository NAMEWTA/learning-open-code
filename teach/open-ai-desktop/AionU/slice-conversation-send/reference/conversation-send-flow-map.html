AionRS 主路径时序

顺序文件动作状态变化
1SendBox/index.tsxsendMessageHandler 拒绝上传中/空输入，拼接引用和 DOM 片段，清空草稿，调用 onSend(finalMessage)。本地输入清空，避免异步发送覆盖新草稿。
2AionrsSendBox.tsxonSendHandler 读取上传文件和 @ 文件，清理选择状态。附件从 UI 选择态变成发送参数。
3useConversationCommandQueue.tsshouldEnqueueConversationCommand 在 isBusy 或已有队列时排队；队列限制包括 20 条、单条 20000 字符、50 个文件、总状态 256KB。忙时不会丢消息，而是等待 runtime gate 可执行。
4AionrsSendBox.tsxexecuteCommand 检查模型，调用 runtimeView.markSendStarted() 和 setWaitingResponse(true)。UI gate 进入本地 pending/processing。
5ipcBridge.ts + httpBridge.tssendMessage.invoke 发 POST /api/conversations/:id/messages，body 映射为 content/files/loading_id/inject_skills。HTTP 2xx 返回 turn_id/runtime/msg_id；非 2xx 抛 BackendHttpError。
6conversationRuntimeViewStore.tslocalSendAccepted 清掉 pending，记录 active turn；localSendFailed 清掉 pending 并重置 gate。发送被接受或失败都不会让按钮无限 loading。
7useAionrsMessage.ts订阅 conversation.responseStream，按 thought/content/tool/permission/error 更新 thought、streamRunning、waitingResponse，并调用 mergeLiveMessage。流式 chunk 进入消息列表；error 会停止 loading。
8useConversationRuntimeView.ts订阅 turnCompleted，匹配当前 conversation_id 后调用 turnCompleted。runtime gate 释放，下一条排队命令可 drain。

ACP 差异路径

阶段ACP 做法与 AionRS 的差异
消息流 hookAcpChat 创建 useAcpMessage 并把 messageState 传给 AcpSendBox。AionRS 的 useAionrsMessage 在 AionrsSendBox 内部使用。
初始消息useAcpInitialMessage 从 sessionStorage 读取 acp_initial_message_${conversation_id} 并发送。这是 guid 创建后自动发送的一条特殊入口。
发送 adapteripcBridge.acpConversation.sendMessage 复用 conversation.sendMessage。HTTP endpoint 与 AionRS 一致，差异在前后处理和消息类型。
错误呈现AcpSendBox 捕获错误后调用 markSendFailed，认证错误会 emit error stream，其余错误添加 center tips，并通过 buildSendFailureError 分类。AionRS 路径主要用 Message.error 和 stream error 处理；ACP 对用户可见错误结构化更明显。
