AionRS 发送主路径

把公共 SendBox、平台 sendbox、runtime gate、HTTP 返回和 WS 流式事件拆开看。

课程 0002 主路径：AionRS 返回总图

1. 入口不是 adapter

用户提交时先进入公共 SendBox。它做空输入、上传中、引用和 DOM 片段处理，然后调用平台层传入的 onSend(finalMessage)。真正知道会话类型的是 AionrsSendBox。

await onSend(finalMessage);
setMessage('');
setSelectedFile([]);
setSelectedFiles([]);
setSelectedAgent(null);

2. 平台层决定立即发送还是排队

AionrsSendBox 先收集上传文件和 @ 文件，再看 runtime gate。忙时走 enqueueConversationCommand()，可发送时才调用 executeCommand()。

if (shouldEnqueueConversationCommand({ isBusy, hasPendingCommands })) {
  enqueueConversationCommand({ content: value, files });
  return;
}
await executeCommand(value, files);

3. 请求和流是两条异步线

发送 HTTP 成功只代表后端接受了 turn；真正的助手内容来自 conversation.responseStream。因此调试时要分别看 markSendAccepted() 和 turnCompleted。

sequenceDiagram
  participant Box as AionrsSendBox
  participant Runtime as runtimeView
  participant HTTP as sendMessage HTTP
  participant WS as responseStream
  participant UI as MessageList
  Box->>Runtime: markSendStarted()
  Box->>HTTP: POST /messages
  HTTP-->>Box: turn_id / msg_id
  Box->>Runtime: markSendAccepted()
  WS-->>UI: message.stream chunks
  WS-->>Runtime: turn.completed
  Runtime-->>Box: gate 可继续发送

4. 检索练习

如果按钮一直 busy，先查 conversationRuntimeViewStore.ts 的 pending/accepted/completed 状态，再查 turnCompleted 是否到达。不要只看 HTTP 请求是否 200。
