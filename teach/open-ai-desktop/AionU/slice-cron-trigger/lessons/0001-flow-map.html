入口点

手动入口在 ScheduledTasksPage：列表页用 useAllCronJobs() 读取全部任务，用户点击“手动创建”时只打开 CreateTaskDialog；点击任务卡进入 /scheduled/:job_id。同一页还有“对话创建”入口，它跳到 /guid 并预填 Cron prompt，后续由 AI 使用注入的 HTTP helper 创建或更新任务。

真正把表单变成任务负载的是弹窗的 handleSubmit。下面是真实连续节选：它先校验表单，生成带 timezone 的 schedule，再在可编辑 agent 配置时解析 agent_config。创建 / 编辑分流放在 短课 0002 单独拆开。

const values = await form.validate();
setSubmitting(true);

const scheduleExpr = scheduleInfo.expr;
const scheduleDesc = scheduleInfo.description;
const schedule = createCronSchedule(scheduleExpr, scheduleDesc);
const assistantValue = typeof values.assistant === 'string' ? values.assistant : selectedAssistantId;
const resolvedExecutionMode: ExecutionMode = isTeamOwnedTask ? 'existing' : execution_mode;

let agent_config: ICreateCronJobParams['agent_config'] | ICronJobUpdateParams['metadata']['agent_config'];
if (canEditAgentConfig) {
  if (!assistantValue) {
    throw new Error(t('cron.page.form.assistantRequired'));
  }
  agent_config = resolveCronAgentConfig({
    agentValue: assistantValue,
    presetAssistants,
    selectedAionrsProvider: geminiCurrentModel
      ? {
          id: geminiCurrentModel.id as string | undefined,
          name: geminiCurrentModel.name,
        }
      : undefined,
    model_id,
    config_options,
    workspace,
    localeKey,
    getMode: resolveAutoApproveModeFromAgentMetadata,
    aionrsModelRequiredMessage: t('cron.page.form.aionrsModelRequired'),
  }).agent_config;
}

链路地图

读这条链时要分清五层：页面层负责入口与反馈；工具层构造 schedule、agent_config；hook 层负责读取和订阅事件；common adapter 把调用映射到 HTTP/WS；会话层在任务执行后展示结果或处理 AI 对话创建后的消息清理。

sequenceDiagram

  participant Page as ScheduledTasksPage

  participant Adapter as ipcBridge.cron
  participant Backend as /api/cron
  participant Detail as TaskDetailPage
  participant Chat as Conversation

  else 编辑

  end

  Hook-->>Page: 更新列表、状态、会话历史标记

  Detail->>Adapter: runNow.invoke({ job_id })
  Adapter->>Backend: POST /api/cron/jobs/:id/run
  Backend-->>Detail: conversation_id
  Detail->>Chat: navigate(/conversation/:id)

状态刷新与边界

列表刷新：useAllCronJobs 首次调用 listJobs，随后订阅 created、updated、removed 事件，本地替换或删除任务。
执行刷新：useCronJobsMap 看到 last_run_at_ms 变化后刷新会话历史、标记未读，并为 new-conversation 执行结果重命名会话。
run-now：TaskDetailPage 用 runningNowRef 防止双击重入，拿到 conversation_id 后轮询会话、补齐 cron_job_id，再导航到会话页。

边界路径：frequency === 'manual' 时 schedule expr 为空，列表和详情页把它当“只允许手动执行”的任务，不显示启停开关；旧任务缺失 schedule.tz 时，读取 hook 会调用 repairCronJobTimeZone 通过 updateJob 补 timezone；AI 对话更新任务的 E2E 明确断言 job id 保持不变，避免“删除再新建”破坏历史。

会话侧的 localCronCommands 不再解析创建命令；注释写明创建、更新、列表由注入 HTTP helper 处理。它只在 AionRS 消息完成后剥离 <think> 内容，必要时替换展示文本。

检索练习

暂停回忆：如果用户说“任务创建成功但列表没出现”，你先查哪两层？

参考答案：先查 CreateTaskDialog 是否真的走到 ipcBridge.cron.addJob.invoke 或 updateJob.invoke；再查 useAllCronJobs/useCronJobSubscription 是否收到 cron.job-created 或初始 listJobs 返回。
