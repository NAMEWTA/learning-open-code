2. 为什么还要轮询会话

后端返回 conversation_id 不等于 renderer cache 已经有完整会话。源码会在 15 秒内轮询 conversation.get，对 new-conversation 模式还会等待 workspace 出现。随后它可能重命名新执行会话，并补齐 cron_job_id / cronJobId，最后写入 SWR cache 再导航。

ipcBridge.cron.runNow 的 adapter 是 POST /api/cron/jobs/:id/run。但当前在 tests/e2e/specs/cron-crud.e2e.ts 和 tests/unit/cron/ 中没有找到 run-now 自动化覆盖；本课把 run-now 作为源码证明，不把它包装成测试已覆盖行为。

4. 检索练习

判断题：“run-now 成功返回 conversation_id 后，页面立刻无条件导航，不处理本地会话 cache。”对还是错？
