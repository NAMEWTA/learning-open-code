一眼图

手动创建
  -> ScheduledTasksPage：打开 CreateTaskDialog
  -> CreateTaskDialog：表单校验、schedule、agent_config、execution_mode
  -> ipcBridge.cron.addJob/updateJob：HTTP 写入 /api/cron/jobs
  -> cron.job-created/job-updated：hook 更新列表、状态、会话历史
  -> TaskDetailPage：读取 getJob、展示 history、runNow、编辑、删除
  -> runNow：POST /api/cron/jobs/:id/run，返回 conversation_id 后导航到会话
  -> Conversation：CronJobManager、ChatHistory、MessageList 显示任务状态或执行结果

AI 对话创建
  -> /scheduled 的 create via chat 跳 /guid 并预填 prompt
  -> AI 使用注入 HTTP helper 创建或更新任务
  -> localCronCommands 只剥离 think 标签，必要时替换展示文本
  -> E2E 轮询 bridge API，验证任务出现在 Scheduled Tasks 页面

入口与层级

层文件职责读码判断
路由/页面ScheduledTasksPage/index.tsx读取全部任务、打开创建弹窗、进入详情、启停、对话创建跳转。handleCreateViaChat 只跳 /guid；handleCreateManually 打开弹窗。
表单转换CreateTaskDialog.tsx把 name、prompt、frequency、assistant、model、workspace、execution mode 转成创建/更新负载。handleSubmit 是创建与编辑的唯一主入口。
工具函数cronUtils.ts、resolveCronAgentConfig.ts构造带 timezone 的 cron schedule，生成执行会话标题，解析 agent 配置。createCronSchedule 默认用当前 IANA timezone，失败回退 UTC。
hook 状态useCronJobs.ts提供单会话、全局列表、会话状态 map、关联会话列表和通用 CRUD action。三个事件订阅 created、updated、removed 是列表刷新的主线。
adapter/APIipcBridge.ts把 listJobs/addJob/updateJob/removeJob/runNow 映射到 Cron HTTP，并把 job 事件映射成 WS emitter。ipcBridge 名字保留旧形状，但 Cron 域主传输是 HTTP + WS。
详情触发TaskDetailPage.tsx读取单任务、启停、run-now、删除、编辑、展示关联会话。runningNowRef 是同步防重入边界。
会话侧localCronCommands.ts、useAionrsMessage.tsAI 对话创建/更新由注入 helper 完成；renderer 完成消息后剥离 think 标签并合并展示。不要把 localCronCommands 误读成任务创建实现。
测试证据cron-crud.e2e.ts、tests/unit/cron/*验证 AI 创建/修改/删除、页面可见、状态刷新、timezone 和状态标签。E2E 覆盖用户可见链路，unit 覆盖 hook 与工具函数边界。

adapter/API contract

export const cron = {
  listJobs: httpGet<ICronJob[], void>('/api/cron/jobs'),
  listJobsByConversation: httpGet<ICronJob[], { conversation_id: string }>(
    (p) => `/api/cron/jobs?conversation_id=${encodeURIComponent(p.conversation_id)}`
  ),
  getJob: httpGet<ICronJob | null, { job_id: string }>((p) => `/api/cron/jobs/${p.job_id}`),
  addJob: httpPost<ICronJob, ICreateCronJobParams>('/api/cron/jobs'),
  updateJob: httpPut<ICronJob, { job_id: string; updates: ICronJobUpdateParams }>(
    (p) => `/api/cron/jobs/${p.job_id}`,
    (p) => ({
      name: p.updates.name,
      description: p.updates.description,
      enabled: p.updates.enabled,
      schedule: p.updates.schedule,
      message: p.updates.target?.payload.text,
      execution_mode: p.updates.target?.execution_mode,
      agent_config: p.updates.metadata?.agent_config,
      conversation_title: p.updates.metadata?.conversation_title,
      max_retries: p.updates.state?.max_retries,
    })
  ),
  removeJob: httpDelete<void, { job_id: string }>((p) => `/api/cron/jobs/${p.job_id}`),
  runNow: httpPost<{ conversation_id: string }, { job_id: string }>((p) => `/api/cron/jobs/${p.job_id}/run`),

来源：ipcBridge.ts。这里是 renderer 与 backend 的 contract 边界。

主路径分解

阶段动作状态变化证据文件
列表加载useAllCronJobs 调 listJobs，再修复缺失 timezone。页面从 loading 到 jobs / empty。useCronJobs.ts、repairCronJobTimeZone.ts
手动创建CreateTaskDialog 校验表单，构造 schedule 和 agent_config，调用 addJob。后端创建后通过 cron.job-created 回推，列表追加任务。CreateTaskDialog.tsx、useCronJobs.ts
编辑任务同一个弹窗传入 editJob，构造 updates 调 updateJob。任务 id 不变，列表和详情替换为 updated job。CreateTaskDialog.tsx、cron-crud.e2e.ts
启停任务列表页 handleToggleEnabled 调 hook 的 pauseJob/resumeJob。enabled 变 false/true；状态标签变 paused/active。ScheduledTasksPage/index.tsx、CronStatusTag.dom.test.tsx
立即执行详情页 runNow 调用 adapter，等待返回 conversation_id。补齐执行会话信息，导航到 conversation 页面。TaskDetailPage.tsx；当前未找到自动化测试覆盖。
执行后刷新cron.job-updated 或 cron.job-executed 到达。会话历史刷新，新执行可标记 unread，详情历史重新拉取。useCronJobs.ts
AI 对话创建AI 使用注入 HTTP helper；完成消息由 processLocalCronResponse 清理展示。创建/更新结果通过 bridge API 和任务页验证，不靠本地解析文本。localCronCommands.ts、cron-crud.e2e.ts

异常与边界矩阵

边界触发点处理测试/证据
手动任务frequency === 'manual' 生成空 cron expr。列表和详情判断 !job.schedule.expr，不显示启停开关，只允许 run-now。ScheduledTasksPage/index.tsx、TaskDetailPage.tsx
run-now 双击用户快速点击立即执行。runningNowRef 在 React state 更新前同步阻断第二次调用。TaskDetailPage.tsx
缺失 timezone旧任务有 cron expr 但没有 schedule.tz。repairCronJobTimeZone 调 updateJob 补当前 timezone，失败则返回原 job。repairCronJobTimeZone.test.ts、useCronJobs.dom.test.ts
任务执行失败或 missedstate.last_status 为 error / missed。hook 与状态标签都把它视为 error；但 disabled 优先显示 paused。CronStatusTag.dom.test.tsx
AI 更新任务对话中要求修改已有任务。E2E 等待同一个 job id 的名称变化，断言不是新建任务。cron-crud.e2e.ts
删除任务详情页删除。任务不可见且 getJob 返回 null；原始会话仍可访问。cron-crud.e2e.ts
关联会话来源详情页历史需要展示任务相关会话。useCronJobConversations 直接调 backend 列表，不依赖可能陈旧的侧边栏 cache。useCronJobs.dom.test.ts
