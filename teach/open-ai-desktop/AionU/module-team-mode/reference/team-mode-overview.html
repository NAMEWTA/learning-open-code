1. 模块职责与位置

一句话职责：Team Mode 是 AionU renderer 中的多 Agent 团队工作台，它把 Team 记录、成员 tab、权限模式、Team run 状态、workspace 和现有 conversation runtime 组装成一个用户可操作的团队页。

职责面主文件说明
侧栏入口open-ai-desktop/AionU/packages/desktop/src/renderer/components/layout/Sider/TeamSiderSection.tsx展示 Team 列表、创建按钮、pin / rename / delete 菜单和待确认徽标。
路由入口open-ai-desktop/AionU/packages/desktop/src/renderer/pages/team/index.tsx从 /team/:id 读取 team id，并调用 ipcBridge.team.get 加载团队记录。
页面总装open-ai-desktop/AionU/packages/desktop/src/renderer/pages/team/TeamPage.tsx组装 ChatLayout、workspace、TeamTabs、TeamPermissionProvider 和每个 assistant slot。
会话复用open-ai-desktop/AionU/packages/desktop/src/renderer/pages/team/components/TeamChatView.tsx按 conversation type 复用 ACP / aionrs Chat，只注入 team send 与 team runtime。
Team 契约open-ai-desktop/AionU/packages/desktop/src/common/types/team/teamTypes.ts定义 TTeam、TeamAssistant、Team run ack / event / slot work 等共享类型。
字段映射open-ai-desktop/AionU/packages/desktop/src/common/adapter/teamMapper.ts把后端 lead、working、thinking、legacy agent 字段映射到前端 assistant-first 模型。
REST / WS 桥接open-ai-desktop/AionU/packages/desktop/src/common/adapter/ipcBridge.ts导出 team 域所有 HTTP provider 和 WebSocket emitter。

架构位置：Team Mode 位于 renderer-core 的页面层之内，向下依赖 common-adapter 的 ipcBridge.team，向旁边复用 conversation-runtime 的 Chat、SendBox、MessageList、模型选择和 workspace 壳层。它不直接实现 aioncore 的 Team 调度算法。

3. 接口清单

接口以 open-ai-desktop/AionU/packages/desktop/src/common/adapter/ipcBridge.ts 的 export const team 为准；类型主要来自 open-ai-desktop/AionU/packages/desktop/src/common/types/team/teamTypes.ts，创建和新增成员 payload 映射来自 open-ai-desktop/AionU/packages/desktop/src/common/adapter/teamMapper.ts。

3.3 关键类型

类型用途注意点
TTeamTeam 记录包含 assistants，并为旧调用保留 agents alias。
TeamAssistantslot 记录包含 slot_id、conversation_id、role、backend、assistant identity、pending confirmations。
TeamAssistantInput创建 Team / 添加成员输入后端从 assistant_id 派生 runtime backend；toBackendAssistant() 要求 assistant_id 必填。
TeammateStatusUI 状态点working、thinking、tool_use 都归一为 active。
ITeamRunAck发送后即时 ackuseTeamRunView 会把 ack 转成 run event，避免等待 WebSocket 才显示 busy。
ITeamRunEvent团队 run 快照包含 active child 数、pending wake 数、starting child 数和 slot work 列表。
ITeamSlotWorkslot 级工作状态决定某个成员的 sendbox 是否被 gate、stop 是否可触发。
ITeamChildTurnEvent成员 child turn 状态用于把 slot 的 active turn 加入或移出 childTurnsBySlot。

4. 关键依赖

依赖类型为什么关键
React Router页面入口/team/:id 是 Team 工作台的唯一路由入口。
SWR数据缓存Team list、Team detail、成员 conversation 都通过 SWR 缓存和 mutate 刷新。
ipcBridge.team业务桥所有 Team CRUD、session、run、消息、事件都从这里进入 REST / WS。
open-ai-desktop/AionU/packages/desktop/src/common/adapter/teamMapper.ts字段映射消除 backend wire contract 和 renderer 模型之间的字段差异。
Conversation runtimeUI 复用Team Mode 复用 ACP / aionrs Chat 和 SendBox，只注入 Team send 与 runtime gate。
Assistants catalog创建入口TeamCreateModal 从统一助手目录选择 leader，并解析默认 model。
Cron 模块清理依赖删除成员或 Team 前必须删除成员会话关联的 Cron job，避免悬挂任务。
Workspace 组件项目上下文Team workspace 可来自用户选择目录，也可回退到 leader conversation 的临时 workspace。
localStorage轻量持久化保存 Team section 展开状态、pin、active slot、成员排序、pending permission 计数快照。
Arco Design / i18nextUI 与文案modal、message、tooltip、表单、按钮、错误文案和多语言依赖它们。

5. 调用示例

示例 A：创建 Team

open-ai-desktop/AionU/packages/desktop/src/renderer/components/layout/Sider/TeamSiderSection.tsx 打开 open-ai-desktop/AionU/packages/desktop/src/renderer/pages/team/components/TeamCreateModal.tsx。modal 校验名称和 leader assistant，调用 resolveDefaultTeamAgentModel()，最后执行 ipcBridge.team.create.invoke({ name, workspace, assistants })。创建成功后侧栏 refreshTeams()，并导航到 /team/{id}。

示例 B：打开 Team 页面

open-ai-desktop/AionU/packages/desktop/src/renderer/components/layout/Router.tsx 命中 /team/:id 后进入 open-ai-desktop/AionU/packages/desktop/src/renderer/pages/team/index.tsx，用 ipcBridge.team.get.invoke({ id }) 读取 TTeam。open-ai-desktop/AionU/packages/desktop/src/renderer/pages/team/TeamPage.tsx 再用 TeamTabsProvider 和 TeamPermissionProvider 包住团队工作台。

示例 C：leader 发送消息

open-ai-desktop/AionU/packages/desktop/src/renderer/pages/team/components/TeamChatView.tsx 给 AcpChat / AionrsChat 注入 teamSendMessage。leader slot 调用 ipcBridge.team.sendMessage，收到 ITeamRunAck 后调用 useTeamRunView.applyAck()，随后继续用 WebSocket 事件校准。

示例 D：成员 stop

open-ai-desktop/AionU/packages/desktop/src/renderer/pages/team/components/teamSendRuntime.ts 先判断当前 slot_id 是否有 active turn、pending wake 或 starting child。若有，stop handler 调用 ipcBridge.team.pauseSlotWork；若 backend 报 “no active team run to pause”，则触发 getRunState reconcile。

用户在 leader tab 输入
  → TeamChatView.teamSendMessage
  → ipcBridge.team.sendMessage.invoke
  → team.sendMessage HTTP provider
  → ITeamRunAck 立即更新 useTeamRunView
  → team.run* / team.childTurn* WebSocket 事件持续校准 slot 状态
  → teamRuntime.runtimeGate 控制每个 slot 的 sendbox 是否可继续发送

7. 空白与风险

项目观察影响
