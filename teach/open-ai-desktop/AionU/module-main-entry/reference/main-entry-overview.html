1. 模块职责

职责落点说明
启动前置index.ts 顶层 import 与顶层逻辑Chromium 配置、Sentry、console、PATH 修复、单实例锁、Squirrel 安装器退出。
主生命周期编排handleAppReady()先短路处理版本输出，再初始化 main 基础设施、启动 backend、恢复缩放与窗口尺寸，并进入剩余模式分支。
运行形态分流isVersionMode / isResetPasswordMode / isWebUIMode同一入口支持桌面窗口、无头 WebUI、CLI 重置密码和版本输出。
退出收尾installQuitCleanup()统一处理托盘销毁、backend 停止、pet 窗口销毁和退出超时保护。
故障状态传播markBackendStartupFailed() + classifyBackendStartupFailure()把 backend 启动异常转成 renderer 可读取的结构化信息。

2. 架构位置

回看全局关系时，建议先打开 L0 项目总览课 或 L0 架构参考页。本页只聚焦 Electron main 入口，不重复解释 AionU 的整体模块地图。

上游输入

命令行参数、操作系统协议唤醒、Electron 生命周期事件、环境变量和上次保存的用户偏好。

本模块输出

BrowserWindow、WebHost、backend 端口、启动失败状态、托盘和退出清理回调。

不负责的内容

不直接承载 renderer 业务、不实现 backend 细节、不解释 deep link / tray / pet 的完整内部逻辑。

4. 内部分层结构

层次代表代码边界判断
启动前置层configureChromium、fix-path、单实例锁、Sentry这些逻辑必须在任何窗口和存储初始化之前发生。
编排层handleAppReady()只负责顺序和条件，不实现 backend / tray / WebHost 内部细节。
模式分支层版本、重置密码、WebUI、桌面模式按运行形态选择“是否创建窗口、是否保持无头存活、是否调用 CLI helper”。
生命周期回调层open-url、activate、window-all-closed、before-quit响应系统事件，但收尾工作继续委托给 helper。
辅助器层backendStartup.ts、quitCleanup.ts、backendStartupFailure.ts把复杂语义做成小而可测单元，减轻入口文件的认知负担。

5. 关键依赖

依赖类型为什么关键
electron运行时框架提供 app、BrowserWindow、ipcMain、powerMonitor 等主进程能力。
@aionui/web-host模块内核心协作者同时提供 BackendLifecycleManager 和 startWebHost()，把桌面 backend 与 WebUI 复用串起来。
ProcessConfig / initializeProcess()本地基础设施存储、语言、窗口尺寸、zoom factor 等都依赖它先完成初始化。
classifyBackendStartupFailure()错误分类器把 runtime、安装损坏、数据库损坏等失败整理成 renderer 可消费的结构。
fix-path / hydrateWindowsProcessPath()平台兼容保证 GUI 启动出的本地 CLI、nvm Node 和 PATH 行为一致。
restoreDesktopWebUIFromPreferences()体验增强桌面窗口创建后异步恢复 WebUI 偏好，说明 WebUI 不只是 CLI 入口，也是桌面能力延展。

6. 调用与阅读示例

推荐阅读顺序：

1. 先读 index.ts:619-624，确认为什么 --version 会在 initializeProcess() 之前直接退出。

2. 再读 index.ts:643-724，确认为什么 backend 启动一定发生在 initializeProcess() 之后。

3. 再读 index.ts:747-902，对照剩余三个运行分支，判断“哪个模式会创建窗口，哪个不会”。

4. 最后读 quitCleanup.ts:49-89，把 before-quit 事件和真正清理动作对上号。

app.whenReady()
  -> handleAppReady()
  -> if (--version) exit
  -> initializeProcess()
  -> startBackendOrExit()
  -> { resetpass | webui | desktop-window }
  -> installQuitCleanup() 托底退出
