工具调用确认全链路

15 分钟短课：学会从一条需要确认的 ACP / tool 消息，判断它经过哪些层、停在哪个安全边界、确认后如何继续传播。

入口：acp_permission / permission 主参考：tool-permission-flow-map 回链：L0 总览

1. 先定位入口

本切片从“会话里出现需要用户确认的消息”开始。ACP 平台在 useAcpMessage.ts 收到 acp_permission 后把消息合并进列表；aionrs 平台遇到 wire type acp_permission 但 payload 是 legacy confirmation 时，会改标成 permission。两者进入同一个 MessageList，但不是同一个 UI 组件。

先记住一个判断：acp_permission 是 ACP 专用确认卡；permission 是通用 / legacy 确认卡；acp_tool_call 只是工具执行状态和结果展示。

2. 五层调用时序

sequenceDiagram
  participant Stream as ACP / aionrs stream
  participant Recovery as pending recovery
  participant Merge as Messages hooks
  participant List as MessageList
  participant Card as Permission UI
  participant Adapter as common adapter
  participant Backend as backend
  Stream->>Merge: acp_permission / permission
  Recovery->>Merge: 缺失 pending confirmation
  Merge->>List: 合并并按 call_id 去重
  List->>Card: 分派到 MessageAcpPermission / MessagePermission
  Card->>Adapter: confirmMessage / confirmation.confirm
  Adapter->>Backend: POST /confirmations/:call_id/confirm
  Backend-->>Merge: confirmation.remove / tool update / turn.completed
  Merge-->>List: 移除卡或展示后续状态
ACP stream / recovered pending -> hooks.ts 合并消息：permission 按 call_id 去重 -> MessageList.tsx 分派：permission / acp_permission / acp_tool_call -> 用户在确认卡选择 option -> common adapter POST /api/conversations/:id/confirmations/:call_id/confirm -> backend 发 remove / tool update / turn.completed，UI 移除或展示后续状态

安全边界在 adapter / preload 一侧更清楚：preload/main.ts 只暴露受控 electronAPI.emit/on 和少量原生 helper；确认业务不是让 renderer 直接拿 Node 权限，而是通过 common adapter 的 conversation contract 发给 backend。

3. 不要把确认卡和工具结果混在一起

确认 UI

MessageAcpPermission 读取 tool_call 和 options，选择后调用 conversation.confirmMessage.invoke()。如果没有 tool_call，它直接不渲染。

MessagePermission 读取 IConfirmation，选择 proceed_always 时带上 always_allow。

结果 UI

MessageAcpToolCall 展示 pending / in progress、diff、文本或图片结果。它说明工具正在做什么或做完了什么，不负责批准。

MessageToolGroup 是普通工具组展示；它可内嵌 confirmationDetails，并提供 Cancel 等 outcome。

4. 边界练习

检索问题：如果页面刷新后权限卡消失，但 backend 仍有 pending confirmation，哪一层负责补回？它如何避免重复补卡？

参考答案：AcpChat 和 AionrsChat 挂载 usePendingConfirmationsRecovery()；这个 hook 调 conversation.confirmation.list，把缺失项构造成 permission 消息，并用 call_id 检查已有卡，收到 confirmation.remove 后再移除。
