1. 模块职责与位置

职责主文件说明
定时任务管理页packages/desktop/src/renderer/pages/cron/ScheduledTasksPage/index.tsx展示任务卡片、空态、keep-awake 开关、创建入口、启停操作。
任务详情页packages/desktop/src/renderer/pages/cron/ScheduledTasksPage/TaskDetailPage.tsx展示任务状态、执行说明、关联会话历史、立即执行、编辑、删除。
创建 / 编辑弹窗packages/desktop/src/renderer/pages/cron/ScheduledTasksPage/CreateTaskDialog.tsx把频率、助手、模型、workspace、执行模式转换成 ICreateCronJobParams 或 ICronJobUpdateParams。
会话页入口提示packages/desktop/src/renderer/pages/cron/components/CronJobManager.tsx在会话标题栏显示已有任务状态，并跳转到详情页；无任务时不显示空态图标。
侧边栏状态索引packages/desktop/src/renderer/pages/cron/useCronJobs.ts按 conversation_id 建立任务 map，处理未读执行、状态图标和会话历史刷新。
API 调用面packages/desktop/src/common/adapter/ipcBridge.ts暴露 ipcBridge.cron，把 renderer 调用映射到 /api/cron/* 与 WS 事件。

2.2 `pages/cron/index.ts` 导出

导出类型用途
CronJobIndicatorReact 组件会话历史中显示 active、paused、error、unread、unconfigured 等图标。
CronJobStatus类型'none' | 'active' | 'paused' | 'error' | 'unread' | 'unconfigured'。
CronJobManagerReact 组件会话标题栏的任务入口。
useCronJobsHook读取单个 conversation 关联的任务。
useCronJobsMapHook为会话历史建立 conversation_id → jobs 映射。

2.3 `ipcBridge.cron` API

接口传输后端路径 / 事件用途
listJobs.invoke()HTTP GET/api/cron/jobs读取全部任务。
listJobsByConversation.invoke({ conversation_id })HTTP GET/api/cron/jobs?conversation_id=...读取某个会话的关联任务。
getJob.invoke({ job_id })HTTP GET/api/cron/jobs/:job_id读取单任务详情。
addJob.invoke(params)HTTP POST/api/cron/jobs创建任务。
updateJob.invoke({ job_id, updates })HTTP PUT/api/cron/jobs/:job_id更新名称、启停、schedule、message、execution mode、agent_config、重试参数。
removeJob.invoke({ job_id })HTTP DELETE/api/cron/jobs/:job_id删除任务，但 E2E 证明原会话仍可访问。
runNow.invoke({ job_id })HTTP POST/api/cron/jobs/:job_id/run立即执行，返回 { conversation_id } 后由详情页导航。
saveSkill.invoke({ job_id, content })HTTP POST/api/cron/jobs/:job_id/skill保存由任务转成的 skill 内容。
hasSkill.invoke({ job_id })HTTP GET/api/cron/jobs/:job_id/skill返回布尔值，判断任务是否已有 skill。
deleteSkill.invoke({ job_id })HTTP DELETE/api/cron/jobs/:job_id/skill删除任务关联 skill。
onJobCreated.on(handler)WebSocketcron.job-created任务创建事件。
onJobUpdated.on(handler)WebSocketcron.job-updated任务更新或执行状态变化事件。
onJobRemoved.on(handler)WebSocketcron.job-removed任务删除事件。
onJobExecuted.on(handler)WebSocketcron.job-executed执行完成事件，状态为 ok、error、skipped 或 missed。

关联会话接口位于 ipcBridge.conversation.listByCronJob.invoke({ cron_job_id })，实际路径是 /api/cron/jobs/:cron_job_id/conversations。

3. 内部分层结构

packages/desktop/src/renderer/pages/cron/ ├── index.ts ← barrel 导出：组件与核心 hooks ├── useCronJobs.ts ← 单会话、全局列表、会话映射、任务会话历史 hooks ├── cronUtils.ts ← schedule 格式化、时区、标题、状态与 ID 归一 ├── repairCronJobTimeZone.ts ← 缺失 tz 的一次性修复器 ├── components/ │ ├── CronJobManager.tsx ← 会话头部任务入口 │ └── CronJobIndicator.tsx ← 会话历史图标 └── ScheduledTasksPage/ ├── index.tsx ← 全部任务列表页 ├── TaskDetailPage.tsx ← 任务详情、立即执行、历史会话 ├── CreateTaskDialog.tsx ← 创建 / 编辑弹窗 ├── CronStatusTag.tsx ← 列表与详情状态标签 ├── resolveCronAgentConfig.ts ← assistant → agent_config └── jobAgentMeta.ts ← 任务卡片上的 agent 名称与头像
层职责典型依赖
路由层把 /scheduled 与 /scheduled/:job_id 接到懒加载页面。React Router、module-renderer-core
页面层渲染列表、详情、弹窗和用户操作反馈。Arco Design、i18n、assistant/model hooks
状态层封装任务读取、事件订阅、未读标记、关联会话刷新。ipcBridge.cron、emitter、localStorage
工具层格式化 schedule、修复 timezone、归一 snake/camel ID、构造 agent_config。Intl API、assistant catalog、provider list
桥接层把 renderer 能力落到 HTTP 与 WebSocket。module-common-adapter、backend `/api/cron/*`

4. Hook 与工具函数

函数 / 组件返回或行为边界说明
useCronJobs(conversation_id?)jobs、loading、error、hasJobs、activeJobsCount、hasError、pauseJob、resumeJob、deleteJob、updateJob无 conversation_id 时返回空列表；依赖 WS 事件刷新。
useAllCronJobs()全部任务、加载态、activeCount、hasError、公共 actions列表页使用；actions 会同步本地状态。
useCronJobsMap()conversation_id → jobs map、状态查询、未读标记、refetch侧边栏 / 会话历史使用；新执行会触发 chat.history.refresh。
useCronJobConversations(job_id?)某任务关联的会话列表详情页历史栏使用；监听任务事件和 conversation.listChanged。
createCronSchedule(expr, description){ kind: 'cron', expr, tz, description }使用当前 IANA 时区；取不到时回退 UTC。
repairAllCronJobTimeZonesOnce()启动后修复旧任务缺失的 schedule.tz在 packages/desktop/src/renderer/main.tsx 认证 ready 后调用，并去重并发请求。
resolveCronJobId(extra)从 cron_job_id 或 cronJobId 读取任务 ID兼容旧字段和新字段。
resolveCronAgentConfig(input)构造写入后端的 agent_configaionrs 必须有 provider 与 model；普通 assistant 写入 model/config/workspace。
getJobAgentMeta(job, assistants, logos)任务卡片/详情页的 agent 名称、头像、emoji 或 fallback优先 assistant catalog，legacy job 回退到 agent_type。

5. 关键依赖

依赖用途风险点 / 读码提示
module-common-adapterCron HTTP/WS 调用面。不要被 ipcBridge 名字误导；Cron 域主要不是 Electron IPC。
module-renderer-core路由、Layout、侧边栏入口。/scheduled 与 /scheduled/:job_id 都在 renderer 路由树挂载。
module-conversation-runtime任务来源会话、执行后会话、消息内 artifact 跳转。删除任务不应删除原会话；E2E 明确验证此行为。
configService + systemSettings列表页 keep-awake 开关和系统设置。先更新本地 UI，失败后回滚本地配置。
assistant/model hooks创建弹窗中的助手、模型、workspace 配置。aionrs 模式必须从 provider list 选择可用模型。
Arco DesignForm、Select、Switch、Modal、Tag、Tooltip、Message。大量表单分支在 UI 层，业务负载最终集中到 submit。
emitter 与 localStorage会话历史刷新与未读执行状态。未读状态按 conversation_id 存储在 aionui_cron_unread。

6. 调用示例

示例 A：手动创建任务

const schedule = createCronSchedule('0 9 * * *', 'Daily at 09:00');
await ipcBridge.cron.addJob.invoke({
  name: 'Morning Greeting',
  schedule,
  prompt: 'Send a friendly greeting',
  conversation_id: '',
  created_by: 'user',
  execution_mode: 'new_conversation',
  agent_config,
});

agent_config 由同一表单流程中的 resolveCronAgentConfig() 生成。对应文件：packages/desktop/src/renderer/pages/cron/ScheduledTasksPage/CreateTaskDialog.tsx。

示例 B：立即执行并进入会话

const result = await ipcBridge.cron.runNow.invoke({ job_id: job.id });
if (result?.conversation_id) {
  navigate(`/conversation/${result.conversation_id}`);
}

对应文件：packages/desktop/src/renderer/pages/cron/ScheduledTasksPage/TaskDetailPage.tsx。

示例 C：列表页启停任务

if (job.enabled) {
  await pauseJob(job.id);
} else {
  await resumeJob(job.id);
}

pauseJob 与 resumeJob 内部都调用 ipcBridge.cron.updateJob.invoke。
