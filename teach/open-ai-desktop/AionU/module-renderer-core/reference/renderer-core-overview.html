1. 模块定位

Renderer 核心位于 packages/desktop/src/renderer/，职责不是承载单个业务功能，而是为所有页面提供统一启动序列、上下文状态、路由守卫和 UI shell。它在整体架构中的位置介于 preload/common adapter 之上、具体页面之下：向上接收桌面环境和 backend 的事件，向下把这些能力组织成 React 可消费的 provider 与导航框架。

L0 回链：如果要先确认整个仓库里 renderer 处在什么层级，先回看 00-overview 参考总览，再进入本页看局部骨架。

能力面主文件职责
根挂载main.tsx初始化 Sentry、i18n、样式、配置 bootstrap、provider 栈与根渲染。
路由分发components/layout/Router.tsx认证保护、懒加载页面、旧路由重定向、默认落点。
UI shellcomponents/layout/Layout.tsx标题栏、侧栏、移动端折叠、全局导航、系统事件接入。
启动准备services/bootstrapRenderer.ts等待 renderer 配置初始化，避免空配置进入业务页面。
共享上下文层hooks/context/*.tsx认证、主题、布局、导航历史、反馈、会话历史等全局共享状态。
预览上下文层pages/conversation/Preview/context/PreviewContext.tsxPreviewProvider 的真实定义位置，负责预览面板与标签页状态，不属于 hooks/context。

2. 对外接口与入口清单

入口 / 接口形式说明
createRoot(...).render(...)DOM 挂载点把 renderer 接到 `#root`，并根据 `window.__backendStartupFailure` 选择正常应用或失败对话框。
AppProvidersProvider 组合器按 `Auth → Theme → Preview → Feedback` 顺序包裹全局上下文。
ConfigUI 配置包装器为 Arco Design 注入主题色和当前语言 locale。
<Router layout={...} />路由入口组件从根入口把会话历史 provider 包起来的 `Layout` 交给路由层承载。
bootstrapRendererConfig()启动服务函数等待 `configService.initialize()`，出错只记录日志，不阻断启动流程。
useAuth()上下文接口给出 `ready`、`status`、登录/登出/刷新能力，驱动路由守卫。
LayoutContext上下文接口暴露 `isMobile`、`siderCollapsed` 与折叠控制，供 shell 内组件消费。
NavigationHistoryProvider上下文接口在应用内部维护 back/forward 栈，给标题栏等控件使用。

3. Provider 栈

顺序Provider核心状态 / 作用为什么放在这里
1AuthProvider认证状态、用户、登录登出、刷新路由守卫和后续启动闸门都依赖它。
2ThemeProvider主题模式、样式状态进入任意页面前先确定视觉主题。
3PreviewProvider预览上下文对会话与文档预览跨页共享。
4FeedbackProvider反馈相关 UI 与上下文包住运行时故障弹窗与后续页面。
5ConversationHistoryProvider会话历史共享状态只在主应用布局下需要，因此不放到最顶层。

6. Shell 级 hooks

Hook事件源对 shell 的影响
useDeepLink()ipcBridge.deepLink.received白名单允许的深链可直接跳到会话、团队或模型设置页。
useNotificationClick()ipcBridge.notification.clicked点击系统通知后导航回对应会话。
useBrowserNotification()会话响应流 + 浏览器 Notification API后台页面收到完成或确认事件时弹通知。
useConversationShortcuts()window.keydown桌面环境下支持 Ctrl+Tab 切会话、Ctrl/Cmd+T 新建会话。
useDirectorySelection()目录选择对话框上下文把目录选择器的宿主 context 挂到 shell 顶层。

7. 关键依赖

依赖用途在本模块的体现
React 19 / React DOM根渲染与 hooks 机制createRoot、provider、lazy/Suspense。
React Routerhash 路由与嵌套路由HashRouter、Routes、Navigate、Outlet。
Arco DesignUI 基础设施ConfigProvider、Modal、布局组件。
i18next多语言切换 Arco locale、生成错误文案与资源标签。
configService配置初始化驱动 renderer bootstrap 的准备阶段。
ipcBridge桌面事件桥托盘、日志流、深链、通知点击、任务停止等能力接入 shell。

8. 测试与阅读示例

E2E：导航主链

tests/e2e/specs/navigation.e2e.ts 验证 guid 页、设置页与 URL 级跨页切换，说明路由和 shell 是可观察的产品主路径。

Unit：品牌返回逻辑

tests/unit/renderer/LayoutSiderBrandHome.dom.test.tsx 验证设置页点击品牌字样会回到最近的非设置路径，直接对应 Layout.tsx 的 shell 行为。

Unit：浏览器通知

tests/unit/renderer/useBrowserNotification.dom.test.tsx 证明 shell hooks 不只是监听系统事件，还会在点击通知时把用户带回会话。

推荐阅读顺序：main.tsx 看启动闸门，Router.tsx 看入口分发，Layout.tsx 看共享交互；之后再按 hook 名称回跳到 hooks/system、hooks/context，以及 pages/conversation/Preview/context/PreviewContext.tsx。
