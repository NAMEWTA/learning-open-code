Mobile Companion：手机只是控制面，不是第二个桌面

目标：15 分钟内能说清移动端入口、host 首页和 RPC client 的分工。

先看三层

Orca Mobile 是桌面 runtime 的 companion。它不在手机上跑 Git、PTY 或 agent，而是把 paired desktop 当作 host， 通过 WebSocket RPC 查看 worktree、打开 session、发送 terminal input、读文件和驱动任务。读源码时先分三层： 根布局负责路由和连接池，首页负责 host/worktree 聚合，RPC client 负责加密传输和重连。

mobile/ ├── app/ │ ├── _layout.tsx ← Expo Router 根布局、deep link、通知跳转、RpcClientProvider │ └── index.tsx ← host 列表、resume card、账号用量、任务入口 └── src/ └── transport/ └── rpc-client.ts ← E2EE WebSocket RPC、请求/订阅、重连和 stream replay

入口先装连接池

_layout.tsx 把整个 Stack 包在 RpcClientProvider 里，所以所有页面都共享同一套 per-host 连接池。它还处理 orca://pair deep link：冷启动时读 initial URL，热启动时监听 URL event， 解析到 pairing code 后跳到 confirm screen。通知点击也在这里转成 host/worktree 路径。

这个设计让 pairing、首页、host detail 和 session screen 不需要各自开 WebSocket。页面只通过 hook acquire host client， provider 负责复用、重连和前后台恢复。

首页是 host 状态聚合器

index.tsx 从本地存储读 paired hosts，用 useAllHostClients() 获取每个 host 的共享 client。 连接后它请求 stats.summary、worktree.ps、accounts.list， 同时读取 settings、preflight 和 Linear 状态来决定 Tasks 入口。首页还会把 worktree 和账号快照缓存下来， 让冷启动先显示上一次数据，再用桌面新响应覆盖。 

const allClients = useAllHostClients(hostIds)
const lastActive = pickResumeWorktree(worktrees)

Resume card 的选择也不是随便拿第一个 worktree：先看首页外层记录的手机本地 last visited；没有命中时再交给 pickResumeWorktree()，由它优先选桌面 active worktree，最后按最近 terminal output 兜底。

RPC client 是真实边界

rpc-client.ts 连接桌面 endpoint 后先发 E2EE hello，收到 ready 后用 device token 认证。 sendRequest() 会等待 connected，再发带 id、method、params 的加密 RPC；subscribe() 保存 streaming listener，断线重连后会重放还没取消的订阅。它还处理 terminal/browser 二进制帧、连接超时、activity probe、 授权重试和有限 backoff。

如果你在 session 页看到终端输入没反应，第一步不是看按钮样式，而是判断：host 是否 connected、订阅是否 replay、 terminal.send 是否被加密发出、桌面 runtime 是否返回错误。

检索练习：手机扫码后提示连接超时，你会先查哪里？A. 首页 host card；B. pairing confirm；C. RPC client； D. desktop runtime server。

参考路径：先看 pairing confirm 是否正确解析 offer 并调用 connect()，再看 RPC client 的 WebSocket open、 E2EE handshake 和 pairing overall timeout。确认手机端 endpoint/token/public key 都正确后，再进入 desktop runtime server。 更完整的 pairing、host storage、session route 和测试佐证见 模块总览参考。
