移动端 Companion 模块总览参考

定位：L1 模块速查。配合 导览短课 使用。

职责边界

层 核心文件 职责 边界不变量
根布局 mobile/app/_layout.tsx 注册 Expo Router Stack，安装 RpcClientProvider，处理 pairing deep link、通知跳转和 splash。 所有路由共享同一个 per-host client provider。
Pairing 路由 mobile/app/pair.tsx、mobile/app/pair-scan.tsx、mobile/app/pair-confirm.tsx 接受 QR/deep link/paste code，连接桌面 runtime，验证 status.get，保存 host。 扫码 UI 是入口之一；L1 只定位边界，完整 QR 扫码链路留到 L2 mobile pairing flow。
Pairing payload mobile/src/transport/pairing.ts 解析 orca://pair、query/hash code 和裸 base64 desktop QR payload。 pairing 阶段有独立整体超时；连接成功但保存失败要显示本地保存错误。
Host 存储 mobile/src/transport/host-store.ts 保存 host metadata、读取 host 列表、重命名/移除 host、更新 lastConnected。 native device token 进 SecureStore；AsyncStorage 只存 metadata。
连接池 mobile/src/transport/client-context.tsx 每 host 一个共享 RpcClient，提供 acquire/release、forceReconnect、closeHost 和状态订阅。 页面切换不应重复打开 N 个 WebSocket；foreground/network 恢复会通知 live clients。
RPC 传输 mobile/src/transport/rpc-client.ts E2EE handshake、加密请求、stream subscribe、terminal/browser binary frame、重连和 stream replay。 请求必须等 connected；订阅断线后 replay；授权失败先有限重试，再进入 auth-failed。
首页 mobile/app/index.tsx 聚合 hosts、worktrees、stats、account usage、task providers、resume card 和 host 操作。 缓存只用于冷启动 seed；connected 后以桌面 RPC 响应覆盖。
Resume fallback mobile/src/worktree/resume-worktree.ts 在首页 last visited 未命中时，从 worktree 列表中选择桌面 active、最近输出或第一项。 last visited 是 mobile/app/index.tsx 的外层策略；helper 只负责 fallback 选择。
Host detail mobile/app/h/[hostId]/index.tsx 展示 worktree list，读写桌面 UI 偏好，执行 worktree activate/sleep/remove。 view settings 与桌面 shared UI store 同步，不只存在手机本地。
Session route mobile/app/h/[hostId]/session/[worktreeId].tsx 承载 terminal、session tabs、browser、file、markdown、source-control 等移动端工作区面板。 真实状态来自 runtime subscriptions；手机端只是渲染和发送操作。

关键流程

流程 入口 主调用 读法
配对 orca://pair、QR scan、paste code connect()、status.get、saveHost() 先验证 URL/code 解析，再看连接日志和 host-store 保存边界。
首页刷新 mobile/app/index.tsx stats.summary、worktree.ps、accounts.list 首页是多 host 聚合层；不要把它误读成 session 业务实现。
Worktree 列表 mobile/app/h/[hostId]/index.tsx repo.list、worktree.ps、ui.get、ui.set 列表和过滤排序偏好要与桌面端共享 UI 状态对齐。
打开会话 mobile/app/h/[hostId]/session/[worktreeId].tsx terminal.subscribe、session.tabs.subscribe、terminal.send 先看 transport 是否 connected，再看 stream 是否收到 ready/snapshot。
前后台恢复 client-context.tsx、rpc-client.ts notifyForeground()、activity probe、reconnect backoff 半开连接靠 foreground probe 触发恢复，超过重试上限后需要用户或 foreground nudge。
协议兼容与调试 mobile/README.md 记录本地开发、mock server、实机调试、protocol compatibility 和 validation 命令。 协议兼容由 status.get 与 protocol-version 常量兜底；细节归 shared/L2 继续拆。

核心调用示例

response = await client.sendRequest('status.get')
await saveHost({
  id: hostId,
  endpoint: offer.endpoint,
  deviceToken: offer.deviceToken,
  publicKeyB64: offer.publicKeyB64
})

pairing confirm 的最小成功条件是：offer 可解析，WebSocket/E2EE/device token 可通过 status.get， 然后 host metadata 和 token 能保存。后续首页、host detail、session screen 都只是在这个 host profile 上 acquire 共享 client。

测试佐证

行为 测试文件 不变量
pairing URL/code 解析 mobile/src/transport/pairing.test.ts 支持 query/hash/bare code，拒绝相似但错误的 route。
pair confirm 状态 mobile/src/transport/pair-confirm-state.test.ts 缺 code、非法 payload 和合法 offer 都进入明确状态。
pairing 连接尝试 mobile/src/transport/pairing-connection-attempt.test.ts 初次 pairing 有整体超时和 dispose，避免无限 Connecting。
RPC client mobile/src/transport/rpc-client.test.ts 连接超时会关闭 socket；stale socket 事件不污染新连接；stream dispose 会发 unsubscribe。
实网恢复 harness mobile/src/transport/rpc-client-live-recovery.test.ts foreground probe 能回收半开连接，parked retry loop 能被 foreground nudge 唤醒。
Resume card mobile/src/worktree/resume-worktree.test.ts 测试覆盖 fallback helper：优先桌面 active worktree，再 fallback 到最近输出，最后取第一项；手机 last visited 是首页外层策略。
