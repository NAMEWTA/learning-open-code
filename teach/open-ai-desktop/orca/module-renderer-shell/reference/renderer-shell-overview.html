Renderer Shell 模块总览参考

定位：L1 模块速查。配合 导览短课 使用。

职责边界

层 核心文件 职责 边界不变量
启动入口 src/renderer/src/main.tsx 导入全局 CSS，安装 crash diagnostics，应用主题，挂载 i18n、错误边界和 App。 没有 #root 直接失败；根级 recoverable error 不应让整个 Electron 窗口静默空白。
应用 shell src/renderer/src/App.tsx 总装配页面、modal、window controls、启动水合、session writer、beforeunload flush 和 runtime graph sync。 水合成功前不打开可写 session 门；失败时保留磁盘上的旧 session。
状态总线 src/renderer/src/store/index.ts 用 Zustand 组合 repo、worktree、terminal、tabs、settings、browser、editor、SSH、runtime 等 slice。 store 是 renderer 的内存事实来源；跨进程能力仍只能经 preload 的 window.api。
统一 tab 模型 src/renderer/src/store/slices/tabs.ts 维护 unifiedTabsByWorktree、group、active group 和 split layout，修正 legacy terminal/browser/editor 状态。 tab、group、layout 必须原子更新，否则 split pane、active tab 和持久化 session 会分裂。
runtime graph 发布 src/renderer/src/runtime/sync-runtime-graph.ts 收集 mounted terminal panes、后台 automation PTY 和 mobile session tab snapshots，发给 main runtime。 不直接 import store；通过 getter 注入避免 store 构造循环。
IPC 事件消费 src/renderer/src/hooks/useIpcEvents.ts 把 main 进程事件映射为 store 更新、toast、导航或 UI 状态变化。 renderer 响应事件，但不越过 preload 直接访问 main 能力。

启动链路

阶段 位置 行为 读源码时的判断点
React bootstrap main.tsx 记录 bootstrap breadcrumb，安装 diagnostics，应用系统主题，挂载根组件。 根错误边界包住 App，不是包住单个页面。
基础数据读取 App.tsx 先串行读取 settings；提前发起 UI、keybindings、onboarding；repos 完成后让 project/folder scope 与 worktrees、lineage 扫描部分重叠。 并行化只发生在依赖满足后的子链路，避免远端 host 阻塞整个启动链。
session 水合 App.tsx 用 repos 推导 runtime hosts，读取各 host session partition，水合 workspace、tabs、editor、browser。 水合失败不写默认空 session；只让 UI 进入降级可见状态。
终端恢复 App.tsx 先恢复 SSH connection，再等待 first-window services，最后 reconnect persisted terminals。 终端恢复依赖 main runtime/daemon 服务，不是纯 renderer 内存操作。
持久化解锁 App.tsx 成功完成依赖步骤后设置 hydration succeeded，并通过 session write subscriber 写回 patch。 writer 与 JSX 解耦，避免 tab/file/browser 高频变动触发 App re-render。

关键调用点

调用点 依赖 说明
setRuntimeGraphStoreStateGetter(useAppStore.getState) App.tsx、sync-runtime-graph.ts App 把 store getter 注入 runtime graph helper，避免 helper 反向 import store。
setRuntimeGraphSyncEnabled(workspaceSessionReady) App.tsx、workspace session 状态 只有 session ready 后才允许发布窗口 graph，防止 runtime 看到半水合布局。
createSessionWriteSubscriber() App.tsx、workspace session helpers 订阅 store patch 并写到本地/远端 workspace session partition。
createTabsSlice() store/index.ts、tabs.ts 为 shell 提供创建、激活、关闭、split、reconcile 和 hydrate tabs 的 action。 
const graph: RuntimeSyncWindowGraph = {
  tabs: [],
  leaves: [],
  mobileSessionTabs: buildMobileSessionTabSnapshots(state, systemPrefersDark)
}

const result = await window.api.runtime.syncWindowGraph(graph)

runtime graph 同步不是把整个 Zustand store 发给 main，而是构造一个窗口运行时投影：terminal tab、pane leaf、 active leaf、layout 和移动端可消费的 session tab snapshots。main runtime 再用返回值更新 renderer 侧的 orchestration 状态。
