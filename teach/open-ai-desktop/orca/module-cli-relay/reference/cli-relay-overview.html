CLI 与 Relay 模块参考

职责定位

src/cli 是 Orca 的本地命令行入口，负责把用户输入规范化为 runtime RPC 或本地启动行为；src/relay 是远端 SSH 主机上的轻量 daemon，负责把 PTY、文件、Git、preflight、agent hook 和 workspace session 能力通过 framed JSON-RPC 暴露给桌面端。

在整体架构中的位置

orca CLI
├── parseArgs / validateCommandAndFlags
├── RuntimeClient
└── desktop runtime 或 headless runtime

desktop main / SSH
└── deploy relay.js on remote host
    ├── protocol frame
    ├── RelayDispatcher
    └── Pty/Fs/Git/Preflight/AgentExec handlers

回到 L0：CLI 与 relay 位于 项目总览图 的脚本化和远端执行边界。

源码索引

文件 职责 阅读提示
src/cli/index.ts CLI 入口、特殊 shim、help、校验、RuntimeClient 创建。 先看 main() 的控制流。
src/cli/dispatch.ts 汇总 handler group 并按 commandPath 分发。 查一个命令落地位置时从这里反查。
src/cli/specs/index.ts 聚合 browser、project、file、computer、orchestration 等命令规格。 判断命令和 flag 是否合法。
src/cli/runtime-client.ts 兼容 barrel，导出拆分后的 runtime client。 实际实现继续进入 src/cli/runtime/。
src/relay/relay.ts relay 进程入口、参数、connect mode、daemon mode、socket 和 handler 装配。 后续 L4 会专门讲 reconnect。
src/relay/protocol.ts frame header、handshake、keepalive、JSON-RPC message 和 payload 上限。 这是 relay 与桌面端互通的最低层契约。
src/relay/dispatcher.ts 多客户端 dispatcher、请求/通知、ack、超时和 stale client 管理。 理解断线和重连行为必须读这里。

对外接口清单

接口族 入口 说明
CLI 命令 COMMAND_SPECS + dispatch() 规格定义合法命令，dispatch 查找对应 handler。
Runtime RPC RuntimeClient CLI 调用桌面/headless runtime 的主要通道。
Relay frame encodeFrame()、FrameDecoder 独立于 Electron 的远端传输格式。
Relay JSON-RPC RelayDispatcher.onRequest()、onNotification() 远端 handler 注册点，服务 PTY、FS、Git 等能力。
Grace reconnect --connect、Unix socket、handshake 桌面端重连到仍持有 PTY 的 relay 进程。

关键依赖

方向 依赖对象 说明
CLI 依赖 args、specs、runtime、handlers 参数解析与命令规格先保证输入合法，再由 RuntimeClient 或 handler 执行。
CLI 被谁调用 用户、agent、脚本、特殊 shim orca 是自动化入口，agent-teams-tmux 和 claude-teams 是兼容桥。
Relay 依赖 protocol、dispatcher、PTY/FS/Git/preflight handlers 协议层解帧，dispatcher 路由，handler 执行远端能力。
Relay 被谁调用 main SSH 服务、远端 orca --connect 桥、relay socket client 桌面端部署 relay，重连时通过 socket 接回仍持有 PTY 的进程。

简单调用示例

orca browser click --selector "#run"
├── src/cli/index.ts       ← parseArgs + validateCommandAndFlags
├── RuntimeClient          ← 连接 desktop/headless runtime
├── src/cli/dispatch.ts    ← commandPath: browser click
└── browser handler        ← 组装 runtime 请求并输出结果
