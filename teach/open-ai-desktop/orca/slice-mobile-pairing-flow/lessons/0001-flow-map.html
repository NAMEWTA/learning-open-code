Mobile Pairing Flow：手机如何拿到桌面 runtime 会话

目标：15 分钟内把二维码、E2EE 握手、host 保存和 Resume 串成一条链路。

先抓一条主路径

Orca mobile pairing 从桌面 Settings 开始。renderer 调 mobile:getPairingQR， main 进程选一个可达地址，再让 runtime 生成 mobile scope 的 pairing offer。二维码和复制码都是同一个 orca://pair?code=... URL；桌面/shared payload 包含 endpoint、deviceToken、publicKeyB64 和 optional scope。

desktop Settings -> src/main/ipc/mobile.ts mobile:getPairingQR(scope: "mobile") -> src/main/runtime/runtime-rpc.ts createPairingOffer() -> src/main/runtime/device-registry.ts pending device token -> src/shared/pairing.ts orca://pair?code=...

手机端：三种入口收敛到同一个 offer

手机端入口有 QR scan、deep link 和 paste。mobile/app/pair-scan.tsx 直接从相机或输入框解析；mobile/app/_layout.tsx 和 mobile/app/pair.tsx 把 orca://pair deep link 重定向到 /pair-confirm。 这些路径最终都调用 mobile/src/transport/pairing.ts，得到 mobile 本地 PairingOffer。 注意这个 mobile schema 当前不暴露 scope，授权仍由 token registry 和 status.get 的 deviceScope 证明。

解析成功还不是配对成功。确认页和扫码页都会创建临时 RpcClient， 用 offer 里的 public key 先走 E2EE 握手，再用 deviceToken 认证，最后请求 status.get。 只有这个请求成功，手机才认为桌面 runtime 可达且 token 有效。

保存和恢复是第二条链

status.get 成功后，手机创建 host id/name，并调用 saveHost()。 mobile/src/transport/host-store.ts 把 endpoint、publicKeyB64、lastConnected 作为 metadata 写入 AsyncStorage；deviceToken 在 native 上写入 Expo SecureStore。保存失败会显示 Pairing succeeded but couldn't save the host， 因为这已经不是网络或认证错误。 

PairingOffer
  -> connect(endpoint, deviceToken, publicKeyB64)
  -> E2EE hello/ready/auth/authenticated
  -> status.get
  -> saveHost(metadata in AsyncStorage, token in SecureStore)
  -> router.replace("/h/:hostId")

后续恢复不会重新扫码。App 根部的 RpcClientProvider 为每个 host 共享一个 client； 首页 mobile/app/index.tsx 调 loadHosts() 后 prime hosts， 再用保存的 token 和 public key 重新连接。foreground 或网络恢复时，client 会发 status.get probe 或重置 reconnect backoff。

runtime 边界：scope 和 allowlist

桌面 runtime 的 WebSocket 先由 E2EEChannel 校验 token，认证通过后才把明文 RPC 交给 dispatcher。 token 对应的 device scope 决定授权：mobile scope 只能调用 MOBILE_RPC_METHOD_ALLOWLIST 里的方法； runtime scope 用于 web/desktop remote clients。status.get 返回时会被注入 deviceScope， 让旧保存的 pairing 也能知道自己属于哪类客户端。

检索练习

手机扫码后提示 “Pairing succeeded but couldn't save the host”，你应该先查网络/E2EE，还是查本地持久化？为什么？

先查本地持久化。这个文案在 mobile/app/pair-confirm.tsx 和 mobile/app/pair-scan.tsx 的 status.get 成功之后才出现； 失败点是 saveHost()，也就是 AsyncStorage metadata 或 SecureStore token 写入。 网络、token、E2EE 已经通过了临时 client 的验证。完整来源见 移动端配对与会话恢复参考。

主要来源：src/main/ipc/mobile.ts、 src/main/runtime/runtime-rpc.ts、 mobile/app/pair-confirm.tsx、 mobile/src/transport/rpc-client.ts。
