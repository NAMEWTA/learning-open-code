# 使命：Orca 项目总览

## 为什么
学习者希望把 Orca 当作一个真实的大型 Electron + React + CLI/Relay 系统来拆解，而不是只停留在 README 功能列表。掌握总览后，后续阅读任一模块时都能先判断它属于桌面主进程、渲染层、远端 relay、CLI、移动端还是共享协议层。

## 成功的样子
- 能用一张架构图说明 Orca 如何把桌面窗口、CLI、远端 relay、移动端和 agent worktree 串在一起。
- 能看到一个源码路径时快速判断它的运行环境、主要依赖和学习顺序。
- 能从总览进入后续 L1/L2 主题，而不在 6000 多个源码文件中迷路。

## 约束条件
- 本主题是批量生成模式下的 L0 锚点，不等待用户访谈。
- 单节课必须在 15 分钟内完成，细节清单分流到参考文档。
- 以当前子模块 commit `61bd98db6faacb8baffa0de369b187c0e40d662a` 为准。

## 不在范围内
- 不逐文件讲解 `src/main`、`src/renderer` 或移动端实现。
- 不覆盖所有 CLI 命令、IPC handler、React 组件和测试用例。
- 不评价产品路线或发布策略，只讲当前源码结构。
