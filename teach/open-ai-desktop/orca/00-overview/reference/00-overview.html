Orca 项目总览参考

项目定位

Orca 是用于并行 agent 开发的 Electron 桌面应用和配套运行时。README 将它描述为可把 Codex、Claude Code、OpenCode、Pi 等 CLI agent 放在并列 worktree 中运行、比较和合并结果的 AI orchestrator。源码上，它由桌面主进程、React 渲染层、安全 preload 桥、命令行入口、远端 relay、移动端 companion 和共享协议层组成。

核心设计决策

决策 含义 阅读提示
多进程边界清晰 UI、主进程系统能力、preload 契约、CLI 和 relay 各自承担不同信任边界。 先确认代码运行在哪个进程，再追依赖。
Worktree 是核心抽象 并行 agent 工作以隔离 worktree 为单位组织，Git、terminal、review 和任务状态围绕它展开。 后续切片应优先覆盖创建、切换、清理和 source-control。
远端 SSH 是一等场景 AGENTS.md 明确要求考虑 SSH；relay 保持 PTY、文件、Git 和 hook 能力。 不要把本地路径、shell、平台按 macOS 写死。
Agent provider 可扩展 Codex、Claude、OpenCode、Pi、Gemini、Copilot 等以 provider/hook/runtime 服务接入。 阅读 provider 时同时看 shared 类型和 hook service。
测试覆盖跨平台风险 大量单测和 E2E 保护 terminal、SSH、Git、browser、renderer 恢复和性能路径。 学习行为预期时优先找同名 `.test.ts` 或 `tests/e2e`。

源码豁免清单

范围 类型 豁免理由
node_modules/、构建缓存、输出目录 第三方依赖 / 构建产物 不属于仓库手写源码，后续覆盖率扫描应排除。
resources/**/*.png、*.jpg、*.gif、*.webp、*.woff2 二进制资源 作为视觉或字体资产引用即可，不按源码逐行讲解。
docs/assets/、previews/*.html 文档资产 / 手工预览 在 L0 或相关 UI 主题中说明用途，通常不进入函数级覆盖。
mobile/assets/、原生构建中间产物 移动端资源 / 平台输出 只需在 mobile 模块总览中标注资产用途。
*.lock、发行配方、纯配置文件 依赖锁定 / 发布配置 配置主题中概述职责即可，不逐条讲依赖解析。

层级 候选主题 建议原因
