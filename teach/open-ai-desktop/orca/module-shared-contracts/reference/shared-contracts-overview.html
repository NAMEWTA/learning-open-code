L1 参考文档 · shared 契约层

共享协议与类型模块参考

职责定位

src/shared 维护跨 renderer、main、preload、CLI、relay、mobile 的共享契约：静态类型、运行时 schema、协议版本、capability、路径/id 解析和纯函数。它降低各端字段漂移的风险，是阅读垂直切片时的共同地基。

在整体架构中的位置

renderer UI       → imports shared UI/runtime/worktree types
main process      → imports shared IPC/runtime/SSH/session contracts
CLI               → directly reuses shared runtime envelope/status contracts
mobile            → reuses many shared domain types and mirrors transport compatibility locally
relay             → mirrors selected protocol pieces when it must stay Electron-free

回到 L0：shared 位于 项目总览图 的所有进程之间，负责让数据形态先稳定下来。

核心类型簇

类型簇 代表文件 覆盖内容
Runtime RPC src/shared/runtime-rpc-envelope.ts、src/shared/runtime-types.ts 响应 envelope、keepalive、runtime status、terminal/browser/mobile session graph。
SSH 与 Relay src/shared/ssh-types.ts SSH target、连接状态、relay grace period、远端 PTY lease、端口转发。
Workspace / Worktree src/shared/worktree-id.ts、src/shared/workspace-session-schema.ts worktree id 拆分、folder workspace suffix、session JSON 恢复校验。
协议兼容 src/shared/protocol-version.ts runtime protocol version、客户端/服务端最小兼容版本、capability 清单。
业务聚合 src/shared/types.ts Repo、Worktree、Tab、Terminal、Git、Browser、Notification 等共享业务模型。

内部结构与依赖

分层 代表文件 依赖 / 下游
协议版本层 src/shared/protocol-version.ts 被 desktop、CLI、mobile/headless 兼容检查读取；mobile 侧有本地镜像版本文件。
运行时 schema 层 src/shared/runtime-rpc-envelope.ts 依赖 zod；被 CLI/runtime client 用来识别成功、失败和 keepalive frame。
领域类型层 src/shared/runtime-types.ts、src/shared/types.ts 被 renderer store、main IPC、preload API、CLI 和移动端页面共同引用。
持久化 schema 层 src/shared/workspace-session-schema.ts 依赖 zod 和 sibling 类型；保护磁盘 session 恢复不会污染 React 状态。
ID / 路径工具层 src/shared/worktree-id.ts 被 main、renderer 和 source-control/worktree 逻辑用于 repo/worktree 映射。
SSH relay 契约层 src/shared/ssh-types.ts 被 main SSH 服务、settings UI、relay 配置请求和端口转发状态共同使用。

关键接口速查

接口/函数 签名或形态 用途
RuntimeRpcEnvelopeSchema Zod union：success / failure / keepalive 运行时校验 runtime RPC frame。
isKeepaliveFrame(frame) unknown → RuntimeRpcKeepaliveFrame 识别 runtime keepalive，避免把心跳当业务响应。
getRepoIdFromWorktreeId() string → string 从 worktree id 中提取 repo id。
splitWorktreeIdForFilesystem() string → ParsedWorktreeId | null 去掉 folder workspace UUID suffix，得到真实文件系统路径。
RUNTIME_CAPABILITIES 字符串 capability 清单 声明 runtime 能力，供客户端判断兼容特性。
SSH_RELAY_CONFIGURE_GRACE_TIME_METHOD 'relay.configureGraceTime' relay grace 配置方法名，连接 main 与 relay 的协议约定。

阅读规则

  • 跨端字段先找 shared，再找调用端；不要只在 renderer 或 main 内搜索。
  • 涉及磁盘恢复的数据优先看 Zod schema 和容错策略。
  • 涉及 CLI/mobile/headless 的变更优先看 src/shared/protocol-version.ts 的兼容规则。
  • relay 需要 Electron-free 时可能镜像协议定义，阅读时要检查注释中的同步关系。
