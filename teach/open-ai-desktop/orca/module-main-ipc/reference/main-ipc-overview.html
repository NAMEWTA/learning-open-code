主进程 IPC 边界模块总览参考

定位：L1 模块速查。配合 导览短课 使用。

职责边界

层 核心文件 职责 边界不变量
preload 桥 src/preload/index.ts 把 renderer 可用能力组织成 api，用 contextBridge.exposeInMainWorld() 暴露。 renderer 只看到封装后的函数，不直接拿到 Node/Electron main 能力。
总注册 src/main/ipc/register-core-handlers.ts 一次性注册 app、filesystem、runtime、settings、browser、GitHub/GitLab、Linear/Jira 等核心 domain handler。 module-level registered 防止重复注册；trusted webContents ID 每次仍会更新。repo/worktree handlers 由窗口装配层注册。
终端边界 src/main/ipc/pty.ts 处理 PTY spawn/write/resize/kill、renderer delivery、provider 路由、pane serializer 和 mobile driver 防线。 本地 spawn 等 daemon startup barrier；SSH/headless 路径不等桌面 daemon。
文件与 Git src/main/ipc/filesystem.ts 注册文件读写、搜索和 git:* channel；watcher 由 src/main/ipc/filesystem-watcher.ts 单独注册。 connectionId 存在时走 SSH provider，否则走本地 Git；commit message 在 IPC 边界先校验。
移动端配对 src/main/ipc/mobile.ts 生成 QR/runtime pairing URL、列设备、列 runtime grant、撤销设备和报告 WebSocket ready。 移动端 QR 用 scope: 'mobile'；web/runtime link 用 scope: 'runtime'。

接口清单

接口族 preload API main channel 说明
PTY api.pty.spawn/write/resize/kill pty:spawn、pty:write、pty:resize、pty:kill spawn/kill 用 invoke；write/resize 用 send/on，降低高频空回复。
PTY 事件 api.pty.onData/onExit/onReplay pty:data、pty:exit、pty:replay main 将 provider 数据批量转发给当前 trusted renderer。
Git api.git.status/commit/stage/discard git:status、git:commit、git:stage、git:discard 当前源码中由 src/main/ipc/filesystem.ts 注册，按 connectionId 分流本地或 SSH provider。
Mobile renderer 通过 mobile UI 调 main channel mobile:getPairingQR、mobile:getRuntimePairingUrl、mobile:revokeDevice handler 依赖 OrcaRuntimeRpcServer，因为设备 registry 和 pairing offer 归它所有。
核心注册 无直接 renderer API registerCoreHandlers() 由窗口装配调用，确保 domain handlers 与 trusted window 状态就绪。

关键依赖与调用示例

调用链 关键依赖 读法
renderer → preload → PTY ipcRenderer、ipcMain、IPtyProvider、OrcaRuntimeService 先看 preload 参数形状，再看 src/main/ipc/pty.ts 中 provider 选择和 runtime driver 防线。
renderer → preload → Git Store、Git runner、SSH filesystem/git provider 不要找旧清单里的 git.ts；直接看 src/main/ipc/filesystem.ts 的 git:* handlers。
renderer → mobile IPC → runtime RPC OrcaRuntimeRpcServer、device registry、QRCode、networkInterfaces mobile handler 只是 desktop UI 的桥，真正 token 和 E2EE 物料在 runtime RPC server。 
const offer = rpcServer.createPairingOffer({
  address: ip,
  rotate: args?.rotate,
  name: `Mobile ${new Date().toLocaleDateString()}`,
  scope: 'mobile'
})

这是移动端 QR 的最小调用示例：IPC handler 负责选择可达地址和生成 QR，runtime RPC server 负责生成带 scope 的 pairing offer。

测试佐证

行为 测试文件 不变量
总注册幂等 src/main/ipc/register-core-handlers.test.ts 再次调用会更新 trusted IDs，但不会重复注册 IPC handlers。
PTY 启动屏障 src/main/ipc/pty-startup-barrier-ordering.test.ts 本地 spawn 在解析 provider 前等待 startup barrier，防止抢在 daemon provider 替换前运行。
daemon 路径 parity src/main/ipc/pty.test.ts daemon-backed PTY 也注入 OpenCode、Codex、proxy、agent hook 等环境，与本地 provider 行为对齐。
Git 本地/SSH 分流 src/main/ipc/filesystem.test.ts connectionId 存在时走 SSH provider；空 commit message 在进入 provider 前就被拒绝。
移动端 pairing src/main/ipc/mobile.test.ts 网络地址会优先 tailnet；mobile 和 runtime scope 分开；pending runtime grant 也可撤销。
