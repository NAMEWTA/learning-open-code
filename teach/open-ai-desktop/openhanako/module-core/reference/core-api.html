1. 引擎核心 HanaEngine

1.3 init() 生命周期详解

执行顺序：
0. 数据迁移（config scope → preferences、provider → added-models.yaml、model overrides）
0d. 统一版本号驱动迁移 (migrations.js)
1. Pi SDK + ModelRegistry 初始化 + refreshAvailable()
2. AgentManager.initAllAgents() -- 扫描所有 agent 目录，初始化焦点 agent
2b. repairChannelCursorProjection() -- 补齐频道游标
3. ResourceLoader + SkillManager 初始化
4. ChannelManager + BridgeSessionManager 初始化
5. SessionCoordinator + VisionBridge + 插件系统初始化
异常：
- initAllAgents 中焦点 agent 初始化失败不阻塞启动（catch 后继续），用户可进应用修配置
- 非焦点 agent 初始化失败仅 warning 日志，不影响启动
- Pi SDK 初始化失败会抛出，阻止启动

1.4 dispose() 生命周期详解

清理顺序：
1. StudioCronService.stop()
2. SkillManager.unwatch()
3. ResourceWatchRegistry.dispose()
4. 所有 session 的 discardSessionRuntime()
5. AgentManager.disposeAll()
6. 通知服务、语音识别服务停止
7. Win32LegacySandbox 清理
8. SessionManifestStore 关闭、事件总线移除

1.5 边界条件与约束

null/empty 处理：
- getAgent(null) 返回 null
- switchAgent("") 抛出 "agentId is required"
- prompt("") 空文本不发送请求，直接返回
- abort() 在无活动 stream 时静默忽略
- sessionPath 为 null/undefined 时，多数方法回退到 this.currentSessionPath

并发控制：
- Agent 切换通过 _switchQueue (Promise chain) 串行化
- Session 创建/切换通过 SessionCoordinator 内部队列保证原子性

超时：
- LLM 调用通过 AbortSignal 控制超时（callText 的 timeoutMs 参数）
- 默认无超时限制，由调用方注入 AbortController.signal
