1. 认证路由

1.1 OAuth 认证 — server/routes/auth.ts

源码: server/routes/auth.ts (338 行) | 工厂函数: createAuthRoute(engine)

支持两种 OAuth 流程：授权码流程（用户粘贴授权码，如 Anthropic）和设备码流程（服务端轮询，如 MiniMax）。内部维护 pendingFlows Map 追踪进行中的流程，10 分钟未完成自动清理。

路由总表

方法路径鉴权说明
POST /api/auth/oauth/start OAuth bootstrap 启动 OAuth 登录流程
POST /api/auth/oauth/callback OAuth bootstrap 提交授权码（授权码流程）
GET /api/auth/oauth/poll/:sessionId OAuth bootstrap 轮询登录状态（设备码流程）
GET /api/auth/oauth/status chat 查询所有 OAuth provider 登录状态
POST /api/auth/oauth/logout chat 登出指定 provider
GET /api/auth/oauth/:provider/custom-models chat 获取 OAuth provider 自定义模型列表
POST /api/auth/oauth/:provider/custom-models chat 添加自定义模型
DELETE /api/auth/oauth/:provider/custom-models/:modelId chat 删除自定义模型

POST /auth/oauth/start

请求体
{ provider: string }
成功响应 (200)
{
  sessionId: string,     // UUID v4
  url: string,           // OAuth 授权页面 URL
  instructions?: string, // 设备码流程时为 user_code
  polling?: boolean      // true = 设备码流程
}

边界条件

  • provider 缺失返回 400 {"error":"provider is required"}
  • 同一 authKey 已有进行中的 flow 时复用，返回已有 URL（防止重复创建）
  • 已完成的 flow（有 result）被自动清理后重新创建
  • 5 分钟超时：urlPromise 未 resolve 时拒绝并清理
  • 支持 usesCallbackServer 模式（如 OpenAI Codex）：本地启动回调服务器，不走设备码 polling

POST /auth/oauth/callback

请求体
{ sessionId: string, code: string }
成功响应 (200)
{ ok: true }

错误

  • 400 — {"error": "No pending login flow"} (sessionId 无效)
  • 500 — {"error": "..."} (loginPromise 失败)

GET /auth/oauth/poll/:sessionId

响应 (200)
{ status: "pending" }
// 或
{ status: "done" }
// 或
{ status: "error", error: string }

错误

  • 400 — {"status":"error","error":"No pending login flow"}

GET /auth/oauth/status

响应 (200)
{
  [providerId: string]: {
    name: string,
    loggedIn: boolean,
    modelCount: number
  }
}

POST /auth/oauth/logout

请求体
{ provider: string }
响应
{ ok: true }

OAuth 自定义模型子路由

方法路径请求/响应
GET /auth/oauth/:provider/custom-models 响应: { models: string[] }
POST /auth/oauth/:provider/custom-models 请求: { modelId: string }；已存在则幂等返回
DELETE /auth/oauth/:provider/custom-models/:modelId 响应: { ok: true, models: string[] }

自定义模型变更后自动调用 engine.refreshModels() 刷新模型列表。

1.2 Web 会话认证 — server/routes/web-auth.ts

源码: server/routes/web-auth.ts (192 行) | 工厂函数: createWebAuthRoute({ hanakoHome, authService, ... })

Web 客户端（浏览器）的会话管理。支持 token 凭证登录和本地账号密码登录两种方式，通过 HttpOnly Cookie 维持会话。

路由总表

方法路径鉴权说明
POST /api/web-auth/login public Web 登录
GET /api/web-auth/session public 检查当前会话状态
POST /api/web-auth/logout public 登出

POST /web-auth/login

请求体
{
  credential?: string,  // token 凭证（与 username/password 二选一）
  username?: string,    // 本地账号用户名
  password?: string,    // 本地账号密码
  clientKind?: string   // 客户端类型（影响 scope profile），默认 "mobile"
}
成功响应 (200)
{
  ok: true,
  expiresAt: string,        // ISO 8601
  principal: {
    kind, credentialKind, connectionKind, trustState,
    serverId, serverNodeId, userId, studioId, deviceId,
    credentialId, platformAccountId, officialServiceKind,
    authMethod, scopes: string[]
  }
}
Set-Cookie
hana_web_session={secret}; Path=/; HttpOnly; SameSite=Strict; Max-Age=1209600

边界条件

  • 同时提供 credential 和 username/password：优先使用 credential (token)
  • 密码登录要求 connectionKind === "local" 或 HTTPS 安全上下文
  • 密码验证失败返回 403；token 验证失败返回 400/403（取决于 authService）
  • Session TTL 默认 14 天 (14 * 24 * 60 * 60 * 1000)

GET /web-auth/session

响应 (200)
{ authenticated: true, principal: {...} }
// 或
{ authenticated: false, principal: null }

从 Cookie header 中提取 hana_web_session 并验证。

POST /web-auth/logout

响应
{ ok: true }

撤销服务端 session + 清除客户端 Cookie（Max-Age=0）。

1.3 WebSocket 票据认证 — server/routes/ws-auth.ts

源码: server/routes/ws-auth.ts (32 行) | 工厂函数: createWebSocketAuthRoute({ ticketService })

POST /api/ws-ticket

鉴权
需要有效的 HTTP 请求 principal（通过 bearer token 或 web session）
scope: chat (POST)
响应 (200)
{
  ticket: string,      // 一次性 WS 连接票据
  expiresAt: string    // ISO 8601 过期时间
}

错误

  • 403 — {"error": "missing_principal"}

使用流程

// 1. HTTP 请求获取 ticket
const { ticket } = await fetch("/api/ws-ticket", {
  method: "POST",
  headers: { Authorization: "Bearer <token>" }
}).then(r => r.json());

// 2. 用 ticket 连接 WebSocket
const ws = new WebSocket(`ws://localhost:${port}/ws?wsTicket=${ticket}`);

Config 写入特殊处理

Secret 字段掩码

  • config 中的 api_key 等敏感字段使用掩码值 (sk-****...****abc)
  • 写入时掩码值不覆盖实际密钥，只更新非掩码字段
  • 通过 resolveSecretPatch / maskObjectSecrets 处理

Inline Provider Credential

  • config 中可内联 provider 凭证 (api_key, base_url)
  • 读取时自动清零 (clearInlineProviderCredentialFields)
  • 写入时提取到 providers 独立存储

调用示例

// 列出所有 agent
const agents = await fetch("/api/agents").then(r => r.json());

  method: "PUT",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    model: { id: "claude-sonnet-4-20250514", provider: "anthropic" },
    systemPrompt: "You are a helpful assistant."
  })
});

// 转发前剥离安全凭证：
// - query: pluginIframeTicket, pluginSurfaceSession
// - header: X-Hana-Plugin-Surface-Session
// 注入：

插件 surface session 认证

  • 插件 route 鉴权 policy 类型为 plugin_route：需要 owner 或匹配的 plugin surface principal

13.3 会话流状态存储 — server/session-stream-store.ts

源码: server/session-stream-store.ts (254 行)

维护单个 session 的流式事件 ring buffer。用于断线重连时 replay 已发送的事件。

核心函数

函数签名说明
createSessionStreamState (opts?: { maxEvents?, maxBytes?, maxEventBytes? }) => StreamState 创建初始流状态（默认 maxEvents=5000, maxBytes=8MB, maxEventBytes=256KB）
beginSessionStream (state, streamId?) => string 开始新一轮流式回复，返回 streamId
appendSessionStreamEvent (state, event) => { streamId, seq, event, ts, byteLength } 写入一条流式事件
finishSessionStream (state) => void 结束当前流，清空 events
resumeSessionStream (state, opts?: { streamId?, sinceSeq? }) => ResumeResult 按 seq 恢复事件列表

StreamState 结构

内部状态字段
{
  streamId: string|null,     // 当前活跃 stream ID
  nextSeq: number,           // 下一条事件的序号（从 1 开始）
  isStreaming: boolean,      // 是否正在流式输出
  startedAt: number,         // 开始时间戳
  endedAt: number,           // 结束时间戳
  events: Entry[],           // 事件 ring buffer
  maxEvents: number,         // 最大事件数 (5000)
  maxBytes: number,          // 最大字节数 (8MB)
  maxEventBytes: number,     // 单事件最大字节 (256KB)
  totalEventBytes: number,   // 当前总字节
  droppedEvents: number,     // 被裁剪的事件数
  droppedBytes: number,      // 被裁剪的字节数
  compactedEvents: number    // 被压缩的事件数
}

Resume 逻辑 (resumeSessionStream)

边界情况

  • 无活跃流：返回空 events，streamId: null
  • streamId 不匹配：客户端请求的是旧流，返回 reset: true + 当前流的全部 events
  • sinceSeq 小于最早可用的 seq：标记 truncated: true，从 earliest 开始返回
  • 正常续传：返回 sinceSeq 之后的所有 events

事件裁剪与压缩

  • Ring buffer 裁剪 (trimEvents)：超出 maxEvents(5000) 或 maxBytes(8MB) 时从头部丢弃
  • 单事件压缩 (compactValue)：
    • 字符串超过 8192 字符截断为 [omitted N chars]
    • 数组超过 64 项截断
    • 对象超过 64 key 截断并标记 _omittedKeys
    • 大字段（base64, content, data, details, snapshot, text, thumbnail）超过 256 字符截断
    • 循环引用检测（[circular]）
  • 完全省略：压缩后仍超 maxEventBytes，返回 { type, compacted: true, omitted: true, originalByteLength }
