第 4 课 · 收尾与持久化：Theme + Workspace + Tutorial → setupComplete 完成收尾

🎯 本节目标：学完你能讲清引导的最后三个阶段——Theme 的前端 localStorage 持久化、Workspace 的路径选择与 config 写入、Tutorial 的教学卡片展示，以及最关键的一步：setupComplete 标记如何通过 IPC → Server API → preferences.json 的完整链路，使得后续启动直接跳过引导。

前三课覆盖了从架构播种到 Provider 配置的流程。本课聚焦引导的后半段——步骤 4 到 6 以及最终的 setupComplete 持久化。这些步骤看似简单，但 setupComplete 的写入链路跨越了渲染进程、主进程和服务端三个层级。

一、Step 4：主题选择（ThemeStep）— 纯前端持久化

ThemeStep 是 7 个步骤中最简单的一个——纯前端交互，不涉及任何 API 调用：

// desktop/src/react/onboarding/steps/ThemeStep.tsx
export function ThemeStep({ goToStep }) {
  const [activeTheme, setActiveTheme] = useState(() =>
    registry.migrateSavedTheme(localStorage.getItem(registry.STORAGE_KEY))
  );

  return (
    <StepContainer>
      {OB_THEMES.map(theme => (
        <button
          className={activeTheme === theme ? 'active' : ''}
          onClick={() => { setActiveTheme(theme); setTheme(theme); }}
        >
          {t(`settings.appearance.${themeKey(theme)}`)}
        </button>
      ))}
    </StepContainer>
  );
}

10 个可选主题：warm-paper、coral、midnight、auto、high-contrast、grass-aroma、contemplation、absolutely、delve、deep-think。主题通过 theme-registry 模块和 localStorage 持久化，CSS 变量实时切换。

二、Step 5：工作区设置（WorkspaceStep）— 双路径选择

WorkspaceStep 允许用户选择 Agent 可访问的默认文件夹，提供"使用默认路径"和"自定义路径"两个选项：

// WorkspaceStep.tsx 核心逻辑
export function WorkspaceStep({ hanaFetch, goToStep }) {
  const [defaultPath, setDefaultPath] = useState('');   // 系统推荐
  const [selectedPath, setSelectedPath] = useState(''); // 用户自定义

  useEffect(() => {
    loadDefaultWorkspace(hanaFetch)      // GET /api/config/default-workspace
      .then(path => setDefaultPath(path));
  }, []);

  const onBrowse = useCallback(async () => {
    const folder = await window.platform?.selectFolder?.();
    if (folder) setSelectedPath(folder);
  }, []);

  const onNext = useCallback(async () => {
    await saveWorkspace({
      hanaFetch,
      workspacePath: selectedPath || defaultPath,  // 自定义优先
      defaultPath,
    });
    goToStep(6);
  }, [selectedPath, defaultPath]);
}

saveWorkspace 的 API 实现：

// onboarding-actions.ts
export async function saveWorkspace({ hanaFetch, workspacePath, defaultPath }) {
  const selected = workspacePath.trim();
  if (!selected) throw new Error('workspacePath is required');

  if (selected === defaultPath) {
    await ensureDefaultWorkspace(hanaFetch);   // 确保默认目录存在
  }

  await hanaFetch(`/api/agents/${AGENT_ID}/config`, {
    method: 'PUT',
    body: JSON.stringify({
      desk: {
        home_folder: selected,
        heartbeat_enabled: false,
        heartbeat_interval: DEFAULT_HEARTBEAT_INTERVAL_MINUTES,
      },
    }),
  });
}
💡 workspacePath 为空时使用 defaultPath——这意味着用户可以不用浏览文件夹，直接点击"下一步"使用系统推荐的默认路径（通常是 ~/Desktop/HanaAgent）。

三、Step 6：教学概览（TutorialStep）— 最后一步

TutorialStep 展示 5 张功能卡片（记忆、技能、工作区、文件、多 Agent），用户点击"开始使用"后触发引导完成流程：

// TutorialStep.tsx 核心逻辑
export function TutorialStep({ preview, showError }) {
  const [finishing, setFinishing] = useState(false);

  const onFinish = useCallback(async () => {
    if (preview) { window.close(); return; }
    setFinishing(true);
    try {
      await window.hana.onboardingComplete?.();  // IPC 调用主进程
    } catch (err) {
      showError(t('onboarding.error'));
      setFinishing(false);
    }
  }, [preview]);

  return (
    <StepContainer>
      {/* 5 张教学卡片 */}
      <TutorialCard icon={<MemoryIcon />}   title="记忆" ... />
      <TutorialCard icon={<SkillsIcon />}   title="技能" ... />
      <TutorialCard icon={<WorkspaceIcon />} title="工作区" ... />
      <TutorialCard icon={<JianIcon />}      title="文件" ... />
      <TutorialCard icon={<AgentsIcon />}    title="多 Agent" ... />
      <button disabled={finishing} onClick={onFinish}>
        开始使用
      </button>
    </StepContainer>
  );
}

四、setupComplete 持久化：跨三层进程的写入链

这是整个引导流程的最终一步，跨越三个进程层级：

1. 渲染进程 → 主进程（IPC） TutorialStep.onFinish() → window.hana.onboardingComplete() → IPC: "onboarding-complete" → 主进程 wrapIpcHandler("onboarding-complete") 2. 主进程 → Server（HTTP） completeOnboardingAndOpenMain({ serverPort, serverToken, createMainWindow }) → POST /api/preferences/setup-complete → Server: markSetupComplete() → 写入 ~/.hanako/user/preferences.json { primaryAgent: "hanako", setupComplete: true } 3. 主进程：窗口切换 Server 返回 { ok: true, setupComplete: true } → createMainWindow() // 创建主窗口 → onboardingWindow.close() // 关闭引导窗口

onboarding-completion.cjs 的核心实现：

// desktop/src/shared/onboarding-completion.cjs
async function completeOnboardingAndOpenMain({
  serverPort, serverToken, createMainWindow, fetchImpl,
}) {
  // 步骤 1：向 Server 提交 setupComplete 意图
  await submitOnboardingCompleteIntent({ serverPort, serverToken, fetchImpl });

  // 步骤 2：创建主窗口
  if (typeof createMainWindow !== "function") {
    throw new Error("createMainWindow is not available");
  }
  createMainWindow();
}

完成后，~/.hanako/user/preferences.json 的内容变为：

{
  "primaryAgent": "hanako",
  "setupComplete": true
}

下次启动时，isSetupComplete() 返回 true，主进程直接走 createMainWindow() 分支，跳过整个 Onboarding。

五、检验一下

Q1. ThemeStep 使用什么机制持久化主题选择？

A. 通过 Server API 写入 config.yaml B. 通过 localStorage 和 theme-registry 模块 C. 通过 Electron 的 app.setUserDefault() D. 写入 preferences.json 文件
看答案B。ThemeStep 是纯前端步骤，不涉及任何 API 调用。主题通过 localStorage 存储，theme-registry 模块管理迁移和 CSS 变量切换。

Q2. setupComplete 标记的写入链路经过哪三层？

A. 网络层、数据库层、缓存层 B. 渲染进程(IPC) → 主进程(HTTP) → Server(文件写入) C. 前端 localStorage → IndexedDB → 文件系统 D. Electron → Chromium → Node.js
看答案B。TutorialStep 通过 IPC 调用主进程，主进程通过 HTTP 调用 Server 的 /api/preferences/setup-complete，Server 写入 preferences.json 文件。

Q3. WorkspaceStep 中如果用户不浏览文件夹直接点击下一步，使用什么路径？

A. 抛出错误，要求用户必须选择 B. 使用系统推荐的 defaultPath（通常 ~/Desktop/HanaAgent） C. 使用应用安装目录 D. 使用上一次的路径
看答案B。saveWorkspace 中使用 selectedPath || defaultPath，如果用户未选择自定义路径，自动使用系统推荐的默认路径。

Q4. 引导完成后，下次启动如何跳过 Onboarding？

A. 检查 localStorage 中的 onboardingCompleted 标志 B. isSetupComplete() 读取 preferences.json 的 setupComplete 字段，为 true 则直接 createMainWindow() C. 检查 config.yaml 中的 setupComplete 字段 D. 检查环境变量 SKIP_ONBOARDING
看答案B。这正是三步检测阶梯的第一优先级。setupComplete=true 写入 preferences.json 后，后续每次启动都在 app.whenReady() 的第一分支直接创建主窗口。
