🎯 本节目标：学完你能讲清 Splash 窗口和 Onboarding 窗口的创建时机、生命周期管理、以及 OnboardingApp 如何通过 React 受控组件模式编排 7 个步骤的切换与数据传递。

上一课我们了解了引导系统的架构和播种机制。本课聚焦窗口层的实现——Splash 窗口如何作为加载状态的视觉反馈、Onboarding 窗口如何创建并与 Splash 无缝切换、以及 OnboardingApp 如何作为"中央总线"编排 7 个步骤。

一、Splash 窗口：第一屏视觉反馈

Splash 窗口是用户看到的第一屏——在 Server 启动和数据播种完成之前显示，起到"应用正在加载"的视觉反馈作用。它也是一个计时器——确保最少显示 3 秒，避免"闪一下就消失"的体验：

// desktop/main.cjs:1438-1481
function createSplashWindow() {
  splashWindow = new BrowserWindow({
    width: 380, height: 280,
    resizable: false,
    frame: false,
    transparent: true,             // 透明背景，由 CSS 绘制圆角
    show: false,                   // HTML/CSS 渲染完成后才 show
    webPreferences: {
      preload: path.join(__dirname, "preload.bundle.cjs"),
      contextIsolation: true,
      nodeIntegration: false,
    },
  });
  loadWindowURL(splashWindow, "splash");
  splashWindow.once("ready-to-show", () => splashWindow.show());
}
💡 Splash 最少 3 秒的设计意图：Server 启动+数据播种+config 解析可能非常快（<500ms），如果不加最小显示时间，用户会看到 Splash 一闪而过，产生"应用有问题"的错觉。3 秒是一个心理阈值，让用户感知到"加载-完成"的完整过程。

二、Onboarding 窗口：无缝切换

Onboarding 窗口的创建时机由三步检测阶梯决定。它的 HTML 入口是 onboarding.html，加载三个全局脚本后才挂载 React：

// desktop/main.cjs:3315-3348
function createOnboardingWindow(query = {}) {
  onboardingWindow = new BrowserWindow({
    width: 560, height: 780,
    resizable: false,
    frame: false,
    show: false,
    webPreferences: {
      preload: path.join(__dirname, "preload.bundle.cjs"),
      contextIsolation: true,
      nodeIntegration: false,
    },
  });
  loadWindowURL(onboardingWindow, "onboarding", { query });

  onboardingWindow.once("ready-to-show", () => {
    if (splashWindow && !splashWindow.isDestroyed()) {
      splashWindow.close();        // ← Splash 到 Onboarding 的无缝切换
    }
    onboardingWindow.show();
  });
}

三、OnboardingApp：中央总线编排

OnboardingApp（desktop/src/react/onboarding/OnboardingApp.tsx）是顶层编排组件。它维护 7 个步骤的切换逻辑和全局状态，是典型的 React 受控组件模式：

export function OnboardingApp({ preview, skipToTutorial }) {
  const [serverConnection, setServerConnection] = useState(null);
  const [step, setStep] = useState(skipToTutorial ? 6 : 0);
  const [stepKey, setStepKey] = useState(0);

  // 全局状态：语言、Agent 名称
  const [locale, setLocale] = useState('zh-CN');
  const [agentName, setAgentName] = useState('Hanako');

  // Provider 跨步骤传递：Step 2 → Step 3
  const [providerName, setProviderName] = useState('');
  const [providerUrl, setProviderUrl] = useState('');
  const [providerApi, setProviderApi] = useState('openai-completions');
  const [apiKey, setApiKey] = useState('');

  // 步骤切换（通过 key 变化强制重新挂载）
  const goToStep = useCallback((index) => {
    setStepKey(k => k + 1);
    setStep(index);
  }, []);

  // hanaFetch：所有步骤共享的 API 调用闭包
  const hanaFetch = useCallback((path, opts = {}) => {
    const headers = appendConnectionAuth(serverConnection, opts.headers);
    return fetch(buildConnectionUrl(serverConnection, path), { ...opts, headers });
  }, [serverConnection]);

  // Provider 回调：Step 2 完成后传给 Step 3
  const onProviderReady = useCallback((name, url, api, key) => {
    setProviderName(name); setProviderUrl(url);
    setProviderApi(api); setApiKey(key);
  }, []);

  // 渲染当前步骤
  return (
    <div className="onboarding">
      <div className="onboarding-progress">{/* 7 个圆点指示器 */}</div>
      {step === 0 && <LocaleStep ... />}
      {step === 1 && <NameStep ... />}
      {step === 2 && <ProviderStep ... onProviderReady={...} />}
      {step === 3 && <ModelStep providerName={providerName} ... />}
      {step === 4 && <ThemeStep ... />}
      {step === 5 && <WorkspaceStep ... />}
      {step === 6 && <TutorialStep ... />}
    </div>
  );
}
💡 设计观察：OnboardingApp 的状态管理是典型的 React 受控组件模式。7 个步骤组件之间不直接通信，而是通过 OnboardingApp 的 useState 作为"中央总线"传递数据。ProviderStep 和 ModelStep 的通信通过 onProviderReady 回调完成——Step 2 存入 provider 信息，Step 3 通过 props 读取。

四、Step 0：语言选择（LocaleStep）

LocaleStep（desktop/src/react/onboarding/steps/LocaleStep.tsx）是引导的第一步。支持 5 种语言（zh-CN/zh-TW/ja/ko/en），并提供隐藏的"连接到局域网服务器"面板：

const onNext = useCallback(async () => {
  if (!preview) {
    await saveLocale(hanaFetch, locale);
    // → PUT /api/agents/hanako/config { locale }
  }
  goToStep(1);
}, []);

五、Step 1：身份设定（NameStep）

NameStep（desktop/src/react/onboarding/steps/NameStep.tsx）收集三个关键输入：用户名（必填）、Agent 名称（可选）、记忆功能开关：

const onNext = useCallback(async () => {
  if (preview) { goToStep(2); return; }
  const trimmed = userName.trim();
  if (!trimmed) return;                  // 用户名为空阻塞

  await saveOnboardingIdentity({
    hanaFetch, userName: trimmed, agentName, memoryEnabled,
  });
  // → PUT /api/agents/hanako/config
  //   { user: { name }, agent: { name }, memory: { enabled } }
  goToStep(2);
}, []);

六、检验一下

Q1. Splash 窗口最少显示多长时间？

A. 1 秒 B. 2 秒 C. 3 秒 D. 5 秒
看答案C。Splash 确保最少显示 3 秒，避免因 Server 启动太快导致"闪一下就消失"的体验。3 秒是用户感知"加载-完成"过程的心理阈值。

Q2. OnboardingApp 的步骤切换使用什么机制强制重新挂载组件？

A. 使用 React Router 的路由切换 B. 通过 stepKey 递增变化，React 检测 key 变化后卸载旧组件并挂载新组件 C. 使用 Redux 的全局状态管理 D. 直接操作 DOM 替换元素
看答案B。goToStep() 中 setStepKey(k => k + 1) 改变组件的 key prop，React 识别到 key 变化后会卸载旧组件并挂载新组件，确保步骤组件每次都是全新状态。

Q3. ProviderStep 和 ModelStep 之间如何传递 Provider 信息？

A. 通过 Redux Store 共享 B. 通过 OnboardingApp 的 useState + onProviderReady 回调 C. 通过 localStorage 持久化后读取 D. 两个步骤直接通过 props 互相引用
看答案B。ProviderStep 完成后调用 onProviderReady(name, url, api, key)，OnboardingApp 存入 useState，ModelStep 通过 props 接收 providerName 等值。

Q4. NameStep 中用户名为空时如何处理？

A. 弹出警告对话框 B. 直接 return，不做任何提交，按钮通过 disabled={!userName.trim()} 保持不可用 C. 自动填入默认名称 D. 允许跳过，之后可补充
看答案B。NameStep 通过两重保护确保用户名非空：onNext 中的 if (!trimmed) return 作为运行时保护，"下一步"按钮的 disabled={!userName.trim()} 作为 UI 层保护。
