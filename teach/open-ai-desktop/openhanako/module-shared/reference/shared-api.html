1.2 config-scope.ts

函数 splitByScope(partial)

export function splitByScope(partial: Record<string, unknown>): { global: Array<{ key: string; value: unknown; setter: string }>; agent: Record<string, unknown> }
参数partial -- 前端发来的 config patch 对象
返回值{ global, agent } -- global 为数组（每项含 key/value/setter），agent 为剔除 global 字段后的剩余对象
说明根据 CONFIG_SCHEMA 将 partial config 拆分为 global 和 agent 两部分。对含有嵌套 global 字段的 parent 对象做浅拷贝保护，避免修改原对象。最多支持 2 级嵌套路径。

仅处理 CONFIG_SCHEMA 中 scope='global' 且有 setter 的字段；超过 2 级深度的路径不处理（视为 agent scope）；从 agent 对象中删除已提取的 global 字段后，若 parent 对象变空则一并删除。

// 示例
const { global, agent } = splitByScope({ locale: 'zh', thinking_level: 'high', model: 'gpt-4' });
// global = [{ key: 'locale', value: 'zh', setter: 'setLocale' }, { key: 'thinking_level', value: 'high', setter: 'setThinkingLevel' }]
// agent = { model: 'gpt-4' }

函数 injectGlobalFields(config, engine)

export function injectGlobalFields(config: Record<string, unknown>, engine: Record<string, Function>): void
参数config -- 将被原地修改的 config 对象；engine -- 需提供 schema 中声明的 getter 方法
返回值void（原地修改 config）
说明遍历 CONFIG_SCHEMA 中所有 scope='global' 且有 getter 的字段，调用 engine 上的 getter 方法获取值并注入到 config 对象中。

若 engine 上不存在对应的 getter 方法则跳过该字段；若嵌套路径的 parent 不存在则自动创建空对象；原地修改 config 参数。

函数 resolveSecretPatch({ patch, existing, secretKeys })

export function resolveSecretPatch({ patch, existing, secretKeys }): SecretRecord
参数patch -- 前端提交的更新对象；existing -- 已存储的当前值（默认 {}）；secretKeys -- 敏感键名集合
返回值合并后的新对象。对于敏感键：若 patch 中的值是遮蔽值 ("********") 则从 existing 中恢复原值，否则使用 patch 新值

// 典型使用场景：用户编辑 provider 配置，api_key 字段在前端显示为 "********"

  patch: { api_key: "********", name: "my-provider" },
  existing: { api_key: "sk-real-key-123" },
});
// → { api_key: "sk-real-key-123", name: "my-provider" }

对于敏感键，一旦匹配即不再递归其子节点；路径格式为点分隔字符串。

函数 atomicWriteSync(filePath, content, opts?)

export function atomicWriteSync(filePath: string, content: string, opts?: { mode?: number }): void
参数opts.mode -- 文件权限位（如 0o600 用于敏感凭据文件）
说明原子写入：先写到 <filePath>.tmp，再 rename 覆盖目标文件。防止进程崩溃时产生半写文件。

先通过 writeFileSync 的 mode 参数设置权限；再通过 chmodSync 兜底（捕获异常不抛出）；rename 是原子操作（同一文件系统内）。

函数 safeCopyDir(src, dst)

export function safeCopyDir(src: string, dst: string): void
说明带 rollback 的原子目录复制。流程：1) 复制 src→dst.tmp_{ts}；2) 若 dst 存在则 rename dst→dst.bak_{ts}；3) rename dst.tmp_{ts}→dst；4) 删除 dst.bak_{ts}。步骤 3 失败时 rollback：恢复 dst.bak_{ts}，清理 tmp。

复制过程中任何失败抛出 AppError('FS_COPY_FAILED', { cause, context: { src, dst } })。

支持符号链接目录（junction on Win32）；覆盖前先 chmod 目标文件为 0o644（Windows NTFS 上忽略异常）。

3.3 log-redactor.ts

函数 redactLogText(value, options?)

export function redactLogText(value: unknown, options?: { homeDir?: string; extraPaths?: string[] }): string
参数options.homeDir -- 用户主目录（替换为 ~）；options.extraPaths -- 额外需遮蔽的路径列表
返回值遮蔽后的字符串

遮蔽规则覆盖 9 类敏感信息：

  1. Base64 data URI：data:...;base64,... -> data:$1;base64,[redacted]
  2. URL 内嵌凭据：https://user:pass@host -> https://[credentials]@host
  3. URL 查询参数密钥：token、api_key、secret 等参数值遮蔽
  4. Authorization Header：Bearer token 遮蔽
  5. Cookie Header：完整 cookie 值遮蔽
  6. 赋值式密钥：api_key=xxx / api_key: "xxx" 右侧遮蔽
  7. 已知 API Key 格式：sk-*、AKIA*、gsk_*、ghp_*、glpat-*、xox* 等
  8. PII：信用卡号、中国身份证号、美国 SSN、邮箱地址
  9. 长随机字符串：40 字符以上的 base64/hex 类 token

函数 redactLogValue(value, options?, state?)

export function redactLogValue(value: any, options?: any, state?: { depth?: number; seen?: WeakSet<object> }): any
参数state.depth -- 当前递归深度（最大 8）；state.seen -- 循环引用检测
返回值递归遮蔽后的新对象

原始类型（number/boolean/bigint）直接返回；symbol/function 转字符串后遮蔽；Error 对象保留 name/message/stack/code 并遮蔽；数组递归处理；普通对象按 SENSITIVE_OBJECT_KEY_RE 正则（匹配 30+ 个敏感键名）直接遮蔽整个值；深度超过 8 返回 "[MaxDepth]"；循环引用返回 "[Circular]"。

6.2 error-bus.ts

类 ErrorBus

export class ErrorBus { constructor(); addBreadcrumb(crumb: Record<string, unknown>): void; report(error: unknown, extra?: ReportExtra): void; subscribe(listener: ErrorBusListener): () => void; // 返回取消订阅函数 }
说明进程级错误事件总线。提供 breadcrumb 追踪、错误去重（5 秒窗口内相同 fingerprint 只报一次）、自动路由（根据 severity 和 error code 分发到 statusbar/boundary/toast）、日志输出（带脱敏）。

去重机制：基于 extra.dedupeKey || error.code 生成 fingerprint，5 秒内重复上报被静默丢弃；fingerprint 缓存超过 200 条时自动清理过期条目。

自动路由：WS_DISCONNECTED → statusbar；severity='critical' → boundary；其余 → toast。

Listener 隔离：每个 listener 在 try/catch 中执行，单个 listener 异常不会影响其他 listener 或总线本身。

Breadcrumb：最多保留 50 条，超出时 shift 最旧的。report 时自动附加当前 breadcrumb 快照。

全局单例 errorBus

export const errorBus = new ErrorBus();
说明进程级单例 ErrorBus 实例。所有模块共享同一个错误总线。

6.3 retry.ts

函数 withRetry(fn, opts?)

export async function withRetry(fn: () => Promise<any>, opts?: { maxAttempts?: number; // 默认 3 baseDelayMs?: number; // 默认 1000 maxDelayMs?: number; // 默认 30000 signal?: AbortSignal; shouldRetry?: (err: AppError) => boolean; // 自定义重试判断 }): Promise<any>
说明使用 AWS 推荐的 decorrelated jitter 算法：delay = min(maxDelay, random(baseDelay, previousDelay * 3))。每次重试前检查 signal 是否已 aborted。

最后一次尝试失败时抛出 AppError；若 signal 已 aborted 则立即抛出。

shouldRetry 未提供时使用 error.retryable 属性判断；sleep 支持 AbortSignal 中断（通过 addEventListener('abort', ...) 实现）。

// 示例：最多 5 次尝试，初始延迟 500ms
const result = await withRetry(() => fetch(url), {
  maxAttempts: 5,
  baseDelayMs: 500,
  signal: abortController.signal,
  shouldRetry: (err) => err.retryable && err.httpStatus !== 401,
});

export function isNoProxyMatch(targetUrl: string | URL, noProxy: string): boolean

函数 resolveProxyForUrl(targetUrl, config, env?)

export function resolveProxyForUrl(targetUrl: string | URL, config: any, env?: typeof process.env): string
返回值代理 URL 字符串，直连时返回 ""
说明根据代理配置为目标 URL 解析代理地址。direct 模式直接返回空；system 模式从环境变量读取；manual 模式按协议分发（http→httpProxy/httpsProxy，ws→wsProxy 等）。若目标 URL 匹配 no_proxy 规则则返回空。

协议回退链：http 优先 httpProxy → httpsProxy；https 优先 httpsProxy → httpProxy；ws 优先 wsProxy → httpProxy → httpsProxy；wss 优先 wssProxy → httpsProxy → httpProxy → wsProxy。
