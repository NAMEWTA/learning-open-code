1. 架构位置与分层结构

┌─────────────────────────────────────────────────────────┐
│                    Desktop / CLI                         │  ← 用户界面层（Electron / TUI）
├─────────────────────────────────────────────────────────┤
│  Server (HTTP API + WebSocket + Bridge)                  │  ← 网络服务层
├─────────────────────────────────────────────────────────┤
│  Core (编排层)                                           │
│  agent-launcher, agent-executor, session-gateway,        │
│  hub-manager, plugin-loader, desk-daemon, ...            │  ← 业务编排
├─────────────────────────────────────────────────────────┤
│  Lib (本模块 — 功能实现层)                           │
│  ┌──────────┬──────────┬──────────┬────────────────────┐│
│  │ 记忆身份  │ 工具执行  │ 安全沙盒  │ AI / LLM 接入      ││
│  │ 桥接集成  │ 书桌自动化 │ 文件技能  │ 通知/日记/实验/... ││
│  └──────────┴──────────┴──────────┴────────────────────┘│
├─────────────────────────────────────────────────────────┤
│  Pi SDK ( @mariozechner/pi-* )                          │  ← 外部 AI Agent SDK
├─────────────────────────────────────────────────────────┤
│  Node.js / Electron / TypeScript                        │  ← 运行时
└─────────────────────────────────────────────────────────┘

分层原则：

  • lib/ 模块之间允许相互引用，但依赖方向遵循 "基础 → 上层" 原则
  • 顶层 .ts 文件通常是无副作用的工具类/服务类，可被任何子目录引用
  • 子目录内部可自由组织，但不应形成循环依赖
  • lib/ 不依赖 core/ 或 server/，仅依赖 shared/ 共享类型和 @mariozechner/pi-* SDK

export default async function(host) {
  host.phase("调研阶段");

  // 并行搜索多个数据源
  const [webResult, docResult] = await host.parallel([
    () => host.agent("搜索最新的 React 19 文档...", {
      label: "web-search",
      toolFilter: ["web_search", "web_fetch"],
    }),
    () => host.agent("分析项目中的组件使用情况...", {
      label: "code-analysis",
      toolFilter: ["read", "grep", "find"],
    }),
  ]);

  host.phase("总结阶段");

  // 流水线处理：先分析再翻译
  const finalReport = await host.pipeline(
    [webResult, docResult],
    async (data) => host.agent(`分析：${data}`),
    async (analysis) => host.agent(`翻译为中文：${analysis}`),
  );

  host.log(`剩余 token 预算：${host.budget.remaining()}`);
  return finalReport;
}

附录：关键设计原则

文件即数据库

通道、记忆、会话文件、技能组合等均以 Markdown 或 JSON 文件存储，不引入外部数据库。文件系统本身就是持久化层。

Pi SDK 单一入口

所有对 @mariozechner/pi-* 的导入必须通过 lib/pi-sdk/index.ts，其余模块不得直接导入 SDK。这确保了版本升级时的改动隔离。

工厂模式

工具、命令运行器、Provider 插件、Cron 调度器等均采用工厂函数创建，依赖通过参数注入，便于测试和替换。

调度与执行解耦

Cron 调度器只负责时间判断和触发回调，不直接调用 LLM。LLM 调用由 core/ 编排层在回调中执行。

延迟执行（Lazy Evaluation）

Workflow 的 agent() 返回 Proxy/Thenable，推迟到 await 才真正执行。这允许检测未等待的调用并保证执行顺序。

幂等与去重

通知、桥接消息、延迟结果均带 10 分钟 TTL 的幂等键，防止重复投递。
