async compileToday(summaryManager, outputPath, resolvedModel, opts?)

编译今天的 timeline 条目 → today.md。水位线增量机制：只重扫新增/修订过的摘要，将属于当前逻辑日的 timeline 条目作为 delta 喂给 LLM。
export async function compileToday( summaryManager: SessionSummaryManager, outputPath: string, resolvedModel: { model: string, api: string, api_key: string, base_url: string }, opts?: { since?: any, statePath?: string } ): Promise<"compiled" | "skipped">
参数类型说明
summaryManagerSessionSummaryManager摘要管理器，提供 getAllSummaries() 或 getSummariesInRange()
outputPathstringtoday.md 输出路径
resolvedModelobjectLLM 模型配置（model/api/api_key/base_url）
opts.sinceany强制重置水位线的时间戳
opts.statePathstring自定义 today-state.json 路径，默认 memoryDir/today-state.json
边界条件： ① 逻辑日切换（state.logicalDate !== 当前逻辑日）→ 草稿与水位线重置，新一天从空白开始； ② 同一逻辑日、无新增/修订摘要 → 保留现有草稿，不落水位线（等下次 delta）； ③ 从无水位线（老用户升级/状态丢失）→ 一次性扫描所有摘要中的今天条目，与已有草稿合并； ④ 空 sessions + 无水位线 → 清空 today.md 陈旧内容。
语义：返回 "compiled" 表示已处理（可能输出为空），"skipped" 表示无需操作。每条 timeline 事件标注"新增"或"取代先前相关记述"。
// 典型调用（由 memory-ticker 驱动） const result = await compileToday( summaryManager, path.join(memoryDir, "today.md"), resolvedModel, ); // result: "compiled" | "skipped"

async compileDaily(summaryManager, dailyDir, logicalDate, resolvedModel, opts?)

编译已结束那天 → memory/daily/{logicalDate}.md。一天只落一次盘（指纹去重），输入优先使用该逻辑日的 timeline 条目。
export async function compileDaily( summaryManager: SessionSummaryManager, dailyDir: string, logicalDate: string, // YYYY-MM-DD resolvedModel: object, opts?: { since?: any, todayDraftPath?: string } ): Promise<"compiled" | "skipped">
参数类型说明
summaryManagerSessionSummaryManager仅用于草稿缺失时的回落编译
dailyDirstringmemory/daily 目录
logicalDatestringYYYY-MM-DD，要编译的那个逻辑日
resolvedModelobjectLLM 模型配置
opts.todayDraftPathstringtoday.md 草稿路径，缺省时视为草稿不可用
边界条件： ① 当天没有任何内容 → 零占位（不产文件也不留悬空标题），返回 "skipped"； ② 草稿缺失（todayDraftPath 不存在或为空）→ 显式 warn 日志后回落到按当天 session 摘要编译； ③ 两者都没有 → 维持零占位，不产文件； ④ 指纹命中 → 跳过，返回 "skipped"。
异常：LLM 调用失败时向上抛出，不会静默吞错。

class PathGuard

路径权限校验器。定义四种访问级别 BLOCKED / READ_ONLY / READ_WRITE / FULL，所有路径先经过 realpath 解析符号链接再匹配区域。
export class PathGuard { constructor(policy: SandboxPolicy) getAccessLevel(rawPath: string): AccessLevel check(absolutePath: string, operation: "read"|"write"|"delete"): { allowed: boolean, reason?: string } }
方法说明
constructor(policy)从 deriveSandboxPolicy() 结果或兼容旧格式初始化。mode === "full-access" 时跳过所有检查
getAccessLevel(rawPath)按 11 级优先级匹配（BLOCKED 文件/目录 → READ_ONLY agent → READ_ONLY 全局 → READ_WRITE agent → READ_WRITE 全局 → hanakoHome → workspace → policyWritablePaths → 外部），返回 AccessLevel 字符串
check(absolutePath, operation)操作权限检查。read 需 READ_ONLY+，write 需 READ_WRITE+，delete 需 FULL
边界条件： ① 路径不存在 → 递归向上找最近的存在的祖先目录做 realpath，再把不存在的段拼回去（支持 mkdir -p 场景）； ② fullAccess 模式 → check() 始终返回 { allowed: true }； ③ 路径解析完全失败 → 返回 BLOCKED。
// 典型使用 const guard = new PathGuard(sandboxPolicy); const { allowed, reason } = guard.check("/home/user/project/src/file.ts", "write"); if (!allowed) throw new Error(reason);

AccessLevel 常量

export const AccessLevel = { BLOCKED: "blocked", READ_ONLY: "read_only", READ_WRITE: "read_write", FULL: "full", };

sync classifyExecCommand(command, opts?)

对 shell 命令进行风险分类。返回 ExecCommandClassification — 这是命令执行前安全检查的核心入口。
export function classifyExecCommand( command: string, opts?: { platform?: NodeJS.Platform } ): ExecCommandClassification // 返回类型 interface ExecCommandClassification { kind: "safe" | "probe" | "mutation" | "dangerous" | "unknown"; reason: string; executable?: string; unsupportedSyntax?: boolean; errorCode?: string; }
风险等级含义示例
safe只读操作，无副作用git status、Get-ChildItem、ls
probe环境探测命令node --version、which python
mutation文件修改但非破坏性mkdir、cp、git push（非 force）

unknown无法分类空命令、复杂管道
边界条件： ① 空命令 → unknown / "empty-command"； ② Windows 平台检测到 POSIX heredoc 语法 → unknown + unsupportedSyntax=true + errorCode； ③ git push -f / --force / --all / --mirror → dangerous。

API 方法说明
agent(prompt, opts?)调度一个子 agent 执行任务。返回类 Promise 对象（lazy：await 时才真正执行）。支持 label/model/agentType/toolFilter/access/schema 选项
parallel(thunks)并行执行一组 thunk（Promise.all 包装，单个失败返回 null 不中断）
pipeline(items, ...stages)流水线处理：每个 item 依次经过所有 stage，单个 stage 失败返回 null
log(message)输出日志消息到 onProgress + onAgentEvent
phase(title)设置当前阶段标题（弱分组标签，best-effort）
workflow(script, childArgs?)嵌套子 workflow（共享 limiter/signal/budget）
边界条件： ① agent() 返回的是 Proxy 对象，必须 await，否则 .result 会抛出提示； ② 所有 agent 调用结束后 runtime 会检查是否有未 await 的调用（assertNoUnawaitedAgentCalls）； ③ budget 耗尽（spent >= total）→ 拒绝新 agent 调用； ④ journal 回放模式 → cache hit 不消耗 limiter slot，瞬间返回； ⑤ opts.access="read"/"write" → 自动映射 toolFilter 权限。
异常： ① agent() prompt 非空字符串 → Error； ② opts 含不支持的 key → Error（列出合法 key）； ③ agentType 无法解析 → Error； ④ 未 await agent() → Error（在脚本结束时由 assertNoUnawaitedAgentCalls 触发）。
// workflow 脚本中的典型用法 export default async function(api) { api.phase("调研阶段"); const research = await api.agent("搜索相关资料", { agentType: "researcher", access: "read" }); api.phase("编写阶段"); const draft = await api.agent("根据资料编写报告", { access: "write" }); api.log("报告完成"); return draft; }
