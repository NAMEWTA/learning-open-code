StoreInstance 存储实例路由图谱

一句话：StoreInstance 是后端连接配置的业务表；RagPO 保存选中的实例 ID；摄入和检索阶段由 VectorStoreFactory 或 SearchEngineFactory 查表、解析 config、选择已注册的 PgVector、Milvus、Elasticsearch 实现。

源码校正：用户指定的 persistence/store 包在当前源码中不存在；StoreInstancePO 和 StoreInstanceMapper 实际位于 persistence/admin。

维度约束

检查点源码行为失败信息
embeddingModelIdEmbeddingModelDimensionService.getEmbeddingDimension 优先读取模型配置中的 embeddingDimension，其次调用一次 embedding 探测，最后回退 1024 并回写。模型不存在时抛出 Embedding 模型不存在。
vectorStoreInstanceIdVectorDimensionConstraintService 回查 StoreInstancePO 并按类型取后端最大维度。实例不存在时抛出 存储实例不存在。
dimensionOfVectorModel必须非空，不能小于 64，不能超过模型上限，也不能超过后端上限。按具体超限原因抛出 SnailAiCommonException。
后端类型最大维度说明
PG_VECTOR2000当前 Admin 约束层限制 PgVector 最大维度为 2000。
MILVUS4096当前约束层限制为 4096。
ELASTICSEARCH4096向量模式下限制为 4096。

Factory 路由

VectorStoreFactory

步骤源码行为排查点
输入create(RagPO) 读取 dimensionOfVectorModel；为空时再通过 EmbeddingModelDimensionService 获取模型维度。确认 RagPO.embeddingModelId 和维度是否正确。
查实例getStoreInstanceOrThrow 用 vectorStoreInstanceId 查询 StoreInstancePO。实例不存在会抛 向量库实例不存在。
类型转换StoreInstanceTypeEnum.fromType 后用枚举名转 VectorStoreType。PG_FULLTEXT 不能转成向量实现。
注册表从 VectorStoreFactory.REGISTER 取构造器。未注册抛 向量库类型未注册。
缓存缓存 key 是 vectorStoreInstanceId + "_" + embeddingModelId；实例更新时间或模型更新时间变化会重建。改了连接参数后看 updateDt 是否变更。

SearchEngineFactory

步骤源码行为排查点
输入forStoreInstance(searchEngineInstanceId) 要求实例 ID 非空；但保存期的 KnowledgeService.validateKnowledgeStorage 只在 ID 非空时校验搜索实例。启用 searchEngineEnable 时也要检查 searchEngineInstanceId 是否真的入库。
类型转换StoreInstanceTypeEnum 的枚举名传给 SearchEngineEnum.fromType。当前 SearchEngineEnum 只有 ELASTICSEARCH。
注册表EsSearchLifecycle 把 ELASTICSEARCH 绑定到 ElasticsearchSearchEngine。PG_FULLTEXT 会走类型不存在或未注册错误。

RAG 调用点

场景类与方法调用行为错误影响
知识库创建KnowledgeService.create先校验存储实例，再校验维度，最后保存 RagPO。校验失败时知识库不入库。
文档摄入DocumentPipeline.dualWrite先 vectorStoreFactory.create(knowledge)，分批写入向量；再按搜索开关写入全文索引。向量失败会使文档处理失败；搜索写入失败只记录 warning。
向量检索VectorSearchHandler.handle用 VectorStoreFactory.create(ctx.getKnowledge()) 构造后端，检索 rag_ 加知识库 ID。后端构造或检索异常会中断当前 pipeline。
BM25 检索Bm25SearchHandler.handle仅在 searchEngineEnable=true 时调用 SearchEngineFactory.forStoreInstance。搜索实例缺失或未注册会抛异常；未启用则直接跳过。
知识库删除KnowledgeService.delete尝试调用 deleteByIndexName 清理向量后端，再删除 chunk、document、knowledge。向量清理失败只记录 warning，不阻断业务库删除。
文档重处理DocumentPipeline.cleanExistingData按旧 vectorId 清理向量，再删除旧 chunk。清理失败只记录 warning，后续仍删除业务 chunk。

Fallback 与错误路径

场景当前行为调试入口
未传 vectorStoreInstanceIdKnowledgeService.validateKnowledgeStorage 抛 请选择向量库实例；运行时没有自动取默认实例。Admin 创建知识库请求体。
默认实例isDefault=true 会清理同分类其他默认值；列表按默认值优先排序。StoreInstanceService.clearDefaultInCategory。
向量分类类型不合法向量分类只接受 PG_VECTOR、MILVUS、ELASTICSEARCH。StoreInstanceService.validateCategoryType。
搜索分类类型不合法搜索分类当前只接受 ELASTICSEARCH。SEARCH_TYPES 常量。
PG_FULLTEXT业务枚举存在，但当前搜索工厂没有注册实现。SearchEngineEnum 和 EsSearchLifecycle。
向量 indexName 为空AbstractSnailAiVectorStore.add/search 抛 indexName 不能为空。检查 IndexNameBuilder.KNOWLEDGE 参数。
向量查询文本为空AbstractSnailAiVectorStore.search 返回空列表。检查 RagSearchRequestVO.query。
搜索写入失败DocumentPipeline.dualWrite 捕获异常并记录 SearchEngine insert failed (non-fatal)。搜索实例配置、ES bulk 日志、索引创建日志。
搜索启用但未传 searchEngineInstanceId保存期：KnowledgeService.validateKnowledgeStorage 只在 searchEngineInstanceId != null 时校验搜索实例，所以 searchEngineEnable=true 且 ID 为空可能入库。运行期：DocumentPipeline 搜索双写会调用 SearchEngineFactory.forStoreInstance(null)，异常被捕获为 non-fatal；Bm25SearchHandler 检索时同样调用该工厂，但这里没有捕获，会抛出 搜索引擎实例 ID 不能为空。同时查 sai_rag.search_engine_enable 和 sai_rag.search_engine_instance_id，不能只看开关。
搜索索引不存在ElasticsearchSearchEngine.search 检查 index 不存在则返回空列表。确认摄入时是否启用 searchEngineEnable。
搜索写入 indexName 为空SearchAddRequest.indexName == null 时，ElasticsearchSearchEngine.insert 立即抛 SearchEngineException；如果是空字符串 ""，当前代码不会在 insert 入口拦截，会更晚落到 ES ensureIndex、bulk 等调用层失败。排查 IndexNameBuilder.KNOWLEDGE 的参数和调用方是否传入空字符串。
后端连接禁用PgVector 的 PgDataSourceFactory、Milvus 的 MilvusClientFactory、Elasticsearch Vector 的 EsClientFactory 都会在 enabled=false 时抛 IllegalArgumentException；Elasticsearch Search 的 ElasticsearchSearchEngine.createClient 当前不检查 enabled。向量侧先查 sai_store_instance.config.enabled；搜索侧不能把 enabled=false 当作已被工厂拦截。

调试 checklist

顺序检查项期望结论
1查 sai_rag.id 对应的 vector_store_instance_id、search_engine_enable、search_engine_instance_id、dimension_of_vector_model。确认知识库确实绑定了目标实例和维度。
2查 sai_store_instance.id 的 category、type、config、status。向量实例必须是分类 1；搜索实例必须是分类 2。
3对照 StoreInstanceTypeEnum 与 VectorStoreType 或 SearchEngineEnum。确认类型有对应工厂注册。
4计算 IndexNameBuilder.KNOWLEDGE。RAG 统一访问 rag_ 加知识库 ID。
5按后端检查真实对象。PgVector 查表；Milvus 查 collection；Elasticsearch 查 index。
6文档摄入失败时先看向量写入日志。向量失败会使文档失败，搜索失败不会。
7检索无 BM25 结果时确认 searchEngineEnable 和摄入时的搜索写入日志。未启用或写入失败都会导致全文索引没有数据。
8改了实例连接参数后检查 update_dt。VectorStoreFactory 依赖实例与模型更新时间触发缓存重建。

读法建议 先用短课建立链路，再用本图谱定位具体文件；真正调试时按 checklist 从业务库字段查到后端对象。
