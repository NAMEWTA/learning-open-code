RAG 文档上传摄入参考图谱

一句话：Snail AI 的 RAG 摄入链路先把上传或 URL 导入的文件落到资源表和文档表，状态置为 PENDING；定时任务再调用 DocumentPipeline 解析、切块、写 chunk、向量库和可选全文搜索索引，最后把文档改为 SUCCESS 或 FAILED。

源码优先：RagDocumentStatus 有 PARSING，但当前主路径没有使用该状态；全文搜索写入失败是非致命 warning；删除和重处理清理路径中未看到 RAG 侧调用 SearchEngine.delete。

入口 API

方法路径入口方法摄入行为
POST/document/uploadRagDocumentController.upload传统直传，走 LocalFileImportStrategy 读字节和 hash，再写正式资源和 PENDING 文档。
POST/document/upload/previewuploadPreview批量上传预览，写 DOCUMENT_PREVIEW 临时资源，返回每个文件的去重决策。
POST/document/upload/commituploadCommit根据 token 回放预览状态，重新做冲突判定，再新建、跳过或覆盖旧文档。
DELETE/document/upload/preview/{token}cancelUploadPreview删除预览阶段临时资源并失效 token。
POST/document/import/urlimportFromUrlURL 导入，限制 http/https、拒绝 localhost 和私网地址，最大读取 50 MB。
POST/document/{id}/reprocessreprocess把已有文档状态重置为 PENDING，等待定时任务重跑。
DELETE/document/{id}delete清理资源、chunk、未被其他 chunk 引用的向量，然后删除文档行。

  participant ChunkDB as RagChunkMapper

  participant Model as ModelRuntimeHandler
  participant Vector as 向量/搜索存储

  Admin->>Ctrl: upload / import / commit / reprocess
  Ctrl->>Service: 校验请求并执行去重决策
  Service->>DocDB: 新建或重置 status=PENDING
  Task->>DocDB: select PENDING limit 5
  Task->>Pipe: processDocument(documentId)
  Pipe->>DocDB: claimPendingDocument 抢占为 PROCESSING
  Pipe->>Parser: ResourceService.load(resourceId), parser.parse(inputStream)
  alt parser 异常或解析内容为空
    Pipe->>DocDB: update FAILED(errorMsg)
    Note over Pipe,ChunkDB: cleanExistingData 尚未执行，旧 chunk 可能仍在
  else 解析内容非空
    Pipe->>Pipe: cleanExistingData(旧 chunk + 未共享向量)
    Pipe->>Chunker: chunk(content, RagConfigDO.ChunkParams)
    alt 没有生成 chunk
      Pipe->>DocDB: update FAILED(No chunks generated)
    else 生成 chunks
      Pipe->>ChunkDB: 插入 sai_rag_chunk 行
      Pipe->>VecFactory: create(RagPO)
      opt dimensionOfVectorModel 为空
        VecFactory->>Dim: getEmbeddingDimension(embeddingModelId)
      end
      VecFactory->>Model: buildEmbeddingModel(modelConfig)
      Pipe->>Vector: SnailAiVectorStore.add(rag_{ragId})
      opt searchEngineEnable=true
        Pipe->>Vector: SearchEngine.insert(rag_{ragId})
      end
      Pipe->>DocDB: update SUCCESS(chunkCount=N)
    end
  end

核心 pipeline 与策略

阶段核心文件判断点
导入策略LocalFileImportStrategy.java、UrlImportStrategy.java直传读取 MultipartFile；URL 导入校验公网目标、状态码和大小限制。
去重决策DocumentDedupChecker.java按知识库内文件名、内容 hash 或二者查询冲突，再映射为 reject、skip、overwrite。
文档调度DocumentProcessingTask.java每 10 秒按创建时间取 5 条 PENDING 文档。
状态抢占DocumentPipeline.claimPendingDocument条件更新要求当前仍是 PENDING，成功后进入 PROCESSING。
解析DocumentParserFactory.java按 fileType 选择 PDF、DOCX、Excel、Markdown、HTML 或 TXT parser；不支持会抛异常并失败。
切块DocumentChunkingService.java读取 RagConfigDO.ChunkParams，支持 default、delimiter、regex、smart。
智能切块SmartChunkStrategy.javachunkModelId 为空会回退为原文；模型配置不存在、非 CHAT 模型、API Key 不可用会直接抛异常并导致文档 FAILED；只有模型调用后返回空、JSON 解析失败或调用异常等场景会 fallback（回退）。
向量双写VectorStoreFactory.java、ModelRuntimeHandler.java、AbstractSnailAiVectorStore.java根据知识库的向量库实例和 embedding 模型创建后端；VectorStoreFactory 通过 ModelRuntimeHandler.buildEmbeddingModel(...) 构建 embedding，add 委托 Spring AI 完成向量化和写入。
向量维度EmbeddingModelDimensionService.javadimensionOfVectorModel 为空时先读模型扩展配置，再尝试一次 embedding 探测维度，仍失败则使用默认维度并回写配置。
搜索双写SearchEngineFactory.java、ElasticsearchSearchEngine.java仅在 searchEngineEnable=true 时写全文索引，插入失败不阻断文档成功。

状态字段

对象字段意义源码行为
RagDocumentStatusPENDING=0待处理上传、URL 导入、commit 新建、reprocess 都会进入或回到该状态。
RagDocumentStatusPARSING=1解析中枚举存在，主 pipeline 当前未设置。
RagDocumentStatusPROCESSING=2处理中claimPendingDocument 抢占成功后设置。
RagDocumentStatusSUCCESS=3完成chunk 写入、向量写入完成后设置，同时写 chunkCount 并清空 errorMsg。
RagDocumentStatusFAILED=4失败知识库缺失、解析空内容、无 chunk、parser 或向量链路异常会写入该状态和错误信息。
RagChunkPOvectorId向量库 ID新 chunk 生成 UUID；同 RAG 内内容 hash 已存在时复用已有 vectorId。
RagChunkPOcontentHashchunk 去重键由 chunk 内容 SHA-256 生成，索引为 (rag_id, content_hash)。

表与索引

存储表或索引关键字段用途
业务库sai_ragembedding_model_id、dimension_of_vector_model、vector_store_instance_id、search_engine_enable、search_engine_instance_id、config知识库摄入配置来源。
业务库sai_resourcestorage_key、biz_type、biz_id、access_url保存文件存储位置和业务归属；预览阶段使用 DOCUMENT_PREVIEW。
业务库sai_rag_documentrag_id、resource_id、content_hash、status、error_msg、chunk_count摄入状态和文档级去重记录；索引包含 idx_rag_doc_rag、idx_rag_content_hash、idx_rag_name。
业务库sai_rag_chunkdocument_id、paragraph_index、chunk_index、content、vector_id、content_hash保存可检索文本片段和向量引用；索引包含 idx_rag_chunk_document、idx_chunk_rag_hash。
业务库sai_store_instancecategory、type、config向量库和搜索引擎实例配置。
业务库sai_model_configmodel_type、adapter_key、api_endpoint、api_key、config_jsonembedding 模型配置来源。
向量库rag_{ragId}id、content、metadata.ragId/documentId/chunkId由 IndexNameBuilder.KNOWLEDGE 生成，供向量检索使用。
搜索库rag_{ragId}content、metadataES 全文索引，使用 cjk_analyzer，供 BM25 检索使用。

异常补偿边界

场景当前行为排查入口
预览后用户关闭页面token 30 分钟后失效；孤儿资源任务每小时清理超过 1 小时仍为 DOCUMENT_PREVIEW 的资源。DocumentUploadPreviewService.cleanupExpiredTokens、OrphanPreviewResourceTask
commit 时冲突变化重新判定冲突，返回 CONFLICT_CHANGED，不消费 token，允许用户重新确认。isConflictChanged
覆盖旧文档先清理旧文档资源、chunk 和未共享向量，再插入新文档。cleanupDocumentInternal
解析异常或解析为空文档进入 FAILED，写入错误信息；因为 cleanExistingData 在解析非空后才执行，重处理场景的旧 chunk 和旧向量可能仍在。parseContent、processDocument
切块结果为空解析已通过且旧数据已清理，但没有生成新 chunk；文档进入 FAILED。DocumentChunkingService.chunk
向量库或 embedding 失败sai_rag_chunk 已先插入，异常再从 dualWrite 冒泡到 pipeline 外层；文档进入 FAILED，新 chunk 可能已留在业务库。dualWrite、VectorStoreFactory、ModelRuntimeHandler
全文搜索写入失败记录 warning，文档仍可成功；之后 BM25 可能缺失该文档。DocumentPipeline.dualWrite
重处理已有文档reprocess 只把状态重置为 PENDING；真正重跑时先 parseContent，解析非空后才 cleanExistingData，再切块、插入新 chunk、向量/搜索双写。搜索索引清理未在 RAG 路径中看到。RagDocumentService.reprocess、cleanExistingData
删除文档资源删除和向量删除是 best effort，失败只记录 warning；最后删除 chunk 和文档行。RagDocumentService.cleanupDocumentInternal

调试 checklist

问题检查点对应文件
上传接口返回成功但一直没解析sai_rag_document.status 是否为 0；定时任务是否启动；日志是否出现 pending 扫描。DocumentProcessingTask.java
文档变成 FAILED先看 error_msg，再按资源读取、parser、chunker、向量库顺序排查。DocumentPipeline.java
文件类型不支持确认 file_type 是否被 parser 的 supports 接受。DocumentParserFactory.java 和各 parser
chunk 数为 0检查解析后文本是否为空，或切块配置是否让所有段落为空。DocumentChunkingService.java
smart 切块让文档失败先区分“模型配置前置校验失败”和“模型调用后 fallback（回退）”：配置不存在、非 CHAT、API Key 不可用会失败；返回空、JSON 解析失败或调用异常通常回退。SmartChunkStrategy.java
向量没有写入检查 vector_store_instance_id、embedding_model_id、模型 API Key、维度和后端注册；维度为空时会走 EmbeddingModelDimensionService。VectorStoreFactory.java、ModelRuntimeHandler.java、EmbeddingModelDimensionService.java
chunk 有 vectorId 但检索不到确认向量索引名是 rag_{ragId}，再查具体 PgVector、Milvus 或 ES vector 后端。IndexNameBuilder.java
BM25 查不到但向量能查到确认 search_engine_enable 和 search_engine_instance_id，搜索写入失败可能只留下 warning。SearchEngineFactory.java、ElasticsearchSearchEngine.java
重处理后全文索引疑似残留RAG 清理路径未看到 search delete 调用，需直接核查搜索后端索引文档。DocumentPipeline.cleanExistingData
