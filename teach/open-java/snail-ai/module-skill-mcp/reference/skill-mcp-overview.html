Skill 与 MCP 能力模块地图

这份参考用于快速定位 Snail AI 中 Skill、MCP Server 和工具 resolver 的源码边界。短课负责建立主链路，本文负责放完整清单、时序和排查入口。

1. 一句话边界

Admin 端保存 Skill 包、MCP Server 配置和 Agent 绑定关系；Server 端 Agent 责任链把绑定关系转为 ChatDispatchRequest 描述符；Agent Client 端 resolver 把描述符转为 Spring AI ToolCallback，最后由 DefaultChatClientFactory.defaultTools 注入 ChatClient。

Admin 配置 ├─ SkillController / SkillService ├─ McpServerController / McpServerService └─ AgentService 写入关联表 ↓ Server Agent Chain ├─ McpHandler 生成 McpServerDescriptor ├─ SkillAgentChatHandler 生成 SkillDescriptor └─ LlmCallHandler 组装 ChatDispatchRequest ↓ gRPC /chat/dispatch Agent Client ├─ ToolRuntime.resolve ├─ ClientMcpToolResolver 转 MCP ToolCallback ├─ ClientSkillToolResolver 注册 read_skill └─ DefaultChatClientFactory.defaultTools

2. 源码分层

open-java/snail-ai/ ├── docs/ │ ├── guide/skill/ ← Skill 用户文档 │ ├── guide/mcp/ ← MCP 用户文档 │ └── sql/snail_ai_schema.sql ← Skill/MCP 表结构 ├── snail-ai-server/ │ ├── snail-ai-server-admin/ ← Admin API、VO、Service │ └── snail-ai-server-features/ ← Agent chain、Skill callback、MCP service ├── snail-ai-commons/ ← DTO、gRPC URI、MCP 枚举 └── snail-ai-agent/ └── snail-ai-agent-executor/ ← ToolRuntime、resolver、tool、ChatClient 注入

3. Admin API 完整入口

3.1 SkillController

入口方法与路径请求输入职责与结果
上传 Skill 包POST /skill/uploadfile multipart 参数，必须是 zip。解压 ZIP，查找 SKILL.md，解析 frontmatter，写入 SkillPO，并把支撑文件写入 SkillFilePO。
分页查询 SkillGET /skill/pageSkillQueryVO：page、size、keyword。按名称关键字分页查询，返回 PageResult<List<SkillResponseVO>>。
删除 SkillDELETE /skill/{id}路径参数 id。删除 Skill、Agent 关联、支撑文件记录和磁盘文件。
下载 Skill 包GET /skill/{id}/download路径参数 id。把 DB 中的 SKILL.md 和支撑文件重新打包为 zip 流。
全部 Skill 列表GET /skill/list无。返回全部 Skill，用于 Agent 选择下拉。
在线创建 SkillPOST /skillSkillCreateRequestVO。创建空 Skill 记录，skillContent 初始为空，后续由文件编辑器补内容。
Skill 详情GET /skill/{id}路径参数 id。读取单个 SkillResponseVO；不存在时抛出“技能不存在”。
更新 Skill 元数据PUT /skill/{id}SkillUpdateRequestVO。更新名称或描述；发生变化时自增版本号。
读取文件树GET /skill/{skillId}/files路径参数 skillId。把重建的 SKILL.md 和 sai_skill_file 组装成树。
读取文件内容GET /skill/{skillId}/files/content路径参数 skillId，查询参数 path。读取 SKILL.md 或支撑文件内容，返回内容、编码和大小。
保存文件内容PUT /skill/{skillId}/files/content路径参数 skillId，查询参数 path，请求体 SkillFileContentRequestVO。保存 SKILL.md 或支撑文件；保存后自增版本并重算大小。
新建文件或目录POST /skill/{skillId}/filesSkillCreateFileRequestVO。新建支撑文件或目录标记文件，设置 hasFiles 并自增版本。
删除支撑文件DELETE /skill/{skillId}/files路径参数 skillId，查询参数 path。删除支撑文件；禁止删除 SKILL.md；重新计算 hasFiles 和版本。
重命名支撑文件PUT /skill/{skillId}/files/renameSkillRenameFileRequestVO。把旧路径记录复制到新路径并删除旧记录；自增版本。

3.2 McpServerController

入口方法与路径请求输入职责与结果
分页查询 MCP ServerGET /mcp-server/pageMcpServerQueryVO：分页、关键字、传输类型、时间范围。按条件分页查询，返回 PageResult<List<McpServerResponseVO>>。
MCP Server 详情GET /mcp-server/{id}路径参数 id。读取单个 MCP Server；不存在时抛出 MCP Server not found。
创建 MCP ServerPOST /mcp-serverMcpServerRequestVO。保存名称、传输类型、端点、命令、参数、环境变量、认证配置和能力列表。
更新 MCP ServerPUT /mcp-server/{id}路径参数 id，请求体 McpServerRequestVO。按非空字段更新配置；变更传输类型时会清空旧连接字段。
删除 MCP ServerDELETE /mcp-server/{id}路径参数 id。删除 Agent 关联后删除 MCP Server 记录。
测试连接POST /mcp-server/{id}/test-connection路径参数 id。按传输类型创建临时 McpSyncClient，执行 initialize 与能力验证，更新最近连接时间。
全部 MCP Server 列表GET /mcp-server/list无。返回全部 MCP Server，用于 Agent 选择下拉。

Agent 绑定不在上面两个控制器里：POST/PUT /agent 通过 AgentRequestVO.skillIds、mcpServerIds、skillEnabled、mcpEnabled 写入 Agent 和关联表；POST /agent/{id}/chat 的 disabledMcpServerIds、disabledSkillIds 只影响本次对话。

4. 持久化与 DTO/枚举

类别对象关键字段用途

Skill 文件SkillFilePOskillId、filePath、content、encoding保存 scripts、references、examples 等支撑文件。
Skill 绑定AgentSkillPOagentId、skillIdAgent 与 Skill 多对多关系。
MCP 主表McpServerPOtransportType、baseUri、endpoint、command、args、envVars、authType保存 MCP Server 连接配置。
MCP 绑定AgentMcpServerPOagentId、mcpServerIdAgent 与 MCP Server 多对多关系。
分发合同ChatDispatchRequestmcpServers、skills、agentConfig、systemPromptServer 到 Client 的核心 gRPC 载荷。
Skill 内容回调SkillContentRequest
SkillContentResponse		skillId、skillContent、version、filesClient read_skill 回调 Server 获取完整内容。

5. Server 责任链时序

顺序Handler对 Skill/MCP 的影响
10InitContextHandler加载 Agent 配置，后续才能判断 mcpEnabled、skillEnabled。
20ConversationHandler会话管理，不直接处理能力。
30ModelResolveHandler解析模型配置，不直接处理能力。
40McpHandler若 Agent 启用 MCP，读取绑定 MCP Server，过滤本次禁用 ID，生成 McpServerDescriptor。
50SystemPromptHandler生成基础 system prompt。Skill resolver 后续会在 Client 侧追加 Skill 列表。
60RagHandlerRAG 预检索或提示词注入。ToolRuntime 还会过滤名为 RAG MCP 的描述符。
70SkillAgentChatHandler若 Agent 启用 Skill，读取绑定 Skill，过滤本次禁用 ID，生成 SkillDescriptor；源码仍调用 skillPrompt(s.getSkillContent()) 把正文带在描述符里。
75ContextCollectorHandler收集历史、记忆和目标 Client。
80LlmCallHandler把 mcpServerDescriptors、skillDescriptors 放入 ChatDispatchRequest，通过 /chat/dispatch 分发。

6. Client resolver 边界

组件输入输出边界说明
ChatSessionRuntimeChatDispatchRequestClientChatExecutionRequest会话壳，先解析工具，再执行聊天；completion/error 后关闭 MCP resolver 并递减活跃计数。
ToolRuntime完整 dispatch 请求ToolResolution按 MCP、base、RAG、Skill、custom 顺序汇总工具。
ClientMcpToolResolverMcpServerDescriptor 列表MCP 暴露的 ToolCallback每次对话连接 MCP Server，SyncMcpToolCallbackProvider 转工具；SSE 被跳过。
BaseToolResolverAgent ID、conversationIdshell、http_request始终注册，工作目录按 Agent 与会话隔离。
ClientRagToolResolverAgentConfig 中 RAG 字段rag_search仅智能调用模式注册；强制模式由 Server 预检索。
ClientSkillToolResolverSkillDescriptor 列表read_skill追加 Skill 名称、描述和临时目录列表到 system prompt，并注册读取完整 Skill 的工具；当前源码不把 descriptor.skillPrompt 直接拼成 prompt 正文。
ReadSkillToolSkill 名称Skill 正文字符串用描述符中的名称匹配 ID 和版本，通过 RpcClient.fetchSkillContent 回调 Server 重新拉取正文，缓存后异步写支撑文件到临时目录。
CustomToolCallbackProviderSpring 容器 Bean业务 @Tool 回调启动时扫描非内部工具包 Bean，缓存后每次会话追加。
DefaultChatClientFactory模型配置、Advisor、toolsChatClient用 defaultTools 注入工具，并加 ToolCallingAdvisor。

7. Skill 运行时序

上传或在线创建 Skill └─ SkillService 写 SkillPO / SkillFilePO / version ↓ Agent 创建或更新 └─ AgentService 调 updateAgentSkills 写 AgentSkillPO ↓ 用户发起对话 └─ SkillAgentChatHandler 读取 SkillPO，过滤 disabledSkillIds ↓ LlmCallHandler └─ ChatDispatchRequest.skills，SkillDescriptor 仍携带 skillPrompt ↓ ToolRuntime └─ ClientSkillToolResolver 追加 Skill 列表，不直接注入 skillPrompt 正文，注册 read_skill ↓ 模型调用 read_skill └─ ReadSkillTool -> RpcClient.fetchSkillContent ↓ Server 回调 └─ SkillContentCallbackHandler 返回 skillContent 与 files ↓ Client └─ 缓存正文，异步物化支撑文件，返回 Skill 指令

渐进式披露边界要看 Client 源码，而不是只看 DTO：SkillAgentChatHandler 构造的 SkillDescriptor 仍包含 skillPrompt(s.getSkillContent())，但当前 ClientSkillToolResolver.buildSkillsList 只使用名称、描述和临时目录来生成 Skill 列表，未把 skillPrompt 作为 prompt 正文直接注入。模型需要完整指令时调用 read_skill；ReadSkillTool 再通过 RPC 回调 SkillContentCallbackHandler 重新拉取 skillContent 和支撑文件。

8. MCP 运行时序

创建 MCP Server └─ McpServerService 写 McpServerPO ↓ 可选连接测试 └─ createTestClient -> initialize -> listTools ↓ Agent 绑定 MCP └─ AgentService 或 /agent/{id}/mcp-servers 写 AgentMcpServerPO ↓ 用户发起对话 └─ McpHandler 读取 McpServerPO，过滤 disabledMcpServerIds ↓ LlmCallHandler └─ ChatDispatchRequest.mcpServers ↓ ToolRuntime └─ ClientMcpToolResolver 创建 McpSyncClient ↓ Spring AI └─ SyncMcpToolCallbackProvider 暴露工具给 ToolCallingAdvisor ↓ completion 或 error └─ ToolResolution.close 关闭 MCP client

9. 易错点

现象优先检查判断依据
Agent 配了 Skill 但模型不知道skillEnabled、sai_agent_skill、disabledSkillIds、ClientSkillToolResolver 日志Server 需要生成 SkillDescriptor，Client 需要追加 Skill 列表并注册 read_skill。
read_skill 找不到 Skill传入名称是否与 Skill 列表完全一致ReadSkillTool 用 Skill 名称做 map key，错误时返回可用名称列表。
Skill 支撑文件不可见SkillContentCallbackHandler 返回的 files、Client 临时目录、异步物化日志正文同步返回，支撑文件在虚拟线程中异步写入。
MCP SSE 不生效McpTransportTypeEnum.SSE 与 resolver 日志当前 SDK 2.x 下 SSE 在 Admin 测试中报错，在 Client resolver 中告警后跳过。
MCP 认证配置没生效ClientMcpToolResolver 的 transport 构建代码描述符包含 authType、authConfig，但当前 Client resolver 没把它写入 transport。
对话中某个 MCP/Skill 消失请求体的 disabledMcpServerIds、disabledSkillIds禁用 ID 只影响当前对话，不删除绑定关系。
工具集合里没有 RAG MCPToolRuntime.filterRagMcpDescriptor名为 SystemConstants.RAG_MCP_SERVER_NAME 的 MCP 描述符会被移除，RAG 由专用 resolver 处理。
自定义 @Tool 重复或缺失CustomToolCallbackProvider.afterPropertiesSet扫描发生在启动后，且排除 com.aizuda.snail.ai.agent.core.tool 内部工具包。

10. 排查路线

问题类型最短源码路线
Admin 保存失败SkillController 或 McpServerController -> 对应 Service -> PO/Mapper。
Agent 绑定没生效AgentRequestVO -> AgentService.create/update -> updateAgentSkills 或 updateAgentMcpServers。
Server 没下发描述符AgentChatService -> AgentChatContext -> McpHandler 或 SkillAgentChatHandler -> LlmCallHandler。
Client 没工具ChatSessionRuntime -> ToolRuntime -> 具体 resolver -> DefaultChatClientFactory。
Skill 内容回调失败ReadSkillTool -> RpcClient.fetchSkillContent -> UriConstants.CALLBACK_SKILL_CONTENT -> SkillContentCallbackHandler。
MCP 连接失败McpServerService.testConnection 与 ClientMcpToolResolver.connectMcpServer 对照，看 Admin 测试和 Client 实际连接是否一致。

测试空白：当前未发现这些核心类的直接单元测试。做 L2/L3 深挖时，建议先补 ClientSkillToolResolver、ReadSkillTool、ClientMcpToolResolver、ToolRuntime 的窄单元测试，再做端到端集成验证。
