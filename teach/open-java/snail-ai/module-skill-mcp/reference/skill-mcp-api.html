Skill/MCP 模块构建 API

本页是 L3 微观构建字典，聚焦 snail-ai-feature-skill 的 Maven 坐标、直接依赖，以及它与 slice-mcp-skill-tools 运行时各阶段的边界。skill-mcp-overview.html 负责全链路地图；本文只回答「这个 jar 包里有什么、被谁依赖、在工具注入链路里承担哪一段」。

层级：L3 微观 API 源码 commit：2ae56c0 覆盖：1 个 pom + 2 个 Java 类

2. 直接依赖与职责

源文件：snail-ai-feature-skill/pom.xml

依赖坐标版本来源在本模块中的实际用途运行时落点
com.aizuda:snail-ai-commons-core ${revision} SkillContentRequest、JsonUtil、MCP 枚举等公共 DTO/工具 SkillContentCallbackHandler 解析回调 body
com.aizuda:snail-ai-commons-grpc ${revision} GrpcRequestHandler、UriConstants、GrpcSnailAiResult SkillContentCallbackHandler 注册 /callback/skill/content
com.aizuda:snail-ai-biz-template ${revision} SkillMapper、SkillFileMapper、AgentSkillMapper 及 PO SkillHandler 按 Agent 绑定读 SkillPO；回调 Handler 读正文与支撑文件
org.springframework.ai:spring-ai-mcp ${spring-ai.version} → 2.0.0 当前源码未直接使用；MCP Client 在 feature-model 与 Agent Client resolver 中引用 见下文「MCP 运行时不在本 jar」
org.springframework.ai:spring-ai-client-chat ${spring-ai.version} → 2.0.0 当前源码未直接使用；工具注入在 Agent Client ToolRuntime 完成 见 L2 切片 Client 段
org.projectlombok:lombok ${lombok.version} 编译期注解；provided 作用域 —

依赖声明 vs 实际代码：pom.xml 声明了 Spring AI MCP 栈，但 snail-ai-feature-skill 目录下没有任何 import org.springframework.ai 或 io.modelcontextprotocol。MCP 连接、SyncMcpToolCallbackProvider 和 read_skill 工具注册分别落在 snail-ai-feature-model（Admin 测试）、snail-ai-feature-agent（描述符生成）和 snail-ai-agent-executor-core（Client resolver）。不要把本 jar 误当成 MCP 运行时模块。

3. 谁依赖本模块

消费方依赖方式使用的公共 API
snail-ai-feature-agent 直接 Maven 依赖 注入 SkillHandler，在 SkillAgentChatHandler @Order(70) 中读取 Agent 绑定 Skill 并生成 SkillDescriptor
snail-ai-server-admin 直接 Maven 依赖 随 Spring Boot 组件扫描加载 SkillContentCallbackHandler；Admin CRUD 本身在 SkillService（admin 包），不在本 jar
snail-ai-starter（可执行 Server） 传递依赖自 admin 启动时注册 gRPC 回调路由

4. 本模块公开 API

4.1 SkillHandler — Server 侧 Skill 读取

snail-ai-feature-skill/src/main/java/com/aizuda/snail/ai/features/skill/handle/SkillHandler.java

方法签名入参返回 / 副作用调用方
getSkillsWithContentForAgent List<SkillPO> getSkillsWithContentForAgent(Long agentId) Agent 主键 按 AgentSkillPO 关联查 SkillPO 全量列表（含 skillContent）；无绑定时返回空列表 SkillAgentChatHandler

4.2 SkillContentCallbackHandler — Client 回调读正文

snail-ai-feature-skill/src/main/java/com/aizuda/snail/ai/features/skill/handle/SkillContentCallbackHandler.java

契约项值
接口GrpcRequestHandler
URIUriConstants.CALLBACK_SKILL_CONTENT → /callback/skill/content
请求 DTOSkillContentRequest：skillId（必填）、agentId（可选，当前 Handler 未使用）
成功响应 JSONskillContent、version、files[]（每项含 filePath、content、encoding）
失败GrpcSnailAiResult.status=0；缺 skillId 或 Skill 不存在时返回错误消息
Client 触发ReadSkillTool → RpcClient.fetchSkillContent

5. 与 slice-mcp-skill-tools 运行时的关系

L2 切片 mcp-skill-tools-flow-map.html 描述一次对话中 MCP 与 Skill 工具从绑定到 cleanup 的完整链路。下表把各阶段映射到具体 Maven 模块，并标出 snail-ai-feature-skill 的参与点。

运行时阶段责任模块关键类feature-skill 是否参与
Admin 保存 Skill / MCP 配置 snail-ai-server-admin SkillService、McpServerService 否（仅被 admin 依赖以注册回调 Bean）
Agent 绑定写入关联表 snail-ai-server-admin AgentService.updateAgentSkills / updateAgentMcpServers 否
对话时 MCP 描述符生成 snail-ai-feature-agent + snail-ai-feature-model McpHandler、McpToolService 否；MCP 读取走 McpToolService.getMcpServersForAgent
对话时 Skill 描述符生成 snail-ai-feature-agent + 本模块 SkillAgentChatHandler → SkillHandler 是：提供绑定 Skill 的 SkillPO 列表
组装 ChatDispatchRequest snail-ai-feature-agent LlmCallHandler 间接：消费上一步写入的 skillDescriptors
Client MCP 工具连接 snail-ai-agent-executor-core ClientMcpToolResolver 否；使用 spring-ai-mcp 在 Client 侧
Client Skill 工具注册 snail-ai-agent-executor-core ClientSkillToolResolver、ReadSkillTool 否；仅消费描述符，不依赖本 jar
read_skill 回调拉正文 本模块 + commons-grpc SkillContentCallbackHandler 是：Server 侧唯一 Skill 运行时回调
工具汇总与 MCP cleanup snail-ai-agent-executor-core ToolRuntime、ChatSessionRuntime 否
slice-mcp-skill-tools 主链路（模块视角） Admin (skill/mcp CRUD) └─ feature-skill：不参与 ↓ Agent 绑定 └─ feature-skill：不参与 ↓ Server Chain ├─ McpHandler ← feature-model (McpToolService) └─ SkillAgentChatHandler ← feature-skill (SkillHandler) ★ ↓ LlmCallHandler → ChatDispatchRequest ↓ Client ToolRuntime ├─ ClientMcpToolResolver (spring-ai-mcp) └─ ClientSkillToolResolver + ReadSkillTool ↓ RPC SkillContentCallbackHandler ← feature-skill ★

6. 构建与排查速查

问题先查哪里判断依据
Skill 描述符为空SkillHandler.getSkillsWithContentForAgent 上游：skillEnabled、AgentSkillPO本模块只读 DB，不负责绑定写入
read_skill 回调 404 / 无 HandlerServer 是否引入 snail-ai-feature-skill 且扫描到 SkillContentCallbackHandler缺依赖则 gRPC 路由无 /callback/skill/content 处理器
MCP 工具缺失McpHandler、ClientMcpToolResolver与 feature-skill 无关；勿在本模块找 MCP 代码
想给 feature-skill 加 MCP 逻辑对照 feature-model/McpToolService 与 feature-agent/McpHandler现有分层把 MCP 放在 model + agent，Skill 基础设施保持薄层
spring-ai-mcp 为何在 pom 里git history / 后续扩展当前 2 个 Java 类均未引用；可能是预留或历史遗留，改动前用 mvn dependency:analyze 验证

官方 architecture.md 把 feature-skill 描述为「技能管理（SkillService、McpServerService）」——与当前源码不一致：CRUD 在 admin，MCP 服务在 feature-model，本 jar 仅提供 Server 运行时读取与回调。以本页与源码为准。
