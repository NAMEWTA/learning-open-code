2. 请求 DTO 字段去向

OpenApiChatRequest 字段校验当前去向排查含义
agentId@NotNullAgentChatCommand.agentId，后续 InitContextHandler 加载 Agent。Agent 不存在会在责任链早期写错误并终止。
openId@NotBlankOpenApiUserResolver 用 appId + openId 解析平台 UserPO，同时写入 AgentChatCommand.openId。OpenId 无效 属于认证类异常，返回 status=5001。
conversationId@NotBlankAgentChatCommand.conversationId，用于会话记录、短期历史、路由选择和最终 done。同一会话复用会影响短期历史。
content@NotBlankAgentChatCommand.content，最终成为 ChatDispatchRequest.userMessage。Client DefaultPromptFactory 会把它追加为最后一个 UserMessage。
disabledMcpServerIds无McpHandler 过滤 MCP server 后生成 mcpServers descriptor。仅当 Agent 启用 MCP 且绑定 server 时可见效果。
disabledSkillIds无SkillAgentChatHandler 过滤 skill 后生成 skills descriptor。仅当 Agent 启用 Skill 且有 skill 内容时可见效果。
sid无为空时服务端生成 UUID，进入 AgentChatCommand.sid、ChatDispatchRequest.sid、回流 ChatStreamResponse.sid。用于区分同一用户同一会话下的并发提交。
deepPlanEnabled无当前 OpenApiChatService 未消费。传入不会影响当前 chat 链路。
webSearchEnabled无当前 OpenApiChatService 未消费；WebSearchHandler 在当前源码中也未注册 @Component。传入不会启用 web search。
timeout无当前 OpenApiChatService 未消费；同步等待固定 300_000ms。同步超时不能通过请求体字段调整。

5. Writer 行为与响应差异

场景writer转发内容完成条件边界
OpenAPI 流式 FluxBridgeChatStreamWriter，OpenApiChatService 内部类。 能解析为 ChatStreamResponse 的 text 会累积 content 并发 event: text；thinking 发 event: thinking；其他已知/未知类型按原 type 发出。 complete() 时发一次 event: done，data 为 {"conversationId": "...", "fullText": "..."}，然后 complete Flux。 parse 失败的非 JSON chunk 被忽略；completion chunk 不直接发给 OpenAPI，而是触发 Server 侧持久化后由 gRPC completion 转成 done。
OpenAPI 同步 CollectingChatStreamWriter。 能解析为 text 的 chunk 追加 content；无法解析的非 JSON data 原样追加；thinking 和 completion 不追加。 complete() 或 completeWithError() 释放 latch；等待超过 300 秒抛 SnailAiException("对话响应超时")。 同步响应只有 conversationId、content、durationMs，不返回 token usage、thinking 或 cacheTokens。
Client gRPC 回流 ChatStreamObserver。 text/thinking 转成 JSON 后写入上游 writer；completion 用于持久化；error 写纯文本 [ERROR] ...。 Client observer.onCompleted() 后触发 writer.complete。 error chunk 不是 OpenAPI JSON error event；流式侧可能只在服务层 catch 异常时看到 event: error。

7. 错误码和异常边界

问题当前返回 / 行为优先查哪里
缺少认证头SnailAiAuthenticationException，OpenAPI exception handler 返回 Result.fail(5001, message)。OpenApiAuthInterceptor#preHandle、OpenApiExceptionHandler#onAuthException。
App 不存在、禁用、token 错误同为 status=5001，message 分别说明应用认证失败、应用已禁用或 token 验证失败。AppPO.status、缓存过期时间、header 是否写反。
openId 无效OpenApiUserResolver 抛认证异常，返回 status=5001。open_api_user 映射和 platformUserId。
参数缺失MethodArgumentNotValidException 或 BindException，返回 Result.fail("字段 message")，默认 status=0。OpenApiChatRequest 上的 @NotNull、@NotBlank。
没有可用 ClientContextCollectorHandler 写纯文本 [ERROR] 没有可用的客户端实例... 并 complete；流式 OpenAPI bridge 不会把非 JSON chunk 转为 error event。Agent 的 appId、Client 心跳、ClientInstanceManager#getAliveInstances。
Client 模型执行失败Client 发送 ChatStreamResponse.error(sid, "LLM_FAILED", message)；Server ChatStreamObserver 记录并写纯文本 [ERROR] ...。ChatDispatchStreamingHandler#handleError、ChatStreamObserver#handleError。
同步等待超时CollectingChatStreamWriter#awaitAndGetFullText 超时，服务层抛 SnailAiException("对话响应超时")。Client 是否 onCompleted、模型是否一直无 completion、同步固定 300 秒。

9. 调试 checklist

症状检查顺序
HTTP 直接失败确认 URL 是否包含 /snail-ai context path；确认两个 OpenAPI header；看 OpenApiExceptionHandler 返回的 status/message。
认证通过但用户失败确认 OpenApiSessionUtils.current().appId 对应的 openId 映射存在，且 platformUserId 指向真实 UserPO。
Agent 链没有输出按 Handler 顺序查 terminated：Agent 是否存在、模型配置是否可组装、Agent 是否配置 appId、是否有 alive Client。
SSE 有 text 但没有最终内容看 Client 是否发送 completion 并 observer.onCompleted()；OpenAPI done 来自 writer.complete，不来自 completion event。
SSE 只看到 done 或空 fullText排查上游是否写了纯文本 Error: 或 [ERROR]；流式 bridge parse 失败会忽略非 JSON chunk。
同步一直等待确认 CollectingChatStreamWriter.complete() 是否被调用；同步接口不使用请求体 timeout，固定 300 秒。
usage 没有记录看模型最终流式 chunk 是否包含 Spring AI Usage，再看 TokenUsageCollectorAdvisor 日志和 ChatStreamObserver#handleCompletion。
SDK 调用异常SDK 的 OpenApiHttpInvokeHandler 会自动加认证头；普通 HTTP status >= 400 抛异常，SSE status >= 400 会发 OpenApiChatStreamEvent.error("HTTP ...")。
