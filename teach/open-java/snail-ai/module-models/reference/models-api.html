OpenAiChatModel 扩展点

项目将 Spring AI 的 OpenAiChatModel 放在 chat-core/.../org/springframework/ai/openai/OpenAiChatModel.java，供 OpenAiCompatibleChatModelAdapter 通过 OpenAiChatModel.builder().options(...).build() 创建。相对上游 Spring AI，Snail AI 重点扩展了兼容性与流式合并。

构建入口

API说明
OpenAiChatModel.builder()Builder 模式；可注入 OpenAiChatOptions、ToolCallingManager、ObservationRegistry。
OpenAiCompatibleChatModelAdapter.buildOptions把 ChatModelSpec + ConfigExtAttrsDTO 映射为 OpenAiChatOptions：baseUrl、apiKey、model、timeout、temperature、topP、topK、maxTokens、penalty、stop、seed、extraBody。
默认超时configJson.timeoutMs 缺失或 ≤0 时使用 300_000 ms。

请求体扩展：extraBody

在 createRequest 路径中，若 OpenAiChatOptions.extraBody 非空，会转为 additionalBodyProperties 写入 OpenAI Java SDK 请求——用于 DeepSeek、Qwen 等 OpenAI-compatible 厂商的专有字段（如 enable_thinking）。配置来源：ConfigExtAttrsDTO.extraBody → adapter → options → 请求体。

流式 ChunkMerger（内部类）

方法职责
hasToolCall / toolCallsDone检测流式 chunk 是否处于 tool call 片段。
mergeChunks合并同一逻辑响应的多个 ChatCompletionChunk。
mergeToolCalls按 index 合并 tool call 参数片段；兼容 DeepSeek 等 provider 在续传 delta 上发送空字符串 id 的行为。
chunkToChatCompletion将最终 chunk 转为 ChatCompletion；保留 delta 的 _additionalProperties（如 reasoning_content）。

测试锚点：OpenAiChatModelChunkMergerTest 验证 reasoning_content 在 chunkToChatCompletion 后仍存在于 message additional properties——推理模型场景排查时可对照此测试。
