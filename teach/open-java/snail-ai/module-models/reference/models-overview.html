模型适配层参考

一句话：snail-ai-models 是 Chat、Embedding、Rerank 三类模型能力的适配层：它把业务配置转成 Spec，由 Runtime 按 adapterKey 选择 Adapter，再创建具体 SDK 模型或 HTTP 客户端。返回 短课、API 速查 或 L0 项目地图。

源码优先：当前内置 Chat/Embedding provider 都是 openai-compatible，Rerank provider 是 qwen-rerank。providerKey 会保存在 spec 中，但当前内置 provider 的选择键是 adapterKey。

错误与扩展检查表

场景当前行为排查入口
缺少 adapterKeyModelAdapterDefaults.resolve 按模型类型回退默认值。ModelAdapterDefaults
缺少 baseUrl、apiKey 或 modelKeyruntime 抛出 INVALID_MODEL_CONFIG。*ModelRuntime.validate
没有匹配 adapterruntime 抛出 ADAPTER_NOT_FOUND。adapter 的 adapterKey() 与 starter Bean 注册
provider 构建失败非 ModelAdapterException 会被包装为 MODEL_BUILD_FAILED。具体 provider 的 create(spec)
Rerank 空输入documents 为 null 或空列表时，Qwen HTTP client 直接返回空列表。QwenRerankModelAdapter.HttpRerankApiClient.rerank
Rerank 响应无可用结果响应 body 为 null、含 error 字段、无 results 或结果为空时，Qwen HTTP client 记录日志并返回空列表；RAG 侧看到空结果后按原分数排序降级。QwenRerankModelAdapter.HttpRerankApiClient、DefaultRerankService
Rerank HTTP/网络异常adapter 不捕获该异常；异常向外抛出，经 ModelRuntimeHandler 包装后由 DefaultRerankService 捕获并降级。ModelRuntimeHandler.rerank、DefaultRerankService.rerank

新增 provider 最小步骤 实现对应 *ModelAdapter，返回唯一 adapterKey()；把实现注册为 Spring Bean；注册同能力的 ModelAdapterDescriptor；必要时扩展 ModelAdapterDefaults 的默认值；最后用 Server 或 Agent 侧 spec factory 做一次端到端构建验证。
