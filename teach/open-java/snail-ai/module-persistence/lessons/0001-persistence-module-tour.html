1. 职责：把三类存储隔开

在 L0 项目地图 里，persistence 是 Server 能力落地到外部系统的适配层。它不是一个单独业务流程，而是三类存储边界：业务库保存用户、模型配置、Agent、RAG 文档、chunk 和存储实例；向量存储保存可相似度检索的 chunk 或记忆；搜索存储保存 BM25 全文索引。RAG L1 已经讲过摄入和检索依赖 vector/search 后端，本课只解释这些后端怎样被抽象出来。

2. 位置：Server Persistence 的三块

snail-ai-server-persistence/ ├── snail-ai-biz-storage/ ← MyBatis-Plus 业务库 PO、Mapper、方言 XML ├── snail-ai-vector-storage/ ← SnailAiVectorStore 模板与 PgVector/Milvus/ES 实现 └── snail-ai-search-storage/ ← SearchEngine 模板与 Elasticsearch 实现

业务库模板里有共享 PO 和 Mapper，MySQL、PostgreSQL、达梦模块只补少量方言 XML。向量库模板暴露 SnailAiVectorStore、VectorStoreFactory、IndexNameBuilder 和请求/结果 DTO；三个实现模块在生命周期启动时注册 PgVector、Milvus、Elasticsearch 构造器。搜索库更窄：模板暴露 SearchEngine 和 SearchEngineFactory，当前实现只有 Elasticsearch。

3. 后端选择：StoreInstance 是开关

StoreInstanceController
  ↓
StoreInstanceService → sai_store_instance(category, type, config)
  ↓
RagPO.vectorStoreInstanceId / searchEngineInstanceId
  ↓
VectorStoreFactory / SearchEngineFactory
  ↓
PgVector / Milvus / Elasticsearch

sai_store_instance 的 category 区分向量库和搜索引擎，type 区分 PG_VECTOR、MILVUS、ELASTICSEARCH、PG_FULLTEXT，config 保存连接 JSON。创建知识库时，KnowledgeService.create 把 vectorStoreInstanceId 写入 RagPO；若启用搜索引擎，再写 searchEngineInstanceId。后续摄入时 DocumentPipeline 用这些 ID 双写向量库和搜索引擎，检索时 VectorSearchHandler 与 Bm25SearchHandler 再用相同 ID 读回来。

4. 最小示例：一个知识库选 PgVector + ES

POST /store-instance  →  category=1,type=1,config={PgVector连接}
POST /store-instance  →  category=2,type=3,config={Elasticsearch连接}
POST /rag             →  vectorStoreInstanceId=1,
                         searchEngineEnable=true,
                         searchEngineInstanceId=2

这个例子说明 Snail AI 的后端切换主要是运行期配置，不是把 RAG 代码改成某个数据库专用实现。真正的注意点有两个：向量库还要匹配 embedding 模型维度，VectorDimensionConstraintService 会按模型和后端取有效上限；搜索侧虽然枚举里有 PG_FULLTEXT，但当前 SearchEngineEnum 只注册 Elasticsearch。完整清单放在 持久化参考。

判断练习：不看上文，判断三个文件应该归到哪一层：RagMapper.java、MilvusVectorLifecycle.java、ElasticsearchSearchEngine.java。
参考答案：RagMapper.java 属于业务库模板；MilvusVectorLifecycle.java 属于向量库实现注册；ElasticsearchSearchEngine.java 属于全文搜索实现。若把后两者都叫“RAG 模块”，说明还需要把业务功能和存储适配拆开看。

主要推荐来源：先读 snail-ai-server-persistence/pom.xml 和 StoreInstanceController.java，再用 持久化参考 对照接口清单。遇到某个后端实现不清楚时，继续向 agent 追问对应 L2/L3 切片。
