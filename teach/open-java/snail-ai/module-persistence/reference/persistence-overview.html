持久化与存储适配模块参考

一句话：persistence 模块把 Snail AI 的业务数据、向量索引和全文索引分成三套适配边界；业务功能通过 mapper、SnailAiVectorStore 和 SearchEngine 编程，而不是直接依赖具体数据库客户端。

源码优先：StoreInstanceTypeEnum 中存在 PG_FULLTEXT，VectorStoreType 中存在 MONGO，但当前模块只看到 PgVector、Milvus、Elasticsearch 三种向量实现，以及 Elasticsearch 一种搜索实现。

常见判断

问题判断
看到 Mapper 或 PO 应归哪层？通常归业务库模板；再看是否有方言 XML 补 SQL 差异。
看到 VectorStoreFactory.REGISTER 应归哪层？向量存储后端注册；具体类型由 lifecycle 在启动阶段放入注册表。
看到 SearchEngineFactory.REGISTER 应归哪层？全文搜索后端注册；当前只有 Elasticsearch。
RAG 选择后端发生在哪里？Admin 创建知识库时保存实例 ID；摄入和检索阶段再按 ID 创建具体后端。
PgVector 和 PostgreSQL 业务库是不是同一个概念？不是。业务库保存配置和文档元数据；PgVector 是向量索引后端，可以与业务库同实例部署，也可以独立配置。

后续 L2 候选 store-instance 管理切片、RAG 摄入双写切片、向量后端实现切片、搜索后端实现切片、业务库方言切片。
