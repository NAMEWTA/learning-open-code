# 使命：记忆功能模块

## 为什么
学习者需要理解 Snail AI 如何把对话历史放回下一次模型调用中，并能判断当前源码中哪些是已经落地的短期记忆路径，哪些是长期记忆的架构契约。掌握这个边界后，后续排查上下文丢失、窗口过大、记忆未注入等问题时不会把责任误归到模型层。

## 成功的样子
- 能说清 `snail-ai-feature-memory` 在 Server 侧架构中的位置，以及它和 Agent 责任链的关系。
- 能定位 `ShortTermMemoryStore` 的两个实现，并解释 `memory` 与 `db` 存储模式的取舍。
- 能区分短期历史、长期记忆 DTO、记忆枚举和 `ChatDispatchRequest.memoryContext` 各自承担的边界。

## 约束条件
- 本主题是 L1 模块导览，只讲模块职责、分层、公共类清单和一个调用点。
- 课程控制在 15 分钟内完成，长表格和源码索引放在 reference。

## 不在范围内
- 不展开完整长期记忆抽取、向量化、召回和压缩切片。
- 不讲 Admin 记忆管理页面和所有 VO 的逐字段行为。
