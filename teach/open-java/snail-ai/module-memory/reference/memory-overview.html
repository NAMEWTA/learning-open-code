公共类与接口清单

类/接口所在模块职责运行边界
ConversationMemoryDTOfeature-memory/dto长期记忆条目的传输对象，包含 agent、user、conversation、type、content、tags、vectorId、score、status、accessCount 等字段。长期记忆契约；Admin VO 会引用。
MemoryMetricsfeature-memory/dto记录抽取耗时、候选数量、新增、更新、删除、跳过数量。记忆操作统计契约。
MemoryOperationItemfeature-memory/dto单条记忆操作结果，包含 memoryId、vectorId 和 memory 文本。记忆新增/更新结果契约。
ShortTermHistoryQueryfeature-memory/dto短期历史查询参数，包含 conversationId、agentId、userId、shortTermMemorySize。ShortTermMemoryStore.loadHistory 入参。
ShortTermMessagefeature-memory/dto短期历史消息，只有 role 和 content。store 与 Agent 链之间的轻量消息模型。
ShortTermMemoryStorefeature-memory/store短期记忆端口：append、loadHistory、evict、storeType。Agent 链依赖接口，不依赖具体 store。
InMemoryShortTermMemoryStorefeature-memory/store用 ConcurrentHashMap + LinkedList 维护每个 conversation 的有界队列，cache miss 时从 DB 回填。snail-ai.memory.short-term.store-type=memory 时激活。
DbShortTermMemoryStorefeature-memory/store每次从 AgentConversationRecordMapper 按 agent、conversation、user 查询最近 N 条记录，再倒序为时间正序。store-type=db 或缺省匹配时激活。
MemoryRetrieveRequestcommons-core/dto/memory长期记忆召回请求，包含 agentId、userId、conversationId、query、memoryConfigId、modelId、topK。跨模块公共契约；当前 L1 不证明完整链路。
ShortTermMemoryRequestcommons-core/dto/memory加载短期对话历史请求，包含 conversationId 和 windowSize。RpcClient.loadShortTermHistory 的公共 DTO。

短期记忆调用路径

ContextCollectorHandler(@Order 75) 位于 SkillAgentChatHandler(@Order 70) 之后、LlmCallHandler(@Order 80) 之前运行。此时 MCP、RAG、Skill 等上下文已经由前序 Handler 放入 AgentChatContext，它在真正分发前收集目标 Client 和短期历史。

ConversationHandler(@Order 20)
  └─ 插入当前 user 对话记录，作为 DB store 后续读取来源

McpHandler(@Order 40) / RagHandler(@Order 60) / SkillAgentChatHandler(@Order 70)
  └─ MCP、RAG、Skill 上下文就绪

ContextCollectorHandler(@Order 75).handle(ctx)
  ├─ 校验 appId、App 状态、可用 Client
  ├─ 根据 agent.shortTermMemorySize 计算窗口，默认 20
  ├─ 若 memoryEnabled 不是 false
  │   ├─ shortTermMemoryStore.append(conversationId, "user", content, window)
  │   │   ├─ memory 模式：写入 JVM 队列
  │   │   └─ db 模式：空实现，依赖 ConversationHandler 已写的对话记录
  │   └─ shortTermMemoryStore.loadHistory(query, window)
  └─ ctx.setHistoryMessages(...)

LlmCallHandler(@Order 80).buildDispatchRequest(ctx)
  ├─ 写入 historyMessages
  └─ 保留 memoryContext 字段

ChatStreamObserver.handleCompletion(data)
  └─ ChatResultPersistService.persistAsync(...)
      ├─ 插入 assistant 对话记录
      └─ shortTermMemoryStore.append(conversationId, "assistant", fullText, window)

store 实现对比

实现激活条件读路径写路径适用场景
InMemoryShortTermMemoryStoresnail-ai.memory.short-term.store-type=memory优先读 JVM 内存；无数据时按 agent、conversation、user 从 DB 取最近 N 条并回填。追加到当前节点内存队列，超过窗口删除最早消息。单机、本地开发、低延迟场景。
DbShortTermMemoryStorestore-type=db 或配置缺省每次从 AgentConversationRecordMapper 查询最近 N 条，再反转为时间正序。append 当前为空实现；当前用户消息依赖 ConversationHandler 已写入的对话记录，assistant 消息依赖完成后的结果持久化链路。多实例部署、需要跨节点共享历史的场景。
