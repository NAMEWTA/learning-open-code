Server API 入口模块导览

这一课只做一件事：建立 Server API 入口层的边界感。学完后，你应能判断请求是 Admin API 还是 OpenAPI、该带哪组认证头、返回 JSON 还是 SSE，并知道下一步该从哪个 Controller 继续追踪。

学习目标：识别入口边界 预计 12 分钟 主来源：open-java/snail-ai/snail-ai-server/snail-ai-server-admin/src/main/java/com/aizuda/snail/ai/admin/interceptor/AuthenticationInterceptor.java open-java/snail-ai/snail-ai-server/snail-ai-server-openapi/src/main/java/com/aizuda/snail/ai/openapi/interceptor/OpenApiAuthInterceptor.java open-java/snail-ai/snail-ai-server/snail-ai-server-openapi/src/main/java/com/aizuda/snail/ai/openapi/controller/OpenApiChatController.java

1. 入口层负责什么

在 L0 总览里，请求从 Admin 或 OpenAPI 进入 Server，然后才进入 Agent 责任链、RAG 或其他业务服务。本课只看入口层：Controller 负责接收 HTTP 参数、选择同步或流式响应；Interceptor 负责认证和把当前身份放进 ThreadLocal；Service 才开始做业务编排。源码上，Server 聚合模块下有两个入口子模块：snail-ai-server-admin 面向管理后台和内部配置操作，snail-ai-server-openapi 面向第三方系统调用。外部 URL 还会叠加 application.yml 中的 /snail-ai context path，所以 Java 常量里的 /openapi/v1 对外通常表现为 /snail-ai/openapi/v1。

snail-ai-server/ ├── snail-ai-server-admin/ ← 管理后台 REST/SSE 入口 │ ├── controller/ ← Agent、RAG、模型、用户、应用等 Controller │ └── interceptor/ ← Admin 登录态认证 └── snail-ai-server-openapi/ ← 第三方系统 OpenAPI 入口 ├── controller/ ← 用户、智能体、会话、对话、嵌入 Token ├── interceptor/ ← App ID + Token 鉴权 ├── stream/ ← 同步对话收集器 └── config/ ← OpenAPI MVC 拦截器注册

2. 两套认证不要混用

Admin API 使用 Snail-Ai-Auth。它由 AuthenticationInterceptor 读取，先看目标方法是否带 @LoginRequired，再解析 JWT、查询用户、校验角色，并把用户写入 UserSessionUtils；请求结束后清理 ThreadLocal。注意：Admin 拦截器挂在 /**，但排除 /openapi/** 和 /files/**，且真正强制登录依赖方法注解。

OpenAPI 使用 Snail-Ai-App-Id 与 Snail-Ai-Token。OpenApiAuthInterceptor 会查应用、要求状态为启用、用常量时间比较 Token，再写入 OpenApiSessionUtils。它由 MVC 配置注册到 /openapi/**，因此第三方系统不要把 Admin 的 Snail-Ai-Auth 拿来调用 OpenAPI。

3. 一个请求入口示例

以 OpenAPI 流式对话为例，入口是 POST /snail-ai/openapi/v1/agent/chat。请求先过 App 鉴权，再到 OpenApiChatController#chat，Controller 返回 Flux<ServerSentEvent<String>>，把服务层产生的事件类型映射成 SSE event。服务层会解析外部用户、构造 Agent 对话命令，并通过流式 writer 把下游 chunk 送回 HTTP；责任链内部怎么补上下文，本课先不展开。

curl -N -X POST http://localhost:8900/snail-ai/openapi/v1/agent/chat
  -H "Content-Type: application/json"
  -H "Snail-Ai-App-Id: your-app-id"
  -H "Snail-Ai-Token: your-app-token"
  -d '{"agentId":1,"openId":"external-user-001","conversationId":"conv-001","content":"你好"}'

同步对话走 /agent/chat/sync，边界不同：它用 CollectingChatStreamWriter 等待流完成并拼接完整文本，再一次性返回 Result<OpenApiChatSyncResponse>。因此调试响应问题时，先判断接口是“边推边返回”还是“收集后返回”。完整接口清单见 Server API 入口参考，上游定位可回看 项目总览课。
