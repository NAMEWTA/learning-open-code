Server API 入口模块参考

一句话定位：Server API 入口层把 HTTP 请求划分为 Admin API 与 OpenAPI 两类，完成认证、参数绑定、响应类型选择，再把请求交给 Admin/OpenAPI service 或 Agent 对话服务。

模块结构与关键依赖

snail-ai-server/ ├── pom.xml ← 聚合 Persistence、Features、Admin、OpenAPI ├── snail-ai-server-admin/ ← 管理后台入口模块 │ ├── controller/ ← 13 个 Controller │ ├── interceptor/ ← 登录态拦截器与 MVC 注册 │ └── pom.xml ← 依赖 Feature、Persistence、Web、JWT └── snail-ai-server-openapi/ ← 外部集成入口模块 ├── controller/ ← 5 个 OpenAPI Controller ├── interceptor/ ← App 鉴权拦截器 ├── stream/ ← 同步响应收集器 └── pom.xml ← 依赖 server-admin，复用对话服务能力
模块入口职责主要依赖方向
snail-ai-server-admin管理后台用户、应用、模型、智能体、RAG、资源、Skill、MCP 的 REST/SSE 入口。依赖 commons-core、feature-agent/model/memory/rag/skill/resource、commons-grpc、MyBatis-Plus、Spring Web、java-jwt。
snail-ai-server-openapi第三方系统调用入口，包括用户同步、智能体查询、会话管理、流式/同步对话和嵌入 Token。依赖 snail-ai-server-admin 与 java-jwt，复用 Admin 侧 Agent 对话服务和持久化能力。

认证与路径边界

入口外部 URL 范围Header认证行为会话存放
Admin API/snail-ai/**，但 MVC 注册排除 /snail-ai/openapi/** 与 /snail-ai/files/**Snail-Ai-Auth拦截器命中后检查目标方法是否带 @LoginRequired；带注解才解析 JWT、查用户、按 RoleEnum 校验角色。UserSessionUtils
OpenAPI/snail-ai/openapi/**；源码注册路径是 context path 内的 /openapi/**Snail-Ai-App-Id
Snail-Ai-Token		每个 OpenAPI 请求都查 App，要求状态为启用，使用 MessageDigest.isEqual 比较 Token，并缓存 App 5 分钟。OpenApiSessionUtils
文件访问/snail-ai/files/**不走 Admin 拦截器由 FileAccessController 自己处理访问逻辑；不要把它误判成普通 Admin 受保护接口。无统一入口会话

易错点：Snail-Ai-Auth 是 Admin API 和智能体对话会话 Token 的 Header，不是 OpenAPI 外部集成的应用凭证。第三方系统调用 OpenAPI 必须使用 Snail-Ai-App-Id 与 Snail-Ai-Token。

流式与同步响应边界

入口返回类型边界说明
POST /agent/{id}/chatFlux<String>，MediaType.TEXT_EVENT_STREAM_VALUEAdmin 智能体对话。Controller 内部桥接 ChatStreamWriter 到 FluxSink<String>，把下游 chunk 推给浏览器。
POST /agent/create/streamFlux<String>Admin 智能体创建流，使用 AgentCreateStreamHttpAdapter.MEDIA_TYPE 编码。
POST /rag/qa/streamFlux<String>Admin RAG 问答流，由 RAG QA 适配器编码输出。
POST /openapi/v1/agent/chatFlux<ServerSentEvent<String>>OpenAPI 流式对话。事件类型来自 OpenApiChatStreamEvent.type，数据来自 data 字段。
POST /openapi/v1/agent/chat/syncResult<OpenApiChatSyncResponse>OpenAPI 同步对话。CollectingChatStreamWriter 使用 CountDownLatch 等待完成，默认最多等待 300000 ms。

请求入口示例

POST /snail-ai/openapi/v1/agent/chat
  1. DispatcherServlet 接收请求
  2. OpenApiWebMvcConfiguration 匹配 /openapi/**
  3. OpenApiAuthInterceptor 校验 Snail-Ai-App-Id / Snail-Ai-Token
  4. OpenApiChatController#chat 绑定 OpenApiChatRequest
  5. OpenApiChatService#chatStream 解析 openId 并构造 AgentChatCommand
  6. FluxBridgeChatStreamWriter 把下游 chunk 转成 OpenAPI 事件
  7. Controller 转成 ServerSentEvent 返回给调用方

源码中的 OpenApiChatRequest 对 agentId、openId、conversationId 和 content 都有必填校验；编写调用示例或测试时不要只参考旧文档片段，应以当前 DTO 为准。

下一步追踪 需要完整端点与 VO 字段时查阅 L3 API 速查；从 Controller 进入 service 后，再转向 Agent 责任链或 RAG 主题。
