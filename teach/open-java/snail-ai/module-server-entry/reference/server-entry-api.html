Server Admin/OpenAPI 对外 API 参考

L3 速查：按业务域列出 snail-ai-server-admin 与 snail-ai-server-openapi 的全部 REST 端点、请求/响应 VO 字段与流式事件类型。默认外部 URL 前缀为 /snail-ai（context path）。返回 L1 入口概览 · 短课 · Admin 对话切片 · OpenAPI 对话切片。

Admin 13 Controller OpenAPI 5 Controller 3 种 Admin 流式 OpenAPI SSE
响应包装 流式事件 用户/认证 智能体 对话/会话 RAG 知识库 记忆 VO Skill MCP 模型 应用 资源/存储 OpenAPI 异常与基础设施

认证易错：Admin 使用 Snail-Ai-Auth（JWT）；OpenAPI 使用 Snail-Ai-App-Id + Snail-Ai-Token。二者不可混用。带 @LoginRequired 的 Admin 端点才校验登录态。

通用响应包装

类型字段说明
Result<T>status, message, data多数 Admin/OpenAPI JSON 响应。成功通常 status=200。
PageResult<T>继承 Result，额外 page, size, total分页列表。部分 Admin 接口直接返回 PageResult，由 GlobalRestfulResponseBodyAdvice 透传。
LoginResponseVOusername, nickname, token登录接口不包 Result，直接返回 JWT 字符串于 token。
裸类型—AiModelController 部分接口返回 Long/boolean/List，由 Advice 自动包成 Result。
流式—TEXT_EVENT_STREAM 或 application/x-ndjson 响应不被 Advice 包装。

流式事件类型

入口Media-Type帧格式事件 type载荷字段
POST /agent/{id}/chat text/event-stream 行分隔 JSON（经 ChatStreamWriter） text, thinking type, content（ChatStreamEvent）
POST /agent/create/stream application/x-ndjson 每行一个 JSON + \n start, field_done, done, error type, field（name/description/greeting/presetQuestions/instruction）, data, agentId, message
POST /rag/qa/stream application/x-ndjson 每行一个 JSON + \n text, done, error type, data, message
POST /openapi/v1/agent/chat text/event-stream 标准 SSE：event + data text, thinking, done, error data 为字符串；done 的 data 为 JSON：conversationId, fullText
POST /openapi/v1/agent/chat/sync application/json 同步收集流式输出 — OpenApiChatSyncResponse：conversationId, content, durationMs

记忆（VO 预备，无独立 Controller）

源码现状：vo/memory/ 下已定义完整记忆 VO，但 snail-ai-server-admin 当前没有 MemoryController。记忆能力通过智能体配置字段 memoryEnabled、memoryConfigId、shortTermMemorySize 在 AgentRequestVO 中启用，运行时由 feature-memory 模块处理。

SearchMemoryRequestVO检索query, agentId, userId, conversationId
MemoryItemResponseVO记忆条目id, title, content, memoryType, tags, relevanceScore
