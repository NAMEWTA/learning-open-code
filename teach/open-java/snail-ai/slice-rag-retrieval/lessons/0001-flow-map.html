RAG 对话检索注入流程地图

这一课只做一件事：从一次 Agent 对话进入 RAG 判断开始，画清检索如何被触发、如何经过向量/BM25、融合、rerank 和 finalize，最后怎样进入模型上下文。学完后，你应该能判断问题卡在调用模式、检索 pipeline，还是 prompt/tool 注入。

1. 对话先进入 RAG 判断

AgentChatService.chat 创建上下文后交给有序责任链。SystemPromptHandler 先把 Agent 指令放进 systemPrompt，随后 RagHandler 检查 ragEnabled、ragIds 和 ragCallMode。这里的核心判断只有两个：FORCED 直接服务端检索，SMART 只写入可用知识库说明，让 Client 端模型自己决定是否调用工具。

Agent 对话AgentChatService.chat
提示词基础SystemPromptHandler
RAG 开关ragEnabled 与 ragIds
模式分流RagCallModeEnum
LLM 分发LlmCallHandler

2. 强制模式立即检索

强制调用时，RagHandler.parallelSearch 用虚拟线程并行遍历绑定知识库。每个知识库都会构造 RagSearchRequestDTO：ragId 来自 Agent 绑定列表，query 直接使用本轮用户消息 ctx.getContent()。随后进入统一的 RagSearchService.search。

配置ConfigResolveHandler
向量召回VectorSearchHandler
BM25 召回Bm25SearchHandler
融合/重排HybridFusionHandler
RerankHandler
补全输出FinalizeHandler

QueryRewriteHandler 当前是空实现，所以不要把问题改写当成已生效能力。向量检索使用索引名 rag_{ragId}，BM25 只有在 searchEngineEnable=true 时执行。BM25 有结果才触发融合；rerank 只有 rerankEnabled=true 且存在模型 ID 才调用模型。最终 FinalizeHandler 补齐 chunk 内容、文档名、邻近切片，并做 lost-in-the-middle 重排。真实源码片段见 参考图谱的源码片段。

3. 注入有两条路径

强制调用

RagHandler 把检索结果格式化成编号资料，追加到 systemPrompt 的 Knowledge Base Reference 段落。LlmCallHandler 把它放进 ChatDispatchRequest.systemPrompt，Client 的 DefaultPromptFactory 再转成 SystemMessage。

智能调用

RagHandler 只追加 Available Knowledge Bases。Client 的 ClientRagToolResolver 在非强制模式注册 RagSearchTool，模型调用 rag_search 后，经 RpcClient.searchRag 回到 RagSearchCallbackHandler。

Server: RagHandler.buildSmartPrompt 写知识库清单
Server: LlmCallHandler 分发 ChatDispatchRequest
Client: ToolRuntime 解析并注册 rag_search
Model: 按需调用 rag_search，工具 schema 参数是 ragId 与 queryQuestion
Client: RagSearchTool 调用 RpcClient.searchRag
Server: RagSearchCallbackHandler 执行同一套检索 pipeline
Model: 工具返回编号资料，继续生成回答

4. 快速练习

暂停回忆：不看上面的图，判断三件事：FORCED 模式会不会在 Client 注册 rag_search？智能调用的工具参数名是什么？如果 BM25 没有结果，是否还会执行融合？
参考答案：FORCED 不注册工具，因为服务端已经预检索并写入 systemPrompt；智能调用的自然语言提示可以称为 query，但 Java 工具 schema 的参数名是 queryQuestion，工具会 trim 后回调 Server；BM25 结果为空时 HybridFusionHandler 直接跳过，最终主要沿用向量结果。

主要推荐来源：先读 RagHandler.java 和 ClientRagToolResolver.java，再对照 检索注入参考图谱 查 pipeline、DTO、配置字段和排障 checklist。遇到配置空指针、rerank 降级或工具不触发时，继续向 agent 追问对应源码段。
