成功主路径：从 dispatch 到 completion

本课只练一个判断：一次正常的 /chat/dispatch 会如何从请求解析、模型流订阅，走到最终 completion 回写。

三步主线

入口绑定ChatDispatchStreamingHandler 解析 body，取 requestId 和 sid，把四类 consumer 交给 runtime。
会话执行ChatSessionRuntime 增加活跃计数，解析工具，再调用 ClientChatExecutor.executeStream()。
终态回写ClientChatExecutor 在 Flux complete 时生成 ChatCompletionResult，handler 写回 completion。

这条路径上，text 和 thinking 是中途增量；completion 是终态摘要。判断是否成功，不要只看前端有没有零散文字，还要看最终是否收到包含 fullText、fullThinking、token 和耗时的 completion。

源码锚点

入口的关键动作在 handle()：解析请求后，consumer 只转发事件，不直接调用模型。

ChatDispatchRequest dispatchRequest = parseDispatchRequest(request.getBody());
String requestId = dispatchRequest.getRequestId();
String sid = dispatchRequest.getSid();

chatSessionRuntime.execute(ChatSessionRequest.builder()
    .dispatchRequest(dispatchRequest)
    .textConsumer(text -> handleTextChunk(reqId, sid, text, observer))
    .thinkingConsumer(thinking -> handleThinkingChunk(reqId, sid, thinking, observer))
    .completionConsumer(completion -> handleCompletion(reqId, requestId, sid, completion, observer))
    .errorConsumer(error -> handleError(reqId, sid, error, observer))
    .build());

执行器的完成信号来自 Reactor 订阅的第三个回调。前面的 Advisor 已把 text、thinking、usage 累积进 ClientStreamExecutionContext，所以 complete 时才能一次性取出完整结果。

.subscribe(
    response -> {
    },
    request::acceptError,
    () -> request.acceptCompletion(state.toCompletionResult())
);

completion 的真正含义

completion 不等于“最后一个 text chunk”。它是 ClientStreamExecutionContext.toCompletionResult() 的快照：完整文本来自 StreamChunkForwarderAdvisor，完整 thinking 来自 ThinkingCollectorAdvisor，usage 来自 TokenUsageCollectorAdvisor。

正常发送时，handleCompletion() 先 observer.onNext() 发送 status 1 和 JSON data，随后在同一个 try 内调用 observer.onCompleted() 结束 gRPC 流。也就是说，Server 侧看到 completion 后，才有机会进入持久化链路。

检索练习

暂停回忆：如果你要证明一次调用走完成功主路径，至少要在日志或断点里确认哪三个位置？

参考答案：先确认 ChatDispatchStreamingHandler.handle() 成功解析出 requestId/sid；再确认 ClientChatExecutor 的 complete 回调执行 state.toCompletionResult()；最后确认 handleCompletion() 发出 ChatStreamResponse.completion() 并调用 observer.onCompleted()。

主要来源建议读 ChatDispatchStreamingHandler.java、ClientChatExecutor.java 和 ClientStreamExecutionContext.java。不清楚的地方可以继续让 agent 带你沿调用栈查。
