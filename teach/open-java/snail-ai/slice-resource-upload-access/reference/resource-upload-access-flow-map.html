资源上传和访问参考图谱

一句话：Snail AI 的资源链路由 ResourceController 接收 Admin 上传请求，ResourceAdminService 补齐当前用户和业务字段，ResourceService 选择默认存储策略并写入 sai_resource，随后通过 /files/{storageKey} 由 FileAccessController 公开代理读取 LOCAL 或 MINIO 文件。

边界提醒：/resource/* 有 @LoginRequired；/files/** 在拦截器中被排除，访问凭证实际是包含 UUID 的 storageKey。EXTERNAL 资源保存外部 accessUrl，当前文件访问控制器不会代理它。

入口 API

方法路径入口方法行为权限边界
POST/resource/uploadResourceController.upload接收 MultipartFile 和 ResourceUploadRequestVO，委托 ResourceAdminService.upload。@LoginRequired 默认 ADMIN。
DELETE/resource/{id}ResourceController.delete先做资源授权检查，再调用 ResourceService.delete 删除文件和资源行。@LoginRequired 默认 ADMIN；服务层还保留非管理员所有者校验。
GET/resource/pageResourceController.page按 bizType、originalName、bizId、时间范围分页查询。@LoginRequired 默认 ADMIN；服务层对非管理员追加 creatorId 过滤。
GET/files/**FileAccessController.access从 URI 提取 storageKey，反查资源行并从 LOCAL 或 MINIO 读取流。拦截器排除该路径；不看资源 ID 和登录 token。

源码片段

ResourceController 只定义入口

@PostMapping("/upload")
@LoginRequired
public Result<ResourceResponseVO> upload(@RequestParam("file") MultipartFile file,
                                          ResourceUploadRequestVO request) {
    return Result.ok(resourceAdminService.upload(file, request));
}

@DeleteMapping("/{id}")
@LoginRequired
public Result<Void> delete(@PathVariable("id") Long id) {
    resourceAdminService.delete(id);
    return Result.ok(null);
}

ResourceService.upload 串起存储和资源表

ResourceStorageService storage = storageFactory.getDefault();
String storageKey = storage.store(bizType, originalName, inputStream);
String mimeType = MimeTypeUtils.detect(originalName);

ResourcePO resource = ResourcePO.builder()
        .storageKey(storageKey)
        .originalName(originalName)
        .fileSize(fileSize)
        .mimeType(mimeType)
        .storageType(storage.getType())
        .bizType(bizType)
        .bizId(bizId)
        .creatorId(creatorId)
        .build();
resourceMapper.insert(resource);

String accessUrl = storage.getAccessUrl(resource.getId(), storageKey);
resource.setAccessUrl(accessUrl);
resourceMapper.updateById(resource);

关键对象和字段

字段或对象来源作用排查意义
storageKeyStorageKeyGenerator.generate形如 {bizType}/{yyyy}/{MM}/{dd}/{uuid} 加可选扩展名，是存储后端对象 key，也是 /files/** 的访问凭证。404 时先确认 URL 中的 key 和表里的 storage_key 完全一致。
accessUrlResourceStorageService.getAccessUrlLOCAL 和 MINIO 当前都返回 /files/{storageKey}；EXTERNAL 直接保存外部 URL。判断是否应走文件控制器。
storageType默认配置或直接建记录上传走 snail-ai.resource.storage-type，支持策略是 LOCAL 和 MINIO；枚举还有 EXTERNAL。/files/** 只代理 LOCAL 和 MINIO。
bizType上传请求或调用方用于路径分区和业务归属，包含 GENERAL、AVATAR、ATTACHMENT、DOCUMENT、DOCUMENT_PREVIEW。头像绑定、RAG 文档和临时预览资源都依赖它区分语义。
creatorIdUserSessionUtils.currentUserSession记录上传者。服务层分页和删除授权会用它做所有者边界。
mimeTypeMimeTypeUtils.detect按扩展名或 JDK 猜测生成响应 Content-Type。影响浏览器预览、下载和 Content-Disposition。

storageKey 和 URL 规则

阶段源码事实结果
生成StorageKeyGenerator 会清理 bizType 为小写字母、数字、下划线或短横线，扩展名只保留小写字母数字。DOCUMENT_PREVIEW 会变成路径前缀 document_preview。
日期路径使用当前日期 yyyy/MM/dd。对象 key 天然按日期分区。
随机凭证文件名主体是 UUID。公开访问依赖 key 难猜，而不是登录态。
访问校验FileAccessController.STORAGE_KEY_PATTERN 要求 bizType/yyyy/MM/dd/uuid 加可选扩展名的形态。不符合格式、URL 解码失败或前缀不是 /files/ 都返回 404。
下载参数download=true 或 MIME 不在 inline 白名单时使用 attachment。图片、PDF、TXT、CSV、Markdown 可 inline；其他默认下载。

存储策略差异

类型写入位置读取方式URL安全边界
LOCALsnail-ai.resource.upload-dir，默认 ./upload/resource。Files.newInputStream(resolveAbsolute(storageKey))。/files/{storageKey}resolveAbsolute 归一化后要求仍在根目录内，防路径穿越。
MINIOsnail-ai.resource.minio.bucket，默认 snail-ai。MinioClient.getObject(bucket, storageKey)。/files/{storageKey}首次使用时按 endpoint/accessKey/secretKey 建 client，并确保 bucket 存在。
EXTERNAL不是上传策略；例如 OpenApiUserService 直接写外部头像 URL。不由 ResourceStorageFactory 读取。外部 accessUrlFileAccessController 的 isProxyStorage 会拒绝，返回 404。

权限和响应头

位置规则意义
ResourceController三条接口都有 @LoginRequired；注解默认 role=ADMIN。当前 Admin 资源管理入口按管理员权限保护。
ResourceAdminService.requireAuthorizedResource资源不存在报错；非管理员且不是 creatorId 时拒绝。即使入口权限以后放宽，服务层仍保留所有者边界。
SnailJobWebMvcConfigurerAdapter认证拦截器排除 /openapi/** 和 /files/**。公开文件访问不依赖登录态。
FileAccessController添加 Cache-Control: public, max-age=86400 和 X-Content-Type-Options: nosniff。允许公共缓存一天，并降低 MIME 嗅探风险。
OriginalControllerReturnValue文件访问方法标注该注解。绕过统一 Result 包装，直接返回 ResponseEntity<InputStreamResource>。

表结构速查

表字段用途
sai_resourceid资源自增主键，Admin 删除和业务绑定使用。
sai_resourcestorage_key唯一索引 uk_storage_key，文件访问入口按它反查。
sai_resourcestorage_type默认 LOCAL，SQL 注释写 LOCAL/MINIO。
sai_resourceaccess_url前端直接使用的访问链接。
sai_resourcebiz_type、biz_id业务归属索引 idx_biz。
sai_resourcecreator_id上传者索引 idx_creator。

调试 checklist

问题检查点对应源码
上传接口直接 401 或权限不足确认请求头 Snail-Ai-Auth 有效，当前用户是否满足 @LoginRequired 默认管理员角色。AuthenticationInterceptor.java、LoginRequired.java
上传成功但响应没有可用 URL检查 ResourceService.upload 是否插入后成功回写 accessUrl，以及默认存储策略是否实现 getAccessUrl。ResourceService.java、ResourceStorageService.java
/files/** 返回 404确认 URL 解码后以 /files/ 开头、storageKey 符合正则、表中有同名 storage_key。FileAccessController.extractStorageKey
EXTERNAL 资源访问失败不要走 /files/**；直接使用资源行的外部 accessUrl。FileAccessController.isProxyStorage、OpenApiUserService.java
本地文件 404 或读取失败检查 snail-ai.resource.upload-dir、磁盘文件是否存在、路径是否被归一化后越过根目录。LocalResourceStorageService.java
MinIO 上传或读取失败检查 endpoint、accessKey、secretKey、bucket 和对象 key；首次访问会创建 bucket。MinioResourceStorageService.java、ResourceConfig.java
删除后文件仍残留文件或对象删除失败只记录 warning；资源行最终仍会删除，需要查存储后端日志。ResourceService.delete
分页看不到资源核对 bizType、originalName、bizId、时间范围和当前用户角色；服务层可能追加 creatorId 条件。ResourceAdminService.page、BaseQueryVO.java
头像绑定失败用户资料更新只接受 bizType=AVATAR 的资源。UserService.updateUserAvatar
RAG 文档下载 URL 不对RAG 响应直接使用资源行 accessUrl 作为 previewUrl，下载 URL 只是追加 download=true。RagDocumentService.toResponseVO
