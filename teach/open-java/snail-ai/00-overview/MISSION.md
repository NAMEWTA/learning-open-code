# 使命：Snail AI 项目总览

## 为什么
学习者希望系统掌握 Snail AI 的整体架构和源码分层，后续能够按模块继续深入 Server 责任链、Agent Client、RAG、记忆、模型适配和持久化实现。这个总览主题的现实目标是建立一张可信的项目地图，避免后续学习时在 Maven 多模块、文档站和内置静态前端之间迷路。

## 成功的样子
- 能用一句话说明 Snail AI 的项目定位，并区分 Server 与 Agent Client 的职责边界。
- 能根据顶层目录判断一个功能大概率落在哪个 Maven 模块中。
- 能说出当前技术栈的关键版本、存储后端和主要扩展点。
- 能从本主题参考文档中提取后续 L1 模块课和 L2 垂直切片候选。

## 约束条件
- 本次为批量生成模式，不等待用户访谈，使用面向源码学习的默认使命。
- 每节 lesson 必须保持 15 分钟内完成，复杂清单放入 reference 文档。
- 当前目标只覆盖 L0 全局地图，不展开单个 Handler、Advisor 或持久化实现细节。

## 不在范围内
- 不讲具体部署排障、生产容量规划或安全加固操作。
- 不逐行讲解 RAG、记忆、MCP、Skill、OpenAPI 等功能实现。
- 不覆盖已编译前端静态包、图片资源和自动生成 gRPC Java 类的细节。
