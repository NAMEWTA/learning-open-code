Snail AI 项目总览参考

一句话描述：Snail AI 是基于 Spring Boot 4 + Spring AI 的企业级 AI 智能体平台，采用 Server-Agent 分布式架构，把多模型、Agent 编排、RAG、记忆、MCP/Skill、OpenAPI 和后台管理整合为一套可扩展系统。

版本注意：当前根 POM 的 revision 为 1.0.0；文档站配置和部分架构文档仍标注 0.0.6。后续课程以当前工作区源码、根 POM 与实际配置为准。

核心设计哲学与架构决策

决策含义后续学习入口
Server-Agent 分离Server 做业务编排和状态管理，Client 做模型与工具执行，二者通过 gRPC 连接。L2：对话请求从 Controller 到 Client
双链路扩展Server 端用 Handler Chain 组织业务上下文，Client 端用 Spring AI Advisor 组织执行期横切能力。L1：Agent 责任链；L1：Client Advisor
模型能力分类型适配Chat、Embedding、Rerank 拆成独立 Maven 聚合，避免把所有 provider 混在一个模块里。L1：模型适配层
存储后端可替换业务库、向量库、搜索引擎都通过模板和具体实现分离，支持多后端选择。L1：Persistence 与 Store Instance
RAG 和记忆都是上下文能力它们不直接替代模型调用，而是在请求链路中检索、过滤、注入上下文。L2：RAG 检索注入；L2：记忆抽取与召回

学习路线索引

先看 L1：Agent 责任链再看 L2：OpenAPI Chat随后补 RAG 与 Memory最后看 Persistence 与 Models

判断任意文件归属时可用三问：它是否处理 HTTP/gRPC 入口？它是否补上下文或执行业务能力？它是否是模型、存储或资源后端的具体适配？这三问能把大多数文件放回正确的 L1/L2 学习主题中。
