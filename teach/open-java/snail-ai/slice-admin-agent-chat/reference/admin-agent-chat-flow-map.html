Admin 智能体流式对话切片参考

这份参考用于排查 Admin 智能体流式对话。它把入口端点、关键 DTO、Handler 顺序、gRPC URI、stream writer、失败/取消/usage/持久化边界和调试 checklist 放在一处。

1. 入口端点与认证上下文

项目当前源码事实源码
HTTP 入口 POST /agent/{id}/chat，Controller 类级路径是 /agent，方法路径是 /{id}/chat。 open-java/snail-ai/snail-ai-server/snail-ai-server-admin/src/main/java/com/aizuda/snail/ai/admin/controller/AgentController.java
响应类型 produces = MediaType.TEXT_EVENT_STREAM_VALUE，返回 Flux<String>。 AgentController#chat
认证 @LoginRequired(role = RoleEnum.USER) 触发 Admin 拦截器读取 Snail-Ai-Auth。 AuthenticationInterceptor.java
用户上下文 拦截器将 UserPO 写入 UserSessionUtils；请求完成后清理 ThreadLocal。 UserSessionUtils.java
请求体 conversationId、content 必填；disabledMcpServerIds、disabledSkillIds 可选。 AgentChatRequestVO.java
Controller 输出口 FluxBridgeChatStreamWriter 实现 ChatStreamWriter，send 调 sink.next，complete 调 sink.complete。 AgentController.java

2. 关键 DTO 与共享上下文

类型方向关键字段使用边界
AgentChatRequestVO HTTP 请求体 conversationId、content、禁用 MCP/Skill 列表 Admin Controller 的入参校验模型。
AgentChatCommand Controller 到服务 agentId、conversationId、content、streamWriter、sid 把 Web 层参数转成应用服务命令。
AgentChatContext Server 责任链内部 user、agent、modelConfig、mcpServerDescriptors、skillDescriptors、historyMessages、targetClient 所有 Handler 共享的接力棒；失败时用 terminated 短路。
ChatDispatchRequest Server 到 Client requestId、sid、Agent/model/MCP/Skill/history/system prompt/server port LlmCallHandler 组装，gRPC body 用 JSON 字符串传输。
ChatStreamResponse Client 到 Server type、content、fullText、fullThinking、token、duration、error gRPC streaming 的业务 data；类型为 text、thinking、completion、error。
ChatResultPersistCommand Server completion 到持久化 fullText、thinkingText、input/output/cache tokens、memory 开关、shortTermWindow ChatStreamObserver.handleCompletion 创建，交给虚拟线程异步持久化。

3. Server Handler 顺序

OrderHandler职责失败边界
10InitContextHandler取当前用户或显式请求用户，按 agentId 加载 Agent。Agent 不存在时写错误、complete、置 terminated。
20ConversationHandler按 conversationId 创建会话并保存用户消息，状态为 RUNNING。没有显式捕获异常；数据库异常会向外抛。
30ModelResolveHandler选择 Agent 绑定模型或默认 chat 模型，组装 ModelConfig。未配置模型或配置加载失败时终止。
40McpHandler加载 Agent MCP Server，过滤本次禁用列表，生成 Client 描述符。解析 JSON 失败时回退空列表或空 Map。
50SystemPromptHandler以 Agent instruction 初始化 system prompt。instruction 为空时使用默认助手提示。
60RagHandler强制模式在 Server 检索并追加 prompt；智能模式追加可用知识库说明。知识库不存在或检索失败时记录日志，通常不终止主链路。
70SkillAgentChatHandler加载 Skill 内容，过滤本次禁用列表，生成 Skill 描述符。Skill 为空时直接跳过。
75ContextCollectorHandler校验 app、选择活跃 Client、写入用户短期记忆、加载历史消息。缺 appId、app 停用、无 Client 时写错误并终止。
80LlmCallHandler组装 ChatDispatchRequest，调用 GrpcChannelUtil.sendServerStreaming。缺 targetClient 或 modelConfig 时终止；gRPC 发起异常会向外抛。

执行器 AgentChatChainService 只是遍历列表，不会主动 break。每个 Handler 必须自己检查 ctx.isTerminated()，这是调试“错误已写出但后续还执行吗”的关键。

4. gRPC URI 与 Client 执行

阶段组件关键事实源码
Server 发起 GrpcChannelUtil.sendServerStreaming 构造统一 GrpcSnailAiRequest，metadata.uri 为 /chat/dispatch，body 为 ChatDispatchRequest JSON。 GrpcChannelUtil.java
业务 URI UriConstants.CHAT_DISPATCH 常量值是 /chat/dispatch，方向是 Server 到 Client。 UriConstants.java
Client gRPC server ClientGrpcServer 注册 unary 和 server-streaming service，监听 Agent Client 本地端口。 ClientGrpcServer.java
Client 分发 ClientRequestDispatcher 读取 metadata.uri，委托 GrpcRequestDispatcher.dispatchStreaming。 ClientRequestDispatcher.java
聊天 handler ChatDispatchStreamingHandler 只支持 /chat/dispatch，解析请求后把 text/thinking/completion/error 映射为 GrpcSnailAiResult.data。 ChatDispatchStreamingHandler.java
运行壳 ChatSessionRuntime 增加 active chat 计数，解析工具，执行流式聊天，在 completion/error 后清理工具连接并减少计数。 ChatSessionRuntime.java
工具解析 ToolRuntime 解析 MCP、基础工具、智能 RAG 工具、Skill 工具和业务 @Tool。 ToolRuntime.java
模型执行 ClientChatExecutor 构建 Prompt 和 ChatClient，写入 Advisor context，订阅 stream().chatResponse()。 ClientChatExecutor.java
Advisor TokenUsageCollectorAdvisor、ThinkingCollectorAdvisor、StreamChunkForwarderAdvisor 分别收集 usage、thinking 和文本 chunk，写入 ClientStreamExecutionContext。 SnailAiAgentAutoConfiguration.java

5. 流式回写、取消与持久化边界

边界当前行为排查提示
text chunk Client 发 ChatStreamResponse.text(sid, text)；Server ChatStreamObserver.handleText 序列化后追加换行并写入 ChatStreamWriter.send。 前端没有增量输出时，看 Client 是否触发 StreamChunkForwarderAdvisor。
thinking chunk Client 从 metadata 或底层响应对象抽取 thinking；Server 过滤空文本后按 thinking 类型写回。 依赖模型是否返回 reasoning/thinking metadata。
completion Client 将 fullText、fullThinking、tokens、duration 发回后调用 observer completed；Server 收到 completion 只触发持久化，不把 completion 事件写给 HTTP 客户端。 助手消息和 usage 缺失时，先确认 completion 是否到达 Server。
error Client error 会以 ChatStreamResponse.error 回传；Server 写 [ERROR] message，但不触发 completion 持久化。 只看到用户消息没有助手消息，可能是 error 路径。
HTTP 取消 Controller 的 sink.onCancel(() -> {}) 当前为空；writer 会避免向已取消 sink 写入，但没有主动取消已经发起的 gRPC 调用。 用户关闭页面不等于 Client 模型调用立即停止。
usage TokenUsageCollectorAdvisor 只在流式响应 metadata 中存在 usage 且 totalTokens 大于 0 时写入 state。 usage 为 0 可能是模型或配置未返回流式 usage，不一定是持久化失败。
用户消息持久化 ConversationHandler 在 gRPC 分发前插入 USER 记录。 这解释了“用户消息有、助手消息没有”的常见现象。
助手消息持久化 ChatResultPersistService 在虚拟线程中插入 ASSISTANT 记录，写 thinking 和 token。 失败会记录日志并抛出到虚拟线程内，不会补发 SSE。
短期记忆 用户消息在 ContextCollectorHandler 追加；助手消息在 completion 后由 ChatResultPersistService 追加。 上下文缺历史时，分别检查这两个写入点。
每日统计 updateUsageStat 当前递增 message_count，并在无记录时插入当天统计。 统计的是消息/会话计数；token 数保存在对话记录中。

6. 调试 checklist

症状优先检查判断依据
HTTP 401 或未登录 Snail-Ai-Auth header、JWT audience、用户是否存在、角色是否满足 USER。 AuthenticationInterceptor 只对带 @LoginRequired 的方法强制认证。
SSE 建立后立即错误 InitContextHandler、ModelResolveHandler、ContextCollectorHandler 的错误文本。 这些错误会直接写 stream writer 并 complete。
找不到 Client Agent 的 appId、应用启用状态、Client 心跳、ClientInstanceManager.getAliveInstances。 Client 心跳每 10 秒上报；Server 认为 40 秒未心跳过期。
Client 收不到任务 LlmCallHandler 日志中的目标 host/port、UriConstants.CHAT_DISPATCH、Client gRPC 端口。 Server 到 Client 是 gRPC server-streaming，不是 HTTP。
Client 收到任务但无文本 ChatDispatchStreamingHandler、ChatSessionRuntime、工具解析异常、模型配置校验、ClientChatExecutor error consumer。 error 会通过 ChatStreamResponse.error 回传。
文本有但 thinking 没有 模型是否支持 reasoning/thinking metadata，ThinkingCollectorAdvisor 是否能识别 key。 当前兼容 reasoningContent、thinking、reasoning 和部分 additionalProperties。
文本有但 usage 为 0 模型流式 usage 是否启用，最终 chunk 是否包含 usage，TokenUsageCollectorAdvisor 日志。 Advisor 只在 totalTokens 大于 0 时写入 token。
用户消息有、助手消息无 是否收到 completion、ChatStreamObserver.handleCompletion 是否执行、ChatResultPersistService 日志。 用户消息在 gRPC 前保存；助手消息只在 completion 后保存。
关闭页面后模型还在跑 AgentController.chat 中 sink.onCancel 当前为空。 取消只阻止继续写已取消 sink，没有向 gRPC 或 Client 传播取消信号。
