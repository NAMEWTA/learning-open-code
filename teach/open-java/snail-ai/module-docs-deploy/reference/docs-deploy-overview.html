SQL 初始化边界

脚本数据库关键内容风险
docs/sql/snail_ai_schema.sqlMySQL22 张 sai_* 表；默认管理员 admin/admin123；默认测试应用 snail-ai-agent-demo。重复导入前确认是否清库；生产必须改默认账号和 token。
docs/sql/snail_ai_schema_pgsql.sqlPostgreSQL同样 22 张业务表，使用 BIGSERIAL 和注释语句。需要与 PostgreSQL 驱动和连接串匹配；PgVector 扩展另行验证。
docs/sql/snail_ai_schema_dameng.sql达梦从 MySQL 版本转换，使用 IDENTITY 等达梦写法。当前文件在本地识别为 ISO-8859 文本，中文注释显示乱码；使用前要验证编码与执行结果。

表域 用户与 OpenAPI：sai_user、sai_openapi_user；模型：sai_model_provider、sai_model_config、sai_model_usage_stat；智能体：sai_agent、sai_agent_conversation、sai_agent_conversation_record、sai_agent_usage_stat、sai_user_agent；RAG：sai_rag、sai_rag_document、sai_rag_chunk；MCP 和 Skill：sai_mcp_server、sai_agent_mcp_server、sai_skill、sai_skill_file、sai_agent_skill；应用与资源：sai_app、sai_client_node、sai_store_instance、sai_resource。

启动顺序

阶段操作验证点
1. 构建边界确认 JDK 21、Maven、根 pom.xml 和 snail-ai-starter/pom.xml。能产出 snail-ai-starter/target/snail-ai-server-exec.jar。
2. 数据库选择 MySQL 或 PostgreSQL，导入匹配 SQL。业务表存在，默认管理员可用。
3. 依赖组件按需要启动 PgVector、Milvus、Elasticsearch、MinIO。端口和健康检查正常。
4. Server使用 Maven、JAR 或 Compose 启动 Snail AI Server。curl http://localhost:8900/snail-ai/actuator/health。
5. Client 与模型配置应用、模型、Agent Client 和 OpenAPI 认证。Client 节点在线，对话或 OpenAPI 请求可返回。

CI 与发布 workflow

文件触发流程产物
.github/workflows/docker-image.yml推送 v* tagcheckout，JDK 21，Maven cache，mvn -B package -Dmaven.test.skip=true，读取项目版本，Docker Buildx，登录 DockerHub，使用 snail-ai-starter/ 上下文构建。DOCKER_NAMESPACE/snail-ai:latest 和 :VERSION，平台为 linux/amd64、linux/arm64。
.github/workflows/maven-publish.yml推送 v* tagcheckout，JDK 21，Maven cache，package，配置 Maven Central 和 GPG，执行 mvn deploy -P release -f pom.xml。Maven Central 发布包。

常见部署风险

风险为什么重要先查哪里
文档与 Compose 描述不一致可能误以为 Compose 只启动依赖，实际也会启动 Server。docs/docker/docker-compose.yaml
部署路径漂移部分源文档仍提根目录 script/，不能作为当前文件位置依据。当前 checkout 以 docs/docker/docker-compose.yaml 和 docs/sql/ 为准
数据库脚本混用MySQL、PostgreSQL、达梦语法不同，混用会导致初始化失败。docs/sql/ 三个具体脚本
默认密码和密钥进入生产默认 MySQL、MinIO、管理员、加密密钥都不适合生产。application.yml、Compose env、生产部署清单
gRPC 端口未开放后台可访问不代表 Agent Client 能连接。snail-ai.server.grpc-port、18888 防火墙
本地资源目录未持久化上传文件、Skill、日志可能随容器或进程目录丢失。Compose volumes、snail-ai.resource.*、SNAIL_AI_SKILL_UPLOAD_DIR
把初始化 SQL 当升级 SQL已有生产库可能被覆盖或出现重复字段、重复数据。docs/deploy/upgrade.md、Release Notes
Docker 构建缺少 JARDockerfile 使用 ADD ./target/snail-ai-server-exec.jar app.jar。snail-ai-starter/pom.xml、docker-image.yml
达梦脚本编码未验证当前本地识别为 ISO-8859 文本，注释显示乱码。docs/sql/snail_ai_schema_dameng.sql
