模型配置到运行时调用速查

这份 reference 用于排查 Snail AI 的模型调用：配置字段、API Key 解密、provider 目录、runtime 接口、Chat/Embedding/Rerank 差异、异常传播和降级策略都放在这里。

当前源码事实：用户给出的 DispatchModelConfigAssembler 优先路径在 snail-ai-feature-agent 下不存在；真实类位于 snail-ai-server-features/snail-ai-feature-model/src/main/java/com/aizuda/snail/ai/model/adapter/server/DispatchModelConfigAssembler.java，并由 ModelResolveHandler import 使用。

2. 配置字段速查

对象字段进入 runtime 的方式来源
ModelConfigInfoDTOid缓存 key、日志和 modelConfigId 追踪。ModelConfigHandler.buildConfigInfoDTO
ModelConfigInfoDTOproviderKey必须非空，进入 spec；当前内置 adapter 选择不靠它。ServerModelSpecFactory.requireConfig
ModelConfigInfoDTOadapterKey空值由 ModelAdapterDefaults.resolve 补默认值，是 runtime 选择 adapter 的主键。ServerModelSpecFactory.resolveAdapterKey
ModelConfigInfoDTOmodelType必须匹配 CHAT、EMBEDDING 或 RERANKER。ServerModelSpecFactory.requireConfig
ModelConfigInfoDTOapiEndpoint进入 spec 的 baseUrl；缺失会报非法参数或无效配置。ServerModelSpecFactory、*ModelRuntime.validate
ModelConfigInfoDTOmodelKey进入 provider 的 model 字段；Chat/Embedding 传给 OpenAI-compatible options，Rerank 传给 HTTP body。OpenAiCompatible*Adapter、QwenRerankModelAdapter
ModelConfigInfoDTOencryptedApiKeyServer 侧通过 ModelConfigHandler.decryptApiKey 解密；Client 侧不再解密。ModelRuntimeHandler.decryptApiKey、DispatchModelConfigAssembler
ModelConfigInfoDTOconfigJson进入 spec；provider 读取 timeout、采样参数、embedding 维度、rerank path。ConfigExtAttrsDTO
ChatDispatchRequest.ModelConfigapiEndpoint、apiKey、modelKeyClient validator 要求非空，随后转为 ChatModelSpec。ClientModelConfigValidator
ChatDispatchRequest.ModelConfigadapterKeyClient 侧空值按 Chat 默认补为 openai-compatible。ClientChatModelSpecFactory
ConfigExtAttrsDTO 字段ChatEmbeddingRerank
timeoutMs默认 300000 ms。默认 60000 ms。默认 60000 ms，连接超时最多 10000 ms。
maxRetries当前内置 Chat adapter 未映射。映射到 OpenAiEmbeddingOptions.maxRetries，默认 2。当前内置 Rerank adapter 未映射。
responseFormat、stream字段存在于 ConfigExtAttrsDTO，并随 configJson 抵达 ChatModelSpec；但当前内置 OpenAiCompatibleChatModelAdapter 未读取它们，未映射成实际 runtime 参数。不使用。不使用。
temperature、topP、topK、maxTokens映射到 OpenAiChatOptions。不使用。不使用。
frequencyPenalty、presencePenalty、stopSequences、seed、extraBody映射到 OpenAiChatOptions。不使用。不使用。
embeddingDimension、encodingFormat不使用。映射到 OpenAiEmbeddingOptions；调用时的 dimensions 还会进入 EmbeddingRequest。不使用。
rerankPath不使用。不使用。拼接到 baseUrl，默认 /rerank。

4. 运行时接口

层接口或类职责典型错误
Server 配置ModelConfigHandler读取模型配置、解密 API Key、获取默认模型。配置不存在、密钥解密失败。
Server specServerModelSpecFactory按类型生成 ChatModelSpec、EmbeddingModelSpec、RerankModelSpec。INVALID_PARAMETER。
Server 门面ServerModelFacade把三类 spec 交给对应 runtime。透传 runtime 或 adapter 异常。
RuntimeChatModelRuntime、EmbeddingModelRuntime、RerankModelRuntime校验 spec，按 adapterKey 选择 adapter。INVALID_MODEL_CONFIG、ADAPTER_NOT_FOUND、MODEL_BUILD_FAILED。
Provider SPIChatModelAdapter、EmbeddingModelAdapter、RerankModelAdapter把 spec 翻译成 SDK 模型或 HTTP client。provider 构建异常。
Client specClientChatModelSpecFactory把下发的 ChatDispatchRequest.ModelConfig 转成 ChatModelSpec。IllegalArgumentException。
Client factoryDefaultChatClientFactory构建 ChatClient，注入 tools、advisor 和 tool context。进入 Client error 回调。

5. 三类模型差异

维度ChatEmbeddingRerank
Server 本地入口ModelRuntimeHandler.chat、chatStream、SmartChunkStrategy 的 ChatClientBuilder。ModelRuntimeHandler.embed、embedBatch、buildEmbeddingModel。ModelRuntimeHandler.rerank。
Agent Client 入口DefaultChatClientFactory。当前 Agent Client 切片未下发 Embedding runtime。当前 Agent Client 切片未下发 Rerank runtime。
真实调用ChatClient.prompt(...).call() 或 stream().chatResponse()。EmbeddingModel.call(new EmbeddingRequest(...))。RerankApiClient.rerank(query, documents, topN)。
返回结构文本内容、流式 chunk、completion token 统计。SnailEmbeddingResponse，包含 vectors、dimensions、costTimeMs。List<RerankResultItem>，包含 index 和 score。
usage 边界Client 流式路径由 TokenUsageCollectorAdvisor 提取并通过 ChatStreamResponse.completion 回传；Server 本地 chat 当前只返回文本。响应类有 usage 字段，但当前 convertEmbeddingResponse 主要填 vectors、dimensions、costTimeMs。无 token usage，只有排序分数。
缓存与复用ChatClientBuilder 按配置 ID 缓存 5 分钟，并用 API Key hash 判断是否失效。当前 builder 每次构建。当前 builder 每次构建。
典型降级SmartChunkStrategy 调用失败后保留原文切片；Agent Client 错误变成 error chunk。EmbeddingModelDimensionService 探测失败后使用并持久化默认维度 1024。DefaultRerankService 空结果或异常后按原 score 排序。

6. 异常与降级矩阵

位置触发条件当前行为排查入口
ModelResolveHandlerAgent 未配置模型且无默认 CHAT 模型。向 stream 写 Error: 未配置对话模型，complete，并设置 terminated。模型配置表、默认模型标记、provider 启用状态。
DispatchModelConfigAssembler配置缺失、密钥解密失败、字段校验失败。异常被 ModelResolveHandler 捕获，输出 Error: 模型配置加载失败。ModelConfigHandler、ServerModelSpecFactory。
ClientModelConfigValidator下发配置缺少 apiEndpoint、apiKey 或 modelKey。抛 IllegalArgumentException，Client 通过 ChatStreamResponse.error 回 Server。ChatDispatchRequest.ModelConfig。
*ModelRuntime.validatespec 缺少 adapterKey、baseUrl、apiKey 或 modelKey。抛 ModelAdapterException，错误码为 INVALID_MODEL_CONFIG。ChatModelRuntime、EmbeddingModelRuntime、RerankModelRuntime。
*ModelRuntime.build没有 adapter 的 adapterKey() 与 spec 匹配。抛 ModelAdapterException，错误码为 ADAPTER_NOT_FOUND。starter 自动装配、provider 依赖、adapterKey 大小写。
Server path caller-visible wrapper*ModelRuntime 在 validate/build/create 期间抛出 ModelAdapterException。Server builder 会捕获并包装成调用方可见的 ModelCallException：Chat 侧 ChatClientBuilder 使用 CHAT_CLIENT_BUILD_FAILED；Embedding/Rerank builder 当前都使用 CHAT_MODEL_BUILD_FAILED。ChatClientBuilder、EmbeddingClientBuilder、RerankClientBuilder。
ModelRuntimeHandler.decryptApiKey解密返回空或抛异常。抛 ModelCallException.API_KEY_DECRYPT_FAILED。密文、密钥配置、CryptoHelper。
ModelRuntimeHandler.chatStream流式订阅过程中模型报错。记录日志并调用请求中的 onError consumer。RagQAService 或调用方传入的 error handler。
QwenRerankModelAdapter响应 body 为 null、含 error、无 results 或结果为空。返回空列表；上层 RAG 降级。provider 响应体、rerankPath、模型名。
DefaultRerankServicererankModelId 为空、结果为空、ModelCallException 或其他异常。按候选原 score 降序截断。RerankHandler、知识库 rerankModelId、搜索参数。
EmbeddingModelDimensionService无法通过 API 探测 embedding 维度。使用默认维度 1024 并尝试写回配置。configJson.embeddingDimension、embedding API 日志。
ChatStreamObserverClient 返回 ChatStreamResponse.error。记录错误并向 Server writer 写纯文本 [ERROR] ...。Client 日志、error chunk、gRPC status。

7. 调试 checklist

症状检查顺序
Agent 对话直接提示未配置模型查 Agent 的 chatModelId；若为空，查是否有启用 provider 下的默认 CHAT 模型。
Agent 对话提示模型配置加载失败查 ModelConfigInfoDTO.modelType 是否为 CHAT，providerKey、apiEndpoint、modelKey 是否为空，API Key 是否能解密。
Client 报模型字段缺失打印 ChatDispatchRequest.ModelConfig，确认 apiEndpoint、apiKey、modelKey 已由 Server 组装。
找不到 adapter确认 adapterKey 是否为 openai-compatible 或 qwen-rerank；确认对应 starter 和 provider 模块在依赖中。
Chat usage 一直为 0查模型流最终 chunk 是否有 Usage；查 TokenUsageCollectorAdvisor 是否在 advisor 链中；非 OpenAI-compatible 的 cache token 可能为 0。
Embedding 向量库维度不对先看 configJson.embeddingDimension；为空时看探测 API 是否成功；失败会回退并写入 1024。
Rerank 没有效果但搜索没报错看 rerankEnabled、rerankModelId、enterRerankCount、resultCount；再看 provider 是否返回空结果，空结果会静默降级排序。
RAG 智能切片输出原文查 chunkModelId 是否为 CHAT，API Key 是否可用；SmartChunkStrategy 调用失败会保留原文。

最短定位法 先问：这是 Agent Client 路径还是 Server 本地路径？再问：当前 modelConfigId 经过的是 ChatModelRuntime、EmbeddingModelRuntime 还是 RerankModelRuntime？最后用 adapterKey 和 provider starter 验证落点。
