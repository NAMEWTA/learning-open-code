Maven 坐标 — snail-ai-feature-resource/pom.xml

字段值说明
groupIdcom.aizuda继承自根 POM open-java/snail-ai/pom.xml。
artifactIdsnail-ai-feature-resource资源领域 feature 模块，packaging=jar。
version${revision}与 monorepo 统一版本号；根 POM 在 dependencyManagement 中登记该 artifact。

模块定位：纯领域 JAR，不含独立 @SpringBootApplication。运行时由 snail-ai-server-admin 引入并参与组件扫描，ResourceService、存储策略 Bean 在 Admin 进程中注册。

ResourceService — 领域服务 API

类路径：com.aizuda.snail.ai.features.resource.ResourceService。标注 @Service、@RequiredArgsConstructor；写操作带 @Transactional。

方法签名返回行为摘要
upload (InputStream inputStream, String originalName, long fileSize, String bizType, Long bizId, Long creatorId) ResourcePO ① storageFactory.getDefault() ② store 得 storageKey ③ MimeTypeUtils.detect ④ insert 资源行 ⑤ getAccessUrl 回写 accessUrl。
load (Long resourceId) InputStream requireResource 后按 storageType 取策略，load(storageKey)。
delete (Long resourceId) void 资源不存在则直接返回；存储删除失败只 log.warn，随后删 DB 行。
getById (Long resourceId) ResourcePO 或 null 标准 selectById，不抛异常。
getByStorageKey (String storageKey) ResourcePO 或 null FileAccessController 反查资源元数据用。
updateBizType (Long resourceId, String bizType) void 预览提交时 DOCUMENT_PREVIEW → DOCUMENT；资源不存在则静默返回。
requireResource (Long resourceId) ResourcePO 不存在抛 SnailAiException("Resource not found: …")。

ResourcePO 关键字段 — sai_resource

字段类型说明
idLong自增主键；getAccessUrl 生成 URL 时可用。
storageKeyString存储后端唯一键；公开访问凭证，格式见 StorageKeyGenerator。
originalName / fileSize / mimeTypeString / Long / String上传元数据。
storageTypeStringLOCAL、MINIO、EXTERNAL。
accessUrlString对外 URL；LOCAL/MINIO 多为 /files/{storageKey}，EXTERNAL 为外部地址。
bizType / bizIdString / Long业务分区与关联 ID。
creatorIdLong上传用户；删除授权依据之一。

存储策略 API

ResourceStorageService — 策略接口

String getType();
String store(String bizType, String fileName, InputStream inputStream);
InputStream load(String storageKey);
void delete(String storageKey);
String getAccessUrl(Long resourceId, String storageKey);
boolean exists(String storageKey);
实现类getType()备注
LocalResourceStorageServiceLOCAL写入 snail-ai.resource.upload-dir；getAccessUrl 返回 /files/{storageKey}。
MinioResourceStorageServiceMINIO对象 key 与 storageKey 一致；访问 URL 形式与 LOCAL 相同，由 Admin 代理读取。

EXTERNAL 无对应 ResourceStorageService 实现；OpenAPI 等场景直接插入 ResourcePO，accessUrl 指向外部地址，不经 ResourceService.upload。

ResourceStorageFactory

方法行为

getDefault()get(resourceConfig.getStorageType())；上传链路默认入口。
