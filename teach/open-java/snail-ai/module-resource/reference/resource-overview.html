资源管理模块参考

一句话：资源模块把二进制文件存储、资源元数据和公开访问 URL 抽成通用底座；上层模块通过 resourceId、accessUrl 或外部 URL 复用它。

返回短课 · 构建 API 速查 · 上传访问切片 · L0 项目地图

边界提醒：/resource/** 是登录态管理接口；/files/** 被认证拦截器排除，访问控制主要依赖不可预测的 UUID storageKey、路径格式校验和存储类型限制。

Controller 与接口边界

入口方法权限职责
ResourceControllerPOST /resource/upload@LoginRequired上传单个 multipart 文件；请求可带 bizType、bizId。
ResourceControllerDELETE /resource/{id}@LoginRequired删除资源；非管理员只能删除自己创建的资源。

RagDocumentControllerPOST /document/upload/commit@LoginRequired把临时资源提升为正式 DOCUMENT，或跳过/覆盖后清理。

存储策略与 URL

类型实现或来源写入方式访问 URL代理限制
LOCALLocalResourceStorageService写入 ResourceConfig.uploadDir 下的相对路径。/files/{storageKey}可由 FileAccessController 代理；本地读取前会 normalize 并校验不能逃出根目录。
MINIOMinioResourceStorageService使用 MinIO SDK 上传到配置 bucket。/files/{storageKey}可由 FileAccessController 代理；首次使用时懒加载 client 并确保 bucket 存在。
EXTERNALOpenAPI 外部头像直接写入资源表不上传文件，storageKey 和 accessUrl 都可为外部 URL。外部 URLFileAccessController 不代理该类型。

StorageKeyGenerator 会清理业务类型和扩展名，生成 {bizType}/{yyyy}/{MM}/{dd}/{uuid}.{ext}。FileAccessController 的正则也要求这个结构，因此随意拼接或传自增资源 ID 都无法访问。

持久化字段与索引

字段含义常见来源
id资源主键，自增。ResourceMapper.insert 后生成。
storage_key后端文件唯一键；表上有唯一键 uk_storage_key。StorageKeyGenerator.generate 或外部 URL。
original_name用户上传时的原始文件名。Multipart 文件名或外部头像昵称派生名。
file_size文件大小字节数。上传文件大小；外部头像为 0。
mime_type媒体类型。MimeTypeUtils.detect。
storage_typeLOCAL、MINIO、EXTERNAL。存储策略或 OpenAPI 外部头像。
access_url前端访问或下载链接。LOCAL/MINIO 为 /files/{storageKey}；EXTERNAL 为外部 URL。
biz_type业务类型，含 GENERAL、AVATAR、ATTACHMENT、DOCUMENT、DOCUMENT_PREVIEW。上传请求或业务服务。
biz_id关联业务对象 ID。如 RAG ID、用户 ID 或 Agent ID。
creator_id创建人 ID。登录态用户；部分后台路径可能为空。

ResourceMapper 只继承 MyBatis-Plus BaseMapper<ResourcePO>，资源查询大多通过 Java 侧 LambdaQueryWrapper 构造。SQL 脚本还定义了 idx_biz(biz_type, biz_id) 和 idx_creator(creator_id)，对应管理端业务筛选和非管理员创建人过滤。

访问 URL 与权限边界

检查点源码事实排查建议
登录态/files/** 被 SnailJobWebMvcConfigurerAdapter 排除；/resource/** 方法使用 @LoginRequired。访问文件失败时不要只查登录态，先查 URL 是否符合 storageKey 格式。
访问凭证FileAccessController 从 URI 提取 storageKey，不接受 resourceId。不要把 /files/123 当作资源访问方式。
格式校验storageKey 必须是小写业务前缀、日期、UUID 和可选扩展名。若业务类型或扩展名被自定义污染，先看 StorageKeyGenerator 清理规则。
存储类型只代理 LOCAL 和 MINIO；EXTERNAL 返回 404。外部头像直接使用外部 accessUrl。
响应方式图片、PDF、文本、CSV、Markdown 可 inline；其他或 ?download=true 走 attachment。浏览器预览/下载行为异常时查 MIME 和 download 参数。
安全头返回 X-Content-Type-Options: nosniff，并设置 public max-age 86400 秒缓存。内容更新后若旧文件被缓存，先确认 URL 是否复用同一个 storageKey。

上层模块关系

上层能力资源模块参与点关键文件注意事项
RAG 直传文档RagDocumentService.uploadResourceIfPresent 把原始 bytes 上传为 DOCUMENT，再把资源 ID 写入 RagDocumentPO.resourceId。open-java/snail-ai/snail-ai-server/snail-ai-server-admin/src/main/java/com/aizuda/snail/ai/admin/service/rag/RagDocumentService.java文档行与资源行分离；RAG 状态、chunk 和向量不存放在资源表。
RAG 预览上传DocumentUploadPreviewService.preview 先落 DOCUMENT_PREVIEW；commit 时转为 DOCUMENT，skip/cancel 时删除。open-java/snail-ai/snail-ai-server/snail-ai-server-admin/src/main/java/com/aizuda/snail/ai/admin/service/rag/DocumentUploadPreviewService.java孤儿预览资源由定时任务兜底清理。
RAG 解析管线DocumentPipeline.parseContent 通过 ResourceService.load(document.getResourceId()) 获取文件流。open-java/snail-ai/snail-ai-server/snail-ai-server-features/snail-ai-feature-rag/src/main/java/com/aizuda/snail/ai/features/rag/pipeline/DocumentPipeline.java解析失败时应同时检查 sai_rag_document.resource_id、sai_resource.storage_type 和后端文件是否存在。
RAG 内容解析 SPIResourceDocumentContentResolver 作为 admin 层 @Primary 实现，也通过资源库加载文档流。open-java/snail-ai/snail-ai-server/snail-ai-server-admin/src/main/java/com/aizuda/snail/ai/admin/config/ResourceDocumentContentResolver.java当前 DocumentPipeline 直接注入 ResourceService，该 SPI 代表扩展边界但不是本管线唯一事实来源。
用户头像UserService.updateUserAvatar 要求资源存在且 bizType=AVATAR，再保存到 UserPO.resourceId。open-java/snail-ai/snail-ai-server/snail-ai-server-admin/src/main/java/com/aizuda/snail/ai/admin/service/UserService.java普通资源不能直接当头像绑定。
OpenAPI 外部头像OpenApiUserService.createExternalAvatarResource 创建 EXTERNAL 资源记录并绑定用户。open-java/snail-ai/snail-ai-server/snail-ai-server-openapi/src/main/java/com/aizuda/snail/ai/openapi/service/OpenApiUserService.java外部头像不上传文件，不由 /files/** 代理。
Agent 展示头像AgentPO.avatar 是字符串 URL；后端当前不强制保存为资源 ID。open-java/snail-ai/snail-ai-server/snail-ai-server-persistence/snail-ai-biz-storage/snail-ai-biz-template/src/main/java/com/aizuda/snail/ai/persistence/agent/po/AgentPO.javaAgent 可以显示资源 URL，但持久化边界不同于用户头像。

排查路径

上传失败
  ├─ 看 ResourceAdminService 是否拿到 MultipartFile 和当前用户
  ├─ 看 ResourceConfig.storageType 是否匹配已注册 ResourceStorageService
  └─ 看 Local/MinIO store 是否抛 SnailAiException

有资源行但文件打不开
  ├─ 查 sai_resource.storage_key 是否符合 /files/** 正则
  ├─ 查 storage_type 是否为 LOCAL 或 MINIO
  ├─ 查后端文件或 MinIO object 是否存在
  └─ 查响应 MIME 与 download 参数

RAG 文档无法解析
  ├─ 查 sai_rag_document.resource_id
  ├─ 查 ResourceService.load 是否能读出 InputStream
  └─ 再进入 parser、chunker、向量写入链路
