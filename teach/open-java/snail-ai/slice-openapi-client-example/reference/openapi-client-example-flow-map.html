3. 配置、URL 和认证头

配置/字段SDK 使用点排查含义
snail-ai.openapi.enabledSnailAiOpenApiAutoConfiguration 条件装配。为 false 时，OpenApiDemoController 也不会启用。
snail-ai.openapi.server-hostOpenApiHttpInvokeHandler#buildBaseUrl。为空时回退到 snail-ai.server.host。
snail-ai.openapi.web-port拼接 Server HTTP 端口。Server 默认是 8900，不是 gRPC 的 18888。
snail-ai.openapi.prefix追加到 base URL。Server 默认 context path 是 /snail-ai，SDK 默认 prefix=snail-ai 正好对应这段。
snail-ai.app-id写入 Snail-Ai-App-Id。它不是调用 /demo 时传入的 Header。
snail-ai.token写入 Snail-Ai-Token。Server 会和应用表里的 token 做常量时间比较。
snail-ai.openapi.chat-timeout-msSDK SSE 请求超时。只影响客户端等待流的时间；Server 同步 chat 仍固定等待 300000 ms。
 
private String buildBaseUrl() {
    String host = openApiProperties.getServerHost();
    if (host == null || host.trim().isEmpty()) {
        host = agentProperties.getServer().getHost();
    }
    String protocol = openApiProperties.isHttps() ? "https" : "http";
    int port = openApiProperties.getWebPort();
    String prefix = openApiProperties.getPrefix();
    host = host.replaceAll("/$", "");
    StringBuilder url = new StringBuilder(protocol).append("://").append(host);
    if ((openApiProperties.isHttps() && port != 443) || (!openApiProperties.isHttps() && port != 80)) {
        url.append(":").append(port);
    }
    if (prefix != null && !prefix.trim().isEmpty()) {
        if (!prefix.startsWith("/")) {
            url.append("/");
        }
        url.append(prefix);
    }
    return url.toString();
}

@PostMapping("/agent/{agentId}/chat/sync")
public Result<OpenApiChatSyncResponse> chatSync(@PathVariable Long agentId,
                                                 @RequestBody OpenApiChatRequest request) {
    request.setAgentId(agentId);
    log.info("Sync chat request: agentId={}, content={}", agentId, request.getContent());
    return chatClient.chatSync(request);
}

@GetMapping(value = "/agent/{agentId}/chat/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public Flux<ServerSentEvent<String>> chatStream(@PathVariable Long agentId,
                                                 @RequestParam String openId,
                                                 @RequestParam String content,
                                                 @RequestParam(required = false) String conversationId) {
    OpenApiChatRequest request = new OpenApiChatRequest();
    request.setAgentId(agentId);
    request.setOpenId(openId);
    request.setContent(content);
    request.setConversationId(conversationId);
    return chatClient.chatStream(request)
            .map(event -> ServerSentEvent.<String>builder(event.getData())
                    .event(event.getType())
                    .build());
}

5. 请求字段与 Server 边界

字段示例/SDK 写入Server 当前约束排查含义
agentId示例从 path 补到 OpenApiChatRequest。@NotNull。不要只在 URL path 有值，最终请求体也必须有值。
openId流式示例从 query 写入；同步示例依赖请求体。@NotBlank，并由 OpenApiUserResolver 做 appId + openId 解析。映射不存在会返回认证类错误 status=5001。
conversationId流式示例 query 标为可选；同步示例依赖请求体。当前源码是 @NotBlank。文档和示例的“可选”说法需要按源码校正。
content流式示例从 query 写入；同步示例依赖请求体。@NotBlank。最终进入 Server 的 AgentChatCommand.content。
disabledMcpServerIds可由 SDK 请求体透传。Server 转入 Agent 链后由 McpHandler 消费。只影响本次请求的 MCP 可用集合。
disabledSkillIds可由 SDK 请求体透传。Server 转入 Agent 链后由 SkillAgentChatHandler 消费。只影响本次请求的 Skill 可用集合。
sidSnailAiOpenApi.ChatBuilder 当前没有暴露 setter；直接构造 DTO 可以设置。Server 为空时生成 UUID。并发提交排查时可在直接 DTO 调用中显式设置。
timeoutDTO 字段存在。Server chat 服务当前未消费。同步超时不能靠请求体调整。

6. SSE 消费与事件类型

private void parseSseStream(BufferedReader reader, FluxSink<OpenApiChatStreamEvent> sink) throws Exception {
    String currentEvent = "text";
    String line;
    StringBuilder dataBuffer = new StringBuilder();

    while ((line = reader.readLine()) != null) {
        if (sink.isCancelled()) {
            break;
        }
        if (line.startsWith("event:")) {
            currentEvent = line.substring(6).trim();
        } else if (line.startsWith("data:")) {
            dataBuffer.append(line.substring(5));
        } else if (line.isEmpty()) {
            if (!dataBuffer.isEmpty()) {
                sink.next(OpenApiChatStreamEvent.of(currentEvent, dataBuffer.toString()));
                dataBuffer.setLength(0);
            }
            currentEvent = "text";
        }
    }
}
事件Server 产生方式SDK 看到的 data调用方处理
textOpenApiChatService 解析 ChatStreamResponse.TYPE_TEXT 后发出。一个 JSON 字符串，内容来自 ChatStreamResponse。解析后拼接 content，或直接转发给浏览器。
thinkingServer 解析 ChatStreamResponse.TYPE_THINKING 后发出。一个 JSON 字符串，内容来自 ChatStreamResponse。按产品策略显示或忽略。
doneChatStreamWriter.complete() 时由 Server 发出。{"conversationId":"...","fullText":"..."}。关闭流、落库完整文本或更新 UI 状态。
errorServer 流式服务 catch 异常或 writer error 时发出。{"message":"..."}。显示错误并停止等待后续 chunk。

SDK 的 SSE parser 只识别 event: 和 data: 行。如果认证或参数校验在进入流前返回普通 JSON envelope，HTTP 状态又不是 4xx/5xx，当前 parser 可能不会产生 error event。此时用 curl -v 看响应体，或看 Server 的 OpenApiExceptionHandler 日志。

7. 同步响应

层次返回类型成功字段错误处理
Server ControllerResult<OpenApiChatSyncResponse>status=1、message=success、data.conversationId、data.content、data.durationMs。OpenApiExceptionHandler 返回 Result.fail(...)。
SDK 类型化接口Result<OpenApiChatSyncResponse>直接反序列化 Server JSON。HTTP 状态 >= 400 抛 SnailAiException；业务失败仍是 Result。
SnailAiOpenApi.ChatBuilder#executeOpenApiChatSyncResponse只返回 data。Result.status != 1 时抛 SnailAiException。
示例 ControllerResult<OpenApiChatSyncResponse>把 SDK 类型化接口结果原样返回给调用方。没有额外包装或降级。

同步响应不包含 token usage、thinking、cache tokens 或工具轨迹。Server 同步路径通过 CollectingChatStreamWriter 聚合 text chunk，最终字段只来自 OpenApiChatSyncResponse。

8. 常见 HTTP/SSE 错误

症状当前源码行为优先检查
/demo 请求能进来，但 Server 认证失败。SDK 发 Server 请求时才加 Snail-Ai-App-Id 和 Snail-Ai-Token。示例工程 snail-ai.app-id、snail-ai.token 是否和 Server 应用管理一致。
404 或打到错误端口。SDK base URL 由 serverHost、webPort、prefix 拼出。web-port 是否是 8900；prefix 是否匹配 Server context path snail-ai。
同步返回 status=5001。Server 认证拦截器或 OpenApiUserResolver 抛 SnailAiAuthenticationException。应用状态、token、openId 映射、关联平台用户。
同步返回 status=0 参数错误。OpenApiExceptionHandler 返回字段校验消息。agentId、openId、conversationId、content 是否都有值。
SSE 直接完成但没有任何 event。SDK parser 没解析到 data: 行时不会发 event。看原始响应是否是普通 JSON envelope；检查认证和参数校验是否在流前失败。
SSE 收到 error event。可能来自 Server 流式服务的 emitError，也可能是 SDK 在 HTTP 状态 >= 400 或网络异常时合成。区分 data.message、HTTP 状态、SDK Debug 日志和 Server 日志。
同步一直等待。Server 同步路径等待 CollectingChatStreamWriter.complete()，固定最多 300000 ms。Client 是否回调完成，Agent 链是否提前写了非 JSON 错误，Server 是否调用 writer complete。
文档示例无法编译。当前源码的 SnailAiOpenApi.ChatBuilder 没有 listener(...)。改用 stream().subscribe(...) 或直接注入 OpenApiChatClient。

9. 调试 checklist

顺序检查项判断标准
1确认示例工程是否启用 OpenAPI SDK。snail-ai.openapi.enabled=true，OpenApiDemoController Bean 生效。
2确认 Server HTTP 地址。SDK 日志中的 base URL 应类似 http://127.0.0.1:8900/snail-ai。
3确认认证凭证来源。检查示例工程配置里的 app-id/token，不是调用 /demo 的 Header。
4确认请求体完整。agentId、openId、conversationId、content 不为空。
5用直接 curl 对照 Server。同样的 OpenApiChatRequest 直接打 /snail-ai/openapi/v1/agent/chat 能看到 SSE。
6打开 SDK Debug 日志。com.aizuda.snail.ai.openapi.client 能打印请求 URL、HTTP 状态和 SSE 异常。
7查看 Server OpenAPI 日志。认证失败看 OpenApiAuthInterceptor 和 OpenApiExceptionHandler；流中失败看 OpenApiChatService。
8再进入已有 Server 切片。如果 Server OpenAPI 已经进入 chat 服务，但无模型输出，转到 slice-openapi-chat 查 Agent 链和 Client 执行。
