Snail AI Starter 参考

定位：snail-ai-starter 是 Snail AI Server 的 Spring Boot 启动壳。它聚合服务端入口、存储适配、Actuator、MCP Server 能力，承载默认配置、日志配置和内置管理后台静态资源。

模块结构

snail-ai-starter/ ├── pom.xml ← 聚合依赖、打包产物和主类声明 └── src/ └── main/ ├── java/ │ └── com/ │ └── aizuda/ ← Spring Boot 启动代码根包 └── resources/ ├── admin/ ← 已构建管理后台静态资源 ├── application.yml ← HTTP、DB、gRPC、资源、记忆配置 └── logback-boot.xml ← 控制台与滚动文件日志

边界：admin/assets/** 是构建产物，适合用于确认静态资源交付路径，不适合逐文件讲解业务逻辑。

关键源码索引

文件职责学习时怎么用
snail-ai-starter/pom.xml声明 starter 自身、聚合依赖、最终文件名 snail-ai-server、Spring Boot repackage 主类与 exec classifier。判断一个模块是否被装进 Server 进程，以及可执行 JAR 的实际文件名。
snail-ai-starter/src/main/java/com/aizuda/snail/ai/starter/SnailAiSpringbootApplication.javaSpring Boot 主类，扫描 com.aizuda.snail.ai，启用配置属性、异步、调度，并设置默认时区。定位应用启动入口和 Bean 扫描边界。
snail-ai-starter/src/main/java/com/aizuda/snail/ai/starter/listener/StartListener.java监听 ContextRefreshedEvent，打印 Logo，触发所有自定义 Lifecycle.start()。排查存储注册、搜索/向量后端初始化等启动钩子。
snail-ai-starter/src/main/java/com/aizuda/snail/ai/starter/listener/EndListener.java监听 ContextClosedEvent，触发所有自定义 Lifecycle.close()。排查关闭时资源释放和连接清理。
snail-ai-starter/src/main/resources/application.yml默认 HTTP、数据库、gRPC、静态资源、资源存储、Skill、加密、短期记忆和 MyBatis-Plus 配置。排查端口、路径、存储、上传目录和记忆模式。
snail-ai-starter/src/main/resources/logback-boot.xml控制台 appender、INFO/WARN/ERROR 按级别滚动文件、异步日志队列。判断日志输出路径、保留天数和日志级别分流。

依赖与运行时装配

依赖进入 starter 后带来的能力边界说明
snail-ai-server-admin管理后台 REST/SSE 接口。具体 Controller 和服务逻辑属于 Server API 入口主题。
snail-ai-server-openapi外部集成 OpenAPI。鉴权和对话接口不在 starter 内实现。
snail-ai-mysql-storage默认 MySQL 业务存储适配。默认数据源仍由 application.yml 决定。
snail-ai-postgres-storagePostgreSQL 业务存储适配。配置需切换 driver 与 JDBC URL。
snail-ai-dm-storage达梦存储适配。当前配置中仅保留示例注释，生产使用需专项验证。
spring-boot-starter-actuator健康检查和运行时信息端点。用于部署验证，例如 /actuator/health。
spring-ai-starter-mcp-server-webmvcSpring AI MCP Server WebMVC 能力。MCP 工具和业务绑定应进入 Skill/MCP 专题。

反向依赖：其他 Maven 模块不反向依赖 snail-ai-starter；根 pom.xml 只是把它列为 reactor 模块。部署脚本、浏览器入口和 Agent Client 连接最终依赖 starter 暴露的运行时边界：HTTP :8900/snail-ai 与 gRPC :18888。

启动生命周期

main()
  -> setDefaultTimeZone("Asia/Shanghai")
  -> SpringApplication.run(...)
  -> 扫描 com.aizuda.snail.ai 下的 Bean
  -> ContextRefreshedEvent
  -> StartListener 调用所有 Lifecycle.start()
  -> 应用运行
  -> ContextClosedEvent
  -> EndListener 调用所有 Lifecycle.close()
阶段触发点典型影响
Spring Boot 主启动SnailAiSpringbootApplication.main装配 classpath 中所有 Snail AI 模块。
自定义启动钩子StartListener注册 PgVector、Milvus、Elasticsearch 等存储工厂实现。
Spring 生命周期钩子SmartLifecycleServer 端 gRPC 服务监听 snail-ai.server.grpc-port。
自定义关闭钩子EndListener关闭数据源、连接或其他自定义资源。

默认配置速查

配置项默认值用途修改时注意
server.port8900Admin API、OpenAPI、SSE 和静态后台共用 HTTP 端口。反向代理和文档示例需同步。
server.servlet.context-path/snail-aiHTTP 上下文路径。静态资源引用也按该路径生成。
spring.web.resources.static-locationsclasspath:admin/内置管理后台静态资源目录。不要把它误认为前端源码目录。
spring.profiles.activedev默认激活 dev profile。当前 starter 目录未提供额外 profile 文件，实际覆盖通常来自启动参数或外部配置。
spring.datasource.*MySQL、本地 snail_ai 库、root/123456业务关系库连接。生产环境必须改密码和连接地址。
snail-ai.server.grpc-port18888Agent Client 连接 Server 的 gRPC 端口。与防火墙、Client 配置和部署网络一起检查。
snail-ai.skill.upload-dir./upload/skillsSkill 上传和存放目录。生产环境应挂载持久化磁盘。
snail-ai.resource.storage-typeLOCAL资源文件使用本地存储。多节点部署通常需要切换到 MinIO。
snail-ai.memory.short-term.store-typememory短期记忆保存在应用内存。重启丢失；需要持久化时切换为 db。
logging.configclasspath:logback-boot.xml指定日志配置文件。日志路径默认为 ./data/log/snail-ai/。

启动与部署边界

打包名核验：snail-ai-starter/pom.xml 设置 <finalName>snail-ai-server</finalName>，Spring Boot repackage 设置 <classifier>exec</classifier>，所以实际可执行包是 target/snail-ai-server-exec.jar。若部署目录中出现 snail-ai-starter.jar，应明确它是部署时重命名后的文件。

场景命令或入口starter 负责什么不负责什么
本地启动mvn spring-boot:run -pl snail-ai-starter启动 Spring Boot Server、暴露 HTTP 和静态后台。不启动 MySQL、PgVector、Milvus、Elasticsearch、MinIO。
访问后台http://localhost:8900/snail-ai从 classpath:admin/ 返回前端壳和资源。不提供前端源码构建流程。
Agent Client 连接18888通过配置提供 Server gRPC 端口。Client 节点启动和模型调用属于 Agent Client。
JAR 部署java -jar target/snail-ai-server-exec.jar以同一主类和配置运行 Spring Boot repackage 产物。进程守护、HTTPS、外部依赖、备份策略和部署时重命名属于部署层。

排查入口

现象先看哪里下一步
浏览器打不开后台server.port、context-path、static-locations确认 admin/index.html 是否在 classpath 中。
Agent Client 连接失败snail-ai.server.grpc-port再查 gRPC Server Bean、网络和 Client 配置。
启动后数据库报错spring.datasource.*确认数据库已启动并导入 SQL。
上传文件丢失snail-ai.resource.*、snail-ai.skill.upload-dir检查本地目录或 MinIO 挂载。
日志找不到logging.config、logback-boot.xml查看 ./data/log/snail-ai/。
RAG 或模型调用异常starter 只确认配置是否装载进入对应 Server、Models 或 Agent Client 主题继续排查。
