MCP 与 Skill 工具怎样注入一次对话

本课只完成一个目标：从 Agent 绑定 MCP/Skill，追到模型实际可见的 ToolCallback，再追到对话结束时的资源清理。

层级：L2 垂直切片 预计 15 分钟 主线：Server -> Client -> Model

1. 先看完整链路

Snail AI 的 Server 不直接把 MCP 或 Skill 变成模型工具。Server 只确认 Agent 绑定关系，把可执行线索压成描述符，再通过 /chat/dispatch 发给 Agent Client。真正连接 MCP server、注册 read_skill、注入 ChatClient，都发生在 Client。

绑定AgentService 写 sai_agent_mcp_server 与 sai_agent_skill
描述符McpHandler 与 SkillAgentChatHandler 读取绑定并过滤禁用 ID
分发LlmCallHandler 把描述符放入 ChatDispatchRequest
入口ChatDispatchStreamingHandler 解析 JSON 并启动 runtime
解析ToolRuntime 生成 MCP callbacks 与 read_skill
调用DefaultChatClientFactory 用 defaultTools 暴露给模型
清理ChatSessionRuntime 在完成或错误时关闭 MCP resolver

2. 描述符不是工具

McpHandler 只把 McpServerRef 转成 McpServerDescriptor：字段包括 transportType、baseUri、endpoint、command、args、envVars、authType 和 authConfig。SkillAgentChatHandler 只生成 SkillDescriptor：字段包括 id、name、description、version、skillPrompt。

判断技巧：如果 Server 日志里 descriptor 已经存在，但模型看不到工具，下一站不是继续查 McpHandler，而是查 Client 的 ToolRuntime.resolve 和对应 resolver。

3. Client 才建立工具回调

ToolRuntime 的顺序很关键：先移除内置 RAG MCP descriptor，再调用 ClientMcpToolResolver 连接外部 MCP server；然后注册基础 shell、http_request，按 RAG smart mode 注册 rag_search，最后由 ClientSkillToolResolver 追加 Skill 系统提示并注册 read_skill。

模型选择技能：系统提示列出可用 Skill 名称、描述和支撑文件目录。
模型调用工具：模型发起 read_skill(skillName)，Spring AI 的 ToolCallingAdvisor 调用 ReadSkillTool。
Client 回调 Server：ReadSkillTool 调用 RpcClient.fetchSkillContent，URI 是 /callback/skill/content。
Server 返回内容：SkillContentCallbackHandler 查询 sai_skill 与 sai_skill_file，返回 skillContent、version、files。
Client 物化文件：ReadSkillTool 立即返回 skillContent，并用虚拟线程把支撑文件写入临时目录。

模型真正能调用工具，是因为 DefaultChatClientFactory 把所有 callbacks 传给 defaultTools，并添加 ToolCallingAdvisor。对话 completion 或 error 后，ChatSessionRuntime.cleanup 会让活跃计数减一，并调用 ToolResolution.close，进而关闭本轮 ClientMcpToolResolver 持有的 MCP clients。

4. 检索练习

暂停回忆：如果一次对话里模型能看到 Skill 名称，但调用 read_skill 后拿不到支撑文件，你按什么顺序查？

参考答案与反馈

先查 Server 端 SkillContentCallbackHandler 是否从 sai_skill_file 返回了 files；再查 Client 端 ReadSkillTool.materializeFiles 是否因路径、编码或权限失败；最后确认模型是否在 read_skill 返回后立刻访问文件，因为支撑文件写入是异步的。不要先查 SkillAgentChatHandler，它只下发描述符。

主要阅读来源：McpHandler.java、SkillAgentChatHandler.java、ToolRuntime.java、ClientMcpToolResolver.java、ClientSkillToolResolver.java、ReadSkillTool.java、DefaultChatClientFactory.java。下一步打开 切片速查，按 DTO 字段、transport 边界和 checklist 复盘；遇到任一阶段不清楚，可以继续向 agent 追问具体调用栈。
