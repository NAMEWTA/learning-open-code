MCP 与 Skill 工具注入速查

这份参考用于排查一次 Snail AI 对话里的工具从哪里来、什么时候变成 ToolCallback、哪些 transport 当前可用、认证字段是否生效，以及结束时哪些资源会被清理。

1. 垂直切片总图

绑定AgentService.create 或 AgentService.update 保存 mcpServerIds 与 skillIds。
描述符McpHandler 生成 McpServerDescriptor；SkillAgentChatHandler 生成 SkillDescriptor。
gRPC dispatchLlmCallHandler 把描述符装入 ChatDispatchRequest，通过 /chat/dispatch 发给 Client。
Client resolverChatDispatchStreamingHandler 启动 ChatSessionRuntime，再调用 ToolRuntime.resolve。
ToolCallbackClientMcpToolResolver 连接 MCP；ClientSkillToolResolver 注册 read_skill。
模型调用DefaultChatClientFactory 注入 defaultTools 与 ToolCallingAdvisor。
cleanupChatSessionRuntime.cleanup 调用 ToolResolution.close，关闭本轮 MCP clients。

2. 绑定入口与表字段

阶段源码关键字段含义
Agent 请求 AgentRequestVO mcpEnabled、skillEnabled、mcpServerIds、skillIds 开关控制对话时是否读取绑定；ID 列表控制绑定哪些 MCP server 或 Skill。
创建 Agent AgentService.create mcpServerService.updateAgentMcpServers、skillService.updateAgentSkills 插入 AgentPO 后，如果请求带 ID 列表，就写关联表。
更新 Agent AgentService.update mcpServerIds != null、skillIds != null 传入列表时先删旧关联再插入新关联；传 null 表示不改绑定。
MCP 关联表 AgentMcpServerPO agentId、mcpServerId McpToolService.getMcpServersForAgent 按 agentId 读取。
Skill 关联表 AgentSkillPO agentId、skillId SkillHandler.getSkillsWithContentForAgent 按 agentId 读取。

3. Server 下发的 DTO 与字段

DTO字段来源Client 用途
ChatDispatchRequest mcpServers McpHandler 写入 ctx.mcpServerDescriptors ToolRuntime 交给 ClientMcpToolResolver。
ChatDispatchRequest skills SkillAgentChatHandler 写入 ctx.skillDescriptors ClientSkillToolResolver 构建 Skill 列表并注册 read_skill。
McpServerDescriptor id、name、transportType McpServerPO 通过 McpServerRef 暴露 选择 transport，日志定位 server，建立 MCP client。
McpServerDescriptor baseUri、endpoint McpHandler.toDescriptor Streamable HTTP transport 的连接地址。
McpServerDescriptor command、args、envVars McpHandler.toDescriptor 解析 JSON Stdio transport 的进程命令、参数和环境变量。
McpServerDescriptor authType、authConfig McpHandler.toDescriptor 解析 JSON 当前 Client resolver 未消费；只作为下发字段存在。
SkillDescriptor id、name、description、version、skillPrompt SkillPO name 用于 read_skill 参数匹配；version 用于临时目录路径。
SkillContentRequest skillId、agentId ReadSkillTool 当前 ReadSkillTool 设置 skillId，agentId 为 null。
SkillContentResponse skillContent、version、files SkillContentCallbackHandler 返回完整 Skill 指令；支撑文件由 Client 异步写入临时目录。

4. Client resolver 顺序

顺序代码位置行为失败处理
0 ToolRuntime.filterRagMcpDescriptor 移除名称等于内置 RAG MCP server 的 descriptor。 直接修改 dispatchRequest.mcpServers 列表。
1 ClientMcpToolResolver.resolve 逐个连接 MCP server，用 SyncMcpToolCallbackProvider 生成 callbacks。 单个 server 失败只记录 warn，继续处理其他 server。
2 BaseToolResolver.resolve 注册 shell 与 http_request，工作目录按 agentId/conversationId 隔离。 目录创建失败只记录 warn，工具仍可能注册。
3 ClientRagToolResolver.resolve 仅在 RAG 启用、存在 ragIds 且非 forced mode 时注册 rag_search。 外层 addResolvedTools 捕获异常并记录 warn。
4 ClientSkillToolResolver.resolve 向 systemPrompt 追加 Skill 渐进式披露说明，并注册 ReadSkillTool。 失败时记录 warn，返回已解析或空工具列表。
5 CustomToolCallbackProvider.getToolCallbacks 追加 Spring 容器中非内部包的本地 @Tool Bean。 扫描发生在初始化阶段；运行时返回缓存数组。
6 DefaultChatClientFactory.build 把 callbacks 传给 defaultTools，并添加 ToolCallingAdvisor。 如果 callbacks 为空，模型只能普通对话。

5. MCP transport 边界

transport枚举值Server 连接测试Client resolver结论
SSE McpTransportTypeEnum.SSE = 1 McpServerService.createTestClient 抛出 deprecated 异常。 ClientMcpToolResolver.connectMcpServer 记录 warn 并返回 null。 字段保留，当前不可用。文档仍有 SSE 说明，但源码按 MCP SDK 2.x deprecated 处理。
Streamable HTTP McpTransportTypeEnum.STREAMABLE_HTTP = 2 使用 HttpClientStreamableHttpTransport；支持 baseUri 加 endpoint，或仅 endpoint。 同样使用 HttpClientStreamableHttpTransport 并调用 client.initialize()。 当前主路径。缺少地址时不会注册工具。
Stdio McpTransportTypeEnum.STDIO = 3 使用 ServerParameters、StdioClientTransport；需要 command。 根据 command、args、envVars 启动本地进程并初始化。 当前可用。排查重点是命令路径、参数 JSON、环境变量和进程权限。

6. 认证现状

当前源码事实：authType 与 authConfig 会在 Admin 保存、在 McpHandler 下发到 McpServerDescriptor，但 ClientMcpToolResolver 和 McpToolService 构造 HTTP/Stdio transport 时没有读取这些字段。因此“配置已存在”和“请求已带认证头”不是一回事。
位置源码事实排查含义
McpAuthTypeEnum 枚举为 NONE(0)、API_KEY(1)、OAUTH(2)、BASIC(3)。 与文档中 Bearer/OAuth 编号描述存在差异，调试时以源码枚举为准。
McpServerService.create 保存 authType，把 authConfig 序列化为 JSON。 数据库有配置不代表 transport 会自动带 header。
McpHandler.toDescriptor 把 authType 和解析后的 authConfig 放入 descriptor。 gRPC 下发层没有丢字段。
ClientMcpToolResolver 连接 Streamable HTTP 与 Stdio 时只读取地址、命令、参数、环境变量。 认证失败时不要误判为 descriptor 丢失；当前更可能是 runtime 未实现认证注入。

7. Skill support files 边界

阶段源码字段注意点
上传 Skill SkillService.upload skillContent、hasFiles、version 上传 zip 时解析 SKILL.md frontmatter，正文去掉 frontmatter 后存入 skillContent。
保存支撑文件 SkillService.saveSkillFilesFromZip filePath、content、encoding 当前保存为 utf-8 文本；在线编辑也写 sai_skill_file。
读取 Skill SkillContentCallbackHandler.handle skillContent、version、files 按 skillId 查主表和文件表，返回给 Client。
注册工具 ClientSkillToolResolver.buildSkillsList 支撑文件目录为 skillTempDir/skillId/version 模型在系统提示里只能看到目录路径和 Skill 摘要，完整指令必须通过 read_skill 获取。
物化文件 ReadSkillTool.materializeFiles SKILL.md 与每个 SkillFile 写入发生在虚拟线程中，read_skill 先返回内容；文件访问存在短暂竞态窗口。
重复读取 ReadSkillTool.contentCache skillId -> skillContent 同一 ReadSkillTool 实例内重复调用命中缓存，不再回调 Server。

8. 模型工具调用与 cleanup

阶段源码关键行为
构建 ChatClient DefaultChatClientFactory.build 创建模型、创建 ToolCallingManager、添加 ToolCallingAdvisor、调用 defaultTools。
执行流式调用 ClientChatExecutor.streamResponses 把 Prompt、Advisor 参数和工具化后的 ChatClient 连接起来。
模型发起 tool call Spring AI ToolCallingAdvisor Snail AI 的边界是把 callbacks 提供给 Spring AI；具体调用分发由 Spring AI 接管。
正常结束 ChatSessionRuntime.completionConsumer 先转发 completion，再执行 cleanup(currentResolution)。
异常结束 ChatSessionRuntime.errorConsumer 先转发 error，再执行相同 cleanup；启动阶段异常也会 cleanup。
MCP 关闭 ToolResolution.close 与 ClientMcpToolResolver.close 逐个调用 McpSyncClient.close，然后清空 connectedClients。

9. 失败与调试 checklist

现象优先检查判断依据下一步
模型看不到任何 MCP 工具 AgentService 绑定、McpHandler descriptor、ToolRuntime.resolveMcpTools Server 有 descriptor 但 Client warn 连接失败，说明问题在 transport 或地址。 检查 transportType、baseUri、endpoint、command、Client 运行环境网络。
SSE MCP 没注册 ClientMcpToolResolver.connectMcpServer SSE 当前记录 deprecated warn 并返回 null。 改用 Streamable HTTP 或 Stdio 配置。
配置了认证仍失败 McpHandler 下发字段、ClientMcpToolResolver transport 构造 当前 resolver 不消费 authType 与 authConfig。 先确认 MCP server 是否允许无认证或通过环境变量认证；需要源码改造时从 transport builder 注入 header 开始设计。
Skill 名称可见但没有 read_skill ClientSkillToolResolver.resolve 如果 request.getSkills() 为空，不会注册 ReadSkillTool。 回查 SkillAgentChatHandler 是否被 skillEnabled、禁用 Skill ID 或关联表过滤。
read_skill 返回 Skill not found ReadSkillTool.skillRegistry registry key 是 Skill 的 name，参数必须完全匹配。 看系统提示里的 Skill 名称，确认模型调用参数没有别名或翻译。
read_skill 回调失败 RpcClient.fetchSkillContent、GrpcClientInvokeHandler、SkillContentCallbackHandler URI 是 /callback/skill/content；Server 返回 status != 1 会抛 CallbackServerErrorException。 查 Server 是否能按 skillId 找到 SkillPO，以及 Client 回调 channel 是否可用。
支撑文件不存在 SkillContentCallbackHandler 返回 files，再查 ReadSkillTool.materializeFiles 主内容同步返回，文件异步写入；日志可能有 materialize warn。 检查文件路径、父目录创建、编码是否为 base64 或 utf-8，以及模型是否过早访问。
对话结束后 MCP 进程未退出 ChatSessionRuntime.cleanup 与 ClientMcpToolResolver.close completion/error 正常触发时会关闭本轮 resolver；如果流没有 completion/error，cleanup 可能不执行。 查 ChatDispatchStreamingHandler 是否发送 completion/error，Client stream 是否卡住，MCP client close 是否抛异常。
RAG MCP 被过滤 ToolRuntime.filterRagMcpDescriptor 名称等于内置 RAG MCP server 的 descriptor 会被移除。 确认是否应改走 ClientRagToolResolver 的 rag_search，不要误判为外部 MCP 丢失。
