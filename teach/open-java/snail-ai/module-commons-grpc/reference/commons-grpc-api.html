SnailSpringContext

方法签名约束与返回值边界 / 异常调用示例
getBeanFactory() static ListableBeanFactory getBeanFactory() 优先返回 BeanFactoryPostProcessor 注入的 factory，否则回退 ApplicationContext。 两者都未注入时抛 SnailAiException。 SnailSpringContext.getBeanFactory().getBean(MyService.class)
getContext() static ApplicationContext getContext() 可能为 null（Spring 尚未回调 setApplicationContext）。 非 Spring 环境或未装配时为 null。 var ctx = SnailSpringContext.getContext();
getBean* static <T> T getBean(String name)
static <T> T getBean(Class<T>)
static <T> T getBean(String, Class<T>)		 按名称或类型取 Bean；失败时记录 error 日志并返回 null（不抛异常）。 BeansException、NullPointerException 被吞掉；调用方必须判空。 var svc = SnailSpringContext.getBean(RpcClient.class);
getBeanByType static <T> T getBeanByType(Class<T> clazz) 直接委托 getBeanFactory().getBean(clazz)；Bean 不存在时抛 Spring 异常。 与 getBean(Class) 不同：这里会抛异常而不是返回 null。 SnailSpringContext.getBeanByType(JsonUtil.class)

Bean 名固定为 snailAiSpringContext，@Order(HIGHEST_PRECEDENCE) 保证尽早拿到 factory。
