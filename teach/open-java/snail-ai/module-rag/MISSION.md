# 使命：RAG 功能模块

## 为什么
学习者需要把 Snail AI 的 RAG 能力放回 Server 架构中理解，能快速判断文档摄入、检索召回、重排和 Prompt 注入分别落在哪一层源码里。掌握这个边界后，后续才能安全地下钻到上传摄入切片和对话检索切片，而不会把整个 RAG 链路塞进一节巨型课。

## 成功的样子
- 能说清 RAG 模块在 Snail AI 中承担“上下文能力”而不是“模型调用本身”的职责。
- 能从 Admin 入口定位到摄入管线、检索管线、策略层和存储依赖。
- 能判断一个 RAG 相关变更应该放进 `module-rag` 总览、`slice-rag-ingestion` 还是 `slice-rag-retrieval`。
- 查阅 `reference/rag-api.html` 时，能根据 DTO 字段、枚举解析规则和策略接口签名，快速判断该改哪一层代码而不误触 Admin 或存储实现。

## 约束条件
- 本主题是 L1 模块导览，lesson 必须控制为 15 分钟短课。
- 只覆盖模块边界和源码分层，不展开完整摄入链路或完整检索链路。
- 输出限定在 `teach/open-java/snail-ai/module-rag/`。

## 不在范围内
- 不逐行讲解上传预览、去重、TOCTOU 提交和文档清理细节。
- 不逐行讲解向量库、BM25、融合算法、Rerank 模型调用和 Prompt 注入实现。
- 不讲前端管理台构建产物和截图资源。
