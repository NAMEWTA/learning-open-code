RAG DTO/策略/工具 API 速查

DTO

ChunkDTO — 分片输出单元

字段类型说明
paragraphIndexint一级段落序号（delimiter/regex/smart 切分后）。
chunkIndexint段内二级递归切分序号。
contentString切片正文。
tokenCountint估算 token 数，由 TokenAwareChunker 填充。

由 ChunkStrategy.chunk(ChunkContext) 产出，DocumentPipeline 持久化为 RagChunkPO 并向量化。

KnowledgeInfoDTO — 知识库摘要

字段类型说明
idLong知识库 ID。
nameString显示名称。
descriptionString描述。
documentCountInteger文档数统计。
chunkCountIntegerchunk 数统计。

类仅有 5 个 private 字段，无 Lombok、无手写 getter/setter；当前仓库内无引用点，疑似预留 DTO。若对外暴露需补访问器或改用 KnowledgeResponseVO（Admin 层）。

RagSearchRequestDTO — 检索请求（对外主契约）

字段类型约束说明
ragIdLong@NotNull知识库 ID；JSON 可用别名 knowledgeId（@JsonAlias）。
queryString@NotBlank检索问题文本。
debugBoolean可选为 true 时响应携带 RagStageMetricsDTO 各阶段耗时。

Admin 入口：POST /rag/search（RagSearchController）将 RagSearchRequestVO 映射为本 DTO 后调用 RagSearchService.search。

RagSearchResponseDTO — 检索响应

字段类型说明
resultsList<SearchResult>融合 / rerank / finalize 后的命中列表；元素定义在 snail-ai-commons-core。
metricsRagStageMetricsDTO仅当请求 debug=true 时非空；否则序列化省略（@JsonInclude(NON_NULL)）。

ImportRequest / ImportResult — 导入策略入参/出参

类型字段 / 方法说明
ImportRequest ragId目标知识库 ID。
file本地上传 MultipartFile。
url远程 URL 导入地址。
textContent纯文本导入内容。
name自定义文档名（可选）。
sourceType来源类型字符串，见 DocumentSourceTypeEnum。
ImportResult fileName、fileType、sourceType、fileSize导入元数据。
contentHashSHA-256 hex，供去重比对；由 ContentHashUtil.sha256Hex 计算。
rawBytes（@JsonIgnore transient）原始字节，仅文件类导入填充。

textContent 字段存在，但 DocumentSourceTypeEnum 仅定义 UPLOAD / URL，且没有对应的 DocumentImportStrategy 实现。纯文本导入当前不走本工厂路径，应通过 Admin 直写 DB 或扩展新策略。

ChunkContext — 分片策略上下文

字段类型对应配置
contentStringparser 输出的全文。
maxTokensintRagConfigDO.chunkParams.maxTokens
overlapintchunkParams.overlap
customDelimiterStringdelimiter 模式：单分隔符或 JSON 数组。
chunkRegexStringregex 模式：Java 正则。
chunkModelIdLongsmart 模式：Chat 模型 ID。

枚举

枚举值解析 / 行为
ChunkModeEnum DEFAULT、DELIMITER、REGEX、SMART fromValue(String)：忽略大小写；空或未知 → DEFAULT。
FusionStrategy RRF、WEIGHTED_SUM 配置字符串与枚举 strategy 字段对应；由 HybridFusionHandler 选择实现。
RagDocumentStatus PENDING(0)、PARSING(1)、PROCESSING(2)、SUCCESS(3)、FAILED(4) 运行时主枚举；DocumentPipeline、DocumentProcessingTask、RagDocumentService 使用。PARSING 当前管线未写入。
DocumentStatus INIT(0)、PROCESSING(1)、SUCCESS(2)、FAIL(3) 遗留枚举，与 RagDocumentStatus 编码不同；当前摄入/检索主链路未引用。
DocumentSourceTypeEnum UPLOAD、URL fromValue：忽略大小写；未知 → null。
FileStorageTypeEnum LOCAL fromValue：忽略大小写；未知 → null。当前仅本地存储类型。
DedupStrategy NONE(0)、BY_NAME(1)、BY_CONTENT(2)、BY_NAME_OR_CONTENT(3) fromCode(null) → BY_CONTENT；未知 code → BY_CONTENT；matchesByName() / matchesByContent() 辅助维度判断。
DedupAction REJECT(0)、SKIP(1)、OVERWRITE(2) fromCode(null) → REJECT；未知 code → REJECT；命中去重后的默认动作。
DedupMatchType NONE、BY_NAME、BY_CONTENT、BOTH 描述命中维度，供前端展示与 UploadDecision 携带。
UploadDecisionType NEW、REJECT、SKIP、OVERWRITE 预览/提交阶段对每个文件的最终决策类型。

去重决策模型

摄入侧去重由 Admin RagDocumentService 调用；本模块提供判定结果类型，不含 HTTP 层逻辑。详细流程见 slice-rag-ingestion。

DedupResult

成员类型说明
matchTypeDedupMatchType命中维度。
conflictRagDocumentPO库内冲突文档；未命中为 null。
none()静态工厂(NONE, null)
of(type, conflict)静态工厂构造命中结果。
isHit()booleanmatchType != NONE && conflict != null

UploadDecision

工厂方法决策类型触发条件（概念）
newFile()NEW未命中任何去重维度。
reject(r)REJECT命中且知识库 DedupAction=REJECT。
skip(r)SKIP命中且策略为跳过。
overwrite(r)OVERWRITE命中且策略为覆盖旧文档。
isNew()type == NEW

内容哈希统一使用 ContentHashUtil.sha256Hex(byte[]|String)，与 ImportResult.contentHash、BY_CONTENT 策略对齐。

检索管线 API

调度器 RagSearchPipeline 按 Spring @Order 依次调用 List<RagSearchHandler>。Handler 顺序与职责总览见 rag-overview.html；垂直切片见 slice-rag-retrieval。

RagSearchHandler

void handle(RagSearchContext ctx);

每个 handler 读写共享上下文；设置 ctx.terminated=true 可提前结束管线（当前实现较少使用）。

RagSearchContext 字段契约

分组字段类型写入者 / 用途

ragIdLong知识库 ID。

配置knowledgeRagPOConfigResolveHandler

HybridFusion — 融合策略接口

List<SearchResult> fuse(
    List<SearchResult> vectorResults,
    List<SearchResult> bm25Results,
    int outputSize);
实现类对应 FusionStrategy算法要点
RRFFusionRRF按排名计算 1/(k+rank) 累加，不依赖原始分数量纲。
WeightedSumFusionWEIGHTED_SUM各自 max 归一化后按 denseWeight 加权求和。

HybridFusionHandler 仅在 BM25 有结果时执行融合；仅向量命中时直接透传 vectorResults。

策略接口

DocumentImportStrategy — 文档导入

boolean supports(String sourceType);
ImportResult importDocument(ImportRequest request);
实现类supports行为摘要
LocalFileImportStrategyUPLOAD读取 MultipartFile 字节，推断扩展名，计算 contentHash。
UrlImportStrategyURLHTTP 下载远程文件，同上填充 ImportResult。

策略由 Spring 注入 List<DocumentImportStrategy>，按 supports 首个匹配选用。

DocumentParser — 文档解析

boolean supports(String fileType);
String parse(InputStream inputStream);
实现类支持扩展名
PdfParserpdf
WordParserdocx
ExcelParserxlsx、xls
MarkdownParsermd、markdown
HtmlParserhtml、htm
PlainTextParsertxt、text

DocumentParserFactory.getParser(fileType) 按扩展名选择首个 supports 为真的 parser；无匹配抛 IllegalArgumentException("Unsupported file type: …")。

ChunkStrategy — 文档分片

boolean supports(ChunkModeEnum mode);
List<ChunkDTO> chunk(ChunkContext context);
实现类模式一级切分 splitIntoParagraphs
LengthChunkStrategyDEFAULT整篇作为单段落。
DelimiterChunkStrategyDELIMITER按 customDelimiter（或 JSON 数组）切分。
RegexChunkStrategyREGEX按 chunkRegex 切分；空 regex 回退双换行。
SmartChunkStrategySMART调用 Chat 模型输出 JSON 语义段，带并发限制与 fallback。

AbstractChunkStrategy 模板：子类只做一级切分，二级 token 递归统一委托 TokenAwareChunker.chunkParagraphs(paragraphs, maxTokens, overlap)。空内容直接返回空列表。

ChunkStrategyFactory.getStrategy(ChunkModeEnum)：从注入的 List<ChunkStrategy> 取首个 supports 匹配；否则抛 IllegalArgumentException。

工具与 SPI

ContentHashUtil

方法签名说明
sha256Hex(byte[] data) → StringSHA-256 小写 hex；算法不可用抛 SnailAiException。
sha256Hex(String text) → StringUTF-8 编码后哈希；null 输入返回 null。

DocumentContentResolver — 文档内容 SPI

InputStream resolve(RagDocumentPO document);

按文档存储方式加载文件流。TEXT 类型文档内容在 DB content 字段，返回 null。默认实现走 RAG 本地存储；Admin 层可覆盖为资源库实现。

当前 DocumentPipeline 主路径通过 ResourceService 取流，DocumentContentResolver 作为可替换 SPI 保留。

异常速查

入口异常类型触发条件
DocumentParserFactory.getParserIllegalArgumentException无 parser supports(fileType)。
DocumentImportFactory.getStrategyIllegalArgumentException无 strategy supports(sourceType)（如传入未实现的 TEXT）。
ChunkStrategyFactory.getStrategyIllegalArgumentException无 strategy supports(mode)；消息为中文「不支持的切片模式」。
ContentHashUtil.sha256Hex(byte[])SnailAiExceptionJVM 不支持 SHA-256 算法（极罕见）。
RagSearchRequestDTO 校验MethodArgumentNotValidExceptionAdmin @Validated 请求体缺少 ragId 或空 query。

调用示例

检索：RagSearchService

// RagSearchService.search — 与 Admin POST /rag/search 同路径
RagSearchRequestDTO req = new RagSearchRequestDTO();
req.setRagId(12L);
req.setQuery("如何配置向量库？");
req.setDebug(true);

RagSearchResponseDTO resp = ragSearchService.search(req);
// resp.getResults() → List<SearchResult>
// debug=true 时 resp.getMetrics() 含 vector/bm25/fusion/rerank/total 耗时

分片：ChunkStrategyFactory

ChunkContext ctx = ChunkContext.builder()
    .content(parsedText)
    .maxTokens(2000)
    .overlap(0)
    .customDelimiter("\n\n")
    .build();

ChunkStrategy strategy = chunkStrategyFactory.getStrategy(ChunkModeEnum.DELIMITER);
List<ChunkDTO> chunks = strategy.chunk(ctx);

生产路径由 DocumentChunkingService.chunk 封装：从 RagPO.config.chunkParams.mode 解析模式字符串（如 default → ChunkModeEnum.DEFAULT）。

解析：DocumentParserFactory

DocumentParser parser = parserFactory.getParser("pdf");
String text = parser.parse(inputStream);

DocumentPipeline 在 processDocument 中按 RagDocumentPO.fileType 调用。

检索管线：RagSearchPipeline

RagSearchContext ctx = RagSearchContext.builder()
    .originalQuery("用户问题")
    .ragId(12L)
    .debug(true)
    .metrics(new RagStageMetricsDTO())
    .build();

pipeline.execute(ctx);
List<SearchResult> hits = ctx.getResults();
