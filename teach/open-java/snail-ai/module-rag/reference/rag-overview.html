RAG 功能模块参考

一句话：RAG 模块把知识库文档变成 chunk、向量索引和全文索引，并在搜索或问答时召回 SearchResult，供 Admin 调试、知识库问答和 Agent 运行时上下文使用。DTO 字段、枚举解析与策略接口签名见 rag-api.html。

源码优先：官方文档描述了 Query 改写、PARSING 状态、PPT/OCR/CSV 等能力；当前源码中 QueryRewrite 是空实现，文档处理直接抢占为 PROCESSING，parser 清单也没有 PPT/OCR/CSV 专用实现。

Chunker

模式类一级切分二级切分
defaultLengthChunkStrategy整篇作为一个段落。TokenAwareChunker 调用 LangChain4j recursive splitter，并估算 CJK/英文 token。
delimiterDelimiterChunkStrategy按单个分隔符或 JSON 分隔符数组切分。
regexRegexChunkStrategy按 Java regex 切分，空 regex 回退到双换行。
smartSmartChunkStrategy调用 Chat 模型输出 JSON 语义片段，带并发限制、重试和 fallback。

Fusion 与 Rerank

能力类说明
RRFRRFFusion按 1 / (k + rank) 汇总排名，不依赖向量和 BM25 分数量纲。
Weighted SumWeightedSumFusion先按各自最大分归一化，再按 denseWeight 和互补权重合并。
RerankDefaultRerankService通过 ModelRuntimeHandler.rerank 调用 rerank 模型；失败时按原 score 排序降级。
