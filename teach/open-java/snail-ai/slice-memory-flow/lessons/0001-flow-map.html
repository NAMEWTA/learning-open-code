1. 先背主时序

入口服务把 AgentChatCommand 转成 AgentChatContext，AgentChatChainService 逐个执行 Spring 已排序的 AgentChatHandler。记忆切片的关键不是“是否有 memory 模块”，而是这些写读点的先后关系。

用户消息

      └─ AgentChatChainService
          ├─ @Order(10)  InitContextHandler
          ├─ @Order(20)  ConversationHandler
          │   └─ 写入 user 记录到 AgentConversationRecord
          ├─ @Order(50)  SystemPromptHandler
          ├─ @Order(60)  RagHandler 可能追加 systemPrompt
          ├─ @Order(75)  ContextCollectorHandler

          ├─ 长期召回插槽：当前源码未看到注入调用
          └─ @Order(80)  LlmCallHandler
              └─ ChatDispatchRequest → gRPC /chat/dispatch → Client LLM

2. 短期历史为什么不会重复当前消息

ConversationHandler 先把当前用户消息写入会话记录；随后 ContextCollectorHandler 如果 memoryEnabled 没有显式为 false，会把同一条用户消息追加到短期 store，再调用 loadHistory。两个 store 都会返回时间正序历史，并排除最后一条，因为当前用户消息稍后会由 DefaultPromptFactory 作为新的 UserMessage 追加。

DbShortTermMemoryStore 的 append 是空操作，因为用户记录和 assistant 记录已经由会话记录表持久化；它每次从 DB 取最近窗口。InMemoryShortTermMemoryStore 则维护 JVM 内的有界队列，冷启动或 miss 时从 DB 回填。

3. 长期记忆的真实边界

架构文档描述的目标链路是：检索长期记忆、构造“用户历史记忆”、注入系统提示词，然后再调用 LLM。当前源码里只有 AgentChatContext.memoryContext 和 ChatDispatchRequest.memoryContext 字段，但没有看到 MemoryInjectionAdvisor、setMemoryContext 或长期检索服务被主链调用；Client 侧 DefaultPromptFactory 也没有读取 memoryContext。

排查结论：当前可执行链路里，长期记忆召回注入不是已闭环能力。若用户说“开启 memory 后长期偏好没生效”，优先检查是否根本没有注入点，而不是先调向量阈值。

4. completion 后才更新 assistant 侧记忆

Client 完成模型调用后发送 ChatStreamResponse.TYPE_COMPLETION。Server 的 ChatStreamObserver#handleCompletion 调用 ChatResultPersistService#persistAsync，在虚拟线程里插入 assistant 会话记录、追加 assistant 到 ShortTermMemoryStore，再更新用量统计。因此“本轮 assistant 回复”只能影响下一次历史加载，且异步写入可能比前端流结束稍晚。

Client completion
  └─ ChatStreamObserver.handleCompletion
      └─ ChatResultPersistService.persistAsync
          ├─ insert assistant AgentConversationRecord
          ├─ shortTermMemoryStore.append(assistant)
          └─ update AgentUsageStat

下一轮用户消息
  └─ ContextCollectorHandler.loadHistory
      └─ 读到上一轮 user + assistant，排除当前 user

参考答案与反馈

第一题：之后。用户消息先由 ConversationHandler 写入，ContextCollectorHandler 再追加短期 store 并读取历史，store 读取时排除最后一条当前用户消息。第二题：不会；当前 DefaultPromptFactory 只组装 systemPrompt、historyMessages 和当前 userMessage。继续排查时查 记忆链路速查，也可以向 agent 追问某个字段的具体消费点。
