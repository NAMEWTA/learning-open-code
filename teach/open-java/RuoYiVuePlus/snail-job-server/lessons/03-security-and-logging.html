第3课：安全防护与日志体系

本课目标：理解服务端如何保护 Actuator 敏感端点，以及日志系统的三层架构（控制台/文件/异步）和"客户端发送 → 服务端接收"的远程日志通道。学完本课，你能说清服务端的安全边界和日志流转全链路。

1. 安全防护：Actuator Basic Auth 机制

1.1 为什么需要保护 Actuator？

在 application.yml 中，Actuator 被配置为暴露全部端点：

management:
  endpoints:
    web:
      exposure:
        include: '*'
  endpoint:
    health:
      show-details: ALWAYS

这意味着任何人都可以通过 /snail-job/actuator/* 访问健康检查、环境变量、线程堆栈等敏感信息。必须有防护机制。

1.2 两层防护架构

SecurityConfig + ActuatorAuthFilter 构成了经典的过滤器级 Basic Auth 校验：

SecurityConfig (@Configuration) │ │ 读取配置: │ @Value("${spring.boot.admin.client.username}") │ @Value("${spring.boot.admin.client.password}") │ ▼ FilterRegistrationBean │ filter: ActuatorAuthFilter(username, password) │ urlPatterns: /actuator, /actuator/* │ ▼ ActuatorAuthFilter.doFilter() │ ├─ 1. 获取 Authorization 请求头 ├─ 2. 验证 "Basic " 前缀 ├─ 3. Base64 解码用户名:密码 ├─ 4. 比对 username 和 password ├─ 匹配 → filterChain.doFilter() 放行 └─ 不匹配 → 401 Unauthorized + WWW-Authenticate 头

1.3 源码精读：ActuatorAuthFilter

// ActuatorAuthFilter.java 核心逻辑
@Override
public void doFilter(ServletRequest req, ServletResponse res,
                     FilterChain chain) throws IOException {
    HttpServletRequest request = (HttpServletRequest) req;
    String authHeader = request.getHeader("Authorization");

    // 校验 1: 请求头必须存在且以 "Basic " 开头
    if (authHeader == null || !authHeader.startsWith("Basic ")) {
        response.setHeader("WWW-Authenticate", "Basic realm=\"realm\"");
        response.sendError(401, "Unauthorized");
        return;
    }

    // 校验 2: Base64 解码并提取 username:password
    String base64 = authHeader.substring("Basic ".length());
    byte[] decoded = Base64.getDecoder().decode(base64);
    String credentials = new String(decoded, StandardCharsets.UTF_8);
    String[] parts = credentials.split(":");

    // 校验 3: 格式必须为 "user:pass"
    if (parts.length != 2) { ... return 401; }

    // 校验 4: 用户名密码精确匹配
    if (!username.equals(parts[0]) || !password.equals(parts[1])) {
        ... return 401;
    }

    // 认证通过，放行
    chain.doFilter(request, response);
}
认证凭据的来源：用户名和密码来自 @monitor.username@ / @monitor.password@ Maven 属性（在父 pom 中定义）。Spring Boot Admin Server 回调 Actuator 端点时，会携带 metadata.username 和 metadata.userpassword 构造 Basic Auth 头，所以 Admin Server 可以无缝通过校验。
安全提示：这是一个轻量级的防护方案。Basic Auth 在非 HTTPS 环境下用户名密码是 Base64 编码（非加密）传输，生产环境建议配合 Nginx 反向代理启用 HTTPS，或替换为更安全的认证方案。

2. 日志体系：三层架构 + 远程日志

2.1 logback-plus.xml 全貌

服务端的日志配置定义了 6 个 Appender，形成 控制台 | 文件 | 异步 | 远程 四管齐下的日志输出体系：

Appender 名称类型作用
console ConsoleAppender 控制台彩色输出（带时间、线程、级别、类名着色）
file_console RollingFileAppender 控制台日志写入文件 console.log，按天滚动，保留 1 天，INFO 级别以上
file_info RollingFileAppender INFO 级别精确文件，保留 60 天
file_error RollingFileAppender ERROR 级别单独文件，保留 60 天（排查线上故障核心入口）
async_info AsyncAppender 异步包装 file_info，队列 1024，丢弃阈值 100
async_error AsyncAppender 异步包装 file_error，队列 1024，丢弃阈值 100
snail_log_server_appender SnailJobServerLogbackAppender 接收客户端远程日志，持久化到数据库

2.2 日志文件目录结构

./logs/ruoyi-snailjob-server/ ├── console.log ← 控制台输出（保留 1 天） ├── console.2026-07-04.log ├── info.log ← INFO 日志（保留 60 天） ├── info.2026-07-04.log ├── error.log ← ERROR 日志（保留 60 天，线上排障第一入口） └── error.2026-07-04.log

2.3 远程日志：客户端 → 服务端的日志通道

这是 SnailJob 日志体系中最有特色的设计——客户端任务执行日志实时上报到服务端：

日志流转链路
客户端启动时，SnailJobConfig.onStarting() 监听 SnailClientStartingEvent 事件
动态挂载 SnailLogbackAppender 到 Logback 根 Logger
任务执行过程中产生的 INFO/WARN/ERROR 日志被 SnailLogbackAppender 拦截
日志通过 Netty（17888 端口）实时发送到服务端
服务端 SnailJobServerLogbackAppender 接收并持久化到 MySQL
Dashboard 管理界面从数据库查询展示任务执行日志

客户端侧 (ruoyi-common-job)

// SnailJobConfig.java - 客户端启动时挂载远程日志 appender
@EventListener(SnailClientStartingEvent.class)
public void onStarting(SnailClientStartingEvent event) {
    LoggerContext lc = (LoggerContext) LoggerFactory.getILoggerFactory();
    // 创建 SnailLogbackAppender 实例
    SnailLogbackAppender<ILoggingEvent> ca = new SnailLogbackAppender<>();
    ca.setName("snail_log_appender");
    ca.start();
    // 挂载到根 Logger
    Logger rootLogger = lc.getLogger(Logger.ROOT_LOGGER_NAME);
    rootLogger.addAppender(ca);
}

服务端侧 (logback-plus.xml)

<!-- 接收客户端远程日志的 appender -->
<appender name="snail_log_server_appender"
          class="com.aizuda.snailjob.server.common.appender
                            .SnailJobServerLogbackAppender">
</appender>

<root level="info">
    <appender-ref ref="snail_log_server_appender" />
</root>
设计巧思：远程日志通道使得运维人员无需 SSH 到每台客户端机器查看日志。所有任务执行日志集中存储在服务端数据库中，通过 Dashboard 即可统一检索。在微服务/分布式任务场景下，这极大地降低了排障成本。
