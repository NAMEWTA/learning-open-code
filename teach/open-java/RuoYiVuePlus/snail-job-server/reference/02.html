速查 #02：配置体系与通信模型

双端口通信模型

┌─────────────────────────────────┐ │ SnailJob 服务端 │ │ │ 浏览器 / Dashboard ─│─► :8800 HTTP (Tomcat) │ │ ├── Dashboard 管理界面 │ │ ├── REST API │ │ └── Actuator 健康端点 │ │ │ ruoyi-common-job ─│─► :17888 TCP (Netty) │ (客户端) │ ├── 心跳上报 │ │ ├── 任务执行结果回传 │ │ ├── 远程日志 (LogbackAppender) │ │ └── 任务分派指令接收 │ └─────────────────────────────────┘
端口协议用途访问者
8800HTTP (Tomcat)Dashboard 管理界面、REST API、Actuator浏览器、Admin Server
17888TCP (Netty)客户端通信：心跳、日志上报、任务分派ruoyi-common-job 客户端

application.yml 核心配置速查

服务端口与路径

server:
  port: 8800
  servlet:
    context-path: /snail-job        # Dashboard → :8800/snail-job/

静态资源托管

spring:
  web:
    resources:
      static-locations: classpath:admin/   # 框架内置 Dashboard 前端资源

MyBatis-Plus 持久化

mybatis-plus:
  typeAliasesPackage: com.aizuda.snailjob.template.datasource.persistence.po
  global-config:
    db-config:
      where-strategy: NOT_EMPTY
      logic-delete-value: 1
      logic-not-delete-value: 0
  configuration:
    map-underscore-to-camel-case: true

要点 typeAliasesPackage 指向 SnailJob 框架内置 PO，非 RuoYi 项目代码。这些 PO 管理调度元数据表（任务定义、重试记录、调度配置、工作流 DAG、节点信息）。

application-dev.yml SnailJob 核心配置

配置项默认值含义
server-port17888Netty 客户端通信端口
retry-pull-page-size100每批次拉取重试任务条数
log-storage7任务执行日志保留天数
merge-log-days1合并日志天数窗口
merge-log-num500合并日志最大条数阈值
bucket-total128核心 分桶总数。任务哈希分配，每桶一个 Akka Actor 调度
load-balance-cycle-time10 (秒)负载均衡周期，检查各桶负载并跨节点重分配
retry-max-pull-parallel2重试任务拉取并行线程数
summary-day7Dashboard 仪表盘容错天数

bucket-total 128 的分布式调度语义

所有任务 ──一致性哈希──► 128 个桶 (Bucket) │ 每个桶 ──对应──► 1 个 Akka Actor │ 集群节点通过 Actor 协议协商各自负责的桶 │ 每 10s (load-balance-cycle-time) 重分配 │ └── 无中心化负载均衡

Dockerfile JVM 调优

FROM bellsoft/liberica-openjdk-rocky:21.0.11-cds

EXPOSE 8800
EXPOSE 17888

ENTRYPOINT java \
           -Djava.security.egd=file:/dev/./urandom \
           -XX:+HeapDumpOnOutOfMemoryError \
           -XX:+UseZGC \               # 低延迟垃圾收集器
           ${JAVA_OPTS} \
           -jar app.jar

使用 ZGC 的原因：调度系统对延迟敏感，ZGC 的亚毫秒级暂停时间避免 GC 抖动影响任务调度精度。
