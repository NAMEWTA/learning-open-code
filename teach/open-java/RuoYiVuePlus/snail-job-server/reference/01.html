速查 #01：服务端架构与启动原理

模块定位

维度说明
所属层ruoyi-extend 扩展层（独立部署的 Spring Boot 应用）
同级模块ruoyi-monitor-admin、ruoyi-snailai-server
进程关系与 ruoyi-admin 主应用不共享进程
源码文件数5 个 Java/配置源码（启动类 + 2 安全过滤器 + 3 yml）
设计哲学"薄封装 + 重框架" — 其余全部由 snail-job-server-starter 自动配置提供

3 个核心依赖

依赖版本作用
snail-job-server-starter2.0.0服务端核心引擎：Akka Actor 调度、Netty 通信、Dashboard API、持久化层
scala-library2.13.9Scala 运行时，支撑 Akka Actor 模型
spring-boot-admin-starter-client-注册到 Spring Boot Admin 监控中心

重要 pom.xml 排除了 starter 自带的 scala-library，显式声明 2.13.9，统一管理版本避免传递依赖冲突。

跨包启动引导（核心设计）

package org.dromara.snailjob;  ← 对外声明的包

@SpringBootApplication
public class SnailJobServerApplication {

    public static void main(String[] args) {
        SpringApplication.run(
            com.aizuda.snailjob.server.SnailJobServerApplication.class,  ← 实际运行的类
            args
        );
    }
}
角色类作用
引导器 点火开关org.dromara.snailjob.SnailJobServerApplicationJVM 入口，@SpringBootApplication 让 IDE 识别为 Spring Boot 应用
发动机 真正上下文com.aizuda.snailjob.server.SnailJobServerApplication以框架包为基准进行自动配置扫描，加载全部内部 Bean

原因：SnailJob 框架的自动配置类、@ComponentScan、@EntityScan 都声明在 com.aizuda.snailjob.server 包下。若直接用 org.dromara.snailjob 下的类作为入口，框架自身 Bean 将扫描不到。

启动流程追踪

1. org.dromara.snailjob.SnailJobServerApplication.main() │ (JVM 入口) │ 2. SpringApplication.run(com.aizuda.snailjob.server.SnailJobServerApplication.class) │ (以框架包为基准启动 Spring Boot) │ 3. 自动配置扫描 com.aizuda.snailjob.server.* ├── 加载 Akka Actor 系统（调度引擎核心） ├── 启动 Netty Server（17888 端口，客户端通信） ├── 初始化 MyBatis-Plus（连接 MySQL，管理调度元数据） ├── 启动 Dashboard HTTP Server（8800 端口，管理界面） └── 启动 HTTP 重试/定时/工作流调度器 │ 4. ApplicationReadyEvent → 服务端就绪，等待客户端注册
