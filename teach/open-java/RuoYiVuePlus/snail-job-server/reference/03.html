两层防护组件

组件类型职责
SecurityConfig@Configuration读取凭据，注册 FilterRegistrationBean，绑定 URL 模式
ActuatorAuthFilterjavax.servlet.Filter四步 Basic Auth 校验

ActuatorAuthFilter 四步校验链

请求 → /actuator/** │ ├─ 校验1: Authorization 请求头是否存在且以 "Basic " 开头 │ 失败 → 401 + WWW-Authenticate 头 │ ├─ 校验2: 截取 "Basic " 之后的 Base64 字符串并解码 │ 失败 → 401 │ ├─ 校验3: 解码结果是否包含且仅包含一个 ":" 分隔符 │ 失败 → 401 │ └─ 校验4: username 和 password 精确匹配配置值 失败 → 401 成功 → chain.doFilter() 放行

认证凭据来源

spring.boot.admin.client:
  username: @monitor.username@      # Maven 属性，父 pom 定义
  password: @monitor.password@
  instance:
    metadata:
      username: ${...username}       # Admin Server 回调时携带
      userpassword: ${...password}   # 构造 Basic Auth 头

安全提示 Basic Auth 在非 HTTPS 下为 Base64 编码（非加密），生产环境应配合 Nginx HTTPS 或更安全的认证方案。

日志体系：7 个 Appender

Appender类型关键参数用途
consoleConsoleAppender-控制台彩色输出
file_consoleRollingFileAppender按天滚动，保留 1 天控制台日志落盘 → console.log
file_infoRollingFileAppenderINFO+，保留 60 天INFO 级别文件 → info.log
file_errorRollingFileAppenderERROR+，保留 60 天排障入口 ERROR 单独文件 → error.log
async_infoAsyncAppender队列 1024，丢弃阈值 100异步包装 file_info
async_errorAsyncAppender队列 1024，丢弃阈值 100异步包装 file_error
snail_log_server_appenderSnailJobServerLogbackAppender-接收客户端远程日志，持久化到数据库

日志目录结构

./logs/ruoyi-snailjob-server/ ├── console.log ← 保留 1 天 ├── console.2026-07-04.log ├── info.log ← 保留 60 天 ├── info.2026-07-04.log ├── error.log ← 保留 60 天 ★ 线上排障第一入口 └── error.2026-07-04.log

远程日志通道（核心特色）

客户端 → 服务端日志流转链路
步骤位置动作
1客户端 SnailJobConfig监听 SnailClientStartingEvent，动态挂载 SnailLogbackAppender 到 Logback 根 Logger
2客户端任务执行任务 INFO/WARN/ERROR 日志被 SnailLogbackAppender 拦截
3客户端 Netty日志通过 17888 端口实时发送到服务端
4服务端 LogbackSnailJobServerLogbackAppender 接收并持久化到 MySQL
5Dashboard从数据库查询展示任务执行日志

客户端侧代码片段

@EventListener(SnailClientStartingEvent.class)
public void onStarting(SnailClientStartingEvent event) {
    LoggerContext lc = (LoggerContext) LoggerFactory.getILoggerFactory();
    SnailLogbackAppender<ILoggingEvent> ca = new SnailLogbackAppender<>();
    ca.setName("snail_log_appender");
    ca.start();
    Logger rootLogger = lc.getLogger(Logger.ROOT_LOGGER_NAME);
    rootLogger.addAppender(ca);
}

服务端侧配置

<appender name="snail_log_server_appender"
          class="com.aizuda.snailjob.server.common.appender
                        .SnailJobServerLogbackAppender">
</appender>

<root level="info">
    <appender-ref ref="snail_log_server_appender" />
</root>

设计价值 无需 SSH 到每台客户端机器。所有任务日志集中存储在服务端数据库，Dashboard 统一检索——在微服务/分布式任务场景下极大降低排障成本。
