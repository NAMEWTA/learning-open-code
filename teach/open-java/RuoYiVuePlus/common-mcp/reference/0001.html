速查 01 · ruoyi-common-mcp 模块全景

配套课程：lessons/0001.html · 使命：读懂 ruoyi-common-mcp MCP 通信模块

一句话定位

ruoyi-common-mcp 不是 MCP 协议的 Java 实现（由 Spring AI MCP 2.0.0 完成），而是一层「项目级便利封装」——把 McpSyncClient 的遍历/调用/结果包装做得更方便，附带按 Server 名称分组的返回格式。

MCP 协议核心概念

概念说明
MCP (Model Context Protocol)Anthropic 提出 → 行业标准：AI 应用（Host/Client）与外部工具/数据源（Server）之间的通信协议
三大原语Tools（工具调用，类似 RPC）、Resources（资源读取，类似 REST GET）、Prompts（提示模板）
传输层JSON-RPC 2.0 消息，支持 SSE / stdio / Streamable HTTP

模块全部内容（4 类 + 1 清单）

文件作用
pom.xml依赖 spring-ai-starter-mcp-client + spring-ai-starter-mcp-server-webmvc + ruoyi-common-core
config/McpAutoConfiguration.java条件装配：有 McpSyncClient 时创建 McpClientTemplate Bean（第 2 课）
core/McpClientTemplate.java封装：6 个便捷方法（listTools / callTool x2 / readResource x2 / findClient），第 2 课
core/McpToolCallResult.java模型：工具调用结果的标准化 Record（第 3 课）
core/McpResourceReadResult.java模型：资源读取结果的标准化 Record（第 3 课）
META-INF/spring/AutoConfiguration.imports发现：声明 McpAutoConfiguration 为自动配置类

四个类 = 四件事

类职责解决
McpAutoConfiguration条件装配依赖即生效，无连接不报错
McpClientTemplate操作封装避免业务代码直接遍历 Client 列表
McpToolCallResult工具结果模型统一格式 + 附加 serverName
McpResourceReadResult资源结果模型统一格式 + 附加 serverName

记忆锚点

  • MCP = AI 应用的「USB-C 接口」——统一 Client-Server 通信标准。
  • Spring AI MCP 做协议实现，本模块只做便利封装。
  • 每个操作都有两个版本：广播版（所有 Server）和 单点版（指定 Server）。
  • @ConditionalOnBean(McpSyncClient.class) → 无连接 = 不创建模板 = 不报错。
  • 这是 RuoYi ruoyi-common-* 包装第三方库的标准范式：自动配置 + 模板封装 + 统一结果模型。
