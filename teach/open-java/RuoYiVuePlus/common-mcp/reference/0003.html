McpToolCallResult vs McpResourceReadResult

McpToolCallResultMcpResourceReadResult
字段数42
serverName✅ String✅ String
error 标记✅ boolean❌ 无
文本内容List<McpSchema.Content>List<McpSchema.ResourceContents>
结构化数据✅ Object structuredContent❌ 无
工厂方法入参McpSchema.CallToolResultMcpSchema.ReadResourceResult

工具调用完整链路

1. 业务代码: mcpClientTemplate.callTool("weather-server", "get_forecast", Map.of("city","北京")) 2. McpClientTemplate: findClient("weather-server") → Optional<McpSyncClient> .map(client → client.callTool(new CallToolRequest(name, args))) 3. Spring AI MCP (SDK 层): CallToolRequest → JSON-RPC 消息 → SSE/stdio → MCP Server MCP Server → JSON-RPC 响应 → CallToolResult 4. McpClientTemplate: McpToolCallResult.of("weather-server", callResult) → Record{"weather-server", isError?, content[], structuredContent} 5. 业务代码: if (result.error()) { 读取 content 中的错误文本 } else { 读取 content(文本) 或 structuredContent(结构化数据) }

解析 ToolCallResult 的防御性代码模式

// 1. 先查 Optional 是否存在
if (resultOpt.isEmpty()) { /* Server 名称写错 */ }

// 2. 再查 error 标记
McpToolCallResult result = resultOpt.get();
if (result.error()) { /* 工具执行失败 */ }

// 3. 解析文本内容
String text = result.content().stream()
    .filter(c -> "text".equals(c.type()))
    .map(c -> c.text())
    .collect(Collectors.joining("\n"));

// 4. 解析结构化内容（可选）
if (result.structuredContent() instanceof Map map) {
    // 强转后用
}

解析 ResourceReadResult 的防御性代码模式

// 1. 先查 Optional 是否存在
if (resultOpt.isEmpty()) { /* Server 名称写错 */ }

// 2. 遍历 contents 列表
McpResourceReadResult result = resultOpt.get();
for (ResourceContents rc : result.contents()) {
    if (rc instanceof TextResourceContents textRc) {
        System.out.println(textRc.uri() + ": " + textRc.text());
    } else if (rc instanceof BlobResourceContents blobRc) {
        byte[] data = Base64.getDecoder().decode(blobRc.blob());
        // 处理二进制数据
    }
}

典型使用模式速查

需求调用方式
查看所有 Server 的工具listTools() → Map<Server名, 工具列表>
调用已知 Server 的工具callTool("server", "tool", args) → Optional
向所有 Server 发起同一操作callTool("tool", args) → Map（广播）
读取已知 Server 的资源readResource("server", "uri") → Optional
检查某个 Server 是否已连接findClient("server").isPresent()

注意：Boolean.TRUE.equals(isError)

McpToolCallResult 的 error 字段来自 CallToolResult.isError()， 返回 Boolean 包装类型（可能为 null）。 源码用 Boolean.TRUE.equals(result.isError()) 防止 NPE。 → 如果 isError 为 null → 视为 false（调用未出错）。

记忆锚点

  • content 是文本片段列表（展示用），structuredContent 是结构化 JSON 对象（程序化处理用）。
  • 解析 ResourceContents 必须用 instanceof 判断 TextResourceContents / BlobResourceContents。
  • 完整调用链 5 层：业务代码 → Template → SDK → 传输层 → MCP Server。
  • 防御性编程三步：检查 Optional → 检查 error → 解析 content。
