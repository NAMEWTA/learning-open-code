速查 02 · McpClientTemplate 操作速查

配套课程：lessons/0002.html

它是什么

封装 List<McpSyncClient> 的遍历操作，提供 6 个便捷方法。业务代码通过它间接使用 MCP，不必直接操作 McpSyncClient 列表。由 McpAutoConfiguration 条件创建。

注入方式

@Autowired
private McpClientTemplate mcpClientTemplate;

6 个方法速查

方法模式返回找不到 Client
listTools()广播Map<String, List<String>>空 Map
callTool(toolName, args)广播Map<String, McpToolCallResult>空 Map
callTool(server, tool, args)单点Optional<McpToolCallResult>Optional.empty()
readResource(uri)广播Map<String, McpResourceReadResult>空 Map
readResource(server, uri)单点Optional<McpResourceReadResult>Optional.empty()
findClient(serverName)—Optional<McpSyncClient>Optional.empty()

条件装配逻辑

@AutoConfiguration ← 自动配置类声明 @ConditionalOnBean(McpSyncClient.class) ← 前提：有 MCP 连接 @ConditionalOnMissingBean ← 允许用户自定义覆盖 ↓ 只有项目 yml 中配置了 MCP Server 连接 → Spring AI MCP 创建 McpSyncClient Bean → 本模块才创建 McpClientTemplate

广播 vs 单点

广播版单点版
参数无 serverName 参数第一个参数是 serverName
行为遍历所有 Client，逐个调用先 findClient 再调用
返回Map（key=Server 名称）Optional
适用所有 Server 都声明了同名工具/资源已知目标 Server

getServerName 内部实现

client.getServerInfo().name() → 如果 serverInfo 为 null 或 name 为 null → 返回 "unknown"（防御性编程，避免 NPE）

记忆锚点

  • findClient + McpSchema.CallToolRequest + McpToolCallResult.of() = 一次工具调用的三个关键步骤。
  • 广播版适合「所有 Server 都支持」的操作（如 health check）；单点版适合已知目标的操作。
  • @ConditionalOnBean 是「按需装配」的关键——依赖不报错，用到才创建。
  • @ConditionalOnMissingBean 是「可覆盖」的关键——用户自定义优于自动配置。
