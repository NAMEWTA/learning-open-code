第 3 课 · 工具类与校验注解 Utils / Validate

本节目标：学完你能熟练使用 SpringUtils、ServletUtils、MessageUtils、MapstructUtils、ValidatorUtils 五个核心工具类，理解 @DictPattern、@EnumPattern、@Xss 三个自定义校验注解的实现原理，以及 AddressUtils 基于 ip2region 的离线 IP 定位机制。

core 模块的工具类不是随意的静态方法集合——它们大多扩展自 Hutool，并注入了 Spring 容器感知能力。自定义校验注解则填补了 Jakarta Bean Validation 在业务校验场景的空白。今天我们从 Spring 容器桥接器开始。

一、SpringUtils — Spring 容器的静态访问桥

SpringUtils 位于 org.dromara.common.core.utils.SpringUtils，继承自 Hutool 的 SpringUtil，是整个项目中引用最频繁的工具类之一。它通过实现 ApplicationContextAware 接口，在 Spring 容器启动时自动注入 ApplicationContext，之后任何地方都能通过静态方法访问容器中的 Bean。

@Component
public class SpringUtils extends SpringUtil implements ApplicationContextAware {

    @Override
    public void setApplicationContext(ApplicationContext applicationContext) {
        // 容器启动时回调，持有 ApplicationContext 引用
    }
}

扩展能力（超出 Hutool SpringUtil 的部分）

方法功能使用场景
containsBean(String name)判断容器是否包含指定名称的 Bean条件装配、可选依赖检查
isSingleton(String name)判断 Bean 是否为单例Bean 生命周期管理
getType(String name)获取 Bean 的类型（不实例化）泛型推断、反射操作
getAopProxy(Object invoker)获取对象的 AOP 代理需要 AOP 增强时获取代理对象
isVirtual()检测当前 JVM 是否运行在虚拟线程上Java 21+ 虚拟线程兼容判断
注意：SpringUtils.getBean() 是 Hutool 提供的基础方法，获取不到时抛 NoSuchBeanDefinitionException。如果业务中 Bean 可能不存在，应先用 containsBean() 判空。

二、ServletUtils — Web 层静态工具

ServletUtils 位于 org.dromara.common.core.utils.ServletUtils，继承自 Hutool 的 JakartaServletUtil，基于 Spring 的 RequestContextHolder 获取当前请求上下文：

// 核心方法一览
getRequest()            // 获取当前 HttpServletRequest
getResponse()           // 获取当前 HttpServletResponse
getParameter(String name)  // 获取请求参数（支持 GET/POST）
getClientIP()           // 获取客户端真实 IP（含代理穿透）
renderString(response, string)  // 直接向响应流写入字符串
renderString(response, string, contentType) // 指定 Content-Type 写入

getClientIP() 的 IP 获取逻辑值得关注——它不是简单地取 request.getRemoteAddr()，而是按顺序检查多个代理头：

X-Forwarded-For → Proxy-Client-IP → WL-Proxy-Client-IP
→ HTTP_CLIENT_IP → HTTP_X_FORWARDED_FOR → RemoteAddr

这确保了在 Nginx、Apache、CDN 等多层代理场景下也能获取到真实客户端 IP。

三、MessageUtils — 国际化消息解析

MessageUtils 位于 org.dromara.common.core.utils.MessageUtils，是 i18n 消息解析的静态入口：

// 核心方法
public static String message(String code, Object... args) {
    MessageSource messageSource = SpringUtils.getBean(MessageSource.class);
    return messageSource.getMessage(code, args,
        LocaleContextHolder.getLocale());  // 自动识别当前请求的语言环境
}

优雅降级机制——当 code 在 i18n 资源文件中找不到时，不会直接崩掉，而是返回 defaultMessage 或 code 本身作为回退值。

调用链： UserException("user.not.exists", userId) → getMessage() → MessageUtils.message("user.not.exists", [userId]) → MessageSource.getMessage(code="user.not.exists", args=[userId], locale=zh_CN) → 查找 messages_zh_CN.properties → "用户[id={0}]不存在" → 格式化 → "用户[id=12345]不存在"

四、MapstructUtils — 对象转换工具

MapstructUtils 位于 org.dromara.common.core.utils.MapstructUtils，对 MapStruct 的 Converter 接口做了静态封装：

public class MapstructUtils {
    // 从 Spring 容器获取 MapStruct 生成的 Converter 实现类
    private final static Converter CONVERTER = SpringUtils.getBean(Converter.class);

    // 单对象转换
    public static <S, T> T convert(S source, Class<T> targetClass) {
        return CONVERTER.convert(source, targetClass);
    }

    // 列表转换
    public static <S, T> List<T> convert(List<S> sourceList, Class<T> targetClass) {
        return CONVERTER.convert(sourceList, targetClass);
    }

    // Map 转 Bean
    public static <T> T convert(Map<String, Object> map, Class<T> beanClass) {
        return CONVERTER.convert(map, beanClass);
    }
}
设计洞察 — 为什么 Converter 要从 Spring 容器获取？ MapStruct 在编译期生成 Converter 接口的实现类（如 ConverterImpl），这个实现类需要用 @Mapper(componentModel = "spring") 注册为 Spring Bean。MapstructUtils 通过 SpringUtils.getBean(Converter.class) 拿到的是 Spring 管理的单例，确保了 MapStruct 的 Mapper 生命周期与 Spring 容器一致。

五、ValidatorUtils — 手动触发校验

ValidatorUtils 位于 org.dromara.common.core.utils.ValidatorUtils，提供了在代码中手动触发 Jakarta Bean Validation 的能力：

public class ValidatorUtils {
    private static final Validator VALIDATOR = SpringUtils.getBean(Validator.class);

    // 支持分组校验
    public static void validate(Object object, Class<?>... groups) {
        Set<ConstraintViolation<Object>> violations = VALIDATOR.validate(object, groups);
        if (!violations.isEmpty()) {
            throw new ConstraintViolationException(violations);
        }
    }
}

这里配合 ValidatorConfig 实现了 Fail-Fast 模式——一旦发现第一个校验失败就立即停止后续校验，避免用户看到一堆错误信息。配置要点：

// ValidatorConfig 中的关键配置
validatorFactory.getValidator().validate(object)
// 配置了 failFast(true) → 第一个失败就停止

典型使用场景：Controller 层已经有了 @Validated 自动校验，但在 Service 层处理非 HTTP 入口（如 MQ 消息、定时任务）时，需要手动调用 ValidatorUtils.validate() 来校验数据。

六、自定义校验注解体系

Jakarta Bean Validation 提供了 @NotNull、@Size、@Pattern 等标准注解，但在业务场景中我们需要更具体的校验能力。core 模块为此设计了三个自定义注解：

6.1 @DictPattern — 字典值校验

@DictPattern 位于 org.dromara.common.core.validate.dicts.DictPattern，校验字段值是否属于指定字典类型的合法值：

// 注解声明
@Target({FIELD, PARAMETER})
@Retention(RUNTIME)
@Constraint(validatedBy = DictPatternValidator.class)
public @interface DictPattern {
    String dictType();           // 字典类型，如 "sys_user_sex"
    String message() default "字典值不合法";
    Class<?>[] groups() default {};
    Class<? extends Payload>[] payload() default {};
}

// 校验器实现（简化）
public class DictPatternValidator implements ConstraintValidator<DictPattern, String> {
    @Override
    public boolean isValid(String value, ConstraintValidatorContext context) {
        if (value == null) return true;  // null 放行，由 @NotNull 处理
        DictService dictService = SpringUtils.getBean(DictService.class);
        // 查询字典数据，判断 value 是否在合法值集合中
        return dictService.selectDictDataByType(dictType)
            .stream().anyMatch(d -> d.getDictValue().equals(value));
    }
}

使用示例：

@DictPattern(dictType = "sys_user_sex", message = "性别不合法")
private String sex;  // 必须是字典 sys_user_sex 中定义的值
注意：DictPatternValidator 中通过 SpringUtils.getBean(DictService.class) 获取字典服务，而非通过 @Autowired 注入。这是因为 Jakarta Validator 的 ConstraintValidator 生命周期由 ValidatorFactory 管理，不受 Spring 管理——@Autowired 会失效。通过静态工具类桥接是务实方案。

6.2 @EnumPattern — 枚举值校验

@EnumPattern 位于 org.dromara.common.core.validate.enums.EnumPattern，校验字段值是否属于指定枚举类的合法值。它是 @Repeatable 的，允许在同一字段上标注多个规则：

// 注解声明
@Target({FIELD, PARAMETER})
@Retention(RUNTIME)
@Repeatable(EnumPattern.List.class)  // 支持重复标注
@Constraint(validatedBy = EnumPatternValidator.class)
public @interface EnumPattern {
    Class<? extends Enum<?>> enumClass();  // 目标枚举类
    String fieldName() default "value";      // 用于匹配的字段名
    String message() default "枚举值不合法";
}

// 校验器核心逻辑
public class EnumPatternValidator implements ConstraintValidator<EnumPattern, String> {
    @Override
    public boolean isValid(String value, ConstraintValidatorContext context) {
        // 通过反射遍历枚举的所有值，提取指定字段的值进行匹配
        Enum<?>[] enumConstants = annotation.enumClass().getEnumConstants();
        for (Enum<?> e : enumConstants) {
            Field field = e.getClass().getDeclaredField(annotation.fieldName());
            field.setAccessible(true);
            if (field.get(e).equals(value)) return true;
        }
        return false;
    }
}

使用示例：

@EnumPattern(enumClass = UserStatus.class, fieldName = "code")
private String userStatus;  // 必须是 UserStatus 枚举中 code 字段的合法值

核心实现原理是 反射遍历枚举常量——在 isValid() 中拿到枚举类 enumClass 的 getEnumConstants()，逐个反射读取 fieldName 字段的值与入参比对。这种设计使得同一个校验注解可以适配任意枚举类，不需要为每个枚举单独写 Validator。

6.3 @Xss — XSS 攻击防护

@Xss 位于 org.dromara.common.core.xss.Xss，检测字段值中是否包含 HTML 标签，防止跨站脚本攻击：

@Target({FIELD, PARAMETER})
@Retention(RUNTIME)
@Constraint(validatedBy = XssValidator.class)
public @interface Xss {
    String message() default "不能包含HTML标签";
}

// 校验器
public class XssValidator implements ConstraintValidator<Xss, String> {
    @Override
    public boolean isValid(String value, ConstraintValidatorContext context) {
        // 利用 Hutool 的 RE_HTML_MARK 正则匹配 HTML 标签
        return !ReUtil.contains(HtmlUtil.RE_HTML_MARK, value);
    }
}

底层依赖 Hutool 的 HtmlUtil.RE_HTML_MARK 正则表达式来检测 <script>、<img>、<a> 等 HTML 标签。简单但有效——如果值中包含任何 HTML 标签，校验直接失败。

七、IP 定位工具 — 离线 IP 到地理位置

AddressUtils 和 RegionUtils 位于 org.dromara.common.core.utils，基于 ip2region 离线数据库实现 IP 地址到地理位置的转换：

// 典型调用
String ip = "120.229.45.82";
String location = AddressUtils.getRealAddressByIP(ip);
// → "中国|广东省|深圳市|移动"

设计特点：

  • 离线查询：不依赖外部 API，ip2region 数据库文件（.xdb）内嵌在项目中
  • 内存搜索：将整个索引加载到内存中，查询速度极快
  • 缓存优化：RegionUtils 提供 indexBlockPtr 等缓存机制，避免重复解析
ip2region 查询流程： IP 字符串 → RegionUtils.parseIP(ip) → 查找 indexBlockPtr 缓存 → 未命中：VectorIndex 二分查找 IP 范围 → 命中：返回 region 数据索引 → 从 xdb 文件读取 region 字符串 → 返回 "国家|区域|省份|城市|ISP"

八、检验一下（提取练习）

Q1. ValidatorUtils.validate() 的 Fail-Fast 模式是什么意思？

A. 校验失败时立即回滚事务 B. 发现第一个校验失败后立即停止后续校验，不收集全部错误 C. 校验速度更快 D. 跳过所有校验直接返回成功
看答案B。Fail-Fast 模式通过 ValidatorConfig 中配置 failFast(true) 实现，一旦发现第一个 ConstraintViolation 就停止，避免给用户返回一长串错误信息。

Q2. 为什么 DictPatternValidator 中用 SpringUtils.getBean(DictService.class) 而不是 @Autowired 注入？

A. @Autowired 性能更差 B. Jakarta Validator 的 ConstraintValidator 生命周期由 ValidatorFactory 管理，不受 Spring 管理，@Autowired 会失效 C. 这是代码风格偏好 D. DictService 不是 Spring Bean
看答案B。ConstraintValidator 实例由 ValidatorFactory 创建和管理，不在 Spring 容器中，因此 @Autowired 不会生效。通过 SpringUtils 静态桥接是最直接的解决方案。

Q3. 关于 @EnumPattern，以下哪个说法正确？

A. 每个枚举类都需要单独写一个 Validator B. 它通过反射读取枚举常量的指定字段值进行匹配，支持任意枚举类 C. 只能校验 String 类型的值，不能校验 Integer D. 不支持 @Repeatable，同一字段只能标注一次
看答案B。EnumPatternValidator 在 isValid() 中用反射提取枚举常量的指定字段（fieldName 默认 "value"），与入参比对。@Repeatable 支持同一字段多次标注，e.g. 同时校验两个不同的枚举类。

Q4. @Xss 注解的底层检测机制是什么？

A. 调用外部 XSS 检测 API B. 使用 Hutool HtmlUtil.RE_HTML_MARK 正则匹配 HTML 标签 C. 转义所有特殊字符 D. 使用 OWASP ESAPI 库进行检测
看答案B。XssValidator 通过 Hutool 的 ReUtil.contains(HtmlUtil.RE_HTML_MARK, value) 检测字段值中是否包含 HTML 标签，简单直接。

Q5. SpringUtils 继承自哪个类？它比父类多了哪些能力？

A. 继承 SpringUtil，多了 containsBean、isSingleton、getType、getAopProxy、isVirtual B. 继承 ApplicationContextAware，多了 getBean 方法 C. 继承 Object，是一个全新的工具类 D. 继承 BeanUtils，多了容器感知能力
看答案A。SpringUtils extends SpringUtil（Hutool），在父类基础上扩展了 Bean 定义查询（containsBean/isSingleton/getType）、AOP 代理获取（getAopProxy）和虚拟线程检测（isVirtual）等能力。
