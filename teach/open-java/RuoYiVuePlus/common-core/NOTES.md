# NOTES — 教学偏好暂存

> 记录用户在课程中表达的教学偏好，供后续课程设计参考。

- 用户要求「全面详细讲解，结合实际代码」——课程应大量引用真实文件路径、类名、关键代码片段，并追踪跨模块的真实调用链，不停留在抽象描述。
- 偏好「鸟瞰图先行」风格——每课先用模块全景图定位当前课程主题在 core 模块中的位置，再分步拆解具体类的设计。
- 课程需要「短小精悍」——每课几分钟可完成，聚焦一个具体学习目标，避免信息过载。
- 每个课程必须包含交互式元素（选择题、填空题、分析题）来进行提取练习。
- 每个课程末尾需要推荐一手资料（源码路径、官方文档链接）和 AI 追问提示。
- 课程间互相链接（上一课/下一课导航）。
- 51 个 Java 文件需要合理分配到 8 节课中，每节课约 6-8 个关键类。
- `reference/` 下的 HTML 是对应课程的压缩速查表（cheat sheet），内容应高度凝练。
