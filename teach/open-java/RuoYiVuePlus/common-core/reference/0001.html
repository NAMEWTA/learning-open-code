速查 01 · ruoyi-common-core 核心全景

配套课程：lessons/0001.html · 鸟瞰 core 模块的四大支柱

一句话定位

ruoyi-common-core 是所有模块的技术基石。65 个 Java 文件分成四大支柱：统一响应体 (R/PageResult)、分层异常体系 (BaseException/ServiceException)、常量与枚举 (6接口+7枚举)、工具类集合 (18+ 工具类)。

四大支柱速览

支柱包路径代表类核心职责
① 统一响应domain/R, PageResultAPI 返回信封：code/msg/data
② 异常体系exception/BaseException, ServiceExceptionBaseException → i18n; ServiceException → 占位符
③ 常量枚举constant/ + enums/Constants, BusinessStatusEnuminterface 常量 + 状态机枚举
④ 工具服务utils/ + config/ + validate/StringUtils, SpringUtils可复用工具、校验、自动配置

R<T> 工厂方法

R.ok() → {code:200, msg:"操作成功", data:null} R.ok(data) → {code:200, msg:"操作成功", data:obj} R.fail() → {code:500, msg:"操作失败"} R.fail(msg) → {code:500, msg:自定义} R.fail(code, msg) → {code:自定义, msg:自定义} R.warn(msg) → {code:601} (HttpStatus.WARN)

异常体系双分支

特性BaseException 分支ServiceException
国际化支持 (code→MessageUtils)不支持 (直接存消息)

可继承是 (FileException→子类)final，不可继承

记忆锚点

  • 四大支柱：响应体 → 异常 → 常量枚举 → 工具类，按依赖顺序由内向外。
  • R 的三个字段：code (int), msg (String), data (T 泛型) — 没有更多。
  • BaseException 的 getMessage() 走 MessageUtils.message(code, args) → i18n 解析。
  • ServiceException 用 {} 占位符（Hutool StrFormatter），BaseException 用 i18n key。
  • 常量用 interface 而非 class — 利用字段隐式 public static final。
