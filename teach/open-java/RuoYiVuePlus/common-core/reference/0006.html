速查 06 · 配置与线程池

配套课程：lessons/0006.html · ThreadPoolConfig、ApplicationConfig、ValidatorConfig

一句话定位

三个 @AutoConfiguration 类通过 AutoConfiguration.imports 自动加载：ApplicationConfig 开启 AOP+异步，ThreadPoolConfig 提供定时任务线程池+虚拟线程适配，ValidatorConfig 设置 fail-fast 校验。

三个配置类速览

配置类核心注解/方法作用
ApplicationConfig@EnableAspectJAutoProxy + @EnableAsync启用 AOP 和异步方法
ThreadPoolConfigscheduledExecutorService() + destroy()定时任务线程池 + 优雅关闭
ValidatorConfigvalidator() → failFast(true)Fail-Fast 参数校验模式

ApplicationConfig — 纯注解驱动

类体为空！全部配置通过注解完成： @AutoConfiguration → Spring Boot 3.0+ 自动配置 @EnableAspectJAutoProxy → 启用 @Aspect (日志/数据权限/限流) @EnableAsync(proxyTargetClass = true) → 启用 @Async (CGLIB代理) 依赖者: @Log, @DataScope, @RepeatSubmit, @RateLimiter, 异步日志

ThreadPoolConfig — 虚拟线程适配

核心线程数: Runtime.availableProcessors() + 1 (I/O密集型最佳实践) 线程池选择: SpringUtils.isVirtual() == true → 虚拟线程: Executors.newThreadPerTaskExecutor SpringUtils.isVirtual() == false → 平台线程: ScheduledThreadPoolExecutor (daemon) 优雅关闭 (@PreDestroy): shutdown() → awaitTermination(120s) → shutdownNow() → awaitTermination(120s) → 四段式关闭，超过240秒强杀。catch 中恢复中断标志。

ValidatorConfig — Fail-Fast

@AutoConfiguration(before = ValidationAutoConfiguration.class) 核心: failFast(true) → 遇到第一个校验失败立即返回 Fail-Fast vs Fail-All: UserDTO { name:null, email:"bad", age:-1 } Fail-Fast → 只返回: "姓名不能为空"（第一条错误） Fail-All → 返回: "姓名不能为空; 邮箱格式错误; 年龄必须>0" 选择理由: 前端已有独立校验，后端只需防绕过。减少不必要的校验执行。

辅助工厂类

RegexPatternPoolFactory → extends Hutool PatternPool → 预编译 7 个正则 Pattern 为静态常量 (DICTIONARY_TYPE, ID_CARD_LAST_6 等) YmlPropertySourceFactory → 让 @PropertySource 加载 .yml 文件 → @PropertySource(value="xxx.yml", factory=YmlPropertySourceFactory.class)

记忆锚点

  • @AutoConfiguration 非 @Configuration — Spring Boot 3.0+ 新规范，通过 imports 文件自动加载。
  • CPU+1 线程数 — I/O 密集型任务的黄金法则。
  • 虚拟线程的价值 — 阻塞时被 JVM 挂起释放 OS 线程，少量 OS 线程支撑大量并发。
  • destroy() 的四段式 — shutdown → 等120s → shutdownNow → 再等120s → 强杀。
  • failFast(true) — 快速失败，配合前端校验使用，性能更好。
