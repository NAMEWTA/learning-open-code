速查 03 · 工具类下篇 — Spring 桥接工具

配套课程：lessons/0003.html · SpringUtils、ServletUtils、MessageUtils、ValidatorUtils、TreeBuildUtils

一句话定位

五个Spring 基础设施的「桥」——让非 Spring 管理的代码也能方便访问 Spring 容器的 Bean。

SpringUtils — 任何地方拿 Bean

继承: extends cn.hutool.extra.spring.SpringUtil (Hutool) 注册: @Component + ApplicationContextAware → 启动时自动注入 核心方法: SpringUtils.getBean(DictService.class) → 获取 Bean (最常用) SpringUtils.containsBean("name") → Bean 是否存在 SpringUtils.getAopProxy(bean) → 获取 AOP 代理 SpringUtils.isVirtual() → 检测虚拟线程环境

ServletUtils — 随时获取请求信息

继承: extends cn.hutool.extra.servlet.JakartaServletUtil 原理: 基于 RequestContextHolder (Spring 的 ThreadLocal 包装) 核心方法: getClientIP() → 多层代理穿透取真实 IP (6个代理头依次尝试) isAjaxRequest(req) → 四层判断 (Accept/Header/URI后缀/参数) renderString(resp, json) → 写 JSON 响应体 getParameter("name") → 类型安全参数获取+默认值

MessageUtils — 一行代码 i18n

唯一公开方法: MessageUtils.message(code, args) → MESSAGE_SOURCE.getMessage(code, args, LocaleContextHolder.getLocale()) → 找不到 key? 优雅降级返回原 code (不抛异常) 消费方: BaseException.getMessage() → MessageUtils.message(code, args) 完整链路: throw new UserException("user.not.exists", userId) → BaseException.getMessage() → MessageUtils.message("user.not.exists", [123]) → i18n: user.not.exists=用户{0}不存在 → "用户123不存在"

ValidatorUtils — 手动触发校验

ValidatorUtils.validate(dto) → 不分组 ValidatorUtils.validate(dto, AddGroup.class) → 新增分组 校验失败 → 抛 ConstraintViolationException → 全局异常处理器 → R.fail 内部获取: private static Validator VALID = SpringUtils.getBean(Validator.class) 注意: 这个 Validator 是 ValidatorConfig 配置的 fail-fast 版

TreeBuildUtils — DP 建树

继承: extends cn.hutool.core.lang.tree.TreeUtil 核心算法 build(items, parentId, classifier, action): ① groupingBy(classifier) → 子节点映射表 (DP 表, O(n)) ② 遍历 + 回溯挂载 → 构建树结构 ③ 返回 parentId 对应的子树 DEFAULT_CONFIG: nameKey="label" (适配 Element Plus 前端树组件) 额外方法: buildMultiRoot(自动识别根节点), getLeafNodes(递归取叶子)

记忆锚点

  • 「桥」模式：SpringUtils.getBean() 让 ConstraintValidator（非Spring Bean）能调 DictService。
  • getClientIP() — 可变参数 otherHeaderNames 支持扩展自定义代理头。
  • MessageUtils 优雅降级 — i18n key 找不到返回原 code，不崩系统。
  • TreeBuildUtils 的 DP — groupingBy 将 O(n²) 嵌套查找变成 O(n) 哈希查找。
