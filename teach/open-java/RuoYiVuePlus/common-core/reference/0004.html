速查 04 · 自定义校验注解

配套课程：lessons/0004.html · @Xss、@DictPattern、@EnumPattern 的设计原理

一句话定位

三个自定义 Jakarta Bean Validation 注解，分别校验 XSS 脚本、字典值合法性、枚举值合法性。每个注解 = 注解声明 + Validator 实现 + 分组标记接口 三件套。

三注解速览

注解Validator校验内容复杂度
@XssXssValidator不含 HTML 标签 (RE_HTML_MARK)低（无 initialize）
@DictPatternDictPatternValidator值在指定字典类型中存在中（查 DictService）
@EnumPatternEnumPatternValidator值在指定枚举字段中存在高（反射+缓存+@Repeatable）

@Xss — 最简校验器

注解: @Constraint(validatedBy = {XssValidator.class}) 校验: ReUtil.contains(HtmlUtil.RE_HTML_MARK, value) → 含 <script> 标签 → false → "不允许任何脚本运行" 空值: value 为空 → return true (让 @NotBlank 处理)

@DictPattern — 字典值校验（依赖倒置）

注解参数: dictType="sys_user_sex", separator="," 校验器: DictPatternValidator.isValid() → SpringUtils.getBean(DictService.class) ← 非 Spring Bean 拿 Bean → dictService.getDictLabel(dictType, value, separator) → 能查到 label → true; 查不到 → false 关键设计: core 只定义 DictService 接口，实现由业务模块提供。

@EnumPattern — 反射+缓存（最复杂）

支持 @Repeatable 多规则组合 initialize(): → enumClass.getEnumConstants() → 遍历枚举常量 → ReflectUtils.invokeGetter(e, fieldName) → 反射取字段值 → values.add(String.valueOf(fieldValue)) → 缓存到 HashSet isValid(): → values.contains(value) → O(1) 查找 注解参数: type=DeviceType.class, fieldName="device"

分组标记接口

AddGroup / EditGroup / QueryGroup — 空标记接口 (marker interface) DTO 使用: @NotBlank(groups = {AddGroup.class}) → 仅新增校验 @NotNull(groups = {AddGroup.class, EditGroup.class}) → 新增+编辑 Controller 指定: @Validated(AddGroup.class) @RequestBody UserDTO dto

记忆锚点

  • 空值放行约定：三个 isValid() 对空值都 return true，让 @NotBlank 处理空值。
  • SpringUtils.getBean() 桥接：Validator 不是 Spring Bean，通过静态方法拿 Bean。
  • @EnumPattern 的 O(1)：initialize() 一次性反射提取→HashSet，isValid() 只需 contains()。
  • 扩展新注解：必须包含 message(), groups(), payload() 三个标准属性。
