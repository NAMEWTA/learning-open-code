第 6 课 · SocialAuthStrategy 第三方登录 微信接入

🎯 本节目标：理解 SocialAuthStrategy 如何通过 JustAuth 库实现第三方登录的统一抽象，并掌握从零接入微信认证的完整步骤——包括 JustAuth 配置、前端授权流程、回调处理、账号绑定和登录态签发。

RuoYi 的第三方登录策略基于 JustAuth（一个整合了 20+ 平台的 OAuth 库）。SocialAuthStrategy 只是一个策略壳，真正的 OAuth 授权流程由 JustAuth 完成。

一、架构全景：三层协作

二、SocialAuthStrategy 源码逐行解读

SocialAuthStrategy.java · ruoyi-admin/.../service/impl/SocialAuthStrategy.java

@Service("social" + IAuthStrategy.BASE_NAME)  // bean 名 = "socialAuthStrategy"
@RequiredArgsConstructor
public class SocialAuthStrategy implements IAuthStrategy {

    private final SocialProperties socialProperties;       // ← 所有平台的OAuth配置
    private final ISysSocialService sysSocialService;      // ← 绑定关系CRUD
    private final SysUserMapper userMapper;
    private final SysLoginService loginService;

    @Override
    public LoginVo login(String body, SysClientVo client) {
        // 步骤1：解析请求体
        SocialLoginBody loginBody = JsonUtils.parseObject(body, SocialLoginBody.class);
        ValidatorUtils.validate(loginBody);

        // 步骤2：通过 JustAuth 完成 OAuth 授权
        // source = "wechat_open" | "gitee" | "github" | ...
        // socialCode = 微信回调返回的 code
        // socialState = 微信回调返回的 state（防CSRF）
        AuthResponse<AuthUser> response = SocialUtils.loginAuth(
            loginBody.getSource(), loginBody.getSocialCode(),
            loginBody.getSocialState(), socialProperties);

        if (!response.ok()) {
            throw new ServiceException(response.getMsg());
        }
        AuthUser authUserData = response.getData();

        // 步骤3：查绑定关系
        // authId = "wechat_open" + "oXXXX_user_openid"
        List<SysSocialVo> list = sysSocialService
            .selectByAuthId(authUserData.getSource() + authUserData.getUuid());

        if (CollUtil.isEmpty(list)) {
            // 未绑定 → 需要先绑定才能登录
            throw new ServiceException("你还没有绑定第三方账号，绑定后才可以登录！");
        }

        // 步骤4：加载用户 → 构建登录态 → 签发token
        SysUserVo user = loadUser(list.getFirst().getUserId());
        LoginUser loginUser = loginService.buildLoginUser(user);
        loginUser.setClientKey(client.getClientKey());
        loginUser.setDeviceType(client.getDeviceType());
        SaLoginParameter model = IAuthStrategy.buildLoginParameter(client);
        LoginHelper.login(loginUser, model);

        // 步骤5：返回token给前端
        LoginVo loginVo = new LoginVo();
        loginVo.setAccessToken(StpUtil.getTokenValue());
        loginVo.setExpireIn(StpUtil.getTokenTimeout());
        loginVo.setClientId(client.getClientId());
        return loginVo;
    }
}
⚠️ 注意：SocialAuthStrategy 不是直接处理 OAuth 回调的。它处理的是"已有绑定关系的用户用第三方账号登录"的场景。OAuth 授权和绑定是分开的两步：
① 绑定阶段：用户已登录系统 → 点击绑定微信 → 跳转授权 → 回调 POST /auth/social/callback → SysLoginService.socialRegister() 写入数据库
② 登录阶段：用户未登录 → 点击微信登录 → 跳转授权 → 回调拿到 code → POST /auth/login (grantType=social) → SocialAuthStrategy.login()

三、接入微信认证的完整步骤

第 1 步：在微信开放平台注册应用

  1. 访问 微信开放平台
  2. 创建网站应用或移动应用
  3. 获取 AppID 和 AppSecret
  4. 配置授权回调域（例如 your-domain.com）

第 2 步：添加 JustAuth 配置

在 application.yml 中添加微信开放平台的配置：

# 第三方登录配置
justauth:
  type:
    wechat_open:                           # ← source 名称
      client-id: wx1234567890abcdef        # AppID
      client-secret: abc123def456ghi789    # AppSecret
      redirect-uri: https://your-domain.com/auth/social/callback

SocialProperties.java · ruoyi-common/ruoyi-common-social/.../config/properties/SocialProperties.java

// SocialProperties 是一个 Map 结构
// key:   平台标识（"wechat_open"、"gitee"、"github"...）
// value: SocialLoginConfigProperties { clientId, clientSecret, redirectUri }
@Data
@ConfigurationProperties(prefix = "justauth")
public class SocialProperties {
    private Map<String, SocialLoginConfigProperties> type;
}

第 3 步：前端调用授权 URL 获取接口

用户点击"微信登录"按钮时，前端调用：

GET /auth/binding/wechat_open
→ 返回: { "data": "https://open.weixin.qq.com/connect/qrconnect?..." }

后端 AuthController.authBinding() 的实现：

@GetMapping("/binding/{source}")
public R<String> authBinding(@PathVariable("source") String source) {
    SocialLoginConfigProperties obj = socialProperties.getType().get(source);
    if (ObjectUtil.isNull(obj)) {
        return R.fail(source + "平台账号暂不支持");
    }
    AuthRequest authRequest = SocialUtils.getAuthRequest(source, socialProperties);
    String authorizeUrl = authRequest.authorize(AuthStateUtils.createState());
    return R.data(authorizeUrl);  // 前端拿到URL后执行 window.location.href
}

第 4 步：处理微信回调 — 绑定账号

用户在微信授权页面点击"同意"后，微信重定向回你的回调地址。前端拿到 URL 参数中的 code 和 state：

POST /auth/social/callback
Body: {
  "source": "wechat_open",
  "socialCode": "041abc123...",   // 微信返回的 code
  "socialState": "xyz..."         // 微信返回的 state
}

→ AuthController.socialCallback():
  1. StpUtil.checkLogin() — 确保当前用户已登录（绑定需要已登录状态）
  2. SocialUtils.loginAuth() — JustAuth 用 code 换 access_token + 用户信息
  3. loginService.socialRegister(authUserData) — 写入 sys_social 绑定表

第 5 步：第三方登录 — SocialAuthStrategy

绑定完成后，下次用户就可以直接用微信登录了：

POST /auth/login
Body: {
  "clientId": "eureka",
  "grantType": "social",
  "source": "wechat_open",
  "socialCode": "041abc123...",
  "socialState": "xyz..."
}

→ IAuthStrategy.login() → bean="socialAuthStrategy"
→ SocialAuthStrategy.login()
  1. JustAuth 验证 code 有效性，拿到 openid
  2. selectByAuthId("wechat_open" + openid) → 找到绑定的 userId
  3. loadUser(userId) → buildLoginUser → LoginHelper.login → 返回 token

第 6 步：在 sys_client 表添加 social 授权类型

确保客户端配置中包含 social 授权类型，否则 AuthController 会在第一步就拦截：

-- sys_client 表的 grant_type 字段需要包含 social
UPDATE sys_client SET grant_type = 'password,sms,email,social'
WHERE client_id = 'eureka';
⚠️ 常见陷阱：如果 sys_client.grant_type 字段中没有 "social"，AuthController 会直接返回 "授权类型不正确"，不会走到 SocialAuthStrategy。

四、JustAuth 的统一抽象模型

JustAuth 将所有平台的 OAuth 流程抽象为统一的接口，你不需要为每个平台写不同的代码：

// 所有平台共用同一个调用方式，只是 source 不同
SocialUtils.loginAuth(source, code, state, socialProperties);
// source = "wechat_open" → JustAuth 内部用 AuthWeChatOpenRequest
// source = "gitee"       → JustAuth 内部用 AuthGiteeRequest
// source = "github"      → JustAuth 内部用 AuthGithubRequest

// 返回统一的 AuthUser 对象
AuthUser {
    uuid:       "oXXXX_user_openid",   // 平台用户唯一标识
    username:   "用户昵称",
    nickname:   "用户昵称",
    avatar:     "https://...",
    source:     "WECHAT_OPEN",         // 平台标识
    token:      { accessToken, ... }
}

五、接入其他平台（GitHub、Gitee、QQ）

接入新平台的步骤完全一致——这就是策略模式 + JustAuth 统一的威力：

步骤微信GitHubGitee
1. 注册应用微信开放平台GitHub OAuth AppsGitee 第三方应用
2. application.ymlwechat_open:github:gitee:
3. 代码修改不需要修改任何 Java 代码
4. 数据库grant_type 包含 social
💡 核心洞察：SocialAuthStrategy 之所以能做到不改代码就接入新平台，是因为它将"平台差异"完全委托给了 JustAuth 库和配置文件。SocialUtils.loginAuth() 根据 source 参数动态选择 JustAuth 的 AuthRequest 实现——这是策略模式的递归应用：RuoYi 层用策略选择认证方式（密码/短信/社交），JustAuth 层用策略选择 OAuth 平台（微信/GitHub/Gitee）。

六、账号绑定表结构

第三方登录依赖 sys_social 表存储平台账号与系统用户的绑定关系：

sys_social 表（核心字段）：
  id          BIGINT     主键
  user_id     BIGINT     系统用户ID（关联 sys_user）
  auth_id     VARCHAR    平台+用户标识（如 "WECHAT_OPENoXXX..."）
  source      VARCHAR    平台标识（"wechat_open"）
  open_id     VARCHAR    平台用户唯一标识（微信的 openid）
  user_name   VARCHAR    平台用户名
  nick_name   VARCHAR    平台昵称
  avatar      VARCHAR    平台头像URL
