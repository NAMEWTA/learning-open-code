DromaraApplication 关键代码

启动入口 — 28 行
@SpringBootApplication
public class DromaraApplication {
    public static void main(String[] args) {
        SpringApplication application = new SpringApplication(DromaraApplication.class);
        application.setApplicationStartup(new BufferingApplicationStartup(2048));
        application.run(args);
    }
}

启动流程分解

步骤机制作用
1@SpringBootApplication= @SpringBootConfiguration + @EnableAutoConfiguration + @ComponentScan
2new SpringApplication(clazz)构造实例以设置定制项（不直接调用静态 run）
3BufferingApplicationStartup(2048)Spring Boot 3 启动性能追踪，通过 /actuator/startup 查看
4application.run(args)触发完整启动：环境准备 → Context 创建 → 自动配置 → Bean 初始化 → Jetty 启动

两个入口类对比

DromaraApplicationDromaraServletInitializer
用途内嵌 Jetty (jar 包部署)外部 Servlet 容器 (war 包部署)
基类无SpringBootServletInitializer
触发时机main() 直接调用容器启动时回调 configure()
使用场景开发 / 微服务部署传统应用服务器部署

认证相关核心配置

1. 验证码配置 (CaptchaProperties)

application.yml — captcha 段
captcha:
  enable: true          # 全局开关，控制 PasswordAuthStrategy 中图片验证码
  type: math            # math(计算) / char(字符)
  numberLength: 1       # 数学验证码位数
  charLength: 4         # 字符验证码长度

2. 密码策略配置 (SysLoginService @Value)

application.yml — user.password 段
user:
  password:
    maxRetryCount: 5    # 最大错误次数
    lockTime: 10        # 锁定时间（分钟）

Redis key: PWD_ERR_CNT_KEY + username，TTL = lockTime。过期自动清零，无需手动解锁。

3. Sa-Token 会话配置

配置项值含义
token-nameAuthorization请求头中 token 的 key
is-concurrenttrue / false是否允许同一账号并发登录
is-sharetrue / false并发登录时是否共用一个 token
jwt-secret-key自定义JWT 签名密钥

4. 客户端独立超时 (sys_client 表)

IAuthStrategy.buildLoginParameter()
SaLoginParameter model = new SaLoginParameter();
model.setDeviceType(client.getDeviceType());
model.setTimeout(client.getTimeout());           // token 有效期(秒)
model.setActiveTimeout(client.getActiveTimeout()); // 无操作过期(秒)
model.setExtra(LoginHelper.CLIENT_KEY, client.getClientId());
