核心入口: AuthController.login()

POST /auth/login — 请求流转
@SaIgnore   // 登录接口不需要鉴权
@PostMapping("/login")
@ApiEncrypt  // 请求体加密传输
public R<LoginVo> login(@RequestBody String body) {
    LoginBody loginBody = JsonUtils.parseObject(body, LoginBody.class);
    // → 提取 clientId, grantType
    // → 校验客户端合法性与状态
    // → IAuthStrategy.login(body, client, grantType)  ← 策略分发
    // → 延时 5s 发送欢迎消息
    return R.ok(loginVo);
}
⚠️ @SaIgnore 标注在 AuthController 类上，所有接口无需登录即可访问。其他 Controller 默认需要鉴权。

IAuthStrategy 接口设计（核心）

静态工厂方法 + 实例方法委托
public interface IAuthStrategy {
    String BASE_NAME = "AuthStrategy";  // Bean 名称后缀

    // ── 静态工厂：核心路由逻辑 ──
    static LoginVo login(String body, SysClientVo client, String grantType) {
        String beanName = grantType + BASE_NAME;        // "password" → "passwordAuthStrategy"
        if (!SpringUtils.containsBean(beanName)) {       // 安全检查
            throw new ServiceException("授权类型不正确!");
        }
        IAuthStrategy instance = SpringUtils.getBean(beanName);
        return instance.login(body, client);  // 委托给具体策略
    }

    // ── 构建 Sa-Token 登录参数 ──
    static SaLoginParameter buildLoginParameter(SysClientVo client) { ... }
    static SaLoginParameter buildLoginParameter(SysClientVo client, Consumer<SaLoginParameter> customizer) { ... }

    // ── 子类实现 ──
    LoginVo login(String body, SysClientVo client);
}

完整请求链路

POST /auth/login → AuthController.login() ├── 1. JsonUtils.parseObject(body, LoginBody.class) ├── 2. clientService.queryByClientId(clientId) │ → 校验: grantType 是否在客户端授权列表内？状态是否正常？ └── 3. IAuthStrategy.login(body, client, grantType) ├── beanName = grantType + "AuthStrategy" ├── SpringUtils.containsBean(beanName) → true/false ├── instance = SpringUtils.getBean(beanName) └── instance.login(body, client) ├── 解析 body → 具体请求对象 ├── 执行认证校验（验证码/密码/OAuth） ├── buildLoginUser → LoginUser ├── LoginHelper.login(loginUser, model) → Sa-Token 签发 token └── return LoginVo {accessToken, expireIn, clientId}

buildLoginParameter 共享方法

所有策略签发 token 前统一调用
static SaLoginParameter buildLoginParameter(SysClientVo client, Consumer<SaLoginParameter> customizer) {
    SaLoginParameter model = new SaLoginParameter();
    model.setDeviceType(client.getDeviceType());
    model.setTimeout(client.getTimeout());
    model.setActiveTimeout(client.getActiveTimeout());
    model.setExtra(LoginHelper.CLIENT_KEY, client.getClientId());
    if (ObjectUtil.isNotNull(customizer)) {
        customizer.accept(model);  // 策略可追加自定义参数（如 openid）
    }
    return model;
}

设计要点：接口静态方法 + Consumer 回调 = 函数式扩展。策略通过 lambda 注入差异逻辑，无需子类化。

新增第六种认证方式（两步）

// 步骤 1：新建实现类 @Service("ldapAuthStrategy") public class LdapAuthStrategy implements IAuthStrategy { @Override public LoginVo login(String body, SysClientVo client) { // 1. 解析 LdapLoginBody // 2. 调用 LDAP 认证 // 3. buildLoginUser → LoginHelper.login // 4. return LoginVo } } // 步骤 2：sys_client 表 grant_type 字段包含 "ldap" // 无需修改 AuthController、IAuthStrategy、任何已有策略 —— 开闭原则

客户端校验两道防线

防线校验逻辑失败返回
第 1 道clientId 存在 AND grantType 在 client.grantType 列表中"授权类型不正确!"
第 2 道client.status == NORMAL ("0")"授权类型已被停用!"

注意：即使策略实现类存在，若客户端未授权该 grantType，请求也会被拦截。修改数据库即可动态控制，无需重启。
