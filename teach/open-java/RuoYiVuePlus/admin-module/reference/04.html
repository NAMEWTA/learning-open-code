双层事件架构全景

LoginHelper.login(loginUser, model) └── Sa-Token 框架：签发 token │ ├── UserActionListener.doLogin() ← 第一层：Sa-Token 钩子 │ └── context.publishEvent(new UserLoginSuccessEvent(...)) │ │ │ └── UserLoginSuccessListener ← 第二层：Spring 事件监听 │ └── handleLoginSuccess() │ ├── 解析 User-Agent → 浏览器+OS │ ├── IP + 归属地解析 (ip2region) │ ├── 写入 Redis 在线用户缓存 │ ├── recordLoginInfo() → LoginInfoEvent → DB 日志 │ └── updateLastLoginInfo() → sys_user 表 │ ├── doLogout() ← 主动退出 ├── doKickout() ← 被踢下线 └── doReplaced() ← 被顶下线

第一层: UserActionListener (Sa-Token 钩子)

桥梁角色：Sa-Token 事件 → Spring 事件
@Component
public class UserActionListener implements SaTokenListener {

    @Override
    public void doLogin(String loginType, Object loginId,
                        String tokenValue, SaLoginParameter loginParameter) {
        SpringUtils.context()
            .publishEvent(new UserLoginSuccessEvent(loginId, tokenValue, loginParameter));
    }

    @Override
    public void doLogout(String loginType, Object loginId, String tokenValue) {
        RedisUtils.deleteObject(CacheNames.ONLINE_TOKEN_KEY + tokenValue);
    }

    @Override
    public void doKickout(String loginType, Object loginId, String tokenValue) {
        RedisUtils.deleteObject(CacheNames.ONLINE_TOKEN_KEY + tokenValue);
    }

    @Override
    public void doReplaced(String loginType, Object loginId, String tokenValue) {
        RedisUtils.deleteObject(CacheNames.ONLINE_TOKEN_KEY + tokenValue);
    }
}

三种退出场景对比

回调触发场景配置关联操作
doLogout()用户主动退出AuthController.logout()删 Redis 在线缓存
doKickout()管理员踢人StpUtil.kickout(loginId)删 Redis 在线缓存
doReplaced()新登录顶掉旧会话is-concurrent: false删 Redis 在线缓存

    Object loginId,

第二层: UserLoginSuccessListener (业务副作用)

5 项业务操作
@Component
public class UserLoginSuccessListener {
    @EventListener
    public void handleLoginSuccess(UserLoginSuccessEvent event) {
        // 1. User-Agent 解析
        UserAgent userAgent = UserAgentUtil.parse(
            ServletUtils.getRequest().getHeader("User-Agent"));

        // 2. IP 归属地解析
        String ip = ServletUtils.getClientIP();
        String location = AddressUtils.getRealAddressByIP(ip);  // ip2region

        // 3. 组装在线用户信息 + 写入 Redis
        UserOnlineDTO dto = new UserOnlineDTO();
        dto.setIpaddr(ip);
        dto.setLoginLocation(location);
        dto.setBrowser(userAgent.getBrowser().getName());
        dto.setOs(userAgent.getOs().getName());
        dto.setTokenId(tokenValue);
        dto.setUserName(username);
        // ... 更多字段
        RedisUtils.setCacheObject(ONLINE_TOKEN_KEY + tokenValue, dto, Duration.ofSeconds(timeout));

        // 4. 发布登录日志事件 (异步写入 DB)
        loginService.recordLoginInfo(username, LOGIN_SUCCESS, "user.login.success");

        // 5. 更新用户表登录信息
        loginService.updateLastLoginInfo(userId, ip);
    }
}

三层关注点分离

层级组件职责耦合对象
框架层UserActionListener监听 Sa-Token 生命周期，翻译为 Spring 事件Sa-Token API
业务层UserLoginSuccessListener在线信息、IP 归属地、浏览器解析、Redis 缓存Redis、ip2region、Hutool UA
日志层LoginInfoEvent + LogListener异步写入登录日志到数据库数据库、线程池

退出登录完整流程

AuthController.logout() └── loginService.logout() ├── recordLoginInfo(username, LOGOUT, "退出成功") → LoginInfoEvent → DB └── finally { StpUtil.logout() } └── UserActionListener.doLogout() → deleteObject(online_token:xxx) 关键设计：recordLoginInfo 可能抛异常，StpUtil.logout() 在 finally 中确保执行。 两层 try-catch 吞掉 NotLoginException —— "没有登录态也应正常返回"
⚠️ 设计优势：AuthController 只需 10 行，不含副作用代码。新增副作用只需加一个 @EventListener 方法，不改现有代码。测试隔离——可独立测试策略逻辑（mock 监听器），也可独立测试监听器（手动发布事件）。
