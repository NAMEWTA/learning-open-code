一、模块本质

ruoyi-ai 是 Snail AI SDK 的 Spring Boot 适配器——不实现 AI 能力，通过自动配置、条件装配和双异常处理体系将外部 AI 智能体平台嵌入 RuoYi-Vue-Plus。后端仅 3 个 Java 文件（~210 行）。

六、SSE 流式数据流

fetch() POST /snail-ai/agent/{id}/chat/stream
  → response.body.getReader()              // ReadableStream
    → TextDecoder 解码字节块
      → buffer.split(/\r?\n\r?\n/)         // SSE 事件分割
        → event: thinking → onThinking()
        → event: message  → onMessage() → setMessages(追加 content)
        → event: done     → onDone() → invalidateQueries()
        → event: error    → onError()

九、常见问题

Q: snail-ai.enabled=false 时是否有性能影响？
A: 无。@ConditionalOnProperty 阻止了所有 AI 相关 Bean 的加载和 gRPC 连接建立。启动时间、内存占用完全不受影响。

Q: 为什么不统一使用 iframe 嵌入？
A: React 端选择原生组件是为了完全控制 UI 和交互体验（与 RuoYi 风格统一）。Vue 端选择 iframe 是为了零维护成本。两种路线各有适用场景，不是技术优劣问题。

Q: 如何关闭用户注册端点但保留 AI 对话？
A: 设置 snail-ai.enabled=true + snail-ai.open-api.enabled=false。
