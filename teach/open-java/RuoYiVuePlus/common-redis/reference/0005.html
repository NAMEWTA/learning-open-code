速查 05 · 二级缓存 Caffeine + Redis

配套课程：lessons/0005.html · 文件：manager/PlusSpringCacheManager · manager/CaffeineCacheDecorator · config/CacheConfig

三件套关系

@EnableCaching (CacheConfig) → PlusSpringCacheManager (自定义 CacheManager, 解析 cacheName#ttl#maxIdle#maxSize#local) → CaffeineCacheDecorator (装饰 Redis Cache, 包一层本地 Caffeine) → TransactionAwareCacheDecorator (事务提交后写)

缓存名 # 语法

"cacheName#ttl#maxIdle#maxSize#local" Ex: "dict#1h#30m#200#1" → 缓存名=dict, TTL=1小时, 最长空闲=30分钟, 最大200条, 启用本地缓存

Caffeine 本地缓存配置

  • expireAfterWrite(30s) — 写后 30 秒自动过期
  • maximumSize(1000) — 最多 1000 条
  • initialCapacity(100)

CaffeineCacheDecorator 读写逻辑

操作逻辑
读 getcaffeine.get(key, () -> cache.get(key)) — 先本地，无则回源 Redis 并自动缓存本地
写 put先 caffeine.invalidate 清本地 → 再 cache.put 写 Redis（防旧本地值残留）
删 evict先 cache.evictIfPresent 删 Redis → 成功则 caffeine.invalidate
清 clear先清同命名空间下的本地 key → 再 cache.clear

选 RMap 还是 RMapCache

  • 无 TTL/SIZE → RMap（不过期）
  • 有 TTL/SIZE → RMapCache（带过期策略）

当前状态

  • CacheConfig.cacheManager() 未传 caffeine → 默认未启用二级缓存
  • 想开启需传入：new PlusSpringCacheManager(caffeine())
  • 适合：字典/配置/读多写少 + 能忍 30s 陈旧
  • 不适合：实时性要求高 / 频繁写 / 集群下需即时一致

→ 术语：二级缓存、CaffeineCacheDecorator（见 GLOSSARY.md）
