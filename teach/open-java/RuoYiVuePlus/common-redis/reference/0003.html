速查 03 · @RateLimiter 限流

配套课程：lessons/0003.html · 文件：annotation/RateLimiter · aspectj/RateLimiterAspect · enums/LimitType

链路

@RateLimiter 方法 → RateLimiterAspect.doBefore() @Before → getCombineKey() 拼 key → RedisUtils.rateLimiter(key,rateType,count,time,timeout) → RRateLimiter.trySetRate + tryAcquire → 成功:剩余令牌(≥0) / 失败:-1 → 抛 ServiceException

注解参数

参数默认含义
key""SpEL 业务维度，如 #user.userId
time60时间窗口（秒）
count100窗口内允许次数
limitTypeDEFAULT限流类型
timeout86400策略存活（秒），过期重置
message{rate.limiter.message}{}格式国际化

三种 LimitType（重点别记反）

类型key 加效果RateType
DEFAULT—全集群共享一个桶（控总量）OVERALL
IP客户端 IP每个 IP 一个桶OVERALL
CLUSTER实例 id每个后端实例一个桶（控单机）PER_CLIENT

key 组成

RATE_LIMIT_KEY + requestURI + ":" + [IP/实例id + ":"] + 业务key(SpEL求值)

要点

  • 令牌桶 = Redisson RRateLimiter，Lua 原子扣令牌，分布式安全（不会超发）。
  • trySetRate 幂等：仅桶不存在时设速率。
  • SpEL 支持 #参数、#{模板}、@bean 引用。
  • 限流 = 控"频率"（区别于 @RepeatSubmit 控"重复"，见第 4 课）。

→ 术语：令牌桶、SpEL 动态 key、LimitType（见 GLOSSARY.md）
