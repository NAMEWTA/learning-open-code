速查 04 · @RepeatSubmit 防重提交

配套课程：lessons/0004.html · 文件：annotation/RepeatSubmit · aspectj/RepeatSubmitAspect

链路（三个切面通知）

1. @Before doBefore() → 校验间隔≥1s → key = url + md5(token + ":" + 参数JSON) → setObjectIfAbsent(key, "", interval毫秒) ├─ true → 存 ThreadLocal → 放行 └─ false → 抛 ServiceException(重复提交) 2. @AfterReturning doAfterReturning() → 返回 success → 不删（留键锁定） → 返回 非success → 删 key（允许重试） 3. @AfterThrowing doAfterThrowing() → 删 key（允许重试）

注解参数

参数默认含义
interval5000重复判断窗口（与 timeUnit 配合），≥1000ms
timeUnitMILLISECONDS间隔时间单位
message{repeat.submit.message}{}格式国际化

核心设计：成功留键、失败删键

请求结果key 处理效果
返回 SUCCESS保留 key 等过期窗口内重复提交被拦
返回非 SUCCESS删 key可立即重试
抛异常删 key可立即重试

与 @RateLimiter 区别（核心）

@RateLimiter@RepeatSubmit
保护系统（不被刷爆）数据（不重复落库）
控制频率（N次/时间窗）同一操作的重复
底层RRateLimiter 令牌桶setIfAbsent 占位
成功后令牌随时间恢复锁键到过期

易忘点

  • key = token + 参数MD5 + url，三方定义"同一操作"边界。
  • KEY_CACHE 是 ThreadLocal，在同线程前置↔后置间传 key。
  • 过滤掉 Multipart/Request/Response/BindingResult 等不可序列化参数。
  • 参考 美团 GTIS 防重系统 思路。

→ 术语：幂等、setIfAbsent（见 GLOSSARY.md）
