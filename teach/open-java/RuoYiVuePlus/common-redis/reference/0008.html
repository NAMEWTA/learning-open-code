速查 08 · 异常处理、依赖边界与全局复盘

配套课程：lessons/0008.html · 8 节闭环完结

RedisExceptionHandler

  • @RestControllerAdvice，只兜 LockFailureException
  • 锁竞争失败 → 返回 HTTP_UNAVAILABLE + "业务处理中，请稍后再试"
  • 不处理 Redis 连接异常（归全局异常处理器）

自动装配（AutoConfiguration.imports）

RedisConfig          → Redisson 序列化/前缀/连接
CacheConfig          → @EnableCaching + Caffeine + PlusSpringCacheManager
RateLimiterConfig    → RateLimiterAspect (after=RedisConfiguration)
IdempotentConfig     → RepeatSubmitAspect (after=RedisConfiguration)

依赖边界

依赖角色边界
redisson★ 核心客户端模块基石，所有工具类 CLIENT 的源头
caffeine本地一级缓存可选，不传则二级缓存不启用
lock4j分布式锁锁归 Lock4j 管，异常归 RedisExceptionHandler 兜
sa-token读 token 名仅 RepeatSubmit 一处引用
ruoyi-common-core工具类/异常/常量redis ← core 单向依赖
ruoyi-common-jsonJSON 序列化RedisConfig 的 Jackson3 Codec、RepeatSubmit 的参数序列化

全局复盘：5 大区域一句话

区域一句话
config配好序列化/前缀/连接 → 决定所有操作的底层规则
handler加透明前缀 + 兜锁失败 → 挂在框架扩展点上的小钩子
aspectj/annotation提供限流（控频率）和防重（控重复）两个注解保护
manager搭 Caffeine+Redis 二级缓存，缓存名 # 语法传参数
utils四扇门：裸操作/注解缓存/队列/发号 → 业务日常入口
