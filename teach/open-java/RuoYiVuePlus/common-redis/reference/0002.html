装配方式

@AutoConfiguration → 返回 RedissonAutoConfigurationCustomizer
→ 在 Redisson starter 创建客户端"前"插一脚改 config（扩展而非替换）

三处定制 + 两个全局参数

定制代码解决什么
序列化CompositeCodec(StringCodec, json, json)key 可读、value 存复杂对象带类型
key 前缀setNameMapper(KeyPrefixHandler)多项目共用 Redis 不撞 key，业务无感
连接模式useSingleServer / useClusterServers哪个配置块非空用哪个
线程setThreads / setNettyThreads默认 CPU 核数 × 2
脚本缓存setUseScriptCache(true)Redisson 多基于 Lua，缓存减少网络传输

序列化关键

  • activateDefaultTyping(..., NON_FINAL)：JSON 里额外存全类名，反序列化能还原类型。
  • LocalDateTime 用 yyyy-MM-dd HH:mm:ss 格式（SimpleModule 自定义）。

反序列化安全（重点）

// 当前：全放行（等价 Jackson 2.x LaissezFaire）
BasicPolymorphicTypeValidator.builder()
    .allowIfSubType((c, clazz) -> true).build()

// 官方建议：包名白名单收紧
BasicPolymorphicTypeValidator.builder()
    .allowIfBaseType("org.dromara")
    .allowIfSubType("org.dromara").build()
  • 带类型信息 + 数据源不可信 = 反序列化漏洞风险。
  • Jackson 3.x 禁用了 LaissezFaireSubTypeValidator 外部引用。

RedissonProperties（yml: redisson.*）

  • keyPrefix、threads、nettyThreads
  • singleServerConfig / clusterServersConfig（二选一）
  • 集群多：readMode=SLAVE、subscriptionMode=MASTER
