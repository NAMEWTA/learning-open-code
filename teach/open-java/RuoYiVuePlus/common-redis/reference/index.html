定位

基于 Redisson 的缓存服务模块，把 Redis 包装成开箱即用的 缓存管理 / 限流 / 防重提交 / 分布式队列 / 发号器 五大能力。不使用 Spring RedisTemplate，全面采用 Redisson 分布式对象（RBucket/RMap/RAtomicLong/RRateLimiter 等）。

术语速查

术语解释
Redisson把 Redis 包装为 Java 分布式对象（Map/Set/Queue/AtomicLong/限流器等），本模块基石
CompositeCodec编解码器：key 用 StringCodec（redis-cli 可读）、value 用 JSON（存复杂对象并还原类型）
KeyPrefixHandler实现 NameMapper，存时加前缀、取时去前缀，对业务透明防 key 冲突
令牌桶（RRateLimiter）Redis Lua 原子维护令牌桶，tryAcquire 扣令牌，耗尽返回 -1 触发限流
LimitTypeDEFAULT（全局）/ IP（按来源 IP）/ CLUSTER（按实例），决定限流 key 维度和 RateType
幂等（防重提交）setObjectIfAbsent 原子占位：成功保留键到过期、失败立即删键允许重试
二级缓存Caffeine 本地（30s/1000条）+ Redis 分布式，读优先本地，写先失效本地再写 Redis
RBlockingQueue分布式阻塞队列（LPUSH+BRPOP），适合轻量临时任务，重量级场景用 MQ
RIdGenerator分布式发号器，按时间分片 key（如 P20260630），生成 P20260630000001 格式业务单号

常用代码

RedisUtils 基础操作：

// 对象读写
RedisUtils.setObject("user:1", user, Duration.ofMinutes(30));
User user = RedisUtils.getObject("user:1");

// Map 操作
RedisUtils.setCacheMapValue("config:system", "smtp_host", "smtp.example.com");
String host = RedisUtils.getCacheMapValue("config:system", "smtp_host");

// 原子计数器
long orderNo = RedisUtils.incrAtomicValue("order:seq", 1);

@RateLimiter 限流：

// 全局限流：所有请求共享，每秒 10 次
@RateLimiter(key = "'global'", time = 1, count = 10)
public Result sendSms(String phone) { ... }

// IP 限流：每个 IP 独立限流
@RateLimiter(key = "#phone", time = 60, count = 3, limitType = LimitType.IP)
public Result sendSms(String phone) { ... }

@RepeatSubmit 防重：

// 5 秒内同一参数不可重复提交
@RepeatSubmit(interval = 5000, message = "请勿重复提交")
@PostMapping("/order")
public Result submitOrder(@RequestBody OrderDTO order) { ... }

分布式发号器：

// 生成业务单号，格式: P2026070500001
String orderNo = SequenceUtils.nextId("P", "yyyyMMdd");
