CacheUtils（Spring Cache 命名空间）

操作签名
读get(cacheNames, key)
写put(cacheNames, key, value)
删evict(cacheNames, key)
清clear(cacheNames)
  • ⚠ 与 RedisUtils 核心区别：CacheUtils 走 Spring CacheManager（和 @Cacheable 同套抽象，key/序列化一致）；RedisUtils 走裸 RedissonClient（key 自己拼）。混用会"对不上"。
  • 通过 CacheManagerHolder 懒持有 CacheManager。

QueueUtils（分布式队列）

队列类型入队出队订阅消费
普通addQueueObjectgetQueueObject(poll)subscribeBlockingQueue
优先级addPriorityQueueObjectgetPriorityQueueObject—
  • 底层：RBlockingQueue / RPriorityBlockingQueue
  • 轻量级：适合临时小数据任务；重量级请用 MQ
  • 不支持延迟队列：Redisson 已废弃（代码注释）
  • 需 Redis 5.x+

SequenceUtils（发号器）

方法格式过期典型用途
getDateId("P")P202606300000011天订单号
getDateTimeId("O")O202606301530450000011分钟流水号
getPaddedDateId同 getDateId + 补01天定长单号
key = 前缀 + 时间戳（如 "P20260630"）
每过一天/分钟 → 新 key → 旧 key 自动过期 → 不会堆积
RIdGenerator.tryInit(1,1) + nextId() → Lua 原子递增
  • 必须用带 prefix 的版本（无参版已 @Deprecated）
  • DEFAULT_MIN_ID_CAPACITY_BITS = 6，缺位自动左补 0
  • 底层：RIdGenerator，基于 Lua 原子递增，支持步长预取

四个工具类选型一句话

用哪个当你要
RedisUtils裸 key-value / 集合 / 原子计数 / 发布订阅 / 限流
CacheUtils读 @Cacheable 注解缓存的数据（同命名空间）
QueueUtils轻量临时任务队列（不是生产级消息）
SequenceUtils生成带业务前缀+时间戳的分布式唯一单号

→ 术语：RIdGenerator、RBlockingQueue（见 GLOSSARY.md）
