速查 01 · ruoyi-common-redis 模块全景

配套课程：lessons/0001.html · 使命：全面读懂 ruoyi-common-redis 缓存服务模块

一句话定位

它是 Redisson 的业务封装层：把偏底层的 Redisson 包装成 RuoYi 业务可直接用的
「缓存 / 限流 / 防重 / 分布式队列 / 发号器 / 原子值」等开箱能力。

5 大区域 + 关键类

区域职责关键类
config 配置启动装配，定地基规则RedisConfig(序列化/前缀/单机集群)、CacheConfig、RateLimiterConfig、IdempotentConfig、RedissonProperties
handler 处理器挂在框架扩展点的钩子KeyPrefixHandler(key 加/去前缀)、RedisExceptionHandler(兜 Lock4j 锁失败)
aspectj/annotation 切面+注解把注解变成拦截行为@RateLimiter+RateLimiterAspect、@RepeatSubmit+RepeatSubmitAspect、LimitType
manager 缓存管理二级缓存核心PlusSpringCacheManager、CaffeineCacheDecorator
utils 工具业务最常用入口RedisUtils、CacheUtils、QueueUtils、SequenceUtils

四个工具类怎么选

用它场景底层对象
RedisUtils裸操作：存取对象/集合/Map、过期、限流、发布订阅、原子计数RBucket/RList/RSet/RMap/RAtomicLong/RTopic
CacheUtils操作 @Cacheable 那套 Spring Cache 命名空间Spring CacheManager
QueueUtils轻量分布式队列（普通/优先级/订阅消费）RBlockingQueue
SequenceUtils分布式唯一单号，如 P20260630000001RIdGenerator

RedisConfig 三处定制（必记）

  • 序列化 CompositeCodec：key=String（cli 可读）、value=JSON（存复杂对象）
  • 前缀 setNameMapper(KeyPrefixHandler)：所有 key 自动加项目前缀，业务无感
  • 双模式：singleServerConfig/clusterServersConfig 哪个非空用哪个
  • 安全：Jackson 3.x 用 BasicPolymorphicTypeValidator 防反序列化漏洞（建议改包名白名单）

两个注解对比

@RateLimiter@RepeatSubmit
意图控访问频率（防刷爆）防同一操作重复提交（防数据错）
底层Redisson RRateLimiter 令牌桶setObjectIfAbsent（SETNX+过期）
keyURI+(IP/实例)+SpEL keyURI+token+参数 MD5
幂等关键令牌耗尽返回 -1 → 抛异常成功留键、失败/异常删键

二级缓存

@Cacheable → PlusSpringCacheManager → [CaffeineCacheDecorator(本地30s)] → RedissonCache(Redis)
缓存名格式：cacheName#ttl#maxIdle#maxSize#local（参数直接写注解里，不改配置类）

易混点

  • RedisUtils（裸 key 自己拼）≠ CacheUtils（Spring Cache 命名空间，和 @Cacheable 同源）——选错会"对不上"。
  • 工具类用 SpringUtils.getBean 取 client，必须容器就绪后才能用（运行期工具，非启动期组件）。
  • 限流"控频率"、防重"控重复"——别混。
