动态路由速查

路由分层结构

config/config.ts:
  /login, /register, /social-callback  ← 无布局独立页
  / → BasicLayout                       ← 有布局的管理页
    /index                              ← 仪表盘
    /user/profile                       ← 个人信息
    * → DynamicPage                     ← 动态路由 catch-all

动态路由完整流程


① 用户登录 → getRouters() API
② 后端返回 BackendRoute[]（树形，按用户权限过滤）
③ reloadMenus(): backendRoutes + menuData + permissions
④ 用户访问 /system/user
⑤ UmiJS 匹配 * → DynamicPage
⑥ findRouteByPath(backendRoutes, '/system/user')
⑦ migratedPages[component] → React 组件
⑧ 渲染页面

BackendRoute 类型

{
  name, path, component, redirect?,
  meta: { title, icon?, visible, isLink?, isFrame?, isCache? },
  children?: BackendRoute[]  // 递归
}

两层权限控制

层级实现效果
菜单级后端 getRouters 过滤无权限的菜单不显示在侧边栏
按钮级前端 permissions.includes()无权限的按钮不渲染

meta 字段

字段作用
visible: false隐藏菜单（路由仍可用）
isLink: true外链（打开新标签）
isFrame: true内嵌 iframe
isCache: true页面缓存（keep-alive）
