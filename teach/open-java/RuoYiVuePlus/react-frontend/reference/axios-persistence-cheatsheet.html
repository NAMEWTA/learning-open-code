HTTP 请求与持久化速查

Axios 拦截器链路


组件调用 API
  ↓
① 请求拦截器
  ├── 附加 Authorization: Bearer <token>
  ├── 参数序列化（GET）
  ├── 防重复提交（500ms 去重）
  └── 加密（可选：RSA 加密 AES Key + AES 加密 body）
  ↓
② 发送 HTTP 请求
  ↓
③ 响应拦截器
  ├── 解密（可选）
  ├── code=200 → 返回 data
  ├── code=401 → 弹出重登 Modal（防重复弹窗）
  └── 其他 → 统一错误提示
  ↓
组件拿到数据

请求 Header 控制选项

Header作用
isToken: false跳过 Token 附加（如登录接口）
isEncrypt: true对请求体进行 RSA+AES 加密
repeatSubmit: false允许重复提交

两层加密原理


① 生成随机 AES 密钥（16位）
② AES(body, aesKey) → encryptedBody（快，适合大体积数据）
③ RSA(aesKey, 公钥) → encryptedKey（安全，只加密小体积密钥）
④ 发送 { body: encryptedBody, key: encryptedKey }
⑤ 后端 RSA 私钥解密 → 得 aesKey
⑥ 后端 AES 解密 encryptedBody → 得明文
