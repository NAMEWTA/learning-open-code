04 · Hook（下）：useRef、useMemo、useCallback 与自定义 Hook

胜利条件：能说出 useRef 和 useState 的本质区别（不触发重渲染）；知道什么时候用 useMemo/useCallback 避免不必要的计算；能读懂 ruoyi-react 自定义 Hook 的逻辑并为自己写一个。

一、useRef：一个"不触发重渲染"的容器

useRef 存一个值，你改了 不会触发重渲染。最常用于：引用 DOM 元素、保存定时器 ID、记住"上一次的值"。

useState vs useRef：本质区别

useStateuseRef
变值后触发重渲染不触发重渲染
读取valueref.current
更新setValue(v)ref.current = v
典型场景影响 UI 的数据DOM 引用、定时器、缓存、标记位

ruoyi-react 中 useRef 的用法

src/api/request.ts 中的防重复提交就是典型用例：

// 标记位：防止同一个请求重复提交
const isRelogin = { show: false };

// 缓存上一次请求的标识，用来 500ms 内去重
const requestCache = new Map();

src/hooks/useTableSelection.ts — 表格多选：

const useTableSelection = () => {
  const [selectedRowKeys, setSelectedRowKeys] = useState<Key[]>([]);
  const selectedRowsRef = useRef<any[]>([]);  // 保存完整的行数据，但不需要触发重渲染

  const onSelectChange = (keys: Key[], rows: any[]) => {
    setSelectedRowKeys(keys);     // keys 要显示 → useState
    selectedRowsRef.current = rows; // rows 存在 ref 里，操作时用
  };

  return { selectedRowKeys, selectedRowsRef, onSelectChange };
};

二、useMemo：缓存计算结果

useMemo 把昂贵的计算缓存起来，只有依赖变了才重新算。不是所有值都用 useMemo 包——只在计算确实耗时、或者要传给子组件作 props 时才用。 
// 语法
const memoizedValue = useMemo(() => {
  return 昂贵的计算(依赖1, 依赖2);
}, [依赖1, 依赖2]);  // 只有这些变了才重新计算

// ruoyi-react 真实例子：BasicLayout.tsx 中过滤菜单
const menuData = useMemo(() => {
  // 从后端路由树递归生成 ProLayout 需要的菜单格式
  return generateMenuData(backendRoutes);
}, [backendRoutes]);  // 路由不变，菜单格式不重新算

什么时候用？一个简单的判断标准

// ❌ 不需要 useMemo：计算很轻
const fullName = firstName + lastName;

// ✅ 需要 useMemo：复杂数组操作，且在 JSX 中直接使用
const filteredList = useMemo(() => {
  return bigArray.filter(...).sort(...).map(...);
}, [bigArray, keyword]);

三、useCallback：缓存函数引用

useCallback 是 useMemo 的函数版——缓存函数本身，避免每次渲染都创建新函数。只在函数作为 props 传给用 React.memo 包裹的子组件时才有意义。 
// 语法：useCallback(函数, 依赖数组)
const handleClick = useCallback(() => {
  doSomething(count);
}, [count]);  // count 变了才创建新函数

// 等价于
const handleClick = useMemo(() => {
  return () => doSomething(count);
}, [count]);
ruoyi-react 的大多数回调函数并未使用 useCallback，因为 ProTable 等 Ant Design 组件已经内部处理了渲染优化。不要盲目包装！

四、自定义 Hook：封装可复用的逻辑

自定义 Hook 就是用 use 开头的普通函数，内部可以调用其他 Hook。它封装逻辑，不封装 UI。

解剖 ruoyi-react 的自定义 Hook

src/hooks/useDict.ts — 最重要、最常用的自定义 Hook：

// 自定义 Hook = use 开头 + 调用其他 Hook + 返回数据/方法
export const useDict = (dictType: string) => {
  const { dictCache, setDictCache } = useDictStore();      // 用 Zustand store
  const queryClient = useQueryClient();                     // 用 react-query

  return useQuery({                                         // 用 react-query 的 Hook
    queryKey: ['dict', dictType],
    queryFn: async () => {
      if (dictCache[dictType]) {
        return dictCache[dictType];  // 缓存命中
      }
      const res = await getDicts(dictType);
      setDictCache(dictType, res.data);
      return res.data;
    },
    staleTime: 10 * 60 * 1000,  // 10 分钟内不重新请求
    enabled: !!dictType,
  });

  // 调用方：
  // const { data: statusOptions } = useDict('sys_normal_disable');
  // 拿到 [{value: '0', label: '正常'}, {value: '1', label: '停用'}]
};

src/hooks/useTableSelection.ts — 第二种典型模式：

export const useTableSelection = () => {
  const [selectedRowKeys, setSelectedRowKeys] = useState<Key[]>([]);
  const selectedRowsRef = useRef<any[]>([]);

  const onSelectChange = useCallback((keys: Key[], rows: any[]) => {
    setSelectedRowKeys(keys);
    selectedRowsRef.current = rows;
  }, []);

  const clearSelection = useCallback(() => {
    setSelectedRowKeys([]);
    selectedRowsRef.current = [];
  }, []);

  return { selectedRowKeys, selectedRowsRef, onSelectChange, clearSelection };
};

自定义 Hook 的模式总结

模式例子特征
数据获取型useDict内部调用 API + 缓存，返回 { data, isLoading, error }
状态封装型useTableSelection封装 useState + useCallback，返回 { 状态 + 操作方法 }
副作用封装型useLoading封装 useEffect + useState，返回 { loading, execute }
工具转换型useDateRangeQuery接收日期组件值，转成查询参数
