03 · State 与 Hook（上）：useState 和 useEffect

胜利条件：能用 useState 管理组件内部状态并解释触发重渲染的机制；能用 useEffect 在正确的时机执行副作用（挂载、更新、卸载），理解依赖数组的三种形态。

一、为什么需要 State？Props 不够吗？

Props 是只读的，父传子。State 是组件自己的可读写内存——State 变了，React 就自动重新渲染。
Props（只读，父→子） login.tsx → <LoginForm loading={loading} /> ↑ 这个值是父组件的 state State（可读写，组件内部） Screenfull.tsx → const [isFullscreen, setIsFullscreen] = useState(false); ↑ 读 ↑ 写

二、useState：组件的"变量"

语法解构

const [value, setValue] = useState(初始值);
//     ↑       ↑            ↑
//   当前值   更新函数      初始值（只在第一次渲染时用）

ruoyi-react 真实案例

src/pages/login.tsx

const LoginPage: React.FC = () => {
  // 3 个 state：各自独立，各自的 setter 只影响自己
  const [loading, setLoading] = useState(false);
  const [errorMsg, setErrorMsg] = useState('');
  const [rememberMe, setRememberMe] = useState(true);

  const handleSubmit = async (values: LoginParams) => {
    setLoading(true);                        // 1. 开始加载 → 按钮转圈
    setErrorMsg('');                         // 2. 清除旧错误
    try {
      const res = await login(values);
      setToken(res.data.access_token);       // 3. 存 token
      history.push('/index');                // 4. 跳转
    } catch (error) {
      setErrorMsg(error.message);            // 5. 显示错误
    } finally {
      setLoading(false);                     // 6. 停止加载
    }
  };

  return (
    <div>
      {errorMsg && <Alert message={errorMsg} />}
      <LoginForm
        loading={loading}         {/* state 作为 props 传下去 */}
        onSubmit={handleSubmit}
      />
    </div>
  );
};

useState 三条铁律

✅ 永远用 setter 更新 
setCount(count + 1);   // 对！触发重渲染
// count = count + 1;  // 错！不会重渲染
❌ setter 是异步的——不要连续读 
// ❌ 错误：count 是闭包里的旧值
setCount(count + 1);
setCount(count + 1);  // 还是旧值 +1，不是 +2

// ✅ 正确：用函数式更新
setCount(prev => prev + 1);
setCount(prev => prev + 1);  // 现在 +2 了
✅ State 不可变——对象/数组要整个替换 
// ❌ 错误
user.name = '新名字';
setUser(user);  // React 不会检测到变化！

// ✅ 正确
setUser({ ...user, name: '新名字' });  // 新对象

三、useEffect：在渲染之后"做点别的事"

useEffect 让你在 React 渲染完 DOM 之后执行副作用——数据请求、订阅、操作 DOM、定时器。它是函数组件连接"外部世界"的窗口。

依赖数组的三种形态（最重要！）

// 形态 1：无依赖数组 → 每次渲染都执行（几乎不用）
useEffect(() => {
  console.log('每次渲染都执行');
});

// 形态 2：空数组 [] → 只在组件挂载时执行一次
useEffect(() => {
  console.log('等价于 class 组件的 componentDidMount');
  fetchData();                 // 初始数据加载
  const timer = setInterval(() => {}, 1000);
  return () => clearInterval(timer);  // 清理函数 → 卸载时执行
}, []);

// 形态 3：有依赖 [a, b] → 挂载时 + a 或 b 变化时执行
useEffect(() => {
  console.log('count 变了，现在是', count);
  document.title = `点击了 ${count} 次`;
}, [count]);

ruoyi-react 真实案例

src/pages/system/user/index.tsx — 典型的列表页数据加载模式：

const UserPage: React.FC = () => {
  const [searchParams, setSearchParams] = useState({ pageNum: 1, pageSize: 10 });

  // 组件挂载 + searchParams 变化 → 重新请求数据
  useEffect(() => {
    fetchUserList(searchParams);
  }, [searchParams]);  // 依赖 searchParams

  // 组件卸载 → 清理（取消请求、移除监听等）
  useEffect(() => {
    return () => {
      // cleanup
    };
  }, []);
};

src/layouts/BasicLayout.tsx — 布局挂载时的初始化：

useEffect(() => {
  // 挂载时：加载菜单、初始化推送
  reloadMenus();
  initMessageBox();
  initPush();

  // 卸载时：清理推送连接
  return () => {
    closePush();
  };
}, []);  // 空数组 = 只执行一次

四、一个完整追踪：用户点击搜索 → 发生了什么

用户点击搜索按钮 ↓ onSubmit 回调 → setSearchParams({ userName: '张三', pageNum: 1 }) ↓ searchParams state 变了 → React 标记重渲染 ↓ 组件用新的 searchParams 重新渲染 ↓ useEffect 发现 [searchParams] 依赖变了 ↓ 执行 fetchUserList({ userName: '张三', pageNum: 1 }) ↓ API 返回数据 → setData(res.rows) ↓ data state 变了 → 再次重渲染 → 表格显示新数据
