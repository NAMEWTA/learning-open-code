01 · JSX：React 的模板语言

胜利条件：看到任意 JSX 代码片段能区分"HTML 部分"和"JavaScript 表达式部分"，理解 {} 的作用，并能写出带条件渲染和列表渲染的简单 JSX。

一、JSX 是什么？一句话

JSX 是 JavaScript 的语法扩展，让你在 JS 文件中直接写类似 HTML 的标签。它不是字符串模板，而是会被编译成 React.createElement() 调用。

看看 ruoyi-react 的真实入口——登录页面的 return 语句：

src/pages/login.tsx（简化）

return (
  <div className="login-container">
    <h1>若依管理系统</h1>
    <LoginForm
      onSubmit={handleSubmit}
      loading={loading}
    />
  </div>
);

这就是 JSX。它不是 HTML——它是写在 .tsx 文件里的、看起来像 HTML 的 JavaScript 表达式。

二、JSX 三条核心规则

规则 1：{} 是"切换到 JS 模式"

大括号内任何合法的 JS 表达式都能被执行：

// ✅ 表达式——能放到赋值语句 = 右边的就是表达式
<span>{user.name}</span>           // 变量
<span>{1 + 2}</span>                 // 运算
<span>{items.length}</span>           // 属性访问
<span>{getUserName()}</span>          // 函数调用
<span>{isAdmin ? '管理员' : '普通用户'}</span>  // 三元
❌ 常见错误：在大括号里写 if 语句 
<span>{if (isAdmin) { '管理员' }}</span>  // 语法错误！
✅ 正确做法：用三元表达式或提前计算 
<span>{isAdmin ? '管理员' : '普通用户'}</span>

规则 2：class → className，属性用 camelCase

因为 JSX 本质是 JavaScript，HTML 属性名必须转成 JS 的驼峰命名：

HTMLJSX
class="btn"className="btn"
onclick="handler()"onClick={handler}
tabindex="0"tabIndex={0}
for="name"htmlFor="name"

规则 3：必须有一个根元素（或 Fragment）

// ❌ 错误：两个相邻标签
return (
  <h1>标题</h1>
  <p>内容</p>
);

// ✅ 方案 A：用 <div> 包裹
return (
  <div>
    <h1>标题</h1>
    <p>内容</p>
  </div>
);

// ✅ 方案 B：用空标签 <>...</>（不产生额外 DOM）
return (
  <>
    <h1>标题</h1>
    <p>内容</p>
  </>
);

三、条件渲染：React 怎么"if"

React 没有 v-if 指令，而是用 JavaScript 本身的能力：

// 方式 1：三元表达式（二选一）
{loading ? <Spin /> : <Table data={list} />}

// 方式 2：&& 短路（只渲染或完全不渲染）
{hasError && <Alert message="出错了" />}

// 方式 3：提前 return（整体控制）
if (!token) {
  return <Redirect to="/login" />;
}
return <PageContainer>...</PageContainer>;

看 ruoyi-react 的真实例子——BasicLayout 中检查 token：

src/layouts/BasicLayout.tsx

// 没有 token → 跳转登录页
if (!getToken()) {
  return ;
}
// 有 token → 正常渲染布局
return <ProLayout {...props}>...</ProLayout>;

四、列表渲染：map 是你的 for 循环

React 没有 v-for，用 Array.map() 把数据数组映射成 JSX 数组：

// ruoyi-react 菜单渲染的核心逻辑
{menuData.map((item) => (
  <Menu.Item key={item.path} icon={item.icon}>
    {item.name}
  </Menu.Item>
))}

key 属性：每个列表项必须有一个唯一且稳定的 key，React 用它来追踪哪个项变了、加了、删了。

❌ 不要用数组索引当 key（列表重排序时会出 bug） 
{list.map((item, index) => <div key={index}>...)}
✅ 用数据的唯一 ID 
{list.map((item) => <div key={item.id}>...)}
