08 · Zustand 状态管理（下）：ruoyi-react 的 6 个 Store 详解

胜利条件：能说出 6 个 store 各自的职责和关键字段；能在代码中快速判断"这个数据应该放在哪个 store"；理解 permissionStore 的 reloadMenus 是整个应用初始化的核心方法。

6 个 Store 全景图

Store职责数据来源持久化核心字段
userStore.ts当前登录用户API getInfo()否userInfo, setUserInfo

一、userStore — 最简 Store

src/stores/userStore.ts

interface UserStore {
  userInfo: UserInfo | null;       // 用户详情
  setUserInfo: (info: UserInfo) => void;
}

export const useUserStore = create<UserStore>((set) => ({
  userInfo: null,
  setUserInfo: (userInfo) => set({ userInfo }),
}));

使用场景：

// 登录后写入
useUserStore.getState().setUserInfo(res.data);

// 顶栏读取
const { userInfo } = useUserStore();
<span>{userInfo?.nickName}</span>

二、permissionStore — 最核心的 Store 🔑

src/stores/permissionStore.ts

interface PermissionStore {
  // 数据
  backendRoutes: BackendRoute[];     // 后端返回的原始路由树
  menuData: MenuDataItem[];          // 格式化后的菜单数据
  permissions: string[];             // 权限标识列表 ['system:user:list', ...]

  // 核心方法——应用启动时调用
  reloadMenus: () => Promise<void>;
  // 1. 并行请求 getInfo() + getRouters()
  // 2. getInfo() → userStore.setUserInfo(userInfo)
  // 3. getRouters() → 格式化菜单 → menuData
  // 4. 提取权限标识 → permissions[]
  // 5. 保存原始路由树 → backendRoutes
}

reloadMenus 的时间线：

BasicLayout mount → reloadMenus() → Promise.all([getInfo(), getRouters()]) → getInfo() 返回 → userStore.setUserInfo(...) → getRouters() 返回 → 格式化菜单逻辑 → 递归遍历路由树 → 过滤 meta.visible === false 的 → 图标名称 → 图标组件 → 生成 menuData[] → 提取 permissions[] → set({ backendRoutes, menuData, permissions })

使用场景：

// BasicLayout 渲染菜单
const { menuData } = usePermissionStore();
<ProLayout menuDataRender={() => menuData} />

// DynamicPage 匹配页面组件
const { backendRoutes } = usePermissionStore();
const matchedRoute = findRouteByPath(backendRoutes, pathname);

三、appStore — 持久化的布局设置

src/stores/appStore.ts

export const useAppStore = create(
  persist(              // ← 关键！页面刷新后设置仍在
    (set) => ({
      // 布局设置
      layoutSettings: {
        navType: 'side',          // side | top | mix
        sideTheme: 'dark',        // dark | light
        fixedHeader: true,
        showLogo: true,
        tagsView: true,
        // ...
      },

      // UI 偏好
      darkMode: false,
      locale: 'zh_CN',
      componentSize: 'middle',

      // Actions
      setLayoutSettings: (settings) => set({ layoutSettings: settings }),
      toggleDarkMode: () => set((s) => ({ darkMode: !s.darkMode })),
      setLocale: (locale) => set({ locale }),
    }),
    { name: 'layout-setting-react' }
  ),
);

五、dictStore — 字典数据缓存

src/stores/dictStore.ts + src/hooks/useDict.ts

// store 只负责缓存
dictCache: Record<string, DictItem[]>;  // {'sys_normal_disable': [{value:'0',label:'正常'},...]}
setDictCache: (type, data) => set({ dictCache: { ...get().dictCache, [type]: data } });

// useDict hook 负责获取逻辑（含 react-query 的 10 分钟 staleTime）
const { data: statusOptions } = useDict('sys_normal_disable');
// 第一次：调 API → 写 dictCache → 返回
// 第二次（10分钟内）：从 react-query 缓存返回，不调 API

七、Store 之间如何协作？

场景：用户登录 login.tsx: setToken(token) ← localStorage useUserStore.getState().setUserInfo(user) ← userStore BasicLayout.tsx mount: permissionStore.getState().reloadMenus() ← permissionStore → 内部调 getInfo() → userStore.setUserInfo(...) ← 更新 userStore → 内部调 getRouters() → set({ backendRoutes, menuData }) 场景：用户退出 Dropdown.Item onClick: removeToken() ← localStorage useUserStore.getState().logout() ← userStore (清 userInfo) usePermissionStore.getState().reset() ← permissionStore (清路由) useTagsViewStore.getState().delAllViews() ← tagsViewStore (清标签) history.push('/login') ← 跳转
