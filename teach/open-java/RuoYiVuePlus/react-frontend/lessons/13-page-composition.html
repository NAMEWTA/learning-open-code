一、一个标准 CRUD 页面的 6 层结构

以 src/pages/system/user/index.tsx 为范本：

<PageContainer> ← ① 页面容器（面包屑 + 标题） <ProTable ← ② 核心表格 headerTitle="用户列表" rowKey="userId" columns={columns} ← ③ 列定义（含搜索、渲染） request={fetchUserList} ← ④ 数据源（自动分页） toolBarRender={...} ← ⑤ 工具栏（新增、导出） rowSelection={...} ← 多选 /> <EditModal /> ← ⑥ 编辑弹窗（新增/编辑共用） <DetailDrawer /> ← 详情抽屉 </PageContainer>

二、columns：一列包含什么？

ProTable 的 columns 定义最值得细读——它不仅定义表格显示，还定义搜索表单：

const columns: ProColumns<User>[] = [
  {
    title: '用户名称',
    dataIndex: 'userName',
    // --- 搜索配置 ---
    valueType: 'text',
    fieldProps: { placeholder: '请输入用户名称' },
    // --- 表格渲染 ---
    render: (_, record) => (
      <a onClick={() => handleDetail(record.userId)}>{record.userName}</a>
    ),
  },
  {
    title: '状态',
    dataIndex: 'status',
    // --- 搜索（下拉选择）---
    valueType: 'select',
    valueEnum: {               // ← 枚举值映射
      0: { text: '正常', status: 'Success' },
      1: { text: '停用', status: 'Error' },
    },
    // --- 表格渲染 ---
    render: (_, record) => (
      <Switch
        checked={record.status === '0'}
        onChange={(val) => handleChangeStatus(record.userId, val ? '0' : '1')}
      />
    ),
  },
  {
    title: '创建时间',
    dataIndex: 'createTime',
    valueType: 'dateRange',   // ← 搜索时是日期范围选择器
    hideInSearch: false,      // ← 显示在搜索栏
    hideInTable: false,       // ← 显示在表格
    render: (_, record) => <span>{record.createTime}</span>,
  },
  {
    title: '操作',
    valueType: 'option',      // ← 操作列（不参与搜索）
    render: (_, record) => [
      <Button key="edit" onClick={() => handleEdit(record)}>编辑</Button>,
      <Button key="delete" danger onClick={() => handleDelete(record.userId)}>删除</Button>,
    ],
  },
];

valueType 决定了搜索控件类型

valueType搜索表单渲染为ruoyi-react 使用场景
textInput 文本框用户名、手机号搜索
selectSelect 下拉框状态、性别等枚举搜索
dateRangeDatePicker.RangePicker创建时间范围搜索
dateDatePicker单个日期搜索
treeSelectTreeSelect 树形选择部门选择
option不参与搜索操作列
money数字输入框金额搜索

三、request：ProTable 的数据源

request 是 ProTable 最强大的能力——你只需传一个返回约定格式的异步函数，ProTable 自动处理分页、排序、搜索：

// ProTable 约定的函数签名
const fetchUserList = async (params: {
  current?: number;     // ProTable 传入当前页码
  pageSize?: number;     // ProTable 传入每页条数
  userName?: string;     // 搜索栏的值自动合并进来！
  status?: string;
  createTime?: [string, string];
  // ... 所有 column 中 hideInSearch !== true 的 dataIndex 都会出现在这里
}) => {
  const res = await getUserList({
    pageNum: params.current!,
    pageSize: params.pageSize!,
    ...params,          // 搜索条件直接展开
  });

  return {
    data: res.data.rows,       // 表格数据
    total: res.data.total,     // 总条数（ProTable 自动算分页）
    success: res.code === 200, // 是否成功
  };
};

// 使用
<ProTable
  request={fetchUserList}
  // ProTable 自动：
  // - 首次加载 → request({ current: 1, pageSize: 20 })
  // - 点击搜索 → request({ current: 1, pageSize: 20, userName: '张三' })
  // - 翻页     → request({ current: 2, pageSize: 20, userName: '张三' })
/>

四、弹窗表单：新增/编辑共用模式

const EditModal: React.FC<{
  visible: boolean;
  record?: User;                        // 编辑时有值，新增时 undefined
  onClose: () => void;
  onSuccess: () => void;
}> = ({ visible, record, onClose, onSuccess }) => {
  const [form] = Form.useForm();
  const isEdit = !!record;

  // 编辑时回填表单
  useEffect(() => {
    if (visible && record) {
      form.setFieldsValue(record);
    }
    if (visible && !record) {
      form.resetFields();              // 新增时重置
    }
  }, [visible, record]);

  const handleSubmit = async () => {
    const values = await form.validateFields();
    if (isEdit) {
      await updateUser(record.userId, values);  // 更新
    } else {
      await addUser(values);                    // 新增
    }
    message.success(isEdit ? '修改成功' : '新增成功');
    onSuccess();  // 通知父组件刷新列表
  };

  return (
    <Modal
      open={visible}
      title={isEdit ? '编辑用户' : '新增用户'}
      onCancel={onClose}
      onOk={handleSubmit}
    >
      <Form form={form} layout="vertical">
        <Form.Item name="userName" label="用户名称" rules={[{ required: true }]}>
          <Input />
        </Form.Item>
        <Form.Item name="status" label="状态">
          <Radio.Group>
            <Radio value="0">正常</Radio>
            <Radio value="1">停用</Radio>
          </Radio.Group>
        </Form.Item>
        {/* 新增时不显示密码字段 */}
        {!isEdit && (
          <Form.Item name="password" label="密码" rules={[{ required: true }]}>
            <Input.Password />
          </Form.Item>
        )}
      </Form>
    </Modal>
  );
};

五、字典数据如何在表格中使用

字典是 ruoyi-react 最常用的数据模式——把 { code: '0', label: '正常' } 这种枚举数据存后端，前端通过 useDict 获取并在表格中渲染：

const UserPage = () => {
  // 获取字典
  const { data: statusDict } = useDict('sys_normal_disable');
  const { data: sexDict } = useDict('sys_user_sex');

  const columns = [
    {
      title: '状态',
      dataIndex: 'status',
      render: (_, record) => {
        // 把 '0' 映射成 '正常'
        const item = statusDict?.find(d => d.value === record.status);
        return <Tag color={item?.color}>{item?.label}</Tag>;
      },
    },
    // 如果只是简单映射，可用 valueEnum
    {
      title: '性别',
      dataIndex: 'sex',
      valueEnum: sexDict?.reduce((acc, d) => {
        acc[d.value] = { text: d.label };
        return acc;
      }, {} as Record<string, { text: string }>),
    },
  ];
};

六、完整数据流图（用户管理 CRUD）

页面加载 useDict('sys_normal_disable') → 获取字典（cache 10min） ↓ ProTable mount → request(current:1, pageSize:20) ↓ getUserList({ pageNum: 1, pageSize: 20 }) → request.ts 拦截器附加 token → GET /system/user/list?pageNum=1&pageSize=20 ← { code: 200, data: { rows: [...25条], total: 125 } } ↓ ProTable 渲染 25 行 × N 列 - 状态列：render(status) → 从 statusDict 翻译 → <Tag> - 操作列：render → <Button>编辑 <Button>删除 用户点击"新增" → setEditVisible(true); setCurrentRecord(undefined) → EditModal visible=true, record=undefined → form.resetFields()（空表单） ↓ 用户填写 → 点击确定 → form.validateFields() → { userName, status, password } → addUser(values) → POST /system/user → 后端保存 → 返回成功 → message.success('新增成功') → onSuccess() → setEditVisible(false) + tableRef.reload() → ProTable 自动重新 request → 表格刷新
