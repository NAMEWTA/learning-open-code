06 · 路由系统与布局：ProLayout 如何撑起整个后台

胜利条件：能画出 ruoyi-react 的路由层级结构（静态路由→BasicLayout→子路由→DynamicPage）；理解 ProLayout 的核心 props；能解释菜单从后端路由树到左侧导航栏的完整转换过程。

一、两级路由：静态 vs 动态

ruoyi-react 的路由分两层：静态路由写在 config.ts 里（登录、首页等固定页），动态路由从后端 API 拿到、由 DynamicPage 在运行时匹配。
config/config.ts 中的 routes │ ├── /login → pages/login.tsx (无布局，独立页) ├── /register → pages/register.tsx (无布局) ├── /social-callback → pages/socialCallback.tsx (无布局) ├── /401, /404 → pages/error/ (无布局) │ └── / → BasicLayout ← 所有管理页面共用此布局 ├── / → redirect → /index ├── /index → pages/index.tsx (仪表盘) ├── /user/profile → pages/user/profile (个人信息) └── * (catch-all) → pages/dynamicPage.tsx (动态路由)

二、BasicLayout：整个后台的"壳"

src/layouts/BasicLayout.tsx（500+ 行）是 ruoyi-react 的心脏。

核心结构（精简）

const BasicLayout: React.FC = () => {
  // ===== 1. 数据层：从 Store 获取状态 =====
  const token = getToken();
  const { userInfo } = useUserStore();
  const { menuData, backendRoutes, reloadMenus } = usePermissionStore();
  const { layoutSettings } = useAppStore();

  // ===== 2. 副作用层：挂载时初始化 =====
  useEffect(() => {
    if (!getToken()) return;          // 无 token → 不加载
    reloadMenus();                    // 调后端 API：getInfo() + getRouters()
    initPush();                       // 初始化实时推送
    return () => closePush();         // 卸载清理
  }, []);

  // ===== 3. 没有 token → 跳转登录 =====
  if (!token) {
    return <Redirect to="/login" />;
  }

  // ===== 4. 渲染 ProLayout =====
  return (
    <ProLayout
      // --- 数据 ---
      title="若依管理系统"
      logo="..."
      menu={{ defaultOpenAll: true }}
      menuDataRender={() => menuData}       // 菜单数据

      // --- 布局模式 ---
      layout={layoutSettings.navType}       // side(侧边) / top(顶部) / mix(混合)
      navTheme={layoutSettings.sideTheme}   // dark / light
      fixedHeader={layoutSettings.fixedHeader}

      // --- 右侧操作区 ---
      actionsRender={() => [
        <MenuSearch />,
        <MessageBox />,
        <Screenfull />,
        <LocaleSelect />,
        <SizeSelect />,
        <LayoutSettings />,
      ]}

      // --- 用户下拉菜单 ---
      avatarProps={{
        src: userInfo?.avatar,
        title: userInfo?.nickName,
        render: (_, dom) => (
          <Dropdown menu={{ items: userMenuItems }}>{dom}</Dropdown>
        ),
      }}

      // --- 标签页（多tab）---
      {...(tagsView && {
        tagRender: () => <TagsView />,
      })}
    >
      // 子路由的内容渲染在这里
      <Outlet />
    </ProLayout>
  );
};

三、ProLayout 的 7 个核心 Props

Prop作用ruoyi-react 传了什么
menuDataRender左侧菜单的数据源从 permissionStore 的 menuData（后端路由树格式化后的数组）
layout布局模式side / top / mix，用户可在设置面板切换
navTheme侧边栏主题dark / light
actionsRender顶栏右侧操作区搜索、消息、全屏、语言、尺寸、设置
avatarProps右上角头像+下拉菜单用户头像、昵称、下拉（个人中心/退出）
children内容区（页面主体）<Outlet />——子路由渲染在这里
tagRender标签页栏（可选）<TagsView />——多 tab 切换

四、菜单数据流：后端路由 → 侧边栏

① 后端 API: GET /system/menu/getRouters?userId=1 返回→ BackendRoute[] （树形，含 component:'@/views/system/user/index'） ② permissionStore.reloadMenus() - 调 getInfo() 拿用户信息 → userStore - 调 getRouters() 拿路由树 → permissionStore.backendRoutes ③ 菜单格式化（menu.tsx） - 遍历 backendRoutes 树 - 过滤隐藏菜单（meta.visible=false） - 转成 ProLayout menuDataRender 需要的格式 - 图标处理（Ant Design Icon + Iconify 混合） - 设置 key/path/name/icon/children ④ ProLayout 渲染 - menuDataRender 返回格式化后的菜单数组 - ProLayout 渲染为左侧导航栏

五、布局设置持久化

用户调整了主题、布局模式后，这些偏好需要记住。src/stores/appStore.ts 负责：

export const useAppStore = create(
  persist(                    // ← Zustand 的 persist 中间件！
    (set) => ({
      layoutSettings: {
        navType: 'side',
        sideTheme: 'dark',
        fixedHeader: true,
        showLogo: true,
        tagsView: true,
        // ...
      },
      setLayoutSettings: (settings) => set({ layoutSettings: settings }),
    }),
    {
      name: 'layout-setting-react',  // localStorage 的 key
    },
  ),
);

这就是"持久化"的一部分——布局设置自动同步到 localStorage，刷新后不丢失。

六、实战检验

  4. 修改布局设置（如切换暗黑模式），观察 localStorage 中该 key 的变化
