二、Token 完整生命周期

第 1 站：登录 → 拿到 token

src/pages/login.tsx

const handleSubmit = async (values: LoginParams) => {
  setLoading(true);
  try {
    // 1. 调登录 API（请求体被 RSA+AES 加密！）
    const res = await login({
      username: values.username,
      password: values.password,
      uuid: captchaInfo.uuid,
      code: values.code,
    });

    // 2. 存入 localStorage
    setToken(res.data.access_token);
    // → localStorage.setItem('Admin-Token', token_value)

    // 3. 记住我（可选）
    if (values.rememberMe) {
      setCookie('username', values.username);      // 存用户名到 Cookie
      localStorage.setItem('rememberMe', 'true');
    }

    // 4. 跳转
    history.push(from || '/index');
  } catch (error) {
    setErrorMsg(error.message);
  } finally {
    setLoading(false);
  }
};

三、auth.ts：Token 存取工具

src/utils/auth.ts

const TOKEN_KEY = 'Admin-Token';

export function getToken(): string | null {
  return localStorage.getItem(TOKEN_KEY);
}

export function setToken(token: string): void {
  localStorage.setItem(TOKEN_KEY, token);
}

export function removeToken(): void {
  localStorage.removeItem(TOKEN_KEY);
}
为什么用 localStorage 而不是 sessionStorage？
localStorage 在新标签页、浏览器重启后仍存在（取决于"记住我"策略）。sessionStorage 关标签页就没了。后端通过 token 过期时间控制安全性，前端只管存取。

四、Zustand persist：自动化的持久化

appStore 和 tagsViewStore 使用 Zustand 的 persist 中间件，声明式持久化——不用手写 localStorage：

// appStore: 布局设置持久化
export const useAppStore = create(
  persist(
    (set) => ({
      darkMode: false,
      layoutSettings: {...},
      toggleDarkMode: () => set(s => ({ darkMode: !s.darkMode })),
    }),
    {
      name: 'layout-setting-react',  // ← localStorage key
      // Zustand 自动：
      // - 每次 state 变化 → JSON.stringify → localStorage
      // - 应用启动时 → localStorage.getItem → JSON.parse → 初始化 state
    }
  )
);

五、"记住我"功能追踪

// 登录时：勾选"记住我"
→ setCookie('username', values.username)      // Cookie（可设过期时间）
→ localStorage.setItem('rememberMe', 'true')

// 下次打开登录页：检查是否记住
useEffect(() => {
  if (localStorage.getItem('rememberMe') === 'true') {
    const savedUsername = getCookie('username');
    if (savedUsername) {
      form.setFieldsValue({ username: savedUsername });
      setRememberMe(true);
    }
  }
}, []);

六、安全性考量

存储位置风险ruoyi-react 的应对
localStorage TokenXSS 攻击可读取后端 token 有过期时间；登录请求体加密
localStorage 布局设置低风险仅 UI 偏好，无敏感数据
Cookie 用户名XSS 可读取仅存用户名（非密码），可选功能
