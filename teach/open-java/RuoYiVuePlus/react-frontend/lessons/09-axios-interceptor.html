09 · HTTP 请求层：Axios 拦截器链与请求封装

胜利条件：能画出"组件调 API → 请求拦截器 → 发送 → 响应拦截器 → 组件"的完整链路；理解 ruoyi-react 在拦截器中附加了哪些逻辑（token、加密、去重）；能解释 401 处理、RSA+AES 两层加密的原理。

一、一个请求的完整旅程

src/api/request.ts 是 ruoyi-react HTTP 层核心。每个 API 调用都走这条链：

组件调用 service.get('/system/user/list', { params }) ↓ ① 请求拦截器（request interceptor） ├── 附加 Token → headers.Authorization = 'Bearer xxx' ├── 参数序列化 → GET 请求的 params 转成 URL query string ├── 防重复提交 → 检查是否 500ms 内相同请求 └── 加密（可选）→ RSA 加密 AES Key + AES 加密请求体 ↓ ② 发送 HTTP 请求到后端 ↓ ③ 响应拦截器（response interceptor） ├── 解密（可选）→ AES 解密响应体 ├── 处理 401 → 弹出重新登录 Modal ├── 处理其他错误 → 统一错误提示 └── 返回 res.data ↓ 组件拿到数据 → setState → 渲染

二、请求拦截器：出门前"打扮"请求

src/api/request.ts 中的请求拦截器：

service.interceptors.request.use((config) => {
  // 1. 附加 Token（除非标记 isToken: false）
  if (getToken() && config.headers?.isToken !== false) {
    config.headers.Authorization = 'Bearer ' + getToken();
  }

  // 2. GET 请求参数序列化
  if (config.method === 'get' && config.params) {
    config.paramsSerializer = (params) => qs.stringify(params, {
      arrayFormat: 'repeat',  // ids=[1,2,3] → ids=1&ids=2&ids=3
    });
  }

  // 3. 防重复提交（500ms 内相同 URL + 参数 + 方法的请求去重）
  if (config.headers?.repeatSubmit !== false) {
    const requestKey = JSON.stringify({ url, method, params, data });
    if (requestCache.has(requestKey)) {
      return Promise.reject(new Error('重复请求'));
    }
    requestCache.set(requestKey, true);
    setTimeout(() => requestCache.delete(requestKey), 500);
  }

  // 4. 加密（标记 isEncrypt: true 时触发）
  if (config.headers?.isEncrypt) {
    const aesKey = generateAesKey();          // 随机 16 位 AES 密钥
    const encryptedBody = aesEncrypt(JSON.stringify(config.data), aesKey);
    const encryptedKey = rsaEncrypt(aesKey, publicKey);
    config.data = { body: encryptedBody, key: encryptedKey };
  }

  return config;
});

三、RSA + AES 两层加密原理

为什么不用单纯的 RSA 或 AES？因为 RSA 不适合加密大量数据，AES 的密钥传输不安全。两层加密取各自所长：AES 加密数据体（快），RSA 加密 AES 密钥（安全）。
客户端 服务端 │ │ │ 1. 生成随机 AES 密钥 (16位) │ │ 2. AES 加密请求体 │ │ encryptedBody = AES(body, aesKey) │ │ 3. RSA 加密 AES 密钥 │ │ encryptedKey = RSA(aesKey, 公钥) │ │ │ │ 4. 发送 { body: encryptedBody, │ │ key: encryptedKey } │ ├──────────────────────────────────────→│ │ │ 5. RSA 私钥解密 → 拿到 AES 密钥 │ │ 6. AES 密钥解密 body → 拿到明文

公钥在 .env.development 中配置，私钥在后端。

四、响应拦截器：回家后"检查"响应

service.interceptors.response.use((response) => {
  // 1. 解密（如果响应被加密）
  if (response.config.headers?.isEncrypt) {
    const decrypted = aesDecrypt(response.data.body, aesKey);
    response.data = JSON.parse(decrypted);
  }

  // 2. 检查业务 code
  const { code, msg } = response.data;
  if (code === 200) {
    return response.data;  // 成功，返回数据
  }

  // 3. 处理各种错误码
  if (code === 401) {
    handleReLogin();  // → 弹出重新登录 Modal
  } else {
    message.error(msg || '请求失败');
  }

  return Promise.reject(new Error(msg));
}, (error) => {
  // 4. 网络错误
  message.error('网络连接失败');
  return Promise.reject(error);
});

五、401 处理：一个精妙的防重复设计

let isRelogin = { show: false };  // 模块级变量，防重复弹窗

const handleReLogin = () => {
  if (isRelogin.show) return;     // 已经弹着了，不再弹
  isRelogin.show = true;

  Modal.confirm({
    title: '系统提示',
    content: '登录状态已过期，请重新登录',
    onOk: () => {
      isRelogin.show = false;
      removeToken();
      history.push('/login');
    },
    onCancel: () => {
      isRelogin.show = false;
    },
  });
};

如果 3 个 API 同时返回 401，只有第一个会弹出 Modal，后两个因为 isRelogin.show === true 直接 return。

六、API 调用封装：从原始 axios 到业务调用

API 定义分两层：

// 第 1 层：request.ts 导出带拦截器的 service
export const service = axios.create({ baseURL, timeout });
// ...拦截器配置...

// 第 2 层：业务 API（src/api/system/user/index.ts）
export const getUserList = (params: UserQuery) =>
  service.get<R<PageResult<User>>>('/system/user/list', { params });

export const addUser = (data: AddUserParams) =>
  service.post<R<void>>('/system/user', data);

export const updateUser = (data: UpdateUserParams) =>
  service.put<R<void>>('/system/user', data);

export const deleteUser = (ids: string[]) =>
  service.delete<R<void>>(`/system/user/${ids.join(',')}`);

七、API 类型体系（响应包装）

src/api/types.ts

// 后端统一响应格式
interface R<T> {
  code: number;
  msg: string;
  data: T;
}

// 分页查询参数
interface PageQuery {
  pageNum: number;
  pageSize: number;
  [key: string]: any;  // 其他搜索条件
}

// 分页结果
interface PageResult<T> {
  rows: T[];
  total: number;
}
