12 · 动态路由（下）：DynamicPage 匹配与权限控制

胜利条件：能画出 DynamicPage 的完整匹配流程（URL→路由树查找→migratedPages映射→渲染）；理解 migratedPages 映射表的作用和为什么存在；能独立添加一个新的页面映射到动态路由中。

一、DynamicPage：动态路由的"调度中心"

在 config.ts 中，path: '*'（catch-all）指向 src/pages/dynamicPage.tsx。所有不在静态路由中的 URL，都会进入这个组件，由它在运行时决定渲染哪个页面。
用户访问 /system/user ↓ UmiJS 路由匹配 ├── /login? → 不匹配 ├── /index? → 不匹配 ├── /user/profile? → 不匹配 └── * (catch-all) → 匹配！→ 渲染 DynamicPage ↓ DynamicPage 运行时匹配 ↓ 找到 → 渲染 SystemUserPage 找不到 → 显示迁移占位卡片

二、DynamicPage 核心逻辑

src/pages/dynamicPage.tsx

const DynamicPage: React.FC = () => {
  const pathname = useLocation().pathname;                    // 当前 URL
  const { backendRoutes } = usePermissionStore();             // 后端路由树

  // === 第 1 步：在路由树中查找 ===
  const matchedRoute = useMemo(() => {
    return findRouteByPath(backendRoutes, pathname);
  }, [backendRoutes, pathname]);

  // === 第 2 步：没找到 → 404 ===
  if (!matchedRoute) {
    return <Result status="404" title="页面不存在" />;
  }

  // === 第 3 步：外链模式 ===
  if (matchedRoute.meta?.isLink && matchedRoute.path) {
    return <IframePage src={matchedRoute.path} />;
  }

  // === 第 4 步：iframe 嵌入模式 ===
  if (matchedRoute.meta?.isFrame) {
    return <IframePage src={matchedRoute.path} />;
  }

  // === 第 5 步：查找对应的 React 组件 ===
  const componentKey = matchedRoute.component;
  // componentKey 格式: '@/views/system/user/index'（Vue 风格路径）
  const PageComponent = migratedPages[componentKey];

  if (PageComponent) {
    return <PageComponent />;     // ✅ 找到！渲染 React 页面
  }

  // === 第 6 步：没有 React 组件→显示占位 ===
  return <MigrationPlaceholder route={matchedRoute} />;
};

三、migratedPages：路径→组件的映射表

这是 ruoyi-react 的迁移架构核心。后端返回的 component 路径仍是 Vue 时代的 @/views/system/user/index，而 migractedPages 把 Vue 路径映射到实际的 React 组件。

四、findRouteByPath：递归路由树查找

// 在路由树（含嵌套 children）中查找匹配 pathname 的路由
function findRouteByPath(
  routes: BackendRoute[],
  targetPath: string
): BackendRoute | null {
  for (const route of routes) {
    // 精确匹配
    if (route.path === targetPath) {
      return route;
    }
    // 递归查找子路由
    if (route.children?.length) {
      const found = findRouteByPath(route.children, targetPath);
      if (found) return found;
    }
  }
  return null;
}

五、权限控制的两层过滤

ruoyi-react 的权限控制分两层：

第 1 层：菜单级（后端控制）

后端 getRouters 已经根据用户角色过滤了路由树——用户没有权限的菜单根本不返回。所以侧边栏自然看不到。

第 2 层：按钮级（前端指令）

src/stores/permissionStore.ts 中的 permissions 数组（从路由树提取的权限标识）用于控制页面内按钮的显隐：

// permissions 数组示例
['system:user:list', 'system:user:add', 'system:user:edit', 'system:user:remove']

// 在页面中判断
const { permissions } = usePermissionStore();
const hasAddPermission = permissions.includes('system:user:add');

return (
  <>
    {hasAddPermission && <Button>新增用户</Button>}
  </>
);

六、如何新增一个动态路由页面？

假设后端已经配好菜单，前端只需两步。

步骤 1：创建页面组件

// src/pages/report/index.tsx
const ReportPage: React.FC = () => {
  return (
    <PageContainer>
      <ProCard title="报表管理">
        {/* 页面内容 */}
      </ProCard>
    </PageContainer>
  );
};
export default ReportPage;

步骤 2：在 DynamicPage 中注册映射

// src/pages/dynamicPage.tsx
import ReportPage from './report';

const migratedPages = {
  // ... 已有映射
  '@/views/report/index': ReportPage,  // ← 新增这一行
};

Done！刷新页面，访问 /report 就能看到新页面了。

七、完整的动态路由追踪

1. 管理员在后台给"张三"分配了"用户管理"和"角色管理"菜单 2. 张三登录 → getRouters 返回： [{ path: '/system', children: [ { path: '/system/user', component: '@/views/system/user/index' }, { path: '/system/role', component: '@/views/system/role/index' }, ] }] （注意：没有 /system/menu，因为张三没权限） 3. reloadMenus → backendRoutes + menuData → 侧边栏只有 2 项 4. 张三点击"用户管理"→ history.push('/system/user') 5. UmiJS 路由匹配 → '*' catch-all → DynamicPage 6. DynamicPage: findRouteByPath(backendRoutes, '/system/user') → 找到 { path: '/system/user', component: '@/views/system/user/index' } 7. migratedPages['@/views/system/user/index'] → SystemUserPage 8. 渲染 <SystemUserPage />

八、实战检验

  1. 打开 src/pages/dynamicPage.tsx，逐行阅读完整代码，找出 migratedPages 映射表
  2. 在 Chrome DevTools 中给 DynamicPage 的 matchedRoute 处打断点，点击不同菜单观察 matchedRoute 的值
  3. 人为改变 mappedPages 中某个 key，访问该页面，观察是否显示迁移占位卡片
