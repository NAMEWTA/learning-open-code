11 · 动态路由（上）：后端路由树与菜单生成

胜利条件：能说出 ruoyi-react 动态路由的完整流程（后端 API → 路由树 → 菜单格式化 → ProLayout 渲染）；理解 BackendRoute 的数据结构和 meta 字段的含义；能解释为什么路由不走前端 routes.ts 而是从后端获取。

一、为什么需要动态路由？

后台管理系统的菜单因人而异——普通用户只能看自己的菜单，管理员才能看所有菜单。如果路由写死在前端，每个用户都会看到全部页面（即使打不开）。动态路由让后端根据用户权限返回他能访问的路由树，前端据此渲染菜单。

二、数据源头：后端返回什么？

API: GET /system/menu/getRouters，后端返回一个树形结构：

src/api/types.ts — BackendRoute 类型

export interface BackendRoute {
  name: string;            // 路由名称，如 'System'
  path: string;            // 路由路径，如 '/system'
  component: string;       // 前端组件路径，如 '@/views/system/user/index'
  redirect?: string;       // 重定向
  query?: string;          // 查询参数
  meta: {
    title: string;         // 显示标题，如 '用户管理'
    icon?: string;         // 图标名称，如 'user' 或 'ant-design:user-outlined'
    visible: boolean;      // 是否在菜单中显示（false=隐藏但路由可用）
    isLink?: boolean;      // 是否外链
    isFrame?: boolean;     // 是否 iframe 内嵌
    isCache?: boolean;     // 是否缓存页面
  };
  children?: BackendRoute[]; // 子路由（递归结构）
}

三、reloadMenus：路由加载的起点

src/stores/permissionStore.ts

reloadMenus: async () => {
  // 1. 并行请求：用户信息 + 路由树
  const [userRes, routeRes] = await Promise.all([
    getInfo(),
    getRouters(),
  ]);

  // 2. 写入 userStore
  userStore.getState().setUserInfo(userRes.data);

  // 3. 拿到后端路由树
  const backendRoutes = routeRes.data;  // BackendRoute[]

  // 4. 格式化成菜单数据
  const menuData = generateMenuData(backendRoutes);

  // 5. 提取权限标识
  const permissions = extractPermissions(backendRoutes);

  // 6. 更新 store
  set({ backendRoutes, menuData, permissions });
}

四、菜单格式化：从 BackendRoute[] → MenuDataItem[]

src/api/menu.ts（或 permissionStore 内部）——这是整个流程中最"繁琐"的环节：

// 核心转换逻辑（简化）
function generateMenuData(routes: BackendRoute[]): MenuDataItem[] {
  return routes
    .filter(route => route.meta?.visible !== false)  // 过滤隐藏菜单
    .map(route => {
      const menu: MenuDataItem = {
        key: route.path,                    // ProLayout 用 path 当 key
        path: route.path,
        name: route.meta?.title,
        icon: resolveIcon(route.meta?.icon), // 图标名 → 图标组件
      };

      // 递归处理子路由（如果还有嵌套）
      if (route.children && route.children.length > 0) {
        menu.children = generateMenuData(route.children);
      }

      return menu;
    });
}

// 图标解析：支持两种格式
function resolveIcon(iconName?: string): React.ReactNode {
  if (!iconName) return null;

  // 格式 1: "ant-design:user-outlined" → 使用 @iconify/react
  if (iconName.includes(':')) {
    const [prefix, name] = iconName.split(':');
    return <Icon icon={`${prefix}:${name}`} />;
  }

  // 格式 2: "user" → 从 @ant-design/icons 按名映射
  const AntIcon = antdIconsMap[iconName];
  return AntIcon ? <AntIcon /> : null;
}

五、完整数据流

用户登录 → 跳转首页 ↓ BasicLayout mount ↓ useEffect(() => { reloadMenus() }, []) ↓ Promise.all([getInfo(), getRouters()]) │ ├── getInfo() → { user: { nickName, avatar, roles, permissions } } │ → useUserStore.setUserInfo(user) │ └── getRouters() → BackendRoute[] │ ├── 保存原始树 → permissionStore.backendRoutes │ （DynamicPage 用来匹配页面组件） │ ├── generateMenuData() → MenuDataItem[] │ → permissionStore.menuData │ → ProLayout.menuDataRender 使用 │ → 渲染左侧导航栏 │ └── extractPermissions() → string[] → permissionStore.permissions → 按钮级权限控制用

六、meta 字段详解

字段作用影响
visible: false隐藏菜单项不在侧边栏显示，但路由可用（直接输 URL 可访问）
isLink: true外链点击打开新窗口/新标签到外部 URL
isFrame: true内嵌 iframe在页面上用 iframe 加载 path 指向的 URL
isCache: true缓存页面页面切换后组件不被销毁（keep-alive），状态保留
icon菜单图标支持 ant-design 图标（如 user）和 Iconify 格式（如 ant-design:user-outlined）
