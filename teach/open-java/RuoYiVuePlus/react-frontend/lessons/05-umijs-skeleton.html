05 · UmiJS 项目骨架：理解 ruoyi-react 的目录与约定

胜利条件：能在 ruoyi-react 项目中快速定位任意功能所在的目录；理解 UmiJS 的约定大于配置理念；能解释 app.tsx 和 config/config.ts 各自的作用。

一、UmiJS 是什么？为什么用它？

UmiJS 是蚂蚁集团出品的企业级 React 框架，类似前端的 Spring Boot——帮你搭好脚手架、路由、构建、插件，你只需关注页面和业务。

ruoyi-react 用的是 @umijs/max（"max" = 满配版），内置了路由、状态管理、请求、国际化等能力。

核心哲学：约定大于配置。 文件放在特定目录就自动拥有特定能力，不需要手工注册。

二、目录树详解：每个目录干什么

三、两个核心配置文件

config/config.ts — 构建+路由配置（静态的）

config/config.ts 是 UmiJS 的配置中心：

export default defineConfig({
  // 1. 插件（UmiJS 的能力都是插件提供的）
  plugins: ['@umijs/max-plugin-openapi'],

  // 2. 路由定义
  routes: [
    { path: '/login', component: './login' },   // 登录页，无布局
    {
      path: '/',
      component: '@/layouts/BasicLayout',       // 嵌套路由：所有 /xxx 页面都套这个布局
      routes: [
        { path: '/', redirect: '/index' },
        { path: '/index', component: './index' },
        { path: '/user/profile', component: './user/profile' },
        { path: '*', component: './dynamicPage' },  // 动态路由
      ],
    },
  ],

  // 3. 路径别名
  alias: { '@': './src' },

  // 4. 代理（开发环境）
  proxy: {
    '/dev-api': {
      target: 'http://localhost:8080',
      changeOrigin: true,
    },
  },
});

src/app.tsx 是 React 应用的"入口逻辑"——它在应用启动时执行：

四、约定大于配置：哪些"约定"你必须知道

约定规则例子
文件即路由src/pages/ 下每个文件自动成为路由pages/login.tsx → 可路由 /login
布局嵌套routes 嵌套定义 = 布局嵌套子路由自动套 BasicLayout
导出名约定app.tsx 导出的固定函数名会被调用getInitialState、rootContainer
路径别名@/ = src/import { useDict } from '@/hooks/useDict'
TypeScript开箱即用，.tsx 文件直接支持所有组件都是 .tsx

五、src/.umi：自动生成的"黑箱"

UmiJS 会在 src/.umi/ 目录下自动生成文件，包括：路由、类型、入口文件等。绝不要手动修改这个目录。

你只需知道：这个目录的存在使 import { history, useModel } from 'umi' 等写法成为可能——它们都是 UmiJS 自动生成的导出。

六、一个请求的完整路径

浏览器访问 http://localhost:8000/system/user ↓ UmiJS 路由匹配（config/config.ts 中的 routes 配置） ↓ 匹配到 /system/user → 不是静态路由 → 走 catch-all '*' ↓ 渲染 BasicLayout 布局（嵌套路由的父组件） ↓ 在 BasicLayout 内部渲染 DynamicPage 组件 ↓ DynamicPage 从 permissionStore 的后端路由树中查找匹配的路由 ↓ 找到 component: '@/views/system/user/index' → 通过 migratedPages 映射到 React 组件 ↓ 渲染 SystemUserPage 组件

七、实战检验

  1. 打开 config/config.ts，找到 routes 数组，数出有多少条静态路由
  2. 打开 src/app.tsx，找到 getInitialState，看它返回了什么（这是全局初始状态的类型）
  3. 打开 package.json，找到 scripts 部分，看 dev 命令是什么
  4. 在终端运行 cat src/.umi/umi.ts（如果有的话），看一眼自动生成的入口文件长什么样
