02 · 组件与 Props：React 的积木系统

胜利条件：能区分函数组件和普通函数，理解 props 是"父传子的只读参数"，能写出接收 props 并有 TypeScript 类型标注的组件。

一、组件就是返回 JSX 的函数

React 组件 = 一个函数，接收 props（属性），返回 JSX。首字母必须大写。

最简单的例子——先看 ruoyi-react 中一个真实组件：

src/components/layout/Screenfull.tsx（简化）

// 函数组件：首字母大写，接收 props，返回 JSX
const Screenfull: React.FC = () => {
  const [isFullscreen, setIsFullscreen] = useState(false);

  const toggle = () => {
    if (!screenfull.isEnabled) {
      message.warning('您的浏览器不支持全屏');
      return;
    }
    screenfull.toggle();
    setIsFullscreen(!isFullscreen);
  };

  return (
    <Tooltip title="全屏">
      <FullscreenOutlined onClick={toggle} />
    </Tooltip>
  );
};

这就是一个完整的 React 组件：以函数形式存在，内部有自己的状态和逻辑，最终返回一段 JSX。

二、Props：父组件到子组件的数据流

React 的数据流是单向的——只能从父传子，通过 props：

// ===== 子组件：声明需要什么 props =====
interface UserCardProps {
  name: string;        // 必传
  age?: number;        // 可选（有 ?）
  onDelete: () => void; // 回调函数也是 props！
}

const UserCard: React.FC<UserCardProps> = ({ name, age, onDelete }) => {
  return (
    <div className="user-card">
      <span>{name}</span>
      {age && <span>（{age}岁）</span>}
      <Button onClick={onDelete}>删除</Button>
    </div>
  );
};

// ===== 父组件：传入 props =====
const UserList = () => {
  return (
    <div>
      <UserCard
        name="张三"
        age={28}
        onDelete={() => console.log('删除张三')}
      />
      <UserCard
        name="李四"           {/* age 可选，不传 */}
        onDelete={() => console.log('删除李四')}
      />
    </div>
  );
};

Props 核心规则

规则说明错误示例
只读！子组件绝不能修改 propsprops.name = '新名字'
任意 JS 类型字符串、数字、布尔、对象、数组、函数都可以—
解构接收用 { name, age } 代替 props.name—
children 特殊 prop写在标签之间的内容见下方

children：标签之间的内容

// 定义
const Card: React.FC<{ title: string; children: React.ReactNode }> = ({ title, children }) => {
  return (
    <div className="card">
      <h2>{title}</h2>
      <div>{children}</div>   {/* 这里渲染"标签之间的内容" */}
    </div>
  );
};

// 使用
<Card title="用户信息">
  <p>这里是卡片内容</p>     {/* 自动传给 children */}
  <Button>操作</Button>
</Card>

三、ruoyi-react 中的真实 Props 模式

模式 1：组件 + 样式 props

src/pages/index.tsx — ProCard 组件接收各种配置 props：

<ProCard
  title="用户信息"
  extra={<Button>更多</Button>}
  colSpan={12}        // 占 12/24 栅格宽度
  loading={loading}   // 加载状态
>
  {content}
</ProCard>

模式 2：Hook 返回值作为 props 传入

src/pages/system/user/index.tsx — 这是最核心的模式，贯穿整个项目：

// Hook 返回状态 + 操作方法
const { loading, data, pagination, search, reset } = useUserList();

// 每个值作为 props 传给子组件
<ProTable
  loading={loading}           // 布尔
  dataSource={data}           // 数组
  pagination={pagination}     // 对象
  onSubmit={search}           // 函数
  onReset={reset}             // 函数
/>

模式 3：回调函数 props（子→父通信的唯一方式）

// 父：传入回调
<EditModal
  visible={visible}
  userId={currentUser?.id}
  onClose={() => setVisible(false)}     // 回调1
  onSuccess={() => {                     // 回调2
    setVisible(false);
    refresh();                           // 成功后刷新列表
  }}
/>

// 子：在适当时机调用回调
const EditModal: React.FC<Props> = ({ visible, userId, onClose, onSuccess }) => {
  const handleSubmit = async (values) => {
    await updateUser(userId, values);
    message.success('更新成功');
    onSuccess();  // 通知父组件：我成功了
  };
  return <Modal open={visible} onCancel={onClose}>...</Modal>;
};

四、组件的"身份证"：TypeScript 类型标注

ruoyi-react 使用 TypeScript，每个组件的 props 都有接口定义。阅读代码时，先找 interface 就能知道这个组件需要什么数据：

// 标准模式
interface ComponentNameProps {
  // 数据 props
  userId?: number;
  record?: UserRecord;

  // 控制 props
  visible: boolean;
  loading?: boolean;

  // 回调 props（命名约定：on + 事件名）
  onClose: () => void;
  onSubmit?: (values: UserFormData) => Promise<void>;
  onSuccess?: () => void;
}

const ComponentName: React.FC<ComponentNameProps> = (props) => { ... };
