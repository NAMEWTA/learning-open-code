07 · Zustand 状态管理（上）：核心概念与模式

胜利条件：能解释"为什么需要全局状态管理"；掌握 Zustand create() 的三个要素（state、getter、action）；能读懂任意一个 ruoyi-react store 文件的结构。

一、为什么 useState 不够？

useState 的状态属于单个组件。当多个组件需要共享同一份数据（用户信息、权限路由、布局设置），就需要全局状态管理——把数据从组件树中"提"出来。
场景：用户信息在 10 个地方用到（顶栏头像、个人中心、登录日志...） ❌ 用 props 逐层传递： LoginPage → App → BasicLayout → Header → Avatar （中间每层都要接收并转发，即使自己不用） ✅ 用 Zustand Store： LoginPage → useUserStore.setState({ userInfo }) Avatar → useUserStore(state => state.userInfo) （任意组件直接读写，不需要中间层转发）

二、Zustand 是什么？

Zustand 是"状态"的德语。它是一个极简的 React 全局状态管理库：

对比维度ReduxZustand
核心概念数store、reducer、action、dispatch、selector、middleware、Provider...create + set + get，三个
是否需要 Provider是（<Provider store={store}>）不需要（直接 import 使用）
代码量多（action types + reducers + mapStateToProps...）少（一个 create() 搞定）
TypeScript 支持需要额外类型定义原生支持

三、create() 三步法

import { create } from 'zustand';

// 步骤 1：定义类型（store 长什么样）
interface UserStore {
  // state——数据
  userInfo: UserInfo | null;
  token: string;

  // actions——修改数据的方法
  setUserInfo: (info: UserInfo) => void;
  logout: () => void;
}

// 步骤 2：create() 创建 store
export const useUserStore = create<UserStore>((set, get) => ({
  // 初始 state
  userInfo: null,
  token: '',

  // action：修改 state
  setUserInfo: (info) => set({ userInfo: info }),

  // action：一次性改多个字段
  logout: () => set({ userInfo: null, token: '' }),
}));

// 步骤 3：在组件中使用（就像用 useState 一样！）
function Avatar() {
  const userInfo = useUserStore((state) => state.userInfo);

  return <span>{userInfo?.nickName}</span>;
}

set 的两种写法

// 方式 1：直接传对象（合并到 state，浅合并）
set({ userInfo: newInfo });

// 方式 2：传函数（当前 state → 新 state）
set((state) => ({ count: state.count + 1 }));

get：在 action 里读当前 state

const useStore = create((set, get) => ({
  count: 0,
  increment: () => set({ count: get().count + 1 }),  // get() 拿到最新值
}));

四、三个核心设计模式

模式 1：selector 精准订阅（性能关键！）

// 组件只订阅 userInfo，token 变了不会导致它重渲染
const userInfo = useUserStore((state) => state.userInfo);

// ❌ 不要拿整个 store（任何字段变了都会重渲染）
const store = useUserStore();

模式 2：persist 中间件 → 自动同步到 localStorage

import { persist } from 'zustand/middleware';

export const useAppStore = create(
  persist(
    (set) => ({
      theme: 'light',
      setTheme: (t) => set({ theme: t }),
    }),
    { name: 'app-settings' }  // localStorage 的 key
  )
);
// 刷新页面后 theme 自动从 localStorage 恢复

模式 3：在组件外读写 store

// Zustand store 是普通 JS 对象，可以在任何地方读写
import { useUserStore } from '@/stores/userStore';

// ✅ 在 API 拦截器中读
const token = useUserStore.getState().token;

// ✅ 在工具函数中写
useUserStore.getState().logout();

// ✅ 在 axios 响应拦截器中
if (response.status === 401) {
  useUserStore.getState().logout();
}

五、一个完整的数据流追踪

用户登录成功 ↓ login.tsx: setToken(res.data.access_token) ← localStorage login.tsx: useUserStore.getState().setUserInfo({...}) ← Zustand ↓ BasicLayout.tsx: const userInfo = useUserStore(s => s.userInfo); ← 订阅 avatarProps.src = userInfo.avatar ← 渲染头像 ↓ 用户点击右上角头像 → 退出登录 ↓ Dropdown.Item onClick: useUserStore.getState().logout(); ← Zustand action removeToken(); ← localStorage 清除 history.push('/login'); ← 跳转 ↓ BasicLayout 检测到 token 为空 → return <Redirect to="/login" />
