注解 × 消费者映射

注解消费者方向关键属性
@CellMerge CellMergeHandler → CellMergeStrategy 仅导出 index(-1自动), mergeBy(依赖字段)
@ExcelDictFormat ExcelDictConvert + ExcelDownHandler 导入+导出 dictType(查库), readConverterExp(内联表达式), separator
@ExcelEnumFormat ExcelEnumConvert + ExcelDownHandler 导入+导出 enumClass, codeField("code"), textField("text")
@ExcelDynamicOptions ExcelDownHandler 仅导出 providerClass(ExcelOptionsProvider实现)
@ExcelRequired DataWriteHandler 仅导出 fontColor(RED)
@ExcelNotation DataWriteHandler 仅导出 value(批注内容)

@ExcelDictFormat 两条路径

dictType 有值 → DictService.getDictLabel/Value() → 查数据库字典表 readConverterExp 有值 → parseConverterExp("0=男,1=女") → 解析内联Map 两者都为空 → 返回空字符串 导入时反向查：label → code（getDictValue / reverseByExp）

@ExcelEnumFormat 双缓存策略

ENUM_MAP_CACHE: Field → Map<code, text> (导出用) ENUM_REVERSE_MAP_CACHE: Field → Map<text, code> (导入用) 首次访问 → ReflectUtils.invokeGetter 反射读取枚举属性 → 缓存 导入校验：用户填的文本必须在反向Map中，否则抛异常并列出所有允许值

@CellMerge 合并条件

相邻两行 当前字段值相同 AND mergeBy中指定的所有依赖字段值也相同 → 合并 空值(null/"") → 中断合并区间 mergeBy示例：合并"部门名称"时依赖"部门编号"，防止同名不同部门被错误合并

记忆锚点

  • 6 个注解中，只有 @ExcelDictFormat 和 @ExcelEnumFormat 同时被 Converter 和 WriteHandler 消费。
  • Converter 管「值怎么转」，WriteHandler 管「下拉选项怎么生成」——同一注解服务两个目的。
  • @ExcelRequired + @ExcelNotation 仅影响导出模板的视觉效果，不参与数据转换。
  • @ExcelDynamicOptions 是策略模式扩展点，适合下拉数据来自动态查询的场景。

→ 相关术语见 GLOSSARY.md：@CellMerge, @ExcelDictFormat, @ExcelEnumFormat, @ExcelDynamicOptions
