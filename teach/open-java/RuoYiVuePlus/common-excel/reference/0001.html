速查 01 · ruoyi-common-excel 模块全景

配套课程：lessons/0001.html

一句话定位

ruoyi-common-excel 是基于 Fesod Sheet 的项目化适配层——通过实现其 Converter / WriteHandler / ReadListener 扩展点，注入 RuoYi 的字典服务、枚举体系、校验框架。

四层架构

层包文件数
注解层annotation/6：CellMerge, ExcelDictFormat, ExcelEnumFormat, ExcelDynamicOptions, ExcelRequired, ExcelNotation
转换器层convert/3：ExcelBigNumberConvert, ExcelDictConvert, ExcelEnumConvert
处理器层core/ + handler/10：DefaultExcelListener, ExcelListener, ExcelResult, DefaultExcelResult, CellMergeStrategy, CellMergeHandler, ExcelDownHandler, DropDownOptions, ExcelOptionsProvider, DataWriteHandler
工具层utils/2：ExcelBuilder, ExcelWriterWrapper

导出链路（自动注册的处理器）

ExcelBuilder.of → createWriterBuilder(): 始终注册: ExcelBigNumberConvert + DataWriteHandler + ExcelDownHandler 可选注册: CellMergeStrategy (仅 merge=true 时) → builder.doWrite(data)

导入链路

ExcelBuilder.read → ReadBuilder.doRead(): → new DefaultExcelListener(validate, failFast) → FesodSheet.read → builder.sheet().doRead() → 逐行 invoke → onException分类处理 → getExcelResult()

记忆锚点

  • 21 个文件 = 6 注解 + 3 转换器 + 10 处理器/接口 + 2 工具类。
  • 模块不直接操作 POI API，全部通过 Fesod Sheet 的扩展点（Converter / WriteHandler / ReadListener）。
  • 导出是「注解声明 → 转换器值转换 + 处理器结构处理」，导入是「监听器逐行解析 + JSR303 校验 + 结果收集」。
  • ExcelBuilder 是唯一对外入口，提供 of() / writer() / template() / read() 四个静态方法。

→ 相关术语见 GLOSSARY.md：Fesod Sheet, Converter, WriteHandler, AnalysisEventListener
