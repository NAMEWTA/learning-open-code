DefaultExcelListener 核心行为

方法做什么
invoke(data, ctx)若 validate → ValidatorUtils.validate(data)；通过后 add 到 list
onException(ex, ctx)区分 ExcelDataConvertException(行+列+表头) / ConstraintViolationException(仅行号) / 兜底；收集错误；failFast=true 时抛异常
invokeHeadMap(map, ctx)保存列索引→表头名映射，用于错误信息定位
doAfterAllAnalysed(ctx)打日志

两个开关

isValidate (默认true): 控制是否执行 JSR303 校验，可关闭以提升性能 failFast (默认true): 控制首次错误时是否立即终止 true → onException 中 throw ExcelAnalysisException false → 收集全部错误后返回汇总 ExcelResult

DefaultExcelResult.getAnalysis() 三种回执

successCount==0 → "读取失败，未解析到数据" errorCount==0 → "恭喜您，全部读取成功！共X条" errorCount>0 → "共X条，成功导入Y条，错误Z条"

完整调用

ExcelBuilder.read(is, clazz) .validate(bool).failFast(bool).sheetNo(n) .doRead() // 返回 ExcelResult // 内部：new DefaultExcelListener(validate, failFast) // → FesodSheet.read → registerReadListener // → sheet().doRead() → 逐行回调 → getExcelResult() .doReadSync() // 返回 List<T>（不含错误信息） .doReadAll() / doReadAllSync() // 读取全部 Sheet

记忆锚点

  • 两种异常处理：ExcelDataConvertException（精确到单元格）vs ConstraintViolationException（精确到行）。
  • failFast 开关：true=快速失败，false=柔性收集——业务场景决定。
  • headMap 不是数据的一部分，而是用于错误提示——「第3行-第2列-表头性别: 解析异常」。
  • 自定义监听器：继承 DefaultExcelListener 或实现 ExcelListener，通过 ReadBuilder.listener() 注入。
