ExcelDownHandler 四种下拉数据源

① @ExcelDictFormat.dictType → DictService.getAllDictByDictType() ② @ExcelDictFormat.readConverterExp → 解析 "0=男,1=女" 取标签 ③ @ExcelEnumFormat.enumClass → EnumUtil.getFieldValues() ④ @ExcelDynamicOptions.providerClass → SpringUtils.getBean().getOptions() ⑤ 外部 DropDownOptions 参数 选项 > 20 或总字符 > 255 → dropDownWithSheet (隐藏Sheet+名称管理器) 否则 → dropDownWithSimple (内联显式列表约束)

级联下拉原理（INDIRECT）

1. 隐藏Sheet首行横向写一级选项 → 名称管理器指向该行 2. 每个一级选项创建一个子名称管理器 → 指向对应列的二级数据 3. 一级列校验 = 公式约束引用名称管理器 4. 二级列校验 = INDIRECT(一级列单元格) → 用户选"北京" → INDIRECT("北京") → 找名称管理器"北京" → 展示二级选项

CellMergeHandler 合并算法

逐字段、逐行扫描： - 空值 → 收口上一段合并区间，当前行不参与 - 首次遇到 → 缓存 (值, 起始行) - 值相同 AND mergeBy依赖字段全相同 → 扩展区间 - 值不同 OR 依赖不同 → 收口旧区间 + 开始新区间 - 最后一行 → 收口最后一段 产出: List<CellRangeAddress> → 交给 CellMergeStrategy 执行

CellMergeStrategy 两步执行

afterSheetCreate: 提前批量写入所有合并区域 (sheet.addMergedRegion) → 避免 "row already written to disk" 错误 merge (每格写入后): 若单元格在合并区域内且非首行 → cell.setBlank() → 清空非首行的重复值，让合并区域视觉上干净

DataWriteHandler 表头处理

仅当 context.getHead()==true 时生效: - 单元格设为文本格式 (format index=49) - 表头字体加粗 - @ExcelRequired → 表头字体颜色 = fontColor (默认 RED) - @ExcelNotation → 创建 POI Comment 批注 (XSSFClientAnchor)

记忆锚点

  • 下拉只在前 1000 行生效（FIRST_DATA_ROW_INDEX=1, LAST_DATA_ROW_INDEX=1000）。
  • 简单下拉（内联）有 255 字符限制，超限自动切换隐藏 Sheet 方案。
  • 级联下拉依赖 Excel 的 INDIRECT 函数 + 名称管理器，这是 Excel 原生特性，不是 Java 逻辑。
  • 合并策略的「预占坑」设计是为了兼容 POI 的写入时序限制。
