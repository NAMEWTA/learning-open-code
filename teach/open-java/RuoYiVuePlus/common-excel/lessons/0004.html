第 4 课 · 导出处理器链：下载一个 Excel 经过了哪些 WriteHandler 处理器层 ②

🎯 本节目标（对应使命 Success #5 + #7）：学完你能完整讲清导出链路中三条 WriteHandler 链各自的职责——ExcelDownHandler 的四种下拉数据源与「简单列表 vs 隐藏 Sheet」两种实现策略、CellMergeHandler + CellMergeStrategy 的合并算法与执行时序、DataWriteHandler 的表头样式标注。

导出链路有 3 条 WriteHandler 线，全部由 ExcelBuilder.createWriterBuilder() 自动注册。本课逐一拆解。

一、三条 WriteHandler 线总览

处理器实现的接口回调时机做什么
ExcelDownHandler SheetWriteHandler Sheet 创建后 为每列生成数据校验下拉（4种数据源）
CellMergeStrategy AbstractMergeStrategy + SheetWriteHandler Sheet 创建后 + 每格写入后 先批量写入合并区域，再逐格清空非首行内容
DataWriteHandler SheetWriteHandler + CellWriteHandler 每格写入后（仅表头行） 表头字体标红（必填）+ 添加批注

二、ExcelDownHandler —— 下拉选项四种数据源

位置：core/ExcelDownHandler.java（445 行，模块最长的文件）

它的 afterSheetCreate() 遍历实体类的每个字段，根据注解类型从四种数据源收集下拉选项：

// afterSheetCreate() 中的遍历逻辑 — ExcelDownHandler.java:101-155 for (字段 : 实体类的全部字段) { ① @ExcelDictFormat: if (dictType 有值) → getDictService().getAllDictByDictType(dictType) → 取全部标签 else if (readConverterExp 有值) → 解析 "0=男,1=女" → 取等号右边的值列表 ② @ExcelEnumFormat: EnumUtil.getFieldValues(enumClass, textField) → 取所有枚举的 text 值 ③ @ExcelDynamicOptions: SpringUtils.getBean(providerClass).getOptions() → 取运行时提供的数据 ④ DropDownOptions（外部传入）: 遍历 dropDownOptions 列表 → 内部调用上面的 dropDownLinkedOptions 等方法 if (options 非空): if (选项数 > 20 或 总字符数 > 255): → dropDownWithSheet() — 隐藏 Sheet 方案 else: → dropDownWithSimple() — 内联列表方案 }

2.1 简单下拉：dropDownWithSimple()

// ExcelDownHandler.java:182-187
private void dropDownWithSimple(DataValidationHelper helper, Sheet sheet,
                                 Integer celIndex, List<String> value) {
    this.markOptionsToSheet(helper, sheet, celIndex,
        helper.createExplicitListConstraint(ArrayUtil.toArray(value, String.class)));
}

使用 POI 的 createExplicitListConstraint，直接把下拉选项写成逗号分隔的字符串。Excel 限制：总字符数不能超过 255。适合选项少且短的场景。

2.2 隐藏 Sheet 下拉：dropDownWithSheet()

当选项太多或太长时，切换为更高级的方案：

// ExcelDownHandler.java:337-374 ① 创建一个隐藏的 Sheet（名称 options_0, options_1, ...） ② 把选项值逐行写入这个隐藏 Sheet 的 A 列 ③ 用 workbook.createName() 创建名称管理器，指向隐藏 Sheet 的 A1:AN ④ 用 createFormulaListConstraint(nameName) 在主表列上创建数据校验 ⑤ workbook.setSheetHidden(...) 隐藏辅助 Sheet

这是 Excel 下拉选项的标准高级用法——通过名称管理器（Name Manager）引用另一个 Sheet 的单元格区域作为下拉数据源，完全避开 255 字符限制。

2.3 级联下拉：dropDownLinkedOptions()（最复杂的逻辑）

位置：ExcelDownHandler.java:194-315（120 行，约占整个文件的 1/4）

// 级联下拉的核心思路（以「省份 → 城市」为例）： ① 创建隐藏 Sheet "linkedOptions_0" ② 第一行横向写入所有一级选项（如 北京 | 上海 | 广东） ③ 创建名称管理器 "linkedOptions_0" → 引用 =linkedOptions_0!$A$1:$C$1 ④ 一级列的数据校验 = 公式列表约束 "linkedOptions_0" → 用户在一级列（省份）只能选 A1:C1 中的值 ⑤ 为每个一级选项创建子名称管理器： 名称管理器 "北京" → 引用 =linkedOptions_0!$A$2:$A$5 名称管理器 "上海" → 引用 =linkedOptions_0!$B$2:$B$4 ... ⑥ 二级列（城市）每一行的数据校验 = =INDIRECT(A2) → Excel 的 INDIRECT 函数：取 A2 格的值（如"北京"）， 找到名为"北京"的名称管理器，展示其引用的区域作为下拉选项 ⑦ 按列填充二级数据：把每个一级选项对应的二级数据按行排列
💡 INDIRECT 函数是级联下拉的关键。Excel 的 INDIRECT(text) 把文本当作单元格引用或名称管理器来解析。当用户在 A2 选了"北京"，B2 的校验公式 =INDIRECT(A2) 就解析为 =INDIRECT("北京") → 找到名称管理器"北京" → 展示"北京"下的二级选项。

2.4 markDataValidationToSheet() —— 统一的校验挂载

// ExcelDownHandler.java:399-418
private void markDataValidationToSheet(DataValidationHelper helper, Sheet sheet,
                                        DataValidationConstraint constraint,
                                        CellRangeAddressList addressList) {
    DataValidation dataValidation = helper.createValidation(constraint, addressList);
    if (dataValidation instanceof XSSFDataValidation) {
        dataValidation.setSuppressDropDownArrow(true);
        dataValidation.setErrorStyle(DataValidation.ErrorStyle.STOP);
        dataValidation.createErrorBox("提示", "此值与单元格定义数据不一致");
        dataValidation.createPromptBox("填写说明：",
            "填写内容只能为下拉中数据，其他数据将导致导入失败");
    }
    sheet.addValidationData(dataValidation);
}

注意：下拉校验挂载到前 1000 行（FIRST_DATA_ROW_INDEX=1 到 LAST_DATA_ROW_INDEX=1000）。注释明确说明：超过 1000 行需要分次导出，否则 Excel 打开会很慢。

三、CellMergeHandler + CellMergeStrategy —— 合并的「分析-执行」分离

这两个类是模块中「分析」与「执行」分离的典范：

CellMergeHandlerCellMergeStrategy
角色分析器（静态工具类）执行器（Fesod Sheet WriteHandler）
核心方法handle(List<?> rows) → List<CellRangeAddress>afterSheetCreate() + merge()
做的事扫描数据行，计算哪些区域需要合并将合并区域写入 Sheet，清理非首行重复值

3.1 CellMergeHandler.handle() —— 合并算法

// CellMergeHandler.java:62-125
public List<CellRangeAddress> handle(List<?> rows) {
    // ① 获取所有带 @CellMerge 的字段及其列索引
    Map<Field, FieldColumnIndex> mergeFields = getFieldColumnIndexMap(rows.get(0).getClass());

    // ② 逐字段、逐行扫描
    for (字段 : mergeFields) {
        for (i = 0; i < rows.size(); i++) {
            当前行字段值 = ReflectUtils.invokeGetter(rows.get(i), 字段名);

            if (值为空) {
                // 空值 = 合并中断 → 收口上一段合并区间
                appendMergeResult(result, 上次缓存的RepeatCell, i-1, 列号);
                continue;
            }
            if (缓存中无此字段) {
                // 首次遇到 → 开始一个新的合并区间
                缓存.put(字段, RepeatCell.of(当前值, i));
                continue;
            }

            缓存值 = 缓存.get(字段).value();
            // ③ 检查合并条件：值相同 + 依赖字段也相同
            if (当前值.equals(缓存值) && isMerge(当前行, 上一行, cellMerge)) {
                // 继续扩展当前合并区间
            } else {
                // ④ 中断当前区间，开始新区间
                appendMergeResult(result, 缓存RepeatCell, i-1, 列号);
                缓存.put(字段, RepeatCell.of(当前值, i));
            }
        }
        // ⑤ 处理最后一组
        appendMergeResult(result, 缓存RepeatCell, rows.size()-1, 列号);
    }
    return result;
}
核心空值中断：如果某行的合并字段值为 null 或空字符串，handle() 会立即收口上一段合并区间，并且该行不参与合并。这意味着导出时如果一个分组内有空值行，合并会被打断。
核心getFieldColumnIndexMap() 的列索引推断优先级：
@CellMerge.index() → @ExcelProperty.index() → 反射遍历顺序。同时会跳过 serialVersionUID、@ExcelIgnore 标注的字段、以及 @ExcelIgnoreUnannotated 下未标注 @ExcelProperty 的字段。

3.2 CellMergeStrategy —— 为什么要分两步执行？

// CellMergeStrategy.java:89-98 — afterSheetCreate()
public void afterSheetCreate(WriteWorkbookHolder wwb, WriteSheetHolder wsh) {
    // 提前写入所有合并区域
    for (CellRangeAddress item : cellList) {
        sheet.addMergedRegion(item);
    }
}

// CellMergeStrategy.java:68-79 — merge()
protected void merge(Sheet sheet, Cell cell, Head head, Integer relativeRowIndex) {
    // 单元格写入后，如果该格在合并区域内且不是首行 → 清空内容
    for (CellRangeAddress cellAddresses : cellList) {
        if (cellAddresses.isInRange(cell) && rowIndex != cellAddresses.getFirstRow()) {
            cell.setBlank();
        }
    }
}
⚠️ 为什么合并区域要在 afterSheetCreate 提前写入？
POI 的限制：如果在数据写入过程中动态添加合并区域，可能导致 "Attempting to write a row in the range that is already written to disk" 错误。所以 CellMergeStrategy 的策略是：先在 Sheet 创建时一次性写入所有合并区域（只占位置，此时格子里还没数据），后续 Fesod Sheet 正常逐格填充数据（首行格子正常写入，非首行格子被 merge() 清空）。 这是一种「预占坑」策略。

Q1. ExcelDownHandler 在什么条件下从「简单下拉」切换为「隐藏 Sheet」方案？

A. 选项数量 > 10 B. 选项数量 > 20 或 所有选项总字符数 > 255 C. 总是使用隐藏 Sheet 方案 D. 选项中有中文时
看答案B。代码 ExcelDownHandler.java:148：if (options.size() > 20 || totalCharacter > 255)。简单下拉用 Excel 的显式列表约束，有 255 字符的硬限制；隐藏 Sheet 方案没有这个限制。

Q2. 级联下拉中，二级选项的 Excel 校验公式是什么？

A. =VLOOKUP(A2, options, 2, FALSE) B. =INDIRECT(A2) C. =IF(A2="", "", A2) D. 直接列出所有二级选项
看答案B。=INDIRECT(A2)。INDIRECT 是 Excel 内置函数，把文本参数解析为单元格引用或名称管理器。当用户在 A2 选了一级选项（如"北京"），INDIRECT 就去找名为"北京"的名称管理器，返回其关联的二级选项列表。

Q3. CellMergeStrategy 为什么要把合并区域的写入拆成 afterSheetCreate + merge 两步？

A. 为了代码更清晰 B. 先在 afterSheetCreate 预占坑防止 POI 写入冲突，再在 merge 中清空非首行重复值 C. 没有特殊原因 D. 因为 Fesod Sheet 只能分两步注册 WriteHandler
看答案B。POI 的 addMergedRegion() 需要在数据写入前完成，否则会报 "row already written to disk" 错误。而清空非首行重复值的逻辑需要在单元格写入后才能执行（才能知道哪些单元格需要清空）。
