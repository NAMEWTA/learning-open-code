一、ExcelBuilder —— 模块的唯一对外 API

位置：utils/ExcelBuilder.java（1110 行，模块最长的文件）

ExcelBuilder 是一个 final class，包含三个内部类（ReadBuilder、TemplateBuilder）和一个私有枚举：

ExcelBuilder<T> ├── 导出 API │ ├── static of(data, headType) → ExcelBuilder // 列表导出 │ ├── static writer(headType) → ExcelBuilder // 自定义写出 │ └── static template(templatePath) → TemplateBuilder // 模板填充 ├── 导入 API │ └── static read(is, clazz) → ReadBuilder // 导入读取 ├── 链式配置 (导出) │ ├── sheetName / sheetNo / merge / options / password │ ├── needHead / automaticMergeHead │ ├── includeFields / excludeFields / includeIndexes / excludeIndexes │ ├── orderByIncludeColumn / columnWidth / rowHeight │ ├── registerWriteHandler / registerConverter │ ├── zip / zip(pageSize) │ └── toResponse(response) / toStream(os) / toStream(os, consumer) └── 链式配置 (导入 — ReadBuilder) ├── validate / failFast / listener ├── sheetNo / sheetName / headRowNumber ├── ignoreEmptyRow / password / autoTrim / autoStrip / numRows ├── registerConverter └── doRead() / doReadSync() / doReadAll() / doReadAllSync()
💡 DSL 设计的亮点：ExcelBuilder 的方法全部返回 this，实现了流畅的链式调用。但注意它是 有状态的建造者——配置顺序（在 createWriterBuilder() 前还是后）无所谓，关键是最后调用 toResponse() / toStream() 时才真正执行。

二、createWriterBuilder() —— 所有默认处理器的注册点

这是整个模块最重要的一段初始化代码（ExcelBuilder.java:472-521）：

private ExcelWriterBuilder createWriterBuilder(OutputStream outputStream) {
    ExcelWriterBuilder builder = FesodSheet.write(outputStream, headType)
        .autoCloseStream(false)
        // ═══ 始终注册 ═══
        .registerConverter(new ExcelBigNumberConvert())  // ① 大数字精度保护
        .registerWriteHandler(new DataWriteHandler(headType))  // ② 批注+必填
        // 列宽策略
        if (columnWidth == null) {
            builder.registerWriteHandler(new LongestMatchColumnWidthStyleStrategy());
        } else {
            builder.registerWriteHandler(new SimpleColumnWidthStyleStrategy(columnWidth));
        }
        // 行高策略
        if (headRowHeight != null || contentRowHeight != null) {
            builder.registerWriteHandler(new SimpleRowHeightStyleStrategy(...));
        }
        // ═══ 始终注册 ═══
        .registerWriteHandler(new ExcelDownHandler(options));  // ③ 下拉
        // ④ CellMergeStrategy 在 writeSheet() 中仅在 merge=true 时注册
        // ... 应用其他配置项
        return builder;
}
关键注册顺序有意义吗？
Fesod Sheet 按注册顺序依次调用各 WriteHandler 的回调。但本模块的几个处理器互相独立——ExcelDownHandler 只管数据校验、CellMergeStrategy 只管合并区域、DataWriteHandler 只管表头样式——所以注册顺序不影响最终结果。

三、ExcelWriterWrapper —— 安全包装的价值

位置：utils/ExcelWriterWrapper.java（248 行）

public record ExcelWriterWrapper<T>(ExcelWriter excelWriter) {
    public void write(Collection<T> data, WriteSheet writeSheet) {
        excelWriter.write(data, writeSheet);
    }
    // ... 其他 write/fill 方法
    // 注意：没有暴露 excelWriter.finish() !
}

这是一个 Java Record，对 Fesod Sheet 的 ExcelWriter 做了安全包装。关键设计：

  • 不暴露 finish()：防止业务代码提前关闭输出流。
  • 不暴露 ExcelWriter 本身（Record 的字段是私有的，只有一个 getter 方法）。
  • 提供 writeContext()：允许业务代码获取写出上下文做一些高级操作。
  • 内置 sheetBuilder() / tableBuilder() 静态工厂：方便创建 WriteSheet 和 WriteTable。
💡 为什么不用 Lombok @Delegate？Record 的开销极低，且编译器会保证所有字段 final、自带 equals/hashCode/toString。这里的包装层只有薄薄 248 行，但对于防止「调用方不小心关了流」这种事，它提供了编译期安全保证。

四、ExcelBigNumberConvert —— 15 位精度的分水岭

位置：convert/ExcelBigNumberConvert.java（81 行）

// convertToExcelData() — 关键的判断
public WriteCellData<Object> convertToExcelData(Long object, ...) {
    if (ObjectUtil.isNull(object)) {
        return new WriteCellData<>("");
    }
    String str = Convert.toStr(object);
    if (str.length() > 15) {
        // 超过15位 → 按字符串写出，前面加制表符防止科学计数法
        return new WriteCellData<>(str);
    }
    // 15位以内 → 按数值类型写出
    WriteCellData<Object> cellData = new WriteCellData<>(new BigDecimal(object));
    cellData.setType(CellDataTypeEnum.NUMBER);
    return cellData;
}
1为什么是 15 位？
Excel 的数值精度是 15 位有效数字。超过 15 位的数字（如 18 位身份证号、雪花算法生成的 Long 型 ID）如果按数值写入，会被 Excel 截断并显示为科学计数法（如 1.23457E+17），导致数据失真。
2解决方案：文本单元格
长度 > 15 时，以字符串形式写入单元格。Excel 打开后会将其视为文本，完整保留所有位数。这也是 ExcelBuilder 在 createWriterBuilder() 中始终注册这个 Converter 的原因——它是基础数据保护，不需要用户选择。

五、完整实战：导出代码

// ====== 场景一：标准列表导出（最常用） ======
@PostMapping("/export")
public void export(HttpServletResponse response, SysUser user) {
    // 1. 查数据
    List<SysUser> list = userService.selectUserList(user);

    // 2. 导出
    ExcelBuilder.of(list, SysUser.class)
        .sheetName("用户数据")
        .merge()                         // 合并相同值的列
        .excludeFields(List.of("password", "salt"))  // 排除敏感字段
        .toResponse(response);
}

// ====== 场景二：带下拉选项的模板导出 ======
@PostMapping("/exportTemplate")
public void exportTemplate(HttpServletResponse response) {
    List<DropDownOptions> options = List.of(
        // 简单下拉
        new DropDownOptions(2, List.of("男", "女")),
        // 级联下拉：省份 → 城市
        DropDownOptions.buildLinkedOptions(
            provinceList, 3, cityList, 4,
            Province::getId, City::getProvinceId,
            Province::getName
        )
    );

    ExcelBuilder.of(Collections.emptyList(), SysUser.class)
        .sheetName("用户导入模板")
        .options(options)
        .needHead(true)
        .toResponse(response);
}

// ====== 场景三：大数据量 ZIP 分页导出 ======
@PostMapping("/exportLarge")
public void exportLarge(HttpServletResponse response) {
    List<Order> list = orderService.selectAll(); // 假设有 5000 条

    ExcelBuilder.of(list, Order.class)
        .sheetName("订单数据")
        .zip(1000)  // 每 1000 条一个 Excel 文件，打包成 ZIP
        .toResponse(response);
    // → 浏览器下载 订单数据.zip，内含 5 个 xlsx 文件
}

// ====== 场景四：自定义写出（多 Sheet） ======
@PostMapping("/exportMultiSheet")
public void exportMultiSheet(HttpServletResponse response) {
    List<User> users = userService.list();
    List<Order> orders = orderService.list();

    ExcelBuilder.writer(null)
        .toResponse(response, wrapper -> {
            // Sheet 1: 用户数据
            wrapper.write(users, ExcelWriterWrapper.buildSheet(0, "用户"));
            // Sheet 2: 订单数据
            wrapper.write(orders, ExcelWriterWrapper.buildSheet(1, "订单"));
        });
}

六、完整实战：导入代码

// ====== 场景一：标准导入（异步校验） ======
@PostMapping("/importData")
public R<Void> importData(@RequestPart("file") MultipartFile file) throws Exception {
    ExcelResult<SysUser> result = ExcelBuilder
        .read(file.getInputStream(), SysUser.class)
        .failFast(false)    // 全量收集错误，不因一个错就终止
        .doRead();

    if (CollUtil.isEmpty(result.getErrorList())) {
        userService.saveBatch(result.getList());
        return R.ok("全部导入成功，共 " + result.getList().size() + " 条");
    }
    // 部分成功：记录错误日志
    log.warn("导入部分失败: {}", result.getAnalysis());
    if (CollUtil.isNotEmpty(result.getList())) {
        userService.saveBatch(result.getList());
    }
    return R.fail(result.getAnalysis());
}

// ====== 场景二：严格校验导入 ======
@PostMapping("/importStrict")
public R<Void> importStrict(@RequestPart("file") MultipartFile file) throws Exception {
    try {
        ExcelResult<SysUser> result = ExcelBuilder
            .read(file.getInputStream(), SysUser.class)
            .failFast(true)   // 一个错就终止
            .headRowNumber(2) // 表头占 2 行
            .ignoreEmptyRow(true)
            .doRead();
        userService.saveBatch(result.getList());
        return R.ok();
    } catch (ExcelAnalysisException e) {
        return R.fail("导入失败：" + e.getMessage());
    }
}

// ====== 场景三：自定义监听器（导入后做额外业务校验） ======
@PostMapping("/importWithCheck")
public R<Void> importWithCheck(@RequestPart("file") MultipartFile file) throws Exception {
    ExcelListener<SysUser> customListener = new DefaultExcelListener<>(true, false) {
        @Override
        public void invoke(SysUser data, AnalysisContext context) {
            // 先做标准校验
            super.invoke(data, context);
            // 再做业务校验
            if (userService.existsByPhone(data.getPhonenumber())) {
                throw new ServiceException("手机号已存在：" + data.getPhonenumber());
            }
        }
    };

    ExcelResult<SysUser> result = ExcelBuilder
        .read(file.getInputStream(), SysUser.class)
        .listener(customListener)
        .doRead();

    userService.saveBatch(result.getList());
    return R.ok(result.getAnalysis());
}

// ====== 场景：基于模板文件填充数据 ======
@PostMapping("/exportByTemplate")
public void exportByTemplate(HttpServletResponse response) {
    Map<String, Object> data = Map.of(
        "users", userService.list(),   // {users.字段} 在模板中引用
        "dept", deptService.list()
    );

    ExcelBuilder.template("templates/用户报表.xlsx")
        .filename("用户报表")
        .multiList(data)    // 多列表填充模式
        .toResponse(response);
}

// TemplateBuilder 支持三种模式：
// .data(list)       — LIST: 单表数据，模板中用 {.属性}
// .multiList(map)   — MULTI_LIST: 多表数据，模板中用 {key.属性}
// .multiSheet(list) — MULTI_SHEET: 多 Sheet 数据

Q2. ExcelBigNumberConvert 在什么条件下将 Long 值以字符串形式写入 Excel？

A. 值大于 999999 B. 值的字符串长度 > 15 C. 总是以字符串形式写入 D. 值以 0 开头时
看答案B。if (str.length() > 15)。Excel 数值精度为 15 位，超过则必须用文本格式避免精度丢失。这是处理雪花算法 ID（18-19 位）等长数字的标准做法。

Q3. ExcelWriterWrapper 没有暴露 ExcelWriter 的哪个方法？为什么？

A. write()——防止写入错误数据 B. finish()——防止业务代码提前关闭输出流 C. fill()——防止模板填充失败 D. 所有方法都暴露了
看答案B。ExcelWriterWrapper 是 Record，只通过包装方法暴露了 write() / fill() / writeContext()，特意没有暴露 finish()。finish() 由调用方（ExcelBuilder.toStream() 中的 try-with-resources）统一管理。

Q4. 填空题：导入用户数据，遇到错误不终止，读取第 2 个 Sheet。

A. read, sheetNo, failFast B. of, sheetNo, validate C. read, sheetName, failFast D. read, sheetNo, zip
