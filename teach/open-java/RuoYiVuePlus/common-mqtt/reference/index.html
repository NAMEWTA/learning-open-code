ruoyi-common-mqtt 速查表

Maven: org.dromara:ruoyi-common-mqtt | ruoyi-common/ruoyi-common-mqtt/src/main/java/org/dromara/common/mqtt/

定位

基于 Mica MQTT 的 MQTT 客户端适配模块。仅 3 个 Java 文件，通过 Spring Boot 自动配置实现零代码接入 MQTT Broker，用虚拟线程优化 IO 性能，统一管理连接生命周期与消息监听。

核心文件

类名路径职责
MqttAutoConfigurationconfig/MqttAutoConfiguration.java自动配置入口，注册两个监听器和 MqttClientCustomizer，由 mqtt.client.enabled 控制开关
MqttClientCustomizerconfig/MqttClientCustomizer.java定制 MqttClientCreator，注入虚拟线程执行器替换默认线程池
MqttClientConnectListenerconfig/MqttClientConnectListener.java连接生命周期监听（onConnected / onDisconnect），断连时支持凭证轮换
MqttClientGlobalMessageListenerconfig/MqttClientGlobalMessageListener.java全局消息监听，接收所有已订阅 Topic 的消息并记录日志

术语速查

术语解释
MQTT轻量级发布订阅消息协议（OASIS 标准），基于 TCP/IP，专为物联网低带宽场景设计
Mica MQTTDromara 出品的 MQTT 客户端/服务端框架，基于 t-io，本模块只做客户端
发布订阅模型Publisher 发消息到 Broker 的 Topic，Subscriber 订阅 Topic 后接收，双方空间解耦
BrokerMQTT 中央消息代理服务器（如 EMQX、Mosquitto），负责消息路由和订阅关系维护
Topic消息分类标签，用 / 分层，支持通配符 +（单层）和 #（多层），无需预先创建
QoS消息交付质量：QoS 0（至多一次）、QoS 1（至少一次）、QoS 2（恰好一次）
MqttClientCreatorMica MQTT 的客户端构建器，链式设置连接参数和回调，Spring 容器自动注入
虚拟线程Java 21 轻量级线程，替换默认线程池使大量并发连接不受 OS 线程数限制
client-idMQTT 协议客户端唯一标识，同一 Broker 不可重复，配置于 mqtt.client.client-id

常用代码

发送消息（需注入 Mica MQTT 的 MqttClient）：

// 注入 MqttClient（来自 Mica MQTT 自动配置）
@Autowired
private MqttClient mqttClient;

// 发布消息到指定 Topic
mqttClient.publish("sensor/temperature/room1",
    "{\"temp\": 25.6}".getBytes(StandardCharsets.UTF_8),
    QoS.AT_LEAST_ONCE);

订阅 Topic（在 MqttClientConnectListener.onConnected 中订阅）：

@Override
public void onConnected(ChannelContext context, boolean isReconnect) {
    log.info("MQTT 已连接, isReconnect={}", isReconnect);
    // 订阅多个 Topic
    client.subscribe("sensor/#", QoS.AT_LEAST_ONCE);
}

application.yml 配置示例：

mqtt:
  client:
    enabled: true
    ip: 127.0.0.1
    port: 1883
    client-id: "000001"
    username: admin
    password: public
    version: mqtt_5          # 支持 mqtt_3_1 / mqtt_3_1_1 / mqtt_5
    keep-alive-secs: 60
    session-expiry-interval-secs: 3600
