速查 01 · MQTT 协议与模块全景

配套课程：lessons/0001.html · 使命：读懂 ruoyi-common-mqtt 模块

一句话定位

ruoyi-common-mqtt 不是 MQTT 框架，而是把 Mica MQTT 收编进 RuoYi 自动配置体系的「接入垫片」——自己不发 MQTT 报文，只补三件事。

MQTT 核心概念

概念说明类比
Broker中央消息代理，接收消息 + 按 Topic 路由邮政总局
TopicUTF-8 分层路径，通配符 + / #收件地址
Publisher发消息到 Broker 的某 Topic寄信人
Subscriber向 Broker 声明订阅哪些 Topic收信人
QoS0=至多一次，1=至少一次，2=恰好一次平信/挂号/签收
Keep Alive心跳间隔（秒），超时 Broker 判离线定期报平安
Persistent SessionBroker 记住订阅和离线消息，重连后补发出差回来收信

模块全部内容

文件作用
pom.xml依赖 mica-mqtt-client-starter + ruoyi-common-core + ruoyi-common-json
config/MqttAutoConfiguration.java装配：注册监听器 + 虚拟线程定制器（第 2 课）
listener/MqttClientConnectListener.java连接：onConnected / onDisconnect 回调（第 3 课）
listener/MqttClientGlobalMessageListener.java消息：onMessage 全局消息处理（第 3 课）
META-INF/spring/...AutoConfiguration.imports发现：声明 MqttAutoConfiguration 为自动配置类

三个类 = 三件事

类职责解决
MqttAutoConfiguration装配入口零配置接入 + 配置开关 + 虚拟线程 IO 优化
MqttClientConnectListener连接生命周期连接/断线统一管理 + 凭证轮换扩展点
MqttClientGlobalMessageListener全局消息入口所有 Topic 消息的统一处理点

激活链路

yml: mqtt.client.enabled=true → @ConditionalOnProperty 满足 → MqttAutoConfiguration 加载（.imports 文件发现） → 3 个 @Bean 注册（两个监听器 + 虚拟线程定制器） → Mica MQTT 用定制后的 Creator 连接 Broker → onConnected() → onMessage() 就绪

Topic 通配符速记

通配符匹配示例
+单个层级device/+/temp → device/001/temp
#多层级（须在末尾）device/# → device/001/temp, device/001/status

MQTT 5 vs 3.1.1

  • Session 过期：5 精确控制秒数，3.1.1 仅持久/非持久
  • 错误反馈：5 有原因码（Reason Code），3.1.1 仅成功/失败
  • 用户属性：5 支持键值对元数据，3.1.1 不支持
  • 消息过期：5 支持 message-expiry-interval，3.1.1 不支持

记忆锚点

  • MQTT 脏活全在 Mica MQTT，本模块 零行 MQTT 协议实现。
  • 模块 不暴露 Controller，是被业务依赖的底层通信能力层。
  • MQTT = Pub/Sub + Topic + QoS + Keep Alive + Persistent Session + Last Will。
  • 发布订阅的核心 = 空间解耦 + 时间解耦 + 一对多广播。
