类注解

注解作用关键参数
@AutoConfiguration声明为 Spring Boot 自动配置类无（由 .imports 文件发现）
@ConditionalOnProperty配置开关，默认关闭value="mqtt.client.enabled", havingValue="true", matchIfMissing=false

三个 Bean 方法

Bean 名称返回类型参数注入职责
mqttClientConnectListenerMqttClientConnectListenerMqttClientCreator（由 Mica MQTT Starter 创建）注册连接生命周期监听器
mqttClientGlobalMessageListenerMqttClientGlobalMessageListener无注册全局消息监听器
mqttClientCustomizerMqttClientCustomizer无（Lambda 实现函数式接口）替换三组线程池为虚拟线程

虚拟线程定制器分析

线程池名称前缀类型职责

装配流程

应用启动 → Spring Boot 扫描所有 jar 的 .imports 文件 → 发现 MqttAutoConfiguration → 检查 mqtt.client.enabled ├─ false/missing → 跳过 └─ true → MqttClientCustomizer 先执行（替换线程池） → 创建 ConnectListener + GlobalMessageListener → Mica MQTT 连接 Broker → 进入监听循环

发现机制

META-INF/spring/ org.springframework.boot.autoconfigure.AutoConfiguration.imports → 内容：org.dromara.common.mqtt.config.MqttAutoConfiguration → 替代 Spring Boot 2.x 的 spring.factories

虚拟线程 vs 平台线程

维度平台线程虚拟线程
创建开销~1MB 栈~数百字节
数量上限数千数百万
阻塞行为占用 OS 线程释放底层平台线程
适用场景CPU 密集IO 密集（MQTT 场景）

记忆锚点

  • 自动配置的发现 → .imports 文件，不是 @ComponentScan 或 @Import。
  • 总开关 → @ConditionalOnProperty，未配置时默认关闭。
  • 虚拟线程 → IO 密集型场景的最优解，阻塞时不占 OS 线程。
  • MqttClientCreator → 由Mica MQTT 创建，本模块只注入使用。
  • corePoolSize 在虚拟线程场景下不重要，只是满足 API 约定。
