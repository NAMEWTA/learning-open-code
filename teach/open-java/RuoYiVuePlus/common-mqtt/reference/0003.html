速查 03 · 监听器与消息收发

配套课程：lessons/0003.html · 使命：读懂 ruoyi-common-mqtt 模块

两个监听器对比

维度MqttClientConnectListenerMqttClientGlobalMessageListener
实现接口IMqttClientConnectListenerIMqttClientGlobalMessageListener
关注点连接生命周期消息处理
触发频率低频（每次连接/断连一次）高频（每条消息一次）
构造参数需要 MqttClientCreator无参数

MqttClientConnectListener 回调速查

方法参数关键逻辑
onConnected ChannelContext, boolean isReconnect isReconnect=false → 首次连接（可做初始化/订阅）
isReconnect=true → 重连（恢复状态）
onDisconnect ChannelContext, Throwable, String remark, boolean isRemove throwable!=null → 异常断连（需告警）
throwable==null → 正常断开
可动态更新 clientId/username/password

MqttClientGlobalMessageListener 参数速查

参数类型说明
contextChannelContext通道上下文（含连接信息）
topicString实际匹配到的 Topic（非通配符）
messageMqttPublishMessageMQTT PUBLISH 报文（含 QoS/Retain/MessageId）
payloadbyte[]原始字节载荷（需按约定解码）

消息收发 API 速查

操作API返回
发布消息mqttClientCreator.publish(topic, payload)CompletableFuture<Void>
订阅 TopicmqttClientCreator.subscribe(topic)void
订阅 + QoSmqttClientCreator.subscribe(topic, qos)void
取消订阅mqttClientCreator.unsubscribe(topic)void

实战代码骨架

// 发布消息
@Autowired MqttClientCreator creator;

creator.publish("ruoyi/device/001/command",
    "{\"action\":\"shutdown\"}".getBytes(StandardCharsets.UTF_8))
    .whenComplete((v, e) -> log.info("发送{}", e==null?"成功":"失败"));

// 扩展监听器
@Component
class MyListener implements IMqttClientGlobalMessageListener {
    public void onMessage(ChannelContext ctx, String topic,
                          MqttPublishMessage msg, byte[] payload) {
        String body = new String(payload, StandardCharsets.UTF_8);
        if (topic.startsWith("ruoyi/device/") && topic.endsWith("/data")) {
            // 处理设备数据
        }
    }
}

// 订阅 Topic
@PostConstruct
void init() {
    creator.subscribe("ruoyi/device/+/data");
    creator.subscribe("ruoyi/device/+/status");
}

Topic 命名规范建议

ruoyi/device/{deviceId}/data — 设备数据上报 ruoyi/device/{deviceId}/status — 设备状态上报 ruoyi/device/{deviceId}/command — 下发给设备的指令 ruoyi/device/{deviceId}/event — 设备事件（告警/故障） ruoyi/broadcast/{type} — 广播消息（全体设备）

常见扩展点

  • 凭证轮换：onDisconnect 中更新 clientId/username/password
  • 消息分流：onMessage 中按 topic.startsWith() 路由
  • 格式转换：payload 解码 JSON / Protobuf 后分发
  • 订阅恢复：onConnected(isReconnect=false) 中重新订阅

记忆锚点

  • 连接监听器 = 低频 + 连接生命周期 + 凭证轮换扩展点。
  • 消息监听器 = 高频 + 所有 Topic 统一入口 + 按 topic 分流业务。
  • isReconnect 区分首次连接和重连（决定是否需恢复订阅）。
  • payload 是 byte[]，需按约定编码解码。
  • 发布返回 CompletableFuture，可异步处理结果。
  • publish() 自动借用 Mica MQTT 的连接池，无需手动管理。
