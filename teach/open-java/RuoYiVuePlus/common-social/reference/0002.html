JustAuth OAuth 四阶段

阶段 1: authorize(state) → 生成授权 URL + state 存缓存 阶段 2: 用户在 OAuth 平台授权 → 回调 redirect_uri 附 code + state 阶段 3: login(callback) → 校验 state → getAccessToken(code) → AuthToken 阶段 4: getUserInfo(token) → AuthUser{uuid, username, avatar, ...}

SocialUtils 两个核心方法

方法签名做什么
loginAuth()(source, code, state, socialProperties) → AuthResponse分发 + 组装 Callback + 委托 login()
getAuthRequest()(source, socialProperties) → AuthRequest配置读取 → AuthConfig 构建 → switch 创建对应平台实例

getAuthRequest() switch 关键分支

平台构造方式特殊参数
github / gitee / weibo / baidu / ...new AuthXxxRequest(builder.build(), STATE_CACHE)标准无特殊
alipay_walletnew AuthAlipayRequest(builder.build(), alipayPublicKey, STATE_CACHE)支付宝公钥
microsoftnew AuthMicrosoftRequest(builder.tenantId(tenantId).build(), STATE_CACHE)租户 ID
wechat_enterprisenew AuthWeChatEnterpriseQrcodeV2Request(builder.agentId(agentId).build(), STATE_CACHE)应用 agentId
dingtalknew AuthDingTalkV2Request(builder.build(), STATE_CACHE)定制版（同包补丁）
gitea / maxkey / topiamnew AuthXxxRequest(builder.build(), STATE_CACHE)自定义提供者

自动装配四件套

① @AutoConfiguration → 声明自动配置类 ② AutoConfiguration.imports → Spring 发现机制（META-INF/spring/...） ③ @EnableConfigurationProperties(SocialProperties.class) → yml 绑定 ④ @Bean AuthStateCache → 注册 AuthRedisStateCache 实例

STATE_CACHE 获取方式

private static final AuthRedisStateCache STATE_CACHE = SpringUtils.getBean(AuthRedisStateCache.class); // 静态工具类无法依赖注入 → 用 SpringUtils 从容器获取
