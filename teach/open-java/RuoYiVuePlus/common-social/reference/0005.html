速查 05 · 完整社交登录链路

配套课程：lessons/0005.html · 以 Gitee 登录为例

链路六帧

帧 1: 前端 → GET /auth/login?source=gitee 帧 2: 后端 → SocialUtils.getAuthRequest("gitee") → new AuthGiteaRequest(config, STATE_CACHE) 帧 3: 后端 → authorize(state) → state 存 Redis + 返回授权 URL → 302 重定向 帧 4: 用户 → Gitee 页面授权 → 回调 redirect_uri?code=xxx&state=xxx 帧 5: 后端 → SocialUtils.loginAuth() → login(callback) → state 校验 → code→token → token→user 帧 6: 后端 → AuthUser 注册/登录 → 签发 token → 返回前端

各环节关键类与 Redis 操作

SocialUtils.loginAuth() 内部展开

loginAuth(source, code, state, socialProperties) ├── authRequest = getAuthRequest(source, socialProperties) │ └── switch(source) → new AuthGiteaRequest(config, STATE_CACHE) ├── callback = new AuthCallback(code, state) └── authRequest.login(callback) ← JustAuth AuthDefaultRequest.login() ├── state 校验 → Redis EXISTS social_auth_code:{state} ├── getAccessToken(callback) → POST token endpoint │ └── Form: client_id + client_secret + code + grant_type + redirect_uri └── getUserInfo(authToken) → GET userInfo endpoint └── Header: Authorization: Bearer {accessToken}

记忆锚点

  • 整个链路 ruoyi-common-social 只负责「生成授权 URL」和「处理回调」两个环节——中间用户和第三方平台的交互不参与。
  • Redis 只在两个地方使用：① authorize 时 SET state ② login 时 EXISTS 校验 state。
  • 完整链路涉及 6 个模块、10+ 个关键类——但核心只有 SocialUtils（调度） + AuthRedisStateCache（缓存） + 平台 Request/Source（执行）。
