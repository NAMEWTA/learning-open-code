配套课程：lessons/0001.html · 使命：读懂 ruoyi-common-social 社交登录模块

一句话定位

ruoyi-common-social 是 JustAuth OAuth 框架之上的「调度 + 装配 + 扩展」三合一适配层——不实现 OAuth 协议，只做统一调度、Redis 状态缓存、自动装配，外加 3 个自定义提供者。

配置驱动模型

yml: justauth.type.{platform}.{clientId, clientSecret, redirectUri, ...} → SocialProperties (Map<String, SocialLoginConfigProperties>) → SocialUtils.getAuthRequest() 读取 → 构建 AuthConfig → 创建 AuthRequest

记忆锚点

  • 13 个 Java 文件，3+1 层结构（装配 + 调度 + 扩展 + 补丁）。
  • OAuth 协议全部由 JustAuth 实现，模块本身零行协议代码。
  • 模块不暴露 Controller，是被 ruoyi-admin 依赖的能力层。
  • 27 个平台共用一个 switch + 一套 AuthConfig 模型。
  • 新增平台：yml 配置 + switch case → 完成，调用方零改动。

→ 相关术语见 GLOSSARY.md：JustAuth、AuthRequest、AuthDefaultRequest、SocialUtils、AuthRedisStateCache
