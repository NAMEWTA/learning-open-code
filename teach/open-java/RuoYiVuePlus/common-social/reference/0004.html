企业微信 OAuth 流程差异

企业微信流程 (5 步，非标准 4 步)： ① 获取企业 access_token (corpId + corpSecret，不需要 code) ② code + access_token → 获取 userId + user_ticket ③ userId → 获取用户基础信息 ④ user_ticket → 获取用户敏感信息 (可选) ⑤ 组装 AuthUser 返回 关键类：AbstractAuthWeChatEnterpriseRequest (抽象父类)

企业微信关键差异点

差异标准 OAuth2企业微信
凭证名clientId / clientSecretcorpId / corpSecret
token 级别用户级企业级
token 获取需传 code不需 code（code 留给后续）
用户信息获取1 次 API2 次 API（userId → 详情）
敏感信息不区分user_ticket 二次获取
必需字段access_tokenuserid 必须存在，否则抛异常

同包补丁注意事项

  • 包名必须精确匹配 me.zhyd.oauth.request
  • 可能被迫使用框架的依赖（如 fastjson 而非项目统一的 JsonUtils）
  • JustAuth 版本升级时需检查 protected 方法签名是否变化
  • 适合快速修复 / 增强，不适合作为长期维护策略
