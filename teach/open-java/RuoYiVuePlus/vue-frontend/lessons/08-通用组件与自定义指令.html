08 通用组件与自定义指令

RuoYi-Vue-Plus 前端提供了 20+ 个通用组件和 3 个自定义指令，覆盖了中后台开发中最常见的交互场景。学完本课后，你将能说出每个组件的用途，并在开发 CRUD 页面时直接复用它们。

核心组件深入

Pagination — 分页组件

<!-- 使用示例 -->
<pagination
  v-show="total > 0"
  :total="total"
  v-model:page="queryParams.pageNum"
  v-model:limit="queryParams.pageSize"
  @pagination="getList"
/>

Pagination 组件通过简单的 v-model:page 和 v-model:limit 双向绑定到 queryParams，在 @pagination 事件中调用列表刷新方法。它内部封装了 el-pagination 的布局预设。

RightToolbar — 表格工具栏

<right-toolbar
  v-model:showSearch="showSearch"
  :columns="columns"
  @queryTable="getList"
  @refresh="getList"
/>

四个功能按钮：搜索切换（控制搜索表单显示/隐藏）、刷新、列设置（显示/隐藏表格列）、导出（触发表格数据导出）。showSearch 与 useSearchToggle composable 配合使用。

SvgIcon — SVG 图标组件

<svg-icon icon-class="user" />  <!-- 引用 src/assets/icons/svg/user.svg -->
<svg-icon icon-class="system" class-name="custom-class" />

底层依赖 Vite 插件 vite-plugin-svg-icons-ng，在构建时将 src/assets/icons/svg/ 目录中的所有 SVG 文件注册为 symbol，通过 <use> 引用。这避免了每个 SVG 产生一个 HTTP 请求。

自定义指令

1. v-hasPermi — 权限按钮控制

<el-button v-hasPermi="['system:user:add']">新增</el-button>

已在第 6 课详解，核心是通过 parentNode.removeChild(el) 从 DOM 中移除无权限的按钮元素。

2. v-hasRoles — 角色按钮控制

<el-button v-hasRoles="['admin', 'common']">管理操作</el-button>

同样使用 DOM 移除策略，检查用户的 roles 数组中是否包含指定角色，或是否为 admin/superadmin。

3. v-copyText — 一键复制

<span v-copyText="'要复制的文本内容'">点击复制</span>

<!-- 也可传入回调 -->
<span v-copyText="['复制内容', callback] | v-copyText:callback">复制</span>

实现原理：

export default {
  beforeMount(el, { value, arg }) {
    if (arg === 'callback') {
      el.$copyCallback = value;
    } else {
      el.$copyValue = value;
      const handler = () => {
        copyTextToClipboard(el.$copyValue);   // 核心复制逻辑
        if (el.$copyCallback) {
          el.$copyCallback(el.$copyValue);    // 复制后回调
        }
      };
      el.addEventListener('click', handler);
    }
  }
};

function copyTextToClipboard(input) {
  const element = document.createElement('textarea');
  element.value = input;
  element.setAttribute('readonly', '');
  element.style.position = 'absolute';
  element.style.left = '-9999px';  // 隐藏 textarea
  document.body.append(element);
  element.select();
  document.execCommand('copy');    // 经典 Clipboard API
  element.remove();
}
注意： document.execCommand('copy') 已被标记为废弃，现代浏览器推荐使用 navigator.clipboard.writeText()。但由于 Clipboard API 需要 HTTPS 或 localhost 环境，execCommand 作为降级方案仍然可用。
与 React 对比： React 中没有自定义指令的概念，按钮级权限通过条件渲染 {hasPerm('xxx') && <Button>} 实现，复制功能通过自定义 Hook（如 useCopyToClipboard()）返回 copy() 函数。Vue 的指令方案将交互逻辑从组件逻辑中分离，使模板更加声明式。
最佳实践：
1. Pagination + RightToolbar + useTableSelection + useSearchToggle 组合使用，可快速搭建标准 CRUD 页面
2. FileUpload 需要后端 OSS 配置正确才能工作
3. IconSelect 依赖 Element Plus 图标库的完整导入（auto-import 已处理）
4. 工作流组件要在 workflow 模块页面中使用，不要跨模块引用
