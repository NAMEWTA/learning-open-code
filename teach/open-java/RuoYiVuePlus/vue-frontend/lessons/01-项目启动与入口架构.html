01 项目启动与入口架构

当一个 Vue 3 应用启动时，一切从 main.ts 开始。本课你将追踪 main.ts 的完整启动链，理解这个生产级中后台前端的初始化骨架。学完本课后，你将能说清楚从 createApp 到 app.mount('#app') 之间发生了什么。

启动入口 main.ts：11 步初始化序列

以下是精简后的启动代码，保留了关键步骤的语义结构：

import { createApp } from 'vue';
import App from './App.vue';

// 步骤1-4: 样式与图标注册（无运行时副作用，纯导入）
import 'virtual:uno.css';                     // UnoCSS 原子 CSS
import 'element-plus/theme-chalk/dark/css-vars.css'; // Element Plus 暗黑模式变量
import '@/assets/styles/index.scss';           // 全局 SCSS 样式入口
import 'virtual:svg-icons-register';           // 注册 SVG 图标精灵

// 步骤5-9: 运行时模块导入
import i18n from '@/lang/index';               // 国际化实例
import ElementIcons from '@/plugins/svgicon';  // SVG 图标组件
import directive from './directive';           // 自定义指令注册函数
import plugins from './plugins/index';         // 全局方法插件（$tab/$modal/$auth...）
import router from './router';                 // Vue Router 实例
import store from './store';                   // Pinia store 实例

// 步骤10: 创建 Vue 应用实例
const app = createApp(App);

// 步骤11: 插件顺序注册
app.use(HighLight);        // 代码高亮
app.use(ElementIcons);     // SvgIcon 组件
app.use(store);            // Pinia
app.use(router);           // Vue Router
app.use(i18n);             // Vue I18n
app.use(VxeTablePlugin);   // VxeTable 表格组件
app.use(plugins);          // 自定义全局插件
directive(app);            // 注册自定义指令

app.mount('#app');
与 React 对比： React 18 用 createRoot(document.getElementById('root')).render(<App />)，所有 Provider/Redux store/react-router 需要在 JSX 中嵌套；Vue 3 则通过 app.use() 链式注册插件，每个插件通过 install(app) 接收 app 实例进行全局注入，风格更扁平。

App.vue：最薄根组件

App.vue 仅做两件事：提供 Element Plus 全局配置上下文，并渲染 <router-view />：

<template>
  <el-config-provider :locale="appStore.locale" :size="appStore.size">
    <router-view />
  </el-config-provider>
</template>

<script setup lang="ts">
import { useAppStore } from '@/store/modules/app';
import { useSettingsStore } from '@/store/modules/settings';
import { handleThemeStyle } from '@/utils/theme';

const appStore = useAppStore();

onMounted(() => {
  nextTick(() => {
    handleThemeStyle(useSettingsStore().theme); // 初始化主题 CSS 变量
  });
});
</script>

el-config-provider 将 Element Plus 的语言包和组件尺寸注入到整个组件树中。这是 Element Plus 的"根上下文"，类似 React 的 Context Provider。

注意： App.vue 不包含路由守卫逻辑。路由守卫定义在独立的 src/permission.ts 文件中，通过 import './permission' 的副作用导入执行——这是 Vue 生态中常见的"模块即行为"模式。

插件注册链：installPlugin 函数

在 src/plugins/index.ts 中，自定义插件将 $tab、$modal、$cache、$download、$auth 五个全局方法挂载到 app.config.globalProperties：

export default function installPlugin(app: App) {
  app.config.globalProperties.$tab = tab;
  app.config.globalProperties.$modal = modal;
  app.config.globalProperties.$cache = cache;
  app.config.globalProperties.$download = download;
  app.config.globalProperties.$auth = auth;
}

这些全局属性在模板中可直接访问 $auth.hasPermi('xxx')，在 Options API 中通过 this.$auth 调用。在 Composition API 中则建议直接从 @/plugins/auth 导入（无需 this）。

指令注册：directive 函数

// src/directive/index.ts
export default (app: App) => {
  app.directive('copyText', copyText);
  app.directive('hasPermi', hasPermi);
  app.directive('hasRoles', hasRoles);
};

三个注册函数的调用顺序（plugins -> directive）不会影响运行时行为，因为一个在 globalProperties 上操作，一个在指令注册表操作。

全局样式体系

入口样式文件 src/assets/styles/index.scss 使用 SCSS 的 @use 模块系统按顺序加载所有样式层：

@use './tokens/theme';        // 浅色主题 CSS 变量
@use './tokens/theme-dark';   // 暗黑主题 CSS 变量（通过 html.dark 选择器切换）
@use './base/document';       // 文档根样式
@use './layout/surface';      // 布局表面样式
@use './components/page-shell'; // 页面壳层样式
@use './components/table-toolbar'; // 表格工具栏样式
@use 'animate.css';           // 动画库

由于 @use 保证模块只加载一次，后加载的 CSS 规则在同优先级下覆盖前面的，因此 theme-dark 在 theme 之后加载是为了能被暗黑模式正确覆盖。

启动时序全景图

完整的初始化时间线：

1. Vite 解析 main.ts，执行所有顶层 import（副作用代码执行）
2. import './permission' → 注册 router.beforeEach（此时守卫已生效）
3. createApp(App) → 创建应用实例
4. app.use(store) → Pinia 安装，但 store 尚未初始化（Pinia 按需激活）
5. app.use(router) → Router 安装，首次路由解析立即触发
6. app.use(i18n) → I18n 安装，$t() 方法可用
7. app.mount('#app') → 首次渲染 App.vue
8. App.vue 渲染 → <router-view /> 开始匹配当前 URL
9. 路由守卫 beforeEach 触发 → 判断是否有 Token
10. 有 Token → 拉取用户信息 → 生成动态路由 → 放行
11. 无 Token → 跳转登录页
关键理解： router.beforeEach 在 import './permission' 时就已注册，早于 app.use(router)。这是因为 Vue Router 允许在安装前注册守卫——守卫注册到的是同一个 router 实例。首次导航在 app.use(router) 时触发，此时守卫早已就位。
