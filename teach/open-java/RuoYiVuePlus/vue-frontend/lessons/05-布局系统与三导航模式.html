05 布局系统与三导航模式

RuoYi-Vue-Plus 的布局系统支持 LEFT（左侧导航）、TOP（顶部导航）、MIX（混合导航）三种模式，通过 settingsStore.navType 动态切换。学完本课后，你将能画出 Layout 组件树的结构，并解释三种布局模式在 DOM 层面的异同。

Layout 组件树

Layout (src/layout/index.vue)
├── Sidebar (侧边栏)           — LEFT/MIX 模式显示
│   ├── Logo (Logo 组件)
│   ├── el-scrollbar (滚动容器)
│   └── el-menu
│       └── SidebarItem (递归渲染菜单项)
│           └── SidebarItem (子菜单递归自身)
├── main-container (主区域)
│   ├── layout-header
│   │   ├── Navbar (顶部导航栏)
│   │   │   ├── Hamburger (折叠按钮)
│   │   │   ├── Breadcrumb (面包屑)
│   │   │   └── TopBar (顶部菜单，TOP 模式)
│   │   └── TagsView (标签页)
│   └── AppMain (页面内容区)
│       └── <router-view /> (含 keep-alive + transition)
└── Settings (右侧设置抽屉)

Layout 模板：条件渲染驱动三模式

<template>
  <div :class="classObj" class="app-wrapper">
    <!-- 移动端遮罩层 -->
    <div v-if="device === 'mobile' && sidebar.opened"
         class="drawer-bg" @click="handleClickOutside" />

    <!-- 侧边栏：仅 LEFT 或 MIX 模式显示 -->
    <side-bar v-if="showSidebar" class="sidebar-container" />

    <div class="main-container">
      <div class="layout-header">
        <navbar @set-layout="setLayout" />
        <tags-view v-if="needTagsView" />
      </div>
      <app-main />
      <settings ref="settingRef" />
    </div>
  </div>
</template>

关键计算属性 showSidebar 决定了侧边栏的可见性：

const layout = computed(() => settingsStore.navType);

const showSidebar = computed(() => {
  if (sidebar.value.hide) return false;    // 全局隐藏标志
  return layout.value === NavTypeEnum.LEFT
      || layout.value === NavTypeEnum.MIX; // TOP 模式不显示侧边栏
});

NavTypeEnum 枚举

export enum NavTypeEnum {
  LEFT = 'left', // 左侧导航：侧边栏固定显示，顶部无菜单
  TOP  = 'top',  // 顶部导航：侧边栏隐藏，菜单在顶栏水平排列
  MIX  = 'mix'   // 混合导航：侧边栏 + 顶栏同时显示
}

SidebarItem：递归菜单组件

侧边栏菜单使用 SidebarItem 组件递归渲染，这是整个布局中最精妙的部分：

<template>
  <div v-if="!item.hidden">
    <!-- 情况1: 只有一个可见子路由 → 直接渲染为菜单项 -->
    <template v-if="hasOneShowingChild(item, item.children) && !item.alwaysShow">
      <app-link :to="resolvePath(onlyOneChild.path)">
        <el-menu-item :index="resolvePath(onlyOneChild.path)">
          <svg-icon :icon-class="onlyOneChild.meta.icon" />
          <template #title>
            <span class="menu-title">{{ onlyOneChild.meta.title }}</span>
          </template>
        </el-menu-item>
      </app-link>
    </template>

    <!-- 情况2: 多个子路由 → 渲染为 el-sub-menu + 递归 children -->
    <el-sub-menu v-else :index="resolvePath(item.path)" teleported>
      <template #title>
        <svg-icon :icon-class="item.meta.icon" />
        <span>{{ item.meta?.title }}</span>
      </template>
      <sidebar-item
        v-for="child in item.children"
        :key="child.path"
        :item="child"
        :is-nest="true"
        :base-path="resolvePath(child.path)"
        class="nest-menu"
      />
    </el-sub-menu>
  </div>
</template>

hasOneShowingChild 的判断逻辑：如果一个父路由只有一个可见的子路由，则跳过父级菜单项，直接显示子菜单项。这避免了"系统管理 -> 用户管理"这种只有一项的冗余折叠。但如果设置了 alwaysShow: true，则强制显示父级。

与 React 对比： React 中递归路由菜单通常通过 useRoutes() 配置 children 属性在 Route 层级实现，UI 组件渲染由不同的 <Outlet /> + Sidebar render 函数组合。Vue 这边的 SidebarItem 递归模式更加直观——它同时是"路由配置"和"UI 组件"的混合体，通过 hidden、alwaysShow、meta.icon 等字段控制。

TopBar：顶部导航

在 TOP 和 MIX 模式下，菜单由 TopBar 组件渲染为水平排列的胶囊按钮：

<el-menu mode="horizontal" class="topbar-menu">
  <!-- 可见菜单（前 N 个） -->
  <sidebar-item
    v-for="menuRoute in topMenus"
    :item="menuRoute"
    :base-path="menuRoute.path"
    popper-class="topbar-menu-popper"
  />
  <!-- 溢出菜单（"更多菜单"下拉） -->
  <el-sub-menu v-if="moreRoutes.length > 0" index="more">
    <template #title><span>更多菜单</span></template>
    <sidebar-item v-for="menuRoute in moreRoutes" ... />
  </el-sub-menu>
</el-menu>

TopBar 通过 visibleNumber 计算属性自适应屏幕宽度：

const visibleNumber = ref(5);
const topMenus = computed(() =>
  permissionStore.sidebarRouters
    .filter(f => !f.hidden)
    .slice(0, visibleNumber.value)
);
const moreRoutes = computed(() =>
  permissionStore.sidebarRouters
    .filter(f => !f.hidden)
    .slice(visibleNumber.value)  // 超出部分放入"更多菜单"
);

function setVisibleNumber() {
  let width = document.body.getBoundingClientRect().width;
  if (width >= 1000) width -= 420;
  visibleNumber.value = Math.max(1, Math.floor(width / 3 / 92) + 2);
}

响应式断点：桌面/移动端切换

Layout 组件通过 useWindowSize 监听窗口宽度变化，以 992px 为断点：

const { width } = useWindowSize();
const WIDTH = 992;

watch(width, w => {
  if (w - 1 < WIDTH) {
    useAppStore().toggleDevice('mobile');
    useAppStore().closeSideBar({ withoutAnimation: true }); // 移动端关闭侧边栏
  } else {
    useAppStore().toggleDevice('desktop');
  }
}, { immediate: true });

在 mobile 模式下，侧边栏通过绝对定位覆盖在主内容上方，点击遮罩层可关闭。

AppMain：KeepAlive 与过渡动画

页面内容区通过组件名实现缓存控制：

<!-- AppMain.vue -->
<section class="app-main">
  <router-view v-slot="{ Component, route }">
    <transition name="fade-transform" mode="out-in">
      <keep-alive :include="cachedViews">  <!-- 只缓存 tagsView 中列出的组件名 -->
        <component :is="Component" :key="route.path" />
      </keep-alive>
    </transition>
  </router-view>
</section>

cachedViews 来自 tagsView store，是当前所有标签页的组件 name 列表。未被缓存的页面每次导航都会重新挂载。组件 name 由 createCustomNameComponent 保证注入。

三种布局模式的 DOM 差异：
LEFT：有 .sidebar-container，固定左侧，main-container 通过 margin-left 留出空间
TOP：无 .sidebar-container，main-container 占满宽，TopBar 在 Navbar 中渲染
MIX：有 .sidebar-container + TopBar，main-container 既要有 margin-left 又要容纳顶部菜单
