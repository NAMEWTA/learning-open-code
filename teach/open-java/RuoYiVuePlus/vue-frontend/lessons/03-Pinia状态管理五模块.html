03 Pinia 状态管理五模块

RuoYi-Vue-Plus 前端使用 Pinia（Vue 3 的官方状态管理库）管理 7 个 store 模块。其中 5 个核心模块（app、user、permission、settings、tagsView）承担了应用运行时的状态中枢。所有 store 均采用 Setup Store 语法（与 Composition API 风格一致）。学完本课后，你将能画出这 5 个 store 的职责边界和数据流向。

五大 Store 职责一览

Store核心状态持久化职责
user token, roles, permissions, name, avatar token 存 localStorage 登录/注销/获取用户信息，维护角色权限数组
permission routes, sidebarRouters, topbarRouters 无 从后端拉取菜单生成路由表，区分侧边栏/顶栏/默认路由
app sidebar(状态), device, language, size, locale sidebarStatus, language, size 侧边栏开/关、设备类型、Element Plus 语言包和组件尺寸
settings theme, navType, tagsView, sideTheme, dark... layout-setting 主题色、布局模式、标签页开关等所有可配置项
tagsView visitedViews, cachedViews, iframeViews 可选 (tagsViewPersist) 标签页的增删改查、缓存管理、持久化恢复

User Store：认证与权限的锚点

user store 是所有权限判断的数据源——后面的 permission store、auth 插件、v-hasPermi 指令都依赖它的 roles 和 permissions 数组：

export const useUserStore = defineStore('user', () => {
  const token = ref(getToken());            // 从 localStorage 恢复
  const name = ref('');                     // 登录名
  const nickname = ref('');                 // 显示名
  const avatar = ref('');
  const roles = ref<Array<string>>([]);       // 角色编码：['admin', 'common']
  const permissions = ref<Array<string>>([]); // 权限编码：['system:user:list', ...]

  const login = async (userInfo) => {
    const [err, res] = await to(loginApi(userInfo));
    const data = res.data;
    setToken(data.access_token);             // 存入 localStorage
    token.value = data.access_token;         // 同步到响应式状态
  };

  const getInfo = async () => {
    const [err, res] = await to(getUserInfo());
    const data = res.data;
    roles.value = data.roles;              // 来自后端 /system/user/getInfo
    permissions.value = data.permissions;  // 来自后端
    if (data.roles.length === 0) {
      roles.value = ['ROLE_DEFAULT'];      // 回退角色
    }
    name.value = data.user.userName;
    nickname.value = data.user.nickName;
  };

  const logout = async () => {
    await logoutApi();
    token.value = '';
    roles.value = [];
    permissions.value = [];
    removeToken();                          // 清除 localStorage
  };

  return { token, name, roles, permissions, login, getInfo, logout };
});
注意 await-to-js 模式： const [err, res] = await to(promise) 是 Go 风格的错误处理，避免了 try/catch 嵌套。该库将 promise 的 reject 转为 [Error, undefined]，resolve 转为 [null, Data]。代码中先判断 if (res) 再解构，隐式跳过了错误分支。

Permission Store：路由表的管理者

permission store 不直接持有权限数据（权限在 user store 中），而是持有路由表：

export const usePermissionStore = defineStore('permission', () => {
  const routes = ref<RouteRecordRaw[]>([]);       // 完整路由表
  const sidebarRouters = ref<RouteRecordRaw[]>([]); // 侧边栏菜单
  const topbarRouters = ref<RouteRecordRaw[]>([]);  // 顶栏菜单
  const defaultRoutes = ref<RouteRecordRaw[]>([]);  // 默认路由

  const generateRoutes = async () => {
    const res = await getRouters();              // 调后端
    const data = Array.isArray(res.data) ? res.data : [];

    // structuredClone 三份独立数据，分别用于不同 Layout 模式
    const sidebarRoutes = filterAsyncRouter(structuredClone(data));
    const rewriteRoutes = filterAsyncRouter(structuredClone(data), undefined, true);
    const defaultRoutes = filterAsyncRouter(structuredClone(data));

    // 过滤本地动态路由
    const asyncRoutes = filterDynamicRoutes(dynamicRoutes);
    asyncRoutes.forEach(route => { router.addRoute(route); });

    // 存入状态
    setRoutes(rewriteRoutes);
    setSidebarRouters(constantRoutes.concat(sidebarRoutes));
    setTopbarRoutes(defaultRoutes);

    duplicateRouteChecker(asyncRoutes, sidebarRoutes);
    return rewriteRoutes;
  };
  // ...
});
与 React 对比： React 生态中路由权限通常在 react-router 的 loader 或自定义 <AuthRoute> 中处理，路由表由 JSX 声明。Vue/Pinia 方案将路由表作为响应式状态管理，侧边栏和顶栏直接通过 computed 从 store 中派生 UI，不需要额外的 Provider 传递。

App Store：UI 壳层状态

app store 管理侧边栏状态、设备类型、Element Plus 国际化配置：

export const useAppStore = defineStore('app', () => {
  const sidebarStatus = useStorage('sidebarStatus', '1'); // 自动 localStorage 读写
  const sidebar = reactive({
    opened: sidebarStatus.value ? !!+sidebarStatus.value : true,
    withoutAnimation: false,
    hide: false
  });
  const device = ref('desktop');   // 'desktop' | 'mobile'
  const size = useStorage('size', 'default');    // Element Plus 组件尺寸
  const language = useStorage('language', 'zh_CN');

  const locale = computed(() => {
    return { zh_CN: zhCN, en_US: enUS }[language.value];
  });

  const toggleSideBar = (withoutAnimation) => {
    sidebar.opened = !sidebar.opened;
    sidebarStatus.value = sidebar.opened ? '1' : '0'; // 同步到 localStorage
  };

  return { device, sidebar, language, locale, size, toggleSideBar, ... };
});

useStorage 来自 VueUse 库，它把 ref() 的值自动同步到 localStorage——任何对 ref 的写入都会自动持久化，页面刷新后自动恢复。这是比手动 watch + localStorage.setItem 更优雅的方案。

Settings Store：主题与布局配置

settings store 使用 useStorage 将用户定制化的布局偏好持久化：

const storageSetting = useStorage('layout-setting', {
  tagsView: true,      // 显示标签页
  fixedHeader: true,   // 固定头部
  sidebarLogo: true,   // 显示 Logo
  navType: NavTypeEnum.LEFT, // 左侧导航
  theme: '#409EFF',    // 主题色
  sideTheme: 'theme-dark',   // 侧边栏深色主题
  // ...
});

// 每个独立配置项也通过 useStorage 引用 storageSetting 中的键
const theme = ref(storageSetting.value.theme);
const navType = ref(storageSetting.value.navType || NavTypeEnum.LEFT);

TagsView Store：标签页管理

tagsView store 维护三个核心数组：

  • visitedViews：用户打开过的所有标签页
  • cachedViews：被 keep-alive 缓存的组件 name 列表
  • iframeViews：内嵌的 iframe 标签页
export const useTagsViewStore = defineStore('tagsView', () => {
  const visitedViews = ref<TagView[]>([]);
  const cachedViews = ref<string[]>([]);
  const iframeViews = ref<TagView[]>([]);

  const addView = (view) => {
    addVisitedView(view);  // 去重后加入 visitedViews
    addCachedView(view);   // 若非 noCache，加入 cachedViews
  };

  const delView = (view) => {
    delVisitedView(view);  // 从 visitedViews 移除
    delCachedView(view);   // 从 cachedViews 移除
  };

  const delOthersViews = (view) => {
    // 只保留 affix 标签和当前标签
    visitedViews = visitedViews.filter(v => v.meta?.affix || v.path === view.path);
  };

  const loadPersistedViews = () => {
    // 若开启 tagsViewPersist，从 localStorage 恢复标签页
    loadVisitedViews().forEach(view => { visitedViews.push(...); });
  };
});
数据流向图： user.getInfo() 填充 roles/permissions → permission.generateRoutes() 依据这些权限过滤菜单 → 路由注入后 Layout 组件从 permission store 读取 sidebarRouters 渲染侧边栏 → tagsView store 追踪当前打开的所有页面 → app store 控制侧边栏折叠/设备适配 → settings store 控制主题/布局模式。
