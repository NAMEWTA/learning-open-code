02 路由系统与动态菜单

RuoYi-Vue-Plus 的路由系统是"常量路由 + 动态路由"的双层架构：常量路由（登录页、首页、404 等）在构建时已注册，动态路由在用户登录后从后端接口拉取并调用 router.addRoute() 注入。学完本课后，你将能完整追踪从后端菜单数据到侧边栏渲染的全链路。

常量路由：constantRoutes

常量路由在 src/router/index.ts 中预先定义，包含以下部分：

export const constantRoutes: RouteRecordRaw[] = [
  { path: '/redirect', component: Layout, hidden: true, children: [...] },
  { path: '/social-callback', hidden: true, component: ... },
  { path: '/login', component: () => import('@/views/login.vue'), hidden: true },
  { path: '/register', component: () => import('@/views/register.vue'), hidden: true },
  { path: '/:pathMatch(.*)*', component: () => import('@/views/error/404.vue'), hidden: true },
  { path: '/401', component: () => import('@/views/error/401.vue'), hidden: true },
  {
    path: '', component: Layout, redirect: '/index',
    children: [
      { path: '/index', component: () => import('@/views/index.vue'),
        name: 'Index', meta: { title: '首页', icon: 'dashboard', affix: true } }
    ]
  },
  {
    path: '/user', component: Layout, hidden: true,
    children: [
      { path: 'profile', component: ..., name: 'Profile', meta: { title: '个人中心' } }
    ]
  }
];

所有常量路由都设置 hidden: true（除首页外），意味着它们不会出现在侧边栏菜单中，但路由本身可用。

动态路由：dynamicRoutes（空数组 + 本地验证）

// 动态路由，基于用户权限动态去加载
export const dynamicRoutes: RouteRecordRaw[] = [];

dynamicRoutes 在本项目中是空数组。它的设计意图是通过 filterDynamicRoutes 函数根据 route.permissions 和 route.roles 字段判断本地路由是否可被有权限的用户访问。如果有需求在本地定义权限路由，在这个数组中填入路由对象即可，框架会自动过滤。

路由守卫：permission.ts 的 beforeEach

路由守卫是登录态校验和动态路由注入的核心：

import { getToken } from '@/utils/auth';
import router from './router';

const whiteList = ['/login', '/register', '/social-callback', ...];

router.beforeEach(async (to, from) => {
  NProgress.start();

  if (getToken()) {
    // === 有 Token ===
    if (to.path === '/login') {
      return { path: '/' }; // 已登录用户访问登录页 → 重定向首页
    }

    if (useUserStore().roles.length === 0) {
      // 角色为空 → 说明 userInfo 还没拉取，现在拉取
      await useUserStore().getInfo();          // 获取用户信息 + 角色/权限列表
      const accessRoutes = await usePermissionStore().generateRoutes(); // 拉取菜单生成路由
      accessRoutes.forEach(route => {
        router.addRoute(route);                // 动态注入路由
      });
      return { path: to.path, replace: true }; // hack: 确保 addRoute 生效后重新匹配
    }
    return true; // 角色已存在，直接放行
  } else {
    // === 无 Token ===
    if (whiteList.includes(to.path)) {
      return true; // 白名单放行
    }
    return `/login?redirect=${encodeURIComponent(to.fullPath)}`; // 重定向登录页
  }
});
Hack 模式说明： return { path: to.path, replace: true } 是一段经典的 Vue Router 动态路由 hack。因为 router.addRoute() 不会自动触发当前导航的重新匹配，必须通过返回一个新的导航对象来触发重新解析。这个 hack 会导致目标路由经过两次 beforeEach，第二次时 roles.length 已不为 0，直接放行。
与 React 对比： React Router v6 没有官方的路由守卫概念，通常通过封装 <AuthRoute> 组件或在 loader 中检查认证态来实现。Vue Router 的 beforeEach 提供了更干净的全局拦截点，支持异步 resolve 和路由重定向。

generateRoutes：后端菜单 -> 前端路由的转换核心

这是整个路由系统最关键的函数，位于 src/store/modules/permission.ts：

const generateRoutes = async (): Promise<RouteRecordRaw[]> => {
  // 1. 调用后端接口获取菜单树
  const res = await getRouters(); // GET /system/menu/getRouters
  const data = Array.isArray(res.data) ? res.data : [];

  // 2. 深克隆三份数据，用于不同的展示场景
  const sdata = structuredClone(data);    // 侧边栏展示用
  const rdata = structuredClone(data);    // 注册路由用（含组件重写）
  const defaultData = structuredClone(data); // 默认路由

  // 3. filterAsyncRouter 将后端字符串组件名 → 真实 Vue 组件
  const sidebarRoutes = filterAsyncRouter(sdata);           // 侧边栏菜单
  const rewriteRoutes = filterAsyncRouter(rdata, undefined, true); // 注册路由
  const defaultRoutes = filterAsyncRouter(defaultData);     // 顶栏菜单

  // 4. 从本地 dynamicRoutes 中过滤有权限的路由并 addRoute
  const asyncRoutes = filterDynamicRoutes(dynamicRoutes);
  asyncRoutes.forEach(route => { router.addRoute(route); });

  // 5. 存入 store，供 Layout 组件消费
  setRoutes(rewriteRoutes);
  setSidebarRouters(constantRoutes.concat(sidebarRoutes));
  setTopbarRoutes(defaultRoutes);

  // 6. 路由 name 重复检测
  duplicateRouteChecker(asyncRoutes, sidebarRoutes);

  return rewriteRoutes;
};

filterAsyncRouter：组件字符串 → 真实组件的映射

后端返回的菜单数据中，component 字段是字符串（如 'system/user/index'），需要转换为 Vue 的异步组件加载函数：

// 在 store 初始化时预建查找表
const modules = import.meta.glob('./../../views/**/*.vue');
const viewModuleMap = new Map<string, () => Promise<any>>();
for (const path in modules) {
  const dir = path.substring(
    path.indexOf('/views/') + 7,
    path.lastIndexOf('.vue')
  );
  viewModuleMap.set(dir, modules[path]);
}

const filterAsyncRouter = (asyncRouterMap, lastRouter?, type = false) => {
  return asyncRouterMap.filter(route => {
    // 特殊组件直接赋值
    if (route.component?.toString() === 'Layout') {
      route.component = Layout;
    } else if (route.component?.toString() === 'ParentView') {
      route.component = ParentView;
    } else if (route.component?.toString() === 'InnerLink') {
      route.component = InnerLink;
    } else {
      // 从查找表中获取异步组件
      route.component = loadView(route.component, route.name);
    }
    // 递归处理子路由
    if (route.children?.length) {
      route.children = filterAsyncRouter(route.children, route, type);
    }
    return true;
  });
};

import.meta.glob 是 Vite 的静态导入语法，相当于 Webpack 的 require.context。它在构建时将所有匹配的 .vue 文件路径转化为一个键值对对象，运行时从 Map 中 O(1) 查找对应的 loader 函数。

路由 Name 重复检测：duplicateRouteChecker

当本地 dynamicRoutes 和后端菜单中包含相同 name 的路由时（keep-alive 依赖 name 作为缓存 key），会导致页面 404。框架通过展平所有路由并检查 name 唯一性来防御：

function duplicateRouteChecker(localRoutes, routes) {
  const allRoutes = flatRoutes([...localRoutes, ...routes]);
  const nameList = [];
  allRoutes.forEach(route => {
    const name = route.name?.toString() ?? '';
    if (!name) return;
    if (nameList.includes(name)) {
      console.error(`路由名称: [${name}] 重复, 会造成 404`);
      ElNotification({ title: '路由名称重复', message: ..., type: 'error' });
      return;
    }
    nameList.push(name);
  });
}
常见坑点： 如果你在本地 dynamicRoutes 中定义了与后端菜单 name 相同的路由，前端不会报编译错误，但运行时 keep-alive 会混乱，导致页面 404。这个检测器在开发时帮你尽早发现。

createCustomNameComponent：为异步组件注入 name

由于 import.meta.glob 加载的是包装后的异步组件，其自身没有 name 选项，而 keep-alive 需要 name 作为缓存 key。该工具用一个有 name 的壳层组件包裹：

export function createCustomNameComponent(loader, options = {}) {
  const { name } = options;
  let component = null;
  let wrappedComponent = null;

  return async () => {
    if (!component) {
      const { default: loadedComponent } = await loader();
      component = loadedComponent;
    }
    if (!wrappedComponent) {
      wrappedComponent = defineComponent({
        name,
        render() { return h(component); }
      });
    }
    return wrappedComponent;
  };
}
