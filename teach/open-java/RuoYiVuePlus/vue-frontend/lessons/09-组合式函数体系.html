09 组合式函数体系

RuoYi-Vue-Plus 前端将常见的交互模式抽取为 10 个 composables（Vue 3 组合式函数），按场景分为表格、表单、弹窗、树、异步加载五类。学完本课后，你将能利用这些 composables 快速组装标准 CRUD 页面的交互逻辑。

├── form/                   # 表单场景

Composable返回的核心数据典型使用场景
useTableSelection ids, selectedRows, single, multiple 表格批量删除/导出——通过 ids 传给后端
useTableSortQuery handleSortChange, resetSort 表格列排序——同步 orderByColumn 和 isAsc 到查询参数
useSearchToggle showSearch, toggleSearch 搜索表单的显示/隐藏（配合 RightToolbar 的搜索按钮）
useSearchReset resetQuery 重置搜索表单 + 重置分页参数到初始值
useDateRangeQuery dateRange, applyDateRange, resetDateRange 日期范围选择器 → 展开为 beginTime/endTime 参数
useDialogState dialog, openDialog, closeDialog 任何弹窗的打开/关闭状态管理
useFormDialog dialog, openDialog, closeDialog, resetForm 新增/编辑弹窗——打开时自动重置表单
useTreeTableExpand isExpandAll, handleToggleExpandAll 树形表格的全展开/全收起
useTreeCollapsed treeCollapsed, toggleCollapsed 左侧树形筛选面板的折叠/展开
useLoading loading, setLoading, withLoading 任何异步操作的加载态管理

核心 Composable 深入

useTableSelection — 表格多选

export function useTableSelection<T, ID extends string | number>(
  getRowId: (row: T) => ID
) {
  const ids = ref<ID[]>([]);
  const selectedRows = ref<T[]>([]);
  const single = ref(true);    // 禁用单个操作按钮的条件
  const multiple = ref(true);  // 禁用批量操作按钮的条件

  const handleSelectionChange = (selection: T[]) => {
    selectedRows.value = selection;
    ids.value = selection.map(getRowId); // 提取 ID 数组
    single.value = selection.length !== 1;
    multiple.value = selection.length === 0;
  };

  return { ids, selectedRows, single, multiple, handleSelectionChange };
}

使用示例：

const { ids, single, multiple, handleSelectionChange } =
  useTableSelection<UserVO>(row => row.userId);

// 模板中
<el-table @selection-change="handleSelectionChange">
<el-button :disabled="single" @click="handleUpdate(ids[0])">修改</el-button>
<el-button :disabled="multiple" @click="handleBatchDelete(ids)">批量删除</el-button>
设计精妙处： single 和 multiple 的语义是"禁用按钮"——当选中 1 条时 single=false（不禁用单个操作），当选中 0 条时 multiple=true（禁用批量操作）。这与 Element Plus 的 disabled 属性直接绑定，避免了额外的取反逻辑。

useSearchReset — 搜索重置

export function useSearchReset<T extends Record<string, any>>(options) {
  const { queryFormRef, queryParams, pageNumKey, pageSizeKey,
          initialPageNum = 1, initialPageSize, resetExtras, afterReset } = options;

  const resetQuery = () => {
    queryFormRef.value?.resetFields();    // ElForm 重置
    if (queryParams?.value) {
      if (pageNumKey) {
        queryParams.value[pageNumKey] = initialPageNum; // 分页归位
      }
      if (pageSizeKey && initialPageSize !== undefined) {
        queryParams.value[pageSizeKey] = initialPageSize;
      }
    }
    resetExtras?.();  // 额外的自定义重置逻辑
    afterReset?.();   // 重置后的回调（如重新查询）
  };

  return { resetQuery };
}

useFormDialog — 表单弹窗

export function useFormDialog<T extends Record<string, any>>(options) {
  const { form, formRef, initialFormData, initialTitle } = options;
  const { dialog, openDialog: baseOpen, closeDialog, setTitle }
    = useDialogState(initialTitle);

  const resetForm = () => {
    form.value = { ...initialFormData };  // 恢复初始数据
    formRef?.value?.resetFields();        // 重置表单校验
    formRef?.value?.clearValidate?.();    // 清除校验错误
  };

  const openFormDialog = (title?: string) => {
    resetForm();                           // 先重置再打开
    baseOpen(title);
  };

  return { dialog, resetForm, openDialog: openFormDialog, closeDialog, setTitle };
}

使用示例（新增用户弹窗）：

const { dialog, openDialog, closeDialog } = useFormDialog({
  form: ref<UserForm>({ ...initialFormData }),
  formRef: ref<ElFormInstance>(),
  initialFormData: { userName: '', status: '0', roleIds: [], postIds: [] },
  initialTitle: '新增用户'
});

// 打开新增弹窗
openDialog('新增用户');

// 打开编辑弹窗（先填充 form，再 openDialog 或直接 showDialog）
showDialog('编辑用户'); // 不重置表单，保留已填充的数据

useLoading — 异步加载态

export function useLoading(initialValue = false) {
  const loading = ref(initialValue);

  const withLoading = async <T>(task: () => Promise<T>) => {
    loading.value = true;
    try {
      return await task();
    } finally {
      loading.value = false;  // 无论成功还是失败都关闭 loading
    }
  };

  return { loading, setLoading, withLoading };
}

withLoading 是高阶函数包装器，自动处理 loading 状态生命周期：

const { loading, withLoading } = useLoading();

const getList = async () => {
  const result = await withLoading(() => listUser(queryParams));
  // loading 自动为 true → false，无需手动管理
};

Composables 组合使用模式

一个标准的 CRUD 页面通常同时使用 4-5 个 composables：

// ===== 典型 CRUD 页面模板 =====

// 1. 表格选择
const { ids, selectedRows, single, multiple, handleSelectionChange }
  = useTableSelection(row => row.userId);

// 2. 搜索区控制
const { showSearch, toggleSearch } = useSearchToggle(true);
const { resetQuery } = useSearchReset({
  queryFormRef: queryForm,
  queryParams: queryParams,
  pageNumKey: 'pageNum',
  pageSizeKey: 'pageSize',
  afterReset: getList
});

// 3. 日期范围
const { dateRange, applyDateRange, resetDateRange }
  = useDateRangeQuery();

// 4. 表单弹窗
const { dialog, openDialog, closeDialog } = useFormDialog({
  form: form,
  formRef: userFormRef,
  initialFormData: { ... }
});

// 5. 加载态
const { loading, withLoading } = useLoading();

// 6. 查询列表
const getList = async () => {
  await withLoading(async () => {
    const params = applyDateRange(queryParams.value); // 日期范围展开
    const res = await listUser(params);
    userList.value = res.data.rows;
    total.value = res.data.total;
  });
};
与 React 对比： React 中的自定义 Hook 和 Vue 的 composable 设计理念完全一致——都是将可复用逻辑提取为纯函数。区别在于：Vue 的 composable 中 ref 和 computed 自动追踪依赖，而 React 的 Hook 依赖显式的依赖数组和在组件顶层调用顺序的约束（Rules of Hooks）。Vue 的响应式系统使 composable 的使用更灵活（可在条件分支中调用、可嵌套在普通函数中）。
