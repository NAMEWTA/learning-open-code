10 构建配置与环境管理

RuoYi-Vue-Plus 前端基于 Vite 8 构建，集成了一套完整的工程化工具链：UnoCSS 原子 CSS、自动导入、SVG 精灵图、Gzip/Brotli 压缩、多环境配置、国际化 i18n。学完本课后，你将能理解和修改 Vite 构建配置，并在多环境之间正确切换。

Vite 配置：vite.config.ts

export default defineConfig(({ mode, command }) => {
  const env = loadEnv(mode, process.cwd());  // 加载 .env 文件

  return {
    base: env.VITE_APP_CONTEXT_PATH,          // 部署路径

    resolve: {
      tsconfigPaths: true,                    // 自动从 tsconfig.json 读取路径别名
      extensions: ['.mjs', '.js', '.ts', '.jsx', '.tsx', '.json', '.vue']
    },

    plugins: createPlugins(env, command === 'build'), // 插件工厂函数

    build: {
      chunkSizeWarningLimit: 1500,            // 大 chunk 警告阈值 (1500KB)
    },

    server: {
      host: '0.0.0.0',
      port: Number(env.VITE_APP_PORT),        // 默认 80
      open: true,                             // 自动打开浏览器
      proxy: {
        [env.VITE_APP_BASE_API]: {            // '/dev-api'
          target: 'http://localhost:8080',    // 代理到后端
          changeOrigin: true,
          ws: true,                           // WebSocket 代理
          rewrite: path => path.replace(
            new RegExp('^' + env.VITE_APP_BASE_API), ''
          )                                   // /dev-api/xxx → /xxx
        }
      }
    },

    css: {
      preprocessorOptions: {
        scss: {
          // SCSS 全局变量/混入注入（本项目未启用 additionalData）
        }
      },
      postcss: {
        plugins: [autoprefixer()]             // 自动添加浏览器前缀
      }
    }
  };
});
loadEnv 机制： Vite 的 loadEnv(mode, cwd) 会在 process.cwd() 目录下加载 .env.[mode] 和 .env.[mode].local 文件，并自动过滤 VITE_ 前缀的变量暴露给客户端代码。其他前缀的变量仅在 vite.config.ts 中可用。

Vite 插件链

插件在 vite/plugins/index.ts 中通过工厂函数统一创建：

export default (viteEnv, isBuild = false) => {
  const vitePlugins = [];
  vitePlugins.push(vue());                          // 1. @vitejs/plugin-vue — Vue SFC 编译
  vitePlugins.push(createUnoCss());                  // 2. UnoCSS — 原子 CSS 引擎
  vitePlugins.push(createAutoImport());              // 3. unplugin-auto-import
  vitePlugins.push(createComponents());              // 4. unplugin-vue-components
  vitePlugins.push(createCompression(viteEnv));      // 5. Gzip/Brotli 压缩
  vitePlugins.push(createSvgIconsPlugin());          // 6. SVG 精灵图
  vitePlugins.push(createSetupExtend());             // 7. setup 扩展
  vitePlugins.push(viteCheckTransitionPlugin());     // 8. 过渡动画检查
  return vitePlugins;
};

插件 3+4：自动导入（unplugin-auto-import + unplugin-vue-components）

// auto-import.ts
AutoImport({
  imports: ['vue', 'vue-router', '@vueuse/core', 'pinia'],
  resolvers: [ElementPlusResolver({ importStyle: false })],
  vueTemplate: true,
  dts: 'src/types/auto-imports.d.ts'     // 生成类型声明文件
})

// components.ts
Components({
  resolvers: [ElementPlusResolver({ importStyle: false })],
  dts: 'src/types/components.d.ts'
})

这两个插件使开发者可以 无需手动 import 直接使用 Vue API 和 Element Plus 组件：

// 无需 import { ref, computed, watch, onMounted } from 'vue'
const count = ref(0);            // auto-import 自动处理
const doubled = computed(() => count.value * 2);

// 无需 import { ElButton, ElDialog } from 'element-plus'
<el-button type="primary">确定</el-button>  // components 插件自动解析
类型提示： 自动生成的两个 .d.ts 文件（auto-imports.d.ts 和 components.d.ts）为 IDE 提供类型提示，使自动导入的 API 不会报"未定义"错误。如果开发时遇到类型报错，检查这两个文件是否已生成（运行一次 pnpm dev 即可）。

插件 2：UnoCSS

// uno.config.ts
export default defineConfig({
  shortcuts: {
    'panel-title': 'pb-[5px] font-sans leading-[1.1] font-medium text-base
                    text-[#6379bb] border-b border-b-solid
                    border-[var(--el-border-color-light)] mb-5 mt-0'
  },
  presets: [
    presetWind3(),          // Tailwind CSS v3 兼容预设
    presetAttributify(),    // 属性化模式（如 text="red"）
    presetTypography(),     // 排版预设（prose 类）
    presetWebFonts()        // Web 字体预设
  ],
  transformers: [
    transformerDirectives(),    // @apply 指令支持
    transformerVariantGroup()   // 变体分组（如 hover:(bg-red text-white)）
  ]
});

UnoCSS 在构建时扫描代码中使用的工具类，只生成实际用到的 CSS，无需 PurgeCSS。

// .env.development

国际化 i18n

// src/lang/index.ts
import { createI18n } from 'vue-i18n';

const i18n = createI18n({
  globalInjection: true,      // 全局注入 $t() 方法
  allowComposition: true,     // 允许在 setup 中使用
  legacy: false,              // 使用 Composition API 模式
  locale: getLanguage(),      // 从 localStorage 读取 'zh_CN' | 'en_US'
  messages: {
    zh_CN: zh_CN,             // 中文语言包
    en_US: en_US              // 英文语言包
  }
});

export const getLanguage = (): LanguageEnum => {
  const language = useStorage('language', LanguageEnum.zh_CN);
  return language.value || LanguageEnum.zh_CN;
};

语言包文件（src/lang/zh_CN.ts、src/lang/en_US.ts）使用嵌套 JSON 结构组织：

// 使用示例
{{ $t('login.username') }}     // 模板中
const { t } = useI18n();        // setup 中
t('system.user.addSuccess')     // 逻辑中
国际化与 Element Plus 的联动： Element Plus 的 el-config-provider 通过 locale prop 接收语言包。app store 中的 locale 计算属性将 language 值映射为 Element Plus 的语言包对象（zhCN / enUS），从而实现项目国际化与组件库国际化的联动。
与 React 对比： React 生态中 Vite 配置差异不大（同样使用 @vitejs/plugin-react 替代 @vitejs/plugin-vue）。但自动导入方案在 React 中不常见——React JSX 需要显式 import 组件和 Hooks（eslint-plugin-react-hooks 要求）。UnoCSS 在 React 中同样可用。国际化方面，React 常用 react-intl 或 next-intl（Next.js），与 Vue I18n 的模式差异较大。
