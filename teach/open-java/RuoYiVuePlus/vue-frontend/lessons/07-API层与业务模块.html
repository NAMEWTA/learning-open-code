07 API 层与业务模块

RuoYi-Vue-Plus 的 API 层按照后端模块进行目录划分，每个模块内部采用统一的三文件结构：types.ts（类型定义）、index.ts（API 函数）。学完本课后，你将能快速在项目中定位 API 函数，并遵循规范新增模块接口。

API 函数标准模式

以用户管理模块为例，API 函数遵循以下规范：

// src/api/system/user/index.ts
import type { UserForm, UserQuery, UserVO } from './types';
import type { AxiosPromise } from '@/utils/api-types';
import request from '@/utils/request';

/** 查询用户列表 */
export const listUser = (query: UserQuery): AxiosPromise<PageResult<UserVO>> => {
  return request({
    url: '/system/user/list',
    method: 'get',
    params: query
  });
};

/** 获取用户详情 */
export const getUser = (userId?: string | number): AxiosPromise<UserInfoVO> => {
  return request({
    url: '/system/user/' + parseStrEmpty(userId),
    method: 'get'
  });
};

/** 新增用户 */
export const addUser = (data: UserForm) => {
  return request({
    url: '/system/user',
    method: 'post',
    data: data
  });
};

/** 删除用户 */
export const delUser = (userId: Array<string | number> | string | number) => {
  return request({
    url: '/system/user/' + userId,
    method: 'delete'
  });
};

/** 重置密码（需加密） */
export const resetUserPwd = (userId: string | number, password: string) => {
  return request({
    url: '/system/user/resetPwd',
    method: 'put',
    headers: {
      isEncrypt: true,       // 密码需要加密传输
      repeatSubmit: false     // 禁止重复提交
    },
    data: { userId, password }
  });
};
命名约定：
- 查询列表：listXxx (GET, 带 params)
- 查询单条：getXxx (GET, id 拼接 URL)
- 新增：addXxx (POST, 带 data)
- 修改：updateXxx (PUT, 带 data)
- 删除：delXxx (DELETE, id 拼接 URL)
- 导出/特殊操作：自由命名

类型定义模式

每个模块的 types.ts 定义前端使用的类型，与后端 DTO/VO 对齐：

// src/api/system/user/types.ts

/** 用户信息（getInfo 接口返回） */
export interface UserInfo {
  user: UserVO;
  roles: string[];        // 角色编码数组
  permissions: string[];   // 权限编码数组
}

/** 用户查询参数 */
export interface UserQuery extends PageQuery {
  userName?: string;
  phoneNumber?: string;
  status?: string;
  deptId?: string | number;
}

/** 用户列表项 */
export interface UserVO extends BaseEntity {
  userId: string | number;
  userName: string;
  nickName: string;
  email: string;
  status: string;
  deptName: string;
  roles: RoleVO[];
}

/** 用户表单 */
export interface UserForm {
  userId?: string;
  userName: string;
  password: string;
  status: string;
  postIds: string[];
  roleIds: string[];
}

全局共享类型定义在 src/api/types.ts：

/** 登录请求 */
export interface LoginData {
  username?: string;
  password?: string;
  clientId: string;
  grantType: string;
}

/** 登录响应 */
export interface LoginResult {
  access_token: string;
}

/** 分页返回结果 */
export interface PageResult<T = any> {
  total: number;
  rows: T[];
}

AxiosPromise 类型包装

项目定义了 AxiosPromise<T> 工具类型来保持 API 返回值的类型清晰：

// src/utils/api-types.ts
export type AxiosPromise<T = any> = Promise<RuoYiAjaxResult<T>>;

export interface RuoYiAjaxResult<T = any> {
  code: number;
  msg: string;
  data: T;
}

在业务代码中这样消费：

const [err, res] = await to(listUser(query));
if (res) {
  const { rows, total } = (res as RuoYiAjaxResult<PageResult<UserVO>>).data;
  // rows: UserVO[], total: number  → 类型安全
}

登录 API 的特殊配置

登录接口使用特殊的 header 标志：

export function login(data: LoginData): AxiosPromise<LoginResult> {
  return request({
    url: '/auth/login',
    headers: {
      isToken: false,       // 登录时不需要 token（还没登录！）
      isEncrypt: true,       // 密码需要加密传输
      repeatSubmit: false    // 禁止快速重复登录
    },
    method: 'post',
    data: params
  });
}

与 React 对比： React 项目通常使用 axios + react-query（TanStack Query）的组合，API 函数返回 promise 后由 query hooks 管理缓存和重取。RuoYi-Vue-Plus 没有引入 VueQuery，而是通过 Pinia store（如 dict store）手动管理 API 数据的本地缓存，好处是减少依赖，代价是缓存策略需要手动维护。
