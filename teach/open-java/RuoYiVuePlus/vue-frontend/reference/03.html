数据流向

user.getInfo() → roles[], permissions[] 填充 → permission.generateRoutes() → getRouters() → filterAsyncRouter() → sidebarRouters[] / topbarRouters[] → Layout 组件消费 sidebarRouters → SidebarItem 递归渲染菜单 → 用户点击菜单 → 路由跳转 → tagsView.addView() → visitedViews[] + cachedViews[] → AppMain 的 keep-alive :include="cachedViews" settings.theme / navType → Layout classObj 计算 → CSS 变量切换 app.sidebar.opened → 侧边栏折叠/展开 → main-container margin-left 变化

关键 API 速查

// === user store ===
useUserStore().login(userInfo)     // 登录 → setToken
useUserStore().getInfo()           // 拉取用户信息 + roles + permissions
useUserStore().logout()            // 清除 token + roles + permissions

// === permission store ===
usePermissionStore().generateRoutes() // 拉取后端菜单 → 生成路由表

// === app store ===
useAppStore().toggleSideBar()      // 折叠/展开
useAppStore().toggleDevice('mobile'|'desktop')

// === settings store ===
useSettingsStore().navType         // 'left' | 'top' | 'mix'
useSettingsStore().theme           // '#409EFF'

// === tagsView store ===
useTagsViewStore().addView(view)   // 打开标签
useTagsViewStore().delView(view)   // 关闭标签
useTagsViewStore().cachedViews     // keep-alive 白名单

关键知识点

知识点说明
语法 Setup Store与 Composition API 风格一致，ref/reactive/computed 即状态
持久化 useStorageVueUse 提供，ref 值自动同步 localStorage，页面刷新自动恢复
错误处理 await-to-jsconst [err, res] = await to(promise)，Go 风格，避免 try/catch
路由 structuredClone ×3permission store 用三份独立数据分别服务侧边栏/注册路由/顶栏
注意：permission store 不持有权限数据（权限在 user store），它只持有路由表。await-to-js 模式下需先判断 if(res) 再解构。
