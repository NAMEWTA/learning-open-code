速查 04：Axios 封装与接口加密

拦截器链全景

请求 → service.interceptors.request ├── Content-Language 注入 ├── Authorization: Bearer <token> (isToken !== false) ├── GET params → URL query string ├── 防重复提交检查 (sessionStorage, 500ms, POST/PUT, repeatSubmit !== false) └── RSA+AES 加密 (isEncrypt=true, POST/PUT) ├── 生成随机 32 位 AES Key (crypto.getRandomValues) ├── encrypt-key header ← RSA 加密 AES Key └── body ← AES-ECB/PKCS7 加密请求体 ↓ 发送 ↓ 响应 ← service.interceptors.response ├── RSA 解密 encrypt-key → 得到 AES Key → AES 解密响应体 ├── 二进制/blobl → 直接返回 ├── code===401 → 重新登录弹窗（isRelogin 全局锁防重复） ├── code===500 → ElMessage.error + createHandledError ├── code===601 (WARN) → ElMessage.warning + createHandledError ├── code!==200 → ElNotification.error + createHandledError └── code===200 → resolve(res.data)

自定义 Header 标志

Header含义典型场景
isToken: false跳过 Token 注入登录接口（还没 token）
isEncrypt: true启用 RSA+AES 加密登录、重置密码等敏感接口
repeatSubmit: false关闭防重复提交允许快速重复提交的接口（注意语义反转）

加密三函数

// 1. 生成随机 AES Key
generateAesKey(): WordArray
  → crypto.getRandomValues(new Uint8Array(32))
  → CryptoJS.enc.Utf8.parse(hexString.slice(0, 32))

// 2. AES 加密请求体
encryptWithAes(message, aesKey): string
  → CryptoJS.AES.encrypt(message, aesKey, { mode: ECB, padding: Pkcs7 })

// 3. RSA 加密 AES Key
encrypt(txt): string
  → new JSEncrypt() + setPublicKey(env.VITE_APP_RSA_PUBLIC_KEY)
  → encryptor.encrypt(txt)

// createHandledError — 标记错误已被拦截器处理
function createHandledError(msg) {
  const err = new Error(msg);
  err.isHandled = true;
  return err;
}

关键知识点

知识点说明
实例 axios.create()自定义实例，独立拦截器链，不污染全局 axios
防重复 sessionStorage500ms 内同 url+data 判定为重复。注意 repeatSubmit: false 是"允许重复"
加密 RSA+AES 混合RSA 加密 AES 密钥（header），AES 加密请求体。每次新随机密钥
模式 ECB vs CBC当前使用 ECB 模式，非最安全但实现简单，配合每次随机密钥可接受
锁 isRelogin.show防止多个 401 响应同时弹窗，模块级单例锁
注意：AES-ECB 模式不隐藏数据模式（相同明文产生相同密文），但配合每次随机密钥可缓解。生产环境建议切 CBC/GCM。防重复提交用 sessionStorage 而非模块变量，可跨路由复用。
