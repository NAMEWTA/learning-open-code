Vite 8 插件链

vite/plugins/index.ts — 8 个插件按序注册 1. @vitejs/plugin-vue → Vue SFC 编译 2. UnoCSS (vite-plugin) → 原子 CSS（构建时按需生成） 3. unplugin-auto-import → 自动导入 Vue API / VueUse / Pinia 4. unplugin-vue-components → 自动导入 Element Plus 组件 5. vite-plugin-compression → Gzip / Brotli 压缩（仅 build） 6. vite-plugin-svg-icons-ng → SVG 精灵图（symbol 模式） 7. vite-plugin-vue-setup-extend → setup 组件 name 扩展 8. vite-plugin-check-transition → 过渡动画检查

i18n 国际化

// 初始化（src/lang/index.ts）
createI18n({
  globalInjection: true,     // 全局 $t()
  legacy: false,             // Composition API 模式
  locale: getLanguage(),     // localStorage → 'zh_CN' | 'en_US'
  messages: { zh_CN, en_US }
});

// 使用方式
模板: {{ $t('login.username') }}
setup: const { t } = useI18n(); t('system.user.addSuccess')

// 与 Element Plus 联动
appStore.locale = computed(() => {
  return { zh_CN: zhCN, en_US: enUS }[language.value]
});
// → el-config-provider :locale="appStore.locale"

关键知识点

知识点说明
loadEnv 过滤只有 VITE_ 前缀的变量暴露给客户端，其他变量仅在 vite.config.ts 中可用
自动导入 dts 文件auto-imports.d.ts 和 components.d.ts 为 IDE 提供类型提示，pnpm dev 生成
UnoCSS 按需构建时扫描代码中使用的工具类，只生成实际用到的 CSS，无需 PurgeCSS
压缩 VITE_BUILD_COMPRESSgzip / brotli / gzip,brotli，压缩产物 .gz / .br 供 nginx 静态服务
svg symbol 模式所有 SVG 合并为一个 sprite，通过 <use> 引用，避免多个 HTTP 请求
注意：.env.development.local 已在 .gitignore 中，适合存储个人开发配置。自动导入的类型文件若缺失会导致 IDE 报错，运行一次 pnpm dev 即可修复。UnoCSS 的 presetAttributify 模式可让你直接在 HTML 属性中使用工具类。
