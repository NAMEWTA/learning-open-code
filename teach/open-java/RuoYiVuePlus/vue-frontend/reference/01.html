速查 01：项目启动与入口架构

启动链路（11 步）

main.ts import 阶段 ├── 1. virtual:uno.css → UnoCSS 原子样式（构建时生成） ├── 2. element-plus dark/css-vars → 暗黑模式 CSS 变量 ├── 3. @/assets/styles/index.scss → 全局 SCSS（theme → layout → components） ├── 4. virtual:svg-icons-register → SVG 精灵图注册 ├── 5. @/lang/index → i18n 实例 ├── 6. @/plugins/svgicon → SvgIcon 全局组件 ├── 7. ./directive → 自定义指令函数 ├── 8. ./plugins/index → $tab/$modal/$cache/$download/$auth ├── 9. ./router + ./store → Router + Pinia 实例 │ createApp(App) ├── 10. app.use(...) 链：HighLight → ElementIcons → store → router → i18n → VxeTablePlugin → plugins ├── 11. directive(app) → app.mount('#app')

核心文件

文件职责
main.ts组装启动链、注册所有插件和指令
App.vueel-config-provider 根上下文 + router-view
src/permission.ts路由守卫（副作用导入，模块即行为）
src/plugins/index.ts5 个全局方法注入 ($tab/$modal/$cache/$download/$auth)
src/directive/index.ts3 个自定义指令注册 (copyText/hasPermi/hasRoles)
src/assets/styles/index.scssSCSS @use 模块链：theme → theme-dark → document → surface → page-shell → table-toolbar

关键知识点

知识点说明
时序 router.beforeEach 早于 app.use(router)import './permission' 时即注册守卫，首次导航在 use(router) 时触发，守卫早已就位
模式 模块即行为import './permission' 的副作用就是注册路由守卫，无需显式调用函数
插件 app.use() 链式注册每个插件通过 install(app) 接收 app 实例，风格扁平
样式 @use 保证单次加载theme-dark 在 theme 之后加载，通过 html.dark 选择器覆盖浅色变量
对比 vs ReactVue 用 app.use() 扁平注册；React 用 JSX Provider 嵌套

代码速查

// 全局方法注册
app.config.globalProperties.$tab = tab;
app.config.globalProperties.$auth = auth;

// 指令注册
app.directive('hasPermi', hasPermi);
app.directive('hasRoles', hasRoles);
app.directive('copyText', copyText);

// App.vue 核心模板
<el-config-provider :locale="appStore.locale" :size="appStore.size">
  <router-view />
</el-config-provider>
要点：import './permission' 的副作用导入在 createApp 前执行，router 实例已带守卫。Plugins 和 directive 注册顺序不影响行为（操作不同注册表）。
