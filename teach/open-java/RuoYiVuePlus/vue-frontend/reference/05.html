速查 05：布局系统与三导航模式

Layout 组件树

Layout (src/layout/index.vue) ├── Sidebar (v-if="showSidebar") — LEFT/MIX 模式显示 │ ├── Logo │ ├── el-scrollbar │ └── el-menu → SidebarItem (递归) │ └── SidebarItem → SidebarItem → ... ├── main-container │ ├── layout-header │ │ ├── Navbar │ │ │ ├── Hamburger (折叠按钮) │ │ │ ├── Breadcrumb │ │ │ └── TopBar (TOP 模式时显示) │ │ └── TagsView (v-if="needTagsView") │ └── AppMain → <router-view> (keep-alive + transition) └── Settings (右侧设置抽屉)

三模式对比

模式NavTypeEnumSidebarTopBarmain-container
左侧导航LEFT显示，固定左侧隐藏margin-left 留空
顶部导航TOP隐藏显示，水平排列占满宽
混合导航MIX显示，固定左侧显示，水平排列margin-left + 顶部菜单

SidebarItem 递归渲染逻辑

// 三种渲染分支
1. hasOneShowingChild(item) && !item.alwaysShow
   → 直接渲染 <el-menu-item>（跳过只有1个子项的父级）
2. 多个子路由
   → 渲染 <el-sub-menu> + 递归 <sidebar-item>
3. item.hidden === true
   → 不渲染（v-if 跳过）

TopBar 自适应溢出

// 可见数量自适应屏幕宽度

响应式与 KeepAlive

// 桌面/移动端断点: 992px (useWindowSize)
width < 992 → toggleDevice('mobile') + closeSideBar()
width >= 992 → toggleDevice('desktop')

// AppMain.vue 缓存控制
<keep-alive :include="cachedViews">
  <component :is="Component" :key="route.path" />
</keep-alive>
// cachedViews = tagsViewStore.cachedViews (组件 name 列表)
// createCustomNameComponent 保证异步组件有 name

关键知识点

知识点说明
条件渲染 v-if showSidebarTOP 模式伪侧边栏，DOM 中无 .sidebar-container
递归 SidebarItem 自引用组件内部 import 自身，通过 v-for children 传递 item prop
响应式 useWindowSizeVueUse 提供，992px 断点，移动端侧边栏 overlay + 遮罩
缓存 keep-alive include动态数组，只有 tagsView 中的页面被缓存
key route.path确保同一路由不同参数时组件正确重建（而非 path 不变复用）
注意：移动端侧边栏通过绝对定位覆盖内容区，点击遮罩层关闭。keep-alive 的 key 用 route.path 而非 route.name，因为 name 可能被 createCustomNameComponent 包装后不一致。
