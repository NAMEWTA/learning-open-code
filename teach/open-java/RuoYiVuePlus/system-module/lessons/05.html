📋 第5课：动态菜单与前端路由 — SysMenu → RouterVo 的魔法

胜利目标：能完整解释 sys_menu 表中的一行数据如何变成前端 Vue Router 的一段配置，理解三种 menu_type 的路由生成差异。

1. 核心思路：数据库驱动的前端路由

前端不硬编码路由，而是登录后从后端 GET /system/menu/getRouters 获取当前用户可访问的菜单树，动态生成 Vue Router 配置。这样权限变更无需重新部署前端。

2. 三种菜单类型的路由生成

menu_type含义前端表现component 取值
M（目录）侧边栏的折叠分组无实际页面，仅做子路由容器Layout
C（菜单）实际的页面入口点击后加载对应组件system/user/index
F（按钮）页面内的操作按钮不生成路由，仅做权限校验（不生成路由）

3. 完整转换链路


【API 请求】

    │

    │  遍历 SysMenu 树，逐节点转换为 RouterVo
    │  ├── getRouteName()    → RouterVo.name

    │  ├── getComponentInfo() → RouterVo.component
    │  └── 构建 MetaVo       → RouterVo.meta
    │
    ▼
返回 List<RouterVo> → 前端 Vue Router

4. SysMenu 的三个关键计算方法

4.1 getRouteName() — 路由名称

public String getRouteName() {
    String routerName = StringUtils.capitalize(path);  // system/user → SystemUser
    if (isMenuFrame()) {
        routerName = "";  // 顶级菜单如果是内嵌框架，路由名为空
    }
    return routerName;  // 最终 name = "SystemUser" + menuId
}

4.2 getRouterPath() — 路由地址

public String getRouterPath() {
    String routerPath = this.path;
    // 1. 非顶级 + 内链 → 替换域名特殊字符
    if (!isTopParent() && isInnerLink()) {
        routerPath = innerLinkReplaceEach(routerPath);  // http://xxx → /xxx
    }
    // 2. 顶级目录 → 加 / 前缀
    if (isTopParent() && TYPE_DIR.equals(menuType) && NO.equals(isFrame)) {
        routerPath = "/" + this.path;
    }
    // 3. 顶级菜单（内嵌框架模式）→ 路由设为 /
    else if (isMenuFrame()) {
        routerPath = "/";
    }
    return routerPath;
}

4.3 getComponentInfo() — 组件路径

public String getComponentInfo() {
    String component = "Layout";  // 默认使用布局组件
    // 1. 有自定义组件 + 非内嵌框架 → 使用自定义组件
    if (StringUtils.isNotEmpty(this.component) && !isMenuFrame()) {
        component = this.component;  // system/user/index
    }
    // 2. 无组件 + 非顶级 + 内链 → InnerLink
    else if (StringUtils.isEmpty(this.component) && !isTopParent() && isInnerLink()) {
        component = "InnerLink";
    }
    // 3. 无组件 + 父视图 → ParentView
    else if (StringUtils.isEmpty(this.component) && isParentView()) {
        component = "ParentView";
    }
    return component;
}

5. buildMenus() — 递归转换为 RouterVo 树

public List<RouterVo> buildMenus(List<SysMenu> menus) {
    List<RouterVo> routers = new LinkedList<>();
    for (SysMenu menu : menus) {
        RouterVo router = new RouterVo();
        router.setHidden("1".equals(menu.getVisible()));
        router.setName(menu.getRouteName() + menu.getMenuId());
        router.setPath(menu.getRouterPath());
        router.setComponent(menu.getComponentInfo());
        router.setMeta(new MetaVo(
            menu.getMenuName(), menu.getIcon(),
            !menu.getIsCache(),      // 是否缓存（取反）
            menu.getPath(),
            menu.getActiveMenu()
        ));

        // 🔑 四种子路由处理策略
        if (hasChildren && TYPE_DIR.equals(menu.getMenuType())) {
            // 策略1: 目录+有子菜单 → 嵌套路由，递归构建 children
            router.setAlwaysShow(true);
            router.setRedirect("noRedirect");
            router.setChildren(buildMenus(children));
        } else if (menu.isMenuFrame()) {
            // 策略2: 顶级菜单+内嵌框架 → 创建包装路由
            router.setMeta(null);
            // 创建子路由指向实际页面
            router.setChildren([...]);
        } else if (menu.getParentId().equals(TOP) && menu.isInnerLink()) {
            // 策略3: 顶级内链 → 特殊路由处理
            router.setPath("/");
            router.setChildren([内链组件]);
        }
        // 策略4: 其他情况（如按钮类型F）→ 不生成路由
    }
    return routers;
}
四种路由构建策略总结：
  1. 目录 + 子菜单 → Layout 嵌套路由（标准多级菜单）
  2. 菜单（内嵌框架） → 包装路由指向 iframe 页面
  3. 内链 → 特殊处理域名转路径，生成 InnerLink 组件
  4. 按钮 → 跳过，不生成路由
