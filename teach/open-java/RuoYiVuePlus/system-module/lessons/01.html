1. ruoyi-system 在项目中的位置

RuoYi-Vue-Plus 采用 多模块微服务架构，ruoyi-system 属于 ruoyi-modules 下的业务模块之一：

RuoYi-Vue-Plus/
├── ruoyi-common/          # 公共模块（工具、安全、缓存、日志等）
│   ├── ruoyi-common-core       # 核心工具、常量、异常
│   ├── ruoyi-common-security   # Sa-Token 安全配置
│   ├── ruoyi-common-mybatis     # MyBatis-Plus 封装
│   ├── ruoyi-common-redis       # Redis 工具
│   ├── ruoyi-common-log         # 操作日志注解
│   ├── ruoyi-common-excel       # Excel 导入导出
│   └── ...
├── ruoyi-api/             # 远程调用 API 接口定义
├── ruoyi-modules/         # 业务模块（微服务）
│   ├── ruoyi-system       # 👈 系统管理模块（本课程焦点）
│   ├── ruoyi-gen          # 代码生成模块
│   ├── ruoyi-job          # 定时任务模块
│   └── ruoyi-file         # 文件服务模块
└── ruoyi-gateway/         # 网关模块
关键认知：ruoyi-system 是整个平台的管理中枢。用户、角色、菜单、部门、权限、配置、日志——所有后台管理能力都在这里。

2. 分层架构

ruoyi-system 遵循经典的 Controller → Service → Mapper 三层架构，同时引入了领域驱动设计（DDD）的轻量实践：

ruoyi-system/src/main/java/org/dromara/system/
│
├── controller/            # 控制器层（REST API 入口）
│   ├── monitor/           # 监控类接口：缓存、操作日志、登录日志、在线用户
│   └── system/            # 系统管理接口：用户、角色、菜单、部门、岗位、配置等
│
├── service/               # 服务接口层
│   ├── ISysUserService.java
│   ├── ISysRoleService.java
│   ├── ISysMenuService.java
│   ├── ISysDeptService.java
│   ├── ISysPermissionService.java
│   ├── ISysDataScopeService.java
│   └── ...
│   └── impl/              # 服务实现层
│       ├── SysUserServiceImpl.java
│       ├── SysRoleServiceImpl.java
│       ├── SysMenuServiceImpl.java
│       └── ...
│
├── mapper/                # 数据访问层（MyBatis-Plus Mapper 接口）
│   ├── SysUserMapper.java
│   ├── SysRoleMapper.java
│   └── ...
│
├── domain/                # 领域对象
│   ├── SysUser.java       # 实体（Entity）
│   ├── SysRole.java
│   ├── bo/                # 业务对象（Business Object = 入参 DTO）
│   │   ├── SysUserBo.java
│   │   └── ...
│   └── vo/                # 视图对象（View Object = 出参 DTO）
│       ├── SysUserVo.java
│       └── ...
│
├── event/                 # Spring 事件定义
│   ├── OnlineUserCleanEvent.java
│   └── OssConfigChangeEvent.java
│
├── listener/              # 事件监听器 & Excel 导入辅助
│   ├── OnlineUserCleanListener.java
│   ├── OssConfigChangeListener.java
│   └── SysUserImportListener.java
│
└── runner/                # 应用启动后初始化
    └── SystemApplicationRunner.java

3. 模块依赖全景

从 pom.xml 看 ruoyi-system 的依赖关系：

依赖用途
ruoyi-common-core核心工具类、常量定义、基础异常
ruoyi-api远程调用接口定义（如 UserService、RoleService）
ruoyi-common-doc接口文档（Knife4j/Swagger）
ruoyi-common-mybatisMyBatis-Plus 封装、分页、数据权限插件
ruoyi-common-translation数据翻译（如 ID → 名称）
ruoyi-common-oss对象存储（头像、文件上传）
ruoyi-common-log操作日志注解 @Log
ruoyi-common-excelExcel 导入导出
ruoyi-common-sms短信发送
ruoyi-common-securitySa-Token 认证授权
ruoyi-common-webWeb 层基础配置
ruoyi-common-sensitive数据脱敏
ruoyi-common-encrypt数据库字段加解密
ruoyi-common-push消息推送
设计亮点：ruoyi-system 通过 ruoyi-api 模块暴露远程调用接口，其他微服务可以通过 Feign/Dubbo 调用 UserService、RoleService 等，实现跨服务的用户/角色/部门数据查询。

4. 数据流全景

一个典型的请求在 ruoyi-system 中的完整链路：


HTTP Request
    │
    ▼
┌──────────────────────────────────────┐
│  Gateway (ruoyi-gateway)             │  ← 路由转发、鉴权
└──────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────┐
│  Controller                          │  ← @SaCheckPermission 权限校验
│  SysUserController.list()            │     @Log 日志记录
└──────────────────────────────────────┘     @RepeatSubmit 防重复
    │
    ▼
┌──────────────────────────────────────┐
│  Service                             │  ← 业务逻辑、唯一性校验
│  SysUserServiceImpl.selectPage()     │     @Transactional 事务管理
└──────────────────────────────────────┘     @Cacheable/@CacheEvict 缓存
    │
    ▼
┌──────────────────────────────────────┐
│  Mapper                              │  ← MyBatis-Plus 查询
│  SysUserMapper.selectPageUserList()  │     数据权限插件自动注入 SQL
└──────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────┐
│  MySQL + Redis                       │  ← 数据存储 + 缓存
└──────────────────────────────────────┘

5. 核心设计模式速览

模式体现位置说明
RBACSysUser → SysRole → SysMenu用户通过角色获得菜单/按钮权限
数据权限拦截器common-mybatis 插件 + SysDataScopeService根据角色 dataScope 自动拼接 SQL WHERE 条件
事件驱动OnlineUserCleanEvent → OnlineUserCleanListener角色变更后踢出相关在线用户
静态代理（AOP）@Cacheable 通过 AopProxy 调用解决同类方法调用不走代理的问题
模板方法BaseEntity（createBy, createTime, updateBy, updateTime）所有实体统一继承，自动填充审计字段
策略模式DataScope 5 种数据范围不同 dataScope 值对应不同的 SQL 过滤策略
