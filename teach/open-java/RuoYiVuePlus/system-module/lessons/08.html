🎛️ 第8课：Controller 设计规范 — 注解驱动的横切关注点

胜利目标：能独立写出一个符合 ruoyi 规范的 Controller 方法，正确使用所有横切注解。

1. Controller 注解全景

以一个典型的用户修改方法为例，解码所有注解的职责：

@Validated                          // ① 开启方法参数校验
@RequiredArgsConstructor             // ② Lombok：生成 final 字段的构造函数注入
@RestController                      // ③ REST 控制器 = @Controller + @ResponseBody
@RequestMapping("/system/user")      // ④ 统一 URL 前缀
public class SysUserController extends BaseController {  // ⑤ 继承基类

    /**
     * 修改用户。
     */
    @SaCheckPermission("system:user:edit")  // ⑥ 功能权限校验
    @Log(title = "用户管理",                  // ⑦ 操作日志
         businessType = BusinessType.UPDATE)
    @RepeatSubmit()                          // ⑧ 防重复提交
    @PutMapping                              // ⑨ RESTful PUT 方法
    public R<Void> edit(
        @Validated                            // ⑩ 参数校验
        @RequestBody SysUserBo user) {        // ⑪ 请求体自动绑定

        userService.checkUserAllowed(user.getUserId());    // ⑫ 业务校验
        userService.checkUserDataScope(user.getUserId());  // ⑬ 数据权限校验
        deptService.checkDeptDataScope(user.getDeptId());

        // ⑭ 业务唯一性校验
        if (!userService.checkUserNameUnique(user)) {
            return R.fail("修改用户失败，登录账号已存在");
        }

        return toAjax(userService.updateUser(user));  // ⑮ 操作结果转 R
    }
}

2. 每个注解的深度解析

⑥ @SaCheckPermission — 功能权限

// 单一权限 — 拥有 system:user:list 即可
@SaCheckPermission("system:user:list")

// 多权限 OR — 拥有任一即可
@SaCheckPermission(value = {"system:user:add", "system:user:edit"}, mode = SaMode.OR)

// 多权限 AND — 必须全部拥有
@SaCheckPermission(value = {"system:user:add", "system:user:edit"}, mode = SaMode.AND)

⑥ @SaCheckRole — 角色校验

// 菜单管理只允许超管操作
@SaCheckRole(SystemConstants.SUPER_ADMIN_ROLE_KEY)  // "super_admin"

// 多角色 OR
@SaCheckRole(value = {"super_admin", "common"}, mode = SaMode.OR)
权限 vs 角色的使用区分：
  • @SaCheckPermission — 细粒度控制（增删改查导出导入），大多数方法使用
  • @SaCheckRole — 粗粒度控制（只允许超管），仅菜单管理使用

⑦ @Log — 操作日志

@Log(title = "用户管理", businessType = BusinessType.UPDATE)
// businessType 枚举值：
//   INSERT(新增), UPDATE(修改), DELETE(删除),
//   EXPORT(导出), IMPORT(导入), GRANT(授权),
//   OTHER(其他), CLEAN(清理)

// 实现机制：AOP 切面拦截，自动记录：
//   - 操作人、操作时间
//   - 请求 URL、方法名、参数
//   - 返回结果、耗时
//   → 写入 sys_oper_log 表

⑧ @RepeatSubmit — 防重复提交

@RepeatSubmit()  // 默认间隔 5000ms，基于 Redis 分布式锁
// 原理：将 "URL + 参数 + 用户" 的 hash 作为 Redis key
// 如果在间隔时间内重复请求，返回"不允许重复提交"错误
// 只用于写操作（POST/PUT/DELETE），读操作不需要

⑮ BaseController.toAjax() — 结果转换

// BaseController 提供的工具方法
protected R<Void> toAjax(int rows) {
    return rows > 0 ? R.ok() : R.fail("操作失败");
}
// 将 Service 返回的影响行数转为统一响应格式

3. Controller 中常见的安全校验模式

// 模式1：不允许操作超管
userService.checkUserAllowed(user.getUserId());
// → if (userId == 1) throw ServiceException("不允许操作超级管理员用户")

// 模式2：不允许删除自己
if (ArrayUtil.contains(userIds, LoginHelper.getUserId())) {
    return R.fail("当前用户不能删除");
}

// 模式3：数据权限校验
userService.checkUserDataScope(userId);
// → 通过 SQL 查询验证当前用户是否有权限看到目标用户

// 模式4：唯一性校验
if (!userService.checkUserNameUnique(user)) {
    return R.fail("登录账号已存在");
}

4. 特殊的注解组合

密码加密：@ApiEncrypt

@ApiEncrypt  // 标记接口需要加解密处理
@PutMapping("/resetPwd")
public R<Void> resetPwd(@RequestBody SysUserBo user) {
    // 前端传来的密码是加密的，通过 @ApiEncrypt 切面自动解密
    // 然后再 BCrypt 哈希存入数据库
    user.setPassword(BCrypt.hashpw(user.getPassword()));
    return toAjax(userService.resetUserPwd(user.getUserId(), user.getPassword()));
}

Excel 导入：@RequestPart

@PostMapping(value = "/importData", consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
public R<Void> importData(
    @RequestPart("file") MultipartFile file,   // 文件上传
    boolean updateSupport                       // 是否覆盖已有数据
) throws Exception {
    ExcelResult<SysUserImportVo> result = ExcelBuilder
        .read(file.getInputStream(), SysUserImportVo.class)
        .listener(new SysUserImportListener(updateSupport))  // 自定义监听器
        .doRead();
    return R.ok(result.getAnalysis());
}

忽略数据权限：DataPermissionHelper.ignore()

// 个人中心查看自己的信息时，不应该受数据权限限制
SysUserVo user = DataPermissionHelper.ignore(() ->
    userService.selectUserById(LoginHelper.getUserId())
);

5. Controller 代码规范清单

规范项要求示例
类注解@Validated + @RequiredArgsConstructor + @RestController + @RequestMapping标准模板
依赖注入private final + @RequiredArgsConstructor（构造器注入）不用 @Autowired
读方法只加 @SaCheckPermission，不加 @Log/@RepeatSubmitlist(), getInfo()
写方法@SaCheckPermission + @Log + @RepeatSubmit（完整三件套）add(), edit(), remove()
返回类型统一返回 R<T>R<Void>, R<PageResult>, R<List>
参数校验方法参数加 @Validated 触发 JSR303 校验配合 Bo 中的 @NotBlank 等注解
安全校验顺序注解校验 → 业务校验 → 操作不可跳过任何一环
