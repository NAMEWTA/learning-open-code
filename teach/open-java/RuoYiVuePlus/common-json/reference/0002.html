BigNumberSerializer 精度保护

要点说明
继承NumberSerializer（Jackson 内建数字序列化器）
安全范围[-9007199254740991, 9007199254740991]（JS Number.MAX/MIN_SAFE_INTEGER）
判断逻辑value.longValue() 在范围内 → 数字输出；范围外 → writeString() 字符串输出
注册类型Long.class、Long.TYPE（基本类型）、BigInteger.class
BigDecimal 不同用 ToStringSerializer（小数不需要整数精度保护）
注解@JacksonStdImpl — Jackson 内部优化标记

两个反序列化器对比

CustomDateDeserializerCustomLocalDateTimeDeserializer
返回类型java.util.Datejava.time.LocalDateTime
转换方法parse.toJdkDate()parse.toLocalDateTime()
解析委托Hutool DateUtil.parse(text.trim())
空值处理if (null || isBlank()) return null
支持格式yyyy-MM-dd HH:mm:ss、yyyy/MM/dd、时间戳、ISO 等（Hutool 自动识别）

核心代码片段

// BigNumberSerializer 核心判断
if (value.longValue() >= MIN_SAFE_INTEGER
    && value.longValue() <= MAX_SAFE_INTEGER) {
    super.serialize(value, gen, provider);  // 数字
} else {
    gen.writeString(value.toString());       // 字符串
}

// 反序列化器核心（两个类完全相同，仅返回类型不同）
String text = p.getString();
if (text == null || text.isBlank()) return null;
DateTime parse = DateUtil.parse(text.trim());
return parse.toJdkDate();          // Date 版
return parse.toLocalDateTime();    // LocalDateTime 版

序列化端 LocalDateTime 输出格式

JacksonConfig 注册：LocalDateTimeSerializer(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"))
保证输出固定格式，而非 Jackson 默认数组格式 [2026,7,5,14,30,0]

设计策略

  • 输入端宽容：反序列化用 Hutool 多格式自动识别
  • 输出端严格：序列化用固定格式 + 精度保护
  • 薄适配器模式：两个反序列化器共享同一套解析能力（Hutool），各自只做一层薄适配

→ 术语：BigNumberSerializer、CustomDateDeserializer、CustomLocalDateTimeDeserializer（见 GLOSSARY.md）
