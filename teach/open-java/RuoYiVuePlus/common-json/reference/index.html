ruoyi-common-json 速查表

Maven: org.dromara:ruoyi-common-json | ruoyi-common-json/src/main/java/org/dromara/common/json/

定位

基于 Jackson 3.x (tools.jackson) 的开箱即用序列化模块，提供三驾马车：JacksonConfig（序列化配置中心）、JsonEnhancementConfig（字段增强引擎）、JsonUtils（业务工具门面），以及 @JsonPattern 校验注解。

术语速查

术语解释
Jackson (tools.jackson)Maven Shade 重打包的 Jackson 3.x，避免版本冲突，功能与 FasterXML 一致
JsonMapperJackson 3.x 入口类，不可变，通过 JsonMapper.builder() 构建，替代 ObjectMapper
SimpleModuleJackson 模块化扩展，注册自定义 Serializer/Deserializer 的载体
BigNumberSerializerLong 超 JS 安全整数范围（±9007199254740991）时转 string，范围内保持 number
三阶段生命周期collect（收集字段）→ prepare（批量 IO 消 N+1）→ process（渲染替换），JsonEnhancementContext 作数据总线
JsonFieldProcessor字段处理器接口，supports() 判定 + 三阶段 default 方法，实现者按需覆盖
JsonValueEnhancer增强引擎，递归扫描对象树 → IdentityHashMap 防循环引用 → 处理后二次增强
@JsonPatternJakarta Bean Validation 注解，校验字符串是否合法 JSON，type 选 OBJECT/ARRAY/ANY
JsonTypeJSON 类型枚举：OBJECT（{}）、ARRAY（[]）、ANY（任意），驱动校验策略

常用代码

序列化

// 对象 → JSON 字符串
String json = JsonUtils.toJsonString(user);

// 序列化时排除敏感字段
String safe = JsonUtils.toJsonStringExcludeFields(user, "password", "salt");

反序列化

// JSON → 对象
User user = JsonUtils.parseObject(jsonStr, User.class);

// JSON → 带泛型的对象
List<User> users = JsonUtils.parseArray(jsonStr, User.class);

// JSON → Map
Map<String, Object> map = JsonUtils.parseMap(jsonStr);

// JSON → List<Map>
List<Map<String, Object>> list = JsonUtils.parseArrayMap(jsonStr);

JSON 校验

// 业务代码中判断字符串是否为合法 JSON
boolean valid = JsonUtils.isJson(input);
boolean isObj = JsonUtils.isJsonObject(input);
boolean isArr = JsonUtils.isJsonArray(input);

// Bean Validation 注解方式
public class ConfigDTO {
    @NotBlank
    @JsonPattern(type = JsonType.OBJECT)  // 必须是 JSON 对象
    private String extraConfig;
}

JSON 字段动态删除

// 递归删除 JSON 中的指定字段（适合日志脱敏）
String cleaned = JsonUtils.removeFields(jsonStr, "password", "phone");
