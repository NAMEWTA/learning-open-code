@JsonPattern 校验链路

@JsonPattern(type=JsonType.XXX) → JsonPatternValidator.initialize(annotation) 读 type → isValid(value, ctx): 空值 → 返回 true（交给 @NotBlank） 非空 → switch(jsonType): ANY → JsonUtils.isJson(value) OBJECT → JsonUtils.isJsonObject(value) ARRAY → JsonUtils.isJsonArray(value)

JsonType 枚举

值允许的格式示例
OBJECTJSON 对象{"a":1}
ARRAYJSON 数组[1,2,3]
ANY（默认）对象或数组上面两种都行

使用示例

// 只允许 JSON 对象
@JsonPattern(type = JsonType.OBJECT, message = "必须是 JSON 对象")
@NotBlank
private String config;

// 只允许 JSON 数组
@JsonPattern(type = JsonType.ARRAY, message = "必须是 JSON 数组")
private String tags;

// 任意 JSON 类型（默认）
@JsonPattern
private String extraData;

底层：readTreeQuietly 模式

private static JsonNode readTreeQuietly(String str) {
    if (StringUtils.isBlank(str)) return null;
    try { return JSON_MAPPER.readTree(str); }
    catch (Exception e) { return null; }
}
// 异常 → null，调用方只需判断 null，不必 try-catch

#规则
1永远用 JsonUtils 而非 new ObjectMapper()
2@JsonPattern 配合 @NotBlank 使用
3Processor 的 attribute key 用全限定类名前缀
4泛型反序列化用 TypeReference 而非 Class
5toJsonStringExcludeFields 是递归删除，注意嵌套同名字段
6process() 中不做 IO，IO 放 prepare()
7@JsonPattern 只校验格式，不校验内容语义
