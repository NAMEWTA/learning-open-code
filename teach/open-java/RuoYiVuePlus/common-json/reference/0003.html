三阶段生命周期

阶段调用频率职责典型操作
collect()每个字段一次收集需要处理的 key从 fieldContext 读值 → 存到 context.attributes
prepare()每次响应一次批量预处理一次性查数据库/Redis → 结果写入 context
process()每个字段一次返回替换值从 context 取 prepare 的结果 → 返回新值
关键设计意图：消除 N+1 查询
如果 process 里查库 → 100 个字段 = 100 次查询
collect 收集 → prepare 批量查 1 次 → process 从内存取 = 1 次查询

四个类职责

类角色关键设计点
JsonEnhancementContext数据总线（黑板模式）LinkedHashMap attributes 共享数据、processingRequired 短路标记
JsonFieldContext字段快照（Java Record）不可变、getAnnotation() 读取字段注解驱动 supports()
JsonFieldProcessor扩展点（接口）4 个 default 方法，实现者只覆盖关心的阶段
JsonValueEnhancer引擎（下节课详讲）驱动三阶段、递归遍历、属性缓存

JsonFieldProcessor 接口合约

public interface JsonFieldProcessor {
    default boolean supports(JsonFieldContext fc) { return true; }
    default void collect(JsonFieldContext fc, JsonEnhancementContext ctx) {}
    default void prepare(JsonEnhancementContext ctx) {}
    default Object process(JsonFieldContext fc, Object value,
                           JsonEnhancementContext ctx) { return value; }
}

JsonEnhancementContext 关键方法

getAttribute(key)               // 读取上下文属性
getOrCreateAttribute(key, sup)  // 惰性创建（computeIfAbsent）
setAttribute(key, value)        // 写入上下文属性
markProcessingRequired()        // 标记需要处理（短路优化）
isProcessingRequired()          // 是否需要增强
containsAttribute(key) / removeAttribute(key)

写 Processor 的 attribute key 命名约定

使用 全限定类名作为前缀 以避免多 Processor 冲突
例："com.example.DeptProcessor:collect:keys" 而非 "keys"

易混点

  • supports() 是字段-Processor 级别过滤（每个字段每个 Processor 独立判断）
  • processingRequired 是请求级别标记（整个响应只要有一个字段需要增强就为 true）
  • process() 返回原 value = 不修改；返回 null = 将字段值置为 null

→ 术语：JsonEnhancementContext、JsonFieldContext、JsonFieldProcessor、三阶段生命周期（见 GLOSSARY.md）
