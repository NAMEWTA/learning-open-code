enhance() 主流程

1. 快速短路：null / JsonNode / 无处理器 → 原样返回 2. new JsonEnhancementContext → collectValue(body, ctx, visited) 3. 检查 processingRequired == false？ → 原样返回（短路优化） 4. processors.forEach(p → p.prepare(ctx)) ← 批量预处理 5. return renderValue(body, ctx, visited) ← 构建增强 JSON 树

递归遍历：collectValue / renderValue 的类型分发

类型collect 行为render 行为
null返回返回 nullNode()
Map遍历 values 递归创建 ObjectNode，遍历 entry 递归
IterableforEach 递归创建 ArrayNode，forEach 递归
数组Array.get() 循环递归Array.get() 循环递归
简单值返回（跳过）valueToTree(value)
POJO遍历属性 → collectField → 递归遍历属性 → process 处理链 → renderValue

关键设计点

循环引用保护

IdentityHashMap<Object, Boolean> visited  // 引用相等（==）判断
.put(value, TRUE) != null → 已访问 → 跳过
finally { visited.remove(value) }          // 退栈清理，允许不同路径重新访问
为什么用 IdentityHashMap 而非 HashSet？循环引用关心「同一个内存对象」而非 equals 相等

属性元数据缓存

ConcurrentHashMap<Class<?>, List<PropertyMetadata>> propertyCache
// 每个 Class 的 PropertyMetadata 只解析一次，线程安全

属性解析（resolveProperties）

使用 Jackson ClassIntrospector（而非纯反射）→ 尊重 @JsonProperty/@JsonIgnore 等注解
优先 getter，回退 field → member.fixAccess(true) → 强制可访问

简单值判定（isSimpleValue）

基本类型 / CharSequence / Number / Boolean / Character
/ Date / Temporal / Enum / UUID / Class
→ 不递归遍历，直接序列化

二次增强（enhanceTranslatedValue）

process() 返回的复杂对象 → 重新走 collect-prepare-process 流程
确保「替换后对象中的嵌套字段」也能被增强

转换器排除（supports(converterType)）

排除 ByteArrayHttpMessageConverter（文件下载-二进制）
排除 StringHttpMessageConverter（已经是字符串）
排除 ResourceHttpMessageConverter（静态资源）
→ 正确性保障，防止增强框架遍历非对象数据

处理链 process 顺序

processors 按 @Order / Ordered 排序（AnnotationAwareOrderComparator.sort）
→ 对同一字段，多个 Processor 按排序顺序依次处理

完整调用链路

Controller 返回 → ResponseBodyAdvice.beforeBodyWrite() → jsonValueEnhancer.enhance(body) → collect(collectValue 递归) → prepare(批量) → render(renderValue 递归) → Spring MVC 用 Jackson 写出增强后的 JsonNode 树

→ 术语：JsonValueEnhancer（见 GLOSSARY.md）
