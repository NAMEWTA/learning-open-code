# Teaching Notes — ruoyi-common-satoken

- 学习者背景：已学完 ruoyi-common-redis 和 ruoyi-common-sms，对 Redis 操作和模块自动装配模式有基础认知。本模块天然对比 PlusSmsDao 的双层缓存设计。
- 课程顺序设计：先鸟瞰（全景）→ 配置与 DAO（基础设施）→ 权限验证（核心逻辑）→ 异常处理与工具（边界与易用性）→ 实战（链路串联），从静态到动态、从组件到流程。
- 对比式教学：利用学习者已有的 sms 模块知识，在 DAO 设计、配置模式、异常处理三处做横向对比，降低新知识的认知负担。
- 代码优先：每节课的核心概念必须有源码行号引用，避免空谈设计理念。"为什么这样写"比"写的是什么"更重要。
- 练习设计：选择题后跟解析，不只说"选C"，而是解释为什么A/B/D错误。最后一课的实战题要能独立完成链路追踪。
- 不追求大而全：聚焦 5 个 Java 文件 + 1 个 yml 配置，不扩展到 Sa-Token 的注解鉴权、OAuth2 等周边特性。
