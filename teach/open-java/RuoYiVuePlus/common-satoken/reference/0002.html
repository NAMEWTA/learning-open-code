common-satoken.yml 全量配置

sa-token:
  # token 名称（同时也是 cookie 名称）
  token-name: Authorization
  # token 前缀，与 token-name 连用
  token-prefix: Bearer
  # token 有效期（单位：秒），默认30天，-1 代表永久有效
  timeout: 86400
  # token 最低活跃频率（单位：秒），超时无操作则冻结
  activity-timeout: 1800
  # 是否允许同一账号多地同时登录（true=允许, false=不允许）
  is-concurrent: true
  # 是否共用一个 token
  is-share: true
  # 是否输出操作日志
  is-log: false
  # token 风格（uuid/simple-uuid/random-32/random-64/random-128/tik）
  token-style: simple-uuid
  # 是否在初始化配置时打印版本字符画
  is-print: false
  # jwt 秘钥（使用 jwt 时必填）
  jwt-secret-key: abcdefghijklmnopqrstuvwxyz

Caffeine 缓存配置

Cache<String, Object> cache = Caffeine.newBuilder()
    .maximumSize(10_000)            // 最多缓存 10,000 条
    .expireAfterWrite(30, TimeUnit.SECONDS)  // 写入后 30 秒过期
    .recordStats()                  // 开启统计（命中率等）
    .build();

读写时序

读路径（READ）: 请求 → PlusSaTokenDao.get(sessionKey) ├─ cache.getIfPresent(sessionKey) → 命中? 直接返回 └─ 未命中 → redisTemplate.opsForValue().get(sessionKey) ├─ Redis null? → 返回 null └─ Redis 有值 → cache.put(sessionKey, value) → 返回值 写路径（WRITE）: 请求 → PlusSaTokenDao.set(sessionKey, value, timeout) ├─ redisTemplate.opsForValue().set(sessionKey, value, timeout, TimeUnit.SECONDS) └─ cache.put(sessionKey, value)

维度 PlusSmsDao PlusSaTokenDao
