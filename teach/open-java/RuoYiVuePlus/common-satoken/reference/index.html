定位

基于 Sa-Token 的认证鉴权模块，集成 JWT 简单模式 + Redis 持久化 + Caffeine 两级缓存 的生产级加固方案。替代传统 Spring Security + OAuth2，提供登录认证、权限鉴权、Token 管理、踢人下线、账号封禁等能力。

核心文件

类名路径职责
SaTokenConfigconfig/SaTokenConfig.java自动配置入口，注册 StpLogic/StpInterface/SaTokenDao/防火墙等 Bean
PlusSaTokenDaodao/PlusSaTokenDao.java数据持久层，Redis + Caffeine 两级缓存替代默认内存存储，支持分布式部署
SaPermissionImplservice/SaPermissionImpl.java实现 StpInterface，getPermissionList/getRoleList 从 LoginUser 提取权限码和角色
LoginHelperhelper/LoginHelper.java登录工具类，封装 login/logout/getLoginUser/getUserId 等常用操作
SaTokenExceptionHandlerhandler/SaTokenExceptionHandler.java全局异常处理，区分 NotLoginException 子类型返回中文提示
LoginUserdomain/LoginUser.java登录用户信息对象，含 userId/userName/deptId/角色列表/权限列表
StpLogicJwtForSimple（sa-token-jwt 插件）JWT 简单模式 StpLogic 实现，Token 变为可跨语言解析的 JWT 字符串

术语速查

术语解释
Sa-TokenJava 轻量级权限认证框架，提供登录认证、权限鉴权、Token 管理、踢人下线等
JWT自包含 Token 格式，Token 自身携带基础用户数据，服务端无需查库即可验证
StpLogic认证逻辑核心类，掌管登录/退出/Token 创建/会话管理等所有底层实现
StpInterface权限扩展接口，getPermissionList/getRoleList 在每次鉴权时被回调
SaTokenDao数据持久层接口，本模块用 PlusSaTokenDao 替换为 Redis+Caffeine 两级缓存
StpLogicJwtForSimpleJWT 简单模式实现，Token 变为 Base64 JWT，可被任何语言解析
Caffeine高性能本地缓存库，在 PlusSaTokenDao 中缓存 Token/Session，降低 Redis QPS 压力
Token Session与 Token 绑定的会话对象，同一账号多设备登录时各自独立，存储 LoginUser 等数据
NotLoginException未登录异常，子类型：TOKEN_TIMEOUT/BE_REPLACED/KICK_OUT/TOKEN_FREEZE 等
Token 前缀Bearer 前缀，HTTP 头 Authorization: Bearer xxxxx，Sa-Token 读取时自动剥离

常用代码

登录与获取用户信息：

// 登录
LoginHelper.login(userId, username, loginUser);

// 获取当前登录用户
LoginUser loginUser = LoginHelper.getLoginUser();
Long userId = LoginHelper.getUserId();
String username = LoginHelper.getUsername();

// 退出登录
LoginHelper.logout();

权限校验（注解方式）：

// 角色校验
@SaCheckRole("admin")
@GetMapping("/admin/dashboard")
public Result dashboard() { ... }

// 权限码校验
@SaCheckPermission("system:user:list")
@GetMapping("/system/user/list")
public Result userList() { ... }

// 角色或权限（满足其一即可）
@SaCheckPermission(value = "system:user:add", orRole = "admin")
@PostMapping("/system/user")
public Result addUser() { ... }

手动权限校验：

// 判断是否登录
if (StpUtil.isLogin()) { ... }

// 检查角色
StpUtil.checkRole("admin");

// 检查权限
StpUtil.checkPermission("system:user:delete");

// 踢人下线
StpUtil.kickout(loginId);

Token 配置（common-satoken.yml）：

sa-token:
  token-name: Authorization
  token-prefix: "Bearer"
  timeout: 86400              # Token 有效期（秒）
  active-timeout: 1800        # 临时有效期内无操作则过期
  is-concurrent: true         # 允许多设备同时登录
  is-share: false             # 多设备不共享 Token
  is-log: true                # 开启日志
