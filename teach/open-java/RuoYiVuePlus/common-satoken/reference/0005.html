速查 05 · 完整链路与模式切换

配套第 5 课学习

登录完整调用链

POST /login → SysLoginController.login() → SysLoginService.login() → 校验用户名/密码/验证码 → 构建 LoginUser 对象 → LoginHelper.login(loginUser) → fillRequestContext(loginUser) // 自动填充IP/地点/设备 → StpUtil.login(loginId, parameter) // Sa-Token核心登录 → StpLogicJwtForSimple.createLoginSession() → 生成 JWT Token → PlusSaTokenDao.setObject() // 写Redis+失效Caffeine → TokenSession.extra: userId等 // 写入常用字段 → TokenSession.object: LoginUser // 写入完整用户对象 → 返回 Token 给前端

鉴权调用链

GET /system/user/list (Header: Authorization: Bearer {jwt}) → Sa-Token Interceptor → 从Header读取Token, 剥离"Bearer "前缀 → StpLogicJwtForSimple 解析JWT(验签+提取loginId) → @SaCheckPermission("system:user:list") → StpUtil.checkPermission() → SaPermissionImpl.getPermissionList(loginId) → [路径1] LoginHelper.getLoginUser() → LoginUser.permissions → [路径2] PermissionService.getMenuPermission(userId) → 权限码匹配 → 通过/403 → Controller 执行业务逻辑

JWT模式 vs Redis模式 对比

对比维度JWT模式(当前)Redis模式
Token格式JWT (eyJhb...)随机字符串
StpLogic实现StpLogicJwtForSimpleStpLogicForDefault
Token验证方式本地JWT验签查Redis验证存在
踢人下线机制清除Redis TokenSession(需等Caffeine 5秒过期)直接删除Redis中的Token
每次请求Redis查询仅查TokenSession(数据)每次验证Token存在+查Session
网关友好度高(网关可独立验JWT)低(网关必须转发到后端)
切换方法去掉 sa-token-jwt 依赖 + 改 StpLogic Bean加上依赖 + 改 Bean

模式切换操作清单

  1. pom.xml: 添加/移除 sa-token-jwt 依赖
  2. SaTokenConfig.java: 修改 stpLogic() Bean 的返回值
  3. common-satoken.yml: （无需修改，配置通用）
  4. PlusSaTokenDao.java: （无需修改，存储层不变）
  5. SaPermissionImpl.java: （无需修改，权限层不变）
  6. 重启应用验证

常见问题排查

问题现象可能原因排查步骤
登录成功但接口返回401 TokenSession未写入 / Caffeine未失效 检查Redis中TokenSession / 清除Caffeine
踢人后用户仍能访问 Caffeine 5秒缓存未过期（仅JWT模式） 等待5秒或重启 / 手动清Caffeine
@SaCheckPermission 一直403 权限码未分配 / SaPermissionImpl返回空 检查权限表 / 调试 getPermissionList 返回值
切换JWT模式后登录失败 缺少JWT密钥配置 检查 sa-token.jwt-secret-key 配置
分布式环境下登录状态不一致 Redis未共享 / 各实例Caffeine缓存不一致 检查Redis连接 / 确认Caffeine仅本地
