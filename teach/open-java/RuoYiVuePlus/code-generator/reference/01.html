核心数据结构

表关键字段说明
gen_tabledataName, tableName, className, packageName, moduleName, businessName, functionName, functionAuthor, tplCategory, frontendType, options(JSON)生成配置主表。options 存 JSON 扩展配置（导出/状态/唯一/排序/树参数），避免为每个可选功能加列
gen_table_columncolumnName, columnType, javaType, javaField, isPk, isIncrement, isRequired, isInsert, isEdit, isList, isQuery, queryType, htmlType, dictType, sort字段配置明细。所有 is* 字段值 "1"/"0"，通过 isPk()/isInsert() 等 boolean 辅助方法供模板使用

生成的 13~16 个产出文件

分类数量文件
后端 Java8Domain → VO → BO → Mapper → Service → ServiceImpl → Controller + Mapper XML
前端 TS/Vue/React4api/index.ts → types.ts → index.vue/tsx（或 index-tree 变体）
数据库 SQL1按方言自动选：mysql/oracle/postgres/sqlserver 菜单 INSERT SQL
核心链路一句话：用户导表 → AnyLine 读元数据 → GenUtils 推断类型 → FreeMarker 渲染模板 → ZIP 打包下载

关键设计思想

  • 数据与模板分离：元数据（gen_table + gen_table_column）与渲染模板（.ftl）彻底解耦，加新语言/框架只需加模板目录
  • 类型推断引擎：数据库类型 + 字段长度 + 字段名约定 + 业务语义，四维联合推断
  • 动态数据源适配：AnyLine 与 baomidou 桥接——多个框架共享一个连接池的经典案例
