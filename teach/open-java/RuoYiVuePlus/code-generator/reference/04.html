FreeMarker 引擎初始化（关键配置）

static {
    TemplateConfig config = new TemplateConfig(UTF_8, "", CLASSPATH);
    config.setCustomEngine(FreemarkerEngine.class);
    TEMPLATE_ENGINE = TemplateUtil.createEngine(config);

    // ⚠️ 必须设置！否则雪花ID 1234567890 → "1,234,567,890"
    freemarkerEngine.getConfiguration().setNumberFormat("computer");

    // 预加载所有固定路径的模板
    TEMPLATE_MAPPER = PathNamedTemplate.form(TEMPLATE_ENGINE, TEMPLATE_PATHS);
}

PathNamedTemplate：模板的"身份证"

设计说明
装饰器模式包装 FreeMarker Template 对象，同时携带 pathName（模板路径标识）
pathName如 "fm/java/controller.java.ftl"，用于日志、预览展示、文件路径计算
delegate.render()委托给 FreeMarker 引擎执行实际渲染
为什么要 pathName？两个不同框架的模板可能渲染出相同文件路径，需靠模板路径本身区分

模板选择：三维组合

getTemplateList(tplCategory, dsName, frontendType)
  │
  ├─ 后端（固定 8 个，与前端类型无关）
  │   domain.java.ftl → vo.java.ftl → bo.java.ftl
  │   → mapper.java.ftl → service.java.ftl
  │   → serviceImpl.java.ftl → controller.java.ftl
  │   → mapper.xml.ftl
  │
  ├─ 前端 API + TS 类型（按 frontendType 拼接）
  │   "fm/" + frontendType + "/api.ts.ftl"
  │   "fm/" + frontendType + "/types.ts.ftl"
  │
  ├─ SQL 模板（按数据库方言）
  │   isMySQL()? → fm/sql/mysql.sql.ftl
  │   isOracle()? → fm/sql/oracle.sql.ftl
  │   isPostgreSql()? → fm/sql/postgres.sql.ftl
  │   isSqlServer()? → fm/sql/sqlserver.sql.ftl
  │
  └─ 前端页面（按 tplCategory + 通配符发现扩展名）
      crud  → classpath*:fm/{type}/index.*.ftl
      tree  → classpath*:fm/{type}/index-tree.*.ftl

buildContext() 输出变量速查

分组变量来源
基础packageName, moduleName, businessName, ClassName, className, functionName, author, datetimeGenTable 字段
包路径basePackage, permissionPrefix包名计算
列信息columns, pkColumn, importListGenTable + 每列 javaType
列特征needImageUpload, needEditor, needSelect, needSwitchField, needParseTime, hasBetween, needAddDateRange ...setColumnFeatureContext() 扫描 columns
导出enableExportoptions JSON
状态enableStatus, statusColumn, statusField, statusActiveValue, statusInactiveValueoptions + 字段匹配
唯一校验enableUnique, uniqueColumnsoptions + 字段匹配
排序enableSort, sortColumn, sortFieldoptions + 字段匹配
字典dicts, dictsNoSymbol, needDict扫描所有列 dictType
树表treeCode, treeParentCode, treeName, treeRootValue, treeRootValueJavaLiteral, treeAncestorsField, treeOrderField ...options + 类型推断
菜单parentMenuId, menuIdsoptions + IdGeneratorUtil
setColumnFeatureContext() 设计亮点：预先扫描所有列的 htmlType，生成 needEditor、needImageUpload 等布尔开关。模板只需 <#if needEditor> 一个判断，不需要在渲染时遍历 columns 列表做复杂过滤。

添加新前端框架（以 Svelte 为例，4 步）

步骤操作
1. 创建模板在 fm/svelte/ 下创建 4 个 .ftl 文件：api.ts.ftl, types.ts.ftl, index.svelte.ftl, index-tree.svelte.ftl
2. 自动发现前端页面通过 classpath*:fm/svelte/index.*.ftl 通配符自动发现，无需改 Java 代码
3. 验证路径输出目录自动为 svelte，页面目录为 views（非 react 分支）
4. 使用gen_table 中将 frontendType 设为 svelte，生成时自动拼接路径
约定优于配置：加新前端框架 = 在 fm/ 下建目录 + 放 4 个 .ftl 文件。不需要改 Java 代码、不需要注册、不需要配置。
