# NOTES — 教学偏好暂存

> 记录用户在课程中表达的教学偏好，供后续课程设计参考。

- 用户要求「全面详细讲解，结合实际代码」——课程应大量引用真实文件路径、行号、可运行的代码片段，并追踪跨模块的真实调用链（如 `ruoyi-demo` 模块中 `DocumentMapper` → `EsCrudController` 的完整链路），不停留在抽象描述。
- 偏好「鸟瞰图先行」风格（沿用 teach-ruoyi-auth / teach-ruoyi-sms 的全景图 + 分步拆解 + 提取练习的结构）。
- 本模块文件极少（仅 2 个 Java 类），但承载了「可插拔模块」和「环境后处理器」两个通用模式的讲解——课程设计上每个模式单独一节课，第 3 课结合 DEMO 代码做实战收尾。
