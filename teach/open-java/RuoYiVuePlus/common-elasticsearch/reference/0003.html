ActuatorEnvironmentPostProcessor 核心逻辑

class ActuatorEnvironmentPostProcessor implements EnvironmentPostProcessor, Ordered { postProcessEnvironment(env, app): enable = env.getProperty("easy-es.enable", "false") System.setProperty( "management.health.elasticsearch.enabled", enable) getOrder(): return Ordered.HIGHEST_PRECEDENCE // Integer.MIN_VALUE }

为什么需要这个类

  • Spring Boot Actuator 内置 ElasticsearchHealthIndicator，默认启用
  • 如果 easy-es.enable=false 但健康检查仍开着 → 健康检查连接 ES 失败 → health 返回 DOWN
  • 此处理器在启动最早阶段把 easy-es 开关同步到健康检查开关，避免误报

关键技术点

技术点说明
EnvironmentPostProcessorSpring Boot SPI，在 Environment 准备后、Context 刷新前执行
Ordered.HIGHEST_PRECEDENCE值为 Integer.MIN_VALUE，确保最先执行
spring.factories注册方式：org.springframework.boot.EnvironmentPostProcessor=类名
System.setProperty直接设 JVM 系统属性，绕过 Spring Environment 体系，确保 Actuator 能读到

ES CRUD 开发模板（三文件骨架）

1. 实体类

// 放在 org.dromara.xxx.domain 包
@Data
public class Product {
    private String id;      // ES 文档 ID（String 类型！）
    private String name;
    private BigDecimal price;
}

2. Mapper 接口 ⚠ 必须在 esmapper 包下!

// 放在 org.dromara.xxx.esmapper 包
package org.dromara.xxx.esmapper;
public interface ProductMapper extends BaseEsMapper<Product> {
}

3. Controller / Service 使用

@RequiredArgsConstructor
@RestController
public class ProductController {
    private final ProductMapper productMapper;

    // 精确查询
    public Product getByName(String name) {
        LambdaEsQueryWrapper<Product> w =
            new LambdaEsQueryWrapper<>();
        w.eq(Product::getName, name);
        return productMapper.selectOne(w);
    }

    // 模糊搜索
    public List<Product> search(String kw) {
        LambdaEsQueryWrapper<Product> w =
            new LambdaEsQueryWrapper<>();
        w.like(Product::getName, kw);
        return productMapper.selectList(w);
    }

    // 插入
    public Integer add(Product p) {
        return productMapper.insert(p);
    }

    // 更新(按ID)
    public void update(Product p) {
        productMapper.updateById(p);
    }

    // 删除(按ID)
    public Integer delete(String id) {
        return productMapper.deleteById(id);
    }
}

常用查询条件速查

条件代码ES DSL 对应
精确匹配w.eq(Entity::getField, val)term
模糊匹配w.like(Entity::getField, val)match/wildcard
大于w.gt(Entity::getField, val)range gt
大于等于w.ge(Entity::getField, val)range gte
小于w.lt(Entity::getField, val)range lt
小于等于w.le(Entity::getField, val)range lte
IN 条件w.in(Entity::getField, vals)terms

application.yml 关键配置

easy-es:
  enable: true              # 必须设为 true
  address: localhost:9200   # ES 集群地址（逗号分隔）
  schema: http
  # username:               # 生产环境建议启用认证
  # password:
  global-config:
    print-dsl: true         # 开发时打开，看生成的 DSL
    db-config:
      id-type: customize    # ID 由应用生成（与实体 String id 配合）
      field-strategy: not_null
      refresh-policy: immediate

排障清单

  • Mapper 注入失败 → 检查包名是否以 esmapper 结尾
  • 查询结果为空 → 确认 print-dsl: true 打开后看控制台打印的 DSL 是否正确
  • 启动报 ES 连接超时 → 确认 ES 服务已启动（curl localhost:9200 测试）
  • ID 类型问题 → ES 文档 ID 是 String，不要用 Long
  • 实体类缺少无参构造 → Lombok @Data 自动生成，无需担心
