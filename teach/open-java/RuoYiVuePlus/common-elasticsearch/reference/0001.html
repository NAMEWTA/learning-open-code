速查 01 · ruoyi-common-elasticsearch 模块全景

配套课程：lessons/0001.html · 使命：读懂 ruoyi-common-elasticsearch 搜索引擎模块

一句话定位

ruoyi-common-elasticsearch 不是搜索引擎框架，而是把 Dromara 开源框架 Easy-Es(3.0.2) 收编进 RuoYi 技术栈的「可插拔适配垫片」——默认关闭、一行配置开启，自己不发 ES 请求，只补装配扫描和健康检查同步。

模块全部内容

文件作用
pom.xml只依赖 easy-es-boot-starter（底层 ES 客户端 7.17.28）
config/EasyEsConfiguration.java装配：@EsMapperScan 扫描注册 EsMapper（第 2 课）
config/ActuatorEnvironmentPostProcessor.java同步：easy-es.enable → management.health.elasticsearch.enabled（第 3 课）
META-INF/spring/AutoConfiguration.imports发现：声明 EasyEsConfiguration 为自动配置类
META-INF/spring.factories注册：声明 ActuatorEnvironmentPostProcessor 为环境后处理器

两个类 = 两件事

类职责关键注解/接口注册方式
EasyEsConfiguration扫描注册 EsMapper@AutoConfiguration + @ConditionalOnProperty + @EsMapperScanAutoConfiguration.imports
ActuatorEnvironmentPostProcessor同步健康检查开关EnvironmentPostProcessor + Ordered.HIGHEST_PRECEDENCEspring.factories

DEMO 使用现场（5 个 API）

API方法Easy-Es 调用
/es/select?title=精确查询wrapper.eq(Document::getTitle, title) → selectOne
/es/search?key=模糊搜索wrapper.like(Document::getTitle, key) → selectList
/es/insert插入文档documentMapper.insert(document)
/es/update更新文档documentMapper.updateById(document)
/es/delete/{id}删除文档documentMapper.deleteById(id)

记忆锚点

  • 模块只有 2 个 Java 类，没有一行发送 ES 请求的代码——真正通信靠 Easy-Es。
  • @EsMapperScan("org.dromara.**.esmapper") 是 Mapper 能被发现的唯一入口——Mapper 必须放在 esmapper 包下。
  • easy-es.enable 默认 false——可插拔设计，不需要 ES 的项目零开销。
  • 两个注册文件走不同 SPI 通道：.imports → 自动配置、spring.factories → 环境后处理器。
  • 这是所有 ruoyi-common-* 包装第三方库的标准范式。
