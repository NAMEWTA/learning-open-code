EasyEsConfiguration 三注解

注解作用失效后果
@AutoConfiguration标记为自动配置类（proxyBeanMethods=false，支持 before/after 排序）即使 .imports 文件列出了，Spring 也不把它当配置类处理
@ConditionalOnProperty(value="easy-es.enable", havingValue="true")条件开关：仅当 easy-es.enable=true 时生效类被完全跳过，@EsMapperScan 不执行，EsMapper 不会注册为 Bean
@EsMapperScan("org.dromara.**.esmapper")扫描指定包下所有 BaseEsMapper 子接口，生成代理并注册 BeanEsMapper 接口即使写对了也不会被扫描，注入时报 NoSuchBean

两个注册文件对比

Spring Boot 启动与 ES 组件的时序

1️⃣ EnvironmentPostProcessor 执行 └── ActuatorEnvironmentPostProcessor: easy-es.enable → management.health.elasticsearch.enabled 2️⃣ AutoConfiguration 加载 └── .imports → EasyEsConfiguration → 检查 @ConditionalOnProperty ├─ true → @EsMapperScan 扫描 → 代理 Bean 注册 └─ false → 跳过，零开销 3️⃣ ApplicationContext 刷新 └── 实例化单例 Bean ├── DocumentMapper（代理对象） ├── EsCrudController（注入 DocumentMapper） └── ElasticsearchHealthIndicator（读取 management.health.elasticsearch.enabled）

Mapper 扫描路径规则

  • 扫描模式：org.dromara.**.esmapper
  • ** 匹配任意层级目录
  • 匹配条件：包路径以 org.dromara 开头，以 esmapper 结尾
  • 被扫描的接口必须 extends BaseEsMapper

常见问题排障

现象最可能的原因
DocumentMapper 注入失败 NoSuchBean①Mapper 不在 esmapper 包下 ②easy-es.enable 不是 true ③忘记依赖 ruoyi-common-elasticsearch
easy-es.enable=true 但 Mapper 仍未被扫描检查 @EsMapperScan 的包路径是否正确，确认 Mapper 接口 extends BaseEsMapper
健康检查报 ES 连接失败（项目不用 ES）确认 easy-es.enable=false，并确认 management.health.elasticsearch.enabled 也被同步为 false（检查 ActuatorEnvironmentPostProcessor 是否被加载）
