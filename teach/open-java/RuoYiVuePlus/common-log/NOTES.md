# NOTES — 教学偏好暂存

> 记录用户在课程中表达的教学偏好，供后续课程设计参考。

- 用户要求「全面详细讲解，结合实际代码」——课程应大量引用真实文件路径、行号、可运行的代码片段，并追踪跨模块的真实调用链，不停留在抽象描述。
- 偏好「鸟瞰图先行」风格（沿用 teach-ruoyi-auth 的全景图 + 分步拆解 + 提取练习的结构）。
- 学习者：全栈背景（Java 后端 + Vue/React 前端）。
- 目标：读懂整体设计与流程，不要求大量动手编码。
- 课程风格：以追踪真实代码调用链 + 解释设计动机为主；练习偏「读代码答问题 / 复述链路 / 对比设计」。
- 语言：简体中文。
- ruoyi-common-log 模块特色：代码量小（仅 8 个源文件）、结构清晰，教学重点应放在 AOP + 事件解耦 + 跨模块协作的设计思想上，而非逐行解释代码。
- 关键跨模块链路：`@Log 注解 Controller` -> `LogAspect` (common-log) -> `publishEvent` -> `@EventListener + @Async` (ruoyi-system) -> 写入数据库。模块边界感受是教学亮点。
