速查 01 · ruoyi-common-log 模块全景

一句话定位：通过 AOP 切面自动拦截 @Log 注解，采集操作日志并以 Spring 事件方式异步入库的通用日志模块。

模块文件清单（7 个文件）

文件路径职责
Log.javaannotation/声明式日志注解，标记 Controller 方法
BusinessType.javaenums/定义 10 种业务操作类型
BusinessStatus.javaenums/操作结果：SUCCESS / FAIL
OperatorType.javaenums/操作人来源：后台/手机端/其他
LogAspect.javaaspect/AOP 切面核心：拦截、采集、填充、发布事件
OperLogEvent.javaevent/操作日志事件对象（22 个字段）
LoginInfoEvent.javaevent/登录事件对象（5 个字段）

三层架构

┌─ 注解层 ─────────────────────────────────────┐ │ @Log(title, businessType, operatorType, │ │ isSaveRequestData, isSaveResponseData) │ └──────────────────┬──────────────────────────────┘ ▼ @Around 拦截 ┌─ 切面层 (LogAspect) ─────────────────────────┐ │ doAround() → handleLog() │ │ ├ 采集 IP / URL / 用户 / 设备 / 浏览器 │ │ ├ 设置注解参数（title, businessType...） │ │ ├ 收集请求参数 & 排除敏感字段 │ │ ├ 计算耗时 (StopWatch) │ │ └ 发布 OperLogEvent │ └──────────────────┬──────────────────────────────┘ ▼ Spring Event ┌─ 事件层 ─────────────────────────────────────┐ │ OperLogEvent / LoginInfoEvent │ │ └→ 监听器（外部模块）异步消费入库 │ └──────────────────────────────────────────────┘

两个事件对比

维度OperLogEventLoginInfoEvent
触发方式AOP 自动（@Log 注解）手动 publishEvent
字段数量22 个（含用户/设备/耗时/结果等）5 个（username, status, message, request, args）
数据来源LoginHelper + ServletUtils + 注解调用方手动填充
是否自动采集是，切面全自动采集否，由业务代码自行填充
典型场景增删改查、导出、授权等操作登录成功/失败记录

记忆锚点

  • @Around 双路径：切面中 try/catch 分别走成功路径和异常路径，但最终都调用同一个 handleLog()
  • Spring Event 解耦：切面只负责发布事件，不直接操作数据库 — 监听器在 ruoyi-system 模块中消费
  • Ordinal 映射：枚举值以 ordinal() 写入 DB，而不是枚举名称字符串
  • 参数自动过滤：自动排除 MultipartFile、HttpServletRequest、HttpServletResponse、BindingResult
  • 异常保护：handleLog 内部 catch 所有异常只打印日志，不影响业务主流程执行
