速查 03 · LogAspect 处理流程

doAround 完整流程（成功 + 异常双路径）

@Around("@annotation(controllerLog)") ┌─ doAround(joinPoint, controllerLog) ─────────────────────────┐ │ StopWatch start() │ │ ┌─ try ───────────────────────────────────────────────────┐ │ │ │ jsonResult = joinPoint.proceed() // 执行目标方法 │ │ │ │ handleLog(joinPoint, log, null, jsonResult, sw) // 成功 │ │ │ │ return jsonResult │ │ │ └────────────────────────────────────────────────────────┘ │ │ ┌─ catch (Exception e) ──────────────────────────────────┐ │ │ │ handleLog(joinPoint, log, e, null, sw) // 失败 │ │ │ │ throw e // 异常继续向上抛，不吞 │ │ │ └────────────────────────────────────────────────────────┘ │ └──────────────────────────────────────────────────────────────┘ 注意：成功路径 jsonResult 非 null，异常路径 jsonResult=null 成功路径 exception=null，异常路径 exception 非 null

handleLog 7 步处理

步骤操作关键代码
1. 创建事件new OperLogEvent，初始 status=SUCCESS.ordinal()new OperLogEvent()
2. 采集网络信息获取 IP、URL、ClientKey（长度限制）ServletUtils.getClientIP()
request.getRequestURI()
3. 采集用户/设备从 LoginHelper 获取用户、部门、设备、浏览器、OSLoginHelper.getLoginUser()
4. 设置异常状态如果 e != null，status=FAIL，记录 errorMsgoperLog.setStatus(FAIL.ordinal())
5. 设置方法信息className.methodName()、请求方式 GET/POST 等joinPoint.getSignature().getName()
6. 注解参数处理委托 getControllerMethodDescription() 填充 title/businessType/参数getControllerMethodDescription(...)
7. 耗时 & 发布stopWatch.stop()，计算 costTime，SpringUtils 发布事件SpringUtils.context().publishEvent(operLog)

参数收集决策树

isSaveRequestData == true ? ├─ paramsMap 不为空？ │ ├ 不为空 → 剔除 EXCLUDE_PROPERTIES + excludeParamNames → JSON 序列化 │ └ 为空 + 请求方式是 PUT/POST/DELETE？ │ ├ 是 → argsArrayToString(joinPoint.getArgs()) → 过滤后拼接 │ └ 否 → 参数为空字符串 └ isSaveRequestData == false → 跳过参数收集

isFilterObject / isFilterValue 过滤类型

过滤目标说明
MultipartFile文件上传对象，不能序列化
HttpServletRequest请求对象，循环引用无法序列化
HttpServletResponse响应对象，循环引用无法序列化
BindingResult校验结果对象，非业务数据
包含以上类型的数组/集合/Map递归检查容器内元素
