扩展指南

  1. 新增 BusinessType：在 BusinessType.java 枚举末尾追加新值（如 AUDIT），不可在中间插入，否则 ordinal 错位导致历史数据错误。
  2. 创建自定义事件：继承 Serializable，使用 @Data 注解，定义所需字段。参考 OperLogEvent 结构。
  3. 添加监听器：使用 @EventListener 注解或实现 ApplicationListener<YourEvent>，在 onApplicationEvent() 中处理业务逻辑（写入 DB、发送通知等）。
  4. 发布事件：通过 SpringUtils.context().publishEvent(event) 手动发布，或在 AOP 切面中自动发布。
  5. 异步处理：监听器方法上加 @Async 注解，避免阻塞主业务流程。

事件发布方式对比

维度AOP 自动发布（OperLogEvent）手动发布（LoginInfoEvent）
触发机制@Log 注解 + @Around 切面业务代码显式调用 publishEvent
数据采集全自动（IP/用户/参数/耗时）手动填充所有字段
代码侵入性低（加注解即可）高（需编写发布代码）
灵活性受限于注解属性完全自由控制
适用场景Controller 操作记录登录、定时任务、非 HTTP 触发
发布代码示例无需手动代码SpringUtils.context().publishEvent(new LoginInfoEvent(...))
