速查 02 · @Log 注解与枚举速查

@Log 注解属性

属性名类型默认值说明
titleString""操作模块名称，如"用户管理"
businessTypeBusinessTypeBusinessType.OTHER业务操作类型
operatorTypeOperatorTypeOperatorType.MANAGE操作人类别（后台/手机端）
isSaveRequestDatabooleantrue是否保存请求参数
isSaveResponseDatabooleantrue是否保存响应结果
excludeParamNamesString[]{}排除不记录的请求参数名

BusinessType 枚举（完整）

Ordinal枚举值中文含义典型场景
0OTHER其他未分类操作
1INSERT新增添加用户、角色等
2UPDATE修改编辑数据
3DELETE删除删除记录
4GRANT授权分配角色、权限
5EXPORT导出数据导出
6IMPORT导入数据导入
7FORCE强退强制退出用户
8GENCODE生成代码代码生成器
9CLEAN清空数据清空表/缓存

OperatorType 枚举

Ordinal枚举值中文含义
0OTHER其他
1MANAGE后台用户
2MOBILE手机端用户

BusinessStatus 枚举

Ordinal枚举值中文含义
0SUCCESS成功
1FAIL失败

Ordinal 映射说明

数据库存储的是枚举的 ordinal() 整数值，而非枚举名称字符串。例如 BusinessType.INSERT 入库为 1。
注意：如果修改枚举值顺序（增删项），历史数据将错位，必须在末尾追加新值。

典型用法示例

// Controller 方法上标注 @Log(title = "用户管理", businessType = BusinessType.INSERT, operatorType = OperatorType.MANAGE, excludeParamNames = {"password", "salt"}) @PostMapping public R addUser(@RequestBody User user) { ... }
// 切面自动拦截流程： @Around → doAround() → joinPoint.proceed() // 执行目标方法 → handleLog() // 采集日志 ├ title → operLog.title = "用户管理" ├ businessType → operLog.businessType = 1 (ordinal) ├ operatorType → operLog.operatorType = 1 (ordinal) ├ isSaveRequestData = true → 收集请求参数 (排除 password, salt) └ isSaveResponseData = true → 收集响应 JSON
