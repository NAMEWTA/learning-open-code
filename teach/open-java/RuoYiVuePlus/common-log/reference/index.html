ruoyi-common-log 速查表

Maven: org.dromara:ruoyi-common-log | ruoyi-common-log/src/main/java/org/dromara/common/log/

定位

基于 声明式注解 + AOP 切面 + Spring 事件 三层设计的零侵入操作日志记录模块。仅 7 个 Java 类（1 注解 + 1 切面 + 3 枚举 + 2 事件），让业务 Controller 加一个 @Log 注解即可自动记录操作日志，存储由 system 模块异步消费。

核心文件

类名路径职责
@Logannotation/Log.java操作日志注解，6 个属性：title、businessType、operatorType、isSaveRequestData、isSaveResponseData、excludeParamNames
LogAspectaspect/LogAspect.javaAOP 切面，@Around 拦截 @Log 方法：StopWatch 计时 → proceed() → 填充上下文 → 发布 OperLogEvent
BusinessTypeenums/BusinessType.java操作类型枚举：OTHER/INSERT/UPDATE/DELETE/GRANT/EXPORT/IMPORT/FORCE/GENCODE/CLEAN
OperatorTypeenums/OperatorType.java操作人类别：OTHER/MANAGE/MOBILE，与 BusinessType 正交（做什么 vs 谁做的）
BusinessStatusenums/BusinessStatus.java操作状态：SUCCESS(0)/FAIL(1)，LogAspect 根据是否有异常自动赋值
OperLogEventevent/OperLogEvent.java操作日志事件，23 字段（title/businessType/operUrl/operIp/userId/operParam/costTime 等），由 system 模块异步消费
LoginInfoEventevent/LoginInfoEvent.java登录事件，5 字段（username/status/message/request/args），由 SysLoginService 发布

术语速查

术语解释
@Log零侵入操作日志入口，标注在 Controller 方法上即可自动记录
LogAspectAOP 环绕切面：拦截 → 计时 → 执行 → 组装事件 → 发布，异常时状态翻转为 FAIL
BusinessType回答「做了什么操作」：INSERT/UPDATE/DELETE/EXPORT 等 10 种
OperatorType回答「从哪个端操作」：MANAGE（后台）/MOBILE（手机）/OTHER
BusinessStatus回答「执行结果如何」：SUCCESS(0) / FAIL(1)，异常时自动翻转
OperLogEventSpring 事件载体（非实体），23 字段描述一次完整操作，由 SysOperLogServiceImpl 消费
LoginInfoEvent登录事件载体，由 SysLoginService 发布，SysLoginInfoServiceImpl 消费
事件解耦common-log 只管发事件，system 模块 @EventListener + @Async 异步消费写库
excludeParamNames@Log 属性，排除请求参数中的敏感字段（如 password），不写入日志
操作日志 vs 系统日志操作日志=业务审计（@Log/数据库），系统日志=技术运行（slf4j/文件）

常用代码

基础用法：Controller 方法加 @Log

@PostMapping("/add")
@Log(title = "用户管理", businessType = BusinessType.INSERT)
public R<Void> add(@Validated @RequestBody SysUser user) {
    userService.save(user);
    return R.ok();
}

排除敏感参数

@PutMapping("/resetPwd")
@Log(title = "用户管理", businessType = BusinessType.UPDATE,
     excludeParamNames = {"password", "oldPassword"})
public R<Void> resetPwd(@RequestBody ResetPwdDTO dto) {
    userService.resetPwd(dto);
    return R.ok();
}

完整调用链路

Controller @Log → LogAspect @Around 拦截
  → StopWatch 开始计时
  → proceed() 执行目标方法
  → handleLog() 组装 OperLogEvent
      ├─ IP/URL/用户/方法名/请求方式 自动获取
      ├─ 解析 @Log 注解（title/businessType/operatorType）
      ├─ 保存请求参数（排除 excludeParamNames + 系统常量 EXCLUDE_PROPERTIES）
      ├─ 保存响应结果（isSaveResponseData=true 时）
      └─ 异常时 status=FAIL 并记录 errorMsg
  → SpringUtils.context().publishEvent(operLog)
  → SysOperLogServiceImpl.recordOper() @EventListener + @Async 消费
  → 异步写入 sys_oper_log 表
