02 API 加解密通道

本节目标：理解 API 加解密通道的完整工作流程：@ApiEncrypt 注解控制、CryptoFilter 拦截协调、双 Wrapper 的请求解密与响应加密机制，以及 RSA+AES 混合加密的密钥协商过程。

1. 通道入口：CryptoFilter

API 加解密通道在 Servlet Filter 层 工作，属于 Web 容器层面的拦截。核心类是 org.dromara.common.encrypt.filter.CryptoFilter，实现 javax.servlet.Filter。

Filter 在 Spring Security 过滤器链中注册，拦截所有请求。根据请求方法和 @ApiEncrypt 注解决定是否解密请求体、是否加密响应体。

2. @ApiEncrypt 注解

标记在 Controller 方法上，控制加解密行为：

@ApiEncrypt(response = true)   // 需要加密响应体
@PostMapping("/save")
public R<Void> save(@RequestBody User user) { ... }

@ApiEncrypt                    // 默认 response=false，只解密请求体
@PostMapping("/login")
public R<String> login(@RequestBody LoginBody body) { ... }

只有一个属性 response()：false（默认）时只解密请求，true 时同时加密响应。

3. 请求解密：DecryptRequestBodyWrapper

org.dromara.common.encrypt.filter.DecryptRequestBodyWrapper 继承 HttpServletRequestWrapper，构造时执行解密：

  1. 从请求头获取 encrypt-key（RSA 公钥加密后的 AES 密钥）
  2. 用 RSA 私钥解密 encrypt-key，得到 Base64 编码的 AES 密钥
  3. Base64 解码得到 AES 密钥明文
  4. 读取原始请求体（密文）
  5. 用 AES 密钥解密得到 JSON 明文
  6. 将明文字节缓存为 ByteArrayInputStream，支持重复读取
// 密钥协商（客户端侧）
const aesKey = randomBytes(24);                      // 1. 生成随机 AES 密钥
const encryptedKey = rsaEncrypt(aesKey, publicKey);  // 2. 用 RSA 公钥加密
headers['encrypt-key'] = base64(encryptedKey);       // 3. 放入请求头
const body = aesEncrypt(JSON.stringify(data), aesKey);// 4. AES 加密请求体

4. 响应加密：EncryptResponseBodyWrapper

org.dromara.common.encrypt.filter.EncryptResponseBodyWrapper 继承 HttpServletResponseWrapper，关键方法 getEncryptContent()：

  1. 用 ByteArrayOutputStream 拦截并缓存原始响应内容
  2. 生成 24 字节随机 AES 密钥
  3. Base64 编码 AES 密钥，再用 RSA 公钥加密，放入响应头 encrypt-key
  4. 用 AES 密钥加密业务响应内容
  5. 设置 Content-Length 为加密后的长度（避免分块传输问题）

5. 完整密钥协商流程

客户端                                服务端
  |                                     |
  |--[1] 生成随机 AES Key (24字节)------>
  |--[2] RSA公钥加密 AES Key ----------->
  |--[3] AES加密请求体 ---------------->
  |         POST /api/xxx               |
  |         Header: encrypt-key          |
  |         Body: AES密文                |
  |                                     |--[4] RSA私钥解密 encrypt-key
  |                                     |--[5] Base64解码得 AES Key
  |                                     |--[6] AES解密请求体 -> JSON明文
  |                                     |--[7] Controller处理业务
  |                                     |--[8] AES加密响应体
  |                                     |--[9] RSA公钥加密新AES Key -> 放响应头
  |<--[10] 响应头 encrypt-key ---------|
  |<--[11] AES密文响应体 -------------|
  |--[12] RSA私钥解密 encrypt-key ----->
  |--[13] AES解密响应体 -> JSON明文 --->
设计洞察：之所以用 RSA（非对称）+ AES（对称）混合加密，是因为 RSA 加密大数据很慢，AES 加密快但密钥分发不安全。混合方案取长补短：用 RSA 安全传输 AES 密钥（仅 24 字节），用 AES 高速加密业务数据（可能数 KB）。每次请求生成随机 AES 密钥，实现一次一密。
