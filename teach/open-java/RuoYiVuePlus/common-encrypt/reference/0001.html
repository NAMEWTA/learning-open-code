算法速查

算法类型密钥是否真加密Encoder
BASE64编码无否Base64Encryptor
AES对称password (16/24/32B)是AesEncryptor
RSA非对称publicKey + privateKey (≥1024bit)是RsaEncryptor
SM2非对称(国密)publicKey + privateKey是Sm2Encryptor
SM4对称(国密)password (恰好16B)是Sm4Encryptor

ENC_ 前缀机制

加密时：EncryptorManager.encrypt(value, ctx) → if (value.startsWith("ENC_")) return value; // 已加密，跳过 → encryptor.encrypt(value, encode) → return "ENC_" + encryptedValue; 解密时：EncryptorManager.decrypt(value, ctx) → if (!value.startsWith("ENC_")) return value; // 未加密，跳过 → value = value.substring(4); // 去除前缀 → return encryptor.decrypt(value);

目的：防止对已加密值重复加密（双重加密），以及防止对未加密值误解密。

配置示例（application.yml）

# API 加密通道（RSA+AES 混合） api-decrypt: enabled: true header-flag: encrypt-key public-key: MIGfMA0GCSqGSIb3DQEBAQUAA4GNADCBiQKBgQC... private-key: MIICdgIBADANBgkqhkiG9w0BAQEFAASCAmAwggJcAgEAAoGBA... # DB 字段加密通道（可选五种算法） mybatis-encryptor: enable: true algorithm: aes # 默认算法：BASE64|AES|RSA|SM2|SM4 password: your16bytekey! # AES/SM4 的密钥 public-key: MIGfMA0G... # RSA/SM2 的公钥 private-key: MIICdgIB... # RSA/SM2 的私钥 encode: base64 # 编码方式：BASE64|HEX

关键注意事项

  • Base64 不是加密：生产环境不要用 BASE64 作为 DB 字段加密算法，它是明文编码
  • API 通道不支持自定义算法：CryptoFilter 内部硬编码 RSA+AES，不经过 AlgorithmType 选择
  • RSA 密钥最低 1024 位：CryptoFilter 构造时校验，不满足则启动失败
  • 注解覆盖 yml：@EncryptField 的 algorithm=SM4 会覆盖 yml 中配置的 algorithm=AES
  • Snapshot 恢复：入参加密后原始值会被恢复，业务层始终操作明文
  • 反射缓存：encryptorMap 和 fieldCache 均为 ConcurrentHashMap，线程安全
