速查 05 · 加密上下文与工厂速查

配套课程：lessons/0005.html

一句话

「@EncryptField 注解 → EncryptContextFactory.create(field) 合并默认值 → EncryptContext 五字段 → EncryptorManager 双缓存调度 → IEncryptor 执行加解密」

EncryptContext 五字段

字段类型用途
algorithmAlgorithmType算法：BASE64 / AES / RSA / SM2 / SM4
passwordString对称密钥，AES/SM4 使用
publicKeyString非对称公钥，RSA/SM2 加密使用
privateKeyString非对称私钥，RSA/SM2 解密使用
encodeEncodeType编码方式：BASE64 / HEX

EncryptContextFactory 优先级逻辑

属性注解值结果
algorithmAlgorithmType.DEFAULT使用 defaultProperties.getAlgorithm()
algorithmAlgorithmType.AES使用 AES
password""（空字符串）使用 defaultProperties.getPassword()
password"my-key"使用 "my-key"
encodeEncodeType.DEFAULT使用 defaultProperties.getEncode()
publicKey""使用 defaultProperties.getPublicKey()
privateKey""使用 defaultProperties.getPrivateKey()

规则：枚举用 == DEFAULT 判等；字符串用 StringUtils.isBlank() 判空。

EncryptorManager 双缓存架构

EncryptorManager │ ├── encryptorMap: Map<EncryptorCacheKey, IEncryptor> │ ├── key = (algorithm, encode, password, publicKey, privateKey) │ └── value = 加密器实例（AesEncryptor / RsaEncryptor / ...） │ └── fieldCache: Map<Class<?>, Set<Field>> ├── key = 实体类 Class 对象 └── value = 标记了 @EncryptField 且类型为 String 的字段集合

ENCRYPT_HEADER 机制

常量值
Constants.ENCRYPT_HEADER"ENC_"
操作条件行为
encrypt()值以 "ENC_" 开头直接返回原值（已加密，跳过）
encrypt()值不以 "ENC_" 开头加密 + 在前面加上 "ENC_" 前缀
decrypt()值不以 "ENC_" 开头直接返回原值（明文，跳过）
decrypt()值以 "ENC_" 开头去掉前缀 + 解密

EncryptorCacheKey 结构

private record EncryptorCacheKey( AlgorithmType algorithm, // 如 AES EncodeType encode, // 如 BASE64 String password, // 如 "secret-key" String publicKey, // 如 "MIGfMA0..." String privateKey // 如 "MIICdg..." ) // record 类型 = 不可变 + 自动 equals/hashCode = 完美 HashMap key

getFieldCache 过滤规则

public Set<Field> getFieldCache(Class<?> sourceClazz) { 跳过 ← isPrimitive() // int, long, boolean ... 跳过 ← isArray() // int[], String[] ... 跳过 ← isEnum() // 枚举类 跳过 ← isAnnotation() // 注解类 跳过 ← isPrimitiveOrWrapper() // Integer, Long, Boolean ... 跳过 ← name.startsWith("java.") // java.lang.String, java.util.Date ... 否则 → fieldCache.computeIfAbsent(sourceClazz, this::getEncryptFieldSetFromClazz) }
