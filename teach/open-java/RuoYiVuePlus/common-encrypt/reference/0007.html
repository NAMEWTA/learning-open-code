常见陷阱

  • enabled vs enable：API 层是 enabled，DB 层是 enable，两个不同的字段名，写反了开关不生效
  • 密钥独立配置：api-decrypt 和 mybatis-encryptor 的密钥完全独立，不存在"共用一个密钥配置"的设计
  • 默认算法陷阱：EncryptorProperties 默认 algorithm=BASE64（仅编码，非加密）。如果只开关不配算法，数据不会真正被加密
  • 扫描路径：EncryptorManager 通过 MybatisPlusProperties.typeAliasesPackage 扫描实体类。如果实体类不在该路径下，@EncryptField 不会被识别
  • 启动校验局限性：validateEncryptorProperties 只校验全局默认算法所需的密钥。字段级别的算法覆盖独立于全局校验
  • 两个拦截器 vs 一个：入参加密和解密是两个拦截器，独立注册。调试时可以临时注释掉一个来定位问题方向
