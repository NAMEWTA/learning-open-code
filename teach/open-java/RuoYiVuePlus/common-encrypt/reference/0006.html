ENCRYPT_HEADER + 重复加密防护

Constants.ENCRYPT_HEADER = "ENC_" 加密流程： encrypt(value): if value.startsWith("ENC_") → return value // 幂等：已加密 result = encryptor.encrypt(value) return "ENC_" + result 解密流程： decrypt(value): if !value.startsWith("ENC_") → return value // 幂等：明文 stripped = value.removePrefix("ENC_") return encryptor.decrypt(stripped)

注意事项（常见坑）

  • 只支持 String 类型：getEncryptFieldSetFromClazz 过滤条件包含 field.getType() == String.class，Integer/Long/Date 等类型的字段即使标记了 @EncryptField 也会被忽略
  • @Inherited 行为：父类 Entity 的加密字段会被子类继承。如果不想让子类加密，需要在子类中移除该字段或用非 String 类型重写
  • 注解优先于 yml 默认值：注解上显式指定的 algorithm/password/encode 会覆盖 yml 全局配置，但 DEFAULT 或空字符串会回退到全局配置
  • 加密后无法范围查询：加密字段的 WHERE >、<、BETWEEN、LIKE 等条件会失效。只能做精确匹配（WHERE =）。如果需要模糊搜索，应在应用层解密后过滤
  • 数据库字段类型需为 VARCHAR：加密结果是文本（base64/hex 编码），需要足够长度的 VARCHAR 列。不要对加密字段建唯一索引（两次加密同一明文不一定得到同一密文，除非算法是确定性的）
  • 密钥管理：密码/密钥不应硬编码在注解中。建议注解留空，统一在 yml 中配置并通过配置中心管理，仅对需要独立密钥的高敏感字段在注解上指定
