关键陷阱与注意事项

陷阱说明
Base64 是编码不是加密 Base64Encryptor 不使用任何密钥，任何人都可解码。生产环境不要用它保护敏感数据。它的主要用途是开发调试时的占位算法。
SM4 密钥必须恰好 16 字节 EncryptUtils 中 SM4 的校验是 password.length() != 16（精确相等），而不是像 AES 那样允许 16/24/32。注意中文密码的 UTF-8 字节长度不等于字符数。
AES 密钥支持 16/24/32 分别对应 AES-128/192/256。Hutool 使用 password.getBytes(UTF_8) 直接作为密钥字节，所以密码必须是恰好 16/24/32 个 UTF-8 字节。
RSA 密钥最低 1024 位 CryptoFilter 构造时调用 validateRsaPublicKey/PrivateKey 校验。不满足则应用启动失败（Fail-Fast）。生产环境建议使用 2048 位以上。
RSA/SM2 构造器校验公私钥完整性 RsaEncryptor 和 Sm2Encryptor 的构造器中检查 publicKey 和 privateKey 是否同时提供，任一缺失直接抛异常。但 AesEncryptor 和 Sm4Encryptor 不在构造器中校验 password（留给 EncryptUtils 处理）。
解密不需要 encodeType IEncryptor.decrypt(String value) 不接收 EncodeType 参数。Hutool 的 decryptStr() 方法能自动识别密文是 Base64 还是 Hex 编码。
EncryptorCacheKey 使用 Record Java 16+ 的 Record 类型自动实现 equals()/hashCode()，天然适合作为 HashMap 的 Key。加密器相同配置会被复用，避免重复创建。

加密器创建与缓存流程

调用链： MybatisEncryptInterceptor.intercept() → EncryptedFieldProcessor.encrypt(sourceObject) → handle() 递归扫描所有 @EncryptField 字段 → EncryptContextFactory.create(field) // 注解+yml 合并 → EncryptorManager.encrypt(value, encryptContext) → if (value.startsWith("ENC_")) return value; // 幂等性保护 → registAndGetEncryptor(encryptContext) → EncryptorCacheKey.of(encryptContext) // 构建缓存键 → encryptorMap.computeIfAbsent(key, k -> ReflectUtil.newInstance(k.algorithm().getClazz(), encryptContext)) // 缓存未命中则反射创建，命中则复用 → encryptor.encrypt(value, encryptContext.getEncode()) → return "ENC_" + encryptedValue;
