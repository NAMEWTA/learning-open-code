ruoyi-common-encrypt 速查表

Maven: org.dromara:ruoyi-common-encrypt | ruoyi-common-encrypt/src/main/java/org/dromara/common/encrypt/

定位

提供双通道透明加解密能力：API 加密通道（Servlet Filter 层拦截请求/响应）和 DB 字段加密通道（MyBatis Interceptor 层拦截 SQL 入参/出参），支持 AES、RSA、SM2、SM4、Base64 五种算法。两条通道独立配置、独立开关，可按需组合使用。

核心文件

类/接口类型职责
IEncryptor接口加密算法顶层抽象，定义 algorithm() / encrypt() / decrypt()
AbstractEncryptor抽象类加密器基类，接收 EncryptContext 做配置校验与注入
AesEncryptor实现类AES 对称加密（密钥 16/24/32 位），国标常用
RsaEncryptor实现类RSA 非对称加密（公钥加密 + 私钥解密），密钥协商场景
Sm2Encryptor实现类SM2 国密非对称加密，等同 RSA 的非对称用法
Sm4Encryptor实现类SM4 国密对称加密（密钥 16 位），等同 AES 的对称用法
Base64Encryptor实现类Base64 编码器（非加密），仅做编码/解码
AlgorithmType枚举枚举 5 种算法并绑定加密器实现类，DEFAULT(null) 走 yml 全局配置
EncodeType枚举编码方式：BASE64 / HEX / DEFAULT
EncryptContextDTO加密参数载体（algorithm / password / publicKey / privateKey / encode）
EncryptContextFactory工厂类从 @EncryptField 注解 + yml 默认配置合成 EncryptContext，注解优先、默认值兜底
EncryptorManager管理器加解密调度中心，维护 encryptorMap（按算法+密钥缓存加密器）和 fieldCache（按 Class 缓存注解字段）两级缓存
EncryptedFieldProcessor处理器递归遍历对象 @EncryptField 字段并加解密，返回 FieldSnapshot 列表；支持嵌套、集合、Map，IdentityHashMap 防循环引用
FieldSnapshot内部类记录字段原始值，restore() 恢复原值——保证入参加密只对 SQL 生效，业务对象不受污染
EncryptUtils工具类纯静态工具，封装 Hutool SecureUtil / SmUtil，额外提供 MD5 / SHA256 / SM3 哈希
@ApiEncrypt注解标记 Controller 方法，response=true 时同时加密响应
@EncryptField注解标记实体 String 字段，@Inherited 支持继承链；可逐字段覆盖算法/密钥/编码

术语速查

术语一句话解释避免使用
双通道架构API 通道（Servlet Filter + Wrapper）与 DB 通道（MyBatis Interceptor）独立配置、独立开关的加解密设计「两种加密方式」
IEncryptor加密算法顶层接口，定义 encrypt(value, encodeType) / decrypt(value)「加密算法」
AlgorithmType枚举 5 种算法并绑定加密器实现类，EncryptorManager 通过 clazz 字段反射实例化—
EncryptContext加密参数的 DTO 载体，在 ContextFactory → Manager → Encryptor 之间流转—
EncryptContextFactory注解字段 + yml 默认值合成 EncryptContext，规则：注解值优先、默认值兜底—
EncryptorManager加解密调度中心，两级缓存（加密器实例 + 注解字段），加解密时检查 ENCRYPT_HEADER 前缀防重复「加密工具类」
EncryptedFieldProcessor递归处理对象 @EncryptField 字段加解密，生成 FieldSnapshot 用于事后还原—
FieldSnapshot字段原始值快照，restore() 确保 SQL 后业务对象还原——加密只对 SQL 生效—
ENCRYPT_HEADER加密结果前缀常量（来自 ruoyi-common-core），标记「已加密」防重复加密—
EncryptUtils纯静态加解密工具，封装 Hutool SecureUtil / SmUtil，含 MD5 / SHA256 / SM3 哈希—
CryptoFilterAPI 加密入口 Filter，密钥协商：客户端 RSA 加密随机 AES 密钥 → 服务端 RSA 解密 → AES 加解密业务报文—
DecryptRequestBodyWrapper包装 HttpServletRequest，RSA 解密 AES 密钥 → AES 解密请求体—
EncryptResponseBodyWrapper包装 HttpServletResponse，生成随机 AES 密钥加密响应体，RSA 加密该 AES 密钥放响应头—
MybatisEncryptInterceptorMyBatis Plugin，拦截 ParameterHandler.setParameters，SQL 前加密字段 → SQL 后 FieldSnapshot.restore() 还原—
MybatisDecryptInterceptorMyBatis Plugin，拦截 ResultSetHandler.handleResultSets，查询结果密文字段解密还原—

常用代码

DB 字段加密 — 实体类

@Data
public class User {
    @EncryptField                           // 走 yml 全局算法
    private String phone;

    @EncryptField(algorithm = AlgorithmType.SM4, password = "my-sm4-key-16ch")
    private String idCard;                  // 逐字段覆盖算法+密钥
}

API 加密 — Controller

@ApiEncrypt                                 // 只解密请求体
@PostMapping("/save")
public R<Void> save(@RequestBody User user) { ... }

@ApiEncrypt(response = true)                // 同时加密响应
@PostMapping("/query")
public R<User> query(@RequestBody QueryReq req) { ... }

手动加解密

// 由 EncryptorManager 注入加密器后使用
encryptorManager.encrypt("13800138000");    // 返回 "ENCRYPT_HEADERxxxx..."
encryptorManager.decrypt(encryptedValue);   // 返回 "13800138000"

// 静态工具类直接调用
EncryptUtils.aesEncrypt(plain, key, EncodeType.BASE64);
EncryptUtils.rsaDecrypt(cipher, privateKey);

yml 配置示例

# DB 字段加密
mybatis-encryptor:
  enable: true
  algorithm: aes          # 默认算法
  password: your-aes-key  # 默认密钥
  encode: base64          # 默认编码

# API 加解密
api-decrypt:
  enabled: true

核心调用链

API 通道:  POST body → CryptoFilter → DecryptRequestBodyWrapper(RSA解AES密钥→AES解body) → Controller → @ApiEncrypt(response=true) → EncryptResponseBodyWrapper(AES加body→RSA加AES密钥放响应头)

DB 通道:   SQL 执行前 → MybatisEncryptInterceptor → EncryptedFieldProcessor.encrypt() → EncryptorManager.encrypt() → SQL 执行 → MybatisEncryptInterceptor → FieldSnapshot.restore()
           SQL 查询后 → MybatisDecryptInterceptor → EncryptedFieldProcessor.decrypt() → EncryptorManager.decrypt()
