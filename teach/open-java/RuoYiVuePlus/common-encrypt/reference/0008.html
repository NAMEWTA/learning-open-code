速查 08 · 完整加密链路速查

对应课程：第 8 课 · 实战追踪：完整加解密链路的追踪与调试

四阶段流程图

Phase 1 — 请求解密 (API 入向) POST /api/user + header:encrypt-key → CryptoFilter.doFilter() → method=POST/ PUT ? check headerFlag → DecryptRequestBodyWrapper(request, privateKey, headerFlag) → headerRSA → RSA解密 → Base64解码 → AES密钥 → 读请求体 → AES解密 → 明文JSON字节 → Controller 收到明文 Phase 2 — 入参加密 (DB 入向) userService.save(user) → ParameterHandler.setParameters() → MybatisEncryptInterceptor.intercept() → encryptedFieldProcessor.encrypt(parameterObject) → getFieldCache() 获取 @EncryptField 字段 → contextFactory.create(field) 组装 EncryptContext → encryptorManager.encrypt(value, context) → check ENC_前缀 → 否 → registAndGetEncryptor → 实例化加密器 → encryptor.encrypt() → 拼接 "ENC_" + 密文 → 创建 FieldSnapshot(原值) → field.set(target, 密文) → invocation.proceed() → SQL INSERT 密文 → finally: snapshot.restore() → 恢复原值 Phase 3 — 出参解密 (DB 出向) userService.getById(id) → ResultSetHandler.handleResultSets() → MybatisDecryptInterceptor.intercept() → invocation.proceed() → ResultSet → Java对象(密文) → encryptedFieldProcessor.decrypt(result) → getFieldCache() 获取 @EncryptField 字段 → encryptorManager.decrypt(value, context) → check ENC_前缀 → 是 → 去除 "ENC_" 前缀 → registAndGetEncryptor → 缓存命中 → encryptor.decrypt() → 明文 → field.set(result, 明文) → Controller 拿到明文 Phase 4 — 响应加密 (API 出向) @ApiEncrypt(response=true) → CryptoFilter: responseFlag=true → EncryptResponseBodyWrapper(response) → 内建 ByteArrayOutputStream 捕获输出 → chain.doFilter() → Spring MVC 写JSON到缓存 → getEncryptContent(response, publicKey, headerFlag) → generateAesPassword() 随机24字节 → Base64编码 → RSA公钥加密 → 放响应头encrypt-key → getContent() 读缓存 → AES加密 → 设置Content-Length → response.getWriter().write(加密内容) → 前端解密

ENCRYPT_HEADER 生命周期

添加: EncryptorManager.encrypt() → Constants.ENCRYPT_HEADER + encryptResult → 写入数据库的值 = "ENC_xxxx" 检查: EncryptorManager.encrypt() 入口 → if (startsWith("ENC_")) return value; (防重复加密) 检查: EncryptorManager.decrypt() 入口 → if (!startsWith("ENC_")) return value; (非加密值跳过) 去除: EncryptorManager.decrypt() 解密前 → StringUtils.removeStart(value, "ENC_")

常见错误速查

现象原因修复
403 "没有访问权限"有 @ApiEncrypt 但无 encrypt-key 请求头前端添加 encrypt-key 头；或去掉 @ApiEncrypt 注解
请求体解密乱码前后端 RSA/AES 密钥不匹配确认前后端使用同一对 RSA 密钥；AES 密钥长度正确
DB 字段未加密（无 ENC_ 前缀）拦截器未注册 / 实体类未扫描到检查 enable=true；检查 typeAliasesPackage 路径
DB 字段双重加密（ENC_ENC_）Snapshot 恢复失败，实体被多次 save检查 finally 块执行；避免同一实体复用
查询结果仍是密文解密拦截器未生效 / ENC_ 前缀不匹配检查 enable=true；确认数据库值是 ENC_ 开头
响应体为空Wrapper 未缓存 / 加密后未写回检查 doFilter 后的 write 调用；检查 flushBuffer
前端解密响应失败前后端 RSA 密钥对不匹配后端公钥加密 → 前端需对应私钥解密
启动抛 IllegalArgumentException全局算法所需密钥缺失补全 password 或 publicKey+privateKey；或用 BASE64
