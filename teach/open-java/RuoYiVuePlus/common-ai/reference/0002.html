速查 02 · SnailAiConfig 自动配置

配套课程：lessons/0002.html · 使命：读懂 ruoyi-common-ai AI 公共模块

源码速览

@AutoConfiguration @ConditionalOnProperty(prefix = "snail-ai", name = "enabled", havingValue = "true") @EnableSnailAiAgent @EnableSnailAiOpenApi public class SnailAiConfig { }

类体为空——所有工作由 4 个注解完成。

4 个注解拆解

注解作用不满足时的后果
@AutoConfiguration 声明为 Spring Boot 3.x 自动配置类，通过 .imports 文件被自动发现 不会作为自动配置加载（但可以手动 @Import）
@ConditionalOnProperty(prefix="snail-ai", name="enabled", havingValue="true") 开关控制：只有 snail-ai.enabled=true 时才激活 整个配置类不加载，AI 栈零开销
@EnableSnailAiAgent 激活聊天网关：注册 Controller、会话管理、消息管道 无聊天接口，/api/snail/chat/** 不可用
@EnableSnailAiOpenApi 激活 OpenAPI 客户端：注册模型调用、API Key 认证 无法调用外部 AI 模型

AutoConfiguration.imports

# 文件位置：META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports # 内容仅一行： org.dromara.common.ai.config.SnailAiConfig

启动时序

1. Spring Boot 扫描每个 jar 的 .imports 文件 2. 发现 SnailAiConfig → 加载类 → 评估 @ConditionalOnProperty 3. snail-ai.enabled 未配置 / =false → 跳过 4. snail-ai.enabled=true → 执行 @EnableSnailAiAgent → 注册聊天 Bean → 执行 @EnableSnailAiOpenApi → 注册 API Bean 5. AI 栈全部就绪

与 SmsAutoConfiguration 对比

维度SnailAiConfigSmsAutoConfiguration
@Bean 方法0 个2 个
需要 @Primary否是（压过 SmsDaoDefaultImpl）
需要 after 排序否是（after DataRedisAutoConfiguration）
简洁原因框架不暴露需替换的接口需替换 SmsDao 实现 + 等 Redis 就绪

启用步骤

# application.yml snail-ai: enabled: true # 开关 openai: # 模型配置（具体 key 参看 Snail AI 官方文档） api-key: sk-xxxx base-url: https://api.openai.com

记忆锚点

  • 4 注解协同：@AutoConfiguration 被发现 → @ConditionalOnProperty 守门 → @Enable* 激活具体能力。
  • 类体为空：所有 Bean 由 @Enable* 内部创建，SnailAiConfig 只负责「总开关」。
  • 开关价值：AI 是非必选模块，@ConditionalOnProperty 避免「没配 API Key 但强制初始化」。
