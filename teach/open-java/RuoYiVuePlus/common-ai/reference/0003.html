三个关键设计

设计注解/代码含义
定点拦截 @RestControllerAdvice(assignableTypes = SnailAiChatGatewayController.class) 只拦截 /api/snail/chat/**，不影响 RuoYi 其它 Controller
最高优先级 @Order(Ordered.HIGHEST_PRECEDENCE) 确保先于全局 GlobalExceptionHandler 匹配异常，防止 Result 被覆盖为 R
响应格式隔离 返回 Result<Void> Snail AI 前端 SDK 按 Result(status/message/data) 解析——返回 R 则前端错误提示失效

8 种异常三层分类

层级异常提示语特殊处理
业务层SnailAiAuthenticationException认证失败，请重新登录status=5001，前端可跳登录页
业务层BaseSnailAiExceptionAI 服务请求失败，请稍后再试业务异常兜底
模型层ModelCallException模型调用失败，请稍后再试含原始异常，log.warn 记堆栈
参数层BindException / MethodArgumentNotValidException / HandlerMethodValidationException（拼接校验错误）三合一处理 @Valid 校验
参数层ConstraintViolationException（拼接约束违反）@Validated 方法参数校验

参数层IllegalStateExceptionAI 会话状态异常，请刷新后重试引导用户刷新而非重试

记忆锚点

  • assignableTypes 定边界：API 路径不同 → 消费者不同 → 响应格式不同 → 必须隔离。
  • @Order 保优先：AI 异常处理器必须比全局处理器先匹配，否则 Result 变 R。
  • 内外有别：预期异常 log.warn，未预期 log.error；对外 message 永远不含堆栈。
  • 三层分类：业务层（框架自身异常）→ 模型层（外部调用失败）→ 参数层（请求不合法）→ 最终兜底。
  • 与 SMS 对比：SMS 把异常「拉入」R 格式；AI 把异常处理「适配」到第三方 Result 格式。方向相反，原因相同。
