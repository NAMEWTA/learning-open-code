速查 01 · ruoyi-common-ai 模块全景

配套课程：lessons/0001.html · 使命：读懂 ruoyi-common-ai AI 公共模块

一句话定位

ruoyi-common-ai 不是 AI 框架，而是把开源 AI Agent 框架 Snail AI Agent（aizuda）收编进 RuoYi 技术栈的「适配垫片」——自己不写一行 AI 逻辑，只补装配和异常两件事。

模块全部内容

文件作用
pom.xml依赖 3 个 Snail AI starter（chat / executor / openapi） + ruoyi-common-core
config/SnailAiConfig.java装配：4 注解激活整个 AI 栈（第 2 课）
handler/SnailAiChatExceptionHandler.java兜底：定点拦截 AI 聊天异常 → SDK Result 格式（第 3 课）
META-INF/spring/...AutoConfiguration.imports发现：声明 SnailAiConfig 为自动配置类

两个类 = 两个职责

类职责解决
SnailAiConfig开关 + 激活snail-ai.enabled=true 一键启停，零配置接入
SnailAiChatExceptionHandler定点异常兜底AI 聊天异常返回 SDK Result 格式，不影响 R 格式接口

3 个 Starter 职责

Starter负责
chat-starter聊天网关（/api/snail/chat/**）、会话管理、消息处理
executor-starter执行器调度、任务执行、多步骤推理
openapi-starterOpenAPI 客户端、模型 HTTP 调用、API Key 认证

记忆锚点

  • 模块零行 AI 调用逻辑——核心能力全在 Snail AI Agent 框架里。
  • 模块不暴露 Controller——SnailAiChatGatewayController 由框架内部自动注册。
  • 与 SMS 模块的关键差异：AI 模块多了「响应格式边界隔离」——因为 Snail AI 有独立前端 SDK，必须保持 Result 格式。
  • snail-ai.enabled=true 是唯一需要手动配置的开关。
