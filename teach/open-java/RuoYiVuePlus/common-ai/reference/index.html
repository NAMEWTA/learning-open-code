ruoyi-common-ai 速查表

Maven: org.dromara:ruoyi-common-ai | 源码: RuoYi-Vue-Plus/ruoyi-common/ruoyi-common-ai/

定位

对 Snail AI Agent（aizuda 社区开源 AI Agent 框架）的项目化适配层。模块仅 2 个 Java 类，通过自动配置实现 snail-ai.enabled=true 即用，通过定点异常处理器保持 Snail AI 前端 SDK 的 Result 响应格式不被 RuoYi 通用 R 格式覆盖（响应格式边界隔离）。

装配套件范式自动配置响应格式边界隔离2 个类

核心文件

类名路径职责
SnailAiConfig org.dromara.common.ai.config 自动配置类，通过 @AutoConfiguration + @ConditionalOnProperty + @EnableSnailAiAgent + @EnableSnailAiOpenApi 实现「配置开关控制、三合一自动激活」
SnailAiChatExceptionHandler org.dromara.common.ai.handler 定点异常处理器，用 @RestControllerAdvice(assignableTypes=...) 精确绑定 Snail AI 聊天网关，@Order(HIGHEST_PRECEDENCE) 最高优先级，覆盖 8 种异常，全部返回 Snail AI SDK 的 Result<Void> 格式

装配套件范式ruoyi-common-* 包装第三方库的标准模式：依赖即生效 + 开关控制 + 异常格式转换

常用代码

启用 AI 模块（application.yml）

snail-ai:
  enabled: true

自动配置核心逻辑

@AutoConfiguration
@ConditionalOnProperty(prefix = "snail-ai", name = "enabled", havingValue = "true")
@EnableSnailAiAgent
@EnableSnailAiOpenApi
public class SnailAiConfig {
}

定点异常处理（关键设计）

@RestControllerAdvice(assignableTypes = SnailAiChatGatewayController.class)
@Order(Ordered.HIGHEST_PRECEDENCE)
public class SnailAiChatExceptionHandler {

    @ExceptionHandler(SnailAiAuthenticationException.class)
    public Result<Void> handleAuthException(SnailAiAuthenticationException e) {
        return Result.fail(5001, "认证失败，请重新登录");
    }

    @ExceptionHandler(BaseSnailAiException.class)
    public Result<Void> handleBaseException(BaseSnailAiException e) {
        return Result.fail(5000, "AI 服务请求失败");
    }
}

与 ruoyi-common-sms 的范式对比

// 相同点：都用自动配置 + 异常处理器收编第三方框架
// 不同点：AI 模块多了「响应格式边界隔离」
//   - AI 必须返回 Result (status/message/data)，因为前端 SDK 按此解析
//   - SMS 可以走 RuoYi 通用的 R (code/msg/data)
//   - AI 无需自定义缓存（Snail AI 不依赖项目 Redis）
