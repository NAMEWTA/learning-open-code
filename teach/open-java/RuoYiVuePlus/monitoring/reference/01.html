一句话核心

monitor-admin 同时作为 SBA Server 和 SBA Client，通过「自注册模式」实现自我监控——打开后台就能看到监控中心自身的健康状态。

三层协作模型

层角色职责关键产物
第 1 层：SBA Server展示与聚合接收客户端上报，在 Web UI 展示@EnableAdminServer + 内嵌 Vue.js UI
第 2 层：SBA Client注册与上报启动时注册，定时心跳 + 状态上报spring.boot.admin.client.url
第 3 层：Actuator数据源暴露 /health、/metrics、/env 等端点management.endpoints.web.exposure.include

关键理解：Client 不产生数据，只是信使——它从 Actuator 采集数据，打包上报给 Server。

核心注解：@EnableAdminServer 自动装配了什么

自动装配项作用
AdminServerAutoConfiguration注册 InstanceRepository（内存存储实例）、启动 HTTP 端点接收上报
AdminServerWebConfiguration挂载 SBA 管理界面（Vue.js SPA），默认路径 /，可配 context-path
NotifierAutoConfiguration扫描所有 Notifier Bean，事件发生时自动调用
UI 静态资源内嵌 spring-boot-admin-server-ui jar

6 种实例状态码

状态码含义中文映射典型触发条件
UP正常在线服务上线/health 返回 UP
OFFLINE离线服务离线心跳超时 / 主动注销
DOWN宕机服务下线/health 返回 DOWN（如 DB 断连）
RESTRICTED受限服务受限限流触发 / 功能降级
OUT_OF_SERVICE停服（计划内）停止服务状态手动标记，预发布验证
UNKNOWN未知服务未知异常刚注册未上报 / 网络中断

核心依赖

<!-- Server：提供管理后台 UI -->
<dependency>
    <groupId>de.codecentric</groupId>
    <artifactId>spring-boot-admin-starter-server</artifactId>
</dependency>

<!-- Client：将自己注册到 Server -->
<dependency>
    <groupId>de.codecentric</groupId>
    <artifactId>spring-boot-admin-starter-client</artifactId>
</dependency>

启动入口

@EnableAdminServer
@SpringBootApplication
public class MonitorAdminApplication {
    public static void main(String[] args) {
        SpringApplication.run(MonitorAdminApplication.class, args);
        System.out.println("Admin 监控启动成功");
    }
}
