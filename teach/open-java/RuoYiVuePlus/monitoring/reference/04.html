速查 · 第 4 课：CustomNotifier 事件驱动通知器 速查

一句话核心

继承 AbstractEventNotifier，覆盖 doNotify()，用 instanceof 过滤 InstanceStatusChangedEvent，用 switch 表达式将 6 种 SBA 状态码映射为中文业务语义并记录日志。

完整源码速览

@Slf4j
@Component
public class CustomNotifier extends AbstractEventNotifier {

    protected CustomNotifier(InstanceRepository repository) {
        super(repository);
    }

    @Override
    protected Mono<Void> doNotify(InstanceEvent event, Instance instance) {
        return Mono.fromRunnable(() -> {
            if (event instanceof InstanceStatusChangedEvent) {
                String registName = instance.getRegistration().getName();
                String instanceId = event.getInstance().getValue();
                String status = ((InstanceStatusChangedEvent) event)
                    .getStatusInfo().getStatus();
                String serviceUrl = instance.getRegistration().getServiceUrl();
                String statusName = switch (status) {
                    case STATUS_UP              -> "服务上线";
                    case STATUS_OFFLINE         -> "服务离线";
                    case STATUS_RESTRICTED      -> "服务受限";
                    case STATUS_OUT_OF_SERVICE  -> "停止服务状态";
                    case STATUS_DOWN            -> "服务下线";
                    case STATUS_UNKNOWN         -> "服务未知异常";
                    default                     -> "未知状态";
                };
                log.info("Instance Status Change: 状态名称【{}】, "
                    + "注册名称【{}】, 实例ID【{}】, 状态【{}】, 服务URL【{}】",
                    statusName, registName, instanceId, status, serviceUrl);
            }
        });
    }
}

继承体系

Notifier (接口)                          ← 定义 notify(InstanceEvent) → Mono<Void>
  └── AbstractEventNotifier (抽象类)       ← 提供骨架：event 类型过滤 + 去重检查
        ├── 持有 InstanceRepository        ← SBA 的内存数据库
        └── 定义 doNotify() 模板方法        ← 子类覆盖
              └── CustomNotifier (本类)     ← 只处理 InstanceStatusChangedEvent

SBA 事件类型速查

事件类型触发时机本 Notifier 是否关注
InstanceRegisteredEvent新实例首次注册否
InstanceDeregisteredEvent实例注销否
InstanceStatusChangedEvent实例状态变化（UP→DOWN 等）是 ★
InstanceInfoChangedEvent元数据或 info 端点信息变更否
InstanceEndpointsDetectedEvent探测到新 actuator 端点否

状态码 → 中文映射表

SBA 状态码中文映射业务含义严重度
STATUS_UP服务上线/health 返回 UP，一切正常正常
STATUS_OFFLINE服务离线实例从注册中心移除（心跳超时/主动注销）警告
STATUS_DOWN服务下线/health 返回 DOWN（进程可能还在但功能坏了）严重
STATUS_RESTRICTED服务受限限流触发或部分功能降级警告
STATUS_OUT_OF_SERVICE停止服务状态手动标记，计划内维护信息
STATUS_UNKNOWN服务未知异常刚注册未上报 / 网络中断警告
default未知状态防御性编程——未来版本新增状态码时不抛异常—

Reactor Mono 操作速查（本类涉及的）

Mono 操作含义本类是否使用
Mono.fromRunnable()将同步代码包装成 Mono<Void>，代码在订阅时执行是
Mono.fromCallable()包装有返回值的同步代码为 Mono<T>否
Mono.just()包装已知值否
.subscribeOn(Schedulers.boundedElastic())将慢操作调度到弹性线程池否（扩展时建议）

事件处理完整数据流

1. Client 心跳上报 → Server 更新 Instance 状态
2. Server 对比新旧状态 → 发现变化（如 UP→DOWN）
3. Server 发布 InstanceStatusChangedEvent 到事件总线
4. 事件总线遍历所有 Notifier Bean
5. AbstractEventNotifier.notify(event)
   ├→ event type 过滤 → 匹配，继续
   └→ doNotify(event, instance)
6. CustomNotifier.doNotify()
   ├→ instanceof InstanceStatusChangedEvent ✓
   ├→ 提取 registName, instanceId, status, serviceUrl
   ├→ switch 映射 status → 中文
   └→ log.info(...)
7. 日志落盘 → ELK/Splunk 可搜索到状态变化

设计要点

为什么返回 Mono<Void> 而非 void？SBA 事件总线是响应式的，所有 Notifier 返回 Mono 以支持非阻塞调度。多个 Notifier 可并发执行，慢操作不阻塞事件分发。扩展真实告警时，用 .subscribeOn(Schedulers.boundedElastic()) 将慢 I/O 挪到弹性线程池。
静态导入常量的意义：import static ...StatusInfo.* 将 STATUS_UP/DOWN 等常量直接导入命名空间，避免写 StatusInfo.STATUS_UP 冗长前缀。switch 表达式（Java 14+）箭头语法自动 break，不穿透。
default 分支是防御性编程：如果 SBA 未来版本新增状态码（如 STATUS_STARTING），不会因为缺少 case 分支而抛异常。日志仍然打印，状态名为"未知状态"。
当前实现只做 log.info()，不涉及真正异步。扩展为真实告警（邮件/钉钉/企微/短信）时，必须在 Mono.fromRunnable() 后链式调用 .subscribeOn() 指定调度器，否则慢 I/O 会阻塞事件分发线程。
