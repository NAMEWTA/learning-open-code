一句话核心

用 6 项安全策略保护 SBA 管理后台：依赖 AdminServerProperties 动态获取 context-path，实现路径无关的安全过滤链。

完整源码速览

@EnableWebSecurity
@Configuration
public class SecurityConfig {

    private final String adminContextPath;  // 从 AdminServerProperties 获取

    public SecurityConfig(AdminServerProperties adminServerProperties) {
        this.adminContextPath = adminServerProperties.getContextPath();
    }

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        SavedRequestAwareAuthenticationSuccessHandler successHandler =
            new SavedRequestAwareAuthenticationSuccessHandler();
        successHandler.setTargetUrlParameter("redirectTo");
        successHandler.setDefaultTargetUrl(adminContextPath + "/");

        PathPatternRequestMatcher.Builder mvc =
            PathPatternRequestMatcher.withDefaults();

        return http
            // ① .headers(h -> h.frameOptions(disable))
            // ② .authorizeHttpRequests(auth -> auth
            //         .requestMatchers(mvc.matcher(ctx+"/assets/**"),
            //                          mvc.matcher(ctx+"/login")).permitAll()
            //         .anyRequest().authenticated())
            // ③ .formLogin(f -> f.loginPage(ctx+"/login")
            //                      .successHandler(successHandler))
            // ④ .logout(l -> l.logoutUrl(ctx+"/logout"))
            // ⑤ .httpBasic(Customizer.withDefaults())
            // ⑥ .csrf(AbstractHttpConfigurer::disable)
            .build();
    }
}

6 项安全策略逐项速查

#策略配置方式原因风险 / 注意
① 禁用 frameOptions header.frameOptions(disable) SBA 日志查看器等面板在 iframe 中渲染，X-Frame-Options: DENY 会导致白屏 clickjacking 风险。SBA 通常内网部署，风险可控。公网部署建议用 CSP frame-ancestors
② 放行静态资源 + 登录页 .requestMatchers("/assets/**","/login").permitAll() /assets/** 不放行则登录页无样式；/login 不放行则死循环（需登录才能看登录页） 其余所有请求均需认证
③ 表单登录 .formLogin().loginPage().successHandler() 人（运维人员）通过浏览器登录。SavedRequest 登录后回到拦截前的页面 使用 SBA 自带登录页，非自定义
④ 注销 .logout().logoutUrl() POST /admin/logout 清除 SecurityContext 和 Session 依赖 CSRF 已禁用
⑤ HTTP Basic 认证 .httpBasic(withDefaults()) 程序（SBA Client）通过 HTTP 头传递凭证注册 + Server 回调 actuator 与表单登录并存，服务不同类型的"用户"
⑥ 禁用 CSRF .csrf(disable) SBA Vue.js UI 的 AJAX 请求不带 CSRF token；Client HTTP Basic 也不支持 CSRF SBA 官方推荐配置，非偷懒

两种认证方式的使用方

认证方式使用者场景
表单登录（formLogin）人（运维人员）浏览器打开 SBA 管理后台，输入账号密码
HTTP Basic（httpBasic）程序（SBA Client）Client 向 Server 注册；Server 回调 Client 的 actuator 端点

安全链路全景

┌──────────────────────────────────────────────┐
│  浏览器访问 http://localhost:9090/admin       │
│  ├─ 未登录 → 拦截到 /admin/login              │
│  ├─ 登录成功 → SavedRequest → 回原页面         │
│  ├─ /assets/** → permitAll（样式/JS 加载）     │
│  └─ 其他 API → 必须认证                       │
├──────────────────────────────────────────────┤
│  SBA Client 注册                              │
│  ├─ POST /admin/instances                     │
│  ├─ Authorization: Basic <base64(user:pass)> │
│  └─ httpBasic() 认证通过 → 写入仓库            │
├──────────────────────────────────────────────┤
│  frameOptions: disable（iframe 正常加载）       │
│  CSRF: disable（AJAX 不报 403）                │
└──────────────────────────────────────────────┘

关键设计决策

为什么注入 AdminServerProperties 而不是硬编码 /admin？——无论你把 SBA 挂载到 /admin、/monitor 还是 /，所有安全规则自动适配。改一处 YML，安全规则自动跟随。
为什么用 PathPatternRequestMatcher？——Spring Security 6.x 新默认匹配器，性能优于 Ant 风格，天然支持 {variable} 路径变量。RuoYi-Vue-Plus 全系统统一使用。
禁用 frameOptions 的安全影响——页面可被其他网站用 iframe 嵌入（clickjacking 风险）。SBA 是内部运维工具，通常在内网，攻击面有限。公网部署建议用 CSP 替代。
