YML 四段结构总览

段落分隔符配置内容核心配置项
段 1（顶部）基础服务server.port: 9090, spring.application.name
段 2--- # 监控中心服务端配置SBA Servercontext-path: /admin, spring.security.user.*
段 3--- # Actuator 监控端点Actuator 数据暴露include: '*', show-details: ALWAYS
段 4--- # 监控配置SBA Client 自注册url, service-host-type: IP, metadata

段 4（Client 自注册）逐行解释 — 本课核心

spring.boot.admin.client:
  enabled: true                         # 启用客户端注册
  url: http://localhost:9090/admin      # ★ 指向自己！实现自注册
  instance:
    service-host-type: IP                # ★ 用 IP 注册，避免 hostname DNS 依赖
    metadata:                            # 自定义元数据
      username: ${spring.boot.admin.client.username}
      userpassword: ${spring.boot.admin.client.password}
  username: @monitor.username@           # Client 注册凭证（Maven变量）
  password: @monitor.password@           # Client 注册凭证（Maven变量）

启动流程（自注册时序）

1. Spring Boot 启动 → 加载所有 AutoConfiguration
2. SBA Server AutoConfig → 启动内嵌容器，监听 9090，挂载 /admin UI
3. SBA Client AutoConfig → 读取 url=http://localhost:9090/admin
4. Client 向该 URL POST 注册请求 → Server 写入 InstanceRepository
5. Client 启动心跳线程 → 定时向 Server 报告状态
6. 打开 http://localhost:9090/admin → 看到 monitor-admin 自身

service-host-type 三种模式对比

值注册地址格式适用场景
IP（本项目）http://192.168.1.100:9090生产环境首选，无 DNS 依赖
HOSTNAME（默认）http://my-server:9090内网 DNS 完善时可用
CANONICAL_HOST_NAMEhttp://my-server.example.com:9090需要 FQDN 的场景

Actuator 配置三件套

配置项作用不配的后果
include: '*'暴露所有 actuator 端点后台只看到在线/离线，无指标
show-details: ALWAYS健康检查返回完整细节看不到哪个组件挂了
logfile.external-file指定日志文件路径后台日志查看器无内容

两套认证体系区分（易混淆点）

认证配置位置用途
后台登录认证spring.security.user.name/password人（运维）通过浏览器登录 SBA 管理后台
Client 注册认证spring.boot.admin.client.username/password程序（SBA Client）向 Server 注册 + Server 回调 Client actuator 端点

常见翻车点

context-path 与 client.url 不同步：改了 context-path: /admin 但忘记同步改 client.url 中的路径，导致 Client 注册到错误地址，后台看不到任何实例。
生产环境用 HOSTNAME 模式：hostname 解析失败时，其他微服务无法访问健康端点。生产环境强烈建议用 service-host-type: IP。
metadata 凭证传递：Client 把自己的账密通过 metadata 告诉 Server，Server 拿着这些凭证去调 Client 的受保护 Actuator 端点。这是 SBA 框架层面的机制。

完整 YML（速览）

server.port: 9090
spring.application.name: ruoyi-monitor-admin

--- # 监控中心服务端配置
spring.security.user.name: @monitor.username@
spring.security.user.password: @monitor.password@
spring.boot.admin.ui.title: RuoYi-Vue-Plus服务监控中心
spring.boot.admin.context-path: /admin

--- # Actuator 端点
management.endpoints.web.exposure.include: '*'
management.endpoint.health.show-details: ALWAYS
management.endpoint.logfile.external-file: ./logs/ruoyi-monitor-admin.log

--- # Client 自注册
spring.boot.admin.client:
  enabled: true
  url: http://localhost:9090/admin
  instance.service-host-type: IP
  instance.metadata.username: ${...client.username}
  instance.metadata.userpassword: ${...client.password}
  username: @monitor.username@
  password: @monitor.password@
