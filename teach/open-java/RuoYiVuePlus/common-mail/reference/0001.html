速查 01 · ruoyi-common-mail 模块全景

配套课程：lessons/0001.html · 使命：读懂 ruoyi-common-mail 邮件模块

一句话定位

ruoyi-common-mail 不是邮件发送库，而是对 Hutool JakartaMail 的 Builder 模式流畅封装 —— 三层架构：MailProperties（配置映射）→ MailConfig（装配Bean）→ MailBuilder（流式发送）。

模块全部内容

文件作用
pom.xml依赖 ruoyi-common-core + jakarta.mail-api + angus-mail
config/MailConfig.java装配：注册 MailAccount Bean（第 2 课）
config/properties/MailProperties.java映射：yml 的 mail.* → POJO + toMailAccount() 转换（第 2 课）
core/MailBuilder.java发送：流式 Builder API（第 3 课）
META-INF/spring/...AutoConfiguration.imports发现：登记 MailConfig 为自动配置类

三层架构 = 三件事

层类职责
数据层MailProperties绑定 yml 10 个属性，提供 toMailAccount() 转换
装配层MailConfig激活属性绑定，根据 mail.enabled 开关条件创建 MailAccount Bean
API 层MailBuilder流式链式调用，send() 结束构建并发送

与 ruoyi-common-sms 对比

维度mailsms
底层库Hutool JakartaMailsms4j 3.3.5
异常处理无（抛原生异常）SmsExceptionHandler（@RestControllerAdvice）
缓存无（一次性IO）PlusSmsDao（Redis 拦截计数）
启用方式@ConditionalOnProperty(mail.enabled)无条件装配 + @Primary 覆盖
API 风格Builder 链式调用直接调 sms4j API
Properties 类✅ MailProperties❌（sms4j 内部解析）

最短发送链路

yml(mail.enabled=true, host, port...) → MailProperties 绑定 → MailConfig.mailAccount() 创建 MailAccount Bean → MailBuilder.of().to().subject().text().send() → validate() 校验 → resolveMailAccount() 拿 Bean / 克隆覆盖 → JakartaMail.create(account).send() → SMTP 服务器

记忆锚点

  • 模块 零行 SMTP 协议代码，全委托 Hutool + Jakarta Mail。
  • MailBuilder 是 private 构造器 + 静态工厂 of()，标准的 Builder/Fluent API 实现。
  • mail.enabled=false 时不创建 MailAccount Bean，send() 会抛 NoSuchBeanDefinitionException。
  • 与 sms 模块的最大不同：mail 有 Builder 模式和条件开关，sms 有异常处理器和 Redis 缓存。
