速查 02 · MailConfig / MailProperties 配置详解

配套课程：lessons/0002.html

一句话

MailProperties 绑定 yml 的 10 个 mail.* 属性，toMailAccount() 把它们转成 Hutool MailAccount；MailConfig 用 @ConditionalOnProperty(mail.enabled) 控制整个模块的启停。

MailProperties 的 10 个字段

字段yml key类型典型值
enabledmail.enabledBooleanfalse / true
hostmail.hostStringsmtp.163.com
portmail.portInteger465(SSL) / 587(STARTTLS)
authmail.authBooleantrue
usermail.userStringxxx@163.com
passmail.passStringSMTP 授权码
frommail.fromStringxxx@163.com 或 名称<xx@xx.com>
starttlsEnablemail.starttlsEnableBooleantrue
sslEnablemail.sslEnableBooleantrue
timeoutmail.timeoutLongnull / 10000
connectionTimeoutmail.connectionTimeoutLongnull / 5000

toMailAccount() 逐个 set 的 10 个调用

account.setHost(host) .setPort(port) .setAuth(auth) .setFrom(from) .setUser(user) .setPass(pass) .setSocketFactoryPort(port) ← 注意：= port，SSL 同端口 .setStarttlsEnable(starttlsEnable) .setSslEnable(sslEnable) .setTimeout(timeout) .setConnectionTimeout(connectionTimeout)

MailConfig 的 4 个注解

注解位置作用
@AutoConfiguration类上标记自动配置类，通过 .imports 文件被发现
@EnableConfigurationProperties(MailProperties.class)类上注册 MailProperties 为 Bean + 触发 yml 绑定
@BeanmailAccount() 方法创建 MailAccount Bean，存入 Spring 容器
@ConditionalOnProperty(value="mail.enabled", havingValue="true")mailAccount() 方法开关：只有 mail.enabled=true 时才创建 Bean；false 时容器中无 MailAccount，send() 会抛 NoSuchBeanDefinitionException

启动时序（5 步）

Boot 启动 → 扫 .imports → 发现 MailConfig → @EnableConfigurationProperties → MailProperties 绑定 yml → @ConditionalOnProperty 检查 mail.enabled ├─ true → @Bean mailAccount() → new MailAccount ← toMailAccount() └─ false → 跳过，不创建 Bean → 启动完成

与 SmsAutoConfiguration 对比

MailConfigSmsAutoConfiguration
加载顺序无约束after = DataRedisAutoConfiguration
Bean 策略条件创建（@ConditionalOnProperty）无条件创建（@Primary 覆盖默认）
注册 Bean 数1 个2 个

记忆锚点

  • @EnableConfigurationProperties 不写，MailProperties 字段全是 null。
  • @ConditionalOnProperty 不满足 → 无 MailAccount Bean → send() 炸。
  • socketFactoryPort = port 是因为 SSL SMTP 握手和通信同一端口。
  • yml 里 pass 填的是 SMTP 授权码，不是邮箱登录密码。
