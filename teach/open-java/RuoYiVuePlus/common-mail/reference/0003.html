速查 03 · MailBuilder 流式构建器

配套课程：lessons/0003.html

一句话

400 行的 私有构造器 + 静态工厂 + 链式 API 的 Builder 模式实现——设参数（to/cc/subject/text/html/files/image）→ 校验 → 拿账户（可能克隆）→ JakartaMail.send() → 关闭图片流 → 返回 message-id。

两个静态工厂入口

方法使用场景
MailBuilder.of()常规发送，使用 yml 全局 MailAccount Bean
MailBuilder.of(MailAccount)多账户/动态账户发送

send() 执行路径（7 步）

① validate() — 校验收件人/标题/正文必填 ② resolveMailAccount() — 解析账户 ├─ mailAccount!=null && 无覆盖 → 返回传入的 ├─ mailAccount==null && 无覆盖 → SpringUtils.getBean(MailAccount.class) └─ 有覆盖(from/user/pass) → ObjectUtil.clone + 字段覆盖 ③ JakartaMail.create(account).setUseGlobalSession(...) ④ setTos / setCcs / setBccs / setTitle / setContent / setHtml / setFiles ⑤ try { images.forEach(mail::addImage) } ⑥ mail.send() → 返回 message-id ⑦ finally { images.forEach(IoUtil::close) }

条件返回值

记忆锚点

  • private 构造器：强制通过 of() 创建，符合 Builder 模式规范。
  • 覆盖字段 → 克隆账户：不污染全局 Bean，ObjectUtil.clone() 基于序列化做深拷贝。
  • files() 是覆盖式：第二次调用会覆盖第一次的附件列表。
  • try-finally 关图片流：防止资源泄漏，即使发送失败也关。
  • to/cc/bcc 支持逗号分号混合分隔：splitAddress() 中先 replace(';',',') 再 splitTrim。
  • text() 设 html=false，html() 设 html=true，content(str,bool) 自由指定。
