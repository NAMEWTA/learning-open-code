速查 05 · API 文档生成实战与排错

配套课程：lessons/0005.html · 使命：读懂 ruoyi-common-doc 接口文档模块

从零到文档的完整步骤

1. yml 确认：springdoc.api-docs.enabled = true 2. 写 Controller（@RestController + JavaDoc /** */ + 可选 SaToken 注解） 3. 启动 → 访问 /swagger-ui/index.html 4. 验证 /v3/api-docs JSON 输出

依赖关系图

ruoyi-common-doc ├── ruoyi-common-core (工具类) ├── springdoc-openapi (文档扫描+生成+UI) │ ├── spring-boot-web-server (读 contextPath) │ └── jackson-module-kotlin (Kotlin 序列化) ├── therapi-runtime-javadoc (JavaDoc 运行时读取) └── sa-token (可选——有则解析权限，无则跳过)

问题 1：Swagger UI 无 API 显示

检查点排查方法
基础端点访问 /v3/api-docs 看 paths 是否为空
扫描路径Controller 包名是否在 @SpringBootApplication 扫描范围内
注解Controller 是否有 @RestController / @Controller
配置开关springdoc.api-docs.enabled 是否为 true

问题 2：无 JavaDoc 中文描述

检查点排查方法
注释格式必须用 /** ... */，不是 // 或 /*
Maven 编译检查 class 文件同目录是否有 .javadoc.json
插件配置父 pom 中 therapi-runtime-javadoc 插件是否正确配置
JavadocProvider注入的 Optional<JavadocProvider> 是否为 empty

问题 3：无权限说明

检查点排查方法
sa-token jarclasspath 是否有 sa-token 依赖（最常见原因）
注解位置方法上 OR 类上都支持（supports 检查 both）
@SaIgnore 短路是否被 @SaIgnore 拦截导致不再解析权限
反射异常resolvePermissionAnnotation 的 try-catch 空吞异常——临时加日志排查
Bean 注册SaTokenAnnotationMetadataJavadocResolver 是否被注册为 Bean
启动日志debug 模式下检查 "SaTokenAnnotationJavadocResolver init success" 日志

path 前缀防重复关键代码

static class PlusPaths extends Paths { } // 空子类，只做标记 openApiCustomizer lambda: if (oldPaths instanceof PlusPaths) return; // ← 防重复的核心 PlusPaths newPaths = new PlusPaths(); oldPaths.forEach((k, v) → newPaths.addPathItem(contextPath + k, v)); openApi.setPaths(newPaths);
