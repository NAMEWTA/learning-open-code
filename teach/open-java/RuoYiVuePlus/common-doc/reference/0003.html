JavadocOperationCustomizer 两阶段

阶段一：取 JavadocProvider → getMethodJavadocDescription(method) → getFirstSentence(desc) → operation.summary（首句） → desc → operation.description（全文） 阶段二：遍历 javadocResolvers → .sorted() → 按 getOrder() 升序 → .filter(supports(hm)) → 只调支持的 → .map(resolve(hm, op)) → 执行解析 → .filter(notBlank) → 过滤空结果 → description.append → 拼到原描述末尾

ClassTagOperationCustomizer Tag 替换流程

customize(op, hm): ① 有 @Tag/@Tags 注解？→ 用注解值（尊重显式声明）→ return ② 无注解 → getClassJavadocTagName(beanType) 取类 JavaDoc 第一行（IoUtil.readLines → filter notBlank → findFirst） ③ autoTagName = splitCamelCase(beanType.getSimpleName()) 例: SysUserController → "Sys User Controller" ④ shouldUseClassJavadocTag？operation 无 tags 或包含 autoTagName → true ⑤ removeOperationTag(autoTagName) + addOperationTag(javadocTagName) replacedAutoTagNames.add(autoTagName) tags.putIfAbsent(javadocTagName, new Tag(...)) customise(openApi): ① openApi.getTags().removeIf(tag → replacedAutoTagNames.contains(tag.getName())) ② tags 中有但 openApi.tags 中没有的 → addTagsItem

JavadocResolver 扩展点接口

方法职责
supports(HandlerMethod)判断是否处理该方法（如：有 @SaCheckPermission？）
resolve(HandlerMethod, Operation)解析并返回追加的描述文本
getOrder()排序（default = LOWEST_PRECEDENCE）
compareTo(JavadocResolver)按 order 比较（default 实现）

AbstractMetadataJavadocResolver 工具方法

泛型 <M> 管理元数据对象 Supplier<M> 每次 resolve 创建新实例（避免状态污染） 注解检测：hasMethodAnnotation / hasClassAnnotation / hasAnnotation 值读取： getMethodAnnotationValueMap / getClassAnnotationValueMap 支持 Class 参数和字符串类名两种方式

新增 Resolver 三步法

1. 写类继承 AbstractMetadataJavadocResolver<YourMeta> 实现 supports() + resolve(hm, op, meta) 2. 在任意 @Configuration 中注册 @Bean 3. Spring 自动注入到 List<JavadocResolver> → 零修改现有代码
