速查 02 · SpringDocConfig 注解与 Bean 注册

配套课程：lessons/0002.html · 使命：读懂 ruoyi-common-doc 接口文档模块

类上 4 个注解

注解作用为什么
@RequiredArgsConstructor为 final 字段生成构造器构造器注入 ServerProperties
@AutoConfiguration(before=SpringDocConfiguration)在本配置类在 SpringDoc 默认配置之前加载确保 @ConditionalOnMissingBean(OpenAPI) 能覆盖默认 Bean
@EnableConfigurationProperties(SpringDocProperties)激活配置属性绑定从 yml springdoc.* 段读取配置
@ConditionalOnProperty(name="springdoc.api-docs.enabled")条件开关false 时整个模块跳过

SpringDocProperties 映射关系

字段yml 路径目标 OpenAPI 属性
InfoProperties infospringdoc.info.*openApi.info
ExternalDocumentation externalDocsspringdoc.external-docs.*openApi.externalDocs
List<Tag> tagsspringdoc.tags[0].nameopenApi.tags

Components componentsspringdoc.components.security-schemes.*openApi.components

InfoProperties 为什么是内部类

原生 io.swagger.v3.oas.models.info.Info 没有 Spring Boot 元数据注解，IDE 无法 yml 补全。 自定义 InfoProperties 每个字段有 Javadoc → Spring Boot 扫描生成 spring-configuration-metadata.json → IDE 智能提示。

PlusPaths 防重复机制

步骤逻辑
第 1 次回调oldPaths 是普通 Paths → 创建 PlusPaths → 路径加前缀 → setPaths
第 N 次回调oldPaths instanceof PlusPaths → return（跳过）
PlusPaths 是空子类（5 行），不做任何逻辑，只用作 instanceof 标记。标记信息附着在对象类型上，不受 lambda 重建影响。

openApi() 安全方案构建

components.securitySchemes 的 keySet → 为每个 key 创建 SecurityRequirement → 设到 openApi.security → Swagger UI 出现 "Authorize" 按钮，每个请求自动带 Authorization header。
