为什么用字符串类名而非 import

ruoyi-common-doc 不依赖 sa-token jar。 用 ClassLoaderUtil.loadClass("cn.dev33.satoken.annotation.SaCheckPermission") 动态加载： classpath 有 sa-token → Class 实例正常 → 解析注解 → 生成权限文档 classpath 无 sa-token → loadClass 返回 null → supports() 返回 false → 静默跳过 这是 可选依赖（Optional Dependency） 的标准实现模式。

加载的 4 个注解

常量全限定类名用途
SA_CHECK_PERMISSION_CLASScn.dev33.satoken.annotation.SaCheckPermission权限校验
SA_CHECK_ROLE_CLASScn.dev33.satoken.annotation.SaCheckRole角色校验
SA_IGNORE_CLASScn.dev33.satoken.annotation.SaIgnore忽略校验
SA_CHECK_LOGIN_CLASScn.dev33.satoken.annotation.SaCheckLogin需登录（仅加载，未用于 supports）

supports() 判断逻辑

hasAnnotation(hm, SA_CHECK_ROLE_CLASS) // 方法或类上有 @SaCheckRole || hasAnnotation(hm, SA_CHECK_PERMISSION_CLASS) // 方法或类上有 @SaCheckPermission || hasAnnotation(hm, SA_IGNORE_CLASS) // 方法或类上有 @SaIgnore → true 时该 Resolver 介入 注意：不含 @SaCheckLogin —— 因为 metadata 默认就显示「需要登录」。

resolve() 三段式解析

1. @SaIgnore？→ metadata.ignore = true → return "忽略权限检查"（短路） 2. resolvePermissionCheck()：方法注解 → 类注解（叠加） 反射读 value/mode/type/orRole → metadata.addPermission() 3. resolveRoleCheck()：方法注解 → 类注解（叠加） 反射读 value/mode/type → metadata.addRole() → return metadata.toMarkdownString()

SaTokenSecurityMetadata 四种输出策略

条件输出
ignore = true权限策略：忽略权限检查
!ignore && permissions 空 && roles 空权限策略：需要登录
permissions 非空权限校验：`perm1` & `perm2`
或角色：`role1`
roles 非空角色校验：`role1` | `role2`

端到端示例

Controller: /** 系统用户管理 */ public class SysUserController { /** 查询用户列表 */ @SaCheckPermission({"system:user:list","system:user:query"}, mode=AND) @GetMapping("/list") public TableDataInfo list() { ... } } Swagger UI 输出: 分组: "系统用户管理" GET /list 摘要: "查询用户列表" 详情: "查询用户列表 访问权限 权限校验：`system:user:list` & `system:user:query`"
