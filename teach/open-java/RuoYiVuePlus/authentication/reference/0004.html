SaInterceptor 三道关卡（SecurityConfig:81）

① StpUtil.checkLogin()验 token，无效抛 NotLoginException
② 客户端一致性请求头 clientid == token extra clientid，否则 401「-100 客户端ID与Token不匹配」
③ validateClientAccessRulesaccessPath 路径 + ipWhitelist IP（来自 sys_client），不符抛 NotPermissionException

放行：excludes 白名单 + @SaIgnore 注解。

StpInterface 权限加载（SaPermissionImpl:32）

注解来源
@SaCheckLogintoken 会话
@SaCheckPermissiongetPermissionList → menuPermission
@SaCheckRolegetRoleList → rolePermission

优化：查当前用户优先从内存 LoginUser 取（登录已装好），跨用户才走 PermissionService。

异常 → 响应（SaTokenExceptionHandler）

NotPermission/NotRole403 没有访问权限
NotLogin · TOKEN_TIMEOUT401 登录已过期
· BE_REPLACED401 已在其他设备登录被下线
· KICK_OUT401 被管理员强制下线
· TOKEN_FREEZE401 账号被冻结

401 vs 403

401=认证失败(没登录/失效)→前端重登；403=授权失败(登录了权限不够)。客户端不一致抛 401 因属认证问题。

→ 术语：StpInterface、SaInterceptor、权限码
