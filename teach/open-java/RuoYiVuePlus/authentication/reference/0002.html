分发机制（IAuthStrategy:31）

beanName = grantType + "AuthStrategy" → SpringUtils.getBean(beanName)。不用 switch，靠 Bean 名约定 → 开闭原则。

sys_client 关键字段

grantType该端允许的授权类型（逗号分隔）
deviceType设备类型，Sa-Token 多设备维度
timeout/activeTimeouttoken 固定/活跃超时
accessPath/ipWhitelist访问路径/IP 限制（第4课生效）
status0 正常 1 停用

AuthController 两道客户端校验（:83）

① client 存在且 grantType 包含该类型 → 否则 fail；② status 正常 → 否则 blocked。通过才分发。

新增登录方式只动三处

1) 新策略类 + 对的 Bean 名；2) sys_client.grantType 加值；3) 前端传新 grantType。Controller/分发/LoginHelper 不动。

→ 术语：grantType、授权策略、sys_client（见 GLOSSARY.md）
