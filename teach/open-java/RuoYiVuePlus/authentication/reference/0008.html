核心洞察

两个前端面对同一后端契约，职责分工相同，仅实现工具不同。接新端（如 Flutter）= 前端按契约重写四职责 + 后端 sys_client 加一行配置（沿用已有 grantType 则零改代码）。

全栈链路（密码登录）

store.login →(+clientId+grantType,AES加密)→ AuthController(@ApiEncrypt) → CryptoFilter解密 → 查sys_client → IAuthStrategy分发 → 策略验身份 → buildLoginUser → LoginHelper/StpUtil.login(JWT+Redis) → LoginVo → setToken → getInfo → 动态路由。后续请求 Bearer+clientid → SaInterceptor三关 → @SaCheckPermission/StpInterface → 异常 401/403 → 前端重登。
