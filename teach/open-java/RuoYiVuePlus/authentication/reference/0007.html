四职责分工

职责文件
存 token（Admin-Token）utils/auth.ts（useStorage）
登录态/用户信息store/modules/user.ts（Pinia）
请求加工+401utils/request.ts
路由准入+动态路由permission.ts（beforeEach）

登录态

login 只存 access_token；getInfo 另拉 roles(路由权限)+permissions(按钮权限)+user。logout 清空+removeToken。

请求拦截器

默认带 clientid 头（第4课校验用）；有 token 且非 isToken 加 Bearer；isEncrypt 做 AES+RSA；非 repeatSubmit 的 post/put 做 500ms 防重。

401 处理

isRelogin.show 幂等锁（并发只弹一次）；ElMessageBox 询问而非强跳，可留页面。对应后端 SaTokenExceptionHandler。

路由守卫 beforeEach

无 token→白名单放行/否则跳登录；有 token 且 roles 空→getInfo→generateRoutes→addRoute→放行；roles 已有→直接放行。刷新会重走（store 内存态清空）。

权限两级

路由级=roles；按钮级=permissions + v-hasPermi（含 *:*:* 放行）。前端隐藏是体验，后端 @SaCheckPermission 才是安全（纵深防御）。

social 前端

/auth/binding/{source}→跳转→回调 /social-callback(白名单)→callback(grantType=social)。

→ 术语：动态路由、纵深防御、v-hasPermi
