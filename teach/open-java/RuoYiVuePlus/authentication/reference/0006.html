三种「加密」别混淆

名字解决算法
密码存储库脱也无明文BCrypt 单向
token身份凭证JWT 签名
接口加解密传输不被窃听AES+RSA 混合

一次登录三者同时用，正交。

混合加密为何

AES 快但传钥难；RSA 安全传钥但慢。→ AES 加密 body，RSA 公钥加密 AES 密钥。同 TLS 思路。

前端（request.ts:155）

① generateAesKey() 一次性密钥 → ② encrypt(base64(key)) RSA 加密放 encrypt-key 头 → ③ encryptWithAes(body) AES(ECB/Pkcs7)。

后端（CryptoFilter:64）

Filter 层（Controller 之前）：有 encrypt-key 头 → DecryptRequestBodyWrapper（RSA 私钥解 AES 密钥 → AES 解 body）。有 @ApiEncrypt 但无加密头 → 403 拒绝。response()=true 则响应也加密。

为何在 Filter 层

请求体是一次性流，@RequestBody 读完就没了。Wrapper 重写 getInputStream 返回明文，下游 MVC 无感。故登录 Controller 能用 @RequestBody String 拿明文。

@ApiEncrypt 接口

/auth/login、/auth/register（强制加密）。

→ 术语：ApiEncrypt、AES+RSA 混合加密、CryptoFilter
