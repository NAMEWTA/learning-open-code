LoginHelper.login() 三件事（:54）

① fillRequestContext补 IP/浏览器/OS
② StpUtil.login(loginId, model+extra)签发 token，塞 userId/userName/deptId/clientid
③ TokenSession.set(LOGIN_USER_KEY, loginUser)完整 LoginUser 存会话

loginId 格式

userType + ":" + userId（如 sys_user:1）→ 支持多用户体系，前缀隔离权限。

JWT vs Redis 分工（核心）

JWT(token本身)Redis(PlusSaTokenDao)
存loginId+extra,签名完整LoginUser+会话状态
角色身份凭证,可离线验签会话数据+可控状态
价值少查库能踢人/改权限即时生效/查在线

配置：StpLogicJwtForSimple（简单模式）= JWT 身份 + Redis 会话。

PlusSaTokenDao 多级缓存

Caffeine(本地,5s 过期) + Redis。getCacheValue 本地没有才查 Redis；写操作 invalidate 本地。高并发挡 Redis 往返，5s 保证踢人延迟可接受。

取登录用户

LoginHelper.getLoginUser() → StpUtil.getTokenSession().get(LOGIN_USER_KEY)；轻量字段 StpUtil.getExtra("userId")。

satoken 配置要点

token-prefix:Bearer · is-read-header:true · is-read-cookie:false（杜绝 CSRF）· dynamic-active-timeout:true
