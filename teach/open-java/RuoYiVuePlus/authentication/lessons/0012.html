第 12 课 · permission.ts 动态路由：后端 JSON → Vue Router plus-ui-vue

🎯 本节目标：学会追踪 permission.ts 如何将后端返回的菜单 JSON 数组逐步转换为 Vue Router 可用的路由表。重点理解三步变换：拉取路由 → filterAsyncRouter（JSON→组件）→ router.addRoute（注册）。

一、预建视图查找表：O(1) 定位 .vue 文件

// store/modules/permission.ts:14-21
const modules = import.meta.glob('./../../views/**/*.vue');
const viewModuleMap = new Map<string, () => Promise<any>>();
for (const path in modules) {
  const viewsIndex = path.indexOf('/views/');
  if (viewsIndex === -1) continue;
  const dir = path.substring(viewsIndex + 7, path.lastIndexOf('.vue'));
  viewModuleMap.set(dir, modules[path] as () => Promise<any>);
}

import.meta.glob 是 Vite 的静态文件扫描 API。它在构建时把 views/ 下所有 .vue 文件路径收集到一个对象：

// 构建产物等价于：
{
  "/views/system/user/index.vue": () => import("/views/system/user/index.vue"),
  "/views/system/role/index.vue": () => import("/views/system/role/index.vue"),
  ...
}

然后将路径标准化为 key（去掉 /views/ 前缀和 .vue 后缀），存入 Map——后续 loadView("system/user/index", "User") 直接 viewModuleMap.get("system/user/index")，O(1) 查找。

⚠️ 为什么不能用 Vite 的动态路径拼接？ import(\`./views/\${path}\`) 会产生运行时未知模块，Vite 无法在构建时分析，导致生产环境 404。这就是 import.meta.glob 存在的意义——编译时确定所有可能路径。

二、generateRoutes：动态路由生成流水线

// store/modules/permission.ts:55-75
const generateRoutes = async (): Promise<RouteRecordRaw[]> => {
  const res = await getRouters();                       // ① GET /getRouters
  const data = Array.isArray(res.data) ? res.data : [];
  const sdata = structuredClone(data);                  // ② 深拷贝三份

  const sidebarRoutes = filterAsyncRouter(sdata);       // ③ 侧边栏路由（组件不变）
  const rewriteRoutes = filterAsyncRouter(rdata, ..., true); // ④ 扁平化重写路由
  const defaultRoutes = filterAsyncRouter(defaultData); // ⑤ 顶部栏路由

  const asyncRoutes = filterDynamicRoutes(dynamicRoutes);// ⑥ 本地动态路由按角色过滤
  asyncRoutes.forEach(route => { router.addRoute(route); });

  setRoutes(rewriteRoutes);                              // ⑦ 存入 store
  return rewriteRoutes;
};

这段代码的关键设计——为什么需要 structuredClone 深拷贝三份？因为 filterAsyncRouter 会原地修改路由对象（把字符串 component 替换成真正的组件函数），一份数据只能走一条改造流水线。

三、filterAsyncRouter：字符串 component → 真实组件

// store/modules/permission.ts:83-109
const filterAsyncRouter = (asyncRouterMap, lastRouter?, type = false) => {
  return asyncRouterMap.filter(route => {
    if (route.component?.toString() === 'Layout') {
      route.component = Layout;           // ① 特殊字符串 → 布局组件
    } else if (route.component?.toString() === 'ParentView') {
      route.component = ParentView;       // ② 特殊字符串 → 父视图组件
    } else if (route.component?.toString() === 'InnerLink') {
      route.component = InnerLink;        // ③ 特殊字符串 → 内链组件
    } else {
      route.component = loadView(route.component, route.name); // ④ 普通路径 → 懒加载视图
    }
    if (route.children?.length) {
      route.children = filterAsyncRouter(route.children, route, type); // ⑤ 递归
    } else {
      delete route.children;              // ⑥ 叶子节点清理
    }
    return true;
  });
};

后端返回的 JSON 中 component 字段是字符串（如 "system/user/index"），这里一一替换为真实的 Vue 组件函数。三种特殊字符串对应三种布局外壳，其他路径通过 loadView 从 viewModuleMap 中查找。

filterChildren 方法（第 111-122 行）负责路径拼接：递归地将子路由的 path 与父路由拼接成全路径。这是为了后端存储时只存相对路径，前端使用时需要绝对路径。

四、filterDynamicRoutes：本地路由 × 角色权限

// store/modules/permission.ts:141-155
export const filterDynamicRoutes = (routes: RouteRecordRaw[]) => {
  const res: RouteRecordRaw[] = [];
  routes.forEach(route => {
    if (route.permissions) {
      if (auth.hasPermiOr(route.permissions)) { res.push(route); }
    } else if (route.roles) {
      if (auth.hasRoleOr(route.roles)) { res.push(route); }
    }
  });
  return res;
};

本地定义的动态路由（前端写死的路由配置），需要与当前用户的角色/权限做交集。这与后端返回的路由不同——后端返回的路由已经是按用户权限过滤过的。auth.hasPermiOr 和 auth.hasRoleOr 是通过 permissions 和 roles 两个数组做 .some() 包含判断。

练习：暂停回忆

❓ 为什么 generateRoutes() 调用 structuredClone(data) 三次，而不是一次 data 引用多次？

点击查看答案

filterAsyncRouter 会原地修改路由对象的 component 属性（字符串→函数），也会修改 children 结构。三条流水线需要路由对象保持不同的改造结果：rewriteRoutes 需要扁平化路径（type=true），sidebarRoutes 保留原始层级，defaultRoutes 用于顶部栏。如果共享引用，后执行的会覆盖先执行的结果。
