10 个追问覆盖的知识点

  1. request.ts 请求拦截器全解析（token 添加、防重提交、AES 加密流程）
  2. 响应拦截器：401 弹窗机制、isRelogin 互斥锁、响应解密、通用下载
  3. Pinia Setup Store 语法 + await-to-js 错误处理模式
  4. 动态路由：import.meta.glob → filterAsyncRouter → router.addRoute
  5. @ApiEncrypt 注解通过 Servlet Filter 而非 Spring AOP 实现的原因
  6. StpUtil.getTokenSession().get(LOGIN_USER_KEY) 的三层存储模型
  7. 强制下线：NotLoginException 类型码（BE_REPLACED / KICK_OUT / TIMEOUT）的生成与处理链路
  8. SaPermissionImpl 双源权限策略：Token-Session 缓存 → 数据库查询降级
  9. useStorage<null | string>(TokenKey, null) TypeScript 泛型 + VueUse 响应式持久化
  10. 前后端白名单对比：前端 whiteList vs 后端 @SaIgnore vs security.excludes

关键洞察

  • Filter vs Interceptor vs AOP 的选型法则：需要操作原始 HTTP 流 → Filter；需要知道目标方法但不操作流 → Interceptor；只增强方法 → AOP。
  • Sa-Token 的异常驱动状态机：不返回 null/空值，而是抛出携带类型码的 NotLoginException，由全局 @ExceptionHandler 统一转换。
  • 前端白名单 ≠ 安全措施：前端路由守卫是 UX 优化，后端拦截器才是安全边界。
  • StpInterface 双源策略：优先从当前 Session 的 LoginUser 取权限（O(1)），跨用户查询时才降级到数据库。
