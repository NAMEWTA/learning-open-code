# Mission: 读懂 RuoYi-Vue-Plus 全栈登录鉴权认证体系

## Why
学习者要能在脑中完整重建 RuoYi-Vue-Plus 这套系统「从用户点击登录到带着 token 访问受保护接口」的全链路，理解后端 Sa-Token 认证体系与前端（Vue/React 两套 UI）登录鉴权对接的设计动机，达到能给同事讲清楚、能看懂面试题、能在此基础上做判断的程度。重点是**读懂设计与流程**，不是从零造轮子。

## Success looks like
- 能用一张图 + 一段话讲清「一次密码登录」从前端到后端再回到前端的完整调用链，并说出每一环用到的关键类/文件。
- 能解释「客户端（sys_client）+ grantType + 策略模式」这套设计解决了什么问题，以及新增一种登录方式时系统的哪几处会参与。
- 能讲清 Sa-Token 如何签发 token、LoginUser 存在哪里、JWT 简单模式与 Redis 各自承担什么角色。
- 能讲清统一拦截器（SecurityConfig）如何做登录校验 + 客户端一致性校验 + 访问路径/IP 白名单，以及 StpInterface 如何按需加载权限。
- 能说清五种登录方式（password / sms / email / social / xcx）+ 注册 + 三类验证码（图形/短信/邮箱）各自的校验要点与差异。
- 能讲清前端（Vue 的 Pinia + axios + router 守卫，React 的 Zustand + axios + UmiJS 布局守卫）如何存 token、加请求头、处理 401、做路由与按钮级权限，并能对比两套实现的异同。
- 能说清接口加解密（ApiEncrypt + AES/RSA）在登录链路中的位置与作用。

## Constraints
- 学习者是全栈背景，后端 Java + 前端 Vue/React 都要覆盖，篇幅大致并重。
- 目标是「读懂」而非「能改」——课程以追踪真实代码调用链、解释设计动机为主，练习以「读代码答问题 / 复述链路」为主，不要求大量动手编码。
- 全部讲解基于仓库真实代码（已逐文件核对），引用具体文件路径与类名。
- 交互语言：简体中文。

## Out of scope
- 数据权限（data scope）、多租户隔离的完整实现细节——仅在与认证交叉处点到为止。
- 菜单/角色/部门 CRUD 的管理后台业务逻辑。
- 微服务（Cloud）版本的网关鉴权——本仓库为单体 admin 版本。
- 从零搭建一套新的鉴权框架。
