速查 06 · MapReduce 任务

配套课程：lessons/0006.html

一句话

MapReduce = Map 任务 + @ReduceExecutor 汇总阶段：所有子 Map 完成后，Reduce 收集结果做最终聚合

MapReduce vs Map 的增量

Map 任务（上一课）: @MapExecutor（根）→ doMap → @MapExecutor(taskName)（子）× N → 各自返回 MapReduce 任务（本课）: @MapExecutor（根）→ doMap → @MapExecutor(taskName)（子）× N → 所有子任务完成（屏障） → @ReduceExecutor → reduceArgs.getMapResult() → 汇总 → 返回最终结果

Reduce 方法模板

@ReduceExecutor public ExecuteResult reduceExecute(ReduceArgs reduceArgs) { // 收集所有子 Map 的返回值 int total = reduceArgs.getMapResult().stream() .mapToInt(i -> Integer.parseInt((String) i)) // ⚠️ 注意类型转换 .sum(); return ExecuteResult.success(total); }

ReduceArgs

getMapResult() → List<?> — 所有子 Map 返回值的集合 getTaskBatchId() → 本次执行批次 ID getJobId() → 任务定义 ID

执行时序

① 根 Map 执行（拆分数据） ② 子 Map × N 并行执行（处理各片数据） ③ 屏障：等待所有子 Map 完成（有失败则 Reduce 不执行） ④ Reduce 执行（聚合所有子结果） ⑤ 返回最终汇总结果

常见陷阱

  • 类型转换：子 Map 返回 int 但 Reduce 中可能收到 String——框架序列化可能改变类型
  • 全部成功才 Reduce：任一个子 Map 失败，Reduce 不会执行
  • 屏障等待：Reduce 等待最慢的那个子 Map——分片要均衡

Map vs MapReduce 选型

场景选 Map选 MapReduce
每个分片发通知，各自记录结果✅
统计所有订单总金额✅
批量导出独立报表文件✅
数据对账，要汇总差异报告✅

→ 相关术语：@ReduceExecutor、ReduceArgs、getMapResult()、屏障（barrier）、类型转换陷阱
