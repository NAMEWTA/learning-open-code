速查 02 · 基础任务：注解式 vs 继承式

配套课程：lessons/0002.html

注解式任务最小模板

@Component @JobExecutor(name = "任务名") public class XxxJob { public ExecuteResult jobExecute(JobArgs jobArgs) { // 业务逻辑 return ExecuteResult.success("成功"); } }

继承式任务最小模板

@Component public class XxxJob extends AbstractJobExecutor { @Override protected ExecuteResult doJobExecute(JobArgs jobArgs) { return ExecuteResult.success("成功"); } }

两种方式对比

维度注解式继承式
注册name 即任务名，自动注册到服务端需在后台手动创建并填全限定类名
方法public jobExecute(JobArgs)protected doJobExecute(JobArgs)
扩展性轻量，适合简单任务可基类统一处理异常/日志/钩子
推荐度⭐⭐⭐ 90% 场景⭐ 有统一基类诉求时

JobArgs 核心方法

getJobParams() → 服务端配置的任务参数 getWfContext() → DAG 工作流上下文 Map getWfContext(key) → 按 key 取上下文值 appendContext(k, v) → 向上下文追加数据 getTaskBatchId() → 本次执行批次 ID getJobId() → 任务定义 ID

ExecuteResult 工厂方法

ExecuteResult.success() → 成功 ExecuteResult.success(obj) → 成功 + 携带返回对象 ExecuteResult.failure() → 失败 ExecuteResult.failure("原因") → 失败 + 原因描述
