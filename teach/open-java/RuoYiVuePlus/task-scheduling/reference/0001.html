模块依赖关系

ruoyi-job/pom.xml ├── ruoyi-common-json ← JSON 序列化 └── ruoyi-common-job ← SnailJob 客户端装配 └── pom.xml ├── snail-job-client-starter ← 客户端启动器 ├── snail-job-client-job-core ← 任务调度核心 API ├── spring-boot-autoconfigure ← @AutoConfiguration └── ruoyi-common-core ← RuoYi 核心工具

SnailJobConfig 四个关键注解

注解作用
@AutoConfigurationSpring Boot 3.x 自动装配入口
@ConditionalOnProperty(prefix="snail-job", name="enabled", havingValue="true")总开关，默认关
@EnableScheduling开启 Spring 定时任务线程池
@EnableSnailJob扫描 @JobExecutor，向服务端注册任务

SnailJob 客户端启动钩子

SnailClientStartingEvent → onStarting() ├── 获取 Logback LoggerContext ├── 创建 SnailLogbackAppender（远程日志 appender） ├── 挂到 Root Logger └── 效果：所有日志同时上报 SnailJob 服务端

客户端→服务端完整链路

应用启动 → @EnableSnailJob 扫描 @JobExecutor → 向服务端注册(name→类映射) → 挂载 SnailLogbackAppender（远程日志） → 等待服务端调度 → 执行 jobExecute → 上报结果+日志 → 服务端记录

易混点

  • ruoyi-common-job（自动装配层）≠ ruoyi-job（Demo 层）——依赖方向：job → common-job
  • SnailJobLog.LOCAL（本地文件）≠ SnailJobLog.REMOTE（远程上报）——两者都写最保险
  • enabled: false = 整个 SnailJobConfig 不加载，不是「加载了但不执行」

→ 相关术语见 GLOSSARY.md：SnailJob、@JobExecutor、ExecuteResult、SnailLogbackAppender、DAG 工作流
