速查 04 · 静态分片

配套课程：lessons/0004.html

一句话

静态分片 = 服务端预先定义分片规则 → 下发 jobParams 给执行器 → 各执行器按参数范围处理数据

最小模板

@Component @JobExecutor(name = "testStaticShardingJob") public class TestStaticShardingJob { public ExecuteResult jobExecute(JobArgs jobArgs) { String jobParams = Convert.toStr(jobArgs.getJobParams()); // 解析分片参数（如 "1,10000"） String[] split = jobParams.split(","); Long fromId = Long.parseLong(split[0]); Long toId = Long.parseLong(split[1]); // 处理 fromId ~ toId 范围的数据 return ExecuteResult.success(); } }

执行模型

服务端配置 N 个分片，每个分片有自己的 jobParams → 调度时下发到集群 → 各实例认领分片（快的多领） → 每个分片独立执行、独立上报结果 → 分片数 ≠ 实例数（解耦）

常见 jobParams 格式

格式示例场景
起止范围"1,10000"按 ID 分片
日期范围 JSON'{"start":"2026-01","end":"2026-06"}'按月份分片
分片索引+总数"shard:0,total:3"执行器自己算范围
分表名"TABLE_0,TABLE_1"按表分片
⚠️ 分片必须不重不漏：如果分片 0 写 "1,10000"，分片 1 写 "5000,15000"——ID 5000~10000 被重复处理。这是最容易出错的地方。

静态分片 vs 动态分片（Map）

维度静态分片动态分片（Map）
分片规则定义服务端后台配置执行器代码中决定
灵活度固定，改规则要改后台动态，数据变分片自动变
适合策略稳定（如每天按日期跑）数据量不可预测

→ 相关术语：jobParams、分片参数、静态分片、动态分片、不重不漏
