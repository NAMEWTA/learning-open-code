速查 07 · DAG 工作流

配套课程：lessons/0007.html

一句话

DAG 工作流 = 多个任务节点编排成有向无环图 → 无依赖节点并行 → 有依赖节点等待上游 → 通过 wfContext 在节点间传递数据

本模块 DAG Demo 拓扑

wechatBillTask ──┐ ├──→ summaryBillTask alipayBillTask ──┘ （微信和支付宝并行拉账单 → 两者都完成后汇总）

wfContext（工作流上下文）核心操作

操作代码说明
写入jobArgs.appendContext("key", value)追加上下文键值对
读全部jobArgs.getWfContext()获取整个 Map
读单个jobArgs.getWfContext("key")按 key 取值

数据传递最佳实践

上游写入: jobArgs.appendContext("data", JsonUtils.toJsonString(dto)); 下游读取: String json = (String) jobArgs.getWfContext("data"); MyDto dto = JsonUtils.parseObject(json, MyDto.class);

wfContext vs jobParams

维度wfContextjobParams
来源前置节点写入（动态）服务端配置（静态）
生命周期一个工作流实例内本次任务执行
可写是（appendContext）否（只读）
用途传递中间计算结果传递初始执行参数

DAG vs MapReduce 拓扑对比

维度MapReduceDAG
拓扑固定 Map→Reduce任意 DAG（多源汇聚、多路分发）
数据流子 Map 返回值 → Reduce 聚合wfContext 在任意节点间共享
并行度子 Map 并行，Reduce 串行无依赖节点全部并行
场景同类型数据并行计算异构任务编排（不同类型节点）

关键规则

  • 无依赖的节点并行执行（如微信和支付宝同时跑）
  • 任一上游失败 → 下游不执行
  • 上下文在工作流实例内共享，实例间隔离
  • 复杂对象用 JSON 序列化传递，不用 Java 原生序列化
