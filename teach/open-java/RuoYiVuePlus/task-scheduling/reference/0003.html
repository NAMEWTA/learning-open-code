速查 03 · 广播任务

配套课程：lessons/0003.html

一句话

广播任务 = 所有在线客户端实例各执行一次（普通任务只选一个实例执行）

执行模型

服务端触发 → 向所有在线客户端下发指令 → 各实例独立执行 → 各自上报结果

最小模板

@Component @JobExecutor(name = "testBroadcastJob") public class TestBroadcastJob { @Value("${snail-job.port}") // 注入端口作为实例标识 private int clientPort; public ExecuteResult jobExecute(JobArgs jobArgs) { // 每个实例都执行此逻辑 return ExecuteResult.success(); } }

适用场景

✅ 适合❌ 不适合
全实例缓存清理（各自清自己的 JVM 缓存）全局只做一次的事（如发全员邮件）
配置热刷新（各自从配置中心拉取）写数据库（会导致重复写入）
JVM 指标采集（各自的堆内存/线程不同）幂等操作（广播本身就是多次执行）
广播+分片组合（每个实例各管一段数据）分布式锁竞争（每个实例都抢锁无意义）
⚠️ 广播 ≠ 幂等：广播任务在所有实例上执行，如果你的业务逻辑有全局副作用，会导致重复执行。使用前确认「每个实例都执行一次是不是你想要的」。

广播 + 分片组合模式

服务端广播 → 每个实例收到分片参数（jobParams） → 各处理自己那一段数据 适用于：「既要每台机器都参与计算，又要数据不重复处理」

→ 相关术语：广播任务、实例标识、全量下发、幂等性
