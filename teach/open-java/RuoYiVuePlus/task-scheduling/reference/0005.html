速查 05 · Map 任务（动态分片）

配套课程：lessons/0005.html

一句话

Map 任务 = 根 Map 动态拆分数据 → MapHandler.doMap() 分发子任务 → 子 Map 并行执行 → 结果各自返回（不汇总）

三个角色

角色注解方法签名职责
根 Map@MapExecutor（无 taskName）ExecuteResult root(MapArgs, MapHandler)拆分数据，调用 doMap 分发
子 Map@MapExecutor(taskName="xxx")ExecuteResult sub(MapArgs)处理一片数据，返回结果
MapHandler框架注入doMap(List, taskName)将数据列表分发给集群中的子任务

最小模板

@Component @JobExecutor(name = "testMapJob") public class TestMapJob { @MapExecutor // ← 根 Map：拆分数据 public ExecuteResult rootMap(MapArgs mapArgs, MapHandler mapHandler) { List<List<?>> partitions = splitData(); // 拆分成 N 片 return mapHandler.doMap(partitions, "subTask"); // 分发给子任务 } @MapExecutor(taskName = "subTask") // ← 子 Map：处理一片 public ExecuteResult subMap(MapArgs mapArgs) { List<?> data = (List<?>) mapArgs.getMapResult(); // 拿到一片数据 Object result = process(data); // 处理 return ExecuteResult.success(result); } }

执行模型

根 Map 执行 → 拆分数据 → doMap(list, taskName) 提交 N 个子任务 → 子任务进入 SnailJob 队列 → 集群中执行器认领（负载均衡） → 各子任务并行执行 → 各自独立返回 ExecuteResult → 没有汇总阶段（需要汇总请用 MapReduce）

关键注意事项

  • doMap 是异步分发——根任务返回时子任务可能还在排队
  • 一个类里只能有一个不带 taskName 的 @MapExecutor（根任务）
  • 子任务失败不影响其他子任务（各自独立）
  • Map 不汇总——如果需要最终聚合结果，升级到 MapReduce

→ 相关术语：@MapExecutor、MapHandler、doMap()、MapArgs、动态分片
