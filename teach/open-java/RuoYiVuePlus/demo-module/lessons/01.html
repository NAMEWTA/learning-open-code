第 1 课 · 标准 CRUD 模板：用 TestDemoController 快速搭建 REST 接口 核心模板

🎯 本节目标（对应使命 Success #1）：学完你能逐行读通 TestDemoController，把它的每一层注解（权限、校验、日志、防重、Excel）说出来，并在新业务模块中直接复制这套模板。

ruoyi-demo 模块里有 20 个 Controller 演示了框架的几乎所有通用能力，而 TestDemoController 是其中最基础也最完整的「标准单表 CRUD 模板」——分页查询、单条详情、新增、修改、删除、Excel 导入导出，七个端点覆盖了日常业务 90% 的操作。这一课我们把它解剖清楚。

一、Controller 骨架：六行注解五个秘密

先看类声明部分（源码路径 ruoyi-demo/.../controller/TestDemoController.java）：

@Validated                          // ① 开启方法级参数校验
@RequiredArgsConstructor            // ② Lombok 构造器注入
@RestController                     // ③ REST 风格 (自动 @ResponseBody)
@RequestMapping("/demo/demo")       // ④ 统一 URL 前缀
public class TestDemoController extends BaseController {  // ⑤ 继承基础控制器

    private final ITestDemoService testDemoService;   // ⑥ 业务接口注入
}
注解作用去掉后会怎样
@Validated启用 Spring 方法级校验，配合 @NotNull、@NotEmpty 等使用路径参数和查询参数校验失效
@RequiredArgsConstructorLombok 自动生成包含所有 final 字段的构造函数需手动写 @Autowired
extends BaseController继承 toAjax()、success()、startPage() 等工具方法需要手动构造 R<Void> 响应

二、分页查询：权限 + 校验 + 分页三件套

@SaCheckPermission("demo:demo:list")
@GetMapping("/list")
public R<PageResult<TestDemoVo>> list(
    @Validated(QueryGroup.class) TestDemoBo bo,
    PageQuery pageQuery
) {
    return R.ok(testDemoService.queryPageList(bo, pageQuery));
}

这个方法的三个关键点：

1权限控制：@SaCheckPermission("demo:demo:list") 是 Sa-Token 的注解，当前用户必须拥有 demo:demo:list 权限字符串才能访问。权限字符串的命名惯例是 模块:实体:操作。
2参数校验分组：@Validated(QueryGroup.class) 只校验 TestDemoBo 中标记了 QueryGroup 的字段，和新增/编辑分开，互不干扰。
3分页封装：PageQuery 是框架对 MyBatis-Plus 分页的封装，自动读取 pageNum 和 pageSize 请求参数。PageResult<TestDemoVo> 统一包装分页结果。

三、新增与修改：防重、日志、校验分组

// ===== 新增 =====
@SaCheckPermission("demo:demo:add")
@Log(title = "测试单表", businessType = BusinessType.INSERT)
@RepeatSubmit(interval = 2, timeUnit = TimeUnit.SECONDS, message = "{repeat.submit.message}")
@PostMapping()
public R<Void> add(@RequestBody TestDemoBo bo) {
    ValidatorUtils.validate(bo, AddGroup.class);   // 手动校验
    return toAjax(testDemoService.insertByBo(bo));
}

// ===== 修改 =====
@SaCheckPermission("demo:demo:edit")
@Log(title = "测试单表", businessType = BusinessType.UPDATE)
@RepeatSubmit                                      // 使用默认配置
@PutMapping()
public R<Void> edit(@Validated(EditGroup.class) @RequestBody TestDemoBo bo) {
    return toAjax(testDemoService.updateByBo(bo));
}

新增和修改的注解差异展示了两种校验风格：

  • 新增：ValidatorUtils.validate(bo, AddGroup.class) 手动调用校验工具，适用于需要在 Service 层或非 Controller 环境校验的场景。
  • 修改：@Validated(EditGroup.class) 声明式自动校验，Spring 在进入方法体前完成校验。
  • 防重提交：@RepeatSubmit(interval = 2, timeUnit = TimeUnit.SECONDS) 表示同一用户 2 秒内不能重复提交，国际化提示信息通过 message = "{repeat.submit.message}" 从 messages.properties 读取。
锦囊：ValidatorUtils.validate() 的手动校验方式和 @Validated 注解方式等效，差别在于手动校验可以用在 Service、工具类等任意位置，注解只能用在 Spring 管理的 Bean 方法参数上。

四、删除：逻辑删除 + 批量操作

@SaCheckPermission("demo:demo:remove")
@Log(title = "测试单表", businessType = BusinessType.DELETE)
@DeleteMapping("/{ids}")
public R<Void> remove(
    @NotEmpty(message = "主键不能为空")
    @PathVariable Long[] ids
) {
    return toAjax(testDemoService.deleteWithValidByIds(
        Arrays.asList(ids), true
    ));
}

注意三个细节：

  • 路径参数接收 Long[] 数组，前端传逗号分隔即可，Spring 自动解析。
  • @NotEmpty 保证数组不为空，比 @NotNull 更严格。
  • 第二个参数 true 表示开启数据校验（检查数据是否存在），这是 BaseController 提供的模板方法。
  • 实际执行的是逻辑删除（MyBatis-Plus @TableLogic），不会真正从数据库删除数据。

五、Excel 导入导出：两行代码搞定

// ===== 导入 =====
@PostMapping(value = "/importData", consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
public R<Void> importData(@RequestPart("file") MultipartFile file) throws Exception {
    ExcelResult<TestDemoImportVo> excelResult = ExcelBuilder
        .read(file.getInputStream(), TestDemoImportVo.class)
        .validate(true)       // 开启数据校验
        .doRead();            // 执行读取
    List<TestDemo> list = MapstructUtils.convert(excelResult.getList(), TestDemo.class);
    testDemoService.saveBatch(list);
    return R.ok(excelResult.getAnalysis());   // 返回导入分析结果
}

// ===== 导出 =====
@PostMapping("/export")
public void export(@Validated TestDemoBo bo, HttpServletResponse response) {
    List<TestDemoVo> list = testDemoService.queryList(bo);
    ExcelBuilder.of(list, TestDemoVo.class)
        .sheetName("测试单表")
        .toResponse(response);     // 直接写入 HttpServletResponse
}
注意：导入使用 @RequestPart("file") 而非 @RequestParam，配合 consumes = MULTIPART_FORM_DATA_VALUE，这是 Swagger3 能正确生成文件上传 API 文档的前提。

六、树表 CRUD：TestTreeController 的差异

TestTreeController（源码 controller/TestTreeController.java）与 TestDemoController 几乎一模一样的注解模式，唯一区别：

  • 树表查询返回 R<List<TestTreeVo>> 而非分页结果，支持前端渲染树形结构。
  • 使用 TestTreeBo、TestTreeVo 而非 TestDemoBo、TestDemoVo。
  • 其余注解（@SaCheckPermission、@Log、@RepeatSubmit、@Validated 分组校验）模式完全一致。

这说明框架的设计哲学：单表 CRUD 和树表 CRUD 共用同一套注解模板，只需替换实体类和权限字符串，代码结构完全不变。

七、TestDemo 实体：MyBatis-Plus 的四个关键注解

@Data
@EqualsAndHashCode(callSuper = true)
@TableName("test_demo")           // 映射数据库表
public class TestDemo extends BaseEntity {

    @TableId(value = "id")        // 主键
    private Long id;

    @OrderBy(asc = false, sort = 1)   // 默认排序
    private Integer orderNum;

    @Version                       // 乐观锁版本号
    private Long version;

    @TableLogic                    // 逻辑删除标记
    private Long delFlag;
}

BaseEntity 父类统一提供了 createTime、updateTime、createBy、updateBy 等审计字段，所有业务实体只需要关注业务字段本身。

实战建议：在新业务模块中搭建 CRUD 时，直接复制 TestDemoController 作为模板，全局替换类名、权限字符串和 URL 前缀，再调整字段和校验规则，5 分钟即可上线一个功能完整的 REST 接口。
