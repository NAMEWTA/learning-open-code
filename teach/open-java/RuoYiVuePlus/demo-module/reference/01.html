Excel 导入导出代码速查

// 导入
@PostMapping(value = "/importData", consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
public R<Void> importData(@RequestPart("file") MultipartFile file) throws Exception {
    ExcelResult<ImportVo> excelResult = ExcelBuilder
        .read(file.getInputStream(), ImportVo.class)
        .validate(true).doRead();
    List<Entity> list = MapstructUtils.convert(excelResult.getList(), Entity.class);
    service.saveBatch(list);
    return R.ok(excelResult.getAnalysis());
}

// 导出
@PostMapping("/export")
public void export(@Validated Bo bo, HttpServletResponse response) {
    List<Vo> list = service.queryList(bo);
    ExcelBuilder.of(list, Vo.class).sheetName("表名").toResponse(response);
}

实体层四个关键注解

注解作用来源
@TableName("test_demo")映射数据库表名MyBatis-Plus
@TableId("id")标记主键字段MyBatis-Plus
@TableLogic逻辑删除标记（delFlag 字段），删除操作只更新标记不物理删除MyBatis-Plus
@Version乐观锁版本号，更新时自动检查版本一致性MyBatis-Plus
@OrderBy(asc=false, sort=1)查询默认排序规则MyBatis-Plus
extends BaseEntity继承 createTime/updateTime/createBy/updateBy 等审计字段框架基类

注解层全景图

@SaCheckPermission→@Log→@RepeatSubmit→@Validated→R<T>→toAjax()

权限拦截 → 记录日志 → 防重检查 → 参数校验 → 统一响应。每一层各司其职。

树表 CRUD 差异（TestTreeController）

与单表 CRUD 注解模式完全一致，仅两点不同：查询返回 R<List<TreeVo>>（非分页）、使用 TestTreeBo/Vo 而非 TestDemoBo/Vo。框架设计哲学：单表与树表共用同一套注解模板。
