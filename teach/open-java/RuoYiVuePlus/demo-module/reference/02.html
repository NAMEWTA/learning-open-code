Spring Cache 三件套速查

注解触发时机方法体执行?一句话
@Cacheable查询前命中缓存时不执行查询时自动缓存结果
@CachePut方法执行后始终执行写入数据库后刷新缓存
@CacheEvict方法执行后始终执行删除数据后清除缓存

cacheNames 增强格式

"demo:cache#60s#10m#20#1" — 框架扩展的 # 分隔格式：

段位12345
含义缓存名TTL (60s)maxSize (10 条)maxIdleTime (20s)淘汰策略 (1=条目数)

也可用 CacheNames.DEMO_CACHE 常量，值相同。

警告：缓存注解严禁与数据权限等筛选功能一起使用，否则会造成缓存击穿和数据不一致。

@Cacheable 声明式 vs RedisUtils 手动式

@Cacheable

✅ 一行注解零侵入
✅ AOP 自动管理
❌ 仅 Spring Bean 方法
❌ 不够灵活

RedisUtils

✅ 任意位置可用
✅ 精细控制过期时间
✅ 支持所有 Redis 数据结构
❌ 需手动管理一致性

@Lock4j 声明式 vs LockTemplate 编程式

维度@Lock4j 注解LockTemplate 编程
代码量1 行~10 行 try-finally
释放锁自动（方法结束释放）必须手动 releaseLock()
获取失败抛异常，全局异常处理器接管返回 null，可自定义降级
适用简单互斥，标准 CRUD需自定义超时/降级/重试
底层Redisson 分布式锁（看门狗自动续期）
// 注解式
@Lock4j(keys = {"#key"})
@GetMapping("/test")
public R<String> test(String key) { ... }

// 编程式
LockInfo lockInfo = lockTemplate.lock(key, 30000L, 5000L, RedissonLockExecutor.class);
if (null == lockInfo) throw new RuntimeException("业务处理中");
try { /* 业务 */ } finally { lockTemplate.releaseLock(lockInfo); }

@RateLimiter 三种限流粒度

limitType计数维度典型场景示例
默认（全局）接口全局防止数据库整体过载@RateLimiter(count=2, time=10)
LimitType.IP请求 IP防爬虫、防单用户刷接口@RateLimiter(count=2, time=10, limitType=LimitType.IP)
LimitType.CLUSTER服务实例多实例各自限流互不影响@RateLimiter(count=2, time=10, limitType=LimitType.CLUSTER)

key 支持 SpEL：#变量 获取简单值，#{表达式} 计算取值。示例：key="#value" 按参数值限流，key="#{#value!=1?1:0}" 条件取值。
