一、@Sensitive 五种脱敏策略

策略输入示例输出示例脱敏规则
SensitiveStrategy.ID_CARD210397198608215431210397**********31保留前6位+后2位
SensitiveStrategy.PHONE17640125371176******71保留前3位+后2位
SensitiveStrategy.ADDRESS北京市朝阳区某某四合院1203室北京市******保留前3个字符
SensitiveStrategy.EMAIL17640125371@163.com1**********@163.com保留第1字符+@及之后
SensitiveStrategy.BANK_CARD62264569523514528536226**********853保留前4位+后3位
@Sensitive(strategy = SensitiveStrategy.PHONE, roleKey = "common", perms = "system:user:query")
private String phone;  // 有 common 角色 AND system:user:query 权限时看明文，否则脱敏

条件脱敏参数

参数含义备注
roleKey拥有指定角色时不脱敏两者同时存在时为 AND 关系
perms拥有指定权限时不脱敏
管理员例外：框架默认管理员不脱敏（显示所有明文）。如需修改，重写 SensitiveService 实现。
核心原则：脱敏发生在 Jackson 序列化阶段（返回 JSON 给前端时），数据库里存的始终是明文。脱敏管「看」，加密管「存」——两者互补而非替代。

二、@EncryptField 数据库字段加密

四种加密算法

算法类型所需配置适用场景
AlgorithmType.AES对称password通用字段加密，性能好
AlgorithmType.RSA非对称privateKey + publicKey高安全场景
AlgorithmType.SM2国密非对称privateKey + publicKey信创/国产化要求
AlgorithmType.SM4国密对称password信创/国产化要求
@EncryptField(algorithm = AlgorithmType.RSA,
    privateKey = "MIICdQI...", publicKey = "MIGfMA0...")
private String testKey;  // RSA 非对称加密

@EncryptField(algorithm = AlgorithmType.AES, password = "10rfylhtccpuyke5")
private String value;    // AES 对称加密

透明加解密流程

Java 明文→@EncryptField 拦截→DB 密文→DB 密文读取→@EncryptField 拦截→Java 明文

Service 和 Controller 操作的始终是明文，加密解密由 MyBatis 拦截器自动处理，业务代码完全无感。

启用开关：需要 mybatis-encryptor.enable=true 才生效。不指定 algorithm 或不写任何参数时，走 YML 默认配置。Controller 中应通过 @Value("${mybatis-encryptor.enable}") 做前置检查。

三、国际化 i18n 三种方式

方式代码适用
代码获取MessageUtils.message("user.register.success")业务提示、邮件内容
校验注解@NotBlank(message="{not.null}")单个字段校验失败时
Bean 校验@Validated TestI18nBo bo + 实体字段注解整个对象批量校验

消息 key 定义在 messages.properties，扩展语言添加 messages_en_US.properties，框架根据 Accept-Language 请求头自动切换。
