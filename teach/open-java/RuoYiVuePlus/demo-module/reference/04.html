速查 04 · 消息通信全景 消息通道

对应课程 第 4 课：Redis Pub/Sub / WebSocket / MQTT / 优先级队列 — 四种消息通信方式的差异与选型

一、Redis Pub/Sub — 最轻量发布订阅

// 发布
RedisUtils.publish("channel", "message", consumer -> log.info("published"));
// 订阅
RedisUtils.subscribe("channel", String.class, msg -> log.info("received: {}", msg));

特点

零依赖（Redis 自带）| 毫秒级延迟 | 不持久化（离线消息丢失）| 无 Ack（fire-and-forget）

场景：缓存失效通知、配置刷新广播 — 允许偶尔丢失但要求极低延迟。

二、WebSocket — 点对点 + 全服广播

PushPayloadDTO payload = PushPayloadDTO.of(PushTypeEnum.MESSAGE, PushSourceEnum.BACKEND, msg, null);
// 点对点
messageService.publishMessage(List.of(userId), payload);
// 全服广播
messageService.publishAll(payload);

特点

全双工长连接 | 支持点对点和广播 | 低延迟 | 需维护连接状态（心跳/重连）| 仅在线用户 | 无持久化

场景：站内信、实时通知、在线用户踢出。

PushPayloadDTO.of() 四参数

参数类型含义
pushTypePushTypeEnum消息类型：MESSAGE / NOTICE / WARNING
pushSourcePushSourceEnum来源：BACKEND（后端推送）/ FRONTEND（前端触发）
messageString消息内容
extendObject扩展参数（携带业务数据）

三、MQTT — IoT 场景标准协议

// 发布
client.publish("/test/client", "hello".getBytes(StandardCharsets.UTF_8));

// 订阅 QoS 1 + 通配符
@MqttClientSubscribe(value = "/qos1/#", qos = MqttQoS.QOS1)
public void subQos1(String topic, byte[] payload) { ... }

// 占位符主题：${productKey} → +（单层通配符）
@MqttClientSubscribe("/sys/${productKey}/${deviceName}/thing/sub/register")
public void thingSubRegister(String topic, byte[] payload) { ... }

// JSON 自动反序列化
@MqttClientSubscribe(value = "/test/json", deserialize = MqttJsonDeserializer.class)
public void testJson(String topic, MqttPublishMessage message, TestDemo data) { ... }

参数自动映射规则 (mica-mqtt 2.4.5+)

参数类型映射内容
String（第一个）Topic（主题名）
MqttPublishMessage原始消息（含 MQTT5 props）
byte[] / ByteBufferPayload 原始字节
其他自定义类型JSON 反序列化后的业务对象

特点

QoS 三级保证（0/1/2）| 主题通配符（# 多层 / + 单层）| 自动反序列化 | 离线消息（Broker 持久化）| 需外部 Broker（EMQX/Mosquitto）

场景：IoT 设备通信、车联网、智能家居 — 需可靠送达、离线补收。

四、优先级队列 — 轻量级 MQ 替代

QueueUtils.addPriorityQueueObject("queueName", data);      // 入队（按 orderNum 排序）
PriorityDemo data = QueueUtils.getPriorityQueueObject("queueName");  // 出队（按优先级）
QueueUtils.removePriorityQueueObject("queueName", data);   // 删除指定元素
QueueUtils.destroyPriorityQueue("queueName");              // 销毁队列

特点

零外部依赖（Redis Sorted Set）| 支持优先级排序 | 集群安全（同一条只消费一次）| 无 Ack（消费即删除）| 无死信队列

场景：订单按优先级处理、延迟任务调度 — 不想引入 RabbitMQ/RocketMQ 等重量级 MQ。

集群提醒：优先队列消费即删除无 Ack，如果业务处理失败消息就丢了。源码注释明确提醒：做好事务补偿。重要业务建议使用真正的 MQ。
