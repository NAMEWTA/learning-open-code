速查 04 · SmsExceptionHandler 异常兜底

配套课程：lessons/0004.html

一句话

@RestControllerAdvice 全局拦截 SmsBlendException，对内记详细日志，对外回统一 R.fail 友好提示。内外有别。

源码速览

代码含义
@RestControllerAdvice= @ControllerAdvice + @ResponseBody；全局拦截所有 Controller 异常
@ExceptionHandler(SmsBlendException.class)只接 SmsBlendException，其它异常由别处处理
log.error(requestURI, e)给运维：记录哪个接口触发的异常 + 完整堆栈
R.fail(500, "短信发送失败，请稍后再试...")给用户：固定友好文案，不泄露内部细节
HttpServletRequest request自动注入，拿 requestURI 精确定位

SmsBlendException 结构

字段值
code厂商错误码
message错误描述
requestId厂商请求 ID（排查用）

两道防线

  • 防线 1：发送方法返回 SmsResponse，业务代码 isSuccess() 主动判断（如 CaptchaController）。
  • 防线 2：抛 SmsBlendException，被本 Handler 全局兜底。

记忆锚点

  • 短信异常处理就近放在引入 sms4j 的模块，谁依赖谁获得兜底。
  • 内外有别是核心纪律：日志记全，用户看简。
  • 异常处理器本身也是 SmsAutoConfiguration 里注册的 Bean——三课闭环。
