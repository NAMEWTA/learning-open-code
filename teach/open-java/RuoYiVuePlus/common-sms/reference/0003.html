速查 03 · SmsAutoConfiguration 自动配置

配套课程：lessons/0003.html

一句话

用 .imports 被发现 + after 排顺序 + @Primary 压默认 三件套，让业务模块「加一行依赖、不写任何配置」就能用上短信。

三件套

机制位置解决的问题
.imports 文件 META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports 怎么被「发现」？ Boot 3.x 扫所有 jar 的这个文件，登记自动配置类
@AutoConfiguration(after={DataRedisAutoConfiguration}) SmsAutoConfiguration.java 类上 怎么保证顺序？ 显式声明在 Redis 配置之后才加载，PlusSmsDao 才能安全用 RedisUtils
@Primary smsDao() 方法上 怎么压过 sms4j 默认版？ 有两个 SmsDao Bean 时声明本 Bean 为首选

注册的两个 Bean

@Bean 方法返回作用
smsDao()PlusSmsDao实现 SmsDao（@Primary 替换默认）
smsExceptionHandler()SmsExceptionHandler全局异常兜底

启动时序

Boot 启动 → 扫 .imports → 发现 SmsAutoConfiguration → 排到 DataRedisAutoConfiguration 之后加载 → 注册 PlusSmsDao(@Primary) + SmsExceptionHandler → sms4j 内部注入 SmsDao 时拿到 Redis 版 → 业务模块已可用 — 零配置

记忆锚点

  • 这是所有 ruoyi-common-* 的标准接入模板。
  • 依赖即生效的秘密全在 .imports 文件。
  • @Primary 不删除 sms4j 的默认 Bean，只声明「有歧义选我」。
