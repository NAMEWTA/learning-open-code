它是什么

实现 sms4j 的 SmsDao 接口，把短信「拦截/重试计数」的存取转发给项目统一的 RedisUtils。一层薄薄的「接口适配 + 加前缀转发」，无业务逻辑。

为什么必须替换默认实现

sms4j 默认 SmsDaoDefaultImplRuoYi PlusSmsDao
存储JVM 内存 ConcurrentHashMap + TimerRedis（RedisUtils）
集群❌ 各节点各算各的，限流被放大 N 倍✅ 共享计数，minute-max 真正生效

记忆锚点

  • 所有 key 加前缀 GlobalConstants.GLOBAL_REDIS_KEY = "global:"（GlobalConstants.java:13）。
  • 替换动机一句话：内存缓存不跨节点 → 集群下防刷击穿 → 改用共享 Redis。
  • 体现「依赖倒置」：sms4j 只声明 SmsDao 接口，把「用什么存」交给使用方。
  • 「定义实现」在本课，「@Primary 注册启用」在第 3 课，合起来才完成替换。
