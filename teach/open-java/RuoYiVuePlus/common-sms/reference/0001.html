速查 01 · ruoyi-common-sms 模块全景

配套课程：lessons/0001.html · 使命：读懂 ruoyi-common-sms 短信模块

一句话定位

ruoyi-common-sms 不是短信框架，而是把开源框架 sms4j(3.3.5) 收编进 RuoYi 技术栈的「适配垫片」——自己不发短信，只补三件事。

模块全部内容

文件作用
pom.xml只依赖 sms4j-spring-boot-starter + ruoyi-common-redis
config/SmsAutoConfiguration.java装配：注册另外两个类（第 3 课）
core/dao/PlusSmsDao.java缓存：用 RedisUtils 实现 SmsDao（第 2 课）
handler/SmsExceptionHandler.java兜底：捕获 SmsBlendException → R.fail（第 4 课）
META-INF/spring/...AutoConfiguration.imports发现：声明 SmsAutoConfiguration 为自动配置类

三个类 = 三件事

类职责解决
SmsAutoConfiguration注册 Bean依赖即生效，零配置接入
PlusSmsDaoRedis 缓存拦截计数集群下生效
SmsExceptionHandler异常兜底失败回统一友好响应

记忆锚点

  • 发短信的脏活全在 sms4j，本模块零行厂商 API 调用。
  • 模块不暴露 Controller，是被业务（如 CaptchaController）依赖的能力层。
  • 这是所有 ruoyi-common-* 包装第三方库的标准范式：依赖成熟库 + 补「装配/缓存/异常」三件套。

→ 相关术语见 GLOSSARY.md：sms4j、SmsBlend、SmsDao、自动配置垫片
