全链路一句话

SmsFactory.getSmsBlend("config1").sendMessage(...) 发送 → 发成功存 Redis(global:captcha_codes:{phone}=code) → SmsAuthStrategy 用同 key 取回比对 → 匹配则登录

完整链路

步骤文件动作
1CaptchaController.java:62smsCode() 入口，@RateLimiter 限流
2同方法内生成 4 位随机码，SmsFactory.getSmsBlend("config1") 取厂商发送器
3sms4j 内部 ⇒ PlusSmsDao查拦截计数（minute-max / account-max）
4sms4j ⇒ 厂商 APIsendMessage() 调阿里/腾讯等 SDK
5a失败 throw → SmsExceptionHandler全局兜底 R.fail("短信发送失败...")
5b成功 → RedisUtils.setCacheObject存 global:captcha_codes:{phone}=code（5 分钟过期）
6SmsAuthStrategy.java:82RedisUtils.getCacheObject(key) 取回比对
7同方法匹配 → LoginHelper.login() 签发 token

三道防刷

  1. @RateLimiter(key="#phoneNumber", time=60, count=1) — 接口级限流
  2. sms4j minute-max / account-max — 框架级拦截（计数走 PlusSmsDao → Redis）
  3. 验证码 Redis key 5 分钟过期 — 时效性限制

加新厂商（能力在配置、不碰代码）

yml blends 下新增一段： hw1: supplier: huawei # 厂商标识 access-key-id: ... ... 代码里 SmsFactory.getSmsBlend("hw1").sendMessage(...) 即可用
