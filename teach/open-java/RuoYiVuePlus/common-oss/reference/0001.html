速查 01 · ruoyi-common-oss 模块全景

配套课程：lessons/0001.html · 使命：读懂 ruoyi-common-oss 对象存储模块

一句话定位

ruoyi-common-oss 是一层 S3 兼容存储的轻量封装工厂——把 AWS S3 SDK 的 S3AsyncClient / S3TransferManager / S3Presigner 三剑客组装成一条 OssClient 接口，业务代码透明切换阿里云 OSS / 腾讯云 COS / 七牛云 / MinIO，一套 API，换配置不换代码。

模块全部内容（18 个 Java 类，10 个子包）

子包类职责
client (3)OssClient（接口）
AbstractOssClientImpl（抽象类）
DefaultOssClientImpl（实现类）		定义 60+ 操作 / 实现 70+ 方法骨架 / 组装 S3 三剑客
config (4)Config, OssClientConfig, AccessControlPolicyConfig, OssAsyncExecutorConfig配置对象层：endpoint/bucket/region/ACL/线程池
constant (1)OssConstantRedis Key 前缀、默认配置 Key
enums (1)AccessPolicyPRIVATE / PUBLIC_READ_WRITE / PUBLIC_READ
exception (1)S3StorageException统一 OSS 操作异常（RuntimeException）
factory (1)OssFactory客户端单例缓存、配置校验、按 configKey 加锁创建/销毁
io (1)OutputStreamDownloadSubscriber适配器：Consumer<ByteBuffer> → WritableByteChannel
model (4)Options, PutObjectResult, GetObjectResult, HandleAsyncResult上传选项 / 上传结果 / 下载结果 / 异步结果包装

核心设计模式

模式位置要点
模板方法AbstractOssClientImpl + DefaultOssClientImpl父类 70+ 方法，子类只覆写 doInitialize()
工厂OssFactoryConcurrentHashMap 缓存 + ReentrantLock 按 configKey 粒度
策略AccessPolicy 枚举type 字符串 → BucketCannedACL + ObjectCannedACL
BuilderOssClientConfig, Options@Builder / @Accessors(chain=true) 链式构建
适配器OutputStreamDownloadSubscriberConsumer<ByteBuffer> 适配到 WritableByteChannel

核心调用链

业务代码 → OssFactory.instance() ← 读 Redis 配置 → OssProperties → OssClientConfig → ossClient.upload(key, file) ← DefaultOssClientImpl（继承 AbstractOssClientImpl） → s3TransferManager.upload() ← AWS S3 SDK（封装 S3AsyncClient） → CompletableFuture.join() → PutObjectResult(url, key, eTag, size) → 存入 sys_oss 表

上传通路（6 种输入 → 1 个漏斗）

File / Path / InputStream / byte[] / RandomAccessFile / ReadableByteChannel → 各自转为 AsyncRequestBody（fromPath / fromChannel 等） → 收敛到私有方法 bucketUpload(bucket, key, AsyncRequestBody body, Options options) → doCustomUpload(body, requestBuilder, listeners) → s3TransferManager.upload(UploadRequest) → .join() 等待 → PutObjectResult

下载通路

GetObjectRequest(bucket + key) → s3TransferManager.download(DownloadRequest) → AsyncResponseTransformer（toFile / OutputStreamDownloadSubscriber） → WritableByteChannel / OutputStream / File → GetObjectResult(contentLength, contentType)

记忆锚点

  • 继承链：OssClient（接口 60+ 方法）→ AbstractOssClientImpl（骨架 70+ 方法）→ DefaultOssClientImpl（只覆写 doInitialize()）。
  • 三剑客：doInitialize() 里组装 S3AsyncClient（连接池）+ S3TransferManager（上传/下载）+ S3Presigner（预签名 URL）。
  • 工厂缓存：OssFactory.CLIENT_CACHE（ConcurrentHashMap）按 configKey 缓存客户端，配置不变则复用。
  • 上传漏斗：6 种输入类型最终收敛到私有方法 bucketUpload(bucket, key, AsyncRequestBody body, Options options)。
  • 与 SMS 对比：SMS 3 个类 vs OSS 18 个类，但骨架相同：依赖成熟库 + 补「配置装配 + 客户端缓存 + 异常兜底」。
  • 模块不暴露 Controller，是被业务层（如 SysOssController）依赖的公共能力层。
