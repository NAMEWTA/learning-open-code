DefaultOssClientImpl：doInitialize() 步骤

步骤操作关键参数
1. 校验凭证检查 accessKey、secretKey 非空—
2. 解析端点endpointUrl（API 调用）、domainUrl（文件访问）、region默认 Region.US_EAST_1
3. 构建凭证StaticCredentialsProvider.create(AwsBasicCredentials.create(ak, sk))—
4. 构建 S3AsyncClient凭证 + 端点 + region + 路径风格 + 校验和策略见下方详解
5. 构建 HTTP 客户端NettyNioAsyncHttpClient：连接超时 60s、获取超时 30s、最大并发 100、等待队列 1000—
6. 校验和策略WHEN_REQUIRED（请求和响应均仅当服务端要求时）省 CPU
7. 构建 S3TransferManager基于 S3AsyncClient—
8. 构建 S3Presigner使用 domainUrl（不是 endpointUrl）作为端点—
9. 创建 asyncExecutor虚拟线程（Java 21+）或 ScheduledThreadPool（降级）由配置决定

OssFactory：instance() 加锁流程

步骤操作说明
1RedisUtils.getCacheObject(DEFAULT_CONFIG_KEY)从 Redis 取当前生效的配置 key
2CacheUtils.get("sys_oss_config", configKey)从缓存读配置 JSON
3JsonUtils.parseObject(json, OssProperties.class)JSON → POJO
4OssClientConfig.formProperties(properties)POJO → 客户端配置对象（字段映射 + 策略解析）
5getClientLock(configKey).lock()获取该 configKey 专属的 ReentrantLock
6CLIENT_CACHE.get(configKey)第二次检查缓存（DCL）
7a若存在且 verifyConfig(newConfig) == true直接返回缓存的客户端实例
7b若存在但 verifyConfig(newConfig) == falsecloseClient() 关闭旧实例 → 创建新 DefaultOssClientImpl
7c若不存在创建新 DefaultOssClientImpl → 放入 CLIENT_CACHE
8lock.unlock()（finally）释放锁

关键数据结构

数据结构类型用途
CLIENT_CACHEConcurrentHashMap<String, OssClient>按 configKey 缓存客户端实例
CLIENT_LOCKSConcurrentHashMap<String, ReentrantLock>按 configKey 提供独立锁，避免不同厂商互相阻塞
initializedAtomicBoolean保证每个客户端只初始化一次（close 后重置可再次初始化）

记忆锚点

  • 三层继承链：OssClient（接口/契约）→ AbstractOssClientImpl（抽象类/收敛）→ DefaultOssClientImpl（实现类/初始化）。新增厂商只需再写一个 doInitialize()。
  • 两个私有方法收敛 70+ 公开方法：bucketUpload(AsyncRequestBody) + doCustomDownload()。所有上传重载本质都是「把输入转成 AsyncRequestBody，然后调同一个私有方法」。
  • 工厂的 DCL + per-key 锁：并发安全 + 不互相阻塞 + 配置变更自动重建客户端，实现零停机热更新。
  • 虚拟线程优先：Java 21+ 用虚拟线程，低版本退到 ScheduledThreadPool。同一套代码适配不同 JDK。
  • InputStream → 临时文件技巧：S3 传输需要 contentLength，InputStream 不提供 → 先落盘再上传，finally 清理。用空间换确定性。
  • close() 顺序：先停传输 → 再断连接 → 再关辅件 → 最后收线程。顺序错了可能导致任务丢失或连接泄漏。
