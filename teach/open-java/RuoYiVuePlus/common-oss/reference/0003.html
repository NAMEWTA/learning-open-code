速查 03 · 配置体系：OssProperties → OssClientConfig 转换链

配套课程：lessons/0003.html · 使命：读懂 ruoyi-common-oss 对象存储模块

一句话定位

将数据库 JSON 字符串（前端可编辑的弱类型字段）通过 OssProperties → OssClientConfig.formPropertiesBuilder() 转换为不可变、强类型、Optional 包装的内部配置对象，供 DefaultOssClientImpl 安全使用。

配置转换全链路

数据库 sys_oss_config │ │ JSON 字段：endpoint, domainUrl, prefix, accessKey, secretKey, bucketName, region, isHttps("Y"/"N"), accessPolicy("0"/"1"/"2") v Redis（key: global:sys_oss:default_config） │ v CacheUtils.get(DEFAULT_CONFIG_KEY) → JSON 字符串 │ v JsonUtils.parseObject(jsonStr, OssProperties.class) → @Data POJO (9 个字段) │ v OssClientConfig.formProperties(ossProperties) │ ├ .useHttps(​SystemConstants.YES.equals(isHttps)) —— "Y"→true │ ├ .usePathStyleAccess(resolvePathStyleAccess(properties)) —— 自动推断 │ ├ .region(parseRegion(regionString)) —— String→Region │ ├ .accessControlPolicyConfig(resolveAccessControlPolicy(str)) —— String→record │ ├ 其余字段直传(endpoint/domain/accessKey/secretKey/bucket/prefix) v OssClientConfig (11 final 字段，@Builder 构建，不可变) │ v DefaultOssClientImpl 构造函数

方法签名职责

toBuilder OssClientConfigBuilder toBuilder() 转为 Builder：允许基于当前配置局部修改

关键设计意图

  • 两层配置分离：OssProperties（外部 JSON 对等物，弱类型）→ OssClientConfig（内部强类型，不可变）——解耦存储格式与运行时代码。
  • Optional 包装：所有可能为 null 的字段用 Optional.ofNullable() 包裹，调用方链式处理空值，避免 NPE。
  • @EqualsAndHashCode：用于配置变更检测——OssFactory 比较新旧 config，决定复用还是重建客户端。
  • 防御性拷贝：copy() 方法保证客户端持有配置快照，外部修改不影响运行中的实例。
  • 向后兼容：resolvePathStyleAccess 通过 endpoint 字符串推断 URL 风格，是旧版配置缺少 pathStyle 字段的妥协方案。
  • fail-fast：AccessPolicy.formType() 找不到匹配直接抛 S3StorageException，不在运行时默默使用错误策略。
