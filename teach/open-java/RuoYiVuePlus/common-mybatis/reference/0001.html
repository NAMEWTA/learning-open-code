一句话定位

ruoyi-common-mybatis 是所有业务模块与数据库之间的唯一持久层通道，34 个 Java 文件分布在 9 个包中，按功能聚合成四大支柱。

四大支柱速查表

支柱核心类一句话描述
一：BaseMapperPlus BaseMapperPlus<T,V>、BaseEntity、InjectionMetaObjectHandler 双泛型 Mapper + 审计字段自动填充 + Entity→VO 自动转换
二：数据权限 @DataPermission、PlusDataPermissionInterceptor、PlusDataPermissionHandler 注解声明 → AOP 拦截 → MyBatis 注入 WHERE 条件（三层协作）
三：增强查询构建器 LambdaQueryBuilder、LambdaJoinQueryBuilder、LambdaCrudChainWrapper、SubQuery 纯 Java Lambda 写类型安全的联表+子查询+聚合+方言兼容
四：多数据源+SQL日志 DataBaseHelper、SqlLogInterceptor @DS 动态切换数据源 + 自动识别数据库类型 + 完整 SQL 还原日志

必须记住的设计动机（4 点）

  1. BaseMapperPlus 的双泛型不是炫技——它消除了 Service 层中每个查询方法末尾的 MapstructUtils.convert(entity, VoClass.class) 重复代码。
  2. 数据权限的三层架构不是过度设计——切面（AOP）负责「什么时候需要过滤」，拦截器（MyBatis Plugin）负责「在哪过滤」，处理器负责「怎么生成过滤条件」。职责分离使得每一层可以独立测试和替换。
  3. LambdaQueryBuilder 和 LambdaCrudChainWrapper 不是重复——前者是独立的查询构建器，适合在 Service 层构建复杂查询条件；后者是 Mapper 绑定的链式包装器，适合在 Mapper 方法内快速 CRUD。
  4. SqlLogInterceptor 不是日志框架的替代品——它的核心价值是将 ? 占位符还原为实际参数值，输出可直接拷贝到数据库客户端执行的完整 SQL，大幅降低调试成本。
