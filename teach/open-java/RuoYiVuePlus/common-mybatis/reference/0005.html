五、使用场景决策

我应该用哪个 Builder？

你的查询场景是什么？ │ ├── 单表查询（条件筛选/列表） │ ├── 使用独立的 QueryBuilder → LambdaQueryBuilder │ │ QueryBuilder.lambda(SysUser.class).eq(...).build() │ │ │ └── 在 Mapper 中链式调用 → LambdaCrudChainWrapper │ mapper.lambda().eq(...).list() │ ├── 需要聚合统计（SUM/COUNT/GROUP BY） │ ├── 单表聚合 → LambdaQueryBuilder.selectSum/Max/...() │ └── 联表聚合 → LambdaJoinQueryBuilder.selectSum/Max/...() │ ├── 需要 JOIN 多张表 │ └── 只有一条路 → LambdaJoinQueryBuilder │ QueryBuilder.lambdaJoin("alias", Entity.class) │ .leftJoin(...) │ .list(VoClass.class) │ ├── 需要增删改操作 │ └── 只有一条路 → LambdaCrudChainWrapper │ mapper.lambda().eq(...).update(entity) │ mapper.lambda().eq(...).delete() │ └── 需要子查询（IN / EXISTS / 标量子查询） ├── LambdaQueryBuilder → eqSub / inSub / existsSub / selectSub └── LambdaJoinQueryBuilder → 同上，但需传表别名
tip：LambdaQueryBuilder 和 LambdaCrudChainWrapper 的功能有重叠。简单规则：如果查询需要传给其他方法（如 Service 层的公共查询方法），用 LambdaQueryBuilder（它返回纯 Wrapper）；如果查询紧跟着执行（list/page/count），用 LambdaCrudChainWrapper（它链式到底）。
