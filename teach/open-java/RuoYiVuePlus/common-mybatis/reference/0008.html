三、关键设计模式速查

设计模式应用位置说明
Fluent API (链式调用)LambdaQueryBuilder、LambdaCrudChainWrapper每个条件方法返回 this，支持无限链式调用
Self-Type 泛型LambdaQueryCondition<T, Children>Children 泛型让接口 default 方法返回值类型正确推导
组合 + 委派LambdaQueryBuilder 持有 LambdaQueryWrapper不继承 MP 类，通过组合包装增强功能
AOP + Interceptor 双层拦截DataPermissionAdvice + PlusDataPermissionInterceptorAOP 捕获注解，Interceptor 改写 SQL，权限与业务解耦
策略模式（枚举策略）DataScopeType 的 sqlTemplate 字段每种数据范围有不同的 SpEL 模板，"策略"由枚举值承载
工厂模式QueryBuilder.lambda() / lambdaJoin()静态工厂方法，隐藏底层包装器创建细节
适配器模式BaseMapperPlus 对原生 BaseMapper在原生 Mapper 上增加 VO 转换、链式 CRUD 能力
条件装配@ConditionalOnProperty 控制 SqlLogInterceptor配置驱动，不启用时不创建 Bean

四、API 入口速查表

想要做什么使用方式
单表查询（条件 + 聚合）QueryBuilder.lambda(Entity.class).eq(...).selectSum(...).build()
多表 JOIN 查询QueryBuilder.lambdaJoin("alias", Entity.class).leftJoin(...).list(Vo.class)
Mapper 层链式增删改查baseMapper.lambda().eq(...).list() / .update(entity) / .delete()
数据权限自动过滤Mapper 方法上加 @DataPermission({@DataColumn(key="deptName", value="dept_id")})
切换数据源@DS("slave") 或 @DS("#paramName")
分页 + 多字段排序PageQuery.build() → mapper.selectPage(page, wrapper)
子查询（标量子查询）builder.selectSub(SubEntity.class, sub -> sub.selectCountAll().eqColumn(...), Vo::getField)
EXISTS 子查询过滤builder.existsSub(SubEntity.class, sub -> sub.selectCountAll().eqColumn(...))
跨数据库 FIND_IN_SETbuilder.findInSet(value, column) → 自动方言适配
SQL 日志调试mybatis-plus.sql-log.enabled=true
全局关闭逻辑删除mybatis-plus.enableLogicDelete=false

十、故障排查决策树

遇到问题？ │ ├── 数据权限不生效？ │ ├── Mapper 方法有 @DataPermission 吗？ → 没有 → 加上 │ ├── 当前登录用户是超管吗？ → 是 → 预期行为，换非超管测试 │ ├── 被 @InterceptorIgnore 跳过了吗？ → 是 → 移除或设 false │ └── ThreadLocal 被清了吗？ → 检查 DataPermissionHelper 生命周期 │ ├── 查询结果为空？ │ ├── sql-log.enabled=true 查看完整 SQL │ ├── SQL 中有 "1 = 0" 吗？ → 有 → SpEL 变量缺失 │ │ └── 检查 @DataColumn key 是否匹配 sqlTemplate 变量 │ └── 数据被逻辑删除？ → enableLogicDelete=false 临时关闭 │ ├── "未找到数据源"？ │ ├── @DS("xxx") 的 xxx 在 YAML 中存在吗？ │ └── dynamic.strict=true 时名称必须完全一致 │ └── 联表查询报错？ ├── LambdaJoinQueryBuilder 的条件方法传入表别名了吗？ └── selectAs 的字段映射正确吗？
