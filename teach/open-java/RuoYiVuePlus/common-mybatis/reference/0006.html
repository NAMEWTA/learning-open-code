三、findInSet 方言对照表

数据库生成的 SQL函数
MySQL（默认）find_in_set('{value}', {column}) <> 0find_in_set
Oracleinstr(','||{column}||',' , ',{value},') <> 0instr
PostgreSQL(select strpos(','||{column}||',' , ',{value},')) <> 0strpos
SQL Servercharindex(',{value},' , ','+{column}+',') <> 0charindex
核心设计：所有方言都在值/字段前后加逗号，避免部分匹配误判。如字段值为 '12,123' 时，instr(',12,123,', ',12,') 精确匹配 "12" 而非误匹配 "123" 中的 "12"。

四、@DS 注解使用速查

写法含义示例场景
@DS("slave")静态指定数据源名称读从库的 Mapper 或 Service 方法
@DS("#paramName")SpEL 引用方法参数代码生成器连接用户指定的数据源
@DS("#obj.field")SpEL 引用对象属性GenTable.dataName 动态路由
@DS("")空字符串 — 回退 primary清空线程数据源标记，使用默认数据源
优先级：方法级 @DS > 类级 @DS > spring.datasource.dynamic.primary

六、SqlLogInterceptor 执行流程

intercept(Invocation) ├─ StatementHandler ← PluginUtils.realTarget() ├─ BoundSql + MappedStatement ├─ startTime = now() ├─ try: result = invocation.proceed() │ └─ printSql(ms, boundSql, elapsed, null) └─ catch: printSql(ms, boundSql, elapsed, e); throw e │ ▼ buildFullSql(config, boundSql) ├─ sql.replaceAll("\\s+", " ").trim() ├─ buildParameterValues() │ ├─ 遍历 ParameterMapping 列表 │ ├─ 从 additionalParameter / parameterObject / MetaObject 取值 │ └─ formatParameter() — 按类型格式化 │ └─ replacePlaceholders(sql, params) └─ 逐字符扫描，仅替换引号外的 ? 占位符 → buildLogMessage(ms, elapsed, fullSql, throwable) → printChunk(message) ├─ output="console" → System.err (ReentrantLock 加锁) └─ output="log" ├─ ≤8000 字符 → log.info(message) └─ >8000 字符 → 分片输出 (sqlLogId={UUID} part={n}/{total})
