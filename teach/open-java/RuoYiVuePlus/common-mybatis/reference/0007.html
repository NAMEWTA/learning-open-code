一、数据权限四阶段执行链路

【阶段1：AOP 注解捕获】 DataPermissionPointcut 匹配 @DataPermission → DataPermissionAdvice.invoke() → 获取注解（方法级优先 → 类级 → JDK 代理接口级） → DataPermissionHelper.setPermission(dataPermission) 存入 ThreadLocal → invocation.proceed() → finally: DataPermissionHelper.removePermission() 清理 【阶段2：MyBatis 拦截器检查】 PlusDataPermissionInterceptor.beforeQuery() / beforePrepare() → InterceptorIgnoreHelper.willIgnoreDataPermission() 检查忽略标记 → dataPermissionHandler.invalid() 检查 ThreadLocal 中是否有 @DataPermission → parserSingle(sql, msId) 解析并改写 SQL 【阶段3：SQL 改写入口】 PlusDataPermissionHandler.getSqlSegment(where, msId, isSelect) → LoginHelper.isSuperAdmin() → 超管直接返回原 where → 非超管 → buildDataFilter() 【阶段4：数据过滤构建】 buildDataFilter() → 确定连接词（SELECT → OR，UPDATE/DELETE → AND） → 构建 NullSafeStandardEvaluationContext（null 变量返回 "-1"） → currentAccess() 解析 @SaCheckPermission/@SaCheckRole → scopeRoles() 筛选用户角色 → 按 dataScope 值匹配 DataScopeType → SpEL 解析 sqlTemplate → JSqlParser 构建 WHERE 条件 → 多角色用 joinStr 连接

code枚举值SpEL 模板（简化）SQL 效果
"1"ALL（空字符串）不追加任何条件 — 查看全部数据

放置位置适用场景优先级查找位置

五、自定义 DataScopeType 五步法

步骤操作关键代码
1. 加枚举在 DataScopeType 添加新的 code + sqlTemplate + elseSqlDEPT_AND_LINKED("7", " ... ", " 1 = 0 ")
2. 加接口在 ISysDataScopeService 声明方法Set<Long> getDeptAndLinked(Long deptId);
3. 加实现在 SysDataScopeServiceImpl 实现查询关联部门表，通过 @sdss.xxx() 调用
4. 配角色角色管理页面设置 dataScope 值sys_role.data_scope = "7"
5. 加注解Mapper 方法/类上加注解@DataPermission({@DataColumn(key="deptName", value="dept_id")})
扩展原理：buildDataFilter() 通过 DataScopeType.findCode(role.getDataScope()) 动态匹配枚举。新增后框架自动识别，无需修改框架代码。

六、五种常见故障速查

故障现象最可能原因快速修复
数据权限不生效未加 @DataPermission 注解 / 超管账号测试检查 Mapper 注解；换非超管账号测试
列表始终返回空SpEL 变量缺失 → elseSql 生成 "1 = 0"检查 @DataColumn key 是否匹配 sqlTemplate 变量名；确保 SpEL 上下文变量已设置
"未找到数据源" 500 错误@DS 值与 YAML 配置不匹配 / 数据源未定义检查 application-dev.yml 中 datasource 名称和 @DS 注解值一致
ThreadLocal 数据串扰DataPermissionHelper 未清理确保 afterCompletion / finally 中调用 removePermission()
SQL 日志无参数值mybatis-plus.sql-log.enabled 未开启设置 enabled=true；确认 logImpl 为 NoLoggingImpl 避免冲突

十、Mapper 代码组织最佳实践

// 推荐模式 @Mapper @DataPermission({ // 类级数据权限（所有方法共享） @DataColumn(key = "deptName", value = "dept_id"), @DataColumn(key = "userName", value = "create_by") }) public interface SysUserMapper extends BaseMapperPlus<SysUser, SysUser> { // 使用 Lambda 查询：类型安全 + 可读性强 default List<SysUser> selectByDept(Long deptId) { return selectList(QueryBuilder.lambda(SysUser.class) .eq(SysUser::getDeptId, deptId) .build()); } // @InterceptorIgnore 跳过特定拦截器（如数据权限、租户） @InterceptorIgnore(dataPermission = "true") List<SysUser> selectAllIgnorePermission(); // 方法级注解覆盖类级（如某些方法需要不同的权限字段） @DataPermission({ @DataColumn(key = "userName", value = "user_id") // 方法级覆盖 }) List<SysUser> selectOwnData(); }
核心设计原则：AOP + MyBatis Interceptor 实现"权限检查"与"业务查询"彻底解耦。业务代码不需要写任何 WHERE dept_id = ?，数据权限是透明注入的。双重控制 = @DataPermission（数据级） + @SaCheckPermission（功能级）。
