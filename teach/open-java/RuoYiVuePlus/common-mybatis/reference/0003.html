三层架构总览

层类角色关键方法
注解层 @DataPermission / @DataColumn 声明"谁需要数据权限" value() / joinStr()
切面层 DataPermissionPointcut + DataPermissionAdvice 匹配方法 → 建立上下文 matches() / invoke()
拦截器层 PlusDataPermissionInterceptor 拦截 SQL → 注入 WHERE beforeQuery() / beforePrepare()
处理器层 PlusDataPermissionHandler 生成权限 SQL 条件 getSqlSegment() / buildDataFilter()

Pointcut + Advice 执行流

DataPermissionPointcut.matches(Method, Class) ├─ 1. method.getAnnotation(DataPermission.class) != null ? │ → YES: return true │ → NO: go step 2 ├─ 2. targetClass.getAnnotation(DataPermission.class) != null ? │ → YES: return true │ → NO: go step 3 └─ 3. targetClass 是 JDK Proxy ($Proxy 开头)? → YES: 遍历接口方法找 @DataPermission → NO: return false DataPermissionAdvice.invoke(MethodInvocation) ├─ findAnnotation(invocation) → DataPermission │ 优先: 方法注解 → 类注解 → Proxy接口注解 ├─ DataPermissionHelper.setPermission(注解, args) │ ├─ THREAD_LOCAL.set(dataPermission) // ThreadLocal │ └─ SaStorage: {user, params, ...} // Sa-Token 上下文 ├─ invocation.proceed() // 执行目标方法 → 触发拦截器 └─ finally: DataPermissionHelper.removePermission()

Interceptor 方法执行表

方法SQL 类型关键步骤
beforeQuery() SELECT ① InterceptorIgnoreHelper 检查
② handler.invalid() 有效性检查
③ JSqlParser parserSingle() 解析
beforePrepare() UPDATE / DELETE ① 同上检查忽略和有效性
② JSqlParser parserMulti() 解析
③ 角色条件用 AND 拼接
processSelect() SELECT 遍历 FROM 表 → handler.getSqlSegment() → 注入 WHERE
processUpdate() UPDATE handler.getSqlSegment() → AND 到原 WHERE
processDelete() DELETE 同上
