定位

企业级持久层模块（34 个 Java 文件），提供 BaseMapperPlus 增强映射、数据权限三层架构、增强查询构建器、多数据源、SQL 日志 五大核心能力，是所有业务模块 CRUD 的数据层骨架。

术语速查

术语解释
BaseMapperPlus双泛型 Mapper 接口，提供 Entity→VO 转换、批量插入/更新、链式 CRUD
数据权限三层架构切面层（收集注解）→ 拦截器层（触发处理）→ 处理器层（生成 SQL 并注入）
@DataPermission标记需要数据过滤的方法/类，含 @DataColumn 数组和 AND/OR 连接方式
DataScopeType6 种数据范围，每种绑定 SpEL 模板 SQL 和兜底 SQL（elseSql 通常为 1=0）
DataPermissionHelper用 ThreadLocal + SaStorage 桥接三层通信，ignore() 在子查询中关闭权限
LambdaQueryBuilder标准 CRUD + 聚合 + 子查询构建器，eqIfPresent/likeIfText 智能判空
雪花 ID用网卡 MAC 作 workerId 防集群冲突的分布式主键生成策略
SqlLogInterceptor还原完整 SQL 并输出日志，超长 8000 字符分片，控制台模式用 ReentrantLock 防穿插

常用代码

定义实体与 Mapper：

// 实体继承 BaseEntity 获得审计字段自动填充
@Data
@TableName("sys_user")
public class SysUser extends BaseEntity {
    @TableId(type = IdType.ASSIGN_ID)  // 雪花 ID
    private Long userId;
    private String userName;
}

// Mapper 继承 BaseMapperPlus，指定 VO 类型
public interface SysUserMapper extends BaseMapperPlus<SysUser, SysUserVo> {
}

Lambda 查询构建器：

// 单表查询
List<SysUser> users = QueryBuilder.lambda(SysUser.class)
    .eq(SysUser::getStatus, "0")
    .likeIfText(SysUser::getUserName, keyword)
    .orderByAsc(SysUser::getCreateTime)
    .list();

// 联表查询
List<SysUserVo> list = QueryBuilder.lambdaJoin(SysUser.class)
    .selectAll(SysUser.class)
    .leftJoin(SysDept.class, SysDept::getDeptId, SysUser::getDeptId)
    .eq(SysUser::getStatus, "0")
    .list(SysUserVo.class);

数据权限注解使用：

@DataPermission({
    @DataColumn(key = "deptName", value = "dept_id"),
    @DataColumn(key = "userName", value = "user_id")
})
List<SysUserVo> selectUserList();
