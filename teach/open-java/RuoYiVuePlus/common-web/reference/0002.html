速查 02 · XSS 防护体系

配套课程：lessons/0002.html

一句话定位

XssProperties(配置开关+白名单) → XssFilter(路由决策: 放行还是清洗) → XssHttpServletRequestWrapper(内容清洗: HtmlUtil.cleanHtmlTag)

三层防线

层类职责
配置层XssPropertiesenabled 总开关 + excludeUrls 白名单
Filter层XssFilter判断是否过滤: GET/DELETE 放行, 白名单放行, 其余包装
Wrapper层XssHttpServletRequestWrapper重写 getParameter/getParameterMap/getInputStream, 在每个点执行 cleanHtmlTag

XssFilter 放行规则

条件行为
请求方法为 GET 或 DELETE直接放行（无请求体，XSS 风险低）
URL 匹配 excludeUrls 白名单直接放行（如富文本编辑器路径）
其他情况创建 XssHttpServletRequestWrapper 包装后继续

XssHttpServletRequestWrapper 清洗点

方法清洗方式
getParameter(name)HtmlUtil.cleanHtmlTag(value).trim()
getParameterValues(name)遍历数组，逐个 cleanHtmlTag + trim
getParameterMap()copy → 逐 entry 清洗（防容器不可改 Map）
getInputStream()JSON? → 读取 body → cleanHtmlTag → 重建 ByteArrayInputStream
非 JSON → 返回原始流

HtmlUtil.cleanHtmlTag() 效果

输入输出
<script>alert(1)</script>空字符串
<div onclick="x">text</div>text
<b>bold</b>bold
正常文本正常文本

配置示例

xss: enabled: true # @ConditionalOnProperty 控制 Bean 是否注册 excludeUrls: - /system/notice/* # 富文本公告保留 HTML - /tool/swagger/*

记忆锚点

  • 三层防线：配置(开关+白名单) → Filter(路由) → Wrapper(清洗)
  • GET/DELETE 自动放行——它们没有请求体，XSS 载荷主要在 POST/PUT body 中
  • getParameterMap 先 copy 再改——防止容器 UnmodifiableMap 抛异常
  • 清洗策略是全量去标签，不是白名单保留——富文本路径必须排白
  • @ConditionalOnProperty 控制的是 Bean 是否存在，不是 Filter 内部开关——xss.enabled=false 时 XssFilter 不存在于 Filter 链中
  • XssHttpServletRequestWrapper 的 isFinished() 固定返回 true（body 已完整读入内存）

→ 相关术语见 GLOSSARY.md：XssFilter、XssHttpServletRequestWrapper、XssProperties、@ConditionalOnProperty
