速查 03 · 请求包装与过滤器注册

配套课程：lessons/0003.html

核心问题

HttpServletRequest 的 InputStream 基于网络 Socket 流，只能读一次。Interceptor 读了，Controller 就为空 → Required request body is missing。

RepeatableFilter + RepeatedlyRequestWrapper

组件职责
RepeatableFilter判断请求类型 → 仅 JSON 请求创建 RepeatedlyRequestWrapper
RepeatedlyRequestWrapper构造时 IoUtil.readBytes(is, false) 缓存 body → getInputStream() 每次从 byte[] 重建 ByteArrayInputStream

两个 Wrapper 对比

RepeatedlyRequestWrapperXssHttpServletRequestWrapper
目标让 body 可重复读清洗 HTML 标签
body 处理缓存原始 byte[]，每次重建流读取 → cleanHtmlTag → 重建流
编码设置UTF-8不处理
getParameter委托父类（不变）cleanHtmlTag + trim
isFinishedbais.available() == 0固定 true
JSON 判断Filter 判断Wrapper 自己判断

Filter 注册与顺序

FilterConfig 中: 1. RepeatableFilter @FilterRegistration(name="repeatableFilter", urlPatterns="/*") [无 order → 默认] 2. XssFilter @FilterRegistration(name="xssFilter", order=HIGHEST_PRECEDENCE+1) @ConditionalOnProperty(value="xss.enabled", havingValue="true")

记忆锚点

  • 必须先缓存再清洗：RepeatableFilter 在前（缓存原始），XssFilter 在后（清洗）——顺序不可颠倒
  • IoUtil.readBytes(is, false)：false = 读完后不关闭流（容器管理）
  • 每次 getInputStream() 都 new ByteArrayInputStream(body)：保证每个消费者拿到独立流，互不干扰
  • 只对 JSON 请求包装：form 参数不依赖 InputStream，multipart 有自己的机制
  • 拦截器通过 instanceof RepeatedlyRequestWrapper 检查后安全读取 body
