配套课程：lessons/0001.html · 使命：读懂 ruoyi-common-web Web基础设施模块

一句话定位

ruoyi-common-web 是 RuoYi 的 Web 层安全与可观测性基础设施——17 个类覆盖 Filter 安全清洗、Interceptor 耗时统计、全局异常兜底、响应字段增强、验证码与国际化五大域。

五大能力域 + 自动装配

域关键类在请求链位置
过滤器链RepeatableFilter + XssFilterServlet 层，Controller 之前
拦截器PlusWebInvokeTimeInterceptorSpring MVC 层，Controller 前后
异常处理GlobalExceptionHandlerController 抛异常后
响应增强ResponseEnhancementAdviceController 返回后、序列化前
通用组件BaseController + WaveAndCircleCaptcha + I18nLocaleResolver贯穿请求周期

请求处理全链路

HTTP 请求 → RepeatableFilter (JSON?→缓存body到byte[]) → XssFilter (GET/DELETE跳过，否则清洗HTML标签) → PlusWebInvokeTimeInterceptor.preHandle (记录参数+开始计时) → Controller (业务逻辑) ├─ 正常: ResponseEnhancementAdvice → jsonValueEnhancer.enhance() └─ 异常: GlobalExceptionHandler → log.error + R.fail() → JSON 序列化 → 返回客户端

I18nConfigI18nLocaleResolverbefore = WebMvcAutoConfiguration
ResourcesConfigCORS / Interceptor / ExceptionHandler / ResponseAdvice / 日期转换实现 WebMvcConfigurer

记忆锚点

  • Filter 负责环境准备（缓存body、清洗XSS），Interceptor 负责横切逻辑（日志、计时）。
  • RepeatableFilter 必须先于 XssFilter：先缓存原始数据，再清洗。
  • Interceptor 读 JSON body 依赖 RepeatableFilter 的包装——原生 HttpServletRequest 的 InputStream 只能读一次。
  • 正常返回走 Advice，异常走 Handler——同一次请求两者不共存。
