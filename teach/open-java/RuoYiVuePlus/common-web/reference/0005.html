核心原则：内外有别

内部: log.error("请求地址'{}',异常信息, 错误编号: {}", uri, errorId, e) ↑ 完整堆栈 ↑ 错误编号 外部: R.fail("发生系统异常，请联系管理员 [错误编号: " + errorId + "]") ↑ 友好提示 不暴露技术细节

ResponseEnhancementAdvice

supports(returnType, converterType) → jsonValueEnhancer.supports(converterType) ? 继续 : 跳过 beforeBodyWrite(body, ..., contentType, ...) → contentType 兼容 JSON? → jsonValueEnhancer.enhance(body) → 否则 → return body (文件下载等不增强)

BaseController 三个方法

方法代码
toAjax(int rows)rows > 0 ? R.ok() : R.fail()
toAjax(boolean result)result ? R.ok() : R.fail()
redirect(String url)StringUtils.format("redirect:{}", url)

记忆锚点

  • 内外有别：log.error(详情+堆栈) + R.fail(友好提示+错误编号)，绝不暴露技术细节给客户端
  • SseException 返回 String：SSE 不能依赖 @ResponseBody 自动序列化
  • IOException 区分 SSE：SSE 路径静默（断线正常），其他记日志
  • AsyncRequestTimeoutException 空处理：SSE 超时是预期行为
  • 4 种校验异常：Bind / MethodArgumentNotValid / ConstraintViolation / HandlerMethodValidation —— 全部用 StreamUtils.join 合并错误消息
  • ResponseEnhancementAdvice 是路由层：web 模块管"何时增强"，json 模块管"如何增强"
