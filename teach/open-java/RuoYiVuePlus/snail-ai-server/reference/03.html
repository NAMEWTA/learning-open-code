速查 03 · 配置驱动的装配 配置核心

application.yml 配置全景（70 行，7 个配置块）

1. HTTP 服务与静态资源

server:
  port: 8900
  servlet:
    context-path: /snail-ai

spring:
  application:
    name: ruoyi-snailai-server
  servlet.multipart:
    enabled: true
    max-file-size: 50MB       # RAG 文档上传
    max-request-size: 100MB
  web.resources:
    static-locations: classpath:admin/   # 管理后台静态页面

2. gRPC 通信

snail-ai:
  server:
    grpc-port: 18888          # ruoyi-ai 适配器的 AI 能力调用入口

gRPC 基于 HTTP/2 + Protobuf：体积小、解析快、支持双向流（支撑 SSE 流式对话）。仅限内网使用。

3. Skill 上传与 AES 加密

snail-ai:
  skill:
    upload-dir: ./upload/skills
  crypto:
    secret-key: ${SNAIL_AI_CRYPTO_KEY:0123456789abcdef0123456789abcdef}
    iv: ${SNAIL_AI_CRYPTO_IV:fedcba9876543210fedcba9876543210}

${ENV:default} 语法：优先读环境变量，未设置用默认值。生产环境必须通过环境变量覆盖默认密钥。

4. 资源存储：LOCAL vs MinIO

snail-ai:
  resource:
    storage-type: LOCAL       # 当前使用本地存储
    upload-dir: ./upload/resource
    minio:                    # MinIO 预留配置
      endpoint: http://localhost:9000
      access-key: minioadmin
      secret-key: minioadmin
      bucket: snail-ai
"配置预留"模式：切换存储后端只需改 storage-type: MINIO + 真实 MinIO 连接信息。无需改代码。Starter 内部通过策略模式/条件装配切换实现。

5. 短期记忆策略

snail-ai:
  memory:
    short-term:
      store-type: db          # memory（进程内存）| db（MySQL 持久化）
策略存储性能持久性场景
memoryJVM 内存（如 ConcurrentHashMap）极高重启丢失单机开发/测试
dbMySQL（MyBatis-Plus）较低（JDBC+SQL+IO）持久化生产/分布式

6. MyBatis-Plus 配置

mybatis-plus:
  typeAliasesPackage: com.aizuda.snail.ai.persistence
  global-config.db-config:
    capital-mode: false
    logic-delete-value: 1          # 逻辑删除标记
    logic-not-delete-value: 0
  configuration:
    map-underscore-to-camel-case: true   # user_name → userName
    cache-enabled: true

该配置段存在的原因：store-type: db 需要 MyBatis-Plus 做短期记忆持久化。实体类在 com.aizuda.snail.ai.persistence（starter 内部）。

7. Actuator 全端点暴露

management:
  endpoints.web.exposure.include: '*'
  endpoint.health.show-details: ALWAYS
  endpoint.logfile.external-file: ./logs/ruoyi-snailai-server/console.log
端点路径提供信息
health/snail-ai/actuator/health健康状态（DB 连接、磁盘空间）
env/snail-ai/actuator/env环境变量和配置（含敏感信息）
metrics/snail-ai/actuator/metricsJVM 内存、GC、线程池指标
logfile/snail-ai/actuator/logfile应用日志内容（console.log）

环境配置双层结构

文件内容激活方式
application.yml公共配置（端口、AI 能力、MyBatis-Plus、Actuator）始终加载
application-dev.yml数据源（HikariCP 连接池）+ Admin 客户端注册spring.profiles.active: dev
application-prod.yml同上，生产环境差异spring.profiles.active: prod

Profile 切换通过 @profiles.active@ Maven 占位符实现，构建时由 Maven profile 替换。

环境配置核心内容（以 dev 为例）

spring:
  datasource:
    type: com.zaxxer.hikari.HikariDataSource
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/ry-vue?...
    username: root
    password: root
    hikari:
      minimum-idle: 10
      maximum-pool-size: 20

spring.boot.admin.client:
  enabled: true
  url: http://localhost:9090/admin
  username: @monitor.username@    # Maven 构建时替换
  password: @monitor.password@
安全提醒：全端点暴露 + show-details: ALWAYS 意味任何人都能查看敏感运维信息。这就是 ActuatorAuthFilter（HTTP Basic Auth）存在的理由——详见速查 04。
