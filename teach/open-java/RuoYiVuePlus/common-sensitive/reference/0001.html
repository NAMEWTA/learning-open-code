速查 01 · ruoyi-common-sensitive 模块全景

配套课程：lessons/0001.html · 使命：读懂 ruoyi-common-sensitive 数据脱敏模块

一句话定位

ruoyi-common-sensitive 不是脱敏算法库，而是「注解驱动 + 策略模式 + JSON 管线织入」的脱敏调度框架——5 个 Java 文件，一行正则都没有，真正的脱敏算法在 hutool 里。

模块全部内容（5 个 Java 文件）

文件作用对应课
annotation/Sensitive.java声明：标注字段需要脱敏（strategy 必填，roleKey/perms 可选）第 2 课
core/SensitiveStrategy.java算法：17 种脱敏策略枚举，每种持有一个 Function<String,String>第 2 课
handler/SensitiveJsonFieldProcessor.java桥接：实现 JsonFieldProcessor，在 Jackson 序列化时拦截并替换字段值第 3 课
config/SensitiveConfig.java装配：@AutoConfiguration + @Bean 注册处理器第 3 课
core/SensitiveService.java条件：接口，按角色/权限决定是否对当前用户脱敏第 3 课

核心调用链（一条请求的脱敏全路径）

Controller return VO → Jackson ObjectMapper 序列化 → JsonValueEnhancer 接管（ruoyi-common-json） → SensitiveJsonFieldProcessor.supports() 检测 @Sensitive 注解 → SensitiveJsonFieldProcessor.process() → sensitive.strategy() // 取枚举项 → .desensitizer() // 取 Function → .apply(text) // 执行脱敏 → JSON 输出脱敏后的值

五个类 = 五个角色

类角色核心方法/属性
@Sensitive声明脱敏意图strategy() 必填，roleKey()/perms() 可选
SensitiveStrategy提供脱敏算法desensitizer() → Function<String,String>
SensitiveJsonFieldProcessor织入序列化管线supports() 检测注解，process() 执行替换
SensitiveConfig装配到 Spring@Bean 注册处理器
SensitiveService差异化脱敏条件isSensitive(roleKey[], perms[]) → boolean

依赖关系

ruoyi-common-sensitive ├── ruoyi-common-core → DesensitizedUtils + hutool └── ruoyi-common-json → JsonFieldProcessor + JsonFieldContext

记忆锚点

  • 模块零行业务代码侵入——开发者只在 VO 字段上加注解，不在 Controller/Service 里写脱敏调用。
  • 脱敏发生在 JSON 序列化输出阶段——数据库存明文，不影响内部查询和数据分析。
  • 脱敏不修改原 Java 对象——序列化完成后 VO 字段值仍然是明文。
  • 没有 SensitiveService 实现类时，无条件脱敏；有实现类时按角色/权限条件脱敏。
  • 新增脱敏策略只需在 SensitiveStrategy 枚举中加一项——不动处理器、不动配置、不动任何调用方。
