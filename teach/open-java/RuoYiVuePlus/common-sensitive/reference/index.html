ruoyi-common-sensitive 速查表

Maven: org.dromara:ruoyi-common-sensitive | 源码: ruoyi-common-sensitive/src/main/java/org/dromara/common/sensitive/

定位

注解驱动的数据脱敏模块（5 个 Java 文件）。在 JSON 序列化管线中对敏感字段自动掩码——数据库存明文，返回前端时才脱敏。

术语速查

术语解释
数据脱敏对敏感数据变形处理，保留格式特征前提下隐藏真实内容。在 JSON 序列化输出阶段执行
@Sensitive 注解标注 VO 字段触发脱敏，strategy() 指定策略，roleKey()/perms() 按角色/权限条件化脱敏
SensitiveStrategy枚举类型，17 种脱敏策略，每个持有一个 desensitizer 函数，底层调 hutool DesensitizedUtil
SensitiveJsonFieldProcessor实现 JsonFieldProcessor 接口，连接「注解声明」与「策略执行」的桥梁，在 Jackson 序列化时拦截字段值
JsonFieldProcessorruoyi-common-json 定义的核心扩展点，三阶段生命周期：collect → prepare → process
SensitiveService条件脱敏接口，按用户角色/权限决定是否执行脱敏（管理员看明文、普通用户看掩码）
DesensitizedUtilHutool 脱敏工具类，提供 11 种常用脱敏方法
DesensitizedUtilsRuoYi 自定义扩展，新增 mask() 通用灵活脱敏和 maskHighSecurity() 高安全脱敏

常用代码

// 1. VO 字段标注脱敏
public class UserVO {
    @Sensitive(strategy = SensitiveStrategy.PHONE)
    private String phonenumber;  // 134****4567

    @Sensitive(strategy = SensitiveStrategy.ID_CARD)
    private String idCard;       // 110***********1234

    @Sensitive(strategy = SensitiveStrategy.EMAIL,
               roleKey = "admin", perms = "system:user:query")
    private String email;        // 仅非管理员脱敏
}

// 2. 自定义脱敏规则（使用 DesensitizedUtils）
// 前3位可见 + 4个* + 后2位可见
String masked = DesensitizedUtils.mask("ABC1234567DE", 3, 2, 4);
// 结果: "ABC****DE"

// 3. 高安全脱敏（中间全部掩码）
String highMasked = DesensitizedUtils.maskHighSecurity("18812345678", 3, 4);
// 结果: "188****5678"
