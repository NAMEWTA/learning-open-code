@Sensitive 注解

标注在 VO 字段上，声明该字段需要脱敏。strategy 必填，roleKey/perms 可选（默认无实现时忽略）。
属性类型必填说明
strategySensitiveStrategy是脱敏策略，从 17 种枚举中选一个
roleKeyString[]否角色标识符，多个为 OR 关系。需配合 SensitiveService 实现类使用
permsString[]否权限标识符，多个为 OR 关系。需配合 SensitiveService 实现类使用

DesensitizedUtils 两个核心方法

方法签名效果
maskmask(value, prefixVisible, suffixVisible, maskLength)灵活脱敏：前N可见 + M个* + 后N可见。边界自动适配。
maskHighSecuritymaskHighSecurity(value, prefixVisible, suffixVisible)高安全脱敏：前N后N可见，中间用*填满。适合 token/私钥。

注解使用方式

// 最简：无条件脱敏 @Sensitive(strategy = SensitiveStrategy.PHONE) private String phone; // 按角色条件脱敏（需 SensitiveService 实现类） @Sensitive(strategy = SensitiveStrategy.PHONE, roleKey = "admin") private String phone; // 按权限条件脱敏 @Sensitive(strategy = SensitiveStrategy.EMAIL, perms = "system:user:list") private String email; // 多个条件 @Sensitive(strategy = SensitiveStrategy.ID_CARD, roleKey = {"admin","sub_admin"}, perms = "system:user:query") private String idCard;

记忆锚点

  • strategy 必填，无默认值 —— 忘了填编译就报错。
  • 策略本质是 Function<String, String> —— 策略即枚举即函数。
  • STRING_MASK 中间掩码固定长度，MASK_HIGH_SECURITY 中间掩码填满剩余空间。
  • 没有 SensitiveService 实现时，roleKey 和 perms 写了也不生效。
  • 脱敏只对 String 类型字段生效，Integer/Long 类型不处理。
