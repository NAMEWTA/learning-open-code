SensitiveJsonFieldProcessor 核心逻辑

实现 JsonFieldProcessor，在 Jackson 序列化时拦截带 @Sensitive 注解的 String 字段，按策略执行脱敏替换。

关键方法

方法逻辑备注
supports(JsonFieldContext)fieldContext.getAnnotation(Sensitive.class) != null有 @Sensitive 注解的字段才进入 process
process(fieldContext, value, context)1. 取注解 2. 检查 String 类型 3. 检查 SensitiveService 条件 4. 执行 strategy.desensitizer().apply(text)非 String 字段直接返回原值

SensitiveService 的三种状态

sensitiveService 状态@Sensitive 上的 roleKey/perms脱敏结果
null（默认，未注入）随意（写了也忽略）始终脱敏
有值，isSensitive 返回 true写了且匹配脱敏
有值，isSensitive 返回 false写了但用户不匹配不脱敏（返回明文）

SensitiveService 实现模板

@Component public class DefaultSensitiveService implements SensitiveService { @Override public boolean isSensitive(String[] roleKey, String[] perms) { // 无 roleKey 和 perms → 无条件脱敏 if (roleKey.length == 0 && perms.length == 0) return true; // 角色检查（OR 关系） for (String role : roleKey) { if (StpUtil.hasRole(role)) return true; } // 权限检查（OR 关系） for (String perm : perms) { if (StpUtil.hasPermission(perm)) return true; } return false; // 都不满足 → 不脱敏 } }

新增自定义脱敏策略（2 步）

步骤操作
1. 在 SensitiveStrategy 枚举中加一项CUSTOM_SAFETY_CODE(s -> DesensitizedUtils.maskHighSecurity(s, 3, 2)),
2. 在 VO 字段上使用@Sensitive(strategy = SensitiveStrategy.CUSTOM_SAFETY_CODE)

JsonFieldProcessor 三阶段生命周期

collect(每个字段一次) → prepare(全局一次) → process(每个字段一次) ↑ 脱敏处理器只用了 process 阶段
阶段脱敏处理器的行为
collect未重写 —— 脱敏不需要批量预处理，每个字段独立处理即可
prepare未重写 —— 脱敏不需要批量 IO 查询
process重写 —— 检查注解 + 条件判断 + 执行脱敏

记忆锚点

  • 处理器通过 @Order(100) 保证在其他处理器（字典翻译等）之后执行。
  • @Autowired(required = false) 让 SensitiveService 成为可选依赖——不提供也能正常运行。
  • 处理器只重写 supports() 和 process()，不参与 collect/prepare 阶段。
  • 新增策略 = 枚举加一项 + VO 字段上使用，不动处理器和配置。
  • SensitiveConfig 无任何条件注解，引入依赖即生效。
