第 8 课 · 观察者模式 — Spring Events 事件驱动 行为型模式

🎯 本节目标：掌握 Spring 事件机制中的观察者模式实践，理解 @EventListener 与 @Async 的配合方式，并能对比前端 Vue 3 中的响应式观察。

观察者模式解决的核心问题：一个对象的状态变化，需要自动通知多个依赖对象，且发布者不需要知道订阅者是谁。在 Spring 中，这个模式被内建为 ApplicationEvent 机制，项目中大量用于日志记录、登录通知等解耦场景。

一、项目中的三类事件

1.1 OperLogEvent — 操作日志事件

打开 ruoyi-common/ruoyi-common-log/src/main/java/org/dromara/common/log/event/OperLogEvent.java：

@Data
public class OperLogEvent implements Serializable {

    @Serial
    private static final long serialVersionUID = 1L;

    private Long operId;           // 日志主键
    private String title;          // 操作模块
    private Integer businessType;  // 业务类型（0其它 1新增 2修改 3删除）
    private String method;         // 请求方法
    private String requestMethod;  // 请求方式
    private Integer operatorType;  // 操作类别
    private String operName;       // 操作人员
    private Long userId;           // 操作用户ID
    private String operUrl;        // 请求url
    private String operIp;         // 操作地址
    private String operParam;      // 请求参数
    private String jsonResult;     // 返回参数
    private Integer status;        // 操作状态（0正常 1异常）
    private String errorMsg;       // 错误消息
    private LocalDateTime operTime;// 操作时间
    private Long costTime;         // 消耗时间
}
1 事件本质是 DTO：事件类就是一个携带数据的 POJO。它实现了 Serializable（异步场景下可能需要序列化），字段就是需要传递给监听器的信息。没有业务逻辑。 OperLogEvent.java

1.2 LoginInfoEvent — 登录信息事件

打开 ruoyi-common/ruoyi-common-log/src/main/java/org/dromara/common/log/event/LoginInfoEvent.java：

@Data
public class LoginInfoEvent implements Serializable {

    private String username;            // 用户账号
    private String status;              // 登录状态 0成功 1失败
    private String message;             // 提示消息
    private HttpServletRequest request; // 请求体（含 IP、UserAgent 等）
    private Object[] args;              // 其他参数
}
2 携带请求上下文：LoginInfoEvent 包含 HttpServletRequest，让监听器可以提取 IP、浏览器等信息。这说明事件可以携带任意复杂对象——不仅是简单值。 LoginInfoEvent.java

1.3 UserLoginSuccessEvent — 自定义业务事件

打开 ruoyi-admin/src/main/java/org/dromara/web/event/UserLoginSuccessEvent.java：

// Java 14+ record：不可变事件载体
public record UserLoginSuccessEvent(
    Object loginId,
    String tokenValue,
    SaLoginParameter loginParameter
) {}
3 Record 作为事件载体：使用 Java Record 定义事件天然不可变，避免监听器意外修改事件数据。这是比 @Data 更安全的选择——事件一旦发布就不应该被修改。 UserLoginSuccessEvent.java

二、监听器：事件的消费者

2.1 UserLoginSuccessListener — 经典 @EventListener

打开 ruoyi-admin/src/main/java/org/dromara/web/listener/UserLoginSuccessListener.java：

@RequiredArgsConstructor
@Component
@Slf4j
public class UserLoginSuccessListener {

    private final SysLoginService loginService;

    @EventListener  // ← 只需这一个注解，无需实现任何接口
    public void handleLoginSuccess(UserLoginSuccessEvent event) {
        // 从事件中提取登录参数
        SaLoginParameter loginParameter = event.loginParameter();
        UserAgent userAgent = UserAgentUtil.parse(
            ServletUtils.getRequest().getHeader("User-Agent"));
        String ip = ServletUtils.getClientIP();

        // 构建在线用户 DTO 并缓存到 Redis
        UserOnlineDTO dto = new UserOnlineDTO();
        dto.setIpaddr(ip);
        dto.setLoginLocation(AddressUtils.getRealAddressByIP(ip));
        dto.setBrowser(userAgent.getBrowser().getName());
        dto.setOs(userAgent.getOs().getName());
        dto.setTokenId(event.tokenValue());
        // ... 更多字段设置

        RedisUtils.setCacheObject(
            CacheNames.ONLINE_TOKEN_KEY + tokenValue, dto,
            Duration.ofSeconds(loginParameter.getTimeout()));

        // 记录登录日志 + 更新最后登录信息
        loginService.recordLoginInfo(username, Constants.LOGIN_SUCCESS, "登录成功");
        loginService.updateLastLoginInfo(userId, ip);
    }
}
4 @EventListener 自动识别：Spring 根据方法参数类型自动匹配事件——只要方法签名是 void handleXxx(XxxEvent event)，Spring 就能在 XxxEvent 发布时调用它。不需要手动 addObserver()。UserLoginSuccessListener.java:41
5 一个监听器可以做多件事：登录成功后同时做了：① 缓存在线用户到 Redis；② 记录登录日志；③ 更新最后登录信息。所有副作用被集中在一个监听器里，Controller 只发事件，不关心后续。UserLoginSuccessListener.java:42-68

2.2 服务层兼做监听器

项目中 SysOperLogServiceImpl 和 SysLoginInfoServiceImpl 本身就是监听器——它们通过 @EventListener 监听自己关心的事件并持久化到数据库。

// SysOperLogServiceImpl.java — 服务类同时是监听器
@Service
public class SysOperLogServiceImpl implements ISysOperLogService {

    @EventListener  // 监听操作日志事件
    public void handleOperLog(OperLogEvent event) {
        // 将事件转换为 SysOperLog 实体，写入数据库
        SysOperLog operLog = new SysOperLog();
        BeanUtils.copyProperties(event, operLog);
        baseMapper.insert(operLog);
    }
}
💡 为什么不让 Controller 直接调 Service？ 解耦。Controller 只发事件，不关心谁在监听、监听者有哪几个。未来你要加「操作日志同步到 ES」或「异常操作短信告警」，只需新增一个监听器，Controller 一行代码都不用改。这就是观察者模式的核心价值——发布者与订阅者彼此透明。

三、@Async — 让日志不再阻塞业务

⚠️ 关键设计决策：日志记录绝对不能阻塞登录响应。用户登录时等待 200ms 写日志是完全不可接受的。解决方案：@Async + @EnableAsync。 
// 配置异步事件处理
@Configuration
@EnableAsync
public class AsyncConfig {
    // Spring 会为 @EventListener + @Async 的方法使用独立线程池
}

// 监听器方法上标注 @Async
@Async
@EventListener
public void handleOperLog(OperLogEvent event) {
    // 这段代码在独立线程中执行，不会阻塞发布者
    baseMapper.insert(operLog);
}
6 异步事件的标准配置：@EnableAsync 开启异步支持，@Async 标注在 @EventListener 方法上。Spring 会将事件投递到线程池异步执行。发布事件的线程（即处理用户请求的线程）立即返回。AsyncConfig.java

四、事件发布端在哪？

项目中事件发布主要在 AOP 切面中：

// LogAspect.java（概念示意）—— 环绕通知发布操作日志事件
@Around("@annotation(operLog)")
public Object around(ProceedingJoinPoint point, Log operLog) {
    // ... 方法执行前记录时间
    Object result = point.proceed();  // 执行业务方法
    // ... 方法执行后构建事件并发布
    OperLogEvent event = new OperLogEvent();
    event.setOperName(...);
    event.setStatus(...);
    event.setCostTime(...);
    SpringUtils.getApplicationContext()
        .publishEvent(event);  // ← Spring 事件发布
    return result;
}

// SysLoginService.java — 登录成功后发布事件
SpringUtils.getApplicationContext()
    .publishEvent(new LoginInfoEvent(username, status, message, request, args));

五、前端观察者：Vue 3 响应式系统

// watch — 显式观察单个数据源
watch(() => userStore.token, (newToken, oldToken) => {
  if (!newToken) router.replace('/login');  // token 消失 → 跳转登录

// watchEffect — 自动追踪所有依赖

  // 内部访问了 anyRef → 当 anyRef 变化时自动重新执行
  console.log('当前用户角色:', userStore.roles);

// computed — 衍生状态（声明式观察）

六、自测

1. Spring 的 @EventListener 和传统的 Observer 接口相比，最大优势是什么？

2. 为什么日志记录的监听器必须加 @Async？

3. 一个事件发布后，有 3 个监听器都订阅了它。如果其中一个监听器抛出异常，会怎样？

📖 点击查看答案

1. B — Spring 的 @EventListener 是基于注解的方法级声明，不需要实现 Observer 接口、不需要在发布者中维护订阅者列表、不需要手动 addObserver/removeObserver。容器自动管理。
2. B — 日志记录涉及数据库 IO，通常在 50-200ms。如果同步执行，登录接口的响应时间就要加上这段时间。@Async 将日志写入放到独立线程池，用户请求立即返回。
3. B — 同步执行时，一个监听器抛异常会中断 Spring 的事件广播循环，后续监听器不会执行。使用 @Async 后，每个监听器在独立线程中执行，异常隔离，互不影响。生产环境建议对非关键监听器（如日志）使用 @Async。
