第 20 课 · 配置管理与自动装配

RuoYi-Vue-Plus 有 21 个 common 模块，每个模块都有自己的配置类、过滤器、拦截器和 Bean 注册。这些模块如何被 Spring Boot 自动发现和装配？本课拆解 @AutoConfiguration、AutoConfiguration.imports SPI 机制、@ConfigurationProperties 以及项目统一的依赖注入风格。

本课目标：阅读本课后，你能解释 AutoConfiguration.imports 文件如何替代 spring.factories，能编写符合项目规范的 @ConfigurationProperties 配置类，并理解为什么项目全面使用 @RequiredArgsConstructor 进行构造器注入。

1. @AutoConfiguration — Spring Boot 3.x/4.x 的新注解

Spring Boot 2.x 时代，自动配置类用 @Configuration 注解，通过 META-INF/spring.factories 注册。Spring Boot 3.x 引入了 @AutoConfiguration 注解和新的 SPI 文件格式：

// Spring Boot 2.x (旧)
META-INF/spring.factories:
org.springframework.boot.autoconfigure.EnableAutoConfiguration=\
com.example.MyConfig

// Spring Boot 3.x/4.x (新)
META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports:
com.example.MyConfig
关键区别：@AutoConfiguration 比 @Configuration 多了一个语义——它告诉 Spring Boot 这是一个"自动配置类"，会在用户自定义 Bean 之前加载，且支持 @AutoConfigureBefore / @AutoConfigureAfter 控制加载顺序。RuoYi-Vue-Plus 所有 common 模块的配置类一律使用 @AutoConfiguration。

配合 @ConditionalOnProperty 实现按需加载

// 仅当配置中 api-decrypt.enabled=true 时才注册加密过滤器
@AutoConfiguration
@ConditionalOnProperty(value = "api-decrypt.enabled", havingValue = "true")
@EnableConfigurationProperties(ApiDecryptProperties.class)
public class ApiDecryptAutoConfiguration {
    @Bean
    public DecryptFilter decryptFilter() {
        return new DecryptFilter();
    }
}
@EnableConfigurationProperties：在配置类上标注此注解，可以将 @ConfigurationProperties 类注册为 Spring Bean 并完成属性绑定。项目通常在 AutoConfiguration 类上使用它，而不是在每个 Properties 类上加 @Component。

4. YmlPropertySourceFactory — 突破 Spring 限制

定义文件：
RuoYi-Vue-Plus/ruoyi-common/ruoyi-common-core/src/main/java/org/dromara/common/core/factory/YmlPropertySourceFactory.java

Spring 的 @PropertySource 注解默认只支持 .properties 文件，不支持 YAML。RuoYi-Vue-Plus 自定义了 YmlPropertySourceFactory，使模块级别的 common-{module}.yml 配置文件能被加载：

// 自定义 PropertySourceFactory，解析 YAML 文件
public class YmlPropertySourceFactory implements PropertySourceFactory {
    @Override
    public PropertySource<?> createPropertySource(
            String name, EncodedResource resource) throws IOException {
        YamlPropertiesFactoryBean factory = new YamlPropertiesFactoryBean();
        factory.setResources(resource.getResource());
        Properties properties = factory.getObject();
        return new PropertiesPropertySource(
            name != null ? name : resource.getResource().getFilename(), properties);
    }
}

// 使用方式：在配置类上通过 @PropertySource 加载 YAML
@PropertySource(value = "classpath:common-redis.yml",
                factory = YmlPropertySourceFactory.class)

5. 依赖注入规范：@RequiredArgsConstructor

RuoYi-Vue-Plus 全面采用 Lombok 的 @RequiredArgsConstructor 进行构造器注入，而非字段注入的 @Autowired：

// 项目标准写法 —— 构造器注入
@RequiredArgsConstructor  // 为所有 final 字段生成构造器
@RestController
public class SysNoticeController {
    private final ISysNoticeService noticeService;     // final + 构造器注入
    private final ISysUserService userService;
}

// 反模式 —— 字段注入（项目中不这么写）
@RestController
public class BadController {
    @Autowired
    private ISysNoticeService noticeService;  // 字段注入，不推荐
}
方案优点缺点
构造器注入（项目标准）不可变（final）、便于单元测试、避免 NPE依赖多时代码稍显啰嗦（Lombok 自动生成解决）
@Autowired 字段注入代码简洁无法 immutable、不便测试、隐藏依赖、Spring 容器耦合

6. 配置分层架构

application.yml ← 公共配置（端口、数据源、日志） └── application-{profile}.yml ← 环境配置（dev/test/prod 差异化） common-{module}.yml ← 各 common 模块的默认值 ├── common-redis.yml 通过 @PropertySource + YmlPropertySourceFactory 加载 ├── common-encrypt.yml ├── common-mail.yml └── ...
覆盖优先级：application-{profile}.yml 中的配置会覆盖 common-{module}.yml 的默认值。如果你想在 dev 环境把 Redisson 从单机模式改为集群模式，只需在 application-dev.yml 中覆盖对应属性，无需修改 common-redis.yml。

自测题

问题 1：Spring Boot 3.x 中，自动配置类的注册文件是什么？

A. META-INF/spring.factories B. META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports C. META-INF/spring-autoconfigure.xml D. application.yml
查看答案

答案：B。Spring Boot 2.x 使用 spring.factories（选项 A），Spring Boot 3.x 起改用新的 imports 文件格式，一行一个全限定类名。项目中 21 个 common 模块都各自包含此文件。

问题 2：自定义 YmlPropertySourceFactory 解决了什么问题？

查看答案

Spring 原生的 @PropertySource 注解只能加载 .properties 文件，无法直接加载 YAML。RuoYi-Vue-Plus 通过自定义 PropertySourceFactory 实现，利用 YamlPropertiesFactoryBean 将 YAML 转为 Properties，使各 common 模块可以通过 @PropertySource 加载自己的 common-{module}.yml 默认配置。

问题 3：以下哪种依赖注入方式是 RuoYi-Vue-Plus 的项目标准？

A. @Autowired private ISysUserService userService; B. private final ISysUserService userService; + @RequiredArgsConstructor C. 通过 SpringUtils.getBean() 每次手动获取
查看答案

答案：B。项目统一使用 Lombok @RequiredArgsConstructor 生成构造器，所有依赖声明为 private final。字段注入（选项 A）无法保证不可变性且不便单元测试。SpringUtils（选项 C）是 Service Locator 反模式，只在少数无法使用依赖注入的场景（如静态工厂方法）中才合理使用。
