三、GoF 行为型模式 行为型

模式核心文件一句话
策略模式 IAuthStrategy.java
PasswordAuthStrategy.java (×5)
SensitiveStrategy.java
request.ts (加密策略)		 认证策略：接口上静态工厂方法按 grantType 动态路由到 5 种具体策略。脱敏策略：枚举常量携带 lambda 函数。三种实现形态：接口+Bean、枚举+Lambda、函数参数。

AbstractEncryptor.java		 AbstractOssClientImpl.initialize() 定义初始化骨架（检查→调用→标记），子类只需实现 doInitialize()。构造器中做前置校验也是模板方法的变体。

UserLoginSuccessListener.java
OperLogEvent.java		 登录成功后发布事件→监听器异步记录在线状态和信息。关键设计决策：@Async 确保日志写入不阻塞登录响应。
责任链模式 CryptoFilter.java
XssFilter.java
RepeatableFilter.java
PlusWebInvokeTimeInterceptor.java
MybatisEncryptInterceptor.java		 三层 Filter → Interceptor → MyBatis Plugin 形成立体责任链。顺序至关重要：加密过滤器必须在可重复读过滤器之前。

状态模式 SensitiveStrategy.java
UserStatus.java
TaskStatusEnum.java		 SensitiveStrategy 枚举每个常量携带脱敏函数，是枚举版的状态/策略混合体。业务状态枚举（用户状态、任务状态）控制业务流转。

四、架构设计模式 架构

模式核心文件一句话
分层架构 pom.xml (all)
ruoyi-common/*
ruoyi-modules/*		 5 层严格分层：基础设施基础 → 基础设施能力 → 横切组装 → 业务模块 → API 合约 → 入口。杜绝循环依赖，底层不知道上层存在。
依赖倒置 ruoyi-api/pom.xml
ruoyi-api/.../UserService.java

多级缓存 CaffeineCacheDecorator.java
PlusSpringCacheManager.java
TransactionAwareCacheDecorator		 Caffeine L1 (纳秒) + Redis L2 (毫秒) + 事务感知装饰器。读：L1→L2→DB；写：事务暂存→L2更新→L1失效。
事件驱动 *Event.java (4 files)
*Listener.java (5+ files)		 操作日志、登录信息、工作流事件通过 Spring Events 异步处理，核心业务不感知日志记录等副作用。
模块化架构 22 common modules
6 business modules
ruoyi-api contract layer		 22 个 common 模块分 7 大功能组，无 ruoyi-framework 巨石模块。每个模块职责单一，可独立演进。

六、工程规范 规范

规范核心文件一句话
包结构 org.dromara.{module}.{layer} 统一包结构：controller/ → service/impl/ → mapper/ → domain/{bo,vo,dto}/。命名：IServiceName + ServiceNameImpl。
统一响应 R.java
PageResult.java		 R<T> 统一 JSON 响应 {code, msg, data}，状态码 200/500/601。PageResult<T> 统一分页 {total, rows}。
异常体系 BaseException.java
ServiceException.java
GlobalExceptionHandler.java		 BaseException（i18n）+ ServiceException（final + 格式化）。GlobalExceptionHandler 统一拦截 15+ 种异常类型，返回标准 JSON。
自动装配 *Config.java (15+ files)
AutoConfiguration.imports (22 files)		 @AutoConfiguration + AutoConfiguration.imports SPI 注册。YmlPropertySourceFactory 使 @PropertySource 支持 YAML。
构造器注入 所有 @RequiredArgsConstructor 统一 @RequiredArgsConstructor 实现构造器注入，final 字段保证不可变性，杜绝字段注入的循环依赖隐患。
