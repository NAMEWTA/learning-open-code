# NOTES -- 教学偏好暂存

> 记录用户在课程中表达的教学偏好，供后续课程设计参考。

- 用户要求「读源码，真实代码引用，漂亮HTML+练习，简体中文」——课程应大量引用真实文件路径、类名、方法签名及关键代码片段，并设计可动手的提取练习（选择/填空/简答）。
- 偏好「鸟瞰图先行」风格（沿用 teach-ruoyi-sms 的全景图 + 分步拆解 + 提取练习的结构）。
- 翻译模块与 sms 模块的教学结构一致：5 节课 + 5 份速查文档。第 1 课为全景鸟瞰，第 5 课为实战扩展与性能优化。
- 模块有 11 个 Java 文件，比 sms 模块（3 个类）复杂度高——需注意每节课信息密度适中，不要一次性塞太多实现细节。
