速查 01 · ruoyi-common-translation 模块全景

配套课程：lessons/0001.html · 使命：读懂 ruoyi-common-translation 翻译模块

一句话定位

ruoyi-common-translation 是「注解驱动的字段值后处理器」——在 JSON 序列化时自动把 ID/编码替换成人类可读的名称，不查数据库，通过 SPI 接口调用远程服务。

模块结构（11 文件 = 4 层）

层文件作用
声明层@Translation + @TranslationType标注「哪个字段要翻译 / 翻译类型」
契约层TranslationInterface<T>SPI 接口：单值 + 批量翻译
引擎层TranslationJsonFieldProcessorcollect→prepare→process 三阶段调度
实现层5 个 *TranslationImpl对接 UserService/DeptService/DictService/OssService
装配层TranslationConfig@AutoConfiguration 收集实现 + 注册 processor
常量TransConstant5 个内置类型常量

核心工作流

Controller 返回 VO → Jackson 序列化管线拦截 → collect: 收集所有 @Translation 字段的原始值，按(type,other)分组 → prepare: 每组调一次 translationBatch() 批量查询 → process: 将翻译结果写回字段值 → 前端收到已翻译的 JSON

内置翻译类型

类型常量翻译依赖服务
USER_ID_TO_NAME用户ID → 用户名UserService
USER_ID_TO_NICKNAME用户ID → 昵称UserService
DEPT_ID_TO_NAME部门ID → 部门名DeptService
DICT_TYPE_TO_LABEL字典值 → 标签DictService
OSS_ID_TO_URLOSS ID → URLOssService

记忆锚点

  • 翻译发生在 JSON 序列化阶段，不是 SQL 查询或前端渲染阶段。
  • 翻译模块不查数据库——它通过 ruoyi-api 的远程服务接口获取翻译数据。
  • 核心价值：批量收集 + 批量翻译，避免 N+1 查询。
  • 开发者只需在 VO 字段上加 @Translation(type="...") 注解即可，零代码侵入。
