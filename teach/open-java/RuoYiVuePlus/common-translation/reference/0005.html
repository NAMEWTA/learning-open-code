速查 05 · 实战自定义翻译实现与性能优化

配套课程：lessons/0005.html

自定义翻译 4 步接入

步骤操作关键点
1. 定义常量在 TransConstant 或项目常量类中添加 type 字符串避免硬编码拼写错误
2. 实现接口implements TranslationInterface<String>
+ @TranslationType(type="xxx")
+ @Component（或自动扫描）		必须覆盖 translationBatch()
3. 标注字段VO 字段上 @Translation(type="xxx", mapper="源字段")type 与实现类的 @TranslationType.type 一致
4. 验证启动项目，调用含翻译字段的 API不需要修改任何配置类

性能优化要点

要点说明
覆盖 translationBatch避免 default 实现导致的 N+1 退化
空值快速返回翻译键为 null 时跳过，不调用远程服务
异常不影响主流程翻译失败返回原值，不中断整个请求
利用远程服务缓存不要在翻译实现中重复建缓存
只翻译必要字段同类型翻译键自动去重，不需手动合并

常见陷阱

陷阱1: 不覆盖 translationBatch() → N+1退化 陷阱2: translationBatch()内逐条调远程API → 假批量 陷阱3: 翻译实现直接查数据库 → 破坏分层 陷阱4: 滥用翻译注解 → 数据量爆炸

扩展方向

  • 枚举值翻译：枚举名 → 中文标签
  • 国际化：字段 key → 对应语言文本（other 传 locale）
  • 脱敏：手机号/身份证 → 掩码显示
  • 外部API富化：IP → 地理位置、经纬度 → 地址

接入检查清单

  • ✓ 实现类有 @TranslationType(type="xxx")
  • ✓ 实现类是 Spring Bean
  • ✓ 覆盖了 translationBatch()
  • ✓ VO 上 @Translation(type) 与实现类一致
  • ✓ mapper 字段存在且有 getter
