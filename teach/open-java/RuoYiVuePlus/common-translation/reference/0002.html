速查 02 · @Translation 与 @TranslationType 注解详解

配套课程：lessons/0002.html

@Translation 三参数

参数必填作用解析方法
type是翻译类型标识，与 @TranslationType.type 匹配getTranslation(type)
mapper否翻译源字段名（不填=当前字段值）resolveSourceValue() → ReflectUtils.invokeGetter(owner, mapper)
other否额外参数（字典类型、业务分类等）作为 translation(key, other) 的 other 参数

翻译键来源解析逻辑

resolveSourceValue(fieldContext, translation): if (mapper 非空) → ReflectUtils.invokeGetter(owner, mapper) // 从 owner 的 mapper 字段取值 else → fieldContext.value() // 用当前字段的值

典型使用模式

模式示例翻译键来源
直译当前字段@Translation(type="user_id_to_name")
private String createBy;		createBy 字段的值(如 "1")
mapper 映射@Translation(type="user_id_to_name", mapper="createBy")
private String createByName;		owner.createBy 的值
字典带 other@Translation(type="dict_type_to_label", other="sys_user_status")
private String status;		status 字段的值, other="sys_user_status"

@TranslationType 与映射建立

TranslationConfig.@Bean → 收集 List<TranslationInterface>（Spring 注入所有实现） → TranslationJsonFieldProcessor 构造器遍历每个实现 → 读取 @TranslationType 注解的 type 值 → 建立 Map: type字符串 → 实现实例 → 存入 Collections.unmodifiableMap
  • @TranslationType 只有 type() 一个参数，标注在实现类上。
  • @Inherited 元注解：子类可继承父类的 TranslationType 声明。
  • 映射在 Spring 启动时建立，运行时直接查 Map。
