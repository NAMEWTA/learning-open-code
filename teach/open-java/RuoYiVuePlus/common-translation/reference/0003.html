TranslationInterface 方法一览

方法类型说明
translation(key, other)抽象（必须实现）单值翻译，返回翻译结果
translationBatch(keys, other)默认（建议覆盖）批量翻译。默认逐条调用 translation() + warn 日志
collectLongIds(keys)默认（工具）将 Set<Object> → Set<Long>，支持逗号分隔字符串拆分
parseLongIds(ids)默认（工具）逗号分隔字符串 → List<Long>
joinMappedValues(ids, mapper)默认（工具）"1,2,3" + {1→A,2→B,3→C} → "A,B,C"

三阶段流程速查

阶段调用频率核心逻辑数据写入
supports每字段检查是否有 @Translation 注解—
collect每字段解析翻译键 → 按(type,other)分组去重收集context.ATTR_BATCHES
prepare每次响应1次遍历 batches → 每组调 translationBatch()context.ATTR_RESULTS
process每字段查 results → 命中返回翻译值，未命中退化单值查询返回替换值

关键数据结构

TranslationBatchKey(type, other) → batches: Map<BatchKey, Set<Object>> (type+other → 翻译键集合) → results: Map<BatchKey, Map<Object,Object>> (type+other → (键→翻译值)) translationMap (构造时建立，不可变): Map<String, TranslationInterface<?>> (type字符串 → 实现实例)

记忆锚点

  • 三阶段 = 收(collect) → 查(prepare) → 换(process)
  • prepare 才是真正查数据的地方，collect 只是收集键，process 只是取值替换
  • TranslationBatchKey 的 (type, other) 双键分组保证不同 other 的翻译互不干扰
  • 不覆盖 translationBatch → warn 日志 + 退化为 N+1 查询
