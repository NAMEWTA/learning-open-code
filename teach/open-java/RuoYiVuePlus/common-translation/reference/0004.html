标准实现模式

@AllArgsConstructor @TranslationType(type = TransConstant.XXX) public class XxxTranslationImpl implements TranslationInterface<String> { private final RemoteService remoteService; // translation(): 处理单值(Long/String多值) public String translation(Object key, String other) { ... } // translationBatch(): 必须覆盖 // A. collectLongIds(keys) → Set<Long> // B. remoteService.batchQuery(ids) // C. 遍历keys → buildValue(key, resultMap) public Map<Object,String> translationBatch(Set<Object>keys, String other) { ... } // buildValue(): 多值逗号分隔支持 private String buildValue(Object src, Map<Long,String> m) { if (src instanceof String ids) return joinMappedValues(ids, m::get); // "1,2,3" → "A,B,C" return m.get(Convert.toLong(src)); } }

三类翻译的差异

类型代表特征
用户系列UserName, Nickname有单值+批量两套API，buildValue 标准模式
实体系列DeptName, OssUrl单值和多值调同一 API（远程自己处理逗号分隔）
字典系列DictType依赖 other 参数，全量加载后内存匹配

记忆锚点

  • 所有实现都覆盖了 translationBatch，不存在 N+1 退化
  • 所有实现都支持逗号分隔多值输入（"1,2,3" → "张三,李四,王五"）
  • 只有字典翻译需要 other 参数
  • collectLongIds + joinMappedValues + buildValue 是三个标准件
