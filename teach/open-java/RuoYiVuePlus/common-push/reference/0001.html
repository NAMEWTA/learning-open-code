配套课程：lessons/0001.html · 使命：读懂 ruoyi-common-push 消息推送模块

一句话定位

ruoyi-common-push 通过 SSE + WebSocket 双通道互斥选一提供消息推送，用 @ConditionalOnMessageTransport 条件装配按配置激活通道，用 Redis Pub/Sub 实现集群跨节点投递。

核心设计

  • 双通道互斥：运行时只有一个 PushSessionManager Bean，由 message.transport 配置决定。
  • 接口统一：PushSessionManager 定义 subscribe / send / publish 方法，两种实现接口相同。
  • 存储对称：Map<userId, Map<token, Connection>>，同 token 新连接顶替旧连接（发 KICKED 通知）。
  • 跨节点投递：publish → Redis global:message → 各节点 MessageTopicListener → 本节点 sendMessage。

记忆锚点

  • 改一行配置即可切换通道：message.transport: sse|websocket。
  • SSE = 单向（服务端→客户端）、HTTP 长连接、EventSource 自动重连。
  • WebSocket = 双向、独立 ws:// 协议、握手拦截器鉴权、无自动重连。
  • 两个通道的 SessionManager 内部结构完全对称。
  • 跨节点靠 Redis Pub/Sub，不是靠 Nginx/网关广播。
