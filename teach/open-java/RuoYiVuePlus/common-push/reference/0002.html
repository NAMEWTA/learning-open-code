connect() 核心流程

① prepareSseResponse() 设 4 个响应头（Content-Type=text/event-stream, UTF-8, Cache-Control=no-cache, X-Accel-Buffering=no） ② StpUtil.getTokenValue() + LoginHelper.getUserId() 取用户信息 ③ sessionManager.connect(userId, token) ├─ computeIfAbsent(userId) → ConcurrentHashMap ├─ remove(token) 顶替旧连接 → sendKickedMessage ├─ new SseEmitter(sseTimeout=24h) ├─ 注册 onCompletion/onTimeout/onError → 清理 └─ emitter.send(comment("connected"))

心跳：sseMonitor()

  • 每 60s（heartbeatInterval）执行一次
  • 向每个 emitter 发送 comment("heartbeat")
  • 发送失败 → complete() + remove
  • 用 comment 而非 data → 前端不触发 onmessage

消息发送

方法范围实现
sendMessage(userId, message)指定用户遍历 userId 的 map，逐一 emitter.send(event.name("message").data(...))
sendMessage(message)所有本地收集所有 userId，ThreadUtils.virtualInvokeAll 并发发送
publishMessage(dto)集群RedisUtils.publish("global:message", dto)

存储结构

Map<Long, Map<String, SseEmitter>> USER_TOKEN_EMITTERS userId → {token → emitter} 空 emitterMap → 移除 userId（防止内存泄漏）
