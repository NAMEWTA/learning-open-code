三层装配结构

层配置类条件作用
1MessageAutoConfigurationmessage.enabled=true总开关 + 激活 MessageProperties 绑定
2MessageSseConfiguration@ConditionalOnMessageTransport("sse")注册 SSE 三件套（Manager + Listener + Controller）
2MessageWebSocketConfiguration@ConditionalOnMessageTransport("websocket")注册 WS 四件套（Manager + Handler + Interceptor + Listener）

MessageTransportCondition.matches() 逻辑

① metadata.getAnnotationAttributes() → 取出 value=("sse"|"websocket") ② context.getEnvironment().getProperty("message.enabled") → 默认 true ③ context.getEnvironment().getProperty("message.transport") → 默认 "sse" ④ return enabled && expected.equalsIgnoreCase(transport)

属性默认值说明

AutoConfiguration.imports 清单

org.dromara.common.push.config.MessageAutoConfiguration org.dromara.common.push.config.MessageSseConfiguration org.dromara.common.push.config.MessageWebSocketConfiguration 三个都列出，但 @ConditionalOnMessageTransport 裁决谁实际生效

配置切换命令

message.transport: sse → SSE 配置类生效，WS 跳过 message.transport: websocket → WS 配置类生效，SSE 跳过 message.enabled: false → 全部静默关闭

→ 相关术语见 GLOSSARY.md：@ConditionalOnMessageTransport、MessageTransportCondition、MessageAutoConfiguration、MessageProperties
