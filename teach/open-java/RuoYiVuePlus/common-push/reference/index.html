ruoyi-common-push 速查表

Maven: org.dromara:ruoyi-common-push | ruoyi-common/ruoyi-common-push/src/main/java/org/dromara/common/push/

定位

SSE / WebSocket 双通道消息推送模块（17 个 Java 文件）。通过 @ConditionalOnMessageTransport 条件装配按配置二选一激活通道，用 Redis Pub/Sub 实现集群跨节点消息投递，对外暴露统一的 PushSessionManager 接口和 PushHelper 工具类。

术语速查

术语解释
SSEServer-Sent Events，HTTP 长连接单向推送，自动重连，仅文本，Spring 用 SseEmitter 实现
WebSocket全双工双向通信协议（RFC 6455），HTTP Upgrade 握手，支持文本和二进制，无自动重连
条件装配@ConditionalOnMessageTransport + MessageTransportCondition，按配置二选一激活通道
Redis Pub/Sub 跨节点publishMessage() → RedisUtils.publish(TOPIC) → 各节点 MessageTopicListener 收到 → 本地投递
PushSessionManager统一接口，SSE 和 WebSocket 管理器均实现，使 PushHelper 可无感切换通道
顶替旧连接同用户+同 token 新连接通过 KICKED 消息驱逐旧连接（SSE 靠 remove、WS 靠 close）
心跳保活SseEmitterSessionManager 定时 sseMonitor() 发送心跳，清理失效连接

常用代码

业务层推送消息：

// 向指定用户发送消息
PushHelper.sendMessage(1L, "您有新的审批通知");

// 向多个用户发送
PushHelper.sendMessage(Arrays.asList(1L, 2L, 3L), "系统维护通知");

// 广播给所有在线用户
PushHelper.sendMessage("系统将于 22:00 进行维护");

// 跨节点推送（集群环境）
PushHelper.publishMessage(Arrays.asList(1L, 2L), "订单已发货");

前端 SSE 连接：

// 前端建立 SSE 连接
const eventSource = new EventSource('/message/sse/connect?token=xxx');
eventSource.onmessage = (event) => {
    console.log('收到消息:', event.data);
};

application.yml 配置：

message:
  enabled: true
  transport: sse           # sse 或 websocket，二选一
  path: /message
  allowed-origins: "*"
  sse-timeout: 86400000    # SSE 超时 ms，默认 24h
  heartbeat-interval: 60000 # 心跳间隔 ms
