文件作用

WebSocket 生命周期

阶段方法核心逻辑
握手前beforeHandshake()LoginHelper.getLoginUser() + StpUtil.getTokenValue() → attributes
连接建立afterConnectionEstablished()校验用户信息 → ConcurrentWebSocketSessionDecorator 包装 → webSocketSessionManager.connect()
消息接收handleTextMessage()ping→pong / 其他→包装 PushDTO 并 publishMessage
连接关闭afterConnectionClosed()webSocketSessionManager.disconnect()
传输异常handleTransportError()log.error → disconnect 或 closeSession

PlusWebSocketInterceptor.beforeHandshake()

LoginUser loginUser = LoginHelper.getLoginUser(); String tokenValue = StpUtil.getTokenValue(); attributes.put(LOGIN_USER_KEY, loginUser); attributes.put(LOGIN_TOKEN_KEY, tokenValue); return true; // 始终允许握手

handleTextMessage 消息分流

消息内容 == "ping" → webSocketSessionManager.sendMessage(session, "pong") 消息内容 != "ping" → 构建 PushDTO(CUSTOM, CLIENT, payload, null) → webSocketSessionManager.publishMessage(dto)

并发安全

ConcurrentWebSocketSessionDecorator(session, sendTimeLimit=10s, bufferSizeLimit=64KB) 保证多线程同时写 session 时线程安全

会话监控

sessionMonitor() 每 60s 执行 检查 session.isOpen() → 已关闭则 remove （被动清理，不主动发心跳——协议层 Ping/Pong 由容器维护）
