09 · ruoyi-common-excel：动态导入导出引擎

核心问题：如何在 10 行代码内实现带数据校验、字典翻译、下拉选项、批注、单元格合并的 Excel 导入导出？

模块定位：21 个 Java 文件。基于 Apache Fesod Sheet（EasyExcel 替代方案，Apache 基金会项目）。提供统一的 Builder API，链式调用一行代码完成导出/导入/模板填充。

1. ExcelBuilder — 统一入口

四种构造方式

// ① 普通导出
ExcelBuilder.of(dataList, EntityClass.class)
    .sheetName("用户列表")
    .toResponse(response);

// ② 自定义写出（不预设数据，在回调中分批写）
ExcelBuilder.writer(EntityClass.class)
    .toStream(os, wrapper -> {
        wrapper.write(dataPage1);
        wrapper.write(dataPage2);
    });

// ③ 模板导出
ExcelBuilder.template("templates/user_template.xlsx")
    .data(dataList)
    .toResponse(response);

// ④ 导入读取
ExcelResult<EntityClass> result = ExcelBuilder.read(inputStream, EntityClass.class)
    .validate(true).failFast(true)       // 启用校验 + 快速失败（默认已开启）
    .headRowNumber(1)
    .doRead();                   // 返回含成功列表+错误列表的结果

2. 大数据分批 ZIP 导出

// 每 999 条一个 Excel 文件，打包成 ZIP 下载
ExcelBuilder.of(largeData, Entity.class)
    .sheetName("导出数据")
    .zip(999)  // 每个Excel最多999条，超量自动分文件
    .toResponse(response);

// 内部实现：ListUtil.partition(data, pageSize) → 每个分页独立 .xlsx → ZipOutputStream
// 只有 1 页时自动退化为普通导出

3. 三种模板模式

模式模板占位符适用场景
LIST{.属性名}单表多数据（如用户列表）
MULTI_LIST{key.属性名}多表多数据（用户+日志）
MULTI_SHEET每 sheet 独立多 sheet 报表

4. 导入校验与错误处理

// DefaultExcelListener — 默认导入监听器
//   - 自动启用 Bean Validation
//   - 快速失败：failFast=true（第一条错误即停止）
//   - 错误格式："第{行}行-第{列}列-表头{header}: 解析异常"
//   - DefaultExcelResult.getAnalysis() → "共X条，成功导入X条，错误X条"

5. 自动注册的组件

toResponse() 和 toStream() 自动注册以下组件，无需手动配置：

  • ExcelBigNumberConvert — Long 超 15 位转字符串（防 Excel 精度丢失）
  • DataWriteHandler — 批注 + 必填标记（红色表头）
  • LongestMatchColumnWidthStyleStrategy — 自适应列宽
  • ExcelDownHandler — 下拉选项生成

6. 链式配置总览

ExcelBuilder.of(data, Entity.class)
    .sheetName("导出数据")        // Sheet 名称
    .merge()                     // 启用单元格合并
    .options()                   // 启用下拉选项
    .password("123456")          // 工作簿密码保护
    .columnWidth(20)             // 固定列宽（不设时自动适配）
    .rowHeight(25)               // 行高
    .includeFields("id", "name") // 仅包含指定列
    .excludeFields("password")   // 排除指定列
    .toResponse(response);       // 输出到 HTTP 响应

前端下载方式：后端 toResponse(response) 设置 Content-Disposition 头，前端调用 request.ts 的通用 download() 方法（POST + responseType: 'blob'）。

7. 验证要点
