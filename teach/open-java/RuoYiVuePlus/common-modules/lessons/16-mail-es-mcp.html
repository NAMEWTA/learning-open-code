16 · ruoyi-common-mail + elasticsearch + mcp：邮件、搜索与AI协议

核心问题：邮件发送、ES 搜索、MCP 协议——这三个"小而精"的模块如何为项目提供关键的外部集成能力？

Part A: ruoyi-common-mail — 邮件发送

模块定位：3 个 Java 文件。基于 Jakarta Mail + Hutool 封装。Builder 模式，链式调用。支持 HTML 模板邮件、内联图片、附件、多收件人。

MailBuilder 流式 API

// 纯文本邮件
MailBuilder.of()
    .from("noreply@example.com")
    .to("user1@example.com,user2@example.com")  // 逗号分隔多收件人
    .cc("manager@example.com")                   // 抄送
    .subject("系统通知")
    .text("您的订单已发货")
    .send();

// HTML 模板邮件 + 内联图片
MailBuilder.of(mailAccount)
    .to("user@example.com")
    .subject("欢迎注册")
    .html("<h1>欢迎！</h1><img src='cid:logo'>")
    .image("logo", logoInputStream)  // 内联图片（cid 引用）
    .files(new File("welcome.pdf"))   // 附件
    .send();

配置属性

mail:
  enabled: true
  host: smtp.example.com
  port: 465
  auth: true
  user: noreply@example.com
  pass: yourPassword
  from: RuoYi <noreply@example.com>
  starttlsEnable: true
  sslEnable: true

条件装配：@ConditionalOnProperty(value = "mail.enabled", havingValue = "true")，未配置邮件服务器时自动跳过。

Part B: ruoyi-common-elasticsearch — ES 搜索引擎

模块定位：仅 2 个 Java 文件。基于 Easy-Es（国产 ES ORM，API 风格类似 MyBatis-Plus）。极简设计——所有 ES 操作由 Easy-Es 框架处理。

两个配置类

// 1. EasyEsConfiguration — 扫描 ES Mapper
@EsMapperScan("org.dromara.**.esmapper")  // 扫描所有模块的 ES Mapper

// 2. ActuatorEnvironmentPostProcessor — 健康检查联动
// 读取 easy-es.enable → 同步设置 management.health.elasticsearch.enabled

使用方式

// 类似 MyBatis-Plus 的 Lambda 查询
esMapper.lambda()
    .eq(Document::getTitle, "搜索关键词")
    .page(page);

Part C: ruoyi-common-mcp — MCP 协议集成

模块定位：4 个 Java 文件。基于 Spring AI MCP（Model Context Protocol）。封装了 MCP Client 的工具调用和资源读取，为 AI 模块提供标准化的外部工具集成能力。

核心组件 McpClientTemplate

// 查询所有 Server 的工具列表
Map<String, List<String>> tools = mcpClientTemplate.listTools();
// {"server1": ["toolA", "toolB"], "server2": ["toolC"]}

// 调用所有 Server 上的同名工具
Map<String, McpToolCallResult> results = mcpClientTemplate.callTool("weather", arguments);

// 调用指定 Server 上的工具
McpToolCallResult result = mcpClientTemplate.callTool("server1", "weather", args);

// 读取资源
McpResourceReadResult resource = mcpClientTemplate.readResource("file://docs/readme.md");

MCP 协议概念

概念说明
ToolAI 可调用的工具（如天气查询、数据库查询）
ResourceAI 可读取的资源（如文件、配置）
Prompt可复用的提示模板

Spring AI 负责 MCP Client 的连接管理和初始化（Connection 层），本模块只做应用层封装，避免业务代码直接操作 MCP 协议细节。

验证要点

  1. 配置 mail 参数后，写一个测试发送邮件的方法验证 MailBuilder API
  2. 开启 easy-es.enable，创建一个 Document 实体类并实现 ES Mapper
  3. 配置一个 MCP Server 连接，通过 McpClientTemplate.listTools() 查看可用工具
