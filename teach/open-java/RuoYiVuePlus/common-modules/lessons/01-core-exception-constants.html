01 · ruoyi-common-core 核心基石（上）：异常体系与常量设计

核心问题：一个拥有 24 个模块的大型项目中，如何保证所有模块的错误处理一致、常量不重复、状态码统一？答案就在 ruoyi-common-core —— 整个项目的「宪法」模块。

模块定位：ruoyi-common-core 是依赖树的根。它不依赖任何其他 ruoyi 模块，但被所有 23 个模块传递依赖。65 个 Java 文件，23 个包。Maven 坐标：org.dromara:ruoyi-common-core。

1. 异常体系：三层设计

RuntimeException
├── BaseException (i18n 异常基类)
│   ├── FileException (文件异常)
│   │   ├── FileNameLengthLimitExceededException
│   │   └── FileSizeLimitExceededException
│   └── UserException (用户异常)
│       ├── CaptchaException (验证码错误)
│       └── CaptchaExpireException (验证码过期)
├── ServiceException (通用业务异常，项目中 42 个文件抛出)
└── SseException (SSE 推送专用异常)

BaseException — 支持国际化的异常

public class BaseException extends RuntimeException {
    private String module;        // 模块名
    private String code;          // i18n 消息键
    private Object[] args;        // 消息参数
    private String defaultMessage; // 回退消息

    @Override
    public String getMessage() {
        return MessageUtils.message(code, args); // 优先查 i18n
        // 找不到时回退到 defaultMessage
    }
}

ServiceException — 最常用的业务异常

// 链式调用，支持占位符
throw new ServiceException("用户【{}】不存在", username);
throw new ServiceException().setMessage("操作失败").setDetailMessage("库存不足");

被 42 个文件引用，是项目中最常用的运行时异常。在 GlobalExceptionHandler（common-web 模块）中被统一捕获并返回前端。

关键设计：BaseException 和 ServiceException 是两条独立分支，不互相继承。BaseException 走 i18n 翻译，ServiceException 直接返回消息字符串。选择哪一个取决于是否需要国际化支持。

对应前端：request.ts 响应拦截器中 code === 500 分支通过 ElMessage.error(msg) 展示给用户。

CacheNames 的创新设计

// 缓存名内联配置：name#TTL#maxIdleTime#maxSize
String DEMO_CACHE = "demo:cache#60s#10m#20";  // TTL=60s，空闲最大10min，最多20条
String SYS_USER_NAME = "sys_user_name#30d";    // TTL=30天
// 注意：v6 中 SYS_CONFIG 为无后缀的基础缓存名，DEMO_CACHE 是带策略的示例

3. 领域模型：R<T> 统一响应

public class R<T> {
    int code;     // 200/500/601
    String msg;
    T data;
}

// 使用示例
R.ok(data);           // code=200
R.fail("操作失败");    // code=500
R.warn("请先完善资料"); // code=601

整个项目 55 个文件引用此类。前端 plus-ui-vue/src/utils/request.ts 响应拦截器根据 code 值决定 Promise.resolve 还是 reject。

枚举值用途

UserStatusOK("0") / DISABLE("1") / DELETED("2")用户状态流转

BusinessStatusEnumCANCEL→DRAFT→WAITING→FINISH→INVALID→BACK→TERMINATION（7 个状态）工作流状态机，被 workflow 模块 25+ 文件引用

5. 被哪些模块调用

所有 23 个 common 子模块、ruoyi-admin、ruoyi-modules（system/gen/job/demo/workflow）均传递依赖 common-core。核心引用关系：

  • StringUtils → 109 个文件使用（所有模块）
  • R<T> → 55 个文件使用（所有 Controller 层）
  • SpringUtils → 43 个文件使用（跨模块 Bean 查找）
  • ServiceException → 42 个文件使用（system、workflow、admin、common-web、common-encrypt）
