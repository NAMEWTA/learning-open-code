14 · ruoyi-common-social：第三方登录 OAuth2.0 集成

核心问题：如何在一个系统中同时支持微信、QQ、GitHub、企业微信、钉钉、支付宝等 26 种第三方登录？

模块定位：13 个 Java 文件。基于 JustAuth（me.zhyd.oauth）开源 SDK。支持 26 个平台，包括 3 种私有化部署的自定义扩展（Gitea/MaxKey/TopIAM）。

1. 支持的 26 个平台一览

类别平台
社交QQ、微信开放平台、微信公众号、微博、抖音、人人
企业协作钉钉 V2、企业微信（扫码登录）
代码托管GitHub、Gitee、GitLab、Coding
私有 GitGitea（自定义扩展）
开放平台百度、淘宝、支付宝、华为 V3、阿里云
职场领英（LinkedIn）
技术社区开源中国、Stack Overflow
企业 IDaaSMicrosoft (Entra ID)、MaxKey（开源IAM）、TopIAM（开源身份治理）

2. OAuth2.0 授权码模式流程

浏览器                                        后端                          第三方平台
  │                                            │                              │
  │  1. GET /auth/social?source=github         │                              │
  │──────────────────────────────────────────→│                              │
  │                                            │  2. 生成 authorizeUrl         │
  │  3. 302 重定向到 GitHub 授权页              │─────────────────────────────→│
  │←──────────────────────────────────────────│                              │
  │                                                         用户授权           │
  │  4. 回调 redirectUri?code=xxx&state=xxx  │                              │
  │──────────────────────────────────────────→│                              │
  │                                            │  5. 用 code 换 access_token   │
  │                                            │─────────────────────────────→│
  │                                            │  6. 用 access_token 拿用户信息 │
  │                                            │─────────────────────────────→│
  │                                            │  7. 根据 source+uuid 查绑定     │
  │                                            │  8. LoginHelper.login() 签发Token│
  │  9. 返回 LoginVo (含 Admin-Token)          │                              │
  │←──────────────────────────────────────────│                              │

3. State 防 CSRF

// AuthRedisStateCache — State 存储在 Redis
public void cache(String key, String value) {
    RedisUtils.setCacheObject(
        GlobalConstants.SOCIAL_AUTH_CODE_KEY + key, value,
        Duration.ofMinutes(3)  // 3 分钟过期
    );
}
// 回调时比对 state 参数 → 一致才继续 → 防止 CSRF 攻击

4. 账号绑定机制

// 首次登录 → "你还没有绑定第三方账号" → 引导绑定
// 已绑定   → authId = source + uuid (如 "GITHUB12345678")
//          → 查 sys_social 表找到系统用户 ID
//          → 通过 Sa-Token 签发 Token

// 表结构 sys_social:
//   id, user_id (系统用户ID), auth_id (第三方唯一标识),
//   source (平台, 如github), access_token, expire_in, ...

5. 三种私有化扩展模式

Gitea（自建 Git）：通过 serverUrl 配置私有部署地址，实现 authorize/accessToken/userInfo 三个端点。

# 配置示例
justauth:
  type:
    gitea:
      client-id: xxx
      client-secret: xxx
      redirect-uri: http://localhost:8080/auth/callback
      server-url: https://gitea.example.com

MaxKey / TopIAM（企业 IDaaS）：同理适配私有部署的身份认证平台，展示了如何处理非标准 OAuth2.0 差异（如 scope 参数的空格分隔、Basic Auth header 等）。

6. 前端对应

登录页 plus-ui-vue/src/views/login.vue 中的第三方登录图标区，通过 api/login.ts 中 grantType: 'social' 参数调用 /auth/social/callback 接口实现社交登录。

7. 验证要点

  1. 在 application.yml 中配置 GitHub 的 client-id 和 client-secret
  2. 访问 /auth/social?source=github，观察 OAuth2.0 完整跳转流程
  3. 查看 sys_social 表结构，理解 authId = source+uuid 的绑定设计
