15 · ruoyi-common-log + doc：操作日志与API文档

核心问题：如何用一行注解记录任何 Controller 方法的操作日志？如何自动生成带有权限说明的 OpenAPI 文档？

Part A: ruoyi-common-log — 操作日志

@Log 注解

@Log(title = "用户管理", businessType = BusinessType.DELETE)
@DeleteMapping("/{userIds}")
public R<Void> remove(@PathVariable Long[] userIds) {
    // 自动记录：谁、什么时候、删除了什么、耗时多久
    return toAjax(userService.deleteUserByIds(userIds));
}
属性说明
title模块名称
businessTypeOTHER/INSERT/UPDATE/DELETE/GRANT/EXPORT/IMPORT/FORCE/GENCODE/CLEAN
operatorTypeOTHER/MANAGE/MOBILE
isSaveRequestData是否保存请求参数（默认 true）
isSaveResponseData是否保存响应参数（默认 true）
excludeParamNames排除的请求参数字段名

LogAspect — AOP 切面实现

@Around("@annotation(controllerLog)")
public Object around(ProceedingJoinPoint point, Log controllerLog) {
    StopWatch stopWatch = new StopWatch();  // 计时开始
    // 执行目标方法
    Object result = point.proceed();
    // 无论成功异常都执行 handleLog()
    handleLog(point, controllerLog, result, stopWatch, exception);
}

// handleLog 组装 OperLogEvent（24 个字段）:
//   operId, title, businessType, businessTypes, method, requestMethod, operatorType,
//   operName, userId, deptId, deptName, clientKey,
//   deviceType, browser, os, operUrl, operIp, operLocation,
//   operParam, jsonResult, status, errorMsg, operTime, costTime

// 通过 SpringUtils.context().publishEvent(operLog) 发布事件
// 日志记录异常不影响业务执行（try-catch 包裹）

前端对应：plus-ui-vue/src/views/monitor/operlog/ 操作日志页面，调用 api/monitor/operlog.ts 的 list/delete/clean 等接口。

LoginInfoEvent — 登录日志

// 在登录 Service 中使用无参构造 + setter 模式发布
LoginInfoEvent event = new LoginInfoEvent();
event.setUsername(username);
event.setStatus(status);
event.setMessage(message);
// ... 设置其他字段（request、args 等）
SpringUtils.context().publishEvent(event);
// 异步监听器记录到数据库 sys_logininfor 表

Part B: ruoyi-common-doc — API 文档生成

技术栈：SpringDoc (OpenAPI 3.0) + Knife4j。8 个 Java 文件。通过自定义的 ClassTagOperationCustomizer 和 JavadocOperationCustomizer 增强文档质量。

核心特性

特性实现
类级标签增强从 Controller 类 JavaDoc 第一行提取标签名，替代 SpringDoc 的类名分裂标签
方法 JavaDoc 增强从方法 JavaDoc 提取摘要（首句）和完整描述注入 Operation
权限说明自动生成解析 @SaCheckRole、@SaCheckPermission、@SaIgnore 注解，生成 Markdown 权限说明
路径前缀处理自动为所有 API 路径添加 Servlet Context Path 前缀

SaTokenAnnotationMetadataJavadocResolver

// 自动解析 Controller 方法上的 Sa-Token 注解
@SaCheckPermission("system:user:list")
@SaCheckRole("admin")
public R<List<SysUserVo>> list(...) { ... }

// 在 Swagger 文档中自动生成：
// > **权限要求**
// > - 权限码: `system:user:list`
// > - 角色: `admin`

配置方式

# 通过 springdoc.api-docs.enabled 控制开关
springdoc:
  api-docs:
    enabled: true
  swagger-ui:
    path: /swagger-ui.html
  info:
    title: RuoYi-Vue-Plus API
    version: 6.0.0

验证要点

  1. 在 Controller 方法上加 @Log 注解，执行操作后在「操作日志」页面查看记录
  2. 启动项目后访问 /swagger-ui.html，观察 API 文档中的权限说明
  3. 查看 OperLogEvent 的 22 个字段中哪些是从当前请求/登录用户自动获取的
