08 · ruoyi-common-encrypt：API接口加解密与数据库字段加密

核心问题：敏感数据（身份证号、银行卡、手机号）在传输和存储时如何加密？前端到后端的安全通道如何建立？

模块定位：25 个 Java 文件。实现两大加密场景：API 接口加解密（RSA+AES 混合，保护传输层）和数据库字段加解密（MyBatis 拦截器，保护持久层）。@EncryptField 注解支持的算法：BASE64、AES、RSA、SM2、SM4（5 种）；EncryptUtils 工具类额外支持 MD5、SHA256、SM3 哈希算法。

1. 两大加密场景对比

2. API 接口加解密流程

前端请求加密:
  1. 前端随机生成 AES 密钥（24字节）
  2. 用 RSA 公钥加密 AES 密钥 → 放入 encrypt-key 请求头
  3. 用 AES 密钥加密请求体 JSON → 替换 config.data
  4. 发送请求

后端解密 (CryptoFilter):
  1. 从 Header 读取 encrypt-key → RSA 私钥解密 → 得到 AES 密钥
  2. 读取请求体 → AES 解密 → 得到原始 JSON
  3. 包装为 DecryptRequestBodyWrapper → 传递给 Controller

后端响应加密:
  1. EncryptResponseBodyWrapper 捕获 Controller 输出
  2. 随机生成新的 AES 密钥 → 加密响应体
  3. RSA 公钥加密该 AES 密钥 → 写入 encrypt-key 响应头
  4. 前端从响应头取 AES 密钥 → 解密响应体

前端对应代码（plus-ui-vue/src/utils/request.ts 请求拦截器）：

// 当 VITE_APP_ENCRYPT === 'true' 且请求 isEncrypt === 'true' 时
const aesKey = generateAesKey();  // 随机 24 字节
config.headers['encrypt-key'] = rsaEncrypt(aesKey, publicKey);
config.data = aesEncrypt(JSON.stringify(config.data), aesKey);

3. 数据库字段加解密

// 在实体类字段上标注
public class SysOss {
    @EncryptField(algorithm = AlgorithmType.AES, password = "mySecretKey")
    private String accessSecret;  // 存储时自动加密，查询时自动解密
}

执行流程：

  1. 入加密（MybatisEncryptInterceptor）：拦截 ParameterHandler.setParameters，在 SQL 执行前加密实体字段。执行后通过 FieldSnapshot.restore() 恢复原始值（防止内存污染）。
  2. 出解密（MybatisDecryptInterceptor）：拦截 ResultSetHandler.handleResultSets，在结果返回前解密。
  3. 加密头标识：加密后的字符串前缀 Constants.ENCRYPT_HEADER，防止重复加密。
  4. 加密器缓存：EncryptorManager 使用 ConcurrentHashMap 缓存加密器实例，相同算法+编码+密钥的组合复用。

4. 5 种算法速查（@EncryptField 注解支持）

算法类型密钥要求典型场景
BASE64编解码无非敏感数据混淆（代码默认值）
AES对称加密16/24/32位密钥数据库字段加密（推荐）
RSA非对称加密公私钥对，≥1024位API 传输密钥交换
SM2国密非对称公私钥对政务/信创场景
SM4国密对称16位密钥政务/信创场景

注意：MD5、SHA256、SM3 哈希算法存在于 EncryptUtils 工具类中（用于数据完整性校验/密码存储），但不在 AlgorithmType 枚举中，不能用于 @EncryptField 注解。

5. 配置继承链

注解值 > YAML 全局配置

// 注解级别（字段级精准控制）
@EncryptField(algorithm = AlgorithmType.SM4, password = "fieldSecret")

// YAML 全局配置（兜底默认值）
mybatis-encryptor:
  enable: true
  algorithm: AES             # 代码默认值 BASE64，生产建议覆写为 AES 或 SM4
  password: globalSecret
  encode: BASE64

EncryptContextFactory.create(Field field) 负责合并注解值和全局配置，注解优先。

6. 验证要点

  1. 在 application.yml 中开启 api-decrypt.enabled: true，用前端登录页面观察 Network 面板中的加密请求体
  2. 找一个标注了 @EncryptField 的实体字段，在数据库直接查看存储的是否为密文（ENC_ 前缀）
  3. 关闭 mybatis-encryptor.enable，观察数据库中新插入的敏感字段是否为明文
