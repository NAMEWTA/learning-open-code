10 · ruoyi-common-excel（下）：注解驱动的列映射与下拉框

核心问题：导出 Excel 时如何自动将数据库中的「0/1」翻译为「男/女」？导入时如何提供下拉选项限制用户输入？如何合并相同内容的单元格？

1. 六大自定义注解

注解作用示例
@ExcelDictFormat字典翻译dictType="sys_user_gender" 或 readConverterExp="0=男,1=女"
@ExcelEnumFormat枚举翻译enumClass=UserStatus.class
@ExcelDynamicOptions动态下拉选项providerClass=XxxProvider.class（运行时查询）
@CellMerge单元格合并mergeBy={"deptId"} — 同一部门数据合并
@ExcelNotation批注value="请勿修改此列"
@ExcelRequired必填标记表头字体变红色

2. @ExcelDictFormat — 字典翻译的两种模式

// 模式A：通过字典服务查询（dictType 非空时）
@ExcelDictFormat(dictType = "sys_user_gender")
private String sex;
// 导出：0 → "男"（查 DictService.getDictLabel()）
// 导入："女" → 1（查 DictService.getDictValue()）

// 模式B：内联表达式（readConverterExp 直接解析）
@ExcelDictFormat(readConverterExp = "0=正常,1=停用,2=删除")
private String status;
// separator 可指定分隔符，默认 ","

前端对应：plus-ui-vue 中 <dict-tag :options="sys_user_gender" :value="row.sex" />

3. @CellMerge — 单元格合并

// 按部门名合并：同一部门的行共享一个部门名单元格
@CellMerge(mergeBy = {"deptId"})
@ExcelProperty("部门名称")
private String deptName;

// CellMergeHandler 计算合并区间：
//   遍历数据行 → 找出相同值区间 → 生成 CellRangeAddress 列表
//   空值自动中断合并区间
// CellMergeStrategy 在 Sheet 创建时预写入合并区域

4. ExcelDownHandler — 下拉框的三种策略

策略触发条件实现方式
固定值下拉选项数 ≤20 且总字符 ≤255createExplicitListConstraint（Excel 原生数据验证）
隐藏 Sheet 引用选项数 >20 或总字符 >255创建隐藏 options_N Sheet + 名称管理器引用
级联下拉一级选项数 >10隐藏 Sheet + 名称管理器 + INDIRECT() 公式

下拉范围：仅前 1000 行（LAST_DATA_ROW_INDEX = 1000），防止大数据量导致 Excel 打开缓慢。
错误提示：设置"此值与单元格定义数据不一致"提示框 + "填写内容只能为下拉中数据"输入提示。
