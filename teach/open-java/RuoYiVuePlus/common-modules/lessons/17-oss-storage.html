17 · ruoyi-common-oss：对象存储抽象层

核心问题：开发环境用本地存储、测试环境用 MinIO、生产环境用阿里云 OSS——如何做到切换存储方案而不改一行业务代码？

模块定位：18 个 Java 文件。基于 AWS S3 SDK 2.x 统一抽象——因为几乎所有云厂商的 OSS 都兼容 S3 协议（MinIO、阿里云 OSS、腾讯云 COS、七牛 Kodo、华为云 OBS）。一个接口适配所有存储。

1. 适配器模式架构

OssClient (接口)
    ↑
AbstractOssClientImpl (模板方法基类)
    ↑
DefaultOssClientImpl (S3 协议默认实现)
    ↑
OssFactory (工厂 — ConcurrentHashMap 缓存客户端实例)

未来扩展: AliyunOssClientImpl / MinioOssClientImpl 等

2. OssClient 统一操作契约

// 两套 API 风格
//   bucketXxx(bucket, key, ...) → 显式指定存储桶
//   xxx(key, ...)               → 使用默认存储桶

// 上传（支持多种数据源）
ossClient.upload("avatar.png", Path.of("/tmp/avatar.png"));
ossClient.upload("data.json", inputStream);
ossClient.upload("config.txt", "content".getBytes());

// 下载
ossClient.download("avatar.png", Path.of("/tmp/download.png"));

// 预签名 URL（临时访问链接）
String url = ossClient.presignGetUrl("report.pdf", Duration.ofHours(1));
// → https://bucket.s3.amazonaws.com/report.pdf?X-Amz-...

// 删除
ossClient.delete("temp.txt");

// 自动路径生成
String key = ossClient.buildPathKey("business", "avatar.png");
// → business/2024/07/05/uuid-xxxx.png  (自动追加日期+UUID)

3. OssFactory — 客户端工厂

// ConcurrentHashMap 缓存：configKey → OssClient
// ReentrantLock 保证每个 configKey 的线程安全

// 获取默认配置的客户端
OssClient client = OssFactory.instance();

// 获取指定配置的客户端（支持多存储源）
OssClient aliyunClient = OssFactory.instance("aliyun_oss");
OssClient minioClient = OssFactory.instance("minio_oss");

// 配置变更自动检测
// client.verifyConfig(newConfig) → 比较配置 → 不同则关闭旧客户端重建

前端对应：plus-ui-vue/src/views/system/oss/ 文件管理页面，上传组件调用 OSS API。前端通过后端预签名 URL 直接上传到 OSS（绕过业务服务器）。

4. 大文件分片上传

// S3TransferManager 基于 S3AsyncClient
// 支持大文件自动分片上传（默认分片 8MB）
// 异步非阻塞上传，Netty NIO HTTP 客户端
// 配置: 连接超时 60s，获取超时 30s，最大并发 100

5. 访问策略

策略值说明
PRIVATE0私有访问（需签名 URL）
PUBLIC_READ_WRITE1公开读写
PUBLIC_READ2公开读（最常见）

6. 自动推断路径风格

// resolvePathStyleAccess() 自动推断
// 包含阿里云/腾讯云/七牛/华为云等关键词 → 域名风格 (virtual-hosted-style)
//   如: https://bucket.s3.cn-north-1.amazonaws.com.cn/key
// 其他情况 → 路径风格 (path-style)
//   如: https://s3.amazonaws.com/bucket/key

7. 配置示例

# MinIO 示例
oss:
  endpoint: http://localhost:9000
  access-key: minioadmin
  secret-key: minioadmin
  bucket: my-bucket
  region: us-east-1
  use-https: false
  access-policy: PUBLIC_READ

8. 验证要点

  1. 本地启动 MinIO，配置 OSS 属性后测试文件上传和预签名 URL 访问
  2. 在 OssFactory.instance() 打断点，观察 ConcurrentHashMap 中的客户端缓存
  3. 用 S3TransferManager 上传一个大文件（>100MB），观察自动分片过程
