Part A: ruoyi-common-bom — 版本管理

模块定位：0 个 Java 文件，纯 POM 项目。packaging=pom。统一管理所有 24 个 common 模块的版本号，是 Maven BOM（Bill of Materials）模式的标准实践。 
<dependencyManagement>
    <dependencies>
        <!-- 统一版本: ${revision} = 6.0.0-BETA -->
        <dependency><artifactId>ruoyi-common-core</artifactId><version>${revision}</version></dependency>
        <dependency><artifactId>ruoyi-common-web</artifactId><version>${revision}</version></dependency>
        <dependency><artifactId>ruoyi-common-mybatis</artifactId><version>${revision}</version></dependency>
        ... (24 个模块)
    </dependencies>
</dependencyManagement>

使用方式：业务模块只需引入 BOM，无需指定具体版本：

<dependencyManagement>
    <dependency>
        <artifactId>ruoyi-common-bom</artifactId>
        <version>${revision}</version>
        <type>pom</type>
        <scope>import</scope>
    </dependency>
</dependencyManagement>

Part C: 一个请求的全生命周期（前后端全链路）

用户点击「用户管理 → 查询」
    │
    ▼
【前端】plus-ui-vue
  ├─ src/views/system/user/index.vue     — 页面组件
  ├─ src/api/system/user/index.ts        — API 调用 list(query)
  ├─ src/utils/request.ts                — axios 请求拦截器
  │    ├ 设置 Content-Language (I18nLocaleResolver 读取)
  │    ├ 设置 Authorization: Bearer <token> (Sa-Token)
  │    ├ 重复提交检测 (sessionStorage)
  │    └ AES 加密请求体 (可选, common-encrypt)
  │
  ▼  HTTP POST /system/user/list  {pageNum:1, pageSize:10, ...}
  │
【后端 Filter 链】  common-web
  ├─ XssFilter → XssHttpServletRequestWrapper (防XSS)
  ├─ RepeatableFilter → RepeatedlyRequestWrapper (缓存body)
  ├─ CorsFilter → Access-Control-* 头
  │
【后端 Interceptor】 common-web + common-security
  ├─ PlusWebInvokeTimeInterceptor → 记录请求参数+计时
  ├─ SaInterceptor.checkLogin() → Token 验证 (common-satoken)
  ├─ clientId/IP白名单校验 (common-security)
  │
【Controller】 ruoyi-admin
  └─ SysUserController.list(PageQuery, SysUserBo)
      ├ 参数校验 (@Validated, common-core ValidatorConfig)
      │
【Service】 ruoyi-system
  └─ SysUserServiceImpl.selectPageUserList()
      ├ @Log 注解 → LogAspect 记录操作日志 (common-log)
      │
【Mapper】  common-mybatis
  ├─ BaseMapperPlus.selectVoPage()
  ├─ @DataPermission → 数据权限过滤 (common-mybatis)
  │    ├ DataPermissionHelper → ThreadLocal 注解缓存
  │    ├ PlusDataPermissionInterceptor → JSqlParser 注入 WHERE
  │    └ SpEL 模板解析 → 部门条件注入 SQL
  ├─ PaginationInnerInterceptor → 物理分页 (common-mybatis)
  │
【Database】
  └─ SELECT * FROM sys_user WHERE (数据权限条件) LIMIT 0,10
  │
【Response 增强】 common-web
  ├─ ResponseEnhancementAdvice.beforeBodyWrite()
  ├─ JsonValueEnhancer.enhance(body)
  │    ├ TranslationJsonFieldProcessor → 字段翻译 (common-translation)
  │    └ SensitiveJsonFieldProcessor → 字段脱敏 (common-sensitive)
  ├─ Jackson 序列化 (common-json)
  │    ├ BigNumberSerializer → Long 转 String
  │    └ LocalDateTimeSerializer → yyyy-MM-dd HH:mm:ss
  │
  ▼  HTTP Response: {"code":200,"msg":"操作成功","data":{"rows":[...],"total":50}}
  │
【前端】plus-ui-vue
  ├─ src/utils/request.ts 响应拦截器
  │    ├ AES 解密 (可选)
  │    ├ code==200 → Promise.resolve(res.data)
  │    └ code!=200 → ElMessage.error / 重新登录
  ├─ el-table 渲染用户列表
  └─ v-hasPermi 指令控制按钮
