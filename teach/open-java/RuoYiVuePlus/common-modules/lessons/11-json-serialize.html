11 · ruoyi-common-json：Jackson 配置与响应增强框架

核心问题：Java Long 类型在 JavaScript 中会精度丢失、日期格式不统一、JSON 序列化时需要自动翻译和脱敏——如何统一解决？

模块定位：13 个 Java 文件。使用 tools.jackson（Spring Boot 3.x 重命名后的 Jackson）。两大核心能力：Jackson Module 自定义和三阶段响应增强流水线。

1. JacksonConfig — 自定义序列化配置

// 注册的自定义 SimpleModule：
// 序列化方向（Java → JSON）
Long / long     → BigNumberSerializer    // 超 JS 安全范围输出字符串
BigInteger      → BigNumberSerializer    // 同上
BigDecimal      → ToStringSerializer     // 始终输出字符串
LocalDateTime   → LocalDateTimeSerializer // yyyy-MM-dd HH:mm:ss

// 反序列化方向（JSON → Java）
LocalDateTime   → CustomLocalDateTimeDeserializer // 多格式自动识别
Date            → CustomDateDeserializer          // 使用 Hutool DateUtil.parse()

BigNumberSerializer 核心逻辑

if (value >= MIN_SAFE_INTEGER && value <= MAX_SAFE_INTEGER) {
    gen.writeNumber(value);  // 安全范围 → 数字
} else {
    gen.writeString(value.toString()); // 超出 → 字符串
}
// JS: Number.MAX_SAFE_INTEGER = 9007199254740991

前端对应：雪花 ID 在 JS 中会被截断。通过此序列化器，超长 ID 自动以字符串形式输出，前端直接使用字符串接收。

2. JsonUtils — JSON 工具类

// 对象 ↔ JSON
JsonUtils.toJsonString(obj);
JsonUtils.parseObject(json, User.class);
JsonUtils.parseObject(json, new TypeReference<List<User>>() {});

// 快捷解析
JsonUtils.parseMap(json);       // → Dict (Hutool)
JsonUtils.parseArray(json, User.class);  // → List<User>
JsonUtils.parseArrayMap(json);  // → List<Dict>

// 高级功能
JsonUtils.toJsonStringExcludeFields(obj, "password", "salt"); // 排除敏感字段
JsonUtils.isJson(str);          // 判断是否为合法 JSON

3. JsonValueEnhancer — 三阶段响应增强流水线

核心接口：JsonFieldProcessor，三个生命周期方法： 
collect (每个字段一次)  →  prepare (整个响应一次)  →  process (每个字段一次)

处理流程：

  1. collect — 递归扫描响应对象树，收集需要处理的字段 key，存入 JsonEnhancementContext
  2. prepare — 批量处理（如批量查询数据库），消除 N+1 问题。例如：一次查出所有 userId→userName 映射，而非每个 userId 单独查询
  3. process — 返回替换后的字段值。返回 null 置空，返回原值不修改

内置处理器：

处理器作用来源模块
TranslationJsonFieldProcessor字段翻译（用户/部门/字典/OSS）common-translation
SensitiveJsonFieldProcessor字段脱敏（手机/身份证/邮箱等）common-sensitive

技术细节：

  • 使用 IdentityHashMap 防循环引用
  • 使用 ConcurrentHashMap 缓存类型元数据（Jackson Introspector 解析的可序列化属性）
  • 处理器按 @Order 排序（AnnotationAwareOrderComparator）
  • 处理器返回复杂对象时，自动调用 enhanceTree() 二次增强
  • 排除 ByteArray/String/Resource 类型的 HttpMessageConverter

4. JSON 校验注解

// @JsonPattern — 校验字符串是否为合法 JSON
@JsonPattern(type = JsonType.OBJECT)  // 必须是 JSON 对象
private String configJson;

@JsonPattern(type = JsonType.ARRAY)   // 必须是 JSON 数组
private String listJson;

@JsonPattern  // 默认 ANY：对象或数组均可
private String anyJson;

5. 验证要点

  1. 找一个包含雪花 ID 的 API 响应，在浏览器 Console 中查看 JSON，确认 Long 类型 ID 是否为字符串格式
  2. 在 JacksonConfig 中打断点，观察序列化模块的注册过程
  3. 找一个有 @Translation 注解的 VO 类，追踪 JsonValueEnhancer.enhance() 的执行过程
