07 · ruoyi-common-mybatis（下）：查询构造器、插件链与多数据源

核心问题：复杂的联表查询、子查询、聚合统计如何用 Lambda 表达式写，而不是手写 SQL？分页、乐观锁、SQL 日志这些横切关注点如何组织？

1. QueryBuilder — 查询构造器入口

// 单表查询
LambdaQueryBuilder<SysUser> qb = QueryBuilder.lambda(SysUser.class);
qb.eq(SysUser::getStatus, "0")
  .likeIfText(SysUser::getUsername, keyword)
  .betweenIfPresent(SysUser::getCreateTime, start, end)
  .orderByAsc(SysUser::getId);

// 联表查询（基于 MyBatis-Plus-Join）
LambdaJoinQueryBuilder<SysUser> jqb = QueryBuilder.lambdaJoin(SysUser.class);
jqb.leftJoin(SysDept.class).on(SysUser::getDeptId, SysDept::getId)
   .selectAll(SysUser.class)
   .selectAs(SysDept::getDeptName, SysUserVo::getDeptName)
   .eq(SysUser::getStatus, "0");

2. 子查询支持

// 子查询构造
SubQuery<SysUserRole> subQuery = SubQuery.of(SysUserRole.class, paramFormatter);
subQuery.select(SysUserRole::getUserId)
       .eq(SysUserRole::getRoleId, roleId);

// 在主子查询中使用
LambdaCrudChainWrapper<SysUser, SysUserVo> wrapper = userMapper.lambda();
wrapper.inSub(SysUser::getId, subQuery);  // 用户ID IN (子查询)

3. PageQuery — 分页实体

public class PageQuery {
    private Integer pageSize;      // 默认 Integer.MAX_VALUE（查全部，前端分页组件通常传具体值）
    private Integer pageNum;       // 默认 1
    private String orderByColumn;  // 排序字段
    private String isAsc;          // asc/desc

    // 支持多字段各自排序
    // isAsc: "asc,desc"
    // orderByColumn: "id,createTime"
    // → ORDER BY id ASC, createTime DESC
    public Page<T> build() { ... }
}

项目中有 60 个文件使用 PageQuery 作为分页查询入参。前端分页组件（Element Plus Pagination）提交的参数直接映射到此对象。

4. MyBatis 插件链顺序

MybatisPlusInterceptor 链:
  [1] PlusDataPermissionInterceptor     → 数据权限 WHERE 注入
  [2] PaginationInnerInterceptor        → 物理分页改写（自动识别数据库类型）
  [3] OptimisticLockerInnerInterceptor  → 乐观锁版本号处理（@Version）

独立拦截器:
  SqlLogInterceptor                     → 条件启用，输出完整 SQL+参数+耗时
  InjectionMetaObjectHandler            → insertFill/updateFill 自动填充
  PlusPostInitTableInfoHandler          → 逻辑删除全局开关管理
  MPJSqlInjector                        → MyBatis-Plus-Join 联表方法注入

SQL 日志拦截器：启用 mybatis-plus.sql-log.enabled=true 后，自动将 ? 占位符替换为实际参数值，输出完整 SQL。超长 SQL（>8000字符）按 sqlLogId 分片输出。

5. 多数据源切换

底层依赖：baomidou dynamic-datasource-spring-boot4-starter。通过 @DS 注解声明数据源，DynamicRoutingDataSource 在运行时路由。 
// 声明式切换
@DS("slave")
public interface SlaveUserMapper extends BaseMapperPlus<SysUser, SysUserVo> { }

// 程序式切换
DynamicDataSourceContextHolder.push("slave");
try { ... } finally { DynamicDataSourceContextHolder.poll(); }

DataBaseHelper 通过实际连接获取 DatabaseMetaData，按数据源名称缓存数据库类型（MySQL/Oracle/PG/SQLServer），实现跨数据库方言适配。

6. LambdaQueryCondition — 便捷条件封装

// 自动处理 null/空值判断
wrapper.eqIfPresent(User::getStatus, status);      // status 非空才添加
wrapper.likeIfText(User::getUsername, username);    // username 有文本才添加
wrapper.betweenIfPresent(User::getAge, min, max);  // min/max 均非空才添加
wrapper.inIfNotEmpty(User::getDeptId, deptIds);    // 集合非空才添加

这些便捷方法消除了业务代码中大量的 if (xxx != null) { wrapper.eq(...) } 样板代码。

7. 验证要点

  1. 找到 SysDeptMapper 中使用 QueryBuilder.lambda() 的代码，观察链式条件构建
  2. 开启 SQL 日志：设置 mybatis-plus.sql-log.enabled=true，控制台将输出完整 SQL
  3. 如果有多数据源，在 Slave 数据库的 Mapper 上添加 @DS("slave")，确认读写分离生效
