05 · ruoyi-common-mybatis（上）：实体基类与增强 Mapper

核心问题：如何在每个实体中自动填充 createTime/updateTime/createBy 等审计字段，而无需在每次 INSERT/UPDATE 时手动设置？

模块定位：34 个 Java 文件，基于 MyBatis-Plus 深度定制。提供自动填充、增强 Mapper、数据权限、链式查询、多数据源感知、SQL 日志等企业级持久层能力。被 ruoyi-admin、ruoyi-system、ruoyi-gen、ruoyi-demo 等模块引用。

1. BaseEntity — 实体基类

public class BaseEntity implements Serializable {
    @TableField(fill = FieldFill.INSERT)
    private Long createDept;       // 创建部门

    @TableField(fill = FieldFill.INSERT)
    private Long createBy;         // 创建人

    @TableField(fill = FieldFill.INSERT)
    private LocalDateTime createTime;  // 创建时间

    @TableField(fill = FieldFill.INSERT_UPDATE)
    private Long updateBy;         // 更新人

    @TableField(fill = FieldFill.INSERT_UPDATE)
    private LocalDateTime updateTime;  // 更新时间
}

项目中有 22 个实体类继承 BaseEntity。

2. InjectionMetaObjectHandler — 自动填充处理器

// insertFill 逻辑：
1. 判断 metaObject.originalObject instanceof BaseEntity
2. 若是 → 直接设置 createTime/updateTime = 当前时间
          从 LoginHelper 获取当前用户 → 填充 createBy/updateBy/createDept
          用户不存在时默认填充 -1
3. 若非 → 使用 strictInsertFill 按字段名注入 createTime/updateTime

// updateFill 逻辑：
设置 updateTime = 当前时间，updateBy = 当前用户ID

如何获取当前用户？ InjectionMetaObjectHandler 通过 LoginHelper.getUserId() 获取。而 LoginHelper 又通过 Sa-Token 的 ThreadLocal 获得当前登录用户。这意味着即使不在 Controller 层，也能拿到用户信息。

3. BaseMapperPlus — 增强 Mapper 接口

// 泛型：T=实体类型, V=VO类型
public interface BaseMapperPlus<T, V> extends BaseMapper<T> {

    // 实体到VO自动转换（使用 MapstructUtils）
    V selectVoById(Serializable id);
    List<V> selectVoList(Wrapper<T> wrapper);
    List<V> selectVoPage(Page<T> page, Wrapper<T> wrapper);

    // 批量操作封装
    boolean insertBatch(Collection<T> entities);
    boolean updateBatchById(Collection<T> entities);
    boolean insertOrUpdateBatch(Collection<T> entities);

    // 链式 CRUD 入口
    LambdaCrudChainWrapper<T, V> lambda();

    // 自定义转换查询
    <C> List<C> selectObjs(Wrapper<T> wrapper, Function<T, C> mapper);
}

项目中有 32 个 Mapper 接口继承 BaseMapperPlus（而非原生 BaseMapper）。

4. LambdaCrudChainWrapper — 链式 CRUD

// 业务层使用示例（无需写 SQL）
userMapper.lambda()
    .eq(User::getStatus, UserStatus.OK)
    .likeIfText(User::getUsername, query.getUsername())
    .betweenIfPresent(User::getCreateTime, startTime, endTime)
    .voPage(page);  // 自动转换实体→VO

// 更新链
userMapper.lambda()
    .set(User::getStatus, UserStatus.DISABLE)
    .eq(User::getId, userId)
    .update();

// 聚合函数
userMapper.lambda()
    .selectCount(User::getId)  // COUNT(id)
    .eq(User::getDeptId, deptId)
    .count();

5. 逻辑删除配置

# common-mybatis.yml
mybatis-plus:
  global-config:
    dbConfig:
      logicDeleteValue: 1      # 已删除
      logicNotDeleteValue: 0   # 未删除
      insertStrategy: NOT_NULL  # 非空插入
  enableLogicDelete: true       # 全局开关（PlusPostInitTableInfoHandler 处理）

PlusPostInitTableInfoHandler 在表信息初始化后统一控制逻辑删除开关。当 enableLogicDelete=false 时，反射设置所有 TableInfo.withLogicDelete=false。
