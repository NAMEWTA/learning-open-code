03 · ruoyi-common-web：请求处理全链路

核心问题：一个 HTTP 请求从浏览器发出到 Controller 处理完毕，中间经历了什么？XSS 过滤、请求体缓存、跨域处理、日志拦截、统一响应包装——这些横切关注点如何组织？

模块定位：17 个 Java 文件，7 个包。提供 Web 层全链路基础设施，通过 Spring Boot 3.x @AutoConfiguration 自动装配，对 ruoyi-admin 零侵入——admin 模块无需任何 import 即可享受所有能力。

1. 请求处理全链路

HTTP Request
    │
    ▼
┌─────────────────────────┐  ORDER: HIGHEST_PRECEDENCE+1
│  XssFilter              │  跳过 GET/DELETE + excludeUrls
│  → XssHttpServletRequestWrapper │  清理 HTML 标签（防 XSS）
└──────────┬──────────────┘
           ▼
┌─────────────────────────┐  ORDER: 默认
│  RepeatableFilter       │  仅 JSON 请求
│  → RepeatedlyRequestWrapper │  缓存 body 字节数组（支持多次读取）
└──────────┬──────────────┘
           ▼
┌─────────────────────────┐
│  CorsFilter (Spring)    │  处理 OPTIONS 预检，添加 Access-Control-* 头
└──────────┬──────────────┘
           ▼
┌─────────────────────────────┐
│  PlusWebInvokeTimeInterceptor │  preHandle: 记录参数(脱敏) + 启动计时器
│                              │  afterCompletion: 计时 + 清理 ThreadLocal
└──────────┬──────────────────┘
           ▼
┌──────────────────┐
│  DispatcherServlet │  → Controller 处理
└──────────┬───────┘
           ▼
  正常: ResponseEnhancementAdvice（翻译/脱敏后处理）
  异常: GlobalExceptionHandler（16 个 @ExceptionHandler）
           │
           ▼
  HTTP Response (JSON)

2. 四大自动配置类

配置类注册的 Bean关键条件
FilterConfigXssFilter + RepeatableFilterxss.enabled=true 时才注册 XssFilter
CaptchaConfigCaptchaProperties 属性绑定无条件
I18nConfigI18nLocaleResolver在 WebMvcAutoConfiguration 之前加载
ResourcesConfigCorsFilter + GlobalExceptionHandler + 拦截器 + 响应增强实现 WebMvcConfigurer

所有配置类通过 META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports 自动发现，无需任何 @ComponentScan。

3. XssFilter — 防跨站脚本注入

// 跳过逻辑
if (GET 或 DELETE 请求) → 直接放行
if (请求路径匹配 excludeUrls) → 直接放行

// 核心逻辑
XssHttpServletRequestWrapper wrapper = new XssHttpServletRequestWrapper(request);
// 内部使用 HtmlUtil.cleanHtmlTag(value).trim() 剥离 <script> 等标签
// 覆盖 getParameter / getParameterValues / getParameterMap / getInputStream

前端 request.ts 中也有配套的请求体加密（AES+RSA 混合），双重保护。

4. RepeatableFilter — 请求体可重复读

Servlet 的 InputStream 默认只能读一次。但 PlusWebInvokeTimeInterceptor 需要记录请求参数日志，就需要再次读取。解决方案：

// 在构造时将请求体缓存到 byte[] body
RepeatedlyRequestWrapper wrapper = new RepeatedlyRequestWrapper(request);
// 每次 getInputStream() 返回新的 ByteArrayInputStream(body)
// 同时设置 request/response 字符编码为 UTF-8

5. PlusWebInvokeTimeInterceptor — 请求日志与计时

// preHandle:
[PLUS]开始请求 => URL[...],参数类型[json],参数:[{"username":"admin","password":"***"}]

// afterCompletion:
[PLUS]结束请求 => URL[...],耗时:[23]毫秒

自动脱敏 password/oldPassword/newPassword/confirmPassword 字段。JSON 参数截断至 4000 字符。

6. I18nLocaleResolver — 国际化

// 从请求头 content-language 解析 Locale
// 例如: content-language: zh_CN  →  Locale("zh", "CN")
// 缺省回退到系统默认 Locale

前端 request.ts 请求拦截器自动设置 Content-Language = getLanguage()（来自 store/app.ts）。
