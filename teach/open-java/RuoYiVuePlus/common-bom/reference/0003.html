速查 03 · 接入 BOM 与版本升级操作手册

配套课程：lessons/0003.html · 使命：读懂 ruoyi-common-bom

操作一：新增模块接入 BOM（4 步）

步骤文件操作
1. 创建模块ruoyi-common/ruoyi-common-xxx/pom.xml新建 POM，parent=ruoyi-vue-plus，依赖的 common 不写 version
2. 注册聚合ruoyi-common/pom.xml在 <modules> 中添加 <module>ruoyi-common-xxx</module>
3. 注册 BOMruoyi-common/ruoyi-common-bom/pom.xml在 <dependencyManagement> 中添加一个 <dependency> 块
4. 消费引用业务模块 pom.xml加 <dependency> 不写 version（可选）
记忆口诀：创建 → 聚合 → BOM → 消费

操作二：统一升级版本（1 步）

改 BOM properties 一个数字： <revision>6.1.0</revision> → 22 个依赖全部自动跟随 → mvn clean install 生效

常见错误速查

错误原因解决
version is missing依赖没在 BOM 的 dependencyManagement 中声明检查 BOM 是否有该依赖；或手写 version
版本号没更新只改了 BOM，未重新 mvn installmvn clean install -pl ruoyi-common/ruoyi-common-bom
发布的 POM 仍有 ${revision}flatten 插件未生效确认子模块继承了父 POM 的 pluginManagement
模块不参与构建未加到聚合 POM 的 modules在 ruoyi-common/pom.xml 的 <modules> 中添加

检查清单

  • ✓ 新模块 POM 的 version 来自 parent 继承（不写或写 ${revision}）
  • ✓ 聚合 POM 的 modules 中有新模块
  • ✓ BOM 的 dependencyManagement 中有新模块声明
  • ✓ 业务模块引用时不写 version
  • ✓ mvn clean install 构建成功
  • ✓ 用 mvn help:effective-pom 查看解析后的版本号是否正确

诊断命令

# 查看 Maven 解析后的完整 POM（所有变量已替换）
mvn help:effective-pom -pl ruoyi-admin

# 查看依赖树（看清每个依赖的版本来源）
mvn dependency:tree -pl ruoyi-admin

# 只构建 BOM 模块（快速验证 POM 正确性）
mvn clean install -pl ruoyi-common/ruoyi-common-bom
