ruoyi-common-bom 速查表

Maven: org.dromara:ruoyi-common-bom (packaging=pom) | 源码: RuoYi-Vue-Plus/ruoyi-common/ruoyi-common-bom/

定位

RuoYi-Vue-Plus 项目依赖版本管理的总调度中心。纯 POM 构件（packaging=pom，无 Java 代码），通过 Maven BOM 机制集中管理全部 22 个 ruoyi-common-* 子模块的统一版本，实现「一处声明，全局生效」。依赖方通过 scope=import 导入即可自动获得版本号。

BOMdependencyManagement版本治理CI Friendly

核心文件

文件名路径职责
pom.xml（BOM） ruoyi-common/ruoyi-common-bom/pom.xml BOM 定义 POM：packaging=pom，在 <dependencyManagement> 中声明所有 common 模块的统一版本（${revision}）
pom.xml（父 POM） pom.xml（项目根） 父 POM 通过 <scope>import</scope> 导入 BOM，让所有子模块自动获得版本号
pom.xml（common 聚合） ruoyi-common/pom.xml 聚合 POM：用 <modules> 声明子模块构建顺序（管构建顺序，不管版本）

术语速查

术语一句话解释
BOM（Bill of Materials）Maven 特殊 POM（packaging=pom），在 <dependencyManagement> 中集中声明依赖版本，不产出 JAR/WAR
dependencyManagement声明依赖版本的区块，不实际引入依赖到 classpath，只定义「如果要用，版本应该是多少」
${revision}Maven 3.2+ CI Friendly 版本变量，替代硬编码版本号，配合 flatten-maven-plugin 在打包时替换为真实版本
flatten-maven-pluginMojohaus 插件（1.7.3），resolveCiFriendliesOnly 模式只替换 ${revision}/${sha1}/${changelist} 三个变量
scope=import仅在 <dependencyManagement> 中使用，把目标 BOM 的版本声明「合并」到当前 POM
版本仲裁Maven 依赖冲突自动选择规则：①路径最近优先 ②同深度先声明优先。BOM 通过锁定版本绕过仲裁的不确定性
聚合 POM vs BOM聚合 POM 用 <modules> 管构建顺序；BOM 用 <dependencyManagement> 管版本一致性。本项目两者分离
GAVMaven 坐标三要素（GroupId:ArtifactId:Version），BOM 通过 GAV 精确锁定「谁、是什么版本」
type=pom依赖声明中 <type>pom</type> 表示该依赖产物是 POM 文件而非 JAR，导入 BOM 时必须指定
packaging=pom模块打包类型为 POM（不产出二进制构件），聚合模块和 BOM 模块均使用此类型

常用代码

BOM 的 pom.xml 结构

<?xml version="1.0" encoding="UTF-8"?>
<project>
  <modelVersion>4.0.0</modelVersion>
  <groupId>org.dromara</groupId>
  <artifactId>ruoyi-common-bom</artifactId>
  <version>${revision}</version>
  <packaging>pom</packaging>

  <dependencyManagement>
    <dependencies>
      <dependency>
        <groupId>org.dromara</groupId>
        <artifactId>ruoyi-common-core</artifactId>
        <version>${revision}</version>
      </dependency>
      <!-- ... 其余 21 个依赖 ... -->
    </dependencies>
  </dependencyManagement>
</project>

父 POM 导入 BOM（关键：scope=import）

<dependencyManagement>
  <dependencies>
    <dependency>
      <groupId>org.dromara</groupId>
      <artifactId>ruoyi-common-bom</artifactId>
      <version>${revision}</version>
      <type>pom</type>
      <scope>import</scope>  <!-- 核心！合并 BOM 的版本声明 -->
    </dependency>
  </dependencies>
</dependencyManagement>

子模块省写 version（版本由 BOM 仲裁）

<dependencies>
  <dependency>
    <groupId>org.dromara</groupId>
    <artifactId>ruoyi-common-core</artifactId>
    <!-- 无需 <version>，由 BOM 的 dependencyManagement 提供 -->
  </dependency>
</dependencies>

版本链路追踪

// 完整链路：
// 1. BOM 声明: ruoyi-common-core.version = ${revision}
// 2. flatten-maven-plugin 替换: ${revision} → 6.0.0-BETA
// 3. 父 POM scope=import 导入 BOM
// 4. 子模块引用时无需写 version
// 5. Maven 仲裁: 从最近的 dependencyManagement（父 POM 导入的 BOM）取版本
