版本仲裁链路

ruoyi-common-bom 声明 22 个版本 → 父 POM scope=import 导入 → 父 POM dependencyManagement 包含全部 22 个版本 → 子模块 dependencies 只写 groupId:artifactId（不写 version） → Maven 从最近 dependencyManagement 取 version → 全模块统一版本，无分裂 ✓

记忆锚点

  • BOM 只管版本，聚合 POM 只管构建顺序——两者职责分离。
  • 22 个依赖全用 ${revision}，改版号 = 改 BOM properties 一行。
  • 子模块不写 version → version 来自最近 dependencyManagement → BOM 通过 scope=import 在父 POM 提前「占位」→ 仲裁规则保证 BOM 版本胜出。
  • POM 共 186 行，无 Java 代码，纯 XML 配置。
