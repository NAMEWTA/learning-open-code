速查 01 · ruoyi-common-job 模块全景

配套课程：lessons/0001.html · 使命：读懂 ruoyi-common-job 定时任务模块

一句话定位

ruoyi-common-job 不是任务调度框架，而是把开源框架 SnailJob（Aizuda 出品）的客户端接入 RuoYi 技术栈的「适配垫片」——自己只做自动装配 + 开关控制 + 远程日志三件事。

模块全部内容

文件作用
pom.xml依赖 snail-job-client-starter + snail-job-client-job-core + ruoyi-common-core
config/SnailJobConfig.java43 行：4 个注解 + 1 个事件监听方法，完成客户端接入（第 2 课）

一个类 = 四件事

机制职责解决
@AutoConfiguration自动装载依赖即生效，零配置接入
@ConditionalOnProperty开关控制yml 一键启停，开发可关闭
@EnableSnailJob客户端启动连接 Server、扫描任务、接收调度
onStarting() + @EventListener远程日志挂载 SnailLogbackAppender 到根 Logger

记忆锚点

  • 调度的脏活全在 SnailJob，本模块零行调度逻辑。
  • 模块不暴露 Controller，是被业务（ruoyi-job）依赖的能力层。
  • 写了任务后，要在 SnailJob Server 控制台创建对应任务，填写 c ron 和「执行器名称」（必须与 @JobExecutor.name 一致）。
  • 用 SnailJobLog.REMOTE 输出关键日志，运维在控制台就能看到。
