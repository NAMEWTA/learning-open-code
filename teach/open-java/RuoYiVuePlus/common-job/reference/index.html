Maven: org.dromara:ruoyi-common-job | ruoyi-common-job/src/main/java/org/dromara/common/job/

定位

对开源分布式任务调度框架 SnailJob 的「客户端接入适配」模块。仅 1 个 Java 类（43 行），通过 Spring Boot 自动配置实现「依赖即生效」，用 @ConditionalOnProperty 实现开关控制，用事件监听挂载远程日志 Appender。

核心文件

类名路径职责
SnailJobConfigconfig/SnailJobConfig.java模块唯一类，@AutoConfiguration + @ConditionalOnProperty + @EnableScheduling + @EnableSnailJob 四注解自动接入，@EventListener 挂载 SnailLogbackAppender

关键注解一览

注解作用
@AutoConfiguration声明为 Spring Boot 自动配置类，通过 AutoConfiguration.imports 自动加载
@ConditionalOnProperty仅 snail-job.enabled=true 时生效，配置文件一键开关
@EnableScheduling启用 Spring 原生调度（与 SnailJob 互补用于简单场景）
@EnableSnailJobSnailJob 客户端开关注解，触发客户端全部自动装配（连接服务端、注册执行器、接收调度）
@JobExecutor标注任务类，name 指定任务名，服务端按此名称调度
@EventListener监听 SnailClientStartingEvent，在客户端启动时挂载远程日志 Appender

术语速查

术语解释
SnailJobAizuda 社区分布式任务调度与重试框架，Server + Client 架构
SnailClientStartingEvent客户端启动事件，SnailJobConfig.onStarting() 在此挂载日志 Appender
SnailLogbackAppenderLogback 日志追加器，任务日志自动回传 SnailJob 服务端控制台
SnailJobLog双通道日志工具：LOCAL 输出本地，REMOTE 同时回传服务端
@JobExecutor标注任务执行器，约定方法名 jobExecute(JobArgs)，返回 ExecuteResult
JobArgs任务执行参数 DTO，携带 jobParams、wfContext、shardingArgs 等
ExecuteResult任务执行结果封装，.success(data) 或 .failure(msg)，客户端据此更新状态
@ConditionalOnPropertySpring Boot 条件装配，prefix + name + havingValue 控制是否生效

服务端操作流程

1. SnailJob 控制台 → 任务管理 → 新建任务
2. 任务名称填写 @JobExecutor(name) 的值（如 "cleanExpiredLogs"）
3. 配置 Cron 表达式、路由策略、阻塞策略等
4. 启用任务 → 调度触发 → 客户端执行 → 控制台查看日志
