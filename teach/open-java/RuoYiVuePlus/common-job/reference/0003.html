任务代码模板

@Component
@JobExecutor(name = "任务唯一标识")
public class XxxTask {
    public ExecuteResult jobExecute(JobArgs jobArgs) {
        // 1. 获取参数（可选）
        // String params = (String) jobArgs.getJobParams();

        // 2. 执行业务逻辑
        // doSomething(params);

        // 3. 记录日志（推荐用 REMOTE 回传 Server）
        SnailJobLog.REMOTE.info("任务执行中...关键信息");

        // 4. 返回结果
        return ExecuteResult.success("任务完成");
        // 或 return ExecuteResult.failure("失败原因");
    }
}

核心约定

元素必须满足错了会怎样
类注解@Component 必须（注册 Spring Bean）+ @JobExecutor(name) 必须SnailJob 找不到任务类
name 值必须与 Server 控制台「执行器名称」完全一致调度指令到达但找不到执行器
方法参数JobArgs jobArgs反射调用失败
返回类型ExecuteResult反射调用失败
日志用 SnailJobLog.REMOTE 而非 @Slf4j 的 Logger日志不回传 Server

任务类型对比

类型必需注解关键 API适用场景
普通任务@JobExecutorjobExecute(JobArgs)单一操作定时执行
广播任务@JobExecutorjobExecute(JobArgs) + 服务端选「广播」所有节点刷新缓存
静态分片@JobExecutorjobArgs.getJobParams() 解析分片参数按范围批量处理
Map 任务+ @MapExecutormapHandler.doMap(data, "taskName")动态拆分并行执行
MapReduce+ @MapExecutor + @ReduceExecutor+ reduceArgs.getMapResult() 聚合拆分+汇总
DAG 工作流@JobExecutorjobArgs.getWfContext() / appendContext()多步骤有依赖编排

完整执行链路

写代码（@JobExecutor） → 配 yml（enabled=true + Server 地址） → 启 Server（8800/snail-job） → 启客户端应用 → Server 控制台创建任务（填执行器名称 = name） → 等 cron 触发 → Server 分发 → 客户端执行 → 日志回传（SnailLogbackAppender + SnailJobLog.REMOTE） → 控制台查看结果

常见故障排查

现象排查顺序
任务不执行① enabled: true？② Server 在运行？③ token 一致？④ group 存在？⑤ name 拼写一致？
控制台无日志① 是否用了 SnailJobLog.REMOTE？② Appender 是否挂载成功（看启动日志）？
任务报错① 参数类型是否匹配？② 业务异常是否被 catch 了（应抛出或返回 failure）？

记忆锚点

  • @JobExecutor(name = "xxx") 的 name = Server 控制台的「执行器名称」——必须一致。
  • 用 SnailJobLog.REMOTE 而非 @Slf4j 的 Logger 输出关键日志——运维在控制台才能看到。
  • 任务执行失败有两种处理方式：抛 RuntimeException（SnailJob 记录失败状态）或 return ExecuteResult.failure("原因")。
  • Map 和 MapReduce 的拆分逻辑写在代码里（mapHandler.doMap()），静态分片的拆分参数在服务端配。
  • DAG 工作流通过 jobArgs.appendContext() / getWfContext() 传递数据，在 Server 控制台拖拽编排。
