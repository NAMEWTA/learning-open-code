速查 03 · AllUrlHandler URL 扫描与实战配置

配套课程：lessons/0003.html

它是什么

实现 InitializingBean，在容器启动后从 RequestMappingHandlerMapping 自动收集全系统 URL，将 {xxx} 路径变量替换为 * 通配符，供 SecurityConfig 中的 SaRouter.match() 使用。

核心流程

Spring 启动 → RequestMappingHandlerMapping 扫描所有 @Controller @RequestMapping → AllUrlHandler.afterPropertiesSet() 执行 → getHandlerMethods() 获取所有路径映射 → 正则 \{(.*?)\} 替换 {xxx} 为 * → HashSet 去重 → 存入 urls 列表 → SecurityConfig.addInterceptors() 中 SaRouter.match(urls) 消费

路径替换示例

原始路径替换后
/system/user/{userId}/system/user/*
/system/dept/{deptId}/system/dept/*
/monitor/cache/{cacheName}/monitor/cache/*
/system/user/list/system/user/list（无变化）

设计要点

设计决策原因
实现 InitializingBean 而非 @PostConstruct语义更明确——阅读者一眼知道「初始化时会做事」
用 SpringUtils.getBean() 而非 @Autowired延迟获取，避免 Bean 创建顺序依赖；按名称获取更精确
HashSet 去重多个 HandlerMethod 可能映射到相同路径模式
替换为 * 而非保留 {xxx}SaRouter 使用 AntPathMatcher，* 是单段通配符，{xxx} 是字面量
正则 \{(.*?)\}非贪婪匹配任意花括号内容，通用性强

excludes vs AllUrlHandler

AllUrlHandler.getUrls()security.excludes
方向正向：需鉴权的路径反向：不需鉴权的路径
数据源自动扫描 @RequestMappingyml 手动配置
使用处SaRouter.match(urls).excludePathPatterns(excludes)

实战：新增排除路径

# application-dev.yml security: excludes: - /login - /captcha/** - /health # ← 新增公开接口路径

记忆锚点

  • AllUrlHandler 零维护成本：新增 Controller 无需改配置，自动纳入鉴权范围。
  • 路径替换目的：让 SaRouter 能匹配带变量的实际请求路径。
  • InitializingBean 保证执行时机：所有 Bean 初始化完毕、@RequestMapping 注册完毕之后。
  • excludePathPatterns 只排除 Interceptor，不影响 ContextFilter。
  • 需要排除新接口：只改 yml，不动代码。
