速查 02 · SecurityConfig 自动配置与鉴权链

配套课程：lessons/0002.html

SecurityConfig 类结构

@AutoConfiguration + @EnableConfigurationProperties(SecurityProperties) + implements WebMvcConfigurer → 3 个安全 Bean/Override 方法

3 个安全组件总览

组件类型触发时机核心功能
saTokenContextFilterRegistration Servlet Filter（重注册） 请求最早 初始化 SaTokenContext，覆盖 REQUEST+ASYNC+ERROR
addInterceptors → SaInterceptor Spring Interceptor Filter 之后、Controller 之前 四步鉴权链：匹配→登录→clientId→白名单
getSaServletFilter Servlet Filter 仅 /actuator/** Basic Auth 保护运维端点

鉴权链四步（addInterceptors 核心）

步骤代码失败
1. 路由匹配SaRouter.match(allUrlHandler.getUrls())不检查，放行
2. 登录校验StpUtil.checkLogin()NotLoginException
3. clientId 校验equalsAny(tokenCid, headerCid, paramCid)NotLoginException("-100")
4a. 路径白名单StringUtils.matches(requestPath, accessPathList)NotPermissionException
4b. IP 白名单NetUtils.isMatchIpRule(rule, clientIp)NotPermissionException

ContextFilter 重注册关键参数

参数值意义
urlPatterns/*覆盖全部请求
dispatcherTypesREQUEST, ASYNC, ERRORSSE/WebSocket 场景不会丢上下文
asyncSupportedtrue支持异步请求
orderHIGHEST_PRECEDENCE最先执行

SecurityProperties 配置

yml: security.excludes → @ConfigurationProperties(prefix="security") → SecurityProperties.excludes → excludePathPatterns()

记忆锚点

  • ContextFilter 重注册动机：默认只在 REQUEST 下初始化 → SSE/WebSocket 等 ASYNC 场景会报 SaTokenContext 未初始化。
  • clientId 校验动机：防止 token 跨客户端盗用（PC 的 token 在 APP 上用会被拒绝）。
  • validateClientAccessRules 两个检查都是 配置了才校验（不配 = 不限制）。
  • actuator 的账密来自 spring.boot.admin.client.username/password。
