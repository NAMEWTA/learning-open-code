速查 01 · ruoyi-common-security 模块全景

配套课程：lessons/0001.html · 使命：读懂 ruoyi-common-security 安全配置模块

一句话定位

ruoyi-common-security 不是认证框架，而是站在 ruoyi-common-satoken（基础能力）肩膀上的「应用级安全横切层」——三道防线扎在请求链的三个要冲。

模块全部内容

文件作用
pom.xml只依赖 ruoyi-common-satoken 一个兄弟模块
config/SecurityConfig.java装配中心：注册上下文过滤器 + 路由拦截器 + actuator 过滤器（第 2 课）
config/properties/SecurityProperties.java配置类：暴露 security.excludes 排除路径（第 2 课）
handler/AllUrlHandler.javaURL 扫描：收集全系统 URL 并替换路径变量为 *（第 3 课）

三件套 = 三道关卡

组件职责请求链位置
SecurityProperties提供 yml 可配排除路径启动时加载
SecurityConfig注册 3 个安全 Bean请求到达 → 三条防线
AllUrlHandler自动收集全系统 URL启动时执行，产出供 SecurityConfig 使用

请求穿过三条防线

客户端 → ContextFilter(初始化上下文) → SaInterceptor(登录+clientId+白名单) → ActuatorFilter(Basic Auth) → Controller

与 ruoyi-common-satoken 的关系

  • satoken 模块：封装 Sa-Token API，提供 LoginHelper、StpUtil 等基础能力。
  • security 模块：决定「对谁、何时、按什么规则」调用这些能力。
  • 核心公式：satoken 给你枪，security 告诉你枪架在哪。

记忆锚点

  • 只有 3 个 Java 类，但扎在请求链的 3 个关键位置。
  • ContextFilter 重注册的动机是 ASYNC/ERROR 分发场景（SSE、WebSocket、异步 Controller）。
  • 鉴权链四步：匹配→登录→clientId→路径/IP白名单，任一步失败即拦截。
  • AllUrlHandler 用 InitializingBean 保证全量 URL 在容器就绪后收集。
