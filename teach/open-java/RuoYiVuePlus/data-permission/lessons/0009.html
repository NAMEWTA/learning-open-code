核心难点：父子联动下的祖先补全

这是 React 版与 Vue 版最大的差异，也是最考验理解的地方。

问题背景：antd 的 Tree 在 checkStrictly=false（父子联动）时，checkedKeys 回调 不包含半选状态的父节点。但后端 updateRolePermission 需要收到完整的被选中节点列表（含祖先），否则下次回显无法还原。

Vue 的 el-tree 有 getHalfCheckedKeys() API，一行搞定。antd Tree 没有这个 API，所以需要手写。

解决方案（useRolePermission.tsx:379-408）：

// 提交前调用：父子联动时补全祖先
function getTreeCheckedKeysWithAncestors(checkedKeys, nodes, connect) {
  if (!connect) return checkedKeys;   // 非联动模式直接用原始checkedKeys
  return normalizeIds([...checkedKeys, ...collectAncestorKeys(checkedKeys, nodes)]);
}

// DFS 遍历，找出所有"子节点被勾选"的祖先节点
function collectAncestorKeys(checkedKeys, nodes) {
  const checkedKeySet = new Set(checkedKeys.map(String));
  const ancestors = [];

  const visit = (node, path) => {
    const ownKey = node.key;
    let subtreeChecked = checkedKeySet.has(String(ownKey));

    node.children?.forEach(child => {
      if (visit(child, [...path, ownKey])) {
        subtreeChecked = true;  // 子树中有被勾选的节点 → 标记当前节点
      }
    });

    if (subtreeChecked) {
      ancestors.push(...path);  // path里是当前节点的所有祖先
    }
    return subtreeChecked;
  };

  nodes.forEach(node => visit(node, []));
  return normalizeIds(ancestors);  // 去重返回
}
为什么这样做？ DFS 从叶子向上回溯：如果某个节点的子树中有任何被勾选的节点，就把这个节点的所有祖先加入结果。这样后端收到的是"叶子 + 路径上的所有祖先 ID"，下次 deptTreeSelect 返回的 checkedKeys 才能正确还原父子联动状态。
