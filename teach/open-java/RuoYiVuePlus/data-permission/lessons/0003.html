03 六种数据范围 — DataScopeType 枚举与 SpEL 模板

本节目标：你能背出六种数据范围的含义、知道每种最终生成什么样的 SQL 片段、能从 SpEL 模板直接推断出渲染结果。

枚举全貌

DataScopeType.java:26-56 定义了完整的规则引擎。每个枚举值由三个字段构成：code（前端传的 1-6）、sqlTemplate（SpEL 模板）、elseSql（兜底 SQL）。

SpEL 模板语法逐字解析

模板用 #{...} 作为表达式分隔符（不是 Spring 默认的 #{}，而是 TemplateParserContext 配合 SpelExpressionParser 解析的）。

模板里的三个"变量来源"：

写法来源示例渲染结果（假设值）
#deptName注解 key→value 映射注解 key="deptName" value="d.dept_id"d.dept_id（一个列名字符串）
#userName同上注解 key="userName" value="create_by"create_by
#user.deptIdLoginUser 对象属性当前用户 deptId=103103
#user.userId同上当前用户 userId=11
#roleIdHandler 循环注入当前遍历的角色 roleId=100100
@sdss.getRoleCustom(#roleId)Spring Bean 方法调用角色100勾了部门 101,102101,102
@sdss.getDeptAndChild(#user.deptId)同上deptId=103 及其子部门103,104,105

逐一理解每种范围

code=1 ALL — 全部数据

模板为空字符串。在 Handler 中遇到 ALL 直接 return StringUtils.EMPTY（Handler:148-149），即不加任何过滤条件。实际效果等同于超管。

code=2 CUSTOM — 自定义数据

模板: #{#deptName} IN ( #{@sdss.getRoleCustom( #roleId )} )
渲染: dept_id IN ( 101, 102 )

@sdss.getRoleCustom(roleId) 查 sys_role_dept 表，返回逗号分隔的部门 id 串（SysDataScopeServiceImpl:42-54）。这是唯一需要前端勾选部门树的数据范围。

code=3 DEPT — 仅本部门

模板: #{#deptName} = #{#user.deptId}
渲染: dept_id = 103

最简单直接：用等值比较，只看当前用户所属部门的记录。

code=4 DEPT_AND_CHILD — 本部门及以下

模板: #{#deptName} IN ( #{@sdss.getDeptAndChild( #user.deptId )} )
渲染: dept_id IN ( 103, 104, 105 )

@sdss.getDeptAndChild(deptId)（SysDataScopeServiceImpl:64-69）通过 deptMapper.selectDeptAndChildById 用 findInSet 递归查出所有子部门 id。

code=5 SELF — 仅本人

模板: #{#userName} = #{#user.userId}
渲染: create_by = 1

注意这里用的是 #userName（注解 key="userName" → 列名）和 #user.userId（当前用户 ID）。如果注解没声明 key="userName"，这个条件就不会生成。

code=6 DEPT_AND_CHILD_OR_SELF — 部门及以下或本人

模板: #{#deptName} IN ( #{@sdss.getDeptAndChild( #user.deptId )} ) OR #{#userName} = #{#user.userId}
渲染: dept_id IN ( 103, 104, 105 ) OR create_by = 1

组合 4+5：能看到自己部门的及其子部门的数据，或者自己创建的记录。

elseSql 的作用：模板匹配失败时的兜底

当模型的模板里不包含某个注解 key 时，渲染失败。此时 elseSql 生效。ALL 以外所有类型都设了 " 1 = 0 "，即「查不到任何数据」。这是一种安全兜底：宁可少查，不要越权。

核心认知：DataScopeType 是用 SpEL 模板实现的规则引擎。新增数据范围 = 加一个枚举值 + 写一条 SpEL 模板 +（如果需要新数据源）在 @Service("sdss") 里加方法。模板里的变量由 Handler 的 StandardEvaluationContext 注入。
