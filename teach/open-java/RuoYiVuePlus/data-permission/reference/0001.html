六环节全链路

#环节核心类做什么
①注解声明@DataPermission + @DataColumn声明哪些列受数据权限控制（key→value 映射）
②AOP 切面DataPermissionAdviceMapper 方法调用前，注解塞入 ThreadLocal
③角色配置SysLoginService / LoginUser登录时查角色 dataScope + sys_role_dept，组装 LoginUser
④MyBatis 拦截器PlusDataPermissionInterceptorbeforeQuery/beforePrepare 检查 ThreadLocal → 改写 SQL
⑤Handler 拼 WHEREPlusDataPermissionHandler遍历角色 → 查 DataScopeType → SpEL 渲染 → 拼片段
⑥最终 SQL—WHERE 原条件 AND (数据权限条件)

注解速查

@DataColumn

String[] key() default "deptName";    // SpEL 变量名
String[] value() default "dept_id";   // 数据库列名（join 时带别名如 d.dept_id）

@DataPermission

DataColumn[] value();         // 一组 DataColumn
String joinStr() default "";  // 多角色拼接符，默认 SELECT→OR, UPDATE/DELETE→AND

常用写法

// 单表双列（最常用）
@DataPermission({
    @DataColumn(key = "deptName", value = "dept_id"),
    @DataColumn(key = "userName", value = "create_by")
})

// 多表 join 带别名
@DataPermission({
    @DataColumn(key = "deptName", value = "d.dept_id"),
    @DataColumn(key = "userName", value = "u.create_by")
})

// 指定 AND 拼接
@DataPermission(joinStr = "AND", value = {
    @DataColumn(key = "deptName", value = "dept_id")
})

Three-Layer Short Circuit

  1. 超管短路：LoginHelper.isSuperAdmin() → 直接返回原 SQL
  2. 无注解短路：ThreadLocal 里没有 @DataPermission → Interceptor 跳过
  3. Ignore 短路：DataPermissionHelper.ignore() → 临时关闭权限

关键约束

  • key/value 数组必须等长，否则抛 ServiceException
  • 注解放在Mapper 接口方法上（default 方法或抽象方法均可）
  • 注解可放类级别（@Target TYPE + METHOD），方法注解优先于类注解
