六种数据范围速查

code枚举SpEL 模板渲染示例（假设参数）
1ALL""无条件，全局不过滤
2CUSTOM#{#deptName} IN ( #{@sdss.getRoleCustom(#roleId)} )dept_id IN (101,102)
3DEPT#{#deptName} = #{#user.deptId}dept_id = 103
4DEPT_AND_CHILD#{#deptName} IN ( #{@sdss.getDeptAndChild(#user.deptId)} )dept_id IN (103,104,105)
5SELF#{#userName} = #{#user.userId}create_by = 1

SpEL 变量来源速查

变量来源类型
#deptName注解 key→value 映射（如 "dept_id"）String（列名）
#userName同上（如 "create_by"）String（列名）
#user.deptIdLoginUser.deptIdLong
#user.userIdLoginUser.userIdLong
#roleIdHandler 循环注入Long
@sdss.getRoleCustom(#roleId)Spring Bean "sdss" 方法String（逗号分隔 id）
@sdss.getDeptAndChild(#user.deptId)同上String（逗号分隔 id）

NullSafe 兜底：变量/属性为 null 时统一返回 "-1"，防止 IN () 或 = null 破坏 SQL。

getSqlSegment 决策树

getSqlSegment(where, isSelect)
 ├─ isSuperAdmin?  → 原样返回 ← 短路！
 ├─ buildDataFilter 返回空? → 原样返回
 ├─ 原 where 非空? → where AND ( dataFilter )
 └─ 原 where 为空? → ( dataFilter )
 finally → DataPermissionHelper.removePermission()

buildDataFilter 六阶段

1. joinStr = isSelect ? "OR" : "AND"
2. 构建 NullSafe SpEL 上下文
3. scopeRoles(user, access) — 接口约束筛选角色
4. 注册 key→value 列名变量
5. 逐角色遍历渲染模板 → conditions 收集
   └─ ALL? → 直接返回空 ← 短路！
   └─ ignore 包裹渲染 ← 防递归
   └─ 模板不匹配? → elseSql 兜底
6. conditions.join → 去开头 joinStr
