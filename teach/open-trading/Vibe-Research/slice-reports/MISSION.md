# 使命：研报上传与管理全链路

## 为什么
理解 Vibe-Research 如何将用户拖拽的研报文件（PDF/Word/图片等）经浏览器端 base64 编码、后端校验归档、按文件名关键词自动打行业标签，最终支持下载和删除的完整数据流。这是个人投研系统中"私有知识库"的核心能力。

## 成功的样子
- 能画出从前端拖拽上传到后端本地落盘的全链路时序图，标注每一层的职责与异常兜底
- 能解释 base64 传输的取舍、行业关键词匹配的设计意图、以及 index.json 并发安全的实现
- 能快速定位 API 端点、错误码与前端组件之间的对应关系

## 约束条件
- 课程控制在 15 分钟内完成，正文 800-1200 字
- 聚焦单次请求的全链路，不展开 FastAPI 框架或其他模块
- 源码文件不超过 3 个

## 不在范围内
- FastAPI 中间件（鉴权/CORS）的通用机制
- 前端 React 组件设计模式（GlassCard、PageHeader 等 UI 组件）
- 其他 API 端点的实现（行情、持仓、AI 对话等）
