# 使命：每日复盘全链路 — 从前端到数据源的端到端追踪

## 为什么
每日复盘是 Vibe-Research 的默认首页，聚合了大盘指数、全球市场、市场情绪、短线情绪、成交额 TOP20、板块资金、自选行情与 AI 复盘八个数据维度。理解这条链路不是看懂一个页面——而是掌握前后端如何通过并行加载、缓存分层、优雅降级协作完成复杂看板的渲染与 AI 闭环。这是理解全栈数据产品架构的最佳入口。

## 成功的样子
- 能画出每日复盘页面 8 路数据从 `loadIndices()` 到腾讯/东财 API 的完整调用图，标注每一层的职责与缓存策略
- 能解释 `pending(done)` 的优雅降级逻辑：如何区分"加载中"与"数据源暂不可用"
- 能说出 AI 复盘的数据组装方式：`dataSummary` 如何从指数行情拼成 prompt，经 `/api/chat` 流式返回
- 遇到页面数据空白时，能沿链路逐层排查：浏览器 Network → Vite 代理 → FastAPI 端点 → market.py 缓存 → 外部 API

## 约束条件
- 学习时间碎片化，每节课控制在 15 分钟内
- 已具备 L1-backend 和 L1-frontend 的基础知识（app.py 端点路由、market.py 数据层、前端页面路由与组件架构）
- 目标读者具备编程基础，无需解释 React hooks 或 FastAPI 装饰器的基础语法

## 不在范围内
- A 股数据工具箱的十层数据架构细节（已在 L1-a-stock-data 覆盖）
- 全球股票数据工具箱的八层架构（已在 L1-global-stock-data 覆盖）
- 资讯雷达、持仓管理、个股研报等其他垂直切片的链路分析（各属独立 L2 主题）
- AI Chat 面板的 function-calling 工具定义与 MCP 协议细节（属 slice-ai-chat）
