主成功路径 — 8 路并行加载、渲染与 AI 复盘闭环

学习目标：15 分钟内能解释 DailyReview 从 useEffect 到 AI 复盘 Markdown 渲染的完整成功路径，理解并行加载、双态占位与流式输出的协作方式。

1. 阶段一：8 路数据并行发射

页面挂载瞬间，loadIndices() 发射 5 路请求，refreshWatch() 发射 1 路——共 6 个 HTTP GET 在同一个事件循环中发出。另有 api.indices() 和 api.globalIndices() 共享 loadIndices 但各自独立 resolve，实际是 5+1=6 路独立 Promise。

// DailyReview.tsx:44-50 — loadIndices 并行发射 const loadIndices = () => { api.indices().then(setIndices).catch(() => setIdxErr(true)); api.globalIndices().then(setGlobalIdx).catch(() => {}); api.marketOverview().then(setOverview).catch(() => {}) .finally(() => setOvDone(true)); api.emotion().then(setEmotion).catch(() => {}) .finally(() => setEmoDone(true)); api.turnoverTop().then(setTurnover).catch(() => {}) .finally(() => setToDone(true)); };

注意 catch 的差异：指数行情失败时设置 idxErr=true 以显示"行情未接通"；其余 4 路失败时静默吞错，渲染层由 pending(done) 兜底。这是分层容错——核心模块显式报错，辅助模块优雅降级。

2. 阶段二：各数据块的条件渲染

数据到达后，React setState 触发重渲染。页面分为 8 个可视化区域，每个区域有自己的空态渲染逻辑：

数据块成功态空态（数据未到）空态（数据源不可用）
大盘指数4 列 GlassCard 网格4 个骨架占位 + "加载中…"4 个骨架 + "行情未接通"（idxErr）
全球市场5 列 GlassCard 网格不渲染（globalIdx.length > 0）整块隐藏
市场情绪宽度/投机+涨跌家数pending(false) → "加载中…"pending(true) → "暂无数据"
短线情绪连板梯队+比率+表格pending(false) → "加载中…"pending(true) → "暂无数据"
成交额 TOP2020 行表格pending(false) → "加载中…"pending(true) → "暂无数据"
板块资金15 行表格+资金轮动同上同上
自选行情网格卡片+实时价格"加上你关注的股票"引导无自选则不显示
AI 复盘Markdown 渲染+保存按钮"点上方按钮"引导文字错误提示+设置引导（needConfig）

核心函数 pending(done) 接收一个布尔值——请求是否已结束（无论成败）。已结束但数据为空 → 数据源不可用；未结束 → 加载中。这个简单的双态模型避免了用户面对"永远转圈"的焦虑。

3. 阶段三：AI 复盘的数据组装与流式渲染

用户点击"让 AI 复盘今天"，触发 runReview()。它不直接调 LLM——而是先把当前 indices 状态拼接成文本摘要 dataSummary，作为 prompt 的一部分发给用户自己的 AI。

// DailyReview.tsx:85-101 — dataSummary 组装与流式调用 const dataSummary = indices.length ? indices.map((i) => `${i.name} ${i.price}（${i.change_pct > 0 ? "+" : ""}${i.change_pct}%）` ).join("；") : "（指数数据未取到）"; const runReview = async () => { if (!hasLlm()) { setNeedConfig(true); return; } setReviewLoading(true); setReview(""); const prompt = `以下是今天 A 股大盘的客观数据：\n${dataSummary}\n\n` + "请用中文做一段当天大盘复盘..."; await chatStream( [{ role: "user", content: prompt }], `今日大盘数据：${dataSummary}`, { onDelta: (t) => setReview((r) => r + t) } ); };

chatStream（llm.ts）做三件事：(1) 从 localStorage 读 LLM 配置；(2) POST /api/chat 发起 NDJSON 流式请求；(3) 逐行解析 {"type":"delta","text":"..."} 事件，回调 onDelta 追加文本。后端 chat.py 将消息连同 context 转发给用户配置的模型（API 接入走 OpenAI 兼容 function-calling，订阅接入 spawn 本机 CLI），边生成边 yield NDJSON 行。前端用 ReactMarkdown + remarkGfm 渲染最终内容，并由 SaveNoteButton 存入 localStorage。

4. 一条完整请求的逐帧回放

时刻 T+0ms useEffect 触发 → loadIndices() + refreshWatch() T+0ms 6 个 fetch() 并行发出（Vite 代理透明转发） T+5ms 后端 FastAPI 收到 6 个 GET，各自委托 market.py/astock.py T+5ms market.py 检查 _CACHE TTL：命中直接返回，未命中调外部 API T+200ms 最快响应到达 → setState → React 首次重渲染（骨架→数据） T+800ms 最慢响应到达 → 所有 ovDone/emoDone/toDone 置 true → 用户看到完整看板 T+user 用户点击"让 AI 复盘今天" T+user runReview() → hasLlm() 检测 → 组装 dataSummary → POST /api/chat (NDJSON) T+user+ 后端 chat.py 调用户 LLM，逐 token 流式返回 T+user+ onDelta 逐块更新 review state → ReactMarkdown 增量渲染 T+done reviewLoading=false → SaveNoteButton 出现

暂停回忆：如果后端 /api/market/overview 返回了数据但 sentiment 字段为 null，前端市场情绪卡片会显示什么？请追踪 DailyReview.tsx 第 264 行的判断条件。

参考答案：{!sentiment?.breadth ? pending(ovDone) : ...}——只要 sentiment 为 null 或缺少 breadth 字段，就进入 pending(ovDone) 分支。此时 ovDone 已为 true（请求已结束），所以显示"暂无数据：可能是非交易时段或数据源暂时不可用"。
