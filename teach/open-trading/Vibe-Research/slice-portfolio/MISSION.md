# 使命：持仓管理全链路

## 为什么
我想理解 Vibe-Research 的持仓管理从前端录入到后端持久化、实时行情叠加浮动盈亏、再到已清仓记录的全链路设计。这不仅是 CRUD，而是一个本地优先、隐私保护、加权平均成本合并的轻量级投资台账系统。掌握这条链路后，我可以设计自己的交易记录工具。

## 成功的样子
- 能画出从前端 Portfolio.tsx 到后端 portfolio.py 的完整数据流时序图
- 能解释加权平均成本合并的逻辑和线程安全保护
- 能说出 /api/portfolio 五个端点（GET/POST/DELETE/close/refresh）的请求格式和响应结构
- 能识别异常路径：非法代码、零股数、后端宕机、行情源失败时的降级行为

## 约束条件
- 时间：15 分钟内完成主课程学习
- 已具备 L1-backend 模块总览基础
- 学习方式：阅读源码 + 链路图 + 速查表

## 不在范围内
- 行情数据源 astock 的实现细节（见 module-a-stock-data 主题）
- React 组件 UI 设计模式（见 module-frontend 主题）
- 后端鉴权中间件和 CORS 配置（见 module-backend 主题）
