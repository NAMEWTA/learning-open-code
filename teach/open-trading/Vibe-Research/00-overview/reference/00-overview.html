4. 核心设计哲学

1. 本地优先

持仓、自选股、上传的研报、API key 只存本地，不上传、不进 Git 仓库。 后端无外部数据库，数据持久化走本地 JSON 文件 + 文件系统。 用户对个人数据拥有完全自主控制权。 （证据：README.md:162、backend/portfolio.py 本地文件读写）

2. 数据只读

后端不预置标的、不推荐股票、不预测涨跌、不给买卖时机、不做主观评分。 连板股 / 成交额榜等均为客观公开榜单数据，只呈现事实。 所有端点只返回数据，不附带任何投资建议。 （证据：README.md:159-162、backend/app.py:4 "只读、无状态、按用户传入代码返回客观数据"）

3. AI 中立

产品不内置模型、不校准输出、无倾向。提供三条出口（订阅 CLI / API key / MCP）由用户自选。 投研分析框架只规定「怎么读数据」，所有分析方向由用户自己配置的 AI 给出。 （证据：README.md:63-65、README.md:126-147）

4. 渐进依赖

核心层（行情 + 研报 / 公告）秒装可用：仅需 fastapi + uvicorn + requests。 重依赖（akshare、mootdx、pandas）惰性导入，缺失时对应端点返回 HTTP 501 + 安装提示， 不拖垮整个服务。用户可按需安装、渐进增强。 （证据：backend/app.py 各端点中 except DependencyMissing: raise HTTPException(501)）

5. 隐私保护

无埋点、不收集用户行为数据。API key 只存浏览器 localStorage，随请求发送给用户自己的后端， 不经过第三方、不进仓库。可选鉴权 VR_API_KEY 环境变量保护公网部署。 持仓 / 关注股 / 研报文件均存在项目本地目录，用户完全掌控。 （证据：README.md:143、backend/app.py:44-45）
