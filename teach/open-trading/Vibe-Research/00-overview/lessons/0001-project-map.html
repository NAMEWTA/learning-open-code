Vibe-Research 项目全景架构与学习地图

学习目标：15 分钟内能说清 Vibe-Research 的定位、技术栈、架构分层和后续阅读路线。

1. 项目定位与核心场景

Vibe-Research 是一个开源的个人 AI 投研看板，主推 A 股，兼看美股和港股。它的核心问题不是"选什么股票"——而是把行情、研报、估值、财务、公告、资金面、资讯全部配齐，装进一个干净的看板，再留一个接口接入你自己的 AI。方向与结论，由你自己配置的模型/agent 决定。

依据 open-trading/Vibe-Research/README.md，它提供 9 个功能页面：每日复盘、资讯雷达、个股数据、自选股、板块中心、我的持仓、我的研报、研究记录、接入 AI。三条 AI 出口（订阅接入/API 接入/MCP）覆盖了从本机 CLI 一键调用到 agent 多步工具调用的全部场景。

4. 核心设计哲学与豁免清单

五条核心设计哲学：

本地优先 — 持仓、关注股、上传的研报、API key 只存本地，不上传、不进仓库。用户数据完全由自己掌控。

数据只读 — 后端只做客观数据整理与公开榜单呈现，不荐股、不预测涨跌、不给买卖时机、不做主观评分。UI 无买卖按钮。

AI 中立 — 不内置任何模型或建议倾向。三条 AI 出口（订阅接入/API 接入/MCP）支持用户自行选择模型，分析方向由你自己的 AI 决定。

渐进依赖 — 行情（腾讯）+ 研报（东财）秒装可用。akshare/mootdx 惰性导入，缺失时端点返回 501 而非崩溃。让用户按需安装，不强加重型依赖。

隐私保护 — 全站无埋点、不收集用户数据。API key 存浏览器 localStorage，随请求发给自己的后端，不上传。持仓数据只在本机。

源码文件豁免清单：L0 导览阶段豁免以下文件——docs/screenshots/*.png（产品截图）、a-stock-data/assets/*（赞助图）、frontend/public/favicon.svg（网站图标）、frontend/package-lock.json（自动生成）、frontend/node_modules/（第三方依赖）、backend/.venv/（虚拟环境）、backend/__pycache__/（字节码）、.gitignore（Git 配置）。这些文件要么是构建产物，要么是二进制静态资源，不决定主架构路径。

暂停回忆：如果 Vibe-Research 后端启动后，调用 /api/news 返回 501，最可能的原因是什么？这个设计的目的是什么？

参考答案：返回 501 说明 akshare 未安装。这是渐进依赖设计的体现——akshare 作为较重依赖被惰性导入，缺失时端点返回 501 + 安装提示而非崩溃，让行情和研报等核心功能不受影响。运行 pip install akshare 即可恢复。
