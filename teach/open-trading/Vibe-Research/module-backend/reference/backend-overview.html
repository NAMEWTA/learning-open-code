Vibe-Research L1 后端参考速查表

本参考页是「后端模块总览」课程的补充速查资料，按模块详解、API 端点分组、缓存策略、中间件链和依赖清单组织。 建议与 0001-backend-module-tour.html 配合使用。

1. 十模块详解

1.1 app.py — 应用入口与路由注册（602行）

FastAPI 应用定义，监听 :8900。职责：创建 FastAPI() 实例、注册 CORS 中间件和可选 API Key 鉴权中间件、 定义 30+ 路由函数、管理全站共享缓存字典。每个路由函数的通用模式：接收参数 → 校验 6 位代码 → 调用数据模块 → 包装为 {"data": ...} → 捕获异常转换为 HTTP 4xx/5xx。启动时调用 pf.start_scheduler(1800) 注册持仓后台刷新。

1.2 astock.py — A股全栈数据层

五层数据源封装，30+ 数据获取函数。核心设计：分级依赖——腾讯行情仅需标准库（永远可用）、 东财研报/公告仅需 requests（轻量必装）、akshare 和 mootdx 惰性导入（缺失时抛 DependencyMissing）。 关键函数：tencent_quote()（批量行情）、full_valuation()（完整估值）、 eastmoney_reports()（研报列表）、kline()（K线需 mootdx）、 stock_news()（新闻需 akshare）、margin_trading()（融资融券）、 dragon_tiger_board()（龙虎榜）、industry_comparison()（行业排名）。 所有东财请求通过 em_get() 统一发，内置 1s 限流间隔。

1.3 gstock.py — 美股/港股数据层

移植自 global-stock-data 的东财域内合规子集。核心设计：push2 优先、push2delay 降级—— 首次请求走 push2.eastmoney.com，掉连后 latch 到 push2delay.eastmoney.com（延时行情，投研够用）。 提供：全球指数快照（道指/标普/纳指/恒生/恒生科技）、美港股个股行情 + 关键财务指标、股票代码搜索。 市场代码映射：105=NASDAQ、106=NYSE、116=HK、177=韩国。所有请求复用 astock.em_get() 的直连策略。

1.4 market.py — 市场情绪与板块资金流

全站共享缓存（5分钟 TTL），数据源故障空结果不缓存（下次请求直接重试）。提供四大接口： get_overview()（市场总览：涨跌家数、大盘宽度、涨停跌停、题材投机）、 get_short_term_emotion()（短线情绪：连板梯队/最高连板/炸板率/封板率/晋级率）、 get_turnover_top()（全市场成交额榜 Top20，push2/push2delay 同款降级）、 get_global_indices()（全球指数快照，用于判断隔夜外围走势）。 市场情绪数据通过 akshare 惰性导入，缺失时降级返回空字典。

1.5 newsradar.py — 资讯雷达

移植自 investment-news，纯标准库 + 线程池，零 key、零个股字段。12 赛道 108 公开 RSS 源配置文件 news_sources.json。工作流：get_radar(force=False) 读本地 JSON 缓存（无缓存返回赛道骨架）、 fetch_radar() 线程池并发抓全部源（约 20-40s）→ 合规过滤（赌/预测市场/加密/色情关键词黑名单） → 按赛道分组、时间倒序 → 写缓存。AI 今日要点不在此模块——复用前端 /api/chat 由用户自己的模型提炼。

1.6 portfolio.py — 持仓管理

本地 JSON 持久化（.cache/portfolio.json，gitignore、不上传）。核心设计： 同代码多次录入按加权平均成本合并、原子写入（先写 .tmp 再 os.replace 防并发读到半截 JSON）、 threading.Lock 串行化。支持持仓增删、已清仓记录、手动刷新和后台 30 分钟定时刷新（start_scheduler()）。 实时盈亏通过行情叠加计算（红涨绿跌，A股口径）。

1.7 chat.py — AI 对话层

OpenAI 兼容 function-calling 循环，兼容豆包/DeepSeek/任意兼容端点。核心机制： 定义 7 个工具（TOOLS 列表：query_quote/query_valuation/query_reports/query_news/query_global_stock/ query_market_overview/query_short_term_emotion），每个工具对应 astock/gstock/market 的具体函数。 投研五维分析框架（估值/资金面/财报质量/行业景气/事件催化）焊进 SYSTEM_PROMPT。 最大 6 轮工具调用防死循环，单次工具结果注入上限 6000 字符（控 token）。 SSRF 防护：禁止私有 IP、禁止非 80/443 端口。

1.8 cli_runtime.py — 订阅 CLI 运行时

检测并 spawn 本机已安装的 AI CLI（Claude Code/Qwen/DeepSeek/Codex），用用户自己的订阅额度作答。 四种 CLI 三种提示词投递方式：Claude 走 system-file（系统提示词写临时文件）、Qwen/Codex 走 stdin、 DeepSeek 走 arg（位置参数）。子进程在临时目录执行、300s 超时兜底、纯文本输出。 不参与 function-calling——订阅接入只适合"数据已在 context 里"的场景。

1.9 mcp_server.py — MCP 服务

纯标准库 JSON-RPC over stdio，复用 chat.TOOLS 定义自动生成 MCP 工具列表。 支持 initialize、tools/list、tools/call 三个方法。 挂进 Claude Code：claude mcp add vibe-research -- python backend/mcp_server.py。 工具返回纯客观数据，判断由调用方 agent 给出。

1.10 myreports.py — 研报管理

用户上传研报，Base64 JSON 传输（不引入 python-multipart 依赖）、存本地 .cache/myreports/。 元数据存 index.json，threading.Lock 防并发覆盖。白名单扩展名校验（pdf/doc/txt/csv/xlsx 等）、 单文件 25MB 上限。文件名关键词自动打行业标签（9 大类：人形机器人/光互联/HBM/AI算力/半导体/新能源/创新药/商业航天/电力电网）， 未命中记"未分类"。

3. 缓存策略总览

缓存层级TTL适用范围实现方式设计依据
行情级 5 分钟 市场情绪、板块资金流、全球指数、行业排名、成交额榜 market.py 全站共享 _CACHE 字典 盘中变化快但 5 分钟内的抖动对投研无意义
公告级 15 分钟 个股公告、资金流、热门概念、互动易 app.py 端点级 _ANN_CACHE / _DC_CACHE 日级数据，但盘中可能有新公告
静态级 30 分钟 估值分位、财务摘要、融资融券、大宗交易、股东户数、分红、龙虎榜、解禁、板块归属 app.py 端点级 _PCT_CACHE / _FIN_CACHE / _DC_CACHE 季级静态数据，几乎不变，减少对东财限流请求

缓存键设计：行情级按功能 key（如 "overview"）缓存；个股级按 (endpoint, code) 二元组缓存， 确保不同代码的缓存互不干扰。market.py 还有一条特殊规则——数据源故障的空结果不缓存（valid() 判否）， 下次请求直接重试，避免把故障结果锁定 5 分钟。

4. 中间件链

每个 /api/* 请求经过两条中间件：

顺序中间件类型行为
1 CORS Starlette 内置 CORSMiddleware 默认全放行（allow_origins=["*"]）；公网部署时通过 VR_ALLOW_ORIGINS 收紧为白名单
2 可选鉴权 自定义 @app.middleware("http") 设了 VR_API_KEY 环境变量后，所有 /api/* 请求（除 OPTIONS 和 health）必须带 Authorization: Bearer <key>；未设则开放
