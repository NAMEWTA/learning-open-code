全球股票数据工具箱模块总览

学习目标：15 分钟内能说清 global-stock-data 的八层架构、市场覆盖范围和 Vibe-Research 后端的移植裁剪策略。

1. 工具箱定位

global-stock-data 是一个自包含的 AI 编程助手 Skill 文件——本质是一份结构化 Markdown + 内嵌 Python 代码。它把分散在 5 个数据源里的美股/港股原始数据整合成 18 个可直接调用的端点，零鉴权、仅依赖 requests。

它的核心价值不是"又一个数据封装库"，而是让 AI 编程助手（Claude Code、Codex、OpenClaw）在美股/港股相关对话中自动激活——你只需要说"看看 AAPL 的财报"，助手就能自动调用对应的数据端点。

一句话总结：global-stock-data 是 Vibe-Research 的全球股票数据引擎，负责美股、港股、韩股的行情查询和财务指标获取。它与 a-stock-data（A股数据引擎）形成姐妹关系，共同构成 Vibe-Research 的完整数据底座。

2. 八层数据架构

global-stock-data v1.0.1 按数据类型分为八层，共 18 个端点，覆盖从实时行情到 SEC 监管文件的完整数据栈。

全球股票数据 · 八层架构 · V1.0.1 │ ├─ Layer 1 · 行情层 新浪 + 腾讯 + 东财 push2 实时报价 25-78 字段 ├─ Layer 2 · K线层 新浪(回溯至1984) + Yahoo chart 日/周/月/分钟 K线 ├─ Layer 3 · 技术指标 MA/EMA+MACD+RSI+KDJ+布林带 纯Python计算，零额外依赖 ├─ Layer 4 · 基本面 东财datacenter+Yahoo+SEC XBRL 三表+关键指标+估值+机构持仓 ├─ Layer 5 · 资金面 东财 push2his 日级主力/大单/中单/小单资金流 ├─ Layer 6 · 期权层 Yahoo crumb 期权链 calls+puts（仅美股） ├─ Layer 7 · SEC Filing EDGAR submissions + XBRL 10-K/10-Q/8-K + 503 GAAP指标（仅美股） └─ Layer 8 · 工具层 东财search + 全市场列表 + CIK 搜索/排名/新闻/ticker映射

八层之中，行情层、基本面层、工具层是最常用的三层。行情层有 3 个数据源互为备份（新浪侧重中文名+EPS+PE，腾讯字段最全 78 个，东财 push2 统一 secid 查询）；基本面层提供中英双版财报（东财中文字段名 + Yahoo 英文结构化报表）；工具层提供全市场 5925+ 只美股和 18000+ 只港股的列表与搜索。

3. 从工具箱到后端：合规移植

Vibe-Research 的后端 backend/gstock.py（183 行）并没有把 SKILL.md 的 18 个端点全部搬进来。它只移植了东财域内合规子集——即不需要科学上网就能在国内服务器访问的数据源。

open-trading/Vibe-Research/ ├── global-stock-data/ │ └── SKILL.md ← 完整八层 18 端点（含 Yahoo/SEC 境外源） └── backend/ └── gstock.py ← 移植子集：东财域内（行情 + 关键指标 + 搜索 + 指数）

移植策略很清晰：

SKILL.md 原始能力gstock.py 移植状态原因
东财 push2 行情（美股/港股）已移植国内直连可用
东财 datacenter 关键财务指标已移植（最新一期）国内直连，中文字段名
东财 search 代码搜索已移植（增强：精确代码匹配）解决"搜 AAPL 混入票据"问题
全球指数（道指/标普/纳指/恒生）已移植（硬编码 5 个指数 secid）A 股看隔夜外围脸色的刚需
新浪/腾讯行情未移植东财 push2 已覆盖，避免冗余
新浪/Yahoo K线未移植前端暂不需要独立 K 线图
技术指标计算未移植前端自己算或后续加
Yahoo 期权/机构持仓/SEC Filing未移植需科学上网，非国内部署刚需

此外，gstock.py 还有一个原工具包未覆盖的扩展：增加了韩股（Kospi/Kosdaq）的行情支持，通过东财 MktNum=177 接入三星、SK 海力士等半导体龙头的实时报价。韩股代码为 6 位数字（如 005930.KS），需显式加 .KS/.KQ 后缀来与 A 股 6 位代码区分。

4. 市场代码规则与调用速览

调用 global-stock-data 端点时，不同市场和不同数据源需要不同的代码格式。核心规则如下：

市场东财 secid 前缀东财 SECUCODEYahoo 格式
美股 NASDAQ105.AAPLAAPL.OAAPL
美股 NYSE106.BABABABA.NBABA
港股116.0070000700.HK0700.HK
韩股177.005930—（无 F10 财务）—

在 Vibe-Research 后端中，调用入口统一为 gstock.us_hk_stock(query)，它会自动解析代码前缀、搜索东财、拉取行情和关键指标：

# 示例 1：查苹果美股 — 自动识别 NASDAQ >>> gstock.us_hk_stock("AAPL") { "code": "AAPL", "name": "苹果", "market": "NASDAQ", "quote": {"price": 198.75, "change_pct": 1.23, ...}, "metrics": {"revenue": 383285000000, "eps": 6.43, "roe": 158.7, ...} } # 示例 2：查腾讯港股 — 数字代码自动补零到 5 位 >>> gstock.us_hk_stock("700") # "700" → "00700" {"code": "00700", "name": "腾讯控股", "market": "HK", ...} # 示例 3：查三星韩股 — 显式 .KS 后缀区分 A 股 >>> gstock.us_hk_stock("005930.KS") {"code": "005930", "name": "三星电子", "market": "KR", ...}

注意韩股没有 F10 财务数据（metrics 为 null），东财对韩股仅提供行情。

暂停回忆：为什么 global-stock-data SKILL.md 有 18 个端点，但 gstock.py 只移植了行情 + 关键指标 + 搜索 + 全球指数？未被移植的端点在什么场景下才需要？

参考答案：移植策略是东财域内合规子集——只选国内直连可用的数据源（东财 push2/datacenter/search）。Yahoo 期权/机构持仓/SEC Filing 需要科学上网，不适合国内服务器部署；新浪/腾讯行情与东财功能重叠，避免冗余；K 线和技术指标不属于当前前端页面的刚需。如果你想在自己的项目中使用这些功能，可以直接调用 SKILL.md 中的 Python 代码段——它们不依赖后端，独立可用。
