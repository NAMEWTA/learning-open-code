1. 模块职责与架构位置

前端是 Vibe-Research 的用户界面层，负责数据展示、用户交互和 AI 对话入口。它本身不存储业务数据、不直接请求外部数据源——所有行情、研报、资讯数据通过后端 API 代理获取，用户配置（AI key、自选股、研究记录）仅存浏览器 localStorage。

3. 数据流通的三种路径

路径一：API 请求（前端 → 后端 → 数据源）

所有行情、估值、研报等数据通过 lib/api.ts 封装的 request() 函数发出。该函数自动完成三件事：注入 Authorization 鉴权头（如果用户在设置页填写了后端访问密钥）、将 HTTP 错误转为 ApiError 供页面优雅降级、通用 /api 前缀由 Vite 代理转发到后端 :8900。示例：

// api.ts 基础请求封装 async function request<T>(path, method="GET", body?) { const headers = { ...authHeaders() }; const resp = await fetch(`/api${path}`, { method, headers, body }); if (!resp.ok) throw new ApiError(…); return (await resp.json()).data as T; } // 使用示例 const indices = await api.indices(); // GET /api/indices const val = await api.valuation("600519"); // GET /api/valuation?code=600519

路径二：AI 流式对话（前端 → 后端 → AI 模型）

lib/llm.ts 提供 chatStream() 函数。它从 localStorage 读取用户配置的 AI 模型信息，将消息 + 上下文 POST 到 /api/chat（后端转发到 OpenAI 兼容接口），逐行解析 NDJSON 事件流，通过 onDelta 回调逐字吐字、onTool 回调展示工具调用。支持 AbortController 中止——用户关闭 AI 面板时不会浪费 API 额度。

路径三：本地持久化（前端 → localStorage）

用户私有数据全部存浏览器 localStorage，不上传、不进仓库。三类数据各有独立工具模块：lib/watchlist.ts（自选股代码数组，支持从任意文本中自动提取 6 位代码）、lib/notes.ts（研究记录，按类型/标题/内容/时间戳结构化存储，最多 200 条）、Settings 页面中的 AI key 与后端访问密钥（通过 loadAccessKey() / loadLlm() 读取）。

对比记忆：行情数据走 API（有后端缓存 5-30min），AI 对话走流式（SSE NDJSON），用户配置走 localStorage（永不过期）。三路径互不干扰——后端挂了行情看不到了，但自选股和研究记录不会丢。

主题系统通过 index.css 中的 HSL CSS 变量实现暗色（默认）和亮色两种外观。:root 定义暗色变量（深蓝黑底 + 暖橙 #F35D2B 主色），.light 类覆盖为亮色变量。useDarkMode() Hook 通过 document.documentElement.classList.toggle("light", !dark) 切换，用户选择存入 localStorage。

暂停回忆：如果用户说"我浏览器里存的自选股、API key 和研究记录怎么能不丢"，你该怎么解释 Vibe-Research 的本地持久化策略？这三类数据分别由哪三个 lib 工具模块管理？

参考答案：所有用户私有数据通过 localStorage 存储浏览器本地，不上传、不进仓库。自选股由 lib/watchlist.ts 管理（纯 6 位代码数组），研究记录由 lib/notes.ts 管理（结构化 Note 对象数组，最多 200 条），AI key 和后端访问密钥由 Settings 页面通过 loadLlm() / loadAccessKey() 读写。只要不清浏览器数据，这些信息不会因后端重启或代码更新而丢失。
