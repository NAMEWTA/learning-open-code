# 使命：A股数据工具箱 (a-stock-data) 模块总览

## 为什么
学习者需要理解 a-stock-data 作为 Vibe-Research 的 A 股数据引擎的核心价值——一个自包含、零第三方数据封装依赖的十层数据架构，覆盖 40 个端点、13 个数据源。掌握这套工具箱的架构后，无论是直接调用 SKILL.md 中的代码片段，还是阅读 backend/astock.py 的移植实现，都能快速定位到正确的数据层和端点。

## 成功的样子
- 能用一句话说明 a-stock-data 的定位：A股全栈数据工具包，十层架构、40 端点、零第三方封装依赖。
- 能画出十层数据架构的分层图：行情/研报/信号/资金面/新闻/基础数据/公告/打板/ETF期权/舆情互动。
- 能解释数据源优先级原则：通达信(mootdx)/腾讯不封IP优先，东财仅用于独有数据且必须走 em_get() 限流。
- 能根据场景快速定位到正确的数据层和端点（如"想看龙虎榜"→信号层→dragon_tiger_board）。
- 能理解 a-stock-data 与 backend/astock.py 的移植关系（SKILL.md → astock.py 的代码溯源）。
- 能手写出一个完整的端点调用示例（如拉取个股估值：腾讯行情 + 同花顺一致预期）。

## 约束条件
- 本主题是 L1 模块总览，不展开每个端点的完整参数说明和异常处理。
- 每节 lesson 保持 15 分钟内完成；端点清单、数据源速查分流到 reference。
- 教学产物只写入 `teach/open-trading/Vibe-Research/module-a-stock-data/`。
- 批量生成模式，MISSION 使用自动生成的默认使命，不交互等待用户输入。

## 不在范围内
- 不展开每一层所有端点的完整 Python 代码（代码在 SKILL.md 中，本课教"怎么找、怎么用"）。
- 不深入东财防封的底层 TCP/HTTP 细节（已在 SKILL.md 的"数据源优先级 & 东财防封"章节详细说明）。
- 不覆盖 V3.3 新增的打板层、ETF期权层、舆情互动层中每个端点的逐个实测（L2 模块再展开）。
- 不分析 backend/astock.py 的内部实现逻辑（属于 L2 backend 模块）。
