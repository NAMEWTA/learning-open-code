5. TradingAgentsGraph 生命周期

阶段方法操作
初始化__init__()创建双 LLM → 创建 ToolNode 字典 → 初始化 5 个组件 → compile graph
运行准备prepare_graph_run()解析 pending 反思 → 按需创建 SqliteSaver → 查询 checkpoint → 构建初始状态
图执行_run_graph() → graph.invoke()LangGraph 按节点-边拓扑执行，debug 模式逐 chunk 打印
运行收尾finalize_graph_run()状态落盘 JSON → 存 pending 决策 → 清 checkpoint → 提取信号
资源释放close_graph_run()关闭 SqliteSaver context → graph 恢复为无 checkpointer 编译

LLM参数字段使用节点调用次数（单次分析）选择理由

Provider 适配：_get_provider_kwargs() 根据 llm_provider 设置不同参数——Google 设 thinking_level，OpenAI 设 reasoning_effort，Anthropic 设 effort——确保深度/快速语义正确映射到各厂商的 thinking 控制字段。

7. Checkpointer API

函数签名返回值说明
get_checkpointer(data_dir, ticker) -> ContextManager[SqliteSaver]上下文管理器打开 {TICKER}.db，创建 SqliteSaver，调用 setup()
checkpoint_step(data_dir, ticker, date) -> int | None步骤编号或 None查询最新 checkpoint 的 step 号
has_checkpoint(data_dir, ticker, date) -> bool布尔值是否存在可恢复的断点
thread_id(ticker, date) -> strSHA256[:16]ticker+date 的确定性 thread_id
clear_checkpoint(data_dir, ticker, date) -> None—删除 ticker+date 在 writes 和 checkpoints 表中的行
clear_all_checkpoints(data_dir) -> int删除的文件数删除 checkpoints/ 下所有 .db 文件
