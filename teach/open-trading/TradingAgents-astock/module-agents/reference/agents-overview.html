4. 结构化输出 Schema

定义在 schemas.py，使用 Pydantic BaseModel。3 个 Schema 对应 3 个决策型 Agent：

Schema使用方字段评级枚举
ResearchPlanResearch Managerrecommendation (PortfolioRating), rationale (str), strategic_actions (str)Buy / Overweight / Hold / Underweight / Sell
TraderProposalTraderaction (TraderAction), reasoning (str), entry_price (float?), stop_loss (float?), position_sizing (str?)Buy / Hold / Sell
PortfolioDecisionPortfolio Managerrating (PortfolioRating), executive_summary (str), investment_thesis (str), price_target (float?), time_horizon (str?)Buy / Overweight / Hold / Underweight / Sell

降级机制：每个 Schema 都配有 render_xxx() 函数将 Pydantic 实例转回 markdown。当 LLM 供应商不支持结构化输出（如某些 Ollama 模型），invoke_structured_or_freetext() 会自动降级为自由文本生成，保证系统在任何 LLM 上都能运行。

5. 质量门控节点

quality_gate.py 位于 7 个 Analyst 完成之后、Bull Researcher 之前。两层检查：

  1. 硬检查（代码层）：报告长度 >= 200 字符、无失败标记（"无法获取"等）、检测表格和数据缺失，输出 A-F 评级
  2. LLM 复审：仅当硬检查失败数 < 4 时触发，将 7 份报告（每份截断至 3000 字符）发给 LLM 做综合质量评估

质量门控的输出 data_quality_summary 被 Bull/Bear Researcher 读取——当某份报告评级为 C/D/F 时，辩论 Agent 应降低对该报告的依赖度。
