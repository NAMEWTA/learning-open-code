2. 命令参考

命令选项说明
tradingagents analyze 无 启动完整 8 步交互式分析流程（默认）
tradingagents analyze --checkpoint --checkpoint 启用断点续跑：每次节点完成后保存状态，崩溃后可恢复
tradingagents analyze --clear-checkpoints --clear-checkpoints 清除所有已保存的检查点，强制全新运行
注意：Typer App 注册了 shell 自动补全（add_completion=True），首次运行后可用 tradingagents --install-completion 安装。

步骤 7：双 LLM 引擎（Quick Think + Deep Think）

分别选择浅层思考模型（快速响应，用于简单任务）和深层思考模型（深度推理，用于分析/辩论）。模型列表从 tradingagents.llm_clients.model_catalog.get_model_options(provider, mode) 按 provider + mode 动态获取。OpenRouter 模式下额外支持实时拉取最新模型列表（_fetch_openrouter_models()）。

4. DEFAULT_CONFIG 配置项速查

配置项类型默认值说明
llm_providerstr"openai"LLM 提供商 key，CLI 运行时由步骤 6 覆盖
deep_think_llmstr"gpt-5.4"深层思考模型，CLI 运行时由步骤 7 覆盖
quick_think_llmstr"gpt-5.4-mini"浅层思考模型，CLI 运行时由步骤 7 覆盖
backend_urlstr or NoneNoneAPI 端点，None 时各 provider client 使用默认值
max_debate_roundsint1Bull/Bear 辩论最大轮次，CLI 运行时由步骤 5 覆盖
max_risk_discuss_roundsint1三方风控讨论最大轮次，CLI 运行时由步骤 5 覆盖
results_dirstr~/.tradingagents/logs分析结果存储目录，可用环境变量 TRADINGAGENTS_RESULTS_DIR 覆盖
data_cache_dirstr~/.tradingagents/cache数据缓存目录，可用环境变量 TRADINGAGENTS_CACHE_DIR 覆盖
output_languagestr"Chinese"报告输出语言，CLI 运行时由步骤 3 覆盖
checkpoint_enabledboolFalse断点续跑开关，由 --checkpoint 选项控制
