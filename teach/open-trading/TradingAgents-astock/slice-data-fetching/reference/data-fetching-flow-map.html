3. config 配置项速查

配置路径类型默认值说明
data_vendors.{category}str"a_stock"按类别指定首选 vendor
tool_vendors.{method}str—按方法指定 vendor（覆盖类别级）
data_cache_dirstr~/.tradingagents/cacheOHLCV 缓存目录
EM_MIN_INTERVALenv"1.0"东财请求最小间隔（秒）

切换 vendor 示例：

# 全局切换 K线数据源到 yfinance
config["data_vendors"]["core_stock_apis"] = "yfinance"

# 仅 get_news 使用 alpha_vantage
config["tool_vendors"]["get_news"] = "alpha_vantage"

# 多 vendor fallback（逗号分隔，按序尝试）
config["data_vendors"]["core_stock_apis"] = "a_stock,alpha_vantage"

4. 异常返回与根因速查

返回/异常信息根因排查方向
K线数据获取失败：mootdx和新浪备用源均不可用 TCP 7709 不可达 + 新浪 HTTP 也失败 检查网络/防火墙/代理
无法通过 mootdx 解析股票名称（通达信服务暂时不可达） mootdx 不通 换用 6 位代码输入
找不到股票 'xxx' 中文名不存在于映射表 检查名称正确性，改用代码
'xxx' 匹配到多只股票 模糊匹配命中多条 输入完整名称或代码
No available vendor for 'get_xxx' 所有 vendor 均失败 检查网络 + config 配置
东财接口返回空 / ConnectionError IP 被封或风控触发 调大 EM_MIN_INTERVAL，等待解封
No data found for A-stock 'xxx' between ... 日期区间无交易数据 正常——非交易日/停牌期间

5. 中文 ticker 解析全链路

用户/LLM 输入 "宝光股份"
  │
  ▼
safe_ticker_component("宝光股份")          ← utils.py:20-47
  │  _HAS_CHINESE_RE.search("宝光股份")    ← 检测到中文
  │  → 调用 resolve_ticker("宝光股份")
  │
  ▼
resolve_ticker("宝光股份")                ← a_stock.py:115-144
  │  _build_name_code_map()               ← a_stock.py:79-112
  │    │  _get_mootdx_client()             ← TCP :7709
  │    │  client.stocks(market=0)          ← 深市全量
  │    │  client.stocks(market=1)          ← 沪市全量
  │    │  → n2c["宝光股份"] = "600379"     ← 精确匹配命中
  │    │  → c2n["600379"] = "宝光股份"
  │    │  缓存至模块级全局变量
  │  → 返回 "600379"
  │
  ▼
safe_ticker_component 接收 "600379"
  │  _TICKER_PATH_RE.fullmatch("600379")   ← 校验通过
  │  → 返回 "600379"
  │
  ▼
后续所有 get_* 方法使用纯代码调用

边界情况：

  • LLM 返回带交易所前缀的代码（SH688017）→ _normalize_ticker 去前缀
  • LLM 返回带后缀的代码（688017.SH）→ _normalize_ticker 去后缀
  • LLM 返回全角数字 → _normalize_ticker 不做处理（可能解析失败）
  • 用户输入空字符串 → ValueError: 输入不能为空
