2. 架构决策记录

决策选择理由
Agent 编排框架LangGraph StateGraph原生支持多节点条件路由、状态持久化、checkpoint 断点续跑
LLM 调用模式双 LLM（quick + deep）成本优化：30-50 次调用中仅 2 次用深度模型
LLM 供应商扩展工厂模式 + lazy import新增供应商只需添加客户端类 + 注册到工厂，不影响已有代码
数据供应商扩展VENDOR_METHODS 路由表每个方法注册多个 vendor 实现，config 切换，fallback 链自动降级
结构化输出Pydantic BaseModelResearch Manager/Trader/Portfolio Manager 用 JSON Schema 约束输出格式
输出语言分层策略用户报告中文，内部辩论英文——保证推理质量
东财接口防封串行限流 + 随机抖动_em_get() 统一节流入口，间隔 >=1s + 0.1~0.5s 抖动 + Keep-Alive 复用
断点续跑SQLite CheckpointerLangGraph checkpoint 机制，崩溃后同名 ticker+date 自动恢复
交易记忆Markdown 日志 + 批量更新pending → resolved 两阶段，下一轮同 ticker 运行时批量回填 alpha 反思

东财防封限流规则（v0.2.11+）：所有东财请求统一走 _em_get() 节流入口——串行限流默认间隔 >=1s + 0.1~0.5s 随机抖动 + Keep-Alive 会话复用。仅东财限流，mootdx/腾讯/新浪/同花顺/财联社/百度不受影响。可通过环境变量 EM_MIN_INTERVAL=1.5 进一步降速。
