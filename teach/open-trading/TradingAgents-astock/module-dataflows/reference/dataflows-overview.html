5. VENDOR_LIST 与路由决策表

3 个 vendor：

Vendor面向市场数据获取方式需要 API Key
a_stockA 股（沪深京）7 个免费直连源否
yfinance美股 / 全球Yahoo Finance API否
alpha_vantage美股 / 全球Alpha Vantage API是（免费层 25 次/天）

路由决策流程：

  1. 调用方调用 route_to_vendor("get_fundamentals", ticker="600379", ...)
  2. 查 TOOLS_CATEGORIES 确定类别 = "fundamental_data"
  3. 查 get_vendor("fundamental_data", "get_fundamentals")：先查 tool_vendors 精确匹配 → 再查 data_vendors 类别匹配 → 默认 "a_stock"
  4. 构建 fallback 链：首选 + 其余可用 vendor（去重）
  5. 遍历 fallback 链调用实现函数，仅 AlphaVantageRateLimitError 触发降级

6. 两条关键链路流程图

中文 ticker 解析链路：

用户输入 "宝光股份" → safe_ticker_component() 检测中文 → resolve_ticker("宝光股份") → _build_name_code_map() → _get_mootdx_client() — 三级 fallback 建连 → TCP 探测 10 个备选服务器 → bestip 测速 fallback → 裸 factory fallback → client.stocks(market=0) + client.stocks(market=1) → 构建 n2c / c2n 字典（缓存至模块全局变量） → 精确匹配 "宝光股份" → "600379" → 返回 "600379"

东财 API 请求链路：

get_fundamentals("600379") → _tencent_quote() — 腾讯 qt.gtimg.cn (非东财，不走限流) → client.finance() — mootdx TCP (非东财) → _em_get(push2.eastmoney.com) — 东财 push2 → 计算距上次请求间隔 → sleep(min_interval + 随机抖动) 若不足 → _EM_SESSION.get() 复用 TCP 连接 → 更新 _em_last_call 时间戳 → _ths_eps_forecast() — 同花顺 HTML 抓取 (非东财)
