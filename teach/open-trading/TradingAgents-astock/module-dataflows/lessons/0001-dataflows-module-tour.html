0001 — 数据获取层模块总览

学习目标：能说出 dataflows 模块的 5 大数据类别、3 种 vendor 策略、7 个数据源及其协议，以及东财防封限流和中文 ticker 解析两条关键链路的实现要点。

1. 模块定位与架构位置

tradingagents/dataflows/ 是整个 TradingAgents-Astock 的数据基座。它位于 Agent 层和 Graph 层之下，负责从 7 个免费数据源拉取原始行情、基本面、新闻、资金流等数据，以结构化文本格式（CSV / Markdown 表格）返回给上层的 7 个 Analyst Agent 使用。

┌─────────────────────────────────┐ │ Agent 层（7 Analyst + 辩论） │ ├─────────────────────────────────┤ │ Graph 层（LangGraph 工作流） │ ├─────────────────────────────────┤ │ dataflows 层 ◀── 本节重点 │ ├─────────────────────────────────┤ │ 7 数据源（HTTP/TCP 直连） │ └─────────────────────────────────┘

模块通过 interface.py 向上层暴露 17 个数据方法（get_stock_data、get_indicators、get_fundamentals 等），归类为 5 大类别：core_stock_apis、technical_indicators、fundamental_data、news_data、signal_data。signal_data 是 A 股专属层（龙虎榜、资金流、北向资金、解禁日历等 8 个方法），yfinance 和 alpha_vantage 无对应实现。

2. Vendor 路由模式

核心设计：策略模式 + 字典路由 + 降级链。调用方只需指定方法名，不感知底层 vendor。

# VENDOR_METHODS 路由注册表（interface.py，节选） VENDOR_METHODS = { "get_stock_data": { "a_stock": get_astock_stock_data, # mootdx TCP "alpha_vantage": get_alpha_vantage_stock, "yfinance": get_YFin_data_online, }, "get_fundamentals": { "a_stock": get_astock_fundamentals, # 腾讯+东财+同花顺 ... }, # signal_data — A 股专属，无 yfinance/AV 实现 "get_dragon_tiger_board": { "a_stock": get_astock_dragon_tiger_board, }, "get_northbound_flow": { "a_stock": get_astock_northbound_flow, }, ... }

route_to_vendor(method, *args) 的执行流程：查 TOOLS_CATEGORIES 确定类别 → 查 config 获取首选 vendor（支持逗号分隔的主备链）→ 构建 fallback 链（首选在前，其余在后）→ 逐个尝试，仅在 AlphaVantageRateLimitError 时触发降级。

切换数据源只需改 config（如 data_vendors.core_stock_apis = "alpha_vantage"），无需修改任何调用方代码。新增 vendor 只需在 VENDOR_METHODS 中注册实现函数即可。

3. 7 数据源全景

数据源协议提供数据故障降级
mootdx TCP :7709 OHLCV K 线、财务快照、F10 股东研究、全市场股票名-代码映射 → 新浪 HTTP K 线
腾讯财经 HTTP (GBK) 实时报价、PE/PB、市值、换手率、涨跌停价 无独立降级
东方财富 HTTP（走 _em_get） 个股信息、龙虎榜、限售解禁、资金流、板块行情、滚动新闻 → 新浪新闻
新浪财经 HTTP K 线 fallback、财报三表（资产负债表/利润表/现金流量表）、个股新闻 fallback —（自身是 fallback）
同花顺 HTTP (GBK) EPS 一致预期、热股题材（人工标注涨停原因）、北向资金实时流向 无独立降级
财联社 HTTP 全球财经快讯（telegraphList），与东财 7x24 互为补充 → 东财全球资讯
百度股市通 HTTP 概念板块归属（申万行业、概念主题、地域分类） 无独立降级

关键依赖：仅 mootdx 需要第三方库（mootdx，TCP 连接），其余 6 个均为标准库 urllib + requests 直连 HTTP，零外部数据服务依赖。

4. 两个关键机制

东财防封限流 — _em_get()：东财对高频请求有严格风控（每秒 >5 次 / 单 IP 并发 ≥10 / 1 分钟 ≥200 次 → 临时封 IP）。所有 eastmoney.com 请求强制走 _em_get() 统一入口：

# a_stock.py — 东财防封限流核心 _EM_SESSION = _requests.Session() # 复用 TCP 连接（Keep-Alive） _EM_MIN_INTERVAL = float(os.environ.get( "EM_MIN_INTERVAL", "1.0")) # 可通过环境变量调大降速 _em_last_call = [0.0] # 模块级上次请求时间戳 def _em_get(url, params=None, headers=None, timeout=15, **kwargs): wait = _EM_MIN_INTERVAL - (time.time() - _em_last_call[0]) if wait > 0: time.sleep(wait + random.uniform(0.1, 0.5)) # 串行限流+随机抖动 try: return _EM_SESSION.get(url, params=params, ...) finally: _em_last_call[0] = time.time()

覆盖 push2 / push2his / datacenter-web / search-api / np-weblist 共 7 个调用点。mootdx (TCP) / 腾讯 / 新浪 / 同花顺 / 财联社 / 百度等非东财源不走此限流。

中文 ticker 解析链路：用户或 LLM 输入中文股票名（如"宝光股份"）→ safe_ticker_component() 检测中文字符 → 调用 resolve_ticker() → _build_name_code_map() 通过 mootdx 拉取沪深两市全量股票列表（缓存至模块级变量）→ 精确匹配/模糊匹配 → 返回 6 位代码 "600379"。
