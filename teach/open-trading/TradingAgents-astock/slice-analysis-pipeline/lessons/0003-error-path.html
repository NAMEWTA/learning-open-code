0003 — 异常与边界路径

目标任务：能说出全链路中 4 类异常场景的触发条件和降级逻辑，并能判断在什么情况下链路会"静默继续"而非报错中断。

前两节沿主成功路径追踪，本节专门覆盖异常和边界场景。TradingAgents-Astock 的设计哲学是"尽力产出，不因局部失败阻塞链路"——理解这一点对后续调试至关重要。

1. 数据获取失败：工具返回空数据

最频繁的异常场景。当某个 Analyst 调用工具（如 get_stock_data、get_fundamentals）返回空结果或错误时，不会抛出异常——工具将错误信息作为消息内容返回。LLM 在后续分析中可能会在报告中写入 [数据缺失] 标记，但 Analyst 节点本身仍然正常退出。

这意味着：即使数据源（如东财接口）临时不可用，链路会继续跑完——只是最终报告质量受影响，体现在质量门控的评分中。

2. 质量门控不通过：跳过 LLM 复审

质量门控（quality_gate.py:150）有一个关键分支：

llm_review = ""
if fail_count < 4:
    try:
        response = llm.invoke(review_prompt)
        llm_review = response.content
    except Exception as e:
        llm_review = f"（LLM 复审失败: {type(e).__name__}: {e}）"
# 否则 llm_review 保持为 ""，summary 中显示"跳过"
边界条件：fail_count >= 4 当 4 份或更多研报未通过硬检查（评级 D 或 F），LLM 复审被直接跳过。这是合理的：如果多数数据都不可用，让 LLM 再审一遍没有意义。

注意：质量门控不通过不会阻断链路。它只写入 data_quality_summary 到 state，但下游 Bull Researcher 仍然正常启动——辩论代理会看到质量低下的报告，但决策过程照常进行。这是一个设计选择：宁可基于不完美数据给出低信心决策，也不在管线中途抛出异常。

3. 分析师未选中：空节点不会创建

setup_graph()（setup.py:29）接收 selected_analysts 参数。如果用户只选了 ["market", "fundamentals"]，那么：

  • 只有 Market 和 Fundamentals 的节点/工具/删除节点被创建
  • 未选中的 Analyst 的对应报告字段（如 sentiment_report）在初始状态中仍存在（为空字符串），后续节点读取它们时发现为空
  • 质量门控中，未运行的 Analyst 报告为空字符串 → 硬检查评级 F（"报告为空"）

边界情况：如果 selected_analysts 为空列表，setup_graph() 直接 raise ValueError——这是整个链路中少数几个会硬中断的检查点。

4. 异常和降级策略总览

场景触发条件行为是否阻断
工具返回空/错 数据源不可用 工具返回错误消息，Agent 继续分析 否
质量门控 LLM 复审失败 LLM 调用异常 捕获异常，写入失败原因，data_quality_summary 仍正常产出 否
质量门控跳过 LLM 复审 4+ 报告评级 D/F 仅保留硬检查结果 否
分析师列表为空 用户传入空列表 raise ValueError 硬中断 是（启动前校验）
延迟反思收益获取失败 yfinance 数据未就绪 返回 None, None, None，跳过该条日志，下次重试 否
checkpoint 恢复 上次运行崩溃 检测已有 checkpoint，直接从上次成功节点继续 否（恢复执行）

判断题

判断以下说法的正误：

  1. "如果市场分析师的数据全部获取失败，链路会在 Market Analyst 节点抛异常并终止。"
  2. "质量门控中，只要有任意一份报告不通过硬检查，LLM 复审就会被跳过。"
  3. "如果用户只选了 2 个 Analyst，质量门控会检测到其余 5 份报告为空并给出 F 评级。"

参考答案：(1) 错误。工具将错误以消息形式返回，Market Analyst 仍然会产出一份可能质量低下的报告，链路不会中断。(2) 错误。只有 fail_count >= 4（即 4 份以上 D/F）时才跳过 LLM 复审。(3) 正确。未运行的 Analyst 的报告字段保持初始化的空字符串，硬检查会评级为 F。
