0001 — 分析决策全链路地图

目标任务：不看源码，凭记忆画出 TradingAgents-Astock 从 propagate("600519", date) 到最终 Buy/Hold/Sell 的 7 阶段节点拓扑图，并说出每阶段的核心职责。

1. 入口：一条主方法承载全链路

一切始于 TradingAgentsGraph.propagate(company_name, trade_date)（trading_graph.py:301）。它调用 _run_graph()，后者执行三段式生命周期：

  1. prepare_graph_run() — 解析待处理日志 → 创建初始状态 → 注入 thread_id
  2. graph.invoke(init_state) — LangGraph 执行整个状态图（7 阶段串联）
  3. finalize_graph_run() — 写结果日志 → 丢进 memory_log → 清 checkpoint → process_signal() 提取评级
关键点：propagate() 本身只是一个委托方法。真正的链路编排在 GraphSetup.setup_graph() 中定义的 LangGraph StateGraph，LangGraph 负责按边和条件路由依次调用节点。

3. 数据如何在各阶段之间流转

所有状态存储在 AgentState（一个 TypedDict，定义在 agent_states.py）中，LangGraph 的每个节点返回一个 dict 局部更新，框架自动合并。核心字段表见 长链路速查，这里关注三条主线：

  • 报告流：阶段 2 的每个 Analyst 将其研报写入 market_report / sentiment_report / ... / lockup_report。阶段 3 的质量门控读取这 7 个字段，阶段 4-7 的辩论/决策代理通过 state["market_report"] 等方式引用它们。
  • 辩论流：阶段 4 的 Bull/Bear 轮次写入 investment_debate_state["history"]，阶段 7 的三方风险辩论写入 risk_debate_state["history"]。Manager 代理读取这些历史作为决策依据。
  • 决策流：阶段 5→6→7 的决策层层递进——Research Plan → Trader Proposal → Portfolio Decision，每个下游节点读取上游的结构化输出。

4. 条件路由：链路在哪里分叉

三个条件路由点决定了链路走向（定义在 conditional_logic.py 和 setup.py）：

路由点条件判断分支
每个 Analyst 后 last_message.tool_calls 是否有 tool_call？ 有 → tools_analyst（取数据再回到 analyst）；无 → Msg Clear（进入下一个 analyst）
Bull/Bear 辩论 debate_state["count"] >= 2*max_debate_rounds？ 是 → Research Manager；否 → 继续辩论（轮流发言）
风险三方辩论 risk_debate_state["count"] >= 3*max_risk_discuss_rounds？ 是 → Portfolio Manager；否 → 继续轮转（A/C/N 循环）
