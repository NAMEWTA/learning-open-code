# 使命：东财防封限流机制深度剖析

## 为什么
用户需要在自己的量化脚本中可靠地调用东方财富数据接口（龙虎榜、研报、资金流等），但东财有严格的风控策略，高频或并发请求会触发临时封IP。理解 `em_get()` 的限流设计原理后，用户能定制自己的风控策略，避免被封，并在被封后知道如何恢复。

## 成功的样子
- 能解释 em_get() 中每条限流措施的动机和权衡——为什么选串行而非令牌桶、为什么最小间隔设 1s
- 能根据自身网络环境和数据需求调整限流参数（调大 EM_MIN_INTERVAL、调整重试策略）
- 遇到 HTTP 000/403/429 时能判断是风控信号还是网络问题，并采取正确应对

## 约束条件
- 学习时间：30 分钟以内（2 节短课）
- 前提：已理解 L1 基础设施层中 em_get() 的基本用法和东财数据源优先级
- 不涉及东财风控的反向工程细节

## 不在范围内
- 东财数据接口的具体业务字段解析（由各模块课程覆盖）
- 代理池 / IP 轮换方案（属于分布式采集优化，超出本课程范围）
- HTTP 协议层面的深入分析（TCP 握手、TLS 细节等）
