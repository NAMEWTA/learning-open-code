板块归属与资金流向

L1 信号层 第 2/3 节 估计时长：12 分钟

1. 东财板块归属 — 一键查全个股概念

eastmoney_concept_blocks(code) 通过东财 push2 的 slist 接口，一次请求拿到个股所属的全部板块——包括行业、概念、地域，混合在一个列表中返回。每个板块带 BK 代码、当日涨跌幅、板块龙头股。

版本背景： V3.2.2 替换了已失效的百度 PAE getrelatedblock（返回 ResultCode 10003）。东财 slist 通过 spt=3 参数一次拿全，零鉴权，已内置 em_get() 限流。 
blocks = eastmoney_concept_blocks("600519")  # 贵州茅台
print(f"共 {blocks['total']} 个板块")
print(blocks["concept_tags"])
# → ['食品饮料','白酒Ⅲ','白酒Ⅱ','贵州板块','酿酒概念','HS300_',...]

返回结构： {total, boards, concept_tags}。boards 列表每项含 name（板块名）、code（BK 码）、change_pct（涨跌幅）、lead_stock（龙头股）。concept_tags 是所有板块名的便捷列表，可直接用于题材归因。

注意： 东财不区分行业/概念/地域类型（混在一起）。板块名本身已自解释——「食品饮料」是行业、「贵州板块」是地域、「酿酒概念」是概念。如需精确分类可按板块名判断，但不推荐额外请求全市场板块清单（clist + m:90+t:1/2/3），因其大页可能触发风控。

2. 分钟级资金流向 — 主力态度实时看

eastmoney_fund_flow_minute(code) 从东财 push2 获取个股分钟级资金流向，将成交按单笔大小拆分为主力、超大单、大单、中单、小单五类净流入，单位 元（非万元）。 

realtime = eastmoney_fund_flow_minute("000858")  # 五粮液
last = realtime[-1]
signal = "bullish" if last["main_net"] > 0 else "bearish"
print(f"主力净流入: {last['main_net']:.0f}元 → {signal}")

# 全天累计主力净流入
total = sum(r["main_net"] for r in realtime)
print(f"全天主力累计: {total/1e4:.0f}万元")

五类资金： super_net（超大单净流入）、large_net（大单净流入）、main_net（主力净流入 = 超大+大单）、mid_net（中单）、small_net（小单）。盘中实时可判断主力态度——持续净流入为 bullish 信号，持续流出为 bearish。

版本背景： V3.1 替换了已下线的百度 PAE fundflow 和 fundsortlist（2026-05 起返回 null）。日级资金流（120 日）见 Layer 4 资金/筹码层，本端点仅覆盖分钟级。

3. 归属 + 资金：板块联动验证

板块归属和资金流向天然互补——查到个股属于某概念后，用资金流验证主力是否真的在买：

验证流程（归属 → 资金） │ ├── eastmoney_concept_blocks(code) │ └── 返回 concept_tags → 得知个股属于哪些题材 │ ├── eastmoney_fund_flow_minute(code) │ └── 返回分钟级资金流 → 主力净流入 > 0 ? │ └── 结论：主力资金 + 题材归属 = 双向确认信号

  1. eastmoney_concept_blocks() 为什么替换了百度 PAE？新接口用哪个关键参数一次拿全？
  2. 资金流向的五类资金中，"主力净流入"由哪两类加总？
  3. 如何用板块归属 + 资金流向做交叉验证？
参考答案：
1. 百度 PAE getrelatedblock 已失效（返回 ResultCode 10003 + 空数组）。东财 slist 通过 spt=3 参数一次拿全行业/概念/地域。
2. 主力净流入 = 超大单净流入（super_net）+ 大单净流入（large_net）。
3. 先查板块归属得知个股题材标签，再用资金流看主力方向——若题材热门且主力持续净流入，信号可信度更高。
