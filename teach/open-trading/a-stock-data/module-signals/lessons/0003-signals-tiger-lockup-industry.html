龙虎榜、解禁预警与行业排名

L1 信号层 第 3/3 节 估计时长：14 分钟

1. 龙虎榜 — 个股席位 + 全市场排名

龙虎榜是信号层的"席位追踪"工具，由两个互补端点构成：dragon_tiger_board() 聚焦单只股票的席位明细，daily_dragon_tiger() 俯瞰全市场的净买额排名。两者均直连东财 datacenter，走 em_get() 限流。

1.1 个股龙虎榜

data = dragon_tiger_board("002475", "2026-05-17", look_back=30)
print(f"近30日上榜 {len(data['records'])} 次")
for r in data['records']:
    print(f"  {r['date']}: {r['reason']} | 净买{r['net_buy']}万")

# 买入席位 TOP5
for s in data['seats']['buy']:
    print(f"  {s['name']}: 买{s['buy_amt']}万 卖{s['sell_amt']}万 净{s['net']}万")

# 机构动向
inst = data['institution']
print(f"机构买{inst['buy_amt']}万 卖{inst['sell_amt']}万 净{inst['net_amt']}万")

返回结构： {records, seats, institution}。records 为历史上榜列表（含上榜原因、净买额、换手率）；seats 分 buy/sell 两个 TOP5 席位列表；institution 通过 OPERATEDEPT_CODE="0" 筛选机构专用席位，汇总买卖净额。

注意： ST 股 5% 涨跌停更容易触发龙虎榜（"连续三日偏离值累计达12%"），科创板 20% 涨跌停则较少触发。上榜原因字段 EXPLANATION 可判断是涨幅触发还是跌幅触发。

1.2 全市场龙虎榜

data = daily_dragon_tiger("2026-05-16")
print(f"{data['date']} 龙虎榜共 {data['total_records']} 条")

# 只看净买入 > 5000万的
data = daily_dragon_tiger("2026-05-16", min_net_buy=5000)
for s in data['stocks'][:5]:
    print(f"  {s['code']} {s['name']}: {s['reason']} | 净买{s['net_buy_wan']}万")

daily_dragon_tiger() 按 BILLBOARD_NET_AMT 降序排列，min_net_buy 参数可过滤小额上榜。非交易日或盘后未更新时返回空列表。

2. 限售解禁 — 未来 90 天预警

lockup_expiry(code, trade_date, forward_days=90) 从东财 datacenter 查询个股的历史解禁记录 + 未来 90 天待解禁，用于规避解禁抛压风险。 

data = lockup_expiry("002475", "2026-07-08")
print(f"历史解禁 {len(data['history'])} 批")
for h in data['history'][:3]:
    print(f"  {h['date']}: {h['type']} 数量={h['shares']} 占比={h['ratio']}%")

if data['upcoming']:
    print(f"未来90天待解禁 {len(data['upcoming'])} 批")
    for u in data['upcoming']:
        print(f"  {u['date']}: {u['type']} 数量={u['shares']}")
else:
    print("未来90天无待解禁 ✓")

限售股类型： 首发原股东限售股份（IPO 后 1-3 年）、首发机构配售股份（战略配售）、定向增发机构配售股份（6-18 个月）、股权激励限售股份。重点关注 ratio（解禁占比）高的批次。

3. 行业板块排名 — 轮动方向一图看

industry_comparison(top_n=20) 从东财 push2 获取约 100 个行业的涨跌幅排名，一次调用即可看清当日行业轮动方向。

版本背景： V3.0 替换了同花顺 401 接口（加了反爬返回 401）。改用东财 push2 clist（m:90+t:2），零鉴权，自带涨跌家数（up_count/down_count）和领涨龙头。 
data = industry_comparison(10)
print(f"共 {data['total']} 个行业")
for r in data['top'][:5]:
    print(f"  {r['rank']}. {r['name']}: {r['change_pct']}% "
          f"涨{r['up_count']}跌{r['down_count']} 领涨{r['leader']}")

关键字段： up_count/down_count（上涨/下跌家数）反映板块内部分化程度——全板块普涨与龙头独涨含义完全不同。leader（领涨股代码）和 leader_change（领涨股涨幅）可快速定位龙头。

检索练习

不翻看上面内容，尝试回答：

  1. dragon_tiger_board() 返回的 institution 字段是如何计算的？
  2. lockup_expiry() 查询未来多少天的解禁？建议重点关注哪个字段？
  3. industry_comparison() 中的 up_count/down_count 为什么比涨跌幅更重要？
参考答案：
1. 从买卖席位明细中筛选 OPERATEDEPT_CODE="0"（机构专用席位），分别汇总买入金额和卖出金额，净额 = 买 - 卖。
2. 默认查询未来 90 天待解禁。重点关注 ratio（解禁数量占总股本比例），占比高则抛压风险大。
3. 涨跌幅只看板块指数（可能被龙头绑架），up_count/down_count 反映板块内多少只股票实际在涨——前者是全板块普涨还是仅龙头独涨，含义完全不同。
