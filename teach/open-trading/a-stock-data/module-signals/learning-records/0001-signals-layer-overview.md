# 学习记录：信号层导览

日期：2026-07-08

## 关键洞察

### 信号层在 a-stock-data 中的定位
信号层是端点最多的数据层（8 个），覆盖"发现机会 → 归属归因 → 资金验证 → 席位跟踪 → 风险预警"的完整信号链路。它位于行情层之上、资金/筹码层之下，本质是把原始行情数据转化为可交易的判断依据。

### 同花顺热点的独特价值
ths_hot_reason() 的 reason 字段由同花顺编辑部人工标注，是 A 股唯一的"官方级"题材归因。与东财概念板块的自动分类不同，人工标签能捕捉到新兴题材（如"Token工厂"这类非标准概念），且盘后 15:30 就更新。

### 北向资金的数据源切换
东方财富北向数据自 2024-08 起净买额字段断供是一个重要的行业性问题。同花顺 hexin API 是当前可行的替代方案，但本地自缓存意味着历史数据只能靠持续运行来积累。首次部署时只有当天数据。

### 端点之间的"版本替换"模式
信号层的 4 个端点在历史上经历过数据源替换：百度 PAE → 东财 slist（板块归属）、百度 PAE → 东财 push2（资金流向）、同花顺 401 → 东财 push2 clist（行业排名）、东财 → 同花顺 hexin（北向资金）。这种"数据源不可靠→换源"的迭代模式是 a-stock-data 的核心工程经验。

### 信号组合验证优于孤立使用
单一信号端点容易产生误判。实际使用中应组合多个端点做交叉验证——例如，热点题材 + 北向流向（资金是否认可题材）、板块归属 + 资金流向（主力是否真的在买）、龙虎榜全市场筛选 + 个股席位深挖。

## 未解决问题
- 东财板块归属不区分行业/概念/地域类型（混合返回），需要 AI 自行根据板块名判断
- 同花顺热点 reason 标签的完整词典未见官方文档
- 北向自缓存的历史覆盖效果需实际运行后评估
