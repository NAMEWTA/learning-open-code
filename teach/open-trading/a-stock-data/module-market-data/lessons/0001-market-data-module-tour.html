行情层模块导览

L1 模块导览 行情层 估计时长：13 分钟

1. 行情层定位：不封 IP 的首选入口

行情层是整个 10 层数据架构的第一层，也是调用频率最高的层——所有估值分析、策略回测、实时监控都从行情数据开始。a-stock-data 为行情层选择了三个不封 IP的数据源，按能力互补分工：

行情层三源分工 │ ├── mootdx (TCP 7709) → K线多周期 + 五档盘口 + 逐笔成交 ├── 腾讯财经 (HTTP GBK) → PE/PB/市值/换手率/涨跌停/指数/ETF └── 百度股市通 (HTTP) → 日K线 + MA5/MA10/MA20 直接返回

关键原则：这三个源都不封 IP，可以放心高频调用，无需经过 em_get() 限流。mootdx 走通达信 TCP 协议（需国内网络），腾讯和百度走普通 HTTP，海外也可用。

选源口诀：K线盘口找 mootdx，估值市值找腾讯，均线直接找百度。

2. mootdx 标准行情（K线 + 盘口 + 逐笔）

mootdx 是行情层的基石，通过通达信 TCP 7709 端口获取原始行情数据。三大核心能力：

能力调用方式返回注意事项
K线数据 client.bars(symbol, frequency, offset) open/close/high/low/vol/amount 返回不复权原始价，跨除权日需自行复权
五档盘口 client.quotes(symbol=[...]) 46 字段含买一~买五/卖一~卖五 支持批量（传列表）
逐笔成交 client.transaction(symbol, date) time/price/vol/buyorsell 非交易时间返回空，仅当日有效

frequency 参数：最关键的一个坑

bars() 的周期参数名是 frequency，不是 category。如果传错参数名，会被 Python 的 **kwargs 静默吞掉，永远退化为默认日线（frequency=9），拿不到分钟数据。 

# frequency 参数实测值表（mootdx 0.11.7）
#   0=5分钟  1=15分钟  2=30分钟  3=60分钟  4=日线
#   5=周线   6=月线   7=1分钟除权  8=1分钟  9=日线(默认)
#  10=季线  11=年线

klines = client.bars(symbol='688017', frequency=9, offset=10)   # 日线 10 根
min1   = client.bars(symbol='688017', frequency=8, offset=240)  # 1分钟 240 根

这个参数名 Bug 在 V3.2.5 才修复——早期版本写成 category，导致 AI 复制代码后拿到日线而非分钟线，排查了很久才发现是参数被 **kwargs 吞掉了。详见 参考文档 frequency 完整表。

3. 腾讯与百度：补齐估值与均线

mootdx 不提供 PE/PB/市值/换手率/涨跌停价——这些是估值分析的核心指标，交给腾讯财经。百度则补齐了 K 线的 MA 均线数据（mootdx K 线不含均线指标），而且服务端直接算好返回，无需本地计算。

腾讯 tencent_quote()：88 字段中的估值精华

腾讯行情接口返回 88 个 ~ 分隔字段，其中对估值最有价值的 6 个：

索引字段示例
39PE(TTM)300.45
44总市值(亿)410.88
45流通市值(亿)410.88
46PB(市净率)11.51
47涨停价258.01
48跌停价172.01

常见踩坑：很多网上教程把索引 43 写成 PB，实测 43 是振幅%，PB 在索引 46。原因是腾讯字段排列中振幅（amplitude）恰好在市值前面，容易被误读。

百度 baidu_kline_with_ma()：自带均线的 K 线

百度的独有优势：返回的 keys 中直接包含 ma5avgprice、ma10avgprice、ma20avgprice，不需要用 pandas 本地计算。注意返回格式中 marketData 是 ; 分隔的多行字符串，需自行 split 解析；ResultCode 在不同场景下可能是 int 或 str 类型，做判断时建议 str(d.get("ResultCode")) != "0"。

4. 三源拼装：一支股票的完整行情快照

实际使用中，你几乎永远需要组合调用三个数据源。下面的代码展示了如何用约 10 行逻辑拼装出一支股票的完整行情画像： 

# ── 三源组合示例：获取一只股票的完整行情画像 ──
code = "688017"

# 1. 腾讯：实时估值（PE/PB/市值/换手率）
q  = tencent_quote([code])[code]
print(f"{q['name']}: {q['price']}元 PE={q['pe_ttm']} PB={q['pb']} 市值={q['mcap_yi']}亿")

# 2. mootdx：K线 + 盘口深度
client = tdx_client()
kl = client.bars(symbol=code, frequency=9, offset=60)  # 近60个交易日
dq = client.quotes(symbol=[code])
# dq[0] 含买一~买五/卖一~卖五共 20 个挂单价量

# 3. 百度：日K均线趋势判断
baidu = baidu_kline_with_ma(code)
# 用最后一行判断：价格在 MA20 之上？MA5 是否上穿 MA10？
print("K线根数:", len(kl), "盘口档位:", len(dq))
场景需要的源获取什么
快速估值判断腾讯PE/PB/市值/涨跌停价
技术分析回测mootdx + 百度K线(OHLCV) + MA5/10/20
盘中交易决策mootdx + 腾讯实时盘口深度 + 实时价格
完整行情画像三者全用实时价+K线+均线+估值

  2. 腾讯行情接口中，索引 43 和索引 46 分别对应什么字段？
  3. 百度 K 线接口相比 mootdx 的独有优势是什么？

参考答案：
1. 参数名是 frequency。传 category=8 会被 **kwargs 静默吞掉，永远退化为默认 frequency=9（日线），拿不到 1 分钟数据。
2. 索引 43 = 振幅%（不是 PB），索引 46 = PB(市净率)。网上很多教程搞反了。
3. 百度返回自带 MA5/MA10/MA20 均线数据，无需本地计算。mootdx 的 K 线只有原始 OHLCV，要均线得自己算。
4. (a) 腾讯(tencent_quote) (b) mootdx(client.quotes) (c) 百度(baidu_kline_with_ma)
