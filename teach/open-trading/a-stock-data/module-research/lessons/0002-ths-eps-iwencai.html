同花顺一致预期EPS + iwencai 语义搜索

L1 研报层 第 2 节 / 共 2 节 估计时长：14 分钟

1. 为什么需要两个补充数据源

东财 reportapi 解决了"有什么研报"和"研报说了什么评级/EPS"的问题，但它不提供机构一致预期（多家机构预测的统计汇总），也做不了跨主题的自然语言检索。这两个缺口分别由同花顺 basic.10jqka.com.cn 和 iwencai OpenAPI 补充。

三个数据源的定位对比

数据源提供什么鉴权限流
东财 reportapi个股/行业研报列表 + PDFReferer 头em_get() 内置
同花顺 basic机构一致预期 EPS（均值/最小/最大）UA 头即可低频无风险
iwencai OpenAPINL 语义搜索研报/公告/新闻API Key + X-Claw未知

2. ths_eps_forecast() — 同花顺一致预期EPS

直连 https://basic.10jqka.com.cn/new/{code}/worth.html，用 pandas.read_html() 解析 HTML 中的"每股收益"表格。返回 DataFrame，核心列为：年度、预测机构数、最小值、均值、最大值。

V3.2.5 修复了一个关键 Bug：原版用 df.iloc[:, 2] 按位置取"均值"列，但不同股票页面表格列数可能不同（部分页面少一列"摊薄每股收益"），导致取到错误列。修复后改用列名匹配——遍历 DataFrame 的 columns，找到含"均值"的列。

使用建议：一致预期 EPS 的可靠性取决于预测机构数量。"预测机构数" < 3 意味着样本太少，均值无统计意义，应标记为低置信度。冷门小票（如部分北交所标的）经常出现这种情况。

同花顺 worth.html 页面编码为 GBK，必须设置 r.encoding = "gbk" 否则中文乱码。请求仅需 UA 头（模拟浏览器），无 Referer 要求。

3. iwencai — NL 语义搜索（唯一能力）

iwencai 是研报层唯一需要 API Key 的数据源。它提供两个核心端点：

端点函数用途
/v1/comprehensive/searchiwencai_search()语义搜索研报/公告/新闻，返回段落级文章列表
/v1/query2dataiwencai_query()NL 数据查询，返回结构化字段（如"贵州茅台 ROE"）

iwencai_search() 的核心参数：query（自然语言查询）、channel（report/announcement/news）、size（默认 10，实测可调到 50）。返回的 articles 按 score 排序，但同一篇研报的不同段落可能重复出现——这正是 dedup_articles() 解决的问题。

X-Claw 鉴权机制

iwencai SkillHub 2.0 强制要求每个请求携带 5 个自定义头，由 _claw_headers() 统一生成：

X-Claw-Call-Type: normal ← 调用类型 X-Claw-Skill-Id: report-search ← 技能标识 X-Claw-Skill-Version: 2.0.0 ← 技能版本 X-Claw-Plugin-Id: none ← 插件标识（独立调用） X-Claw-Plugin-Version: none ← 插件版本（独立调用） X-Claw-Trace-Id: {32字节随机hex} ← 链路追踪ID

secrets.token_hex(32) 为每次请求生成唯一 trace_id，用于服务端链路追踪和审计。配合 Authorization: Bearer {IWENCAI_KEY} 完成双重鉴权。

dedup_articles() 去重逻辑

iwencai 搜索返回的 articles 是段落级的——同一篇研报可能因多个段落命中关键词而返回多条。去重策略：以 uid 为主键（fallback 到 title+publish_date 组合），对同一 uid 仅保留 score 最高的段落，最后按发布日期倒序排列。

iwencai 的唯一价值：NL 主题搜索。"人形机器人 行星滚柱丝杠 2026"这种跨概念、跨行业的主题检索，只有 iwencai 能做。按具体标的搜研报走东财 reportapi 更稳定、更完整。

检索练习

不看屏幕，尝试回答（每题 30 秒）：

  1. ths_eps_forecast() 返回的 DataFrame 中，"均值"列代表什么？V3.2.5 为什么修复了取列方式？
  2. iwencai_search() 的 channel 参数有三个可选值，分别是什么？
  3. _claw_headers() 生成的 5 个头中，哪个是每次请求都不同的？它用什么方式生成？
  4. dedup_articles() 去重的依据是什么？为什么需要去重？
参考答案：
1. "均值" = 机构一致预期 EPS（所有覆盖机构的预测平均值）。V3.2.5 前用 iloc[:, 2] 按位置取列，但不同股票页面的表格列数可能不同，导致取到错误列。修复后改用列名匹配。
2. channel 可选 report（研报）、announcement（公告）、news（新闻）。
3. X-Claw-Trace-Id 每次请求不同，由 secrets.token_hex(32) 生成 32 字节随机十六进制字符串，用于链路追踪。
4. 以 uid 为主键去重，仅保留 score 最高的一条。原因是 iwencai 返回段落级结果，同一篇研报多个段落命中关键词会产生重复，需要合并。
