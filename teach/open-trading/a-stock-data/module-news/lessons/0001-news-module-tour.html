新闻层导览：个股舆情与全球快讯

L1 新闻层 估计时长：14 分钟

1. 新闻层在架构中的位置

新闻层是 a-stock-data 的第五层，位于资金面之上、基础数据层之下。它的职责很纯粹：拉取最新的文本信息，不做分析、不做情感打分——只是把原始新闻送到你手里。

新闻层端点全景（2 活跃 + 1 弃用） │ ├── 个股新闻 ──── eastmoney_stock_news(code) 东财个股相关新闻（标题+摘要+来源） │ 数据源: search-api-web JSONP V3.2.1 修复解析 Bug │ ├── 全球资讯 ──── eastmoney_global_news() 7×24 全球财经快讯（财联社替代方案） │ 数据源: np-weblist 快讯列表 V3.0 新增 │ └── ⚠️ 已弃用 ─── cls_telegraph() 财联社电报（2026-05 API 下线） 旧数据源: cls.cn/nodeapi → 改用 eastmoney_global_news()

新闻层和前面几层有一个本质区别：它返回的是非结构化文本，而非数值数据。这意味着你无法直接用新闻数据回测策略——你需要先阅读、理解、判断。新闻层的价值在于信息速度优势：一条突发新闻可能比K线上的价格变化早几分钟到几小时。

架构定位：新闻层是"舆情监控"而非"舆情分析"。它负责数据获取，不负责 NLP 情感分析、关键词提取、事件分类。这些上层加工由用户自己完成——你可以把新闻数据喂给 LLM 做摘要或打分。

2. 个股新闻 — JSONP 接口与解析陷阱

eastmoney_stock_news(code, page_size=20) 从东财搜索 API直连拉取个股相关新闻。这个端点的技术复杂度不在于数据量，而在于它的接口协议是 JSONP——响应体包裹在 jQuery_news({...}) 回调函数中，需要手动切掉外层才能拿到 JSON。 

# 请求流程（简化版，展示核心逻辑）
code = "688017"

# 1. 构造 JSONP 参数
cb = "jQuery_news"
inner = json.dumps({
    "keyword": code,
    "type": ["cmsArticleWebOld"],
    "param": {"cmsArticleWebOld": {
        "searchScope": "default", "sort": "default",
        "pageIndex": 1, "pageSize": page_size
    }}
})
params = {"cb": cb, "param": inner}

# 2. 发送请求（走 em_get 限流）
r = em_get(url, params=params, headers=headers)

# 3. 解析 JSONP — 切掉 jQuery_news( 和末尾 )
text = r.text
json_str = text[text.index("(") + 1 : text.rindex(")")]
d = json.loads(json_str)

# 4. 提取文章列表
articles = d["result"]["cmsArticleWebOld"]  # 直接就是 list
for a in articles:
    title = re.sub(r'<[^>]+>', '', a["title"])   # 清理 HTML 标签
    content = re.sub(r'<[^>]+>', '', a["content"])[:200]
    print(f"{a['date']} | {a['mediaName']} | {title}")

V3.2.1 关键修复：早期版本代码对 cmsArticleWebOld 的结果做了 .get("list") 操作，但东财实际返回中 cmsArticleWebOld 直接就是文章列表（list of dict），不是 {list: [...]} 嵌套结构。这导致 AttributeError——旧代码试图对一个 list 调用 .get() 方法。修复后的代码直接遍历 cmsArticleWebOld，只有当它为 None 时才 fallback 为空列表 []。

间歇性空返回（#18）：部分大陆住宅 IP 调此接口只会拿到 passportWeb（股民资料）而无 cmsArticleWebOld（文章列表）——这是东财对该 IP 的间歇风控，非代码问题。代码已安全返回空列表，遇到时隔几分钟或换网络重试即可。

3. 全球资讯 — 财联社替代方案

eastmoney_global_news(page_size=50) 从东财快讯频道拉取7x24 全球财经滚动新闻。这是财联社下线后的替代方案——cls.cn 在 2026 年 5 月迁移到 Next.js 架构，旧版公开 API cls.cn/nodeapi/telegraphList 全面返回 404。而东财 np-weblist 是免费、无 key、7x24 滚动的同类替代。 

news = eastmoney_global_news()
for n in news[:10]:
    print(f"{n['time']} | {n['title']}")
    if n['summary']:
        print(f"  → {n['summary'][:120]}")

# 关键字段：
# title   — 快讯标题
# summary — 快讯摘要（text，非 HTML）
# time    — 发布时间（showTime 字段）

# 与个股新闻的关键差异：
# - 不需要 code 参数（全市场）
# - 不走 JSONP（标准 JSON 响应）
# - req_trace 用 uuid4() 做请求追踪

技术细节：np-weblist 接口需要 3 个关键参数——client=web、biz=web_724（7x24 业务线）、fastColumn=102（栏目 ID）。req_trace 用 uuid.uuid4() 生成唯一请求追踪 ID，用于服务端日志关联，对业务无影响。

4. 已弃用端点与风控说明

cls_telegraph() 是本层唯一已弃用的端点。了解它为什么失效，比记住它的代码更有价值：

弃用详情（#14）：财联社（cls.cn）在 2026 年 5 月进行了全站架构迁移——从传统服务端渲染迁移到 Next.js。旧版公开 API cls.cn/nodeapi/telegraphList 在迁移后返回 HTTP 404。新版 API 需要签名认证，无法通过公开 HTTP 请求调用。SKILL.md 中保留了旧代码作为历史参考，注释明确标注"已不可用"。
端点状态原因替代方案
cls_telegraph() 已弃用 cls.cn 迁移 Next.js，旧 API 404 eastmoney_global_news()

间歇风控统一说明：新闻层的两个活跃端点（search-api-web 和 np-weblist）都走 em_get() 限流，继承了基础设施层的串行保护。大陆住宅 IP 可能出现间歇性空返回或 HTTP 000，这属于上游风控而非代码缺陷。代码已对所有异常场景做了安全降级（返回空列表），不会抛出未捕获异常。

  2. V3.2.1 修复的 cmsArticleWebOld 解析 Bug 是什么？旧代码为什么会 AttributeError？
  3. cls_telegraph() 为什么被弃用？替代方案是什么？
  4. 间歇性空返回（缺少 cmsArticleWebOld）是代码问题吗？应该如何应对？
参考答案：
1. 因为接口返回的是 JSONP 格式（jQuery_news({...})），需要切掉外层回调函数名和括号才能拿到纯 JSON。代码用 text.index("(") 到 text.rindex(")") 截取中间部分。
2. 旧代码假设 cmsArticleWebOld 是 {list: [...]} 结构，对它调 .get("list")。但东财实际返回中它就是 list 本身，对 list 调 .get() 触发 AttributeError。修复后直接遍历列表。
3. cls.cn 在 2026-05 迁移到 Next.js，旧 API /nodeapi/telegraphList 返回 404，新版需签名认证。替代方案是 eastmoney_global_news()（东财 7x24 快讯，免费无 key）。
4. 不是代码问题，是东财对部分住宅 IP 的间歇风控。应对：隔几分钟重试、换网络（如手机热点）、或降低请求频率。
