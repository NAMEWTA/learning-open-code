# 使命：新闻层 — 从个股舆情到全球宏观的实时信息获取

## 为什么
价格反映的是已知信息，新闻是信息的最早载体。一条突发新闻可能让股价在几分钟内剧烈波动——掌握新闻层意味着你能比大多数只看K线的交易者更早感知风险与机会。东方财富个股新闻告诉你某只股票"正在发生什么"，全球7x24资讯告诉你"市场环境在如何变化"。两层信息叠加，形成完整的舆情监控体系。

## 成功的样子
- 能用 eastmoney_stock_news() 快速拉取任意个股的最新相关新闻（标题+摘要+来源+时间）
- 能理解 JSONP 接口的解析流程，知道 V3.2.1 修复的 cmsArticleWebOld 解析 Bug 原理
- 能用 eastmoney_global_news() 获取 7x24 全球财经快讯，替代已下线的财联社
- 知道 cls_telegraph() 已弃用的原因（cls.cn 迁移 Next.js，旧 API 404）及替代方案
- 能识别间歇风控的表现（HTTP 000 / 空 / 缺少 cmsArticleWebOld），知道重试策略而非怀疑代码

## 约束条件
- 每次学习 15 分钟以内（短课制），2 个活跃端点 + 1 个弃用端点在 1 节课内覆盖
- 所有端点走 em_get() 限流，基础设施层已封装
- 大陆住宅 IP 间歇风控说明：非代码 Bug，重试/换网络即可

## 不在范围内
- 底层 em_get() 限流实现 — 由 L1-module-infrastructure 覆盖
- 新闻情感分析 / NLP 处理 — 属于上层应用，不在数据获取层讨论
- 个股公告层 — 由后续 module-filings 覆盖
