阶段详解：鉴权、去重与异常处理

L2 垂直切片 主题研报检索 第 2/2 节 估计时长：14 分钟

1. 阶段一：iwencai X-Claw 鉴权与搜索

iwencai 开放平台要求 SkillHub 2.0 协议头，缺一则返回 401。与普通 API 只需 Authorization: Bearer xxx 不同，iwencai 额外要求五个 X-Claw 头： 

headers = {
    "Authorization": f"Bearer {IWENCAI_KEY}",
    "Content-Type": "application/json",
    "X-Claw-Call-Type": "normal",
    "X-Claw-Skill-Id": "report-search",
    "X-Claw-Skill-Version": "2.0.0",
    "X-Claw-Plugin-Id": "none",
    "X-Claw-Plugin-Version": "none",
    "X-Claw-Trace-Id": secrets.token_hex(32),  # 每次请求唯一
}

一个关键细节：X-Claw-Trace-Id 每次请求必须生成新的 32 字节随机 hex，不能用固定值。这是请求追踪和防重放机制的一部分。

异常路径 — 401 未授权：首先检查 IWENCAI_KEY 环境变量是否设置、Key 是否过期。其次确认 X-Claw-Skill-Id 为 "report-search"（不是随意值）。如果刚申请 Key，可能需要等待几分钟生效。
异常路径 — 超时：iwencai 请求设置了 30 秒超时。如果某个冷门关键词搜索耗时过长，建议缩小 size 参数（默认 50 可降到 20），或为关键词列表添加 try/except 包裹，确保一个关键词失败不影响其余。

2. 阶段二：dedup_articles 去重逻辑详解

dedup_articles() 的核心逻辑只有 7 行，但有两个值得深究的设计决策： 

def dedup_articles(articles: list[dict]) -> list[dict]:
    best = {}
    for a in articles:
        uid = a.get("uid", "") or f"{a.get('title','')}|{a.get('publish_date','')}"
        score = float(a.get("score", 0))
        if uid not in best or score > float(best[uid].get("score", 0)):
            best[uid] = a
    return sorted(best.values(), key=lambda x: x.get("publish_date", ""), reverse=True)

决策 1 — uid fallback：当文章没有 uid 字段时，用 title|publish_date 拼接作为去重键。这是兜底策略——两个同标题同日期的文章近乎可以视为同一篇。缺点：如果同一机构同一天发布了两篇不同内容但有相同标题的研报，会误去重（概率极低）。

决策 2 — 按 score 保留最高分：去重冲突时保留 score 最高的一篇，而非最新的。这假设 iwencai 的 score 反映了与查询关键词的相关度——保留最相关段落比保留最新更重要。最终排序按日期倒序，确保使用者先看到最新研报。

关键词无结果降级：某关键词返回空列表时，循环继续执行下一个 query，不会中断。最终 all_articles 只包含有结果的关键词对应的文章。建议打印每个 query 的结果数，方便判断哪些关键词搜索效果不佳。

3. 阶段三与四：东财翻页 + PDF Referer 鉴权

东财 reportapi 的翻页逻辑注意一个坑——接口参数中有三套翻页字段同时存在： 

params = {
    "pageNo": str(page),    # ← 翻页核心参数
    "pageNum": str(page),   # 冗余但需同时传
    "pageNumber": str(page), # 冗余但需同时传
}

三个参数需要同时传同一页码，因为东财后端可能对不同客户端使用不同参数名。单独传一个在某些版本可能不生效。

PDF 下载的 URL 模板固定为：

PDF_TPL = "https://pdf.dfcfw.com/pdf/H3_{info_code}_1.pdf"
# 例: info_code="AP202607081234567890" → H3_AP202607081234567890_1.pdf
异常路径 — PDF 403 Forbidden：东财 PDF 服务器检查 Referer 头，必须包含 https://data.eastmoney.com/。如果忘了传 Referer，直接访问 PDF URL 会返回 403。已下载过的 PDF 会自动跳过（检查本地文件是否存在），避免重复下载。
异常路径 — PDF 内容为空：即使 HTTP 200，返回内容也可能不是有效 PDF。代码检查 len(r.content) >= 1024——小于 1KB 的响应通常是错误页面 HTML，不是真实 PDF。这类 case 返回 None 并跳过。

判断题

问题：以下说法是否正确？"dedup_articles 去重时，如果两篇文章 uid 相同，保留发布日期最新的那一篇。"

错误。去重时保留 score 最高的一篇（与查询关键词最相关），而非最新的。最终排序才是按日期倒序——这是两个独立操作：先按 score 去重，再按日期排序展示。
