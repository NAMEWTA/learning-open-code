主题研报批量检索流程地图

L2 垂直切片 主题研报检索 第 1/2 节 估计时长：12 分钟

1. 这条链路在做什么

输入一个主题（如"人形机器人"），输出该产业链的全部机构研报列表 + 已下载的 PDF。 这条链路串联了 iwencai NL 语义搜索和东财 reportapi 两个数据源，用多关键词策略从不同角度覆盖研究，再通过东财按标的补充，形成对该主题的完整研究视图。

核心思想：iwencai 负责"哪些研报覆盖了这个主题"（广度搜索），东财 reportapi 负责"这些标的还有哪些研报"（深度补充）。两个数据源互补，不可互相替代。

这是研报层（Layer 2）的典型垂直切片——它不修改任何单层代码，而是编排已有能力形成端到端流程。流程入口为以下伪代码：

queries = ["人形机器人产业链深度 2026", "人形机器人减速器 丝杠", "特斯拉Optimus 国产供应链"]
# → iwencai 并行搜索 → dedup_articles 去重 → eastmoney_reports 补充 PDF

3. 双数据源对比

理解两个数据源的差异是正确编排它们的前提：

能力维度iwencai东财 reportapi
搜索方式NL 自然语言（"人形机器人 丝杠"）按股票代码 / 行业码精确查询
覆盖范围全市场研报（含期货、策略报告）东财收录的个股 + 行业研报
鉴权API Key + X-Claw Headers（付费）Referer 头（免费）
返回字段标题、机构、摘要、score、stock_infos标题、机构、EPS 预测、评级、infoCode
PDF 获取不直接提供infoCode 拼 URL 下载
核心场景「这个主题有哪些研报？」「这只股票有哪些研报？」
编排策略：先用 iwencai 做主题搜索获取"种子研报"，从中提取相关股票代码，再用东财补全这些标的的其余研报。这样既有了主题广度，又有了标的深度。

4. 调用时序要点

顺序执行即可，无需并行编排。虽然多关键词可以并行搜索，但 a-stock-data 当前未封装异步请求，按关键词列表顺序循环执行即可：

seen_uids = set()
all_articles = []
for q in queries:
    arts = iwencai_search(q, channel="report", size=50)
    for a in arts:
        uid = a.get("uid", "")
        if uid not in seen_uids:
            seen_uids.add(uid)
            all_articles.append(a)

这里边搜索边去重的好处是节省内存——不需要等所有关键词搜索完再做一次全局去重。后续东财补充阶段按股票代码逐一拉取，每个标的只需翻 1-2 页即可获取近期研报。

注意：东财 reportapi 通过 em_get() 调用，内置 1 秒间隔限流（批量建议调大到 1.5~2 秒）。如果对 10 个标的各翻 5 页，仅网络等待就需要 10 x 5 x 1.5 ≈ 75 秒。实际使用时建议 max_pages=1 获取最近 100 篇即可。

检索练习

问题：以下哪项是 iwencai 能做而东财 reportapi 不能做的事？

  1. 按股票代码查询该股所有研报
  2. 按自然语言主题搜索全市场研报
  3. 下载研报 PDF 原文
  4. 获取机构一致预期 EPS 数据
答案：B。iwencai 的唯一价值是 NL 语义搜索——"人形机器人 行星滚柱丝杠"这种跨主题、跨标的的检索。A 和 C 是东财 reportapi 的能力（按代码查研报 + 通过 infoCode 下载 PDF）。D 是同花顺 ths_eps_forecast 的能力，属于另一个数据源。
