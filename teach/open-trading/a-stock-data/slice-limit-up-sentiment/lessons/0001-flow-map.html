打板情绪分析全链路

L2 垂直切片 打板情绪 第 1/2 节 估计时长：13 分钟

1. 这条链路在做什么

输入一个交易日日期，输出当天的打板情绪全景。这是打板层8个函数的串联实战——从四个涨停板池拉取原始数据，用同花顺涨停揭秘补充题材信息，最终通过情绪速算函数汇总为结构化指标。

核心思想：打板不是选股，是读市场温度。涨停数/炸板率/连板高度/晋级率这四个指标构成了当天的"市场心电图"。单看一只涨停股没有意义，整体格局才是信号。

情绪输出示例：

{
  "date": "20260626",
  "zt_count": 57,     "zb_count": 18,     "dt_count": 5,
  "break_rate": 24.0,  "max_height": 7,
  "ladder": {1: 28, 2: 15, 3: 8, 4: 3, 5: 2, 7: 1}
}

2. 七步调用链路

limit_up_sentiment("20260626") │ ├─ [1] em_zt_pool(date) → 涨停池 (push2ex) ├─ [2] em_zb_pool(date) → 炸板池 (push2ex) ├─ [3] em_dt_pool(date) → 跌停池 (push2ex) ├─ [4] em_yzt_pool(date) → 昨涨停池 (push2ex) [可选] │ ├─ [5] ths_limit_up_pool(date) → 涨停揭秘 (10jqka) │ └─ [6] limit_up_sentiment(date) ├─ 连板梯队统计 (ladder) ├─ 炸板率 = zb / (zt+zb) * 100% ├─ 最高连板 (max_height) └─ 涨跌停对比 (zt/zb/dt counts) 四池同源(push2ex)共享限流，涨停揭秘走独立域名。
步骤数据源关键字段
1. 涨停池东财 push2ex连板数、封板时间、封板资金、炸板次数
2. 炸板池东财 push2ex涨停价、炸板时间、振幅、涨速
3. 跌停池东财 push2ex封单资金、连续跌停天数、板上成交额
4. 昨涨停池东财 push2ex今日涨幅（算晋级率）、昨封板时间
5. 涨停揭秘同花顺 10jqka涨停原因、板型、封板成功率、回封标记
6. 情绪速算纯计算炸板率、连板高度、连板梯队{板数: 家数}

3. 四池通用模式 — _em_zt_api 拆解

东财四池共用同一个底层函数 _em_zt_api()，只有 endpoint 和 sort 参数不同： 

def _em_zt_api(endpoint, sort, date):
    url = f"https://push2ex.eastmoney.com/{endpoint}"
    params = {"ut": ZTB_UT, "dpt": "wz.ztzt",
              "Pageindex": 0, "pagesize": 10000,
              "sort": sort, "date": date}
    r = em_get(url, params=params, ...)  # ← 走em_get限流
    return (r.json().get("data") or {}).get("pool") or []
endpointsort含义返回池
getTopicZTPoolfbt:asc封板时间升序涨停池
getTopicZBPoolfbt:asc封板时间升序炸板池
getTopicDTPoolfund:asc封单资金升序跌停池
getYesterdayZTPoolzs:desc涨速降序昨涨停池
四个关键坑：
1. price 原始值 ×1000（p["p"] / 1000 才得到真实价格）
2. 涨停板时间 fbt/lbt 是 HHMMSS 整数（92500 → "09:25:00"），需 _fmt_zt_time() 转换
3. date 必须传交易日，非交易日 data 返回 null，pool 为空列表
4. 四池共用 dpt=wz.ztzt，只有 sort 不同

4. 涨停揭秘 — 跨源补充

同花顺 ths_limit_up_pool() 提供东财没有的关键字段：涨停原因题材（reason）、板型（一字板/换手板/T字板）、封板成功率（0~1）。

双源互补价值：东财告诉你"涨停了，封了多少钱"，同花顺告诉你"为什么涨停，封得稳不稳"。两者结合才能判断是情绪性涨停（游资强封）还是逻辑性涨停（题材共振）。

练习：链路排序

请将以下步骤按正确的调用顺序排列：

  1. A. 计算炸板率 = zb / (zt+zb)
  2. B. em_zt_pool(date) 拉涨停池
  3. C. 统计连板梯队 ladder
  4. D. ths_limit_up_pool(date) 拉涨停揭秘
  5. E. em_zb_pool(date) 拉炸板池
B → E → D → C → A
先拉取原始数据（四池+揭秘），再统计计算（梯队、炸板率）。拉取顺序可以不严格，但必须先有数据再计算。
