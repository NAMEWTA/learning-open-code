二、基础设施核心函数

函数层级功能关键设计
tdx_client() 基础设施 创建 mootdx 客户端 TCP 探测有序列表 → 显式 server= 绕过 BESTIP bug → 三级 fallback → 明确报错
em_get() 基础设施 东财统一请求入口 串行限流(≥1s+随机抖动) + 会话复用 + 连接级自动重试(429/5xx)
eastmoney_datacenter() 基础设施 东财数据中心统一查询 龙虎榜/解禁/两融/大宗/股东/分红 共用，走 em_get 限流
get_prefix() 基础设施 6位代码 → 市场前缀 6/9→sh，8→bj，其余→sz。所有接口统一使用
EM_SESSION 基础设施 全局 HTTP 会话 Keep-Alive 复用，默认 UA，挂载 urllib3 Retry 适配器
EM_MIN_INTERVAL 基础设施 东财最小请求间隔 默认 1.0s，批量任务建议调大到 1.5-2s

四、估值公式与调研流程

函数/流程功能公式
forward_pe(price, eps) 前向PE price / eps_forecast
pe_digestion(pe, cagr, target=30) PE消化时间 ln(PE / 30) / ln(1 + CAGR)
calc_peg(pe, cagr) PEG 估值 PE / (CAGR * 100)；<1 便宜，1-1.5 合理，>1.5 贵
full_valuation(code) 单票完整估值 实时价 → 一致预期EPS → PE_fwd / PEG / 消化年数（30秒）

四套内置调研流程

流程做什么耗时涉及端点
A — 单票估值实时价 → 一致预期EPS → 前向PE/PEG/消化年数~30秒腾讯行情 + 同花顺EPS
B — 批量对比多只股票横向估值排列~1分钟同A，循环多票
C — 主题研报iwencai NL搜索 + 东财PDF交叉补充~2分钟iwencai + reportapi + PDF
D — 新标的调研覆盖→估值→板块→资金→龙虎榜→解禁→两融~1分钟同花顺EPS + 腾讯 + 东财slist + push2 + datacenter
