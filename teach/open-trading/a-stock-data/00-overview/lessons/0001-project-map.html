a-stock-data 项目总览

L0 项目导览 V3.3.0 估计时长：12 分钟

1. 项目定位

a-stock-data 是一个自包含的 Claude Code Skill 文件（109KB Markdown），将 13 个数据源的 A 股数据整合为 AI 助手直接可用的工具集，内含约 60 个 Python 函数和 10 层架构文档。兼容 Claude Code、Codex、OpenClaw 三大平台。

目标用户是有 Python 基础、希望搭建量化投研数据管线的开发者。所有端点的鉴权、编码、限流已封装——复制代码即用。覆盖 4 套典型场景：单票估值、批量对比、主题研报检索、新标的快速调研。

2. 十层数据架构

a-stock-data 按数据品类垂直分为 10 层，覆盖 40 个端点、13 个数据源。核心原则：mootdx/腾讯不封 IP 优先用；东财仅用于其独有数据（龙虎榜、解禁、两融等），且全部走 em_get() 限流。

3. 核心设计哲学

原则 1：自包含单文件。 整个项目就是一个 SKILL.md——代码与文档同在一处，AI 读取即用，无需克隆仓库或安装额外配置。
原则 2：数据源优先级分明。 mootdx/腾讯不封 IP 优先用，东财仅用于独有数据（约 14 个端点），且全部经 em_get() 限流。
原则 3：零第三方数据封装。 V3.0 移除 akshare，全部直连 HTTP API。减少中间层 = 减少故障点，不再受版本兼容问题影响。
原则 4：东财统一防封。 em_get() 内置串行限流（1s 间隔 + 随机抖动）+ Keep-Alive 复用 + 自动重试。复制任何代码块即自带防封。
原则 5：多平台兼容。 Markdown + 内嵌 Python 的格式兼容 Claude Code/Codex/OpenClaw，也可直接复制脚本独立运行。

4. 源码结构与学习路线

SKILL.md 虽是单文件，但内部有清晰的逻辑分层：

检索练习

不看屏幕，尝试回答（每题 30 秒）：

  1. 10 层架构中哪三层是 V3.3.0 新增的？
  2. 为什么东财排在数据源优先级最末？说出两个原因。
  3. em_get() 做了哪三件事防封 IP？
  4. 查股票的 PE/PB/市值应该用哪个数据源？为什么？
参考答案：
1. 打板层（L8）、ETF 期权层（L9）、舆情互动层（L10），于 V3.3.0 新增。
2. 东财有 IP 风控（每秒 >5 次即封），而 mootdx/腾讯不封 IP——应优先使用。
3. 串行限流（最小 1s 间隔 + 随机抖动）、Keep-Alive 会话复用、连接级自动重试。
4. 用腾讯财经（tencent_quote）——不封 IP，一次请求拿 PE/PB/市值/换手率/涨跌停。
