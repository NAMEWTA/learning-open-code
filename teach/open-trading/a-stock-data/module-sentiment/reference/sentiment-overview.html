一、四端点能力矩阵

端点所属层协议鉴权核心能力输入关键输出字段
cninfo_irm() §10.1 互动易 HTTP POST UA 投资者提问 + 公司官方回复（两步请求） code(6位) question, answer, answerer, ask_time
ths_hot_list() §10.2 同花顺 HTTP GET UA 热度排名 + 概念标签 + 人气标签 + 排名变化 period(hour/day) rank, heat, concepts, tag, rank_chg, pct
em_hot_rank() §10.2 东财 HTTP POST UA 人气排名 + 排名变化 + 名称/价格（经 ulist.np 补全） top(默认50) rank, name, price, pct, rank_chg
em_hot_concept() §10.2 东财 HTTP POST UA 个股被哪些热门概念覆盖 + 命中热度 code(6位) concept, bk(conceptId), hit

二、cninfo_irm() 两步请求详解

步骤端点方法参数位置参数返回值关键字段
1 /newircs/index/queryKeyboardInfo POST body keyWord=code data[0].secid → orgId
2 /newircs/company/question POST query string stockcode, orgId, pageSize, pageNum rows[] → {stockCode, mainContent, attachedContent, attachedAuthor, pubDate}

翻页参数

参数默认值说明
page_size30每页返回条数
page_num1页码，从 1 开始
坑 1：第二步参数必须放 query string（不是 body），否则 HTTP 400。
坑 2：orgId 取自第一步的 secid，即便前缀是 gshk（港股），靠 stockcode 过漉照样拿 A 股问答。
坑 3：最新提问常未回复（answer=None），回复率因公司而异。
坑 4：pubDate 是毫秒时间戳，需 / 1000 转换。

参数类型取值说明

返回字段类型说明

四、em_hot_rank() 参数与字段

参数类型默认说明
topint50返回前 N 只人气最高个股
返回字段类型来源说明
rankintgetAllCurrentList.rk人气排名
codestrgetAllCurrentList.sc 去前缀6 位代码（去 SZ/SH 前缀）
namestrulist.np.f14股票名称（补全得来）
pricefloatulist.np.f2最新价（补全得来）
pctfloatulist.np.f3涨跌幅（补全得来）
rank_chgintgetAllCurrentList.hisRc排名变化（正=上升）

前缀转换规则

# getAllCurrentList 返回 sc 字段格式: SZ000001 / SH600000
# 转 ulist.np secids 格式:
secid = ("0." if sc.startswith("SZ") else "1.") + sc[2:]   # → 0.000001 / 1.600000
坑 1：getAllCurrentList 只返回带前缀代码（SZ/SH），名称/价格必须走 ulist.np 补全。
坑 2：ulist.np 的 diff 偶尔是 dict（按序号为键），需 list(values()) 归一化。
坑 3：ulist.np 需要 Referer: https://quote.eastmoney.com/，否则可能被拒。

六、三源热度交叉验证流程

单一平台的热度数据可能存在算法偏差或信息滞后。建议用以下流程做交叉验证：

# 1. 取同花顺 + 东财双源 TOP N
ths_top = {s["code"]: s["rank"] for s in ths_hot_list("day")[:50]}
em_top  = {s["code"]: s["rank"] for s in em_hot_rank(50)}

# 2. 计算共振集
both = set(ths_top) & set(em_top)           # 双源一致看好
ths_only = set(ths_top) - set(em_top)       # 仅同花顺热（可能偏概念驱动）
em_only  = set(em_top) - set(ths_top)       # 仅东财热（可能偏散户驱动）

# 3. 对共振票做概念归因
for code in list(both)[:5]:
    concepts = em_hot_concept(code)
    # 概念命中 + 互动易是否有最新回应 → 综合判断热度质量
信号组合含义可信度
双源共振 + 概念命中 > 2 个多概念驱动、市场共识强高
双源共振 + 无概念命中可能为事件驱动（财报/公告），非概念轮动中
仅同花顺热可能是同花顺算法加权，需核实低
仅东财热 + rank_chg 大幅上升可能是突发异动，值得跟踪中
