五、limit_up_sentiment() 情绪指标速查

输出字段计算方式含义健康区间
date输入的日期参数分析日期—
zt_countlen(em_zt_pool)涨停家数>30（活跃市场）
zb_countlen(em_zb_pool)炸板家数越少越好
dt_countlen(em_dt_pool)跌停家数<5（无恐慌）
break_ratezb/(zt+zb)*100炸板率%<20%优 / 20-40%中性 / >40%危险
max_heightmax(涨停池 limit_days)市场最高连板≥3（有龙头）
ladder按 limit_days 分组计数连板梯队无断层（连续晋级）

晋级率自算公式： sum(1 for s in em_yzt_pool(date) if s["pct"] >= 9.8) / len(em_yzt_pool(date))

六、已知坑点清单

编号坑点表现修复
P1 价格未 ÷1000 price=25800 而非 25.80 代码中统一 p["p"] / 1000
P2 非交易日 data=null 返回空列表（不抛异常） 确认 date 为交易日；helper 已兜底
P3 同花顺时间格式 first_limit_up_time 是 Unix 秒时间戳，不是 6 位整数 datetime.fromtimestamp(int(ft))
P4 金额单位混淆 封板资金=50000000 是元，不是万元 换算亿元用 / 1e8
P5 _fmt_zt_time 未补零 93000→"93:00:0" 而非 "09:30:00" str(t).zfill(6) 强制 6 位
P6 同花顺 field 勿改 修改 field 参数会导致返回字段错乱 照抄 SKILL.md 的 field 值

八、典型调用链

# 场景 A：快速打板情绪扫描
s = limit_up_sentiment("20260626")
print(f"炸板率{s['break_rate']}% 最高{s['max_height']}板 涨停{s['zt_count']}家")

# 场景 B：题材热度归因
from collections import Counter
reasons = Counter()
for it in ths_limit_up_pool("20260626"):
    for r in it["reason"].split("；"):
        reasons[r.strip()] += 1
print(reasons.most_common(5))  # Top 5 热点题材

# 场景 C：龙头股筛选（连板≥3 + 零炸板 + 封板率>80%）
zt = em_zt_pool("20260626")
ths_map = {it["code"]: it for it in ths_limit_up_pool("20260626")}
candidates = [s for s in zt
              if s["limit_days"] >= 3
              and s["break_times"] == 0
              and ths_map.get(s["code"], {}).get("seal_rate", 0) > 0.8]

# 场景 D：晋级率计算
yzt = em_yzt_pool("20260626")
up = sum(1 for s in yzt if s["pct"] >= 9.8)
print(f"晋级率: {up}/{len(yzt)} = {up/len(yzt)*100:.1f}%")
