同花顺涨停揭秘与打板情绪速算

L1 模块导览 打板层 第 2/2 节 估计时长：14 分钟

1. 为什么需要同花顺？东财四池缺了什么？

东财四池提供了涨停板的结构化数据——谁涨停了、封板时间、连板数、炸板次数。但它缺少两个关键维度：为什么涨停（题材归因）和封板质量（封板成功率、板型分类）。这就是同花顺涨停揭秘做的工作——它是四池的增强源，而非替代源。

双源分工模型 东财 push2ex（四池） 同花顺 data.10jqka.com.cn（涨停揭秘） ├── 涨停列表 + 连板信息 ├── 涨停原因题材（如 "机器人概念"） ├── 封板时间 + 封板资金 ├── 封板成功率（0~1 连续值） ├── 炸板次数 + 振幅 ├── 板型分类（一字板/换手板/T字板） └── 行业归属 └── 封单额 + 回封标记
选源口诀：拉列表找东财，判题材看同花顺，算情绪两个都要。

2. ths_limit_up_pool(date) — 同花顺涨停揭秘

这个函数请求 data.10jqka.com.cn 的涨停揭秘接口，返回每只涨停股的10 个字段。与东财四池不同，它走普通的 requests.get()，不经 em_get() 限流——因为同花顺有自己独立的限流策略。

字段含义示例用途
code / name股票代码 / 名称600519 / 贵州茅台标识
reason涨停原因题材"机器人概念；工业母机"题材归因
board_type板型分类换手板 / 一字板 / T字板判断封板强度
seal_rate封板成功率0.85（85%）封板质量
seal_amount封单额（元）250000000封板资金量
high_days几天几板"2天2板"与东财 zt_stat 对照
first_time首次涨停时间"09:32:15"封板时间判断
break_times炸板次数0封板稳定性
is_again是否回封0 或 1回封标记
pct涨跌幅%10.02涨幅验证

关键参数：field 那串数字是什么？

同花顺接口的 field 参数是一串逗号分隔的内部字段 ID（如 199112,10,9001,330323,...），对应返回数据中 info 数组里每个元素的键。这是同花顺自定义的内部编码，没有公开文档。直接用 SKILL.md 里的值即可，无需修改。

filter=HS,GEM2STAR 控制板块范围：HS = 沪深主板，GEM = 创业板，STAR = 科创板。如果你只想看主板涨停，可以改为 filter=HS。

最大的坑：first_limit_up_time 是 Unix 秒时间戳！与东财返回的 6 位整数不同，同花顺的 first_limit_up_time 是 Unix 秒时间戳（如 1719388800），必须用 datetime.fromtimestamp(int(ft)) 转换，而不是 _fmt_zt_time() 那种切分字符串的方式。如果你用错了转换方式，会得到一个错误的日期。 
# 同花顺时间转换：Unix 秒时间戳 → HH:MM:SS
from datetime import datetime

ft = 1719388800  # 示例 Unix 时间戳
time_str = datetime.fromtimestamp(ft).strftime("%H:%M:%S")
# → "09:32:15"

# 对比：东财时间转换（6位整数 → HH:MM:SS）
def _fmt_zt_time(t):
    s = str(t).zfill(6)
    return f"{s[0:2]}:{s[2:4]}:{s[4:6]}"
# 92500 → "09:25:00"

3. limit_up_sentiment(date) — 打板情绪速算

有了四池的数据，下一步是把数据转化为决策信号。limit_up_sentiment() 是一个轻量级计算函数，它组合涨停、炸板、跌停三池数据，输出 7 个核心情绪指标：

指标计算方式含义
zt_countlen(涨停池)涨停家数
zb_countlen(炸板池)炸板家数
dt_countlen(跌停池)跌停家数
break_ratezb / (zt + zb) * 100炸板率%(越高说明封板越弱)
max_heightmax(涨停池中 limit_days)市场最高连板高度
ladder按 limit_days 分组计数连板梯队 {1板:N, 2板:M, ...}
# 打板情绪温度计示例输出
s = limit_up_sentiment("20260626")
print(f"涨停{s['zt_count']} 炸板{s['zb_count']}(炸板率{s['break_rate']}%) "
      f"跌停{s['dt_count']} 最高{s['max_height']}连板")
print(f"连板梯队: {s['ladder']}")
# 输出示例：
# 涨停48 炸板12(炸板率20.0%) 跌停5 最高6连板
# 连板梯队: {1: 32, 2: 10, 3: 4, 4: 1, 6: 1}
炸板率读法：炸板率 20% 以下 = 市场封板意愿强，赚钱效应好。炸板率 40% 以上 = 封板困难，打板风险高。连板梯队如果有断层（如 1板→3板 直接跳过 2板），说明晋级失败，情绪可能转弱。

晋级率自算方法

晋级率（昨日涨停今日继续涨停的比例）不在 limit_up_sentiment() 中直接提供，但你可以用 em_yzt_pool() 自算： 

# 自算晋级率
yzt = em_yzt_pool("20260626")  # 昨日涨停，今日表现
up_count = sum(1 for s in yzt if s["pct"] >= 9.8)  # 今天继续涨停（含四舍五入）
advance_rate = round(up_count / len(yzt) * 100, 1) if yzt else 0
print(f"晋级率: {advance_rate}% ({up_count}/{len(yzt)})")

4. 实战场景：三池组合打出完整决策链

实际打板决策中，你几乎永远需要组合调用。下面展示一个典型的盘中打板决策流程： 

# ── 实战：打板情绪 + 题材热点 + 封板质量三位一体 ──
date = "20260626"

# 1. 情绪温度：判断今天能不能打
s = limit_up_sentiment(date)
can_trade = s["break_rate"] < 30 and s["zt_count"] > 30
print(f"情绪判断: {'可打板' if can_trade else '回避'} "
      f"(炸板率{s['break_rate']}% 涨停{s['zt_count']}家 最高{s['max_height']}板)")

# 2. 题材归因：找主流热点
ths = ths_limit_up_pool(date)
reasons = {}
for it in ths:
    for r in it["reason"].split("；"):
        reasons[r.strip()] = reasons.get(r.strip(), 0) + 1
hot = sorted(reasons.items(), key=lambda x: x[1], reverse=True)[:3]
print(f"今日三大热点: {hot}")

# 3. 选股：找龙头 + 封板质量高
zt = em_zt_pool(date)
candidates = [s for s in zt if s["limit_days"] >= 2 and s["break_times"] == 0]
print(f"连板且零炸板: {len(candidates)} 只")
for c in candidates[:5]:
    ths_info = next((t for t in ths if t["code"] == c["code"]), {})
    print(f"  {c['name']} {c['zt_stat']} "
          f"封板率{ths_info.get('seal_rate','?')} "
          f"题材:{ths_info.get('reason','?')}")
决策环节用哪个函数看什么指标
能不能打limit_up_sentiment()炸板率、涨停家数、连板高度
打什么方向ths_limit_up_pool()涨停原因题材（归因聚合）
打哪个票em_zt_pool() + ths连板数、炸板次数、封板成功率
风险控制em_dt_pool()跌停家数、连续跌停天数

综合练习

尝试在脑中模拟一次打板决策（不看文档）：

  1. 炸板率 45%、涨停 15 家、最高 2 连板——这个市场适合打板吗？为什么？
  2. 同花顺返回 first_limit_up_time=1719388800，对应北京时间几点几分？（提示：这是 2024 年 6 月某天的秒时间戳）
  3. 连板梯队是 {1: 25, 3: 3, 5: 1}，缺少了什么？这说明什么？
编程任务：写一个函数 advance_rate()，输入日期，返回昨日涨停今日继续涨停的比例（晋级率）。伪代码即可，想想用哪个 pool 函数。
参考答案：
1. 不适合打板。炸板率 45%（约一半涨停被砸开）说明封板困难；涨停仅 15 家说明市场赚钱效应弱；最高仅 2 板说明没有市场龙头，打板容错率极低。
2. 1719388800 对应 2024-06-26 09:00:00（北京时间 UTC+8）。Unix 秒时间戳是从 1970-01-01 00:00:00 UTC 开始计算的秒数。
3. 缺少 2 板和 4 板，出现了"晋级断层"——2 板的股票晋级 3 板全部失败，4 板的股票也全军覆没。说明接力资金不足，昨日涨停的票今天大多没能继续涨停。
编程任务参考：用 em_yzt_pool(date) 获取昨涨停今日表现列表，统计 pct >= 9.8 的数量除以总数。
