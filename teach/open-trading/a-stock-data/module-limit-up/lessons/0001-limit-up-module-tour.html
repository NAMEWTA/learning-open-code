东财涨停板四池 — 涨停、炸板、跌停、昨涨停数据拉取

L1 模块导览 打板层 第 1/2 节 估计时长：14 分钟

1. 打板层定位与 push2ex 源

打板层是 V3.3.0 新增的三层之一，专门覆盖涨停板全维度数据。它的数据源走 push2ex.eastmoney.com——与现有的东财 push2 同源，已纳入 em_get() 统一限流。全部免登录、零鉴权，是一条纯 HTTP GET 的数据通道。

打板层数据源架构 │ ├── 东财 push2ex ──── 涨停池 / 炸板池 / 跌停池 / 昨涨停池（四池） │ └── _em_zt_api(endpoint, sort, date) ← 共用请求 helper │ └── 同花顺 data.10jqka.com.cn ──── 涨停揭秘（题材归因 + 封板成功率） └── ths_limit_up_pool(date) ← 增强源（第 2 节课）
核心事实：四池共用同一个 API 端点模式——push2ex.eastmoney.com/{endpoint}，参数中仅 endpoint 和 sort 不同，dpt=wz.ztzt 和 ut 令牌四池完全相同。

2. 共用 Helper：_em_zt_api()

_em_zt_api(endpoint, sort, date) 是四池的唯一入口函数，封装了请求构造、异常处理和数据提取。它就是一个三参数的请求模板：

参数含义四池取值
endpointAPI 路径尾部getTopicZTPool / getTopicZBPool / getTopicDTPool / getYesterdayZTPool
sort排序字段:方向fbt:asc（涨停/炸板）/ fund:asc（跌停）/ zs:desc（昨涨停）
date交易日YYYYMMDD 字符串，如 "20260626"

固定参数：ut=7eea3edcaed734bea9cbfc24409ed989（东财令牌）dpt=wz.ztzt（板块标识）Pageindex=0、pagesize=10000（一次拉取全部）。所有请求走 em_get() 限流。

坑点：非交易日 data 字段返回 null，helper 通过 (r.json().get("data") or {}).get("pool") or [] 兜底为空列表，不会抛异常。如果你传了非交易日日期，拿到空列表是正常的，不是 bug。 
# _em_zt_api 核心逻辑（简化）
def _em_zt_api(endpoint, sort, date):
    url = f"https://push2ex.eastmoney.com/{endpoint}"
    params = {"ut": ZTB_UT, "dpt": "wz.ztzt", "Pageindex": 0,
              "pagesize": 10000, "sort": sort, "date": date}
    r = em_get(url, params=params, headers=headers, timeout=10)
    return (r.json().get("data") or {}).get("pool") or []

3. 四池逐一拆解

3.1 em_zt_pool(date) — 涨停池

返回当日涨停股票列表，14 个关键输出字段。连板数（limit_days）、几天几板（zt_stat 如 "2天2板"）、封板资金（seal_fund 元）、首末封板时间、炸板次数、所属行业——打板必需的全维度快照。 

for s in em_zt_pool("20260626")[:3]:
    print(f"{s['name']} | {s['zt_stat']} | "
          f"封板{s['seal_fund']/1e8:.2f}亿 | "
          f"首封{s['first_seal']} 末封{s['last_seal']} | "
          f"炸板{s['break_times']}次 | {s['industry']}")

3.2 em_zb_pool(date) — 炸板池

涨停后开板的股票池，额外提供涨停价（limit_price）、振幅（amplitude）和涨速（speed）。炸板池的 sort=fbt:asc（按首次封板时间升序），与涨停池共用同一个排序字段。

3.3 em_dt_pool(date) — 跌停池

跌停池的 sort=fund:asc（按封单资金升序），字段侧重点不同：封单资金（seal_fund）、连续跌停天数（dt_days）、开板次数（open_times）、板上成交额（board_amount）。

3.4 em_yzt_pool(date) — 昨涨停池

昨日涨停、今日表现如何？用来算晋级率和赚钱效应。关键字段：今日涨幅（pct）、昨封板时间（y_first_seal）、昨连板数（y_limit_days）。sort=zs:desc（按涨速降序，把涨得好的排前面）。

池子endpointsort独有字段核心用途
涨停池getTopicZTPoolfbt:asc连板数、封板资金、首/末封时间打板标的筛选
炸板池getTopicZBPoolfbt:asc涨停价、振幅、涨速炸板票回封判断
跌停池getTopicDTPoolfund:asc封单资金、连续跌停天数、板上成交额风险预警
昨涨停池getYesterdayZTPoolzs:desc昨封板时间、昨连板数晋级率/赚钱效应

4. 两个关键坑点和一个辅助函数

坑点 1 — 价格要 ÷1000：东财返回的 price 和 limit_price 是原始值 ×1000 的整数。例如实际涨停价 25.80 元，接口返回 25800。代码中统一 p["p"] / 1000 处理，如果你直接拿来用会差三个数量级。
坑点 2 — 时间格式转换：接口返回的首封/末封时间是 6 位整数，如 92500 表示 09:25:00。_fmt_zt_time() 用 str(t).zfill(6) 补零后切分为 HH:MM:SS。如果返回值不到 6 位（如 93000→093000），不补零会切成 "93:00:0"。 
def _fmt_zt_time(t) -> str:
    """涨停板时间整数 → HH:MM:SS（92500 → 09:25:00）。"""
    s = str(t).zfill(6)
    return f"{s[0:2]}:{s[2:4]}:{s[4:6]}"

# 示例
print(_fmt_zt_time(92500))   # 09:25:00
print(_fmt_zt_time(93000))   # 09:30:00
print(_fmt_zt_time(145500))  # 14:55:00
设计原则：金额单位统一为元（不是万元），与其他层保持一致。封板资金 50000000 = 0.5 亿，换算时 seal_fund / 1e8 得到亿元。

参考答案：
1. endpoint 控制访问哪个池（ZT/ZB/DT/YZT），sort 控制排序字段和方向，date 指定交易日。dpt 四池都是 wz.ztzt，完全相同。
2. 涨停池和炸板池都是 fbt:asc（首封时间升序），昨涨停池是 zs:desc（涨速降序），跌停池是 fund:asc（封单资金升序）。
3. 25.80 元。因为东财返回的价格是原始值 ×1000，需要 ÷1000 恢复为实际价格。
4. 可能原因：(a) 传入的 date 是非交易日（周末/节假日），东财返回 data=null；(b) 当天确实没有涨停股票（极端情况如熔断日）。
