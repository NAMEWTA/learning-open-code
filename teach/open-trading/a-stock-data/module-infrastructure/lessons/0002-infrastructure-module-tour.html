1. em_get() — 东财防封的三件套

em_get() 是整个 a-stock-data 中调用频率最高的基础设施函数。所有东财接口（datacenter、push2、reportapi 等）都通过它发起请求。它自动实现了防封铁律的前三条：

一、串行限流 + 随机抖动：每次请求前检查距离上次调用的间隔。不足 1 秒则 sleep 补偿，再叠加 0.1-0.5 秒的随机抖动。这保证了 QPS 永远不会超过 2，且请求间隔不呈现规律性——反爬系统很难识别。

二、会话复用：全局 EM_SESSION = requests.Session() 对象持久化 TCP 连接，避免每次请求的 TLS 握手开销，同时减少服务端看到的"新连接"数量。

三、连接级自动重试：通过 urllib3 的 HTTPAdapter+Retry 配置，对 429/5xx 做最多 3 次指数退避重试（backoff_factor=0.6）。注意 403 不重试——因为 403 是东财风控信号，重试只会加速封禁。 

def em_get(url, params=None, headers=None, timeout=15, **kwargs):
    """东财统一请求入口：自动节流 + 复用 session + 默认 UA"""
    wait = EM_MIN_INTERVAL - (time.time() - _em_last_call[0])
    if wait > 0:
        time.sleep(wait + random.uniform(0.1, 0.5))  # 限流 + 抖动
    try:
        return EM_SESSION.get(url, params=params,
                              headers=headers, timeout=timeout, **kwargs)
    finally:
        _em_last_call[0] = time.time()                 # 更新时间戳
批量任务调参：默认 EM_MIN_INTERVAL=1.0 秒。当需要循环 100 只股票逐个拉数据时，建议调到 1.5-2 秒。换算：100 只 × 1.5s = 2.5 分钟，安全通过 5 分钟 ≤300 次的阈值。

2. tdx_client() — 绕过 mootdx 0.11.x 的 BESTIP 陷阱

mootdx 是通达信行情的 Python 客户端。0.11.x 版本存在一个隐蔽 bug：全新安装后首次调用 Quotes.factory(market='std') 时，BESTIP.HQ 配置项是空字符串（不是缺失），导致内部拆包失败抛出 ValueError。

为什么不能锁版本？锁 mootdx==0.10.12 在 Python 3.9 下会因 numpy/pandas 二进制不兼容直接崩溃。正确做法是 a-stock-data 的 tdx_client()——用显式 server 参数绕过 BESTIP 机制。

tdx_client() 三级 fallback 流程 _probe() 顺序探测 _TDX_SERVERS 列表 │ ├── 有可达服务器 → Quotes.factory(server=(ip, port)) ✅ 最优路径 │ └── 全部不可达（海外网络） │ ├── fallback 1: Quotes.factory(bestip=True) # 让mootdx自己测速 │ └── 失败（老用户config已填充时成功） │ ├── fallback 2: Quotes.factory(market='std') # 裸调用 │ └── 失败 → RuntimeError("所有服务器不可达") │ └── 明确报错，不挂起、不死等

_TDX_SERVERS 是 10 个实测可用的通达信服务器（IP + 7709 端口），按延迟排序。_probe() 用 TCP 3 次握手（timeout=2s）快速判断可达性，而非等待应用层响应超时。 

# _probe() — 只做 TCP 握手，2 秒超时
def _probe(ip, port, timeout=2.0):
    try:
        with socket.create_connection((ip, port), timeout=timeout):
            return True
    except Exception:
        return False

# 创建客户端 — 一行替代 Quotes.factory()
client = tdx_client()  # 自动三级 fallback，不抛 BESTIP bug

3. get_prefix() + ticker 归一化

这是最简洁但影响最广的基础设施函数。所有数据层的端点都需要区分上证（sh）、深证（sz）、北证（bj）： 

def get_prefix(code: str) -> str:
    if code.startswith(("6", "9")):     # 6xxxxx 上证主板 / 9xxxxx 上证B股
        return "sh"
    elif code.startswith("8"):           # 8xxxxx 北交所
        return "bj"
    else:                                 # 0xxxxx 深证主板 / 3xxxxx 创业板
        return "sz"

Ticker 格式归一化在 get_prefix 调用前完成——剥离 SH/SZ/BJ 前缀和 .SH/.SZ 后缀，统一为纯 6 位数字：

输入格式归一化结果说明
688017688017纯数字 → 直接使用
SH688017688017带前缀 → 剥离 SH
688017.SH688017带后缀 → 剥离 .SH
SZ000001000001深证前缀 → 剥离 SZ
BJ832000832000北证前缀 → 剥离 BJ

4. eastmoney_datacenter() — 一个函数覆盖 6 类数据

东财数据中心（datacenter-web.eastmoney.com）是龙虎榜、限售解禁、融资融券、大宗交易、股东户数、分红送转六类数据的共用入口。它们的 API 结构完全相同——同一 base URL、同一参数格式，只是 reportName 不同：

reportName 值数据类型属于哪层
RPT_DAILYBILLBOARD_DETAILSNEW龙虎榜（个股席位 + 全市场）L3 信号层
RPT_LIFT_STAGE限售解禁日历L3 信号层
RPTA_WEB_RZRQ_GGMX融资融券明细L4 资金/筹码层
RPT_DATA_BLOCKTRADE大宗交易L4 资金/筹码层
RPT_HOLDERNUMLATEST股东户数变化L4 资金/筹码层
RPT_SHAREBONUS_DET分红送转历史L4 资金/筹码层

统一的好处：所有 6 类数据走同一套 columns/filter/sort 参数体系，通过 em_get() 自动限流。新增一种数据类型只需多加一个调用，不增加维护成本。

参考答案：
1. 403 是东财风控信号（主动拒绝），重试无益反而加重封禁风险。正确做法是降频——调大 EM_MIN_INTERVAL 或增加 sleep。
2. 第一级——probe 探测后用 server=(ip, port) 显式传参，完全绕过 BESTIP.HQ 空串。第二、三级只在第一级全部失败时作为兜底。
3. get_prefix("000001") → "sz"（0 开头，深证）。get_prefix("688017") → "sh"（6 开头，上证）。
4. reportName="RPT_DATA_BLOCKTRADE"，查询大宗交易明细。
