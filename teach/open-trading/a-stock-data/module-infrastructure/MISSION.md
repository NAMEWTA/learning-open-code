# 使命：a-stock-data 基础设施与核心机制

## 为什么
学习者在完成 L0 项目总览后，已经了解 a-stock-data 的 10 层架构全貌。但要真正使用这个工具包获取 A 股数据，必须先掌握基础设施层的三大核心机制——东财限流防封（em_get）、mootdx 连接容错（tdx_client）、市场前缀归一化（get_prefix）。这些是「写在第一页」的代码，被所有 10 个数据层依赖，不理解它们就去调用上层函数，轻则请求被封 IP，重则因 mootdx 0.11.x 的 BESTIP bug 直接崩溃。基础设施层是实现 a-stock-data「东财防封」和「mootdx 兼容」两大设计哲学的落地代码，学透它能让你安全、稳定地跑通所有后续数据接口。

## 成功的样子
- 能说出 13 个数据源的封 IP 风险分级，理解「mootdx/腾讯优先、东财仅用于独有数据」的决策逻辑
- 能写出 em_get() 的限流三件套（串行间隔+抖动、会话复用、连接重试），知道批量任务时如何调参
- 能画出 tdx_client() 的三级 fallback 流程图，解释「为什么不能锁版本解决 0.11.x bug」
- 能用 get_prefix() + ticker 归一化规则处理任意格式的股票代码输入
- 知道 eastmoney_datacenter() 如何用一个函数覆盖龙虎榜/解禁/两融/大宗/股东/分红 6 类数据

## 约束条件
- 学习时间：每节短课 15 分钟，共 2 节
- 基础要求：已掌握 L0 项目总览的 10 层架构、5 大设计哲学、源码结构
- Python 基础：理解 requests.Session、函数参数默认值、try/except 异常处理

## 不在范围内
- 不涉及各数据层的具体端点参数（留给 L1 模块课程）
- 不涉及东财 API 的逆向分析和 URL 发现过程
- 不涉及 mootdx 协议层二进制报文解析
