单票估值流程地图

L2 垂直切片 单票估值 第 1/2 节 估计时长：12 分钟

1. 这条链路在做什么

输入一个股票代码，30 秒内拿到完整估值画像。full_valuation(code) 是 a-stock-data 中最具实战价值的垂直切片——它串联了行情层、研报层、基础数据层，把碎片化数据融合为投资决策所需的核心指标。

核心思想：估值不是单一数字，而是一个指标体系——实时价告诉你市场在付多少钱，一致预期 EPS 告诉你机构认为能赚多少，PEG 和消化年数告诉你这个价格是否合理。

返回结果示例：

{
  "name": "绿的谐波", "price": 224.12, "mcap_yi": 410.88,
  "pe_ttm": 300.45, "pb": 11.51,
  "eps_cur": 2.85, "eps_next": 4.92,
  "pe_fwd": 78.6, "cagr_pct": 73.0,
  "peg": 1.08, "digest_years": 1.7,
  "analyst_count": 15
}

3. 逐层详解

1 行情层 — tencent_quote 拿实时价

第一层调用腾讯财经 HTTP 接口 qt.gtimg.cn。按股票代码前缀（sh/sz/bj）拼接 URL，一次请求拉回 gbk 编码文本，按 ~ 分割取值。

关键字段：当前价（索引 3）、PE(TTM)（索引 39）、PB（索引 46）、总市值亿（索引 44）。注意索引 43 是振幅%不是 PB——这是一个常见的陷阱。

为什么选腾讯而不是东财？腾讯接口实测不封 IP，无需限流，可以直接高频调用。a-stock-data 的设计原则是「能用腾讯/通达信的就不用东财」。
字段索引含义示例
当前价3实时成交价224.12
PE(TTM)39滚动市盈率300.45
总市值(亿)44总市值（亿为单位）410.88
PB46市净率11.51

详见：行情层模块导览

2 研报层 — ths_eps_forecast 拿一致预期

第二层直连同花顺 basic.10jqka.com.cn，获取机构对该公司未来年度的 EPS 预测。用 pd.read_html 解析 HTML 表格，搜索包含「每股收益」或「均值」的表格。

V3.2.5 重要修复：同花顺表格列序不固定。旧版代码用 iloc[2] 取「最小值」列当成一致预期，导致前向 PE/PEG 系统性偏差。新版必须按列名匹配「均值」——这才是机构一致预期。

返回的 DataFrame 结构：

列名含义示例
年度预测财年2025
预测机构数给出预测的机构数量15
最小值最悲观预测2.10
均值⭐ 机构一致预期（这才是要用的）2.85
最大值最乐观预测3.50

详见：研报层模块导览

3 估值计算 — 三个核心公式

拿到 price 和 eps_cur / eps_next 后，串行执行三个计算：

指标公式含义
前向 PEprice / eps_cur基于未来年度盈利的市盈率
CAGReps_next / eps_cur - 1盈利复合增速
PEGpe_fwd / (CAGR * 100)成长估值性价比
消化年数ln(pe_fwd / 30) / ln(1 + CAGR)估值回归 30x 需要多少年
30x PE 锚点：这是 A 股成长股的合理估值重力线。所有行业统一用 30x——它不是从公式推导出来的，而是市场经验的归纳。当期 PE 高于 30x 时用消化年数量化"贵了多少"。

4 汇总输出

最终 full_valuation() 返回一个 dict，包含 14 个字段。其中 pe_fwd、cagr_pct、peg、digest_years 为计算字段，其余来自原始数据。

4. 异常路径：EPS 预测为空

并非所有股票都有机构覆盖。小市值公司、ST 股、新上市股票可能没有分析师给出 EPS 预测。当 ths_eps_forecast() 返回空 DataFrame 时：

处理逻辑：eps_cur 和 eps_next 设为 None → pe_fwd 变为 inf → cagr 为 0 → peg 和 digest_years 不可计算。此时 full_valuation() 仍返回行情数据（price、pe_ttm、pb），但估值字段为 None 或 inf——这是预期行为，不是 bug。

另一个常见异常：同花顺表格存在但列名不匹配。代码中通过 _pick(row, "均值") 按列名模糊匹配，如果匹配失败则 eps_cur 保持 None。
