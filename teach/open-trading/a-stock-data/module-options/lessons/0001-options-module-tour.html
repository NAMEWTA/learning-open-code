ETF期权层模块导览

L1 模块导览 ETF期权层 V3.3.0 新增 估计时长：13 分钟

1. ETF期权层定位：交易所预计算，无需本地BSM

ETF期权层是 V3.3.0 新增的三层之一，覆盖 50ETF(510050)、300ETF(510300)、科创50ETF(588000)、500ETF(510500) 四只标的的期权数据。数据源为新浪财经（hq.sinajs.cn），免费直连，最大的优势是：希腊字母和隐含波动率由交易所预先算好，不需要你在本地实现 BSM 模型。

做过期权开发的都知道，自己写 BSM 要处理无风险利率选取、股息率调整、到期日计算——而这些在新浪源中全部由交易所后端完成，你只需要发起 HTTP 请求即可拿到 Delta/Gamma/Theta/Vega/IV。这是本层相比其他期权数据方案最核心的竞争力。

一句话总结：新浪期权 API = 交易所算好的 T 型报价 + 希腊字母 + IV，你只需要传合约代码。

2. 合约清单 — sina_option_codes()

sina_option_codes(underlying, call) 是期权数据的入口函数。传入标的代码和方向，返回按月份分组的合约代码字典： 

codes = sina_option_codes("510050", call=True)
# 返回: {"06": ["10007923","10007924",...], "07": [...], "09": [...], "12": [...]}
# 第一个 key 即为近月合约

near = list(codes)[0]                        # 近月 "06"
atm  = codes[near][len(codes[near]) // 2]     # 中间档 ≈ 平值附近
参数可选值说明
underlying"510050" / "510300" / "588000" / "510500"默认为 "510050"(50ETF)
callTrue（认购）/ False（认沽）对应新浪 OP_UP / OP_DOWN 前缀

内部流程：先从新浪 StockOptionService.getStockName 获取可用月份列表，再逐月调用 _sina_opt_list() 取合约代码（去掉 CON_OP_ 前缀）。月份参数已从 YYYY-MM 转为 YYMM 格式。

3. T型报价 + 希腊字母：两个互补的视角

T型报价：sina_option_tquote()

T 型报价是期权交易员的标准视图——左侧认购、右侧认沽、中间行权价。API 返回的字段覆盖了报价的核心信息：

字段含义字段含义
bid / bid_vol买一价 / 买一量ask / ask_vol卖一价 / 卖一量
last最新成交价strike行权价
open_interest持仓量pct涨跌幅%
volume / amount成交量 / 成交额prev_close昨收
open / high / low今开 / 最高 / 最低limit_up / limit_down涨停价 / 跌停价

注：返回数据需 ≥43 个字段才有效，否则返回空字典。

希腊字母：sina_option_greeks()

希腊字母衡量期权价格对各风险因子的敏感度。对不熟悉期权的开发者，先记住四个词：

希腊字母一句话含义实战意义
Delta标的价格涨 1 元，期权价格变多少方向判断——认购 Delta>0，认沽 Delta<0
GammaDelta 自身的变化速度Gamma 越大，Delta 越不稳定，需要更频繁调仓
Theta每过一天，期权贬值多少买方每天都在"烧钱"——Theta 通常是负数
Vega波动率涨 1%，期权价格变多少恐慌时 Vega 飙升，做多波动率策略的核心指标
IV市场对未来波动的预期高 IV = 市场恐慌/期权贵；低 IV = 市场平静/期权便宜

函数返回结构：delta/gamma/theta/vega/iv/theory(理论价值)/strike/last/volume。注意 iv 字段是小数（0.1735 = 17.35%），不是百分比数值。

4. 四个必须避开的坑

#坑后果解法
1 新浪返回 GBK 编码 中文乱码，字段解析失败 r.encoding = "gbk"
2 缺少 Referer 请求头 返回 403 Forbidden 必须带 Referer: https://stock.finance.sina.com.cn/
3 希腊字母 raw[1:4] 是空串 Delta/IV 全部错位 v = [raw[0]] + raw[4:] ——必须跳过前 3 个空串
4 iv 是小数（非百分比） 显示错误（如 17.35 写成 1735%） 格式化：f"{g['iv']:.2%}" → "17.35%"

底层请求由 _sina_opt_list(param) 统一处理——向 hq.sinajs.cn 发 GET，GBK 解码后去掉 var hq_str_XXX="..." 外壳，返回逗号分隔的字符串列表。tquote 用 CON_OP_ 前缀，greeks 用 CON_SO_ 前缀，合约代码不同的前缀决定了走哪个数据通道。 

# 完整用法：获取 50ETF 近月平值期权的 T 型报价 + 希腊字母
codes = sina_option_codes("510050", call=True)
near  = list(codes)[0]
c     = codes[near][len(codes[near]) // 2]   # 中间档 ≈ 平值
q, g  = sina_option_tquote(c), sina_option_greeks(c)

print(f"{q['name']} 行权价{q['strike']} 最新{q['last']} 持仓{q['open_interest']:.0f}")
print(f"Delta={g['delta']} Gamma={g['gamma']} Theta={g['theta']} Vega={g['vega']} IV={g['iv']:.2%}")
# → 50ETF购6月2750 行权价2.750 最新0.0852 持仓120350
# → Delta=0.6234 Gamma=2.1521 Theta=-0.0032 Vega=0.0015 IV=17.35%

不查文档，尝试回答（每题 30 秒）：

  1. sina_option_codes() 返回的字典 key 是什么格式？第一个 key 有什么特殊含义？
  2. 希腊字母中，哪个参数衡量"时间衰减"？哪个参数衡量"波动率敏感度"？
  3. 新浪源的最大优势是什么？与本地写 BSM 模型相比省了什么？
  4. 希腊字母接口的 raw[1:4] 是什么？不跳过会导致什么后果？
参考答案：
1. key 是 YYMM 格式（如 "06"、"07"、"09"），第一个 key = 近月合约，期权交易中最活跃的月份。
2. Theta = 时间衰减（每过一天期权贬值多少），Vega = 波动率敏感度（波动率涨 1% 期权价格变多少）。
3. 最大优势是交易所预计算——无需自己实现 BSM 模型、选无风险利率、处理股息率和到期日计算。新浪直接返回交易所算好的 Delta/Gamma/Theta/Vega/IV。
4. raw[1:4] 是 3 个空字符串（新浪数据格式的空字段）。不跳过会导致 Delta 读到空串位置，IV 读到 Delta 位置——所有字段全部错位，数据完全不可用。
