二、_cninfo_orgid() — orgId 映射规则（V3.2.2 修复）

三级查找逻辑

_cninfo_orgid(code)
  │
  ├── 1. 检查 _CNINFO_ORGID_MAP（模块级缓存，首次调用时加载）
  │      ↓ 缓存未初始化
  │      HTTP GET szse_stock.json → 6198 条 code→orgId 映射 → 存入缓存
  │      ↓ 查缓存命中
  │      return orgId
  │
  ├── 2. 缓存未命中 → 硬编码 fallback（仅部分老股票适用）
  │      6 开头 → gssh0{code}
  │      8/4 开头 → gsbj0{code}
  │      其他 → gssz0{code}
  │
  └── 3. 返回结果

四、_cninfo_ts_to_date() 时间戳转换

巨潮 announcementTime 返回 Unix 毫秒整数（13 位），不是秒。转换公式： 

from datetime import datetime

def _cninfo_ts_to_date(ts):
    """毫秒时间戳 → 'YYYY-MM-DD'"""
    if isinstance(ts, (int, float)):
        return datetime.fromtimestamp(ts / 1000).strftime("%Y-%m-%d")
    return str(ts)[:10] if ts else ""

# 示例：1719504000000 → 2024-06-28

常见踩坑：如果直接用 datetime.fromtimestamp(ts) 不除以 1000，得到的日期是 50000 多年后的未来时间（毫秒按秒算），排查时容易一头雾水。

七、公告层与新闻层协作示例

# 场景：检测某股是否有减持风险
code = "603259"  # 药明康德

# Step 1：巨潮查减持相关公告
anns = cninfo_announcements(code, page_size=20)
jianchi = [a for a in anns if "减持" in a["title"]]
for a in jianchi:
    print(f"[减持] {a['date']} {a['title']}")
    print(f"  详情: {a['url']}")

# Step 2：如果发现有减持公告，去新闻层看市场反应
# → 东财个股新闻 eastmoney_stock_news(code)
# → 看媒体对减持的解读和情绪面影响

# Step 3：快照查看（不走 HTTP）
# → mootdx F10 最新提示，确认是否有新公告
