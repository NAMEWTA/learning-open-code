# 使命：公告层 — 巨潮公告全文检索+mootdx F10摘要双源体系

## 为什么
学习者需要在 A 股投研中获取上市公司公告——这是基本面分析、事件驱动策略、风险排查的核心数据源。能够通过 Python 直接检索沪深北全量公告原文、按日期翻页、下载 PDF，并理解巨潮 cninfo 的 orgId 查询机制（避免 601xxx 等大量股票查不到公告的坑）。

## 成功的样子
- 能用 cninfo_announcements() 检索任意 A 股的公告全文，按 page_size 翻页，拿到标题/类型/日期/详情链接
- 理解 _cninfo_orgid() 的动态映射机制——为什么不能硬编码 gssx0{code}，以及 szse_stock.json 如何解决 601xxx 公告缺失
- 能用 mootdx F10 快速获取最新公告摘要，作为巨潮的轻量补充
- 能将公告层与新闻层组合使用，形成完整的个股舆情覆盖

## 约束条件
- 学习时间：15 分钟内完成本课
- 基础要求：已掌握 L0 项目总览中的 10 层架构；理解 tdx_client() 和 em_get() 的作用
- 环境：已安装 mootdx、requests，了解 HTTP POST 请求基本用法

## 不在范围内
- 不涉及公告 PDF 的二进制下载与存储细节（由 SKILL.md 的 download_pdf() 覆盖）
- 不涉及巨潮公告的全文文本解析/NLP 提取
- 不涉及董秘问答、互动易等舆情互动层（归 module-sentiment 覆盖）
