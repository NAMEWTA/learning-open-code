# 使命：基础数据层 — mootdx财务快照+F10、东财个股信息、新浪财报三表

## 为什么
学习者在掌握行情层（L1）和基础设施层后，已经能获取实时价格、K线、PE/PB 估值。但要进行基本面分析和财务选股，必须深入基础数据层（L6）——掌握季报快照的 37 个财务字段含义、F10 公司文本资料的九大类结构、东财个股基本信息、以及新浪财报三表的完整提取流程。基础数据层横跨 mootdx(TCP)、东财(HTTP)、新浪(HTTP) 三大数据源，是整个 a-stock-data 中唯一同时使用 tdx_client() 和 em_get() 的数据层，是连接行情数据与财务分析的桥梁。

## 成功的样子
- 能用 mootdx finance() 拉取任意 A 股的 37 字段季报快照，理解 EPS/ROE/净利润/主营收入等核心字段的含义和选股意义
- 能用 mootdx F10 获取九大类公司文本数据（公司概况、股东研究、财务分析等），并应用截断策略节省 70% token
- 能用 eastmoney_stock_info() 获取行业/总股本/流通股/市值/上市日期等个股基本信息
- 能用 sina_financial_report() 提取资产负债表、利润表、现金流量表的完整财务记录，理解 V3.2.1 修复前后的数据提取差异
- 能将三个数据源的财务数据拼装为一只股票的基本面快照

## 约束条件
- 学习时间：每节短课 15 分钟，共 2 节
- 基础要求：已掌握 L0 项目总览、基础设施层（em_get/tdx_client/get_prefix）、行情层（mootdx K线/盘口）
- Python 基础：理解 requests 库、list/dict 遍历、HTTP 参数编码

## 不在范围内
- 不涉及财务指标的计算与选股策略（如杜邦分析、PEG 计算）
- 不涉及 mootdx 协议层二进制报文解析
- 不涉及新浪/东财 API 的逆向分析过程
- 不涉及 F10 文本的 NER 实体识别或结构化解析
