1. 新浪财报三表 — 补齐 mootdx 快照的历史维度

上节课的 mootdx finance() 只返回单期快照，无法做同比环比分析。新浪财报 API 填补了这个空白——sina_financial_report(code, report_type, num=8) 一次拉取最近 N 期完整财务报表，每条记录含几十个会计科目及其同比数据。

report_type报表名称核心科目示例
"fzb"资产负债表总资产、总负债、股东权益、流动资产、固定资产
"lrb"利润表营业收入、营业成本、净利润、扣非净利润
"llb"现金流量表经营活动现金流、投资活动现金流、筹资活动现金流

代码前缀规则：上证 6 开头 → sh，其余 → sz。这个规则与 get_prefix() 一致，但新浪要求的是带前缀的完整代码如 sh600519。

2. V3.2.1 关键修复 — 旧代码为什么永久返回空？

这是本节最重要的工程洞察。V3.2.0 之前的旧代码直接取 result.data.{lrb}：

旧代码 BUG：r.json()["result"]["data"]["lrb"] — 永远返回 None。因为新浪的实际 JSON 结构是 result.data.report_list（按报告期为键的 dict），不是 result.data.lrb。这个 bug 导致 sina_financial_report() 在 V3.2.0 之前完全不可用。
# V3.2.1 修复后的正确提取逻辑（简化示意） report_list = r.json()["result"]["data"]["report_list"] rows = [] for period in sorted(report_list.keys(), reverse=True)[:num]: obj = report_list[period] # 每期一个对象 rec = {"报告期": f"{period[:4]}-{period[4:6]}-{period[6:8]}"} for it in obj.get("data", []): # 遍历该期的 data 数组 rec[it["item_title"]] = it["item_value"] # 科目名→值 if it.get("item_tongbi"): # 有同比则附上 rec[it["item_title"] + "_同比"] = it["item_tongbi"] rows.append(rec)
设计要点：修复后的代码不依赖任何固定的 key 名（如 "lrb"），而是遍历 report_list 的所有期次键，每期内部再遍历 data 数组的 item_title。这意味着即使新浪新增报表类型，代码也不需要修改。

3. 三表的结构差异与使用场景

三表的科目体系完全不同，不能混用 report_type。以下是选表决策：

分析需求用哪张表关键科目
公司赚了多少钱？利润表 lrb营业收入、净利润、扣非净利润
公司欠了多少债？资产负债表 fzb总资产、总负债、资产负债率
赚的钱是真金白银吗？现金流量表 llb经营活动现金流净额（与净利润对比）
盈利质量检验llb + lrb经营现金流 / 净利润 > 1 说明盈利含金量高
常见踩坑：report_type 参数值必须是 "fzb"/"lrb"/"llb"（新浪拼音缩写），不能写成 balance/income/cashflow。拼写错误不会报错，但返回空列表。

4. 跨源数据拼装 — 构建完整基本面快照

下面是三个数据源的实战组合模式——用 mootdx 拿财务快照、东财拿基本信息、新浪拿历史三表，拼成一只股票的完整基本面画像：

# 跨源基本面快照（伪代码，演示组合模式） from mootdx.quotes import Quotes code = "600519" client = tdx_client() # ① mootdx：最新季报核心财务指标 fin = client.finance(symbol=code) snapshot = { "eps": fin.get("eps"), "roe": fin.get("roe"), "bvps": fin.get("bvps"), "profit": fin.get("profit") } # ② 东财：公司元信息 info = eastmoney_stock_info(code) snapshot.update({ "name": info["name"], "industry": info["industry"], "mcap_yi": info["mcap"] / 1e8, "list_date": info["list_date"] }) # ③ 新浪：最近 4 期利润表（看趋势） lrb = sina_financial_report(code, "lrb", num=4) for r in lrb: print(f"{r['报告期']} 净利润={r.get('净利润')}")

组合原则：mootdx 拿指标（快、单期）、新浪拿趋势（慢、多期）、东财拿元信息（行业/市值/上市日期）。三者互补，各取所长。不要跨源重复获取同一类数据——比如 mootdx 的 eps 和新浪利润表的每股收益是同一个值，选 TCP 通道的 mootdx 更快。

  1. V3.2.0 的 sina_financial_report() 为什么永远返回空？修复后用什么方式提取数据？
  2. 如果要判断一家公司"赚的钱是否真金白银"，应该对比哪两张报表的什么科目？
  3. 新浪财报 API 的 paper_code 参数中，代码 000001 对应的前缀是什么？
  4. 跨源拼装时，为什么优先用 mootdx 拿 eps 而不是新浪利润表？
参考答案：
1. 旧代码直接取 result.data.lrb，但新浪实际结构是 result.data.report_list（按报告期为键的 dict）。修复后改为遍历 report_list 的期次键，每期内部再遍历 data 数组的 item_title 提取科目。
2. 对比现金流量表(llb)的"经营活动现金流净额"与利润表(lrb)的"净利润"。经营现金流/净利润 > 1 说明盈利含金量高。
3. sz——0 开头非上证代码走深圳前缀。
4. mootdx 走 TCP 不封 IP、速度快（单次 TCP 请求），且 finance() 一次拿到 37 个字段。新浪利润表虽然也能拿 eps，但需要解析 JSON 结构，效率低。
