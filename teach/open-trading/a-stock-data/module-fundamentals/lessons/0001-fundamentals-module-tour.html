基础数据层（上）— mootdx 快照 + F10 + 东财个股信息

L1 基础数据层 第 1 节 / 共 2 节 估计时长：14 分钟

1. 基础数据层的定位：不是行情，是公司的"身份证"和"体检表"

行情层（L1）给你的是价格、K线、PE/PB——这些是市场的报价。基础数据层（L6）给你的是公司的财务事实：每股赚多少钱、净资产多少、负债率多高、所属行业是什么、什么时候上市的。它横跨三个数据源——mootdx 走 TCP 不封 IP、东财走 HTTP 有风控、新浪走 HTTP 低风险——是唯一同时依赖 tdx_client() 和 em_get() 的数据层。

核心理解：基础数据层回答"这是一家什么样的公司"，行情层回答"市场现在怎么看这家公司"。两者组合才能做基本面定量分析。

2. mootdx 季报快照 — 37 个字段一次拉取

client.finance(symbol='688017') 是获取财报数据最高效的方式——一次 TCP 请求即可拿到最新季报的全部 37 个财务字段。无需翻财报、无需拼接多源数据。

字段含义选股意义
eps每股收益盈利能力核心指标
bvps每股净资产清算价值参考
roe净资产收益率(%)股东回报率
profit净利润绝对盈利规模
income主营收入营收规模
liutongguben流通股本实际可交易股数
zongguben总股本公司总规模
meigugongjijin每股公积金转增股本潜力
meiguweifeipeili每股未分配利润分红潜力
# mootdx 季报快照调用 from mootdx.quotes import Quotes client = tdx_client() # 使用基础设施层包装函数，规避 0.11.x BESTIP bug fin = client.finance(symbol='688017') # market: 0=深圳, 1=上海 # 返回 37 字段 dict，字段名全为拼音（通达信历史遗留） print(f"EPS={fin.get('eps')} ROE={fin.get('roe')}% 净利润={fin.get('profit')}")
注意：finance() 返回的是最新一期季报快照（单期），不是历史序列。如果需要多期对比，需要结合新浪财报三表（下节课讲）。

3. mootdx F10 — 九大类公司文本资料

F10 是通达信为每只股票维护的九大类结构化文本档案，覆盖公司从基本信息到行业分析的完整资料。调用方式：client.F10(symbol='688017', name='类别名')。

类别内容量级
最新提示近期公告摘要、业绩预告小
公司概况主营业务、所属板块、员工数中
财务分析主要财务指标同比环比中
股东研究十大股东、股东变化历史列表大（16000+ chars）
股本结构股本变动、限售解禁时间表中
资本运作增发、并购、分红记录中
业内点评券商研报摘要大
行业分析行业地位、竞争对手小
公司大事历史重大事件时间线大
token 优化技巧（-70%）："股东研究"中的【4.股东变化】章节含大量历史十大股东列表，实测 16000+ chars。传给 LLM 分析时建议只保留最新一期数据。同理，"公司大事"只需最近 3 年，"业内点评"只需最近 5 条。

4. 东财个股基本信息 — 8 字段直连 push2

eastmoney_stock_info(code) 直连东财 push2 API，返回 8 个个股元信息字段。它不走 em_get()限流——因为单次查询只会调用一次，不构成批量风控风险。但如果要批量查询 100+ 只股票，仍需自行间隔。

# 东财个股基本信息 info = eastmoney_stock_info("688017") # → {code, name, industry, total_shares, float_shares, # mcap, float_mcap, list_date, price} print(f"{info['name']}: 行业={info['industry']} " f"总市值={info['mcap']/1e8:.0f}亿 " f"上市={info['list_date']}") # 字段映射（push2 原生 fXX → 输出 key）: # f57→code f58→name f127→industry f84→总股本 # f85→流通股 f116→总市值 f117→流通市值 f189→上市日期

这 8 个字段在基本面分析中用途明确：行业决定赛道，总股本决定市值规模，上市日期判断次新股/老股。它们与 mootdx 的 37 个财务字段互补——一组是公司"出生证"信息，一组是公司"体检报告"数据。

检索练习

不看屏幕，尝试回答（每题 30 秒）：

  1. mootdx 的 finance() 返回的是单期快照还是历史序列？
  2. F10 的"股东研究"类别为什么需要截断？实测文本量有多大？
  3. mootdx 的 37 个财务字段中，哪 3 个最能反映盈利能力？
  4. eastmoney_stock_info() 为什么可以不走 em_get() 限流？
参考答案：
1. 单期快照——只返回最新一期季报数据。多期对比需结合新浪财报三表。
2. 【4.股东变化】章节含大量历史十大股东列表，实测 16000+ chars。截断只保留最新一期可节省约 70% token。
3. eps（每股收益）、roe（净资产收益率）、profit（净利润）——分别代表每股盈利、股东回报率、绝对盈利规模。
4. 单次查询只调用一次 push2 API，不构成批量风控。批量调用 100+ 只股票时仍需自行加间隔。
