新标的调研 — 各阶段详解与降级策略

L2 垂直切片 第 2 节 / 共 2 节 估计时长：14 分钟

1. 机构覆盖与估值（步骤1~2）

步骤1 — ths_eps_forecast(code)：直连同花顺 basic.10jqka.com.cn，返回机构一致预期 EPS 的年度序列。核心输出：年份、预测机构数、EPS均值/最小/最大。如果 预测机构数 < 3，说明市场关注度极低，均值无统计意义——这是最关键的判断信号。

步骤2 — tencent_quote([code])：腾讯财经 HTTP API 返回 PE_TTM、PB、市值（亿）、换手率。腾讯不封IP，可放心高频调用。结合步骤1的未来 EPS 可估算前向 PE，步骤2的 PE_TTM 除以未来 EPS 增长率可得 PEG（合理区间 < 1.5）。

估值速算公式：PEG = PE_TTM / (未来EPS增长率 × 100)。若 ths_eps_forecast 给出今年EPS=1.0，明年=1.5，增长率=50%，PE=60，则 PEG=60/50=1.2（合理偏高）。无机构覆盖时跳过 PEG，仅记录"无一致预期"。
降级1：无机构覆盖。若 ths_eps_forecast 返回空 DataFrame，标记"无机构覆盖"并跳过 PEG 计算。tencent_quote 仍可正常输出 PE/PB/市值，供人工判断。

2. 概念板块与资金流向（步骤3~4）

步骤3 — eastmoney_concept_blocks(code)：走东财 slist（spt=3），一次请求返回个股所属的全部板块——行业、概念、地域三类，附 BK码、板块涨跌幅、龙头股。典型的强势股可能归属20~30个概念，顶端前10个标签反映最热门的题材归因。

步骤4 — eastmoney_fund_flow_minute(code)：走东财 push2，返回当日分钟级资金流向（超大单、大单、中单、小单）。sum(f["main_net"] for f in flow) 累加主力净流入。正值 = 主力买入主导，负值 = 主力出货。注意这个端点的返回数据量可能较大（240分钟 × 4类资金），解析时注意字段名。

降级2：概念板块为空。极少数新股或停牌股可能无板块归属，标记"板块归属待定"。不要用 baidu PAE 替代（已失效，V3.2.2 已确认）。
降级3：分钟级资金流为空。非交易时段（盘前/盘后/周末）此端点返回空。改用 stock_fund_flow_120d(code) 拉近20日主力净流入累积值作为替代判断。

3. 龙虎榜、解禁与两融（步骤5~7）

步骤5 — dragon_tiger_board(code, date)：走东财 datacenter-web，返回近30日上榜记录 + 买卖席位TOP5。核心关注：上榜频率（活跃度）、买入席位中机构占比（机构动向）、卖出席位是否出现知名游资。

步骤6 — lockup_expiry(code, date)：同走 datacenter-web，返回未来90天待解禁批次。关注解禁股数占总股本比例——超过5%属于大额解禁，可能对股价形成压力。同时注意解禁类型（首发原股东 vs 定增机构）。

步骤7 — margin_trading(code)：走 datacenter-web 的融资融券端点，返回日级融资余额（rzye）和融券余额（rqye）。融资余额上升 = 做多情绪升温；融券余额异常高 = 空头聚集。两者比值反映市场多空倾向。

降级4：龙虎榜无记录。标记"近30日未上榜"而非报错。未上榜可能意味着：冷门股无人关注，或大盘股不走游资路线。
降级5：解禁/两融数据异常。datacenter-web 的返回结构偶有字段缺失（如早期接口版本无 rqye 字段）。处理方法：对缺失字段赋 None 并标注"数据缺失"，不中断后续步骤。

4. 降级策略总览与并行优化

以下表格汇总7步的异常场景与降级动作。核心原则：单点失败不中断全链路。

步骤异常场景降级动作严重度
1无机构覆盖标记"无覆盖"，跳过PEG低
2腾讯返回异常记录错误，继续后续步骤中
3概念板块为空标记"板块待定"低
4非交易时段无分钟流改用120日资金流中
5未上榜标记"近30日未上榜"无
6无待解禁标记"90天内无解禁"无
7字段缺失赋None，标注缺失低
并行加速：步骤2（腾讯）与步骤3（东财 slist）可并发——两者是不同的 HTTP 会话和数据源，没有竞态条件。用 Python concurrent.futures.ThreadPoolExecutor(max_workers=2) 可节省约1秒。
不要并行步骤3~7：步骤3~7全部走东财，共享 em_get() 内部的串行锁。如果对这些端点开多线程，会绕过限流锁、短时间发出大量请求，直接触发东财风控封IP。

检索练习 — 降级判断

以下是4个真实调研场景，请判断应执行哪个降级动作：

  1. 周六下午调研 688017，fund_flow_minute 返回空列表
  2. 002352（顺丰控股）在龙虎榜查不到记录
  3. 301xxx 某次新股 ths_eps_forecast 返回空 DataFrame
  4. lockup_expiry 返回的 upcoming 列表为空
参考答案：
1. 降级3：非交易时段，改用 stock_fund_flow_120d 拉近20日累积数据。
2. 降级4：大盘股不走游资路线，标记"近30日未上榜"即可，非异常。
3. 降级1：次新股尚无机构覆盖是正常的（上市时间短），标记"无机构覆盖"并跳过PEG。
4. 无需降级：90天无解禁是好事（无抛压风险），标记"90天内无解禁"为正面信息。
