# 使命：mootdx 客户端三级 fallback 深度剖析

## 为什么
用户在自己的量化环境中使用 mootdx 获取行情数据时，频繁遇到 ValueError 崩溃或 TCP 超时，但不知道为什么。理解 tdx_client() 的三级兜底机制后，用户能自己诊断 mootdx 连接问题、扩展服务器列表、并在不同网络环境（国内/海外）下稳定获取行情。

## 成功的样子
- 能解释 BESTIP.HQ 空串导致 ValueError 的根因——为什么锁版本不是好方案
- 能手写一个简化版的三级 fallback 函数，理解 TCP 探测、bestip 测速、裸 factory 的兜底顺序
- 遇到 mootdx 连接故障时能快速定位问题层级（是服务器不可达？还是 config 损坏？）

## 约束条件
- 学习时间：30 分钟以内（2 节短课）
- 前提：已理解 L1 基础设施层中 tdx_client() 的基本用法
- 不涉及通达信 TCP 二进制协议的解析细节

## 不在范围内
- 通达信行情协议（7709 端口）的二进制数据帧格式
- mootdx 的其他功能模块（财务、F10）——这些在本课程中仅作为背景提及
- 通达信服务器的架设与维护
