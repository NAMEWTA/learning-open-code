批量估值对比全链路

L2 垂直切片 批量估值 第 1/1 节 估计时长：12 分钟

1. 这条链路在做什么

输入一组股票代码，输出一张横向可比估值表。这是 full_valuation() 的自然扩展——将单票四步估值链路乘以N，再通过DataFrame把结果排成一张可排序、可筛选的对比表。

核心思想：估值不是绝对值而是相对值。单看PE=78x没有意义，放在同一板块4只股票中对比才有信号价值。批量对比的本质是"在多只股票之间建立估值坐标系"。

横向对比表输出示例：

name         code     price   pe_ttm   pe_fwd   cagr_pct   peg     digest_years
绿的谐波      688017   224.1   300.5    78.6     73.0       1.08    1.7
中际旭创      300308   168.5    45.2     32.1     41.0       0.78    0.2
胜宏科技      300476    52.3    28.7     21.5     28.0       0.77    0.0
沪电股份      002463    38.9    35.1     25.8     35.0       0.74    0.0

2. 四步调用链路

与单票估值共享核心管道，不同之处在外层控制流和DataFrame聚合：

stocks = ["688017", "300308", "300476", "002463"] ┌─ for code in stocks ────────────────────────────────────┐ │ │ │ [Step 1] tencent_quote(code) ──→ price, pe_ttm, pb │ │ [Step 2] ths_eps_forecast(code) ──→ eps_cur, eps_next │ │ [Step 3] 估值计算 │ │ [Step 4] 汇总到 dict │ │ │ │ try/except 包裹 → 失败打印告警，不中断循环 │ └──────────────────────────────────────────────────────────┘ ↓ [Step 5] pd.DataFrame(results) → 排序 + 打印
步骤与单票估值的差异关键点
1-4. 逐票估值完全相同，放入 for 循环try/except 保证单票失败不阻塞
5. DataFrame聚合批量对比独有按 pe_fwd 或 peg 排序

3. 关键设计决策

1 为什么用串行而非并行？

同花顺 EPS 接口走的是 HTTP 直连，虽然没有 em_get() 的 1rps 硬限流，但大量并发请求仍然可能触发反爬。对于 N<20 的候选池，串行的总耗时可接受（每票 ~2-3s，10 只股票 30s 内完成）。

设计取舍：简单 > 性能。批量对比的目标是快速决策而非高频回测。如果需要 N>50 的筛选，应该先通过 PE/PB 等行情层指标初筛，再用估值全链路精算——而不是对所有股票跑 full_valuation。

2 异常处理：单票失败 ≠ 流程失败

这是批量对比最重要的工程原则。源码中的模式：

for code in stocks:
    try:
        r = full_valuation(code)
        print(f"{r['name']}: PE_fwd={r['pe_fwd']}x PEG={r['peg']}")
    except Exception as e:
        print(f"{code}: 失败 - {e}")

失败来源有三类：(1) 网络超时，(2) 无机构覆盖导致 EPS 解析异常，(3) 停牌股票行情返回异常。每一类都不应该让其他 3 只股票的结果丢失。

常见陷阱：如果在 DataFrame 构建时没有处理好 None 值，sort_values('pe_fwd') 会把没有分析师覆盖的股票（pe_fwd=None）排到最后或直接报错。建议填充 float('inf') 或先 dropna。

3 排序维度的选择

排序维度适用场景注意事项
pe_fwd 升序找估值洼地低PE可能意味着低增长（价值陷阱）
peg 升序找成长性价比PEG<1 = 低估，PEG>2 = 高估
digest_years 升序找估值消化快的0年 = PE已在30x以下，无需消化
cagr_pct 降序找高增长标的高增长需与PEG结合判断是否透支

4. 输出解读

上例中，绿的谐波 PEG=1.08 处于合理区间（0.5~2.0），消化年数 1.7 年意味着若增长保持 73%，1.7 年后 PE 回归 30x。中际旭创 PEG=0.78 更便宜，消化年数仅 0.2 年——因为自身 PE 已接近 30x。

决策框架：优先排除 PEG>2 或消化年数>5 的股票；然后在 PEG 合理区间内选分析师覆盖最多的（共识更可靠）。最终决策还要结合资金面、行业景气、技术面——估值表给出的是安全边际，不是买卖信号。
