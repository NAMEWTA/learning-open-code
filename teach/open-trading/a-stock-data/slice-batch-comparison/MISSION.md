# 使命：批量估值对比全链路

## 为什么
用户已经掌握了单票估值 `full_valuation()` 的使用，但实际投研中需要同时对多只候选股票做横向估值排列——谁的PE最便宜？谁的增长最快？谁的PEG最有性价比？本主题训练用户将单票估值扩展为批量对比流程，构建可排序、可筛选的多股票估值DataFrame。

## 成功的样子
- 输入一个股票代码列表，能输出一份包含PE前向、CAGR、PEG、消化年数的多股票横向对比表
- 理解批量估值中的异常处理模式：单票失败不阻塞其他股票
- 掌握用Pandas DataFrame做横向排序筛选的基本方法

## 约束条件
- 使用 a-stock-data V3.3.0 内置端点，不依赖外部数据封装库
- 东财端点需遵守 em_get() 串行限流规则（批量场景下注意逐票串行调用）
- 部分股票可能无机构覆盖，eps_cur/eps_next 为空，估值字段为 None 属于正常降级

## 不在范围内
- 单票估值公式的详细推导（已在 slice-single-valuation 中覆盖）
- 多线程/异步并发优化（属于性能进阶主题）
- 行业平均估值基准线的自动拉取
