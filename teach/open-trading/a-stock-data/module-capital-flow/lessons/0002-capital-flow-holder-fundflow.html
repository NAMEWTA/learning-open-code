资金面导览（下）：股东户数、分红送转与主力资金流

L1 资金/筹码层 第 2/2 节 估计时长：14 分钟

1. 筹码集中度 — 股东户数是主力的脚印

如果说融资融券看的是"用了多少杠杆"，大宗交易看的是"机构在暗盘做什么"，那么股东户数变化看的是"股票在谁手里"。这是筹码分析中最核心的指标——股东户数持续减少说明股票在从散户向主力集中（吸筹），反之则是主力在派发筹码给散户（出货）。 

data = holder_num_change("600519")
for d in data[:5]:
    print(f"{d['date']}: 股东数={d['holder_num']} "
          f"变化={d['change_ratio']}% "
          f"户均={d['avg_shares']}股")

# 关键字段解读：
# holder_num   — 股东总户数，越少 = 筹码越集中
# change_num   — 环比变化的户数（正值=散户增多）
# change_ratio — 环比变化百分比（负值=集中，正值=分散）
# avg_shares   — 户均持股数 = 流通股本 / 股东户数

数据特征：这是季度级数据，来自报表 RPT_HOLDERNUMLATEST，按 END_DATE 降序排列。A 股上市公司每季度披露股东名册，所以这条数据每年只有 4 个数据点。低频但高价值——一个季度的筹码变化趋势比每天的K线更能揭示主力意图。

判断口诀：股东户数 ↓ + 户均持股 ↑ = 筹码集中 = 主力吸筹信号。股东户数 ↑ + 户均持股 ↓ = 筹码分散 = 主力出货信号。配合股价位置判断更准：低位集中是吸筹，高位集中可能是对倒。

2. 分红送转 — 股东的现金回报

dividend_history(code, page_size=20) 查询个股的历史分红送转记录。分红是长线投资者最关心的指标之一——它不仅影响持仓成本（除权除息后股价下调），还反映了公司的盈利能力和回报股东意愿。 

data = dividend_history("600519")
for d in data[:5]:
    print(f"{d['date']}: 派息={d['bonus_rmb']}元/股 "
          f"转增={d['transfer_ratio']}送股={d['bonus_ratio']} "
          f"进度={d['plan']}")

# 关键字段：
# bonus_rmb      — 每股派息（税前，元）
# transfer_ratio — 每10股转增比例（公积金转股本）
# bonus_ratio    — 每10股送股比例（利润转股本）
# plan           — 分配进度（预案/股东大会通过/实施/完成）

数据源：报表 RPT_SHAREBONUS_DET，按 EX_DIVIDEND_DATE（除权除息日）降序。page_size 默认 20 条，对大部分股票够覆盖 5-10 年的分红历史。ASSIGN_PROGRESS 字段可以判断当前分红方案处于哪个阶段——预案阶段的分红不一定能最终实施。

交易提示：在除权除息日前买入可享受分红，但除权后股价会等比例下调。短期交易者应关注除权日期避免误判成本；长期投资者更关注每股派息的稳定性和增长趋势。

3. 主力资金流 120 日 — 谁在买，谁在卖

stock_fund_flow_120d(code) 是本层唯一不走 datacenter 的端点——它直连东财 push2his API（push2his.eastmoney.com），拉取最近 120 个交易日的日级资金流向，细分为 5 类资金：超大单、大单、中单、小单，以及汇总的主力净流入。 

data = stock_fund_flow_120d("600519")

# 单日明细
last = data[-1]
print(f"{last['date']}: 主力净流入={last['main_net']/1e4:.0f}万")

# 近20日累计主力净流入
recent = data[-20:]
total = sum(d['main_net'] for d in recent)
print(f"近20日主力累计净流入: {total/1e8:.2f}亿")

# 5类资金拆分：
# super_net — 超大单净流入（>100万/笔）
# large_net — 大单净流入（20-100万/笔）
# mid_net   — 中单净流入（4-20万/笔）
# small_net — 小单净流入（<4万/笔）
# main_net  — 主力净流入 = super + large

数据源与限流：push2his 不走 eastmoney_datacenter()，而是通过 em_get() 直连 push2his.eastmoney.com/api/qt/stock/fflow/daykline/get。market_code 判断：6 开头为上海（market_code=1），其余为深圳（market_code=0）。响应是 CSV 格式的 klines 数组，需手动 split(",") 解析。

间歇风控（#18）：push2his 对部分大陆住宅宽带 IP 有连接级风控，表现为 HTTP 000（连接被拒）或返回空——这不是代码问题。遇到时：① 隔几分钟重试；② 换网络（手机热点）；③ 降频（调大 EM_MIN_INTERVAL）。日级资金流的务实替代：仍可用 mootdx 算量价。

4. 三步完成资金面综合诊断

将本节的 3 个端点与上节的 2 个端点组合，形成完整的资金面 5 维度画像： 

code = "600519"

# 步骤 1：筹码结构（季度级，看大趋势）
holders = holder_num_change(code, page_size=4)
recent_change = holders[0]['change_ratio']
chip_status = "集中" if recent_change < 0 else "分散"

# 步骤 2：主力方向（日级，看短期资金）
funds = stock_fund_flow_120d(code)
main_20d = sum(d['main_net'] for d in funds[-20:])
flow_status = "流入" if main_20d > 0 else "流出"

# 步骤 3：回报评估（历史分红稳定性）
divs = dividend_history(code, page_size=10)
avg_bonus = sum(d['bonus_rmb'] for d in divs) / len(divs)
div_status = "高股息" if avg_bonus > 0.5 else "一般"

# 综合输出
print(f"{code} 资金面画像:")
print(f"  筹码: {chip_status}  主力20日: {flow_status}")
print(f"  近5年每股派息均值: {avg_bonus:.2f}元 → {div_status}")

检索练习

不看屏幕，尝试回答（每题 30 秒）：

  1. 如果某股票股东户数从 10 万降到 8 万（-20%），户均持股从 5000 升到 6250，说明什么？
  2. dividend_history() 的 ASSIGN_PROGRESS 字段有哪几种状态？为什么不能只看预案？
  3. stock_fund_flow_120d() 的 market_code 如何判断？push2his 风控时的 HTTP 000 是代码 Bug 吗？
  4. 近 20 日主力累计净流入 5 亿，但近 5 日每天净流出 5000 万——你会如何解读？
参考答案：
1. 股东户数降 20% + 户均持股升 25% = 筹码从散户向主力集中，典型吸筹信号。需结合股价位置判断是低位吸筹还是高位对倒。
2. 预案 → 股东大会通过 → 实施 → 完成。预案阶段的分红可能被股东大会否决或修改，不能直接等同于最终方案。
3. 代码以 6 开头 → market_code=1（上海），其余 → market_code=0（深圳）。HTTP 000 不是代码 Bug，是大陆住宅 IP 间歇风控，重试或换网络即可。
4. 长期（20日）累计仍为流入，但短期（近5日）方向逆转 → 主力可能在减仓。应关注这个趋势是否持续——如果流出加速，可能是顶部信号。
