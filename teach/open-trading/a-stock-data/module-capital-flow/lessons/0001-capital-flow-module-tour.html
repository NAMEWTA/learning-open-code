资金面导览（上）：融资融券与大宗交易

L1 资金/筹码层 第 1/2 节 估计时长：13 分钟

1. 资金面在架构中的位置

资金面/筹码层是 a-stock-data V3.0 新增的第四层，填补了行情层（K线/盘口）与信号层（题材/席位）之间的关键空白——谁在用真金白银交易。行情告诉你价格变了，资金面告诉你为什么变了。

资金面/筹码层 5 端点全景 │ ├── 杠杆资金 ────── margin_trading() 融资余额/买入/偿还 + 融券（日级） │ 数据源: 东财 datacenter RPTA_WEB_RZRQ_GGMX │ ├── 暗盘交易 ──── block_trade() 大宗交易成交价/量 + 买卖方营业部 + 溢价率 │ 数据源: 东财 datacenter RPT_DATA_BLOCKTRADE │ ├── 筹码集中度 ── holder_num_change() 季度股东户数 + 环比变化 + 户均持股 │ 数据源: 东财 datacenter RPT_HOLDERNUMLATEST │ ├── 分红回报 ──── dividend_history() 每股派息/送股/转增 + 进度状态 │ 数据源: 东财 datacenter RPT_SHAREBONUS_DET │ └── 主力方向 ──── stock_fund_flow_120d() 主力/大单/中单/小单日级净流入（120日） 数据源: 东财 push2his / qt/stock/fflow

所有端点都走东财数据源——datacenter 系列走 eastmoney_datacenter()（内调 em_get），push2his 走 em_get() 直连。这意味着所有端点自动享受基础设施层的串行限流保护，无需手动 sleep。

设计意图：这 5 个端点覆盖了 A 股资金分析的 5 个维度——杠杆资金（融资融券）、暗盘交易（大宗）、筹码结构（股东户数）、回报预期（分红）、主力方向（资金流）。组合使用即可完成个股的"资金面综合画像"。

2. 融资融券 — 杠杆资金的态度

margin_trading(code, page_size=30) 从东财 datacenter 拉取日级融资融券明细。融资余额（rzye）是最核心的指标——它代表有多少杠杆资金正押注这只股票会上涨。融资余额持续攀升 = 多头加杠杆；融资余额快速下降 = 杠杆资金出逃。 

data = margin_trading("600519")
for d in data[:5]:
    print(f"{d['date']}: 融资余额={d['rzye']/1e8:.2f}亿 "
          f"买入={d['rzmre']/1e4:.0f}万 融券={d['rqye']/1e4:.0f}万")

# 关键字段解读：
# rzye    — 融资余额（元），杠杆多头持仓总量
# rzmre   — 融资买入额，当日新增杠杆多头
# rzche   — 融资偿还额，当日平仓的杠杆多头
# rqye    — 融券余额（元），融券做空总量
# rzrqye  — 融资融券余额合计

数据源细节：报表名为 RPTA_WEB_RZRQ_GGMX，通过 eastmoney_datacenter() 调用，filter 用 SCODE 过滤个股，按 DATE 降序排列。page_size 默认 30 条（约一个月交易日），可上调获取更长历史。

注意：融券数据（rqye、rqmcl、rqchl）在 2024 年融券新规后普遍趋近于零，大部分中小盘股的融券余额已无参考价值。重点关注融资数据即可。

3. 大宗交易 — 机构的暗盘定价

block_trade(code, page_size=20) 拉取个股的大宗交易记录。大宗交易是机构之间通过专用通道进行的批量股票转让，不经过竞价市场。它的关键信息不是成交价本身，而是相对于当日收盘价的溢价率——溢价买入说明买方愿意出高价吃货（利好），折价卖出可能是股东减持套现（利空）。 

data = block_trade("600519")
for d in data[:5]:
    print(f"{d['date']}: 成交价={d['price']} "
          f"溢价={d['premium_pct']}% "
          f"买方={d['buyer']} 卖方={d['seller']}")

# 关键字段：
# price       — 大宗成交价
# close       — 当日收盘价
# premium_pct — 溢价率（正=溢价成交，负=折价成交）
# buyer       — 买方营业部名称
# seller      — 卖方营业部名称
# vol/amount  — 成交量/成交额

溢价率计算：代码中手动计算 (deal_price / close - 1) * 100，而不是直接取接口字段——因为 datacenter 返回的原始数据可能不含溢价率，需要自己算。这是 a-stock-data 的设计惯例：关键指标自己算，不依赖上游计算。

营业部信息价值：买卖方营业部名称能透露交易性质。如果买方是"机构专用"席位，且溢价成交，通常是机构建仓信号。如果卖方是"XX证券XX营业部"且大幅折价，可能是大股东减持。

4. 快速诊断：两融 + 大宗组合

融资融券和大宗交易分别从杠杆资金和机构暗盘两个角度反映资金态度。组合使用可以形成初步的资金面判断： 

# 诊断：某股票是否有资金在持续流入？
code = "600519"

# 1. 融资趋势
margin = margin_trading(code, page_size=10)
recent_rz = [m['rzye'] for m in margin[:5]]
rz_trend = "上升" if recent_rz[0] > recent_rz[-1] else "下降"
print(f"融资余额趋势: {rz_trend}")

# 2. 大宗溢价方向
blocks = block_trade(code, page_size=5)
premiums = [b['premium_pct'] for b in blocks]
avg_prem = sum(premiums) / len(premiums) if premiums else 0
print(f"近5笔大宗平均溢价: {avg_prem:.1f}%")

# 3. 综合判断
if rz_trend == "上升" and avg_prem > 0:
    print("→ 杠杆加仓 + 溢价成交，资金面偏多")
elif rz_trend == "下降" and avg_prem < -3:
    print("→ 杠杆撤退 + 大幅折价，资金面偏空")

检索练习

不翻看上面内容，尝试回答：

  1. 融资余额（rzye）上升说明什么？融资买入额（rzmre）和融资偿还额（rzche）的区别是什么？
  2. 大宗交易的溢价率为什么要在代码中手动计算而不是直接取接口字段？
  3. 如果某股票融资余额连续 5 天下降，同时大宗交易全部折价成交，你如何解读？
参考答案：
1. 融资余额上升 = 杠杆资金在加仓，多头情绪增强。rzmre 是当日新开仓的融资买入，rzche 是当日平仓的融资偿还，两者相减得到当日净买入。
2. datacenter 原始返回可能不含溢价率字段，a-stock-data 的设计惯例是"关键指标自己算"。代码用 (deal_price / close - 1) * 100 手动计算。
3. 杠杆资金在撤退（降杠杆）+ 机构间折价转让（可能大股东减持），资金面偏空，需警惕短期下跌风险。
