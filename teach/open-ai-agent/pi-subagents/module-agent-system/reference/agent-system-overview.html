Agent 系统模块参考

本页是 L1 短课的速查资料，覆盖发现入口、来源优先级、管理动作、内置角色、关键依赖和测试证据。短课入口：0001 Agent 发现、配置与内置角色模块。

1. 模块职责与架构位置

维度结论证据
一句话职责把 builtin、package、user、project agent/chain 定义与 settings 覆盖合并成运行时可执行配置。open-ai-agent/pi-subagents/src/agents/agents.ts
上游调用方subagent tool、slash、RPC 或 executor 需要角色时，按 cwd 和 scope 请求 discovery。open-ai-agent/pi-subagents/src/agents/agent-scope.ts
下游消费者foreground/background executor 消费已解析的 AgentConfig，不重新决定来源优先级。teach/open-ai-agent/pi-subagents/00-overview/reference/00-overview.html
角色定义格式Markdown frontmatter 存结构化字段，正文作为 system prompt；chain 可用 .chain.md 或 .chain.json。open-ai-agent/pi-subagents/src/agents/frontmatter.ts、open-ai-agent/pi-subagents/src/agents/chain-serializer.ts
sources ├─ bundled agents/ builtin ├─ installed or settings packages package ├─ user dirs: agent home + ~/.agents user └─ nearest project config dirs project ↓ parse frontmatter / chain json ↓ apply defaultModel + agentOverrides + disable flags ↓ merge by scope ↓ filter disabled ↓ runtime AgentConfig / ChainConfig

2. 发现入口与返回值

入口用途关键行为
discoverAgents(cwd, scope)执行路径使用的主发现函数。按 scope 读取 user/project settings，合并 builtin/package/user/project agents，过滤 disabled。
discoverAgentsAll(cwd)管理面和诊断使用。返回分来源列表、chain 列表、chain diagnostics、user/project 目录与 settings 路径；包含 disabled agent，便于 enable/reset。
mergeAgentsForScope(scope, ...)最终 agent 优先级合并。用 Map<name, AgentConfig> 覆盖同名角色。
resolveExecutionAgentScope(scope)执行参数 scope 归一化。只接受 user、project、both；其他值回退为 both。

3. 来源位置与优先级

来源典型位置是否可写优先级说明
builtinopen-ai-agent/pi-subagents/agents/*.md只读最低默认来源；可被 package/user/project 同名角色覆盖。
package当前包、user/project npm 目录、settings packages、全局 npm root。只读覆盖 builtin，但低于 user/project；执行路径中的同名 package 由后加载者覆盖先加载者，诊断/管理视图只保留先发现者。
usergetAgentDir()/agents、~/.agents、PI_SUBAGENT_EXTRA_AGENT_DIRS。可写scope 为 both 时低于 project，高于 package/builtin。
project最近项目根下 .pi/agents，兼容读取 legacy .agents。可写scope 为 both 时最高；scope 为 project 时 user 不参与。
场景最终规则证据
agent 同名冲突builtin < package < user < project，但 scope 可排除 user 或 project。open-ai-agent/pi-subagents/src/agents/agent-selection.ts
package 同名冲突视图差异discoverAgents() 把 package agents 交给 mergeAgentsForScope()，同名 package 后加载覆盖先加载；discoverAgentsAll() 用 packageMap，同名 package 只保留先发现者。open-ai-agent/pi-subagents/src/agents/agents.ts、open-ai-agent/pi-subagents/src/agents/agent-selection.ts
project/user settings 同时存在project settings 优先；scope 为 user 时不读 project settings，scope 为 project 时不读 user settings。open-ai-agent/pi-subagents/src/agents/agents.ts
subagents.defaultModel只填没有显式 model 的 agent；project defaultModel 优先于 user defaultModel。open-ai-agent/pi-subagents/test/unit/agent-overrides.test.ts
builtin agentOverrides直接覆盖对应字段；false 可清除 model、skills、tools 等可清除字段。open-ai-agent/pi-subagents/src/agents/agents.ts
custom agentOverrides只填 frontmatter 没有显式声明的字段；frontmatter 逐字段优先。open-ai-agent/pi-subagents/test/unit/agent-overrides.test.ts
chain 同名或同 scope 格式冲突同一目录内 .chain.json 优先于 .chain.md；返回顺序为 package、user、project。open-ai-agent/pi-subagents/src/agents/agents.ts、open-ai-agent/pi-subagents/test/unit/agent-frontmatter.test.ts

4. Frontmatter 字段速查

字段含义解析位置
name、package本地名与包名前缀合成 runtime name，例如 code-analysis.scout。open-ai-agent/pi-subagents/src/agents/identity.ts
description管理面 list/get 展示，也用于用户选择角色。open-ai-agent/pi-subagents/src/agents/agents.ts
tools逗号列表；mcp: 前缀进入 mcpDirectTools。open-ai-agent/pi-subagents/src/agents/agents.ts
model、fallbackModels、thinking模型与思考配置；settings defaultModel 只填缺省 model。open-ai-agent/pi-subagents/src/agents/agents.ts
systemPromptModereplace 或 append；delegate 默认 append，其余默认 replace。open-ai-agent/pi-subagents/src/agents/agents.ts
inheritProjectContext是否继承项目上下文；delegate 默认 true，其他内置也常显式 true。open-ai-agent/pi-subagents/src/agents/agents.ts
inheritSkills、skills是否继承父技能，以及主动注入哪些技能引用。open-ai-agent/pi-subagents/src/agents/agents.ts 解析 frontmatter，open-ai-agent/pi-subagents/src/agents/skills.ts 负责 skill discovery 与注入文本。
defaultContextfresh 或 fork，控制默认上下文模式。open-ai-agent/pi-subagents/test/unit/agent-frontmatter.test.ts
defaultReads、defaultProgress、interactive角色默认读取文件、是否进度显示、是否交互。open-ai-agent/pi-subagents/src/agents/agent-serializer.ts
maxSubagentDepth限制该 agent 下游再启 subagent 的深度。open-ai-agent/pi-subagents/test/unit/agent-frontmatter.test.ts
completionGuardfalse 时禁用 completion guard；true 默认不序列化。open-ai-agent/pi-subagents/src/agents/agent-serializer.ts
toolBudgetJSON 对象，描述工具预算和阻断规则。open-ai-agent/pi-subagents/src/agents/agent-management.ts
memory声明 project/user scope 的角色持久记忆目录。open-ai-agent/pi-subagents/src/agents/agent-memory.ts
未知字段保存在 extraFields，序列化时可保留多行 block。open-ai-agent/pi-subagents/src/agents/frontmatter.ts

5. 管理动作清单

动作读写作用边界
list读列出可执行 agent、chain、chain diagnostics 和 proactive skill 建议。默认 scope 为 both；隐藏 disabled agent。
get读查看 agent 或 chain 详情，包括 system prompt、tools、skills、memory、toolBudget。可同时查询 agent 与 chain；找不到则返回可用名称。
models读查看 builtin agent 的有效模型、来源和当前 session model 映射。只接受 builtin 名称过滤。
create写创建 user/project agent 或 chain。要求 name 和 description；scope 只能是 user/project；会校验模型、skills、toolBudget。
update写更新 user/project agent 或 chain，支持重命名和 package 变更。builtin/package 只读；同名双 scope 需指定 agentScope。
delete写删除 user/project agent 或 chain 文件。删除 agent 后会提示仍引用它的 chains。
eject写把 builtin/package agent 复制到 user/project，便于完整自定义。目标 scope 已有同名 agent 或 chain 时拒绝。
disable写通过 settings override 写入 disabled: true。若更高优先级 override 获胜，会报告仍未生效。
enable写移除指定 scope 的 disabled 字段。若另一个 scope 仍禁用，会要求切换 scope。
reset写删除 custom agent 文件和 settings override，恢复 bundled 默认。只对有 bundled 默认的 agent 有意义；纯 custom 需 delete。

6. 内置角色列表

角色描述默认特征
context-builder分析需求和代码库，生成 context 与 meta-prompt。tools 含 read/grep/find/ls/bash/write/web_search/intercom；thinking medium。
delegate轻量委派角色，继承父模型，不做默认读取。systemPromptMode: append，inheritProjectContext: true。
oracle高上下文一致性判断角色，保护继承状态并防止漂移。thinking high，defaultContext: fork，只读加 bash/intercom。
planner根据上下文和需求创建实施计划。thinking high，defaultReads: context.md，defaultContext: fork。
researcher自主 web research，搜索、评估并综合研究 brief。tools 含 web_search/fetch_content/get_search_content。
reviewer代码 diff、计划、方案、代码库健康和 PR/issue 验证专家。thinking high，defaultReads: plan.md, progress.md。
scout快速代码库侦察，输出压缩上下文供交接。thinking low，tools 含 read/grep/find/ls/bash/write/intercom。
worker普通实现任务和批准后的 oracle handoff 执行者。thinking high，defaultContext: fork，defaultReads: context.md, plan.md。

7. Chain 与 serializer 边界

文件职责注意点
open-ai-agent/pi-subagents/src/agents/agent-serializer.ts把 AgentConfig 写回 Markdown frontmatter。更新 custom agent 时可用 preserveFrontmatterFields 避免把 settings override 意外写入文件。
open-ai-agent/pi-subagents/src/agents/chain-serializer.ts解析和序列化 .chain.md 与 .chain.json。Markdown chain 用 ## agent 划分步骤；JSON chain 支持 dynamic fanout 并校验 output bindings。
open-ai-agent/pi-subagents/src/agents/identity.ts处理 package 名、runtime name 和 frontmatter name。package: Code Analysis 会规范化为 code-analysis。

8. Skills、memory 与 proactive 建议

模块作用关键规则
open-ai-agent/pi-subagents/src/agents/skills.ts发现 agent 可用 skills，并构造懒加载注入文本。不把 pi-subagents 自身暴露给 child；skill source 优先级为 project、project-settings、project-package、user、user-settings、user-package、extension、builtin。
open-ai-agent/pi-subagents/src/agents/proactive-skills.ts在 list 输出中建议可用 skill-specialist subagent。默认至少 2 个配置引用同一 skill，最多 3 条建议，首选 reviewer；可用 config 关闭。
open-ai-agent/pi-subagents/src/agents/agent-memory.ts为声明了 memory 的 agent 注入持久角色记忆。路径必须在 user/project agent-memory/ 根下；读写 agent 可创建记忆，纯只读 agent 仅在已有文件时注入。

9. 简单配置示例

---
name: implementer
description: TDD implementer
tools: read, grep, bash, edit, write
model: google/gemini-3-pro
defaultContext: fork
memory:
  scope: project
  path: implementer
---

Drive the failing test first, then implement the smallest passing change.

若项目 .pi/settings.json 同时写了 subagents.agentOverrides.implementer.model，上面 frontmatter 的 model 仍优先。若 settings 写了 toolBudget，而 frontmatter 没写 toolBudget，custom override 可以填入该字段。

11. 常见判断题

问题答案
同名 package agent 和 builtin agent 谁赢？package 赢；但 user/project 同名 agent 会再覆盖 package。
为什么 disable 后 list 看不到 agent？discoverAgents 最后过滤 disabled === true；discoverAgentsAll 仍保留它供 enable/reset 使用。
custom agent 的 model 明明在 settings override 里，为什么没生效？如果 frontmatter 显式写了 model，custom override 不覆盖该字段。
想完整修改 builtin prompt，应该 update 还是 eject？builtin 是只读来源。可用 settings override 改部分字段；要完整复制并编辑 prompt，用 eject 到 user/project。
scope 缺失或非法时执行层怎么处理？resolveExecutionAgentScope 回退到 both。
