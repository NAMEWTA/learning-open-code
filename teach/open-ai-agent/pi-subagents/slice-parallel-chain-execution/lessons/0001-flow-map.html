15 分钟串起 parallel 与 chain 执行

学习目标：给定一条 chain 或一组 tasks，你能从前台入口指到 child spawn，并说出并行聚合、{previous} 传递与 workflow graph 在哪一层产生。完整 step 形态表与测试索引见 parallel / chain 参考页。

L2 垂直切片前置：单次前台 run关联：runs 执行模块

1. 入口：top-level parallel 与 chain 分流

open-ai-agent/pi-subagents/src/runs/foreground/subagent-executor.ts 看到 tasks（或 action 别名 parallel）时走 top-level parallel：对每项直接 runSync，details.mode 为 parallel。看到 chain: [...] 则转 open-ai-agent/pi-subagents/src/runs/foreground/chain-execution.ts 的 executeChain，details.mode 为 chain。二者都依赖 mapConcurrent 控制并发，但 chain 还维护 prev、命名输出 outputs 与逐步 summary。

subagent-executor
  ├─ tasks / parallel action → mapConcurrent + runSync（顶层并行）
  └─ chain → executeChain（链式编排 + 图快照）

2. executeChain 主循环：三种 step

executeChain 先 validateChainOutputBindings、createChainDir、resolveChainTemplates，再按 step 类型分支。类型判断在 open-ai-agent/pi-subagents/src/shared/settings.ts：isParallelStep（parallel 数组）、isDynamicParallelStep（expand + collect + 单对象 parallel）、否则为顺序 step。顺序 step 解析模板后单次 runSync，把输出写入 prev 与可选的 as 命名槽。

for each chain step:
  sequential → runSync → prev = child output
  parallel[] → runParallelChainTasks → aggregateParallelOutputs → prev
  dynamic    → materialize → runParallelChainTasks → collect → outputs[collect.as]

3. 并行内核与 dynamic fanout

runParallelChainTasks 对 step 内每个 task 解析 {task}、{previous}、{outputs.x}，再经 mapConcurrent 调 runSync；failFast 会在首个非零 exit 后跳过后续 task。动态扇出在 open-ai-agent/pi-subagents/src/runs/shared/dynamic-fanout.ts：materializeDynamicParallelStep 从上游 structured output 展开数组、渲染 per-item 模板，跑完后 collectDynamicResults 写入 collect.as。空数组可走 skip 或 fail，由 expand.onEmpty 决定。

4. 观测：workflowGraph 与测试

每次 onUpdate 与最终 details 都调用 open-ai-agent/pi-subagents/src/runs/shared/workflow-graph.ts 的 buildWorkflowGraphSnapshot：顺序 step 为 kind: step，静态并行为 parallel-group，动态扇出为 dynamic-parallel-group 并挂 materialized children。行为佐证见 test/integration/chain-execution.test.ts（{previous}、parallel step、fanout、graph 状态）与 test/integration/parallel-execution.test.ts（并发上限与顺序保持）。

检索练习：一条 chain 第二步是 dynamic fanout，且 expand.from.path 指向上一步 as: "items" 的 structured 数组。若该数组为空且 onEmpty: "skip"，第三步收到的 {previous} 大致是什么？

参考答案：固定文案 Dynamic fanout produced 0 results.；同时 outputs[collect.as] 为 [] 的 JSON，workflow graph 上该 dynamic 组标记为 completed 且无 children（见 chain-execution 空 fanout 分支与对应 integration test）。

主要来源：chain-execution.ts、dynamic-fanout.ts、workflow-graph.ts。想沿某条分支下钻（worktree、acceptance、failFast），向 agent 追问并带上 step 配置片段。
