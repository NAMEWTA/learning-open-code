parallel / chain execution 参考图谱

本页承载 L2 切片的长清单：入口分流、step 形态、模板变量、核心函数、workflow graph 节点类型与测试索引。短课 0001-flow-map 只保留全链路指认技能。

1. 总流程

subagent tool 参数
  ↓
subagent-executor.ts
  ├─ tasks / parallel action
  │     → mapConcurrent(items, concurrency, runSync)
  │     → details.mode = "parallel"
  └─ chain: ChainStep[]
        → chain-execution.ts executeChain
        → details.mode = "chain" + workflowGraph

2. ChainStep 三种形态

形态识别条件（settings.ts）执行路径输出传递
顺序 step含 agent，非 parallel / dynamic单步 runSyncprev ← child 输出；可选 as → outputs
静态 parallelisParallelStep：parallel: ParallelTaskItem[]runParallelChainTasksaggregateParallelOutputs → prev；每 task 可选 as
动态 fanoutisDynamicParallelStep：expand + collect + 单对象 parallelmaterializeDynamicParallelStep → runParallelChainTasks → collectDynamicResultsoutputs[collect.as] 为 collected JSON；prev 为聚合文本

3. 模板与上下文变量

占位符含义解析位置
{task}链的原始用户任务executeChain 内 replace
{previous}上一步聚合输出 prev顺序 / parallel task 模板
{outputs.name}命名输出槽内容resolveOutputReferences
{chain_dir}本次 chain 工作目录replace 为 chainDir
{item} / {item.field}dynamic fanout 每项数据resolveItemTemplate（dynamic-fanout.ts）

resolveChainTemplates（settings.ts）为每步生成默认模板：parallel 子 task 默认 {previous}；顺序 step 默认取 task 或 {previous}。

5. workflowGraph 节点类型

kind何时出现关键字段
step顺序 chain stepagent、flatIndex、outputName（as）
parallel-group静态 parallel stepchildren[] 每项 kind: agent
dynamic-parallel-groupdynamic fanout stepdynamic.sourceOutput、collectAs、materialized children
agentparallel / dynamic 子任务status、itemKey（动态）、acceptanceStatus

节点状态优先级：stepStatuses 覆盖 → results[flatIndex]（detached / interrupted / exitCode）→ 当前 currentFlatIndex 为 running → 否则 pending。

6. 并行控制参数

参数作用域默认 / 行为
concurrency单个 parallel / dynamic step默认 MAX_CONCURRENCY；传给 mapConcurrent
failFast单个 parallel step首个失败 exit 后后续 task 标记 skipped（exit -1）
globalConcurrencyLimit整条 chainSemaphore 跨 step 限制同时运行的 child 数
expand.maxItems / dynamicFanoutMaxItemsdynamic fanout限制展开项数，超出则执行前失败
