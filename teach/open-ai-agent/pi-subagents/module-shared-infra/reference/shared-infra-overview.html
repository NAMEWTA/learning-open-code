3. 核心类型与常量

对象用途阅读提示
Details一次 subagent 调用返回给父会话的总明细：mode、results、async metadata、progress、artifacts、workflowGraph、outputs、usage/cost。看父会话工具结果如何承载多个 `SingleResult`。
SingleResult单个 child agent 的结果：exitCode、usage、model、controlEvents、sessionFile、artifactPaths、finalOutput、structuredOutput、acceptance、children。看 foreground/async 最终结果如何归一。
AsyncStatus后台 run 的磁盘状态：runId、mode、state、activity、steps、parallelGroups、workflowGraph、session/output、tokens/cost。状态查询、wait、TUI 和 stale reconciliation 都围绕它。
AsyncJobStateextension 内存中的后台 job 投影：asyncId、asyncDir、status、pid、steps、cursors、nested children。偏 UI/tracker，不等同于磁盘完整状态。
ArtifactPaths一组 run artifact 路径：input、output、jsonl、transcript、metadata。由 `getArtifactPaths` 根据 runId、agent、index 生成。
RESULTS_DIR、ASYNC_DIR、CHAIN_RUNS_DIR、TEMP_ARTIFACTS_DIRscoped temp root 下的后台结果、运行状态、chain 临时目录和 artifacts 目录。temp scope 优先用 uid；无 uid 时使用用户名、home 或 shared。
SUBAGENT_ASYNC_STARTED_EVENT、SUBAGENT_ASYNC_COMPLETE_EVENT、SUBAGENT_CONTROL_EVENTextension event bus 的生命周期和控制事件名。不要手写字符串；从 `types.ts` 导入。
SUBAGENT_ACTIONS管理动作列表：list、get、models、create、update、delete、eject、disable、enable、reset、status、interrupt、resume、steer、append-step、doctor、schedule 等。schema、doctor、管理命令应与该列表保持一致。

4. 文件状态与路径规则

场景规则风险边界
project-local artifacts`getProjectSubagentsDir(cwd)` 返回 `/.pi-subagents`；artifacts 在 `/.pi-subagents/artifacts`，chain runs 在 `/.pi-subagents/chain-runs`。适合项目级可检查产物；测试见 open-ai-agent/pi-subagents/test/unit/artifacts.test.ts。
session fallback`getArtifactsDir(sessionFile)` 返回 session 目录下的 `subagent-artifacts`；无 session 和 project cwd 时使用 temp artifacts。确保无 project cwd 的旧路径仍可工作。
artifact 文件名`getArtifactPaths` 将 agent 名中的非 `\w.-` 字符替换为 `_`，再生成 `_input.md`、`_output.md`、`.jsonl`、`_transcript.jsonl`、`_meta.json`。防止 agent 名污染路径。
status JSON 写入`createAtomicJsonWriter` 写 `.basename.pid.now.random.tmp`，再 rename 到目标；finally 中删除临时文件。避免 reader 看到半写入 JSON；测试覆盖 retry 和清理。
Windows rename 锁默认仅在 win32 启用 retry；可重试错误为 `EACCES`、`EBUSY`、`EPERM`，默认延迟从 10ms 到 4000ms。非重试错误直接抛出。
events JSONL`createJsonlWriter` 对空路径返回 no-op；每行追加换行；超过 `maxBytes` 后停止写；stream 背压时 pause source，drain 后 resume。事件日志不会无限增长，也不会在背压时继续推流。
status 读取`readStatus(asyncDir)` 读取 `status.json` 并按 mtime 缓存，最多保留 50 条。文件缺失返回 null；解析失败抛带路径的错误。
