L1-intercom 短课 0001

Intercom 模块导览：三条协调通道的职责边界

学习目标：读完后能把父子进程消息、native supervisor 通道、结果回传三件事分开，不再把所有 intercom 行为都归因到同一个文件。

先建立一个判断框架

这个模块可以按“谁在协调什么”拆成三条通道。第一条是 bridge：父进程在启动子 agent 前，决定是否把协调提示和工具名注入到 agent 配置。第二条是 supervisor channel：子 agent 运行中遇到阻塞决策时，用文件通道向父进程发请求并等待回复。第三条是 result intercom：子 agent 完成后，把完整输出整理成 grouped payload，经事件总线送回父会话，同时让普通返回值保持简洁。

Bridge
解决“子 agent 应不应该知道如何联系 supervisor”。核心在 open-ai-agent/pi-subagents/src/intercom/intercom-bridge.ts。
Supervisor Channel
解决“运行中的子 agent 怎样请求决策或报告进度”。核心在 open-ai-agent/pi-subagents/src/intercom/native-supervisor-channel.ts。
Result Intercom
解决“完成结果怎样回传，并且不污染普通结果”。核心在 open-ai-agent/pi-subagents/src/intercom/result-intercom.ts。

运行时顺序

1. 启动前：执行器解析 bridge 配置。若 active，就给 agent 追加 intercom、contact_supervisor，并追加带 marker 的系统提示。
2. 启动子进程：参数构建层写入 run id、agent、child index、父 session id 和 supervisor channel 目录等环境变量。
3. 运行中：子端注册 contact_supervisor。当 reason 是 need_decision 或 interview_request 时，请求文件写入后会等待 reply 文件；progress_update 只排队，不等待回复。
4. 父端：父会话启动时开启 supervisor channel 轮询，只展示属于当前 session 的请求，并通过 subagent_supervisor 或 native intercom 回复。
5. 完成后：foreground 或 async 路径把结果整理成 grouped payload。若 delivery 被确认，父侧看到 compact receipt；若未确认或 bridge 关闭，就回退到旧的普通输出。

边界判断口诀

看到“工具和提示为什么存在”，看 bridge。看到“子 agent 还活着并等待 supervisor 决策”，看 native supervisor channel。看到“子 agent 已经完成，完整输出怎样送回父会话”，看 result intercom。

这三个文件之间不是互相替代关系。bridge 只注入能力，不传递请求；supervisor channel 只处理运行中请求，不格式化最终结果；result intercom 只处理完成后的 grouped payload，不替子 agent 做阻塞问答。

更长的消息类型、生命周期、调用方矩阵和测试证据见 Intercom 总览参考页。

检索练习

不看上文，判断下面三个故障应该先读哪个文件：

  1. 子 agent 没有拿到 contact_supervisor 工具。
  2. 子 agent 已发出 need_decision，但父会话没有看到待回复请求。
  3. parallel run 已完成，父会话只看到简短 receipt，看不到完整输出。

参考答案：

  1. 先读 open-ai-agent/pi-subagents/src/intercom/intercom-bridge.ts，确认 bridge mode、orchestrator target 和工具注入。
  2. 先读 open-ai-agent/pi-subagents/src/intercom/native-supervisor-channel.ts，确认环境变量、channel 目录、session 过滤、请求生命周期。
  3. 先读 open-ai-agent/pi-subagents/src/intercom/result-intercom.ts，再看 foreground/async 调用方，因为完整输出应在 grouped payload 里，普通结果会被压缩。
