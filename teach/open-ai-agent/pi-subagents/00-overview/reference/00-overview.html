pi-subagents L0 总览参考

本参考页是后续 L1/L2 课程的索引图谱，不替代短课。它按“入口、发现、执行、后台、控制面、输出”组织源码阅读顺序。

1. 项目定位与技术栈

维度结论证据
一句话给 Pi 父会话增加可委派 subagent 的扩展包，支持单次、并行、链式、后台、slash、RPC 与 intercom 协调。open-ai-agent/pi-subagents/README.md
语言与运行时TypeScript 源码直接由 Node 运行测试，包类型为 ESM。open-ai-agent/pi-subagents/package.json
Pi 集成pi.extensions 指向 open-ai-agent/pi-subagents/src/extension/index.ts，同时暴露 skills 与 prompts。open-ai-agent/pi-subagents/package.json
运行依赖runtime dependencies：@earendil-works/pi-tui、jiti、typebox。open-ai-agent/pi-subagents/package.json
宿主依赖peer/dev dependencies：@earendil-works/pi-agent-core、@earendil-works/pi-ai、@earendil-works/pi-coding-agent。open-ai-agent/pi-subagents/package.json
数据库与持久化无外部数据库；后台运行状态通过 status.json、events.jsonl、output log、result JSON 等文件产物持久化。open-ai-agent/pi-subagents/README.md、open-ai-agent/pi-subagents/src/shared/types.ts
控制面Pi events、slash 命令、RPC bridge、wait tool、native supervisor/intercom 共同组成控制与观测面。open-ai-agent/pi-subagents/src/extension/index.ts
部署安装作为 npm 包安装，README 主路径为 pi install npm:pi-subagents；install.mjs 是 npm bin 入口。open-ai-agent/pi-subagents/README.md、open-ai-agent/pi-subagents/package.json
测试入口npm run test:unit、npm run test:integration、npm run test:e2e、npm run test:all。open-ai-agent/pi-subagents/package.json

6. 架构决策

决策含义主要证据
父会话保留编排权子会话执行具体角色任务；普通 child 不应继续发起 subagent 编排，除非明确配置 fanout。README.md、skills/pi-subagents/SKILL.md、open-ai-agent/pi-subagents/src/shared/types.ts
发现与执行解耦agent discovery 单独处理来源、优先级和覆盖；executor 只消费已解析的 AgentConfig。open-ai-agent/pi-subagents/src/agents/agents.ts、open-ai-agent/pi-subagents/src/runs/foreground/subagent-executor.ts
后台状态文件优先于终端输出async run 通过 status.json、events.jsonl、输出 log 和 result JSON 对外可观测。README.md、open-ai-agent/pi-subagents/src/runs/background/run-status.ts、open-ai-agent/pi-subagents/src/shared/types.ts
控制面多入口同 executor自然语言 tool、slash、RPC、scheduled run 都最终复用 executor 或其控制路径。open-ai-agent/pi-subagents/src/extension/index.ts、open-ai-agent/pi-subagents/src/slash/slash-commands.ts、open-ai-agent/pi-subagents/src/extension/rpc.ts
长任务以 async/wait 管理后台运行可恢复、可等待、可中断；父会话不应靠 sleep 或输出猜测完成状态。skills/pi-subagents/SKILL.md、open-ai-agent/pi-subagents/src/runs/background/wait.ts
