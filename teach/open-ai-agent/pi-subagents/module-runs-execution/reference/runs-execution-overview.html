runs execution 参考图谱

本页承载 L1 的长清单：模块边界、入口、关键依赖、测试佐证和后续 L2 切片。短课只保留分流判断技能。L3 辅助 API 见 runs-execution-api.html。

1. 模块位置与总流程

subagent tool / slash / RPC
  ↓
agent discovery 得到 AgentConfig[]
  ↓
foreground/subagent-executor.ts
  ├─ action → management/status/resume/interrupt/steer/schedule
  ├─ async → background/async-execution.ts → subagent-runner.ts
  ├─ chain → foreground/chain-execution.ts
  ├─ tasks → foreground parallel path
  └─ agent → foreground/execution.ts runSync

一句话职责：runs 模块是执行分流和生命周期观测层。它不负责注册 Pi extension，也不负责发现 agent 定义；它消费已解析的 agent/config/context，把运行结果包装成 `Details`、`SingleResult`、status 文本、async 事件和可恢复的 session/artifact。

3. 对外入口清单

入口形态参数信号进入路径输出
single foreground{ agent, task } 且无 `tasks/chain/action`，`async` 为 falseopen-ai-agent/pi-subagents/src/runs/foreground/subagent-executor.ts → open-ai-agent/pi-subagents/src/runs/foreground/execution.ts 的 `runSync`当前 tool 返回 child 最终输出，details.mode=`single`。
parallel foreground{ tasks: [...] }open-ai-agent/pi-subagents/src/runs/foreground/subagent-executor.ts 内 top-level parallel path，最终多次 `runSync`聚合每个 task 输出，保留顺序，details.mode=`parallel`。
chain foreground{ chain: [...] }open-ai-agent/pi-subagents/src/runs/foreground/chain-execution.ts链式 summary、named outputs、workflowGraph，details.mode=`chain`。
async single{ agent, task, async: true } 或配置强制 asyncopen-ai-agent/pi-subagents/src/runs/background/async-execution.ts 的 `executeAsyncSingle` → detached runner立即返回 run id、async dir、等待/状态提示。
async chain/parallel{ chain: [...], async: true }；top-level `tasks` 会包装成一个 parallel chain stepopen-ai-agent/pi-subagents/src/runs/background/async-execution.ts 的 `executeAsyncChain` → detached runner立即返回 run id、async dir、workflowGraph。
status/fleet/transcript{ action: "status", id?, view? }open-ai-agent/pi-subagents/src/runs/background/run-status.ts；foreground live status 先从 executor state 查当前运行、记忆 foreground、async、nested 的状态文本。
waitwait({ id?, all?, timeoutMs? })open-ai-agent/pi-subagents/src/runs/background/wait.ts阻塞直到目标 async run 完成、失败、暂停、needs_attention、超时或 abort。
控制/恢复resume、interrupt、steer、append-stepexecutor action path + background control channel/status resolver恢复 child session、请求中断、发送 steer、向 async chain 追加 step。
管理动作list、get、models、create、update、delete、eject、disable、enable、reset、doctor、schedule*open-ai-agent/pi-subagents/src/runs/foreground/subagent-executor.ts action 分支；多数转交 agents/extension/background 模块管理结果，details.mode=`management`。
