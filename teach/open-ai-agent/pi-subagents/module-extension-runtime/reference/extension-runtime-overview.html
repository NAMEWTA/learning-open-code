extension runtime 参考页

本页是 L1 速查资料：lesson 讲最小注册链路，本页收纳注册点、源码职责、依赖和测试佐证。阅读时把 src/extension 看成 Pi runtime 适配层，而不是 subagent 执行算法层。

1. 模块边界

维度结论证据
一句话职责把 subagent 运行能力注册进 Pi runtime，并桥接 tool、wait、renderer、slash、RPC、async tracker、supervisor channel 和 session 生命周期。open-ai-agent/pi-subagents/src/extension/index.ts
架构位置位于 Pi host 与内部 runs/agents/slash/intercom/shared 模块之间；向上接 ExtensionAPI，向下调用 executor、watcher、tracker、discovery。open-ai-agent/pi-subagents/package.json、open-ai-agent/pi-subagents/src/extension/index.ts
不负责不实现 single、parallel、chain 的核心执行算法；不实现 agent discovery 规则；不持久化 async 状态文件的全部语义。open-ai-agent/pi-subagents/src/runs/foreground/subagent-executor.ts、open-ai-agent/pi-subagents/src/agents/agents.ts

3. 源码职责索引

文件职责阅读要点
open-ai-agent/pi-subagents/src/extension/index.ts主入口，注册 Pi tools/renderers/events，维护 runtime/session cleanup。先看 child 早退、state 创建、executor 创建、注册点、session start/shutdown。
open-ai-agent/pi-subagents/src/extension/schemas.tsTypeBox schema；导出 SubagentParams 和 WaitParams。keepTopLevelParameterDescriptions 是 provider payload 控制点。
open-ai-agent/pi-subagents/src/extension/rpc.ts事件总线 RPC bridge。协议版本是 1；request/reply/ready event 名固定；spawn 强制 async。
open-ai-agent/pi-subagents/src/extension/config.ts读写 extension config。loadConfig 出错时打印错误并返回空配置；saveConfig 使用 tab 格式化。
open-ai-agent/pi-subagents/src/extension/doctor.ts构建 read-only doctor report。报告 runtime、filesystem、discovery、permission system、intercom bridge；单项失败不会阻断整份报告。
open-ai-agent/pi-subagents/src/extension/tool-description.ts生成 subagent tool description。支持 full、compact、custom；custom 会补强制安全提示。
open-ai-agent/pi-subagents/src/extension/control-notices.ts控制通知去重、延迟、投递。忽略 active_long_running 的直接消息；foreground notice 延迟确认仍需注意再发。
open-ai-agent/pi-subagents/src/extension/fanout-child.tsfanout child 专用注册入口。普通 child 不注册主入口；fanout child 用 WeakSet 防同一 API 重复注册，并禁止变更类管理动作。

4. 内部分层

层代表对象职责
宿主适配层ExtensionAPI、ExtensionContext、ToolDefinition把 Pi runtime 的工具、消息、事件、session 钩子变成 extension 可用接口。
注册编排层registerSubagentExtension、registerFanoutChildSubagentExtension决定当前进程该注册哪些能力，以及如何清理旧 runtime。
契约层SubagentParams、WaitParams、RPC envelope types定义 tool/RPC 可接受的参数和 reply 形状。
桥接层slash bridge、prompt template bridge、RPC bridge、native supervisor channel把不同入口统一转换为 executor 调用或控制事件。
状态观测层async tracker、result watcher、notify renderer、control notice renderer恢复、刷新、通知并渲染后台/控制状态。
执行下沉层createSubagentExecutor、discoverAgents、background run modules真正执行或管理 run；不属于 extension 入口层的核心算法。

6. 简单调用与注册示例

Pi 加载主入口

{
  "pi": {
    "extensions": [
      "./src/extension/index.ts"
    ]
  }
}

上面来自 open-ai-agent/pi-subagents/package.json，其中 ./src/extension/index.ts 是 npm 包内部相对路径；在本工作区中对应的源码文件是 open-ai-agent/pi-subagents/src/extension/index.ts。Pi host 加载该文件后，默认导出的 registerSubagentExtension(pi) 才会注册工具和事件钩子。

RPC spawn 请求

events.emit("subagents:rpc:v1:request", {
  version: 1,
  requestId: "spawn-1",
  method: "spawn",
  params: { agent: "worker", task: "Do work" }
});

rpc.ts 会把它规范化为 { async: true, clarify: false } 的 executor 调用；若显式传 async: false 或 action，会在 dispatch 前返回 invalid_params。

7. 行为佐证

测试文件覆盖事实对阅读的帮助
open-ai-agent/pi-subagents/test/unit/index-child-registration.test.ts父入口会注册工具；普通 child 和 fanout child 下主入口早退；fanout child 不双注册；child-safe 模式阻止变更类管理动作；wait tool 可被配置禁用。证明入口层确实有进程角色边界和注册保护。
open-ai-agent/pi-subagents/test/unit/rpc.test.tsready/ping 元数据、status/interrupt 委托、spawn 强制 async、stop 写 timeout request、跨 session stop 被拒绝。证明 RPC 是控制面桥，不是独立执行实现。
open-ai-agent/pi-subagents/test/unit/schemas.test.tsschema payload 小于阈值，移除嵌套 description，保留 TypeBox metadata，避免 provider-rejected shapes。证明 schema 裁剪是兼容性设计，不是清理样式。
open-ai-agent/pi-subagents/test/unit/doctor.test.ts成功环境输出有界摘要；目录或 discovery 检查失败时继续报告其他部分。证明 doctor 是诊断汇总器，不能假设所有依赖都健康。
open-ai-agent/pi-subagents/test/unit/tool-description.test.tsdescription mode、custom description、安全提示和注册时描述选择。证明 tool 描述生成是可配置接口面的一部分。
open-ai-agent/pi-subagents/test/unit/control-notices.test.ts控制通知投递、去重、foreground 延迟确认。证明控制通知不是简单 sendMessage，而是带状态判定的 runtime 行为。
open-ai-agent/pi-subagents/test/unit/nested-control.test.tsfanout child 的 nested control inbox 行为。证明 child-safe fanout 仍有受控的嵌套控制通道。
