3. 状态与观测 API

run.state（status.json）含义wait 是否继续等
queued / running仍在执行是（除非 needs_attention）
complete成功结束否
failed失败结束否
paused可恢复暂停否（终态对待）
activityState: needs_attention需父会话决策wait 立即返回
API参数要点返回特征
subagent({ action: "status" })无 id → 列出 active runs不阻塞；可 view: "fleet" | "transcript"
wait({ })默认 first-completion阻塞；适合 fleet 滚动
wait({ all: true })等全部 initial set 终态阻塞至 0 个 active initial run
wait({ id })单 run；前缀歧义时报错等价于 waitForAll 对该 id
