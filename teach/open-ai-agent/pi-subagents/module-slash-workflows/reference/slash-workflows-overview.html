Slash 命令与 prompt workflow 模块参考

本页是 L1 参考资料，用于快速查阅命令注册、prompt workflow 模板、入口衔接、边界条件和测试证据。短课只讲主链路；这里保存完整清单。

1. 入口衔接总图

Pi extension 启动 → registerSlashSubagentBridge → registerPromptTemplateDelegationBridge → registerSlashCommands → registerPromptWorkflowCommands 注入 runSlashSubagent → 用户运行 /run、/chain、/parallel、/prompt-workflow 或 /chain-prompts → 解析为 SubagentParamsLike → buildSlashInitialResult 创建 live card → emit subagent:slash:request → slash bridge 调用 executor.execute → progress update 写入 slash-live-state → response finalize 为最终 snapshot
阶段核心文件职责
扩展启动open-ai-agent/pi-subagents/src/extension/index.ts注册 slash bridge、prompt template bridge，并在工具和 renderer 注册后调用 `registerSlashCommands`。
命令注册open-ai-agent/pi-subagents/src/slash/slash-commands.ts注册用户可输入的 slash 命令，并提供 `runSlashSubagent` 作为共同执行出口。
模板命令open-ai-agent/pi-subagents/src/slash/prompt-workflows.ts发现 Markdown prompt，解析 frontmatter，替换参数，注册 `/prompt-workflow` 与 `/chain-prompts`。
事件桥open-ai-agent/pi-subagents/src/slash/slash-bridge.ts监听 `subagent:slash:*` 事件，管理取消信号，把请求交给 executor。
Live stateopen-ai-agent/pi-subagents/src/slash/slash-live-state.ts为 single、parallel、chain 创建 placeholder，接收 progress，保存 final snapshot。
Prompt template bridgeopen-ai-agent/pi-subagents/src/slash/prompt-template-bridge.ts服务 `prompt-template:subagent:*` 委派事件，支持单任务与 tasks 并行 payload。

内置模板都位于 package prompts 目录。当前这些模板只有 `description` frontmatter，没有显式 `subagent`，因此 `parseAgent` 默认使用 `delegate`。运行时可以用 `/prompt-workflow name --subagent reviewer ...` 覆盖。

open-ai-agent/pi-subagents/prompts/parallel-cleanup.mdParallel cleanup review并行运行 deslop 与 verbosity 方向的 review，可选择 autofix。
open-ai-agent/pi-subagents/prompts/parallel-context-build.mdParallel context builders for planning handoff并行 context-builder 产出 request/scope、codebase/patterns、validation/risks。

7. 阅读顺序建议

  1. 先读 open-ai-agent/pi-subagents/src/slash/prompt-workflows.ts：理解 Markdown 如何变成 workflow。
  2. 再读 open-ai-agent/pi-subagents/src/slash/slash-commands.ts：找到 `registerPromptWorkflowCommands` 和 `runSlashSubagent` 的接缝。
  3. 随后读 open-ai-agent/pi-subagents/src/slash/slash-bridge.ts 与 open-ai-agent/pi-subagents/src/slash/slash-live-state.ts：确认事件、进度、最终快照。
  4. 最后读 open-ai-agent/pi-subagents/src/extension/index.ts 的注册片段：确认 bridge 最终接到 executor。
