测试体系模块边界导览

学习目标：15 分钟内能判断某个运行时行为应去 unit、integration 还是 e2e 找佐证，并知道该跑哪条 npm 命令。

1. 一句话职责

test/ 不是产品功能本身，而是整个 extension 的行为佐证层：当 README 或源码注释声称「前台 single 会写 artifact」「async 会落 status.json」「slash 能触发 workflow」时，测试文件给出可重复执行的断言。L0 已把 test/ 标为架构旁路；本课只建立三层测试边界与夹具分工，完整 106 个测试文件索引见 test suite 参考页。

2. 三层结构与 npm 入口

test/
├── unit/           ← 84 个 *.test.ts，纯函数，无 mock Pi
├── integration/    ← 21 个 *.test.ts，mock Pi CLI 管道
├── e2e/            ← 1 个 real-session 全链路
└── support/        ← 夹具：mock-pi、loader、real-session-runner

open-ai-agent/pi-subagents/package.json 把三层拆成四条命令：test:unit 用 --experimental-strip-types 直接跑 unit；test:integration 与 test:e2e 额外加 --experimental-transform-types 并 --import ./test/support/register-loader.mjs，因为源码使用 TypeScript ESM 的 .js 扩展名且部分文件含参数属性；test:all 顺序执行三层。判断口诀：只测单个纯函数 → unit；要跑 spawn/parse/executor 但不碰真 LLM → integration；要验证真实 AgentSession 注册 extension 并回传 tool result → e2e。

3. 夹具：mock-pi 与优雅跳过

integration 的核心是 test/support/mock-pi.ts：它在临时目录生成名为 pi 的壳脚本，把该目录 prepend 到 PATH，于是 runSync 调用的 child CLI 实际执行 mock-pi-script.mjs。测试通过 onCall({ jsonl: [...] }) 队列化响应，模拟 assistant 消息、tool 事件和退出码，无需 API key。若 Pi 包未安装，test/support/helpers.ts 的 tryImport 只对缺失的 bare specifier 返回 null，integration/e2e 用 skip 优雅退出，避免把环境缺依赖误判为代码回归。

const mock = createMockPi();
mock.install();
mock.onCall({ jsonl: [events.assistantMessage("done")] });
// runSync / executor 走完整前台管道
mock.uninstall();

4. 模块佐证与检索练习

读其他 L1 模块时，优先对照这些 integration 文件：runs 执行 → single-execution、parallel-execution、chain-execution、async-execution；slash → slash-commands、slash-live-state；intercom → intercom-result-delivery；TUI → render-widget、render-fork-badge。agent 发现与管理的多条边界落在 unit 层，例如 agent-selection、agent-overrides、recursion-guard。唯一 e2e real-session-subagent.test.ts 用 faux provider 驱动父会话调用 subagent，确认 marker 经真实 stdout 解析回到 tool result。

检索练习：你想验证「chain 步骤里 {previous} 是否正确传递」，应优先打开哪一层、哪个文件？

参考答案：integration 层的 open-ai-agent/pi-subagents/test/integration/chain-execution.test.ts（模板纯逻辑还可看 template-resolution.test.ts）。unit 的 chain-serializer 只覆盖序列化，不跑完整 spawn 管道。

主要来源：open-ai-agent/pi-subagents/package.json、open-ai-agent/pi-subagents/test/support/mock-pi.ts、open-ai-agent/pi-subagents/test/integration/single-execution.test.ts。下一节 L2 建议从「single foreground run 垂直切片 + 对应 integration 用例」开始；不清楚某条断言覆盖什么行为时，可以继续让 agent 对照源码逐段讲解。
