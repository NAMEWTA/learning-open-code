单次前台 subagent run 链路图

学习目标：15 分钟内能按顺序说出一次 { agent, task } 从 Pi tool 到 SingleResult 的 8 个跳点。

前置：extension-runtime · runs-execution · 速查：参考图谱

1. 什么算「单次前台 run」

父 Pi 会话调用 subagent tool，参数只有 agent 与 task，且 async 为 false（或未设、配置也未强制 async）。当前 turn 会阻塞等待 child Pi 进程结束，再把最终输出作为 tool result 返回。这与 parallel（tasks[]）、chain（chain[]）、后台 async 都不同。

{ "agent": "reviewer", "task": "Review src/runs/foreground/execution.ts" }

2. 八跳全链路

  1. 注册与入口 — extension/index.ts 用 createSubagentExecutor 创建执行器，subagent tool 的 execute 调到 executor.execute（经 executeSubagentCollapsed 折叠工具面板）。
  2. 门卫与分流 — subagent-executor.ts 的 execute：无 action、effectiveAsync 为 false、有 agent+task → 进入 runSinglePath；同时检查递归深度与 session spawn 限额。
  3. 单次路径准备 — runSinglePath 查找 AgentConfig、解析 model/skill/output、创建 session 目录与 foregroundControl（供 interrupt/status 用）。
  4. 同步执行壳 — 调用 execution.ts 的 runSync：拼 system prompt、写 artifact 输入、按 fallback 列表准备 model 候选。
  5. spawn child Pi — runSingleAttempt 经 buildPiArgs 生成 CLI 参数，getPiSpawnCommand（pi-spawn.ts）解析可执行文件，spawn 启动子进程。
  6. 流式解析 — 读 child stdout 的 JSONL 行，更新 AgentProgress，经 onUpdate 推给 TUI；可选写 jsonl/transcript artifact。
  7. 进程收尾 — child close 后汇总 exitCode、usage、control event；失败时按策略尝试下一个 model。
  8. 包装返回 — runSinglePath 把 SingleResult 填入 details.mode: "single" 的 AgentToolResult，父会话展开结果卡片。

记忆口诀：注册 → 门卫 → 准备 → runSync → spawn → JSONL → 收尾 → 包装。async 会在第 2 步被 runAsyncPath 截走，不会进入 runSync。

3. 三个关键文件分工

src/
├── extension/index.ts          ← 注册 tool，把调用交给 executor
├── runs/foreground/
│   ├── subagent-executor.ts    ← 分流、限额、runSinglePath
│   └── execution.ts            ← runSync / spawn / JSONL / SingleResult
└── runs/shared/pi-spawn.ts     ← 解析 pi 二进制路径

extension/index.ts 不 spawn 子进程；subagent-executor.ts 不解析 JSONL；execution.ts 不关心 tool schema。边界清楚后，调试时可以先判断卡在哪一层。

4. 测试与练习

test/integration/single-execution.test.ts 用 mock Pi CLI 跑通第 5–7 步：验证 spawn 参数、progress 更新、usage 累计、spawn 限额与未知 agent 错误，无需真实 LLM。

检索练习：用户传入 { "agent": "reviewer", "task": "…", "async": true }，链路在第几步离开前台 single 路径？

参考答案：第 2 步。executor 算出 effectiveAsync === true 后走 runAsyncPath → 后台 async-execution.ts，不会调用 runSync。

主要来源：extension/index.ts、subagent-executor.ts（runSinglePath）、execution.ts（runSync）。细节表见 参考图谱；model fallback、acceptance、control event 将拆为后续短课。有不清楚的地方，可以继续向 agent 追问。
