Frontmatter 到 AgentConfig

学习目标：看懂 .md 角色文件如何变成可合并的 AgentConfig。

1. 解析入口

loadAgentsFromDir 递归扫描目录下非 .chain.md 的 Markdown，对每个文件调用 parseFrontmatter，再映射字段。缺少 name 或 description 的文件会被静默跳过——这是最常见的「agent 怎么找不到」原因之一。

2. parseFrontmatter 行为

解析器支持单行 key: value 与缩进 block 值（如多行 toolBudget）。无开头 --- 时整文件视为正文。block 结束时会剥掉公共缩进，便于序列化写回。

export function parseFrontmatter(content: string) {
  if (!normalized.startsWith("---")) {
    return { frontmatter, body: normalized };
  }
  const endIndex = normalized.indexOf("\n---", 3);
  // …逐行解析 key: value 或 block …
  return { frontmatter, body };
}

3. 字段映射要点

本地 name 与可选 package 合成 runtime name（如 code-analysis.scout）。tools 逗号列表里 mcp: 前缀拆到 mcpDirectTools。解析时还会记录「哪些字段来自 frontmatter」，供后续 custom agent override 判断能否被 settings 覆盖。字段清单见L1 参考 §4。

4. 检索练习

暂停回忆：某 reviewer.md 只有正文、没有 frontmatter 分隔符，文件名是 reviewer.md。它会被加载为 agent 吗？

参考答案：不会。parseFrontmatter 返回空 frontmatter，缺少必需的 name 与 description，loadAgentsFromDir 会 continue 跳过。应补 frontmatter 或检查路径是否在正确的 user/project 目录。
