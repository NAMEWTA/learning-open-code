Settings 覆盖与 scope 合并

学习目标：解释 settings.json 如何改 model，以及四类来源如何合并成最终 agent 列表。

1. 读取 settings

readSubagentSettings 从 user ~/.pi/agent/settings.json 与 project .pi/settings.json 读取 subagents 块。JSON 结构非法（如 defaultModel 非字符串）会抛错，阻断发现。scope 为 user 时不读 project settings，反之亦然。

2. 两层覆盖

先 applySubagentDefaultModel：只给尚无 model 的 agent 填缺省。再分 builtin 与 custom：applyBuiltinOverrides 直接改字段；applyCustomAgentOverride 用 frontmatter 字段集决定是否填充。project override 整体优先于 user；disableBuiltins 可批量禁用 builtin。

export function mergeAgentsForScope(scope, userAgents, projectAgents, builtinAgents = [], packageAgents = []) {
  const agentMap = new Map();
  for (const agent of builtinAgents) agentMap.set(agent.name, agent);
  for (const agent of packageAgents) agentMap.set(agent.name, agent);
  if (scope === "both") {
    for (const agent of userAgents) agentMap.set(agent.name, agent);
    for (const agent of projectAgents) agentMap.set(agent.name, agent);
  }
  // …
  return Array.from(agentMap.values());
}

3. 执行 vs 诊断

discoverAgents 在合并后 .filter(agent => agent.disabled !== true)。discoverAgentsAll 保留 disabled，且 package 同名冲突时只保留先发现者（与执行路径「后加载覆盖先加载」不同）——管理 list 与执行 list 偶有差异，见参考 §3。

4. 检索练习

暂停回忆：user settings 设 defaultModel: A，project settings 设 defaultModel: B，project 存在 custom worker.md 且 frontmatter 写了 model: C。scope 为 both 时 worker 的 model？

参考答案：C。defaultModel 只填缺省；custom agent 的 frontmatter model 不被 override 覆盖。若无 frontmatter model，则为 B（project defaultModel 胜 user）。
