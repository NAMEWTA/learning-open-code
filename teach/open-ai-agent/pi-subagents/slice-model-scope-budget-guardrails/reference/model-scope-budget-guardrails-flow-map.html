1. 调用时序（guardrails 主路径）

settings / extension config / tool params → discoverAgents → modelScope → subagent-executor.execute → resolveTurnBudget / resolveToolBudget → createForkContextResolver (context=fork) → ExecutionContextData { modelScope, turnBudget, toolBudget } → execution.runSingle / chain / parallel → resolveSubagentModelOverride + buildModelCandidates → checkModelScope (explicit=error, inherited=warn) → appendTurnBudgetSystemPrompt → buildPiArgs → PI_SUBAGENT_TOOL_BUDGET env → child Pi: registerSubagentPromptRuntime → rewriteSubagentPrompt + stripParentOnlySubagentMessages → tool_call handler → soft nudge / hard block → subagent-runner: turn count → shouldAbortForTurnBudget

2. modelScope 规则

规则结论证据
配置来源project subagents.modelScope 优先于 useragents.ts · model-scope-settings.test.ts
匹配对象完整 provider/id，glob 仅 *，忽略 thinking 后缀model-scope.ts matchesScopePattern
explicit 越界severity: "error"，抛异常终止model-fallback.ts · model-scope.test.ts
inherited 越界severity: "warn"，默认 console.warn同上
fallback 候选index>0 的 fallback 一律按 inherited 处理buildModelCandidates
enforce 无 allowsettings 解析期拒绝；运行时 checkModelScope 为 no-opparseModelScopeConfig

3. budget 规则

类型解析优先级enforcement 位置默认 block 工具
toolBudgetstep > run params > agent.frontmatter > extension config子进程 subagent-prompt-runtime 的 tool_callread, grep, find, ls
turnBudgetparams > extension configspawn 前写系统提示；runner 计轮 abort—

hard 行为tool: block 列表内工具；turn: 超 soft+grace 后 abortgrace 轮允许 terminal assistant stop

4. prompt / fork 安全边界

机制作用触发条件
CHILD_SUBAGENT_BOUNDARY_INSTRUCTIONS声明子 agent 不得再委派、忽略父编排历史默认 child；非 fanout
stripParentOnlySubagentMessages移除父级 custom 消息与 subagent 工具调用痕迹context 事件；fanout 可保留当前工具历史
stripProjectContext / stripInheritedSkills按 env 剥离项目上下文与 skills 段PI_SUBAGENT_INHERIT_* 为 0
createForkContextResolver从父 session 分支 JSONL；清理 unsafe Anthropic thinkingcontext: "fork" 且父 session 已持久化
thinkingOverride "off"fork 后若剥离了 thinking 块则强制关闭 thinkingfork-context.ts sanitize 后

5. profiles 与 scope 的关系

profiles.ts 生成 subagents.agentOverrides.{agent}.model，经 applySubagentProfile 写入 user settings。它不直接设置 modelScope，但改变了 inherited 模型候选。scope enforcement 仍在第 4 跳 resolveSubagentModelOverride 发生。
