模型范围、预算与安全边界全链路

学习目标：15 分钟内能按顺序说出一次 subagent run 从配置解析到子进程 runtime 的九跳 guardrails 链路。

L2 垂直切片 前置：profiles 模块 速查：参考图谱

1. 切片边界

本课回答：谁限制子 agent 用什么模型、能做多少工具/轮次、看到什么上下文。三类机制分工不同：

  • modelScope — settings 里的 allow 模式，在模型解析时 enforced
  • toolBudget / turnBudget — 限制工具调用次数与 assistant 轮次
  • prompt / fork 安全 — 剥离父编排痕迹，fork 时清理不可重放的 thinking 块

profiles 只负责给 builtin agent 填 model 候选；真正拦越界模型的是 model-scope.ts + model-fallback.ts。

2. 九跳全链路

  1. 配置入口 — .pi/settings.json 的 subagents.modelScope、extension toolBudget/turnBudget，或 tool params 覆盖；profile 经 profiles.ts 写入 agentOverrides.model。
  2. 发现合并 — agents.ts 的 discoverAgents 把 project settings 的 modelScope 盖过 user，随 AgentConfig[] 一起返回。
  3. executor 预处理 — subagent-executor.ts 解析 turn/tool budget、createForkContextResolver（context: fork 时分支父 session），组装 ExecutionContextData。
  4. 模型解析 + scope — model-fallback.ts 的 resolveSubagentModelOverride / buildModelCandidates 调 checkModelScope：explicit 越界抛错，inherited 越界仅 warn。
  5. spawn 前注入 — execution.ts 用 appendTurnBudgetSystemPrompt 拼系统提示；pi-args.ts 把 toolBudget JSON 写入 PI_SUBAGENT_TOOL_BUDGET 环境变量。
  6. 子进程 runtime 注册 — subagent-prompt-runtime.ts 在子 Pi 里 decodeToolBudgetEnv 并监听 tool_call 事件。
  7. prompt 安全重写 — before_agent_start 时 rewriteSubagentPrompt 加 child boundary 指令，按需剥离 project context / skills / 父级 subagent 工具历史。
  8. tool budget 执行 — 达 soft 限发 steer nudge；超 hard 且工具在 block 列表（默认 read/grep/find/ls）则 { block: true }。
  9. turn budget 收尾 — runner 侧按 shouldAbortForTurnBudget 在 soft+grace 后 abort；超限时 formatTurnBudgetOutput 附带 partial output。

记忆口诀：配置 → 发现 → executor → scope 解析 → spawn 注入 → 子 runtime → prompt 重写 → tool 拦截 → turn 收尾。modelScope 在 spawn 前已决定候选；tool/turn budget 分别在子进程与 runner 两侧生效。

3. 三个关键文件分工

src/runs/
├── shared/
│   ├── model-scope.ts          ← 纯函数：allow 模式匹配 + severity
│   ├── tool-budget.ts          ← hard/soft/block 状态机
│   ├── turn-budget.ts          ← 系统提示 + abort 判定
│   └── subagent-prompt-runtime.ts ← 子进程 enforcement
├── foreground/
│   ├── subagent-executor.ts    ← 解析并下发 guardrails
│   └── execution.ts            ← 前台 spawn 拼接
└── shared/fork-context.ts      ← fork 分支 + thinking 清理

model-scope.ts 不读文件；subagent-prompt-runtime.ts 不解析 settings；fork-context.ts 不管预算。卡住时先判断是「模型被拒」「工具被 block」还是「prompt 仍含父编排」。

4. 检索练习

判断题：project settings.json 设 modelScope.enforce: true、allow: ["anthropic/*"]；agent frontmatter 继承父会话模型 deepseek/deepseek-v4，用户未传 --model。run 会在哪一跳、以什么严重度处理？

参考答案：第 4 跳 resolveSubagentModelOverride。模型来源为 inherited，checkModelScope 返回 severity: "warn"，不抛错；若用户显式传 --model deepseek/... 则同跳抛 hard error。完整测试矩阵见参考页 §测试。
