TUI 渲染参考页

本页是 L1 速查资料：lesson 只讲最小阅读主线，本页收纳渲染分支、helper 角色、async widget 状态展示、边界行为和测试证据。

1. 模块边界

维度结论证据
一句话职责把 subagent 执行结果、进度和后台 job 状态渲染成 Pi TUI 可展示的 Component 或 widget 行。open-ai-agent/pi-subagents/src/tui/render.ts
主要输入foreground 使用 AgentToolResult<Details>；background 使用 AsyncJobState[]。open-ai-agent/pi-subagents/src/shared/types.ts
主要输出renderSubagentResult 返回 Text、Markdown 或 Container；renderWidget 调用 ctx.ui.setWidget 安装 widget component。open-ai-agent/pi-subagents/src/tui/render.ts
不负责不运行 child、不写 status 文件、不决定 agent 选择；它只消费已经形成的 result/progress/job state。open-ai-agent/pi-subagents/src/runs/background/async-job-tracker.ts、open-ai-agent/pi-subagents/src/extension/index.ts

2. 源码职责索引

文件职责阅读要点
open-ai-agent/pi-subagents/src/tui/render.ts结果块、async widget、运行态 glyph、状态行、chain/parallel 标签、宽度裁剪。先找导出函数，再沿 compact/expanded 与 single/multi 分支读。
open-ai-agent/pi-subagents/src/tui/render-helpers.ts通用 TUI helper：fuzzy filter、padding、边框行、header/footer、路径缩写、滚动提示。row 会把多行和 tab 规范化，再按可见宽度裁剪。
open-ai-agent/pi-subagents/src/shared/status-format.ts状态文案与聚合：activity age、step status、parallel outcome。它是 result 与 widget 共享的状态语言层。
open-ai-agent/pi-subagents/src/shared/types.ts定义 Details、AgentProgress、AsyncJobState 等渲染输入。不确定字段含义时先查这里，避免从渲染字符串反推数据结构。
open-ai-agent/pi-subagents/src/runs/background/async-job-tracker.ts把 background run summary 映射成 widget state，并在变化时刷新 widget。summaryToJob 决定 widget 能看到哪些字段。
open-ai-agent/pi-subagents/src/extension/index.ts把 result renderer 和 widget 刷新接入 Pi runtime。renderResult 传入 frame；tool_result 事件刷新 async widget。

3. foreground result 渲染分支

条件函数路径显示重点
没有 details 或没有 resultsrenderSubagentResult 内部 plain text 分支单行直接裁剪，多行用 wrapPlainText 包装；fork context 会显示 [fork]。
single 且 collapsedrenderSingleCompactglyph、agent、model/thinking、turn/tool/token/duration、状态行、首行输出、session 和 artifact。
single 且 expandedrenderSubagentResult 的 single expanded 分支task、running live status、recent tools/output、tool call lines、Markdown output、usage、session 和 artifacts。
multi 且 collapsedrenderMultiCompactmode headline、step/agent 计数、chain/parallel 标签、失败/暂停/空输出行、artifact 目录。
multi 且 expandedrenderSubagentResult 的 multi expanded 分支每个 step 的 task、output target、skills、fallbacks、running detail、tool call lines。

场景函数结果
result 状态行resultStatusLinedetached、paused、error、acceptance、empty output、done 统一压成一行。
result glyphresultGlyphrunning 用 braille frame，detached/paused 用 warning square，failure 用 error cross，成功用 check。

5. async widget 渲染链路

阶段函数作用
数据映射summaryToJob从 background run summary 取 mode、agents、steps、parallel groups、tokens、activity、nested children。
安装 widgetrenderWidgetjobs 为空时清除 widget；有 UI 时设置 WIDGET_KEY 对应 component。
component 构建buildWidgetComponent按展开状态选择 buildWidgetLines、compactSingleWidgetLines 或 expanded 详情。
多 job 排序buildWidgetLinesrunning 在前，queued 汇总，finished 在后，超过 MAX_WIDGET_JOBS 时显示隐藏计数。
单 job 详情buildSingleWidgetLines显示 async subagent ... · background、job 状态、step 或泛化 activity。
空间适配fitAdaptiveWidgetLines根据终端 rows/columns 与展开状态，在 full、progressive、single-line 间选择。

6. async widget 布局层级

层级触发条件保留的信息
full折叠状态下行数仍能放入可用 widget 行。完整 job header、activity、step detail、nested summary。
progressive行数不足，但还不至于只剩一行。header、优先 active job、隐藏 job 计数；session 内锁定行数，减少抖动。
single-line终端几乎没有剩余行。总数摘要，如 running、queued、failed、paused、done。
expanded用户展开 tool detail。跳过折叠自适应 session，允许更多 live detail、recent tools、recent output。

9. 边界行为

边界处理方式意义
空输出但没有 output target显示 Done (no text output)，glyph 使用 warning success。避免用户误以为任务有正文输出。
task 中包含 output targetextractOutputTarget 识别 [Write to: ...]、Write your findings to...、Output:。空正文不一定是异常，可能写入了文件。
运行态动画foreground result 可由 frame 推进 glyph；async widget 依赖 progress seed 变化，不按动画 cadence 自刷。foreground 需要平滑 spinner，background 避免无意义刷新。
Unicode 与 ANSI 宽度truncLine 使用 visibleWidth 和 grapheme segmentation，并在省略号前恢复 ANSI 样式。避免 emoji、宽字符或颜色控制码造成错位。
nested childrenformatNestedWidgetLines collapsed 时聚合，expanded 时按预算展开。嵌套 run 不抢占主 job 的第一屏空间。
拥挤终端progressive 和 single-line fallback。保证 widget 不把主对话挤出屏幕。

11. 后续 L2 候选

候选主题本 L1 状态建议学习产物
renderSubagentResult compact/expanded 逐分支只讲入口分支。L2 短课：从 Details 到 result block 的完整读法。
async widget 自适应布局只讲 full/progressive/single-line。L2 参考：拥挤终端下的 row budget、session locking、active job selection。
chain/parallel 标签与 workflow graph span只列关键函数。L2 短课：dynamic fanout result index 如何映射回 logical step。
ANSI/Unicode 宽度裁剪只说明目的。L2 实验：构造宽字符、emoji、ANSI 样式，验证 truncLine 和 row 行宽。
状态语义与测试驱动阅读只收集证据表。L2 练习：从一个失败/暂停/空输出测试反推渲染路径。
