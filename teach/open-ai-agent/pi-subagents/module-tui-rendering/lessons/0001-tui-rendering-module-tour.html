TUI 结果渲染与 async widget 模块导览

学习目标：15 分钟内能读懂执行结果怎样被压缩成终端可读输出，以及后台 async run 怎样变成状态 widget。

1. 两个入口

open-ai-agent/pi-subagents/src/tui/render.ts 不是执行器，它只把已有状态变成 TUI 组件。入口有两类：renderSubagentResult 处理 foreground tool result，它拿到 Details 和每个 child 的 SingleResult；renderWidget 处理 background async jobs，它拿到 AsyncJobState[]，再调用 ctx.ui.setWidget。后台数据来自 open-ai-agent/pi-subagents/src/runs/background/async-job-tracker.ts，结果块接入点在 open-ai-agent/pi-subagents/src/extension/index.ts。

foreground result：tool result 已经回来，渲染器判断 single/multi、compact/expanded、running/done/failed。
background widget：async tracker 轮询或接收事件，转换为 job state，widget 只展示当前摘要。

2. 结果块先判断“显示模式”

读 renderSubagentResult 时，先不要追每个字段。它的分支顺序是：没有 details 就显示普通文本；single 且一个 result 时走单结果；否则走 multi-result。每条路径又由 expanded 决定是紧凑摘要还是完整详情。紧凑单结果最终会调用 resultStatusLine，把错误、暂停、detached、验收状态、空输出压成一行。

function resultStatusLine(result, output) {
  if (result.detached) return result.detachedReason ? `Detached: ${result.detachedReason}` : "Detached";
  if (result.interrupted) return "Paused";
  if (result.exitCode !== 0) return `Error: ${result.error ?? firstOutputLine(output)}`;
  if (result.acceptance?.status && result.acceptance.status !== "not-required") {
    return `Done · acceptance: ${result.acceptance.status}`;
  }
  if (hasEmptyTextOutputWithoutOutputTarget(result.task, output)) return "Done (no text output)";
  return "Done";
}

这段逻辑的价值是“少而稳”：终端第一屏先告诉用户状态，输出预览、session、artifact 路径再作为次级信息出现。

3. async widget 先排序，再降级

async widget 的思路相反：它面对的是一组仍在变化的 jobs。buildWidgetLines 会把 running 放前面，queued 汇总在中间，finished 放后面；单 job 会走 buildSingleWidgetLines，多 job 会显示 Async agents · background。如果终端空间不足，fitAdaptiveWidgetLines 会从 full 降到 progressive，再极限降到 single-line，并用固定 session 高度避免 UI 抖动。

full
行数够时展示完整 job 与 step 摘要。
progressive
行数拥挤时优先保留 active job，并显示隐藏计数。
single-line
几乎没有剩余行时只保留总状态，例如 running/queued/failed。

状态文案不全写在 widget 里。open-ai-agent/pi-subagents/src/shared/status-format.ts 提供 formatActivityLabel、aggregateStepStatus、formatParallelOutcome，所以 result 与 widget 对 “active 2s ago”“1 agent running · 1/3 done” 这类文案保持一致。

4. 检索练习

问题：如果一个 single subagent 正在运行，并且用户没有展开详情，你应该先去哪个函数找“当前 tool + live detail 提示 + output 路径”的显示逻辑？

参考答案：先看 open-ai-agent/pi-subagents/src/tui/render.ts 的 renderSingleCompact。它在 running 分支里调用 compactCurrentActivity，再显示 live detail hint 和 artifact output 路径。若是后台 async widget 的单 job，则看 buildSingleWidgetLines 与 foregroundStyleWidgetStepLines。

主要来源：open-ai-agent/pi-subagents/src/tui/render.ts、open-ai-agent/pi-subagents/src/tui/render-helpers.ts、open-ai-agent/pi-subagents/src/shared/status-format.ts。分支矩阵、helper 角色和测试证据见 TUI 渲染参考页。如果某个状态显示不符合直觉，可以继续向 agent 追问具体分支。
