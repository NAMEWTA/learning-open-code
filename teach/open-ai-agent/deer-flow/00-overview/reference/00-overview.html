本页是 L0 的速查表，用于给后续 L1 模块课、L2 垂直切片、L3 API 参考和 L4 深度剖析建立 backlog。短课入口：0001-project-map.html。

项目定位

维度结论依据
一句话DeerFlow 2 是带 Web/TUI 工作台的 LangGraph super agent harness，负责把模型、skills、tools、subagents、memory、sandbox 和持久化运行时组织成可执行 agent 系统。README.md、backend/README.md
主入口生产与本地开发默认通过 Gateway embedded runtime；backend/langgraph.json 保留给 LangGraph tooling/Studio 兼容。backend/README.md、docker/docker-compose.yaml
核心交互用户在线程中发送消息，Gateway 创建 run，lead agent 选择模型和工具，必要时委托 subagent 或使用 sandbox，结果通过 SSE 更新前端。frontend/src/app/workspace/chats/[thread_id]/page.tsx、backend/packages/harness/deerflow/runtime/runs/worker.py

核心架构决策

判断依据后续学习位置
Gateway 是默认运行外壳，LangGraph 文件是兼容入口。backend/README.md 说明 scripts/Docker 跑 embedded runtime；docker/docker-compose.yaml 的 gateway 服务直接 uvicorn 启动 app.gateway.app:app。L1 Gateway、L2 conversation streaming
能力通过配置与反射装配，而不是硬编码 provider。config.example.yaml 的 models[*].use、tools[*].use 是模块路径；tools.py 使用 resolve_variable 加载。L1 config/tools、L3 model/tool factory
上下文压力通过分层加载和持久提示缓解。README 描述 skills progressive loading；SkillActivationMiddleware、tool_search、summarization 与 durable context 同时存在。L2 skill activation、L4 context engineering
执行安全被放在 sandbox 与文件写入策略中。config.example.yaml 默认 allow_host_bash: false；read_before_write.enabled: true；Docker 可选 AIO/provisioner。L1 sandbox、L2 file lifecycle
长任务是 run、subagent、stream event、workspace change 的组合。runtime/runs/worker.py 处理 stream modes、subagent event buffer、workspace change；前端有 subtask card、workspace changes、token usage。L2 subagent delegation、L2 artifacts/workspace changes

首轮阅读顺序

  1. 先读 README.md 与 backend/README.md，确认项目定位、核心概念和运行拓扑。
  2. 再读 config.example.yaml 的顶层 section，建立“能力来自配置”的地图。
  3. 接着读 backend/app/gateway/app.py，看 HTTP 入口如何接入 runtime。
  4. 随后读 backend/packages/harness/deerflow/agents/lead_agent/agent.py，看 agent 如何被创建。
  5. 最后读 frontend/src/app/workspace/chats/[thread_id]/page.tsx 与 frontend/src/core/api/api-client.ts，把前端流式体验接回后端。

本参考只建立地图，不替代后续模块课。主 Agent 构建 backlog 时可优先选择“聊天流式执行”“lead agent”“Gateway”“sandbox”“subagent”这五条主线。
