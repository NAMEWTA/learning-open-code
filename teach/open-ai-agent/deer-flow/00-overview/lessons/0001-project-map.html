15 分钟建立 deer-flow 的全局项目地图

学习目标：读完后，你能把 deer-flow 放进一张“前端界面、Gateway、agent harness、工具与 sandbox、持久化”的地图里，并知道后续该从哪些模块继续下钻。

1. 一句话定位

DeerFlow 2 是一个开源 super agent harness：它不是只渲染聊天消息的应用，而是把 README.md 中强调的 sub-agents、memory、sandbox、skills 和 tools 组织成可运行的 agent 基础设施。你可以把它理解为“带前端工作台的 LangGraph agent 运行时”。

最小心智模型有三层：前端负责工作区体验；Gateway 负责 HTTP、认证、SSE、线程、上传和运行生命周期；真正的 agent 能力集中在 backend/packages/harness/deerflow/，其中 backend/langgraph.json 把 lead_agent 指到 deerflow.agents:make_lead_agent。

2. 请求如何穿过系统

用户浏览器 ↓ Next.js workspace 页面 frontend/src/app/workspace/ ↓ LangGraph SDK / REST API Gateway FastAPI app ↓ RunManager + StreamBridge lead_agent LangGraph graph ↓ middleware chain tools / subagents / memory / sandbox ↓ SSE values/messages/custom 前端消息、任务卡、artifact、token usage 更新

这个流向由三处互相印证：frontend/package.json 依赖 @langchain/langgraph-sdk；backend/app/gateway/app.py 注册 models、mcp、skills、threads、runs、uploads、scheduled tasks 等 router；backend/packages/harness/deerflow/agents/lead_agent/agent.py 在创建 agent 时选择模型、过滤 tools、装配 skill/search/subagent/memory/summarization 等中间件。

3. 从目录读职责

deer-flow/ ├── backend/ ← Python 后端与 deerflow-harness │ ├── app/ ← FastAPI Gateway、认证、渠道、调度入口 │ └── packages/ ← 核心 harness 包 deerflow.* ├── frontend/ ← Next.js 工作区、文档站、组件和前端状态 │ └── src/ ← App Router、core API、workspace UI ├── docker/ ← 生产/开发 Compose、nginx、provisioner ├── skills/ ← 内置 public Agent Skills ├── scripts/ ← setup、doctor、serve、deploy 等运维脚本 └── config.example.yaml ← 模型、工具、sandbox、memory、database 总开关

技术栈也顺着目录分开：后端是 Python 3.12、FastAPI、LangGraph SDK 与本地 workspace 包 deerflow-harness；前端是 Next.js 16、React 19、TypeScript、Tailwind、React Query；部署侧由 nginx、Gateway、frontend、Redis 和可选 sandbox provisioner 组成，证据在 backend/pyproject.toml、frontend/package.json 与 docker/docker-compose.yaml。

4. 三个架构判断

第一，运行时内嵌在 Gateway。 后端 README 明确说 langgraph.json 主要用于 LangGraph tooling/Studio，脚本和 Docker 默认跑 Gateway embedded runtime。读源码时先找 Gateway，再找 harness。

第二，扩展点配置化。 config.example.yaml 用 models[*].use、tools[*].use 这类模块路径装配模型和工具，tools.py 再用反射加载并去重。新增 provider 或 tool 通常先看配置边界。

第三，长任务靠“压缩 + 分工 + 隔离”。 lead agent 的中间件包含 summarization、memory、subagent limit、token budget、tool output budget；sandbox 配置默认关闭 host bash，说明项目把长上下文、并行委托和执行安全视为一组问题。

检索练习：不看上面的图，回答：如果你要研究“发送一条消息后如何流式返回到界面”，应该先看哪三个位置？
参考答案：前端入口 frontend/src/app/workspace/chats/[thread_id]/page.tsx，Gateway runs/thread_runs router 与 backend/app/gateway/app.py，以及 agent 运行入口 backend/packages/harness/deerflow/runtime/runs/worker.py 和 backend/packages/harness/deerflow/agents/lead_agent/agent.py。如果你答成“只看 README”，说明地图还没落到代码入口上；回到第 2 节重新顺一次箭头。
