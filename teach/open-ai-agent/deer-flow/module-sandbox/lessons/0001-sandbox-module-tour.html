15 分钟理解 sandbox 如何隔离执行、映射路径并支撑 artifact

学习目标：读完后，你能用一条主线解释 agent 为什么只能看到 `/mnt/user-data/...`，工具如何拿到线程 sandbox，以及写到 outputs 的文件为什么能被 Gateway 当作 artifact 取回。

1. 先记住三个目录

L0 已经把 sandbox 放在执行安全与文件产物边界上。DeerFlow 给每个线程准备一组 host 目录，再把它们暴露成稳定的虚拟路径。关键入口是 backend/packages/harness/deerflow/agents/middlewares/thread_data_middleware.py 和 backend/packages/harness/deerflow/config/paths.py。

host: {DEER_FLOW_HOME}/users/{user_id}/threads/{thread_id}/user-data/ │ ├── workspace → sandbox: /mnt/user-data/workspace ← agent 工作区 ├── uploads → sandbox: /mnt/user-data/uploads ← 用户上传文件 └── outputs → sandbox: /mnt/user-data/outputs ← agent 生成产物

这里的重点不是“目录名”，而是边界：agent prompt 和工具参数都应该使用虚拟路径；host 绝对路径只在 Gateway 和 provider 内部出现，并在工具输出中被遮回虚拟路径。

2. sandbox id 是延迟拿到并写回 state 的

backend/packages/harness/deerflow/sandbox/middleware.py 默认 lazy init。也就是说，run 开始时先有 thread_data，第一次调用 bash、ls 或 read_file 这类工具时，ensure_sandbox_initialized() 才会调用 provider 的 acquire(thread_id, user_id)。

ThreadDataMiddleware.before_agent ↓ ThreadState.thread_data = workspace/uploads/outputs host paths ↓ file or bash tool call ↓ ensure_sandbox_initialized(runtime) ↓ SandboxProvider.acquire(thread_id, user_id) ↓ ThreadState.sandbox = {"sandbox_id": "..."} ↓ SandboxMiddleware.wrap_tool_call emits Command(update=...)

最后一步很容易漏：工具内部直接改 runtime.state 不等于 LangGraph 已持久化 state，所以 middleware 会把新 sandbox id 合并进 Command(update=...)。Lead agent 参考页提到的 ThreadState.sandbox 就在这里落地。

3. Local、AIO、E2B 共享同一套 Sandbox API

公共契约在 backend/packages/harness/deerflow/sandbox/sandbox.py：execute_command、read_file、download_file、list_dir、write_file、glob、grep、update_file。provider 可以不同，但工具层只面向这个抽象。

sandbox = ensure_sandbox_initialized(runtime)
thread_data = get_thread_data(runtime)

if is_local_sandbox(runtime):
    validate_local_tool_path(path, thread_data)
    path = resolve_and_validate_user_data_path(path, thread_data)

sandbox.write_file(path, content)

Local provider 不是强隔离 shell，默认 allow_host_bash: false；AIO 用容器或 provisioner；E2B 用云端 micro-VM。它们都必须遵守 `/mnt/user-data` 契约。这个契约由 backend/tests/test_local_sandbox_virtual_path_contract.py 固定：直接调用 LocalSandbox.write_file("/mnt/user-data/workspace/a.txt", ...) 也要成功，而不是只靠工具层临时翻译。

4. artifact 是 outputs 目录的反向读取

当 agent 写入 /mnt/user-data/outputs/report.html，文件实际落在当前线程的 host outputs 目录。Gateway 的 backend/app/gateway/routers/artifacts.py 会通过 resolve_thread_virtual_path(thread_id, path) 把 URL 中的 mnt/user-data/outputs/report.html 转回 host path，再按 MIME 类型返回；HTML、SVG 这类主动内容会强制下载，避免在应用同源下执行。

检索练习：判断下面说法是否正确：用户上传的 `data.csv` 和 agent 生成的 `chart.svg` 都通过同一个虚拟根 `/mnt/user-data` 暴露给 agent，但它们默认落在不同子目录。
参考答案：正确。上传文件在 `/mnt/user-data/uploads`，生成产物应写到 `/mnt/user-data/outputs`。前者由 uploads router 管入口，后者由 artifacts router 管读取；二者共享线程级 user-data 根，但语义不同。
