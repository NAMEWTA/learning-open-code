Sandbox 抽象接口：sandbox/sandbox.py

成员签名返回值边界与异常
Sandbox.__init__ (id: str) 设置 self._id —
Sandbox.id property str 只读；Local 为 local 或 local:{user}:{thread}；AIO/E2B 为 provider 分配的 id
execute_command (command: str, env: dict[str,str]|None=None, timeout: float|None=None) -> str 命令 stdout/stderr 合并文本 env 键须匹配 ^[A-Za-z_][A-Za-z0-9_]*$，否则 ValueError（_validate_extra_env）；Local 用 timeout 杀进程组；远程实现可忽略 timeout
read_file (path: str) -> str UTF-8 文本内容 路径为 sandbox 内绝对路径（虚拟或已解析）；二进制/不存在由各实现决定（Local 抛 OSError/FileNotFoundError）
download_file (path: str) -> bytes 原始字节 路径穿越 → PermissionError；读失败 → OSError（契约要求统一）
list_dir (path: str, max_depth=2) -> list[str] 树形文本行列表 默认深度 2；目录不存在由各实现映射为异常或空
write_file (path: str, content: str, append: bool=False) -> None — append=False 覆盖；只读 mount → PermissionError
glob (path, pattern, *, include_dirs=False, max_results=200) -> tuple[list[str], bool] 匹配路径列表 + 是否截断 根目录不存在 → FileNotFoundError；非目录 → NotADirectoryError
grep (path, pattern, *, glob=None, literal=False, case_sensitive=False, max_results=100) -> tuple[list[GrepMatch], bool] 匹配列表 + 是否截断 跳过二进制、超大文件、symlink escape；正则 ReDoS 防护（行长上限）
update_file (path: str, content: bytes) -> None — Gateway uploads 同步远程 sandbox 时使用；与 write_file 文本路径互补

Provider 工厂：sandbox/sandbox_provider.py

符号签名行为边界
SandboxProvider ABC 类属性：uses_thread_data_mounts、needs_upload_permission_adjustment Gateway uploads 根据后者决定是否 chmod
acquire (thread_id: str|None=None, *, user_id: str|None=None) -> str 创建或复用 sandbox，返回 sandbox_id 抽象方法；thread_id=None 时 Local 返回 legacy local
acquire_async async (...) -> str 默认 asyncio.to_thread(self.acquire, ...) AIO/Boxlite 可覆盖以避免阻塞事件循环
get (sandbox_id: str) -> Sandbox|None 内存查找，不触发 IO 已 release 的 id 返回 None
release (sandbox_id: str) -> None 销毁或归还 warm pool 抽象方法
reset () -> None 清缓存，默认 no-op reset_sandbox_provider() 调用；不 shutdown 活跃 sandbox
get_sandbox_provider (**kwargs) -> SandboxProvider 按 config.sandbox.use 反射构造单例 threading.Lock 保护；构造在锁外，竞态失败者 shutdown() 孤儿实例（#3721）
reset_sandbox_provider () -> None 清单例 + 调 provider.reset() 不 shutdown；活跃 sandbox 可能孤儿
shutdown_sandbox_provider () -> None shutdown + 清单例 应用退出时调用
set_sandbox_provider (provider: SandboxProvider) -> None 测试注入 mock 不 shutdown 被替换的旧实例

配置解析：resolve_class(config.sandbox.use, SandboxProvider)。常见值：deerflow.sandbox.local:LocalSandboxProvider、deerflow.community.aio_sandbox:AioSandboxProvider、deerflow.community.e2b_sandbox:E2BSandboxProvider、deerflow.community.boxlite:BoxliteProvider。sandbox 在 STARTUP_ONLY_FIELDS，修改需重启 Gateway。

测试：test_sandbox_provider_lifecycle.py、test_aio_sandbox_provider.py

Agent 可见工具（LangChain @tool）

所有工具第一个业务参数为 description: str（模型必须先解释意图）。错误统一返回 "Error: ..." 字符串。

工具名签名（除 runtime 外）默认 / 上限Local 额外校验
bash (description, command: str) -> str 输出截断 bash_output_max_chars；超时 bash_command_timeout 需 allow_host_bash；路径校验 + 虚拟路径替换 + cd workspace 前缀；注入 env secrets
ls (description, path: str) -> str list_dir 深度 2；ls_output_max_chars validate_local_tool_path(read_only=True)；disabled skill 拒绝
glob (description, pattern, path, include_dirs=False, max_results=200) -> str max_results 上限钳制 _resolve_local_read_path
grep (description, pattern, path, glob=None, literal=False, case_sensitive=False, max_results=100) -> str 同上 同上
read_file (description, path, start_line=None, end_line=None) -> str read_file_output_max_chars disabled skill 拒绝；二进制 → 友好错误提示
write_file (description, path, content, append=False) -> str 大小上限 _effective_write_file_max_bytes() 写路径校验；文件操作锁 file_operation_lock；受 ReadBeforeWrite 门闩
str_replace (description, path, old_str, new_str, replace_all=False) -> str — 按 (sandbox.id, path) 串行化；ReadBeforeWrite 门闩

异步路径：每个工具注册 .coroutine = _*_async，内部先 ensure_sandbox_initialized_async，再 asyncio.to_thread 执行同步体。

测试：test_write_file_tool_size_guard.py、test_read_before_write_middleware.py、test_sandbox_search_tools.py
