Frontend workspace 模块速查

这份 reference 是 DeerFlow 前端工作台的地图：页面在哪里、组件如何分层、哪些 core API 接到 Gateway 或 LangGraph SDK。 lesson 只保留一条学习主线，完整清单集中放在这里。

模块：workspace 层级：L1 源码：frontend/src

本页是 L1 Frontend workspace 的速查表。短课入口：0001-frontend-workspace-module-tour.html；微观 API：frontend-workspace-api.html；整体锚点见 L0 项目总览。

一眼架构

在 L0 架构图中，本模块位于 Next.js frontend/workspace 层：它把用户可见的聊天、agent 管理、定时任务、artifact 与 sidecar 体验接到 Gateway 和 LangGraph SDK。

用户 URL -> Next.js App Router: src/app/workspace/** -> WorkspaceContent: QueryClient + Sidebar + Banner + CommandPalette -> workspace components: chat / agents / scheduled tasks / side panels -> core hooks: threads / agents / scheduled-tasks / workspace-changes -> core/api: LangGraph SDK client 或 fetchWithAuth -> Gateway: /api/langgraph、/api/threads、/api/agents、/api/scheduled-tasks

目录骨架

frontend/src/ ├── app/ ← Next.js App Router 入口 │ └── workspace/ ← 工作台路由组 │ ├── layout.tsx ← 登录态、Gateway 离线兜底、工作台壳 │ ├── workspace-content.tsx ← QueryClient、Sidebar、Toast、CommandPalette │ ├── page.tsx ← /workspace 重定向到聊天入口 │ ├── chats/ ← 聊天列表与线程页面 │ ├── agents/ ← Agent 管理与 agent 专属聊天 │ └── scheduled-tasks/ ← 定时任务页面 ├── components/ ← React 组件 │ └── workspace/ ← 工作台专用 UI 组件族 └── core/ ← 前端业务数据与 API 适配 ├── api/ ← LangGraph SDK client 与 fetch wrapper ├── threads/ ← thread、run、stream、history ├── tasks/ ← subtask 事件与前端状态 ├── scheduled-tasks/ ← scheduled tasks CRUD └── workspace-changes/ ← run 文件变更摘要与 diff

core API 与状态层

层 关键文件 对外连接 主要用途
统一 SDK client frontend/src/core/api/api-client.ts /api/langgraph 或 NEXT_PUBLIC_LANGGRAPH_BASE_URL 创建 LangGraph Client，注入 CSRF，包装 stream/cancel/joinStream，支持静态 demo client。
REST fetch wrapper frontend/src/core/api/fetcher.ts Gateway REST API 统一带 cookie credentials；POST/PUT/PATCH/DELETE 自动带 X-CSRF-Token；401 跳登录。
线程 API frontend/src/core/threads/api.ts GET /api/threads/{threadId}/token-usage
POST /api/threads/{threadId}/branches		 token usage、conversation branch 等非 SDK 封装接口。
线程 hooks frontend/src/core/threads/hooks.ts LangGraph threads/runs + REST regenerate prepare useThreadStream、历史分页、threads 列表、metadata、token usage、branch mutation。
subtask 状态 frontend/src/core/tasks/context.tsx
frontend/src/core/tasks/api.ts		 GET /api/threads/{threadId}/runs/{runId}/events 把 subagent.step 事件还原成 subtask steps，供消息里的 SubtaskCard 展示。
scheduled tasks frontend/src/core/scheduled-tasks/api.ts
frontend/src/core/scheduled-tasks/hooks.ts		 /api/scheduled-tasks
/api/threads/{threadId}/scheduled-tasks		 定时任务 CRUD、暂停/恢复/触发、runs 查询和 React Query 缓存失效。
agents frontend/src/core/agents/api.ts
frontend/src/core/agents/hooks.ts		 /api/features
/api/agents		 feature flag、Agent 列表、读取、创建、更新、删除、名称检查。
workspace changes frontend/src/core/workspace-changes/api.ts
frontend/src/core/workspace-changes/hooks.ts		 GET /api/threads/{threadId}/runs/{runId}/workspace-changes 加载文件变更摘要和 diff；badge 先不取 diff，面板打开后再取详情。

关键调用示例：聊天提交

ChatPage.handleSubmit
  -> sendMessage(threadId, message)
  -> useThreadStream.sendMessage
  -> uploadFiles(threadId, files)           // 有附件时
  -> thread.submit(
       { messages: buildThreadSubmitMessages(...) },
       {
         threadId,
         streamSubgraphs: true,
         streamResumable: true,
         context: {
           thread_id: threadId,
           thinking_enabled: context.mode !== "flash",
           is_plan_mode: context.mode === "pro" || context.mode === "ultra",
           subagent_enabled: context.mode === "ultra"
         }
       }
     )
  -> LangGraph stream events update thread.messages
  -> MessageList / MessageListItem render UI
  -> invalidate threads list and token usage caches on finish

阅读顺序建议

  1. 先读 frontend/src/app/workspace/layout.tsx 和 frontend/src/app/workspace/workspace-content.tsx，确认所有工作台页面共享什么壳。
  2. 再读 frontend/src/app/workspace/chats/[thread_id]/page.tsx，只追 useThreadChat、useThreadStream、InputBox、MessageList 四个名字。
  3. 最后跳到 frontend/src/core/api/api-client.ts 和 frontend/src/core/threads/hooks.ts，确认数据真正从哪里来。
