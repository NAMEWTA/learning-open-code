sequenceDiagram
    participant U as 用户消息
    participant SAM as SkillActivationMiddleware
    participant DC as DurableContextMiddleware
    participant LLM as 模型
    participant Tool as read_file/bash

    alt 解析失败
      SAM-->>U: AIMessage 错误（不调用 LLM）
    else 激活成功

      SAM->>LLM: override messages
      loop 工具环（多轮 model call）
        LLM->>Tool: 可选 read 辅助资源
        Tool-->>LLM: ToolMessage

        SAM->>SAM: 重算 secret bindings（slash 路径持久）
        SAM->>LLM: 后续 model call
      end
    end

Slash 解析与激活异常

条件行为用户可见文案模式
非 /name 语法（前导空格、无分隔符）不激活，走普通对话（无 middleware 错误）
保留命令 /new、/goal、/help 等parse_slash_skill_reference 返回 None由 channel / 前端处理
skill 未安装返回 AIMessageSkill `/name` is not installed.
已安装但 disabled返回 AIMessage…installed but disabled…
不在 agent allowlist返回 AIMessage…not available for this agent.
路径/读文件不安全返回 AIMessage…could not be loaded safely…
同一用户消息已激活过跳过二次注入（静默）

递进加载与 durable 边界

场景checkpoint 存什么下轮 model 请求
仅 slash 激活通常无 skill_context 条目（除非又 read_file）激活当轮：隐藏块含全文；之后依赖模型记忆或再读
read_file SKILL.mdskill_context[] 仅元数据render_skill_context 紧凑列表
summarization 压缩旧 tool 消息skill_context 仍保留（durable 通道）提醒仍在 durable_context_data
skill 被 operator 禁用条目可能仍在 state_resolve_registry_skill 失败 → secret 不绑定；需重读时工具报错
read 非 SKILL.md 或路径越界不写入 skill_context无 skill 提醒

Request-scoped secrets（slash vs in-context）

来源触发重验证secrets-autonomous opt-out
Slash 激活用户 /skill激活时 enabled+allowlist；路径存 run context豁免（显式仪式）
In-contextskill_context 条目每轮对 live registry 按 path 解析遵守 secrets-autonomous: false

注入值仅来自 context.secrets；registry 加载失败时当次 bind nothing（fail closed）。 审计日志只记 skill/secret 名称，不记值。

与 deferred_discovery 的关系

skills.deferred_discovery: true 时，系统提示词只给 <skill_index> 名称列表； agent 需先 describe_skill 再 read_file——走的是递进加载路径，不是 slash。 Slash 与 deferred 正交：用户随时可用 /name 跳过索引，直接注入该 skill 全文。

测试锚点： backend/tests/test_skill_request_scoped_secrets.py（slash 与 in-context 密钥）、 harness 内 slash 相关单测。 排障顺序：extensions_config 里 skill 是否 enabled → custom agent allowlist → 用户消息是否匹配 ^/([a-z0-9]+(?:-[a-z0-9]+)*)(?:\s+|$) → 隐藏激活消息是否已插入。
