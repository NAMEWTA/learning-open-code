15 分钟串起 skill slash 激活与递进加载

学习目标：用户输入 /chart-visualization 画折线图 或 agent 自行 read_file 读 SKILL.md 时，你能指出正文落在哪条 middleware 链、checkpoint 里又存了什么。 失败分支与密钥绑定表见 skill-activation-flow-map 参考页。

L2 垂直切片 入口：用户消息 /skill-name 关联：module-skills · lead-agent

1. 两条路径：显式仪式 vs 工具递进

DeerFlow 不把全部 SKILL.md 塞进系统提示词。Slash 激活是用户显式仪式： skills/slash.py 解析严格语法 /name 任务， skill_activation_middleware.py 在当次 model call 前插入隐藏的完整正文。 递进加载是 agent 用 read_file 读 skills/public/<name>/SKILL.md（或 custom 目录）； skill_context.py 只把 name、path、description 写入 ThreadState.skill_context，正文不进 checkpoint。

deer-flow/ ├── skills/ │ ├── public/chart-visualization/SKILL.md ← 仓库内示例 skill │ └── custom/… ← 用户安装，gitignore └── backend/packages/harness/deerflow/skills/ ├── slash.py ← 解析 /name └── storage/ ← load_skills + enabled 状态

2. Slash 激活主链（happy path）

SkillActivationMiddleware 挂在 lead middleware 链第 13 位（在 DynamicContext 之后）。 每次 wrap_model_call 从最新消息向前找真实用户 HumanMessage，用 get_original_user_content_text 读原文（InputSanitization 已保留）。 解析成功则 _read_skill_content 读磁盘，生成带 hide_from_ui 的激活提醒，插在用户消息之前。 

/chart-visualization 画月度趋势
  → parse_slash_skill_reference
  → storage.load_skills → enabled + allowlist
  → 读 SKILL.md 全文 → 隐藏 HumanMessage
  → handler(model) 模型本轮看到完整 skill
sequenceDiagram
    participant U as 用户
    participant SAM as SkillActivationMiddleware
    participant ST as SkillStorage
    participant M as LLM

    U->>SAM: HumanMessage "/chart-visualization 任务"
    SAM->>SAM: parse_slash_skill_reference
    SAM->>ST: load_skills + resolve_slash_skill
    ST-->>SAM: Skill + container path
    SAM->>SAM: read SKILL.md + build_activation_reminder
    SAM->>M: messages 插入隐藏激活块 + 原用户消息
    M-->>U: 按 skill 指令执行（工具环多轮 model call）

同一用户消息不会重复激活：_has_existing_activation_for_target 检查前一条是否已是激活提醒。 Slash 还会在 run context 记下容器路径，供后续 tool loop 的 request-scoped secrets 绑定（整 run 有效）。

3. 递进加载：read_file → durable 提醒

当模型自己 read_file 读取 skills 根下的 SKILL.md 时， ToolErrorHandlingMiddleware 在 ToolMessage 上打 skill_context_entry 元数据。 DurableContextMiddleware 在 before_model 用 extract_skills 枚举 AI 调用与 ToolMessage 配对，合并进 ThreadState.skill_context（按 path 去重，保留最近读取）。 之后每轮 model call 注入的是 render_skill_context 生成的紧凑列表——提醒「已加载过，必要时重读文件」，而不是再次嵌入正文。 

read_file(/mnt/skills/public/foo/SKILL.md)
  → ToolMessage + skill_context_entry
  → DurableContextMiddleware → state.skill_context[]
  → 下轮 <durable_context_data> 仅 name/path/description

4. 异常路径与检索练习

解析失败时 middleware 直接返回 AIMessage 错误文案，不调用模型：未安装、已禁用、不在 agent allowlist、路径不安全、保留命令（/new、/goal 等）。 密钥侧：registry 读取失败时 fail closed，当次不注入 secrets。 完整错误表见 参考页 · 异常。

判断题：agent 在三轮前用 read_file 加载过 data-analysis，本轮用户未打 slash，summarization 已压缩旧 tool 消息。下列哪项仍可能出现在下轮 model 请求里？

A. 完整 SKILL.md 正文   B. skill_context 中的 path 提醒   C. 新的 slash 激活块   D. 以上都不会

参考答案：B。正文只在 slash 当轮或当次 read 的 tool 结果里出现；checkpoint 的 skill_context 经 DurableContextMiddleware 投影为紧凑提醒。C 仅当用户本轮发送 /skill。A 不会因 summarization 自动回放全文。

主要来源：backend/AGENTS.md Middleware 链与 Skills System 节。 想继续下钻「secrets 双来源 union」或 deferred_discovery，向 agent 追问即可。
