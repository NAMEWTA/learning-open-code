主路径时序

sequenceDiagram participant M as Lead agent (LLM) participant T as task_tool participant E as SubagentExecutor participant W as run worker participant S as SSE / StreamBridge participant UI as SubtaskCard M->>T: task(description, prompt, subagent_type) T->>T: get_subagent_config（失败则立即 ToolMessage） T->>E: execute_async(prompt, task_id=tool_call_id) T->>S: task_started loop 每 5s 轮询 E-->>T: SubagentResult（status + ai_messages） T->>S: task_running（新 AI 消息） W->>W: _SubagentEventBuffer → subagent.step end T->>S: task_completed / task_failed / … T-->>M: ToolMessage(subagent_status, …) S-->>UI: task_running 累积 steps UI->>UI: parseSubtaskResult → 终态 pill

状态契约（后端 → 前端）

正文 ToolMessage.content 仅供模型阅读；UI 与 durable ledger 读 additional_kwargs。枚举由 contracts/subagent_status_contract.json 与 test_subagent_status_contract.py 双端锁定。

常见边界路径

配置层（未启动后台任务）

  • Unknown subagent type — get_subagent_config 返回 None
  • bash disabled — subagent_type=bash 且 host bash 不允许

执行层

  • Task disappeared — 轮询时 _background_tasks 无条目
  • max_turns_reached — GraphRecursionError；保留 partial result
  • polling_timed_out — 请求取消并 schedule deferred cleanup

前端契约层

  • 有 structured key 但 status 无法映射 → 故意保持 in_progress，提示契约漂移
  • 历史 run 无 live steps → 展开卡片触发 fetchSubtaskSteps
