15 分钟理解 Gateway 的边界和入口

学习目标：读完后，你能判断一个请求是否属于 Gateway 职责，知道入口在哪里，并能把 router、认证、CSRF、runtime 单例和 lead agent 接线放回同一张图。

1. Gateway 在系统里的位置

deer-flow 默认不是单独跑 LangGraph Server，而是让 backend/app/gateway/app.py 创建 FastAPI 应用，并在同一个进程里接入 embedded agent runtime。它对应 L0 总体架构图里的 Gateway 层。nginx 的 /api/langgraph/* 只是改写到 Gateway 原生 /api/*，所以前端、IM channel 和兼容 LangGraph 的调用最终都会碰到这层外壳。

Browser / IM / LangGraph-compatible client ↓ nginx same-origin endpoint Gateway FastAPI app ↓ middleware: auth、CSRF、CORS、trace router family: threads、runs、models、skills、uploads、memory ↓ request dependencies from deps.py RunManager + StreamBridge + checkpointer + store ↓ lead_agent graph and deerflow harness

2. app.py 负责四件事

第一，启动时读取配置、配置日志、预热 token 计数、清理上传暂存文件。第二，进入 backend/app/gateway/deps.py 的 langgraph_runtime()，把 stream_bridge、checkpointer、store、run_manager、thread store、feedback repo 等对象放到 app.state。第三，注册全局中间件：认证默认 fail-closed，CSRF 保护状态变更请求，CORS 只在显式 allowlist 时启用，trace 负责请求关联。第四，挂载所有 router，并提供公开的 /health。

backend/app/gateway/ ├── app.py ← FastAPI app、lifespan、middleware、router 注册 ├── config.py ← GATEWAY_HOST、GATEWAY_PORT、docs 开关 ├── deps.py ← request 级依赖和 runtime 单例访问 ├── auth_middleware.py ← session、internal token、auth-disabled 边界 ├── csrf_middleware.py ← Double Submit Cookie 与 origin 检查 ├── routers/ ← HTTP API 家族，长清单见参考页 └── auth/ ← local、OIDC、JWT、用户仓储

3. 对外接口先按家族看

不要一上来背每个 endpoint。先把接口分成六组：运行主线是 threads、thread_runs、runs；能力配置是 models、mcp、skills、agents、memory；文件与结果是 uploads、artifacts；账户边界是 auth；集成入口是 channels、github_webhooks；观测和辅助是 console、features、feedback、suggestions。完整接口表放在 Gateway 参考页。

最小无认证调用是健康检查：

curl -s http://localhost:8001/health
# {"status":"healthy","service":"deer-flow-gateway"}

除 /health、文档页、少量登录/注册/OAuth 和 webhook 外，业务 API 会受 AuthMiddleware 保护。状态变更请求还会受 CSRFMiddleware 校验；登录、注册、初始化这类首个 auth POST 会由中间件发放 csrf_token cookie。

4. 你该如何继续读源码

第一遍只读 create_app() 和 lifespan()，确认“启动建单例，路由按组挂载”。第二遍读 deps.py 的 getter，理解 router 为什么通过 request dependency 拿 runtime 对象，而不是直接 import 全局变量。第三遍再选择一个 router 下钻，例如 thread_runs.py 研究一次聊天 run，或 auth.py 研究登录会话。

检索练习：不看上文，判断下面三句话是否正确：A. Gateway 只负责 HTTP 转发，不持有运行时单例。B. /api/langgraph/* 在 nginx 后会进入 Gateway。C. 所有 POST 都必须先有 CSRF token。
参考答案：A 错，langgraph_runtime() 会把 RunManager、StreamBridge、checkpointer 和 store 放进 app.state。B 对，nginx 会改写到 /api/*。C 不完全对，登录、注册、初始化等 auth POST 首次调用不做 double-submit 校验，但会检查 hostile Origin，并在响应里设置 CSRF cookie；webhook 与 auth-disabled 路径还有各自的豁免规则，后续认证切片再展开。

主要来源：backend/app/gateway/app.py、backend/app/gateway/deps.py 和 backend/README.md。如果某个接口家族不清楚，向 agent 追问“沿着 Gateway 的某个 router 继续展开”。
