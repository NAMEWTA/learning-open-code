Memory 与 summarization 参考

本页是 L1 模块参考：职责边界、middleware 顺序、配置、数据结构与源码索引。短课入口：0001-memory-context-module-tour.html；L3 API 字典见 memory-context-api.html；整体锚点见 L0 项目总览、lead agent 参考。

模块职责与位置

维度结论源码证据
Memory跨线程、跨 run 的用户画像与 facts；异步 LLM 更新，文件存储于 per-user memory.json。agents/memory/updater.py、agents/memory/storage.py
Summarization单线程内接近上下文上限时压缩 messages，摘要写入 ThreadState.summary_text。agents/middlewares/summarization_middleware.py
Durable context在 summarization 前捕获 delegation / skill 引用；每次 model call 把 summary_text 等投影为隐藏数据块。agents/middlewares/durable_context_middleware.py
Memory 注入读取持久化 memory，以 <memory> 注入当前轮（与 static system prompt 分离）。agents/middlewares/dynamic_context_middleware.py
架构位置三条链路都挂在 lead agent middleware 链上，位于 sandbox / tool 处理之后、clarification 之前。agents/lead_agent/agent.py build_middlewares

内部分层结构

backend/packages/harness/deerflow/ ├── agents/ │ ├── memory/ │ │ ├── queue.py ← debounce 队列，Timer 线程处理 │ │ ├── updater.py ← LLM 抽取 facts / context，原子写盘 │ │ ├── storage.py ← FileMemoryStorage，per-user 隔离 │ │ ├── prompt.py ← 更新 prompt 与 injection token 预算 │ │ ├── message_processing.py ← 过滤 tool 消息，检测 correction │ │ └── summarization_hook.py ← before_summarization → memory 队列 │ ├── middlewares/ │ │ ├── durable_context_middleware.py │ │ ├── summarization_middleware.py ← DeerFlowSummarizationMiddleware │ │ ├── memory_middleware.py │ │ └── dynamic_context_middleware.py ← <memory> 注入 │ └── thread_state.py ← summary_text LastValue 通道 └── config/ ├── memory_config.py └── summarization_config.py

端到端数据流

Turn N — model call DurableContext.wrap_model_call → 注入 summary_text + delegations + skills DynamicContext（同链更早阶段）→ 注入 <memory> + 日期 ↓ Turn N — summarization（若 trigger 满足） memory_flush_hook → queue.add_nowait(即将删除的 messages) DeerFlowSummarizationMiddleware → messages 替换 + summary_text 更新 ↓ Turn N — after_agent MemoryMiddleware → queue.add(本轮 user+final AI) ↓ debounce（默认 30s） MemoryUpdater + LLM → memory.json（跨线程持久） ↓ Turn N+1 DynamicContext 读取 memory.json → 下一轮 <memory>

Lead middleware 顺序（相关片段）

顺序MiddlewareHook作用
12DynamicContextMiddlewarebefore_agent / abefore_agent在 turn 开始时注入日期 + 可选 <memory>（读 memory.json）。
14DurableContextMiddlewarebefore_model / wrap_model_call捕获 delegations、skills；注入 durable 数据块。
15DeerFlowSummarizationMiddlewarebefore_model超阈值压缩 messages；before_summarization 调 memory flush。
19MemoryMiddlewareafter_agent本轮对话入 memory 更新队列。

Summarization 默认 enabled: false；Memory 默认 enabled: true。两者独立开关，可同时启用。

Memory 数据结构（memory.json）

区块字段示例用途
User ContextworkContext、personalContext、topOfMind1–3 句段落级摘要
HistoryrecentMonths、earlierContext、longTermBackground时间分层背景
Factsid、content、category、confidence、source离散事实；注入时按 max_injection_tokens 与 guaranteed_categories 做 token 预算筛选（非固定条数）

主路径：{base_dir}/users/{user_id}/memory.json；custom agent 为 {base_dir}/users/{user_id}/agents/{agent_name}/memory.json。legacy 无 user_id 时回退 agents/{agent_name}/memory.json。绝对 storage_path 会退出 per-user 隔离。

配置速查（config.yaml）

memory.*

键默认说明
enabledtrue关闭后 MemoryMiddleware 与 flush hook 均不入队。
injection_enabledtrue控制 DynamicContext 是否注入 <memory>。
debounce_seconds30队列合并窗口（1–300）。
model_namenullmemory 更新 LLM；null 用默认模型。
max_facts / fact_confidence_threshold100 / 0.7fact 存储上限与置信度门槛。
max_injection_tokens2000注入 prompt 的 token 预算。
token_countingtiktokenchar 可避免 tiktoken 首次下载阻塞。
guaranteed_categories["correction"]高优先级 fact 单独预算，避免被挤掉。

summarization.*

键默认说明
enabledfalse主开关。
triggernulltokens / messages / fraction 阈值，可列表 OR。
keepmessages: 20压缩后保留多少近期历史。
trim_tokens_to_summarize4000送入摘要模型的输入上限。
model_namenull摘要专用模型，建议轻量。
skill_file_read_tool_namesread_file, read, view, cat判定 skill 文件读取的 tool 名，供 DurableContext 捕获。

Gateway Memory API

方法路径用途
GET/api/memory读取当前用户 memory 数据。
POST/api/memory/reload强制从磁盘 reload 缓存。
DELETE/api/memory清空 memory。
GET/api/memory/config返回 memory 配置快照。
GET/api/memory/status配置 + 数据联合状态。
POST/PATCH/DELETE/api/memory/facts...facts CRUD 与 import/export（见 gateway router）。

用户隔离：HTTP 调用走 get_effective_user_id()；IM channel 可通过受信 header 代 owner 操作。
