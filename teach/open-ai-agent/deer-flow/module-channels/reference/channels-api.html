MessageBus 方法

方法签名行为异常 / 边界
publish_inboundasync (msg) → None入队；adapter / webhook fan-out 调用无背压上限；队列无限增长（依赖 manager 消费速度）
get_inboundasync () → InboundMessage阻塞直到下一条入站仅 manager _dispatch_loop 消费
subscribe_outbound(callback) → None注册 async 出站监听；各 Channel.start 订阅 _on_outbound回调异常被 publish_outbound 捕获并记日志，不中断其他 listener
unsubscribe_outbound(callback) → NoneChannel.stop 时移除—
publish_outboundasync (msg) → None顺序 await 所有 listenermanager 在 runs.wait / stream 完成后调用

用户绑定路由：routers/channel_connections.py

backend/app/gateway/routers/channel_connections.py · 同前缀 /api/channels（与运维路由并存）

路由响应认证行为 / 边界
GET /providersChannelProvidersResponse登录用户channel_connections.enabled=false → enabled:false 空列表
GET /connectionsChannelConnectionsResponse登录用户仅当前用户绑定行
POST /{provider}/connectChannelConnectResponse登录用户secrets.token_urlsafe(16)；TTL 600s；每 provider 最多 5 个 pending
POST /{provider}/runtime-configChannelProviderResponseadmin写入凭证 + configure_channel 热启动 worker
DELETE /{provider}/runtime-configChannelProviderResponseadminremove_channel + 清 runtime store
DELETE /connections/{connection_id}204登录用户仅能删自己的连接；不存在 → 404

支持 provider：telegram（deep_link）· slack · discord · feishu · dingtalk · wechat · wecom。绑定码在 adapter 入站前消费，安全靠码机密性而非 allowed_users。

backend/app/gateway/routers/github_webhooks.py · 前缀 /api/webhooks

符号 / 路由签名 / 行为状态码边界

调用示例

# 运维状态
curl -b cookies.txt http://localhost:8001/api/channels/

# 用户发起 Telegram 绑定（需登录 + CSRF）
curl -b cookies.txt -H "X-CSRF-Token: $CSRF" \
  -X POST http://localhost:8001/api/channels/telegram/connect

# GitHub webhook（仅验签，无 cookie）
curl -X POST http://localhost:8001/api/webhooks/github \
  -H "X-GitHub-Event: issue_comment" \
  -H "X-Hub-Signature-256: sha256=..." \
  -d @payload.json
