消息流（速查）

[IM worker] platform event → Channel.publish_inbound → MessageBus queue → ChannelManager (dedupe / bound identity) → langgraph_sdk: threads.create | runs.wait | runs.stream | runs.create → OutboundMessage → Channel.send (流式平台可多次 is_final=false) [GitHub] POST /api/webhooks/github → HMAC verify → fanout_event (per agent binding) → MessageBus (N × InboundMessage) → ChannelManager runs.create (fire_and_forget) → agent 在 sandbox 内 gh comment（非自动 outbound）

Gateway API 清单

运维 — channels.py

方法路径用途
GET/api/channels/返回 service_running 与各 channel enabled/running。
POST/api/channels/{name}/restart管理员重启指定平台 worker（需 admin）。

用户绑定 — channel_connections.py

方法路径用途
GET/api/channels/providers列出可连接 provider、凭证字段、连接状态。
GET/api/channels/connections当前用户的绑定记录。
POST/api/channels/{provider}/connect生成 600s 有效的一次性绑定码与指令。
POST/api/channels/{provider}/runtime-config管理员写入 bot 凭证并热启动 worker。
DELETE/api/channels/{provider}/runtime-config断开 provider 运行时配置。
DELETE/api/channels/connections/{connection_id}用户撤销自己的绑定。

绑定安全模型靠 connect code 机密性（secrets.token_urlsafe(16)、一次性消费），不是 allowed_users 在绑定时拦截。详见 backend/docs/IM_CHANNEL_CONNECTIONS.md。

GitHub — github_webhooks.py

方法路径用途
POST/api/webhooks/github接收 GitHub 投递；豁免 auth/CSRF；HMAC 验签；fan-out 到总线。
响应码场景
401签名错误或缺失。
503fan-out 瞬态失败（GitHub 会重试）。
200 + skippedchannels.github.enabled=false、未知事件、channel service 未启动等永久态。

路由仅在 GITHUB_WEBHOOK_SECRET 已设置，或开发环境 DEER_FLOW_ALLOW_UNVERIFIED_GITHUB_WEBHOOKS=1 时挂载。

GitHub event-driven agents

  • 绑定写在 custom agent 的 config.yaml → github:（repo、installation_id、triggers）。
  • fanout_event 为每个匹配 binding 发布一条 InboundMessage，metadata 带 agent_name / preferred_thread_id。
  • 全局 channels.github.enabled 是运维 kill-switch，与 webhook 路由是否挂载（secret）独立。
  • 安装 token 由 ChannelRunPolicy.credentials_provider 注入 run_context["github_token"]，供 sandbox gh 使用。

Channel 命令（manager 处理）

/new/status/models/memory/goal/bootstrap/help/<skill>

/connect <code> 与 Telegram /start <code> 由各 platform adapter 在入站前消费，不进入 manager 命令分支。
