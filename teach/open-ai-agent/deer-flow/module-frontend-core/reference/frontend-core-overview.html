Frontend core 模块速查

这份 reference 是 DeerFlow 前端数据层的地图：两条出站通道、认证外壳、线程主链路，以及 skills/MCP/uploads 等同构域。 lesson 只保留一条学习主线，完整清单集中放在这里。

模块：frontend core 层级：L1 源码：frontend/src/core

本页是 L1 Frontend core 的速查表。L3 函数签名见 frontend-core-api.html；短课入口： 0001-frontend-core-module-tour.html； 页面层对照见 Frontend workspace 速查； 整体锚点见 L0 项目总览。

一眼架构

在 L0 架构图中，本模块位于 Next.js frontend/core 层：workspace 组件消费 core hooks，core 再经两条通道连 Gateway。

workspace 组件 / 页面 -> core hooks (useThreadStream, useSkills, useMCPConfig, useUploadFiles, …) -> 通道 A: getAPIClient() -> LangGraph SDK -> /api/langgraph -> 通道 B: fetcher.ts -> Gateway REST (/api/skills, /api/mcp/config, /api/threads/…) -> Gateway -> agent runtime / persistence

通道 A：LangGraph SDK

文件 关键导出 职责
frontend/src/core/api/api-client.ts getAPIClient() 创建并缓存 LangGraph Client；onRequest 注入 CSRF； 包装 runs.stream、runs.cancel、runs.joinStream 处理终端 run 与 409 冲突； 静态演示模式返回 mock client。
frontend/src/core/api/stream-mode.ts sanitizeRunStreamOptions 规范化 stream/joinStream 选项，与 Gateway 能力对齐。
frontend/src/core/api/fetcher.ts fetch、readCsrfCookie 虽在 api/ 目录，但属于通道 B REST wrapper； SDK 的 CSRF 注入与这里共享 readCsrfCookie 契约。

通道 B：Gateway REST（fetcher.ts）

凡是不走 LangGraph threads/runs 协议的接口，都通过 frontend/src/core/api/fetcher.ts 的 fetch：credentials: "include"、状态变更方法自动带 X-CSRF-Token、401 跳转登录。

认证层

文件 运行环境 职责
frontend/src/core/auth/server.ts Server Component / layout getServerSideUser()：读 access_token cookie，向 Gateway 求证； 区分 authenticated / unauthenticated / needs_setup / system_setup_required / gateway_unavailable / config_error。
frontend/src/core/auth/AuthProvider.tsx Client 持有展示用 user（RFC-001：不存 JWT）； 提供 logout、refreshUser、applyUser。
frontend/src/core/auth/gateway-config.ts Server 解析 SSR 用的 internal Gateway URL。
frontend/src/core/auth/setup.ts Client 首次 setup 流程辅助。

threads 主链路

文件 关键 API 出站
frontend/src/core/threads/hooks.ts useThreadStream、useThreads、 useInfiniteThreads、useThreadHistory、 useBranchThread、useDeleteThread 通道 A（SDK stream/search）+ 部分通道 B（regenerate prepare、token usage）
frontend/src/core/threads/api.ts fetchThreadTokenUsage、branchThreadFromTurn 通道 B：GET /api/threads/{id}/token-usage、POST /api/threads/{id}/branches
frontend/src/core/threads/types.ts AgentThreadState、AgentThread 线程状态类型定义，供 SDK 泛型与 UI 共用。

其他 core 域速查

域 目录 与 workspace 的关系
agents frontend/src/core/agents/ Agent gallery 与创建向导的数据源；页面在 workspace/agents。
scheduled-tasks frontend/src/core/scheduled-tasks/ 定时任务 CRUD；页面在 workspace/scheduled-tasks。勿与 tasks/ 混淆。
tasks (subtask) frontend/src/core/tasks/ 聊天内 subagent 步骤时间线；由 SubtasksProvider 消费。
workspace-changes frontend/src/core/workspace-changes/ run 级文件变更摘要；badge 组件在 workspace/changes。
artifacts frontend/src/core/artifacts/ artifact 加载/预览；面板 UI 在 workspace/artifacts。
sidecar frontend/src/core/sidecar/ 旁路线程 API 与引用状态；面板 UI 在 workspace/sidecar。
memory frontend/src/core/memory/ 用户记忆设置；设置页 memory 子页消费。
channels frontend/src/core/channels/ IM 连接轮询与 OAuth URL；设置页 channels 消费。
