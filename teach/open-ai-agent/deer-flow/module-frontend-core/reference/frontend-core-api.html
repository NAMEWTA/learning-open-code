Frontend core 微观 API 参考

L3 速查：按源码文件列出 frontend/src/core/ 各域 api.ts / hooks.ts 的公共函数签名、参数约束、返回值与边界行为。 返回 L1 模块速查 · 短课 · L0 总览。

层级：L3 源码：frontend/src/core 通道 A：LangGraph SDK 通道 B：fetcher REST
通道 A — getAPIClient 通道 B — fetcher.fetch React Query hooks
基础设施 threads skills mcp uploads agents memory channels scheduled-tasks tasks workspace-changes models / suggestions / sidecar 本地状态 hooks 调用示例 测试索引

出站契约：凡需认证的 Gateway REST 调用应走 frontend/src/core/api/fetcher.ts 的 fetch（credentials: "include"、 POST/PUT/PATCH/DELETE 自动带 X-CSRF-Token、401 跳转登录）。 LangGraph threads/runs 走 getAPIClient()，其 onRequest 与 fetcher 共享 readCsrfCookie。

基础设施

api/api-client.ts — 通道 A

符号签名返回值 / 行为边界与异常
getAPIClient (isMock?: boolean) => LangGraphClient 按 isMock 缓存单例（"mock" / "default"）；创建时注入 CSRF、包装 stream/cancel/joinStream 静态站模式（isStaticWebsiteOnly()）返回 mock client，threads/runs 走本地 demo 数据
isInactiveRunStreamError (error: unknown) => boolean 识别 Gateway 409「not active on this worker / cannot be streamed」 用于 joinStream 短路，清除 sessionStorage 重连 key
isRunNotCancellableError (error: unknown) => boolean 识别终端 run 的 cancel 409（is not cancellable） cancel 包装器吞掉此错误视为 no-op
clearReconnectRun (threadId: string | null | undefined, runId: string) => void 清除 sessionStorage 键 lg:stream:{threadId} 仅当存储值与 runId 匹配时删除；SSR / 无 threadId 时 no-op

包装行为：runs.stream / joinStream 经 sanitizeRunStreamOptions 规范化；终端状态（success/error/timeout/interrupted）的 joinStream 在预检后短路，避免 isLoading 永久为 true。

测试：frontend/tests/unit/core/api/api-client.test.ts

api/fetcher.ts — 通道 B

符号签名返回值 / 行为边界与异常
fetch (input, init?) => Promise<Response> 带 cookie + CSRF 的 globalThis.fetch 包装 401 → 跳转 /login 并抛 Error("Unauthorized")
readCsrfCookie () => string | null 从 document.cookie 解析 csrf_token SSR（无 document）返回 null

threads/hooks.ts — 核心 hook

function useThreadStream(options: ThreadStreamOptions): {
  thread: StreamResult & { stop: () => Promise<void>; messages: Message[] };
  pendingUsageMessages: Message[];
  sendMessage: (threadId, message: PromptInputMessage, extraContext?, options?) => Promise<void>;
  regenerateMessage: (threadId, messageId, supersededMessageIds?) => Promise<void>;
  isUploading: boolean;
  isHistoryLoading: boolean;
  hasMoreHistory: boolean;
  loadMoreHistory: () => void;
}
参数 / 回调类型含义
threadIdstring | null | undefined当前线程；空表示新对话
displayThreadIdstring | null | undefined侧栏展示用 ID，默认同 threadId
contextLocalSettings["context"]模式、agent、model 等提交上下文
isMockboolean?静态演示模式，跳过历史加载
onSend / onStart / onFinish / onToolEnd回调发送、run 创建、完成、工具结束事件

内部机制：基于 @langchain/langgraph-sdk/react 的 useStream + getAPIClient(isMock)；合并历史（useThreadHistory）、 乐观消息、摘要迁移缓冲；sendMessage 先上传附件再 thread.submit； sendInFlightRef 防并发发送。

调用示例

// 通道 A：流式对话
const { thread, sendMessage } = useThreadStream({
  threadId,
  context: settings.context,
  onFinish: (state) => console.log(state.title),
});
await sendMessage(threadId, { text: "你好", files: [] });

// 通道 B：REST 配置域
const { skills } = useSkills();
await enableSkill("web-search", true);

// 混合：regenerate 走 REST prepare + SDK submit
await regenerateMessage(threadId, messageId);
