Scheduled task 创建→执行 全链路参考

L2 垂直切片速查。短课入口：0001-flow-map.html。模块背景见 Gateway 与 Runtime。

端到端流程图

/workspace/scheduled-tasks (page.tsx) │ useCreateScheduledTask → api.ts POST /api/scheduled-tasks ▼ scheduled_tasks.py :: create_scheduled_task │ validate context_mode / thread access │ schedules.next_run_at(cron|once, timezone) ▼ ScheduledTaskRepository.create → scheduled_tasks 表 │ status=enabled, next_run_at=... ▼ [时间流逝] ScheduledTaskService._run_loop (poll_interval) │ run_once → claim_due_tasks (lease + FOR UPDATE SKIP LOCKED) ▼ dispatch_task(trigger=scheduled|manual) │ 1) 解析 execution_thread_id (fresh UUID 或 reuse thread_id) │ 2) ScheduledTaskRunRepository.create (status=queued) │ 3) launch_scheduled_thread_run → start_run (同聊天 run 路径) │ context: { non_interactive: true, user_id } │ metadata: scheduled_task_id, scheduled_task_run_id ▼ RunManager / worker.run_agent → lead agent 执行 prompt ▼ handle_run_completion → 更新 task_run + 父任务状态 (once 终态)

分层与源码锚点

frontend/ ├── src/app/workspace/scheduled-tasks/page.tsx ← UI：创建表单、列表、run 历史 ├── src/core/scheduled-tasks/api.ts ← REST 客户端 └── src/core/scheduled-tasks/hooks.ts ← TanStack Query 封装 backend/app/ ├── gateway/routers/scheduled_tasks.py ← CRUD、pause/resume/trigger ├── gateway/services.py :: launch_scheduled_thread_run ├── gateway/app.py ← lifespan 内启动 ScheduledTaskService └── scheduler/service.py ← 轮询、认领、dispatch、completion 回写 backend/packages/harness/deerflow/ ├── scheduler/schedules.py ← timezone、cron 规范化、next_run_at └── persistence/ ├── scheduled_tasks/ ← 任务 ORM + Repository + claim └── scheduled_task_runs/ ← 每次执行记录

执行阶段：认领 → dispatch → run

机制行为源码
轮询_run_loop 每 poll_interval_seconds 调 run_onceapp/scheduler/service.py
并发预算全局活跃 run 数不超过 max_concurrent_runscount_active_runs + claim limit
Lease 认领enabled 且到期；或 running 且 lease 已过期可重认领claim_due_tasks
线程选择fresh_thread_per_run → 新 UUID；reuse_thread → 固定 thread_iddispatch_task
Overlap skip默认 overlap_policy=skip：已有活跃 run 则记 skipped task_rundispatch_task
Launch内部用户 + non_interactive，复用 start_runlaunch_scheduled_thread_run
cron 下次launch 成功即写新 next_run_at，父任务回到 enabledupdate_after_launch
once 中间态launch 后父任务保持 running 直至 completiondispatch_task + handle_run_completion

状态与排障线索

现象优先检查
任务从不执行scheduler.enabled；next_run_at 是否未来；状态是否为 enabled / 非 paused
执行记录 skipped上一轮 run 仍 active；overlap_policy
once 卡在 runningGateway 重启后 cancel_stuck_once_tasks；run 是否无终态
手动触发 409任务 status=running 或已有活跃 task_run
agent 不提问预期行为：non_interactive 排除 ask_clarification
