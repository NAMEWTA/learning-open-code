自定义 agent 管理链路参考

L2 垂直切片速查：从 workspace UI 到 Gateway REST、harness 工具与磁盘布局。短课入口： 0001-flow-map.html。相关 L1： frontend workspace、 Gateway、 lead agent。

分层地图

frontend/src/app/workspace/agents/ ← 路由壳、layout 功能开关、new 向导 frontend/src/components/workspace/agents/ ← AgentGallery、AgentCard、禁用页 frontend/src/core/agents/ ← api.ts、hooks.ts、types.ts、feature-cache.ts backend/app/gateway/routers/agents.py ← HTTP CRUD + user-profile backend/packages/harness/deerflow/ ├── config/agents_config.py ← 扫描、加载、resolve 目录 ├── config/agents_api_config.py ← agents_api.enabled ├── tools/builtins/setup_agent_tool.py ← bootstrap 创建（仅 is_bootstrap） └── tools/builtins/update_agent_tool.py ← 聊天内自更新（agent_name 已设）

磁盘落点（per-user，默认在 backend/.deer-flow/users/{user_id}/agents/{name}/）：config.yaml + SOUL.md（+ 可选 memory 子目录）。legacy 共享路径 {base}/agents/{name}/ 只读兼容，写操作需迁移。

创建路径对比

A. UI 向导（当前默认）

sequenceDiagram participant U as 用户 participant UI as new/page.tsx participant API as Gateway agents participant LG as LangGraph run participant T as setup_agent participant FS as 用户 agent 目录 U->>UI: 输入名称并确认 UI->>API: GET /api/agents/check?name= API-->>UI: available + normalized name UI->>LG: sendMessage(is_bootstrap, agent_name) LG->>T: 模型调用 setup_agent(soul, description, …) T->>FS: 写入 config.yaml + SOUL.md UI->>API: GET /api/agents/{name}（重试） API-->>UI: AgentResponse
步骤关键点
名称校验前端 ^[A-Za-z0-9-]+$；后端规范化小写；冲突含 legacy 目录
bootstrap 上下文is_bootstrap: true 使 lead agent 绑定 setup_agent，且 agent_name 注入 runtime context
保存触发菜单「保存」发送隐藏用户消息；onToolEnd(setup_agent) 后拉取 agent
空 soul 拒绝setup_agent 对空 soul 返回 ToolMessage 错误，避免写入不可用 agent（#3549）

B. REST 直创（并行入口）

接口说明
POST /api/agents请求体含 name、description、model、tool_groups、skills、soul；asyncio.to_thread 写盘；409 已存在；失败时清理半成品目录
前端封装createAgent() in api.ts；useCreateAgent mutation（当前向导 UI 未调用）

读取与列表

接口前端后端
GET /api/agentslistAgents / useAgentslist_custom_agents，含 soul
GET /api/agents/{name}getAgent / useAgentload_agent_config + load_agent_soul
GET /api/agents/checkcheckAgentName校验模式 + per-user/legacy 目录存在性
GET /api/user-profile（全局 USER.md，注入所有 custom agents）读 paths.user_md_file

更新路径

路径触发方落盘
PUT /api/agents/{name}REST 客户端 / updateAgent用 model_fields_set 区分省略与显式 null；preserve_non_managed_fields 保留 github: 等
update_agent 工具agent 专属聊天中模型自更新与 Gateway 更新共享校验；原子写 YAML/SOUL
PUT /api/user-profile全局用户画像写 USER.md

skills 语义：null = 继承全部启用 skill；[] = 无 skill；非空列表 = 白名单。

删除

DELETE /api/agents/{name} → shutil.rmtree per-user 目录（config、SOUL、memory）。AgentCard 经 useDeleteAgent 调用；成功后 invalidate ["agents"] 查询。

常见错误与传播

启用自定义 agent 管理

在仓库根 config.yaml 设置：

agents_api:
  enabled: true

修改后 Gateway 热读取配置；前端再次进入 /workspace/agents 会通过 /api/features 自动启用（staleTime: 0）。
