15 分钟摸清自定义 agent 管理链路

本课只教一个目标：当用户在 workspace 里创建或更新自定义 agent 时，你能从 UI 一路追到磁盘上的 config.yaml 与 SOUL.md，并区分「向导 bootstrap」与「REST / 工具更新」两条路径。

L2 垂直切片 入口：/workspace/agents 参考：完整链路图

1. 功能开关与页面入口

所有 agents 页面先经过 frontend/src/app/workspace/agents/layout.tsx：它调用 useAgentsApiEnabled() 查询 GET /api/features，只有 agents_api.enabled=true 才渲染子页面；否则显示禁用页。列表页 frontend/src/app/workspace/agents/page.tsx 只是挂载 AgentGallery，真正的数据来自 frontend/src/core/agents/hooks.ts 里的 listAgents() → GET /api/agents。 

用户打开 /workspace/agents
  → layout 查 /api/features（agents_api.enabled）
  → AgentGallery → useAgents → GET /api/agents
  → Gateway routers/agents.py → list_custom_agents(user_id)

2. 创建：向导 bootstrap 主路径

「新建 agent」进入 frontend/src/app/workspace/agents/new/page.tsx，分两步：先 checkAgentName（GET /api/agents/check），通过后进入聊天步，并以 context.is_bootstrap=true 启动线程。用户对话结束后点「保存」，前端发送隐藏消息触发 lead agent 调用 setup_agent 工具（仅 bootstrap 模式绑定）。工具在 harness 里把 config.yaml 与 SOUL.md 写入 per-user 目录；前端监听 onToolEnd(setup_agent) 再 GET /api/agents/{name} 确认落盘。 

确认名称 → check API
  → sendMessage(is_bootstrap, agent_name)
  → LangGraph run → setup_agent 工具
  → users/{user_id}/agents/{name}/config.yaml + SOUL.md
  → getAgent 重试读取 → 向导完成

同一存储布局也被 Gateway 的 POST /api/agents 使用（REST 直创），但当前 UI 向导走的是工具路径而非 POST。 详细时序与错误码见 参考文档 · 创建。

3. 更新、列表与删除

已存在 agent 的元数据变更有两条路：（1）Gateway PUT /api/agents/{name}，由 frontend/src/core/agents/api.ts 的 updateAgent 封装，写回 YAML 并可选覆盖 SOUL；（2）在 agent 专属聊天里，lead agent 绑定 update_agent 工具做部分字段自更新。列表卡片 frontend/src/components/workspace/agents/agent-card.tsx 用 deleteAgent 调 DELETE /api/agents/{name}。若 agent 仍停在 legacy 共享目录，更新/删除会返回 409 并提示运行 migrate_user_isolation.py——这是本切片最典型的边界错误。

练习：判断请求落点

用户在「新建 agent」向导里点保存后，Network 面板看到 LangGraph stream 里有 setup_agent 工具调用，但没有 POST /api/agents。这是 bug 还是设计如此？

设计如此。向导创建走 bootstrap 聊天 + setup_agent 工具落盘；POST /api/agents 是 Gateway 提供的并行 REST 入口，供 API 客户端或未来 UI 直调。持久化结果相同，都写入 per-user agent 目录并由 agents_config.py 读取。若保存后列表仍空，应查工具是否返回空 soul、agents_api 是否开启，或 getAgent 重试是否超时。

主要来源：本地 open-ai-agent/deer-flow/backend/app/gateway/routers/agents.py 与 frontend/src/app/workspace/agents/new/page.tsx。想继续深挖可让 agent 讲 update_agent 与 REST PUT 的字段合并语义，或对照 Gateway 模块导览。
