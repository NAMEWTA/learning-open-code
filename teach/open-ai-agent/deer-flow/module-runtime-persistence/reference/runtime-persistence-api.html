2. RunRepository（RunStore 实现）

backend/packages/harness/deerflow/persistence/run/sql.py

每个方法各自获取短生命周期 AsyncSession，不在长任务执行期间持有连接。所有带 user_id 的读操作在 SQL 层做行级过滤；user_id=AUTO 时由 resolve_user_id() 解析当前请求用户。

构造

RunRepository(session_factory: async_sessionmaker[AsyncSession])

CRUD 与状态

方法签名要点返回值约束 / 边界
put put(run_id, *, thread_id, assistant_id=None, user_id=AUTO, model_name=None, status="pending", multitask_strategy="reject", metadata=None, kwargs=None, error=None, created_at=None, follow_up_to_run_id=None) None 幂等 upsert：同 run_id 重试不会主键冲突；metadata/kwargs 经 _safe_json 序列化；model_name strip 后截断至 128 字符。

3. RunManager 与 RunRecord

backend/packages/harness/deerflow/runtime/runs/manager.py

进程内 registry + 可选 RunStore 双写。所有变更受 asyncio.Lock 保护；SQLite 写失败有有界指数退避重试（PersistenceRetryPolicy，默认 5 次）。

构造

RunManager(store: RunStore | None = None, *, persistence_retry_policy: PersistenceRetryPolicy | None = None)

核心生命周期 API

方法签名要点返回异常 / 边界
create create(thread_id, assistant_id=None, *, on_disconnect=DisconnectMode.cancel, metadata=None, kwargs=None, multitask_strategy="reject", user_id=None) RunRecord 生成 UUID run_id；store 写入失败会回滚内存记录并 re-raise。
create_or_reject 同 create + model_name RunRecord reject 且线程有 inflight → ConflictError；interrupt/rollback 先取消旧 run 再插入；未知 strategy → UnsupportedStrategyError。
get / aget get(run_id, *, user_id=None) RunRecord | None 内存优先；store hydration 失败记 warning 并返回 None；hydrated 记录带 store_only=True（无本地 task）。
list_by_thread list_by_thread(thread_id, *, user_id=None, limit=100) list[RunRecord] 内存与 store 按 run_id 合并，内存覆盖同 id；按 created_at 降序截断。
set_status set_status(run_id, status: RunStatus, *, error=None) None 未知 run 记 warning；同步 RunStore.update_status。
cancel cancel(run_id, *, action="interrupt") bool action 为 "interrupt" 或 "rollback"；已 interrupted 幂等返回 True；终端态返回 False；store-only run 在本 worker 上可能无法取消。
update_run_completion update_run_completion(run_id, **kwargs) None 更新内存 + 调用 store；token 字段与 repository 对齐。
update_run_progress update_run_progress(run_id, **kwargs) None 仅当记录在内存且非 store_only 时更新内存；始终 best-effort 写 store。
reconcile_orphaned_inflight_runs reconcile_orphaned_inflight_runs(bridge, *, before=None) list[str] 将无本地 task 的 persisted inflight 标为 error；对 retained stream 发 publish_end；返回受影响 run_id 列表。
shutdown shutdown(*, timeout=5.0) None 等待活跃 task 完成或超时后取消。

枚举与异常

类型值 / 含义
RunStatuspending → running → success | error | timeout | interrupted
DisconnectModecancel（SSE 断开取消 run）| continue（后台继续）
ConflictErrormultitask_strategy=reject 时线程已有 active run
UnsupportedStrategyErrorstrategy 不在 reject|interrupt|rollback

4. run_agent 与 RunContext

backend/packages/harness/deerflow/runtime/runs/worker.py

RunContext

@dataclass(frozen=True)
class RunContext:
    checkpointer: Any
    store: Any | None = None          # LangGraph Store
    event_store: Any | None = None    # RunEventStore
    run_events_config: Any | None = None
    thread_store: Any | None = None   # ThreadMetaRepository
    app_config: AppConfig | None = None
    on_run_completed: Any | None = None # 完成回调 hook

run_agent

async def run_agent(
    bridge: StreamBridge,
    run_manager: RunManager,
    record: RunRecord,
    *,
    ctx: RunContext,
    agent_factory: Any,
    graph_input: dict,
    config: dict,
    stream_modes: list[str] | None = None,
    stream_subgraphs: bool = False,
    interrupt_before: list[str] | Literal["*"] | None = None,
    interrupt_after: list[str] | Literal["*"] | None = None,
) -> None
参数约束
bridgeworker 通过 publish(run_id, event, data) 推送 SSE；结束时必须 publish_end。
record必须已注册于 RunManager；record.task 由 Gateway 在 create_task 后赋值。
ctx.checkpointerLangGraph graph 编译与 checkpoint 读写；rollback 依赖 pre-run snapshot。
stream_modes默认 ["values"]；合法集 values|updates|checkpoints|tasks|debug|messages|custom；events 在 Gateway 路径被跳过（需 astream_events）。
config必须含 configurable.thread_id；worker 注入 runtime.context（thread_id、run_id、caller context、app_config）。

执行阶段（简化）：

  1. wait_for_prior_finalizing — interrupt/rollback 新 run 等待旧 run 收尾。
  2. set_status(running) + bridge.publish(metadata) + 捕获 pre-run checkpoint/workspace。
  3. agent.astream(...) — 按 mode 发布 values/updates/messages/custom；journal 写 event store。
  4. goal continuation 循环（若线程有 active goal）。
  5. finally — flush journal、update_run_completion、sync thread_meta、publish_end、延迟 bridge.cleanup。

异常路径：try 块内任何失败都会走 except/finally，确保 SSE 收到 end；rollback 分支在 abort_action=="rollback" 时恢复 pre-run checkpoint。测试：backend/tests/test_run_worker_rollback.py、backend/tests/test_worker_langfuse_metadata.py、backend/tests/test_goal_worker.py。

5. StreamBridge 协议

backend/packages/harness/deerflow/runtime/stream_bridge/base.py

StreamEvent 与哨兵

@dataclass(frozen=True)
class StreamEvent:
    id: str      # SSE Last-Event-ID；单调递增
    event: str   # "metadata" | "values" | "updates" | "messages" | "custom" | "error" | "end" | ...
    data: Any    # JSON 可序列化载荷

HEARTBEAT_SENTINEL  # event="__heartbeat__"
END_SENTINEL        # event="__end__"

抽象方法

方法签名生产者 / 消费者约束
publish async publish(run_id: str, event: str, data: Any) -> None worker 侧；必须线程/协程安全。
publish_end async publish_end(run_id: str) -> None 标记 stream 结束；订阅方随后收到 END_SENTINEL。
subscribe subscribe(run_id, *, last_event_id=None, heartbeat_interval=15.0) -> AsyncIterator[StreamEvent] SSE 侧；超时发 heartbeat；last_event_id 支持断线重连 replay。
cleanup async cleanup(run_id, *, delay=0) -> None delay>0 时延迟释放，给晚到订阅者 drain 窗口。
close async close() -> None 进程 shutdown 时释放全部 backend 资源。

类属性 supports_cross_process：MemoryStreamBridge=False，RedisStreamBridge=True。

MemoryStreamBridge

backend/packages/harness/deerflow/runtime/stream_bridge/memory.py

构造参数默认行为
queue_maxsize256每 run 环形缓冲；溢出时前移 start_offset，旧事件不可 replay。

事件 id 格式 {timestamp_ms}-{seq}；last_event_id 用 seq 做 O(1) 定位，找不到则从最旧 retained 事件重放。

backend/packages/harness/deerflow/runtime/stream_bridge/async_provider.py

6. Checkpointer Provider

LangGraph 线程状态（messages、goal、title、checkpoint）与 DeerFlow ORM 表分离；checkpointer 表由 LangGraph 自己 setup()，不进 Base.metadata。

异步（Gateway lifespan）

backend/packages/harness/deerflow/runtime/checkpointer/async_provider.py

@contextlib.asynccontextmanager
async def make_checkpointer(app_config: AppConfig | None = None) -> AsyncIterator[Checkpointer]
优先级配置源后端
1legacy checkpointer: 段（若存在）memory | sqlite | postgres
2unified database.backend（非 memory）同上，Postgres 用 database.postgres_url
3默认InMemorySaver
后端依赖包连接约束
sqlitelanggraph-checkpoint-sqlite默认 store.db；父目录经 asyncio.to_thread 创建
postgreslanggraph-checkpoint-postgres + psycopg_pool必须提供 connection string；池带 TCP keepalive

缺失可选依赖 → ImportError（消息含安装提示）；Postgres 无 connection string → ValueError。

同步（CLI / 单测）

backend/packages/harness/deerflow/runtime/checkpointer/provider.py

函数签名语义
get_checkpointer()() -> Checkpointer进程级单例；首次调用创建并持有连接上下文。
reset_checkpointer()() -> None关闭连接并清缓存；测试或热重载边界用。
checkpointer_context()() -> Iterator[Checkpointer]每次 with 独立连接，不缓存。

测试：backend/tests/test_checkpointer_none_fix.py。

调用示例（Gateway 模式）

from deerflow.runtime.checkpointer.async_provider import make_checkpointer
from deerflow.runtime.stream_bridge import make_stream_bridge

async with make_stream_bridge(app_config) as bridge, \
           make_checkpointer(app_config) as checkpointer:
    app.state.stream_bridge = bridge
    app.state.checkpointer = checkpointer
    run_mgr = RunManager(store=RunRepository(session_factory))

7. 跨组件约束速查

场景约束违反后果
同线程并发 run默认 multitask_strategy=rejectConflictError → HTTP 409
取消 store-only run本 worker 无 record.taskcancel() 返回 False → API 409
SSE 断开后台on_disconnect=cancel客户端断开触发 run 取消；continue 则后台跑完
progress 写库update_run_progress 仅命中 running 行已完成 run 的 progress 写被 SQL WHERE 忽略
checkpointer 热切换checkpointer / database 在 STARTUP_ONLY_FIELDS改 config 需重启 Gateway
Redis bridge 多 podorphan recovery 不充当跨 pod reaper需 worker ownership/liveness 才能安全共享 DB

接口级问题可继续追问 Agent，例如：“create_or_reject 的 interrupt 与 rollback 在 checkpointer 层有何差异？”或“MemoryStreamBridge 缓冲溢出后 Last-Event-ID 会怎样？”
