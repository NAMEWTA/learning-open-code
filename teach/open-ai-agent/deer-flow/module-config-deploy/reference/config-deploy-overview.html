配置与部署参考

本页承载 L1 配置与部署模块的速查材料：配置文件分工、Makefile 命令、部署路径、Docker 服务与环境变量。短课入口：0001-config-deploy-module-tour.html；微观 API：config-deploy-api.html；整体锚点见 L0 项目总览。

模块边界

维度结论源码依据
职责提供从配置生成、依赖检查、本地/Docker 启动到生产 compose 的整栈编排；不包含 Gateway 业务逻辑与 agent 推理。Makefile、scripts/、docker/
配置入口config.yaml 管主应用；extensions_config.json 管 MCP/skills；.env 供密钥与 compose 变量。config.example.yaml、extensions_config.example.json
统一入口浏览器与 API 经 nginx :2026；Gateway 原生 :8001、Frontend :3000 为内部端口。AGENTS.md、docker/nginx/
非职责不定义 HTTP router、不实现 checkpointer、不构建前端页面组件。backend/app/gateway/、frontend/src/

配置文件地图

文件生成方式主要内容
config.yamlmake config 或手动复制 exampleconfig_version、models、sandbox、database、run_events、stream_bridge、channels、scheduler 等
extensions_config.json手动复制 example 或 Gateway API 写入mcpServers、mcpInterceptors、skills 启用状态
.envmake config 从 .env.exampleBETTER_AUTH_SECRET、DEER_FLOW_INTERNAL_AUTH_TOKEN、代理与镜像变量等
frontend/.envmake config前端鉴权与 Gateway 内部 URL（Docker 场景）

路径与环境变量优先级（简表）

变量作用
DEER_FLOW_PROJECT_ROOT显式指定项目根，影响相对路径解析
DEER_FLOW_CONFIG_PATH覆盖 config.yaml 位置
DEER_FLOW_EXTENSIONS_CONFIG_PATH覆盖 extensions_config.json 位置
DEER_FLOW_HOME运行时数据目录，默认 .deer-flow
DEER_FLOW_SKILLS_PATHskills 目录，默认 skills/
PORTDocker 对外端口，默认 2026

config.yaml 中以 $VAR 开头的值会解析为环境变量。字段级热加载与 restart-required 边界见 backend/docs/CONFIGURATION.md。

Makefile 命令清单

命令脚本 / 行为典型场景
make setupscripts/setup_wizard.py首次安装，交互生成配置
make doctorscripts/doctor.py配置与系统健康检查
make configscripts/configure.py从 example 生成本地配置（已存在则中止）
make config-upgradescripts/config-upgrade.sh合并 example 新增字段到现有 config
make checkscripts/check.py检查 python、uv、node、pnpm、docker 等
make installuv sync + pnpm install + pre-commit安装前后端依赖
make devscripts/serve.sh --dev本地开发，热重载
make startscripts/serve.sh --prod本地生产模式
make dev-daemon / start-daemonserve.sh --daemon后台运行整栈
make stopserve.sh --stop停止本地进程
make docker-startscripts/docker.sh startDocker 开发环境
make docker-stopscripts/docker.sh stop停止 Docker 开发栈
make upscripts/deploy.sh构建并启动 Docker 生产栈
make downscripts/deploy.sh down停止并移除生产容器
make setup-sandboxscripts/setup-sandbox.sh预拉 sandbox 镜像（可选）
make support-bundlescripts/support_bundle.py生成脱敏排障包

部署路径对照

模式入口命令编排文件热重载备注
本地 devmake devscripts/serve.sh是默认日常开发
本地 prodmake startscripts/serve.sh --prod否本地优化构建
本地 daemonmake dev-daemonserve.sh --daemondev 有后台起服务
Docker devmake docker-startdocker-compose-dev.yaml是容器内开发
Docker prodmake updocker-compose.yaml否生产镜像 + Redis
Browser → nginx:2026 ├─ /api/langgraph/* → gateway:8001/api/*（LangGraph 兼容路径） ├─ /api/* → gateway:8001（REST） └─ / → frontend:3000

Docker Compose 服务（生产）

文件：docker/docker-compose.yaml。deploy.sh 会按 config.yaml 的 sandbox 模式附加 overlay（如 DooD、cli-auth）。

服务镜像/构建端口角色
nginxnginx:alpine${PORT:-2026}统一反向代理
frontendfrontend/Dockerfile target prod内部 3000Next.js 生产服务
gatewaybackend/Dockerfile内部 8001FastAPI + embedded runtime
redisredis:7-alpine内部 6379SSE stream bridge 后端
provisionerdocker/provisioner/内部 8002K8s sandbox 模式可选

gateway 关键卷与环境（生产）

挂载 / 变量说明
config.yaml → /app/backend/config.yaml只读主配置
extensions_config.json只读扩展配置
skills/只读 skills 树
DEER_FLOW_HOME → .deer-flow持久化线程与运行时数据
DEER_FLOW_STREAM_BRIDGE_REDIS_URL默认 redis://redis:6379/0
GATEWAY_WORKERS默认 1；多 worker 需接受 run 状态局限

最小验证命令

make check
make doctor
curl -s http://localhost:2026/health
# 或直连 Gateway：
curl -s http://localhost:8001/health

本地 make dev 成功后应能在浏览器打开 http://localhost:2026。Docker make up 默认同端口；若修改 PORT 环境变量，入口随之变化。
