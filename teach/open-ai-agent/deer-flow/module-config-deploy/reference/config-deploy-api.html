配置加载：deerflow/config/app_config.py

路径解析

符号签名行为异常 / 边界
AppConfig.resolve_config_path (config_path: str | None = None) -> Path 优先级：显式参数 → DEER_FLOW_CONFIG_PATH → project_root()/config.yaml → 遗留 backend/config.yaml、repo_root/config.yaml 路径存在但文件缺失 → FileNotFoundError
AppConfig.from_file (config_path: str | None = None) -> Self YAML 解析 → 版本检查 → resolve_env_variables → _apply_database_defaults → 合并 ExtensionsConfig.from_file() → Pydantic 校验 → 同步子系统单例 缺失 $VAR 环境变量 → ValueError；models 为空仅 warning；注释掉的 YAML 段解析为 null 时由 _drop_null_config_sections 回退默认值
AppConfig.resolve_env_variables (config: Any) -> Any 递归替换以 $ 开头的字符串为 os.getenv 结果 变量未设置 → ValueError（主配置 fail-closed）
AppConfig._check_config_version (config_data, config_path) -> None 对比用户 config_version 与向上搜索的 config.example.yaml 用户版本偏低 → logger.warning，建议 make config-upgrade；缺失版本视为 0
get_app_config () -> AppConfig 单例缓存；比较路径 + 内容签名（mtime、size、sha256）自动重载；优先返回 ContextVar 运行时覆盖 set_app_config 注入的自定义实例跳过重载
reload_app_config (config_path: str | None = None) -> AppConfig 强制从磁盘重载并刷新缓存元数据 —
reset_app_config () -> None 清空缓存，下次 get_app_config() 重新加载 测试与切换配置场景
set_app_config (config: AppConfig) -> None 注入 mock / 自定义实例，标记 _app_config_is_custom 跳过后续文件变更检测
push_current_app_config / pop_current_app_config — 请求级 ContextVar 覆盖栈，供单测或特殊执行上下文 栈空时 pop 置 None
get_model_config / get_tool_config / get_tool_group_config (name: str) -> *Config | None O(1) 名称查找，由 _build_name_indexes 在验证后构建 重名保留首个（setdefault 语义）
apply_logging_level (name: str | None) -> None 仅调整 deerflow / app logger 级别 启动时调用；运行时改 log_level 不自动生效

Gateway 消费方式：依赖注入 get_config() 每次请求调用 get_app_config()，故 models、summarization、memory、tools 等可热加载字段在下一条消息生效。lifespan() 保留 startup_config 本地变量做一次性引导（engine、checkpointer、IM channel）。

扩展配置：deerflow/config/extensions_config.py

符号签名 / 行为边界
ExtensionsConfig.resolve_config_path 参数 → DEER_FLOW_EXTENSIONS_CONFIG_PATH → extensions_config.json / 遗留 mcp_config.json → backend/repo 根 未找到 → 返回 None（扩展可选）
ExtensionsConfig.from_file JSON 加载 + resolve_env_variables + 校验 文件缺失 → 空 {mcpServers:{}, skills:{}}；无效 JSON → ValueError
resolve_env_variables（扩展版） $VAR 未设置时返回 "" 而非抛错 与主配置 fail-closed 行为不同，避免 MCP 启动硬失败
McpServerConfig._accept_transport_alias 接受 MCP 规范的 transport 作为 type 别名 两者同时存在时 type 优先

AppConfig.from_file 始终把扩展配置合并进 config.extensions 字段；Gateway MCP/Skills API 另写 extensions_config.json，MCP 缓存通过 mtime 失效检测变更。

调用示例

Python：加载与热加载探测

from deerflow.config.app_config import get_app_config, reload_app_config
from deerflow.config.reload_boundary import is_startup_only_field

cfg = get_app_config()
print(cfg.sandbox.use)
assert not is_startup_only_field("memory")  # 可热加载

# 编辑 config.yaml 后
cfg2 = reload_app_config()
assert cfg2 is get_app_config()

Shell：本地 vs 生产 Docker

make check && make config    # 首次生成配置
make dev                    # serve.sh --dev → localhost:2026
make up                     # deploy.sh build+start
make down                   # deploy.sh down
