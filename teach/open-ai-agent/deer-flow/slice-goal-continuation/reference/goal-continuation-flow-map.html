goal 自动续跑流程参考

L2 垂直切片速查。短课入口：0001-flow-map.html。相关 L1 背景见 runtime-persistence-overview。

源码锚点

文件职责关键符号
agents/goal_state.py类型契约GoalState、GoalEvaluation、GoalBlocker
runtime/goal.pyevaluator 与续跑判定evaluate_goal_completion、should_continue_goal、make_goal_continuation_message、read_thread_goal、write_thread_goal
runtime/runs/worker.pyrun 内续跑编排run_agent、_prepare_goal_continuation_input、_persist_goal_evaluation、_has_durable_goal_turn_receipt

端到端流程

用户 /goal 或 PUT /api/threads/{id}/goal │ └── write_thread_goal() → checkpoint.channel_values.goal (status=active) Gateway POST /runs/stream → start_run() → asyncio.create_task(run_agent()) run_agent() ├── _stream_once(可见 graph_input) └── loop: _prepare_goal_continuation_input() 1. read_thread_goal → 无 active goal 则 return None 2. 读 checkpoint messages + pending_writes 3. _has_durable_goal_turn_receipt → 否则 blocker=run_failed, stand down 4. evaluate_goal_completion (非 thinking 模型, 仅可见对话证据) 5. compute_no_progress_count (assistant 证据签名) 6. 重读 goal + 对话签名 → 竞态则 stand down 7. satisfied → write_thread_goal(None) 清空 8. _stand_down_reason 或 !should_continue_goal → 只写 last_evaluation 9. continuation_count++ → 注入 hide_from_ui HumanMessage _stream_once({messages: [continuation_message]})

GoalBlocker 与续跑决策

blocker含义会触发隐藏续跑？
nonegoal 已满足（satisfied=true）否 — 清空 goal
goal_not_met_yet证据显示未完成，但 agent 还可自主推进是 — 唯一可续跑 blocker
needs_user_inputassistant 在等待用户否
missing_evidence可见证据不足以证明进度否
run_failed回合未 durable 落盘或执行失败否
external_wait等待外部系统否

熔断与默认上限

字段默认值作用
max_continuations8单 goal 隐藏续跑次数硬顶
continuation_count0 → 每次续跑 +1与 max_continuations 比较
max_no_progress_continuations2连续无新可见 assistant 输出则停
no_progress_count按 progress_key 累加键 = blocker + 最新可见 AI 文本 SHA256，防 evaluator 措辞抖动

并发守卫（stand_down_reason 摘录）

原因触发条件
no_durable_end_of_turncheckpoint 无 id、有 pending_writes、或末条可见消息不是 AI
thread_changed_after_evaluationevaluator 运行期间 checkpoint 或可见对话签名变化
thread_changed_before_continuation已排队续跑前用户又发消息
blocked:{blocker}blocker 非 goal_not_met_yet
max_continuations_reached续跑次数达上限
no_progress_detected无进展熔断触发

GoalState 核心字段

字段说明
objective用户设定的目标文本（最长 4000 字符）
status活跃 goal 恒为 active；满足后整条 goal 从 checkpoint 移除
last_evaluation含 satisfied、blocker、reason、progress_key、可选 stand_down_reason

隐藏续跑消息形态

make_goal_continuation_message 生成带 <goal_continuation> 包裹的 HumanMessage，additional_kwargs 含 hide_from_ui: true 与 deerflow_goal_continuation: true。SSE 仍走同一 run_id，前端不展示该条用户消息。

测试锚点：backend/tests/test_goal_worker.py（worker 续跑与竞态）、test_goal_runtime.py（goal.py 纯函数）。排障时先确认 checkpointer 已配置、可见回合已 durable 落盘，再看 last_evaluation.stand_down_reason。
