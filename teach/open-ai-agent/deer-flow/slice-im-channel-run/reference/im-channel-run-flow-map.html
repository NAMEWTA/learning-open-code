端到端分层

层职责关键文件
L0 平台入口收 webhook / 长轮询，解析平台 payloadbackend/app/channels/feishu.py 等 adapter
L1 总线入站队列、出站回调广播backend/app/channels/message_bus.py
L2 服务编排启停 adapter + manager 单例backend/app/channels/service.py
L3 调度器去重、身份校验、thread 映射、调 runbackend/app/channels/manager.py
L4 SDK 客户端内部鉴权 + CSRF，调 LangGraph 兼容 APImanager.py _get_client()
L5 Gateway HTTPthreads / runs 路由，embedded runtimebackend/app/gateway/routers/thread_runs.py 等
L6 Agentlead agent 图执行（与浏览器 run 相同）backend/packages/harness/deerflow/agents/lead_agent/agent.py
L7 运维 API查 channel 状态、管理员重启 workerbackend/app/gateway/routers/channels.py
IM 用户发消息 → platform adapter → MessageBus.publish_inbound → ChannelManager._dispatch_loop → _handle_message → _handle_chat → langgraph_sdk.threads.create | get → langgraph_sdk.runs.wait | stream | create → Gateway start_run / worker.run_agent → make_lead_agent → publish_outbound → adapter 回平台

三种 run 触发模式

模式SDK 调用典型 channelmanager 出站
阻塞等待client.runs.wait(...)Slack、Discord、钉钉（非卡片流）一次 publish_outbound（final）
流式增量client.runs.stream(..., stream_mode=[messages-tuple, values])Feishu、Telegram、WeCom多次 is_final=False + 最终 final
fire-and-forgetclient.runs.create(...)GitHub webhook不等待、通常不 publish 最终回复（agent 自行 gh 回写）

判断依据：CHANNEL_CAPABILITIES / channel 实例的 supports_streaming，以及 CHANNEL_RUN_POLICY[channel].fire_and_forget。

步骤调用说明

异常与边界路径

场景行为用户可见
未绑定身份_get_bound_identity_rejection → _reject_unbound_channel_message提示去设置页完成 connect
同 thread 并发 runSDK ConflictError / "already running"THREAD_BUSY_MESSAGE

时序图（流式 IM）

sequenceDiagram
    participant P as Platform
    participant A as Channel adapter
    participant B as MessageBus
    participant M as ChannelManager
    participant G as Gateway runs API
    participant W as run_agent

    P->>A: user message
    A->>B: publish_inbound
    B->>M: get_inbound
    M->>G: threads.create (if needed)
    M->>G: POST runs/stream
    G->>W: start_run + Task
    loop SSE chunks
        W-->>G: stream events
        G-->>M: messages-tuple / values
        M->>B: publish_outbound (partial)
    end
    M->>B: publish_outbound (final)
    B->>A: outbound callback
    A->>P: edit card / reply
