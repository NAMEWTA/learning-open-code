Tools 与 MCP 模块参考

本页是 L1 模块参考：集中记录职责、架构位置、公共入口、内部分层、依赖、配置表、工具来源和测试锚点。短课入口：0001-tools-mcp-module-tour.html；L3 公共 API 签名与边界见 tools-mcp-api.html；整体位置对照 L0 架构图中的工具扩展入口。

模块职责与位置

维度结论源码证据
一句话职责Tools/MCP 模块把配置工具、内置工具、MCP server 暴露的工具、ACP agent 工具和 agent 专属额外工具组织成可被 LangChain agent 绑定的 BaseTool 集合。backend/packages/harness/deerflow/tools/tools.py
架构位置位于配置系统、Gateway MCP 管理 API、MCP client/cache/session pool 与 lead agent 装配点之间；在 L0 架构图中对应工具扩展入口。lead agent 负责把工具集合再交给 skill policy、deferred tools 和 middleware。backend/packages/harness/deerflow/agents/lead_agent/agent.py
对外入口包级入口是 deerflow.tools:get_available_tools 与 deerflow.mcp 的 cache/client 导出；HTTP 管理入口是 Gateway 的 /api/mcp/*。backend/packages/harness/deerflow/tools/__init__.py、backend/packages/harness/deerflow/mcp/__init__.py、backend/app/gateway/routers/mcp.py

模块边界

维度结论
职责聚合配置工具、内置工具、MCP/ACP 工具为 LangChain BaseTool 集合；不负责 tool 执行 sandbox 或 Gateway 路由。
非职责不实现 lead agent 推理、不管理 thread checkpoint、不渲染前端 MCP 配置 UI（由 Gateway + frontend core 承担）。

工具来源表

来源进入点启用条件排序/优先级注意点
配置工具运行时主配置中的 tools[*].usegroup 匹配，且 host bash 未被安全策略屏蔽。最高，先进入去重集合。cfg.name 与 tool 对象 .name 不一致会记录 warning，实际绑定以 tool 对象名为准。
内置工具BUILTIN_TOOLS默认包含 present_files 与 ask_clarification。排在配置工具之后。skill_manage_tool 仅在 skill evolution 启用时加入。
subagent 工具SUBAGENT_TOOLS运行参数 subagent_enabled 为真。作为内置工具追加。当前暴露 task，状态轮询由后端内部处理。
vision 工具view_image_tool解析后的模型配置 supports_vision 为真。作为内置工具追加。middleware 还要负责把 viewed images 重新注入模型上下文。
MCP 工具get_cached_mcp_tools()extensions 运行时配置有启用的 MCP server，且工具加载成功。排在内置工具之后。进入集合后会被 tag_mcp_tool() 标记，供 deferred setup 识别。
ACP 工具build_invoke_acp_agent_tool()配置中存在 ACP agents。排在 MCP 后。构造失败只记录 warning，不阻断普通工具加载。
agent 额外工具setup_agent、update_agentbootstrap agent 或 custom agent 装配阶段添加。不属于 get_available_tools() 返回值，而是在 lead agent 内追加后再过滤。GitHub webhook channel 会屏蔽 update_agent。

MCP 配置与安全边界

主题行为源码锚点
配置文件MCP server 配置存放在 extensions 运行时配置或兼容的旧 MCP 配置中，也可由 DEER_FLOW_EXTENSIONS_CONFIG_PATH 指定；仓库中提供 extensions_config.example.json 作为字段示例。backend/packages/harness/deerflow/config/extensions_config.py
server schemaMcpServerConfig 支持 enabled、type/transport、command/args/env、url/headers、oauth、tool_call_timeout。backend/packages/harness/deerflow/config/extensions_config.py
stdio需要 command；API 管理路径只允许单个可执行名，默认允许 npx 和 uvx，可用 DEER_FLOW_MCP_STDIO_COMMAND_ALLOWLIST 扩展。backend/app/gateway/routers/mcp.py
HTTP/SSE需要 url；headers 可配置。OAuth server 会在工具发现和工具调用时注入 Authorization。backend/packages/harness/deerflow/mcp/client.py、backend/packages/harness/deerflow/mcp/oauth.py
密钥遮蔽GET 配置时 env、headers 用 *** 遮蔽，OAuth client_secret 和 refresh_token 不返回；PUT 时 masked 值只允许保留已有密钥。backend/app/gateway/routers/mcp.py
缓存刷新配置 mtime 变化会触发 lazy reload；Gateway 更新配置或显式 reset 会调用 reset_mcp_tools_cache()。backend/packages/harness/deerflow/mcp/cache.py、backend/app/gateway/routers/mcp.py
自定义 interceptormcpInterceptors 可声明构造函数路径，加载后与 OAuth interceptor 一起应用到 MCP tool calls。backend/packages/harness/deerflow/mcp/tools.py
stdio session pool只包装 stdio transport，按用户和线程隔离；HTTP/SSE 不做 pool，避免跨 task cleanup 问题。backend/packages/harness/deerflow/mcp/session_pool.py、backend/packages/harness/deerflow/mcp/tools.py
本地输出路径stdio MCP 子进程 cwd 和 temp dir 钉在 thread user-data 树内，返回文本中的本地路径会尽量改写为 /mnt/user-data/... 虚拟路径。backend/packages/harness/deerflow/mcp/tools.py

Deferred tool_search 规则

规则含义源码锚点
只延迟 MCP 来源工具是否 MCP 来源由 deerflow_mcp metadata 决定，不看工具名。backend/packages/harness/deerflow/tools/mcp_metadata.py
必须在 policy 过滤后构建 catalogskill/agent 未允许的 MCP 工具不会出现在 tool_search 搜索结果里。backend/packages/harness/deerflow/tools/builtins/tool_search.py
开启但没有 MCP survivor 时为空 setup不会追加 tool_search，也不会误报。backend/packages/harness/deerflow/tools/builtins/tool_search.py
fail closed如果开启 deferral 且 MCP 工具幸存，但 deferred set 没恢复出来，直接抛错，避免把 MCP schema 悄悄绑定给模型。backend/packages/harness/deerflow/tools/builtins/tool_search.py
promotion 带 catalog hash持久 state 中的 promoted names 只有 hash 匹配当前 catalog 才生效，避免工具变更后旧状态泄漏 schema。backend/packages/harness/deerflow/agents/middlewares/deferred_tool_filter_middleware.py
ToolNode 保留工具，模型暂时看不到 schemamiddleware 从 model binding 请求里移除未 promote 工具，并阻止模型直接调用未 promote 的 deferred tool。backend/packages/harness/deerflow/agents/middlewares/deferred_tool_filter_middleware.py

调用示例

配置工具

tool_groups:
  - name: web
  - name: file:read

tools:
  - name: web_search
    group: web
    use: deerflow.community.ddg_search.tools:web_search_tool

get_available_tools(groups=["web"]) 会保留 group 为 web 的配置工具，并通过 resolve_variable("deerflow.community.ddg_search.tools:web_search_tool") 得到真实 BaseTool 对象。

MCP 工具

{
  "mcpServers": {
    "github": {
      "enabled": true,
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {"GITHUB_TOKEN": "$GITHUB_TOKEN"}
    }
  }
}

启用后，get_mcp_tools() 会用 MultiServerMCPClient 发现工具。若 tool_search.enabled: true，lead agent 会先把这些 MCP 工具名写入 <available-deferred-tools>，再等待模型调用 tool_search 获取具体 schema。
