15 分钟理解 tools/MCP 如何进入 agent 可调用集合

学习目标：读完后，你能说明一个普通配置工具和一个 MCP 工具分别从哪里加载，经过哪些过滤与延迟处理，最后怎样进入 LangChain agent 的 tools 参数。

1. 工具不是单一来源

Tools/MCP 模块位于 lead agent 装配点之前；对照 L0 架构图，它处在“工具扩展入口”这一层。它不负责决定“模型下一步是否调用工具”，而是先回答更底层的问题：这次 agent run 到底有哪些工具可以绑定？核心入口是 backend/packages/harness/deerflow/tools/tools.py 的 get_available_tools()。

这个入口把多类来源合并：运行时主配置中的 tools[*].use 通过反射加载；present_files、ask_clarification 等内置工具直接加入；subagent 只在运行参数启用时加入 task；vision 模型才加入 view_image；MCP 工具从缓存读取；ACP agent 存在时才构造 invoke_acp_agent。最后按名字去重，优先级是配置工具、内置工具、MCP、ACP。

2. agent 装配会再次收窄集合

lead agent 不会原样使用全量工具。backend/packages/harness/deerflow/agents/lead_agent/agent.py 先按 custom agent 的 tool_groups 传给 get_available_tools()，再用 skill allowed-tools policy 过滤，最后才处理 deferred tools。

raw_tools = get_available_tools(
    model_name=model_name,
    groups=agent_config.tool_groups if agent_config else None,
    subagent_enabled=subagent_enabled,
    app_config=resolved_app_config,
)
filtered = filter_tools_by_skill_allowed_tools(raw_tools + extra_tools, skills_for_tool_policy)
final_tools, setup = assemble_deferred_tools(
    filtered,
    enabled=resolved_app_config.tool_search.enabled,
)

这段顺序很重要：先按 agent 和 skill 收窄，再构建 deferred catalog。这样 tool_search 只能搜索当前 agent 已被允许的 MCP 工具，而不会绕过 skill policy。

3. MCP 的特殊处理是“缓存、打标、延迟”

extensions runtime config / Gateway MCP API ↓ deerflow.mcp.cache → get_mcp_tools() ↓ MultiServerMCPClient loads tools ↓ get_available_tools() tag_mcp_tool(...) ↓ lead agent policy filtering ↓ assemble_deferred_tools(...) ├── tool_search disabled: MCP schema 直接绑定给模型 └── tool_search enabled: 只把名称写入 prompt，schema 等 promote 后再暴露

MCP 配置不从运行时主配置的 tools[*] 读取，而是从 extensions 运行时配置读取，Gateway 的 /api/mcp/config 负责管理它；仓库中可对照 extensions_config.example.json 理解字段形状。backend/packages/harness/deerflow/mcp/tools.py 用 MultiServerMCPClient 拉取各 server 工具；stdio server 会被 session pool 包装，保持同一用户线程内的状态，并把本地输出路径改写为 DeerFlow 可访问的虚拟路径。

backend/packages/harness/deerflow/tools/builtins/tool_search.py 则只关心“哪些 MCP 工具要延迟”。开启 tool_search.enabled 时，它追加一个 tool_search 工具，并让 middleware 在 promote 前隐藏 MCP schema。模型先看到名称列表，调用 tool_search("select:...") 后，promoted state 带着 catalog hash 写入图状态，后续模型调用才看见对应 schema。

4. 暂停回忆

检索练习：不看上文，按顺序说出一个 MCP 工具进入 agent 可调用集合的 6 个阶段。
参考答案：读取 extensions 运行时配置里的启用 server；get_mcp_tools() 通过 MCP client 拉取工具；cache 暂存工具；get_available_tools() 取缓存并用 tag_mcp_tool() 打标；lead agent 经过 tool groups 与 skill policy 过滤；assemble_deferred_tools() 根据 tool_search.enabled 决定直接绑定 schema，或追加 tool_search 并由 middleware 延迟暴露。

主要参考来源：backend/packages/harness/deerflow/tools/tools.py、backend/packages/harness/deerflow/mcp/tools.py、backend/packages/harness/deerflow/tools/builtins/tool_search.py。模块地图见 tools-mcp-overview.html；函数签名、参数约束与测试锚点见 L3 tools-mcp-api.html。如果某段边界不清楚，可以继续追问 agent：“从这个阶段继续展开源码”。
