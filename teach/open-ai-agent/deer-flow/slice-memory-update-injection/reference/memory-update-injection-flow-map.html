middleware 链位置（lead agent）

注入在 agent 执行之前，更新在 agent 执行之后——跨回合闭环，非同回合同步。

端到端流程

用户第 N 轮消息 → Gateway run_agent → LangGraph before_agent: DynamicContextMiddleware._inject() ├── _get_memory_context() → get_memory_data(user_id) → format_memory_for_injection └── 首条 HumanMessage ID-swap: SystemMessage(id=原id, date) + HumanMessage(id=__memory) + HumanMessage(id=__user) … model / tools 循环 … after_agent: MemoryMiddleware.after_agent() ├── memory.enabled? thread_id? human+ai 成对? ├── filter_messages_for_memory (跳过 tool 中间态) ├── detect_correction / detect_reinforcement └── queue.add(thread_id, messages, user_id, deerflow_trace_id) threading.Timer(debounce_seconds 默认 30) └── MemoryUpdater.update_memory() ├── sync LLM invoke (独立连接池, 见 #2615) ├── _apply_updates → facts + user 摘要段 └── storage.save → users/{user_id}/memory.json (tmp + rename) 用户第 N+1 轮 → DynamicContextMiddleware 读到新 memory

agents/memory/ 模块骨架

agents/memory/ ├── queue.py ← 全局单例防抖队列 ├── updater.py ← LLM 抽取 + 原子持久化 ├── storage.py ← 文件存储与 cache ├── prompt.py ← 更新 prompt 与注入 token 计数 ├── message_processing.py ← 入队过滤与 correction 信号 └── summarization_hook.py ← 与 summarization 的衔接（本切片不展开）

关键配置（config.yaml → memory）

字段默认作用
enabledtrue总开关；关则不入队、不加载
injection_enabledtrue仅控制注入；可与更新分开关
debounce_seconds30入队后等待多久批处理
max_injection_tokens2000注入预算上限
token_countingtiktokenchar 可避网络冷启动阻塞
model_namenullmemory 更新专用模型；null 用默认
