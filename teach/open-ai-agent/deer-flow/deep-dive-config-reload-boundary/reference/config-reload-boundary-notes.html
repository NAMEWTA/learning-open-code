注册表 API

符号文件用途
STARTUP_ONLY_PREFIXreload_boundary.py固定字符串 "startup-only:"，schema 与扫描器 pivot

签名与缓存

符号行为
_get_config_signature(path)返回 (mtime, size, sha256_hex)；读失败时 digest 为 None
get_app_config()比较路径 + 完整签名；变化则 AppConfig.from_file() 并更新模块级缓存
reload_app_config()强制从磁盘重载（测试 / 显式刷新）
set_app_config()注入自定义实例；跳过后续文件签名检测（_app_config_is_custom）
push_current_app_config()ContextVar 运行时覆盖；优先于模块级单例

日志文案：mtime: … -> … 或 content signature changed。二者都伴随对象重载，但不代表启动单例已更新。

误报场景对照

现象日志/检测实际原因处理
改了 sandbox 仍 local有 reloading 日志AppConfig 新，provider 单例旧重启 Gateway
改了 database.backend 仍 memory有 reloading 日志engine 启动时创建重启
无 X-Trace-Id 响应头可能无 signature 日志TraceMiddleware 构造时读 enhance重启；查 logging 段
改了 models.max_tokens 无效无 reloading文件未保存或路径错误查 DEER_FLOW_CONFIG_PATH、YAML 语法
改了 memory 无新事实有 reloading 日志debounce + 异步 LLM，非边界等待；见 memory 切片
改了 channels 凭证 bot 仍失败有 reloading 日志IM 客户端启动绑定重启

排障决策树

改 config.yaml 后行为不符 │ ├─ 顶层段 ∈ STARTUP_ONLY_FIELDS ? │ ├─ 是 → 重启 Gateway（与有无 reloading 日志无关） │ └─ 否 → 查 get_app_config 是否重载（日志 / 改热字段试探） │ ├─ 未重载 → 路径、保存、YAML 校验 │ └─ 已重载仍异常 → 代码是否用 startup_config？或异步延迟？ │ └─ extensions_config.json → 另走 mtime；见 tools-mcp 模块
