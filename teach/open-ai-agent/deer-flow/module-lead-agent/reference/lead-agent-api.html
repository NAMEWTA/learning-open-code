2. Graph 工厂与 middleware 组装

2.1 make_lead_agent

backend/packages/harness/deerflow/agents/lead_agent/agent.py

def make_lead_agent(config: RunnableConfig) -> CompiledStateGraph
参数含义约束
configLangGraph 运行时配置必须可从中读取 configurable 与 context；graph 根级会写入 metadata 与 tracing callbacks。

运行时键（经 _get_runtime_config 合并）：

键类型默认影响
model_name / modelstr | None—请求级模型；未知名回退默认模型并打 warn。
thinking_enabledboolTrue模型不支持 thinking 时自动降为 False。
reasoning_effort任意None传给 create_chat_model。
is_plan_modeboolFalse为 True 时追加 TodoMiddleware。
subagent_enabledboolFalse控制 task 工具与 SubagentLimitMiddleware。
max_concurrent_subagentsint3单轮 task 调用硬上限。
agent_namestr | NoneNone经 validate_agent_name 规范化；驱动 custom agent config、SOUL、update_agent 工具。
is_bootstrapboolFalse为 True 时走 bootstrap 分支，仅暴露 setup_agent 与 bootstrap skill。
non_interactiveboolFalse为 True 时从工具列表移除 ask_clarification（调度器路径）。
channel_namestr | NoneNonegithub 等 webhook 渠道会 withhold update_agent。
user_idstr | Nonecontextvar用户级 skill 加载；缺省时 get_effective_user_id()。
app_configAppConfig | Noneget_app_config()允许 Gateway 注入热重载配置快照。

返回值：create_agent(...) 编译图，state_schema=ThreadState。

异常：

条件异常
未配置任何模型ValueError("No chat model could be resolved...")
skill 加载失败记录 exception 后 re-raise（allowed-tools policy 前置加载）

追踪不变量：tracing callbacks 必须挂在 config["callbacks"]；图内所有 create_chat_model 必须 attach_tracing=False。

测试：test_lead_agent_model_resolution.py::test_make_lead_agent_attaches_tracing_callbacks_at_graph_root、test_lead_agent_skills.py、test_update_agent_tool.py（webhook withhold）。

2.2 build_middlewares

def build_middlewares(
    config: RunnableConfig,
    model_name: str | None,
    agent_name: str | None = None,
    custom_middlewares: list[AgentMiddleware] | None = None,
    *,
    available_skills: set[str] | None = None,
    app_config: AppConfig | None = None,
    deferred_setup=None,
    user_id: str | None = None,
) -> list[AgentMiddleware]
参数约束返回值
config读取 is_plan_mode、subagent_enabled、max_concurrent_subagents有序 middleware 列表；ClarificationMiddleware 恒为最后一项。
model_name决定 ViewImageMiddleware 是否加入（supports_vision）
deferred_setup非空且含 deferred_names 时追加 DeferredToolFilterMiddleware
custom_middlewares插在 SafetyFinishReasonMiddleware 之前、ClarificationMiddleware 之前

稳定性：名称被 DeerFlowClient 跨模块引用；签名变更需同步 client.py。测试：test_client.py、test_lead_agent_model_resolution.py（loop detection 注入）。

3. SDK 工厂 create_deerflow_agent

backend/packages/harness/deerflow/agents/factory.py

def create_deerflow_agent(
    model: BaseChatModel,
    tools: list[BaseTool] | None = None,
    *,
    system_prompt: str | None = None,
    middleware: list[AgentMiddleware] | None = None,
    features: RuntimeFeatures | None = None,
    extra_middleware: list[AgentMiddleware] | None = None,
    plan_mode: bool = False,
    state_schema: type | None = None,
    checkpointer: BaseCheckpointSaver | None = None,
    name: str = "default",
) -> CompiledStateGraph
参数组合行为异常
仅 model默认 RuntimeFeatures() 自动装配 middleware；state_schema 默认 ThreadState—
middleware=...完全接管链；不得与 features / extra_middleware 同传ValueError
features=...按 feature flag 组装；自动追加 feature 注入工具（如 ask_clarification、task、view_image）guardrail=True 或 summarization=True 无内置默认 → ValueError
extra_middleware通过 @Next/@Prev 锚点插入；未锚定项插在 ClarificationMiddleware 前锚点冲突 / 循环依赖 → ValueError；非 AgentMiddleware → TypeError
plan_mode=True追加 TodoMiddleware—

工具去重：feature 注入工具按 tool.name 去重，用户传入工具优先。

测试：test_create_deerflow_agent.py（最小创建、features 模式、@Next/@Prev、冲突检测）；集成：test_create_deerflow_agent_live.py。

4.1 apply_prompt_template

def apply_prompt_template(
    subagent_enabled: bool = False,
    max_concurrent_subagents: int = 3,
    *,
    agent_name: str | None = None,
    available_skills: set[str] | None = None,
    app_config: AppConfig | None = None,
    deferred_names: frozenset[str] = frozenset(),
    user_id: str | None = None,
    skill_names: frozenset[str] | None = None,
) -> str
参数影响段落边界
subagent_enabledsubagent_section、orchestrator reminder、thinking 提示max_concurrent_subagents 进入硬限制文案
skill_namesdeferred discovery：<skill_index> 而非完整 metadata非 None 时 Skill First 文案引用 describe_skill

class ThreadState(AgentState):

    skill_context: Annotated[list[SkillEntry], merge_skill_context]

5.2 Reducer API

函数签名new is None/empty特殊边界测试类
merge_sandbox (existing, new) → SandboxState | None 保留 existing 不同 sandbox_id → ValueError（fail-closed） TestMergeSandbox
merge_artifacts (list, list) → list[str] 空列表当有效更新 顺序保留去重 TestMergeArtifacts
merge_viewed_images (dict, dict) → dict — new == {} 清空全部 TestMergeViewedImages
merge_todos (list|None, list|None) → list|None new is None 保留 existing（#3123 修复点） 显式空列表覆盖 TestMergeTodos
merge_goal (GoalState|None, GoalState|None) 保留 existing — TestMergeGoal
merge_promoted (PromotedTools|None, PromotedTools|None) 保留 existing catalog_hash 变化时整体替换 test_thread_state_promoted.py
merge_delegations (list, list) → list[DelegationEntry] 保留 existing 终态不被非终态覆盖；最多 50 条 test_delegation_ledger.py
merge_skill_context (list, list) → list[SkillEntry] 保留 existing 按 path 去重；保留最近 8 条；描述截断 500 字符 test_thread_state_reducers.py

常量：TERMINAL_STATUSES = SUBAGENT_STATUS_VALUES；_DELEGATION_LEDGER_MAX_ENTRIES = 50；_SKILL_CONTEXT_MAX_ENTRIES = 8。
