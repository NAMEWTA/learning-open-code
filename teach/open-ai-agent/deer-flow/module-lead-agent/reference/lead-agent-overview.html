lead agent runtime 参考

本页是 L1 模块参考：它把职责、入口、内部结构、中间件清单和源码索引集中在一处。短课入口：0001-lead-agent-module-tour.html。微观 API 字典：lead-agent-api.html（L3）。

模块职责与位置

维度结论源码证据
一句话职责lead agent runtime 把 DeerFlow 的运行时配置装配成 LangChain/LangGraph 可执行 graph：模型、工具、system prompt、middleware 链和 ThreadState。backend/packages/harness/deerflow/agents/lead_agent/agent.py
整体架构位置位于 Gateway run worker 与 tools/subagents/sandbox/memory 之间，是用户消息进入模型与工具生态前的总装配点。backend/app/gateway/services.py、backend/packages/harness/deerflow/runtime/runs/worker.py
LangGraph 注册backend/langgraph.json 将 lead_agent 指向 deerflow.agents:make_lead_agent，并注册 auth 与 checkpointer。backend/langgraph.json
默认前端入口workspace stream hook 使用 assistantId: "lead_agent"，custom agent 仍通过同一 graph 加 agent_name 上下文运行。frontend/src/core/threads/hooks.ts、backend/app/gateway/services.py

对外入口与导出

符号位置用途稳定性判断
make_lead_agent(config)agents/lead_agent/agent.pyLangGraph graph factory，解析 runtime config 并返回 create_agent(...) 结果。核心入口；langgraph.json 依赖签名。
deerflow.agents:make_lead_agentagents/__init__.py包级 lazy export；首次解析时会 prime_enabled_skills_cache()。LangGraph 注册入口。
build_middlewares(...)agents/lead_agent/agent.py组装 lead agent 完整 middleware 链；DeerFlowClient 也复用。跨模块引用，注释要求保持名称稳定。
apply_prompt_template(...)agents/lead_agent/prompt.py生成静态 system prompt，拼入 SOUL、skills、subagent、deferred tools、ACP 与 mounts。prompt 组装主入口。
create_deerflow_agent(...)agents/factory.pySDK 级纯参数工厂，介于 LangChain 原生 create_agent 和配置驱动 make_lead_agent 之间。通用扩展入口。
RuntimeFeatures、Next、Prevagents/features.py声明式 feature gate 与 middleware 插入定位。用于通用工厂和自定义 middleware。
ThreadStateagents/thread_state.pyLangGraph state schema，定义 reducer-backed 运行态字段。agent state 边界。

内部分层结构

backend/packages/harness/deerflow/agents/ ├── __init__.py ← 包级 lazy exports ├── factory.py ← SDK 级 create_deerflow_agent ├── features.py ← RuntimeFeatures 与 Next/Prev ├── thread_state.py ← ThreadState 与 reducer ├── lead_agent/ ← lead graph 装配主线 │ ├── __init__.py ← make_lead_agent re-export │ ├── agent.py ← 模型、工具、middleware、prompt 总装配 │ └── prompt.py ← system prompt 与 skill/subagent prompt section ├── middlewares/ ← 运行时中间件集合 │ ├── tool_error_handling_middleware.py ← 共享 runtime builder │ ├── dynamic_context_middleware.py ← 日期和 memory 动态注入 │ ├── skill_activation_middleware.py ← slash skill 激活 │ └── ... ← 其余 guard、budget、vision、summary └── memory/ ← memory 读写、摘要 hook 与更新队列

装配链路

thread_runs router / channels / scheduler ↓ Gateway services.start_run ↓ resolve_agent_factory → make_lead_agent ↓ _make_lead_agent ├── resolve model + thinking ├── load custom agent config + available skills ├── get_available_tools + skill allowed-tools policy ├── assemble_deferred_tools + describe_skill ├── apply_prompt_template ├── build_middlewares └── create_agent(..., state_schema=ThreadState) ↓ runtime/runs/worker.py → agent.astream(...)

ThreadState 数据边界

字段用途reducer 行为
sandbox保存当前线程 sandbox id。只接受幂等写入；不同 sandbox id 冲突会失败。
thread_dataworkspace、uploads、outputs 路径。由 ThreadData/Uploads/Sandbox 相关 middleware 使用。
artifacts运行产物路径。合并并去重，保持顺序。
todosplan mode 的 todo 列表。新值覆盖旧值，None 表示未触碰。
goalGoal Loop 续跑状态。新值覆盖，未写入时保留。
viewed_imagesview_image 工具读出的图像数据。字典合并；空字典表示清空。
promotedtool_search 已提升的 deferred tools。按 catalog hash 隔离，hash 变化时整体替换。
delegationstask subagent 委托 ledger。按 id 合并，终态不会被非终态覆盖，最多保留 50 条。
skill_context已加载 skill 的引用。按 path 去重，只保留最近 8 条，丢弃正文。
summary_textsummarization 结果。供 durable context 和后续 model call 使用。

中间件清单

基础 runtime middleware

顺序middleware条件职责源码
1InputSanitizationMiddleware总是清洗最后一条真实用户消息，降低 prompt injection 风险。agents/middlewares/input_sanitization_middleware.py
2ToolOutputBudgetMiddleware总是对过大的工具结果做外置化或截断。agents/middlewares/tool_output_budget_middleware.py
3ThreadDataMiddleware总是创建线程级 workspace、uploads、outputs 路径。agents/middlewares/thread_data_middleware.py
4UploadsMiddlewarelead 包含把上传文件信息注入上下文。agents/middlewares/uploads_middleware.py
5SandboxMiddleware总是初始化 sandbox 并写入 ThreadState.sandbox。sandbox/middleware.py
6DanglingToolCallMiddlewarelead 包含修补历史中缺失的 ToolMessage，避免模型看到断裂工具调用。agents/middlewares/dangling_tool_call_middleware.py
7LLMErrorHandlingMiddleware总是重试 transient LLM 错误，并输出可见 fallback。agents/middlewares/llm_error_handling_middleware.py
8GuardrailMiddleware配置启用接入配置化 guardrail provider；由 shared builder 装配。guardrails/middleware.py；装配点在 agents/middlewares/tool_error_handling_middleware.py
9SandboxAuditMiddleware总是审计 bash 命令安全。agents/middlewares/sandbox_audit_middleware.py
10ReadBeforeWriteMiddlewareread_before_write.enabled阻止未读当前版本就写入已有文件。agents/middlewares/read_before_write_middleware.py
11ToolErrorHandlingMiddleware总是把工具异常转换成 error ToolMessage，使 run 可继续。agents/middlewares/tool_error_handling_middleware.py

lead-only middleware

顺序middleware条件职责源码
12DynamicContextMiddleware总是把当前日期和 memory 作为动态 reminder 注入首条 HumanMessage。agents/middlewares/dynamic_context_middleware.py
13SkillActivationMiddleware总是处理 /skill-name 激活与用户级 skill 加载。agents/middlewares/skill_activation_middleware.py
14DurableContextMiddleware总是捕获 delegations 与 loaded skills，注入持久上下文。agents/middlewares/durable_context_middleware.py
15DeerFlowSummarizationMiddlewaresummarization 启用压缩上下文，并在压缩前触发 memory flush hook。agents/middlewares/summarization_middleware.py
16TodoMiddlewareis_plan_mode提供 write_todos 并防止未完成 todo 时过早退出。agents/middlewares/todo_middleware.py
17TokenUsageMiddlewaretoken usage 启用记录 token 并给 AI step 添加归因。agents/middlewares/token_usage_middleware.py
18TitleMiddleware总是首轮后生成线程标题。agents/middlewares/title_middleware.py
19MemoryMiddleware总是run 结束后将对话排队给 memory update。agents/middlewares/memory_middleware.py
20ViewImageMiddleware模型支持 vision把 view_image 结果重新注入模型上下文。agents/middlewares/view_image_middleware.py
21DeferredToolFilterMiddleware存在 deferred tools在 tool_search promote 前隐藏 MCP 工具 schema。agents/middlewares/deferred_tool_filter_middleware.py
22SystemMessageCoalescingMiddleware总是把所有 SystemMessage 合并为一个 leading SystemMessage。agents/middlewares/system_message_coalescing_middleware.py
23SubagentLimitMiddlewaresubagent_enabled裁剪单轮中过量 task 调用。agents/middlewares/subagent_limit_middleware.py
24LoopDetectionMiddlewareloop detection 启用检测重复工具调用循环并终止。agents/middlewares/loop_detection_middleware.py
25TokenBudgetMiddlewaretoken budget 启用强制单 run token 上限。agents/middlewares/token_budget_middleware.py
26custom_middlewares调用方提供在 clarification 前插入扩展 middleware。agents/lead_agent/agent.py
27SafetyFinishReasonMiddlewaresafety finish reason 启用当 provider 安全终止时清理 tool calls，避免继续执行。agents/middlewares/safety_finish_reason_middleware.py
28ClarificationMiddleware总是最后拦截 ask_clarification 并以 interrupt 呈现给用户。agents/middlewares/clarification_middleware.py

辅助文件不一定直接出现在链中：delegation_ledger.py 渲染 task ledger，skill_context.py 生成 skill context 元数据，safety_termination_detectors.py 提供 provider safety detector，tool_call_metadata.py 维护 AIMessage tool-call 元数据一致性。
