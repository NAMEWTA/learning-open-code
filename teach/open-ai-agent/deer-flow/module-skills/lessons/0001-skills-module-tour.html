15 分钟理解 skill 如何被发现、激活、装入 prompt 并约束工具

学习目标：读完后，你能用一条主线解释 DeerFlow skill 从存储发现到 prompt 出现，再到 slash 激活和工具收窄的过程。

1. 先把 skill 看成“延迟加载的工作流文件”

L0 总览已经说过：skill 不是普通 tool，而是一个带 frontmatter 的 skill file 工作流。运行时先发现它的名字、描述、类别、启用状态和文件路径；真正的正文只在任务需要时才读。frontmatter 解析与 .skill 安装流程见 reference 生命周期表（§2 解析、§9 安装）。

在源码里，入口分成两类：存储与解析在 backend/packages/harness/deerflow/skills/，agent 装配在 backend/packages/harness/deerflow/agents/lead_agent/agent.py。读源码时不要从某个 public skill 开始，而要先看 lead agent 怎么拿到可用 skill 列表。

2. 发现：prompt 只给线索，不直接塞正文

load_skills(enabled_only=True) ↓ custom agent skills 白名单过滤 ↓ skills.deferred_discovery ? ├── false: prompt 渲染 name / description / location └── true : prompt 只渲染 <skill_index>，并追加 describe_skill 工具

默认模式下，backend/packages/harness/deerflow/agents/lead_agent/prompt.py 会把可用 skill 的摘要渲染进 <skill_system>，模型看到路径后再用 read_file 读取对应 skill file。如果开启 skills.deferred_discovery，prompt 只给 skill 名字，模型先调用 describe_skill 取描述和位置，再决定是否读取正文。

skills_for_tool_policy = _load_enabled_skills_for_tool_policy(...)
skill_setup = build_skill_search_setup(skills_for_tool_policy, enabled=...)
filtered = filter_tools_by_skill_allowed_tools(raw_tools, skills_for_tool_policy,
                                               always_allowed_tool_names=SKILL_LOADING_TOOL_NAMES)
system_prompt = apply_prompt_template(..., skill_names=skill_setup.skill_names or None)

3. 激活：slash 是用户显式指定

当用户输入 /data-analysis analyze uploads/orders.csv，backend/packages/harness/deerflow/agents/middlewares/skill_activation_middleware.py 会解析最末尾的真实用户消息，检查 skill 是否已安装、已启用、并在当前 agent 白名单内。通过后，它读取完整 skill file，构造一个隐藏的 HumanMessage 放到模型调用前。

这和模型自主发现不同：slash 代表用户本轮明确指定，所以 runtime 直接注入正文；自主发现则只给索引或描述，让模型用 read_file 渐进加载。无论哪条路，disabled skill 都不能被使用。

4. 工具：skill 可以反向收窄 tool set

allowed-tools 是 frontmatter 里的可选字段。若本次加载范围内没有任何 skill 声明它，DeerFlow 保持 legacy allow-all；一旦至少一个 skill 显式声明，运行时就取这些声明的并集过滤工具。lead agent 还会额外保留 read_file，否则模型连 skill 正文都无法加载。

判断题：假设 custom agent 只允许 data-analysis，用户输入 /code-documentation write docs。会发生什么？如果 data-analysis 声明了 allowed-tools: [read_file, python]，工具集合如何变化？
参考答案：slash 解析会发现 code-documentation 不在当前 agent 的 skill 白名单内，返回“not available for this agent”一类错误，不会注入该 skill。工具集合则按当前可用 skill 的显式 allowed-tools 并集收窄，并保留框架级 read_file。如果没有任何可用 skill 显式声明 allowed-tools，才保持 allow-all。
