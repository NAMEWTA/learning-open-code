Skills 系统参考

本页是 L1 模块参考：集中放置职责、架构位置、API、内部分层、生命周期、内置 skill 表与测试锚点。短课入口：0001-skills-module-tour.html；L3 公共 API 速查：skills-api.html；整体位置对照 L0 架构图中的 Skills/Tools 能力扩展层。

模块职责与架构位置

维度结论源码证据
一句话职责Skills 系统把 skill file 目录树变成 runtime 可见的能力索引，并通过 prompt、slash middleware、durable context 和 tool policy 影响 lead agent 行为。backend/packages/harness/deerflow/skills/
整体架构位置位于配置/存储层与 lead agent 之间；在 L0 架构图中对应 Skills/Tools 能力扩展层。Gateway 管理 skill 生命周期，lead agent 在每次创建 graph 时加载启用 skill，prompt 与工具集合据此变化。backend/app/gateway/routers/skills.py、backend/packages/harness/deerflow/agents/lead_agent/agent.py
运行时路径host 侧默认从项目 skills/ 读取；sandbox/container 侧默认挂载到 skills container root，按 public、custom、legacy 分类。backend/packages/harness/deerflow/config/skills_config.py
与 Tools/MCP 的边界Skills 不直接等于工具。它们先指导模型如何工作，再用 allowed-tools 对已加载工具做过滤；MCP 工具发现与 promotion 由 Tools/MCP 模块负责。backend/packages/harness/deerflow/skills/tool_policy.py

生命周期速查

阶段发生什么关键源码常见故障点
1. 配置解析SkillsConfig 解析 host 路径、container 路径和 deferred_discovery。backend/packages/harness/deerflow/config/skills_config.py路径不在预期项目根；container path 与 prompt 中路径不一致。
2. 存储发现load_skills 遍历 public、custom、legacy 分类下的 skill file，解析 frontmatter，按名字去重并排序。storage/skill_storage.py、storage/user_scoped_skill_storage.pyfrontmatter 缺 name 或 description；后发现的同名 skill 覆盖前者。
3. 启用状态合并PUBLIC 走全局 extensions config；CUSTOM/LEGACY 在 user-scoped storage 中叠加用户级状态文件。storage/user_scoped_skill_storage.py、backend/app/gateway/routers/skills.pypublic toggle 影响所有用户；custom toggle 只影响当前用户。
4. agent 白名单custom agent 可配置 skills：省略表示继承所有启用 skill，[] 表示禁用所有，列表表示只允许这些名字。backend/packages/harness/deerflow/agents/lead_agent/agent.pyslash 激活已安装 skill 但当前 agent 不允许。
5. prompt 装入默认渲染 <available_skills> 摘要；开启 deferred_discovery 时只渲染 <skill_index> 并追加 describe_skill。backend/packages/harness/deerflow/agents/lead_agent/prompt.py、skills/describe.pycache 未刷新导致 UI toggle 后短时看见旧 prompt。
6. slash 激活/skill-name 命中后读取完整 skill file，注入隐藏 HumanMessage，并记录审计信息。agents/middlewares/skill_activation_middleware.py、skills/slash.pyreserved slash 命令、disabled skill、路径越界、文件不可读。
7. durable skill context模型用 read_file 读过的 skill file 会被记录为 name/path/description 引用；后续只提醒引用，不持久化正文。agents/middlewares/skill_context.py、agents/thread_state.py读工具名不在 skill_file_read_tool_names 中，导致无法捕获。
8. 工具约束若任一可用 skill 声明 allowed-tools，工具集合按声明并集收窄；lead agent 额外保留 skill loading 工具名。skills/tool_policy.py、agents/lead_agent/agent.py混合 legacy skill 与 explicit skill 时，未声明者不会恢复 allow-all。
9. 安装与演进Gateway 安装 .skill，或 agent 在 skill evolution 开启时通过 skill_manage 管理 custom skill；都会走校验、扫描、历史和 cache 刷新。skills/installer.py、tools/skill_manage_tool.pyZIP traversal、可执行脚本扫描、support file path 越界。

优先级与过滤顺序

层级真实顺序结果源码锚点
Local storage discoveryLocalSkillStorage._iter_skill_files() 按 public、custom、legacy 的枚举顺序遍历；SkillStorage.load_skills() 用 skills_by_name[skill.name] = skill 合并。后出现的同名 skill 覆盖前者，所以同名 custom 可以 shadow public。backend/packages/harness/deerflow/skills/storage/local_skill_storage.py、backend/packages/harness/deerflow/skills/storage/skill_storage.py
User-scoped discoveryUserScopedSkillStorage 总是读取 global public，再读取当前用户 custom；只有当用户 custom 目录没有任何 skill 时，才把 global custom 作为 LEGACY fallback 暴露。一旦用户创建任意 custom skill，legacy fallback 对该用户消失；同名用户 custom 也会覆盖 global public。backend/packages/harness/deerflow/skills/storage/user_scoped_skill_storage.py
Enabled statePUBLIC 的 enabled state 来自全局 extensions config；CUSTOM/LEGACY 先保留全局默认，再与用户级 enabled state 做 AND 合并。PUBLIC toggle 是全局的；CUSTOM/LEGACY toggle 是用户级的，同时不会绕过升级前的全局禁用。backend/packages/harness/deerflow/skills/storage/user_scoped_skill_storage.py、backend/packages/harness/deerflow/config/extensions_config.py
Agent whitelistcustom agent 的 skills 字段在 enabled skill 列表之后过滤：省略表示继承所有 enabled skill，空数组表示不允许任何 skill，列表表示只允许这些名字。slash、deferred discovery、prompt 和 allowed-tools 都只能看到当前 agent 允许的 skill。backend/packages/harness/deerflow/agents/lead_agent/agent.py
Prompt / slash / tool policyprompt 只渲染过滤后的 skill；slash 激活再次校验 enabled 与 whitelist；allowed-tools 在过滤后的 skill 集合上求并集。未被允许的 skill 不会通过 slash 绕过限制；未声明 allowed-tools 的 skill 不会把工具恢复成 allow-all。backend/packages/harness/deerflow/agents/middlewares/skill_activation_middleware.py、backend/packages/harness/deerflow/skills/tool_policy.py

调用示例

显式 slash 激活

/data-analysis analyze uploads/orders.csv

处理顺序：parse_slash_skill_reference 提取名字和剩余任务文本；storage 加载当前用户可见 skills；检查 enabled 与 agent allowlist；读取 container 内的完整 skill file；构造隐藏 <slash_skill_activation> 消息。

deferred discovery 查询

catalog.search("select:data-analysis,deep-research")
catalog.search("+podcast gen")
catalog.search("chart visualization")

这些查询由 describe_skill 工具暴露给模型。它只返回 description、allowed tools 和 file location，不会自动读取完整 skill file。

Gateway 切换 skill 状态

PUT /api/skills/data-analysis
Content-Type: application/json

{"enabled": true}

PUBLIC skill 写全局 extensions config 并清空所有用户 prompt cache；CUSTOM/LEGACY 写当前用户级状态文件并只刷新该用户 cache。
