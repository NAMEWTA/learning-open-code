Subagent 系统参考

本页承载 L1 Subagent 模块的速查材料：职责边界、入口类、内部分层、依赖、调用示例、状态契约、事件映射、executor 责任表和测试线索。L3 API 签名见 subagents-api.html；短课入口：0001-subagents-module-tour.html；L0 锚点见 00-overview.html。

模块职责与架构位置

维度结论源码依据
职责把 lead agent 的 task tool call 转换为后台 subagent 执行，继承运行上下文，直播 step，返回结构化 ToolMessage，并把 step 持久化为 run event。backend/packages/harness/deerflow/tools/builtins/task_tool.py
架构位置位于 lead agent tool 层、runtime stream bridge、run event persistence、前端 workspace subtask card 之间，是长任务 delegation 的执行桥。backend/packages/harness/deerflow/runtime/runs/worker.py、frontend/src/components/workspace/messages/subtask-card.tsx
非职责不做用户意图判定，不允许递归 subagent，不替代 RunManager，不直接渲染 UI。backend/packages/harness/deerflow/tools/builtins/task_tool.py、frontend/src/core/tasks/steps.ts
用户可见结果subtask card 展示 description、运行状态、AI/tool step 时间线、终态结果或错误；刷新后可按 task id 回填历史 step。frontend/src/components/workspace/messages/subtask-card.tsx

内置 subagent

类型用途工具策略默认上限
general-purpose复杂、多步、需要探索与行动的任务。默认继承父 agent 工具，但禁止 task、ask_clarification、present_files。max_turns=150，timeout 使用全局内置默认或覆盖。
bash一组相关命令执行、构建、测试、诊断。仅允许 sandbox 工具 bash、ls、read_file、write_file、str_replace，且 host bash 开关必须允许。max_turns=60，timeout 使用全局内置默认或覆盖。

对外入口与主要类

入口/类角色阅读重点
task_tool()LangChain tool，lead agent 直接调用。配置查找、runtime context 继承、禁用递归 task、启动 executor、轮询并发 stream event。
SubagentConfig单个 subagent 的声明式配置。tools allowlist、disallowed_tools denylist、skills 白名单、model 继承、max_turns、timeout_seconds。
get_subagent_config()registry 解析器。built-in 与 custom agents 合并，再应用 per-agent timeout、model、skills、max_turns 覆盖。
SubagentExecutor执行引擎。创建 agent、加载 skills、过滤工具、构造 state、stream 执行、捕获 step、处理终态。
SubagentResult后台任务状态容器。保存 task_id、trace_id、status、结果、错误、AI/tool step、token usage、cancel event。
SubagentLimitMiddlewarelead agent 输出后的硬限制。单次 AIMessage 只保留前 N 个 task tool call，默认 3，配置范围 2 到 4。
delegation_ledgerdurable context 构造器。只信 ToolMessage 的结构化 status/result 元数据，给 lead agent 渲染“已委托工作”。

调用示例：一次委托的主链路

# 模型输出的 tool_call_id 会成为 task_id
await task_tool(
    runtime=runtime,
    description="调查 run events",
    prompt="检查 subagent step 如何持久化",
    subagent_type="general-purpose",
    tool_call_id="call_123",
)

# task_tool 内部关键动作
config = get_subagent_config("general-purpose")
tools = get_available_tools(subagent_enabled=False)
executor = SubagentExecutor(config=config, tools=tools, thread_id=thread_id)
task_id = executor.execute_async(prompt, task_id="call_123")
writer({"type": "task_started", "task_id": task_id, "description": description})
AI tool call │ ├── task_tool.py │ ├── 解析 subagent config │ ├── 继承 runtime context │ ├── 加载可用工具并关闭 subagent_enabled │ ├── SubagentExecutor.execute_async() │ └── 轮询 SubagentResult，写 task_* stream chunk │ ├── executor.py │ ├── 后台登记 SubagentResult │ ├── 构造 LangGraph agent state │ ├── 流式执行 agent.astream() │ └── 捕获 AIMessage / ToolMessage 为 step 原料 │ ├── step_events.py │ ├── task_started → subagent.start │ ├── task_running → subagent.step │ └── task_completed → subagent.end │ └── frontend subtask card ├── live：消费 task_running └── reload：按 task_id 拉取 subagent.step

关键依赖

依赖如何影响 subagent
runtime.state["sandbox"]、thread_data让 subagent 复用父 agent 的 sandbox 和 workspace 上下文。
runtime.context传递 thread_id、run_id、用户身份、OAuth、channel user、trace id，保证 guardrail 与审计归属一致。
runtime.config["metadata"]提供父模型名、tool groups、available skills、trace metadata。
deerflow.tools.get_available_tools()按模型和 tool groups 取工具，并用 subagent_enabled=False 禁止嵌套 task。
deerflow.skills.storageexecutor 每次 subagent session 加载 enabled skills，再按配置与父 allowlist 过滤。
RunEventStoreruntime worker 把 subagent.step 持久化，前端历史回填依赖它。
contracts/subagent_status_contract.json固定后端与 TypeScript 共享的终态枚举。

状态契约

终态不是从 ToolMessage.content 文本中解析，而是从 ToolMessage.additional_kwargs 读取。文本给模型看，结构化字段给 runtime、ledger 和前端消费。

状态含义元数据字段ledger 行为
completed正常完成。subagent_status，可带 subagent_result_brief 与 subagent_result_sha256。结果可复用，提示 lead 不要重复委托。
failed执行失败。subagent_status，可带 subagent_error。记录失败尝试，可用变化后的计划重试。
cancelled父 run 或用户取消。subagent_status，可带 subagent_error。记录取消尝试，可用变化后的计划重试。
timed_outexecutor 执行超时。subagent_status，可带 subagent_error。记录超时尝试，建议缩小范围或调整 timeout。
polling_timed_outtask_tool 轮询安全网超时。subagent_status，可带 subagent_error。表示后台可能卡住，不能当成功结果复用。
max_turns_reachedLangGraph recursion limit 到达，但可能已有部分结果。同时带 result brief/hash 与 error。它是唯一既 result-bearing 又 error-bearing 的状态。优先复用 partial result，必要时缩小任务或提高 max_turns。

事件与持久化映射

stream chunkrun eventcontent消费者
task_startedsubagent.starttask_id、descriptionlive SSE 创建卡片；run event store 保留起点。
task_runningsubagent.steptask_id、message_index、kind、text、tool_calls 或 tool_namelive timeline 和刷新后 step 回填。
task_completedsubagent.endstatus=completed，可带 bounded result。卡片终态、ledger 结果复用。
task_failedsubagent.endstatus=failed，可带 bounded error。卡片失败、ledger 失败尝试。
task_cancelledsubagent.endstatus=cancelled。卡片取消、ledger 取消尝试。
task_timed_outsubagent.endstatus=timed_out。卡片超时、ledger 超时尝试。

精确终态原因以 ToolMessage.additional_kwargs.subagent_status 为准；例如 max_turns_reached 会通过 task_failed 进入 run event，polling_timed_out 会通过 task_timed_out 进入 run event。

backend/packages/harness/deerflow/runtime/runs/worker.py 的 `_SubagentEventBuffer` 用 put_batch 批量写入，达到 25 条或终态事件时 flush，finally 中也会 flush 剩余事件。

executor 责任表

阶段关键函数/字段说明
登记任务execute_async()、_background_tasks创建 SubagentResult(PENDING)，用 task_id 存入全局后台任务表。
调度执行_scheduler_pool、_submit_to_isolated_loop_in_context()后台线程把 coroutine 提交到持久隔离事件循环，保留 ContextVar。
创建 agent_build_initial_state()、_create_agent()加载 skills、过滤工具、追加 deferred tool section，创建 LangGraph agent。
捕获 stepcapture_new_step_messages()按 values-mode 新增尾部捕获 AIMessage 与 ToolMessage，避免多工具输出只保留最后一条。
正常完成_extract_final_result()从最后一个 AIMessage 提取文本作为 result。
取消/超时cancel_event、request_cancel_background_task()协作式取消；长工具调用只能在下一个 stream iteration 边界观察到。
max turnsGraphRecursionError恢复 partial result，并写入 MAX_TURNS_REACHED。
清理cleanup_background_task()只删除 terminal 或已完成任务，避免后台线程仍在写状态时被移除。

前端 subtask card 消费路径

文件作用
frontend/src/core/threads/hooks.ts处理 live stream 中的 task_started/task_running/task_completed 等事件，更新 subtask state。
frontend/src/core/tasks/steps.ts把 live message 或 persisted run event 标准化为 SubtaskStep，按 message_index 排序并去重合并。
frontend/src/core/tasks/api.ts展开旧卡片时分页请求 /events?event_types=subagent.step&task_id=...。
frontend/src/components/workspace/messages/subtask-card.tsx显示卡片标题、运行/完成/失败状态、AI/tool step 时间线和最终 result。
