本文档是模块总览的配套参考资料，提供完整脚本清单、CI 工作流详情和 npm scripts 映射表。建议先在 0001-scripts-infra-module-tour.html 完成导览课程后再查阅。

1. 完整脚本清单（按职责分类）

1.1 质量门禁脚本

脚本文件功能npm 命令关键实现
check-pinned-deps.mjs 确保所有外部 npm 依赖使用精确版本号 check:pinned-deps 扫描所有 package.json，正则 /^\d+\.\d+\.\d+/ 验证版本格式
check-ts-relative-imports.mjs 禁止 .ts 文件中出现相对路径 .js 导入 check:ts-imports 使用 TypeScript Compiler API 遍历 AST，检测 import/export/import()
check-lockfile-commit.mjs 审查 pre-commit 时 lockfile 的变更 pre-commit hook 直接调用 对比 HEAD 与 staged 的 package-lock.json，输出变更摘要
check-browser-smoke.mjs 验证 pi-ai 在浏览器环境可加载 check:browser-smoke 使用 esbuild 打包后检查 ESM 兼容性

1.2 锁文件生成脚本

脚本文件功能npm 命令输入/输出
generate-coding-agent-shrinkwrap.mjs 为 coding-agent 生成 npm-shrinkwrap.json shrinkwrap:coding-agent 输入：package-lock.json → 输出：packages/coding-agent/npm-shrinkwrap.json
generate-coding-agent-install-lock.mjs 为 Pi 安装器生成独立 lockfile install-lock:coding-agent 输入：package-lock.json → 输出：packages/coding-agent/install-lock/package.json + package-lock.json

1.3 版本与发布脚本

脚本文件功能npm 命令特殊说明
release.mjs 完整发布流程编排（9 步） release:patch / release:minor / release:major 含 changelog 更新、产物再生、二次 check、自动 commit+tag+push
local-release.mjs 本地发布测试（构建 tarball 后隔离安装验证） release:local 支持 --skip-check、--force、--out 参数
publish.mjs npm publish（含幂等检查、build output 验证） publish 先检查 dist/ 存在 → 查询 npm 是否已发布 → npm pack --dry-run 验证
sync-versions.js 同步 workspace 包间依赖版本（lockstep） 由 version:patch/minor/major 自动调用 读取所有 package.json → 验证版本一致 → 更新内包依赖为 ^x.y.z
release-notes.mjs 提取/修复 GitHub Release Notes release:fix-links 从 CHANGELOG 提取指定版本的 release notes

3. CI/CD 工作流详情

工作流文件触发条件核心 Job用途
ci.yml push/PR to main build-check-test 主 CI：ubuntu-latest 上 npm ci → build → check → test
build-binaries.yml tag push v* / workflow_dispatch build → stage-github-release → publish-npm → publish-github-release → cleanup 发布流水线：构建 6 平台二进制 + npm publish + GitHub Release 发布

approve-contributor.yml issue_comment (lgtm/lgtmi) approve-contributor 维护者评论 lgtm/lgtmi 后更新 APPROVED_CONTRIBUTORS 文件
issue-gate.yml issues opened check-contributor 外部贡献者 issue 门禁：未在白名单则关闭
issue-analysis.yml issues labeled "lgtm" analyze 对已批准 issue 进行自动分析
issue-triage-labels.yml issues opened triage 自动为新 issue 添加分类标签
npm-audit.yml schedule (weekly) npm-audit 定期运行 npm audit 检查安全漏洞
remove-inprogress-on-close.yml issues closed remove-label issue 关闭后移除 "in progress" 标签

5. Pre-commit Hook 执行逻辑

# .husky/pre-commit 流程（伪代码）
STAGED_FILES = git diff --cached --name-only

# 步骤 1：检查 lockfile 提交
node scripts/check-lockfile-commit.mjs
# → 若提交了 package-lock.json 且非仅 workspace 变更，则拒绝

# 步骤 2：运行完整质量检查
npm run check
# → biome + pinned-deps + ts-imports + shrinkwrap + install-lock + tsgo

# 步骤 3：条件性浏览器冒烟测试
if STAGED_FILES 包含 packages/ai/* 或 packages/web-ui/*:
    npm run check:browser-smoke

# 步骤 4：重新暂存被 biome format 修改的文件
for file in STAGED_FILES:
    git add $file
