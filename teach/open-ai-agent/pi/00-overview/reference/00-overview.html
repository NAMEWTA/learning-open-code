Pi Agent Harness 参考文档

本文档是 Pi Agent Harness 项目 L0 总览的配套参考资料，提供各包职责详表、设计哲学、源码豁免清单等速查信息。建议先在 0001-project-map.html 完成导览课程，再回到此处查阅细节。

关键设计约束：pi-tui 和 pi-ai 是完全独立的底层包，不依赖任何其他 pi 包。这使它们可以独立发布、独立测试、独立升级。pi-coding-agent 作为面向用户的聚合层，集成所有底层包的的能力。

2. 核心设计哲学

2.1 多 Provider 统一 LLM API（pi-ai）

pi-ai 不绑定任何单一 LLM 服务商。它通过统一的接口抽象，同时支持 OpenAI（Chat Completions + Responses API）、Anthropic（Messages API）、Google（Gemini API）、AWS Bedrock 和 Mistral。 通过 models.generated.ts（自动生成）提供模型元数据，实现自动模型发现与配置。用户只需在配置中声明 provider 和 model，pi-ai 负责路由到正确的 SDK。

2.2 传输抽象与状态管理的 Agent 运行时（pi-agent-core）

pi-agent-core 提供通用的 Agent 运行时，不绑定任何具体工具或 UI。核心概念包括： Transport（传输抽象，解耦 Agent 与通信协议）、 State Management（会话状态管理，支持序列化/反序列化）、 Tool Calling（工具注册和调用流程）、 Attachment（文件/上下文附件支持）。 这使得 Pi 可以同时支持 TUI 交互模式和 RPC 服务模式。

2.3 终端 UI 差异渲染（pi-tui）

pi-tui 不是简单的 console.log 包装。它实现了终端差异渲染（differential rendering），每次只更新屏幕上变化的部分，避免全屏刷新带来的闪烁和性能问题。 同时提供了内联 Markdown 渲染、文本编辑器组件和东亚字符宽度感知布局，使 Agent 的输出在终端中清晰可读。

2.4 自扩展编码 Agent（pi-coding-agent）

pi-coding-agent 的核心设计理念是 自扩展（self-extensible）。它通过扩展系统（extensions）和 Hook 机制， 允许用户添加自定义工具、Provider、Sandbox 和中间件。正如 CONTRIBUTING.md 所强调的："Pi's core is minimal"——核心保持精简，功能通过扩展注入。 这避免了内核膨胀，同时保持了高度灵活性。

2.5 严格的依赖安全策略

Pi 将 npm 依赖变更视为受审查的代码变更。多层次的防护措施包括：

  • 精确版本锁定：所有直接外部依赖使用 save-exact=true，禁止 semver 范围。
  • 同日发布防护：min-release-age=2 阻止 npm 解析发布不足 2 天的包。
  • Lockfile 审查：pre-commit hook 阻止意外的 package-lock.json 提交，除非设置 PI_ALLOW_LOCKFILE_CHANGE=1。
  • Shrinkwrap：发布的 CLI 包附带 npm-shrinkwrap.json，锁定传递依赖。
  • npm audit：CI 中有定时的 npm audit --omit=dev 检查。
  • 生命周期脚本审查：shrinkwrap 生成脚本有一个显式 allowlist，新的 lifecycle-script 依赖需要审查后才能通过检查。
