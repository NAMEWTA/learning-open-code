0001 — pi-coding-agent 模块总览

层级：L1 模块总览 预计时长：12 分钟 学习目标：1 个
学习目标

理解 pi-coding-agent 的内部分层架构、对外接口和扩展机制，能说出 CLI 入口层、核心工具层、交互模式层的职责划分，以及它如何将 pi-ai、pi-agent-core、pi-tui 三个底层包聚合为可用的编码 Agent。

1. 在架构中的位置

pi-coding-agent 包名 @earendil-works/pi-coding-agent，是 Pi monorepo 面向用户的顶层聚合包，同时依赖三个底层包，自身被 orchestrator 依赖：

pi-coding-agent 将三个底层包的能力编织在一起——用 pi-ai 调用 LLM、用 pi-agent-core 管理 Agent 状态与工具调用、用 pi-tui 渲染终端界面。它是 Pi 项目中代码量最大的包，覆盖了从 CLI 参数解析到终端渲染的完整链路。

3. 对外接口清单

3.1 CLI 入口（bin）

package.json 的 bin 字段注册了 pi 命令，指向编译产物 dist/cli.js。用户安装后直接执行 pi "hello" 即可启动。

# 基本使用
pi "帮我写一个快排函数"                  # 发送初始消息
pi --resume                              # 恢复上一次会话
pi --mode rpc                            # 以 RPC 模式启动
pi --extensions ./my-ext.ts              # 加载自定义扩展
pi --list-models                         # 列出可用模型
pi --provider anthropic --model claude-sonnet-4-5  # 指定模型

3.2 包导出（exports）

两个导出路径：

导出路径用途典型消费者
. (默认) 公开所有 API：Agent 会话、扩展系统、工具定义、UI 组件、配置管理、剪切板、Shell 等 SDK 嵌入、扩展开发、orchestrator 包
./rpc-entry 独立 RPC 服务入口脚本，等价于 pi --mode rpc Docker 容器化部署、CI/CD 集成

3.3 扩展系统（Extension API）

pi-coding-agent 最大的设计特色是自扩展架构。核心保持精简，功能通过扩展注入：

  • 自定义工具 — 注册新的 Tool 供 Agent 调用
  • Hook 监听 — 拦截工具调用、会话事件、Provider 请求
  • UI 组件 — 注入自定义消息渲染器、编辑器、状态栏组件
  • 命令注册 — 添加斜杠命令（/foo）
  • 自动补全 — 注册终端输入自动补全提供者

4. 编程式 SDK 调用

pi-coding-agent 不仅提供 CLI 入口，也支持编程式嵌入。以下是一个最小调用示例：

import { createAgentSessionRuntime } from "@earendil-works/pi-coding-agent";
import { SessionManager } from "@earendil-works/pi-coding-agent";

// 1. 创建会话管理器（管理 JSONL 持久化）
const sessionManager = new SessionManager({ storageDir: "./.pi/sessions" });

// 2. 通过工厂函数创建 Agent 会话运行时
const result = await createAgentSessionRuntime(
  (services) => services.createSession({ model: "claude-sonnet-4-5" }),
  { cwd: process.cwd(), agentDir: "./.pi", sessionManager }
);

const { session } = result;

// 3. 发送用户消息并订阅 Agent 事件
const sub = session.events.subscribe((event) => {
  if (event.type === "tool_use") {
    console.log("工具调用:", event.name);
  }
});

await session.prompt("列出当前目录下的所有 TypeScript 文件");
sub.unsubscribe();

关键点：createAgentSessionRuntime 接受工厂函数和会话配置，内部通过依赖注入创建认证存储、会话管理器、扩展运行时和工具集。返回的 AgentSession 通过 session.prompt() 发起对话、session.events.subscribe() 订阅运行时事件。

点击查看参考答案
  1. 入口层（cli.ts / main.ts / cli/）→ 核心层（core/）→ 模式层（modes/）→ 工具层（utils/）。入口层处理 CLI 参数和启动引导，核心层实现 Agent 会话与工具，模式层提供交互式/打印/RPC 三种运行模式，工具层提供跨模块的公用函数。
  2. .（默认）导出全部公开 API，供 SDK 嵌入、扩展开发、orchestrator 包使用；./rpc-entry 是独立 RPC 服务入口，等价于 pi --mode rpc，用于容器化部署和 CI/CD 集成。
  3. 五种能力：自定义工具注册、Hook 事件监听、UI 组件注入、斜杠命令注册、终端自动补全提供。
