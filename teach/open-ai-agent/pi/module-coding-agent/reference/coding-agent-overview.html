pi-coding-agent 参考文档

本文档是 pi-coding-agent 模块 L1 总览的配套参考资料，提供完整源码索引、依赖详表、扩展类型速查和默认配置速查。建议先在 0001-coding-agent-module-tour.html 完成导览课程，再回到此处查阅细节。

1. 完整源码文件索引

1.1 入口文件（顶层）

文件职责知识域

migrations.ts旧版本配置迁移逻辑（deprecation 警告 + 自动迁移）配置

model-registry.ts模型注册表：管理可用模型列表模型

3. 扩展系统核心类型速查

以下类型定义于 core/extensions/types.ts：

类型说明
Extension扩展对象：包含 name、hooks、tools、commands 等注册项
ExtensionFactory扩展工厂函数：(api: ExtensionAPI) => Extension
ExtensionAPI扩展可用的 API：事件发布、UI 组件注册、工具注册、快捷方式等
ExtensionHandler扩展处理器类型别名：Extension | ExtensionFactory
ExtensionRuntime扩展运行时接口：管理扩展生命周期和 Hook 分发
ExtensionContext扩展上下文：提供当前会话的配置、Provider 信息等
RegisteredTool注册的工具定义：名称、描述、参数 schema、执行函数
RegisteredCommand注册的斜杠命令：名称、描述、处理函数
Hook 事件类型BashToolCallEvent、EditToolCallEvent、SessionStartEvent、TurnEndEvent 等 30+ 事件类型

4. CLI 关键参数速查

参数类型说明
--providerstring指定 LLM Provider（anthropic/openai/google 等）
--modelstring指定模型名称
--api-keystring直接传入 API Key
--mode"text"|"json"|"rpc"运行模式：文本输出 / JSON 输出 / RPC 服务
--resumeflag恢复上次会话
--continueflag继续最近一次对话
--sessionstring指定会话名称
--forkstring从指定会话 fork 出新分支
--thinkingstring思考级别：off / minimal / low / medium / high / xhigh
--extensionsstring[]指定加载的扩展路径（可多次使用）
--no-extensionsflag禁用所有扩展
--toolsstring[]启用特定工具（如仅在 bash+read 模式下运行）
--no-toolsflag禁用所有内置工具
--list-modelsflag|string列出可用模型（可指定 provider 过滤）
--system-promptstring自定义系统提示词
--append-system-promptstring[]追加系统提示词（可多次使用）
--exportstring导出会话到 HTML 文件
--skillsstring[]指定加载的 skill 文件
--no-skillsflag禁用所有 skills
--verboseflag详细日志模式
--versionflag显示版本信息
