2. Component 接口与通用模式

所有 TUI 组件均遵循 @earendil-works/pi-tui 框架的标准化接口。这里总结三类核心模式。

2.1 基础 Component 接口

渲染组件模式

class MyWidget implements Component {
  render(width: number): string[] {
    // 返回渲染后的 ANSI 行数组
    // 每行不应超过 width 宽度
  }
  invalidate(): void {
    // 标记缓存失效，下次 render 重建
  }
}

适用: 静态展示组件（如 DynamicBorder、FooterComponent）

可聚焦组件模式 (+ Focusable)

class MySelector implements Component, Focusable {
  private _focused = false;
  get focused(): boolean { return this._focused; }
  set focused(v: boolean) { this._focused = v; }

  handleInput(keyData: string): void {
    // 处理键盘事件，通过 Keybinding 匹配
    // 典型流程: 箭头键导航 + Enter选择 + Esc取消
  }
}

适用: 所有选择器组件 + 编辑器 + 输入框

2.2 Container 组合模式

class MyComposite extends Container {
  constructor() {
    super();
    this.addChild(new DynamicBorder());           // 上边框
    this.addChild(new Text("Title"));              // 标题
    this.addChild(this.someChildComponent);        // 核心内容
    this.addChild(new DynamicBorder());           // 下边框
  }
}

几乎所有组件都通过 Container 树 组合构建布局。 典型布局为：DynamicBorder → Spacer → Header → 核心内容 → Spacer → DynamicBorder。 这种模式在 12 个选择器组件中有 10 个直接使用。

2.3 键盘导航流程

按键Keybinding ID行为
↑ / ↓tui.select.up / down列表项上下移动（通常支持循环首尾）
Entertui.select.confirm确认选择当前项，触发 onSelect 回调
Esc / Ctrl+Ctui.select.cancel取消操作，触发 onCancel 回调
Tabtui.input.tab切换作用域 / 模式（如 session 切换 current/all）
PageUp / PageDowntui.select.pageUp / pageDown页内跳转（SessionSelector、SettingsSelector）
任意字符—传递给搜索输入框，触发模糊筛选

2.4 主题 API

所有组件通过全局 theme 代理对象获取颜色，无需传递 theme 参数：

import { theme } from "../theme/theme.ts";

theme.fg("accent", text)       // 前景色
theme.bg("selectedBg", text)   // 背景色
theme.bold(text)               // 粗体
theme.fg("muted", text)        // 弱化文本
theme.fg("error", text)        // 错误文本

Theme 提供 48 种前景色（ThemeColor）和 6 种背景色（ThemeBg）， 支持 hex 颜色、256 色索引、CSS 变量引用，以及热加载和自动终端主题检测。

3. 核心入口（3 个文件）

class InteractiveMode interactive-mode.ts

交互模式的 顶层编排类（5780 行）。负责：会话生命周期管理、TUI 组件树的组装与切换、输入事件路由、扩展集成、Overlay 管理。

export class InteractiveMode { constructor(session: AgentSession, runtime: AgentSessionRuntime, ...) }

核心职责:

  • 事件路由: 将 AgentSessionEvent（消息到达、工具执行、错误等）转化为 TUI 组件更新
  • 模式切换: 管理 normal / selector-overlay / login / first-time-setup 等界面模式
  • 输入处理: 接收键盘事件，分发给当前活跃的 Focusable 组件
  • 扩展系统桥接: 调用扩展注册的编辑器工厂、自动补全提供者、消息渲染器等
  • 会话持久化: 管理 session 导入/导出/分支/命名等操作

class Theme theme/theme.ts

主题颜色引擎。管理 48 种前景色 + 6 种背景色的 ANSI 转义序列生成。支持 truecolor / 256 色两种模式，以及 hex / 256索引 / CSS变量引用三种颜色值格式。

export class Theme { fg(color: ThemeColor, text: string): string bg(color: ThemeBg, text: string): string bold(text: string): string italic(text: string): string getThinkingBorderColor(level): (str) => string getBashModeBorderColor(): (str) => string }

关键导出函数:

  • initTheme(name?) — 初始化全局主题实例
  • setTheme(name) — 运行时切换主题（含文件监听热重载）
  • getAvailableThemes() — 列出所有可用主题（内置 + 自定义 + 注册）
  • getMarkdownTheme() — 获取 Markdown 渲染所需的主题着色函数集
  • getSelectListTheme() — 获取选择列表的主题着色函数集
  • getEditorTheme() — 获取编辑器的主题着色函数集
  • getSettingsListTheme() — 获取设置列表的主题着色函数集
  • highlightCode(code, lang?) — 基于 cli-highlight 的代码语法高亮
  • getResolvedThemeColors() — 用于 HTML 导出的 CSS 颜色提取

class InteractiveThemeController theme/theme-controller.ts

主题控制器。桥接 SettingsManager（持久化设置）与 Theme（运行时颜色），支持终端背景色自动检测和 light/dark 自动切换。

export class InteractiveThemeController { constructor(ui: TUI, settingsManager, showError, onChanged) applyFromSettings(): Promise<void> setThemeName(name: string): ThemeResult setThemeInstance(theme: Theme): ThemeResult preview(themeSettingOrName: string): void }

4. 选择器组件（12 个文件）

共同模式: 所有选择器组件均遵循 Component + Focusable + Container 模式。 它们都包含： (1) handleInput(keyData) 处理键盘导航； (2) 通过 onSelect / onCancel 回调返回结果； (3) 使用 DynamicBorder 包裹上下边框； (4) 支持模糊搜索过滤（大部分通过 Input 搜索框）。

class SessionSelectorComponent session-selector.ts

会话选择器。支持 current/all 两种作用域、threaded/recent/relevance 三种排序、 named/all 两种名称过滤、树形父子会话展示、正则搜索、会话重命名和删除（trash + unlink 回退）。 内含 SessionSelectorHeader 和 SessionList 两个内部类。

constructor( currentSessionsLoader: SessionsLoader, allSessionsLoader: SessionsLoader, onSelect, onCancel, onExit, requestRender, options?, currentSessionFilePath? )

functions session-selector-search.ts

会话搜索与排序工具函数。包含 filterAndSortSessions、parseSearchQuery（支持 regex re: 前缀和带引号短语精确匹配）、 hasSessionName 等纯函数。定义 SortMode 和 NameFilter 类型。

class ModelSelectorComponent model-selector.ts

模型选择器。支持 all/scoped 两种作用域（scoped 模式下仅显示当前项目可用的模型）、模糊搜索、 当前模型标记（✓ 图标）。选中后自动更新 SettingsManager 中的默认模型。

constructor(tui, currentModel, settingsManager, modelRegistry, scopedModels, onSelect, onCancel, initialSearchInput?)

interface ModelSearchItem functions model-search.ts

模型搜索文本生成工具。定义 ModelSearchItem 接口（id, provider, name）， 提供 getModelSearchText 和 getModelSelectorSearchText（将 provider 前置以优化排序）。

class ConfigSelectorComponent config-selector.ts

包资源配置选择器。管理 extensions / skills / prompts / themes 四种资源的启用/禁用状态。 按 scope（user/project/temporary）和 origin（package/top-level）分组展示，支持多选切换。

class SettingsSelectorComponent settings-selector.ts

设置面板选择器。管理 20+ 项设置（auto-compact、showImages、thinking level、transport、theme、 project trust、HTTP idle timeout 等），使用 SettingsList 基类渲染，支持子菜单导航 （如 thinking level 进入独立 ThinkingSelector）。导出 SettingsConfig、SettingsCallbacks 类型。

class TrustSelectorComponent trust-selector.ts

项目信任选择器。列出信任选项（trust once / always trust / never trust），显示当前保存的决策和继承路径。 支持 j/k 作为 vim 风格的额外导航键。

export type TrustSelection = Pick<ProjectTrustOption, "trusted" | "updates">

class ThinkingSelectorComponent thinking-selector.ts

推理级别选择器。6 个级别（off / minimal / low / medium / high / xhigh）， 每个级别附带预估 token 消耗描述。使用 SelectList 基类渲染。

class ShowImagesSelectorComponent show-images-selector.ts

图片显示选择器。二元选择（Yes, show inline / No, text placeholder）。 使用 SelectList 渲染，2 个选项。

class OAuthSelectorComponent oauth-selector.ts

认证提供者选择器。支持 login / logout 两种模式，展示各 provider 的认证状态 （已登录/未登录/过期），带模糊搜索和 OAuth 类型标记。

export type AuthSelectorProvider = { id: string; name: string; authType: "oauth" | "api_key" }

class ScopedModelsSelectorComponent scoped-models-selector.ts

作用域模型多选器。与 ModelSelectorComponent 不同，此组件支持 多选（toggle 模式）、 enableAll / clearAll / 移动排序操作。导出 ModelsConfig、ModelsCallbacks 类型。 用于扩展注册和项目级模型白名单配置。

class ExtensionSelectorComponent extension-selector.ts

通用字符串选项选择器。为扩展提供轻量级选择交互。支持可选超时倒计时（通过 CountdownTimer）。 比 SelectList 更轻量：直接在内联列表中用箭头导航。

constructor(title, options: string[], onSelect, onCancel, opts?: ExtensionSelectorOptions)

class ThemeSelectorComponent theme-selector.ts

主题选择器。列出所有可用主题（内置 + 自定义），支持实时预览（通过 onSelectionChange 回调触发 onPreview）。 使用 SelectList 渲染。

5. 消息渲染组件（11 个文件）

渲染流程: 消息到达 → InteractiveMode 分发事件 → 创建对应 Component 实例 → 加入 Container 树 → TUI 差分渲染 → 终端输出。 消息组件分为文本型（assistant/user/custom message）和工具型（tool/bash execution）两类。

class AssistantMessageComponent assistant-message.ts

助手消息组件。最复杂的消息渲染器。支持 thinking block 折叠/展开、Markdown 渲染、OSC 133 终端区域标记 （用于终端的 shell 集成——标记输出区域的开始/结束）。内部管理两个 Container 分别渲染思考内容和正文。

constructor(message?, hideThinkingBlock?, markdownTheme?, hiddenThinkingLabel?, outputPad?) setHideThinkingBlock(hide: boolean): void setOutputPad(padding: number): void

class UserMessageComponent user-message.ts

用户消息组件。使用 userMessageBg 背景色包裹 Markdown 内容。 同样应用 OSC 133 终端区域标记。布局为 Box → Markdown。

class UserMessageSelectorComponent user-message-selector.ts

用户消息分支选择器。用于 Fork from Message 功能——选择一个历史用户消息并复制会话路径。 内部 UserMessageList 类负责列表渲染和键盘导航，支持滚动和空列表自动关闭。

class ToolExecutionComponent tool-execution.ts

工具执行显示组件。渲染工具调用和结果的完整生命周期： pending（加载动画）→ partial args（流式参数）→ executing → result（含 diff / 文本输出）。 支持 20 行预览 + 展开完整输出。支持图片内联显示（Kitty / iTerm2 协议）。 导出 ToolExecutionOptions（showImages、imageWidthCells）。

class BashExecutionComponent bash-execution.ts

Bash 命令执行显示组件。独立于通用 ToolExecution 渲染，专为 shell 命令优化： 流式输出追加、ANSI 转义序列清理、输出截断（max bytes / max lines）、 展开/折叠切换、!! 前缀命令的特殊 dim 边框。

constructor(command: string, ui: TUI, excludeFromContext?: boolean) # 关键状态: "running" | "complete" | "cancelled" | "error"

function renderDiff diff.ts

代码差异渲染函数。使用 diff 库进行行级 + 词级差异计算。 支持 行内词级高亮（通过 theme.inverse() 反色显示变化部分）、 制表符替换为空格、行号解析（+123 content 格式）。 导出 RenderDiffOptions 类型。

export function renderDiff(diffText: string, options?: RenderDiffOptions): string[] export type RenderDiffOptions

class CompactionSummaryMessageComponent compaction-summary-message.ts

上下文压缩摘要消息。使用 customMessageBg 背景渲染。折叠态显示压缩前 token 数， 展开态显示完整的 LLM 生成的摘要。标签为 [compaction] 粗体。

class BranchSummaryMessageComponent branch-summary-message.ts

分支摘要消息。与 CompactionSummary 共享相同的折叠/展开模式。 标签为 [branch] 粗体。展开后显示完整的分支对话总结。

class SkillInvocationMessageComponent skill-invocation-message.ts

技能调用消息。渲染 skill 的名称和内容。折叠态显示 [skill] 名称 (hint to expand)， 展开态显示 [skill] 标签 + skill 全名标题 + 完整 Markdown 内容。

class CustomMessageComponent custom-message.ts

自定义消息渲染器。支持扩展通过 MessageRenderer 注册的自定义渲染逻辑。 优先调用自定义渲染器；回退到默认渲染（[customType] 标签 + Markdown 正文）。 使用 customMessageBg 背景色。

constructor(message: CustomMessage<unknown>, customRenderer?: MessageRenderer, markdownTheme?)

class CustomEntryComponent custom-entry.ts

自定义会话条目渲染器。与 CustomMessage 类似但针对 CustomEntry（来自 SessionManager）。 必须提供 EntryRenderer，渲染失败时显示错误消息而非静默。 支持 expanded 状态切换。

Armin 像素画动画彩蛋。31x36 像素 XBM 格式图片，使用半块字符（▀）渲染。 支持 7 种动画效果：typewriter、scanline、rain、fade、crt、glitch、dissolve。 通过 setInterval 驱动帧动画，自动切换效果。

class DaxnutsComponent daxnuts.ts

"POWERED BY DAXNUTS" 彩蛋。32x32 像素 RGB 真彩色图片（hex 编码）， 使用半块字符 + 24位色 ANSI 转义序列渲染完整彩色图片。

class EarendilAnnouncementComponent earendil-announcement.ts

Earendil 项目加入公告横幅。渲染带 accent 边框的卡片：标题 "pi has joined Earendil" + 博客链接 URL + 内嵌 PNG 图片（通过 Image 组件渲染，支持 Kitty/iTerm2 协议）。
