1.2 IPC 子模块（ipc/）

文件职责知识域
protocol.ts协议层：6 种请求/响应类型定义、RequestMap/ResponseMap 映射、ResponseFor 泛型条件类型、encodeMessage/parseRequestLine/parseResponseLine 编解码函数。ProtocolMessage 联合类型覆盖所有消息。IPC 协议
server.tsIPC 服务器：基于 Node.js net.createServer() 的 Unix Socket 服务器。普通请求走请求-响应模式（socket.end 写回），rpc_stream 请求走连接升级模式（维持长连接，双向转发 JSONL）。包含 removeStaleSocketIfNeeded 的存活检测。IPC 服务端
client.tsIPC 客户端：sendIpcRequest() 通过 net.createConnection() 连接 Socket，write 请求 → read 单行响应 → end 断开。使用 settled 标志防止竞态。IPC 客户端

1.3 核心运行时（顶层 src/）

文件职责知识域
serve.ts服务启动入口：创建 Unix Socket 服务器 → supervisor.recoverAfterRestart() 恢复实例 → 启动 Radius 心跳 → 注册 SIGINT/SIGTERM 优雅关闭 → 等待事件循环启动流程
handler.tsIPC 请求路由：handleIpcRequest() 函数重载，按 request.type 分发到 supervisor 的 spawnInstance/listInstances/getInstance/stopInstance/handleRpc/openRpcStream。toInstanceSummary() 将 InstanceRecord 转为 IPC 响应格式。openRpcStream() 桥接 stream handler 和 supervisor。请求路由
supervisor.tsOrchestratorSupervisor 核心类：liveInstances Map（内存在线实例集合）、spawnInstance（启动子进程 + bindRpcProcess + syncInstanceRecord + registerPi）、stopInstance（cleanupAcquiredResources → 移除记录）、handleRpc（转发命令到子进程）、recoverAfterRestart（重启后标记所有在线实例为 stopped）。SESSION_METADATA_COMMANDS 集合控制何时刷新会话元数据。实例管理
rpc-process.tsRpcProcessInstance 类：通过 child_process.spawn 启动 pi-coding-agent RPC 入口。支持 Bun 二进制模式（检测 isBunBinary，不同启动命令）。stdout 按行解析 JSONL，根据 type 字段路由：response → resolve pending promise、extension_ui_request → 调用 uiRequestHandler、其他 → 广播给所有 eventListeners。send() 方法分配唯一 request ID。子进程 RPC
storage.ts文件持久化：loadMachine/saveMachine/deleteMachine（machine.json）、loadInstances/saveInstances/getInstance/upsertInstance/removeInstance（instances.json）。数据目录在 ~/.pi/orchestrator/。持久化
radius.tsRadius 云服务集成（600+ 行）：RadiusPresence 类管理机器和 Pi 的远程注册/心跳。包括：registerMachine（向 Radius API 注册宿主机）、registerPi（注册 Pi 实例，含 capabilities 声明）、disconnectPi（断开连接）。心跳机制使用 setTimeout 循环，404 连续 N 次后触发自动重注册。指数退避算法 computeBackoffDelayMs() 含 25% 随机抖动。认证通过 ~/.pi/agent/auth.json 或 PI_RADIUS_API_KEY 环境变量。远程编排

2. IPC 协议详表

2.1 请求类型

type接口关键字段说明
spawn SpawnRequest cwd: string、label?: string、provider?: string、model?: string 在指定工作目录下创建新的 Pi Agent 实例（子进程）
list ListRequest 无 列出所有实例（含已停止的持久化记录）
status StatusRequest instanceId: string 查询指定实例的状态和元信息
stop StopRequest instanceId: string 停止指定实例并清理资源
rpc RpcRequest instanceId: string、command: RpcCommand 向子进程发送单条 RPC 命令，等待响应后断开
rpc_stream RpcStreamRequest instanceId: string 建立持久双向流连接，持续转发命令和事件

3. 实例状态机

状态含义触发条件下一状态
starting 实例正在创建中（子进程已 spawn，等待就绪） spawnInstance() 被调用 online（就绪）/ error（失败）
online 实例正常运行，RPC 进程可用，Radius 已注册 spawn 流程完成 stopping（主动停止）/ error（意外退出）
stopping 实例正在关闭中（清理 Radius、终止子进程） stopInstance() 被调用 stopped
stopped 实例已停止，资源已释放 stopping 流程完成 （终态）
error 实例异常（子进程意外退出或启动失败） handleUnexpectedRpcExit() 触发 stopped（清理后）

重启恢复策略（recoverAfterRestart()）：

  • 所有 online 或 starting 状态的持久化实例 → 标记为 stopped
  • 对所有实例调用 radiusPresence.disconnectPi() 断开旧的 Radius 连接
  • 更新 lastSeenAt 为当前时间

4. 包依赖详表

包名版本类型用途
@earendil-works/pi-coding-agent^0.80.3内部（唯一 runtime dep）提供 RPC 类型定义（RpcCommand、RpcResponse、AgentSessionEvent 等）和 AuthStorage 类。子进程通过 require.resolve 加载其 rpc-entry。
shx0.4.0devDependency跨平台 shell 命令（clean 脚本：shx rm -rf dist）

与 pi-coding-agent 依赖 18 个外部包相比，pi-orchestrator 的依赖极简，只有一个内部依赖和一个 dev 工具。这反映了它的设计定位：轻量的编排层，核心能力（LLM 调用、Agent 逻辑、终端渲染）全部委托给子进程。

5. CLI 命令速查

命令参数底层 IPC 请求说明

6. Radius 云服务集成速查

API 端点HTTP 方法说明
POST /v1/machines/registerPOST注册宿主机（含 hostname、platform、arch、version、capabilities）
POST /v1/machines/{id}/heartbeatPOST宿主机心跳（含 cwd、socketPath）
POST /v1/machines/{id}/disconnectPOST宿主机断开连接
POST /v1/pis/registerPOST注册 Pi 实例（含 machineId、label、cwd、hostname、pid、transport、sessionId）
POST /v1/pis/{id}/heartbeatPOSTPi 实例心跳
POST /v1/pis/{id}/disconnectPOSTPi 实例断开连接

心跳重试策略：

  • 404 错误：连续 3 次后触发自动重注册（NOT_FOUND_RETRY_THRESHOLD = 3）
  • 其他错误：指数退避重试，间隔 = min(30000, 1000 * 2^(n-1) + random(0, 25%))
  • 最大退避间隔：30 秒（HEARTBEAT_BACKOFF_MAX_MS）
  • 重注册时先重新注册机器，再逐个重注册其下所有在线 Pi 实例

Radius 认证优先级：

  1. ~/.pi/agent/auth.json 中的 OAuth 凭据（由 pi-coding-agent 的 AuthStorage 管理）
  2. 环境变量 PI_RADIUS_API_KEY
