pi-tui 参考文档

本文档是 pi-tui 模块 L1 总览的配套参考资料，提供完整的公共 API 清单、UI 组件目录、源码文件索引和依赖关系详表。建议先在 0001-tui-module-tour.html 完成导览课程，再回到此处查阅细节。

1. 公共 API 完整清单

以下按源码文件分类，列出 src/index.ts 重导出的所有公共符号。

1.1 TUI 核心 — src/tui.ts

导出类型说明
Component接口所有 UI 组件必须实现的接口：render(width)、handleInput?(data)、invalidate()
Container类子组件容器，支持 addChild/removeChild/clear，递归渲染
TUI类差异渲染引擎主类，extends Container。管理 terminal、焦点、Overlay、输入路由
Focusable接口可接收焦点的组件接口，focused: boolean
isFocusable()类型守卫判断组件是否实现 Focusable
CURSOR_MARKER常量光标位置标记（APC 序列），组件在渲染中嵌入以指示 IME 候选窗位置
OverlayAnchor类型Overlay 锚点位置：center、top-left、top-right 等 9 种
OverlayMargin接口Overlay 边距配置：top/right/bottom/left
OverlayOptions接口Overlay 位置/尺寸配置：width、maxHeight、anchor、row、col、margin、visible
OverlayHandle接口Overlay 控制句柄：hide()、setHidden()、focus()、unfocus()
OverlayUnfocusOptions接口unfocus 时的目标组件指定
SizeValue类型尺寸值：绝对数字或百分比字符串如 "50%"

1.2 终端接口 — src/terminal.ts

导出类型说明
Terminal接口终端抽象：start(onInput, onResize)、stop()、write()、rows/columns、光标控制
ProcessTerminal类基于 process.stdin/stdout 的 Terminal 实现，处理 raw mode、鼠标事件、Kitty 键盘协议

1.3 UI 组件 — src/components/

导出类型所属文件说明
Box类box.ts基础容器组件，可包含子组件并用边框包裹
Text类text.ts静态文本显示组件
Editor类editor.ts多行文本编辑器，含撤销栈、剪贴板、自动补全
EditorOptions接口editor.ts编辑器配置：行数、自动补全 Provider、初始文本、自定义补全
EditorTheme接口editor.ts编辑器主题色配置
Input类input.ts单行文本输入组件
Markdown类markdown.tsMarkdown 渲染组件，处理流式不完整标记
MarkdownOptions接口markdown.tsMarkdown 渲染选项
MarkdownTheme接口markdown.tsMarkdown 主题色配置
SelectList类select-list.ts可滚动选择列表
SelectItem类型select-list.ts选择列表项类型
SettingsList类settings-list.ts设置列表（键值对展示）
Loader类loader.ts加载动画指示器
CancellableLoader类cancellable-loader.ts带取消按钮的加载器
Image类image.ts终端图片显示组件（Kitty/iTerm2 协议）
Spacer类spacer.ts垂直空白占位
TruncatedText类truncated-text.ts自动截断的文本组件

1.4 键盘处理 — src/keys.ts

导出类型说明
Key类键对象：name、ctrl、alt、shift、meta
KeyId类型键标识符联合类型
KeyEventType类型键事件类型
parseKey()函数解析键盘输入序列为 Key 对象
matchesKey()函数判断输入序列是否匹配指定快捷键
isKeyRelease()函数判断是否为按键释放事件
isKeyRepeat()函数判断是否为按键重复事件
isKittyProtocolActive()函数检查 Kitty 键盘协议是否已激活
setKittyProtocolActive()函数设置 Kitty 键盘协议激活状态
decodeKittyPrintable()函数解码 Kitty 协议的 printable key 事件

1.5 快捷键绑定 — src/keybindings.ts

导出类型说明
KeybindingsManager类快捷键管理器，支持作用域、冲突检测
TUI_KEYBINDINGS常量TUI 内置快捷键定义符号
getKeybindings()函数获取当前快捷键配置
setKeybindings()函数设置快捷键配置
KeybindingsConfig类型快捷键配置类型
KeybindingDefinition类型单个快捷键定义
KeybindingConflict类型快捷键冲突信息

1.6 工具函数 — src/utils.ts

导出说明
visibleWidth(str)计算字符串在终端中的可见列宽（感知东亚字符宽度）
truncateToWidth(str, width)将字符串截断到指定列宽
sliceByColumn(str, start, end)按列宽切片字符串
wrapTextWithAnsi(str, width)按列宽自动换行（保留 ANSI 转义序列）

来源文件代表导出说明

4. 原生模块

平台文件功能
macOS arm64native/darwin/prebuilds/darwin-arm64/darwin-modifiers.node检测 macOS 修饰键状态（shift/command/control/option），用于判断 Apple Terminal 中的 Shift+Enter
macOS x64native/darwin/prebuilds/darwin-x64/darwin-modifiers.node同上，Intel Mac 版本
Windows arm64native/win32/prebuilds/win32-arm64/Windows on ARM 预留（当前未包含 .node 文件，可能无功能）
Windows x64native/win32/prebuilds/win32-x64/Windows x64 预留（当前未包含 .node 文件）

说明：原生模块是可选功能。当系统不支持或 .node 文件缺失时，isNativeModifierPressed() 静默返回 false，不影响 TUI 正常运行。

5. 构建与测试

命令作用
npm run build使用 tsgo（TypeScript 7.0 原生编译器）编译 tsconfig.build.json
npm run test使用 Node 原生 test runner 运行 test/*.test.ts（dot reporter）
npm run clean删除 dist/ 目录

与其他 pi 包不同，pi-tui 使用 Node 原生测试运行器而非 Vitest。这是因为它更接近系统底层（stdin/stdout、raw mode、ANSI 转义序列），Node 原生工具链更适合测试这些特性。
