0001 — pi-tui 终端 UI 差异渲染库模块总览

层级：L1 模块总览 预计时长：12 分钟 学习目标：1 个

学习目标：理解 pi-tui 的模块职责、架构位置、对外接口和内部分层，能解释差异渲染的核心原理。

1. 模块职责与架构位置

pi-tui（npm 名 @earendil-works/pi-tui）是 Pi Agent Harness 的终端 UI 差异渲染库。 它不是简单的 console.log 封装，而是一个完整的终端 UI 框架，提供：

  • 差异渲染（differential rendering）：每次只更新屏幕上变化的部分，避免全屏刷新带来的闪烁
  • Markdown 解析：基于 marked 18.x，将 LLM 输出的 Markdown 实时转为终端可渲染的行文本
  • 东亚字符宽度感知布局：基于 get-east-asian-width 正确计算 CJK 字符占 2 列宽度
  • 文本编辑器组件：多行编辑，支持撤销/重做、选区、自动补全

在 Pi monorepo 中，pi-tui 处于最底层，与 pi-ai 并列，不依赖任何其他 pi 包：

pi-coding-agent ────→ pi-tui  (依赖关系)
pi-orchestrator ──→ pi-coding-agent

pi-tui 仅依赖外部 npm 包：
  marked (Markdown 解析) + get-east-asian-width (字符宽度)

分层逻辑：

  1. src/ 核心层（tui.ts、terminal.ts、keys.ts、keybindings.ts）提供终端抽象、差异渲染引擎、键盘输入处理
  2. src/components/ 组件层基于 Component 接口实现可组合的 UI 控件，通过 Container 树形挂载
  3. native/ 原生层提供平台特定能力（macOS 修饰键检测），以预编译 .node 形式分发，非必需功能

4. 差异渲染原理

差异渲染是 pi-tui 的核心特性。其核心思想是：每帧渲染后，将新内容与上一帧逐行比较，只把差异区域写入终端，避免全屏清空重绘（\x1b[2J）带来的视觉闪烁。

核心数据结构

TUI 类维护两个关键状态：

  • previousLines: string[] — 上一帧的渲染结果，每行一个字符串（含 ANSI 转义序列）
  • previousWidth / previousHeight — 上一帧的终端尺寸

渲染流程（doRender 方法）

精简后的核心逻辑如下（原方法约 400 行）：

private doRender(): void {
  const width = this.terminal.columns;
  const height = this.terminal.rows;

  // 1. 渲染所有组件，获得新行数组
  let newLines = this.render(width);

  // 2. 首帧或尺寸变化 → 全量渲染
  if (this.previousLines.length === 0 || widthChanged) {
    fullRender();  // \x1b[2J 清屏 + 逐行输出
    return;
  }

  // 3. 找到第一个和最后一个变化的行
  let firstChanged = -1, lastChanged = -1;
  for (let i = 0; i < maxLines; i++) {
    if (oldLine !== newLine) {
      if (firstChanged === -1) firstChanged = i;
      lastChanged = i;
    }
  }

  // 4. 无变化 → 跳过
  if (firstChanged === -1) return;

  // 5. 只重绘变化区域
  //    - 移动光标到 firstChanged 行
  //    - 从 firstChanged 到 lastChanged 逐行输出（\r\x1b[2K + 行内容）
  //    - 如果新内容行数更少，清除多余行
  differentialRender(firstChanged, lastChanged);
}

关键决策点：

  • 终端宽度变化（如用户调整窗口大小）→ 强制全量重绘，因为换行位置全部改变
  • 变化区域溢出视口（firstChanged 在屏幕上方不可见区域）→ 降级为全量重绘
  • 追加模式（新内容仅追加在原内容末尾）→ 优化为从 appendStart 位置直接输出
  • 16ms 最小渲染间隔（MIN_RENDER_INTERVAL_MS）→ 防止高频渲染（如流式输出）导致 CPU 飙升

同步输出保护

所有终端写入包裹在 \x1b[?2026h（开始）和 \x1b[?2026l（结束）之间，这是终端的同步更新协议（synchronized output），确保终端在收到结束标记前不刷新屏幕，防止中间状态闪烁。

4.4 调用示例

pi-tui 的最小使用示例——创建 TUI 实例并挂载组件：

import { TUI, Container, Text, Box } from "@earendil-works/pi-tui";

// 1. 创建 TUI 实例（绑定终端流）
const tui = new TUI({ stdout: process.stdout, stdin: process.stdin });

// 2. 构建组件树
const root = new Container();
root.addChild(new Text("Hello, Pi!"));

// 3. 启动渲染循环
tui.start(root);

关键点：TUI 构造函数接受 {stdout, stdin} 终端流；Container 是组件树的根节点，通过 addChild() 挂载子组件；tui.start() 启动事件循环和差异渲染。
