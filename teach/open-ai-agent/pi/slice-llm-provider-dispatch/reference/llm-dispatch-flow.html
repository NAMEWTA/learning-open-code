Models 集合

interface Models {
  getProviders(): readonly Provider[];
  getProvider(id: string): Provider | undefined;
  getModels(provider?: string): readonly Model<Api>[];
  getModel(provider: string, id: string): Model<Api> | undefined;

  streamSimple(model, context, options?): AssistantMessageEventStream;

Provider 接口

interface Provider<TApi extends Api = Api> {
  readonly id: string;
  readonly name: string;
  readonly baseUrl?: string;
  readonly auth: ProviderAuth;
  getModels(): readonly Model<TApi>[];
  refreshModels?(): Promise<void>;
  stream(model, context, options?): AssistantMessageEventStream;
  streamSimple(model, context, options?): AssistantMessageEventStream;
}

createProvider 输入

interface CreateProviderOptions<TApi> {

3. ModelsError 错误码对照表

错误码触发条件调用方处理建议
"provider" requireProvider() 找不到 model.provider 对应的 Provider 检查 Provider id 是否正确注册；非内置 Provider 需先 setProvider()
"auth" API Key 解析失败（resolveApiKey 内部异常）或 credential store 读写失败 提示用户重新登录；检查 credential store 后端是否可用
"oauth" OAuth token 刷新失败（refresh 调用异常）或 toAuth 转换失败 提示用户重新授权（re-login）；旧凭证保留供重试
"model_source" Provider.refreshModels() 的网络请求失败 模型列表保持在旧状态；可稍后重试 refresh()
"stream" Provider 没有注册 model.api 对应的 API 实现 检查 Provider 的 api 分发表是否包含该 model.api

4. Provider API 分发表

每个 Provider 通过 api 字段声明支持的 API 协议。当一个 Provider 支持多种 API 时，按 model.api 分发：

ProviderAPI 协议懒加载入口SDK / HTTP 库
openai openai-responses api/openai-responses.lazy.ts openai (npm)
anthropic anthropic-messages api/anthropic-messages.lazy.ts @anthropic-ai/sdk
google google-generative-ai api/google-generative-ai.lazy.ts @google/generative-ai
google-vertex google-vertex api/google-vertex.lazy.ts @google-cloud/vertexai
mistral mistral-conversations api/mistral-conversations.lazy.ts @mistralai/mistralai
azure-openai azure-openai-responses api/azure-openai-responses.lazy.ts openai (npm)
amazon-bedrock bedrock-converse-stream api/bedrock-converse-stream.lazy.ts @aws-sdk/client-bedrock-runtime
openai-codex openai-codex-responses api/openai-codex-responses.lazy.ts openai (npm)
openrouter openai-completions api/openai-completions.lazy.ts openai (npm)

设计要点：每个 API 模块都通过 *.lazy.ts 包装为 lazyApi(() => import("./xxx.ts"))，这意味着 SDK 代码仅在第一次 stream 调用时才加载。对于只使用 OpenAI 的用户，Anthropic SDK 永远不会被加载到内存中。

5. 认证来源决策树

resolveProviderAuth(provider, model, credentials, ctx, overrides?)
│
├─ overrides.apiKey 存在？ ──是──▶ resolveApiKey(with override key)
│                                    └─ 成功 → AuthResult
│                                    └─ 失败 → throw ModelsError("auth")
│
├─ credentialStore.read(providerId)
│  ├─ 已存储 credential
│  │  ├─ type="oauth" ──▶ resolveStoredOAuth
│  │  │                    ├─ 未过期 → oauth.toAuth(cred) → AuthResult
│  │  │                    └─ 已过期 → store.modify() 加锁刷新
│  │  │                                 ├─ 刷新成功 → 持久化 → AuthResult
│  │  │                                 └─ 刷新失败 → throw ModelsError("oauth")
│  │  └─ type="api_key" ──▶ resolveApiKey(with stored key)
│  │                          └─ 成功 → AuthResult
│  │
│  └─ 无存储 ──▶ 进入 ambient 分支
│
└─ ambient（环境变量 / AWS profile / ADC 文件）
   └─ provider.auth.apiKey.resolve({credential: undefined})
        ├─ env var 存在 → AuthResult (source: env var name)
        └─ env var 不存在 → undefined (Provider 未配置，非错误)

重要约束：一旦用户存储了凭证（通过 login 流程），就不会再 fallback 到环境变量。如果存储的 OAuth token 刷新失败，旧凭证保留供重试，不会静默切换。
