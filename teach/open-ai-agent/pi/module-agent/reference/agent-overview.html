pi-agent-core 参考文档

本文档是 pi-agent-core 模块 L1 总览的配套参考资料，提供 Agent 类完整 API、AgentEvent 事件类型、AgentLoopConfig 配置项等速查信息。建议先在 0001-agent-module-tour.html 完成模块导览课程，再回到此处查阅细节。

1. Agent 类 API 速查

方法/属性签名说明
prompt() (text: string, images?: ImageContent[]) => Promise<void>
(message: AgentMessage | AgentMessage[]) => Promise<void>		 启动新对话轮次。若 Agent 正在运行则抛出错误。支持文本 + 图片或完整消息数组。
continue() () => Promise<void> 从当前对话末尾继续。最后一条消息必须是 user 或 toolResult 角色。若最后是 assistant，则先 drain steering/follow-up 队列。
steer() (message: AgentMessage) => void 将消息加入 steering 队列。在当前 assistant 回复（含工具调用）完成后注入。
followUp() (message: AgentMessage) => void 将消息加入 follow-up 队列。仅在 Agent 即将停止（无更多工具调用和 steering 消息）时注入。
subscribe() (listener: (event, signal) => void) => () => void 订阅生命周期事件。返回取消订阅函数。监听器按订阅顺序 await，包含在运行结算中。
abort() () => void 中断当前运行。触发 abort signal，导致工具执行终止和流式响应取消。
waitForIdle() () => Promise<void> 等待当前运行及所有 agent_end 监听器完成。无运行中任务时立即 resolve。
reset() () => void 清空对话历史、流式状态、pendingToolCalls、steering 和 follow-up 队列。
clearSteeringQueue() () => void 清空 steering 队列。
clearFollowUpQueue() () => void 清空 follow-up 队列。
hasQueuedMessages() () => boolean 任一队列中有待处理消息时返回 true。
state AgentState（只读属性） 当前状态快照：systemPrompt、model、thinkingLevel、tools、messages、isStreaming、streamingMessage、pendingToolCalls、errorMessage。
signal AbortSignal | undefined 当前运行的 abort signal。无运行中任务时为 undefined。
steeringMode QueueMode（get/set） steering 队列消费模式："all" 一次 drain 全部，"one-at-a-time" 每次 drain 一条。
followUpMode QueueMode（get/set） follow-up 队列消费模式，同 steeringMode。

3. AgentEvent 事件类型详表

3.1 Agent 生命周期

事件类型携带数据触发时机
agent_start无Agent 循环开始
agent_endmessages: AgentMessage[]Agent 循环结束（最后事件）

3.2 Turn 生命周期

事件类型携带数据触发时机
turn_start无新一轮对话开始（一个 assistant 回复 + 工具调用 = 一个 turn）
turn_endmessage、toolResults一轮对话结束

3.3 消息生命周期

事件类型携带数据触发时机
message_startmessage: AgentMessage新消息开始（user、assistant、toolResult 均触发）
message_updatemessage、assistantMessageEvent流式响应中新增内容块（仅 assistant 消息）
message_endmessage: AgentMessage消息完成

3.4 工具执行生命周期

事件类型携带数据触发时机
tool_execution_starttoolCallId、toolName、args工具开始执行
tool_execution_updatetoolCallId、toolName、partialResult工具执行中产生部分结果
tool_execution_endtoolCallId、toolName、result、isError工具执行完成

4. AgentLoopConfig 核心钩子

钩子签名说明
shouldStopAfterTurn(context) => boolean每轮结束后判断是否应停止（如上下文接近限制）。返回 true 则发出 agent_end 并退出。
prepareNextTurn(context) => AgentLoopTurnUpdate每轮结束后修改下一轮的 context/model/thinking 状态。
getSteeringMessages() => AgentMessage[]获取 steering 消息。在当前 assistant 回复的工具调用完成后调用。
getFollowUpMessages() => AgentMessage[]获取 follow-up 消息。在 Agent 即将停止时调用。
beforeToolCall(context, signal?) => BeforeToolCallResult工具执行前调用。返回 { block: true } 阻止执行。
afterToolCall(context, signal?) => AfterToolCallResult工具执行后调用。可覆写 content、details、isError、terminate。
convertToLlm(AgentMessage[]) => Message[]将 AgentMessage[] 转为 LLM 可识别的 Message[]。必须不抛异常。
transformContext(AgentMessage[], signal?) => AgentMessage[]在 convertToLlm 前修改上下文。用于上下文窗口管理。
