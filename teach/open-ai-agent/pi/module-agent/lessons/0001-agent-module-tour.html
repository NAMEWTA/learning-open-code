0001 — pi-agent-core 模块总览

层级：L1 模块总览 预计时长：12 分钟 学习目标：1 个
学习目标

理解 pi-agent-core 的内部分层架构，能说出其核心职责、对外导出接口、以及 Agent 主循环的双层控制流设计。完成本课后，你可以独立创建一个 Agent 实例并启动一轮对话。

1. 模块定位与职责

pi-agent-core（npm 包名 @earendil-works/pi-agent-core）是 Pi 的 Agent 运行时层。它在整个 monorepo 架构中处于中间层位置：

pi-agent-core 的四大核心职责：

  1. 传输抽象 — 通过 Transport 类型和 StreamFn 流函数，将 Agent 循环与具体 LLM 通信协议解耦。
  2. 状态管理 — AgentState 维护会话消息、工具列表、推理级别、流式状态等运行时快照。
  3. 工具调用 — 完整的工具生命周期：参数校验、前置钩子（beforeToolCall）、执行（串行/并行）、后置钩子（afterToolCall）。
  4. 消息队列 — 内置 steering 和 follow-up 双队列，支持在 Agent 运行时动态注入消息。

pi-agent-core 本身不绑定任何具体工具或 UI。它是一个通用框架，pi-coding-agent 在其上注册了 read/write/bash/edit 等编码工具，pi-orchestrator 用它编排多 Agent 协作。

3. 内部分层结构

packages/agent/src/ ├── index.ts ← 主入口，汇总所有公开导出 ├── types.ts ← 核心类型定义（AgentEvent、AgentTool 等） ├── agent.ts ← Agent 类（状态管理 + 生命周期） ├── agent-loop.ts ← Agent 主循环（双层循环控制） ├── node.ts ← Node.js 环境入口 └── harness/ ← 运行时基础设施 ├── messages.ts ← 消息格式（bash、自定义、压缩摘要） ├── prompt-templates.ts ← 提示词模板引擎 ├── system-prompt.ts ← 系统提示词构建 ├── skills.ts ← 技能加载与格式化 ├── types.ts ← Harness 类型定义 ├── env/ │ └── nodejs.ts ← Node.js 文件系统/Shell 适配 ├── session/ ← 会话持久化 │ ├── session.ts ← 会话树构建 │ ├── jsonl-repo.ts ← JSONL 文件仓库 │ ├── jsonl-storage.ts ← JSONL 存储引擎 │ ├── memory-repo.ts ← 内存仓库（测试/临时用） │ ├── memory-storage.ts ← 内存存储引擎 │ ├── repo-utils.ts ← 仓库工具函数 │ └── uuid.ts ← UUID v7 生成 ├── compaction/ ← 上下文压缩 │ ├── compaction.ts ← 压缩算法核心 │ ├── branch-summarization.ts ← 分支摘要生成 │ └── utils.ts ← 压缩工具函数 └── utils/ ├── shell-output.ts ← Shell 输出处理 └── truncate.ts ← 文本截断工具

分层解读（自顶向下）：

  1. agent.ts + agent-loop.ts — 顶层。Agent 类封装面向用户的 API（prompt、continue、steer、subscribe），agent-loop.ts 实现双层循环控制流。
  2. harness/messages.ts — 消息格式层。定义 AgentMessage 的扩展类型（bashExecution、custom、compactionSummary 等），通过 TypeScript 声明合并注入类型系统。
  3. harness/session/ — 持久化层。Session 类将 AgentMessage 树状化存储，支持 fork、merge、compaction 条目。提供 JSONL 和内存两种后端。
  4. harness/compaction/ — 压缩层。当上下文超过 token 预算时，自动摘要旧消息并注入压缩摘要消息，保持对话连续性。
  5. harness/env/nodejs.ts — 适配层。将 Node.js 的 fs 和 child_process 封装为 FileSystem 和 Shell 接口，使 harness 可在不同运行时（Node.js / 浏览器 / 测试 mock）中复用。

4. 创建并运行一个 Agent

以下是最简示例——创建一个 Agent，设置系统提示词，发送一条消息并监听事件：

import { Agent } from "@earendil-works/pi-agent-core";
import { models } from "@earendil-works/pi-ai";

// 1. 创建 Agent 实例
const agent = new Agent({
  initialState: {
    systemPrompt: "你是一个乐于助人的助手。",
    model: models.get("anthropic", "claude-sonnet-4-5"),
    tools: [],
  },
});

// 2. 订阅事件（可选，用于监听消息流）
agent.subscribe(async (event) => {
  if (event.type === "message_update") {
    const text = event.message.content
      .filter(c => c.type === "text")
      .map(c => c.text).join("");
    process.stdout.write(text);
  }
});

// 3. 发送第一条提示词
await agent.prompt("你好，请介绍一下你自己。");

// 4. 等待 Agent 完成
await agent.waitForIdle();

// 5. 查看对话历史
console.log(`共 ${agent.state.messages.length} 条消息`);

关键 API 速查：

  • agent.prompt(text) — 启动新对话轮次，传入文本或消息数组。
  • agent.continue() — 从当前对话末尾继续，用于注入工具结果后恢复。
  • agent.steer(message) — 在当前 assistant 回复完成后注入一条消息。
  • agent.followUp(message) — 在 Agent 即将停止时注入一条跟进消息。
  • agent.subscribe(fn) — 订阅生命周期事件（返回取消订阅函数）。
  • agent.abort() — 中断当前运行。
  • agent.reset() — 清空对话历史和队列。
  • agent.state — 只读状态快照（messages、tools、isStreaming 等）。

检索练习

不看上文，回答以下问题：

  1. pi-agent-core 的四大核心职责分别是什么？
  2. Agent 主循环的双层结构是如何组织的（外层/内层各处理什么）？
  3. steer() 和 followUp() 的区别是什么？
  4. harness 子目录中，session/ 和 compaction/ 各负责什么？
点击查看参考答案
  1. 传输抽象、状态管理、工具调用、消息队列（steering + follow-up）。
  2. 外层循环处理 follow-up 消息（Agent 即将停止时注入）；内层循环处理 tool-call 执行和 steering 消息（当前 assistant 回复完成后注入）。
  3. steer() 在当前 assistant 回复完成后立即注入（不等待 Agent 停止），followUp() 在 Agent 即将停止（无更多工具调用和 steering 消息）时才注入。
  4. session/ 负责会话持久化（树状存储、fork、JSONL/内存后端），compaction/ 负责上下文压缩（token 预算管理、旧消息摘要）。
