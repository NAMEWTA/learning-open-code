一、一句话定位

Hermes Agent 是 Nous Research 打造的自进化 AI Agent 框架——唯一内置完整学习闭环的 Agent：从经验创建技能、在使用中改进技能、跨会话持久化记忆。
Python 3.11+ MIT 协议 自进化学习 20+ 消息平台 六种终端后端 子代理并行

它可在云端 VM 或 $5 VPS 上运行，通过 Telegram 远程操控。支持 OpenAI、Anthropic、OpenRouter、Nous Portal 等多模型提供商，hermes model 一行切换。内置 cron 调度器支持自然语言定时任务，结果推送至任意消息平台。

核心原则：窄腰模型。 每个模型工具随每次 API 调用发送，新增即为永久成本。多数能力通过 CLI 命令+技能、门控工具、插件或 MCP 服务器在边缘实现——这称为 Footprint Ladder（足迹阶梯）。

入口层（cli.py/run_agent.py/ui-tui/web）接收输入；网关层（gateway/hermes_cli）路由到核心；核心层（agent/）对话循环与学习；能力层（tools/providers/plugins/skills）提供工具和模型；支撑层（hermes_state/cron/docker/locales）持久化与部署。

四、设计哲学与源码起点

五条核心设计原则

  1. 自进化学习闭环——Curator 在复杂任务后自动创建技能；技能调用中收集反馈并自我改进。跨会话记忆通过 Honcho 和 FTS5 全文搜索持久化。
  2. 窄腰架构——每个模型工具随每次 API 调用发送，新核心工具的添加门槛极高。能力通过 Footprint Ladder 优先在边缘实现：扩展现有代码 > CLI 命令+技能 > 服务门控工具 > 插件 > MCP 服务器 > 新核心工具。
  3. 跨平台统一核心——同一套 agent/ 服务于 CLI、TUI、Web、Electron 和 20+ 消息平台。Telegram 消息与 CLI 消息经过同一个 conversation_loop。
  4. 子代理并行与委托——通过 delegate_tool 派生隔离子代理执行并行工作流。子代理拥有独立上下文窗口，结果通过结构化协议回传主代理。
  5. 精确依赖锁定——所有核心依赖锁定精确版本（如 openai==2.24.0），杜绝版本范围引入恶意包。2026 年 Mini Shai-Hulud 蠕虫事件后强制执行。
