一、在整体架构中的位置

agent/ 是 Hermes Agent 的中央引擎——所有用户入口（CLI、TUI、Web、Desktop、20+ 消息平台）最终都汇聚到这里，由它驱动一次完整的对话生命周期：接收用户消息 → 构建提示词 → 调用模型 → 解析工具调用 → 执行工具 → 将结果回传模型 → 循环直至模型输出纯文本回复。
Python 3.11+ 110 个 .py 文件 4.1 MB 从 run_agent.py 拆分而来

在 L0 的分层架构中，agent/ 位于网关层之下、能力层（tools/providers）之上，是架构图中承上启下的中枢：

  用户入口 (CLI / TUI / Web / Desktop / API)
      │
      ▼
  网关层 (hermes_cli / gateway / tui_gateway)
      │
      ▼
  ┌─────────────────────────────────────────┐
  │         ★ agent/  (Agent 核心)          │
  │  conversation_loop  tool_executor       │
  │  context_compressor memory_manager      │
  │  prompt_builder      curator            │
  │  agent_init          auxiliary_client   │
  │  chat_completion_helpers                │
  └──────┬──────────────────────┬───────────┘
         │                      │
         ▼                      ▼
  providers/ (模型)      tools/ (终端/浏览器/文件/委托...)

二、内部四层结构

agent/ 110 个模块并非杂乱堆放，而是按职责自然形成四个子层。核心文件如下：

agent/ ← Agent 核心引擎（110 个 .py，4.1 MB） ├── agent_init.py ← 初始化层：AIAgent 构造（103 KB，60+ 参数） ├── prompt_builder.py ← 提示词构建：系统提示词拼装（94 KB） ├── conversation_loop.py ← 对话循环层：主控循环 run_conversation()（295 KB） ├── tool_executor.py ← 工具执行层：串行/并行工具调度（79 KB） ├── chat_completion_helpers.py ← 模型适配层：LLM API 调用与容错（147 KB） ├── auxiliary_client.py ← 辅助客户端：子任务 LLM 路由（315 KB） ├── context_compressor.py ← 上下文压缩层：轨迹压缩与摘要（149 KB） ├── memory_manager.py ← 记忆管理层：MemoryProvider 编排（43 KB） ├── curator.py ← 学习策展层：自动技能维护（83 KB） └── agent_runtime_helpers.py ← 运行时辅助：消息修复/轨迹转换（143 KB）

数据流向：用户消息 → agent_init 构建 AIAgent 实例 → prompt_builder 组装系统提示词 → conversation_loop 进入对话主循环 → chat_completion_helpers 调用 LLM API → tool_executor 执行模型请求的工具 → 结果回传，循环直至完成 → context_compressor 在上下文超限时压缩历史 → memory_manager 在回合间同步记忆。

子层核心文件职责
对话循环层conversation_loop.py驱动一次完整对话回合：模型调用 → 工具分发 → 重试/降级 → 压缩触发 → 后处理
工具执行层tool_executor.py串行/并发调度模型请求的工具调用，处理权限守卫、预算控制、结果持久化
记忆管理层memory_manager.py编排 MemoryProvider 插件，回合前后拉取/同步外部记忆，注入系统提示词
上下文压缩层context_compressor.py在 token 预算内自动压缩对话历史，使用辅助模型生成结构化摘要

三、关键公共接口

以下是 agent/ 对外暴露的核心入口，被 CLI、gateway、TUI 等上层消费：

1. 对话主循环入口

# agent/conversation_loop.py — run_conversation()
def run_conversation(
    agent,                     # AIAgent 实例（由 agent_init 构建）
    user_message: str,         # 用户消息文本
    system_message: str = None,# 自定义系统提示词（可选）
    conversation_history: List[Dict] = None,  # 历史消息
    task_id: str = None,       # 任务隔离 ID
) -> Dict[str, Any]:
    """运行一次完整的对话（含工具调用循环），返回最终响应和消息历史。"""

2. 工具执行引擎

# agent/tool_executor.py — 串行与并发执行
# 两个模块级函数，均以 AIAgent 为第一参数：
def _execute_tool_calls_sequential(agent, ...)  # 串行逐工具执行
def _execute_tool_calls_concurrent(agent, ...)  # 最多 8 线程并发执行

3. 上下文压缩器

# agent/context_compressor.py — ContextCompressor 类
class ContextCompressor:
    """自动上下文窗口压缩。使用辅助模型摘要中间回合，
       保护 head（系统提示词）和 tail（最近消息）。"""

    def maybe_compress(self, messages, ...)  # 按阈值触发压缩
    def compress(self, messages, ...)        # 执行一轮压缩
    def compress_trajectory(self, ...)       # 轨迹文件压缩

四、依赖关系与调用示例

上依赖（谁依赖 agent/）

  • cli.py — CLI 入口，调用 run_conversation()
  • run_agent.py — 库模式入口，持有 AIAgent 类（其 __init__ 实现在 agent_init.py）
  • gateway/ — 消息网关，每个平台适配器最终调用同一个 run_conversation()
  • tui_gateway/ — TUI 网关层

下依赖（agent/ 依赖谁）

  • providers/ — 模型提供商（OpenAI / Anthropic / Gemini 等）
  • tools/ — 工具注册与执行（terminal_tool, browser_tool 等）
  • hermes_state.py — SQLite 会话状态持久化
  • hermes_constants.py — 全局常量（零依赖）
  • plugins/ — 插件系统（browser、memory、kanban 等）

最小调用示例

# 库模式使用 Hermes Agent 核心的最简示例
from run_agent import AIAgent

agent = AIAgent(
    model="gpt-4o",              # 使用的模型
    provider="openai",           # 模型提供商
    system_prompt="你是一个 Python 专家助手。",
)
result = agent.run_conversation("如何用 asyncio 实现并发 HTTP 请求？")
print(result["content"])         # 模型的最终文本回复

仅此 5 行代码就触发了完整的 agent/ 管线：agent_init 初始化 → prompt_builder 构建提示词 → conversation_loop 驱动对话 → chat_completion_helpers 调用 API → tool_executor 执行模型请求的工具 → context_compressor 按需压缩。

  2. run_conversation() 函数的一次调用会触发哪几个 agent/ 子模块的协作？请按顺序列出。
  3. 如果一个对话回合中模型请求调用 3 个工具，tool_executor.py 如何执行它们？
点击查看参考答案

1. 对话循环层（conversation_loop.py）、工具执行层（tool_executor.py）、记忆管理层（memory_manager.py）、上下文压缩层（context_compressor.py）。

2. agent_init.py（构建实例）→ prompt_builder.py（组装提示词）→ conversation_loop.py（主循环）→ chat_completion_helpers.py（调用 LLM）→ tool_executor.py（执行工具）→ context_compressor.py（按需压缩）。

3. 默认串行执行（_execute_tool_calls_sequential），也可切换到并发模式（_execute_tool_calls_concurrent），最多 8 个工作线程。无论哪种模式，结果都组装为 tool result 消息回传给模型。
