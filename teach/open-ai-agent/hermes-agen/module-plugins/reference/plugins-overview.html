3. 生命周期 Hook 完整清单

Hook 名称触发阶段返回值作用典型用途
pre_tool_call工具调用前block/rewrite 指令安全策略、速率限制、审批
post_tool_call工具调用后观察，返回值忽略日志、临时文件追踪 (disk-cleanup)
transform_terminal_output终端输出后返回字符串替换输出输出脱敏、格式化
transform_tool_result工具结果返回后返回字符串替换结果结果过滤、增强
transform_llm_outputLLM 响应后返回字符串替换响应词汇/人格转换
pre_llm_callLLM 调用前注入上下文到用户消息记忆召回注入
post_llm_callLLM 调用后观察，返回值忽略遥测、日志
pre_verify编辑代码后准备停止continue/block 指令代码检查、diff 整理
pre_api_requestAPI 请求前观察请求日志、请求修改（通过 middleware）
post_api_requestAPI 请求后观察响应日志、计费追踪
api_request_errorAPI 请求失败观察错误告警、重试策略
on_session_start会话开始观察会话初始化、欢迎消息
on_session_end会话结束观察清理资源 (disk-cleanup)
on_session_finalize会话最终化观察数据持久化
on_session_reset会话重置观察状态清理
subagent_start子 Agent 启动观察子任务监控
subagent_stop子 Agent 停止观察子任务完成通知
pre_gateway_dispatchGateway 消息分发前skip/rewrite/allow 指令消息过滤、内容改写
pre_approval_request审批请求前观察审批审计日志
post_approval_response审批响应后观察审批结果记录
kanban_task_claimed看板任务领取观察任务分配通知
kanban_task_completed看板任务完成观察任务完成通知
kanban_task_blocked看板任务阻塞观察阻塞告警

4. PluginContext 注册方法全表

方法对接子系统说明
register_tool(name, handler, ...)tools.registry注册 Agent 可调用的工具函数，支持 override
register_hook(hook_name, callback)PluginManager._hooks注册生命周期 Hook 回调
register_middleware(kind, callback)PluginManager._middleware注册行为改写中间件
register_cli_command(name, setup_fn, ...)argparse 子命令注册 hermes <name> CLI 子命令
register_command(name, handler, ...)会话内 Slash 命令注册 /name 会话内命令
register_image_gen_provider(provider)agent.image_gen_registry注册图像生成后端（ImageGenProvider 实例）
register_web_search_provider(provider)agent.web_search_registry注册网络搜索/提取后端
register_browser_provider(provider)agent.browser_registry注册云浏览器后端
register_video_gen_provider(provider)agent.video_gen_registry注册视频生成后端
register_tts_provider(provider)agent.tts_registry注册文字转语音后端
register_transcription_provider(provider)agent.transcription_registry注册语音转文字后端
register_dashboard_auth_provider(provider)hermes_cli.dashboard_auth注册 Dashboard 认证提供者
register_context_engine(engine)agent.context_engine替换内置 ContextCompressor（仅限一个）
register_secret_source(source)agent.secret_sources.registry注册外部密钥源后端
register_platform(name, adapter_factory, ...)gateway.platform_registry注册 Gateway 平台适配器
register_slack_action_handler(id, callback)Slack Bolt App注册 Slack 交互回调
register_skill(name, path, ...)PluginManager._plugin_skills注册插件提供的只读 Skill
register_auxiliary_task(key, ...)PluginManager._aux_tasks注册插件定义的辅助 LLM 任务
inject_message(content)会话消息队列从外部向活跃会话注入消息
dispatch_tool(name, args)tools.registry从插件内部调度工具调用
llm（属性）agent.plugin_llm受信任插件的 LLM 访问外观
profile_name（属性）hermes_cli.profiles返回当前活跃的 Hermes profile 名称

5. 模块间依赖关系

hermes_cli/plugins.py  (PluginManager + PluginContext)
│
├── 依赖上游
│   ├── hermes_cli/config.py        ← 读取 plugins.enabled / plugins.disabled
│   ├── hermes_constants.py         ← get_hermes_home() 解析用户插件路径
│   ├── tools/registry.py           ← register_tool() 委托给工具注册表
│   ├── utils.py                    ← env_var_enabled(), fast_safe_load()
│   └── agent/*/registry.py         ← 各类 Provider 注册（image_gen、web_search 等）
│
├── 扫描目录（非代码依赖）
│   ├── plugins/  ← bundled（177 .py, 8.1MB）
│   ├── ~/.hermes/plugins/  ← user
│   ├── ./.hermes/plugins/  ← project（需环境变量）
│   └── pip entry_points  ← "hermes_agent.plugins" group
│
└── 被下游消费
    ├── cli.py                         ← discover_plugins() 启动时调用
    ├── hermes_cli/main.py             ← 读取 _plugin_commands、_aux_tasks
    ├── gateway/run.py                 ← invoke_hook("pre_gateway_dispatch")
    ├── agent/conversation_loop.py     ← invoke_hook("pre_llm_call" / "post_llm_call")
    ├── tools/（各类 tool）            ← invoke_hook("pre_tool_call" / "post_tool_call")
    ├── hermes_cli/kanban_db.py        ← invoke_hook("kanban_task_*")
    └── cron/scheduler.py              ← invoke_hook("pre_gateway_dispatch")

6. plugin.yaml 标准字段

字段类型必需说明
namestring是插件名称，用于 plugin.enabled 匹配
versionstring否语义版本号
descriptionstring否简短描述，显示在 hermes plugins list
authorstring否作者标识
kindstring否standalone / backend / exclusive / platform / model-provider（默认 standalone）
requires_envlist否所需环境变量列表（如 OPENAI_API_KEY）
provides_toolslist否声明提供的工具名称列表
provides_hookslist否声明使用的 Hook 列表
