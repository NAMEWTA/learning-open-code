# 使命：Hermes Agent 工具层

## 为什么
理解 Agent 如何与操作系统、浏览器、外部服务交互，是掌握 AI Agent 框架设计的核心环节。Hermes Agent 的工具层是它的"手和眼"——111 个 Python 文件、4.4MB 代码量，覆盖终端执行、浏览器自动化、子代理委托、命令审批等完整的工具生命周期。通过剖析这个模块，开发者能理解生产级 Agent 工具系统的分层设计、安全模型和可扩展架构。

## 成功的样子
- 能画出工具层的分层架构图（注册中心 → 蓝图 → 终端/浏览器/委托 → 审批闸门）
- 能解释 `registry.register()` 自注册机制和 `discover_builtin_tools()` 自动发现原理
- 能说出终端工具、浏览器工具、委托工具、审批系统四大子系统的职责和交互关系
- 能定位任意工具模块在源码中的位置并理解其与 Agent 核心的调用链路

## 约束条件
- 单次课程控制在 15 分钟以内
- 聚焦架构层面，不深入单个工具的完整实现细节
- 以中文输出，代码标识符保留英文

## 不在范围内
- 每个工具模块的完整 API 逐行解析（由后续专项课程覆盖）
- MCP 协议工具的完整协议栈解读
- 技能系统（skills_hub）的完整技能生命周期（由 module-skills 覆盖）
- 终端后端（Docker、Modal、SSH）的运维部署细节
