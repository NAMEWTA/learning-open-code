4. 安全防护机制

4.1 网关生命周期阻断（lifecycle_guard.py）

在 create_job 时扫描 prompt + script，正则匹配 4 种命令分支：

# Branch A: hermes gateway restart/stop
hermes gateway restart
# Branch B: launchctl (macOS)
launchctl kickstart ai.hermes.gateway
# Branch C: systemctl (Linux)
systemctl restart hermes-gateway
# Branch D: pkill / kill
pkill hermes gateway

阻断后会抛出 GatewayLifecycleBlocked(ValueError)，CLI 打印红色错误并 exit 1，agent cronjob tool 返回 tool error。

4.2 禁止的工具集

_resolve_cron_disabled_toolsets() 强制禁用三个工具集：

  • cronjob — 防止 cron agent 再创建更多 cron job（递归）
  • messaging — 交互式，需要 live gateway session
  • clarify — 交互式，会阻塞等待用户输入

4.3 Prompt Injection 扫描

_build_job_prompt() 组装完整 prompt（含 skill 内容）后，调用 _scan_assembled_cron_prompt() 扫描。若触发 injection 规则，抛出 CronPromptInjectionBlocked，job 被标记为 blocked 而非静默失败。

4.4 Credential 防泄漏

_guard_job_credential_exfil() 在执行前扫描 job 的 prompt + script 内容，检测是否包含 API key 或敏感凭证泄露模式。
