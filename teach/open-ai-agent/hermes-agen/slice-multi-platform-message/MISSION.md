# 使命：多平台消息全链路

## 为什么
理解一条消息从 Telegram/Discord 等外部平台进入 Hermes Agent，经过路由、处理再投递回去的完整链路。目标是掌握网关三层架构的实际运转机制，能够诊断消息丢失、投递失败等线上问题。

## 成功的样子
- 能画出从外部消息到 Agent 响应投递的完整时序图，标注每一层的类和方法
- 能解释投递失败时的四级回退策略，知道每条消息最终会落到哪里
- 能在源码中定位任意一个环节的入口函数，快速排查链路问题

## 约束条件
- 已学完 L1-gateway（模块总览）和 L1-agent-core（Agent 核心引擎），本课为垂直穿线
- 单次课程 15 分钟，不展开逐个适配器实现细节
- 以 Telegram/Discord 为典型平台示例

## 不在范围内
- 各个平台适配器的完整实现细节（属于 L1 模块课程）
- Agent 对话循环内部逻辑（已由 L1-agent-core 覆盖）
- 定时任务的调度与投递（属于 L2-cron-scheduled-task）
- WebSocket Relay 协议细节（属于 reference）
