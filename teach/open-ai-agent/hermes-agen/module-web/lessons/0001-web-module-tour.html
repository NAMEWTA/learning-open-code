一、在整体架构中的定位

web/ 是 Hermes Agent 的图形化管理门户——通过浏览器提供 Agent 配置、API 密钥管理、会话监控、日志分析等完整管理功能。它是 FastAPI 后端服务的 SPA 前端，构建产物直接打包进 Python 发行版。
TypeScript React 19 Vite Tailwind CSS v4 2.4 MB 18 个页面 27 个组件 18 种语言

在 Hermes 的整体架构中，web/ 是众多入口之一，与 CLI、TUI、Desktop 并列：

  用户 (浏览器)
      │
      ▼
  ┌─────────────────────────────────────────┐
  │  web/ (SPA 前端，Vite + React 19)        │
  │  App.tsx → Routes → 18 Pages             │
  │  lib/api.ts → fetchJSON() → /api/*       │
  └──────────────┬──────────────────────────┘
                 │ HTTP (Vite proxy / 同源)
                 ▼
  ┌─────────────────────────────────────────┐
  │  hermes_cli/web_server.py (FastAPI)      │
  │  静态文件服务 → web_dist/ (生产构建)      │
  │  REST API → /api/status, /api/config...  │
  └──────────────┬──────────────────────────┘
                 │
                 ▼
  agent/ + gateway/ + providers/ + tools/ ...

二、前端架构与目录结构

web/ 采用经典的 SPA 分层结构。核心目录如下：

web/ ← Web 仪表盘（2.4 MB） ├── src/ │ ├── main.tsx ← React 入口：BrowserRouter + Provider 层级 │ ├── App.tsx ← 主布局：侧边栏 + 路由表 + 插件槽位（1357 行） │ ├── pages/ ← 18 个页面组件（路由终点） │ ├── components/ ← 27 个可复用业务组件 │ ├── lib/ │ │ ├── api.ts ← API 客户端：所有后端端点的类型化封装 │ │ ├── gatewayClient.ts ← WebSocket 网关客户端 │ │ └── utils.ts ← cn() 工具、timeAgo 等辅助函数 │ ├── contexts/ ← React Context：Profile / SystemActions / PageHeader │ ├── plugins/ ← 仪表盘插件 SDK：注册、槽位、类型定义 │ ├── i18n/ ← 18 种语言翻译文件 │ ├── themes/ ← 主题预设与 CSS 变量管理 │ └── hooks/ ← 自定义 Hook：侧边栏状态、模态框行为 ├── vite.config.ts ← Vite 配置：dev token 注入、/api 代理 ├── package.json └── index.html

数据流从用户交互开始：页面组件通过 api.ts 中的 fetchJSON() 发送请求 → Vite 开发服务器代理 /api/* 到 http://127.0.0.1:9119（FastAPI 后端）→ 响应经 React 状态管理驱动 UI 更新。生产环境下，Vite 构建输出到 ../hermes_cli/web_dist/，由 FastAPI 直接作为静态文件服务。

四、插件扩展与前后端联动

仪表盘插件系统

web/ 内置了一套插件 SDK（src/plugins/），允许外部脚本注册新页面和 UI 槽位：

// plugins/types.ts — PluginManifest 核心类型
export interface PluginManifest {
  name: string;          // 插件唯一标识
  label: string;         // 显示名称
  icon: string;          // Lucide 图标名
  tab: {
    path: string;        // 路由路径，如 "/my-plugin"
    position?: string;   // "end" | "after:skills" | "before:config"
    override?: string;   // 覆盖内置路由，如 "/chat"
    hidden?: boolean;    // 注册路由但不显示导航项
  };
  entry: string;         // 插件 bundle 的 JS 入口 URL
  has_api: boolean;      // 是否有后端 API 扩展
  slots?: string[];      // 声明的 UI 槽位名称
}

插件通过 /api/dashboard/plugins 端点注册，在 App.tsx 中与内置路由合并。插件页面渲染在 PluginPage 组件中，UI 槽位通过 PluginSlot 注入到布局的 5 个固定位置：header-banner、backdrop、pre-main、post-main、overlay。

开发与构建联动

开发时使用两条命令并行运行：python -m hermes_cli.main web --no-open 启动 FastAPI 后端（端口 9119），npm run dev 启动 Vite 前端（端口 5173）。Vite 插件 hermesDevToken() 自动从后端抓取 session token 注入前端，确保 /api 请求通过认证。生产构建 npm run build 输出到 hermes_cli/web_dist/，随 Python 包一起分发。

  1. web/ 模块的 src/ 目录下有哪些一级子目录？各自负责什么？
  2. 仪表盘插件如何注册一个新页面？PluginManifest 中 tab.position 的 "after:skills" 表示什么？
  3. 开发环境下，Vite 前端如何与 FastAPI 后端通信？session token 是如何注入的？
点击查看参考答案

1. pages/（18 个页面组件）、components/（27 个可复用业务组件）、lib/（API 客户端、工具函数）、contexts/（React Context 状态管理）、plugins/（仪表盘插件 SDK）、i18n/（18 种语言翻译）、themes/（主题预设）、hooks/（自定义 Hook）。

2. 插件通过 /api/dashboard/plugins 端点注册 PluginManifest。tab.position 为 "after:skills" 表示该插件的导航项插入到 /skills 路由之后。插件页面由 PluginPage 组件渲染，路由在 App.tsx 的 buildRoutes() 中与内置路由合并。

3. Vite 开发服务器将 /api/* 请求代理到 http://127.0.0.1:9119（FastAPI 后端）。hermesDevToken() 插件在每次页面加载时从后端抓取 index.html，提取 window.__HERMES_SESSION_TOKEN__ 并重新注入到开发 HTML 中，确保前端 API 请求携带正确的认证头。

如有不清楚的概念，随时向 AI 教师追问——教师可展开讲解 api.ts 的请求拦截机制或 PluginSlot 的渲染流程。
