3. 关键接口与通信协议

3.1 API 客户端 (api.ts)

端点族说明需要 Profile Scope
/api/statusAgent 状态（运行中/停止）、Gateway 健康是
/api/config动态配置读写（从后端读取 JSON Schema）是
/api/envAPI 密钥管理（保存/清除/列表）是
/api/model/*模型信息、切换、辅助模型配置是
/api/skillsAgent 技能 CRUD是
/api/tools/toolsets工具集配置是
/api/mcpMCP 服务器管理是
/api/messaging/*消息平台配置（Telegram 等）是
/api/analytics用量统计（Token/成本）是
/api/sessions会话列表、搜索、删除否
/api/cronCron 作业管理否
/api/system/*系统操作（重启、更新、日志）否
/api/dashboard/plugins仪表盘插件注册表否
/ws/terminal终端 WebSocket（ChatPage 内嵌 PTY）否

3.2 认证机制

  • Session Token：服务端注入 window.__HERMES_SESSION_TOKEN__，前端所有 fetch 请求自动携带 X-Hermes-Session-Token 头。
  • WebSocket Ticket：当 OAuth 认证开启时（__HERMES_AUTH_REQUIRED__），WebSocket 升级前先调用 getWsTicket() 获取一次性 ticket。
  • Management Profile：api.ts 中的 _managementProfile 全局变量，通过 ?profile= 查询参数注入到 profile-scoped 端点。

4. 仪表盘插件系统

4.1 插件注册流程

  1. 插件开发者编写 JS bundle，调用 window.__HERMES_PLUGIN_SDK__.register()
  2. 后端 /api/dashboard/plugins 返回已注册的 PluginManifest 列表
  3. 前端 App.tsx 的 usePlugins() 加载所有 manifest，合并路由和导航项
  4. 插件页面通过 PluginPage 组件动态加载渲染

4.2 UI 槽位

插件可通过 registerSlot(name, component) 向布局注入内容，可用槽位：

槽位名位置用途
header-banner页面顶部（移动端 header 下方）全局通知横幅
header-left侧边栏品牌区域左侧品牌图标扩展
header-right侧边栏底部按钮区额外操作按钮
backdrop全屏固定背景层背景动画/水印
pre-main主内容区之前页面级工具栏
post-main主内容区之后页脚扩展
overlay全屏覆盖层（最顶层）全局弹窗/通知

4.3 PluginManifest 关键字段

字段类型说明
namestring插件唯一标识（如 "my-plugin"）
tab.pathstring路由路径（如 "/my-plugin"）
tab.positionstring?"end" | "after:<path>" | "before:<path>"
tab.overridestring?覆盖内置路由（如覆盖 "/chat"）
tab.hiddenboolean?注册路由但不显示导航项（槽位专用插件）
entrystring插件 JS bundle URL
integritystring?SRI 哈希（防篡改）
has_apiboolean是否有后端 API 扩展
