一、技能系统的定位与设计理念

Hermes Agent 的技能系统是一个渐进式知识文档系统，兼容 agentskills.io 开放标准。技能以 SKILL.md 文件（YAML 前置元数据 + Markdown 正文）形式存在，Agent 按需加载以最小化 token 消耗。这是 Hermes"自进化学习闭环"的核心——Agent 可在执行任务后自动创建技能、在使用中持续改进技能。
agentskills.io 兼容 110 个 Python 文件 14.1 MB 技能仓库 渐进式披露 7 种联合来源

技能系统横跨三个位置：

二、SKILL.md 格式与渐进式披露

每个技能目录的核心是一个符合 agentskills.io 规范的 SKILL.md 文件。以下是典型的技能定义示例：

---
name: blogwatcher                   # 必填，最长 64 字符
description: RSS/Atom feed monitor  # 必填，最长 1024 字符
version: 1.0.0
license: MIT
platforms: [macos, linux]
prerequisites:
  commands: [blogwatcher-cli]
metadata:
  hermes:
    tags: [rss, monitoring]
    category: research
---

# Blog Watcher

Monitor RSS/Atom feeds for updates. Use this skill when...

技能被加载时采用三级渐进式披露以节省 token：

级别工具返回内容Token 消耗
L0 — 列表skills_list()仅名称 + 描述 + 分类~3k tokens（全部）
L1 — 详情skill_view(name)完整 SKILL.md 正文 + 元数据按需（变量）
L2 — 附件skill_view(name, path)指定引用文件（references/scripts/templates 等）按需（变量）

技能目录还支持标准子目录：references/（参考文档）、templates/（输出模板）、scripts/（辅助脚本）、assets/（附属文件，agentskills.io 标准）。

技能加载远不止读取本地 SKILL.md 文件。Hermes 通过 Skills Hub 从 7 种来源联合发现技能：

Skills Hub（skills_hub.py，4110 行）进一步实现了外部来源的适配器，支持 GitHub PAT、gh CLI 和 GitHub App 三种认证方式，安装的技能进入隔离目录，并有 HubLockFile 记录来源证明。

四、策展器 — 自动技能生命周期管理

这是技能系统最具自进化特征的机制。Agent 在执行复杂任务（5 次以上工具调用）后，可自动将工作流创建为技能。策展器在后台管理这些 agent 创建的技能：

  • 标记陈旧：长时间未使用的 agent 技能自动标记为 stale
  • 归档：进一步不活跃则移入 .archive/ 子目录
  • 绝不自动删除：用户可随时从归档恢复
  • 保护列表：内置技能（如 plan）和 hub 安装技能默认永不归档
  • 来源追踪：skill_provenance.py 通过 ContextVar 区分用户前台写入与后台自改进写入

用量数据存储在 ~/.hermes/skills/.usage.json 中，由 skill_usage.py 管理。配置选项 curator.prune_builtins: true 可开启对未使用内置技能的归档。

  2. Skills Hub 有多少种外部技能来源？列出其中 3 种。
  3. 策展器会在什么条件下归档一个技能？被归档的技能是否会被自动删除？
点击查看参考答案

1. L0 返回名称+描述+分类（~3k tokens）；L1 返回完整 SKILL.md 正文；L2 返回指定引用文件（references/scripts 等）。

2. 7 种：官方可选、GitHub 默认 tap、skills-sh 目录、well-known 端点、直链 URL、ClawHub、browse-sh。

3. agent 创建的技能在长时间未使用后标记 stale，若继续不活跃则归档到 .archive/。策展器绝不自动删除任何技能，用户可随时恢复。内置技能和 hub 安装技能默认受保护。
