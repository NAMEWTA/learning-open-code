3. 关键接口契约

3.1 子命令 Builder 接口约定

要素约定
函数签名def build_xxx_parser(subparsers, *, cmd_xxx: Callable, ...)
输出None——仅向 subparsers 注册解析器
Handler 绑定parser.set_defaults(func=cmd_xxx)
导入规则Builder 函数不 import main.py，handler 通过参数注入
文件命名subcommands/<命令名>.py

3.2 命令分发流程

用户输入: hermes gateway start --verbose
  │
  ▼
main.py: _apply_profile_override() → --profile 预处理（修改 HERMES_HOME）
  │
  ▼
main.py: _read_packed_ref() / _sync_bundled_skills_for_startup() → 启动检查
  │
  ▼
_parser.py: build_top_level_parser() → 返回 (parser, subparsers, chat_parser)
  │
  ▼
main.py: 40+ 个 builder 函数向 subparsers 注册子命令
  │
  ▼
parser.parse_args(argv) → Namespace(command='gateway', gateway_command='start', verbose=1)
  │
  ▼
args.func(args) → cmd_gateway(args) → 根据 gateway_command 分发到具体逻辑

3.3 配置分层优先级

层级来源示例
1 (最低)DEFAULT_CONFIGhermes_cli/config.py 中硬编码的默认值
2~/.hermes/config.yaml用户编辑的 YAML 配置文件
3 (最高)环境变量 / .envHERMES_HOME, HERMES_INFERENCE_MODEL 等

损坏自愈机制：config.yaml 解析失败时，_backup_corrupt_config() 自动创建 config.yaml.corrupt..bak 备份，load_config() 降级返回 DEFAULT_CONFIG。同一文件不会重复告警（通过 _CONFIG_PARSE_WARNED 集合去重）。

4. 顶级全局标志（--flag 无需子命令）

标志说明
-z, --oneshot PROMPT单次模式 — 发送 prompt 并仅输出最终回复到 stdout
--tui启动现代 TUI 界面
--cli强制使用经典 prompt_toolkit REPL
-r, --resume SESSION按 ID 或标题恢复会话
-c, --continue [NAME]恢复最近会话或按名称恢复
-m, --model MODEL模型覆盖
--provider PROVIDER提供商覆盖
-s, --skills SKILLS预加载技能
--yolo跳过所有危险命令审批
--safe-mode排障模式 — 禁用所有自定义配置
--ignore-user-config忽略 ~/.hermes/config.yaml
--ignore-rules跳过 AGENTS.md/memory/skills 注入
-w, --worktree在隔离的 git worktree 中运行
--usage-file PATHoneshot 模式后写入 JSON 用量报告

_inherited_flag()：标记为 inherit_on_relaunch = True 的标志会在 CLI 重新执行自身时（如 setup 后启动 chat）自动携带，确保用户在子流程中不会丢失已设置的标志。
