消息网关模块总览 L1 模块课程 -- 多平台消息路由、适配器体系与投递系统

学习目标：理解 gateway 的三层架构，掌握平台适配器接口与消息投递路由规则 | 预估时长：15 分钟 | 前置知识：L0 项目全局地图 | 关联参考：网关速查手册

一、定位：Agent 与外部世界的"窄腰"

gateway 是 Hermes Agent 的消息中枢——所有外部平台的入站消息经过它进入 agent 核心，agent 的响应也经它投递回正确的平台和频道。它实现了"一个核心，20+ 平台"的架构承诺。
70 个 .py 文件 3.5 MB 源码 22+ 平台 WebSocket Relay 插件扩展

回忆 L0 总览中介绍的分层架构：用户通过 Telegram / Discord / Slack 等消息平台与 Agent 交互，所有消息出入都由 gateway/ 层统一处理。这是经典的 Façade 模式——对 agent 核心而言，所有平台看起来一模一样；对平台而言，agent 是一个黑盒。

数据流向：入站消息自底向上——平台适配器接收 → 中继层组装 SessionSource → 投递层送入 agent 核心。出站响应自顶向下——agent 产出 → 投递层解析 target → 中继层路由 → 适配器发送。

四、核心接口与调用链路

4.1 平台适配器契约

所有平台适配器继承 BasePlatformAdapter，必须实现四个抽象方法：

class BasePlatformAdapter(ABC):
    """所有平台适配器的统一基类。"""

    @abstractmethod
    async def connect(self) -> None:
        """建立平台连接（WebSocket / polling / webhook）。"""
        ...

    @abstractmethod
    async def disconnect(self) -> None:
        """断开平台连接并清理资源。"""
        ...

    @abstractmethod
    async def send(self, target: str, content: str,
                   metadata: dict = None) -> SendResult:
        """向指定 target（chat_id/channel_id）发送消息。"""
        ...

    @abstractmethod
    async def get_chat_info(self, chat_id: str) -> dict:
        """查询频道/会话的元信息（名称、类型、成员数）。"""
        ...

4.2 消息投递路由

DeliveryRouter 实现了三级回退策略，确保消息总能到达：

# 投递优先级（简化表示）
class DeliveryRouter:
    def resolve_target(self, target: str, source: SessionSource):
        # 1. 显式靶点：deliver_to="telegram:123456"
        if target and ":" in target:
            return PlatformTarget(target)

        # 2. 平台 home channel：未指定靶点则投到平台默认频道
        home = self.config.get_home_channel(source.platform)
        if home:
            return PlatformTarget(home)

        # 3. 来源回投：回投到消息来源频道
        if source.chat_id:
            return PlatformTarget(f"{source.platform}:{source.chat_id}")

        # 4. 兜底：写入本地文件
        return LocalTarget()

检索练习

读完本课后，尝试不翻看上文回答以下问题：

  1. gateway 的三层架构从底向上分别是什么？每层的核心文件是哪个？
  2. DeliveryRouter 的四级回退策略各是什么？优先级如何排序？
  3. 如果要在 Hermes Agent 中新增一个"飞书"平台适配器，需要继承哪个类、实现哪四个方法？
点击查看参考答案

1. 平台适配层（platforms/base.py）、中继/路由层（relay/ + session.py）、投递层（delivery.py）。

2. 优先级从高到低：显式靶点（platform:chat_id）→ 平台 home channel → 来源回投（source.chat_id）→ 本地文件兜底。

3. 继承 BasePlatformAdapter，实现 connect()、disconnect()、send()、get_chat_info() 四个方法。如果希望支持插件发现，还需实现 check_fn 并通过 platform_registry.register() 注册。
