2. 对外暴露的关键接口

2.1 cli.py — CLI 入口

接口类型说明
main(query, toolsets, model, provider, ...)函数Fire 框架入口，接受所有 CLI 参数。支持 --query 单次执行、--gateway 启动网关、无参数进入交互 REPL
HermesCLI(model, toolsets, ...)类交互式 CLI 主类，继承 CLIAgentSetupMixin + CLICommandsMixin。run() 方法启动 REPL
ChatConsole类终端显示辅助，管理 rich 面板与 ANSI 渲染

2.2 run_agent.py — 运行循环

接口类型说明
AIAgent(base_url, model, ...)类Agent 主类，约 75 个构造参数。__init__ 委托给 agent/agent_init.py
AIAgent.run_conversation(user_message, ...)方法对话主入口，委托给 agent/conversation_loop.py
AIAgent.chat(message, ...)方法简化对话接口，返回最终响应字符串
AIAgent.interrupt(message)方法发送中断信号到运行中的 Agent
AIAgent.switch_model(new_model, ...)方法运行时动态切换模型 / 提供商
AIAgent.close()方法完整资源释放：Session DB、内存 Provider、客户端连接
main(query, model, ...)函数库模式的 CLI 入口（独立于 cli.py），用于 python -m run_agent

2.3 hermes_state.py — 状态持久化

接口类型说明
SessionDB(db_path)类线程安全 SQLite 会话存储。创建/查询/结束会话、消息增删改查
SessionDB.search_messages(query)方法FTS5 全文搜索历史消息，支持短语与布尔查询
SessionDB.search_sessions(query)方法全文搜索会话标题与元数据
SessionDB.list_sessions_rich(...)方法会话列表（含过滤、分页、排序），供 UI 选择器使用
SessionDB.archive_and_compact(...)方法归档旧会话并压缩数据库文件
AsyncSessionDB(db_path)类SessionDB 的异步包装器，所有调用通过 asyncio.to_thread 执行
apply_wal_with_fallback(db_path)函数设置 WAL 日志模式，不兼容文件系统降级到 DELETE

2.4 引导与配置文件

文件关键接口说明
hermes_constants.pyget_hermes_home()、get_config_path()、set_hermes_home_override()配置路径解析，支持 ContextVar 覆盖
hermes_logging.pysetup_logging()、set_session_context()、flush_log_queue()日志系统初始化与会话标记
hermes_time.pynow()时区感知的当前时间，优先 HERMES_TIMEZONE 环境变量
hermes_bootstrap.pyapply_windows_utf8_bootstrap()、harden_import_path()模块级自动执行，无需手动调用

3. 关键依赖关系矩阵

引导层（零依赖，最先导入）
  hermes_bootstrap.py   → 无 hermes 模块依赖
  hermes_constants.py   → 无 hermes 模块依赖
  hermes_logging.py     → hermes_constants
  hermes_time.py        → hermes_constants

CLI / 运行层
  cli.py       → hermes_bootstrap, hermes_cli (CLIAgentSetupMixin, CLICommandsMixin)
               → model_tools, hermes_state, hermes_logging, hermes_constants
  run_agent.py → hermes_bootstrap, hermes_constants
               → agent/agent_init (AIgent.__init__ 委托)
               → agent/conversation_loop (run_conversation 委托)

状态层
  hermes_state.py → hermes_constants, agent/memory_manager

工具 / 扩展层
  model_tools.py    → tools/registry, toolsets
  toolsets.py       → 无 hermes 模块依赖（仅 typing）
  mcp_serve.py      → mcp.server.fastmcp（第三方），无 hermes 模块依赖
  batch_runner.py   → run_agent, model_tools, toolset_distributions
  trajectory_compressor.py → utils, hermes_constants, agent/retry_utils
  utils.py          → 无 hermes 模块依赖

设计要点：引导层（bootstrap + constants）是整个项目的零依赖基础层，确保在任何导入顺序下都能安全工作。cli.py 和 run_agent.py 是平行的两个入口——cli.py 面向交互式终端用户，run_agent.py 面向库调用者，两者最终都通过 agent/ 核心层执行对话。
