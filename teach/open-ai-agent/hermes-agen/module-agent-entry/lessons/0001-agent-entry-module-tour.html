一、一句话定位

入口层是 Hermes Agent 的启动前置区——负贵环境引导、CLI 命令解析、Agent 实例化、状态持久化等一切"在对话循环开始之前"要做的事。它不参与模型推理，但决定推理在什么环境下、以什么配置、用什么状态启动。
cli.py (726KB) run_agent.py (263KB) hermes_state.py (269KB) Fire 框架 SQLite FTS5 零依赖常量

如果 Hermes Agent 是一座工厂，入口层就是门卫、配电间和档案室——门卫负责接待（CLI 解析），配电间负责通电（环境引导与日志），档案室负责存取历史（状态持久化）。真正的生产车间在 agent/ 核心层。

二、文件职责地图

入口层 13 个文件按职责分四组。核心三文件的大小总和超过项目根目录其余文件之和：

项目根目录 (入口层文件) ├── hermes_bootstrap.py ← [引导] 首行导入，Windows UTF-8 修复 ├── hermes_constants.py ← [配置] HERMES_HOME 路径解析，零依赖 ├── hermes_logging.py ← [配置] 旋转日志 + 会话上下文注入 ├── hermes_time.py ← [配置] 时区感知 now() 函数 │ ├── cli.py ← ★ [CLI] Fire 框架入口，200+ 命令注册 ├── run_agent.py ← ★ [运行] AIAgent 类，run_conversation() │ ├── hermes_state.py ← ★ [状态] SQLite FTS5 全文搜索存储 │ ├── model_tools.py ← [工具] 工具定义获取与函数调用分发 ├── toolsets.py ← [工具] 命名工具集组合与解析 │ ├── mcp_serve.py ← [扩展] FastMCP 服务端入口 ├── batch_runner.py ← [扩展] 批量轨迹生成（研究模式） ├── trajectory_compressor.py ← [扩展] 训练数据轨迹压缩 └── utils.py ← [扩展] 原子文件写、真值解析等

引导组（3 个文件）最先被导入，为后续所有代码提供环境基础。CLI 与运行循环（2 个文件）是入口层的核心——cli.py 面向终端用户，run_agent.py 面向库调用者。状态组（hermes_state.py）独立于运行模式，为所有对话提供持久化支撑。

三、关键调用链

以最常见的交互式启动 python cli.py 为例，完整调用链如下：

用户输入 "python cli.py --toolsets web,terminal"
  │
  ▼
[1] hermes_bootstrap.py   ← 模块级 import 自动执行 UTF-8 修复
[2] hermes_constants.py   ← 首次 import 时推算 HERMES_HOME
[3] hermes_logging.py     ← 通过 setup_logging() 建立日志管道
[4] cli.py:fire.Fire(main)← Fire 框架解析参数，调用 main()
  │
  ▼
[5] cli.py:main(toolsets="web,terminal")
  │  └─ 进入交互模式 → HermesCLI() 实例化
  │     └─ 继承 CLIAgentSetupMixin  + CLICommandsMixin
  │
  ▼
[6] cli.py:HermesCLI.run()  ← REPL 循环
  │  └─ 收到用户查询 → model_tools.get_tool_definitions()
  │  └─ 创建 AIAgent(...) → agent.run_conversation(query)
  │
  ▼
[7] agent/conversation_loop.py  ← 对话循环（下一课 L1-agent-core）

两种启动路径：

# 路径 A：CLI 交互模式 -- cli.py 作为主入口
$ python cli.py --toolsets web,terminal
# Fire 解析 → main() → HermesCLI().run() → REPL 循环

# 路径 B：库模式 -- run_agent.py 作为 Python API
from run_agent import AIAgent
agent = AIAgent(base_url="http://localhost:30000/v1",
                model="claude-opus-4-20250514")
response = agent.run_conversation("你好，介绍一下你自己")

关键依赖链：cli.py 导入 hermes_cli/ 框架层 → 框架层导入 run_agent.py → run_agent.py 的 AIAgent.__init__() 委托给 agent/agent_init.py → agent_init.py 导入 hermes_state.py 创建 SessionDB → 最终进入 agent/conversation_loop.py。

四、状态持久化机制

hermes_state.py 的 SessionDB 类是最重要的基础设施之一。它使用单文件 SQLite 数据库（默认 ~/.hermes/state.db），通过 FTS5 全文搜索引擎 实现对历史会话的快速检索，所有写入操作线程安全。

核心数据表

表名说明
sessions会话元数据（id、标题、平台、状态、时间戳）
messages消息内容（角色、内容、token 计数、工具调用）
model_configs每次运行的模型配置快照
messages_ftsFTS5 虚拟表，为 messages 提供全文索引

关键设计决策：WAL 模式（Write-Ahead Logging）允许并发读取不阻塞写入；FTS5 触发器自动同步消息变更到全文索引，无需手动维护；异步包装器 AsyncSessionDB 将所有 SQLite 调用 offload 到 asyncio.to_thread，确保不冻结事件循环。
