CLI 与 npm 包装层参考

用途：快速定位 `codex` 命令从发布包装到 Rust 子命令分发的源码入口。事实来源来自当前本地源码快照。

模块职责

层级 路径 职责
产品入口说明 README.md 说明 Codex CLI 是本地 coding agent，并区分 IDE、desktop app、Codex Web 等入口。
npm 包元数据 codex-cli/package.json 声明 @openai/codex、Node 版本、发布文件和 codex bin 映射。
npm 启动脚本 codex-cli/bin/codex.js 按平台选择 target triple，定位 optional package 或 vendor 中的原生二进制，并转发参数和信号。
Rust CLI crate codex-rs/cli 生成原生命令 codex，用 clap 解析全局选项和子命令，并分派到业务 crate。
独立 exec 入口 codex-rs/exec/src/main.rs 提供 codex-exec 入口，并复用 codex_exec::run_main。

内部分层与关键依赖

层 关键依赖 说明
解析层 clap、clap_complete 定义 MultitoolCli、Subcommand、completion 生成。
配置与上下文层 codex-config、codex-utils-cli、codex-home 合并 -c 覆盖、profile、feature toggles、Codex home。
运行入口层 codex-tui、codex-exec、codex-app-server、codex-mcp-server 把用户意图交给真正的交互、非交互、富客户端或 MCP server 运行时。
扩展管理层 codex-mcp、codex-core-plugins、codex-plugin 管理外部 MCP servers 和 plugin marketplace/安装状态。
平台与诊断层 codex-sandboxing、codex-execpolicy、codex-terminal-detection 处理 sandbox、策略检查、终端能力与本地健康诊断。

简单调用示例

  1. npm install -g @openai/codex 安装 npm 包，并通过 bin 暴露 codex。
  2. codex-cli/bin/codex.js 根据当前平台定位原生二进制，例如 aarch64-apple-darwin/bin/codex。
  3. codex exec "summarize this repo" 的参数被原样转给 Rust 二进制。
  4. codex-rs/cli/src/main.rs 解析到 Subcommand::Exec，再调用 codex_exec::run_main。
