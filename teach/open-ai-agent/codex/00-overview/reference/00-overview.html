Codex L0 项目总览参考

用途：为后续 L1/L2 教学构建 backlog。事实来源来自当前本地源码快照。

项目定位

Codex CLI 是 OpenAI 的本地 coding agent。仓库同时承载命令行入口、终端 TUI、app-server 协议服务、工具/沙箱/配置/模型后端、Python 与 TypeScript SDK，以及 Bazel、CI、发布包装。入口边界以 open-ai-agent/codex/README.md 为准：CLI 本地运行；IDE 走编辑器集成；桌面体验可通过 codex app；云端 agent 是 Codex Web。

核心设计哲学与源码证据

  1. 本地 agent 优先：README.md 把 CLI 定义为本地运行，codex-cli/bin/codex.js 再把 npm 命令转成本机原生二进制。
  2. crate 边界必须克制：AGENTS.md 明确要求抵制向 codex-core 继续堆功能，优先寻找或新增更合适的 crate。
  3. 沙箱和审批是核心边界：docs/sandbox.md 指向安全文档，cli/src/main.rs 暴露 sandbox 子命令，workspace 中有独立 sandboxing、execpolicy 与平台 sandbox crate。
  4. 富客户端通过 app-server 分离：codex-rs/app-server/README.md 说明该接口服务 VS Code 等 rich interfaces，协议围绕 Thread、Turn、Item 和流式通知。
  5. SDK 不重写 agent：TypeScript SDK 明确包装 @openai/codex CLI 并通过 JSONL 事件通信；Python SDK 也围绕启动线程、运行 turn、复用 Codex 认证展开。

源码文件豁免清单

范围 后续覆盖策略 理由
codex-rs/vendor/、third_party/ 默认豁免，仅在构建或依赖补丁课程中引用。 第三方或供应商代码，不是 Codex 架构主线。
patches/ 构建发布 L1 中概览，不逐补丁教学。 服务 Bazel/toolchain 兼容性，细节依赖外部项目。
各子项目锁文件，例如 codex-rs/Cargo.lock、tools/argument-comment-lint/Cargo.lock、scripts/uv.lock、sdk/python/uv.lock、sdk/python-runtime/uv.lock；另含 pnpm-lock.yaml、MODULE.bazel.lock 默认豁免，依赖漂移或发布课程中说明更新规则。 锁文件体量大、机械生成，重点是触发更新的规则。
图片、二进制、生成物、notebook 输出 默认豁免。 不适合作为源码阅读主线。
.github/workflows/ 纳入构建/CI/发布 L1，但不作为 agent 运行链路必读。 它解释交付，不解释运行时业务逻辑。
docs/*.md 作为用户语义入口纳入；跳转型文档只记录入口，不展开外站内容。 当前仓库中的 docs 多数链接到官方文档站。

建议 L1/L2 goal 清单

模块组 L1 建议 L2 垂直切片建议
CLI 与包装层 从 npm 包到 Rust CLI 的启动链路 npm install -g @openai/codex 后执行 codex exec 的路径；codex app-server 子命令分发。
TUI 交互式终端应用结构 用户输入一条 prompt 后，TUI 如何渲染消息、工具进度和 diff。
core Thread/Turn、上下文与模型请求 一次 user turn 如何构造 model-visible context、调用 Responses API、处理流式事件。
tools 工具定义、schema 与执行抽象 shell、apply_patch、MCP tool 如何变成模型可调用工具并回写结果。
sandbox/config 审批、权限 profile 与平台 sandbox workspace-write、network disabled、Seatbelt/Landlock/Windows 的决策路径。
app-server JSON-RPC 协议与富客户端生命周期 SDK/IDE 调用 thread/start、turn/start 并接收 item/* 通知。
SDK Python 与 TypeScript SDK 边界 TypeScript SDK spawn CLI 并消费 JSONL；Python SDK 复用认证并封装 thread/run。
构建与发布 Bazel、Cargo、pnpm、release workflow Rust 多平台二进制如何进入 npm optional package、Homebrew 或 GitHub Release。
扩展系统 skills、plugins、MCP、hooks 一个 skill 如何被发现、注入上下文，并通过 app-server 暴露给客户端。
