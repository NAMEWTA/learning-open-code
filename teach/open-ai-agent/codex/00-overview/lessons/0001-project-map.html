项目学习地图：先看 Codex 的边界

学习目标：15 分钟内判断一个 Codex 问题应该从哪个顶层模块开始读。

一句话定位：Codex CLI 是 OpenAI 的本地 coding agent，用户从终端、IDE、桌面 app 或 SDK 触发工作，核心执行与 agent 编排主要落在 Rust workspace。 主要来源：open-ai-agent/codex/README.md、open-ai-agent/codex/codex-rs/Cargo.toml。

1. 先分清四个入口

顶层 open-ai-agent/codex/README.md 明确把 Codex CLI 定义为“本地运行在你的电脑上的 coding agent”。同一段还把产品入口分开：想在 VS Code、Cursor、Windsurf 中使用，走 Codex IDE；想要桌面体验，运行 codex app 或打开 Codex App；想找云端 agent，则是 Codex Web。学习源码时要先记住：本仓库的重心不是一个网页产品，而是本地 CLI、TUI、app-server、SDK 与发布包装共同组成的本地 agent 系统。

2. 从用户入口看架构

flowchart LR
  User[用户] --> Npm[npm/Homebrew/安装脚本]
  Npm --> Js[codex-cli/bin/codex.js]
  Js --> Cli[codex-rs/cli]
  User --> SDK[Python/TypeScript SDK]
  SDK --> AppServer[codex-rs/app-server]
  Cli --> Tui[codex-rs/tui]
  Cli --> Exec[codex-rs/exec]
  Cli --> AppServer
  Tui --> Core[codex-rs/core]
  AppServer --> Core
  Core --> Tools[codex-rs/tools]
  Core --> Sandbox[codex-rs/sandboxing]
  Core --> Model[model-provider/backend-client]
  Tools --> MCP[MCP/plugins/skills]
  Sandbox --> OS[Seatbelt/Landlock/Windows sandbox]

这张图的读法是：codex-cli/package.json 只暴露 codex 命令；真正的平台分发逻辑在 codex-cli/bin/codex.js，它按系统选择原生二进制再转发参数。进入 Rust 后，codex-rs/cli/src/main.rs 用 clap 定义 exec、review、mcp、app-server、sandbox、resume 等子命令；无子命令时进入交互 TUI。核心 agent 状态、模型请求、工具调用、会话与上下文主要由 codex-rs/core 承担，但仓库规范提醒“抵制继续膨胀 codex-core”，所以后续读源码要关注哪些能力已被拆到独立 crate。

3. 顶层目录只记职责

open-ai-agent/codex/ ├── README.md ← 项目定位、安装方式、产品入口边界 ├── codex-cli/ ← npm 包装层，只发布 bin/codex.js ├── codex-rs/ ← Rust workspace，CLI/TUI/core/app-server 等主体 ├── sdk/ ← Python 与 TypeScript SDK ├── docs/ ← 用户文档入口，多数跳转到官方文档站 ├── BUILD.bazel ← Bazel 平台和根构建入口 ├── MODULE.bazel ← Bazel 模块、Rust toolchain、Cargo 导入与 patches ├── .github/ ← CI、release、SDK 发布 workflow ├── patches/ ← Bazel/toolchain/依赖补丁 ├── scripts/ ← 仓库维护和发布脚本 └── third_party/ ← 第三方集成材料

L0 阶段不需要背目录，而是建立“问题到目录”的映射：安装失败先看 codex-cli 与 release workflow；交互界面先看 codex-rs/tui；IDE 或 SDK 集成先看 codex-rs/app-server；沙箱和审批先看 codex-rs/sandboxing、codex-rs/config、codex-rs/protocol；模型、工具、MCP、skills 则从 core 分流到 tools、codex-mcp、core-skills 与 ext/*。

4. 检索练习

暂停回忆：如果你要研究“VS Code 扩展如何驱动 Codex 的一个 turn 并接收流式事件”，第一站应该读哪两个区域？如果要研究“npm install 后为什么能启动本机二进制”，第一站又是哪两个文件？

参考答案：VS Code/富客户端链路先读 codex-rs/app-server/README.md 和 codex-rs/app-server-protocol，再下钻到 core。npm 启动链路先读 codex-cli/package.json 和 codex-cli/bin/codex.js。还有疑问时，继续追问 agent，让它带你从其中一个入口做 L1 深挖。
