MCP Server 管理链路地图

学习目标：画出从 CLI mcp add 到 MCP 工具在 turn 中可用的完整 6 层链路，说出每层的职责边界和关键决策点。

前置：L1 扩展系统与 MCP 模块导览 | L1 CLI 打包模块导览

1. 链路全景：6 层闭环

一条 codex mcp add my-server -- node server.js 命令，从 CLI 参数到最终模型可见的工具，穿过 6 层。每一层有独立的职责和错误处理边界：

用户 CLI 输入：codex mcp add my-server -- node server.js │ ▼ [1] CLI 层：mcp_cmd.rs — run_add() 解析参数 → 验证 server name → 构造 McpServerConfig 所属：codex-rs/cli/src/mcp_cmd.rs │ ▼ [2] 配置层：mcp_edit.rs — ConfigEditsBuilder load_global_mcp_servers() 读取现有配置 → replace_mcp_servers() 序列化 toml_edit → apply() 写入 ~/.codex/config.toml 所属：codex-rs/config/src/mcp_edit.rs │ ▼ [3] 编目层：mcp/mod.rs — effective_mcp_servers() 从 McpConfig 提取配置 → 应用 auth 规则 → 移除不可用的 Codex Apps 服务器 所属：codex-rs/codex-mcp/src/mcp/mod.rs │ ▼ [4] 连接管理层：connection_manager.rs — McpConnectionManager::new() 遍历 enabled servers → 创建 AsyncManagedClient → JoinSet 并发启动 → 发出 McpStartupUpdate 事件 → 等待所有完成 → 发出 McpStartupComplete 事件 所属：codex-rs/codex-mcp/src/connection_manager.rs │ ▼ [5] 客户端层：rmcp_client.rs — AsyncManagedClient → RmcpClient make_rmcp_client() 创建 transport（stdio/streamable_http） → initialize() 握手 → tools/list 获取工具 → tool_filter 过滤 → 缓存工具列表 所属：codex-rs/codex-mcp/src/rmcp_client.rs + codex-rs/rmcp-client/src/rmcp_client.rs │ ▼ [6] 模型可见层：connection_manager.rs — list_all_tools() 聚合所有 server 工具 → with_server_metadata() → normalize_tools_for_model_with_prefix() → 返回 ToolInfo[] 供 turn 使用 所属：codex-rs/codex-mcp/src/connection_manager.rs

2. 两种 Transport 的分叉点

MCP 服务器通过两种 transport 连接 Codex。从 CLI 参数到连接建立，两种路径在第 1 层分叉，在第 5 层汇合：

特性StdioStreamable HTTP
CLI 参数 -- command args... --url https://...
配置字段 command + args + env + cwd url + bearer_token_env_var + http_headers
连接方式 StdioServerLauncher 启动子进程，通过 stdin/stdout JSON-RPC StreamableHttpClientTransport 通过 HTTP POST + SSE 通信
OAuth 支持 不支持 支持 OAuth 2.0 登录和 bearer token
环境要求 需要 local environment 本地可直接使用 ambient HTTP client

4. 暂停回忆

不看上文，回答三个问题：

  1. 从 CLI mcp add 到 MCP 工具可用，经过哪 6 层？每层的核心模块是什么？
  2. stdio 和 streamable_http 两种 transport 在配置写入时有什么区别？在连接建立时有什么区别？
  3. 工具列表被缓存在哪一层？什么时候会命中缓存？

参考答案：1. CLI 层（mcp_cmd.rs）→ 配置层（mcp_edit.rs）→ 编目层（mcp/mod.rs）→ 连接管理层（connection_manager.rs）→ 客户端层（rmcp_client.rs + rmcp-client）→ 模型可见层（list_all_tools）。2. 配置写入时，stdio 写 command/args/env/cwd 字段，streamable_http 写 url/bearer_token_env_var/http_headers 字段；连接建立时，stdio 通过 StdioServerLauncher 启动子进程用 stdin/stdout JSON-RPC 通信，streamable_http 通过 HTTP POST + SSE 通信并支持 OAuth bearer token。3. Codex Apps MCP 服务器在 CodexAppsToolsCache 中缓存，普通 MCP 服务器在 ManagedClient.tools 中按启动时获取的结果缓存（不主动刷新）。
