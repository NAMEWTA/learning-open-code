# 使命：Skill 上下文预算模型

## 为什么
理解 Codex 如何在有限的 LLM 上下文窗口中管理大量 Skills 的元数据注入——当用户安装了 50+ 个 skills，每个 skill 的描述可能有几百字，Codex 如何确保模型始终能看到关键信息而不超出 token 预算？这是构建高性能 AI Agent 系统的核心工程决策。

## 成功的样子
- 能解释 Codex 的 2% 上下文窗口预算公式及其来源
- 能手绘三级降级策略的决策树：完整注入 → 描述截断 → 仅名称
- 能评估别名压缩在什么场景下有效、什么场景下退化为绝对路径
- 能说出预算模型的两个已知局限与改进方向

## 约束条件
- 源码只读分析，不修改 Codex 代码
- 每节短课 15 分钟内可完成
- 以 Rust 源码为第一手资料，辅以测试用例验证理解

## 不在范围内
- Codex 的整体架构（参见 L0 总览）
- Skill 加载与解析流程（参见 module-extensions-skills-mcp 和 slice-skill-activation）
- Skill 显式注入与 prompt 渲染（属于 injection.rs 的教学主题，本主题聚焦预算模型本身）
