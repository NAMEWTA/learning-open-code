渲染流程概览

build_available_skills(outcome, budget, side_effects)
  │
  ├─ 1. outcome.allowed_skills_for_implicit_invocation()
  │     过滤：已启用 + 允许隐式调用
  │
  ├─ 2. ordered_absolute_skill_lines(&skills)
  │     排序：System(0) > Admin(1) > Repo(2) > User(3)
  │     同 scope 按 name 字母序
  │
  ├─ 3. build_available_skills_from_lines(absolute_lines, budget)
  │     └─ render_skill_lines_from_lines()
  │          ├─ [一级] full_cost ≤ budget → 完整渲染
  │          ├─ [二级] minimum_cost ≤ budget → 描述按预算截断
  │          └─ [三级] minimum_cost > budget → 仅部分 skill 保留
  │
  ├─ 4. 如果绝对路径有截断/丢弃 → 尝试别名压缩
  │     └─ build_aliased_available_skills(outcome, skills, budget)
  │          ├─ build_alias_plan() → 别名根表 + 路径映射
  │          └─ build_available_skills_from_lines(aliased_lines, adjusted_budget)
  │
  └─ 5. aliased_render_is_better(aliased, absolute, budget) ?
        YES → 返回别名方案
        NO  → 返回绝对路径方案

预算公式

// Token 模式（有上下文窗口信息）
budget_tokens = max(1, context_window × 2 / 100)

// Character 模式（回退，无上下文窗口信息）
budget_chars = 8000

// 示例计算
200,000 tokens 窗口 → budget = 200,000 × 0.02 = 4,000 tokens
  99 tokens 窗口   → budget = max(1, 99 × 2 / 100) = 1 token
  无窗口信息       → budget = 8,000 characters

Scope 优先级

Scope优先级值说明
System0（最高）Codex 内置系统 skill，如 skill-creator
Admin1管理员配置的组织级 skill
Repo2项目仓库中的 skill
User3（最低）用户个人 skill

当三级降级需要丢弃 skills 时，User scope 的 skill 最先被丢弃。
