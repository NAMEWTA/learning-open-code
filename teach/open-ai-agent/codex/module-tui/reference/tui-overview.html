TUI 模块速查：边界、入口与后续切片

这份参考文档用于快速定位 `codex-tui` 在整体架构中的位置。它不是完整源码讲解，而是后续 L2/L3/L4 的地图。

一、模块定位

用户终端 / shell
↓ 键盘、粘贴、resize、启动参数
codex-tui：配置启动、事件分发、聊天 UI、输入区、状态展示
↓ AppCommand / app-server protocol
app-server / core runtime：thread、turn、工具执行、审批、模型请求

二、关键源码入口

源文件职责阅读重点
codex-rs/cli/src/main.rs顶层 CLI 分发无子命令时走交互路径：`subcommand == None` 调用 `run_interactive_tui`，最终进入 `codex_tui::run_main`。
codex-rs/tui/Cargo.tomlcrate 边界与依赖确认 `codex-tui` 同时依赖终端库、app-server client、protocol、config、login、state。
codex-rs/tui/src/main.rs`codex-tui` 二进制入口解析 `TopCli`，合并 `-c` overrides，调用 `run_main`，退出后打印 token/resume 信息。
codex-rs/tui/src/cli.rsTUI 参数`prompt`、`--strict-config`、resume/fork 内部字段、`--ask-for-approval`、`--search`、`--no-alt-screen`。
codex-rs/tui/src/lib.rs运行时组装`run_main` 加载配置和认证，选择 app-server target，准备终端模式，进入 `App::run`。
codex-rs/tui/src/app.rs顶层应用状态`App` 字段、`select!` 事件循环、终端事件、thread 事件、app-server 事件分发。
codex-rs/tui/src/chatwidget.rs聊天主体transcript、active cell、状态行、slash command 接收、`submit_op` 到 `AppCommand`。
codex-rs/tui/src/bottom_pane/输入与弹层`BottomPane`、`ChatComposer`、`BottomPaneView`、approval、footer、status surface。
codex-rs/tui/src/status_indicator_widget.rs运行状态组件elapsed、details、interrupt binding、暂停/恢复、动画刷新。
codex-rs/tui/styles.md样式约定header/primary/secondary 文本，cyan/green/red/magenta 的语义使用，避免自定义颜色。

四、核心依赖速查

依赖类别代表 crate在 TUI 中的意义
终端 UIratatui、crossterm绘制 widget、读取键盘/粘贴/resize、控制 alternate screen。
Codex 服务边界codex-app-server-client、codex-app-server-protocol连接 embedded/local daemon/remote app-server，并收发 thread、turn、审批事件。
配置与认证codex-config、codex-login、codex-cloud-config加载配置层、登录限制、云端配置和模型提供商信息。
协议与状态codex-protocol、codex-state、codex-rollout描述权限、sandbox、thread id、状态库和实验开关。

五、边界与上游入口

方向边界说明关键落点
