工具定义与执行抽象参考

用途：作为 L1 源码阅读地图。短课见 0001 工具执行模块导览；上层架构见 L0 项目总览参考。

模块定位

工具执行层位于模型会话与宿主环境之间。上游是模型返回的 Responses API item，下游是本地进程、文件系统、MCP server、插件扩展、子 agent、code mode runtime 或托管工具。它的核心任务是把“模型请求”变成“受策略约束的副作用”，再把结果转回模型可消费的 response item。

最小调用序列

模型请求阶段 build_tool_specs_and_registry → add_tool_sources → append_tool_search_executor → prepend_code_mode_executors → build_model_visible_specs_and_registry 模型返回阶段 ResponseItem → ToolRouter::build_tool_call → ToolRouter::dispatch_tool_call_with_terminal_outcome → ToolRegistry::dispatch_any_with_terminal_outcome → handler.handle(invocation) → ToolOutput::to_response_item

调试工具“不出现”时，先看规划阶段；调试工具“出现但不能执行”时，先看路由与 registry；调试“执行失败”时，再进入 handler/runtime。注意：NewContextWindowHandler、CurrentTimeHandler、SleepHandler 等是 Rust handler 或注册入口名称，不等同于模型侧看到的 wire/tool name。

Shell 执行阅读顺序

  1. codex-rs/core/src/tools/handlers/shell_spec.rs：看 exec_command、write_stdin、shell_command 的参数 schema。
  2. codex-rs/core/src/tools/handlers/shell/shell_command.rs：看 ToolExecutor 实现、参数解析、to_exec_params。
  3. codex-rs/core/src/tools/handlers/shell.rs：看 run_exec_like 如何处理权限、审批、apply_patch 拦截和 ShellRequest。
  4. codex-rs/core/src/tools/runtimes/shell.rs：看 approval key、network approval、sandbox command 和 execute_env。
  5. codex-rs/core/src/exec.rs：看进程、超时、取消、输出 cap 和 delta 事件。
  6. codex-rs/exec-server/README.md：远端环境时看 process/filesystem RPC 生命周期。

Apply Patch 阅读顺序

  1. codex-rs/core/src/tools/handlers/apply_patch_spec.rs：看 custom/freeform 工具规格。
  2. codex-rs/core/src/tools/handlers/apply_patch.rs：看 patch 解析、验证、环境选择、diff consumer、权限路径计算。
  3. codex-rs/core/src/tools/runtimes/apply_patch.rs：看审批 key、sandbox filesystem context、真实 codex_apply_patch::apply_patch 调用。
  4. codex-rs/apply-patch/src/：看 patch grammar、parser、streaming parser、standalone executable。
  5. codex-rs/apply-patch/tests/：看 add/update/delete/move、空 patch、缺失上下文、目录删除失败等行为边界。
