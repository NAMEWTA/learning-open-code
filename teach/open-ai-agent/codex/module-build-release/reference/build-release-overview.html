关键设计决策

  • Linux 使用 musl 而非 glibc：发布二进制静态链接 musl，避免 glibc 版本兼容问题。通过 Zig cc 提供 musl C 交叉编译工具链。
  • Windows 双 ABI：exec 工具链使用 MSVC 以保证 Bazel 辅助二进制正确链接，但 lint/code 目标使用 gnullvm 以保持与仓库实际 cfg 一致。
  • Bazel 与 Cargo 并存：本地开发走 Cargo（更快、更熟悉的 Rust 生态工具），CI 和正式发布走 Bazel（统一依赖管理、远程缓存、跨平台交叉编译）。defs.bzl 中的 codex_rust_crate 宏保证了 Cargo crate 到 Bazel target 的一一映射。
  • runfiles 禁用：通过 --noenable_runfiles 减少 CI 磁盘开销，测试通过 runfiles manifest 而非符号链接树定位依赖。
  • 补丁策略：对 llvm、rules_rs、abseil-cpp、rules_cc 等上游打了 17 个补丁（patches/ 目录），主要解决 Windows gnullvm 工具链兼容和自定义 libc++ 需求。
