反向重建参考文档

本页为速查资料，覆盖 rollout_reconstruction.rs 的核心类型、关键函数签名、数据结构关系图。配合 0001 问题框架 和 0002 核心机制 使用。

1. 输出类型：RolloutReconstruction

这是 reconstruct_history_from_rollout() 的返回值，包含 Resume 所需的全部信息：

history: Vec<ResponseItem>
完整对话历史（从 replacement_history + rollout_suffix 重建）
previous_turn_settings: Option<PreviousTurnSettings>
最近一次 TurnContext 中的模型配置（model、comp_hash、realtime_active）
reference_context_item: Option<TurnContextItem>
当前会话的 TurnContext 基线。若被 Compaction 清除则为 None
world_state_baseline: Option<WorldStateSnapshot>
重建后的完整世界状态快照（从 full snapshot + patches 还原）
window_number: u64
上下文窗口编号（递增计数）
first_window_id: Option<Uuid>
首个窗口的 UUID v7
previous_window_id: Option<Uuid>
前一窗口的 UUID v7
window_id: Option<Uuid>
当前窗口的 UUID v7

2. 中间类型：ActiveReplaySegment

反向扫描期间累积每个 turn 段信息的临时结构：

turn_id: Option<String>
当前 turn 的标识符，由 TurnStarted/TurnComplete 事件提供
counts_as_user_turn: bool
是否包含用户消息边界（UserMessage event 或 InterAgentCommunication）
previous_turn_settings
本 turn 的 TurnContext 中的模型配置
reference_context_item: TurnReferenceContextItem
三段状态：NeverSet（未遇到）→ Latest(TurnContextItem)（已设置）→ Cleared（被 compaction 清除）
world_state_replay
收集本 turn 内所有 WorldState 和 Compaction items 供后续 replay
base_replacement_history
若本 turn 内 Compaction 携带 replacement_history，则记录之
window: Option<ReconstructedWindow>
上下文窗口信息（number、first_id、previous_id、id）

3. TurnReferenceContextItem 三段状态机

enum TurnReferenceContextItem {
    NeverSet,                        // 尚未遇到任何 TurnContext 或 Cleared
    Cleared,                         // 曾被 Compaction 清除
    Latest(Box<TurnContextItem>),     // 当前有效的 TurnContext 基线
}

状态转换规则：

  • NeverSet + 遇到 TurnContext → Latest(...)
  • NeverSet + 遇到 Compacted → Cleared
  • Latest(...) + 遇到 Compacted → Cleared（不覆盖已有值，因为反向扫描从新往旧——一旦看到 Cleared 就不应再回到 Latest）
  • 实际上代码中仅在 matches!(..., NeverSet) 时才写入，因此 Latest 一旦设置就不会被 Cleared 反向覆盖（除非先看到 Cleared = 更旧的数据被抛弃）

4. finalize_active_segment 逻辑

将累积的 segment 信息写入全局状态变量，同时处理 rollback 跳过：

  1. 若 pending_rollback_turns > 0 且 segment 包含用户 turn → 减少计数并丢弃该 segment
  2. 将 world_state_replay 追加到全局 world_state_replay 列表
  3. 若全局 base_replacement_history 尚未设置且 segment 中有 → 记录之
  4. 若全局 window 尚未设置且 segment 中有 → 记录之
  5. 若全局 previous_turn_settings 尚未设置且 segment 是 user turn → 记录之
  6. 若全局 reference_context_item 为 NeverSet 且 segment 中有（user turn 或 Cleared）→ 记录之

6. 已知局限与改进方向

局限描述改进方向
全量 eager load 当前 load_rollout_items 将整个 JSONL 文件读入内存 Vec<RolloutItem> 改为 lazy reverse reader，流式从文件尾部反向读取，避免大文件内存压力
legacy compaction 双循环 没有 replacement_history 的 legacy compaction 需要在正向 replay 阶段额外处理，且需清 reference_context_item 逐步淘汰 legacy 格式，移除备选路径和 saw_legacy_compaction_* 标志
无增量更新 每次 resume 都完整扫描并重建，即便只新增了几行 可记录上次扫描位置（文件 offset），resume 时只扫描新增部分
Window number fallback 若 rollout 中没有 Compaction，window_number 回退为 Compaction 计数 SessionMeta 中携带初始 window 信息可缓解，但并非所有 session 都有
