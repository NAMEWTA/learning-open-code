状态、模型提供商与后端通信参考

用途：作为 L1 源码阅读地图。短课见 0001 状态与后端通信模块导览；上层架构见 L0 项目总览参考。

模块定位

这 10 个 crate 构成 Codex 连接外部世界的底层基础设施。它们位于 core 之下、网络/文件系统之上，按职责分为四层：状态持久化层把 session 数据写入 JSONL rollout 文件和 SQLite；认证密钥层完成 OAuth 登录并将令牌安全存入平台密钥链；模型 API 层抽象 provider 差异并封装 HTTP 通信；配置遥测层从云端拉取配置并记录 trace bundle 供分析。

认证方式枚举

认证方式CodexAuth 变体来源适用场景
API KeyApiKey(ApiKeyAuth)环境变量 OPENAI_API_KEY 或交互式输入标准 OpenAI API 访问
ChatGPT 登录Chatgpt(ChatgptAuth)OAuth 设备码流程 → codex access tokenChatGPT 订阅用户
ChatGPT Auth TokensChatgptAuthTokens(...)浏览器提取的 session token高级用户手动导入
Agent IdentityAgentIdentity(AgentIdentityAuth)ChatGPT 环境 agent identity JWT托管 Codex agent 身份
Personal Access TokenPersonalAccessToken(...)用户生成的 PATCI/CD 非交互环境
Bedrock API KeyBedrockApiKey(BedrockApiKeyAuth)交互式输入Amazon Bedrock 托管模型

以上 6 个变体即为 CodexAuth 枚举的全部成员。ExternalAuth 不属于 CodexAuth 枚举，而是通过 config.toml 中 auth.command 提供的外部令牌机制，由 BearerTokenRefresher 独立管理，走单独的令牌刷新与注入路径。

认证令牌生命周期

┌─ 登录阶段 ─────────────────────────────────────────────┐ │ codex login │ │ → LoginServer::bind (随机端口，接收 OAuth 回调) │ │ → request_device_code (向 /oauth/device_authorization) │ │ → webbrowser::open (打开浏览器授权页面) │ │ → poll token endpoint (轮询直到用户授权或超时) │ │ → save_auth → KeyringStore::save → 平台密钥链 │ └──────────────────────────────────────────────────────────┘ ┌─ 使用阶段 ─────────────────────────────────────────────┐ │ AuthManager::auth() │ │ → 检查内存缓存 (仍然有效?) │ │ → KeyringStore::load → 从平台密钥链读取 │ │ → 检查过期时间 (需要刷新?) │ │ → refresh token → 更新 KeyringStore::save │ │ → 返回 CodexAuth │ │ │ │ ModelProvider::auth() │ │ → AuthManager::auth() │ │ → resolve_provider_auth() │ │ → BearerAuthProvider → Authorization: Bearer <token> │ └──────────────────────────────────────────────────────────┘ ┌─ 登出阶段 ─────────────────────────────────────────────┐ │ codex logout │ │ → logout_with_revoke (向 /oauth/revoke 撤销令牌) │ │ → KeyringStore::delete → 从平台密钥链删除 │ │ → 清除内存缓存 │ └──────────────────────────────────────────────────────────┘

Thread 持久化读写路径

┌─ 写入路径 ─────────────────────────────────────────────┐ │ Thread 创建: │ │ ThreadStore::create_thread │ │ → LocalThreadStore::create_thread │ │ → LiveThread::start (创建 live writer) │ │ → RolloutRecorder::new (初始化压缩 worker) │ │ │ │ Turn 事件追加: │ │ ThreadStore::append_items │ │ → policy::persisted_rollout_items (过滤需持久化事件) │ │ → RolloutRecorder::append (序列化为 JSONL) │ │ → 写入 rollout 文件 │ │ │ │ Flush / Shutdown: │ │ ThreadStore::flush_thread │ │ → LiveWriter::flush (确保数据落盘) │ │ ThreadStore::shutdown_thread │ │ → LiveWriter::close (关闭文件、停止压缩 worker) │ └──────────────────────────────────────────────────────────┘ ┌─ 读取路径 ─────────────────────────────────────────────┐ │ 列表查询: │ │ ThreadStore::list_threads │ │ → LocalThreadStore::list_threads │ │ → rollout::get_threads (文件系统扫描) │ │ → 按 RolloutConfig 过滤 (cwd/provider/source) │ │ → 返回 ThreadsPage (含 cursor 分页) │ │ │ │ 详情读取: │ │ ThreadStore::read_thread │ │ → LocalThreadStore::read_thread │ │ → rollout::read_head_for_summary (读 header) │ │ → 可选: 读取完整历史 items │ │ │ │ 历史加载 (resume): │ │ ThreadStore::load_history │ │ → LiveThread::start (重新打开 rollout 文件) │ │ → 流式读取已有 items │ │ → 返回 StoredThreadHistory 供 core 重建会话 │ │ │ │ 搜索: │ │ ThreadStore::search_threads │ │ → rollout::search_rollout_matches (全文搜索) │ │ → rollout::first_rollout_content_match_snippet │ │ → 返回 ThreadSearchPage │ └──────────────────────────────────────────────────────────┘
