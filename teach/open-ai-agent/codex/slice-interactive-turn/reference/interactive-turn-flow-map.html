事件流快照

一次成功的交互 turn 中，从用户按 Enter 到看到模型回答，核心事件序列如下（简化版）：

  1. TUI 端：用户按 Enter → InputResult::Submitted → ChatWidget::submit_op(AppCommand::UserInput)
  2. TUI 端：AppCommand 进入 app-server session → app-server 内部启动 thread（若首次）并注入 user input
  3. Core 端：CodexThread 收到 input → run_turn() 启动
  4. Core 端：EventMsg::TurnStarted 产出 → 经 app-server 转 ServerNotification::TurnStarted
  5. Core 端：run_sampling_request() → ModelClientSession 发送 Responses API 请求
  6. 网络：SSE 事件流返回（reasoning text → agent message delta）
  7. Core 端：每个 SSE chunk → EventMsg::AgentMessageContentDelta
  8. App-server：转为 ServerNotification::AgentMessageContentDelta → 推送 TUI
  9. TUI 端：ChatWidget 更新 active cell（markdown 渲染 + chunking 控制）
  10. Core 端：模型返回 finished → EventMsg::TurnComplete
  11. App-server：转为 ServerNotification::TurnCompleted → 推送 TUI
  12. TUI 端：ChatWidget 提交 active cell 为 history cell，bottom pane 恢复 composer

典型异常路径

异常类型触发条件处理位置用户可见效果
模型返回错误 API 返回 4xx/5xx client.rs → handle_retryable_response_stream_error() ChatWidget 顶部显示红色错误信息
流中断 WebSocket 断开 / 网络异常 client.rs → WebSocket 重连 / HTTP fallback 自动重试，失败后显示错误
Turn 被中断 用户按 Ctrl+C turn.rs → cancellation_token 触发 流停止、active cell 提交、恢复 composer
Token 超限 上下文窗口不足 turn.rs → run_pre_sampling_compact() 自动 compaction 后重试，或提示 token 不足
