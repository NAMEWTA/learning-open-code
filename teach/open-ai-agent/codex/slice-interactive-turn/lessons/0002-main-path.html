主成功路径详解：从输入到回答

学习目标：15 分钟内理解一次成功 turn 中用户输入提交、模型采样请求、SSE 事件响应和 TUI 增量渲染四个关键环节的代码流转。
依赖：0001 链路地图（请先完成时序图记忆）

本课聚焦 0001 链路地图 中步骤 [4]～[11] 的主成功路径。源码集中在三个文件：tui/src/chatwidget.rs（用户提交）、core/src/session/turn.rs（模型采样循环）、core/src/client.rs（SSE 流管理）。

1. 用户输入提交：ChatWidget → CodexThread

ChatWidget 不直接调用 CodexThread。它通过 submit_op() 将 AppCommand::UserInput 发送到内部 channel，由 App 事件循环中的 handle_event(AppEvent::CodexOp) 统一分发。这条路径之所以存在，是因为 TUI 需要保证事件顺序——用户可能在 turn 运行中继续输入，这些输入进入 pending_steers 队列而非立即发送。 

// tui/src/chatwidget.rs — ChatWidget::submit_op 简化版
pub(crate) fn submit_op<T>(&mut self, op: T) -> bool
where T: Into<AppCommand>,
{
    let op: AppCommand = op.into();
    self.prepare_local_op_submission(&op);
    match &self.codex_op_target {
        CodexOpTarget::Direct(codex_op_tx) => {
            if let Err(e) = codex_op_tx.send(op) {
                tracing::error!("failed to submit op: {e}");
                return false;
            }
        }
        CodexOpTarget::AppEvent => {
            self.app_event_tx.send(AppEvent::CodexOp(op));
        }
    }
    true
}

当 ChatWidget 运行在 App 上下文中时，codex_op_target 为 AppEvent，命令进入 TUI 内部事件总线。App 的 handle_event 将其转发给 app_server_session，后者驱动 CodexThread 的 turn 启动。

2. 模型采样循环：run_turn 的四件事

run_turn()（位于 core/src/session/turn.rs）是单次 turn 的核心编排函数。一次成功的纯文本对话只走四次关键步骤。代码中大量逻辑（tool execution、MCP invocation、compaction）在纯文本场景下不触发。 

// core/src/session/turn.rs — run_turn 的核心循环骨架（简化）
pub(crate) async fn run_turn(
    sess: Arc<Session>,
    turn_context: Arc<TurnContext>,
    input: Vec<TurnInput>,
    prewarmed_client_session: Option<ModelClientSession>,
    cancellation_token: CancellationToken,
) -> CodexResult<Option<String>> {
    // ① 如需，执行预先 compaction
    let _ = run_pre_sampling_compact(&sess, &turn_context, &mut client_session).await;

    // ② 捕获第一个 step context（上下文片段 + 可用工具）
    let first_step_context = sess
        .capture_step_context(Arc::clone(&turn_context)).await;

    // ③ 循环：每次采样 → 处理结果 → 决定是否继续
    loop {
        let step_context = sess.capture_step_context(Arc::clone(&turn_context)).await;

        // 发送 Responses API 请求并接收事件流
        match run_sampling_request(/*...*/).await {
            Ok((output, _input)) => {
                if output.needs_follow_up {
                    // 模型要求调用工具 → 执行工具 → 继续循环
                    continue;
                }
                // ④ 模型给出最终回答 → 结束 turn
                break;
            }
            Err(err) => { /* 错误处理 */ break; }
        }
    }
}

理解这四步就够了：compact（管理上下文窗口）→ step_context（构建本次请求的上下文片段和工具列表）→ sampling（发送 API 请求并接收 SSE 流）→ 判断是否继续（工具调用继续循环，最终回答则结束）。

3. SSE 事件流转与异常处理

模型返回的 SSE（Server-Sent Events）流在 client.rs 中被解析为 ResponseEvent 枚举。纯文本对话中最常见的两种事件是：

  • ResponseEvent::ContentDelta — 模型输出的文本增量（逐 token）
  • ResponseEvent::Finished — 模型完成本次响应

每个 ContentDelta 经过 handle_non_tool_response_item()（stream_events_utils.rs）转换为 EventMsg::AgentMessageContentDelta，通过 app-server 推送到 TUI。ChatWidget 将其追加到当前 active_cell 的 markdown 流中。 

// core/src/stream_events_utils.rs — SSE 文本事件处理（简化）
fn handle_non_tool_response_item(
    ctx: &HandleOutputCtx,
    item: &ResponseItem,
) -> Option<Vec<EventMsg>> {
    match item {
        ResponseItem::Message { content, .. } => {
            let text = raw_assistant_output_text_from_item(item)?;
            Some(vec![EventMsg::AgentMessageContentDelta(
                AgentMessageContentDeltaEvent {
                    delta: text,
                    item_id: item.id().to_string(),
                    turn_id: ctx.turn_context.turn_id.to_string(),
                },
            )])
        }
        _ => None,
    }
}

一条异常路径：Turn 被中断

当用户在模型回答过程中按 Ctrl+C，ChatWidget 发出 AppCommand::Interrupt，最终触发 run_turn() 中的 cancellation_token。这导致 WebSocket 流被取消，turn 循环退出，active cell 提交为 history cell（标记"已中断"），bottom pane 恢复 composer。 

// tui/src/chatwidget.rs — 中断处理（简化）
fn prepare_local_op_submission(&mut self, op: &AppCommand) {
    if let AppCommand::Interrupt { behavior } = op
        && self.turn_lifecycle.agent_turn_running
    {
        if *behavior == InterruptBehavior::RestorePromptIfNoOutput {
            self.arm_cancel_edit();
        }
        if let Some(controller) = self.stream_controller.as_mut() {
            controller.clear_queue(); // 清空流式缓冲
        }
        self.clear_active_stream_tail(); // 清除渲染中的流尾
        self.request_redraw(); // 触发重新渲染
    }
}

4. 检索练习

判断题：以下哪种情况会导致 run_turn() 的循环继续而非结束？

  1. 模型返回 ResponseEvent::Finished
  2. 模型返回 ResponseEvent::FunctionCall（需要调用工具）
  3. 用户按 Ctrl+C 中断
  4. cancellation_token 被取消
参考答案：只有选项 2 会导致循环继续——模型请求函数调用时，run_turn() 会执行工具并将结果送回模型，进入下一轮采样循环。选项 1 结束 turn；选项 3 和 4 都会导致提前退出。
