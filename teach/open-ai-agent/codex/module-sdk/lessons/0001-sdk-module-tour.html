L1 · module-sdk · 15 分钟短课

Python 与 TypeScript SDK 都是在给 Codex CLI 套稳定入口

本课只训练一个判断：看到 SDK 调用时，你能分辨它走的是 Python 的 app-server JSON-RPC 长连接，还是 TypeScript 的 exec JSONL 子进程流。

学习目标：识别 SDK 的两条运行时路径 主要来源：sdk/python/README.md 与 sdk/typescript/README.md 配套参考：sdk-overview

1. 模块职责

sdk/python 和 sdk/typescript 的共同职责，是把 Codex agent 嵌入应用和工作流：创建 thread、运行 turn、读取 final response、消费流式事件，并传入模型、sandbox、工作目录、结构化输出等选项。它们不是模型执行核心，也不直接实现文件编辑、工具调用或推理循环；这些仍由 Codex CLI / Rust 运行时完成。

差异在运行时边界。Python SDK 依赖 openai-codex-cli-bin，启动 codex app-server --listen stdio://，再通过 typed JSON-RPC 请求和通知队列维持长连接。TypeScript SDK 依赖 npm 侧的 @openai/codex CLI binary，每次 turn spawn codex exec --experimental-json，从 stdout 逐行读取 JSONL 事件。

2. 架构位置

应用代码 ↓ SDK public API Python: CodexClient + MessageRouter + app-server JSON-RPC TypeScript: CodexExec + codex exec JSONL ↓ Codex CLI / app-server / exec Rust core、模型、工具、文件系统、MCP、sandbox

放回 L0 项目地图 或 00-overview 参考，SDK 位于“外部应用 / 自动化脚本”和 Codex CLI 之间。它的价值是给宿主语言一个小而稳定的门面：Python 偏完整会话控制和账户登录；TypeScript 偏轻量封装 exec、事件 union 和 Node 环境控制。

3. 最小调用序列

Python 的同步写法先构造客户端，再显式 start thread，最后从 turn handle 拉事件：

from openai_codex import Codex
from openai_codex.generated.v2_all import AgentMessageDeltaNotification

with Codex() as codex:
    thread = codex.thread_start()
    turn = thread.turn("Explain this repository in three bullets.")
    for event in turn.stream():
        if isinstance(event.payload, AgentMessageDeltaNotification):
            print(event.payload.delta, end="")
    result = thread.run("Now summarize the risks.")
    print(result.final_response)

TypeScript 的同一心智模型是 startThread，然后选择 run() 缓冲结果或 runStreamed() 消费事件：

import { Codex } from "@openai/codex-sdk";

const codex = new Codex();
const thread = codex.startThread();
const streamed = await thread.runStreamed("Explain this repository in three bullets.");

for await (const event of streamed.events) {
  if (event.type === "item.completed") console.log(event.item);
}

const result = await thread.run("Now summarize the risks.");
console.log(result.finalResponse);

暂停回忆

不回看上文，回答两个问题：第一，Python SDK 为什么需要 MessageRouter？第二，TypeScript SDK 的 runStreamed() 事件从哪里来？

参考答案：Python 只有一个 app-server stdout 读线程，JSON-RPC response、login 通知、turn 通知会交错出现，MessageRouter 按 request id、login id、turn id 分发到等待者。TypeScript 的事件来自 CodexExec.run() spawn 的 codex exec --experimental-json，Thread.runStreamedInternal() 逐行解析 stdout JSONL 并 yield ThreadEvent。

如果某个概念还模糊，下一次可以直接追问 agent：“带我沿着 Python thread.run() 或 TypeScript runStreamed() 逐行读源码”。
