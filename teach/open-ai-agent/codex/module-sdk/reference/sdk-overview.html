Python codex app-server --listen stdio://；typed JSON-RPC request / notification。 同步与 async 客户端、账户登录、thread 生命周期、turn 启动 / steering / interrupt、通知流和结果收集。 pydantic>=2.12、openai-codex-cli-bin==0.137.0a4。

内部分层与关键流程

层PythonTypeScript
公开门面api.py 暴露 Codex、Thread、TurnHandle。codex.ts 和 thread.ts 暴露 Codex、Thread。
运行时进程client.py resolve pinned binary，spawn app-server --listen stdio://。exec.ts resolve npm native package，spawn exec --experimental-json。
协议转换Pydantic generated models、_to_wire_input()、_sandbox_policy()、_approval_mode_settings()。CLI args、TOML --config serialization、output schema temp file、input normalize。
事件路由MessageRouter 按 response id、login id、turn id、thread goal route 分流。runStreamedInternal() 逐行 JSON.parse，更新 thread id 并 yield。
结果收集_collect_turn_result() 从 item completed、usage updated、turn completed 组装 TurnResult。Thread.run() 从 item.completed 和 turn.completed 组装 Turn。

最小调用序列对照

Python:
Codex()
  -> CodexClient.start()
  -> initialize()
  -> thread_start()
  -> Thread.turn(input)
  -> turn_start()
  -> TurnHandle.stream()
  -> MessageRouter.next_turn_notification()
  -> _collect_turn_result() 或调用者自行消费

TypeScript:
new Codex()
  -> new CodexExec()
  -> startThread()
  -> Thread.runStreamed(input)
  -> CodexExec.run()
  -> spawn codex exec --experimental-json
  -> parse stdout JSONL as ThreadEvent
  -> Thread.run() 可在此基础上缓冲结果

源码阅读顺序

  1. 先读 sdk/python/README.md 和 sdk/typescript/README.md，确认用户体验。
  2. 读 Python __init__.py 与测试 test_public_api_signatures.py，确认根导出不是随意暴露 internals。
  3. 读 Python api.py 的 Codex.thread_start()、Thread.turn()、TurnHandle.stream()、Thread.run()。
  4. 下钻 Python client.py、_message_router.py、_run.py，理解长连接、路由和结果收集。
  5. 读 TypeScript index.ts、codex.ts、thread.ts，确认公开面和 run / runStreamed 差异。
  6. 下钻 TypeScript exec.ts、events.ts、items.ts，理解 CLI 参数、binary resolution、JSONL event union。
  7. 最后抽查 Python tests/test_app_server_streaming.py 与 TypeScript tests/runStreamed.test.ts、tests/exec.test.ts，用测试验证心智模型。
