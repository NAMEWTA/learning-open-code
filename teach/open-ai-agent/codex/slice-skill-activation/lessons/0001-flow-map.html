链路地图：从文件系统扫描到模型上下文注入

学习目标：15 分钟内画出 Skill 从 .codex/skills/ 目录被发现到成为系统提示词 `` 片段的完整时序图。

一句话定位：本切片沿 文件系统扫描 → loader 解析 → SkillsService 快照 → injection/render 组装 → 系统提示词注入 逐层追踪。前置知识：L1 扩展机制模块导览。

1. 时序总览

下面这张 mermaid 时序图覆盖了 Skill 激活的 7 个阶段。建议先扫一遍全图，建立心理模型后再逐层下钻。 

sequenceDiagram
    participant FS as 文件系统
    participant ROOT as skill_roots()
    participant DISCOVER as discover_skills_under_root
    participant PARSE as parse_skill_file
    participant SVC as SkillsService
    participant RENDER as build_available_skills
    participant CTX as Session 系统提示词
    ROOT->>ROOT: 收集 5 种 root 来源 + dedupe 去重
    loop 每个 SkillRoot
        ROOT->>DISCOVER: load_skills_under_root(root)
        DISCOVER->>FS: BFS 扫描目录 (深度≤6, 目录≤2000)
        FS-->>DISCOVER: 发现 SKILL.md 文件列表
        loop 每个 SKILL.md
            DISCOVER->>PARSE: parse_skill_file(path)
            PARSE->>FS: 读取文件内容
            PARSE->>PARSE: extract_frontmatter + serde_yaml
            alt 解析成功
                PARSE-->>DISCOVER: SkillMetadata
            else 解析失败 (scope != System)
                PARSE-->>DISCOVER: SkillError (记录但不阻断)
            end
        end
        DISCOVER-->>ROOT: SkillLoadOutcome {skills, errors}
    end
    ROOT-->>SVC: SkillLoadOutcome 汇总
    SVC->>SVC: filter + disabled_paths 过滤 + 写入缓存
    SVC-->>RENDER: HostSkillsSnapshot
    RENDER->>RENDER: 按 scope 排序、预算计算、描述截断或裁减
    RENDER-->>CTX: AvailableSkills {skill_root_lines, skill_lines}
    CTX->>CTX: AvailableSkillsInstructions::from_available_skills
    CTX->>CTX: developer_sections.push(render())
    Note over FS,CTX: SkillsWatcher 监听文件变化 → 缓存清空 → SkillsChanged 通知

2. 第 1-2 层：Skill Root 收集与目录扫描

skill_roots() 函数（loader.rs L237-254）从 5 种来源收集 Skill 搜索根目录：Project 层（.codex/skills/）、User 层（$CODEX_HOME/skills/ 和 $HOME/.agents/skills/）、System 层（$CODEX_HOME/skills/.system）、Admin 层（/etc/codex/skills/），再加上 Plugin 和 extra_roots。最后 dedupe_skill_roots_by_path 去重。 

// loader.rs - skill_roots() 的核心结构
for layer in config_layer_stack.get_layers(..) {
    match &layer.name {
        ConfigLayerSource::Project { .. } => {
            roots.push(SkillRoot { path: config_folder.join("skills"),
                scope: SkillScope::Repo, .. }); }
        ConfigLayerSource::User { .. } => {
            roots.push(SkillRoot { path: config_folder.join("skills"),
                scope: SkillScope::User, .. });
            roots.push(SkillRoot { path: home_dir
                .join(".agents").join("skills"),
                scope: SkillScope::User, .. });
            roots.push(SkillRoot { path: system_cache_root_dir(..),
                scope: SkillScope::System, .. }); }
        ConfigLayerSource::System { .. } => {
            roots.push(SkillRoot { path: config_folder.join("skills"),
                scope: SkillScope::Admin, .. }); }
        _ => {}
    }
}

discover_skills_under_root() 对每个 root 做 BFS 扫描：跳过隐藏文件（. 开头）、跳过符号链接（System scope）、支持插件 manifest 检测。深度限制 6 层，目录上限 2000 个。只收集文件名为 SKILL.md 的文件。

3. 第 3 层：SKILL.md 解析与异常路径

parse_skill_file() 读文件后，先调 extract_frontmatter() 找 YAML 分隔符 ---，再用 serde_yaml 反序列化为 SkillFrontmatter { name, description, .. }。解析失败时回退到 repair_frontmatter_scalar_fields 尝试修复（如给含冒号的字段加单引号）。同时加载可选的 agents/openai.yaml 元数据文件。 

// loader.rs - parse_skill_file 核心流程 (精简)
let contents = fs.read_file_text(&path_uri, None).await?;
let ParsedSkillFrontmatter { name, description, .. }
    = parse_skill_frontmatter_metadata_inner(&contents,
        || default_skill_name(path))?;
let name = namespaced_skill_name(fs, path, &base_name, ..).await;
let LoadedSkillMetadata { interface, dependencies, policy }
    = load_skill_metadata(fs, path, plugin_root).await;
// 验证长度、构建 SkillMetadata

异常路径：Skill 解析失败时不阻断整个加载流程。Scope 为 System 的 Skill 错误被静默忽略（不写入 outcome.errors），非 System 的 Skill 错误记录为 SkillError 供上层消费。如果 skills 根目录不存在，discover_skills_under_root 返回空的 skill_files 列表，整个 root 被跳过。

4. 第 4-5 层：快照、渲染与系统提示词注入

SkillsService 对 SkillLoadOutcome 做 product 过滤和 disabled_paths 解析后写入缓存。当 Session 组装系统提示词时（session/mod.rs L3266-3291），调用 build_available_skills() → AvailableSkillsInstructions::from_available_skills() → developer_sections.push()。最终以 <skill-instructions> 标签注入模型上下文。SkillsWatcher 监听文件变化，触发缓存清空和 SkillsChanged 通知。 

// session/mod.rs - 系统提示词组装点 (精简)
if turn_context.config.include_skill_instructions {
    let available_skills = build_available_skills(
        turn_context.turn_skills.snapshot.outcome(),
        default_skill_metadata_budget(turn_context.model_info.context_window),
        SkillRenderSideEffects::ThreadStart { .. },
    );
    if let Some(available_skills) = available_skills {
        let skills_instructions = AvailableSkillsInstructions::from_available_skills(
            &available_skills,
            turn_context.model_info.include_skills_usage_instructions,
        );
        developer_sections.push(skills_instructions.render());
    }
}

检索练习：如果用户的 $HOME/.codex/skills/my-skill/SKILL.md 文件存在但未被模型识别，请按本链路从后往前反推可能的问题层。如果 .codex/skills/ 目录存在但为空，Skill 加载是否会报错？

参考答案：
反推链路：模型没有 skill → developer_sections 为空 → build_available_skills 返回 None（allowed_skills 为空）→ outcome.skills 中无此 skill → parse_skill_file 返回 Err 且 scope 为 System 被静默吞掉 OR skill 被 disabled_paths 过滤 → SKILL.md 解析失败（MissingFrontmatter / InvalidYaml / MissingField description）。
.codex/skills/ 为空目录：不会报错。discover_skills_under_root 扫描结果为 0 个 SKILL.md，load_skills_under_root 正常返回空 outcome，只是模型不会有任何可用 Skill。
还有疑问时，向 agent 追问具体日志行，让它带你定位到 loader.rs 的精确阶段。
