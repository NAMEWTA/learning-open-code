Core Runtime L1 参考地图

用途：快速定位 `codex-core` 的职责、公开接口、内部分层、关键依赖与后续 L2/L3 候选。事实来源来自当前本地源码快照。

模块职责

职责 说明 主要源码
线程生命周期 创建、恢复、fork、subagent、线程列表、元数据更新、线程关闭。 codex-rs/core/src/thread_manager.rs、codex-rs/core/src/codex_thread.rs
请求与事件通道 接收 `Op`，产出 `Event` / `EventMsg`，维持 submission 与 event 的相关 id。 codex-rs/protocol/src/protocol.rs
Session/Turn 编排 维护 session 状态、active turn、输入队列、turn context、context window、MCP runtime 与 rollout。 codex-rs/core/src/session/
任务类型 统一普通 turn、review、compact、用户 shell 命令的异步任务生命周期。 codex-rs/core/src/tasks/
模型上下文与后端 构造 model-visible fragments，管理 provider/auth/streaming，会把模型响应映射成 turn item。 codex-rs/core/src/context/、codex-rs/core/src/client.rs、codex-rs/core/src/event_mapping.rs

整体位置

入口层：TUI / exec / app-server / SDK
        ↓
公共 API：core-api re-export ThreadManager、CodexThread、Op、EventMsg 等
        ↓
运行时：codex-core 管理 thread、session、turn、context、task、client
        ↓
边界 crate：protocol、tools、mcp、skills、config、sandboxing、model-provider、rollout、state

读源码时优先确认“这是公共合同、运行时编排，还是边界能力”。公共合同属于 `codex-rs/protocol/src/`；面向外部调用者的窄门面属于 `codex-rs/core-api/src/lib.rs`；运行时状态和调度属于 `codex-rs/core/src/`；工具、沙箱、模型目录、MCP、skills、配置等不要强行归入 core。需要回到全局视角时，先看 L0 总览参考。

对外接口与公共类型清单

类型或函数 来自 阅读用途
ThreadManager、StartThreadOptions、NewThread、ForkSnapshot codex-rs/core/src/thread_manager.rs 线程创建、恢复、fork、subagent 和共享服务装配。
CodexThread、ThreadConfigSnapshot、CodexThreadSettingsOverrides codex-rs/core/src/codex_thread.rs 提交请求、读取事件、注入当前 turn、读取配置快照、操作后台终端。
Op、Event、EventMsg、SessionConfiguredEvent codex-rs/protocol/src/protocol.rs 跨 TUI、app-server、SDK 和 core 的稳定消息合同。
UserInput、ResponseItem、PermissionProfile、AskForApproval codex-rs/protocol/src/ 用户输入、模型项、权限和审批策略的共享结构。
ModelClient、ModelClientSession codex-rs/core/src/client.rs session-scoped provider/auth 状态和 turn-scoped streaming 请求。
TurnContext codex-rs/core/src/session/turn_context.rs 单个 turn 的模型、环境、权限、MCP、skills、动态工具、telemetry 快照。
parse_turn_item codex-rs/core/src/event_mapping.rs 把 Responses API item 转为 UI/app-server 可展示的 turn item。

内部分层结构

层 目录或文件 职责边界
导出面 codex-rs/core/src/lib.rs 显式 re-export core 对外类型，并把多数实现模块保持私有。
线程管理 codex-rs/core/src/thread_manager.rs、codex-rs/core/src/codex_thread.rs 外部入口持有 `Arc<CodexThread>`，内部绑定到 `Session` 和 event channel。
Session 状态 codex-rs/core/src/session/session.rs、codex-rs/core/src/session/mod.rs 保存配置、权限、环境、服务、输入队列、active turn 和 lifecycle hooks。
Turn 构造 codex-rs/core/src/session/turn.rs、codex-rs/core/src/session/turn_context.rs 构造模型请求上下文，执行普通 turn，追踪 token/context window。
任务抽象 codex-rs/core/src/tasks/mod.rs `SessionTask` 统一 run/abort 生命周期，具体任务分布在 regular/review/compact/user_shell。
上下文片段 codex-rs/core/src/context/ 所有 model-visible fragment 的定义区。AGENTS.md 要求这些片段必须 bounded 且有 hard cap。

关键依赖边界

依赖组 代表 crate 说明
协议与配置 codex-protocol、codex-config、codex-features 跨模块类型、配置分层、feature flag，不应私藏在 core。
模型与认证 codex-api、codex-model-provider、codex-login provider 解析、认证、Responses API transport 与 fallback。
工具与安全 codex-tools、codex-sandboxing、codex-execpolicy 工具 schema、命令执行、审批和平台 sandbox 边界。
扩展与上下文 codex-mcp、codex-core-skills、codex-core-plugins、codex-context-fragments MCP、skills、plugins 和上下文片段抽象。
持久化与状态 codex-rollout、codex-thread-store、codex-state 会话 JSONL/SQLite、线程元数据和状态数据库。

简单调用示例

let new_thread = thread_manager.start_thread(config).await?;
let thread = new_thread.thread;

let submission_id = thread.submit(Op::from(vec![UserInput::Text {
    text: "Summarize this repository".to_string(),
    text_elements: Vec::new(),
}])).await?;

loop {
    let event = thread.next_event().await?;
    if matches!(event.msg, EventMsg::TurnComplete(_)) {
        break;
    }
}

这段示例只表达调用形状。真实 app-server 和 TUI 还会处理 trace、client message id、settings overrides、审批响应、工具事件、token usage、thread store 与关闭流程。

后续 L2/L3 候选

层级 候选主题 建议入口
L2 interactive-turn：TUI 提交一次用户输入到 core，再流式渲染事件。 CodexThread::submit、Session::spawn_task、RegularTask
L2 app-server-turn：JSON-RPC thread/turn API 如何驱动同一套 core。 core-api、app-server、protocol::EventMsg
L2 tool-call-execution：模型工具调用如何进入 tools、sandboxing 和审批。 codex-rs/core/src/tools/、codex-rs/tools、exec_policy
L2 skill-activation：skills 如何被发现、预算化并注入 model context。 codex-rs/core/src/skills.rs、codex-rs/core/src/context/available_skills_instructions.rs
L2 session-resume-fork：rollout、thread store 与 fork snapshot 的恢复链路。 codex-rs/core/src/thread_manager.rs、codex-rs/core/src/session/rollout_reconstruction.rs
L3 Core API 参考：逐项整理 `core-api/src/lib.rs` re-export 的公共类型。 codex-rs/core-api/src/lib.rs
L3 Protocol 类型速查：`Op`、`EventMsg`、权限、用户输入、turn item。 codex-rs/protocol/src/protocol.rs、codex-rs/protocol/src/items.rs、codex-rs/protocol/src/models.rs
