Sandbox、权限与配置系统参考总览

本参考服务于 L1 主题 `module-sandbox-config`。短课入口：0001：Sandbox 配置到执行的 15 分钟导览。全局架构入口：L0 参考总览。

模块职责与架构位置

该系统位于“用户/会话配置”和“工具执行”之间，负责把安全意图变成可执行边界。上游是配置层、CLI overrides、requirements、项目 trust；下游是 shell/unified exec、网络代理、平台 sandbox helper。

用户配置 / CLI override / requirements → codex_config：解析 TOML schema → core::config：合并层、选择 profile、应用约束 → protocol：PermissionProfile 与 runtime policy 类型 → codex_sandboxing：选择并变换 sandbox 执行请求 → 平台后端：Seatbelt / Linux helper / Windows sandbox

外部调用如何进入权限系统

app-server 的 JSON-RPC 外部入口先在 codex-rs/app-server/README.md 和 codex-rs/app-server-protocol/src/protocol/ 中定义请求形状，再由 codex-rs/app-server/src/request_processors/ 转成 core config overrides 或 thread settings。换句话说，外部客户端不是直接操作 sandbox 后端，而是通过 `permissions` profile 选择或 legacy `sandboxPolicy` 覆盖进入权限系统。

外部入口权限相关字段进入点
turn/start实验性 permissions 选择本 turn 与后续 turn 的 profile；legacy sandboxPolicy 仍可用，但不能与 permissions 同时出现。codex-rs/app-server/README.md、codex-rs/app-server-protocol/src/protocol/v2/turn.rs、codex-rs/app-server/src/request_processors/turn_processor.rs
thread/start实验性 permissions 选择线程默认 profile；legacy sandbox shorthand 仍可用，响应还带 legacy sandbox 兼容投影。codex-rs/app-server/README.md、codex-rs/app-server-protocol/src/protocol/v2/thread.rs、codex-rs/app-server/src/request_processors/thread_processor.rs
thread/settings/update更新已加载 thread 的 next-turn 权限设置；permissions 与 sandboxPolicy 互斥，成功后可能触发 thread/settings/updated。codex-rs/app-server/README.md、codex-rs/app-server-protocol/src/protocol/v2/thread.rs、codex-rs/app-server/src/request_processors/turn_processor.rs
permissionProfile/list列出可选 permission profile，帮助客户端选择内置或用户定义 profile。codex-rs/app-server-protocol/src/protocol/common.rs、codex-rs/app-server/tests/suite/v2/permission_profile_list.rs
configRequirements/read读取托管配置 requirements，其中 default_permissions 等字段会约束可用权限。codex-rs/app-server-protocol/src/protocol/common.rs、codex-rs/app-server-protocol/src/protocol/v2/config.rs、codex-rs/app-server/src/request_processors/config_processor.rs

内部分层

1. TOML schema 层

codex-rs/config/src/config_toml.rs 定义 `ConfigToml`。与本主题直接相关的字段包括 `approval_policy`、`approvals_reviewer`、`shell_environment_policy`、`allow_login_shell`、`sandbox_mode`、`sandbox_workspace_write`、`default_permissions`、`permissions`。

codex-rs/config/src/permissions_toml.rs 定义 `PermissionsToml` 和 `PermissionProfileToml`。profile 支持 `extends`，父 profile 先合并，子 profile 覆盖父字段；继承会检查 undefined parent、built-in parent 限制和 cycle。边界是：可扩展的内置父 profile 主要是 :read-only 与 :workspace，不能扩展 :danger-full-access。

2. 配置合并与约束层

codex-rs/core/src/config/mod.rs 的 `Config::load_config_with_layer_stack` 是阅读主入口。它合并配置层，处理 CLI overrides，解析项目 trust，选择 legacy `sandbox_mode` 还是 profile 语法，并应用 requirements 约束。最终构造 `Config { permissions: Permissions { ... } }`。

重要判断点：互斥覆盖项会报错；如果 `[permissions]` 已定义但没有 `default_permissions` 且不是 legacy 模式，会报错；requirements 可把不允许的 profile 降级，并清空 active profile 标识。

3. 权限编译层

codex-rs/core/src/config/permissions.rs 负责从 profile 名称编译到运行态 policy。内置 profile 包括 `:read-only`、`:workspace`、`:danger-full-access`。自定义 profile 会经过 `compile_permission_profile`，将 filesystem 条目编译为 `FileSystemSandboxEntry`，将 network.enabled 编译为 `NetworkSandboxPolicy`。

内置 profile文件系统网络用途
:read-only只读通常 restricted默认保守边界
:workspaceworkspace 可写受 `sandbox_workspace_write.network_access` 或 profile network 控制常见开发模式
:danger-full-access不限制enabled显式高风险模式

4. protocol 权限模型层

codex-rs/protocol/src/config_types.rs 提供 `SandboxMode`。codex-rs/protocol/src/models.rs 提供 `PermissionProfile`、`AdditionalPermissionProfile`、`SandboxEnforcement`。codex-rs/protocol/src/permissions.rs 提供 `FileSystemAccessMode`、`FileSystemSandboxPolicy`、`NetworkSandboxPolicy`。

`FileSystemAccessMode` 的冲突优先级是 deny 高于 write，高于 read。`NetworkSandboxPolicy` 只有 restricted 与 enabled。`SandboxEnforcement` 区分 managed、disabled、external。

5. sandbox transform 与平台层

codex-rs/sandboxing/src/manager.rs 的 `SandboxManager` 提供三个关键阶段：`should_sandbox` 判断需求，`select_initial` 选择平台类型，`transform` 生成 `SandboxExecRequest`。它会合并运行时 additional permissions，加入 managed MITM CA 可读根，并为平台后端准备参数。

平台后端关键文件
macOSSeatbelt / `sandbox-exec`codex-rs/sandboxing/src/seatbelt.rs
Linux当前主体链路是 manager → sandboxing/bwrap/landlock → linux-sandbox helper；默认 bubblewrap + seccomp，legacy Landlock 可选codex-rs/sandboxing/src/bwrap.rs、codex-rs/sandboxing/src/landlock.rs、codex-rs/linux-sandbox/src/bwrap.rs、codex-rs/linux-sandbox/src/launcher.rs、codex-rs/linux-sandbox/src/proxy_routing.rs、codex-rs/linux-sandbox/README.md

最小序列：配置到执行

1. 用户设置 default_permissions / sandbox_mode / approval_policy。
2. ConfigToml 读取 TOML，ConfigLayerStack 合并本地、托管、session 层。
3. Config::load_config_with_layer_stack 解析 profile 语法并选择默认 profile。
4. permissions.rs 编译 filesystem/network，生成 PermissionProfile。
5. requirements 约束可能收窄 profile 或 approval policy。
6. Permissions::effective_permission_profile 物化 workspace roots。
7. SandboxManager::select_initial 判断是否需要平台 sandbox。
8. SandboxManager::transform 生成 SandboxExecRequest。
9. 平台后端包裹命令并执行。

关键依赖

  • `codex_config`：配置层、requirements、TOML schema、profile 解析。
  • `codex_protocol`：跨 core、exec、UI 共享的权限和配置类型。
  • `codex_sandboxing`：平台无关 sandbox 选择与命令变换。
  • `codex_network_proxy`：受限网络、domain/unix socket、managed MITM/proxy 配置。
  • `codex_windows_sandbox` 与 `codex-linux-sandbox`：平台专用执行边界。
  • `codex_execpolicy`：命令前缀策略，用于 allow/prompt/forbidden 决策。

推荐源码阅读顺序

  1. docs/config.md 与 docs/sandbox.md：确认用户文档入口。
  2. codex-rs/protocol/src/config_types.rs：先看 `SandboxMode`。
  3. codex-rs/config/src/config_toml.rs：定位 `ConfigToml` 中的 sandbox/profile 字段。
  4. codex-rs/config/src/permissions_toml.rs：理解 `[permissions]` schema、extends、network/domain/unix socket。
  5. codex-rs/core/src/config/mod.rs：读 `Config::load_config_with_layer_stack` 中 profile 选择到 `Permissions` 构造。
  6. codex-rs/core/src/config/permissions.rs：读 `builtin_permission_profile`、`compile_permission_profile_selection`、`compile_filesystem_permission`。
  7. codex-rs/protocol/src/models.rs 与 codex-rs/protocol/src/permissions.rs：理解 runtime 类型和访问判定。
  8. codex-rs/sandboxing/src/manager.rs：读 `select_initial`、`transform`、`compatibility_sandbox_policy_for_permission_profile`。
  9. codex-rs/linux-sandbox/README.md、codex-rs/sandboxing/src/seatbelt.rs、codex-rs/windows-sandbox-rs/src/unified_exec/mod.rs：按平台补细节。
  10. codex-rs/execpolicy/README.md：最后区分命令策略和 sandbox 权限。
