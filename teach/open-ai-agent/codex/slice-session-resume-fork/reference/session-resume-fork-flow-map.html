关键设计决策

  • JSONL + SQLite 双写 — JSONL 是 source of truth（完整事件日志），SQLite 是查询加速层（结构化提取）。两者独立更新，不保证强一致。
  • 反向扫描 + 提前终止 — resume 不需要解析全部 JSONL，找到最近的 replacement_history 和 metadata 后立即停止，保证长会话的 resume 性能。
  • 子树归档不保证原子性 — 父 thread 归档失败则整体失败；后代失败仅 warn。这是有意简化，避免分布式事务复杂度。
  • Ephemeral fork 无持久化 — 仅存在于 app-server 内存中，重启消失。适用于临时实验场景。
