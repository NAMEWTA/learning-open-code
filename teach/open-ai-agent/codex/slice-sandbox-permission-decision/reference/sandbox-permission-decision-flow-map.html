2. 权限 Profile 三层模型

层级Profile 名称文件系统网络典型场景
内置 :read-only:read-only:root = ReadRestricted纯代码审查
内置 :workspace:workspace:root=Read, :workspace_roots=Write, :tmpdir=WriteRestricted日常开发（默认）
内置 :danger:danger-full-accessUnrestrictedEnabled信任项目/系统管理
用户自定义任意名称TOML extends 继承链TOML network.enabled团队共享安全策略

3. execpolicy 决策矩阵

Decision含义优先级典型规则示例
Allow直接执行低allow git *
Prompt用户审批中prompt npm publish *
Forbidden完全阻塞高forbidden rm -rf /

匹配规则：多个规则命中时取最高 Decision（Forbidden > Prompt > Allow）。无规则命中时由 heuristics_fallback 决定。
