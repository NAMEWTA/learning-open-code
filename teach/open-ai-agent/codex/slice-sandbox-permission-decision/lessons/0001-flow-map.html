Sandbox 权限决策链路地图

学习目标：画出从工具调用触发到沙箱就绪的完整 6 阶段时序图，说出每层的职责边界。

前置：L1 Sandbox 配置模块导览

1. 链路全景：6 阶段决策管道

当 Codex Agent 决定执行一个 shell 命令或文件操作时，会经过一条 6 阶段的安全决策管道。每个阶段由不同模块负责，输入输出明确，边界清晰：

工具调用触发 (ToolOrchestrator) │ ▼ [1] execpolicy 策略查询 输入: 命令令牌数组 (["git", "status"]) 输出: Decision::Allow / Prompt / Forbidden 所属: codex-rs/execpolicy/src/policy.rs → Policy::check() │ ▼ [2] permissions 权限编译 输入: profile_name + SandboxWorkspaceWrite 配置 输出: (FileSystemSandboxPolicy, NetworkSandboxPolicy) 所属: codex-rs/core/src/config/permissions.rs → compile_permission_profile_selection() │ ▼ [3] 权限策略合并 (effective profile) 输入: base PermissionProfile + optional AdditionalPermissionProfile 输出: 合并后的 PermissionProfile 所属: codex-rs/sandboxing/src/policy_transforms.rs → effective_permission_profile() │ ▼ [4] sandboxing::select_initial() 沙箱类型选择 输入: &FileSystemSandboxPolicy + NetworkSandboxPolicy + SandboxablePreference + WindowsSandboxLevel + has_managed_network_requirements: bool 输出: SandboxType (None / MacosSeatbelt / LinuxSeccomp / WindowsRestrictedToken) 所属: codex-rs/sandboxing/src/manager.rs → SandboxManager::select_initial() │ ▼ [5] 平台沙箱创建 (transform) 输入: SandboxTransformRequest (命令 + 权限 + 沙箱类型) 输出: SandboxExecRequest (包装后的命令 + env + 平台沙箱参数) 所属: codex-rs/sandboxing/src/manager.rs → SandboxManager::transform() │ ▼ [6] 执行环境就绪 SandboxExecRequest 交给 exec-server 启动子进程 沙箱策略已内嵌在命令包装器中

4. 异常路径概览

决策链路中任一阶段都可能产生非正常结果。以下为两条关键异常路径（第 2 节课详解）：

  • 权限拒绝路径：execpolicy 返回 Forbidden 时，链路在阶段 [1] 终止，工具调用被直接拒绝，不进入后续沙箱创建。
  • 沙箱降级路径：select_initial() 判断不需要沙箱（SandboxablePreference::Auto + 全盘写权限），返回 SandboxType::None，跳过平台沙箱包装。

检索练习

不翻看上文，用 1-2 句话回答：should_require_platform_sandbox() 在什么条件下返回 false?（提示：涉及 FileSystemSandboxKind 的三个变体）

参考答案

should_require_platform_sandbox() 在以下条件同时满足时返回 false（前提：has_managed_network_requirements 为 false）：

  1. 文件系统策略为 ExternalSandbox（无论网络是否启用）；
  2. 网络启用且文件系统策略为 Unrestricted；
  3. 网络启用且文件系统策略为 Restricted 且 has_full_disk_write_access() 为 true。

注意：当网络未启用时，Restricted 和 Unrestricted 均返回 true（源码 policy_transforms.rs:518-522：!network_policy.is_enabled() 分支中仅 ExternalSandbox 被排除）。
