完整 contributor 清单

下面的清单来自 open-ai-agent/codex/codex-rs/ext/extension-api/src/registry.rs。读业务 extension crate 时，先判断它注册到哪一个 contributor 面，再追对应调用点。

registry 字段 / getter 注册方法 运行面
thread_lifecycle_contributors thread_lifecycle_contributor 线程创建、恢复、结束等 thread 生命周期。
turn_lifecycle_contributors turn_lifecycle_contributor 单轮 turn 的开始、结束和中间阶段。
config_contributors config_contributor 基于 host config 补充 extension 配置或派生状态。
token_usage_contributors token_usage_contributor 把扩展相关的 token 使用量纳入统计或展示。
context_contributors prompt_contributor 向模型上下文贡献 prompt/context 片段。
mcp_server_contributors mcp_server_contributor 贡献运行时 MCP server，再交给 MCP connection manager。
turn_input_contributors turn_input_contributor 在 turn input 组装阶段注入内容，例如被选中的 skill。
tool_contributors tool_contributor 贡献 native/model-visible tool 定义与处理逻辑。
tool_lifecycle_contributors tool_lifecycle_contributor 工具调用前后生命周期，可用于观察、改写或联动。
turn_item_contributors turn_item_contributor 按顺序贡献 turn item，影响会话事件流中的可见条目。
approval_review_contributors approval_review_contributor 在审批 review prompt 上给出扩展侧决策。

源码阅读顺序

  1. 先读 open-ai-agent/codex/codex-rs/ext/extension-api/src/lib.rs、open-ai-agent/codex/codex-rs/ext/extension-api/src/registry.rs，建立 contributor 概念。
  2. 读 Skills：open-ai-agent/codex/codex-rs/core-skills/src/model.rs → open-ai-agent/codex/codex-rs/core-skills/src/loader.rs → open-ai-agent/codex/codex-rs/ext/skills/src/extension.rs。
  3. 读 Plugins：open-ai-agent/codex/codex-rs/plugin/src/manifest.rs → open-ai-agent/codex/codex-rs/core-plugins/src/loader.rs → open-ai-agent/codex/codex-rs/core-plugins/src/manager.rs。
  4. 读 MCP：open-ai-agent/codex/codex-rs/core/src/mcp.rs → open-ai-agent/codex/codex-rs/codex-mcp/src/mcp/mod.rs → open-ai-agent/codex/codex-rs/codex-mcp/src/connection_manager.rs → open-ai-agent/codex/codex-rs/rmcp-client/src/stdio_server_launcher.rs。
  5. 读 Hooks：open-ai-agent/codex/codex-rs/hooks/src/lib.rs → open-ai-agent/codex/codex-rs/hooks/src/engine/discovery.rs → open-ai-agent/codex/codex-rs/hooks/src/events/pre_tool_use.rs → open-ai-agent/codex/codex-rs/core/src/hook_runtime.rs。
  6. 最后把四条线汇合到 session：open-ai-agent/codex/codex-rs/core/src/session/turn_context.rs、open-ai-agent/codex/codex-rs/core/src/session/turn.rs、open-ai-agent/codex/codex-rs/core/src/session/mcp.rs。

审计提示

排查扩展未生效时，按“清单是否被发现、配置是否启用、运行面是否贡献、turn 中是否触发”四步定位。Skills 看 catalog 和 selected entries；Plugins 看 loaded plugin 的 capability summary；MCP 看 effective server 与 startup status；Hooks 看 list_hooks、trust status、matcher 和 command output。
