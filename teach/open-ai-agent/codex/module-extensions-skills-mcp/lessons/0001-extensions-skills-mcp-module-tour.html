扩展系统导览：四类入口，一条调用链

学习目标：15 分钟内区分 Skills、Plugins、MCP、Hooks 的职责，并能复述一个最小运行序列。

在 L0 架构里，扩展系统位于 core 与 tools 的边界外侧：它把外部知识、包能力、远端工具和生命周期命令接入一次 agent turn。先回看 L0 项目地图，再把本课作为 L1 导航。 主要来源：open-ai-agent/codex/codex-rs/core-skills/src/service.rs、 open-ai-agent/codex/codex-rs/core-plugins/src/loader.rs、 open-ai-agent/codex/codex-rs/codex-mcp/src/connection_manager.rs、 open-ai-agent/codex/codex-rs/hooks/src/registry.rs。

1. 先分清四个边界

Skills 是模型可读取的能力说明，核心形态是 SKILL.md 和可选 agents/openai.yaml；它回答“何时把哪段指导注入上下文”。Plugins 是打包层，plugin.json 可同时声明 skills、MCP servers、apps、hooks；它回答“一个包贡献了哪些能力”。MCP 是工具连接层，负责启动或连接 server、列出工具、规范化模型可见工具名并路由调用。Hooks 是生命周期命令层，在用户提交、工具前后、压缩、停止等事件上运行本地或插件命令。

2. 内部分层只记三层

codex-rs/ ├── core/ │ └── session/ ← turn 组装、MCP runtime、hook 调用点 ├── core-skills/ ← SKILL.md 发现、解析、快照、显式注入 ├── core-plugins/ ← plugin 包加载、市场、store、能力装配 ├── codex-mcp/ ← MCP catalog、连接管理、工具路由 ├── rmcp-client/ ← stdio/HTTP/OAuth RMCP client ├── hooks/ ← hook schema、发现、执行、结果解析 └── ext/ ├── extension-api/ ← contributor 注册接口 ├── skills/ ← skills 作为内置 extension 接入 └── mcp/ ← hosted apps 与 executor plugin MCP 接入

读源码时不要把 ext/ 误认为“外部插件目录”。它是 Codex 自己的内置 extension crate 集合，通过 ExtensionRegistryBuilder 注册 context、MCP、tool、turn lifecycle 等 contributor。插件包本身由 plugin/ 与 core-plugins/ 解析，再把能力投射到 skills、MCP、hooks 等运行面。

3. 最小运行序列

发现扩展
  SkillsService 扫描 repo/user/system/plugin roots
  Plugins loader 解析 plugin.json 并收集 skills/MCP/hooks
  Hooks engine 从配置层和插件 hook source 发现命令
连接或注入
  ext/skills 贡献可用 skills 列表与显式 skill 内容
  ext/mcp 与 core/mcp 合成 effective MCP servers
  McpConnectionManager 启动 client 并列出工具
调用或触发
  模型 mention skill → 读取 SKILL.md 进入 turn input
  模型调用 MCP tool → 路由到对应 server/tool
  事件到达 hook 点 → dispatcher 匹配并执行命令

这条链路的关键依赖是：配置层决定启用哪些 roots、plugins、MCP、hooks；codex_protocol 提供 SkillScope、HookEventName、事件与提交结构；codex_tools 提供模型工具抽象；rmcp 和 codex_rmcp_client 处理 MCP 协议传输。

4. 检索练习

暂停回忆：下面三个问题先遮住答案回答：1. 插件是不是一种运行时工具协议？2. 模型显式选择一个 skill 后，最直接进入上下文的文件是什么？3. PreToolUse hook 和 MCP tool call 的边界在哪里？

参考答案：1. 不是，插件是包清单和能力聚合层，MCP 才是工具连接协议层。2. 主要是该 skill 的 SKILL.md 主提示内容。3. PreToolUse hook 是工具调用前的生命周期拦截点；MCP tool call 是真正把某个模型工具请求路由到 MCP server 的执行路径。后续源码阅读请打开 扩展系统参考，按“先清单、再运行面、最后触发点”的顺序继续。
