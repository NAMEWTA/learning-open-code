App-server Thread/Turn JSON-RPC 链路速查表

本文档是 链路地图 和 主路径详解 的精炼提取，用于快速查阅方法路由、源码入口和错误码。反复查阅，不做一次性阅读。

一、JSON-RPC 消息类型（rpc.rs）

类型Rust 结构体说明
RequestJSONRPCRequest含 id、method、params、trace，期待响应
NotificationJSONRPCNotification含 method、params，不期待响应
ResponseJSONRPCResponse含 id、result，成功响应
ErrorJSONRPCError含 id、error（code + message + data）

代码注释明确："We do not do true JSON-RPC 2.0, as we neither send nor expect the 'jsonrpc': '2.0' field."（rpc.rs:1-2）

七、初始化握手序列

客户端在调用 thread/start 或 turn/start 之前必须完成 initialize 握手：

客户端 → 服务端：{"id":0,"method":"initialize","params":{...}} 服务端 → 客户端：{"id":0,"result":{...}} 客户端 → 服务端：{"method":"initialized"} ← notification，当前仅进入日志 此后才可调用 thread/start、turn/start 等业务方法。 若跳过 initialize，dispatch_initialized_client_request() 返回 "Not initialized"（message_processor.rs:881）。
