1. app-server-client

源码：codex-rs/app-server-client/src/lib.rs（2319 行，含测试）， src/remote.rs， src/path.rs。 此 crate 为 CLI 界面（TUI、exec）提供统一的进程内/远程 app-server 客户端门面， 封装初始化握手、请求/通知分发、背压事件流与优雅关闭。

1.1 核心类型

AppServerEvent pub enum

pub enum AppServerEvent { Lagged { skipped: usize }, ServerNotification(ServerNotification), ServerRequest(ServerRequest), Disconnected { message: String }, }

客户端事件枚举。Lagged 表示消费端落后，已跳过 skipped 个事件；ServerNotification/ServerRequest 为服务端推送；Disconnected 仅远程客户端使用。

From 实现：From<InProcessServerEvent> for AppServerEvent（L124-134）

测试：L2120-2130（next_event_surfaces_lagged_markers）

TypedRequestError pub enum

pub enum TypedRequestError { Transport { method: String, source: IoError }, Server { method: String, source: JSONRPCErrorError }, Deserialize { method: String, source: serde_json::Error }, }

分层错误类型，将传输失败、服务端 JSON-RPC 错误和响应反序列化失败区分开来，便于调用方决定重试策略。

实现了 Display（L282-304）和 Error（L306-314），Server 变体无 source。

测试：L2074-2102（typed_request_error_exposes_sources）

1.2 InProcessAppServerClient 方法

start pub async fn

pub async fn start(args: InProcessClientStartArgs) -> IoResult<Self>

启动进程内运行时和 worker 任务。内部调用 codex_app_server::in_process::start()，建立 command/event 两个 mpsc 通道，spawn worker。

边界条件：若内部事件队列饱和，服务端请求将被拒绝并返回 overload 错误码 -32001（L571-577），而非静默丢弃。ChatGPT auth token refresh 服务端请求因进程内不支持而被直接拒绝（L544-556）。

测试：L1021-1057（start_test_client）

request pub async fn

pub async fn request(&self, request: ClientRequest) -> IoResult<RequestResult>

发送原始客户端请求并返回 JSON-RPC 结果。请求在 worker 的 detached task 上等待，不阻塞事件循环。

错误：通道关闭时返回 BrokenPipe。

request_typed pub async fn

pub async fn request_typed<T: DeserializeOwned>(&self, request: ClientRequest) -> Result<T, TypedRequestError>

发送请求并解码响应体为具体类型 T。三层错误：Transport（通道关闭）、Server（JSON-RPC 错误）、Deserialize（响应类型不匹配）。

测试：L1260-1271（typed_request_roundtrip_works）、L1273-1291（typed_request_reports_json_rpc_errors）

notify pub async fn

pub async fn notify(&self, notification: ClientNotification) -> IoResult<()>

发送通知（无响应）。

resolve_server_request / reject_server_request pub async fn

pub async fn resolve_server_request(&self, request_id: RequestId, result: JsonRpcResult) -> IoResult<()> pub async fn reject_server_request(&self, request_id: RequestId, error: JSONRPCErrorError) -> IoResult<()>

解析或拒绝待处理的服务端请求。只能使用从当前客户端事件流中获取的 request ID。

next_event pub async fn

pub async fn next_event(&mut self) -> Option<InProcessServerEvent>

返回下一个进程内事件，worker 退出时返回 None。调用方须及时消费，否则触发 Lagged 标记。

shutdown pub async fn

pub async fn shutdown(self) -> IoResult<()>

关闭 worker 和运行时。先丢弃 event_rx 以解除 pending must-deliver 阻塞，然后发送 Shutdown 命令。若 5 秒内未完成则 abort worker 任务（L752-784）。

测试：L2309-2317（shutdown_completes_promptly_without_retained_managers）

1.3 背压与事件优先级

server_notification_requires_delivery（L160-172）定义无损层：

pub(crate) fn server_notification_requires_delivery(notification: &ServerNotification) -> bool

以下通知必须无损投递（阻塞等待消费端排空）：TurnCompleted、ThreadSettingsUpdated、ItemCompleted、ExternalAgentConfigImportCompleted、AgentMessageDelta、PlanDelta、ReasoningSummaryTextDelta、ReasoningTextDelta。

其余（如 CommandExecutionOutputDelta）为尽力投递，队列满时丢弃并递增 skipped_events 计数器。

forward_in_process_event（L193-259）实现了分流逻辑：无损事件阻塞发送，尽力事件使用 try_send。当有积压的 skip 计数时，在下一次无损事件前刷新 Lagged 标记。

测试：L1360-1446（forward_in_process_event_preserves_transcript_notifications_under_backpressure）

3.6 过载保护

enqueue_incoming_message（L217-256）实现了有界队列过载保护：

  • 当 transport_event_tx 满且消息为 Request 时，不丢弃，而是向 writer 发送 overload error（code -32001，message "Server overloaded; retry later."）。
  • 当 overload error 的 writer queue 也满时，丢弃 overload 响应并 warn。
  • 非 Request 消息（Response/Notification）在队列满时阻塞等待（send().await）而非丢弃。

测试：L297-489（overload error 测试）

4.3 CodexClient 核心方法

CodexClient 是测试客户端的核心结构，封装了传输（Stdio/WebSocket）和 JSON-RPC 请求/通知处理。

connect fn

fn connect(endpoint: &Endpoint, config_overrides: &[String]) -> Result<Self>

根据 endpoint 类型建立连接。Stdio: spawn codex app-server 子进程，pipe stdin/stdout。WebSocket: 连接 URL，10s 超时循环重试（L1598-1635）。

initialize / initialize_with_experimental_api fn

fn initialize(&mut self) -> Result<InitializeResponse> fn initialize_with_experimental_api(&mut self, experimental_api: bool) -> Result<InitializeResponse>

发送 initialize 请求并等待响应，然后发送 initialized notification 完成握手。默认 opt-out 了 item delta 通知（NOTIFICATIONS_TO_OPT_OUT，L96-104）。

thread_start / thread_resume fn

fn thread_start(&mut self, params: ThreadStartParams) -> Result<ThreadStartResponse> fn thread_resume(&mut self, params: ThreadResumeParams) -> Result<ThreadResumeResponse>

turn_start fn

fn turn_start(&mut self, params: TurnStartParams) -> Result<TurnStartResponse>

stream_turn fn

fn stream_turn(&mut self, thread_id: &str, turn_id: &str) -> Result<()>

消费通知流直到 TurnCompleted。途中处理：ThreadStarted、TurnStarted、AgentMessageDelta（输出到 stdout）、CommandExecutionOutputDelta（输出 + helper 检测）、ItemStarted、ItemCompleted、TerminalInteraction、McpToolCallProgress。

记录 command_execution_statuses、command_execution_outputs、last_turn_status、last_turn_error_message。

login_account_chatgpt / login_account_chatgpt_device_code fn

fn login_account_chatgpt(&mut self) -> Result<LoginAccountResponse> fn login_account_chatgpt_device_code(&mut self) -> Result<LoginAccountResponse>

fn wait_for_account_login_completion(&mut self, expected_login_id: &str) -> Result<AccountLoginCompletedNotification>

循环消费通知直到匹配的 AccountLoginCompleted。忽略不匹配的 login_id。

get_account_rate_limits / model_list / thread_list fn

标准 JSON-RPC 请求封装。

thread_increment_elicitation / thread_decrement_elicitation fn

fn thread_increment_elicitation(&mut self, params: ThreadIncrementElicitationParams) -> Result<ThreadIncrementElicitationResponse> fn thread_decrement_elicitation(&mut self, params: ThreadDecrementElicitationParams) -> Result<ThreadDecrementElicitationResponse>

引出暂停计数器增减。仅支持 WebSocket 传输（不支持 --codex-bin 的 stdio 路径，因为需要共享的 websocket URL）。

handle_server_request fn

fn handle_server_request(&mut self, request: JSONRPCRequest) -> Result<()>

处理三种服务端请求：CommandExecutionRequestApproval（打印参数，按 command_approval_behavior 决策）、FileChangeRequestApproval（总是 Accept）、ToolRequestUserInput（交互式提示）。不支持的服务端请求返回错误。

4.4 审批行为控制

enum CommandApprovalBehavior { AlwaysAccept, AbortOn(usize), // 在第 N 次审批时 Cancel }

command_approval_behavior 字段控制审批响应：AlwaysAccept 全部接受；AbortOn(n) 在第 n 次审批时返回 Cancel，其余 Accept。用于验证 approval abort 流程。

4.5 传输模式

enum ClientTransport { Stdio { child: Child, stdin: Option<ChildStdin>, stdout: BufReader<ChildStdout> }, WebSocket { url: String, socket: Box<WebSocket<MaybeTlsStream<TcpStream>>> }, }

Stdio: JSONL 行协议（每行一个 JSON-RPC 消息）。WebSocket: tungstenite Text 帧。

Stdio 生命周期：Drop 时先等待 5s 优雅退出，超时则 kill 子进程（L2322-2352）。

4.6 辅助函数

resolve_endpoint fn

fn resolve_endpoint(codex_bin: Option<PathBuf>, url: Option<String>) -> Result<Endpoint>

边界条件：两者同不为 None 时报错 "mutually exclusive"。两者都为 None 时默认 ws://127.0.0.1:4222。

parse_dynamic_tools_arg fn

fn parse_dynamic_tools_arg(dynamic_tools: &Option<String>) -> Result<Option<Vec<DynamicToolSpec>>>

支持直接 JSON 字符串或 @filepath 从文件读取。解析后经 normalize_dynamic_tool_specs 标准化。

BackgroundAppServer struct

fn spawn(codex_bin: &Path, config_overrides: &[String]) -> Result<Self>

后台启动 app-server：先 bind 临时端口获取地址，再 spawn codex app-server --listen ws://addr。Drop 时 kill 子进程。
