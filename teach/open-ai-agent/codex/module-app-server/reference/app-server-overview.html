App-server 模块速查

这份参考文档承接 L1 短课没有展开的 API 分类、源码入口和后续学习候选。需要学习路径时先看 0001-app-server-module-tour；需要回到项目全局位置时看 L0 项目地图 或 00-overview 参考。

模块职责边界

层负责不负责关键文件 / crate
protocol 定义 JSON-RPC 对象、typed request/response、notification、schema 导出；对应 L0 图里富客户端 / SDK 能依赖的 wire contract。 不执行业务逻辑，不管理连接。 rpc.rs、common.rs、v1.rs、protocol/v2/*
transport stdio、WebSocket、unix socket、连接生命周期、入站消息与出站 writer 队列。 不理解 Thread/Turn 语义。 app-server/src/transport.rs、codex-app-server-transport
runtime 加载配置、启动 acceptor、启动 outbound router、启动 processor loop、处理初始化后通知和优雅关闭。 不逐个实现 API 业务。 lib.rs
dispatch 反序列化 ClientRequest、检查 initialize、experimental API、请求序列化队列，并分发到 processor。 不直接渲染客户端 UI。 message_processor.rs
processors 实现 Thread、Turn、FS、Config、MCP、Plugin、Account 等领域方法。 不重新定义 wire protocol。 request_processors/
core/runtime dependencies 真实 agent 执行、配置解析、thread store、MCP、插件、命令执行、模型目录和遥测。 不暴露 app-server wire 方法名。 codex-core、codex-thread-store、codex-mcp、codex-plugin、codex-exec-server

Transport 与生命周期

阶段客户端行为服务端行为常见源码入口
启动 选择 --listen stdio://、ws://IP:PORT、unix://PATH 或 off。 main.rs 解析 CLI，调用 run_main_with_transport_options。 main.rs、lib.rs
连接 打开 stdio、WebSocket 或 unix-socket 代理流。 transport 发送 ConnectionOpened，runtime 建立 connection state 与 outbound writer。 transport.rs、lib.rs
初始化 发送 initialize request，再发送 initialized notification。 initialize request 写入 server-side initialized 状态，记录 client metadata、capabilities、notification opt-out，并触发初始化后服务端动作；initialized notification 当前主要进入日志路径，不作为 initialized gate。 initialize_processor.rs、message_processor.rs
请求执行 发送 thread/start、turn/start 等请求。 按 request serialization scope 排队或并发执行，然后发 response。 message_processor.rs、request_serialization.rs
事件流 持续读取 notification。 通过 outgoing sender 单连接发送或广播，按 experimental/opt-out 过滤。 outgoing_message.rs、transport.rs

源码阅读顺序

  1. README.md：先读 Protocol、Core Primitives、Lifecycle Overview、Initialization。
  2. main.rs：确认 CLI 入口、transport 选项和 session source。
  3. lib.rs：关注 run_main_with_transport_options 中的 transport acceptor、outbound router、processor loop。
  4. message_processor.rs：读 process_request、dispatch_initialized_client_request、handle_initialized_client_request。
  5. protocol/common.rs：查 ClientRequest 和 ServerNotification 的 wire method。
  6. request_processors/：按目标方法前缀进入具体 processor。
  7. tests/suite/v2/：用集成测试验证真实请求样例。
