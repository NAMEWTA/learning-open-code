L2 垂直切片 · 预计 15 分钟内 · 目标：能判断项目完成会触发哪些 dialog、service、task/archive、worklog 与 op-log 边界

项目完成不是一个 reducer action

这条链路的核心判断是：完成项目只是在最终阶段翻转 project flag，真正的收尾发生在它之前和之后。之前要处理未完成任务；之后要从 active context 离开、展示庆祝页，并让 history/worklog 继续能读到 live 与 archived 数据。

1. 路径图

项目菜单 Complete

    ↓ click
WorkContextMenuComponent.completeProject()
  src/app/core-ui/work-context-menu/work-context-menu.component.ts
    ├─ 读取 project 与 ProjectService.getCompletionInfo()

后台边界
  project.reducer          → isDone + doneOn + isArchived
  task.service/actions     → moveToOtherProject / updateTask / moveToArchive
  archive.service          → Daily Summary 归档 done tasks，不是完成项目本身
  worklog.service          → 合并 live task、archive task、timeTracking 输出 history
  op-log capture           → 只捕获 meta.isPersistent 的本地 action

源码锚点：src/app/core-ui/work-context-menu/work-context-menu.component.ts、src/app/features/project/project.service.ts、src/app/features/project/store/project.reducer.ts、src/app/features/archive/archive.service.ts、src/app/features/worklog/worklog.service.ts。

2. 五个判断点

问题 判断 看哪里
项目里还有未完成任务吗？ 有则先弹 resolve dialog；取消就停止整条完成流程。 src/app/core-ui/work-context-menu/work-context-menu.component.ts
用户选 Inbox 还是 Mark Done？ Inbox 移动 top-level task tree，并把已完成 parent 重新打开；Mark Done 标记每个 unfinished task。 src/app/features/project/project.service.ts
是否已经确认完成？ resolution 选择后还要再确认；确认取消时，不移动也不标记任务。 src/app/core-ui/work-context-menu/work-context-menu.component.spec.ts
完成项目会不会把任务写入 archive store？ 不会。它只设置 project 的 done/archived flag；任务归档由 Daily Summary 调用 `moveToArchive`。 src/app/pages/daily-summary/daily-summary.component.ts
哪些东西会进 op-log？ project complete、task move/update、moveToArchive、flushYoungToOld 会；弹窗关闭和路由上下文切换不会。 src/app/op-log/core/persistent-action.interface.ts

3. 收尾边界

completeProject 被派发后，src/app/features/project/store/project.reducer.ts 只改 isDone、doneOn、isArchived。任务还可能留在 live task store，但 src/app/features/tasks/store/task.selectors.ts 会用 archived project ids 把它们挡在活跃任务视图之外。真正把 done tasks 写进 archiveYoung 的路径，是 Daily Summary 的 TaskService.moveToArchive() 加 ArchiveService.moveTasksToArchiveAndFlushArchiveIfDue()。

完成的是当前 active project 时，菜单组件先导航到 /，再打开 fullscreen celebration；这样 MatDialog 不会被项目路由离开时自动关闭。Work context 的 active id 由路由事件和 setActiveWorkContext 更新，本身不是同步事实。History/worklog 则从 TaskArchiveService.load()、live task state 和 timeTracking start/end 组合输出；这部分可以回读 时间追踪 L2。

4. 60 秒自测

场景：当前项目 P 有任务 A 未完成、任务 B 已完成；用户点击 Complete Project，选择“Move to Inbox”，然后确认完成。暂停 60 秒，判断下面三件事：

  1. A 和 B 会被 project reducer 一次性处理吗？
  2. P 的完成动作和 A/B 的移动动作在 op-log 里是同一条 operation 吗？
  3. 完成后 history 页面为什么仍可能看到这个项目相关的历史记录？
参考答案：1. 不会。未完成任务先由 `ProjectService.moveTasksToInbox()` 走 task service 的普通 move/update 路径，之后 project reducer 才处理 `completeProject`。2. 不是同一条。task move/update 和 project complete 都是各自带 persistent meta 的 action。3. worklog 会合并 live task、archive task 和 timeTracking；完成项目不是删除历史数据。
