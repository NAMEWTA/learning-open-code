主流程时序

1. 菜单只在 active、非 Inbox、非 archived project 上显示 Complete
2. completeProject() 读取 project 与 ProjectService.getCompletionInfo()
3. info.unfinishedTasks.length > 0 时弹 DialogCompleteResolveTasks
4. 用户选择 inbox / markDone / cancel
5. 选择有效后再弹 DialogConfirmComponent
6. 确认后 applyResolution()
   - inbox: moveTasksToInbox(topLevelTasksWithUnfinishedWork)
   - markDone: markTasksDone(unfinishedTasks)
7. 若做过 resolution，重新 getCompletionInfo() 让 stats 反映最终任务列表
8. getProjectCompletionStats(topLevelTasks, allTasks, doneOn)
9. ProjectService.complete(projectId, doneOn)
10. 如果完成的是 active work context，先 navigateByUrl("/")
11. 打开 DialogProjectCompleteComponent 展示庆祝、任务数、耗时、工作天数

未完成任务处理矩阵

分支 输入集合 状态动作 为什么这样做
无未完成任务 `unfinishedTasks.length === 0` 跳过 resolve dialog，直接确认完成。 不需要迁移工作。
Move to Inbox `topLevelTasksWithUnfinishedWork` `ProjectService.moveTasksToInbox()` 读取 parent+subtasks，调用 `TaskService.moveToProject()`；如果 parent 本来 done，再 `setUnDone()`。 用完整 top-level tree 保留子任务结构，并让迁移后的工作可继续执行。
Mark Done `unfinishedTasks` `ProjectService.markTasksDone()` 对每个 unfinished task 调用 `TaskService.setDone()`。 明确用户决定把剩余工作也视为完成。
Cancel resolve dialog 返回 `undefined` 直接 return，不确认、不移动、不完成。 避免把未完成工作静默埋进 archived project。
Cancel confirm 已选择 resolution，但确认 dialog 返回 false 在 `_applyResolution()` 之前 return，因此任务也不变。 最终确认是实际写入前的最后闸门。

完成、归档、删除、移动的差异

操作 主入口 写入内容 op-log 语义
完成项目 src/app/features/project/project.service.ts dispatch `completeProject({ id, doneOn })`，project reducer 设置 done/archived 标志。 PROJECT Update。
重开完成项目 src/app/features/project/project.service.ts dispatch `reopenProject({ id })`，清掉 `isDone`、`doneOn`、`isArchived`。 PROJECT Update。不会还原完成前的 task resolution。
移动未完成任务到 Inbox src/app/features/tasks/task.service.ts dispatch `TaskSharedActions.moveToOtherProject`，必要时再 update task 为 undone。 TASK Update，和 project complete 分开记录。
每日归档 done tasks src/app/pages/daily-summary/daily-summary.component.ts `TaskService.moveToArchive()` 派发 `moveToArchive`，live task reducer 删除任务，ArchiveService 写 archiveYoung。 TASK Update。archive side effect 在本地先写，远端由 handler 写。
删除项目 src/app/features/project/project.service.ts dispatch `TaskSharedActions.deleteProject`，删除 project、live tasks、note 引用、current timeTracking；archive handler 清理 archived tasks 与 archived timeTracking。 PROJECT Delete。破坏性强于完成项目。

统计与 history/worklog

src/app/features/project/project-completion-stats.util.ts 只把 top-level tasks 计入任务数与总耗时，因为 parent 已经聚合子任务耗时；但 worked days 会 union parents 和 subtasks 的 timeSpentOnDay。ProjectService.getCompletionInfo() 的 stats 列表会读取 live 和 archived project tasks，避免用户之前手动归档过的工作从庆祝统计中消失。

src/app/features/worklog/worklog.service.ts 在 history/worklog 路由或手动刷新时加载 archive task state 与 live task state，再调用 src/app/features/worklog/util/get-complete-state-for-work-context.util.ts 按 Project/Tag/Today 过滤合并。随后 src/app/features/worklog/util/map-archive-to-worklog.ts 按日期建立 year/month/week/day，插入 parent/subtask entries，并附加 workStart、workEnd。这些起止时间来自 src/app/features/time-tracking/time-tracking.service.ts 合并 current、archiveYoung、archiveOld 后转换出的 legacy map。
