四条路径分流图

A. 普通 remote ops
download service → operation-log-sync service → processRemoteOps()
  → migration → clean-slate filter → vector clock detect
  → no conflict: applyNonConflictingOps()
  → conflict: autoResolveConflictsLWW()

B. incoming SYNC_IMPORT / BACKUP_IMPORT / REPAIR
download 或 upload piggyback → SyncImportConflictGateService
  → no meaningful pending: 静默接受，继续 processRemoteOps()
  → meaningful pending: dialog
     → USE_LOCAL: forceUploadLocalState()
     → USE_REMOTE: forceDownloadRemoteState()
     → CANCEL: 不推进

C. batch 内 SYNC_IMPORT / BACKUP_IMPORT
processRemoteOps() → migration → clean-slate filter
  → hasFullStateOp 只匹配 SYNC_IMPORT/BACKUP_IMPORT
  → applyNonConflictingOps()，跳过普通冲突检测
  → validateAfterSync()

D. REPAIR
ValidateStateService 生成或远端下载 REPAIR 时属于 full-state op 类型
  → 可触发 clean-slate 过滤与 full-state store 清理
  → 但 processRemoteOps() 的 direct full-state 分支不以 REPAIR 为条件

阶段做什么源码关键边界

LWW 路径展开

RemoteOpsProcessingService.detectConflicts()
  → ConflictResolutionService.checkOpForConflicts()
  → CONCURRENT: EntityConflict
  → autoResolveConflictsLWW(conflicts, nonConflicting)
       ├─ planLwwConflictResolutions()
       ├─ partitionLwwResolutions()
       ├─ remote wins: append remote/pending → applyOperations()
       ├─ local wins: createLWWUpdateOp() → appendWithVectorClockUpdate(local)
       ├─ markRejected(local old ops + remote obsolete ops)
       └─ validateAndRepairCurrentState('conflict-resolution')

LWW 的核心不是“直接保留本地状态”。本地胜出时必须创建一条新的 local op，它的 vector clock 合并冲突双方并递增当前 client，下一轮上传后其他客户端才能知道本地胜出的状态。

full-state 边界

类型何时出现本主题中的路径不要误判为
普通 remote ops其他客户端上传的 create/update/delete。下载后进入 `processRemoteOps()`，做 migration、filter、conflict detection。不要跳过冲突检测。
`SYNC_IMPORT`用户导入、server migration、密码变更 clean slate、force upload local state。incoming 阶段先 conflict gate；进入 `processRemoteOps()` 后 direct full-state apply。不要当作普通 update。
`BACKUP_IMPORT`备份恢复或完整状态恢复。与 `SYNC_IMPORT` 一样可触发 direct full-state apply。不要与 file snapshot hydration 混同。
`REPAIR`Checkpoint D 校验发现状态不一致并修复。属于 full-state op 类型，可参与 clean-slate filter 和 full-state store 清理。不要说它触发 `hasFullStateOp` direct 分支；该分支只检查 `SYNC_IMPORT` / `BACKUP_IMPORT`。
file snapshot state文件型 provider 从 seq 0 下载完整 state。走 `hydrateFromRemoteSync(..., false)`，再写 recentOps 防重复。不要创建或套用 `SYNC_IMPORT` clean-slate 语义。
