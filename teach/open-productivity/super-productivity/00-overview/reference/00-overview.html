super-productivity 项目整体架构总览

项目定位

super-productivity 是一个高级待办事项与时间追踪应用，支持 timeboxing、Pomodoro、休息提醒、个人指标、项目/标签/子任务管理，并能与日历、Jira、GitHub、GitLab、Gitea、OpenProject、Linear、ClickUp、Azure DevOps 等系统集成。它强调隐私和本地数据控制：没有强制用户账号，用户决定数据存储与同步位置。

核心设计哲学与架构决策

单一真相优先

dueDay 与 dueWithTime 在新数据中互斥，读取时 dueWithTime 优先。目标是减少调度状态冲突。

虚拟视图不污染实体

TODAY_TAG 的成员来自任务日期字段，而不是写入 task.tagIds。标签排序和成员判定分离。

同步核心保持框架无关

src/app 负责宿主接线，@sp/sync-providers 负责 provider，@sp/sync-core 只放可复用同步原语。

跨切面成本要被显式证明

项目完成选择普通 per-task actions 加项目 flag flip，而不是重新引入昂贵的多实体原子 op。

一套 renderer，多种宿主桥

Web/PWA、Electron、Android、iOS 共用 Angular 主体，平台差异通过条件 effects、IPC 和 Capacitor lifecycle 注入。

源码文件豁免清单

豁免表示 L0/L1 覆盖率不要求逐文件讲解，不表示永远不能读。后续专题需要时仍可抽样或定点阅读。

路径或类型豁免理由例外情况
node_modules/、.tmp/、dist/依赖或构建产物，非项目手写源码。排查构建输出或依赖补丁时才看。
src/assets/icons/、src/assets/bg/、src/assets/snd/、src/assets/fonts/图标、背景、声音、字体等静态资源，大多不承载业务逻辑。主题、资源加载、包体积分析时抽样。
src/assets/i18n/*.json翻译词条体量大，适合按 key 查，不适合逐文件教学。国际化提取、缺失翻译、文案规范专题。
src/assets/bundled-plugins/构建过程复制的内置插件资产，主要由 packages 插件源码生成。插件运行时加载链路需要核对产物时。
docs/screens/、docs/wiki/assets/、docs/promotion/截图和宣传素材，适合作为产品背景，不作为源码结构主体。商店截图流水线或文档发布专题。
android/app/build/、ios/**/build/原生构建产物，不纳入源码讲解。移动端打包失败排查。
e2e/fixtures/、tools/test-data/测试固件与样例数据，按测试场景读取即可。端到端测试专题。
二进制图片、音频、字体、压缩包不可直接通过源码阅读理解，且体积可能较大。资源加载、许可、包体积或 UI 视觉专题。
