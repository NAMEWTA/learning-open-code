super-productivity 的项目地图

一句话：super-productivity 是一个以任务、时间追踪、计划排程和外部工具集成为核心的个人生产力应用；同一套 Angular 前端通过 Web/PWA、Electron 桌面和 Capacitor 移动端运行。

1. 先抓三条主线

读这个仓库时，不要从某个组件一路钻到底。先看三类入口：产品功能从 README.md 来，运行时装配从 src/main.ts 来，用户可见页面从 src/app/app.routes.ts 来。桌面能力再接到 electron/start-app.ts：它创建窗口、托盘、IPC、空闲检测和退出流程。这样你能把“功能清单”“Angular 应用”“桌面宿主”分开看。

2. 总体架构

用户功能
  ↓
Angular SPA：routes / pages / features / root-store
  ↓
NgRx Store + Effects + meta-reducers + op-log
  ↓
平台桥接：Browser/PWA · Electron IPC · Capacitor App lifecycle
  ↓
外部系统：issue providers · calendar · WebDAV/Dropbox/SuperSync · plugins

angular.json 指向 src/main.ts，并把 src/assets、src/manifest.json、src/static 纳入构建；生产和 Web stage 配置启用 ngsw-config.json。package.json 则把前端构建、Electron 编译、Capacitor sync、Playwright e2e、packages 构建串起来。

3. 顶层目录怎么读

super-productivity/ ├── src/ ← Angular 应用与 Web 资产 │ ├── app/ ← 页面、feature、store、插件、同步入口 │ ├── assets/ ← 图标、字体、翻译、内置插件等静态资源 │ └── environments/ ← stage/production 环境替换 ├── electron/ ← 桌面主进程、IPC、窗口、托盘、空闲检测 ├── packages/ ← 插件 API、插件示例、同步核心与服务端包 ├── android/ ← Capacitor Android 外壳与构建配置 ├── ios/ ← Capacitor iOS 外壳与构建配置 ├── docs/ ← Wiki、同步设计、截图与长期计划 ├── e2e/ ← Playwright 场景、页面对象和测试固件 └── tools/ ← 构建、发布、翻译、截图等脚本

架构决策里有几个读码护栏：任务日期字段要保持单一真相；TODAY_TAG 是由日期推导的虚拟标签；同步核心必须和 Angular、NgRx、具体 provider 保持边界；项目完成选择普通 per-task actions，而不是昂贵的跨实体原子操作。这些不是细节，是后续判断改动是否合理的标准。

4. 检索练习

暂停 60 秒，凭记忆回答：如果你要研究“移动端退到后台时如何避免丢失操作日志”，应该先看哪两个入口？

参考答案：先看 src/main.ts 里的 Capacitor appStateChange 和 OperationWriteFlushService 调用，再看 src/app/root-store/feature-stores.module.ts 中按 Android/iOS 条件注册的 effects。桌面相关问题才转向 electron/start-app.ts。
