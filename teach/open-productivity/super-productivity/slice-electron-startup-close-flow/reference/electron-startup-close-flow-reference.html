Electron 启动、窗口创建与关闭前同步全链路参考

这份参考用于排查桌面端启动、窗口显示、IPC、本地能力和关闭前同步问题。核心原则：Electron main process 管进程、窗口、本地能力和 IPC 管道；Angular renderer 管业务状态、同步决策、finish-day UI，并通过 preload bridge 交接。

短课顺序：链路地图、 启动主路径、 关闭前保护。相关 L1： Electron 宿主、 Angular 应用装配、 op-log 同步架构。

一、主进程入口与 app 生命周期

阶段 源码 职责 读码判断
进程入口 electron/main.ts 非 macOS 获取 requestSingleInstanceLock()，成功才调用 startApp()。 多开、协议启动、二次实例问题先看这里。
启动编排 electron/start-app.ts 注册协议、IPC、本地能力、下载、证书策略、ready 回调、idle/powerMonitor、退出钩子。 它是 main process 编排器，不是 Angular bootstrap。
ready 后建窗 electron/start-app.ts appIN.on('ready', () => createMainWin())，再调用 createWindow(...)。 窗口不存在时先确认 ready 是否触发、userData/GPU 前置逻辑是否失败。
退出兜底 electron/start-app.ts before-quit 拦截原生 quit；window-all-closed 调 app.quit() 并设置强制退出兜底。 原生菜单、Dock、快捷键退出都应回到 close 保护。
main process startup main.ts → startApp() → initializeProtocolHandling() → initIpcInterfaces() → app commandLine / userData / GPU guard → app ready → createMainWin() → createWindow() → initPluginOAuth() → processPendingProtocolUrls()

二、窗口生命周期与 renderer 交接

阶段 源码 关键点
创建窗口 electron/main-window.ts new BrowserWindow 设置 preload、nodeIntegration: false、contextIsolation: true、权限白名单和标题栏策略。
加载 renderer electron/main-window.ts 开发环境加载 http://localhost:4200；打包环境加载 dist 下的 HTML 入口。真正的 renderer bootstrap 入口见 src/main.ts。
窗口可显示 electron/main-window.ts ready-to-show 后 show() 并处理焦点。它不等于业务数据加载完成。
业务就绪回报 src/app/core/startup/startup.service.ts Angular 初始化后调用 window.ea.informAboutAppReady()；main 在 IPC.APP_READY 中设置 app ready 并标记 GPU startup 成功。
Angular bootstrap src/main.ts bootstrapApplication(AppComponent, ...) 是 renderer 自己的启动，不由 main process 同步等待。
排障时不要把 ready-to-show、did-finish-load、APP_READY 混成一个事件。前两个属于 Electron 窗口加载层；APP_READY 是业务 bootstrap 后 renderer 主动发回的 IPC。

三、IPC 边界与本地能力

边界 源码 责任
通道常量 electron/shared-with-frontend/ipc-events.const.ts main 和 renderer 共享 IPC 名称，避免魔法字符串散落。
preload bridge electron/preload.ts、electron/electronAPI.d.ts 在隔离上下文里暴露 window.ea，把 renderer 的方法映射到 ipcRenderer.send 或 ipcRenderer.invoke。
IPC 注册入口 electron/ipc-handler.ts、electron/ipc-handlers/index.ts initIpcInterfaces() 注册 app data、app control、system、Jira、快捷键、剪贴板图片、local REST API 等处理器。
控制类事件 electron/ipc-handlers/app-control.ts ipcMain.on 处理 relaunch、exit、show/focus、lock screen、progress bar、settings update。
查询/文件能力 electron/ipc-handlers/app-data.ts、electron/ipc-handlers/system.ts ipcMain.handle 返回 userData、backup path、save dialog、native share 等异步结果。
安全包装 electron/ipc-handler-wrapper.ts 对 renderer 传入路径做 userData containment、文件名和 imageId 校验。renderer 不是可信输入源。

四、关闭前同步与 finish-day 保护

close / quit path user close button / app menu quit / native Cmd+Q → before-quit may prevent default → closeWinAndQuit() marks quit intent → mainWin.close() → close handler ├─ minimize-to-tray allowed → hide window ├─ ids.length > 0 → send NOTIFY_ON_CLOSE(ids) └─ ids.length === 0 → setIsQuiting(true), close again → renderer handles ids ├─ SyncEffects sync() → BEFORE_CLOSE_DONE(SYNC_BEFORE_CLOSE_ID) └─ FinishDayBeforeCloseEffects dialog decision → main removes ids → ids empty → destroy task widget → mainWin.close() → closed → clear refs → window-all-closed → app.quit()
托盘 indicator 菜单的 Quit 当前是另一条直接退出路径：它调用 quitApp()，先设置 isQuiting=true 再 app.quit()，所以不会走 NOTIFY_ON_CLOSE 等待项分支。应用菜单 Quit 和原生 Cmd+Q 仍通过 closeWinAndQuit() 进入关闭前保护。
组件 源码 关闭前行为
等待项仓库 electron/main-window.ts REGISTER_BEFORE_CLOSE 加 ID，UNREGISTER_BEFORE_CLOSE 移除 ID，BEFORE_CLOSE_DONE 清除已完成 ID。
renderer bridge src/app/core/electron/exec-before-close.service.ts schedule()、unschedule()、setDone() 分别映射到 preload 暴露的 before-close IPC。
同步保护 src/app/imex/sync/sync.effects.ts 同步启用且 ready 后注册 SYNC_BEFORE_CLOSE_ID；收到通知后停止当前任务、flush counter、等待一个 macrotask，再调用 sync()，成功或失败都释放关闭。
finish-day 提醒 src/app/features/finish-day-before-close/finish-day-before-close.effects.ts 功能启用后就注册等待项；关闭通知到来后检查当天已完成任务。没有已完成任务会立即释放；有已完成任务时弹窗，选 quit 才释放，选 finish-day 导航到 daily summary 并保持窗口，选 cancel 不释放。
弹窗 UI src/app/features/finish-day-before-close/dialog-finish-day-before-close/dialog-finish-day-before-close.component.ts、src/app/features/finish-day-before-close/dialog-finish-day-before-close/dialog-finish-day-before-close.component.html 返回 cancel、quit、finish-day 三种选择。
行为佐证 src/app/features/finish-day-before-close/finish-day-before-close.effects.spec.ts 测试固定了三种选择的结果：quit 调 setDone，cancel 不关闭，finish-day 导航但不关闭。

五、排障速查

现象 优先检查 常见判断
第二个实例启动后主窗口没有响应协议 URL electron/main.ts、electron/protocol-handler.ts 先看单实例锁和 pending protocol URL 是否等到 window ready 后处理。
窗口显示了，但 GPU startup guard 仍认为启动失败 electron/main-window.ts、electron/gpu-startup-guard.ts ready-to-show 不足以证明 Angular 完成；要等 APP_READY。
点击关闭只是隐藏到托盘 electron/main-window.ts、electron/shared-state.ts 如果 getIsMinimizeToTray() 且不是显式 quit intent，close handler 会 hide。
关闭卡住 electron/main-window.ts、src/app/core/electron/exec-before-close.service.ts 检查 pending ID 是否没有收到 BEFORE_CLOSE_DONE；finish-day 的 cancel/finish-day 本来就不释放。
同步失败后仍退出 src/app/imex/sync/sync.effects.ts 这是显式设计：catch 后提示用户，然后调用 setDone(SYNC_BEFORE_CLOSE_ID)，避免失败把用户困住。
renderer 能调用危险本地能力 electron/preload.ts、electron/ipc-handler-wrapper.ts renderer 只通过 window.ea 触达白名单 IPC；路径和文件名需要在 main 侧再次校验。
