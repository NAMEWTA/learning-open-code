Capacitor 移动端与 PWA 宿主

一句话：super-productivity 复用 Angular renderer，再分别放进浏览器 PWA、Android WebView/Capacitor activity、iOS Capacitor view controller；它和 Electron 宿主一样是平台边界层，但系统能力来自 Capacitor、WebView、AlarmManager、Widget、Swift plugin 和 service worker。全局定位先回到 L0 地图。

1. 先分五层

读移动端代码时，第一步不是问“是不是 Capacitor”，而是问“当前行为在哪一层发生”。src/main.ts 启动 Angular app，也在同一文件里接 service worker、Android back button、Android/iOS appStateChange flush。capacitor.config.ts 决定 native shell 如何包装 dist/browser。android/ 和 ios/ 才拥有真正系统能力。

Angular app
  → routes / NgRx / effects / op-log
Capacitor native shell
  → webDir=dist/browser + App lifecycle + plugin config
PWA / service worker
  → ngsw-config.json + manifest.json, only Web production/stage
Android / iOS feature
  → platform-specific NgRx effects and services
platform bridge
  → window.SUPAndroid or Capacitor plugin method

2. PWA 不是 native

当前源码把 PWA 和 native 明确分开：ServiceWorkerModule.register() 只有在非 Electron、非 native 且 production/stage 时启用；bootstrap 后还会在 Electron/native 环境注销已有 service worker。也就是说，浏览器 PWA 的离线缓存看 ngsw-config.json 和 src/manifest.json；移动壳的后台保存、回退键、提醒权限不在 PWA 层。

if (!IS_ELECTRON && !IS_NATIVE_PLATFORM) {
  register ngsw-worker.js
} else {
  unregister existing service workers
}

ngsw-config.json 预取 app shell、关键图标/字体/英文翻译，懒缓存 assets/media，并对 GitHub、GitLab、Atlassian API 用 freshness cache。manifest.json 则声明 standalone、portrait、scope、start_url 和图标。

3. Native 能力要过桥

Android 的关键边界是 window.SUPAndroid：android/app/src/main/java/com/superproductivity/superproductivity/CapacitorMainActivity.kt 注入 android/app/src/main/java/com/superproductivity/superproductivity/webview/JavaScriptInterface.kt，TypeScript 侧 src/app/features/android/android-interface.ts 给它补类型和 RxJS subject。Android share、widget、foreground tracking、focus mode、native reminder、startup overlay、SuperSync credential 都沿这座桥走。

iOS 边界更偏 Capacitor plugin：ios/App/App/CustomViewController.swift 注册 WebDavHttpPlugin 和 StoreReviewPlugin；src/app/features/ios/ios-interface.ts 只提供 resume subject。通用 mobile 层在 src/app/features/mobile/store/mobile-notification.effects.ts，它只在 IS_NATIVE_PLATFORM 时注册，负责把任务提醒调度给 CapacitorReminderService。

4. 检索练习

暂停 90 秒判断：A. 用户说“安装成 PWA 后刷新仍显示旧图标”。B. Android 用户说“通知 Done 按钮点了，打开 app 后任务状态才变化”。C. iOS WebDAV 同步读到缓存旧数据。分别先看哪一层？

参考答案：A 先看 PWA 层：src/manifest.json、ngsw-config.json 和 src/main.ts 的 service worker 条件。B 先看 Android bridge：Manifest receiver/widget/service 声明、android/app/src/main/java/com/superproductivity/superproductivity/webview/JavaScriptInterface.kt、src/app/features/android/android-interface.ts 与 Android effects 的队列 drain。C 先看 iOS native plugin：ios/App/App/CustomViewController.swift 注册的 ios/App/App/WebDavHttpPlugin.swift。完整入口和边界表见 模块参考。主要来源：src/main.ts、capacitor.config.ts、android/、ios/。
