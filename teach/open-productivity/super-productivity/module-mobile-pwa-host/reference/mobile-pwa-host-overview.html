Capacitor 移动端与 PWA 宿主参考

职责一句话：本模块把 L0 总览中的 Angular SPA 放进 Web/PWA、Android、iOS 三种运行外壳，并为系统能力建立受控桥接。Angular 壳层详见 Angular L1；桌面 IPC 边界详见 Electron L1。

模块骨架

super-productivity/ ├── src/ │ ├── main.ts ← Angular bootstrap、SW/native 生命周期 │ ├── manifest.json ← PWA manifest │ └── app/ │ ├── features/ │ │ ├── android/ ← Android 专用 Angular effects/bridge 类型 │ │ ├── ios/ ← iOS resume/background tracking │ │ └── mobile/ ← iOS/Android 通用 native notification │ └── core/ │ └── platform/ ← Capacitor 平台与提醒服务 ├── capacitor.config.ts ← Capacitor app/webDir/plugins 配置 ├── ngsw-config.json ← Angular service worker 缓存策略 ├── android/ ← Android 原生壳、Gradle、Manifest、services └── ios/ ← iOS 原生壳、Xcode、Swift plugins

入口清单

入口源码职责先看场景
Angular bootstrapsrc/main.ts注册 Angular providers、ServiceWorkerModule、Capacitor App listeners、Android back button、后台 flush。首屏、退后台、service worker、native 生命周期。
Capacitor configcapacitor.config.ts声明 appId、appName、webDir: dist/browser、plugin 配置、Android includePlugins、iOS WebView 行为。native 壳配置、插件是否被打进 Android、safe area/keyboard/status bar。
PWA cachengsw-config.json配置 app shell、关键资产、懒加载静态资源和 API freshness cache。PWA 离线、资源更新、Web API 缓存。
PWA manifestsrc/manifest.json声明 standalone、portrait、start_url、scope、icons、theme/background color。安装体验、启动 URL、图标、显示模式。
Android Manifestandroid/app/src/main/AndroidManifest.xml声明权限、launcher activity、Capacitor activity、OAuth intent、foreground services、reminder receivers、home widget。Android 系统入口、权限、通知、Widget、分享、深链。
Android Capacitor activityandroid/app/src/main/java/com/superproductivity/superproductivity/CapacitorMainActivity.kt注册 native plugins，初始化 WebView，注入 window.SUPAndroid，处理 share/reminder/widget/lifecycle。Android Capacitor 壳、离线模式、桥接事件。
Android legacy activityandroid/app/src/main/java/com/superproductivity/superproductivity/FullscreenActivity.kt旧在线 WebView/TWA 风格入口，按 LaunchDecider 可切到 CapacitorMainActivity。升级迁移、旧数据用户、在线/离线启动模式。
Android bridgeandroid/app/src/main/java/com/superproductivity/superproductivity/webview/JavaScriptInterface.kt暴露 storage、share、foreground service、focus mode、reminder、widget、startup overlay、SuperSync credential 等 native 方法。TypeScript 调 native、native 回调 JS。
Android TS bridgesrc/app/features/android/android-interface.ts读取 window.SUPAndroid，定义接口、Promise wrapper、ReplaySubject/Subject 事件。Angular 侧如何订阅 native 事件或调用 native 方法。
Feature effects 注册src/app/root-store/feature-stores.module.ts按 IS_ANDROID_WEB_VIEW、IS_IOS_NATIVE、IS_NATIVE_PLATFORM 条件注册 Android/iOS/mobile effects。effect 没触发、平台判断错误。
iOS bridge controllerios/App/App/CustomViewController.swift继承 CAPBridgeViewController，注册 WebDavHttpPlugin 与 StoreReviewPlugin。iOS 自定义 native plugin。
iOS WebDAV pluginios/App/App/WebDavHttpPlugin.swift以 URLSession 发 WebDAV 请求并禁用 HTTP cache，避免同步读到旧数据。iOS WebDAV 同步、缓存、网络错误分类。
mobile notificationssrc/app/features/mobile/store/mobile-notification.effects.ts在 native 平台调度任务提醒、due-date、重复任务预提醒、iOS deadline reminder。移动端提醒调度、权限、exact alarm。
构建脚本package.jsondist:android、dist:android:prod、sync:android、sync:ios、dist:ios:prod 串联 Web build 与 Capacitor sync。移动构建链路、webDir 产物同步。

职责边界

层能做什么不能假设什么代表文件
Angular app路由、UI、NgRx、op-log、effects、同步 provider、平台 service 调用。不能直接拥有 Android AlarmManager、home widget、iOS URLSession、系统通知 action。src/main.ts、src/app/root-store/feature-stores.module.ts
PWA/service worker浏览器安装体验、静态资源缓存、Web API freshness cache。native 平台不会使用当前 Angular service worker；源码会在 native/Electron 下注销已有 SW。ngsw-config.json、src/manifest.json
Capacitor shell把 dist/browser 放进 WebView，提供 App lifecycle、插件配置、native plugin 通道。配置文件不是业务逻辑；具体系统行为在 Android Kotlin/Swift 或 Angular effects 中。capacitor.config.ts、android/、ios/
Android nativeWebView 初始化、JavaScriptInterface、foreground service、AlarmManager、receiver、widget、share intent、startup overlay。不能直接修改 Angular store；通常通过 window.SUPAndroid subject、队列或 get-and-clear 方法交给 Angular。android/app/src/main/java/com/superproductivity/superproductivity/CapacitorMainActivity.kt、android/app/src/main/java/com/superproductivity/superproductivity/webview/JavaScriptInterface.kt、android/app/src/main/AndroidManifest.xml
iOS nativeCapacitor view controller、Swift plugin、URLSession、StoreReview、AppDelegate deep link 转发。当前 iOS feature 面较窄，许多提醒调度仍走 Capacitor LocalNotifications，而非自写 Swift receiver。ios/App/App/CustomViewController.swift、ios/App/App/WebDavHttpPlugin.swift、ios/App/App/AppDelegate.swift
平台桥接把 native 事件转成 RxJS subject，把 Angular 请求转成 native 方法调用。桥接不是业务真相；业务状态仍要回到 NgRx/task/sync/op-log 层判断。src/app/features/android/android-interface.ts、src/app/features/ios/ios-interface.ts、src/app/core/platform/capacitor-reminder.service.ts

关键依赖

依赖/机制当前源码用法判断要点
@capacitor/appsrc/main.ts 注册 backButton、appStateChange、appUrlOpen；src/app/features/ios/ios-interface.ts 发 resume。App lifecycle 负责后台 flush 和 resume signal，不等于 PWA lifecycle。
@capawesome/capacitor-background-taskAndroid/iOS 退后台时用 BackgroundTask.beforeExit() 包住 op-log flush。用于争取 native 暂停前的持久化预算。
@capacitor/local-notificationsiOS 和非 Android native fallback 调度提醒；Android Capacitor/legacy WebView 都优先走 SUPAndroid.scheduleNativeReminder 到 AlarmManager。Android 精确提醒优先看 AlarmManager 桥，iOS 看 Capacitor LocalNotifications。
Android AlarmManagerandroid/app/src/main/java/com/superproductivity/superproductivity/service/ReminderNotificationHelper.kt 使用 exact/inexact alarm，触发 android/app/src/main/java/com/superproductivity/superproductivity/receiver/ReminderAlarmReceiver.kt。提醒不是 JS timer；App 关闭后由 OS 广播唤起 receiver。
Android WorkManagerGradle 引入 work-runtime-ktx，native SuperSync credential bridge 支持后台 sync reminder worker。后台服务器检查不依赖 WebView 常驻。
Angular Service Workerngsw-config.json 控制 PWA 缓存；src/main.ts 在 native/Electron 下禁用/注销。PWA 缓存问题不要误查 Android receiver 或 iOS plugin。
Android addJavascriptInterfaceandroid/app/src/main/java/com/superproductivity/superproductivity/CapacitorMainActivity.kt 将 android/app/src/main/java/com/superproductivity/superproductivity/webview/JavaScriptInterface.kt 注入为 SUPAndroid。任何从 native 拼 JS 字符串回调的路径都要留意转义、队列与冷启动时序。

阅读示例

问题阅读路径边界判断
退后台后最近操作丢失。src/main.ts → OperationWriteFlushService → BackgroundTask.beforeExit()。这是 native lifecycle + op-log flush，不是 PWA cache。
Android 回退键从 tab 切换历史里乱跳。src/main.ts backButton listener → src/app/features/android/android-back-button.service.ts。Capacitor App event 进入 Angular service，再按 route/dialog/store overlay 分类。
Android 分享文本生成空任务。android/app/src/main/java/com/superproductivity/superproductivity/CapacitorMainActivity.kt share intent → src/app/features/android/android-interface.ts → src/app/features/android/store/android.effects.ts 的 buildTaskTitle()。native 接收 intent，Angular 决定 task title 和 attachment。
Widget 勾选后任务稍后才更新。android/app/src/main/java/com/superproductivity/superproductivity/widget/TaskListWidgetProvider.kt → WidgetDoneQueue → src/app/features/android/store/android-widget.effects.ts drain。Widget 先写 native 队列，Angular 在 resume 或 drain signal 后读队列并修改 store。
iOS WebDAV 同步读到旧响应。ios/App/App/CustomViewController.swift → ios/App/App/WebDavHttpPlugin.swift。这是 iOS native HTTP cache 策略，不是 Angular service worker。
PWA 访问 GitHub/GitLab/Atlassian API 缓慢或过期。ngsw-config.json dataGroups api-freshness。只影响浏览器 PWA 的 service worker 数据缓存。
