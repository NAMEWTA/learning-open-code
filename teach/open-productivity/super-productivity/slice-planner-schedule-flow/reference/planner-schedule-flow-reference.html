数据层边界速查

层 主文件 保存或派生什么 不是
Task due 字段 src/app/features/tasks/task.model.ts
src/app/root-store/meta/task-shared-meta-reducers/task-shared-scheduling.reducer.ts		 `dueWithTime` 表示具体时间，设置时清 `dueDay`；`dueDay` 表示按日计划。 不是 Schedule view entry，也不是 calendar event。
Planner state src/app/features/planner/store/planner.reducer.ts `PlannerState.days` 只保存 `dayDate -> taskIds`，用于未来日按日计划顺序。 不保存开始时间、duration、日历事件或 blocked block。
Today order src/app/features/work-context/store/work-context.selectors.ts `TODAY_TAG.taskIds` 保存 Today 顺序；membership 由 `dueWithTime` 优先、再由 `dueDay` 推导。 不是 `task.tagIds` 里的真实 tag 关系。
Schedule view entries src/app/features/schedule/schedule.model.ts
src/app/features/schedule/map-schedule-data/map-schedule-days-to-schedule-events.ts		 `ScheduleDay.entries` 和 `ScheduleEvent` 是渲染用派生数据，包含 grid row、duration、overlap、beyond budget。 不是持久状态。
Calendar integration src/app/features/calendar-integration/calendar-integration.service.ts 外部 provider 拉取的 `ScheduleCalendarMapEntry[]`，可被 Planner/Schedule 读取并作为 blocked block。 默认不是 Task；只有显式创建任务或移动可写 provider event 才写回。

Planner 视图合成

PlannerService.days$
  → selectActiveTaskRepeatCfgs
  → selectTodayTaskIds
  → CalendarIntegrationService.calendarEvents$
  → selectAllTasksWithDueTime
  → selectPlannerDays(...)
      ├─ normal tasks: planner.days[day] 或 Today unplanned ids
      ├─ scheduled task items: active tasks with dueWithTime on day
      ├─ repeat projections: TaskRepeatCfg exact day, with/without startTime
      ├─ calendar events: timed goes scheduledIItems, all-day goes allDayEvents
      └─ deadlineTasks: undone deadlineDay/deadlineWithTime grouped by day
Planner 的 `scheduledIItems` 和 Schedule 的 `ScheduleDay.entries` 都是投影，但服务的 UI 不同：Planner 用它给按日计划列显示“定时区”；Schedule 用它生成时间网格、split、overlap 和 beyond budget。

Schedule 投影算法

ScheduleService.createScheduleDaysWithContext()
  → buildScheduleDays()
  → mapToScheduleDays()
      ├─ selectTimelineTasks.unPlanned      Today 未定时任务
      ├─ selectTimelineTasks.planned        所有未完成 dueWithTime 任务
      ├─ plannerDayMap                      未来日按日计划任务
      ├─ repeat cfg with startTime          scheduled repeat blocker
      ├─ repeat cfg without startTime       flow repeat projection
      ├─ calendar events                    calendar blocker / all-day event
      └─ workStartEnd/lunchBreak            non-task blocked blocks
  → createBlockedBlocksByDayMap()
  → createScheduleDays()
  → createViewEntriesForDay()
  → mapScheduleDaysToScheduleEvents()
输入 投影类型 关键文件 排查提示
`Task.dueWithTime` `BlockedBlockType.ScheduledTask` 到 `SVEType.ScheduledTask` src/app/features/schedule/map-schedule-data/create-sorted-blocker-blocks.ts
src/app/features/schedule/map-schedule-data/create-view-entries-for-block.ts		 被视为占用时间的 blocker，会推动流动任务后移或 split。
Today 未定时任务 `SVEType.Task` 或 split 类型 src/app/features/work-context/store/work-context.selectors.ts
src/app/features/schedule/map-schedule-data/create-schedule-view-entries-for-normal-tasks.ts		 按当前任务优先和 Today 顺序流入剩余预算。

Repeat cfg with startTime `ScheduledRepeatProjection` blocker src/app/features/schedule/map-schedule-data/create-sorted-blocker-blocks.ts 如果当天已有真实 repeat instance，则跳过投影防重复。
Repeat cfg without startTime `RepeatProjection` 流动 entry src/app/features/schedule/map-schedule-data/create-view-entries-for-day.ts 像未定时任务一样进入当天空隙，也会被 blocker split。
Calendar event `CalendarEvent` blocker 或 Planner all-day event src/app/features/calendar-integration/calendar-integration.service.ts
src/app/features/schedule/map-schedule-data/create-sorted-blocker-blocks.ts		 全天事件不阻塞时间槽；timed event 会参与 blocked block 合并。
