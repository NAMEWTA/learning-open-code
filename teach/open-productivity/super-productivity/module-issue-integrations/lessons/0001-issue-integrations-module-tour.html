L1 模块总览 · 预计 15 分钟 · 目标：能把外部工作项链路分到正确入口

Issue Provider、日历集成与外部工作项

一句话：这个模块把外部 issue、VTODO、iCal event 和插件工作项转换成本地任务，同时把“读远端”“导入任务”“刷新已有任务”“写回远端”和“日历 time block”拆成几条独立链路。

1. 架构位置

L0 已把 Jira、GitHub、GitLab、Gitea、OpenProject、Linear、ClickUp、Azure DevOps 等列为集成能力；当前源码中内置 provider 是 Jira、GitLab、CalDAV、Calendar、OpenProject、Redmine、Nextcloud Deck、Plainspace，GitHub、ClickUp、Gitea、Linear、Trello、Azure DevOps 已迁移到插件形态。外部工作项最终仍回到任务领域：创建出来的是带 issueType、issueProviderId、issueId 的 Task。

主入口从 src/app/features/issue/issue.service.ts 读。它不会亲自访问每个远端 API，而是按 provider key 找到实现了 IssueServiceInterface 的服务或插件 adapter。

2. 四条主链路

provider 配置
  → issueProvider store 保存 enabled、defaultProjectId、pollingMode、默认标签

外部 issue → 本地任务
  → provider.getAddTaskData() 映射字段
  → IssueService.addTaskFromIssue() 决定项目、标签、backlog、due 字段

轮询刷新
  → PollToBacklogEffects 拉新 issue 进 backlog
  → PollIssueUpdatesEffects 刷新已有 issue task

写回与日历
  → IssueTwoWaySyncEffects 用 adapter 推字段、创建/删除远端 issue
  → CalendarIntegrationEffects 读日历事件并导入
  → TimeBlockSyncEffects 把 scheduled task 写成插件日历 time block

注意区分：provider 配置只是“能不能用、默认落到哪里、多久轮询”；issue-to-task 映射只发生在导入或刷新时；two-way-sync 监听本地任务 action 后写回远端；calendar/time-block 既可以读日历事件，也可以把本地 scheduled task 写成远端日历块。

3. 阅读示例：新 issue 进 backlog

从 src/app/features/issue/store/poll-to-backlog.effects.ts 开始。它等待初始同步完成，然后按当前项目或 pollingMode === 'always' 找启用的 provider：必须有 isAutoAddToBacklog、defaultProjectId 和非零 poll interval。

命中后调用 IssueService.checkAndImportNewIssuesToBacklogForProject()，再由 provider 的 getNewIssuesToAddToBacklog() 拉远端候选项。真正创建任务时回到 addTaskFromIssue()：这里会去重、处理默认项目/标签、避免自动导入继承当前标签，并让 Calendar 与 Plainspace 使用确定性 task id。

如果是本地任务改动要推远端，不走这条链路；去 src/app/features/issue/two-way-sync/issue-two-way-sync.effects.ts 看 adapter、字段映射和 baseline 判断。Jira 在 Electron 下另有 electron/jira.ts 代发请求，解决桌面端 CORS、代理、自签名证书和附件认证 header 问题。

4. 检索练习

暂停 60 秒，凭记忆判断：用户把一个已导入的 Plainspace 任务重新排到明天 10:00。你应该先看 backlog 轮询、issue 更新轮询、two-way-sync，还是 calendar auto-import？为什么？

参考答案：先看 two-way-sync。这个场景是本地任务字段变化要推回远端，入口是 IssueTwoWaySyncEffects，Plainspace 通过 sync adapter 把 dueWithTime 映射到远端 scheduledAt。backlog 轮询只拉新 issue；issue 更新轮询负责从远端拉变化；calendar auto-import 只处理日历事件导入。

主要来源：src/app/features/issue/issue.service.ts、src/app/features/issue/store/poll-to-backlog.effects.ts、src/app/features/issue/two-way-sync/issue-two-way-sync.effects.ts。长 provider 清单见 参考文档；不确定某个行为属于哪条链路时，直接向 agent 追问。
