UI、配置、导入导出与备份入口判断

一句话：本主题教你判断一个问题该进入共享 UI、全局配置、主题/i18n、文件导入导出、op-log 备份恢复，还是 Electron 本地能力。它接在 L0、Angular L1、op-log L1 和 Electron L1 后面，不重复讲启动、同步协议或主进程全景。

1. 设置页不是配置本身

先从 src/app/pages/config-page/config-page.component.ts 读：它只是把 tab、ThemeSelectorComponent、ConfigSectionComponent、同步按钮和自动备份 section 组装成页面。真正的配置 schema 在 src/app/features/config/global-config.model.ts，默认值在 default-global-config.const.ts，表单分组在 global-config-form-config.const.ts 和 form-cfgs/。

设置页保存一次配置
  → ConfigPageComponent.saveGlobalCfg()
  → GlobalConfigService.updateSection()
  → updateGlobalConfigSection(meta.entityType = 'GLOBAL_CONFIG')
  → op-log capture 持久化为 operation
  → GlobalConfigEffects 应用语言、快捷键、Electron settings 等副作用

关键边界：taskWidget、focusModeLocal 是本机设置，走专用 service；普通全局配置才 dispatch updateGlobalConfigSection 并进入 op-log。

2. UI、主题和 i18n 各管一层

src/app/ui/ 放可复用控件、Formly 类型、管道、动画和基础指令；src/app/core-ui/ 是 header、side nav、layout 等应用壳层；全局 SCSS 在 src/styles/。主题运行时由 GlobalThemeService 写 body class、Material CSS vars、背景图和图标；自定义主题由 CustomThemeService 加载内置 CSS 或用户 CSS。语言入口则是 main.ts 的 Translate loader、src/assets/i18n/*.json 和 LanguageService，RTL 方向最终由根组件写入 document.dir。

3. 导入、恢复和自动备份都要过 op-log

src/app/imex/file-imex/file-imex.component.ts 负责文件选择、URL 导入、下载完整备份和隐私导出，但导入不会直接改 NgRx。它调用 BackupService.importCompleteBackup()：先校验/迁移数据，flush pending writes，再在 op-log 锁内生成 OpType.BackupImport，原子替换 ops、state cache、vector clock 和 client id，最后 dispatch loadAllData。

本地自动备份分平台：Electron 由 LocalBackupService 取 StateSnapshotService 快照后调用 window.ea.backupAppData()，实际文件写入在 electron/backup.ts；Android/iOS 走移动端本地存储和备份环。SuperSync 历史恢复也先取 server snapshot，再复用 BackupService.importCompleteBackup()。

4. 检索练习

暂停 60 秒回答：用户说“改语言后文案没有切换”，你先读哪两个入口？用户说“桌面端自动备份没有生成文件”，你先读哪两个入口？用户说“导入 JSON 后同步状态怪异”，你先读哪个恢复入口？

参考答案：语言先看 GlobalConfigEffects.applyLanguageFromState$ 和 LanguageService.setLng()，再查 assets/i18n key。桌面自动备份先看 LocalBackupService._backupElectron()，再看 electron/backup.ts 的 IPC/file guard。导入后同步异常先看 BackupService.importCompleteBackup()，尤其 BACKUP_IMPORT、pending write flush 和 lastServerSeq 重置。完整入口清单见 模块参考。主要来源：src/app/pages/config-page/config-page.component.ts、src/app/op-log/backup/backup.service.ts。

如果某个边界仍不清楚，直接向 agent 追问“这个症状应该从哪个入口读起”。
