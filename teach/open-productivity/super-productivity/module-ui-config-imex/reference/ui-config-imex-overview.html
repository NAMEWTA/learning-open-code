UI、配置、导入导出与本地备份参考

职责一句话：本模块把 L0 总览中的 Angular renderer UI 层、全局配置状态、导入导出入口和本地备份恢复入口串起来；数据一致性落到 op-log L1，桌面文件能力落到 Electron L1，启动与 shell 装配落到 Angular L1。

模块骨架

super-productivity/src/app/ ├── ui/ ← 可复用控件、Formly 类型、管道、动画、基础指令 ├── core-ui/ ← header、side nav、layout、bottom nav 等壳层 UI ├── core/ ← 主题、语言、日期、persistence、Electron bridge 类型 ├── pages/ │ └── config-page/ ← Settings 页面装配与保存分发 ├── features/ │ └── config/ ← 全局配置模型、默认值、表单、store/effects ├── imex/ ← 文件导入导出、同步 UI、自动备份 orchestration └── op-log/ └── backup/ ← 完整快照导出、恢复导入、legacy backup migration
super-productivity/ ├── src/ │ ├── styles/ ← 全局 SCSS、Material 覆盖、主题 mixin、组件级全局样式 │ └── assets/ │ └── i18n/ ← ngx-translate JSON 文案 └── electron/ ├── backup.ts ← 桌面端备份文件 IPC adapter ├── electronAPI.d.ts ← renderer 可见 window.ea 契约 └── shared-with-frontend/ ← backup 文件名、清理策略、IPC 常量等共享代码

职责边界

模块负责不负责
src/app/ui可复用展示和输入构件，例如 duration input、dialog、Formly 类型、Markdown、pipes。业务配置 schema、NgRx 持久化、Electron 文件系统。

src/app/op-log/backup完整状态快照、legacy backup migration、backup import operation、state cache/op-log baseline 替换。设置页按钮、文件选择 UI、桌面备份目录。

src/styles全局 SCSS、CSS variables、Material 覆盖、页面/组件全局样式。运行时主题选择逻辑和配置持久化。
src/assets/i18n翻译 JSON 文案资源。语言选择、RTL 信号、日期 locale 应用。

同步后 sync provider 被远端覆盖。global-config.reducer.tslocal-only-sync-settings.util.ts检查 remote action 是否走 withLocalOnlySyncSettings()，local-only keys 不应被其他设备覆盖。

边界判断表

你看到的问题应该去不要先去
一个按钮、输入框、dialog、pipe 或 Formly 控件行为异常。src/app/ui/ 或对应 feature component。op-log/backup、Electron 主进程。
header、side nav、right panel、移动底栏、布局状态异常。src/app/core-ui/ 和 Angular L1 的 root shell。features/config/form-cfgs。
设置页字段、默认值、保存、远端合并异常。src/app/features/config/。src/styles 或 assets/i18n。
语言、RTL、日期 locale、翻译 key 问题。LanguageService、main.ts、assets/i18n。备份或同步 provider。
主题模式、背景图、body class、Material 颜色变量。core/theme/、src/styles/。配置 reducer，除非问题是保存/恢复。
手动导入、URL 导入、备份下载、隐私导出。src/app/imex/file-imex/。Electron backup adapter，除非是桌面文件写入。
导入/恢复后本地状态、op-log baseline 或 sync seq 异常。src/app/op-log/backup/backup.service.ts 和 op-log L1。设置页 UI。
桌面端真实文件系统、备份目录、路径 guard、IPC。electron/backup.ts 和 Electron L1。Angular Formly 表单。

推荐阅读顺序

  1. 从 src/app/pages/config-page/config-page.component.ts 建立设置页 tab 和保存入口。
  2. 到 src/app/features/config/global-config.model.ts 和 default-global-config.const.ts 确认字段归属与默认值。
  3. 读 src/app/features/config/store/global-config.actions.ts，记住普通全局配置会以 GLOBAL_CONFIG operation 进入 op-log。
  4. 读 src/app/features/config/store/global-config.reducer.ts，特别是 loadAllData 合并默认值和 local-only sync 保护。
  5. 读 src/app/imex/file-imex/file-imex.component.ts，确认用户导入导出入口。
  6. 读 src/app/op-log/backup/backup.service.ts，确认导入恢复不是 UI 行为，而是 op-log baseline 替换。
  7. 桌面备份再跳到 src/app/imex/local-backup/local-backup.service.ts 和 electron/backup.ts。
  8. 主题/i18n 问题分别读 src/app/core/theme/、src/app/core/language/language.service.ts、src/assets/i18n/。
