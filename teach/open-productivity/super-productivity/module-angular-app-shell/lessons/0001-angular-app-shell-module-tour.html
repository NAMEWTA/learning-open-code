Angular 应用装配与全局壳层

一句话：这个模块把同一个 Angular renderer 装配成可运行应用，并在根组件中承载导航、全局状态、主题、快捷键、onboarding、移动端生命周期和页面出口。它位于 L0 全局地图里的 Angular SPA 层，不重复 Electron、packages 或业务领域全景。

1. 先记住装配顺序

读码顺序从 src/main.ts 开始：它调用 bootstrapApplication，把 Angular、NgRx、Material、Translate、service worker、Capacitor、插件初始化和 DataInitService 都接进 provider。然后 provideRouter(APP_ROUTES) 决定 URL 如何进入页面；AppComponent 再把 side nav、header、right panel、router-outlet 和移动底部导航组装成壳层。

src/main.ts
bootstrapApplication(AppComponent, {
  providers: [
    importProvidersFrom(FeatureStoresModule, StoreModule.forRoot(...)),
    provideRouter(APP_ROUTES, withHashLocation(), withPreloading(...)),
    PLUGIN_INITIALIZER_PROVIDER,
    provideZonelessChangeDetection(),
  ],
});

注意：PLUGIN_INITIALIZER_PROVIDER 只是把 'PLUGIN_INITIALIZER' token 接进 DI provider 表，不是 Angular APP_INITIALIZER 执行链。当前实际运行时插件初始化在 StartupService._initPlugins()：AppComponent 构造函数调用 StartupService.init()，延迟初始化任务等待数据初始加载和当前同步结束后，才调用 PluginService.initializePlugins()。

2. 路由只是入口，不是页面壳

src/app/app.routes.ts 暴露 APP_ROUTES：tag/:id/tasks 是主工作视图，project/:id 与 tag/:id 懒加载共享 context routes，config/search/planner/schedule/boards/habits/plugins 等页面从 pages barrel 懒加载。路由守卫负责验证当前 tag/project、默认起始页和 focus overlay 状态。

URL #/tag/TODAY/tasks
  → APP_ROUTES 中的 tag/:id/tasks
  → TagTaskPageComponent
  → AppComponent 模板里的 router-outlet
  → core-ui 壳层提供导航、header、panel、移动底栏

3. 根组件负责全局体验

src/app/app.component.ts 是壳层行为中心：构造函数触发 StartupService.init()，初始化主题和 RTL，等待初始同步后显示笔记/同步安全/加密迁移 banner，并处理键盘布局。模板放置 magic-side-nav、main-header、right-panel、router-outlet、global-progress-bar、focus overlay、onboarding 和背景右键菜单。feature reducers/effects 由 FeatureStoresModule 统一注册，布局状态再由 core-ui/layout 服务和 store 控制；插件页空白或插件加载失败时，优先沿 StartupService._initPlugins() → PluginService.initializePlugins() 继续查。

4. 检索练习

暂停 60 秒回答：如果用户报告“打开 #/planner 后页面空白，但侧栏和 header 还在”，你先看哪两个入口？如果用户报告“移动端退后台后数据丢失”，你先看哪个入口？

参考答案：页面空白先看 APP_ROUTES 中 planner 的懒加载入口，再看 AppComponent 的 router-outlet 是否仍被壳层承载。移动端退后台先看 src/main.ts 的 Capacitor appStateChange 与 OperationWriteFlushService flush 分支。完整入口清单见 模块参考。主要来源：src/main.ts。
