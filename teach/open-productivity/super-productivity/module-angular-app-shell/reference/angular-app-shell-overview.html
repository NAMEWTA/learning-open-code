Angular 应用装配与全局壳层参考

职责一句话：把 L0 总览中的 Angular SPA 层装配成可运行 renderer，并提供 URL 入口、全局 UI 壳层、feature store/effects 注册和平台生命周期接线。

模块骨架

super-productivity/ ├── src/ │ ├── main.ts ← Angular bootstrap 与跨平台 provider │ └── app/ │ ├── app.routes.ts ← 顶层 URL 与页面入口 │ ├── app.component.ts ← 根组件行为与全局体验 │ ├── app.component.html ← side nav/header/panel/router outlet 壳层 │ ├── root-store/ ← NgRx 根级注册 │ │ └── feature-stores.module.ts ← feature reducers/effects 注册表 │ ├── core-ui/ ← 共享 UI 基建 │ │ └── layout/ ← 壳层布局状态与响应式断点 │ └── core/ ← 核心启动与平台服务 │ └── startup/ ← 启动后初始化任务

对外入口清单

入口源码职责定位时机
bootstrapApplication(AppComponent, ...)src/main.ts创建 Angular 应用，集中注册 providers。启动失败、provider 缺失、平台分支异常。
APP_ROUTESsrc/app/app.routes.ts定义 tag/project 工作区、独立页面、插件页和 fallback。从 URL、hash route、默认页跳转反查页面。
AppComponentsrc/app/app.component.ts根组件，承载全局 UI shell、主题、快捷键、onboarding、banner、背景菜单。导航、header、panel、全局体验或壳层状态问题。
FeatureStoresModulesrc/app/root-store/feature-stores.module.ts统一注册 feature reducers 与 effects。某个业务域 action 没响应、effects 未触发、平台条件 effects 排查。
StartupService.init()src/app/core/startup/startup.service.ts执行单实例检查、备份/持久化、主题预加载、Electron ready，并在延迟任务中进入 _initPlugins()。启动后初始化顺序、Electron ready 或插件运行时初始化问题。
LayoutServicesrc/app/core-ui/layout/layout.service.ts桥接 layout store、断点、add task bar、side panel、焦点恢复。壳层布局、移动底栏、面板开关、任务焦点问题。

内部分层结构

层级关键源码读法
providerssrc/main.ts先看 importProvidersFrom(...) 与 standalone providers：NgRx、Material、Translate、HTTP、router、service worker、插件 provider 和 APP_INITIALIZER 都在这里接线。
routersrc/app/app.routes.ts用 URL 匹配 path；注意 hash location、PreloadAllModules、守卫和 lazy component/routes。
root componentsrc/app/app.component.ts、src/app/app.component.htmlTS 负责全局副作用和 shell 状态，HTML 负责把 router outlet 放进导航/面板/进度条/onboarding 布局。
feature stores/effectssrc/app/root-store/feature-stores.module.ts先按 feature key 找 reducer，再看 effects；Android/iOS/Electron effects 会按平台条件注册。
core-ui shellsrc/app/core-ui/layout/布局服务将 store selector 转为 signal，并派发 panel、add task bar、滚动和焦点动作。

关键依赖与启动职责

依赖/机制注册位置在本模块中的作用
Angular standalone bootstrapsrc/main.ts用 bootstrapApplication 启动 AppComponent，并启用 zoneless change detection。
NgRx Store / Effectssrc/main.ts、src/app/root-store/feature-stores.module.ts根 store 使用 meta-reducers；feature module 注册 app state、layout、config、tasks、project、tag、sync、plugins 等 reducer/effects。
Angular Material / CDKsrc/main.ts、src/app/app.component.ts提供 date adapter、dialog 默认值、sidenav、bottom sheet、drag/drop group、menu/icon 等壳层 UI 能力。
Translatesrc/main.ts通过 TranslateModule.forRoot 和 fallback loader 加载 src/assets/i18n/*.json。
Service workersrc/main.tsWeb production/stage 下注册 ngsw-worker.js；Electron 或 Native 平台会注销已有 service worker。
Capacitor lifecyclesrc/main.tsAndroid back button、Android/iOS appStateChange、后台 OperationWriteFlushService flush 和 iOS URL open。
插件初始化src/app/core/startup/startup.service.ts、src/app/plugins/plugin.service.ts当前实际运行时路径是 AppComponent 构造函数调用 StartupService.init()，再由 StartupService._initPlugins() 等待数据加载和同步完成后调用 PluginService.initializePlugins()。PLUGIN_INITIALIZER_PROVIDER 只是注册 'PLUGIN_INITIALIZER' token 的 provider 接线点，不是 Angular APP_INITIALIZER 执行链。
DataInitsrc/main.ts、src/app/core/startup/startup.service.tsDataInitService 在 bootstrap 时实例化，DataInitStateService 被 Android overlay、banner 和 Electron 设置发送等流程等待。

定位示例

症状第一入口第二入口判断标准
#/planner 空白，但全局壳层仍显示。src/app/app.routes.tssrc/app/app.component.html壳层在，先判断 lazy component 或路由数据；壳层不在，回到 root component/bootstrap。
应用首屏停在启动状态。src/main.tssrc/app/core/startup/startup.service.ts先查 APP_INITIALIZER、DataInit、service worker 和平台分支，再看启动后延迟任务。
插件加载失败或插件页空白。src/app/core/startup/startup.service.tssrc/app/plugins/plugin.service.ts按 StartupService._initPlugins() → PluginService.initializePlugins() → plugin metadata/user-data store（src/app/plugins/plugin-meta-persistence.service.ts、src/app/plugins/plugin-user-persistence.service.ts）排查。
Android/iOS 退后台后最近操作丢失。src/main.tssrc/app/root-store/feature-stores.module.ts先查 Capacitor appStateChange 是否 flush op-log，再查移动端条件 effects 是否注册。
任务列表新增后焦点或滚动异常。src/app/app.component.tssrc/app/core-ui/layout/layout.service.ts先查 add task bar 事件与 onTaskAdded，再查布局服务中的聚焦/滚动重试。
某业务 action 没有副作用。src/app/root-store/feature-stores.module.ts对应 feature 的 store 目录先确认 effects 是否注册，再进入业务模块。
