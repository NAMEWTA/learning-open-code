Focus session、break、metric 与当前任务追踪联动

Super Productivity 把专注模式做成本地 timer 状态机，再经 effect 与 currentTaskId 双向锁步。session 完成才写 metric；break 是否暂停追踪取决于配置；idle 会先把 focus session pause 住。完整索引见 参考文档；相邻主题：规划与时间模块、时间追踪链路。

  end

  alt Pomodoro work 计时到点

    FocusFx->>FocusStore: completeFocusSession

  end

首查文件是 src/app/features/focus-mode/store/focus-mode.effects.ts：它以 TaskService.currentTaskId$ 和 focus action 为两条输入轴，负责 session ↔ tracking 的双向同步、break 自动启动与 metric 写入。

2. 三类状态别混读

层保存在哪代表什么常见误判
focus timerfocusMode.timer + screenwork/break 是否在跑、剩余/已用时长它不是任务耗时；Flowtime 的 duration 为 0。
current tasktask.currentTaskId当前正在 wall-clock 追踪哪条任务本地 UI 状态，不带 op-log meta；秒级 tick 另写 timeSpent。
focus metricmetric[day].focusSessions[]已完成 session 的 duration 列表只在 completeFocusSession 后写入，break 本身不进 metric。

Pomodoro 的 break 链是：completeFocusSession → incrementCycle → startBreak。若开启 isPauseTrackingDuringBreak，break 开始前会 unsetCurrentTask 并把 task id 存进 action 的 pausedTaskId，break 完成或 skip 后再 setCurrentTask 恢复。

3. 三个判断点

判断一：用户点任务行 play，focus 没开 overlay 也会联动。查 syncTrackingStartToSession$：work session 暂停会 unPauseFocusSession；break 中手动追踪会 skipBreak；无 session 且开启 autoStartFocusOnPlay 会静默 startFocusSession。

判断二：session 结束是否还追踪任务，看 isPauseTrackingDuringBreak 与 mode strategy。Pomodoro 自动 break 走 autoStartBreakOnSessionComplete$；Flowtime 手动结束走 autoStartFlowtimeBreakOnSessionEnd$，action 顺序刻意避免与 syncTrackingStopToSession$ 竞态。

判断三：metric 何时增加，查 logFocusSession$。它监听 completeFocusSession，用 selectLastSessionDuration 调 MetricService.logFocusSession；duration 低于门槛则 no-op。

4. 60 秒自测

场景：Pomodoro 模式，isPauseTrackingDuringBreak=true。25 分钟 work session 自动结束，用户休息 5 分钟后点「完成休息」，期间没有手动 play 其他任务。请判断：session 结束时 current task 是否还在追踪？metric 何时 +1？break 完成后 tracking 如何恢复？

参考答案：session 自动完成时 autoStartBreakOnSessionComplete$ 会先 unsetCurrentTask 再 startBreak，因此 break 期间不在追踪。metric 在 completeFocusSession 时经 logFocusSession$ 写入当日 focusSessions。用户点完成休息后 completeBreak 触发 resumeTrackingOnBreakComplete$，若当前无 current task 且 action 携带 pausedTaskId，则 setCurrentTask 恢复；若 strategy 允许还会 startFocusSession 开下一轮。

主要来源：Focus 联动速查。若某条 effect 顺序仍不清楚，可以向 agent 追问：“请从 completeFocusSession 列出所有订阅它的 effect 及 dispatch 顺序”。
