状态层边界速查

层主文件保存或派生什么不是
focus timer focus-mode.reducer.ts timer.purpose、elapsed、duration、screen 任务 timeSpent；break 时长不进 metric。
current task task.reducer.ts currentTaskId、lastCurrentTaskId 不带 persistent meta；远端 sync 不直接改 local current task。
wall-clock 追踪 task.service.ts 每秒累加 timeSpentOnDay，batch 后 flush timeTracking op 与 focus timer tick 是两条独立时钟链。
focus metric metric.reducer.ts Metric.focusSessions[] 按日追加 duration 不是 session 进行中实时更新；完成时才 log。

FocusModeEffects 联动矩阵

Effect触发主要 dispatch目的
syncTrackingStartToSession$ currentTaskId$ 变为非空 unPauseFocusSession / skipBreak / startFocusSession 追踪开始 → 恢复或静默启动 session
syncTrackingStopToSession$ currentTaskId$ 变为 null pauseFocusSession 追踪停止 → 暂停 work/break timer
syncSessionPauseToTracking$ pauseFocusSession unsetCurrentTask session 暂停 → 停止追踪（若仍有 current）
syncSessionResumeToTracking$ unPauseFocusSession setCurrentTask / clearResumingBreakFlag session 恢复 → 恢复 paused task
syncSessionStartToTracking$ startFocusSession setCurrentTask / showFocusOverlay session 开始 → 自动追踪或提示选任务
detectSessionCompletion$ work timer 到点停止 completeFocusSession Pomodoro/Countdown 自动完成
autoStartBreakOnSessionComplete$ incrementCycle unsetCurrentTask + startBreak Pomodoro 自动 break
autoStartFlowtimeBreakOnSessionEnd$ endFlowtimeSession completeFocusSession + break 链 Flowtime 手动结束 → 可选 break
resumeTrackingOnBreakComplete$ completeBreak setCurrentTask break 完成 → 恢复 paused task
skipBreak$ skipBreak setCurrentTask + 可选 startFocusSession 跳过 break 并续 session
logFocusSession$ completeFocusSession （无 dispatch）→ MetricService.logFocusSession 写当日 focusSessions
pauseOnIdle$ openIdleDialog pauseFocusSession idle 弹窗前暂停 focus

Mode strategy 差异

Modesession 结束自动 breakbreak 后自动下一 session
Pomodoro timer 到点 → completeFocusSession 是，按 cycle 长短休息 是（shouldAutoStartNextSession）
Flowtime 用户 endFlowtimeSession 配置开启时按 elapsed 比例 否
Countdown timer 到点 → completeFocusSession 否 否；overlay 关闭时走 banner 提醒

配置影响

阅读顺序建议

  1. focus-mode.model.ts — 认清 timer 与 screen 枚举。
  2. focus-mode.reducer.ts — 看 tick 到点如何停 timer。
  3. focus-mode.effects.ts — 从 syncTrackingStartToSession$ 与 syncTrackingStopToSession$ 读双向同步。
  4. focus-mode-strategies.ts — 判断 break / auto-next 是否启用。
  5. metric.service.ts + logFocusSession$ — 确认 metric 写入时机。
