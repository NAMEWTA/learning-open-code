Bridge、OAuth、Node executor 与 issue provider 边界

本课把运行期调用拆成三条线：iframe/postMessage 只是一种 UI 调用通道；OAuth 与 Node executor 要过 Electron 边界；issue provider 注册和 sync adapter 懒创建不是同一个条件。

相关课程：生命周期地图 / 启动初始化。 相邻主题：Issue 导入与双向同步。

调用时序图

plugin.js / iframe PluginAPI
  -> PluginBridgeService.createBoundMethods(pluginId)
     -> app state / task / config / persistence APIs
     -> startOAuthFlow()
        -> PluginOAuthBridgeService
        -> electron/plugin-oauth.ts loopback + system browser
     -> executeNodeScript()
        -> renderer grant check
        -> electron/plugin-node-executor.ts token + webContents check
     -> registerIssueProvider()
        -> PluginIssueProviderRegistryService.register()
        -> maybe immediate IssueSyncAdapterRegistryService.register()

later, issue two-way sync
  -> IssueSyncAdapterResolverService.getAdapter(issueType)
     -> existing adapter OR plugin registry definition
     -> createPluginSyncAdapter() when createIssue/deleteIssue/updateIssue exists

sync adapter 条件必须分两层

层级源码条件含义
Bridge 注册时立即注册 src/app/plugins/plugin-bridge.service.ts createIssue 或 deleteIssue 或 fieldMappings.length && updateIssue 注册 provider 的同时把 sync adapter 放进 registry；普通字段 push 仍需要 fieldMappings 和 updateIssue。
运行期懒创建 src/app/features/issue/two-way-sync/issue-sync-adapter-resolver.service.ts createIssue 或 deleteIssue 或 updateIssue updateIssue-only provider 也可创建 adapter；源码注释说明这是为了支持没有 task field mappings 的直接事件变更。
实际字段 push src/app/plugins/issue-provider/plugin-sync-adapter.service.ts updateIssue 加可 push 的 field mappings adapter 存在不代表每个 task 字段都能 push；字段方向由 mapping 决定。

最小源码锚点

// Bridge immediate registration
if (
  definition.createIssue ||
  definition.deleteIssue ||
  (definition.fieldMappings?.length && definition.updateIssue)
) {
  this._syncAdapterRegistry.register(registeredKey, adapter);
}

// Resolver lazy creation
if (!definition.createIssue && !definition.deleteIssue && !definition.updateIssue) {
  return undefined;
}

异常路径

失败点表现先查
iframe API 调用失败 postMessage 没回或返回 unsupported bridge token、generation、iframe allowlist、host-only API。
OAuth 失败 无 token、state mismatch、loopback 错误 OAuth config、platform client id、Electron loopback server。
Node script 拒绝执行 permission denied 或 grant missing manifest nodeExecution、用户 consent、grant token、webContents 是否还活着。
provider 能搜索但不能同步 搜索 issue 可用，push/delete/create 不触发 adapter 是否存在；再看 capability 与 fieldMappings，而不是只看 provider 是否注册。

60 秒判断题

一个插件只实现了 updateIssue，没有 fieldMappings、createIssue、deleteIssue。Bridge 注册时会立即注册 sync adapter 吗？后续 resolver 有没有可能懒创建 adapter？

参考答案与反馈

Bridge 不会立即注册，因为它要求 create/delete 或 fieldMappings+updateIssue。resolver 可以懒创建，因为它的条件是 create/delete/updateIssue 任一存在；这用于没有 task field mappings 但需要直接推送事件变更的 provider。
