层源码位置负责什么不负责什么

Service 门面src/app/features/tasks/task.service.ts创建默认 `Task`，派发 `TaskSharedActions.addTask` 或 schedule action。不直接维护 project/tag 双向列表。

op-log 捕获src/app/op-log/capture/operation-log.effects.ts把本地 persistent action 转成 operation 写入 `SUP_OPS`。不捕获输入框临时状态或 remote replay。

4. 归属规则

Project

创建默认值来自 active project、全局默认项目或 Inbox。新增主任务时，src/app/root-store/meta/task-shared-meta-reducers/task-shared-crud.reducer.ts 根据 `isAddToBacklog` 和项目是否启用 backlog 写入 `taskIds` 或 `backlogTaskIds`。

普通 Tag

`#tag` 匹配已有 tag 时会写入 `Task.tagIds`。`addTask`/`updateTask` 走 src/app/root-store/meta/task-shared-meta-reducers/task-shared-crud.reducer.ts，新增路径和 `handleTagUpdates()` 会维护普通 `tag.taskIds`；`applyShortSyntax` 走 src/app/root-store/meta/task-shared-meta-reducers/short-syntax-shared.reducer.ts，只改 `Task.tagIds`，普通 tag 视图成员再由 selector 从 task 侧推导。Today 是例外：它不会进入 `Task.tagIds`，只用 `TODAY_TAG.taskIds` 存排序。

Today

`TODAY_TAG` 是虚拟标签。成员由 src/app/features/work-context/store/work-context.selectors.ts 判断：先看今天的 `dueWithTime`，没有 timed due 时才看 `dueDay === todayStr`。

Planner

未来 `dueDay` 会进入 planner day；Today 使用 `TODAY_TAG.taskIds` 保序。`scheduleTaskWithTime` 会清 `dueDay`，写 `dueWithTime` 和 `remindAt`。

5. op-log 持久化边界

动作actionop 类型说明
创建任务src/app/root-store/meta/task-shared.actions.tsCreate TASK`TaskSharedActions.addTask` 带 `meta.isPersistent`。

timed due / remindsrc/app/root-store/meta/task-shared.actions.tsUpdate TASK`scheduleTaskWithTime` 写 `dueWithTime`、清 `dueDay`、写 `remindAt`。
创建新 tagsrc/app/features/tag/store/tag.actions.tsCreate TAG新 tag 本身是独立 operation。
创建重复配置src/app/features/task-repeat-cfg/store/task-repeat-cfg.actions.tsCreate TASK_REPEAT_CFGrepeat cfg 与 task 创建分开持久化。

最终写入由 src/app/op-log/persistence/operation-log-store.service.ts 的 `appendWithVectorClockUpdate` 完成：operation 进入 `ops` store，并在同一事务更新 vector clock。

6. 故障排查

现象优先检查常见原因
输入 `@tomorrow` 后仍出现在 Todaysrc/app/features/tasks/add-task-bar/add-task-bar-parser.service.ts 和 src/app/root-store/meta/task-shared-meta-reducers/task-shared-crud.reducer.ts默认 Today date 未被短语法覆盖，或 timed due 后未走 schedule/update Today 顺序。
`+project` 没有移动项目src/app/features/tasks/short-syntax.ts项目被 archived/hidden、名称前缀未匹配、或任务是 repeatable task 导致 effect 拒绝移动。
`#123` 被当作 tag 或未当作 tagsrc/app/features/tasks/short-syntax.ts数字 tag 在开头、issue task、空格边界规则影响解析。
deadline 被清掉src/app/features/tasks/add-task-bar/add-task-bar-parser.service.ts解析器区分“短语法拥有的 deadline”和“用户手动选择的 deadline”；移除 `!` 语法会清短语法拥有的值。
创建任务后同步没有这条 opsrc/app/op-log/core/persistent-action.interface.ts 和 src/app/op-log/capture/operation-log.effects.tsaction 未带 `meta.isPersistent`、entityId 无效、remote/deferred 路径被过滤，或 op payload 校验失败。
重复任务创建后提醒重复src/app/features/tasks/add-task-bar/add-task-bar.component.tstimed repeat 有专门路径，组件会跳过普通 `scheduleTask`，避免双重 scheduling。
