本地捕获、持久化与 flush 时序

1. src/app/root-store/meta/task-shared.actions.ts
   action.meta.isPersistent = true

2. src/app/op-log/capture/operation-capture.meta-reducer.ts
   本地 action：OperationCaptureService pending +1
   remote action：meta.isRemote=true，跳过 capture
   remote apply 窗口：deferredActions 缓冲

3. src/app/op-log/capture/operation-log.effects.ts
   concatMap 顺序处理
   生成 uuidv7 / clientId / vectorClock / MultiEntityPayload
   validateOperationPayload()
   appendWithVectorClockUpdate(op, "local")
   finally pending -1
   immediateUploadService.trigger()

4. src/app/op-log/persistence/operation-log-store.service.ts
   同一事务写 SUP_OPS.ops 与 SUP_OPS.vector_clock

5. src/app/op-log/sync/operation-write-flush.service.ts
   Phase 1: 等 pending counter 归零
   Phase 2: 拿 LOCK_NAMES.OPERATION_LOG
   之后 upload / conflict detection 才能读完整本地队列

上传时序

src/app/imex/sync/sync-wrapper.service.ts
  ↓ 先 downloadRemoteOps()，再 uploadPendingOps()
src/app/op-log/sync/operation-log-sync.service.ts
  ↓ flushPendingWrites()
  ↓ isWhollyFreshClient() 阻止空白新客户端上传
  ↓ serverMigrationService.checkAndHandleMigration()
src/app/op-log/sync/operation-log-upload.service.ts
  ↓ getUnsynced()
  ↓ full-state ops 走 uploadSnapshot()
  ↓ regular ops 转 SyncOperation；local-only sync settings 先剥离
  ↓ SuperSync 且有 key：OperationEncryptionService.encryptOperations()
  ↓ provider.uploadOps()
  ↓ accepted seq 标记 synced
  ↓ piggybacked ops 返回给 OperationLogSyncService 先处理
  ↓ rejected ops 最后交给 RejectedOpsHandlerService

上传路径的关键边界：server rejected ops 不能先标记 rejected，因为 piggybacked ops 可能正是冲突另一侧；必须先处理 piggybacked remote ops，让冲突检测还能看到本地 pending ops。

下载与远端 apply 时序

src/app/op-log/sync/operation-log-download.service.ts
  ↓ provider.downloadOps(sinceSeq, clientId, pageSize)
  ↓ gapDetected 时只允许一次 reset to 0
  ↓ 用 SUP_OPS applied op ids 过滤重复
  ↓ encrypted ops 需要 getEncryptKey()，否则 DecryptNoPasswordError
  ↓ 返回 newOps + latestServerSeq，但不直接推进游标

src/app/op-log/sync/operation-log-sync.service.ts
  ↓ fileSnapshotOps + snapshotState：必要时 hydrate snapshot
  ↓ fresh client 有 meaningful local data：LocalDataConflictError
  ↓ incoming SYNC_IMPORT：先跑 conflict gate
  ↓ processRemoteOps()
  ↓ newOps 已存入/应用后才 setLastServerSeq()

src/app/op-log/sync/remote-ops-processing.service.ts
  ↓ schema migration
  ↓ SYNC_IMPORT filter
  ↓ full-state op 直接 apply
  ↓ flushPendingWrites()
  ↓ vector clock conflict detection
  ↓ LWW 或 direct apply
  ↓ validateAfterSync()

provider 边界

维度SuperSync: superSyncOps文件型 provider: fileSnapshotOps
入口src/app/op-log/sync-providers/super-sync/super-sync.ts 创建 package provider。src/app/op-log/sync-providers/file-based/file-based-sync-adapter.service.ts 包装 Dropbox/WebDAV/Nextcloud/OneDrive/LocalFile。
远端形态服务端 append-only operation API；上传返回 accepted/rejected 和可能的 piggybacked ops。单个远端同步文件，含 state snapshot、archive、recentOps、vectorClock、syncVersion。
游标packages/sync-providers/src/super-sync/super-sync.ts 按 baseUrl + access token 派生 lastServerSeq storage key。adapter 在 localStorage 中维护 provider 级 syncVersion / synthetic last seq。
加密mandatory E2EE；src/app/op-log/sync/operation-log-upload.service.ts 上传前加密单条 operation payload。文件型 provider 允许明文或加密；adapter 用 src/app/op-log/encryption/encrypt-and-compress-handler.service.ts 处理整个 sync 文件。
压缩packages/sync-providers/src/super-sync/super-sync.ts 对 HTTP upload body gzip，设置 Content-Encoding: gzip。sync 文件可 gzip 后再加密；prefix 记录压缩/加密/模型版本。
冲突信号server 可返回 rejected ops、existingClock、piggybacked ops。rev mismatch、syncVersion reset、snapshot replacement、partial trimming 会触发重试或 gap。

加密与压缩边界

路径处理对象格式与错误
src/app/op-log/sync/operation-encryption.service.tsSuperSync 的 SyncOperation.payload 或 snapshot state。payload 先 JSON stringify，再用 packages/sync-core/src/encryption.ts 的 Argon2id + AES-GCM；结果设置 isPayloadEncrypted=true。解密失败抛 DecryptError。
src/app/op-log/encryption/encrypt-and-compress-handler.service.ts文件型 provider 的整个远端同步文件字符串。prefix 标记压缩、加密和 modelVersion；缺密码抛 DecryptNoPasswordError，JSON parse 失败抛 JsonParseError。
packages/sync-core/src/compression.tsgzip 原语。SuperSync HTTP body 使用 raw gzip bytes；文件型 sync 文件使用 base64 gzip 字符串。

故障排查

现象先查判断
刚操作后立即同步，远端缺变更。src/app/op-log/sync/operation-write-flush.service.ts看 pending counter 是否卡住；再查 src/app/op-log/capture/operation-log.effects.ts 是否写失败或 lock timeout。
同步重复弹冲突，尤其是 Dropbox/WebDAV。src/app/op-log/sync-providers/file-based/file-based-sync-adapter.service.ts检查 syncVersion reset、snapshot replacement、partial trimming；再看 src/app/op-log/sync/operation-log-sync.service.ts 是否正确跳过 dominated snapshot hydration。
收到加密数据但没有密码。src/app/op-log/sync/operation-log-download.service.tsencrypted ops + no key 会抛 DecryptNoPasswordError；如果本地配置显示 encryption enabled 却没有 key，要按 dropped credential 处理。
SuperSync 初次设置时本地 ops 没上传。src/app/op-log/sync/operation-log-upload.service.tsSuperSync mandatory encryption 没有 key 时会跳过上传，避免明文泄露；wrapper 应报告 UNKNOWN_OR_CHANGED / UpdateRemote。
服务端 rejected ops 后本地变更丢失。src/app/op-log/sync/operation-log-sync.service.ts确认 piggybacked ops 是否先处理；rejected ops 是否只在冲突检测后标记 rejected。
远端 apply 后又生成新本地 operation。src/app/op-log/apply/operation-converter.util.ts确认 replay action 有 meta.isRemote=true；capture meta-reducer 应过滤它。
状态校验失败却显示已同步。src/app/op-log/sync/sync-session-validation.service.ts检查入口是否包在 withSession 内；side channels 包括 src/app/op-log/sync/immediate-upload.service.ts 和 src/app/op-log/sync/ws-triggered-download.service.ts。
文件型 provider 发现旧同步文件。src/app/op-log/sync-providers/file-based/file-based-sync-adapter.service.ts如果没有新协议同步文件但有 legacy __meta_，抛 LegacySyncFormatDetectedError，避免新旧协议 silent divergence。
存储空间不足或 IndexedDB 写失败。src/app/op-log/capture/operation-log.effects.ts普通写失败提示 reload；quota exceeded 会尝试 emergency compaction，再失败则提示 storage quota 错误。
