L2 短课 · Today / Project / Tag / WorkContext / MenuTree

任务进入视图的归属判定全链路

本课只训练一个能力：看到一个任务和一次 action 后，能判断它为什么出现在 Today、Project、Tag 或菜单导航里。

学习目标

先把四类“归属”拆开：Today 的成员由日期字段决定；普通 Tag 的成员由 task.tagIds 决定；Project 视图主要读 project.taskIds / backlogTaskIds；MenuTree 只决定 Project/Tag 在导航树里的位置和可见入口。

父级知识见 L1 任务领域总览；任务创建入口见 相关 L2：新增任务链路。

关键源码锚点：src/app/root-store/meta/task-shared.actions.ts、src/app/features/work-context/store/work-context.selectors.ts、src/app/features/menu-tree/menu-tree.service.ts。

判断点

1. 是否属于 Today？先看 dueWithTime 是否落在今天；没有具体时间才看 dueDay === todayStr。TODAY_TAG.taskIds 只保排序，不能当成员真相。
2. 是否属于普通 Tag？看 task.tagIds.includes(tagId)。tag.taskIds 用来保序，selector 会过滤陈旧 id，并把缺失但带 tag 的任务追加到末尾。
3. 是否属于 Project？写侧会维护任务 projectId、项目 taskIds 和 backlogTaskIds。Project 的 active work context 直接暴露这些列表。
4. 是否在菜单里可见？MenuTree 水合 Project/Tag 导航树。Project 若 archived 或 hidden 会从 UI 列表中过滤；普通 Tag 树排除 Today。导航可见性不改变任务成员。

子任务还有一个展示层例外：src/app/features/tasks/util/resolve-display-tag-ids.util.ts 会让未显式打 tag 的子任务继承父任务标签用于展示，但它不改写 task.tagIds。

60 秒自测

不看源码，判断下面三条任务分别会出现在哪里：

  1. A：dueWithTime 是明天 9 点，旧数据里还残留 dueDay === todayStr。
  2. B：tagIds = ['x']，但标签 x 的 taskIds 里没有 B。
  3. C：属于 Project P，P 被隐藏出菜单，但没有 archived。
参考答案与反馈

A 不属于 Today，因为读侧先看 dueWithTime，不会再用残留 dueDay。B 属于标签 x，排序时会被追加到末尾。C 仍属于 Project P 的任务列表，但 P 不会出现在普通项目菜单里；hidden 是导航入口问题，不是任务归属问题。
