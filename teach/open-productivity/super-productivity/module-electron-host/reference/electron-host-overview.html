Electron 桌面宿主参考

模块职责

职责一句话：把 Angular renderer 装入桌面窗口，并在主进程中集中管理单实例、窗口安全、IPC、托盘、idle/powerMonitor、协议 URL、本地文件、插件 Node/OAuth、本地 REST API 和退出清理。

它位于 L0 总览中 Angular renderer 之外的桌面宿主层；Angular L1 负责 renderer 装配，op-log L1 负责同步/持久化边界，本参考只处理 Electron 主进程和桌面专用能力。

模块骨架

super-productivity/electron/ ├── main.ts ← 单实例入口与 startApp 调用 ├── start-app.ts ← 主进程总装配：ready、IPC、窗口、托盘、idle、退出 ├── main-window.ts ← BrowserWindow、preload、导航安全、关闭前协议 ├── navigation-guard.ts ← 应用 origin 判断，阻止非应用页面进入特权窗口 ├── open-url.ts ← file: URL 与本地路径打开 guard ├── shared-with-frontend/ │ └── is-external-url-allowed.ts ← 外链 scheme、本地 file URL、UNC 与可执行扩展 guard ├── preload.ts ← contextBridge 暴露 window.ea ├── electronAPI.d.ts ← renderer 可见的 typed bridge 契约 ├── ipc-handler.ts ← IPC handler 聚合入口 ├── ipc-handlers/ ← app-control、app-data、system、shortcuts、jira ├── indicator.ts ← tray、当前任务、进度和 task widget 联动 ├── local-file-sync.ts ← LocalFile provider 的主进程文件桥 ├── local-rest-api.ts ← 本地 HTTP API 与 renderer 转发 ├── plugin-node-executor.ts ← 插件 Node 执行授权与脚本执行 ├── plugin-oauth.ts ← 插件 OAuth loopback 与系统浏览器 └── protocol-handler.ts ← superproductivity:// 协议 URL 处理

对外入口清单

入口源码职责先读什么
进程入口electron/main.ts非 macOS 单实例锁；未拿到锁立即退出；protocol URL 只抑制普通重复实例日志，不让第二进程进入 startApp()。app.requestSingleInstanceLock()、process.exit(0)

preload bridgeelectron/preload.ts
electron/electronAPI.d.ts		把主进程 IPC 能力包装为 window.ea；提供 renderer 可见类型。contextBridge.exposeInMainWorld('ea', ea)、ElectronAPI

electron/ipc-handlers/index.ts		初始化插件 Node 执行器、app-control、app-data、system、jira、global shortcuts、clipboard image、本地 REST API。initIpcInterfaces()

内部分层结构

层级关键源码边界判断
main process bootstrapelectron/main.ts、electron/start-app.ts只处理 Electron app 生命周期、进程级参数、系统事件和主进程模块初始化。
window hostelectron/main-window.ts、electron/navigation-guard.ts窗口配置、preload 注入、网络 header 钩子、权限、导航和外链策略在这里；Angular route 不在这里定义。
bridge contractelectron/preload.ts、electron/electronAPI.d.ts、src/app/core/window-ea.d.tsrenderer 只看到 window.ea；bridge 包装 ipcRenderer.send、ipcRenderer.invoke 和事件监听。
IPC handlerselectron/ipc-handlers/app-control.ts、electron/ipc-handlers/app-data.ts、electron/ipc-handlers/system.ts、electron/ipc-handlers/global-shortcuts.ts、electron/ipc-handlers/jira.ts主进程实际执行 app 控制、路径、外链、保存对话框、系统快捷键和 Jira 请求。
desktop capability moduleselectron/indicator.ts、electron/idle-time-handler.ts、electron/protocol-handler.ts、electron/local-file-sync.ts、electron/local-rest-api.ts、electron/plugin-node-executor.ts、electron/plugin-oauth.ts需要 OS、Node、HTTP server、child process、Tray 或 shell 的能力都在主进程模块。

单实例与协议 URL 分派

场景源码实际行为
非 macOS 未拿到 single-instance lockelectron/main.ts无论 argv 是否带 superproductivity://，都会执行 process.exit(0)。差异只是普通重复实例打印 Another instance is already running. Exiting.，protocol URL 重复实例不打印这条普通日志。
Windows/Linux 已运行主实例electron/protocol-handler.tsinitializeProtocolHandling() 注册 second-instance，从 commandLine 找 superproductivity:// 并调用 processProtocolUrl()；toggle-visibility 不做预先 focus。
macOS protocol URLelectron/protocol-handler.tsopen-url 事件拦截 superproductivity://，阻止默认处理后转给 processProtocolUrl()。
命令行 cold-start / 测试入口electron/protocol-handler.ts启动中的主实例扫描 process.argv；普通 action 等 whenReady() 后处理，toggle-visibility 只设置 coldStartShowPending，避免新窗口刚显示又被 toggle 隐藏。

窗口网络、权限与外链边界

边界源码要点
主窗口 WebPreferenceselectron/main-window.tswebSecurity: true、nodeIntegration: false、contextIsolation: true，并只通过 preload.js 暴露 window.ea。
请求头改写onBeforeSendHeaders剥离 Origin、Referer、Cookie、sec-ch-*、sec-fetch-*、accept* 等浏览器请求头；特定 GitHub / Office 主机还移除 User-Agent。
CORS 响应改写onHeadersReceived注入 Access-Control-Allow-Origin、Access-Control-Allow-Headers、Access-Control-Allow-Methods 为 *，并把所有 OPTIONS 响应强制为 HTTP/1.1 200 OK。
权限请求setPermissionRequestHandler
setPermissionCheckHandler		只允许 notifications，拒绝 webcam、microphone、geolocation 等不需要的页面权限。
导航与新窗口initWinEventListeners()
electron/navigation-guard.ts		will-navigate 和 will-redirect 只允许应用自身 origin；setWindowOpenHandler 先调用受控外链打开，再返回 { action: 'deny' }，意外子窗口会被销毁。
外链与本地文件electron/shared-with-frontend/is-external-url-allowed.ts
electron/open-url.ts
electron/ipc-handlers/system.ts		外链先过 scheme allowlist；file: 和本地路径走 openLocalPath()，拒绝 UNC / remote file URL，并阻止可执行或脚本扩展交给 OS 打开。
自签证书electron/start-app.ts全局 certificate-error 回调 true 是为了兼容自托管 Jira、GitLab、Redmine 等自签证书服务；注释明确这是降低 TLS 安全性换兼容性，不是安全加强。

依赖/机制所在层在本模块中的作用

阅读示例：从 renderer API 名反查主进程

  1. 看到 renderer 代码调用 window.ea.fileSyncSave(args)，先到 electron/electronAPI.d.ts 确认参数和返回类型。
  2. 到 electron/preload.ts 找同名方法，看到它调用 _invoke('FILE_SYNC_SAVE', args)。
  3. 用 FILE_SYNC_SAVE 到 electron/shared-with-frontend/ipc-events.const.ts 确认 IPC 常量。
  4. 回主进程搜索 IPC.FILE_SYNC_SAVE，落到 electron/local-file-sync.ts 的 ipcMain.handle()。
  5. 继续读 resolveSyncPath()、assertPathOutside() 和 atomic write，判断这是主进程负责的文件系统边界，不是 Angular service 的职责。
