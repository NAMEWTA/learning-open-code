Electron 桌面宿主、窗口与 IPC 边界

一句话：Electron 宿主把 L0 地图里的 Angular renderer 放进桌面窗口，并独占托盘、idle、协议、本地文件、插件 Node/OAuth、本地 REST API 等系统能力。

1. 先分清四个位置

读 Electron 代码时，先不要追完整启动/退出链路，只判断“这段代码在哪个权限层”。主进程拥有 Electron 和 Node 权限；preload bridge 在隔离上下文里把受控 API 暴露给页面；renderer 仍是 Angular 应用；系统能力只能由主进程模块执行。

Electron main process
  → BrowserWindow + ipcMain handlers + desktop modules
preload bridge
  → contextBridge exposes window.ea
Angular renderer
  → calls window.ea methods, receives IPC events
system capability
  → tray / idle / protocol / local file / plugin / local REST

2. 宿主从单实例入口开始

electron/main.ts 很薄：非 macOS 先申请单实例锁；如果没拿到锁，即使命令行带 superproductivity:// URL，这个第二进程也会退出。protocol URL 在这里的唯一差异是：普通重复实例会打印日志，带 URL 的重复实例不打印这条普通日志。

const isLockObtained = app.requestSingleInstanceLock();
if (!isLockObtained) {
  const hasProtocolUrl = process.argv.some((arg) => arg.startsWith(PROTOCOL_PREFIX));
  if (!hasProtocolUrl) {
    console.log('Another instance is already running. Exiting.');
  }
  process.exit(0);
} else {
  startApp();
}

protocol URL 的处理依赖已运行主实例在 initializeProtocolHandling() 注册的 second-instance（Windows/Linux）、open-url（macOS）和命令行 cold-start 扫描。定位启动问题时，先问“进程是否拿到单实例锁”；若是 protocol URL，再去 electron/protocol-handler.ts 看转发链路，而不是期待第二进程进入 startApp()。

3. 窗口不是 renderer 本身

electron/main-window.ts 创建 BrowserWindow，并用 webSecurity: true、preload、nodeIntegration: false、contextIsolation: true 建出第一层安全边界。Angular 页面没有直接 Node 权限；它只能通过 electron/preload.ts 暴露的 window.ea 调主进程。

webPreferences: {
  webSecurity: true,
  preload: path.join(__dirname, 'preload.js'),
  nodeIntegration: false,
  contextIsolation: true,
}

contextBridge.exposeInMainWorld('ea', ea);

第二层边界在 session、权限和导航钩子：onBeforeSendHeaders 剥离 Origin、Referer、Cookie 等请求头；onHeadersReceived 注入 CORS 响应头并把 OPTIONS 强制成 200；权限只允许 notifications。新窗口由 setWindowOpenHandler 拒绝，再交给带 scheme guard 的外链打开路径；file: 链接还会进入本地文件 guard。electron/start-app.ts 全局放行自签证书是兼容自托管服务的取舍，不是安全加强。

这也是阅读 IPC 的诀窍：看到 renderer 侧 window.ea.openExternalUrl()、fileSyncSave() 或 consumePluginNodeExecutionApi()，先回到 preload 的 API 名，再按 IPC channel 找主进程 handler。

4. 检索练习

暂停 60 秒回答：如果一个功能需要读取本地同步文件，应该放在 Angular service、preload bridge，还是 Electron 主进程模块？renderer 能不能直接拿绝对路径自己读写？

参考答案：文件读写属于系统能力，应落在 Electron 主进程模块；renderer 只通过 window.ea 发起受控 IPC。preload bridge 负责暴露 typed 方法，不承载业务状态，也不应该把不受限文件系统能力直接交给页面。完整入口清单见 Electron 宿主参考。
