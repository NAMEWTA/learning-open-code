PWA 缓存与更新全链路地图

这节课只建立一张地图：Web/PWA 靠 ngsw-worker.js 缓存壳层与少量 API，更新检测在首次 sync 前由 InitialPwaUpdateCheckService 触发；Capacitor/Electron 则在 src/main.ts 主动注销 SW。完整入口清单见 参考文档。父级 L1：Capacitor 移动端与 PWA 宿主。

1. 全链路时序（Web/PWA）

sequenceDiagram
  participant Build as prod/stage build
  participant Main as src/main.ts
  participant NGSW as ngsw-worker.js
  participant Config as ngsw-config.json
  participant App as Angular app
  participant Pwa as InitialPwaUpdateCheckService
  participant SW as SwUpdate
  participant Sync as SyncEffects
  Build->>Config: 生成 asset/data groups
  Main->>NGSW: registerWhenStable 或手动 register
  NGSW->>Config: prefetch app shell + lazy assets
  App->>App: DataInit 完成
  Sync->>Pwa: afterInitialUpdateCheck$
  Pwa->>SW: checkForUpdate (8s timeout)
  alt 有新版本且用户确认
    Pwa->>App: window.location.reload()
  else 无更新或跳过
    Pwa-->>Sync: of(undefined)
    Sync->>Sync: 触发 initial sync
  end

关键顺序：数据先落地，再查 SW 更新，最后才 initial sync。不要把桌面 UpdateCheckService（GitHub releases）误当成 PWA 更新入口。

2. 缓存策略：壳层 vs API freshness

ngsw-config.json 分两类：assetGroups 缓存 HTML/JS/CSS/i18n/图标；dataGroups 里唯一的 api-freshness 只覆盖 GitHub/GitLab/Jira REST，策略是 freshness（网络优先，1h 内可回退缓存，10s 超时）。它不缓存 SuperSync、WebDAV 或本地 op-log。

assetGroups
  app / criticalAssets → installMode: prefetch（壳层 + en.json）
  assets / externalAssets → lazy + updateMode: prefetch

dataGroups
  api-freshness → GitHub/GitLab/Atlassian API only

3. 平台分叉：启用 SW 还是注销

src/main.ts 双重 gate：Angular ServiceWorkerModule 与 bootstrap 末尾的手动 register 都要求 production || stage、非 Electron、非 native。否则遍历 getRegistrations() 并 unregister()——Capacitor WebView 虽暴露 navigator.serviceWorker，但 native 通知必须走 NotifyService 的 Capacitor 分支，不能误用 SW 通知路径。

// main.ts — native/Electron 注销
} else if ('serviceWorker' in navigator && (IS_ELECTRON || IS_NATIVE_PLATFORM)) {
  navigator.serviceWorker.getRegistrations().then((registrations) => {
    for (const registration of registrations) {
      registration.unregister();
    }
  });
}

4. 60 秒判断

用户报告：「Android Capacitor 版安装后，离线打不开壳层，且从未弹出 Web 应用更新对话框。」你先查 ngsw-config.json 的 api-freshness、src/main.ts 的 SW 注销分支，还是 UpdateCheckService？再说为什么。

参考答案：先查 src/main.ts 注销分支。Capacitor native 满足 IS_NATIVE_PLATFORM，启动时会 unregister() 全部 SW，因此不会走 PWA 离线壳层，也不会触发 InitialPwaUpdateCheckService。Android 更新应走 Play Store；UpdateCheckService 仅 Electron 有效。主要来源：src/main.ts。不确定某平台是否注册 SW 时，直接向 agent 追问 IS_NATIVE_PLATFORM 判定链。
