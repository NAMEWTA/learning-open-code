启动阶段速查

阶段入口输出失败时表现
LAUNCHER 路由 Manifest → FullscreenActivity.onCreate 切 Capacitor 或继续 legacy 闪屏后消失 → 查 Activity 切换是否 finish() 过快
WebView 预检 WebViewCompatibilityChecker.evaluate OK / WARN / BLOCK 阻断屏；瞬态失败走 WebViewRecovery 自动重开
Capacitor 装配 CapacitorMainActivity.onCreate 插件注册 + SUPAndroid 注入 + overlay bridge.webView == null → 恢复或阻断
Angular 冷启动 main.ts bootstrap android-interface 队列 drain + overlay 交接 分享/Widget 丢事件 → 冷启动 pull 路径未执行
温启动补偿 android.effects.ts onResume$ 提醒/Widget 队列再 drain 仅影响从后台恢复，不替代冷启动 pull

词汇表

术语含义
MODE_OFFLINELaunchDecider 值 2；使用 CapacitorMainActivity 加载离线 bundle。
MODE_ONLINELaunchDecider 值 1；使用 FullscreenActivity 加载在线 SPA。
SUPAndroid注入 WebView 的 JavaScriptInterface 全局名；Angular 通过它调用原生能力。
SupKeyValStorelegacy 在线版 SQLite 存储文件；存在即视为有 legacy 数据。
Connectivity-Free Mode构建配置 LAUNCH_MODE=2 强制离线；见 android/README_OFFLINE.md。
WebViewRecovery瞬态 WebView 初始化失败时经 FullscreenActivity 自动重开一次的恢复机制（#7518）。
