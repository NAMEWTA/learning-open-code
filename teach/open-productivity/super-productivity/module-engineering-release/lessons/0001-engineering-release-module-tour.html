测试、构建、发布与多平台工程化导览

读这个模块时，先沿着 L0 项目地图 的“测试/构建/发布”入口回到 package.json。相邻的 Electron L1、Mobile/PWA L1 与 Plugin L1 分别讲运行时平台边界和插件运行时，本课只看它们在工程化链路中的构建接点。

1. 先读 scripts，而不是先读 workflow

package.json 是工程化入口。build 不是“所有平台发布”，而是桌面默认构建：先 lint，再构建 production Angular renderer，最后运行 Electron 编译。真正产出发布包的链路会落到 dist、dist:android:prod 或 CI 里的 iOS archive/export；dist:ios:prod 本身只做 Web build 与 Capacitor iOS sync。

npm run build
  → buildAllElectron:noTests:prod
    → lint
    → buildFrontend:prod:es6
      → prebuild → env + git-version + build:packages
      → ng build --configuration production
    → electron:build
      → build-wayland-idle-helper + tsc + bundle-preload

2. 把命令分成四类

类别典型命令真实边界
开发serve、start、startFrontend生成 env 后启动 Angular 或 Electron dev 宿主。
测试lint、test、e2e、test:electron覆盖 ESLint/Stylelint、本地 lint rules、Karma、package tests、Playwright。
构建buildFrontend:prodWeb、electron:build、plugins:build分别产出 Web/PWA、Electron main/preload、内置插件 assets。
打包/发布dist、dist:win、dist:android:prod调用 electron-builder、Capacitor sync、Gradle 或平台商店脚本。

3. 从前端产物追到平台产物

angular.json 决定前端输出：Electron 用 .tmp/angular-dist，Web/Capacitor 用 dist/browser；production 与 productionWeb 启用 ngsw-config.json。Electron 的平台包再由 electron-builder.yaml 收进 .tmp/app-builds；Android/iOS 先通过 capacitor.config.ts 的 webDir 指向 dist/browser 同步到原生工程。插件包不是自动散落在源码里，而是由 packages/build-packages.js 构建并复制到 src/assets/bundled-plugins。

4. 检索练习

暂停 60 秒，凭记忆判断：如果你改了一个内置插件的 Vite 构建，并且想确认 PR 与发布都不会漏掉插件资产，应优先看哪三个入口？

参考答案：先看 package.json 的 plugins:build、buildFrontend:e2e 和 build:packages；再看 packages/build-packages.js 如何复制 bundled plugins；最后看 .github/workflows/ci.yml 与 .github/workflows/plugin-tests.yml 是否会构建共享包并验证 E2E 输出目录里的 api-test-plugin manifest。完整入口表见 工程化发布参考。

主要来源：package.json。如果 preCheck、Capacitor sync、electron-builder 或 workflow 分流不清楚，可以直接向 agent 追问下一跳源码。
