应用启动与数据初始化全链路参考

一句话：本切片从 src/main.ts 的 Angular bootstrap 出发，追到 root shell、DataInit/op-log hydration、初始同步信号、迁移提示和插件初始化。关联模块：Angular 壳层 L1、op-log 同步 L1、插件系统 L1。

入口清单

阶段源码入口负责什么排查信号
Angular bootstrapsrc/main.ts调用 bootstrapApplication(AppComponent, ...)，注册根 providers、router、NgRx、HTTP、Translate、service worker、Capacitor 生命周期。provider 缺失、APP_INITIALIZER、service worker、移动后台 flush。
Feature store 注册src/app/root-store/feature-stores.module.ts注册 app state、layout、config、task、project、tag、plugin metadata/user data、sync effects 等。某个 effect 未触发、sync effects 未进入、平台条件 effects 缺失。
路由入口src/app/app.routes.ts定义工作区页面、独立页面、插件页和 wildcard 默认页。URL 能否匹配页面、lazy component 是否加载、守卫是否挡住。
根组件行为src/app/app.component.ts构造函数触发启动服务；等待数据/同步后显示笔记、安全和加密迁移 banner；初始化主题、RTL、键盘布局。shell 已出现但 banner/onboarding/全局体验异常。
根组件模板src/app/app.component.html把 side nav、header、right panel、router-outlet、progress bar、focus overlay、onboarding 放进 shell。壳层在但页面空白时，先查路由/页面；壳层也不在则回 bootstrap。
启动服务src/app/core/startup/startup.service.ts单实例检查、备份恢复提示、持久化、主题预加载、Electron ready、延迟任务和插件初始化。多标签阻塞、首屏闪主题、插件等待、Electron settings 发送。
数据初始化src/app/core/data-init/data-init.service.ts实例化即触发 reInit()，可选 user profile 初始化，再调用 op-log hydrator。首次数据是否加载、是否 dispatch allDataWasLoaded。
首次数据状态src/app/core/data-init/data-init-state.service.ts用 ReplaySubject + shareReplay(1) 暴露只完成一次的初始数据信号。其他服务是否还在等 isAllDataLoadedInitially$。
op-log hydrationsrc/app/op-log/persistence/operation-log-hydrator.service.ts恢复 pending remote ops，迁移 archive，读 state cache，必要时旧库迁移/schema migration/replay，最后 dispatch loadAllData。IndexedDB、快照损坏、旧库迁移、replay 卡住。
旧库迁移src/app/op-log/persistence/operation-log-migration.service.ts无 state cache 时检查 legacy pf 数据；创建自动备份、校验/修复、写 MIGRATION genesis op 和 state cache。迁移弹窗、legacy 数据不可读、迁移锁、多标签。
schema migrationsrc/app/op-log/persistence/schema-migration.service.ts迁移旧 state cache 和旧 operation payload 到当前 schema。schemaVersion 不匹配、operation 被 drop/split、迁移 registry 异常。
同步触发src/app/imex/sync/sync-trigger.service.ts在数据完成后判断 initial sync 是否启用；生成初始同步完成信号和后续自动同步触发器。UI 是否应等待同步、8 秒兜底、SuperSync 是否跳过等待 UI。
同步执行src/app/imex/sync/sync-wrapper.service.ts防并发、打开 sync window、下载远端 ops、上传本地 ops、设置 sync status、处理错误。同步卡在进行中、插件等待当前同步、加密/网络/认证错误。
远端快照落地src/app/op-log/persistence/sync-hydration.service.ts远端同步下载后合并 archive 和实体模型，保存 state cache/vector clock，并 dispatch loadAllData。远端数据下载后 NgRx 未更新、local-only sync 设置被覆盖、快照校验失败。
插件运行时src/app/plugins/plugin.service.ts发现内置/上传插件，读取 enabled 状态，自动启用 issue provider 迁移插件，加载 enabled 插件并触发 ready。插件未发现、未启用、加载失败、Node grant 被拒。

启动时序

浏览器 / Electron / Native 打开 renderer
  ↓
src/main.ts
  ├─ unlock audio / legacy KDF warning / platform flags
  ├─ bootstrapApplication(AppComponent, providers)
  ├─ APP_INITIALIZER 实例化 DataInitService
  │    └─ src/app/core/data-init/data-init.service.ts
  │        └─ src/app/op-log/persistence/operation-log-hydrator.service.ts
  │            ├─ recover pending remote ops / cleanup corrupt ops
  │            ├─ migrate archives from legacy pf if needed
  │            ├─ load state cache or run legacy migration
  │            ├─ schema migration or operation replay
  │            └─ dispatch loadAllData → allDataWasLoaded → DataInitStateService
  └─ AppComponent constructed
       ├─ src/app/core/startup/startup.service.ts init()
       │    ├─ single-instance / backup / persistence / theme
       │    ├─ Electron ready or browser beforeunload
       │    └─ delayed tasks after 1000ms
       │         ├─ tracking reminder / update check / storage / offline banner
       │         ├─ legacy custom theme migration
       │         ├─ rate prompt
       │         └─ wait data loaded + current sync done
       │              └─ src/app/plugins/plugin.service.ts initializePlugins()
       └─ src/app/app.component.html router-outlet 显示路由页面

src/app/imex/sync/sync.effects.ts
  └─ after DataInitStateService → provider ready → initial sync / auto sync
       └─ SyncWrapperService.sync() download first, upload second
            └─ SyncTriggerService.setInitialSyncDone(true)

常见故障排查

故障第一入口第二入口判断方法
应用提示已经在另一个 tab 运行。src/app/core/startup/startup.service.ts浏览器 BroadcastChannel检查 superProductivityTab 通道是否收到 alreadyOpenElsewhere。
首次数据没有出现。src/app/core/data-init/data-init.service.tssrc/app/op-log/persistence/operation-log-hydrator.service.ts看是否进入 hydration、是否 dispatch loadAllData，以及是否记录 IndexedDB 或 recovery 错误。
IndexedDB backing store 错误。src/app/op-log/persistence/operation-log-hydrator.service.tssrc/app/op-log/core/errors/indexed-db-open.error.tshydrator 会先尝试自动 reload 一次，再显示手动恢复说明。
旧版本数据迁移失败。src/app/op-log/persistence/operation-log-migration.service.tssrc/app/core/persistence/legacy-pf-db.service.ts确认 legacy 数据是否可读、自动备份是否创建、repair 后是否仍验证失败。
同步一直阻塞退出或插件等待。src/app/imex/sync/sync-wrapper.service.tssrc/app/imex/sync/user-input-wait-state.service.ts检查 afterCurrentSyncDoneOrSyncDisabled$ 是否在等用户输入、是否超时兜底。
初始同步后 note/safety/encryption banner 没显示。src/app/app.component.tssrc/app/imex/sync/sync-trigger.service.ts确认 afterInitialSyncDoneAndDataLoadedInitially$ 是否发出，或者是否触发 8 秒兜底。
插件页空白但 shell 正常。src/app/core/startup/startup.service.tssrc/app/plugins/plugin.service.ts先看插件初始化是否等待数据/当前同步；再看 discover、enabled、activate、ready 失败状态。
Electron 首次启动后设置没有发给主进程。src/app/core/startup/startup.service.tssrc/app/core/data-init/data-init-state.service.tsElectron 设置发送要等首次数据完成后读取 global config。
Android/iOS 退后台后最近操作丢失。src/main.tssrc/app/op-log/sync/operation-write-flush.service.ts检查 Capacitor appStateChange 是否触发后台 task，并调用 pending writes flush。

继续深入

  1. 如果要展开启动数据落地，读 src/app/op-log/persistence/operation-log-hydrator.service.ts 的 snapshot、full-state op、tail replay、recovery 分支。
  2. 如果要展开初始同步，读 src/app/imex/sync/sync-trigger.service.ts、src/app/imex/sync/sync.effects.ts、src/app/imex/sync/sync-wrapper.service.ts 的等待关系。
  3. 如果要展开插件启动，读 src/app/plugins/plugin.service.ts 的 discover、auto-enable migration plugin、activate、ready failure 处理。
  4. 如果要展开迁移，读 src/app/op-log/persistence/operation-log-migration.service.ts、src/app/op-log/persistence/schema-migration.service.ts、src/app/imex/sync/super-sync-encryption-migration-banner.service.ts 三类迁移边界。
