应用启动与数据初始化路径图

这节课只建立一张启动地图：Angular 先把 provider 和 root shell 装起来，数据加载在 src/app/core/data-init/data-init.service.ts 里异步落到 NgRx，随后同步信号和插件初始化才接力。详细入口清单见 参考文档。

1. 从 bootstrap 看第一批接线

src/main.ts 调用 bootstrapApplication(AppComponent, ...)，集中注册 src/app/root-store/feature-stores.module.ts、NgRx 根 store、src/app/app.routes.ts、Translate、HTTP、service worker 和平台生命周期。最容易误判的是：src/app/plugins/plugin-initializer.ts 的 provider 只是注册 'PLUGIN_INITIALIZER' token；当前实际插件运行时不靠它执行。

src/main.ts
  ├─ bootstrapApplication(AppComponent)
  ├─ importProvidersFrom(FeatureStoresModule, StoreModule.forRoot(...))
  ├─ provideRouter(APP_ROUTES, withHashLocation(), withPreloading(...))
  ├─ APP_INITIALIZER: OperationCaptureService
  ├─ APP_INITIALIZER: instantiate DataInitService
  └─ APP_INITIALIZER: encryption / OAuth handler side effects

2. 数据加载不是等 bootstrap 完成才开始

src/main.ts 的 DataInit initializer 只返回空函数，但它会强制实例化 src/app/core/data-init/data-init.service.ts。该 service 的字段初始化立即执行 reInit()：可选初始化 user profile，再进入 src/app/op-log/persistence/operation-log-hydrator.service.ts。hydrator 优先读 state cache；没有快照时检查旧 pf 数据迁移；必要时迁移 schema 或 replay operations；最后 dispatch loadAllData，再由 src/app/core/data-init/data-init-state.service.ts 广播首次数据完成。

3. 根组件触发启动后任务

src/app/app.component.ts 构造函数马上调用 src/app/core/startup/startup.service.ts 的初始化。这里做单实例检查、本地备份、持久化请求、主题预加载和 Electron ready；1 秒后进入延迟任务：tracking reminder、更新检查、离线 banner、legacy custom theme migration、评分提示，最后才初始化插件。

AppComponent constructor
  → src/app/core/startup/startup.service.ts
  → single-instance / backup / persistence / theme
  → delayed startup tasks
  → wait data loaded + current sync done
  → src/app/plugins/plugin.service.ts

4. 同步与插件的等待边界

src/app/imex/sync/sync.effects.ts 在数据完成后订阅自动同步；src/app/imex/sync/sync-trigger.service.ts 管理“初始同步完成 + 数据加载完成”的 UI 信号，最多等待 8 秒兜底。插件更保守：src/app/core/startup/startup.service.ts 的插件入口等待 DataInitStateService.isAllDataLoadedInitially$ 和 SyncWrapperService.afterCurrentSyncDoneOrSyncDisabled$，再调用 src/app/plugins/plugin.service.ts 发现内置/上传插件、自动启用迁移插件并加载 enabled 插件。

症状第一判断入口
首屏一直像未加载数据数据是否发出首次完成信号src/app/core/data-init/data-init.service.ts

暂停 60 秒判断

只凭这张路径图判断：用户报告“冷启动后侧栏和 header 已出现，但插件页为空；同时控制台没有路由错误”。你先看 src/app/app.routes.ts、src/app/core/data-init/data-init.service.ts、src/app/core/startup/startup.service.ts，还是 src/app/plugins/plugin.service.ts？再说出第二跳。

参考答案：先看 src/app/core/startup/startup.service.ts，因为 shell 已出现且没有路由错误，问题更像启动后插件等待链。第二跳看 src/app/plugins/plugin.service.ts：确认 initializePlugins() 是否发现 manifest、是否加载 enabled 插件、是否把状态置为 error。主要来源：src/app/core/startup/startup.service.ts。不确定某个等待信号时，直接向 agent 追问下一跳源码。
