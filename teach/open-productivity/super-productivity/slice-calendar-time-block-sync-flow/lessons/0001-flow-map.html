日历事件导入与插件 time block 写回全链路

学完这一课，你要能回答：某段代码是在读远端日历、把事件投影到 Planner/Schedule、导入成本地任务，还是把本地 scheduled task 写回插件日历？ 完整索引见 参考文档。 父模块：Issue Provider 与日历集成。 相邻切片：Planner→Schedule 投影。

1. 时序总览

首查读方向入口是 src/app/features/calendar-integration/calendar-integration.service.ts 的 calendarEvents$；写回本地任务到远端日历是 src/app/features/calendar-integration/time-block/time-block-sync.effects.ts。

2. 三类对象别混

对象存在哪里典型入口常见误判
calendar event（读） 远端 + LS 缓存；视图为 ScheduleFromCalendarEvent CalendarIntegrationService 投影到 Schedule 不等于已导入任务；iCal 事件默认不可拖拽写回。
calendar task（导入） 本地 Task，issueType=ICAL，id 为 cal_{providerId}_{eventId} CalendarIntegrationEffects 或 banner / 右键 与 time block 无关；走 issue 导入，不是 timeBlock.upsertEvent。
time block（写） 远端插件日历事件，由 task.id 关联 TimeBlockSyncEffects 需要 isAutoTimeBlock 且插件定义 timeBlock；监听的是 task action，不是 calendar 投影。

3. 三个判断点

判断一（事件为何没出现在 Schedule）：先看 calendarEvents$ 是否拉到数据，再看视图过滤——已链接任务 id、用户 skip、hidden、regex 过滤都会隐藏事件。缓存写在 LS.CAL_EVENTS_CACHE，拉取失败会回退缓存。

判断二（今日事件为何没自动变任务）：首查 src/app/features/calendar-integration/store/calendar-integration.effects.ts。必须同时满足：provider 开启 isAutoImportForCurrentDay、事件是今天、未重复、初始同步完成且不在 sync window。手动添加走同一 IssueService.addTaskFromIssue，但无 sync window 门控。

判断三（排程后为何没写回日历）：首查 TimeBlockSyncEffects。只有启用插件且 pluginConfig.isAutoTimeBlock 为真才会 upsert；单次编辑多个 action 会在 1s 内合并；取消排程、跨日 transfer、删除任务会走 delete。Schedule 拖插件事件则走 CalendarEventActionsService.moveToStartTime，那是改远端已有事件，不是 time block 链路。

4. 60 秒判断题

场景：用户把本地任务 A 拖到明天 10:00（此前无 dueWithTime），同时 Google Calendar 插件已开 auto time block。Schedule 上另有一个可拖拽的插件日历会议块被改到 11:00。哪条链路会写远端？

  1. 两条都走 TimeBlockSyncEffects。
  2. 任务 A 走 time block upsert；会议块走 CalendarEventActionsService pushChanges。
  3. 两条都走 IssueTwoWaySyncEffects 双向同步。
参考答案与反馈

选 2。本地排程触发 scheduleTaskWithTime，由 TimeBlockSyncEffects 合并后调用 definition.timeBlock.upsertEvent。拖拽 Schedule 上的插件日历事件是移动已存在的远端 agenda 事件，经 sync adapter pushChanges 后 triggerRefresh 重拉；iCal 只读事件则不能拖。它们都不经过 issue 双向同步的 baseline push。

主要来源：日历 time block 全链路速查。若某步不清楚，可向 agent 追问：“请从 calendarEvents$ 追到 Schedule blocked block”或“请列出 time block 队列合并规则”。
