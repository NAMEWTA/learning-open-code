L1 模块总览 · 预计 15 分钟 · 目标：能判断任务为什么属于 Today、Project 或 Tag

任务、项目、标签与工作上下文领域

一句话：这个模块负责把任务实体、项目列表、标签列表和当前工作上下文统一成用户看到的工作视图，并通过 NgRx 与 meta-reducers 维护跨实体一致性。

1. 在整体架构中的位置

L0 已说明 super-productivity 的核心产品域是任务、项目、标签、时间追踪和同步；本课只展开最核心的任务归属边界。你可以先回看 L0 项目地图，那里把 src/app/features/、src/app/root-store/ 和 op-log 放在同一条主线上。

读本模块时要带着两个 ADR 护栏：dueWithTime 与 dueDay 在新数据中互斥，读取时前者优先；TODAY_TAG 是虚拟标签，成员由日期字段推导，绝不能写进 task.tagIds。

2. 对外入口与内部层次

UI 组件：task-list / task-detail / add-task-bar / tasks-by-tag
  ↓
Service 门面：TaskService · ProjectService · TagService · WorkContextService
  ↓
NgRx：actions · reducer · effects · selectors · Entity adapters
  ↓
root-store/meta：TaskSharedActions · task-shared meta-reducers
  ↓
op-log capture：只捕获带 PersistentActionMeta 的本地持久化动作

最常用入口是 TaskService：创建、更新、移动、归档、调度和计时都会落成 actions。项目与标签各有 ProjectService、TagService；统一视图由 WorkContextService 暴露 active context、startable tasks 和 done ids。完整入口表放在 参考文档。

关键依赖有四类：NgRx Entity 管实体表；root-store meta 维护任务、项目、标签的联动；work-context selectors 组装当前视图；repeat cfg、reminder、time tracking 通过 actions/effects 依附任务，但不拥有任务归属规则。

3. 阅读示例：任务属于哪里

先看 src/app/features/work-context/store/work-context.selectors.ts。当 active context 是 Tag，selector 会分叉：如果是 TODAY_TAG.id，调用 Today 专用计算；否则交给普通 tag 计算。

activeType === TAG
  ├─ activeId === TODAY_TAG.id → computeOrderedTaskIdsForToday(...)
  └─ 其他 tag                 → computeOrderedTaskIdsForTag(...)

activeType === PROJECT
  └─ 使用 project.taskIds / project.backlogTaskIds

Today 的成员判断不看 task.tagIds：先检查 dueWithTime 是否落在今天；只有没有时间戳时才检查 dueDay === todayStr。普通标签在 src/app/features/tag/store/tag.reducer.ts 中使用 task.tagIds 判断成员，再用 tag.taskIds 保留用户排序。项目视图则来自 src/app/features/project/project.model.ts 的 taskIds 和 backlogTaskIds。

4. 检索练习

暂停 60 秒，凭记忆判断：一个任务同时有 dueWithTime 指向明天、旧数据里的 dueDay 等于今天，并且 tagIds 里没有 TODAY。它应该出现在 Today 吗？应该从哪个文件开始验证？

参考答案：不应该。读取规则是 dueWithTime 优先，且一旦存在就不再回退检查 dueDay。先看 src/app/features/work-context/store/work-context.selectors.ts 的 Today 计算，再对照 ARCHITECTURE-DECISIONS.md 中的两个 ADR。

主要来源：ARCHITECTURE-DECISIONS.md 与 src/app/features/work-context/store/work-context.selectors.ts。如果你不确定某个 action 是 feature reducer 处理，还是 meta-reducer 处理，直接向 agent 追问。
