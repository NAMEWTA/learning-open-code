任务领域模块总览

模块职责

任务领域把 Task、Project、Tag 与当前 WorkContext 组合成可排序、可计时、可同步的工作视图；它的核心边界是“任务实体字段”和“视图归属列表”之间的一致性。

全局位置：本模块位于 L0 架构图中的 src/app/features/ 业务层和 src/app/root-store/ 状态层之间，向下被 op-log capture 持久化。复习入口见 L0 项目总览参考。

源码地图

src/app/ ├── features/ │ ├── tasks/ ← Task 模型、UI、Service、store、effects、附件、提醒、子任务 │ ├── project/ ← Project 模型、Service、store、完成/归档语义 │ ├── tag/ ← Tag 模型、TODAY_TAG、普通标签成员与排序 │ ├── work-context/ ← Project/Tag 统一视图、active context、selectors │ ├── menu-tree/ ← 项目/标签导航树 │ └── task-repeat-cfg/ ← 重复任务配置与实例生成依赖 ├── root-store/ │ └── meta/ ← task shared actions、meta-reducers、op-log 前置规则 └── op-log/ └── capture/ ← PersistentActionMeta 的操作捕获

对外入口与接口清单

入口源码用途注意点
TaskServicesrc/app/features/tasks/task.service.ts创建、更新、删除、移动、归档、调度、计时、查询任务。多数写操作 dispatch TaskSharedActions，跨实体变化常在 meta-reducers 完成。
task.actionssrc/app/features/tasks/store/task.actions.ts当前任务、选中任务、子任务移动、UI-only 更新、时间修正。只有带 PersistentActionMeta 的 action 会被 op-log 捕获。
task.reducersrc/app/features/tasks/store/task.reducer.tsTask Entity state、当前/选中任务、子任务顺序、附件、loadAllData 归一化。不要把跨 Project/Tag 的完整不变量都塞进 feature reducer。
task.selectorssrc/app/features/tasks/store/task.selectors.ts任务实体、当前任务、过期/计划/提醒/重复任务等派生视图。Today 精确成员优先看 work-context selector。
ProjectServicesrc/app/features/project/project.service.ts项目列表、归档/取消归档、完成/重开、任务移入 backlog 或 Inbox。项目完成采用 per-task actions + 项目 flag flip，不是多实体原子 op。
project.actionssrc/app/features/project/store/project.actions.ts新增/更新/归档/完成项目，移动项目任务与 backlog。移动任务 action 的 entityType 多为 TASK，因为影响的是任务排序。
project.reducer/selectorssrc/app/features/project/store/project.reducer.ts、src/app/features/project/store/project.selectors.tsProject Entity state、Inbox 默认项目、项目排序、active/archived 查询。Project.taskIds 与 backlogTaskIds 是项目视图的列表来源。
TagServicesrc/app/features/tag/tag.service.ts标签列表、创建、删除、颜色、顺序和按 id 查询。删除标签标题通过 sidecar 传给 issue sync，避免写入导出的 op-log payload。
tag.actions/reducersrc/app/features/tag/store/tag.actions.ts、src/app/features/tag/store/tag.reducer.ts标签 Entity state、普通标签成员排序、Today 排序。普通标签成员来自 task.tagIds，tag.taskIds 只保序。
TODAY_TAGsrc/app/features/tag/tag.const.ts虚拟 Today 工作上下文。TODAY 不应进入 task.tagIds。
WorkContextServicesrc/app/features/work-context/work-context.service.tsactive context、主列表、backlog、startable/trackable tasks、上下文标题。统一 Project 和 Tag，使 UI 不必分别处理两套视图。
work-context.selectorssrc/app/features/work-context/store/work-context.selectors.tsToday / Project / Tag 工作上下文组装。追踪任务归属时优先读这里。
menu-treesrc/app/features/menu-tree/store/menu-tree.selectors.ts项目树和标签树导航结构。导航层排序与展示，不决定任务成员。
task-repeat-cfgsrc/app/features/task-repeat-cfg/task-repeat-cfg.model.ts重复任务配置、周期、模板子任务、跳过/删除实例。重复任务创建出来仍回到 Task / Project / Tag 规则。
TaskSharedActionssrc/app/root-store/meta/task-shared.actions.ts跨 reducer 的任务共享动作：add/update/delete、schedule、plan today、move project/tag。这是任务域最重要的持久化 action 集合。
META_REDUCERSsrc/app/root-store/meta/meta-reducer-registry.tsop-log capture、bulk replay、undo、CRUD、调度、实体级 cascade、planner 同步、short syntax、LWW。注册顺序是领域不变量的一部分。

内部层次与数据流

用户操作
  ↓
任务 UI：add-task-bar / task / task-list / task-detail-panel
  ↓
Service：TaskService / ProjectService / TagService / WorkContextService
  ↓
Actions：TaskSharedActions + feature actions（带 PersistentActionMeta）
  ↓
Reducers：tasks / projects / tag / workContext / menuTree / taskRepeatCfg
  ↓
Meta-reducers：CRUD、调度、生命周期、project/tag cascade、planner sync
  ↓
Selectors：activeWorkContext、selectTodayTaskIds、project/tag lists
  ↓
op-log capture：本地持久化动作转为可同步 operation

归属判断速查

问题判断来源排序来源核心文件
任务属于 Today 吗？先看 task.dueWithTime 是否是今天；没有时间戳才看 task.dueDay === todayStr。TODAY_TAG.taskIds，只保序，陈旧项会过滤。src/app/features/work-context/store/work-context.selectors.ts

任务属于 Project 吗？task.projectId 与项目列表维护共同决定；视图读取 project.taskIds。project.taskIds 或 project.backlogTaskIds。src/app/features/project/project.model.ts、src/app/features/work-context/store/work-context.selectors.ts
子任务是否单独显示？如果父任务也属于同一 Today/Tag 结果，子任务嵌套在父任务下，不作为顶层项重复出现。父任务的 subTaskIds 与上下文列表共同决定。src/app/features/work-context/store/work-context.selectors.ts、src/app/features/tag/store/tag.reducer.ts

不变量写入侧读取侧风险

op-log 只捕获持久化动作src/app/op-log/capture/operation-capture.meta-reducer.tssrc/app/root-store/meta/meta-reducer-registry.tsUI-only action 不应被同步；远端 replay 期间本地动作会被缓冲。

阅读 checklist

  1. 先读 ARCHITECTURE-DECISIONS.md，确认是否触碰 due 字段、Today、项目完成或同步边界。
  2. 定位用户操作入口：UI 组件通常调 TaskService、ProjectService、TagService 或 WorkContextService。
  3. 看 dispatch 的 action 是否来自 TaskSharedActions，并检查是否带 PersistentActionMeta。
  4. 同时查 feature reducer 和 src/app/root-store/meta/，不要只看 action 文件。
  5. 最后回到 selectors，确认用户实际看到的是 active work context，而不是某个实体字段的原始值。
