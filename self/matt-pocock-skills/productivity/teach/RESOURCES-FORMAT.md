# RESOURCES.md 格式

`RESOURCES.md` 是针对该主题精心筛选的可信资源集合。解释性内容的知识应来源于此，而非参数化的猜测。智慧来自于此处列出的社区。

## 结构

```md
# {主题} 资源

## 知识

- [书籍：《力量训练的科学与实践》—— Zatsiorsky & Kraemer](https://example.com)
  关于训练计划和适应的基础性著作。适用于：任何与周期化、恢复、强度区间相关的内容。
- [文章："我应该练多少？"—— Greg Nuckols（Stronger By Science）](https://example.com)
  关于训练量基准的循证综述。适用于：每个肌群的每周组数目标。

## 智慧（社区）

- [r/weightroom](https://reddit.com/r/weightroom)
  高信噪比的 subreddit，经过严格管理，抵制伪科学。适用于：训练计划 critique、平台期排查。
- 本地：{健身房名称} 的周二力量课
  适用于：举重动作的实时教练反馈。
```

## 规则

- **仅限高信任度资源。** 优先选择一手资料、公认专家、同行评审成果以及有强管理的社区。如果某个资源是用教育包装的营销，就不要收录。
- **为每个条目添加注释。** 一个裸链接三个月后就毫无用处了。添加一行说明：它涵盖什么以及何时使用它。
- **按知识/智慧分组。** 遵循 [SKILL.md](./SKILL.md) 中的哲学。一个资源只出现在一个分组中是可以的。
- **明确标记缺口。** 如果学习目标需要的某个领域没有好的资源，写一个 `## 缺口` 栏目列出缺失的内容。这会驱动未来的搜索。
- **无情地修剪。** 一个被证明是错误、肤浅或偏离目标的资源应该被移除，而不是被淹没。五个精良的资源胜过三十个平庸的。
- **记录社区偏好。** 如果用户选择不加入社区，在此注明，这样未来的课程就不会反复提议。
