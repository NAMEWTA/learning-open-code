# 学习记录格式

学习记录存放在 `./learning-records/` 目录中，使用顺序编号：`0001-slug.md`、`0002-slug.md` 等。按需创建目录——仅在写入第一条记录时才创建。

它们是教学领域的 ADR（架构决策记录）等价物：记录非显而易见的经验教训、关键洞察以及会影响未来课程的已声明先验知识。它们用于计算最近发展区。

## 模板

```md
# {所学或所确立内容的简短标题}

{1-3 句话：学到了什么（或确立了哪些先验知识），以及它为什么对未来的课程重要。}
```

这就是完整格式。一条学习记录可以只有一个段落。其价值在于记录_这个知识点现在已被掌握_以及_它如何改变接下来要教什么_——而不是填充各个栏目。

## 可选栏目

仅在这些栏目能真正增加价值时才包含。大多数记录不需要它们。

- **Status** frontmatter（`active | superseded by LR-NNNN`）——当早期的理解被证明是错误的并被替换时很有用。
- **Evidence**——用户如何展示了理解（回答了一个问题、完成了一个练习、引用了过往经验）。当该判断可能被重新审视时很有用。
- **Implications**——这对未来的课程解锁了什么或排除了什么。当不明显时值得记录。

## 编号

扫描 `./learning-records/` 目录，找到现有最大编号并加一。

## 何时写学习记录

当以下任一情况成立时，写一条记录：

1. **用户展示了对某个非平凡内容的真正理解**——不仅仅是接触过，而是有证据表明他们能正确运用这个概念。这为接下来教什么设定了一个新的底线。
2. **用户披露了先验知识**——"我已经知道 X。"记录下来，这样未来的课程就不会重复教它。同时记录所声称的_深度_。
3. **纠正了一个误解**——用户之前相信了错误的东西，现在明白了为什么。这类记录价值很高：它们能预测相关主题未来的绊脚石。
4. **学习目标因学习而改变**——用户发现自己关心的东西和之前想的不一样。交叉链接到 [[MISSION.md]] 并更新它。

### 什么_不_算

- 仅仅覆盖过的材料。覆盖不等于学习。等待证据。
- 任何已在 [[GLOSSARY.md]] 中简洁记录为术语定义的内容。不要重复。
- 逐节的活动日志。学习记录不是日记——它们是决策级别的洞察。

## 替代

当一条后续记录与之前的记录矛盾时（用户的理解加深了或被纠正了），将旧记录标记为 `Status: superseded by LR-NNNN`，而不是删除它。理解如何演变的历史本身就是有用的信号。
