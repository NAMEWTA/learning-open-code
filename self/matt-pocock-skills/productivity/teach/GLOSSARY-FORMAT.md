# GLOSSARY.md 格式

`GLOSSARY.md` 是本教学工作区的规范语言。所有解释、练习和学习记录都应遵循其术语。构建它本身就是学习的一部分：将一个概念压缩为精炼的定义，是用户理解它的证据。

## 结构

```md
# {主题} 术语表

{用一到两句话描述本术语表涵盖的主题。}

## 术语

**肌肉肥大（Hypertrophy）**：
由反复训练中产生的机械张力和代谢压力驱动的肌肉增长。
_避免使用_：增肌、变大

**渐进超负荷（Progressive overload）**：
通过负荷、训练量或强度，随时间系统性地增加对肌肉的需求。
_避免使用_：更用力、升级

**RPE（自感用力评级，Rate of Perceived Exertion）**：
对一组训练感受的 1–10 自评分数，其中 10 表示力竭，8 表示还剩两次。
_避免使用_：努力分数、强度评级
```

## 规则

- **仅在用户理解某个术语时才添加。** 术语表是压缩知识的记录，而不是用户用来学习的词典。如果用户刚接触一个概念，请等到他们能正确使用它之后再将其收录于此。
- **要有主见。** 当同一个概念存在多个词汇时，选择最好的一个，并将其余列为应避免的别名。这就是语言压缩的方式。
- **保持定义精炼。** 一到两句话。定义术语是什么，而不是它做什么或怎么做。
- **在定义中使用术语表自身的术语。** 一旦某个术语已收录在术语表中，就在所有地方优先使用它——包括在其他定义内部。这正是让复杂术语在后期更容易理解的原因。
- **当自然的聚类出现时，按子标题分组**（例如 `## 解剖学`、`## 编程`）。当术语之间具有内聚性时，扁平列表也可以。
- **明确标记歧义。** 如果某个术语在更广泛的领域中用法不严谨，注明解决方式："在本工作区中，'组（set）'始终指正式组——热身组单独记录。"
- **随着理解加深进行修订。** 用户在第一周写的定义可能到第六周就是错的。就地更新；不要留下过时的条目。
