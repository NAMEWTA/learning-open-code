# 术语表——编写出色的技能

关于什么造就了出色技能的领域模型。技能的存在是为了从随机系统中提取确定性；以下每个术语都是实现这一目标的杠杆。这是 [`writing-great-skills`](SKILL.md) 的公开参考文档。

任何定义中的**粗体术语**本身都在本术语表中有定义；通过其标题可以找到它们。

## 语言

### 可预测性（Predictability）

技能使智能体在每次运行时以相同_方式_行为的程度——相同的过程，而非相同的输出（一个头脑风暴技能应该_可预测地_发散；其 token 会变化，但其行为不会）。所有其他术语为之服务的根本美德——成本和可维护性是它的症状，而非对手。

_避免使用_：一致性、可靠性、稳健性、输出确定性

### 模型调用（Model-Invoked）

一种保留其 **description** 字段的技能，使智能体可以看到它并自主触发——而人类仍然可以输入其名称，因此模型调用始终_包含_用户可达性。不存在仅模型状态：description 只会_增加_智能体发现能力，永远不会移除人类的能力。为了这种可发现性，它在每一轮对话中付出永久的**上下文负载**。可被其他技能调用，因为使其对智能体可发现的 description 也使其可被调用。内容全部为**参考**的模型调用技能也是共享参考的一种归宿：另一个技能可以调用它，因此多个技能需要的参考可以集中存放在一处。仅在智能体必须自行调用该技能时才选择模型调用；如果它从来不会自动触发，就删除 description 并支付零上下文负载。

_避免使用_：能力、工具、功能

### 用户调用（User-Invoked）

一种移除了 **description** 的技能——对智能体不可见，只能由人类输入其名称来调用（用户_独占_，而**模型调用**是用户_和_智能体共享）。以智能体可发现性换取零**上下文负载**。因为没有 description，只有人类可以调用它：没有其他技能可以触发它。

_避免使用_：流程、工作流、命令

### Description

技能的机器可读触发器，也是**模型调用**技能被迫始终保持在上下文中的唯一**上下文指针**。它的存在本身就是调用轴：保留它，技能就是模型调用的（且可被其他技能调用）；删除它，技能就是**用户调用**的，只能由人类调用。模型调用技能的**上下文负载**的来源。

_避免使用_：frontmatter、摘要

### 上下文指针（Context Pointer）

在智能体上下文中持有的一个引用，它指向某些上下文外的材料，并编码了获取它的条件。**description** 是顶层的上下文指针（上下文窗口 → 技能）；指向已披露文件的指针是下一层的同一概念。其措辞——而非目标——决定了智能体_何时_获取——以及_可靠程度_。一个措辞薄弱的指针后面跟着一个必备目标是方差 bug：先修正措辞，只有在强化无效时才将材料内联。

_避免使用_：链接、引用、导入

### 上下文负载（Context Load）

**模型调用**技能对智能体上下文窗口施加的成本——它的 **description** 始终被加载，消耗 token 和注意力。**用户调用**技能通过没有 description 来避免这一点，也是拆分成更多模型调用技能的刹车。

_避免使用_：token 成本、上下文膨胀

### 认知负载（Cognitive Load）

**用户调用**技能对人类施加的成本——他们必须在脑中记住：存在哪些技能以及何时使用每个技能（人类就是索引）。**模型调用**通过让智能体可发现来消除这一点，也是拆分成更多用户调用技能的刹车。不是需要最小化的成本：它是人类自主权的代价，是一些技能保持用户调用的原因。在人类判断重要的地方投入它；在不重要的地方移除它。

_避免使用_：人类索引、负担、开销

### 粒度（Granularity）

你将技能切分的精细程度。更细的切分会消耗两种负载之一：更多**模型调用**技能消耗**上下文负载**（更多 description 挤占上下文窗口并争夺注意力）；更多**用户调用**技能消耗**认知负载**（更多内容需要人类记住和使用）。两种切分指导着划分。按**调用**切分：当你有一个独特的**主导词**来触发它时，拆分出一个模型调用技能——一个你在提示中实际使用的触发词。按**顺序**切分：当一个步骤的**后续步骤**需要隐藏时，将一系列**步骤**拆分出来，因为将其隔离在自己的上下文中会清除后续内容。注意反面：合并序列会将每个步骤的后续步骤暴露给后续内容，导致过早完成。

_避免使用_：分块、模块化

### 路由技能（Router Skill）

一种**用户调用**技能，其职责是指向你的其他用户调用技能——命名每个技能以及何时使用它——这样人类只需要记住一个技能而不是多个。它只能提示，永远不能触发它们：用户调用技能没有 **description**，所以除了人类没有谁能调用它们。当用户调用技能增多时，这是**认知负载**的解药。

_避免使用_：调度器、菜单、注册表、索引、路由流程

### 信息层级（Information Hierarchy）

技能内容按智能体需要它的紧急程度排序——一个单一的阶梯，由两次切分产生：文件内还是指针后，以及步骤还是参考。层级如下：

- **步骤（Steps）**——文件内，主要
- **参考（Reference）**，文件内——次要
- **参考（Reference）**，已披露——在**上下文指针**之后

没有**步骤**的技能只使用下面两层——通常是一个合法的扁平对等集合（例如审查的每条规则都在同一层），这是一种良好的安排，不是坏味道。层级独立于调用方式：一个技能可以是模型调用或用户调用，无论它全是步骤、全是参考，还是两者兼有。当技能有步骤时，应该被披露的文件内参考会将其淹没，并将关注它们变成抛硬币——这是一个方差杠杆，而不仅仅是可读性杠杆。保持阶梯顶部的清晰；尽可能将内容往下推。

_避免使用_：结构、组织、布局

### 就近放置（Co-location）

将智能体一次需要的材料放在一个地方——一个概念的定义、规则和注意事项放在同一个标题下，而不是分散在文件中——这样阅读一部分就会连带看到其相邻内容。**信息层级**的文件内伴侣：层级决定了内容的_深度_；就近放置决定了_什么东西与它相邻_。对于一段**参考**的正确格式没有公式；检验标准是技能应该读起来像为智能体编写的文档，而分组的内容读起来像文档，分散的内容则不是。不同于**重复**：重复是在两个地方重复一个含义，而分散是将一个含义拆散到多处。

_避免使用_：分组、聚类、内聚

### 分支（Branch）

技能可以被调用的一种不同方式——技能处理的一种情况——因此不同的运行会经过不同的路径。一个有多个步骤的技能可能包含多个分支；一个线性的技能则没有。

_避免使用_：路径、情况、分叉

### 渐进式披露（Progressive Disclosure）

将**参考**向下移动到阶梯中——移出 SKILL.md 并放到**上下文指针**之后——使顶部保持清晰。主要不是 token 优化；它是保护**信息层级**的方式。由**分支**授权：只披露某些分支需要的内容，内联每个路径都需要的内容；如果某个指针对于必备材料触发不可靠，先强化其措辞，只有失败时才将其拉回内联。

_避免使用_：懒加载、分块

### 步骤（Steps）

智能体执行的有序动作——当技能有步骤时，它是内容的主要层级，也是它值得放在 SKILL.md 中的部分。不是每个技能都有步骤：一个技能可以全是步骤（`tdd`）、全是**参考**（审查），或者两者兼有，且独立于调用方式。每个步骤都以一个**完成标准**结束，无论清晰还是模糊。

_避免使用_：工作流、指令、编排

### 完成标准（Completion Criterion）

告诉智能体一个工作单元已完成的条件——它用来判断的目标。有两个属性使其成为杠杆，而不仅仅是质量指标。其**清晰度**（智能体能区分完成和未完成吗？）抵抗**过早完成**——一个模糊的界限（"理解达到"）让智能体宣布完成并滑入下一步；这个轴需要_步骤_才能生效，因为过早完成是一个跨步骤的失败。其**要求**（它要求多少）设定了**扎实工作**——"每个修改的模型都考虑到了"迫使彻底工作，而"生成变更列表"则不会——并且这个轴_不受_步骤限制：它也可以约束一组扁平的参考，这就是没有步骤的技能仍然承载穷尽性标准的方式（"每条规则都已应用"）。最强的标准既可检验又穷尽。

_避免使用_：完成条件、退出条件、停止规则

### 后续步骤（Post-Completion Steps）

紧跟当前步骤之后的**步骤**。可见时，它们会拉动智能体走向**过早完成**——看到的越多，拉力越强；防御方法是通过将步骤序列拆分为两个来隐藏它们。

_避免使用_：视野、战争迷雾、前瞻

### 扎实工作（Legwork）

智能体在单个步骤内幕后进行的工作——读取文件、探索代码库、进行修改、挖掘需要的东西而不是推卸给用户。它存在于步骤结构之下：从不写成自己的步骤，潜伏在措辞中，由智能体而非技能控制。是**后续步骤**跨步骤拉力的步骤内对应物。由**主导词**（_全面_、_彻底_）或要求工作穷尽的**完成标准**来提升——包括应用于扁平参考的要求轴，这正是驱动扁平参考技能覆盖其所有层级的力量。当该要求缺失或**过早完成**截断步骤时会变薄。

_避免使用_：范围、努力、勤奋、覆盖

### 参考（Reference）

智能体按需查阅的材料——定义、事实、参数、示例、条件指令。当技能有**步骤**时，它从属于步骤；当技能没有步骤时，它是全部内容；或者它完全存在于任何技能之外——见**外部参考**。通过**上下文指针**获取，是**渐进式披露**的主要候选对象。

_避免使用_：辅助材料、文档、背景

### 外部参考（External Reference）

存在于技能系统之外的**参考**——一个普通文件，没有 **description**、没有**步骤**、不可调用——任何技能都可以指向它。用于不需要自行触发的共享参考的归宿，也是两个**用户调用**技能可以使用的唯一共享归宿，因为两者都没有 description，因此都不能调用对方。

_避免使用_：文档、资源、知识库

### 主导词（Leading Word）

一个紧凑的概念——也称为 *Leitwort*——已经存在于模型的预训练中，智能体在运行技能时用它来思考。它通过调用模型已有的先验知识，用最少的 token 编码一个行为原则（例如 _lesson_、_proximal zone of development_、_fog of war_、_tracer bullets_）。作为一个 token 重复出现，从不作为句子，它在技能中积累分布式定义，并锚定整个行为区域。如果你清楚地定义它，自己造词也行，但一个生造的词不招募任何先验知识——你用定义 token 付出的代价，预训练词免费提供。优先寻找已有的词。

主导词以两种方式服务于**可预测性**。在正文中，它锚定**执行**——智能体每次遇到这个概念时都会采取相同的行为；在扁平参考中，它将注意力集中在一类要查找的事物上，每次运行都会招募正确的检查。在 **description** 中，它锚定**调用**——而且不仅在技能内部：当同一个词出现在你的提示、文档和代码库中时，智能体会将这种共享语言与技能关联起来，更可靠地触发它。用你在想要该技能时实际使用的主导词来措辞 description。

_避免使用_：关键词、术语、母题

### 单一事实来源（Single Source of Truth）

每个含义只存在于唯一权威位置的期望状态，这样对技能行为的更改只需在一处进行。**重复**是对它的违反。

_避免使用_：归宿、规范位置

### 相关性（Relevance）

一行内容是否仍然与技能的功能相关——是保留什么的透镜。一行内容失去相关性，要么是因为它从未与任务相关（纯粹的解释性内容，或应该被披露的**分支**），要么是因为它变得过时：随着它所描述的行为或世界变化而漂移。较短的技能更容易保持相关性，因为每行的检查成本更低。不同于**空操作**：相关性问的是一行是否与任务相关，而不是它是否改变行为。

_避免使用_：承载性、陈旧性、新鲜度

## 失败模式

### 过早完成（Premature Completion）

在当前步骤真正完成之前就结束它，因为智能体的注意力滑向了"完成"而非工作本身。一种跨步骤的失败：它需要**步骤**才会发生——一个没有步骤的技能提前退出不是过早完成，而是在未满足要求下的**扎实工作**薄弱。两种力量之间的拉锯战：可见的**后续步骤**（向前的拉力）和**完成标准**的清晰度（抵抗力——一个清晰、可检验的界限能守住；一个模糊的则会失守）。模糊性是必要条件：一个清晰的界限无论如何都能抵抗拉力，无论后面有多少可见步骤，所以一个从不仓促的步骤不需要防御。对于一个确实需要防御的步骤，有两个杠杆，但按顺序使用：**先强化界限**——它是局部的且成本低。只有当选标准是不可简化的模糊_且_你确实观察到了仓促行为时，才**隐藏后续步骤**——而隐藏只有在真正的上下文边界（用户调用的交接或子智能体派发；内联的模型调用会让后续步骤留在上下文中，什么也清除不掉）下才有效。是扎实工作薄弱的一个原因，但与之不同：即使步骤运行到完全完成，扎实工作也可能薄弱。

_避免使用_：过早关闭、仓促、赶工、走捷径

### 重复（Duplication）

同一个含义被赋予了多个**单一事实来源**。它增加维护成本（改一处，必须改其他处），消耗 token，并膨胀显著性——重复一个含义会让它在阶梯上的权重超过其真实排名。是**主导词**的意外逆操作，主导词通过重复一个 token（而非含义）来有意提升注意力。

_避免使用_：反复、冗余

### 沉积（Sediment）

旧内容层层堆积在技能中从未清理，因为添加感觉安全而删除感觉有风险——于是陈旧的、无关的行不断积累，你必须钻透它们才能找到仍然有效的内容。任何没有修剪纪律的技能的默认命运；**相关性**的缓慢侵蚀，与**重复**的重复含义相对。

_避免使用_：堆积、膨胀、腐化、腐烂

### 蔓延（Sprawl）

一个技能就是太长了——SKILL.md 中的行太多——独立于它们是否陈旧或重复。即使一个全是有效、全是独特内容的技能也可能蔓延。它消耗可读性（智能体在行动前必须跋涉更多内容，注意力在多余内容上变薄）、可维护性（每多一行就多一行需要保持**相关性**）和 token。解药是**信息层级**：将**参考**推到**上下文指针**之后，并按**分支**或顺序拆分，使每条路径只携带自己需要的内容。不同于**沉积**（因陈旧积累导致的长度）和**重复**（因重复含义导致的长度）——蔓延就是长度本身，无论其原因是什么。

_避免使用_：膨胀、长度、规模、冗长

### 空操作（No-Op）

一条指令不改变任何东西，因为模型默认就会这样做——你付出负载来告诉智能体它本来就会做的事。检验标准：一行代码是否改变了相对于默认值的行为？一行代码可以完全**相关**却仍然是空操作。使**主导词**免费的相同先验知识也使空操作毫无价值。

主导词是一种_技术_；空操作是对一行的_裁决_——它们会交叉。一个太弱而无法战胜默认值的主导词就是空操作（_be thorough_ 当智能体本身就足够 thorough），修复方法是使用一个通过裁决的更强的词（_relentless_），而不是换一种技术。因此，空操作检验——它是否改变了相对于默认值的行为？——也是你评估一个主导词是否值得其重复次数的标准。这是相对于模型的，而非相对于读者的：两个人在一行是否是空操作上意见不一致，是对默认值的判断不同，应通过运行技能来解决，而非辩论。

_避免使用_：冗余指令、重申显而易见、啰嗦
