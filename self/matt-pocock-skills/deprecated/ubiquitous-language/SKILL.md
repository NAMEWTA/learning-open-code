# 通用语言

从当前对话中提取并规范化领域术语，形成一致的词汇表，保存到本地文件。

## 流程

1. **扫描对话**，找出与领域相关的名词、动词和概念
2. **识别问题**：
   - 同一词语用于不同概念（歧义）
   - 不同词语用于同一概念（同义词）
   - 模糊或过载的术语
3. **提出规范词汇表**，给出有主见的术语选择
4. **写入 `UBIQUITOUS_LANGUAGE.md`**，保存到工作目录，使用以下格式
5. **在对话中输出摘要**

## 输出格式

编写一个 `UBIQUITOUS_LANGUAGE.md` 文件，结构如下：

```md
# 通用语言

## 订单生命周期

| 术语         | 定义                           | 应避免的别名         |
| ------------ | ------------------------------ | -------------------- |
| **Order**    | 客户购买一件或多件商品的请求   | Purchase, transaction |
| **Invoice**  | 交付后发送给客户的付款请求     | Bill, payment request |

## 人员

| 术语           | 定义                         | 应避免的别名           |
| -------------- | ---------------------------- | ---------------------- |
| **Customer**   | 下订单的个人或组织           | Client, buyer, account |
| **User**       | 系统中的认证身份             | Login, account         |

## 关系

- 一个 **Invoice** 属于且仅属于一个 **Customer**
- 一个 **Order** 产生一个或多个 **Invoice**

## 对话示例

> **开发：**"当 **Customer** 下了一个 **Order**，我们是立即创建 **Invoice** 吗？"
> **领域专家：**"不——**Invoice** 只有在 **Fulfillment** 确认后才生成。如果商品分多个 **Shipment** 发货，一个 **Order** 可以产生多个 **Invoice**。"
> **开发：**"所以如果 **Shipment** 在发货前被取消，就不会有对应的 **Invoice**？"
> **领域专家：**"没错。**Invoice** 的生命周期与 **Fulfillment** 绑定，而不是 **Order**。"

## 标记的歧义

- "account" 曾被用于同时表示 **Customer** 和 **User**——这是两个不同的概念：**Customer** 下订单，而 **User** 是认证身份，可能代表也可能不代表 **Customer**。
```

## 规则

- **要有主见。** 当同一概念存在多个词语时，挑选最佳的一个，将其余列为应避免的别名。
- **明确标记冲突。** 如果对话中某个术语存在歧义使用，在"标记的歧义"部分指出并给出明确的建议。
- **只包含领域专家关心的术语。** 跳过模块或类的名称，除非它们在领域语言中有意义。
- **保持定义精炼。** 最多一句话。定义它是什么，而非它做什么。
- **展示关系。** 使用粗体术语名称，并在明显时注明基数。
- **只包含领域术语。** 跳过通用编程概念（array、function、endpoint），除非它们具有领域特定的含义。
- **将术语分组到多个表格中**，当存在自然聚类时（如按子域、生命周期或角色）。每个分组有自己的标题和表格。如果所有术语属于一个统一的领域，一个表格也可以——不要强制分组。
- **编写对话示例。** 一段简短的对话（3-5 轮），展示术语如何自然地交互。对话应澄清相关概念之间的边界，并展示术语的精确使用。

## 对话示例

> **开发：**"如何在不用 Docker 的情况下测试 **sync service**？"

> **领域专家：**"使用 **filesystem layer** 代替 **Docker layer**。它实现了相同的 **Sandbox service** 接口，但使用本地目录作为 **sandbox**。"

> **开发：**"所以 **sync-in** 仍然创建 **bundle** 并解包？"

> **领域专家：**"没错。**sync service** 不知道它在与哪个层通信。它调用 `exec` 和 `copyIn`——**filesystem layer** 只是将这些作为本地 shell 命令运行。"

</example>

## 重新运行

在同一对话中再次调用时：

1. 读取现有的 `UBIQUITOUS_LANGUAGE.md`
2. 纳入后续讨论中的任何新术语
3. 如果理解有变化，更新定义
4. 重新标记任何新的歧义
5. 重写对话示例以纳入新术语
