# 改进代码库架构

浮现架构摩擦并提出**深化机会**——将浅层模块转化为深层模块的重构。目标是可测试性和 AI 可导航性。

此命令*基于*项目的领域模型，并建立在共享的设计词汇之上：

- 运行 `/codebase-design` 技能获取架构词汇（**模块**、**接口**、**深度**、**接缝**、**适配器**、**杠杆**、**局部性**）及其原则（删除测试、"接口就是测试表面"、"一个适配器 = 假设接缝，两个 = 真实接缝"）。在每条建议中严格使用这些术语——不要偏离到"组件"、"服务"、"API"或"边界"。
- `CONTEXT.md` 中的领域语言为好的接缝命名；`docs/adr/` 中的 ADR 记录此命令不应重新争议的决策。

## 流程

### 1. 探索

首先阅读项目的领域词汇表（`CONTEXT.md`）和你接触区域中的任何 ADR。

然后使用 Agent 工具，`subagent_type=Explore`，遍历代码库。不要遵循僵化的启发式——有机地探索，注意你在哪里遇到摩擦：

- 在哪里理解一个概念需要在许多小模块之间跳转？
- 哪里模块是**浅层的**——接口几乎和实现一样复杂？
- 哪里纯函数仅为可测试性而被提取，但真正的 bug 隐藏在其调用方式中（没有**局部性**）？
- 哪里紧密耦合的模块在其接缝处泄漏？
- 代码库的哪些部分未经测试，或难以通过其当前接口进行测试？

将**删除测试**应用于任何你怀疑是浅层的东西：删除它会集中复杂性，还是只是移动它？"是的，集中"就是你想要的信号。

### 2. 以 HTML 报告呈现候选

编写一个自包含的 HTML 文件到操作系统临时目录，以免任何东西落入仓库。从 `$TMPDIR` 解析临时目录，回退到 `/tmp`（Windows 上为 `%TEMP%`），写入 `<tmpdir>/architecture-review-<timestamp>.html`，使每次运行获得新文件。为用户打开它——Linux 上 `xdg-open <path>`，macOS 上 `open <path>`，Windows 上 `start <path>`——并告诉他们绝对路径。

报告使用**CDN 加载的 Tailwind** 进行布局和样式，以及**CDN 加载的 Mermaid** 用于图/流程/序列可靠传达结构的图表。混合使用 Mermaid 和手工制作的 CSS/SVG 视觉——当关系是图形状的（调用图、依赖、序列）使用 Mermaid，当你想更编辑化（质量图、横截面、折叠动画）时使用手工构建的 div/SVG。每个候选获得一个**前后对比可视化**。要有视觉表现力。

对于每个候选，渲染一张卡片：

- **文件**——涉及哪些文件/模块
- **问题**——为什么当前架构正在造成摩擦
- **解决方案**——用通俗语言描述将要改变什么
- **收益**——以局部性和杠杆的方式解释，以及测试将如何改善
- **前后对比图**——并排，自定义绘制，展示浅层性和深化
- **推荐强度**——以下之一：`强烈`、`值得探索`、`推测性`，渲染为徽章

以**首要推荐**部分结束报告：你首先处理哪个候选及其原因。

**领域用 CONTEXT.md 词汇，架构用 `/codebase-design` 词汇。** 如果 `CONTEXT.md` 定义了"Order"，谈论"Order 接收模块"——不是"FooBarHandler"，也不是"Order 服务"。

**ADR 冲突**：如果候选与现有 ADR 矛盾，仅当摩擦真实到足以值得重新审视 ADR 时才浮现它。在卡片中明确标记（例如警告标注：*"与 ADR-0007 矛盾——但因……值得重新讨论"*）。不要列出 ADR 禁止的每个理论重构。

参见 [HTML-REPORT.md](HTML-REPORT.md) 获取完整的 HTML 框架、图表模式和样式指南。

此时不要提出接口。文件写入后，询问用户："这些中你想探索哪一个？"

### 3. 质询循环

一旦用户选择了一个候选，运行 `/grilling` 技能与他们遍历设计树——约束、依赖、深化模块的形态、接缝后面的内容、哪些测试存活。

副作用在决策结晶时内联发生——在过程中运行 `/domain-modeling` 技能以保持领域模型最新：

- **深化模块以 `CONTEXT.md` 中不存在的概念命名？** 将术语添加到 `CONTEXT.md`。如果文件不存在则惰性创建。
- **在对话中锐化模糊术语？** 立即更新 `CONTEXT.md`。
- **用户以关键理由拒绝候选？** 提供 ADR，框架为：*"想让我将此记录为 ADR，以便未来的架构审查不会重新建议吗？"* 仅当理由确实需要未来的探索者以避免重新建议相同事情时才提供——跳过短暂的理由（"现在不值得做"）和显而易见的理由。
- **想探索深化模块的替代接口？** 运行 `/codebase-design` 技能并使用其设计两次的并行子代理模式。
