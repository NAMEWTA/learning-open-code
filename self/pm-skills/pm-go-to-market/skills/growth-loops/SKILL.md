# 增长循环

## 概述
识别和设计能创造可持续牵引力的增长循环（飞轮）。评估五种经过验证的增长循环机制，以减少对付费获客的依赖，构建产品驱动型增长。

## 使用场景
- 为产品设计增长机制
- 构建可持续的病毒式或推荐式牵引力
- 减少对付费获客的依赖
- 分析竞争对手的增长策略
- 为产品驱动型增长优化产品

## 五种增长循环类型

### 1. 病毒式循环
用户在产品中创建的内容被分享到外部平台，将新用户带回产品。
- **机制**：用户在产品中创建内容 → 分享到社交/外部平台 → 新用户发现并注册
- **示例**：以链接形式分享 Figma 设计、在邮件中分享 Loom 视频
- **优势**：如果内容天然可分享，可实现指数级用户获取
- **挑战**：需要产出高度可分享的内容和强大的分享激励

### 2. 使用式循环
用户在产品内创建内容或价值，然后分享，从而邀请新用户或驱动老用户回访。
- **机制**：用户创建 → 分享创作 → 他人消费 → 成为活跃用户
- **示例**：Twitter 话题串、Medium 文章、公开发布的 Notion 模板
- **优势**：增长直接与产品使用和网络效应绑定
- **挑战**：要求内容创作摩擦极低

### 3. 协作式循环
用户邀请同事共同创作或协作，从而在组织内扩展用户基础。
- **机制**：用户创建 → 邀请同事协作 → 同事发现产品价值
- **示例**：Google Docs 邀请、Figma 团队项目、Slack 频道
- **优势**：深度的组织渗透和高留存率
- **挑战**：最适合协作/团队型产品

### 4. 用户生成式循环
用户通过其他用户的创作发现新内容或功能，然后创建并分享自己的内容。
- **机制**：用户发现内容 → 创建类似内容 → 分享创作 → 他人发现
- **示例**：TikTok、Pinterest、YouTube 趋势驱动创作者参与
- **优势**：创造内容飞轮和网络效应
- **挑战**：需要足够量的优质内容才能持续运转

### 5. 推荐式循环
用户换取奖励、激励或社交认可来邀请其他潜在用户。
- **机制**：用户推荐 → 被推荐者加入 → 推荐者获得奖励 → 分享更多推荐
- **示例**：Dropbox 推荐奖励、Uber 乘客推荐、PayPal 注册奖励
- **优势**：直接激励获客；易于衡量 ROI
- **挑战**：需要提供有价值的激励同时不破坏单位经济模型

## 工作方式

### 第 1 步：定义产品价值
明确用户体验到的核心价值：
- 用户在产品中的主要操作
- 每次用户操作创造的价值
- 网络效应（如有）
- 体验中的摩擦点

### 第 2 步：评估循环匹配度
评估哪些增长循环与产品适配：
- 产品类型（协作型、内容型、工具型等）
- 目标用户的行为和分享习惯
- 已有的网络效应
- 现有用户基础和参与度

### 第 3 步：设计循环机制
创建具体的循环实施方案：
- 触发分享或邀请的触发器
- 参与激励（内在或外在）
- 分享机制的便捷程度
- 从邀请到激活的转化率
- 每位用户的循环重复频率

### 第 4 步：计算循环系数
估算增长速度：
- 每位用户每周期邀请/分享数
- 邀请到新用户的转化率
- 每周期净新增用户
- 每次循环迭代的时间

### 第 5 步：构建循环
首先实施杠杆最高的循环：
- 从产品最自然的循环开始
- 优化信息传递和摩擦
- 衡量循环指标和转化率
- 随时间累积成果

## 输入格式
通过 $ARGUMENTS 传递：
- 产品描述和主要用户操作
- 目标用户人口统计和行为
- 现有的分享/协作功能
- 当前的增长渠道和指标
- 约束条件或机会

## 输出
增长循环分析包括：
- 针对产品对所有 5 种循环类型的排序评估
- 推荐的首选增长循环及实施方案
- 随时间叠加的次要循环
- 关键指标和衡量框架
- 30-60-90 天实施路线图
- 潜在循环系数和增长预测

## 框架
基于 Ognjen Bošković 的增长循环研究。聚焦于通过内置的、产品原生的分享和协作机制实现复合式用户获取。

## 技巧
- 从单一循环开始，掌握后再增加复杂性
- 病毒式循环复合增长最快，但需要时间积累
- 协作式循环能创造最强的留存和 LTV
- 优化阶段每周衡量循环健康状况
- 规模化运营后组合多个循环以实现乘数效应
