# /analyze-feedback -- 用户反馈分析

将大量用户反馈（评论、问卷、支持工单、NPS 回复）处理为结构化的洞察，包含情感分析和细分层面模式。

## 调用方式

```
/analyze-feedback [上传 NPS 回复的 CSV 文件]
/analyze-feedback [粘贴应用商店评论或问卷回复]
/analyze-feedback [上传支持工单导出]
```

## 工作流

### 第 1 步：接收反馈数据

接受任何格式：CSV/Excel（附带反馈文本）、粘贴文本、上传文档。

### 第 2 步：分析

应用 **sentiment-analysis** skill：情感评分、主题提取、频率分析、细分分析、趋势检测

### 第 3 步：生成分析报告

保存为 markdown：整体情感、Top 主题表格、每个主题的深入分析、细分分析、值得注意的引用、随时间趋势、可操作洞察、数据缺口。

### 第 4 步：提供后续建议

- "需要我基于这些反馈模式**创建用户画像**吗？"
- "需要我**将 Top 主题分类为功能请求**吗？"

## 注意事项

- 情感分析是近似的——标注边界情况（讽刺、混合情感、非英文文本）
- 主题提取应寻找请求背后的需求，而非仅表面话题
- 对于 NPS 数据，单独分析批评者（0-6）、中立者（7-8）和推荐者（9-10）
