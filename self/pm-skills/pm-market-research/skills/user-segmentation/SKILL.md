# 用户细分

## 目的
分析多样化的用户反馈，识别至少 3 个独特的行为和基于需求的用户细分群体。基于待完成工作（JTBD）、行为和动机而非仅凭人口统计特征，发现隐藏的客户群体，实现有针对性的产品策略。

## 操作指引

你是一位专家级行为研究员和数据分析师，专精于用户细分和行为聚类。

### 输入
你的任务是基于行为、JTBD 和未满足的需求，为 **$ARGUMENTS** 对用户进行细分。

如果用户提供了反馈数据、访谈、支持工单、产品使用日志、问卷或其他用户数据，请直接读取和分析。在整个用户群体中提取行为模式、动机和需求。

### 分析步骤（逐步思考）

1. **数据准备**：读取和组织所有提供的用户反馈和数据
2. **行为提取**：识别关键行为模式、使用方式和用户旅程
3. **需求分析**：映射每个用户的 JTBD、期望成果和痛点
4. **聚类**：基于行为和需求相似性将用户归为独特的细分群体
5. **验证**：确保群体内部一致、不重叠且可操作
6. **特征刻画**：为每个群体开发丰富的画像，附代表性引述

### 输出结构

对每个识别出的群体（至少 3 个）：

**群体名称与概览**
- 清晰、描述性的群体标识
- 规模：用户基数中的估计数量或百分比
- 一句话简要特征描述

**行为特征**
- 该群体如何使用 $ARGUMENTS（主要使用场景、频率、深度）
- 典型用户旅程和关键触点
- 技术熟练程度
- 与其他工具或工作流的集成

**待完成工作与动机**
- 该群体试图完成的核心工作
- 潜在的动机和期望成果
- 工作的情境和频率
- 该群体眼中的成功是什么

**关键需求与痛点**
- 该群体行为特有的未满足需求
- 阻碍有效完成工作的障碍
- 他们当前采用的替代手段或替代方案
- 痛点的严重程度和频率

**当前产品匹配度**
- $ARGUMENTS 当前在多大程度上服务该群体
- 该群体最看重的功能或能力
- 最令该群体沮丧的差距或局限性
- 继续使用 vs 流失风险的可能性

**差异化价值主张**
- 可为该群体解锁的独特价值
- 能最大化匹配度的功能或体验改进
- 最能引起该群体共鸣的信息传递和定位

**群体优先级**
- 战略重要性：增长潜力、收入影响、与愿景的对齐度
- 实施难度：满足该群体需求的难易程度
- 建议：投入、维持或降级

## 最佳实践

- 基于行为和动机数据而非仅凭人口统计特征进行细分
- 使用来自实际用户反馈的代表性引述和示例
- 确保群体特点鲜明且服务不同的核心需求
- 考虑群体之间的相互依赖关系和优先级权衡
- 标注可能在反馈数据中代表性不足的群体
- 在可用时，对照产品使用或客户数据验证新兴群体
- 考虑相邻行为和跨群体模式
