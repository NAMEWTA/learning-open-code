## 客户旅程地图

绘制从认知到拥护的端到端客户体验地图，识别每个阶段的情绪、痛点和改进机会。

### 背景

你正在为 **$ARGUMENTS** 创建客户旅程地图。

如果用户提供了文件（访谈记录、问卷数据、分析数据、支持工单或现有旅程地图），请先阅读。如果提供了 URL，使用网络搜索了解产品。

### 操作指引

1. **定义画像**：谁在这条旅程中？使用带有 JTBD 的具体画像，而非泛化的用户。

2. **映射旅程阶段**（根据产品调整）：

   | 阶段 | 描述 |
   |---|---|
   | **认知** | 他们如何首次了解到产品？ |
   | **考虑** | 他们评估什么？比较哪些替代方案？ |
   | **获取** | 他们如何注册或购买？ |
   | **引导** | 产品的首次体验——实现价值的时间 |
   | **参与** | 日常使用——养成习惯 |
   | **留存** | 什么让他们持续使用？什么可能导致流失？ |
   | **拥护** | 他们何时以及为何向他人推荐产品？ |

3. **对每个阶段，记录**：

   - **触点**：用户与产品、品牌或团队互动的接触点（网站、邮件、应用内、客服、社交媒体）
   - **用户行为**：他们在此阶段做什么
   - **想法与疑问**：他们脑海中在想什么（"这值得花时间吗？""我怎么……？"）
   - **情绪**：他们的感受（兴奋、困惑、沮丧、愉悦）——用量表评分或使用表情符号
   - **痛点**：摩擦、困惑、流失风险
   - **机会**：如何改善此阶段的体验

4. **识别关键时刻**：
   - **Aha 时刻**：用户首次体验到核心价值的时刻
   - **真相时刻**：他们决定投入或放弃的决策节点
   - **流失触发点**：用户最常见的流失环节

5. **创建旅程地图表格**：

   | 阶段 | 触点 | 用户行为 | 情绪 | 痛点 | 机会 |
   |---|---|---|---|---|---|

6. **推荐优先改进项**：
   - 哪些痛点对转化或留存影响最大？
   - 哪些快赢方案可以立即改善体验？
   - 哪些需要更深入投入但回报最大？

逐步思考。保存为 markdown 文档。如需可视化旅程地图，建议用户以本分析为基础在 Miro 或 FigJam 中创建。
