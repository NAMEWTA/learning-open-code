## 定价策略

设计一个基于价值交付、竞争定位和支付意愿的定价策略。

### 背景

你正在为 **$ARGUMENTS** 制定定价策略。

如果用户提供了文件（竞争对手定价、问卷数据、财务模型或使用数据），请先阅读。如需要，使用网络搜索研究竞争对手定价。

### 操作指引

1. **理解交付的价值**
2. **评估定价模型** — 推荐最佳匹配：

   | 模式 | 最适合 | 示例 |
   |---|---|---|
   | **统一费率** | 简单产品、可预测成本 | Basecamp（$99/月 统一价） |
   | **按席位** | 协作工具、团队产品 | Slack、Figma |
   | **按用量** | 基础设施、API 产品 | AWS、Twilio |
   | **分层定价** | 有明确用户群体的产品 | 大多数 SaaS（Free/Pro/Enterprise） |
   | **免费增值** | 有病毒式/网络效应的产品 | Spotify、Notion |
   | **免费增值 + 用量** | 平台产品 | Vercel、OpenAI API |
   | **基于价值** | 高影响力的企业工具 | Salesforce、Palantir |

3. **分析竞争定价**
4. **设计定价结构** — 2-4 个分层、功能门槛、价值指标、锚定定价、年付折扣（通常 15-20%）
5. **估算价格敏感度** — Van Westendorp 价格敏感度测量法或基于竞争对手定价和价值交付估算
6. **规划定价实验** — A/B 测试定价页面、创始人主导的销售对话、落地页测试
7. **输出定价建议**

逐步思考。保存为 markdown。标注上线前需要验证的假设。
