# /performance-audit-static -- 找出无法扩展的部分

针对 AI 构建代码的聚焦性能审查。代理优化的是「在我的种子数据上能用」，而非「在 100× 行数据下能撑住」。本命令找出随着数据增长会暴露的三种失败模式——过度获取、缺失索引和缺乏缓存——并按工作量和影响排序修复方案。

## 审计

### 1. 视图负载中的过度获取

审查渲染列表和仪表盘视图的组件。识别从数据库获取但前端从未使用的字段、宽表上的 `SELECT *`、缺少分页、缺少懒加载以及冗余加载。

### 2. 缺失或低效的索引

审查生产视图中使用的查询、过滤器和 RPC。基于排序、过滤和 join 条件识别缺失或低效的索引。

### 3. 缓存优化机会

审查频繁调用的路径的端点及数据访问模式，识别前端或后端缓存有帮助的地方，并为每个缓存指定失效规则。

## 输出

按视图、路由或表报告发现：
```
<视图 / 路由 / 表>:
  - 发现 | 建议 | 工作量 | 优先级 | 预期效果
```

## 注意事项

- 按单位工作量的影响排序——一个热表上的缺失索引通常胜过十项微优化。
- 不要标注没有增长路径的理论性低效。
- 本命令仅覆盖性能。授权、注入和数据暴露风险使用 `/security-audit-static`。
