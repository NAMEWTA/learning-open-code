# /write-query -- SQL 查询生成器

用通俗英文描述你需要什么数据，获得优化后的 SQL 查询。支持多种方言，可从上传的文件中读取你的 schema。

## 调用方式

```
/write-query 显示过去 30 天的每日活跃用户，按套餐层级拆分
/write-query 查找上个月注册但从未完成引导的用户
/write-query [上传 schema 图表] 按同期群的试用转付费转化率是多少？
```

## 工作流

### 第 1 步：理解问题

解析用户的自然语言请求以识别：请求的数据、时间范围和粒度、分组和排序偏好、输出期望

### 第 2 步：确定 Schema

- 如有 schema → 将请求映射到具体表和列、识别必要 join
- 如无 schema → 询问数据库类型、从问题推断合理 schema 并请用户确认

### 第 3 步：生成查询

应用 **sql-queries** skill：使用正确方言编写 SQL、优化可读性和性能、包含注释解释关键逻辑、用 CTE 改进复杂查询的可读性、处理边界情况

### 第 4 步：呈现并迭代

呈现查询、返回内容、假设条件、注意事项。提供修改、配套查询、构建仪表盘等后续选项。

## 注意事项

- SQL 中始终包含注释
- 默认可读性优先于技巧性——CTE 优先于嵌套子查询
- 标注在大型数据集上可能较慢的查询并建议优化
- 如果请求模糊（如「活跃用户」），请用户精确定义指标
