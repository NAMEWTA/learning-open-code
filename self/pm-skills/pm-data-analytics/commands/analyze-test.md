# /analyze-test -- A/B 测试分析

以统计学严谨性评估实验成果，并将发现转化为明确的产品决策：发布、继续或停止。

## 调用方式

```
/analyze-test 对照组：4.2% 转化率（n=5000），变体：4.8% 转化率（n=5100）
/analyze-test [上传测试结果的 CSV 文件]
/analyze-test [来自实验平台的截图]
```

## 工作流

### 第 1 步：接收测试数据

接受任何格式：汇总统计数据、原始事件数据（CSV）、实验平台截图、实验结果描述

### 第 2 步：验证测试设计

分析结果前，检查：样本量是否充足？测试是否运行足够长时间？随机化是否干净？测试期间是否有外部因素？

### 第 3 步：分析结果

应用 **ab-test-analysis** skill：统计显著性、效应量、实际显著性、置信区间、细分分析（如数据允许）

### 第 4 步：生成分析

结果摘要表、统计分析、样本量检查、**决策建议：发布/继续/停止**、业务影响估算、注意事项、后续跟进

### 第 5 步：提供后续建议

- "需要我基于这些发现**设计后续实验**吗？"
- "需要我为**特定细分群体**进行分析吗？"
- "需要我**生成 SQL** 在发布后监控该指标吗？"

## 注意事项

- 统计显著性 ≠ 实际显著性
- 在信任结果前始终检查样本比例不匹配（SRM）
- 新奇效应可能夸大短期结果——建议发布后监控 2-4 周
- 如测试检验功效不足，正确答案通常是「延长」而非「无效」
- 如以 CSV 提供数据，使用 Python scipy.stats 生成完整分析
