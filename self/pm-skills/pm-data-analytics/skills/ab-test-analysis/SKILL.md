## A/B 测试分析

以统计学严谨性评估 A/B 测试结果，并将发现转化为明确的产品决策。

### 背景

你正在分析 **$ARGUMENTS** 的 A/B 测试结果。

如果用户提供了数据文件（CSV、Excel 或分析导出），直接读取并分析。必要时生成 Python 脚本进行统计计算。

### 操作指引

1. **理解实验**：
   - 假设是什么？
   - 变更了什么（变体）？
   - 主要指标是什么？有哪些护栏指标？
   - 测试运行了多长时间？
   - 流量分配比例是多少？

2. **验证测试设置**：
   - **样本量**：样本量是否足够检测预期效应量？
     - 使用公式：n = (Z²α/2 × 2 × p × (1-p)) / MDE²
     - 如果检验功效不足（<80%），予以标注
   - **持续时间**：测试是否至少运行了 1-2 个完整业务周期？
   - **随机化**：是否有样本比例不匹配（SRM）的证据？
   - **新奇/首因效应**：是否有足够时间消除初始行为变化的影响？

3. **计算统计显著性**：
   - 对照组和变体的**转化率**
   - **相对提升**：(变体 - 对照组) / 对照组 × 100
   - **p 值**：使用双尾 z 检验或卡方检验
   - **置信区间**：差异的 95% CI
   - **统计显著性**：p < 0.05？
   - **实际显著性**：提升幅度对业务是否有意义？

   如果用户提供了原始数据，生成并运行 Python 脚本进行计算。

4. **检查护栏指标**：
   - 是否有护栏指标（收入、参与度、页面加载时间）恶化？
   - 主要指标获胜但护栏指标恶化，可能不是真正的成功

5. **解读结果**：

   | 结果 | 建议 |
   |---|---|
   | 显著正向提升，无护栏问题 | **发布** — 推全至 100% |
   | 显著正向提升，护栏指标有顾虑 | **调查** — 发布前了解权衡取舍 |
   | 不显著，呈正向趋势 | **延长测试** — 需要更多数据或更大效应量 |
   | 不显著，持平 | **停止测试** — 未检测到有意义的差异 |
   | 显著负向提升 | **不发布** — 回退至对照组，分析原因 |

6. **提供分析摘要**：
   ```
   ## A/B 测试结果：[测试名称]

   **假设**：[我们的预期]
   **持续时间**：[X 天] | **样本量**：[N 对照组 / M 变体]

   | 指标 | 对照组 | 变体 | 提升 | p 值 | 是否显著？ |
   |---|---|---|---|---|---|
   | [主要指标] | X% | Y% | +Z% | 0.0X | 是/否 |
   | [护栏指标] | ... | ... | ... | ... | ... |

   **建议**：[发布 / 延长 / 停止 / 调查]
   **理由**：[原因]
   **后续步骤**：[做什么]
   ```

逐步思考。保存为 markdown 格式。如果提供了原始数据，生成 Python 脚本进行计算。
