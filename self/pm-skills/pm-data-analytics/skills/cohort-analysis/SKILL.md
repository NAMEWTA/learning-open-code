# 同期群分析与留存探索

## 目的
按同期群分析用户参与度和留存模式，识别用户行为、功能采用和长期参与度方面的趋势。将定量洞察与定性研究建议相结合。

## 工作方式

### 第 1 步：读取并验证数据
- 接受包含用户同期群信息的 CSV、Excel 或 JSON 数据文件
- 验证数据结构：同期群标识、时间周期、参与度指标
- 检查缺失值和数据质量问题
- 汇总关键统计信息（同期群规模、日期范围、可用指标）

### 第 2 步：生成定量分析
- 计算同期群留存率和参与度趋势
- 识别留存曲线、流失模式和异常情况
- 计算各同期群的功能采用率
- 计算环比或周期对比变化
- 如有需要，使用 pandas 和 numpy 生成 Python 分析脚本

### 第 3 步：创建可视化
- 生成留存热力图（同期群 vs 时间周期）
- 创建展示同期群变化趋势的折线图
- 构建功能采用对比图表
- 可视化流失节点和参与度趋势
- 输出为交互式图表或静态图片

### 第 4 步：识别洞察与模式
- 发现一个或多个显著模式：
  - 特定同期群的早期流失
  - 后期参与度变化
  - 功能采用聚类
  - 季节性趋势
- 突出令人意外的发现和偏差
- 对比同期群表现以建立基准线

### 第 5 步：建议后续研究
- 推荐定性研究方法：
  - 针对流失用户的目标访谈
  - 针对高参与度同期群的功能使用调查
  - 关键交互模式的会话回放
  - 高留存 vs 低留存同期群的赢/输分析
- 设计后续定量研究
- 建议 A/B 测试或功能实验

## 使用示例

**示例 1：上传 CSV 数据**
```
上传 cohort_engagement.csv，列包含：cohort_month、weeks_active、
user_id、feature_x_usage、engagement_score

请求："分析留存模式，并识别为什么 2025 年 Q4 的同期群
表现不如 Q3"
```

**示例 2：描述数据格式**
```
"我有 2025 年 1 月至 12 月的月度用户同期群数据。每行包含：
同期群日期、用户 ID、购买频率和支持工单数。
分析哪些同期群显示出最佳的长期留存率。"
```

**示例 3：功能采用分析**
```
上传包含同期群采用数据的 feature_usage.xlsx。

请求："比较各同期群对我们的新功能的采用曲线。
哪些同期群采用最快？有什么模式？"
```

## 核心能力

- **数据读取**：导入 CSV、Excel、JSON、SQL 查询结果
- **留存分析**：计算并可视化各时期的留存率
- **同期群对比**：对比不同同期群的指标
- **异常检测**：标注异常模式或流失节点
- **Python 脚本**：生成可复用的分析代码，支持持续分析
- **可视化**：创建热力图、图表和交互式仪表盘
- **研究设计**：建议有针对性的后续研究和访谈方法
- **统计汇总**：提供定量指标和相关性分析

## 获取最佳结果的技巧

1. **包含时间维度**：提供跨多个时间周期的数据
2. **明确定义同期群**：使同期群分组明确（注册月份、功能上线日期等）
3. **提供背景**：说明该期间内的产品变更、发布或事件
4. **多指标覆盖**：包括留存、参与度、功能使用、收入等
5. **充足数据**：至少 3-4 个同期群才能识别有意义的模式
6. **指定输出需求**：要求可视化、Python 脚本或研究建议

## 输出格式

你将获得：
- **数据摘要**：同期群概览和数据质量评估
- **定量发现**：关键指标、留存率和趋势分析
- **可视化**：展示留存曲线、采用模式的图表
- **模式识别**：数据中的 2-3 个重要洞察
- **研究建议**：具体的定性和定量后续研究
- **分析脚本**（如需）：可复现分析的 Python 代码
- **后续步骤**：基于发现排定优先级的行动项
