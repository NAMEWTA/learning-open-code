# pm-product-discovery

面向 PM 的产品探索 skills：创意构思、实验设计、假设验证、功能优先级排序和客户访谈总结。

## Skills（13 个）

- **analyze-feature-requests** — 按主题、战略对齐度、影响、工作量和风险对功能请求列表进行分析和优先级排序。
- **brainstorm-experiments-existing** — 为已有产品设计实验以验证假设。
- **brainstorm-experiments-new** — 为新产品的精益创业实验（原型测试）设计。
- **brainstorm-ideas-existing** — 使用多视角创意法（PM、设计师、工程师）为已有产品进行头脑风暴。
- **brainstorm-ideas-new** — 为处于初期探索阶段的新产品构思功能创意。
- **identify-assumptions-existing** — 从价值、可用性、可行性和可实现性四个维度，识别已有产品中某功能创意的高风险假设。
- **identify-assumptions-new** — 从 8 个风险类别（包括上市策略、战略和团队等）中识别新产品创意的高风险假设。
- **interview-script** — 创建结构化的客户访谈脚本，包含 JTBD 探究问题、热身、核心探索和总结环节。
- **metrics-dashboard** — 定义并设计产品指标仪表盘，包含关键指标、数据源、可视化类型和告警阈值。
- **opportunity-solution-tree** — 构建机会解决方案树（OST），将期望成果映射到机会、解决方案和实验，结构化产品探索过程。
- **prioritize-assumptions** — 使用影响 × 风险矩阵对假设进行优先级排序，并为每项假设建议实验方案。
- **prioritize-features** — 基于影响、工作量、风险和战略对齐度对功能待办列表进行优先级排序。
- **summarize-interview** — 将客户访谈记录总结为结构化模板，包含 JTBD、满意度信号和行动项。

## Commands（5 个）

- `/pm-product-discovery:brainstorm` — 从 PM、设计师和工程师的视角进行产品创意或实验的头脑风暴——适用于已有产品或新产品。
- `/pm-product-discovery:discover` — 运行完整的 product discovery 流程——从创意构思到假设映射再到实验设计。
- `/pm-product-discovery:interview` — 准备客户访谈脚本或将访谈记录总结为结构化洞察。
- `/pm-product-discovery:setup-metrics` — 设计产品指标仪表盘，包含北极星指标、输入指标、健康指标和告警阈值。
- `/pm-product-discovery:triage-requests` — 分析、分类和排序来自客户或利益相关方的功能请求。
