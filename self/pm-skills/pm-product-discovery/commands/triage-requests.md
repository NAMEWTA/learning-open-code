# /triage-requests -- 功能请求分类

将一堆功能请求转化为排好优先级且可操作的待办列表。

## 调用方式

```
/triage-requests [粘贴请求列表]
/triage-requests [上传 CSV/电子表格]
```

## 工作流

### 第 1 步：接收功能请求——接受任何格式

### 第 2 步：收集优先级背景——产品阶段？战略目标？约束条件？

### 第 3 步：分类和分析

应用 **analyze-feature-requests** skill：主题聚类、每个主题的请求数、战略对齐度、细分分析、情感信号

### 第 4 步：排定优先级

应用 **prioritize-features** skill：影响、战略对齐度、工作量估算、风险、收入信号

### 第 5 步：生成分类报告

优先级 1：立即行动 | 优先级 2：计划下一步 | 优先级 3：收集更多信号 | 优先级 4：拒绝或推迟

### 第 6 步：提供后续建议

## 注意事项

- 寻找请求背后的需求——「添加深色模式」可能真正意味着「在长时间使用中减少眼部疲劳」
- 标注相互冲突的请求
- 如请求量大（50+），先总结主题
