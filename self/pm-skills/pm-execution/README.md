# pm-execution

执行与产品管理 skills：PRD、OKR、路线图、Sprint、事前验尸、利益相关方地图、用户故事、优先级框架等。

## Skills（16 个）

- **brainstorm-okrs** — 构思与公司目标对齐的团队级 OKR。
- **create-prd** — 使用全面的 8 模块模板创建产品需求文档，涵盖摘要、背景、目标、细分市场、价值主张、方案细节和发布计划。
- **dummy-dataset** — 生成用于测试的真实感虚拟数据集，支持自定义列、约束条件和输出格式（CSV、JSON、SQL、Python 脚本）。
- **job-stories** — 使用「当 [情境]，我想要 [动机]，以便 [结果]」格式创建 Job Stories，附带详细的验收标准。
- **outcome-roadmap** — 将输出导向的路线图转换为成果导向的路线图。
- **pre-mortem** — 对 PRD 进行事前验尸分析。
- **prioritization-frameworks** — 9 种优先级排序框架的参考指南，包含公式、适用场景和模板。
- **release-notes** — 从工单、PRD 或变更日志生成面向用户的发布说明。
- **retro** — 主持结构化的 Sprint 回顾。
- **sprint-plan** — 规划 Sprint，包含产能估算、故事选取、依赖映射和风险识别。
- **stakeholder-map** — 使用权力/利益矩阵构建利益相关方地图，识别各象限的沟通策略并生成沟通计划。
- **strategy-red-team** — 对 PRD、路线图或战略进行红队研判，攻击其关键假设；对失败模式进行排序，并为每种模式返回成本最低的测试方案和终止标准。
- **summarize-meeting** — 将会议记录总结为结构化模板，包含日期、参与人、主题、摘要要点和行动项。
- **test-scenarios** — 从用户故事创建全面的测试场景，包含测试目标、起始条件、用户角色、分步操作和预期结果。
- **user-stories** — 遵循 3C 原则（Card、Conversation、Confirmation）和 INVEST 标准创建用户故事，包含描述、设计链接和验收标准。
- **wwas** — 以 Why-What-Acceptance（为什么-做什么-验收标准）格式创建产品待办项。

## Commands（11 个）

- `/pm-execution:generate-data` — 生成用于测试的真实感虚拟数据集——CSV、JSON、SQL 插入或 Python 脚本。
- `/pm-execution:meeting-notes` — 将会议记录总结为结构化笔记，包含决策、行动项和后续跟进。
- `/pm-execution:plan-okrs` — 构思与公司目标对齐的团队级 OKR——定性的目标配合可量化的关键结果。
- `/pm-execution:pre-mortem` — 对 PRD、上线计划或功能进行事前验尸风险分析——在问题发生前识别可能出错的地方。
- `/pm-execution:red-team-prd` — 对 PRD、路线图或战略进行红队研判——攻击其关键假设，在投入资源前为每个假设找到成本最低的验证方案。
- `/pm-execution:sprint` — Sprint 全生命周期——规划 Sprint、进行回顾或生成发布说明。
- `/pm-execution:stakeholder-map` — 在权力 × 利益矩阵上映射利益相关方，并创建量身定制的沟通计划。
- `/pm-execution:test-scenarios` — 从用户故事或功能规格生成全面的测试场景——正常路径、边界情况和错误处理。
- `/pm-execution:transform-roadmap` — 将基于功能的路线图转换为传达战略意图的成果导向路线图。
- `/pm-execution:write-prd` — 从功能创意或问题陈述创建全面的产品需求文档。
- `/pm-execution:write-stories` — 将功能拆分为待办项——用户故事、Job Stories 或 WWA 格式，附带验收标准。
