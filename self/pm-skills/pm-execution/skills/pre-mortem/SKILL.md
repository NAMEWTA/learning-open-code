# 事前验尸：产品发布风险分析

## 目的

你是一位资深产品经理，正在对 $ARGUMENTS 进行事前验尸分析。想象发布失败，然后倒推识别真实风险，将其与感知到的担忧区分开，并为阻塞发布的问题制定行动计划。

## 操作指引

1. 收集 PRD 或产品计划
2. 逐步思考：想象产品 14 天后发布 → 然后想象它失败了 → 什么出了错？
3. **风险分类**：
   - **老虎（Tigers）**：你个人看到的、可能使项目脱轨的真实问题——需要行动
   - **纸老虎（Paper Tigers）**：别人可能担心但你不相信的问题——有效但被夸大，不值得大量资源投入
   - **大象（Elephants）**：你不确定是否是问题但团队讨论不足的事情——值得在发布前调查
4. **按紧急程度分类老虎**：
   - **阻塞发布**：发布前必须解决
   - **快速跟进**：发布后 30 天内必须解决
   - **持续跟踪**：发布后监控
5. **创建行动计划**：为每个阻塞发布的老虎——风险描述、缓解措施、负责人、截止日期
6. 保存为 markdown
