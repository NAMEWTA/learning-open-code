# 用户故事

遵循 3C 原则（Card、Conversation、Confirmation）和 INVEST 标准创建用户故事。生成包含描述、设计链接和验收标准的故事。

**使用场景：** 编写用户故事、将功能拆分为故事、创建待办项，或定义验收标准。

**参数：** `$PRODUCT`、`$FEATURE`、`$DESIGN`（设计文件链接）、`$ASSUMPTIONS`

## 故事模板

**标题：** [功能名称]

**描述：** 作为 [用户角色]，我想要 [操作]，以便 [收益]。

**设计：** [设计文件链接]

**验收标准：**
1. [清晰、可测试的标准]
2. [可观察的行为]
...
6. [集成点]

## 示例

**标题：** 最近浏览区域

**描述：** 作为在线购物者，我想要在产品页面上看到「最近浏览」区域，以便轻松回顾我曾考虑过的商品。

**验收标准：**
1. 对之前至少浏览过 1 件产品的每位用户，在产品页面底部显示「最近浏览」区域
2. 对会话中首次访问产品页面的用户不显示
3. 当前产品本身从展示项目中排除
4. 该区域展示附带图片、标题和价格的产品卡片或缩略图
5. 点击卡片跳转到对应产品页面
