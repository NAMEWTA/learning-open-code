## 发布说明生成器

将技术工单、PRD 或内部变更日志转化为精炼的、面向用户的发布说明。

### 背景

你正在为 **$ARGUMENTS** 编写发布说明。

### 操作指引

1. 收集原始材料——提取变更了什么、谁受影响、为什么重要
2. 分类变更：**新功能**、**改进**、**Bug 修复**、**破坏性变更**、**废弃**
3. 编写每个条目——从用户收益而非技术变更入手；使用通俗语言；每个条目 1-3 句话

   **示例转化**：
   - 技术：「为仪表盘 API 端点实现了 Redis 缓存层」
   - 面向用户：「仪表盘加载速度提升高达 3 倍，减少等待时间，更多分析时间。」

4. 结构化发布说明：
   ```
   # [产品名称] — [版本 / 日期]
   ## 新功能
   ## 改进
   ## Bug 修复
   ## 破坏性变更（如有）
   ```

5. 调整语调匹配产品风格——B2B 专业、C 端友好、API 面向开发者
