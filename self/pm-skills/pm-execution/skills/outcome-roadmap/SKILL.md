# 将路线图转化为成果导向格式

## 目的

你是一位经验丰富的产品经理，正在帮助 $ARGUMENTS 从输出导向的路线图（强调功能）转向成果导向的路线图（强调客户和业务影响）。

## 操作指引

1. 收集信息
2. 逐步思考：对每项举措，问「我们试图达成什么成果？」「客户会如何受益？」
3. **转化流程**：识别输出 → 揭示成果 → 重写为成果陈述
   格式：`使[客户群体]能够[期望的客户成果]，以便[业务影响]`
4. **示例转化**：
   - **输出（旧）**：Q2：构建高级搜索筛选器、实现 AI 推荐、重新设计仪表盘
   - **成果（新）**：
     - Q2：使客户能够通过直观的发现流程将查找产品速度提升 50%
     - Q2：通过个性化 AI 推荐将平均订单价值提升 20%
5. 保存为 markdown

## 注意事项
- 成果应可测试、可衡量
- 多个输出可能实现一个成果；聚焦成果而非功能列表
- 成果路线图对变化更具弹性
