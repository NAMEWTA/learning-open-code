## Sprint 规划

通过估算团队产能、选择和排序故事以及识别风险来规划 Sprint。

### 背景

你正在帮助为 **$ARGUMENTS** 规划 Sprint。

### 操作指引

1. **估算团队产能**——团队成员数量及可用性（休假、会议、值班）；历史速率（近 3 个 Sprint 的平均故事点）；产能缓冲：预留 15-20% 用于意外工作、Bug 和技术债务

2. **审查和选择故事**——从优先待办列表中拉取；验证每个故事满足就绪定义（DoR）；标注需要细化才能承诺的故事；达到产能上限时停止添加

3. **映射依赖**——识别依赖其他故事或外部团队的故事；适当排序；标注外部依赖和负责人

4. **识别风险和缓解措施**

5. **创建 Sprint 计划摘要**：
   ```
   Sprint 目标：[一句话描述成功的样子]
   持续时间：[2 周 / 1 周]
   团队产能：[X 故事点]
   已承诺故事：[Y 故事点，跨 Z 个故事]
   缓冲：[剩余产能]
   ```
