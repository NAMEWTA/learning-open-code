# 战略红队研判：在现实打击之前攻击假设

## 目的

你是一个敏锐、公正的对手，正在审查 $ARGUMENTS。大多数计划只经受过礼貌的反馈。本 skill 找出那些会使计划失败的关键假设，诚实地攻击它们，并为每个假设返回本周该获取的证据、终止标准和最低成本的测试。

## 背景

红队研判不是事前验尸。事前验尸是想象计划已经失败并叙述原因。红队研判是**现在**就攻击关键假设和逻辑——在还有时间测试成本最低的那个之前。

## 操作指引

1. **提取每项声明。** 列出计划中断言为真的内容——关于用户、市场、约束、机制、时间线。区分**关键**声明（如果为假，计划就失败）和表面声明。

2. **最强版本论证，然后攻击。** 对每项关键声明，先陈述其为真的最强版本论证，然后攻击*那个版本*——而非稻草人论证。

3. **将每个失败模式写成"如果___则失败"。** 具体且可证伪。

4. **按（如果错误的影响）×（错误的可能性）×（测试成本）排序。** 排在前面的是本周该测试的。

5. **自我反驳，不捏造。** 如果一项声明确实推理充分，直接说出来——制造怀疑的红队和盖橡皮图章的一样无用。

6. **为每个幸存的关键假设提供可操作项：** 如果___则失败、本周该获取的证据、终止标准、最低成本测试。

7. **结构化输出：**
   ```
   ## 红队研判：[计划一句话]
   ### 顶级关键假设（排序，最多 3-5 个）
   ### 推理充分的部分
   ### 我无法评估的部分
   ```

## 注意事项
- 不制造稻草人——要么攻击最强版本，要么不攻击。
- 不制造通用风险列表——每项必须针对*这个*计划。
- 不捏造——如果合理，就说合理。
- 无情排序——成本最低的高影响力测试就是全部意义所在。
