# /write-prd -- 产品需求文档

创建结构化 PRD，对齐利益相关方并指导开发。接受从模糊想法到详细简报的任何输入。

## 工作流

### 第 1 步：理解功能——接受任何形式的输入
### 第 2 步：收集背景

对话式询问关键问题：用户问题、目标用户、成功指标、约束条件、先例、范围偏好

### 第 3 步：生成 PRD

应用 **create-prd** skill 产出 8 模块文档：执行摘要 → 背景与背景信息 → 目标与成功指标（含明确的非目标） → 目标用户与细分 → 用户故事与需求（P0/P1/P2） → 解决方案概览 → 待解决问题 → 时间线与分阶段

### 第 4 步：审核并迭代

保存为 markdown。

## 注意事项

- 对范围要有主见——聚焦的 PRD 优于宽泛模糊的
- 非目标与目标同等重要——它们防止范围蔓延
- 成功指标必须具体
- 待解决问题应是真正未解决的
