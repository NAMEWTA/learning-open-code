# /write-stories -- 待办项生成器

将功能拆分为结构良好的待办项。根据团队偏好从三种格式中选择。

## 格式

- **用户故事** — 「作为 [用户类型]，我想要 [能力]，以便 [收益]」。应用 **user-stories** skill
- **Job Stories** — 「当 [情境]，我想要 [动机]，以便 [结果]」。应用 **job-stories** skill
- **WWA** — 为什么 [战略背景] → 做什么 [交付物] → 验收 [标准]。应用 **wwas** skill

## 工作流

### 第 1 步：接收功能——任何形式
### 第 2 步：确定格式——用户故事/Job Stories/WWA
### 第 3 步：拆解功能——拆分为 5-15 个独立故事，每个在一个 Sprint 内可完成
### 第 4 步：生成故事——已排优先级、附验收标准、故事地图、技术说明
### 第 5 步：提供后续建议

## 注意事项

- 一个故事 = 一个可部署的价值单元
- 验收标准应由 QA 可直接验证，无需额外解释
- 如功能较大（15+ 故事），建议分组成 epics 或阶段
