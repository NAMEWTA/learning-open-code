# /pre-mortem -- 发布前风险分析

想象你的发布已经失败。然后倒推找出原因。应用老虎/纸老虎/大象框架揭示真实风险并制定缓解计划。

## 工作流

### 第 1 步：接收计划
### 第 2 步：风险识别

应用 **pre-mortem** skill：想象产品已发布并失败，生成跨类别的风险——技术、用户、业务、运营、依赖

### 第 3 步：风险分类

**老虎** — 真实、实质性的风险：阻塞发布 / 快速跟进 / 持续跟踪

**纸老虎** — 感觉可怕但被夸大的风险

**大象** — 团队知道但回避讨论的未被言说的风险

### 第 4 步：生成事前验尸报告——风险摘要、阻塞发布的老虎、快速跟进的老虎、纸老虎、房间里的大象、Go/No-Go 检查清单
### 第 5 步：提供后续建议

## 注意事项

- 最佳事前验尸在计划完成 80% 时进行——足够早以改变方向，足够晚以实质有内容
- 大象是最高价值的产出——揭示团队回避讨论的内容
