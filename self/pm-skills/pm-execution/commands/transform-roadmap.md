# /transform-roadmap -- 成果导向路线图

将计划功能列表或输出导向的路线图重写为传达*为什么*而非*做什么*的成果导向路线图。

## 工作流

### 第 1 步：接收当前路线图——任何格式
### 第 2 步：理解战略背景
### 第 3 步：转化每项条目

应用 **outcome-roadmap** skill：识别用户或业务成果 → 重写为成果陈述 → 将服务于同一成果的功能归为一个项目 → 添加成功指标

**转化示例：**
- 「构建 SSO 集成」→ 「减少企业引导摩擦——目标：企业账户实现价值时间提速 50%」
- 「重新设计仪表盘」→ 「帮助高级用户更快发现洞察——目标：洞察获取时间减少 30%」

### 第 4 步：生成转化后的路线图——Now/Next/Later 格式、转化说明
### 第 5 步：审核并提供后续建议

## 注意事项

- 成果应可衡量且有明确的「完成」状态
- 多个功能可映射到一个成果——这是特性而非 bug
- 如果一个输出不能清晰服务于一个成果，标注给用户判断或降级
