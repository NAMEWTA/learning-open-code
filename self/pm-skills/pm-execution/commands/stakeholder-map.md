# /stakeholder-map -- 利益相关方映射与沟通计划

识别项目的所有利益相关方，按影响力和利益映射，并生成确保正确的人在正确的时间获得正确信息的沟通计划。

## 工作流

### 第 1 步：理解项目
### 第 2 步：识别利益相关方——内部、外部、常被遗漏的
### 第 3 步：映射到权力 × 利益矩阵

应用 **stakeholder-map** skill：密切管理 / 保持满意 / 保持知情 / 监控

### 第 4 步：生成沟通计划——利益相关方表格、各象限沟通策略、潜在冲突、上报路径、RACI 矩阵
### 第 5 步：提供后续建议

## 注意事项

- 「密切管理」象限是 PM 花费最多政治资本的地方
- 别忘了下游利益相关方：支持、文档和销售支持团队常在发布时被动应对
