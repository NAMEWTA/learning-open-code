# /test-scenarios -- 测试场景生成器

将用户故事或功能描述转化为 QA 可立即执行的全面测试场景。覆盖正常路径、边界情况、错误处理以及跨浏览器/设备考量。

## 工作流

### 第 1 步：接收输入——用户故事、验收标准、PRD、功能描述
### 第 2 步：生成测试场景

应用 **test-scenarios** skill：正常路径场景、边界情况、错误场景、安全场景（如适用）、性能场景（如适用）

### 第 3 步：结构化输出——场景表（步骤、操作、预期结果）、覆盖矩阵
### 第 4 步：提供后续建议

## 注意事项

- 先正常路径，再叠加边界情况
- 每个验收标准应映射到至少一个测试场景
- 包含正向测试（有效）和负向测试（优雅失败）
