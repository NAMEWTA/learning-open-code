# /plan-okrs -- 团队 OKR 规划

生成结构良好的 OKR，将团队工作与公司战略连接起来。产出 3 套 OKR，包含定性目标和定量关键结果。

## 工作流

### 第 1 步：收集背景
### 第 2 步：生成 OKR

应用 **brainstorm-okrs** skill：3 套 OKR，目标要定性、鼓舞人心、雄心勃勃但可实现；关键结果要定量、可衡量、有时限

### 第 3 步：验证质量

每个 OKR 对照最佳实践检查：目标鼓舞人心？KR 可衡量？目标雄心勃勃但不令人沮丧？

### 第 4 步：呈现并迭代

评分指南：0.0-0.3 重大失误 | 0.4-0.6 有进展但不足 | 0.7-0.9 校准良好的挑战目标 | 1.0 要么完美达成，要么目标不够雄心勃勃

## 注意事项

- OKR 应描述成果而非产出（「将激活率提升 20%」而非「发布引导流程重新设计」）
- 每季度每团队最多 3 个目标——更多意味着更不聚焦
- KR 应是挑战目标——如果你确定能达成，说明不够雄心勃勃
